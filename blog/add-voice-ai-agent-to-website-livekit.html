System Architecture Overview

The solution combines several powerful technologies into one seamless experience. Understanding the components helps troubleshoot any issues during deployment.

The system has five main parts: the Next.js widget interface, LiveKit cloud infrastructure, Python voice agent, speech-to-text transcription, and text-to-speech synthesis. Data flows between these components in real-time using WebRTC connections.

Key Components:

  • Next.js Widget: The embeddable interface visitors interact with
  • LiveKit Cloud: Handles real-time audio/video streaming
  • Python Agent: Processes conversations using AI models
  • STT/TTS: Converts between speech and text
  • LLM: Generates intelligent responses (ChatGPT 4.1 mini)

Agent Deployment Steps

Deploying the voice agent requires setting up the Python backend that will handle conversations. The process is straightforward with the provided template.

Step 1: Configure Agent Settings

Edit the agent.py file to specify your prompt, models, and tools. The critical addition is setting the agent name in both the Python file and .env.local configuration.

Step 2: Deploy to LiveKit Cloud

Using the LiveKit CLI, run lk agent deploy from the agent directory. Verify deployment success in your LiveKit dashboard where the agent should show as "Running".

Pro Tip: Test your agent using LiveKit's sandbox environment before website integration. This lets you verify conversation flows work as expected.

Widget Setup Process

The Next.js widget provides the interface visitors will interact with on your website. Configuration focuses on connecting it to your deployed agent.

Step 1: Install Dependencies

Run npm install in the widget directory to download all required packages. This includes the LiveKit client SDK and interface components.

Step 2: Configure Environment

Update the .env.local file with your LiveKit API credentials and the agent name matching your deployment. Set the connection endpoint to your localhost during testing.

Step 3: Local Testing

Run npm run dev and visit localhost:3000 to verify the widget connects to your agent. Test both chat and voice functionality.

Website Integration Methods

Once tested locally, deploy the widget for production use. Two integration methods provide flexibility for different website setups.

Method 1: Iframe Embed

Deploy the widget to Railway.app and embed it using an iframe. This works well for adding the agent to specific page sections.

Method 2: Floating Widget

Add a script tag to your website's body that creates a floating chat button. Visitors can initiate conversations from any page.

Implementation Tip: For Framer websites, add the script via the Embed component. Other platforms like WordPress have dedicated script injection areas.
