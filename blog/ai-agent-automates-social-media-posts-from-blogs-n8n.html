How the AI Agent Works

The workflow combines three powerful automation components: content extraction, AI transformation, and multi-platform publishing. Here's the high-level process:

  1. Schedule Trigger: The workflow runs automatically on your chosen schedule (every 4 hours in the tutorial)
  2. RSS Feed Reader: Pulls the latest blog post from your specified source
  3. HTML to Markdown: Converts the blog content into clean, structured text
  4. AI Agent: Analyzes the content and generates platform-specific captions and image prompts
  5. Image Generation: Creates social media images using Google's Imagen 2
  6. Multi-Platform Publishing: Uploads the image with tailored captions to all connected social accounts

At 2:45 in the video, you can see the complete workflow visualized in n8n's interface. The entire process happens without any manual intervention once configured.

Setting Up the RSS Feed

The workflow starts by pulling content from an RSS feed. While the tutorial uses TechCrunch as an example, you can use any blog with an RSS feed - including your own company blog.

Key configuration points:

  • Add the RSS Feed node in n8n
  • Enter your blog's RSS URL (append "/rss" to the base URL if unsure)
  • Use the "Set" node to extract just the latest post URL
  • Configure to execute only once to avoid duplicate posts

At 4:30 in the video, there's a crucial tip about RSS feed URLs. Many blogs require adding "/rss" to the end of their URL to access the feed properly. The tutorial shows how to test and validate your RSS feed connection directly in n8n.

Configuring the AI Agent

The AI Agent node is where the magic happens. It takes the blog content and transforms it into social-ready assets. The tutorial uses Google Gemini, but you could substitute OpenAI or other models.

Critical configuration elements:

  1. System Message: Instructs the AI to act as a professional social media strategist
  2. Input: The markdown-formatted blog content
  3. Output Schema: Defines the exact structure needed (image prompt + platform captions)

At 8:15, the video shows the detailed JSON schema that ensures consistent output formatting. This schema specifies:

  • An image prompt for DALL-E/Imagen generation
  • Platform-specific captions (Facebook, Instagram, LinkedIn, etc.)
  • Hashtags tailored to each platform's audience

Pro Tip: Gemini 1.5 Flash was chosen for its 1 million token context window, allowing it to process long-form blog posts that would exceed other models' limits.

Image Generation Process

With the AI-generated image prompt ready, the workflow creates the visual content using Google's Imagen 2. The tutorial shows how to:

  1. Add the Google Gemini Image node
  2. Select Imagen 2 as the model (higher quality than standard options)
  3. Map the AI-generated prompt to the node
  4. Configure image count and dimensions

At 12:30, you can see the generated image in the workflow. The key advantage here is that the image is created specifically for the blog content, not just a generic stock photo. The AI ensures visual relevance to your post.

For businesses with branded visual styles, you can enhance the prompt to include specific color schemes, logo placements, or other branding elements.

Multi-Platform Publishing

The final step publishes the content across all connected social platforms. The tutorial uses a third-party tool called UploadPost to handle the cross-platform posting, but you could adapt this for native APIs.

Configuration highlights:

  • Connect all your social accounts to UploadPost
  • Map the AI-generated captions to each platform
  • Set the generated image as the visual content
  • Specify destination pages/groups where applicable

At 16:45, the video demonstrates how each platform receives unique, optimized content:

  • Facebook gets a conversational caption
  • LinkedIn receives a more professional tone
  • Twitter/X has concise messaging with relevant hashtags
  • Pinterest includes rich pin descriptions

This platform-specific optimization happens automatically, saving you from manually reformatting for each network.

Scheduling & Automation

The workflow's power comes from its fully automated execution. The tutorial sets it to run every 4 hours, but you can adjust this to any interval:

  • For high-frequency accounts: Every 2-6 hours
  • Standard business accounts: 1-3 times per day
  • Low-volume accounts: Weekly or monthly

Key automation features:

  1. Schedule Trigger: Controls how often the workflow runs
  2. Conditional Logic: Only publishes if new content exists
  3. Error Handling: Notifications if any step fails

At 20:10, the video shows how to publish the workflow in n8n to activate the automation. Once live, it will run indefinitely on your schedule without manual intervention.
