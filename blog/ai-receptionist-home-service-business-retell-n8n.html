Retell Setup: Building the Voice Agent Personality

The magic starts with the global prompt at 22:10. We define the AI's role ("voice receptionist for HVAC company"), tone ("helpful, calm, confident"), and guardrails ("no technical diagnosis, no exact pricing"). This creates a consistent brand experience while preventing overpromising.

Key configuration tips: 1) Set max response length to 15 seconds to avoid monologues, 2) Enable continuous speech recognition for interruptible dialog, and 3) Add your service area cities to improve address recognition accuracy by 37%.

n8n Backend: The 3 Critical Workflows

The n8n automation handles the heavy lifting with three interconnected workflows: 1) Address validation (checks service eligibility), 2) Availability checking (queries your calendar), and 3) Appointment booking (creates CRM entries). Each triggers based on the Retell conversation stage.

At 18:45, we see the webhook setup that connects Retell to n8n. The "action" parameter determines which workflow runs - "check_address", "check_availability", or "book_appointment". This single endpoint approach simplifies maintenance while keeping data secure.

Address Validation with Google Maps API

The address checker (25:30) uses Google's Address Validation API to: 1) Confirm real locations, 2) Standardize formatting, and 3) Check driving distance from your business. Invalid addresses trigger re-prompting, while out-of-area calls get graceful exits.

Pro tip: Enable the "Distance Matrix" API to calculate exact drive times. At 27:10, we set the 20-mile radius rule, but you can customize this by travel time (e.g., "under 45 minutes") for more accurate service area definitions.

Emergency Call Handling & Safety Protocols

The safety hazard branch (14:20) shows the AI's emergency response: 1) Immediate problem identification ("Do you smell gas?"), 2) Safety instructions if needed, and 3) Priority dispatch routing. These calls bypass normal booking queues and trigger your on-call protocols.

We configure this by adding emergency-specific calendar IDs in n8n (31:45). When the AI detects an emergency, it checks this separate calendar for next-available slots, ensuring faster response times for critical issues.

Calendar Integration & Appointment Booking

The booking workflow (34:10) demonstrates four key steps: 1) Time slot confirmation, 2) Customer profile creation, 3) Service ticket generation, and 4) Calendar event creation. While we use Cal.com in the demo, the same logic applies to ServiceTitan or HouseCall Pro.

Critical detail at 36:20: We disable email requirements in Cal.com since the system uses phone numbers as primary identifiers. This prevents booking failures when callers don't provide emails during voice interactions.
