How the AI Receptionist Works

The system combines three powerful technologies: Retell AI for natural voice conversations, n8n for workflow automation, and Cal.com for calendar management. When a call comes in:

  1. Retell AI answers with a branded greeting and begins qualifying the caller
  2. The AI checks available time slots by calling your n8n workflow
  3. n8n queries your Cal.com API for real-time availability
  4. The AI offers available times in natural conversation
  5. Once confirmed, n8n creates the booking and sends confirmations

This all happens in seconds, with the caller experiencing a seamless conversation. The workflow maintains context between calls through a Supabase database, creating a continuous customer experience whether they call back later or you reach out to them.

Cal.com Calendar Integration

The n8n workflow needs to communicate seamlessly with your Cal.com account to check availability and create bookings. The video shows two critical API endpoints:

1. Checking Availability

The /api/cal.com/v2/slots endpoint returns available time slots based on your calendar rules. The workflow must include:

  • Authorization header with your API key
  • Correct Cal.com API version
  • Time zone conversion to ISO 8601 format

2. Creating Bookings

The /api/cal.com/v2/bookings endpoint handles actual appointment creation. Key parameters include:

  • Event type ID
  • Start/end times in correct format
  • Attendee email and name
  • Custom meeting description

The video demonstrates how to test these API calls directly in n8n before connecting them to Retell AI.

Building the n8n Workflow

The n8n workflow acts as the bridge between Retell AI and your calendar system. Follow these steps to recreate the workflow shown in the video:

Step 1: Set Up Webhook Trigger

Create a webhook node that Retell AI will call when it needs to check availability or book an appointment. Configure it to accept POST requests with JSON data.

Step 2: Add HTTP Request Nodes

For each Cal.com API endpoint (slots and bookings), create an HTTP Request node with:

  • Correct API URL
  • Authorization headers
  • Proper content-type
  • Parameter mapping from the webhook

Step 3: Implement Error Handling

Add error handling nodes as shown at 14:30 in the video to manage cases where slots aren't available or API calls fail. These ensure the AI can respond appropriately to the caller.

Step 4: Connect to Database

Add Supabase nodes to log call details and maintain conversation history. This enables continuity across multiple interactions with the same caller.

Remember: Test each node individually before chaining them together. The video shows how to use the "Execute Node" feature to debug each step.

Error Handling and Edge Cases

The difference between a frustrating AI experience and a seamless one comes down to how you handle errors. The workflow includes specific nodes to manage:

Time Zone Conflicts

Convert all times to ISO 8601 format with explicit time zones before sending to Cal.com. The video shows how to use n8n's date functions for this conversion.

Double Bookings

Even with real-time checking, slots can sometimes get booked between availability check and confirmation. The workflow includes a verification step to catch this.

API Rate Limits

Cal.com's API has rate limits. The workflow implements retry logic with exponential backoff when hitting these limits.

At 18:40 in the video, you'll see how to test these error scenarios by manually triggering different failure modes and verifying the AI responds appropriately.
