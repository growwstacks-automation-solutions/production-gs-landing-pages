AI Test Agents Explained: Planner, Generator & Healer

Test automation maintenance consumes 30-50% of QA teams' time according to recent studies. Every UI change breaks selectors, every new feature requires test updates, and debugging failures eats into development cycles. Playwright's revolutionary AI agents solve this by automating the entire test lifecycle.

Introduced in Playwright 1.56, these three specialized agents work together in what's called an "agentic loop": The Planner explores your application like a human tester would, the Generator writes production-ready test scripts, and the Healer automatically fixes broken tests when they fail.

Key insight: These aren't just code generators - they're autonomous testing specialists that understand application behavior. The Planner includes context about QA best practices, the Generator knows Playwright's API inside out, and the Healer understands common failure patterns.

Real-World Demo: Automating a Banking App in Minutes

To demonstrate the agents' capabilities, we automated tests for a Parabank banking application - a complex web app with user registration, account management, fund transfers, and bill payments. Traditionally, creating comprehensive tests for such an application would take days of manual work.

The Planner agent began by systematically exploring every feature: registering new users, opening accounts, transferring funds between accounts, and paying bills. It documented each user flow as test scenarios with validation points, creating a detailed 27-test plan covering five core features in under 10 minutes.

At the 4:30 mark in the video tutorial, you can see the Planner automatically recovering from edge cases - when a username was already taken, it intelligently generated alternative credentials to complete registration.

Planner Agent in Action: From Blank Page to 27 Test Cases

The Planner agent works by combining browser automation with large language model capabilities. As it navigates your application, it:

  1. Identifies all interactive elements and possible user journeys
  2. Documents happy paths and error scenarios
  3. Organizes tests by feature area with clear naming conventions
  4. Includes validation points for critical UI elements

For the banking app demo, the Planner created test cases covering:

  • User registration (5 scenarios including error cases)
  • Account management (5 scenarios for different account types)
  • Fund transfers (7 scenarios with various amount validations)
  • Bill payment (6 scenarios including scheduled payments)
  • Transaction history (4 scenarios with filtering options)

Pro tip: The Planner's test plans use markdown formatting, making them perfect documentation artifacts that stay in sync with your actual tests. No more outdated test case documents!

Generator Magic: Turning Natural Language into Executable Tests

The Generator agent transforms the Planner's markdown test plan into production-ready Playwright scripts. It doesn't just create basic recordings - it generates maintainable, well-structured test code with:

  • Proper test organization using describe/it blocks
  • Reusable page objects for common elements
  • Smart locators that resist UI changes
  • Comprehensive assertions for validation

In our demo, the Generator created TypeScript tests that:

  1. Used data-testid attributes where available for robust selectors
  2. Included error handling for network delays
  3. Validated both UI state and API responses
  4. Supported parallel execution with proper test isolation

The Generator's output isn't just technically correct - it follows Playwright best practices that would take junior engineers months to learn.

Self-Healing Tests: How the Healer Agent Fixes Broken Scripts

When tests fail (around 18:30 in the video), the Healer agent springs into action. It doesn't just report failures - it diagnoses the root cause and attempts repairs by:

  1. Analyzing failure screenshots and error messages
  2. Comparing current UI state with expected state
  3. Testing alternative selectors for changed elements
  4. Updating test steps to match new application flows

In our banking app demo, when user registration tests failed due to application issues, the Healer:

  • Identified the username collision error
  • Attempted alternative username generation strategies
  • Recognized when failures were due to application bugs vs test issues
  • Provided detailed diagnostic information for developers

Important note: The Healer can't fix application bugs, but it can make tests resilient to non-breaking UI changes - the #1 cause of test maintenance headaches.

Step-by-Step Setup Guide for Playwright Agents

Implementing these agents in your project takes about 15 minutes. Here's the exact process shown in the tutorial:

Step 1: Install Prerequisites

Ensure you have:

  • Visual Studio Code 1.105+
  • Node.js 18+
  • Playwright 1.56+

Step 2: Initialize Your Project

Run these commands in a new directory:

 npm init playwright@latest npx playwright init agents-loop --vs-code

Step 3: Configure Agents

The installation creates:

  • A test/agents directory with three MD files (planner, generator, healer)
  • An MCP server configuration
  • Sample test specs showing agent usage

Step 4: Start Exploring

In VS Code's GitHub Copilot chat:

  1. Select "Test Planner Agent"
  2. Enter: "Explore [your-app-url] and create test plan for core features"
  3. Let the agent navigate your app and generate the plan

Pro tip: For complex applications, break exploration into smaller chunks by feature area rather than trying to test everything at once.
