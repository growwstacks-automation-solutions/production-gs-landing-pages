Cal.com Integration Setup

Cal.com provides the scheduling infrastructure that makes this automation reliable. Their free tier offers everything needed for individual practitioners, with paid plans available for larger practices.

Implementation requires just three steps:

Step 1: Get Your API Key

Navigate to Cal.com Settings → Developer → API Keys and create a new key. For security, set an appropriate expiration date unless you're comfortable with "Never expires."

Step 2: Identify Your Event Type ID

Each appointment type (consultation, checkup, etc.) has a unique ID visible in the URL when editing that event type in Cal.com. This tells the system which calendar template to use.

Step 3: Configure Time Zone

Critical for avoiding scheduling errors. Verify your Cal.com profile and Retail AI integration both use the same time zone as your physical practice location.

Pro Tip: Create separate event types for different appointment durations (15min, 30min, etc.) with appropriate buffers between sessions.

Rescheduling System with Zapier

While booking is straightforward through Cal.com's direct integration, rescheduling requires Zapier to handle the multi-step process:

Step 1: Catch Hook Setup

Create a Zapier trigger that listens for reschedule requests from Retail AI. This captures the caller's email and requested new time slot.

Step 2: Find Existing Appointment

Use Cal.com's API to lookup upcoming bookings matching the caller's email. This identifies which appointment needs modification.

Step 3: Execute Reschedule

Call Cal.com's reschedule endpoint with the booking ID and new time in ISO 8601 format (critical for proper parsing).

Important: The time format conversion happens in Retail AI before sending to Zapier. At 12:45 in the video, you'll see the exact parameter configuration that ensures proper date handling.
