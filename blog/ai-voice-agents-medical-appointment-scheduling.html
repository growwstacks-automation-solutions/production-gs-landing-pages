How the AI Voice Agent Works

The solution uses AWS Nova Sonic's speech recognition to understand patient requests in natural language. Unlike rigid IVR systems, it handles interruptions ("Um, let me check") and follows conversational flow like a human agent.

Key components shown in the demo:

  1. Natural language understanding - Processes patient speech in real-time
  2. Multi-factor verification - Checks name, DOB, and partial SSN against EHR
  3. Real-time availability lookup - Integrates with practice management systems
  4. Dynamic response generation - Offers alternatives when preferred times are booked

At 2:30 in the video, the AI smoothly handles the patient's initial time request being unavailable and successfully books an alternative slot.

Patient Verification Process

HIPAA-compliant identity verification is critical for medical scheduling. The demo shows the AI agent collecting three verification factors:

3-Point Verification: Full name → Date of birth → Last 4 SSN digits

For verified patient Vadim Bellski (0:45 in video), the system matched all three factors against EHR records. For John Doe (3:20), the mismatch triggered a secure transfer to human staff - demonstrating robust security protocols.

Real-Time Availability Checks

The AI doesn't just take messages - it makes actual bookings by integrating with the practice's scheduling system. In the demo (1:30), when the patient requests "tomorrow 9:00 a.m.", the AI:

  1. Queries real-time provider availability
  2. Identifies no 9am slots available
  3. Offers the earliest alternative (2:00 p.m.)
  4. Books the appointment after confirmation

This eliminates the callback cycle where staff manually check availability and try to reach patients later.

Handling Failed Verification

The demo's second call (3:00) shows the system's security protocols in action when verification fails. For "John Doe", the AI:

  • Politely repeats collected information for confirmation
  • Recognizes the name doesn't match EHR records
  • Declines to proceed with scheduling
  • Offers transfer to human staff

This prevents fraudulent bookings while maintaining patient experience - a critical balance for healthcare providers.

Integration Architecture

The solution's technical architecture (mentioned at video start) combines:

AWS Nova Sonic for speech processing + LiveKit media server + EHR/CRM integrations

This stack handles the complete workflow:

  1. Voice calls routed through LiveKit media server
  2. Nova Sonic converts speech to text and analyzes intent
  3. Backend queries EHR for verification and CRM for availability
  4. Nova Sonic generates natural voice responses

The entire process happens in real-time with sub-second latency, creating a natural conversation flow.
