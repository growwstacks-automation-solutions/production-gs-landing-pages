The $600 Billion Recruitment Automation Pain Point

Recruitment automation solves many manual processes, but one critical pain point has remained untouched until now: fake candidates. These fraudulent applicants cost businesses an estimated $600 billion annually, with single bad hires potentially costing up to $240,000 in lost productivity and replacement expenses.

The Verify module in Make.com represents a breakthrough in automated candidate screening. Developed by recruitment professionals who've experienced this pain firsthand, it surfaces red flags that human screeners might miss during manual resume reviews.

Key insight: Traditional automation focuses on speeding up the hiring process, but Verify adds a crucial quality control layer that prevents wasted time on unqualified or dishonest candidates.

Make.com Verify Module Overview

Verify is a specialized module within Make.com that analyzes resumes for potential red flags. Unlike generic parsing tools, it's specifically trained to detect patterns that indicate:

  • Inconsistent employment dates or suspicious gaps
  • Potentially fabricated education credentials
  • Skill claims that don't match experience levels
  • Other indicators of resume fraud

The module outputs both a risk assessment score and a detailed HTML report that explains any detected issues. These results can be integrated into your existing recruitment workflows at any stage.

Step-by-Step Workflow Setup

The basic Verify workflow follows a simple pattern that can be customized for different recruitment pipelines:

Step 1: Trigger from Resume Submission

Configure the workflow to trigger when a new resume appears in your Dropbox, Google Drive, or other storage location. This ensures every candidate gets screened automatically.

Step 2: File Processing

Use Make.com's Tools module to prepare the resume file for analysis. This converts the document into a format Verify can process while maintaining all original content.

Step 3: Verify Analysis

The core module where the resume gets analyzed. This step requires your Verify API key (covered in the next section) and typically completes in seconds.

Step 4: Result Handling

Configure where the results should go - back to storage as a report, into your CRM, or to trigger specific follow-up actions based on risk level.

Pro tip: Place Verify early in your workflow to filter out questionable candidates before investing time in interviews or assessments.

Verify API Integration Process

Connecting Verify to your Make.com scenario requires just a few simple steps:

  1. Sign up for a Verify account through their developer portal (link in description)
  2. Generate an API key in your account dashboard
  3. Add the Verify module to your Make.com scenario
  4. Paste your API key when prompted
  5. Map the resume content from your file processing step

The entire setup takes less than 5 minutes. Verify uses a credit-based system, so you'll need to purchase a small amount of credits (starting at $1) to begin processing resumes.

Understanding the Results Analysis

Verify provides multiple output formats to suit different recruitment workflows:

  • HTML Report: A visually formatted document highlighting potential issues with explanations
  • Risk Score: A numerical assessment (0-100) of overall resume credibility
  • Detailed Flags: Specific categories where potential issues were detected
  • Raw Data: Structured JSON for integration with other systems

These outputs can be routed to different destinations based on your needs. For example, high-risk resumes might go to a separate review queue, while low-risk candidates proceed directly to scheduling.

Adding Verify to Existing Workflows

One of Verify's strongest features is its modular design. It can be inserted into nearly any existing Make.com recruitment automation with minimal disruption:

  • ATS Integrations: Add Verify between resume submission and candidate scoring
  • Screening Workflows: Use risk scores to prioritize manual reviews
  • Interview Scheduling: Flag questionable candidates before calendar invites go out

The module works particularly well when combined with the Make.com automation workflows we've covered in previous tutorials.
