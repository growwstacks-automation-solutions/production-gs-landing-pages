Configuring Your ElevenLabs AI Receptionist

The foundation of this automation is an ElevenLabs voice agent that sounds indistinguishable from a human receptionist. Here's how to set it up:

Step 1: Create Your Agent

In your ElevenLabs dashboard, create a new agent specifically for appointment booking. Choose a professional voice that matches your brand - studies show customers prefer female voices for service interactions.

Step 2: Add the Webhook Tool

Scroll to the tools section and add a new webhook. Name it "Book" (this becomes the trigger word for appointments). Paste the description exactly as shown at 3:42 in the video - this ensures the AI understands its purpose.

Pro Tip: Set the webhook color to green - this visual cue helps when managing multiple tools.

Step 3: Define Data Collection Fields

The magic happens in the body parameters. You'll need fields for:

  • Full name (text)
  • Phone number (number)
  • Service requested (text)
  • Preferred time (text)
  • Service address (text)
  • Email (text)

Mark all fields as required. The order matters - match the sequence shown at 4:18 in the tutorial to ensure smooth data flow to Make.com.

Creating the Make.com Webhook

With your ElevenLabs agent ready, we now build the automation backbone in Make.com:

Step 1: Set Up the Webhook Module

Create a new scenario in Make.com and add a webhook as the first module. Choose "Custom Webhook" and copy the URL provided. This is what you'll paste into your ElevenLabs agent settings.

Step 2: Add OpenAI for Time Zone Conversion

The critical innovation comes next. Add an OpenAI module connected to your ChatGPT account. At 7:05 in the video, you'll see the exact system prompt that handles time zone conversion:

"Convert [caller's requested time] to [business timezone] accounting for daylight savings if applicable. Return only the converted time in ISO 8601 format."

This single step eliminates the #1 cause of booking errors - time zone mismatches.

Step 3: Test the Connection

Before proceeding, test that data flows correctly from ElevenLabs to Make.com. You should see all defined fields populated when making a test call (demonstrated at 8:30 in the tutorial).

Automating Time Zone Conversion

The time zone module is where this automation shines. Traditional booking systems either:

  • Assume everyone is in the same time zone (leading to missed appointments)
  • Require callers to specify time zones (creating friction)

Our solution automatically detects and converts time zones using OpenAI. Here's how it works:

Real-world example: When a California caller says "tomorrow at 5pm", the system knows they're in PST and converts it to EST if your business is in New York. The calendar event is created in the correct local time slot.

At 10:12 in the video, you'll see the AI handling this conversion seamlessly. The key is configuring the OpenAI module to return the converted time in ISO format, which Google Calendar requires.

Google Calendar Integration That Prevents Double-Booking

The heart of the system is the Google Calendar integration. This isn't just about creating events - it's about intelligent scheduling:

Step 1: Add the Router

The router module (shown at 12:45) creates two paths:

  1. If the time slot is available → Create calendar event
  2. If booked → Offer alternative times

Step 2: Configure Calendar Event

For available slots, we create an event with:

  • Title: "[Service] for [Name]" (e.g., "AC Repair for John Smith")
  • Time: The converted time from OpenAI
  • Duration: Set based on your service (default 60 minutes)
  • Description: All collected customer details

Critical setting: Enable "Require meet.google.com link" if you do virtual consultations. This automatically adds a video meeting link to calendar invites.

Step 3: Handle Booked Slots

For unavailable times, the webhook response (shown at 14:20) politely offers alternatives: "Sorry, we're booked at 5pm. We have availability at 4pm or 6pm - which works for you?"

Automated CRM Logging in Google Sheets

While the calendar integration handles scheduling, we also need to log customer details for follow-up. The Google Sheets module (demonstrated at 16:00) creates a permanent record of every booking with:

  • Customer name and contact info
  • Service requested
  • Booked time (in both time zones)
  • Service address
  • Timestamp of booking

This spreadsheet becomes your de facto CRM. For businesses using Salesforce or HubSpot, you can replace this module with your CRM's API connection.

Pro Tip: Add a "Follow-Up Date" column with a formula that calculates 3 days after the appointment. This creates a ready-made outreach list for satisfaction checks.
