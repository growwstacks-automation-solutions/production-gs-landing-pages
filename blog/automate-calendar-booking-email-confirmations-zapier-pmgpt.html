Zapier MCP: Your Booking Automation Backbone

Zapier's Model Context Protocol (MCP) acts as the critical bridge between your AI agent and business tools. Unlike standard Zapier automations, MCP enables dynamic two-way communication where the AI can both retrieve information (like calendar availability) and take actions (like booking time slots).

The setup demonstrated in the video connects two core services:

  • Google Calendar with 6 critical permissions: Find events, create detailed events, quick add events, delete events, find busy periods, and update events
  • Gmail with send email capability permission for instant confirmations

This combination gives your AI agent everything it needs to manage the entire booking process end-to-end while preventing scheduling conflicts.

Google Calendar Integration That Prevents Double Bookings

The magic happens in the calendar configuration. By enabling "find busy periods" in your MCP setup, the AI agent automatically checks availability before confirming any appointment - eliminating the #1 pain point in manual scheduling.

At 3:12 in the video, you'll see the system correctly identify an already-booked 3pm slot and immediately offer alternative times. This real-time availability check happens:

  1. When the client suggests a time
  2. Before final confirmation
  3. During any rescheduling requests

Implementation tip: Always specify which calendar to use in your system prompt to avoid confusion if multiple calendars are connected.

Building Your PMGPT Booking Agent

The PMGPT agent serves as the conversational interface that guides clients through the booking process. Key configuration points:

  • AI Model: GPT-5.1 provides the ideal balance of conversational ability and tool use reliability
  • Greeting Message: Sets the tone for the interaction (e.g., "Welcome to Dynamic Dental")

The real power comes from connecting this agent to your MCP server (demonstrated at 6:20 in the video). This connection gives the AI access to all your configured tools while maintaining natural conversation.

The Booking Flow Prompt That Gets It Right

Your system prompt is the blueprint for how the agent handles bookings. The example in the video includes critical like:

  • Calendar settings specifying which calendar to use
  • Time zone requirements for all queries
  • Step-by-step booking flow (name → contact info → service type → preferred time)
  • Business hours and default appointment durations

At 5:45, you'll see how the prompt instructs the agent to always check availability before booking using the "find_busy_periods tool. This is the safeguard against double bookings.

Pro tip: Include service durations in your prompt so the agent knows how much time to block (e.g., cleanings 30 minutes, consultations 60 minutes).
