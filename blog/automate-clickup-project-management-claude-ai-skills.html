Claude Skills vs Commands vs Agents

Claude offers three ways to automate workflows: Commands (simple one-off instructions), Agents (dynamic problem solvers), and Skills (structured, reusable workflows). For project management automation, Skills provide the perfect balance of structure and flexibility.

Unlike Commands which are limited to basic operations, Skills can invoke Python scripts, maintain context across sessions, and handle complex multi-step workflows. Compared to Agents which may hallucinate solutions, Skills follow predetermined paths while still adapting to your specific project needs.

12K to 100 Tokens: The Efficiency Breakthrough

The most surprising discovery was the token efficiency of Skills versus MCPs. Where a ClickUp MCP consumes 12,000 tokens when loaded, the equivalent Skill uses just 100 tokens when idle thanks to progressive loading.

This efficiency comes from Skills only loading the specific functionality needed for each operation. The demo at 21:15 shows how the Skill dynamically loads just the "update task" script when making status changes, then unloads it immediately after.

Skill Architecture: Python + Config Files

The Skill uses a modular architecture with Python scripts for core functions (create_task.py, fetch_tasks.py, update_task.py) and Markdown config files for workspace-specific settings. This separation makes the Skill portable across different ClickUp spaces.

Key architectural decisions include:

  • Python scripts for API interactions (avoiding MCP rate limits)
  • Config files for space-specific settings (making the Skill reusable)
  • Progressive loading to minimize token usage
  • Fallback to MCP for complex queries not covered by scripts

Step-by-Step Implementation Guide

Step 1: Set Up the Skill Builder

Install Anthropic's Skill Builder plugin in Claude Code. This provides templates and scaffolding for creating new Skills. Without it, Claude tends to create commands instead of proper Skills.

Step 2: Define Your ClickUp Workflow

Map out exactly which project management tasks to automate. Start with high-frequency, repetitive tasks like status updates before moving to complex workflows.

Step 3: Create Python Scripts

Develop focused Python scripts for each automation task. Keep them simple initially - you can expand functionality later as shown in the demo's evolution.

Step 4: Configure Workspace Settings

Set up your config.md file with space IDs, list mappings, and other workspace-specific parameters that shouldn't be hardcoded in scripts.

Step 5: Test and Iterate

Start with basic functionality and gradually expand. The demo shows how the Skill grew from simple status updates to handling complete project hierarchies.

Implementation tip: Use Claude's feedback during development to identify missing functionality. The demo shows how due date handling was added after Claude flagged it as a gap during testing.
