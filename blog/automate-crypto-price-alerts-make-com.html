The Problem With Manual Crypto Tracking

Crypto traders know the frustration of constantly checking price charts. You set price alerts for Bitcoin at $50,000, Ethereum at $3,000, and Solana at $100 - only to miss the moves when you're sleeping, in meetings, or just not staring at your phone.

The solution? Automated price alerts that work while you don't. This Make.com workflow solves three critical pain points:

  • Missed opportunities: Price moves happen fast - the average crypto sees 3-5 significant daily swings
  • Emotional trading: Manual tracking leads to impulsive decisions rather than strategy execution
  • Platform fatigue: Jumping between CoinGecko, Telegram, and trading apps creates unnecessary complexity

Crypto traders spend 2.7 hours daily manually tracking prices according to a CoinDesk study. This automation reclaims that time while ensuring you never miss a strategic entry or exit point.

How Webhooks Trigger Crypto Automations

The workflow begins with a webhook - essentially a digital doorbell that tells Make.com when to start working. Think of it like setting an alarm clock for your automation rather than waking up to check prices yourself.

Here's how the webhook works in practice:

  1. Make.com generates a unique URL (your webhook endpoint)
  2. You configure this URL to receive price data from CoinGecko's API
  3. When new data arrives, the webhook "rings" and triggers the automation

Webhooks enable real-time processing without constant polling. Unlike scheduled checks that might miss price spikes between intervals, webhooks react immediately to market movements.

Fetching Real-Time Data from CoinGecko

The HTTP module acts as your digital assistant visiting CoinGecko to collect the latest market data. Configured with CoinGecko's API endpoint, it requests:

  • Top 10 coins by market cap (configurable to any number)
  • Current USD prices
  • 24-hour percentage changes
  • Market capitalization data

The API returns this data in JSON format - essentially a structured list where each coin's details are neatly organized. At this point, the raw data looks like a spreadsheet dumped into a single cell - all the information is there, but not yet usable.

Processing the Crypto Data Stream

The iterator module transforms the raw API response into actionable data by:

  1. Splitting the JSON response into individual coin records
  2. Extracting key metrics for each cryptocurrency
  3. Structuring the data for easy threshold checking

This step is crucial because CoinGecko's API returns all coins in one bundle. The iterator unpacks this bundle so we can evaluate each coin's performance individually.

Processing tip: Always set "Parse response" to Yes in the HTTP module configuration. This ensures proper data formatting for downstream processing.

Setting Profit & Loss Thresholds

The router module acts as your automated trading assistant, applying the rules you'd normally track manually:

Condition Action Example
Price increase ≥3% Send profit alert "BTC +4.2% in 24h"
Price decrease ≤-3% Send loss alert "SOL -3.5% in 24h"
No threshold crossed Log to Airtable Silent record-keeping

You can adjust these percentage thresholds to match your trading strategy. Conservative traders might use 5% thresholds while day traders might set 1.5% alerts.

Delivering Alerts to Telegram/Slack/Airtable

The final step formats and delivers notifications based on the router's decisions:

Telegram Alerts

Configured with Telegram's Bot API, these appear as direct messages with emoji indicators (📈 for gains, 📉 for losses) and key details:

  • Coin name and symbol
  • Current price
  • 24-hour change percentage
  • Market cap ranking

Airtable Logging

All coin data gets recorded to a spreadsheet for historical analysis, whether or not thresholds were crossed. This creates an audit trail showing:

  • Timestamp of each check
  • Complete market snapshot
  • Alert status (triggered/not triggered)

Pro tip: Add a Google Sheets integration to calculate moving averages and identify trends across your alert history.
