Basic Automation Workflow

The core automation follows a simple but powerful sequence in n8n:

Step 1: Scheduled Trigger

Configure the timer trigger to run every 1-3 days based on your ideal posting frequency. This replaces your manual posting reminder with an automated system that never forgets.

Step 2: Content Retrieval

The workflow pulls posts from n8n's built-in database tables (alternative to Google Sheets for better reliability). Each post includes:

  • Pre-written title
  • Body content
  • Call-to-action
  • Approval status

Step 3: Field Mapping

An Edit Fields node structures the content for GBP's API requirements, separating title, body, and CTA into distinct elements.

Step 4: Posting Loop

The Loop Item node ensures only one post publishes per trigger, creating a sustainable content drip rather than flooding your profile.

Key benefit: This basic version eliminates the blank page syndrome by working from pre-approved content, while maintaining perfect posting consistency.

Google Business Profile API Setup

The most technical part - but only needs doing once. At 8:45 in the video, we walk through the exact steps to:

1. Enable the API

Search for "Google My Business API" (the legacy name still used in developer tools) and enable it for your project.

2. Request Quota

The hidden hurdle: Google auto-rejects initial quota requests. The solution is to:

  • Submit the quota request form
  • When rejected (as expected), reply to the email with your business details
  • Typically approved within 24-48 hours after manual review

3. Create OAuth Credentials

Set up a web application credential in Google Cloud Console with your redirect URI. This provides the client ID and secret needed for n8n authentication.

4. Find Your Location ID

Available in GBP Settings → Advanced Settings. This tells the API which business location to post to.

Pro tip: Many tutorials skip the quota request follow-up step, causing frustration when auto-rejected. Persistence here is key - the API works perfectly once properly enabled.

Database vs. AI Posting

The workflow supports two content approaches - each with distinct advantages:

Database Method (Recommended)

Uses n8n's built-in tables to store and rotate through pre-approved posts. Benefits:

  • 100% quality control - no unexpected content
  • Easy to import from CSV (shown at 4:20 in video)
  • Posts can be categorized and scheduled strategically

AI Generation Method

Dynamically creates posts using OpenAI's API. Requires careful prompt engineering to:

  • Define post length (e.g., "30 words")
  • Specify content focus ("SEO services")
  • Include mandatory elements (title, body, CTA)

At 14:30, we demonstrate the structured data parser that formats AI output into GBP-ready components. This ensures consistency even with dynamic content generation.

Hybrid approach: Many clients use AI to generate draft posts that get human approval before being added to the database - combining automation efficiency with quality control.

Structured Content Formatting

GBP posts perform best with clear structure. At 18:50, we show the JSON schema that enforces this:

 {   "type": "object",   "properties": {     "title": { "type": "string", "description": "Post headline" },     "body": { "type": "string", "description": "Main content" },     "cta": { "type": "string", "description": "Call-to-action" }   },   "required": ["title", "body", "cta"] }

This structure ensures every post - whether from database or AI - includes:

  • Attention-grabbing title: 5-8 words mentioning local service
  • Scannable body: 25-50 words with concrete benefit
  • Clear CTA: "Call today" or "Book now" with contact method

The parser extracts these components automatically, ready for the GBP API.
