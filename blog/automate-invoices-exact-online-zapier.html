The Invoice Automation Problem

Accounts payable teams waste countless hours manually transferring approved invoices between systems. Each transfer risks data entry errors that create reconciliation headaches. The traditional process looks like this: 1) Someone approves an invoice in your workflow system, 2) An AP clerk rekeys all details into Exact Online, 3) Someone else must verify the entries match.

This manual handoff creates three serious business problems: 67% of accounting errors originate during data transfers between systems, AP staff spend 15-20 minutes per invoice on repetitive entry work, and month-end close gets delayed waiting for all invoices to be processed.

The automation opportunity: By connecting Triggre's approval workflow directly to Exact Online via Zapier, we eliminate all manual data transfer. Approved invoices appear in Exact Online within seconds - with perfect accuracy every time.

How the Integration Works

The automated invoice flow follows a precise sequence: 1) An invoice enters Triggre's system (manually or via OCR), 2) The approval workflow validates the PO number match, 3) Approved invoices trigger a Zapier webhook, 4) Zapier transmits all invoice data to Exact Online via a third-party connector.

At 2:15 in the video tutorial, you can see the exact moment an approved invoice appears in Exact Online - proving the automation works in real-time. The entire process happens without human intervention once configured.

Triggre Flow Configuration

The magic happens in Triggre's Flow Designer where we add two critical components: 1) An approval calculation that sets invoice_approved=true when PO numbers match, and 2) A "Send to Zapier" action that only triggers for approved invoices.

Key configuration points in Triggre:

  1. Add boolean output parameter "invoice_approved" to your ReceiveInvoice flow
  2. Create calculation that sets invoice_approved=true when PO matches
  3. Add conditional logic to only send approved invoices to Zapier
  4. Map all required fields (invoice number, PO, supplier, line items)

Pro Tip: Name your Zapier action descriptively (like "Send invoice to ExactOnline") so it's easily identifiable when setting up the Zap later.

Zapier Setup Steps

Zapier acts as the bridge between Triggre and Exact Online. The zap requires: 1) Triggre as trigger (webhook), 2) Data transformation if needed, 3) Exact Online action via third-party connector.

Critical Zapier configuration details:

  1. Select "Triggre" as trigger app with your specific action name
  2. Test trigger to confirm data flow from Triggre
  3. Add Exact Online action using a premium connector
  4. Map all fields including line items and supplier details
  5. Set up error notifications for failed transmissions

At 4:30 in the video, you'll see the Zapier interface with all field mappings configured - the most crucial part of the setup.

Exact Online Configuration

Exact Online requires proper setup to receive automated invoices. The three essential prerequisites are: 1) All suppliers must exist in Exact with correct codes, 2) Products/services need matching codes, 3) The Zapier connector must have API access.

Most implementation challenges occur when:

  • Supplier contacts aren't linked to the supplier record
  • Product codes differ between systems
  • Required fields in Exact aren't mapped from Triggre

Implementation Note: Spend time upfront verifying all master data matches between systems. This prevents 90% of integration issues.

Approval Logic Explained

The entire automation hinges on Triggre's approval calculation. When an invoice enters the system: 1) The workflow checks for PO number match, 2) A calculation sets invoice_approved=true/false, 3) This boolean determines the invoice's path.

Key aspects of the approval logic:

  • Uses simple "if PO matches then true else false" calculation
  • Output parameter drives all downstream actions
  • Only approved invoices trigger the Zapier integration
  • Rejected invoices route to exception handling

This elegant true/false switch makes the workflow incredibly flexible while maintaining simplicity.

Common Pitfalls to Avoid

After implementing this integration for dozens of clients, we've identified three frequent mistakes: 1) Incomplete field mappings, 2) Missing supplier/product master data, 3) No error handling for failed transmissions.

Implementation best practices:

  1. Create a test supplier in Exact Online for integration testing
  2. Verify all required fields are mapped in Zapier (including line items)
  3. Set up Zapier error notifications to catch transmission failures
  4. Monitor first 100 transactions closely before full rollout

Critical: The Exact Online connector requires specific field formats. Always test with sample data before going live.
