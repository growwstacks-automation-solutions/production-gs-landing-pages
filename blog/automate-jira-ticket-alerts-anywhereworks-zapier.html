The Manual Notification Problem

Development teams using Jira often struggle with inefficient notification workflows. When a new ticket is created, someone must manually copy details like ticket number, title, and link - then paste them into team chat platforms like AnywhereWorks. This process creates several pain points:

  • Delayed awareness: Tickets often sit unnoticed for hours until someone manually shares them
  • Inconsistent formatting: Each team member formats notifications differently, causing confusion
  • Human error: Important tickets get missed when people forget to notify the team

Teams waste 3-5 hours weekly on manual Jira notification tasks that could be fully automated. The Zapier integration shown in this tutorial eliminates this busywork while ensuring every ticket gets properly communicated.

Zapier Setup Overview

The automation follows a simple trigger-action pattern in Zapier:

  1. Trigger: New issue created in Jira (filterable by project, status, etc.)
  2. Action: Send collab message in AnywhereWorks with ticket details

Key benefit: Once set up, this workflow runs automatically for every matching ticket, requiring zero ongoing maintenance from your team.

Configuring the Jira Trigger

The Jira trigger setup involves three critical steps:

Step 1: Connect Your Jira Account

Zapier needs API access to your Jira Cloud instance. You'll provide:

  • Your company's Jira domain URL (e.g. company.atlassian.net)
  • API permissions through OAuth authentication

Step 2: Set Trigger Conditions

Choose "New Issue" as the trigger event, then specify:

  • Project: Which Jira project(s) to monitor
  • Status: Typically "Open" for new tickets
  • Assignee: Optional filter for specific team members

Step 3: Test the Trigger

Zapier will fetch sample tickets to verify the connection works before proceeding to the action setup.

Connecting the AnywhereWorks Action

The action configuration determines how ticket details appear in AnywhereWorks:

Step 1: Authenticate AnywhereWorks

Similar to Jira, you'll authorize Zapier to send messages to your AnywhereWorks account through OAuth.

Step 2: Choose Message Type

Select "Send a Collab Message" as the action type. This posts notifications to specific team channels rather than direct messages.

Step 3: Configure Message Details

Specify:

  • Collab Channel: Which AnywhereWorks channel receives notifications
  • Message Content: The formatted text with ticket details

Pro Tip: Use Zapier's dynamic fields to insert Jira ticket data like key, summary, and assignee directly into your message template.

Message Formatting Tips

Well-formatted notifications help teams quickly understand ticket context. Here's an optimized message structure:

Example Message Template:
New ticket: [Summary]
Project: [Project]
Key: [Clickable Jira Link]
Assignee: [Assignee]
Status: [Status]

To create clickable Jira links:

  1. Get your Jira instance's base URL (e.g. https://company.atlassian.net/browse/)
  2. Concatenate this with the ticket key using Zapier's text formatting tools

This transforms plain ticket numbers into direct links that open the ticket in one click.
