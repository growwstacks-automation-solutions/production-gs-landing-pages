Structuring Your Airtable Base

A properly structured Airtable base is the foundation for successful automation. Your database needs both content fields and workflow status markers that Make.com can read and act upon.

Essential fields include:

  • Title: The campaign name that will appear in Mailchimp
  • Author: Single select field for attribution
  • Excerpt: Short preview text for email subjects
  • Body Content: Long text fields with rich formatting enabled
  • Status: Single select with "Not Yet Uploaded", "Send to Mailchimp", and "Complete" options

Create a filtered view showing only records marked "Send to Mailchimp" - this becomes your automation trigger. At 2:15 in the video, you'll see how this filtering works in practice.

Creating the Mailchimp Template

Mailchimp's drag-and-drop editor simplifies template creation, but we need to prepare it for dynamic content insertion. The key is adding unique placeholder text that Make.com will later replace with Airtable content.

Best practices for automation-ready templates:

  1. Build your layout using Mailchimp's visual editor (no raw HTML initially)
  2. Add placeholder text like "INSERT_TITLE" where dynamic content goes
  3. Export the template as HTML after completing the visual design
  4. Use an HTML editor to copy the full code block for Make.com

Pro Tip: At 4:30 in the tutorial, you'll see how to use the free HTML.net editor to extract your template code without altering the formatting.

Building the Make Automation

The Make.com scenario acts as the bridge between Airtable and Mailchimp, handling content transformation and delivery. This six-module workflow eliminates all manual steps:

  1. Airtable Search: Finds records marked "Send to Mailchimp"
  2. Markdown Conversion: Transforms Airtable rich text to HTML
  3. Template Population: Replaces placeholders with actual content
  4. Campaign Creation: Builds the Mailchimp email with your content
  5. Status Update: Marks the record as complete in Airtable

The most technical step involves mapping Airtable fields to the HTML template. At 7:45 in the video, watch how to use Ctrl+F to quickly locate and replace each placeholder.

Step-by-Step Workflow Breakdown

Step 1: Configure Airtable Trigger

Set up the "Search Records" module to check your filtered view every hour. The automation only processes records with the "Send to Mailchimp" status, preventing duplicate sends.

Step 2: Convert Markdown to HTML

Make.com's markdown modules transform Airtable's formatted text into Mailchimp-compatible HTML. This preserves bold, italics, lists, and links from your original content.

Step 3: Map Content to Template

Paste your exported HTML template into the "Create Campaign" module. Use search-and-replace to swap each placeholder (like INSERT_TITLE) with the corresponding Airtable field.

Step 4: Set Campaign Details

Configure the Mailchimp campaign settings, mapping the subject line, sender info, and audience list. These can be hardcoded or pulled from additional Airtable fields.

Step 5: Update Airtable Status

The final module changes the record's status to "Complete", preventing the automation from reprocessing it. This creates an audit trail of sent campaigns.

In Summary: 1) Trigger on status change → 2) Convert formatting → 3) Populate template → 4) Create campaign → 5) Mark complete. Five steps replace what was previously an hour of manual work.

Testing and Troubleshooting

When first implementing this automation, test with a small Mailchimp segment before sending to your full list. Common issues and solutions:

  • Formatting Errors: Ensure Airtable long text fields have rich formatting enabled
  • Missing Content: Verify all placeholders in the HTML template have matching Airtable fields
  • Duplicate Sends: Confirm your status field updates correctly after processing
  • API Limits: Space out automation runs if hitting Mailchimp's rate limits
