Publication dates require Google searches. Prices change constantly. Before you know it, what should be a 30-second task becomes a 5-minute chore.

The creator in our video transcript experienced this firsthand during "Notion November." After manually dragging book covers into his database, he realized: "That's way too many steps for a nerd of my caliber." This moment of frustration sparked the automation journey we're about to explore.

The Amazon API Solution

The breakthrough came when the creator asked: "What if I could find a way to automate this?" The answer lay in Amazon's Product Advertising API - the same system that powers affiliate links. As an Amazon Associate (which many content creators already are), you get free access to book metadata including:

  • High-resolution cover images
  • Accurate publication dates
  • Current pricing data
  • Author and publisher information
  • ISBN and page counts

In the video at 4:32, the creator explains: "I actually have Amazon influencer and Amazon associate. So both of the above are going to work for me." This existing affiliate status meant no additional approvals were needed - just grabbing his API keys from the Amazon Associates dashboard.

Building the Make.com Workflow

Make.com (formerly Integromat) serves as the automation bridge between Notion and Amazon. The workflow follows this sequence:

Step 1: Trigger on New Notion Database Entries

The system watches your Notion book database for new entries containing Amazon links. At 7:15 in the video, the creator notes: "Make.com trigger a notion database... runs automatically every few minutes."

Step 2: Parse the Book Link

The workflow identifies whether the link points to Amazon, Audible, Goodreads, or another supported platform. Conditional routing handles each type differently.

Step 3: Fetch Metadata via Amazon API

Using your Associate credentials, Make.com queries Amazon's API for the book's details. The video emphasizes: "You need the access key, the secret access key, and my associate ID."

Step 4: Update Notion Database

The retrieved data populates your specified Notion fields automatically. The creator demonstrates this at 9:47 by showing how cover images appear instantly.

Pro Tip: Add error handling to manage API limits and missing data. The workflow can even space out requests to avoid hitting Amazon's rate limits.

Notion Database Setup

Your Notion database needs these key properties to maximize the automation's value:

  • Book Link: URL field for the Amazon/product page
  • Cover Image: Files & media property for automatic cover art
  • Title: Standard text field
  • Author: Text or relation field
  • Publication Date: Date property
  • Current Price: Number field that can update automatically

At 3:18 in the video, the creator experiments with different image placement options: "I was trying to figure out how to get this graphic in here... changing the cover image to be the book image." The final workflow uses a dedicated image property for consistency.
