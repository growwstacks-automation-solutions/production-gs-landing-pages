The API Limitation Problem

Restaurant reservation platforms often restrict API access to protect their systems—leaving staff to manually handle booking changes. This creates a frustrating bottleneck where:

Our client discovered workaround came when a client's reservation platform refused API access for modifications. Rather than accepting the limitation, we built a solution that works within these constraints while still delivers automation benefits.

Key insight: When direct API integration isn't possible, Telegram notifications become your "human API"—alerting staff the moment changes are requested so they can be processed in real-time.

Why Telegram Solves the Notification Challenge

During peak dinner service, restaurant managers can't constantly check emails or CRM systems. Telegram's instant mobile notifications solve this by:

  • Pushing alerts directly to staff phones with a distinctive sound
  • Displaying all booking change details in one glanceable message
  • Requiring no additional app login or navigation

The system we built captures customer requests via Vapi voice agent, structures the data in Make.com, then pushes formatted alerts to designated Telegram groups—creating a seamless handoff between AI and human staff.

The Complete Workflow Overview

When a customer calls to modify their reservation:

  1. 1. Vapi voice agent collects all necessary details (name, phone, booking reference, etc.)
  2. 2. Make.com routes the request cancellation, rescheduling, or modification path
  3. 3. Google Sheets logs the request for record-keeping
  4. 4. Telegram instantly notifies the appropriate staff member

Implementation tip: Use Make.com's router module to split workflows at 2:10 in the video tutorial—this handles all three modification types in one scenario while keeping logic organized.

Configuring the Vapi Voice Agent

The Vapi agent needs specific prompts to handle restaurant modification requests effectively:

You are a restaurant reservation assistant. Strictly collect: - First and last name - Phone number - Current booking details - Requested changes Confirm all details before proceeding

At 4:35 in the video, you'll see how we configured the agent to gracefully handle edge cases like customers changing their minds mid-cancelling after requesting a modification.

Make.com Integration Setup

The Make.com workflow performs three critical functions:

  • Receives webhook from Vapi with raw customer data
  • Structures information into standardized format
  • Routes to appropriate processing path
  • Generates human-readable Telegram messages

The key innovation shown at 7:20 is using ChatGPT via Make.com to transform raw data into perfectly formatted notifications that include all necessary context without overwhelming staff.

Creating the Telegram Notification Bot

Setting up the Telegram bot involves three key steps:

  1. 1. Create bot via BotFather to get API token
  2. 2. Configure Make.com's Telegram module with token
  3. 3. Set up dedicated group for staff notifications
