The Sales Tax Calculation Challenge

Manually calculating sales tax across different US jurisdictions is a nightmare for businesses. While some states like Michigan have simple flat rates (6%), others like Ohio and Indiana have complex layered taxes that vary by county and sometimes city. Certain jurisdictions tax shipping while others don't. Even worse, some zip codes span multiple counties with different rates - meaning the same address could have different tax obligations depending on which side of the county line it falls on.

This workflow solves these problems by automatically determining the exact tax jurisdiction for any US address, calculating the combined state/county/city rate, and applying it correctly to both product costs and shipping (when applicable). The system handles all edge cases that typically require manual research - saving businesses hours per week in administrative work.

Real-world impact: A business processing 50 orders weekly saves approximately 200 hours annually by automating tax calculations that previously took 5-15 minutes per order depending on jurisdiction complexity.

Workflow Overview

The automation follows a clear sequence that eliminates manual tax research. At 3:45 in the tutorial video, you can see the complete flow from form submission to tax calculation to email output.

Step-by-step process:

  1. Customer submits order details via JotForm (or any form platform)
  2. Make.com receives the submission and extracts address, product costs, and shipping
  3. System queries Zip-Tax API with the full address to determine exact jurisdiction
  4. Workflow calculates tax based on product costs + shipping (if taxable in that jurisdiction)
  5. Results are formatted into a clean email with all tax details

The beauty of this system is its flexibility - the JotForm trigger can be replaced with any form system or direct API call, and the email output can be modified to create PDFs, database entries, or CRM updates based on your business needs.

JotForm Setup (3:45 in video)

The form requires just four key fields to enable accurate tax calculations: customer email (for results), full shipping address (street, city, state, zip), total product cost, and shipping amount. As shown at 3:45 in the video, the address field is critical because some zip codes span multiple tax jurisdictions.

Important configuration details:

  • Enable "Submit Another Response" option so users can process multiple orders
  • Use JotForm's free tier which includes API access (no premium features required)
  • Simple validation ensures required fields are completed before submission

The form acts as the user-friendly frontend, while all complex tax logic happens automatically in the background through Make.com's workflow.

Make.com Scenario Configuration

The Make.com scenario begins with a JotForm webhook that triggers on each form submission. This connects directly to your form without requiring complex coding. The setup process involves:

  1. Creating a new scenario in Make.com
  2. Selecting "JotForm" as the trigger app (note: it's "JotForm" not "JotForms")
  3. Choosing "Watch for submissions" as the trigger event
  4. Connecting to your specific form

Once configured, every form submission will automatically flow into the scenario for processing. The system handles all the API connections behind the scenes, allowing you to focus on mapping the data fields needed for tax calculation.

Zip-Tax API Integration

The secret sauce of this workflow is the Zip-Tax API integration, which provides up-to-date tax jurisdiction data for any US address. Key features:

  • Free tier available for low-volume usage (paid plans for high-volume businesses)
  • Returns combined tax rates including state, county, and city layers
  • Identifies whether shipping is taxable in each specific jurisdiction
  • Handles complex cases where zip codes span multiple counties

The API requires just two pieces of information - your API key (available in your Zip-Tax account) and the customer's full address. The HTTP request module in Make.com handles all the technical communication, returning clean data that's easy to parse for the tax calculation step.

Tax Calculation Logic

The workflow uses Make.com's "Set Multiple Variables" module to perform all tax calculations automatically. This handles three critical determinations:

  1. Is shipping taxable? (varies by jurisdiction - some states tax it, others don't)
  2. What's the combined tax rate? (state + county + city rates where applicable)
  3. Total tax owed: Product cost (plus shipping if taxable) × tax rate

The system pulls these values directly from the Zip-Tax API response, ensuring accuracy without manual rate tables. For example, when testing an Indianapolis address (14:30 in the video), the workflow correctly identified:

  • 7% state tax rate (no county or city additions)
  • Shipping was taxable
  • $28 tax on $400 total ($300 products + $100 shipping)

This logic adapts automatically to each address's specific tax rules, eliminating guesswork and potential audit risks from incorrect manual calculations.

Email Output Configuration

The final step formats all tax details into a clear email (18:50 in the video). While the tutorial uses Gmail, this could connect to any email system or CRM. The email includes:

  • Total tax amount due
  • Breakdown of tax rate components (state, county, city if applicable)
  • Clear indication of whether shipping was taxed
  • Jurisdiction names and codes for record-keeping

The raw HTML option in Make.com's email module allows complete customization of the output format. Businesses can modify this to match their branding or include additional order details as needed.

Pro tip: Add a "Submit Another" link to the email (like the JotForm option) allowing staff to quickly process multiple orders without returning to the form.
