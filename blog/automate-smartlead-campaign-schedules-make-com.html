Make.com + Smartlead API Solution

The workflow solves this through three simple Make.com modules that leverage Smartlead's API:

  1. HTTP Request: Retrieves all campaigns from your Smartlead account
  2. Iterator + Filter: Identifies only active campaigns needing updates
  3. HTTP Request: Updates each campaign's schedule with holiday pauses

Unlike manual edits, this approach handles unlimited campaigns in seconds. The JSON-based updates ensure perfect formatting every time, eliminating the syntax errors that plague manual API calls.

Step 1: Retrieving All Campaigns

The foundation is Smartlead's GET /api/v1/campaigns endpoint. In Make.com, we configure an HTTP module with:

  • Base URL: Your Smartlead instance (e.g. https://app.smartlead.ai)
  • Endpoint: /api/v1/campaigns
  • Headers: Authorization: [Your API Key]

At 2:45 in the video tutorial, you'll see how to locate your API key in Smartlead's settings. This first call returns an array of all campaigns - active, paused, and archived.

Pro Tip: Store your API key as a Make.com variable rather than hardcoding it. This simplifies key rotation and keeps credentials secure.

Step 2: Filtering Active Campaigns

The iterator module processes the campaign array, passing only active sequences to the next step. We add a filter with:

 status = "active"

This exclusion prevents wasting API calls on campaigns that don't need updates. The filtered output contains only the campaigns currently sending emails - exactly what we want to pause for holidays.

At 4:30 in the video, notice how archived campaigns automatically skip the update process. This selective handling becomes crucial as your campaign library grows into the hundreds.

Step 3: Updating Schedules

The final HTTP module uses Smartlead's PUT /api/v1/campaigns/{id} endpoint. Each active campaign's ID gets inserted into the URL path dynamically. The JSON body contains the new schedule:

 {   "days_of_week": [1,2,3,4,5], // Monday-Friday   "start_time": "09:00",   "end_time": "17:00",   "timezone": "America/New_York" }

To pause campaigns, we simply set days_of_week to an empty array []. The video at 7:15 shows how to test this with one campaign before applying globally.

Timezone Handling for Global Teams

Smartlead requires IANA timezone strings (e.g. "America/Los_Angeles"). The workflow automatically includes:

  • Your preferred timezone in all update requests
  • Validation to prevent invalid timezone entries
  • Option to specify different timezones per campaign group

This ensures your holiday pauses align with local observance times. For teams spanning multiple regions, we can create parallel workflows for each major timezone.
