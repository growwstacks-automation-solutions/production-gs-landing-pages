Make.com Basics for Beginners

Make.com (formerly Integromat) is a visual automation platform that connects apps through customizable workflows called scenarios. Unlike simpler tools that only handle "if this then that" triggers, Make allows complex data transformations between steps - crucial for adapting Notion's date formats to ContentStudio's API requirements.

When you log into Make, your dashboard shows active scenarios, usage metrics, and template library. Templates offer pre-built workflows you can customize, but we'll build ours manually for complete control. Every scenario consists of modules connected in a flow - starting with a trigger (like new Notion database entries) followed by actions (like scheduling ContentStudio posts).

Setting Up the Notion Trigger

The automation begins when you add content to your Notion calendar. In Make, create a new scenario and select Notion as your trigger app. Choose "Watch Database Items" as the specific trigger - this monitors your content calendar for new entries.

Make will request access to your Notion workspace. During authorization, you can limit access to just your social media calendar database. Once connected, you'll need your Database ID - easily found by searching your database name in Make's interface. Set the polling interval based on how frequently you update the calendar (15-60 minutes works for most teams).

Pro Tip: Use Notion's "Last Edited Time" property as an additional filter to ensure the automation only processes newly added or modified calendar entries, preventing duplicate posts.

Connecting ContentStudio

With the Notion trigger configured, add ContentStudio as your action app. You'll need an API key from ContentStudio's settings - this grants Make permission to schedule posts on your behalf. In Make, create a new ContentStudio connection and paste your API key.

Map the fields from your Notion database to ContentStudio's post requirements: Notion's "Caption" becomes ContentStudio's "Post Text", image URLs map directly, and the posting date determines scheduling. This is where many automations fail - ContentStudio requires dates in YYYY-MM-DD HH:MM:SS format, while Notion often uses different date presentations.

Handling Date Formatting

The date format mismatch between Notion and ContentStudio would cause the automation to fail without intervention. Make solves this with its Tools module. Insert a "Set Variable" module between Notion and ContentStudio, using the formatDate() function to transform the date.

The formula takes Notion's original date and converts it to ContentStudio's required structure. For example: formatDate(NotionDate; "YYYY-MM-DD HH:mm:ss"). You'll then map this formatted date to ContentStudio's scheduling field instead of the raw Notion date.

Timezone awareness: Add a second Tools module to convert dates to your target timezone if your team works across regions. The formula formatDate(NotionDate; "YYYY-MM-DD HH:mm:ss"; "America/New_York") ensures posts schedule correctly for your audience's local time.

Testing and Activating the Workflow

Before running the automation live, test with a sample Notion entry. Make's scenario tester lets you walk through each step: Notion data retrieval, date formatting, and ContentStudio scheduling. Verify the post appears correctly in ContentStudio's planner with the right caption, image, and scheduled time.

Once testing succeeds, activate the scenario and set its execution schedule. For most teams, checking every 15-30 minutes balances responsiveness with API quota conservation. Make will now automatically process new Notion calendar entries, format the data, and schedule ContentStudio posts without any manual intervention.

Advanced Possibilities

This basic workflow can evolve into sophisticated automations. Add error handling modules to catch and notify about formatting issues. Insert approval steps requiring manager sign-off before scheduling. Combine with AI tools to auto-generate post variations from Notion bullet points.

Make's true power emerges when chaining multiple apps. Imagine: Notion triggers the workflow → AI generates post variations → Manager approves in Slack → Scheduled in ContentStudio → Posted metrics log back to Notion. These multi-app workflows automate entire marketing processes end-to-end.

Scalability secret: Agencies use this same workflow template across all client accounts by creating separate scenarios for each Notion database and ContentStudio workspace. Make's organizational tools keep everything manageable at scale.
