The Subtitle Problem Every Creator Faces

Content teams know they should add subtitles to every video - 85% of social media videos are watched without sound. But the manual process kills productivity: record video → send to editor → wait 1-2 days → download files → rename everything → upload to each platform separately. At 3-5 hours per video, most creators either post inconsistently or skip subtitles entirely.

The breakthrough came when we realized the entire workflow follows predictable patterns. The same file transfers, the same naming conventions, the same platform-specific formatting requirements. These repetitive tasks are perfect for automation.

Key insight: Adding subtitles manually costs $75-$150 per video in editing time. This automation reduces that to $2-$5 in AI processing costs while being 3x faster.

How Automation Solves It in 4 Steps

This Make.com workflow transforms a multi-day manual process into a 10-minute automated pipeline. Here's the complete journey from raw video to published content:

Step 1: Automatic Trigger

The system watches a specific Google Drive folder. When a new video appears, it immediately grabs the file and starts processing - no manual initiation needed.

Step 2: AI Caption Generation

Bookoly's transcription AI generates accurate subtitles in multiple formats (SRT, VTT, TXT) with 90-95% accuracy for clear audio.

Step 3: Platform-Specific Formatting

GPT reformats captions for each social platform - longer versions for YouTube/Facebook, concise versions for Twitter/Instagram - all stored in a central database.

Step 4: Organized File Output

Final files are automatically saved to designated Google Drive folders with consistent naming conventions, while temporary files are deleted to save space.

Before & After: What took 3-5 hours of human work now completes in 10-15 minutes with zero active effort from the creator.

Triggering the Workflow

The magic begins with Make.com's Google Drive module. At 2:15 in the video, you'll see how we configure the trigger to monitor a specific "Raw Videos" folder. Key settings:

  • Watches for new files in the designated folder only
  • Filters by video file extensions (MP4, MOV, etc.)
  • Captures all file metadata including uploader info

This trigger eliminates the most common bottleneck - remembering to start the process. Team members simply drop videos where they normally would, and automation handles the rest.

AI Caption Generation with Bookoly

Bookoly's API receives the video file and returns multiple caption formats within minutes. The workflow:

  1. Sends video to Bookoly via HTTP request
  2. Waits for processing completion (typically 3-7 minutes)
  3. Downloads SRT, VTT, and plain text versions
  4. Stores all files with consistent naming conventions

At 5:30 in the tutorial, notice how we handle errors - if Bookoly fails (rare), the workflow retries twice before notifying the team via Slack.

Social Media Formatting with GPT

Raw transcripts need reformatting for each platform. Our GPT module:

  • Creates verbose YouTube descriptions with timestamps
  • Generates concise Instagram captions under 125 characters
  • Produces threaded Twitter versions in 240-character chunks
  • Stores all variations in Google Sheets for future repurposing

The prompt engineering (shown at 8:45) ensures consistent brand voice across platforms while adapting to each medium's requirements.

Automatic File Organization

Chaotic file storage wastes hours. This workflow implements:

  • Dedicated subfolders for originals, subtitles, and social versions
  • Consistent naming: [Date]_[VideoTitle]_[Platform]_[Type]
  • Automatic cleanup of temporary processing files
  • Google Sheets tracker with direct links to all assets

At 12:20, you'll see how we use Make's Google Drive module to maintain this structure automatically, eliminating manual file management.
