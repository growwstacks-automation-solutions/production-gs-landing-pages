The Enquiry Crisis Every Service Business Faces

Picture this: It's Monday morning at a busy handyman service company. The team arrives to find 47 new website enquiries from the weekend - each needing a personalized response with service details, availability questions, and pricing estimates. What should be good news (lots of potential clients!) quickly becomes an operational nightmare.

This was the daily reality for our client - a general maintenance and handyman service provider using their website contact form as the primary lead generation channel. Their standard process:

  1. Customer submits web form with service request details
  2. Enquiry emails flood the shared operations inbox
  3. Staff manually reads each email, researches the service type
  4. Drafts a personalized response with follow-up questions
  5. Sends reply (3-10 minutes per enquiry)
  6. Attempts to track conversations in a messy inbox

The hidden cost: At just 5 minutes per enquiry (conservative estimate), 50 daily enquiries consume over 4 hours of staff time - before any actual work is scheduled. Multiply that across a week, and you're looking at 20+ hours lost to email tennis.

Three Problems Solved With One Automation

The handyman service identified three critical pain points that were eroding their efficiency and customer satisfaction:

1. Inbox Overload

With no automatic sorting or prioritization, legitimate enquiries got buried under spam and low-intent submissions. Important client requests slipped through cracks during busy periods.

2. Response Bottleneck

Each enquiry required manual processing time that scaled linearly with volume - more enquiries meant either longer response times or more staff dedicated to email.

3. No Spam Filtering

Approximately 15-20% of submissions were test messages, pranks, or non-serious enquiries wasting staff time. There was no automated way to identify and filter these.

The breakthrough: By combining Make.com's workflow automation with Zoho Mail's API capabilities and some smart AI processing, we built a system that handles all three problems simultaneously without requiring additional staff.

How The Make.com Automation Works Step-by-Step

The automation triggers the moment a website enquiry arrives in the company's Zoho Mail inbox. Here's the complete sequence:

Step 1: Instant Trigger via Webhook

Instead of polling the inbox on a schedule, we configured a mail hook that forwards enquiries to Make.com immediately upon arrival. This eliminates delays and reduces API calls.

Step 2: Subject Line Filter

The first module confirms the email subject matches the exact website enquiry format, ignoring all other incoming mail (including spam).

Step 3: Artificial Response Delay

A 2-3 minute "sleep" period makes the automated response feel more human (avoiding robotic instant replies that customers distrust).

Step 4: Information Extraction

Make.com's text parser extracts the customer's name, email, requested service type, and any custom message from the enquiry body.

Step 5: Variable Assignment

Key details are stored as variables including the HTML email signature template used for all outgoing responses.

Technical note: The system captures the Zoho Mail message ID from headers - crucial for later manipulating the original email (marking as read/moving to folder).

Smart Categorization With Router Logic

The automation's intelligence comes from its service-type routing system. Here's how it works:

Four service categories: The router analyzes the requested service and assigns each enquiry to one of four pre-defined response tracks (A-D) based on service type.

For example:

  • Category A: Home/office services (drywall, painting, etc.)
  • Category B: Equipment hire (TLBs, scaffolding)
  • Category C: Rubble removal/debris clearance
  • Category D: Emergency call-outs

Each category has its own:

  • Response template with service-specific details
  • Pricing guide attachments
  • Follow-up questions
  • Availability calendar link

The router ensures customers get accurate, relevant information immediately - without staff needing to research each service type.

Professional Responses That Feel Human

The automated replies are designed to:

  1. Personalize with the customer's name and service details
  2. Provide immediate value with service-specific information
  3. Maintain brand voice and professionalism
  4. Guide next steps clearly

HTML formatting secret: By storing the company's email signature as an HTML variable, every auto-response maintains consistent branding without rebuilding the template each time.

The system even handles practical details like:

  • Attaching relevant PDF guides based on service type

Customers report higher satisfaction with these detailed, prompt responses compared to the previous slower manual replies.

Inbox Decluttering With Custom API Actions

After sending the response, the automation performs two crucial inbox management actions:

1. Marks Email as Read

Using Zoho Mail's "Mark as Read" module with the captured message ID, the system prevents duplicate processing of the same enquiry.

2. Moves to Processed Folder

Since Make.com has no native "Move Email" action for Zoho, we implemented a custom API call using Zoho's documentation. This:

  • Keeps the main inbox clean
  • Creates an audit trail
  • Allows for reporting on enquiry volumes

Operational benefit: Staff now see only unprocessed enquiries in their main view, with all completed conversations neatly archived but searchable when needed.
