Key Workflow Components

Trio Chat workflows consist of several core components that work together to automate conversations:

1. Triggers

Defines when the workflow should start - typically when a new conversation begins or when specific conditions are met.

2. Action Nodes

Perform specific tasks like sending messages, asking questions, or updating contact records.

3. Branches

Create conditional paths based on user responses or system data.

The video at 1:10 demonstrates how these components connect to form complete conversation flows.

Implementing Conditional Logic

Conditional branches transform simple chatbots into intelligent conversation handlers. At 1:45 in the demo, you'll see how to create paths that:

  • Route sales inquiries to your pricing information
  • Send support requests to your help desk
  • Provide FAQ answers automatically
  • Escalate complex issues to human agents

Each branch evaluates the conversation context to determine the appropriate next step, creating a dynamic experience that feels personalized to each customer.
