How the Automation Works

The workflow transforms topic ideas into publish-ready videos through five connected steps:

Step 1: Airtable Content Queue

Your video topics live in an Airtable base with status fields (To-Do, In Progress, Done). Make.com monitors this queue for new entries.

Step 2: AI Script Generation

For each new topic, Make.com sends a structured prompt to OpenAI requesting a 1-minute script (180-240 words) divided into scenes.

Step 3: Video Rendering

The JSON2VIDEO module receives the script and renders visuals using predefined templates, adding AI voiceover and transitions.

Step 4: Cloud Storage

Completed videos upload to Cloudinary for permanent storage, with the URL saved back to Airtable.

Step 5: Status Update

The workflow marks the record as Done, preventing duplicate processing.

Key insight: This isn't just automation — it's an entire video production pipeline that runs while you sleep. Queue 10 topics on Monday, wake up to 10 finished videos on Friday.

Setting Up Airtable

Your Airtable base serves as the content management hub. The demo workflow uses these key fields:

  • Topic: The video subject (e.g., "Mental Toughness for Entrepreneurs")
  • Language: For multilingual voiceover support
  • Voice: Preferred AI voice style
  • Status: To-Do/In Progress/Done workflow tracking
  • Cloudinary URL: Final video storage location

Make.com connects via API using your Airtable personal access token. The "Search Records" module looks for entries where Status equals "To-Do," processing them first-in-first-out.

Pro Tip: Add a "Priority" field to sequence your video queue. High-priority topics jump the line when you're preparing for a product launch or trending event.

Configuring OpenAI Scripting

The magic happens in the prompt engineering. The demo workflow uses this structure:

Act as an expert motivational copywriter and visual storyboard artist.   Create a 1-minute (180-240 word) script divided into 3 scenes.   Topic: [Airtable Topic Field]   Tone: Inspirational yet practical   Output Format: JSON with scene details and visual descriptions

Key configuration notes:

  • Using GPT-4-turbo for higher quality output
  • Temperature set to 0.7 for balanced creativity
  • JSON response format for easy parsing
  • Token limit adjusted for 1-minute scripts

Video Rendering with JSON2VIDEO

JSON2VIDEO converts scripts into videos through API calls specifying:

  • Template ID: Predefined visual style
  • Script: The OpenAI-generated content
  • Voice: Selected from 120+ AI voices
  • Resolution: 1080p vertical or horizontal
  • Subtitles: Auto-generated with customizable fonts

The service handles all rendering in the cloud, typically completing 1-minute videos in under 10 minutes. You receive a web URL where the video can be previewed and downloaded.

Cost Saver: JSON2VIDEO offers batch rendering discounts. Queue multiple videos overnight when API rates are lowest.

Cloud Storage Options

While JSON2VIDEO provides temporary hosting, the workflow includes Cloudinary integration for permanent storage. Why this matters:

  • Reliability: JSON2VIDEO may purge older videos
  • Organization: Cloudinary folders keep content sorted
  • Distribution: Direct links for social sharing
  • Backup: Protect against service outages

The Make.com scenario automatically uploads each video to your Cloudinary account using an upload preset. The public URL saves back to Airtable for easy access.
