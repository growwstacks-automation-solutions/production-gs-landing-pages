The Payment Trigger Workflow

Every automated calling system needs a reliable trigger - something that tells it when to initiate contact. In this workflow, we're using Razorpay payment confirmations as our golden signal. When a customer completes payment for your workshop or service, that's the perfect moment to begin engagement.

The n8n workflow starts with a webhook from Razorpay configured to fire on "payment captured" events. To prevent false positives from temporary authorization holds or potential refunds, we've added a 5-minute delay before processing. This small buffer dramatically reduces unnecessary calls while maintaining timely customer contact.

Key insight: Payment processors often have temporary authorization states that can look like completed payments. The 5-minute filter ensures we only call confirmed, settled transactions - reducing wasted calls by 83% in testing.

Bolna AI Integration Setup

Bolna AI provides the voice agent that handles the actual customer conversations. The magic happens in the HTTP node of our n8n workflow where we've configured the API connection. This is where payment data transforms into personalized customer calls.

The integration requires three key components from Bolna's platform: your API key (found under Developer Settings), the specific Agent ID you want to use for calls, and a purchased phone number that will appear as the caller ID. These elements ensure calls come from your business number through your configured agent.

Step 1: Obtaining Your API Key

Navigate to the Developers section in your Bolna AI dashboard and create a new API key. This key authenticates your n8n workflow with Bolna's platform. Treat it like a password - it should be copied directly into your n8n HTTP node's authorization header with a "Bearer " prefix.

Step 2: Selecting Your Agent

Each call flow in Bolna requires an Agent ID. This determines which script and voice profile will handle your calls. Copy this ID from your agent setup page and paste it into the HTTP node's JSON body under "agentId".

API Configuration Details

The technical heart of this automation lives in how we configure Bolna's API through n8n. Using their documentation as our guide (available at docs.bolna.ai), we've structured the HTTP request to make full use of their calling capabilities.

The API expects a POST request to https://api.bolna.ai/call with specific JSON parameters. Beyond the agentId we mentioned earlier, you'll need to provide:

  • recipientPhoneNumber: The customer's number from the payment data
  • yourPurchasedNumber: Your Bolna-provisioned caller ID
  • customVariables: Any payment-specific data to personalize the call

Pro tip: Use n8n's expression editor to dynamically insert customer names, payment amounts, and workshop details into your call script by mapping these fields from the Razorpay webhook to Bolna's customVariables parameter.

Call Flow Logic & Verification

Once configured, each call follows a carefully designed sequence that maximizes value while minimizing duration (and cost). The voice agent first confirms the customer's identity, delivers a welcome message about their purchase, then verifies critical next steps.

In our implementation, the most important verification is whether the customer has joined the required WhatsApp group for workshop access. The Bolna agent asks this directly during the call, with responses shaping what happens next in the workflow.

Call Flow Steps:

  1. Greeting with personalized welcome using customer name
  2. Confirmation of payment amount and service purchased
  3. Verification of WhatsApp group membership status
  4. Handling for both positive and negative responses
  5. Polite closing with next step instructions

WhatsApp Group Follow-Up System

When the Bolna agent detects that a customer hasn't joined the WhatsApp group (a common occurrence), it triggers an automated follow-up sequence. This is where n8n's flexibility shines - we've connected the voice call outcomes to WhatsApp messaging.

The workflow checks the call transcript for specific phrases indicating the WhatsApp group hasn't been joined. When found, it automatically sends an immediate follow-up message with the group invite link. This creates a seamless transition from voice to messaging.

Conversion boost: Businesses using this system report 92% WhatsApp group join rates within 1 hour of payment - compared to 58% with manual processes. The immediate, personalized follow-up makes all the difference.

Automated Feedback Collection

Every customer interaction generates valuable data, and this system captures it automatically. The Bolna agent records whether calls were answered, WhatsApp verification status, and any customer questions or concerns raised during the conversation.

In our n8n workflow, we've added nodes that parse this feedback into structured data stored in Google Sheets. This creates a searchable record of every customer touchpoint, perfect for spotting trends or following up on issues.

Key metrics tracked include:

  • Call answer rates (typically 65-80%)
  • WhatsApp group join confirmation rates
  • Common customer questions or concerns
  • Call duration and cost per interaction
