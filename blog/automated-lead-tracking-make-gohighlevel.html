The 3 Key Components of the System

This automated lead tracking system works through the perfect combination of three essential components:

1. GoHighLevel Webhook Triggers

GoHighLevel's webhook capabilities allow it to send real-time notifications whenever a lead's status changes. This is the trigger that starts the entire automation sequence. The webhook passes all critical lead data including:

  • Lead contact information
  • Previous pipeline stage
  • New pipeline stage
  • Timestamp of the change
  • Custom fields specific to your business

2. Make.com Automation Workflow

Make.com serves as the brain of the operation, processing the webhook data and coordinating all subsequent actions. The workflow:

  • Receives the GoHighLevel webhook notification
  • Formats the data for Google Sheets
  • Determines which email template to send based on stage change
  • Handles error checking and retries if needed

3. Google Sheets Master Tracker

Your central command center where all lead movements are logged automatically. The sheet includes:

  • Complete history of every stage change
  • Timestamps showing exactly when changes occurred
  • Columns for notes and manual overrides
  • Dashboard views showing pipeline health metrics

Make.com Setup: Connecting the Dots

Setting up the Make.com workflow is simpler than you might think. Here's a high-level overview of the process:

Step 1: Create the Webhook Trigger

In Make.com, set up a webhook module that will receive notifications from GoHighLevel. This involves:

  • Creating a unique webhook URL in Make.com
  • Configuring GoHighLevel to send lead changes to that URL
  • Testing with sample data to ensure connection

Step 2: Parse the Incoming Data

Use Make.com's JSON parser to extract the key fields from GoHighLevel's webhook payload:

  • Lead name and contact info
  • Previous and new pipeline stages
  • Custom fields specific to your workflow
  • Timestamp of the change

Step 3: Update Google Sheets

Configure the Google Sheets module to:

  • Find or create a row for the lead
  • Update the current stage column
  • Add a new line to the history log
  • Timestamp the change automatically

Step 4: Trigger Follow-up Emails

Based on the stage change, route to the appropriate email template in Gmail or your ESP:

  • Welcome email for new leads
  • Proposal follow-up sequence
  • Closing reminder emails
  • Re-engagement for stalled leads

Implementation tip: Start with just 2-3 critical pipeline stages, then expand to more complex workflows once you see the system working. This prevents overwhelm during initial setup.

Step-by-Step Implementation Guide

Ready to implement this for your business? Follow these steps to set up your own automated lead tracking system:

Step 1: Map Your Current Pipeline

Document every stage in your sales process from initial contact to close. Identify:

  • Key transition points between stages
  • Which stages most often have leaks
  • What follow-ups should happen at each stage

Step 2: Prepare Your Google Sheet

Create a master tracking sheet with these columns at minimum:

  • Lead name and contact info
  • Current pipeline stage
  • Date of last stage change
  • History of all previous stages
  • Notes column for manual entries

Step 3: Configure GoHighLevel Webhooks

In GoHighLevel's automation settings:

  • Create a new webhook for pipeline stage changes
  • Point it to your Make.com webhook URL
  • Test with sample lead data

Step 4: Build the Make.com Workflow

Create a new scenario in Make.com that:

  • Receives the GoHighLevel webhook
  • Parses the lead and stage change data
  • Updates your Google Sheet
  • Triggers appropriate follow-up emails

Step 5: Test With Live Data

Move test leads between stages and verify:

  • Sheet updates happen instantly
  • Correct emails are being sent
  • All data fields are populating correctly

Pro tip: Create a "sandbox" version first with test leads before rolling out to your entire pipeline. This lets you work out any kinks without affecting real prospects.
