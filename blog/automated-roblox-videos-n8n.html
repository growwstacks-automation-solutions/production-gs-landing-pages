Complete Workflow Overview

The automation connects six specialized AI tools through n8n, creating an end-to-end production pipeline. It starts with a Google Sheet containing raw assets and ends with videos published on YouTube - with zero manual intervention.

Each stage handles a specific creative task: Nano Banana generates photorealistic streamer images, VO3 creates animated scenes, Fal AI merges clips, and JSON to video combines everything with gameplay footage. The workflow maintains quality through structured prompts and validation checks between stages.

Step 1: Data Preparation & Triggers

The foundation is a Google Sheet containing reference images, gameplay videos, and background music links. The workflow starts by fetching the next "to do" row from this sheet, ensuring each execution focuses on one video project.

At 7:20 in the tutorial, you'll see how to configure the Google Sheets node to retrieve only unfinished projects. The system stores all API keys (Kai AI, Fal AI, JSON to video) upfront, preventing authentication issues during automated runs.

Pro Tip: Use both manual and schedule triggers - manual for testing, scheduled for production. This ensures smooth operation once the workflow goes live.

Step 2: AI Character Generation

Nano Banana transforms reference images into Roblox-style streamer characters. The workflow sends the image to Kai AI's endpoint with carefully engineered prompts that maintain consistent style across videos.

The key is dynamic mapping - replacing static values with expressions that reference the Google Sheet data. At 12:45, notice how the HTTP request node pulls the image URL from the sheet rather than using a fixed value.

Step 3: Automated Scene Creation

An AI agent (Claude Sonnet) generates four 8-second scenes with natural dialogue and pacing. The system prompt enforces consistent tone, while the user prompt provides creative direction specific to Roblox content.

VO3 then animates these scenes using the generated character image. At 18:30, you'll see how the workflow handles the rendering delay with wait nodes, then retrieves the finished clips automatically.

Step 4: Smart Video Merging

Fal AI's FFmpeg API merges the individual scenes into one cohesive video. The workflow structures the clip URLs into the required JSON format, sends the merge request, and polls for completion.

This step eliminates the most tedious part of manual editing - aligning scenes and transitions. The automation ensures perfect timing every time, as demonstrated at 22:10 in the video.

Step 5: Final Video Rendering

JSON to video combines the merged streamer footage with gameplay videos and background music. A pre-configured template controls the layout, timing, and transitions - enforcing professional standards automatically.

The system updates the Google Sheet upon completion, marking the project as done and adding the final video URL. This closed-loop design prevents duplicate processing of the same content.

Step 6: Automated YouTube Publishing

Blotato handles the final upload with generated titles, captions, and tags. The workflow passes all metadata from earlier stages, ensuring consistent branding across videos.

At 25:40, notice how the system tests the media upload before creating the post. This validation step prevents failed publishes due to rendering issues or API limits.

Scaling Tip: The same workflow can run multiple times daily by adding more rows to the Google Sheet. Each execution processes the next available project automatically.
