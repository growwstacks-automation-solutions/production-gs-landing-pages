How the OpenMic-to-MyCase Automation Works

The workflow activates the moment an OpenMic call ends. Zapier receives the transcript and applies natural language processing to identify:

  • Client names (first + last)
  • Phone numbers in any format
  • Email addresses mentioned
  • Case type keywords (e.g., "real estate closing")
  • Urgency indicators ("need help immediately")

Step-by-Step Zapier Setup

Step 1: Connect OpenMic as Trigger

In Zapier, select OpenMic as your trigger app. Choose "New Post-Call Summary" as the trigger event. Authenticate using your OpenMic admin credentials.

Step 2: Configure Data Extraction

Map these OpenMic output fields to temporary Zapier variables:

  • call_transcript (raw text)
  • call_duration
  • caller_phone_number

Step 3: Set Up MyCase Action

Add MyCase as your action app. Select "Create Lead" as the action. Connect your MyCase account using API credentials.

Pro Tip: At 5:12 in the video, notice how we add a 10-second delay before creating the MyCase lead. This ensures OpenMic's final transcript processing completes.

Mapping Fields to MyCase Requirements

MyCase expects specific field formats for optimal integration. The workflow transforms extracted data to match:

  • Phone numbers → E.164 format (+15551234567)
  • Names → Proper case (Sammy Harris)
  • Case types → Mapped to your practice areas

The video demonstrates field mapping at 6:30, showing how to handle custom fields for different legal specialties. Estate planning firms can capture will/trust details, while PI attorneys auto-tag accident types.

Testing and Troubleshooting the Workflow
