How the 24/7 Call Logging System Works

The magic happens through three simple components working together:

  1. Vapi Voice Agent - Answers calls with natural conversation when your team is unavailable
  2. Make.com Webhook - Processes the caller data instantly when the conversation ends
  3. Google Sheets - Stores all captured information in an easily sortable format

Here's the call flow in action (timestamp 1:45 in the video): When a client calls after hours, the AI agent greets them professionally, asks for their name and phone number (marked as required fields), and optionally collects their reason for calling. This structured data gets sent to Make.com the moment the call ends.

Step 1: Vapi Tool Configuration

Setting up the voice agent requires careful configuration of the API request in Vapi's tools section (timestamp 3:12):

Critical Configuration Details:

  • HTTP Method: POST (to send data to Make.com)
  • Headers: Content-Type = application/json
  • Request Body: Structured fields for name, phone (E.164 format), call reason, and summary

Pro Tip: Always use E.164 phone number formatting (e.g., +19141234567) to ensure compatibility across all telecom systems and avoid parsing errors.

Step 2: Make.com Webhook Setup

The Make.com automation begins with a custom webhook that receives data from Vapi (timestamp 4:30):

  1. Create a new scenario in Make.com
  2. Add a "Custom Webhook" module as the first step
  3. Copy the webhook URL to your clipboard
  4. Paste this URL into your Vapi tool configuration

This connection point ensures every completed call from Vapi triggers your Make.com automation instantly. The webhook captures all the structured data from the call - name, phone number, call reason, and any notes from the conversation.

Step 3: Google Sheets Integration

The final piece logs everything neatly in Google Sheets (timestamp 6:15):

Google Sheets Module Configuration:

  • Spreadsheet ID: Extract from your sheet's URL (between /d/ and /edit)
  • Sheet Name: Typically "Sheet1" unless renamed
  • Column Range: A:Z to allow for future field additions

Map each field from the webhook to its corresponding column in your sheet. The timestamp automatically records when the call occurred, while the status column helps your team track which leads have been followed up on.

In Summary: 1) Vapi captures call details → 2) Make.com processes the data → 3) Google Sheets stores everything for easy follow-up. Three simple steps to never miss a lead again.
