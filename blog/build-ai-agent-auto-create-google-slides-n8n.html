How the AI Agent Works

The workflow acts as an intelligent agent that takes simple inputs (topic, slide count) and produces a complete presentation. At 2:15 in the video, you can see the agent in action - it asks for three pieces of information then generates seven professional slides automatically.

Behind the scenes, the workflow combines several powerful technologies:

  • n8n form node collects user inputs
  • AI language model generates structured content
  • Google Slides API creates and formats the presentation
  • Error handling logic ensures reliable operation

The magic happens in the transformation from simple topic to complete slides. The AI understands how to structure information for presentations, creating logical flows with titles, subtitles, and bullet points.

Step 1: Setting Up the Input Form

The workflow begins with a simple form that collects three key pieces of information. As shown at 3:50 in the video, we use n8n's Form Trigger node to create this input interface.

Form Fields to Include:

  1. Presentation Title: The name of your slide deck (text field, required)
  2. Topic to Cover: What the presentation should be about (text field, required)
  3. Number of Slides: How many slides to generate (number field, required)

These three inputs give the AI agent everything it needs to create a tailored presentation. The form appears when you execute the workflow, making it user-friendly for non-technical team members.

Step 2: Configuring Google Slides API

To create slides automatically, the workflow needs access to your Google account through the Slides API. At 5:20 in the video, you can see the OAuth setup process.

Key configuration steps:

  1. Create a project in Google Cloud Console
  2. Enable the Google Slides API
  3. Configure OAuth consent screen
  4. Create credentials (Client ID and Secret)
  5. Add these to n8n's Google Slides node

Pro tip: Use a service account if you want the workflow to run without manual authentication. This is ideal for scheduled or triggered presentations.

Once configured, the Google Slides node can create presentations, add slides, insert text, and apply formatting - all through API calls.

Step 3: Building the AI Prompt

The heart of the workflow is the AI prompt that transforms your topic into structured slide content. At 7:10 in the video, we construct this prompt to ensure consistent, high-quality output.

The prompt specifies:

  • Required JSON output format
  • Structure for each slide (title, subtitle, bullet points)
  • Content guidelines (professional tone, clear hierarchy)
  • Topic constraints (stay focused on user's subject)

We connect this to an AI node (like ChatGPT or Claude) with temperature settings tuned for presentation content - not too creative, not too dry. The result is content that's ready for slides with minimal editing.

Step 4: Generating Slide Content

With the prompt ready, the workflow executes the AI call and processes the response. At 9:45 in the video, you can see the AI return perfectly formatted JSON with all slide content.

The workflow handles:

  1. Sending the prompt with user inputs to the AI
  2. Validating the JSON response structure
  3. Extracting individual slide elements
  4. Preparing content for Google Slides API

This is where the magic happens - your simple topic becomes a structured outline with titles, subtitles, and bullet points. The AI understands presentation flow, creating logical progressions from introduction to conclusion.

Step 5: Creating the Presentation

The final step takes the structured content and builds the actual Google Slides presentation. At 12:30 in the video, we configure the Google Slides node to create the deck.

The workflow:

  1. Creates a new presentation with the specified title
  2. Adds the correct number of slides
  3. Inserts titles, subtitles, and bullet points
  4. Applies consistent formatting
  5. Returns the presentation URL when complete

Customization: You can extend this to include your company branding, specific slide layouts, or even custom graphics by modifying the API requests.

Error Handling and Retries

At 14:50 in the video, we implement error handling to make the workflow production-ready. AI responses and API calls can sometimes fail, so we need smart recovery.

The workflow includes:

  • JSON validation to catch malformed AI responses
  • API error detection and automatic retries
  • Fallback content generation if primary method fails
  • Notification system for critical failures

This ensures the workflow completes successfully even if individual components encounter temporary issues. The retry logic is particularly important for API rate limits and occasional AI hiccups.
