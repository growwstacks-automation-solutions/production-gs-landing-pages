Understanding the MCP, ADK and A2A Components

Building AI agents often feels overwhelming because the ecosystem involves multiple specialized components that need to work together. The breakthrough comes when you understand how Google Cloud's three key technologies - MCP, ADK and A2A - each handle a distinct part of the agent lifecycle.

MCP (Model Context Protocol) acts as your agent's toolbelt. It's a standardized way to connect your AI model to external APIs, databases and services. ADK (Agent Development Kit) is the framework that makes building the actual agent feel like normal software development. A2A (Agent-to-Agent protocol) enables your agent to collaborate with others in a secure, standardized way.

Key insight: MCP provides the tools, ADK builds the agent, and A2A enables collaboration. This separation of concerns is what makes Google Cloud's approach both powerful and practical for real-world deployments.

Setting Up Your Google Cloud Environment

Before writing any code, we need to configure our Google Cloud project properly. The tutorial uses Cloud Shell which provides all necessary tools pre-installed, eliminating local environment setup headaches.

First, create a new Google Cloud project with billing enabled. Then activate Cloud Shell from the console. We'll run these critical setup commands:

 # Set project ID export PROJECT_ID=your-project-id gcloud config set project $PROJECT_ID # Enable required APIs gcloud services enable run.googleapis.com \   cloudbuild.googleapis.com \   artifactregistry.googleapis.com \   aiplatform.googleapis.com

These commands configure our project ID and enable Cloud Run, Cloud Build, Artifact Registry and Vertex AI APIs. The API enablement can take a few minutes. Verify your Python version is 3.10+ with python3 --version.

Building and Testing the MCP Server

The MCP server is where we'll expose tools our agent can use. For this demo, we're creating a currency exchange rate tool using FastMCP - a Python framework that simplifies MCP server development.

Clone the tutorial repository and examine the server.py file. The key component is the get_exchange_rate function decorated with @mcp_tool. This registers it as an available tool:

 @mcp_tool def get_exchange_rate(from_currency: str, to_currency: str):     """Fetches current exchange rate between two currencies"""     # Implementation calls external API     return rate

Start the server locally on port 8080 and test it in a new Cloud Shell tab. You should see the current USD to EUR exchange rate, confirming the tool works before we deploy it.

Deploying to Cloud Run

While our MCP server works locally, we need a production-grade deployment. Cloud Run provides automatic scaling, security and reliability with minimal configuration.

The deployment command includes a critical security flag:

 gcloud run deploy currency-agent-mcp \   --source . \   --region us-central1 \   --no-allow-unauthenticated

Why no-allow-unauthenticated? This ensures only authorized services can access our currency tool. We then use Cloud Run Proxy to create a secure tunnel for testing:

 cloud-run-proxy --project $PROJECT_ID \   --target mcp-server

Run the same test script through this proxy to verify the deployed service works identically to our local version. Check Cloud Run logs to confirm the request was processed.

Creating the Agent with ADK

With our MCP server deployed, we can now build the actual AI agent using ADK. The currency-agent.py file contains two critical sections:

  1. System instructions defining the agent's purpose and rules (e.g. "Only answer currency questions")
  2. MCP toolset connecting to our deployed exchange rate service

ADK includes a web interface for testing. Start it on port 8000 and access via Cloud Shell's web preview. When you ask "What is 250 CAD in USD?", the agent:

  1. Understands the currency conversion request
  2. Calls our MCP server's get_exchange_rate tool
  3. Returns the calculated amount

This demonstrates the power of separating tool implementation (MCP) from agent logic (ADK).

Implementing A2A Protocol

Our agent can answer human questions, but the real value comes when it collaborates with other agents. A2A protocol standardizes this communication.

ADK's adk_to_a2a utility converts our agent to A2A-compatible format with two lines of code:

 app = adk_to_a2a(agent) app.run(host='0.0.0.0', port=10000)

This creates an A2A server on port 10000 that:

  • Automatically generates an "agent card" describing capabilities
  • Exposes standardized endpoints for discovery and collaboration
  • Maintains security through Google Cloud's IAM system

The tutorial includes a test client that demonstrates A2A communication by querying our agent programmatically.

Testing the Complete Agent System

With all components running - MCP server on Cloud Run, ADK web interface, and A2A server - we can test the full system:

  1. Human queries via ADK web interface
  2. Programmatic queries via A2A client
  3. Direct tool calls to MCP server

Each access method should return consistent results. At 12:43 in the video, you can see the test client successfully getting exchange rates through the A2A interface.

MCP (Model Context Protocol) is a reverse proxy or API server for API servers in Google Cloud's AI ecosystem. It provides a standardized way to connect AI models to the specific resources, prompts and tools they need to complete tasks.

MCP acts as an interface layer between AI agents and the services they require, handling authentication, rate limiting and protocol translation behind the scenes.

  • Standardizes tool access for AI agents
  • Handles security and protocol translation
  • Enables tool reuse across multiple agents

ADK (Agent Development Kit) is Google Cloud's framework for building and deploying AI agents. It abstracts away much of the complexity of creating production-grade agents.

ADK provides pre-built components for common agent patterns, testing tools, and deployment pipelines that work seamlessly with Google Cloud services.

  • Pre-built components for common patterns
  • Integrated testing tools
  • One-command deployment to Cloud Run

A2A (Agent-to-Agent) protocol enables different AI agents to communicate regardless of their framework, hosting environment or programming language.

It provides enterprise-grade security while allowing agents to discover each other's capabilities through standardized "agent cards" similar to digital business cards.

  • Framework-agnostic collaboration
  • Secure communication channels
  • Capability discovery via agent cards
