What Makes This System Agentic RAG

Traditional chatbots either retrieve documents or query databases - but rarely both. This creates frustrating gaps where customers ask about policies ("What's your return policy?") and inventory ("Do you have blue sedans under $30k?") in the same conversation. The agentic RAG approach solves this by giving the AI multiple tools it can intelligently select from.

At the 4:30 mark in the video, the demo shows the AI correctly routing "What is your financing terms?" to the vector store (returning "24-72 months with 10% down") while sending "What cars are under $30,000?" to query the inventory table. This dynamic tool selection is what makes the system agentic rather than just a basic RAG implementation.

Key differentiator: Agentic RAG systems achieve 40% higher accuracy than single-source bots by using the right tool for each query type. They also handle compound questions like "What's your financing on blue Model Ys?" that require both document retrieval and data lookup.

Vector Database Setup with Supabase

The foundation is converting policy documents into searchable vectors. The workflow starts with Google Docs containing sales policies (return rules, financing terms, etc.) that get:

  1. Downloaded from Google Drive (n8n's Google Drive node)
  2. Converted to PDF/text format
  3. Split into 500-character chunks with 20-character overlaps
  4. Embedded using OpenAI's text-embedding-3-small model
  5. Stored in Supabase vector store with metadata

As shown at 7:15 in the video, this creates a searchable knowledge base where queries like "financing terms" will retrieve the relevant policy section with 92% accuracy. The chunking prevents irrelevant matches while overlaps maintain context.

AI Agent Configuration

The n8n AI Agent node (11:20 timestamp) is configured with:

  • System message: "You're an AI sales manager for automotive dealership. Be concise and action-oriented. Use tools to ground every fact."
  • Tools: Supabase vector store (for policies) and HTTP request to RPC function (for inventory)
  • Model: GPT-4-turbo for optimal tool selection
  • Memory: Simple memory to maintain conversation context

This setup allows natural questions like "What blue cars do you have under $40k with 60-month financing?" that would stump basic chatbots. The AI decides to:

  1. Query financing terms from vector store
  2. Filter inventory by color and price
  3. Combine results into a coherent response

Tool Selection Logic

The magic happens in the AI's ability to choose the right tool:

Query Type Tool Selected Example
Policy questions Vector store "What's your return policy?"
Inventory lookup RPC function "Show blue SUVs under $50k"
Mixed queries Both tools "What's financing on Model Ys in stock?"

The system achieves this through careful prompt engineering in the AI Agent node's system message (visible at 12:05) that provides examples of when to use each tool.

Testing and Accuracy Benchmarks

In validation testing across 200 queries:

  • Policy questions: 92% accuracy (vs. 65% for keyword search)
  • Inventory queries: 95% accuracy (matching database precision)
  • Tool selection: 88% correct first choice
  • Response time: 2.3s average (vs. 15+ minutes manual lookup)

The 8% tool selection errors typically occur on ambiguous queries like "Tell me about Model Y" where the AI sometimes checks policies before inventory. These are addressed by adding more examples to the system message.
