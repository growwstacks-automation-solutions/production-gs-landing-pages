Workflow Overview

The complete system requires just four connected components in Make.com:

  1. Tally Form: Collects student name, email, and question
  2. Google Docs: Stores your FAQ content as the knowledge base
  3. Gemini API: Analyzes questions and finds relevant answers
  4. Gmail: Sends personalized response emails automatically

When a student submits a question (shown at 5:18 in the video), the workflow:

  1. Triggers on new Tally form submission
  2. Pulls the latest FAQ content from Google Docs
  3. Sends question + FAQs to Gemini AI for analysis
  4. Formats the AI response into a professional email
  5. Sends personalized answer back to the student

Setting Up Tally Forms

The first step is creating a simple form to collect student inquiries. At 12:45 in the video, the instructor demonstrates the exact fields needed:

  • Short answer: "What is your name?"
  • Email field: Validates proper email format
  • Long answer: "What is your question?"

In Make.com, you'll:

  1. Create a new scenario
  2. Add the "Tally - Watch new response" module
  3. Authenticate with your Tally account
  4. Select your form from the dropdown

Pro tip: Test your form connection by submitting a test entry and using the "Run once" button in Make.com to verify the data flows correctly (shown at 18:30).

Connecting Google Docs

Your FAQ document serves as the knowledge base for the AI. At 22:10, the video shows how to:

  1. Create a Google Doc with all your FAQs
  2. Organize content with clear headings
  3. Share the doc with your Make.com service account

In Make.com, add the "Google Docs - Get content of a document" module and:

  1. Authenticate with Google
  2. Select "My Drive" as the location
  3. Choose your FAQ document
  4. Enable "Include all tabs content"

The module will output the full text content that Gemini will use to answer questions.

Configuring Gemini AI

The magic happens in the HTTP module that calls Gemini's API (demonstrated at 32:45). You'll need:

  • A Gemini API key from Google AI Studio
  • The API endpoint URL with your key inserted
  • Proper JSON structure for the request

Key configuration steps:

  1. Add an HTTP module set to POST method
  2. Paste your Gemini API URL with key
  3. Set Content-Type to application/json
  4. Add the system instruction prompt (provided at 36:20)
  5. Map the Google Docs content and student question

Critical note: The system instruction tells Gemini to only answer based on the provided FAQs. For unknown questions, it responds: "Great question. Our team will get back to you soon."

Automating Email Responses

The final step sends polished email responses (shown at 48:10). Using the Gmail module:

  1. Authenticate your Gmail account
  2. Map the student's email address from Tally
  3. Create a professional subject line
  4. Build an HTML email template with:
    • Student's name
    • The AI-generated answer
    • Your branding and signature

The video at 51:45 shows the complete template with placeholders that automatically fill with the student's name and personalized answer.

Testing the Workflow

Before going live, thoroughly test each component:

  1. Form submission: Verify Tally data reaches Make.com
  2. Document access: Check FAQ content is fully retrieved
  3. AI responses: Test with various question types
  4. Email delivery: Confirm formatting and spam folder issues

At 55:30, the instructor demonstrates testing with different questions to ensure:

  • Known questions get accurate FAQ answers
  • Unknown questions trigger the fallback response
  • Emails maintain proper formatting

Deployment tip: Once testing is complete, activate the scenario to run automatically on new form submissions (shown at 58:15).
