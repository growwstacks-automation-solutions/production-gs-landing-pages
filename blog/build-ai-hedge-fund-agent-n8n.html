How the AI Hedge Fund Agent Works

Investment analysis typically requires hours of research across multiple data sources - market trends, financial statements, news sentiment, and historical performance. This n8n workflow automates the entire process, delivering a comprehensive hedge fund-style analysis in minutes.

The system works through a Telegram interface where you simply send a stock ticker (like TSLA for Tesla). Behind the scenes, it aggregates data from multiple free APIs, processes it through AI agents modeled after top investors, and returns a detailed report with position sizing, risk ratings, and investment theses.

Complete analysis in 3-5 minutes: While the workflow appears complex, each component serves a specific purpose - data collection, AI processing, risk assessment, and output formatting. The wait nodes between AI agents prevent API rate limits while ensuring thorough analysis.

Free Data Sources Used

The workflow leverages four key data sources, all available through free API tiers suitable for personal use:

  1. Polygon.io - Provides real-time market data, company fundamentals, and financial statements
  2. NewsAPI.org - Delivers recent news articles for sentiment analysis
  3. OpenAI API - Powers the AI analysis using GPT-4-turbo
  4. Telegram Bot API - Handles user interaction and report delivery

Building the n8n Workflow

The workflow follows a logical sequence that mirrors professional investment analysis:

Step 1: Data Collection

Four HTTP request nodes pull data simultaneously from Polygon (market data, fundamentals, financials) and NewsAPI. These run in parallel to minimize execution time.

Step 2: Data Processing

Merge and aggregate nodes combine the datasets, followed by a code node that formats the information for AI consumption. This standardization is crucial for accurate analysis.

Step 3: AI Analysis

Five AI agent nodes process the data sequentially, with wait nodes spacing the requests to avoid API limits. Each agent returns its independent assessment.

Step 4: Risk Synthesis

A final AI agent synthesizes all analyses into a coherent recommendation with position sizing, stop-loss levels, and risk ratings.

Memory nodes maintain context: Each AI agent connects to a simple memory node using the stock ticker as a unique identifier. This ensures continuity across the analysis while preventing data mix-ups.

Telegram Bot Integration

The Telegram interface provides an accessible way to interact with the hedge fund agent. Setting up the bot requires:

  1. Creating a new bot through BotFather (shown at 2:18 in the video)
  2. Configuring the n8n Telegram trigger node with your API key
  3. Setting up the send message nodes to format the output

Because Telegram has a 4000-character message limit, the workflow splits the analysis into four parts: summary, part 1, part 2, and part 3. A code node handles this segmentation automatically.

The video at 28:45 shows how to test and troubleshoot the Telegram integration, including retrieving your chat ID and handling common authentication errors.

Automated Risk Assessment

The final AI agent serves as risk manager, synthesizing all analyses into actionable recommendations. Its report includes:

  • Position sizing (typically 3-6% of portfolio)
  • Risk rating (1-10 scale)
  • Expected annual return
  • Maximum expected loss
  • Stop-loss and profit target levels

At 18:30 in the tutorial, you'll see how this agent weights the different analyses based on market conditions. For example, in volatile markets, it might give more weight to Ray Dalio's risk parity approach over Warren Buffett's long-term perspective.

Real-world example: When analyzing Tesla (TSLA), the system recommended a 6% portfolio allocation with a 7/10 risk rating, suggesting dollar-cost averaging into the position rather than a lump-sum investment.

Performance Optimization Tips

To ensure reliable operation and minimize costs:

  1. Limit executions to 1-2 per day (the workflow is resource-intensive)
  2. Use wait nodes between AI agents (15-45 seconds) to avoid API limits
  3. Monitor OpenAI usage (each analysis costs $0.10-$0.20)
  4. Cache frequent requests by adding a database node for commonly analyzed stocks
