The Structured Output Revolution

Most AI implementations fail because they accept free-form text responses. The breakthrough in this workflow is the structured output parser that forces the AI to return data in a specific JSON format. No more guessing where the subject line ends and the body begins.

The schema defines seven critical components for every communication: 1) Catchy subject line, 2) Summary paragraph, 3) Key points list, 4) Top takeaways, 5) Tools mentioned, 6) Gmail HTML body, and 7) Google Docs markdown content. This structure ensures consistency across all outputs.

Google API Setup (Step-by-Step)

Connecting n8n to Google services requires proper OAuth configuration. Many workflows fail at this step due to incorrect API enabling or redirect URI mismatches. Follow these exact steps to avoid common pitfalls.

Step 1: Enable Required APIs

Navigate to Google Cloud Console and enable three critical APIs: Gmail API, Google Docs API, and Google Drive API (required for Docs functionality). Each takes about 30 seconds to enable.

Step 2: Configure OAuth Consent

Create an OAuth consent screen with your business name and support email. Select "External" user type and add your email as a test user. This step authorizes your app to access Google services.

Step 3: Create Credentials

Generate OAuth client ID credentials for a web application. The crucial detail is using your n8n instance's exact OAuth callback URL. Save the client ID and secret securely - you'll need them in n8n.

Pro Tip: Always publish your app in Google Cloud Console immediately after creation. Unpublished apps will fail authentication with confusing errors.

AI Agent Configuration
