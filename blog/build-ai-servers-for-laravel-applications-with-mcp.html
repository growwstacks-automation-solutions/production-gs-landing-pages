What is Model Context Protocol (MCP)?

If you've used AI assistants like Claude or ChatGPT to interact with applications, you've likely used MCP without realizing it. Model Context Protocol is the standardized language that enables AI systems to understand and work with external applications.

Originally developed by Anthropic for Claude, MCP has become the de facto standard adopted by major AI providers. It solves a critical problem: instead of each application needing to teach AI assistants how to use its unique API, MCP provides a common protocol that all AI systems understand.

Key insight: MCP isn't just for large tech companies. Any Laravel application can implement MCP to make its functionality accessible through AI interfaces, opening up new ways for users to interact with your software.

The Laravel MCP Package

The Laravel MCP package handles all the heavy lifting of implementing the protocol in your application. It provides:

  • Artisan commands for generating MCP components
  • Built-in authentication using Laravel Sanctum
  • Standardized response formatting
  • Tool, prompt and resource scaffolding

This means you can focus on defining what your application can do through MCP, rather than worrying about protocol implementation details. The package ensures your MCP server speaks the language AI assistants expect.

Key MCP Components: Tools, Prompts and Resources

MCP defines three main components that your Laravel application can expose to AI assistants:

1. Tools

Tools represent actions AI assistants can perform with your application. Each tool has:

  • A handle method containing the action logic
  • An input schema defining required parameters
  • A description explaining when to use it

2. Prompts

Prompts guide AI assistants on how to interact with your tools. They serve as system messages that shape:

  • How the assistant should interpret user requests
  • Which tools to use for different scenarios
  • The tone and style of responses

3. Resources

Resources provide read-only access to your application's data. Unlike tools, they don't perform actions but instead:

  • Expose specific data entities
  • Include descriptions of what they contain
  • Can be combined with tools for complete workflows

Creating MCP Elements in Laravel

The Laravel MCP package provides Artisan commands to generate each component type:

Creating a Tool

 php artisan make:mcp-tool GetRecentLinks

This generates a tool class where you implement the handle method and define your input schema.

Creating a Prompt

 php artisan make:mcp-prompt SummarizeLinks

The generated prompt class lets you define the system message that guides AI behavior.

Creating a Resource

 php artisan make:mcp-resource RecentLink

Resources include a handle method to fetch and return your application data.

Each component includes detailed descriptions that help AI assistants understand when and how to use them.

MCP Server Types: Web vs Local

Laravel MCP supports two deployment models:

Web MCP Servers

Public-facing endpoints that:

  • Are accessible to any authorized AI assistant
  • Can be discovered through your application's UI
  • Use standard web authentication flows

Local MCP Servers

Development-focused instances that:

  • Run within your local environment
  • Are useful for testing and internal tools
  • Power packages like Laravel Boost

The same MCP components work in both environments, giving you flexibility in how you expose your application's functionality to AI systems.
