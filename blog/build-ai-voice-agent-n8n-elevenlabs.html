Connecting n8n for Business Logic

While ElevenLabs handles the voice conversation, n8n provides the operational backbone. Our tutorial workflow includes:

Key workflow steps:

  1. Receive order ID and email from ElevenLabs agent
  2. Verify customer against mock database
  3. Retrieve order details including eligibility status
  4. Process refunds under $100 automatically
  5. Update order status in the system

The n8n workflow uses two data tables - one for customer information and another for order details. This simulates connecting to a real e-commerce platform's database while demonstrating the pattern for production implementations.

Building the Refund Processing Agent

Creating an effective voice agent requires careful prompt engineering. Our refund specialist agent includes:

Prompt Structure

  • Personality: Empathetic but policy-compliant
  • Goals: Verify identity → Check eligibility → Process refund
  • Guardrails: Never process invalid refunds or share order info prematurely
  • Tools: Clear instructions for when to use verification and refund tools

The prompt explicitly defines refund rules: under $100 can be automated, $100-$500 requires secondary verification, and over $500 escalates to a supervisor. This structure ensures compliance while handling most routine cases automatically.

Creating Webhook Tools for Order Verification

Webhook tools bridge ElevenLabs to your business systems. We create two critical tools:

1. Verify Identity Tool

  • Collects order ID and email via LLM prompt
  • Sends to n8n to check against customer database
  • Returns order details if verification passes

2. Process Refund Tool

  • Triggers only after successful verification
  • Requires refund reason code (defective, wrong item, etc.)
  • Updates order status in the system

Each tool includes typing sounds during processing to maintain natural conversation flow. The tools are configured with appropriate timeout settings and error handling for production reliability.

Testing and Deploying to a Phone Number

Before going live, we thoroughly test the agent through ElevenLabs' preview interface. The testing process confirms:

  • Accurate transcription of order numbers and emails
  • Correct tool triggering based on conversation flow
  • Proper handling of edge cases (invalid orders, expired refund windows)
  • Natural pacing and conversational repair strategies

Deployment involves connecting a Twilio phone number through ElevenLabs' telephony integration. The tutorial shows how to configure:

  • Account SID and auth token from Twilio
  • Phone number in E.164 format
  • Optimal regional routing for call quality
