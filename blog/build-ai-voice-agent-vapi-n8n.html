Step 1: Vapi Phone Number & Assistant Setup

Vapi provides the telephony infrastructure and AI conversation engine. Start by creating your virtual phone number (timestamp 2:15 in the video):

  1. Log in to Vapi and navigate to Phone Numbers
  2. Click "Create Phone Number" and select your area code
  3. Name your agent (e.g. "Tanya - Appointment Scheduler")
  4. Note the assigned inbound number - this will be your business line

Next, create your AI assistant (timestamp 3:48):

Pro Tip: Use GPT-4-turbo for the best balance of performance and cost. The smaller models may struggle with multi-turn conversations.

Step 2: Crafting the Perfect System Prompt

The system prompt defines your agent's personality and capabilities (timestamp 5:12). An effective prompt includes:

  • Role definition: "You are Tanya, the office coordinator for [Business Name]"
  • Current date context: Provided via {now} expression
  • Conversation flow: One question at a time, specific information gathering
  • Calendar check protocol: Exact phrasing for availability checks
  • Structured data requirements: What to extract from each call

The video includes a complete template you can adapt (timestamp 6:30). Key elements to customize:

 Business Name: [Your Business Here] Agent Name: [Your Agent Name] Available Days: [Your Availability] Meeting Duration: [30/60 minutes]

Step 3: Google Calendar Integration

For real-time appointment scheduling, connect your Google Calendar (timestamp 12:45):

  1. In Vapi Tools, create two Google Calendar tools:
    • Check Availability (for finding open slots)
    • Create Event (for booking appointments)
  2. Use your primary calendar email as the Calendar ID
  3. Set your timezone (e.g. America/Los_Angeles)

Critical: Update the tool names in your system prompt to match exactly what you create in Vapi (timestamp 14:20). Mismatches will prevent the calendar functions from working.

Step 4: Structured Data Extraction

Vapi can output call data in a structured format for your CRM (timestamp 16:05):

  1. Navigate to Assistant → Structured Data
  2. Define the fields to capture (max 10):
    • Caller name, phone, email
    • Appointment date/time
    • Special instructions
    • Full call transcript
  3. Include instructions to send data via webhook to n8n

The structured data prompt template (timestamp 17:30) includes JSON examples showing exactly how data will be formatted for your n8n workflow.

Step 5: Building the n8n Workflow

n8n handles the data processing and CRM integration (timestamp 20:15):

  1. Create a new workflow with Webhook trigger
  2. Add "Respond to Webhook" node (keeps connection open)
  3. Configure filter to only process structured data payloads
  4. Add HTTP node to fetch call recording details

The video shows how to handle a common issue where multiple webhook calls arrive (timestamp 23:40). The solution involves:

Key Fix: Using Vapi's production URL instead of test URL, and disabling unnecessary server messages to prevent duplicate payloads.

Step 6: Testing Calls & Troubleshooting

Before going live, thoroughly test your agent (timestamp 26:00):

  • Call from different numbers to simulate real clients
  • Test edge cases:
    • Requests outside business hours
    • Complex scheduling needs
    • Requests to speak with a human
  • Verify calendar events created correctly
  • Check n8n for complete data capture

Common issues and fixes:

Issue Solution
Agent doesn't recognize calendar slots Verify tool names match exactly in system prompt
Duplicate n8n payloads Disable all server messages except end-of-call
Missing structured data Check property definitions in Vapi

Step 7: Airtable CRM Integration

The final step connects everything to your CRM (timestamp 29:30):

  1. Create Airtable base with fields matching structured data
  2. Generate API key with read/write permissions
  3. Add Airtable node to n8n workflow
  4. Map Vapi fields to Airtable columns

This creates a complete record for every call including:

  • Client contact information
  • Appointment details
  • Call recording link
  • Full transcript

Implementation Tip: The same n8n workflow can push data to HubSpot, Salesforce, or custom databases by adding additional nodes.
