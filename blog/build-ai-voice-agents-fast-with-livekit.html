What Is LiveKit Agent Builder?

Voice AI has traditionally required significant technical expertise to implement - until now. LiveKit's Agent Builder is a game-changing web interface that lets anyone prototype and deploy voice agents directly from their browser. No servers, no complex SDK setup, just pure voice AI functionality at your fingertips.

Think of it as the Figma of voice AI - a visual builder that abstracts away the complexity while maintaining the power of LiveKit's underlying framework. You get to focus on designing your agent's personality and capabilities rather than wrestling with infrastructure.

90% faster prototyping: What used to take days of coding can now be accomplished in minutes through the browser interface, while still maintaining the option to convert to code later for advanced customization.

Key Features of LiveKit's Voice AI Platform

LiveKit stands apart from other voice AI platforms through its model-agnostic approach and enterprise-grade architecture. While the Agent Builder simplifies creation, you're still leveraging the full power of LiveKit's infrastructure.

The platform combines three core components: text-to-speech (voice synthesis), large language models (conversation intelligence), and speech-to-text (transcription). Unlike monolithic solutions, LiveKit lets you mix and match providers for each component based on your needs.

  • Model Flexibility: Choose between 11 Labs, Cartesia, Rhyme, and others for voice synthesis
  • LLM Options: Switch between Gemini, Claude, and OpenAI models for conversation
  • Transcription: Deepgram Nova 3 provides industry-leading speech-to-text accuracy
  • Tool Integration: HTTP actions connect your agent to any API or external service

Getting Started with the Agent Builder

Accessing the Agent Builder is simple - just navigate to cloud.livekit.io and select the new "Build Agent in Browser" option. The interface is divided into four main panes that guide you through the creation process:

  1. Instructions: Define your agent's personality, behavior, and conversation flow
  2. Models and Voice: Configure the text-to-speech, LLM, and speech-to-text components
  3. Actions: Connect external tools and APIs through HTTP requests
  4. Advanced: Set metadata, secrets, and telephony configurations

The magic of LiveKit comes from being able to customize each component of your voice agent's stack. In the Models and Voice pane, you can select different providers for each function:

Pro Tip: For most business applications, we recommend 11 Labs for voice synthesis (most realistic), Gemini Flash for the LLM (fast response times), and Deepgram Nova 3 for transcription (highest accuracy).

Building Your Agent's Prompt

A well-structured prompt is the difference between a robotic interface and a natural conversational experience. LiveKit's prompting guide recommends including these key sections:

  • Identity: Who is your agent? (e.g., "You are Jonas's concierge assistant")
  • Output Rules: How should responses be formatted?
  • Goals: What is the agent's primary purpose?
  • Conversation Flow: Example dialogues to establish tone
  • Tools: What external capabilities does it have?
  • Guardrails: What should it never do or say?

At 12:45 in the tutorial, we build a complete prompt for a calendar concierge agent that can check schedules and appointments. Notice how we define the tool name (get_cal) before creating the actual HTTP action in the next step.

Connecting External Tools and APIs

The real power comes when your voice agent can interact with other systems. LiveKit's HTTP actions let you connect to any API or service. In our example at 15:20, we:

  1. Create a new HTTP tool named "get_cal"
  2. Set the method to POST and add our n8n webhook URL
  3. Define parameters to pass the user's request
  4. Save the tool and reference it in our prompt

This creates a seamless connection where the voice agent can now retrieve real calendar data through our n8n workflow. The same pattern works for CRMs, databases, or any API-accessible service.

Deploying to a Phone Number

LiveKit's newest feature lets you purchase US local numbers directly in the platform. At 18:50 in the video, we:

  1. Buy a phone number with two clicks
  2. Create a dispatch rule to route calls to our agent
  3. Associate the number with our agent's ID
  4. Test the complete system end-to-end

Important Note: While convenient, LiveKit's native phone numbers currently have some reliability limitations. For production systems, we recommend SIP trunking with external providers as covered in our previous tutorials.

Converting Your Agent to Code

The Agent Builder isn't meant to replace LiveKit's SDK - it's an on-ramp. Once you've prototyped your agent, you can:

  • Click "Download Code" to get a Python script of your agent
  • Import this into a full SDK project
  • Add advanced features like workflows, handoffs, and custom integrations
  • Deploy to your own infrastructure for complete control
