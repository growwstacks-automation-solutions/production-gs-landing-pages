Building Your First Agent in Gemini 3 Pro

Creating your voice agent starts in Google's AI Studio. The platform's Build feature lets you describe your agent in natural language and generates all necessary code automatically. For the appointment booking demo, the prompt specified:

  • Premium black interface with gold wave animation
  • Central audio visualizer and live transcription
  • Specific greeting phrase ("Hello, how can I help you today?")
  • Fast American voice option for best performance

After generating the interface, we added business logic through a second prompt that defined the appointment booking requirements: collecting name, email, and desired time, then triggering a webhook to n8n. Gemini automatically implemented conversation flow controls like "Let me check that for you" to improve user experience.

Connecting to n8n for Backend Automation

The real power comes from connecting your voice agent to n8n workflows. At 5:48 in the video, you'll see the webhook setup that passes customer data from Gemini to n8n. The key steps:

  1. Create a webhook trigger in n8n set to POST requests
  2. Copy the webhook URL into your Gemini agent code
  3. Map the required fields (name, email, appointment time)

Pro tip: Use test URLs during development to show the data flow in real-time to clients. Switch to production URLs when deploying live agents.

The n8n workflow receives the customer's requested time in text format, converts it to ISO datetime (critical for calendar integrations), then checks availability before confirming or suggesting alternatives.

Real-Time Calendar Availability Checks

At 9:12 in the tutorial, the n8n workflow uses Google Calendar's "Get Availability" node to verify appointment slots. The configuration includes:

  • Calendar selection (supporting multiple team members/calendars)
  • ISO datetime formatting of the requested time
  • Conditional logic branching based on availability

If the slot is available (boolean true), the workflow creates a calendar event with the customer's name and email as attendee. If unavailable (false), it triggers the alternative times suggestion logic.

The entire availability check and event creation happens in under 2 seconds, allowing the voice agent to maintain natural conversation flow without awkward pauses.

Handling Alternative Appointment Times

When the requested time is unavailable, the workflow (shown at 11:30) automatically checks the next day for openings. The process:

  1. Adds one day to the original request datetime
  2. Scans the calendar for all existing events that day
  3. Analyzes gaps between appointments (business hours 9am-5pm)
  4. Generates polite, concise alternative suggestions using an LLM node

The demo shows how the system identifies three available slots on December 6th when December 5th was fully booked. The response webhook node sends these options back to the voice agent, which presents them conversationally to the customer.
