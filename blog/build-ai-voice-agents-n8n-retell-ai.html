Configuring Your Agent in Retell AI

Retell AI offers four agent types, each suited for different use cases:

1. Conversation Flow Agent

Best for rigid, scripted interactions like FAQs or step-by-step procedures. The agent strictly follows predefined paths without deviation.

2. Single Prompt Agent

Handles free-form conversations using one central prompt. Simpler to set up but less control over responses.

3. Multi-Prompt Agent (Recommended)

Our tutorial uses this flexible approach. It maintains context across follow-up questions while still referencing your documents through n8n.

4. Custom LLM Agent

For enterprises with their own AI models. Connects to your existing LLM infrastructure.

n8n Integration for Document Retrieval

The magic happens when Retell AI passes the caller's question to n8n, which then:

  1. Receives the transcript through a webhook
  2. Extracts the core question using JavaScript
  3. Queries your Pinecone vector database for relevant document chunks
  4. Processes the results through an AI prompt for clarity
  5. Returns the polished response to Retell AI

Pro Tip: The JavaScript code at 8:42 in the video shows how to cleanly extract just the user's question from Retell AI's full transcript data.

This creates a seamless experience where callers hear natural, accurate responses pulled directly from your latest documentation - with no manual scripting required.

Connecting Your Twilio Phone Number

While Retell AI offers numbers, many businesses prefer using their existing Twilio numbers. The connection requires:

  1. Creating SIP credentials in Twilio (shown at 5:18)
  2. Whitelisting Retell AI's IP addresses
  3. Entering the SIP URI in Retell AI's phone number settings

For outbound calls, you'll need to complete Retell AI's verification process. This involves submitting identification documents to comply with telecom regulations.

Complete Workflow Breakdown

The end-to-end n8n workflow (shown at 7:30) consists of:

Step 1: Webhook Trigger

Receives the call transcript from Retell AI with both user questions and agent responses.

Step 2: Question Extraction

JavaScript node isolates just the user's spoken question for processing.

Step 3: Vector Database Query

Pinecone searches your document chunks for relevant information.

Step 4: Response Generation

AI model formats the raw document chunks into a natural spoken response.

Step 5: Return to Retell AI

Structured response goes back through the webhook for voice delivery.

Key Insight: At 9:15, notice how the prompt instructs the AI to "respond with the response field exactly as it appears" - this prevents the model from improvising answers beyond your source documents.
