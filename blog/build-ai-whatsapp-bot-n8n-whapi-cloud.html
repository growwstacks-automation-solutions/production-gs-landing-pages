Step 1: Setting Up Your WhatsApp Webhook

The foundation of any WhatsApp bot is capturing incoming messages. We'll use Whapi.Cloud's webhook feature to instantly forward all WhatsApp messages to our n8n workflow.

In your Whapi.Cloud dashboard, navigate to Webhooks and create a new endpoint pointing to your n8n webhook URL. Make sure to select "Messages" as the event type. This ensures your bot receives every incoming message in real-time.

At 1:15 in the video tutorial, you'll see the exact webhook configuration settings. The key elements are the HTTPS URL, verification token, and selecting the correct events to monitor.

Step 2: Adding Human-Like Emoji Reactions

What separates a robotic interaction from a human one? Small touches like emoji reactions. Our bot will instantly react to messages before processing them - creating a more engaging experience.

In n8n, add an HTTP Request node configured to call Whapi.Cloud's "React to Message" endpoint. You'll need:

  • Method: PUT
  • URL: https://gate.whapi.cloud/messages/reaction
  • Headers: Authorization with your Bearer token
  • Body: message_id (from webhook) and emoji (like "😊")

Pro Tip: Test different emojis based on message sentiment. A thumbs-up for positive messages or thinking face for complex questions makes interactions feel more natural.

Step 3: Processing Messages with OpenAI

Now for the AI magic. We'll pass the user's message to OpenAI with instructions on how to respond. The key is crafting a clear system prompt that defines your bot's personality and knowledge boundaries.

Add an OpenAI node in n8n with:

  • Your OpenAI API key
  • System prompt defining the bot's role (e.g., "You're a friendly customer support assistant for [Business Name]")
  • User message from the webhook as the prompt
  • Model selection (gpt-3.5-turbo works well for most use cases)

At 3:40 in the video, you'll see how to structure the prompt for consistent, on-brand responses that align with your business voice.

Step 4: Sending AI-Generated Replies

The final step is delivering OpenAI's response back to WhatsApp. We'll use another HTTP Request node to call Whapi.Cloud's "Send Message" endpoint.

Configure the node with:

  • Method: POST
  • URL: https://gate.whapi.cloud/messages/text
  • Headers: Authorization with your Bearer token
  • Body: to (user's WhatsApp number) and body (AI-generated text)

This creates a complete loop: message comes in → bot reacts → AI processes → reply sent. The entire process happens in seconds, faster than any human could respond.

Testing and Refining Your Bot

Before going live, thoroughly test your bot with different message types. Send questions, complaints, compliments, and random texts to see how it responds.

Common refinements include:

  • Adjusting the system prompt for more professional or casual tone
  • Adding fallback responses for unclear messages
  • Implementing rate limiting to prevent spam
  • Setting business hours for automated "we'll reply soon" messages

Important: Always disclose you're using an AI bot. A simple "I'm an automated assistant" in your initial message maintains transparency with customers.
