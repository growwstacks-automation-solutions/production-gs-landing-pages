How to Build a Chatbot UI in Power Apps with Gallery and Power Automate

Most business apps need conversational interfaces but struggle with complex development. This Power Apps solution lets you create a fully functional chatbot in hours - not weeks - using galleries for conversation history and Power Automate for AI integration. No coding required.

Essential Chatbot UI Components

Building an effective chatbot interface in Power Apps requires three core components working together. First is the flexible height gallery that will display your conversation history in chronological order. This replaces the need for complex canvas arrangements or multiple controls.

The second component is the text input box where users will type their messages. Position this at the bottom of the screen with a send button for natural mobile-like interaction. The third critical piece is the Power Automate flow that connects your Power App to AI services like Gemini, handling the API calls and response processing.

Pro Tip: Always start with a blank flexible height gallery rather than a pre-configured template. This gives you complete control over the layout and behavior of your chat messages.

Setting Up the Conversation Gallery

The gallery is the foundation of your chatbot interface. Begin by adding a blank flexible height gallery to your Power Apps canvas. Set the border property to make it visible during development (you can remove this later). Size it to occupy most of the screen, leaving room at the bottom for your input controls.

Inside the gallery, add two key controls: a text label for displaying messages and an image control for showing avatars. Configure the text label with auto height enabled and a reasonable font size (20px works well). This ensures messages of varying lengths display properly without truncation or excessive whitespace.

Collecting and Displaying Chat History

Chat history management is where the Collect function becomes essential. Create a collection called 'colChatHistory' with two fields: 'sender' (values: "user" or "bot") and 'message' (the actual text). When a user sends a message, your app will add two records to this collection - the user's input and the bot's response.

Set your gallery's Items property to the collection name (colChatHistory). Power Apps automatically handles the display order, showing the newest messages at the bottom. For the text label in each gallery item, set the Text property to 'ThisItem.message' to display the appropriate content for each chat bubble.

Important: Always clear the input field after adding a message to your collection using the Reset function. This provides a better user experience by preparing the interface for the next message.

Connecting to Power Automate

The real power comes from integrating with Power Automate to connect to AI services. Create a flow that accepts a prompt from Power Apps, calls the Gemini API (or your preferred AI service), processes the JSON response, and returns just the text portion to your app.

In your Power App, use the 'Run' function to call this flow when the send button is pressed. Pass the user's input text as a parameter. The flow will return the AI's response, which you'll then add to your chat history collection with sender="bot". This creates the back-and-forth conversation flow users expect from a chatbot.

Dynamic Image Display by Role

Visual distinction between user and bot messages improves usability. Configure the image control in your gallery to show different avatars based on the sender. Use an If statement in the Image property: 'If(ThisItem.sender = "user", UserImage, BotImage)'.

Upload two images to your Power App - one representing the user (could be a generic person icon) and one for the bot (often a robot or your company logo). Reference these in your If statement. The gallery will automatically display the correct image for each message based on the sender field in your collection.

Testing the Chat Flow

Thorough testing ensures your chatbot works as expected. Start by sending simple messages and verifying they appear in the gallery with the correct avatar. Check that the input field clears after each submission. Test longer messages to confirm the auto-height properties work correctly.

Pay special attention to API response times. Consider adding a loading indicator during Power Automate execution. Test error scenarios too - what happens if the API is unavailable? Implement basic error handling in your flow and display user-friendly messages in the app when issues occur.

Potential UI Enhancements

Once the basic functionality works, consider these UI improvements: Add different background colors for user vs. bot messages. Implement typing indicators when waiting for API responses. Include timestamps for each message. Add a scroll-to-bottom behavior so new messages are always visible.

For enterprise use cases, you might add: Conversation persistence (save chats to Dataverse), Suggested replies/quick responses, File attachment support, or Integration with your CRM or help desk system. These enhancements make your chatbot more powerful and user-friendly.

Key Takeaways

Building a chatbot UI in Power Apps is achievable without complex coding by leveraging galleries for conversation display and Power Automate for AI integration. The flexible height gallery provides the perfect container for dynamic chat content, while collections maintain the conversation history.

In summary: 1) Use a gallery for conversation display, 2) Manage history with collections, 3) Connect to AI via Power Automate, 4) Differentiate messages with dynamic images. This approach delivers professional results with low-code tools.
