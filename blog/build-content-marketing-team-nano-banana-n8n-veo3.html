How the Automation Works

The system acts as your personal content team through a Telegram bot interface. You send requests in natural language, and it automatically determines whether to create images or videos based on your input.

Key components include Nano Banana for image generation, VEO3 for video creation, and n8n as the workflow engine that ties everything together. The magic happens in how these tools are orchestrated to handle different input types:

  • Text without "video" → Generates image using Nano Banana
  • Text with "video" → Creates video using VEO3
  • Image alone → Enhances the image
  • Image + caption → Converts to video

Setting Up Your Telegram Bot

The Telegram bot serves as your always-available content assistant. Creating it takes about 3 minutes:

  1. Open Telegram and search for "BotFather"
  2. Send the command /newbot
  3. Choose a name (e.g., "MyContentCreator")
  4. Select a username ending in "bot"
  5. Copy the provided API token

This token connects your n8n workflow to Telegram. At 6:45 in the video, you'll see exactly where to paste it in the n8n interface to activate the automation.

Configuring Fell AI API

Fell AI powers both the Nano Banana image generation and VEO3 video creation. Setup involves:

  1. Creating a Fell AI account
  2. Generating an API key from the dashboard
  3. Configuring the credential in n8n's HTTP request nodes

The video shows at 8:20 how to set up the authentication header that allows n8n to communicate with Fell AI's services. This one credential powers all the AI content generation in the workflow.

Cost-effective creativity: Fell AI offers free credits to start, with image generation costing just 1-5 cents and videos running 50 cents to $2 - far cheaper than human creators.

Workflow Implementation in n8n

The n8n workflow handles all the logic and processing. After importing the template JSON file (available in the video description), you'll need to:

  1. Apply your Telegram credentials to all send nodes
  2. Configure Fell AI API for the HTTP request nodes
  3. Update any hardcoded bot tokens in image URLs

At 10:15, the video demonstrates how to update the image URL parameters to ensure the workflow can properly access uploaded content through your specific bot.

Testing and Troubleshooting

Before going live, verify everything works correctly:

Common issues and solutions are covered at 13:30 in the video, including credential verification and content generation failures. The workflow includes intelligent polling that checks status every 10 seconds to ensure reliable delivery.
