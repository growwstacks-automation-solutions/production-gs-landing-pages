What Makes Copilot Studio Different

Traditional automation tools require complex scripting or only work with APIs. Copilot Studio changes this by combining natural language processing with computer vision and robotic process automation. The platform understands what you want done in plain English, then figures out how to accomplish it across multiple applications.

At the 2:15 mark in the video, you can see how the agent navigates ServiceNow without any pre-programmed scripts. It logs in, finds the right records, and updates fields - all through natural language instructions. This represents a fundamental shift in how businesses approach automation.

Understanding the Computer Use Feature

The magic happens through Copilot Studio's computer use capability. This allows agents to:

  • Control mouse movements and clicks
  • Type on keyboards
  • Recognize screen elements visually
  • Navigate between applications

Behind the scenes, agents run in a secure Windows 365 environment. This provides isolation from your production systems while allowing access to necessary applications. The 12:45 timestamp shows how credentials can be securely stored in Azure Key Vault rather than hardcoded.

Step-by-Step Building Process

Step 1: Create a New Agent

Start in Copilot Studio by creating a new agent. Give it a descriptive name (like "HR Vacation Booker"), select an icon, and choose the GPT-4 model version. The platform will soon support newer models like GPT-4 Turbo.

Step 2: Define Topics

Topics represent different conversation flows. For our HR agent, we created a "Book Vacation" topic that triggers when someone asks about vacation time.

Step 3: Configure Computer Use

Under the computer use section, specify which applications the agent can access. You can restrict it to specific websites or desktop apps for security.

Step 4: Add Connectors

Connect to external systems like ServiceNow through pre-built connectors or custom APIs. The demo uses both direct API calls and UI interactions.

Pro tip: Start with simple tasks and gradually increase complexity. The 7:20 mark shows how we built the vacation booking flow incrementally.

Working with Connectors and APIs

Copilot Studio supports two integration approaches:

  1. Direct API connections: For systems like ServiceNow that provide robust APIs
  2. UI automation: For legacy systems without APIs

The HR agent combines both methods. It uses ServiceNow's API to query records but falls back to UI interactions when API access isn't available. At 14:30 in the video, you can see how the agent handles both scenarios seamlessly.

Testing and Deployment Considerations

Before deploying any Copilot agent:

  • Test with non-production data first
  • Implement proper error handling
  • Set content moderation levels appropriately
  • Configure credential storage securely

The demo shows these best practices in action. Notice how at 16:45, the agent gracefully handles incorrect inputs by asking for clarification rather than failing.
