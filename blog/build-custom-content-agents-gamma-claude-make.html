The Personalized Content Challenge

Businesses know personalized content converts better - but creating custom materials for each lead is time-consuming and expensive. Marketing teams face a dilemma: generic content gets poor engagement while customized materials require manual work that doesn't scale.

This workflow solves that problem by automating the entire content creation process. When a lead requests information through a form, the system instantly generates a professionally designed, brand-consistent document tailored to their specific needs - all without human intervention.

68% of marketers say personalization improves customer relationships, yet only 22% have automated systems to deliver customized content at scale.

Workflow Overview

The automated content agent follows a clear sequence from request to delivery. It begins when a lead submits information through an Airtable form (or alternative like Google Forms). This triggers the Make.com workflow which coordinates all subsequent steps.

Claude API receives the lead's information and uses a custom skill to determine how to personalize the content. It then generates instructions for Gamma's design AI, which creates the final branded document. The system emails the content to the lead while logging everything in Airtable and notifying your team via Slack.

Building the Claude Skill

The Claude skill is the brains of the operation - it ensures content stays on-brand and relevant to each request. Creating an effective skill requires careful planning and testing to get right.

Start by uploading a sample PDF of your existing content to Claude. Use the conversation interface to explain how you want the content customized for different scenarios. Claude will generate a skill file that includes brand guidelines, content structure rules, and personalization instructions.

Key tip: Always specify that the skill must be API-compatible. Regular Claude skills won't work in automated workflows without this crucial adaptation.

Gamma Setup & Configuration

Gamma handles the design aspects that Claude can't. Proper setup requires creating both a custom theme (for branding) and templates (for content structure).

Your Gamma theme should include your logo, brand colors, and preferred fonts. Templates define the structure of your outputs - whether PDFs, slide decks, or web pages. The workflow uses Gamma's API to combine these elements with Claude's customized content for professional-looking results.

Design advantage: Gamma produces significantly better visual outputs than Claude's native PDF builder, with proper layouts, image placement, and typography.

Make.com Integration

Make.com orchestrates the entire workflow, connecting all the components. The platform's visual interface makes it easy to design the automation without coding.

The workflow begins with an Airtable trigger, progresses through Claude and Gamma modules, then concludes with delivery steps. Make's Gamma module is particularly powerful - it's more fully featured than what's available in other automation tools like Zapier or n8n.

Key configuration points in Make:

  1. Set Claude to use your custom skill for each request
  2. Map form fields to appropriate content personalization points
  3. Configure Gamma to use your branded theme and templates
  4. Set delivery preferences (email, Slack notifications, etc.)

Content Delivery System

The workflow includes multiple delivery methods to ensure content reaches both the requester and your team. Gmail handles customer-facing delivery while Slack provides internal notifications.

Each email includes a personalized message and link to view the generated content. The Gamma URL allows recipients to view the material online and optionally export it as a PDF or PowerPoint. Meanwhile, Slack notifications keep your team informed about new requests and the content being generated.

Tracking benefit: The system logs all generated content in Airtable, creating a searchable database of what materials have been sent to whom.
