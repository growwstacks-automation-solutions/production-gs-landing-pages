n8n Setup Options: Cloud vs Self-Hosted

The first decision when building an n8n chatbot is choosing between cloud hosting (n8n's paid service) or self-hosting on your own server. The cloud version starts at $20/month but limits you to 5 active workflows - not ideal if you're serving multiple clients.

Self-hosting with a provider like Hostinger costs just $7/month (80% off during Black Friday) and gives you unlimited workflows. More importantly, it keeps each client's data separate on their own server - critical for security and compliance.

Cost comparison: Self-hosting saves 72% versus n8n cloud ($7 vs $26/month). The tradeoff is slightly more setup work, but services like Hostinger automate most of the server configuration.

Creating Your First AI Agent in n8n

n8n's AI Agent node provides everything needed for a basic chatbot in one place. Start by adding a Chat Trigger node, then connect it to an AI Agent node. The magic happens when you select your large language model (LLM) - options include GPT-4, Claude, Gemini and 560+ others via OpenRouter.

At 4:20 in the video, you'll see how to connect an OpenAI API key. The key insight? Use OpenRouter as a single point for all your AI models - it's like Uber Eats for LLMs, letting you access multiple models with one API key instead of managing separate accounts.

Adding Memory and System Prompts

Basic chatbots forget conversations immediately. n8n solves this with simple memory functionality that stores up to 5 conversation turns by default (expandable with databases). Enable it by connecting a session ID to your AI Agent node.

System prompts (shown at 12:45) define your chatbot's personality and capabilities. Example: "You are a helpful assistant that tells a joke after each message." This stays constant while the user's messages change. Well-crafted system prompts reduce the need for complex coding.

Connecting Tools Like Google Sheets

n8n truly shines when connecting your chatbot to other tools. At 18:30, we add a Google Sheets tool that lets the AI update task statuses. The key is proper tool naming and descriptions:

  1. Name: "Task Updater"
  2. Description: "Updates Google Sheet with task number and status"
  3. Required fields: task_number, status

The AI automatically uses this tool when users mention tasks, proving you don't need to hardcode every interaction.

Webhook Integration for Websites

Webhooks (demonstrated at 32:10) connect your n8n chatbot to websites. The workflow:

Data flow: Website → Webhook (POST request) → n8n processes with AI → JSON response → Website displays reply. This two-way communication enables real-time chatting.

Critical components are the chat message and a unique chat ID for memory. The response must be properly formatted JSON - at 45:20 we fix common quotation mark issues that break the response.

Embedding the Chatbot on Your Site

n8n offers two embedding options (shown at 38:50):

  1. Hosted chat: Simple iframe with basic styling
  2. Embeddable widget: Custom JavaScript for full design control

For professional sites, use AI tools to generate branded chat interfaces (CSS/HTML) that connect to your n8n webhook. The video shows how to implement this in GoHighLevel at 41:30 - the same method works for WordPress, Shopify, and other platforms.
