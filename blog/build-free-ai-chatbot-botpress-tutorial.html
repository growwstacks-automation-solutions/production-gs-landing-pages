Botpress Basics: 2 Node Types You Need

Botpress simplifies chatbot building with just two node types that cover every possible interaction. Standard nodes create predictable, step-by-step flows perfect for collecting structured data like reservation details. Autonomous nodes tap into AI to handle free-form conversations using your knowledge base.

At 1:45 in the video, you'll see how the restaurant chatbot uses both types: an autonomous node for greeting and menu questions, then standard nodes to collect booking details systematically. This combination delivers both natural conversation and reliable data collection.

Step 1: Building Your Greeting Node

The greeting node serves as your chatbot's first impression and information hub. By making it an autonomous node connected to your knowledge base, it can answer hundreds of potential questions without you scripting each one individually.

Start by giving your AI assistant a clear role description like "You are a friendly restaurant assistant who greets customers, answers menu questions, and guides them to reservations." Then upload your menu PDF or text file to the knowledge base - the AI will automatically extract and reference this information.

Pro tip: Add transition conditions like "When the user mentions 'reservation', move to the booking node" to create natural conversation flows between autonomous and standard nodes.

Step 2: The Reservation Collection Node

This standard node uses three simple cards to collect all necessary booking details systematically. The email card asks for contact information, the date/time card handles scheduling, and the party size card captures the number of guests.

For each card, you'll create a variable to store the customer's response. These variables then feed into both the confirmation system and your booking records. At 3:20 in the tutorial, you can see how the execution card generates a unique booking ID using one line of JavaScript code (provided in the video description).

Step 3: Automated Confirmation System

Nothing destroys trust faster than uncertainty after a booking. This node combines email and in-chat confirmations to reassure customers instantly. The send email card inserts all the variables you collected (date, time, party size, booking ID) into a professional template.

The final text card within the chat provides immediate confirmation and any special instructions. At 6:10 in the video, notice how the same variables appear in both the email and chat message - ensuring consistency across communication channels.
