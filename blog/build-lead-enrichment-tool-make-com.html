Workflow Overview

Christian's automation solves two specific enrichment needs for his sales team:

  1. Facility Type: Identifying whether each company operates at a single location or multiple locations
  2. Company Description: Generating a 3-4 sentence summary about each business

The workflow follows this architecture:

1. Input: Google Sheets with basic lead information
2. Processing: Make AI web researcher gathers data
3. Output: Structured JSON with enrichment fields
4. Update: Original spreadsheet gets enhanced data

At 7:30, Christian explains the duplicate handling logic: "There are multiple contacts associated with each company... by adding a company spreadsheet, I can get rid of the duplicates so we only search each row one time."

Step 1: Setting Up Your Data

The tutorial begins with two connected Google Sheets:

  • Contacts Sheet: Contains all individual leads (name, email, company, etc.)
  • Companies Sheet: Deduplicated list of organizations (one row per company)

At 9:15, Christian demonstrates the deduplication process using Google Sheets' "Remove duplicates" feature, reducing 36 rows to just 20 unique companies - immediately cutting the required processing by 44%.

Pro Tip: Add a "Processed" column (Christian uses "SENT") to track which records have been enriched, preventing duplicate work when rerunning the automation.

Step 2: AI Web Research Module

The core innovation comes at 12:30 with Make.com's AI web researcher. Christian explains: "This is like Perplexity on steroids... it generates exact responses that you need it to."

Key configuration points:

  1. Prompt Engineering: The tutorial shows how to craft prompts that return structured JSON output
  2. Error Handling: Default values for when information isn't available
  3. Validation: Manual spot-checking of initial results (shown at 16:45)

At 18:20, Christian refines the output format: "I'm going to ask for an output in JSON format with the first piece being the facility type... and the second piece being the company description."

Step 3: JSON Processing

Make.com's built-in JSON parser (featured at 19:50) eliminates what previously required a separate module. Christian notes: "Huge upgrade from Make by adding this in... we can parse it directly from here."

The workflow extracts:

  • facility_type: "single" or "multi"
  • company_description: 3-4 sentence summary

At 22:30, the tutorial shows how to handle array outputs when multiple contacts belong to one company using Make.com's array aggregator - a crucial technique for efficiency.

Step 4: Updating Records

The final step writes the enriched data back to both sheets:

  1. Companies Sheet: Marks records as processed (Christian uses "SENT" in caps)
  2. Contacts Sheet: Adds the new facility type and description fields

At 25:10, Christian troubleshoots a common issue: "We took the row number of the wrong spreadsheet... we should have taken the row number here." This demonstrates real-world problem solving when building automations.

Performance Note: The final version processes about 60 leads/hour. At 27:30, Christian mentions: "I'm just going to let this run one row every minute and it should be done in 15 minutes."
