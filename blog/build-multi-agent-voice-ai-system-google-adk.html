The Fundamental Voice Assistant Problem

Every business considering voice AI faces the same frustrating trade-off: speed versus depth. Traditional single-agent systems excel at quick responses to simple queries like "What's the weather today?" but collapse when users ask for trend analysis or comparative data. Conversely, systems that deliver detailed responses force users to endure lengthy verbal explanations of data that should be visual.

The demo transcript reveals this perfectly - when asked about weekly weather trends, a standard voice assistant recites every temperature point verbally. This creates cognitive overload rather than insight. The same applies to stock prices, sales reports, or any data-rich query where patterns matter more than individual data points.

Key insight: Voice is optimal for conversational flow but terrible for structured data delivery. The human brain processes visual information 60,000x faster than text/audio, making charts indispensable for complex analysis.

How Multi-Agent Architecture Solves It

Google's Agent Development Kit (ADK) enables a breakthrough approach: specialized agents working in concert. In our implementation:

  1. Live Agent: Gemini 2.5 Flash Native Audio handles real-time voice interaction with natural 1-2 sentence responses
  2. Detail Agent: Gemini 2.5 Flash analyzes conversation transcripts to generate visual augmentations only when needed

This separation of concerns delivers both instant gratification (voice responses in <500ms) and deeper insight (auto-generated charts for complex queries). The system intelligently routes each query component to the optimal processing path.

Key System Components

The architecture combines several cutting-edge technologies:

1. Gemini Live Models

Specialized versions of Gemini 2.5 Flash optimized for real-time audio with:

  • Native audio I/O (no separate speech-to-text/text-to-speech)
  • Built-in transcript generation aligned with audio
  • Response streaming for sub-second latency

2. ADK Framework

Provides critical infrastructure:

  • Agent memory and session management
  • Tool integration (like Google Search)
  • Multi-agent coordination

3. WebSocket Streaming

Bidirectional real-time communication for:

  • Audio chunk transmission
  • Transcript synchronization
  • Visual data delivery

Live Agent Configuration

The Live Agent uses specific optimizations for conversational flow:

Critical instruction: "Keep responses brief (1-2 sentences max), conversational, and focused on immediate answers. Defer detailed analysis to subsequent processing."

Technical implementation highlights:

 agent = create_agent(     name="Google Search Agent",     model="gemini-2.5-flash-native-audio-preview",     tools=[google_search_tool],     instructions=live_agent_instructions,     response_modalities=["audio", "text"],     enable_input_transcript=True,     enable_output_transcript=True )

This configuration ensures the agent excels at its primary role - fluid voice interaction - while preparing data for the Detail Agent's analysis.

Detail Analysis Agent Workflow

Triggered after each Live Agent turn completes, this agent:

  1. Evaluates Need: Runs Analyze Trend Need tool to assess whether visualization adds value
  2. Gathers Data: Uses Google Search for historical/contextual information when needed
  3. Formats Output: Generates Chart.js-compatible JSON for frontend rendering

The decision logic focuses on three criteria:

  • Explicit user requests for trends/comparisons
  • Implied need based on query scope (e.g. "last week" vs "today")
  • Data complexity exceeding verbal explanation limits

Real-Time Data Flow Architecture

The system orchestrates multiple parallel data streams:

Dual accumulation: Transcripts stream to the client for real-time display while simultaneously building memory context for the Detail Agent.

Key flow components:

  1. User audio → PCM encoding → WebSocket → Live Agent
  2. Agent response → Parallel streams:
    • Audio chunks → Web Audio API playback
    • Transcript text → UI display + session memory
  3. Turn completion → Transcript bundle → Detail Agent
  4. Analysis results → Structured JSON → Chart rendering

Frequently Asked Questions

Common questions about multi-agent voice systems

Single-agent systems force a trade-off between conversational speed and data depth. They either deliver fast but shallow responses (like basic weather info) OR detailed but slow explanations (listing every data point verbally).

Google's ADK framework solves this by separating these functions across specialized agents - one optimized for real-time conversation, another for structured data analysis.

  • Voice agents excel at natural dialogue flow
  • Analysis agents generate visualizations from the same conversation
  • No compromise on either interaction quality or data depth

The system uses two specialized agents working in sequence:

1) Live Agent: Handles real-time voice interaction using Gemini 2.5 Flash Native Audio, focusing on 1-2 sentence natural responses. 2) Detail Agent: Processes the full conversation transcript afterward to generate visual augmentations when needed.

  • Agents share memory via session transcripts
  • Each agent has distinct tools and instructions
  • Frontend combines both outputs seamlessly

Any query needing both instant answers and deeper analysis benefits from this architecture. Common examples include:

Weather (current conditions + weekly forecasts), financial data (stock quotes + historical trends), business metrics (daily numbers + quarterly comparisons). The system automatically detects when visual augmentation adds value beyond the verbal response.

  • Time-series data queries
  • Comparative analysis requests
  • Any question implying "show me" rather than "tell me"

The Detail Agent runs an Analyze Trend Need tool that evaluates transcripts using three criteria:

1) Explicit requests for trends/comparisons 2) Implied need based on query timeframe (e.g. "last week" vs "today") 3) Data complexity exceeding reasonable verbal explanation limits. Simple queries like current weather skip visualization.

  • Uses semantic analysis of user intent
  • Considers data density thresholds
  • Configurable sensitivity for different domains

The system uses WebSockets for low-latency bidirectional communication:

User audio gets encoded as PCM data sent to the Live Agent. Responses stream back as both audio (via Web Audio API) and synchronized transcripts. All processing happens asynchronously to prevent audio lag while maintaining transcript accuracy.

  • WebSocket protocol for real-time data
  • Web Audio API for instant playback
  • Async processing prevents blocking

The Detail Agent outputs structured JSON matching Chart.js requirements:

Includes pre-formatted labels, values, axes titles and written insights. This eliminates frontend data transformation, allowing direct rendering of professional charts from the agent's output. The system essentially delivers "visual responses" as ready-to-display packages.

  • Standardized JSON schema
  • Pre-computed chart configurations
  • Embedded insights as annotations
