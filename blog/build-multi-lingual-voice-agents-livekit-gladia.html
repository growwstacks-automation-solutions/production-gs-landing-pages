Step-by-Step Setup Process

Setting up your multilingual voice agent requires just a few key steps. The entire process can be completed in under 30 minutes, even if you're new to voice technology.

Step 1: Create Your LiveKit Project

Begin by creating a new project in LiveKit. Name it something identifiable like "Gladia Multilingual" to keep your projects organized. This creates the foundation for your voice agent's communication capabilities.

Step 2: Obtain Your API Keys

You'll need three critical API keys: Gladia for transcription, OpenAI for response generation, and ElevenLabs for text-to-speech. The Gladia key is particularly important - it's what enables the automatic language detection and transcription.

Step 3: Configure the Agent Files

Replace the default agent.py file with our customized version that includes Gladia and ElevenLabs integration. This modified file contains the multilingual prompt template you can customize for your specific needs.

Step 4: Set Up the Environment

Run the UV Sync command to establish your virtual environment, then execute the agent download command to pull all necessary dependencies. This prepares your system for local testing.

Step 5: Launch and Test

With everything configured, start your agent locally using the UV Run command. Test by speaking different languages - the system should transcribe and respond appropriately without any manual intervention.

Pro Tip: For faster response times, consider using ElevenLabs' Fast 2.5 model instead of the default Multilingual v2. This reduces latency significantly while maintaining quality.
