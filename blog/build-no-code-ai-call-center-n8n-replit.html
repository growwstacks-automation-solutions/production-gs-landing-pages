3-Part System Architecture Overview

This solution combines three powerful tools into a seamless workflow:

1. Replit Dashboard (Frontend)

The user interface where businesses upload lead lists (CSV files), schedule calls, and view analytics. Built with Replit's new natural language app builder, it requires no traditional coding.

2. n8n Workflow Engine (Backend)

Acts as the central nervous system - triggering calls at scheduled times via ElevenLabs API, processing call results, and updating Google Sheets with transcripts and outcomes.

3. ElevenLabs Voice AI

Handles the actual phone conversations with human-like quality (4.1s average response time). Configured with custom call flows, fallback responses, and integration points to n8n.

Technical note: All components connect via webhooks and APIs. No local installation required - the entire system runs in the cloud with minimal setup.

Configuring ElevenLabs Voice Agent

ElevenLabs provides the voice AI capabilities. Here's how to set it up:

Step 1: Create Agent

After signing up (free tier available), create a new "blank agent" and name it appropriately for your use case (e.g., "Sales Qualifier").

Step 2: System Prompt

This is the brain of your agent. A well-structured prompt should:

  • Define the agent's role and capabilities
  • Include conversation flow logic (if X then Y)
  • Specify tone and handling of objections
  • Connect to your knowledge base (upload FAQs/documentation)

Step 3: Webhook Configuration

Under "Security" settings, add your n8n webhook URL to connect call triggers and results. This enables two-way communication between the voice agent and your backend.

Pro tip: Use GPT-4.1 (not 5) for the LLM unless you need complex reasoning. The latency difference (1.2s vs 3.8s) significantly impacts call flow.

Building the n8n Backend Workflow

The n8n workflow handles four critical functions:

1. Call Triggering

Receives schedule from Replit → Calls ElevenLabs API with lead details → Initiates outbound call

2. Call Monitoring

Listens for call completion → Retrieves full transcript → Performs sentiment analysis

3. Result Processing

Determines call outcome (booked/not booked) → Identifies reasons for unsuccessful calls → Generates improvement suggestions

4. Data Storage

Updates Google Sheets with all call details → Syncs with Replit dashboard → Optionally pushes to CRM

Key node: The "Combine Transcript" JavaScript node merges segmented call transcripts into a single readable format while preserving timing data.

Creating the Replit Analytics Dashboard

Replit's natural language app builder makes dashboard creation surprisingly simple:

Core Components

  • Lead Upload: CSV import with validation (required fields, email format checking)
  • Scheduling Interface: Table view of leads with time/date picker
  • Analytics Dashboard: Shows call volume, success rates, AI-generated insights
  • Google Sheets Connector: Built-in integration for real-time data sync

The entire interface can be built by describing what you want in plain English. For example:

"Build a lead scheduling dashboard with CSV upload, editable schedule times, and analytics showing total calls, booked calls, and success rate percentage."

Replit's AI will generate the code and provide a working prototype typically within 15-30 minutes.

Connecting All Components

The magic happens when these systems work together:

Workflow Sequence

  1. User uploads leads CSV in Replit dashboard
  2. Replit stores data in Google Sheets and displays in scheduling interface
  3. At scheduled time, n8n triggers ElevenLabs call via webhook
  4. ElevenLabs handles conversation, sends transcript back to n8n
  5. n8n analyzes results, updates Google Sheets
  6. Replit dashboard refreshes with latest analytics

Integration tip: Use unique lead IDs throughout the system to maintain data consistency across all platforms.
