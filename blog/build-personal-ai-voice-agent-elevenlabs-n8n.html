ElevenLabs Agent Configuration

Creating your voice agent begins in the ElevenLabs platform. After logging in, navigate to the Agents section and click "New Agent". You'll choose between different template types:

  • Blank Assistant - Full customization from scratch
  • Business Agent - Pre-configured for professional use cases

The critical configuration elements are:

  1. System Prompt - Defines how your agent introduces itself
  2. Core Instructions - The operational rules and capabilities
  3. Personality Settings - Voice tone and response style

ElevenLabs provides an auto-generator for the personality profile. For our demo, we created "Alfred" - an efficient, discreet assistant modeled after the iconic Batman character. This gives the agent recognizable characteristics while maintaining professionalism.

Connecting to n8n Workflows

The real power comes when connecting ElevenLabs to n8n's automation capabilities. In the ElevenLabs interface, we add a custom webhook tool that points to our n8n workflow URL.

The n8n workflow has several key components:

Step 1: Webhook Trigger

This receives the voice command from ElevenLabs and initiates the workflow. Configure it to accept POST requests with JSON data containing the transcribed command.

Step 2: AI Processing Node

We use Open Router to access multiple AI models. For simple requests, GPT-4-turbo works well. Complex tasks may require Claude 3 or Gemini 1.5 Pro for better context handling.

Step 3: Application Modules

These connect to your business apps - Gmail for emails, HubSpot for CRM, Perplexity for research. Each module handles specific task types.

Pro Tip: Start with just email functionality, then add CRM and research modules once the basic workflow is stable.
