Step 1: Document Preprocessing

Municipal PDFs contain tables, footnotes, and formatting that confuse basic text extraction. The solution? Python's PyPDF2 library with custom post-processing:

Key Technical Steps:

  1. Extract raw text while preserving section hierarchies (1.1, 1.2, etc.)
  2. Clean artifacts like page numbers, headers/footers that disrupt context
  3. Rebuild logical flow where PDFs split clauses across pages
  4. Output structured markdown with headings for semantic processing

Pro Tip: Add metadata tags during extraction (e.g. #water_main, #backfill_requirements) to enable categorical filtering in later searches.

Step 2: Chunking Strategies

Throwing entire documents at an LLM is like bringing a textbook to an open-book test - inefficient and expensive. Effective chunking acts as curated study notes for the AI:

3 Municipal-Specific Approaches:

  1. Clause-based chunks (ideal for standards): Split at every numbered clause (3.1, 3.2) with 10% overlap
  2. Hierarchical chunks: Parent section + all child clauses as one unit
  3. Table-aware chunks: Extract tabular data separately with descriptive headers

The student's implementation used clause-based chunking, resulting in 1,200+ searchable segments from what was originally 400 PDF pages. This reduced average response latency from 12 seconds to under 3.

Step 3: LLM Integration

While any modern LLM can power RAG, municipal applications demand precision. After testing 4 models, the student found:

Model Benchmark Results:
GPT-4 Turbo: 88% accuracy
Claude 3 Opus: 93% accuracy
Gemini 1.5: 85% accuracy
Mixtral: 79% accuracy

Claude's superior performance on technical Q&A justified its higher cost. The integration pipeline:

  1. User question → vector similarity search
  2. Top 3 chunks retrieved → injected into prompt
  3. LLM synthesizes answer with citations
  4. Confidence scoring → web fallback if under 70%

Step 4: System Prompt Engineering

The secret sauce? A 287-word system prompt that eliminated hallucination:

Key Prompt Elements:
- "You are a senior municipal engineer..."
- "Answers must include clause citations..."
- "If uncertain, respond 'This requires manual verification...'"
- "Format: numbered steps, max 150 words..."

This reduced follow-up clarification questions by 62% compared to generic prompts. The prompt also encoded municipal-specific heuristics like prioritizing safety factors in calculations.

Step 5: Validation Framework

Production readiness required rigorous testing. The student created:

  1. Golden Dataset: 53 manually answered questions covering all document sections
  2. Accuracy Matrix: Scoring for correctness, citation accuracy, and completeness
  3. Edge Cases: Ambiguous queries, outdated standards, conflicting clauses

After 3 iterations adjusting chunk sizes and prompts, the system achieved 93% accuracy - surpassing the senior engineers' 90% target. The validation framework now serves as a regression test suite for future updates.
