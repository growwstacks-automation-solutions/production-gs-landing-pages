The Problem With Text-Only AI Responses

Most AI agents today return plain text responses that require manual interpretation. When asking about financial data, customer metrics, or operational KPIs, you get walls of text that need to be copied into spreadsheets or visualized separately. This creates friction in decision-making and slows down business processes.

TheSys C1 changes this paradigm by enabling AI agents to return fully-formed UI components instead of raw text. At 2:15 in the video, you can see the dramatic difference - where a traditional AI response would show "Q4 revenue was $287,000", TheSys C1 generates an interactive chart showing monthly trends, product breakdowns, and comparison metrics.

Key insight: Users complete tasks 40% faster with visual interfaces compared to text-only chatbots according to TheSys internal testing.

How TheSys C1 Solves This Problem

TheSys C1 acts as middleware between your AI agent and the end user. Instead of connecting directly to OpenAI or another model provider, you route requests through TheSys API. Their system analyzes the response and converts it into appropriate UI components based on the data structure and content.

The transformation happens in three stages:

  1. Data extraction: The AI response is parsed to identify numerical data, categorical information, and actionable insights
  2. Component selection: TheSys determines the most effective visualization for each data type (charts for trends, tables for detailed records, cards for summaries)
  3. UI generation: Components are assembled into a cohesive interface with interactive elements

At 4:30 in the video, you can see how this works in practice when asking "Which product is causing this decline?" - the response includes both a bar chart showing revenue by product and a table with cancellation rates.

Step-by-Step Setup Guide

Implementing this solution requires just a few configuration steps:

Step 1: Set Up TheSys C1 Account

Create a free account at thesis.dev and navigate to the API keys section. Generate a new key and note the base URL (api.thesis.dev/v1).

Step 2: Configure n8n Credentials

Create new OpenAI credentials in n8n but replace the base URL with TheSys endpoint and paste your API key. This routes all requests through TheSys instead of directly to OpenAI.

Step 3: Embed the Chat Widget

Copy the widget code from TheSys documentation and add it to your HTML page. The only customization needed is pointing the webhook URL to your n8n workflow.

At 9:20 in the video, you can see the complete n8n workflow with just four nodes: chat trigger, AI agent, Google Sheets tool, and Slack integration.

Configuring Streaming Responses

For a more ChatGPT-like experience, you can enable streaming responses where components render progressively. This requires three simple settings:

  1. Set streaming: true in the widget configuration
  2. Select "streaming" as the response mode in the n8n chat node
  3. Enable streaming in the AI agent node settings

The video shows this in action at 11:35 where charts and tables populate gradually rather than appearing all at once. This creates a more engaging user experience especially for complex queries.
