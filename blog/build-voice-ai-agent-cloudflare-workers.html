What Makes Cloudflare's Voice SDK Different

Most voice AI solutions require stitching together multiple APIs (speech-to-text, LLMs, and text-to-speech services. Cloudflare's new voice SDK bundles everything into one package that runs directly on their edge network edge.

The SDK includes three key capabilities:

  • withVoice: Full voice agent with STT + TTS
  • withVoiceInput: Speech-to-text only option
  • VoiceClient: Framework-agnostic JavaScript client

Best of all - it uses Workers AI models so there's no need for external API keys or per-call pricing - everything runs on Cloudflare's global network.

The 3 Key Components of Our Voice Agent

Our customer support agent handles three core tasks: order lookups, return requests, and inventory checks. Here's how the pieces fit together:

1. The Voice Agent Class

Extends Cloudflare's base VoiceAgent class with:

  • Transcriber (using Workers AI's DeepGram model)
  • TTS method for responses
  • onCallStart hook for greetings

2. The Tool System

Three Zod-validated tools with:

  • Clear descriptions with example formats
  • Fake API endpoints for demo purposes
  • Normalization for order numbers

3. The LLM Configuration

Using Gemma model from Workers AI with:

  • Custom system prompt
  • Markdown suppression
  • 5-step limit to prevent runaways
