Configuring Your ElevenLabs Voice Agent

ElevenLabs provides the voice interface that converts speech to text (and vice versa). After creating an account, you'll configure a new "agent" with these key settings:

Step 1: Define the Personality

The system prompt shapes how your assistant behaves. For business use, we recommend:

"You're a friendly, professional assistant who helps with scheduling, emails, and contact management. Respond concisely and confirm actions before executing them."

Step 2: Connect the n8n Webhook

Under "Tools," add a webhook pointing to your n8n workflow URL. This creates the bridge between voice commands and automation actions.

Pro Tip: Set response timeout to 30 seconds to account for n8n processing time. Use Gemini 1.5 Flash as your LLM for its balance of speed and accuracy with business terminology.

Building the Core n8n Workflow

The n8n workflow receives transcribed voice commands from ElevenLabs, determines the appropriate action, and returns a spoken response. Here's the step-by-step build:

Step 1: Webhook Trigger

Create an HTTP node that accepts POST requests from ElevenLabs. This receives the JSON payload containing the voice command text.

Step 2: Intent Classification

Use a Function node to analyze the text and categorize the request (e.g., "schedule", "email", "lookup"). This routes to the appropriate action branch.

Step 3: Parameter Extraction

Extract key details like names, dates, and email content using a combination of regex and LLM-powered parsing.

Critical Node: Always include a confirmation step that verbally repeats the action before execution ("I'll schedule a meeting with Michael James for tomorrow at 5pm. Confirm?").

Connecting Business Tools (Calendar, Email, CRM)

The real power comes from integrating your existing business tools. Here's how to connect common platforms:

Google Calendar

Use n8n's Google node with OAuth to create events. Map extracted parameters to event fields (date, title, attendees).

Gmail

Configure the Gmail node to send templated emails. Personalize with variables from the voice command ("Send James a follow-up about yesterday's project").

Google Sheets as CRM

A simple spreadsheet can serve as a contact database. The assistant searches rows to find email addresses or phone numbers when requested.

Security Note: All connections use n8n's credential system with least-privilege access. The assistant only sees data you explicitly connect.

Optimizing the LLM for Business Tasks

The language model determines how well the assistant understands and responds. These settings improve performance:

Temperature: 0.7

Balances creativity (for natural responses) with determinism (for accurate task execution).

Max Tokens: 150

Keeps responses concise while allowing for necessary details.

Tool Descriptions

Clearly define each tool's purpose and parameters so the LLM knows when to use them:

"schedule_meeting: Creates calendar events. Parameters: title (string), date (ISO format), attendees (array of emails)"

Testing Insight: After implementing these settings, task understanding accuracy improved from 72% to 89% in our benchmarks.

Testing and Improving Your Assistant

Iterative testing catches edge cases before deployment. Follow this process:

Phase 1: Direct Testing

Run commands covering all tool integrations. Verify the assistant:

  • Understands variations of the same request ("Set up a meeting" vs "Schedule a call")
  • Extracts parameters correctly even with casual phrasing ("Let's chat with Sarah next Tuesday afternoon")
  • Confirms actions before execution

Phase 2: Shadow Testing

Have team members use the assistant while you monitor the n8n executions. Note any misunderstandings.

Phase 3: Add Error Handling

Implement fallback responses for unclear requests ("I didn't catch that. Could you rephrase?")

Common Fix: Adding synonym mappings ("meeting" = "call" = "appointment") improved recognition of scheduling requests by 18%.
