Setting Up n8n With WhatsApp Business API

The foundation is connecting n8n to WhatsApp's Business API. While the approval process takes 1-2 days, the technical setup is straightforward:

Step 1: Create Your WhatsApp Business Account

Navigate to Facebook Business Manager and register for WhatsApp Business API access. You'll need to verify your business domain and provide basic company information.

Step 2: Generate API Credentials

Once approved, generate your permanent access token and phone number ID from the WhatsApp API settings. These credentials will authenticate your n8n workflow.

Step 3: Connect to n8n

In your n8n dashboard, add the WhatsApp node and enter your credentials. Test the connection to ensure messages can be both sent and received.

Pro tip: Use a dedicated phone number for your AI agent - mixing human and automated responses on the same number can confuse customers.

Configuring the AI Brain (GPT-4)

The magic happens when we connect WhatsApp to a powerful language model. For this workflow, we're using OpenAI's GPT-4 turbo for its balance of speed and accuracy.

Step 1: Add the OpenAI Node

Drag the OpenAI node into your n8n workflow. Connect your API key (available from OpenAI's dashboard) and select the gpt-4-turbo model.

Step 2: Craft Your System Prompt

This is where you define the AI's personality and capabilities. A good starter prompt:

"You are Sam, a friendly customer support agent for [Your Business]. Answer questions concisely using the knowledge base. For pricing inquiries, say 'Our pricing starts at $X. Would you like me to connect you with a specialist?' Always end by offering to schedule a call."

Step 3: Set Response Parameters

Configure temperature (0.7 for balanced creativity), max tokens (120 keeps responses concise), and presence penalty (1.0 avoids repetition).

Adding Conversation Memory

Basic chatbots frustrate users by forgetting context between messages. Our AI agent maintains a 10-message memory window:

Step 1: Add Context Memory

Implement a simple JSON variable that stores the last 10 message pairs (user input + AI response). This gives GPT the conversation history.

Step 2: Configure Context Window

Set your memory node to trim older messages automatically, keeping only the most recent relevant exchanges.

Step 3: Test Memory Functionality

Send a sequence like "What's your pricing?" followed by "And for enterprise?" to verify the AI remembers the context.

Implementation note: At 4:32 in the video tutorial, we debug a common memory initialization error by ensuring the JSON array starts empty.

Testing and Debugging Your Workflow

Before going live, thorough testing prevents embarrassing errors. Follow this checklist:

1. Test Basic Functionality

Send sample messages covering your most common inquiries. Verify responses match expectations.

2. Check Edge Cases

Test with misspellings, abrupt topic changes, and complex multi-part questions.

3. Validate Escalation Paths

Ensure "speak to a human" requests properly route to your team.

4. Monitor Response Times

Aim for sub-3-second replies - slower responses break the chat illusion.

Debugging tip: Use n8n's execution viewer to inspect exactly what data passes between nodes when errors occur.
