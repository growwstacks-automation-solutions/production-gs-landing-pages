The Problem With Government Service Information

Applicants looking to relocate to Germany face a maze of visa documentation scattered across government websites. Whether it's for work, study, or family reunification, finding clear answers about requirements and procedures can be overwhelming. The information exists - but it's buried in PDFs, spread across multiple agencies, and often written in bureaucratic language.

This creates significant frustration for applicants who need quick, reliable answers to questions like "What documents do I need?" or "How much does the visa cost?" Without a centralized, conversational interface, people waste hours searching and still end up uncertain about their next steps.

80% of visa applicants report spending more than 5 hours searching for information across multiple government websites before finding what they need.

Solution Overview - Voice AI + RAG

The winning solution from the hackathon combined real-time voice interaction with Retrieval Augmented Generation (RAG) to create an AI avatar that could answer visa and immigration questions conversationally. This approach provides instant answers to common questions while still being able to handle complex, specific queries by searching official documentation.

The system was built in just 3 hours during the hackathon, demonstrating how quickly these AI components can be integrated to solve real problems. It uses ElevenLabs for voice interaction, Superbase for vector storage, and a carefully designed workflow to ensure accurate, helpful responses.

Technology Stack Breakdown

The solution combines several cutting-edge AI technologies into a seamless experience:

  • ElevenLabs - Handles the entire voice interaction pipeline from speech-to-text to text-to-speech
  • Superbase - Vector database for storing and retrieving official documentation chunks
  • RAG Architecture - Combines pre-built FAQ knowledge with semantic search of official sources
  • WebSockets - Enables real-time streaming of voice responses

At 2:15 in the video, you can see how these components work together to provide a natural conversation flow while maintaining accuracy.

Workflow Architecture

The system follows a smart decision-making process for answering questions:

  1. First checks the pre-built FAQ knowledge base for common questions
  2. If the answer isn't found, activates the RAG tool to search official documents
  3. Performs semantic search to find relevant information chunks
  4. Generates a conversational answer while citing the source

Key Insight: This two-tiered approach ensures fast responses for common questions while still handling complex queries accurately.

Building the Knowledge Base

The system starts by scraping and processing official government documentation:

  • HTTP requests download relevant pages and documents
  • Content is cleaned and converted to markdown format
  • Documents are split into logical chunks for semantic search
  • Each chunk is embedded and stored in the vector database

This process creates a searchable knowledge base that maintains connections back to the original source material, ensuring answers can be verified.

Voice Interaction Pipeline

The voice interface makes the system accessible to anyone:

  1. User speaks their question into the microphone
  2. ElevenLabs converts speech to text
  3. System processes the query through the knowledge base
  4. Generated answer is converted back to natural-sounding speech
  5. Response streams in real-time through WebSockets

The multilingual capabilities mean the system can interact with users in their preferred language while still pulling from official sources.

Testing Results and Accuracy

During testing, the system successfully answered both simple and complex questions:

  • Correctly stated the EU Blue Card requires a university degree and job offer with minimum salary
  • Accurately provided the 37 euro fee for EID cards
  • Handled follow-up questions about eligibility and procedures

When the system didn't know an answer (like the exact cost of another document), it clearly indicated it was retrieving the information rather than guessing.
