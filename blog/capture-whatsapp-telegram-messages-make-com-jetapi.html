JetAPI Webhooks Overview

JetAPI acts as a bridge between your messaging apps and Make.com. Instead of polling for new messages (which creates delays and API limits), JetAPI uses webhooks to push messages to Make.com instantly when they occur.

The setup requires three main components: a webhook trigger in Make.com, your JetAPI account connected to WhatsApp/Telegram, and the configuration linking these systems. What makes JetAPI particularly powerful is its ability to capture both incoming and outgoing messages, giving you complete visibility into all conversations.

Step 1: Create Your Make.com Webhook

Begin by creating a new scenario in Make.com with a Webhook as the trigger module. This generates a unique URL that will receive messages from JetAPI.

  1. In Make.com, create a new scenario
  2. Add a Webhook module as the trigger
  3. Copy the generated webhook URL (you'll need this for JetAPI)
  4. Save the scenario but don't run it yet

Pro Tip: Name your webhook clearly (e.g., "WhatsApp-Telegram Messages") as you'll likely create multiple webhooks for different purposes.

Step 2: JetAPI Account Setup

If you don't have a JetAPI account yet, create one at jetapi.io. The free tier is sufficient for testing this integration.

After signing up, authorize your messaging channels:

  • For WhatsApp: Scan the QR code with your phone and wait for chat synchronization
  • For Telegram: Log in with your phone number and enter the received code

Both authorizations typically complete within 2-3 minutes. Keep the app open during this process - closing it may interrupt the connection.

Step 3: Connect Webhook in JetAPI

With your Make.com webhook URL copied and JetAPI authorized, it's time to connect them:

  1. In JetAPI, open the sidebar and go to Settings → Webhooks
  2. Click "Add Webhook"
  3. Paste your Make.com webhook URL
  4. Select which message types to receive (WhatsApp, Telegram, or both)
  5. Click "Create"

Test the connection by sending a message through either WhatsApp or Telegram (depending on your setup). Within seconds, you should see the message data appear in Make.com when you run your scenario once.

Understanding the Message Payload

Each message captured through JetAPI delivers a comprehensive JSON payload to Make.com. Key data points include:

  • Message content: The actual text sent or received
  • Sender information: Phone number (WhatsApp) or chat ID (Telegram)
  • Direction: Whether the message was incoming or outgoing
  • Timestamp: Exact time the message was sent/received
  • Platform: Identifies WhatsApp or Telegram

This rich dataset enables sophisticated filtering and routing in your Make.com automations. For example, you could route customer inquiries to one workflow while internal team messages go elsewhere.
