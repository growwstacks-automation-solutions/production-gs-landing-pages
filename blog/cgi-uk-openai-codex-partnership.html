From 200 to 20,000 Seats: CGI UK's OpenAI Journey

When ChatGPT took the world by storm in 2023, CGI UK faced the same challenge as every enterprise: how to move beyond experimental AI tools to meaningful business transformation. Russell, Head of AI at CGI UK, described their approach as "two intertwined programs" - enhancing client services with AI while simultaneously using the technology to improve internal operations.

What began as a 200-seat ChatGPT trial rapidly evolved into a comprehensive partnership with OpenAI. By , over 20,000 CGI consultants globally were using OpenAI technologies daily, with 8,000 in the UK alone. This included widespread adoption of Codex across their developer teams, fundamentally changing how software was built.

Key Insight: CGI's success came from treating AI adoption as both a client service enhancement and internal transformation initiative from day one. This dual focus prevented the common pitfall of AI remaining siloed in innovation labs.

The Shift to Agentic Delegation in Coding

The evolution of AI coding tools has progressed through three distinct phases, as explained by OpenAI's Pat Pap during the CGI Experiment conference. Early tools like GitHub Copilot (built on GPT-3) focused on simple code completions - helpful but limited in scope.

With ChatGPT's emergence, developers gained conversational AI that could handle more complex requests, though still requiring significant oversight. Today's phase of agentic delegation represents a fundamental shift - AI agents that can run autonomously for hours, following plans, testing outputs, and iterating on solutions.

As Pat demonstrated with his paint-by-numbers algorithm example (timestamp 12:45 in the video), modern Codex implementations can:

  • Run continuous improvement cycles (20+ versions in his demo)
  • Access and modify project files autonomously
  • Execute and evaluate test cases
  • Maintain context across long-running sessions

How Codex Handles Long-Running Development Tasks

The secret to Codex's ability to handle complex, long-duration tasks lies in its iterative execution loop. Unlike traditional AI that provides one-shot responses, Codex operates through continuous cycles of:

  1. Model Inference: Analyzing the current task and objectives
  2. Tool Execution: Running commands to gather context (reading files, checking environments)
  3. Context Building: Updating its understanding of the project state
  4. Action Taking: Implementing changes or creating new versions

In the paint-by-numbers demo, this loop allowed Codex to:

  • Start with a baseline algorithm (V0)
  • Measure performance metrics (region size percentage improved from 16% to 32%)
  • Generate 20 improved versions autonomously
  • Identify V8 as the optimal balance for their needs

Implementation Tip: The plans.md file Pat created provided crucial structure for Codex's autonomous operation. This simple text file outlined the goal, current state, workflow, and success criteria - enabling hours of unsupervised improvement.

Measured Productivity Gains from AI Coding Agents

While CGI UK hasn't released specific productivity metrics, OpenAI shared revealing data about the impact of Codex at scale. Their internal measurements show:

  • Developer headcount growth continues (contrary to fears of AI replacing jobs)
  • Pull request volume per developer has increased dramatically
  • Code contribution velocity accelerated across all projects

This aligns with Pat's observation that "software engineers are definitely here to stay" but their work is fundamentally changing. Developers spend less time on routine coding and more on:

  1. Defining problems and success criteria for AI agents
  2. Reviewing and refining AI-generated solutions
  3. Integrating components into larger systems

The paint-by-numbers demo illustrated this shift perfectly - Pat spent minutes defining the problem and success metrics, then let Codex work autonomously for 22 minutes to produce the solution.

5 Best Practices for Scaling AI Coding Agents

Based on CGI UK's experience and OpenAI's recommendations, these practices maximize the value of Codex implementations:

1. Create Detailed Plans for Agents

The plans.md file in the demo provided structure for autonomous operation. Effective plans include:

  • Clear objectives and success metrics
  • Current state assessment
  • Step-by-step workflow instructions
  • Stopping conditions (e.g., "stop after 20 versions")

2. Ensure Accessible Testing Frameworks

Codex needs the ability to validate its work. CGI UK integrated testing into their Codex deployment so agents could:

  • Run automated tests on each iteration
  • Measure performance against defined metrics
  • Use test results to guide improvements

3. Remove Unnecessary Approval Bottlenecks

Agentic delegation requires autonomy. CGI UK streamlined approval processes to allow Codex to:

  • Access necessary systems and files
  • Execute commands without manual intervention
  • Iterate rapidly without waiting for human review

4. Implement Structured File Systems

Long-running tasks require persistent context. CGI UK's implementation ensured:

  • Clear project directory structures
  • Version-controlled artifacts
  • Machine-readable documentation

5. Prepare for New Interfaces

Traditional IDEs aren't optimized for agent oversight. CGI UK adopted tools like:

  • The Codex app for agent management
  • Enhanced version control visualization
  • Automated change summaries

The Future of AI-Assisted Development

OpenAI's Pat Pap highlighted several emerging trends that will shape the next phase of AI coding:

Faster Models

The upcoming Spark models promise significantly faster inference speeds, reducing the time for complex tasks. In the demo, Pat showed how Spark could instantly analyze a conference agenda PDF and generate a song about the event.

Multi-Agent Systems

Projects like OpenClow demonstrate the potential of multiple specialized agents working in concert. CGI UK is exploring similar architectures where different agents handle specific aspects of development.

New Abstraction Layers

As Andrej Karpathy noted, developers will need to master new concepts like:

  • Agent/sub-agent relationships
  • Prompt engineering at scale
  • Context management across long sessions
  • Tool and plugin integration

Horizon Insight: The most successful organizations will treat AI coding adoption as a fundamental skills transformation, not just tool implementation. Developer roles will evolve toward AI oversight and system architecture.
