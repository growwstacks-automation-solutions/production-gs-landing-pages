Two Approaches to AI in HighLevel

HighLevel offers two distinct ways to implement AI - through standalone conversational agents or ChatGPT functions within workflows. Many users default to one approach without understanding the strategic implications of each.

Conversational AI agents act like virtual team members with memory, knowledge bases, and goal tracking. ChatGPT functions are single-purpose tools that execute specific prompt-based tasks within workflows. The key is knowing when each approach delivers maximum efficiency.

Key insight: Conversational agents cost more but handle complex interactions better, while ChatGPT functions are cheaper for simple, repetitive tasks but lack memory and sophisticated goal tracking.

When Conversational AI Agents Shine

Conversational agents excel in scenarios requiring context retention across multiple interactions. At 12:35 in the video, we see how they maintain conversation history while ChatGPT functions treat each prompt as independent.

Three situations where conversational agents outperform:

  1. Knowledge-base dependent responses - Agents can reference uploaded documents, crawled websites, or manually entered FAQs
  2. Goal-oriented conversations - Built-in tracking of conversation objectives like lead qualification
  3. Fallback scenarios - Automatic switching to secondary models when primary responses fail

The video demonstrates how agents handle Instagram DM conversations naturally, remembering previous interactions - something impossible with standalone ChatGPT functions.

Strategic Uses for ChatGPT Functions

ChatGPT functions within workflows are ideal for simple, repetitive tasks where conversation history isn't needed. At 8:12 in the tutorial, we see them processing Instagram DMs without maintaining context between messages.

Best applications for ChatGPT functions:

  • Quick information retrieval from static knowledge sources
  • Simple automated replies to common inquiries
  • Prompt chaining within complex workflows
  • Cost-sensitive applications (typically 3-5x cheaper per execution)

Pro tip: Chain multiple ChatGPT functions together to create sophisticated micro-apps within workflows - like the voice call analysis example at 15:47 in the video.

Knowledge Bases vs Prompt Engineering

The most significant architectural difference lies in how each approach handles information. Conversational agents use dedicated knowledge bases that can be updated independently of the agent logic.

ChatGPT functions require all knowledge to be embedded in prompts, making maintenance more challenging. As shown at 6:18, updating an agent's knowledge base takes seconds, while modifying ChatGPT functions may require editing multiple workflow steps.

This makes conversational agents better for:

  • Frequently changing information (pricing, availability)
  • Large reference materials (product manuals, policies)
  • Multi-source knowledge aggregation

Model Flexibility Differences

While ChatGPT functions primarily use OpenAI models, conversational agents offer more flexibility. Voice AI agents can currently use Claude and Gemini models, with text-based options coming soon.

This model diversity provides:

  1. Fallback capabilities - Automatic switching when primary models underperform
  2. Cost optimization - Matching model complexity to task requirements
  3. Specialized performance - Using Claude for analytical tasks, Gemini for creative ones

The video demonstrates how this flexibility creates more robust automation solutions compared to single-model ChatGPT functions.

  • 50+ daily conversations
  • Knowledge-base intensive interactions
  • Complex multi-step engagements

  • Low-volume applications

Cost saver: Use ChatGPT functions for internal workflows and conversational agents for customer-facing interactions to optimize your AI spend.

Reporting Capabilities Compared

Conversational agents provide native analytics on:

  • Engagement metrics
  • Message volume per contact
  • Channel performance
  • Goal completion rates

ChatGPT functions require custom reporting implementation through:

  • Additional workflow steps
  • External analytics integration
  • Manual data collection

This makes conversational agents preferable for scenarios requiring detailed performance tracking and optimization.

Key Takeaways

Implementing AI effectively in HighLevel requires matching the tool to the task. Conversational agents handle complex, knowledge-intensive interactions while ChatGPT functions excel at simple, repetitive workflow tasks.

In summary: Use conversational AI agents for customer-facing interactions requiring memory and knowledge bases. Use ChatGPT functions for internal workflows and simple automations. Combine both approaches for sophisticated solutions.
