Prerequisites

Before beginning the setup, ensure you have these components ready:

  • Access to a virtual private server (VPS) - we'll use Hostinger as our example but any provider works
  • A working n8n instance (either on the same VPS or accessible from it)
  • Terminal access to your VPS (either direct or via SSH)
  • A Claude Code account or API key

At 2:15 in the video, the instructor demonstrates how to access the terminal in Hostinger - if you're using a different provider, you may need to create SSH keys and log in from your local terminal instead.

Installing Claude Code on Your VPS

The first technical step is installing Claude Code on your VPS. The process varies slightly depending on your operating system:

Step 1: Access Your Terminal

In Hostinger, open the terminal from your dashboard. Other providers may require SSH access from your local machine.

Step 2: Run the Installation Command

For Ubuntu (as shown at 3:40 in the video), use:

 curl -L "https://claude-code.ai/install.sh" | sh

Windows and other Linux distributions have different commands available in Claude's official documentation.

Step 3: Complete Initial Setup

After installation, run claude to start the setup wizard. Choose dark/light mode preference and authenticate using either:

  • Your Claude account (generates a browser authentication link)
  • API key (if you prefer not to link your account)

Pro tip: The authentication code shown at 5:15 is single-use and expires quickly - have your browser ready before starting this step.

Creating SSH Credentials for n8n

To allow n8n to execute Claude commands, we need to create secure SSH credentials:

Step 1: Generate SSH Keys

Run this command in your terminal (shown at 6:50):

 ssh-keygen -t ed25519

When prompted, create a secure passphrase and confirm it. This generates public and private keys in your ~/.ssh directory.

Step 2: Add Public Key to Authorized Keys

Navigate to your SSH directory and append the public key:

 cd ~/.ssh cat id_ed25519.pub >> authorized_keys

Step 3: Create n8n Credential

In n8n, create a new SSH Private Key credential with:

  • Host: Your server's hostname (find with hostname -I)
  • Username: Typically "root" or check with whoami
  • Private Key: Contents of your id_ed25519 file
  • Passphrase: The one you created earlier

At 9:30, the video shows exactly where to paste each value in n8n's interface.

Password Authentication Alternative

If you prefer not to use SSH keys, you can create a password-based credential instead:

Step 1: Create SSH Password Credential

In n8n, select "SSH Password" when creating a new credential and provide:

  • Host: Same as before (hostname -I result)
  • Username: Your VPS username
  • Password: Your VPS login password

Step 2: Test Both Credentials

The video demonstrates at 11:20 that both credential types should work, though SSH keys are more secure. Choose the method that best fits your security requirements.

Security note: While password authentication is simpler to set up, SSH keys provide stronger security for production environments. Consider your use case when choosing.

The Crucial Symbolic Link Step

At 12:45, the instructor performs the most critical - and often overlooked - step: creating a symbolic link to make Claude accessible to n8n's non-interactive sessions.

Step 1: Find Claude Installation Path

Run which claude to locate where Claude is installed (typically /usr/local/bin/claude).

Step 2: Create the Symbolic Link

Execute this command (shown at 13:10):

 ln -s /usr/local/bin/claude /usr/bin/claude

This creates a shortcut that n8n can access during workflow execution.

Without this step, your n8n workflows will fail because the SSH node won't be able to find the Claude command in its limited environment path.

Testing the Integration

With everything set up, let's verify the integration works:

Step 1: Create Test Workflow

In n8n, create a new workflow with an SSH node connected to your credential.

Step 2: Configure Simple Claude Command

As shown at 14:30, use this command format:

 claude -p "Hello, how are you doing?"

The -p flag tells Claude to print the response directly.

Step 3: Execute and Verify

Run the workflow - you should see Claude's response in the node output. At 15:00, the video demonstrates a successful response appearing in n8n.

Troubleshooting tip: If you get errors, first verify Claude works in your terminal directly, then check your SSH credentials in n8n are correct, and finally confirm you created the symbolic link properly.

Advanced Usage & Conversation Sessions

Beyond simple commands, you can leverage Claude's full capabilities in n8n:

Ongoing Conversations

Use the -r flag to maintain conversation context across executions:

 claude -r "Continue our discussion about AI ethics"

Agent Creation

Create specialized sub-agents for different tasks in your workflows:

 claude --agent "SEO content analyzer" -p "Review this meta description"

Exploring All Options

View all available arguments with:

 claude --help
