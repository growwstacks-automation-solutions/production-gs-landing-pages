What Are Agent Teams in Claude Opus 4.6?

Traditional AI coding assistants work like a single developer - sequentially tackling backend logic, then frontend design, then testing. Claude Opus 4.6 changes this paradigm completely with agent teams that work in parallel, each with specialized roles and responsibilities.

The breakthrough comes from Anthropic's ability to spin up completely separate Claude Code instances that maintain their own context while coordinating through a central control plane. This means you can have:

Specialized agents: Backend developers, frontend designers, security reviewers, and even debate-focused agents arguing over design directions - all working simultaneously in their own terminal sessions.

Unlike previous versions where sub-agents shared context, these are independent instances that must document their decisions in the shared claude.md file. This creates accountability and visibility into the development process.

How to Set Up Agent Teams

Setting up agent teams requires a few configuration steps to unlock this experimental feature:

  1. Edit settings.json: Located in your .claude directory, add the agent teams configuration shown in the documentation
  2. Install T-Max: This terminal multiplexer helps visualize multiple agent sessions (brew install t-max on Mac)
  3. Use iTerm 2: The recommended terminal emulator for managing agent team windows

At 2:15 in the video, you can see the exact configuration being added to enable agent teams. The key advantage of this setup is being able to watch all your specialized agents working in their own dedicated terminal sessions while coordinating through the main Claude Code instance.

Building a Global Tide App With Agent Teams

The demo shows how agent teams can build a complete application from scratch. With one prompt requesting a global tide and sea temperature web app, Claude Opus 4.6 created three specialized agents:

  • Backend Developer: Focused on integrating global tide APIs and temperature data
  • Frontend Developer: Built a responsive dark mode UI with data visualization
  • Skeptic Reviewer: Played devil's advocate on security and UX without writing code

What's remarkable is how these agents coordinate. The backend and frontend developers must get plan approval from the skeptic before writing any code. All decisions get documented in claude.md for visibility. The result was a fully functional tide tracking app that worked globally - from California to England to Cyprus - with accurate tide graphs and temperature displays.

How the claude.md File Coordinates Agents

The claude.md file serves as the central nervous system for agent teams coordination. Every agent documents:

  • Research findings and API choices
  • Design decisions and alternatives considered
  • Skeptic reviews and security concerns
  • Approval workflows between agents

Installing glow (brew install glow) provides beautiful markdown rendering of this file. At 6:15 in the video, you can see how the skeptic agent documented blocking issues, strongly recommended fixes, and nice-to-have improvements - creating a complete audit trail of the development process.

Recommended Tools for Managing Agent Teams

While agent teams work in standard terminals, these tools enhance the experience:

  1. iTerm 2: Provides better window management for multiple agent sessions
  2. T-Max: Terminal multiplexer that visualizes agent coordination
  3. Glow: Renders the claude.md documentation beautifully
  4. VS Code: For reviewing and extending the generated code

The combination creates a development environment where you can watch specialized AI agents work in parallel while maintaining visibility into their decision-making process through the shared documentation.
