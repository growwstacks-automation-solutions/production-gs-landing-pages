Most companies deploy AI agents with generic settings that feel robotic and impersonal. The secret? Agent Settings Overview

Salesforce's Agent Force Builder provides a centralized control panel for all AI Agent configurations. Unlike generic chatbot builders, this specialized interface gives you precise control over how your AI interacts with customers across every touchpoint.

The left-hand panel contains seven key sections (timestamp 1:15 in video): Topics for managing conversation flows, Data for knowledge base integration, Connections for channel setup, Context for variables management, Language for multilingual support, Events for conversation logging, and Checklist for deployment readiness.

Pro Tip: Before adjusting settings, use Conversation Preview to test how changes affect actual interactions. This live testing environment lets you validate adjustments immediately without affecting production.

Language & Tone Configuration

Nothing breaks customer trust faster than an AI that sounds robotic or culturally tone-deaf. Salesforce solves this with granular language and tone controls that maintain brand consistency across all conversations.

In the Language settings (timestamp 3:45), first set your default language - this serves as the baseline for all responses. Then add secondary languages using the intuitive picker interface. For our example with Opal Media, we enabled French, German and Japanese to serve international clients.

The tone selector (casual, neutral, or formal) is equally powerful. A casual setting makes the agent more conversational ("Hey there! How can I help?"), while formal uses more professional language ("Good day. How may I assist you?"). Match this to your brand voice guidelines.

Event Logs for Troubleshooting

Most companies deploy AI agents blindly, with no visibility into why conversations succeed or fail. Salesforce's enhanced event logs (timestamp 6:20) provide unprecedented transparency into your agent's decision-making process.

When enabled during agent creation, these logs capture the complete conversation transcript along with metadata like response timestamps and - most importantly - the reasoning context behind each decision. This is troubleshooting gold when responses miss the mark.

Critical Insight: Event logs reveal not just what the agent said, but why it chose that response. This lets you identify gaps in instructions, knowledge base inaccuracies, or context variables that need adjustment.

Custom Welcome & Error Messages

First impressions matter - especially with AI. Generic "Hello, how can I help?" greetings feel impersonal, while technical error messages frustrate customers. Salesforce lets you customize both in Agent Force Studio (timestamp 8:10).

The welcome message (800 character limit) should accomplish three things: Identify as AI, explain your purpose, and invite engagement. For example: "Hi, I'm Opal Media's AI event assistant. I can answer questions about venues, catering, and logistics, or connect you with our team. What brings you here today?"

Error messages should be human and helpful. Instead of "Input not recognized," try "Hmm, I didn't quite get that. Could you try rephrasing? Here are examples of what I understand: [sample queries]." This maintains the conversation flow.

Data 360 Requirement

One critical limitation often surprises administrators: using knowledge articles or documents as data sources requires Data 360 (timestamp 10:30). This Salesforce feature must be enabled in your org before the agent can reference uploaded content.

Without Data 360, your agent can only use pre-configured topics and flows - severely limiting its knowledge base. The implementation team should verify Data 360 status during planning to avoid last-minute roadblocks.

Implementation Tip: If Data 360 isn't available, consider using Salesforce CMS content instead, which doesn't require this license. The trade-off is less structured knowledge management.

Frequently Asked Questions

Common questions about Salesforce AI Agent configuration

The most important settings to configure are language selection (default and additional languages), conversation tone (formal, neutral, or casual), event logging for troubleshooting, and custom welcome/error messages.

These settings ensure your AI Agent communicates in your brand voice and provides visibility into conversations. Language and tone affect every customer interaction, while event logs help you continuously improve performance.

  • Language: Set default + additional languages
  • Tone: Formal, neutral, or casual brand voice
  • Event Logs: Enable for conversation visibility

In the Agent Force Builder left panel, click Language settings. Set your default language first, then use the language picker to select additional languages.

Move languages to the Selected column to activate them. The agent will automatically detect and respond in the customer's preferred language from your enabled options. Test each language with native speakers before deployment.

  • Default language serves as baseline
  • Use picker to add secondary languages
  • Test with native speakers pre-launch

Enhanced event logs provide detailed conversation transcripts including user prompts, timestamps, agent responses, and the reasoning context behind each decision.

This visibility is crucial for testing and troubleshooting agent behavior before deployment and for continuous improvement after launch. You can see exactly where conversations go off track and adjust instructions accordingly.

  • See full conversation transcripts
  • View agent reasoning context
  • Identify and fix response gaps

Welcome messages are configured in Agent Force Studio (not Builder). Open your agent details, select System Messages, and edit the welcome message field (800 character limit).

Keep it friendly, transparent about being an AI, and clear about how you can help. Example: "Hi, I'm your AI event planning assistant. I can answer questions about venues, catering, and logistics. What would you like to know?" Test different versions to optimize engagement.

  • 800 character limit
  • Be transparent about being AI
  • Clearly state how you can help

Using knowledge articles or other documents as data sources for your AI Agent requires Data 360 to be enabled in your Salesforce org.

This feature allows the agent to reference and ground responses in your official company knowledge base for more accurate, trustworthy answers. Without Data 360, you're limited to pre-configured topics and flows rather than dynamic knowledge base integration.

  • Data 360 license required
  • Enables knowledge base integration
  • Alternative: Use Salesforce CMS content

The tone setting (formal, neutral, or casual) shapes the language style of all agent responses. A formal tone uses more professional language, casual is more conversational and friendly, while neutral strikes a balance.

Choose the tone that best matches your brand voice and customer expectations. Financial services might prefer formal ("How may I assist you?"), while lifestyle brands often choose casual ("Hey there! What's up?"). Test different tones with your audience.

  • Formal: Professional language
  • Casual: Conversational style
  • Neutral: Balanced approach

Error messages should be human, helpful, and guide the customer forward. Avoid technical jargon like "input not recognized" which frustrates users.

Instead, use friendly language that maintains the conversation flow: "Hmm, I didn't quite get that. Could you try asking in a different way?" or "Let me try that again - could you give me more details about what you need?" Keep it brief while maintaining your brand voice.

  • Avoid technical error codes
  • Guide user to rephrase
  • Maintain brand voice
