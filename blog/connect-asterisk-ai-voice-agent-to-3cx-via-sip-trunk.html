Why Connect 3CX to an AI Voice Agent?

Business phone systems often become bottlenecks during peak hours, leaving customers waiting or requiring expensive staffing solutions. An AI voice agent integrated via SIP trunk can handle routine inquiries instantly while seamlessly transferring complex cases to human agents.

Prerequisites for SIP Trunk Setup

Before configuring the connection, you'll need admin access to both systems and basic networking knowledge. The two servers must be able to communicate over your network or the internet.

Essential requirements include:

  • A running Asterisk server (FreePBX recommended) with AI agent configured
  • 3CX PBX with admin privileges
  • Static IP or DNS for both servers
  • Open SIP ports between the systems (default 5060)

FreePBX Trunk Configuration

The first configuration happens on the Asterisk/FreePBX side. You'll create an inbound SIP trunk that will accept connections from your 3CX system.

Follow these steps:

Step 1: Create New SIP Trunk

Navigate to Connectivity → Trunks in FreePBX and create a new SIP trunk named "from-3CX" or similar.

Step 2: Configure Peer Details

Set the peer type to "friend" and configure authentication with a username and secret. The username will be used later in 3CX configuration.

Step 3: Set Context

Modify the context to "from-internal" or similar to route calls properly through your dialplan.

Pro tip: Use a dedicated extension range (like 3000-3099) for AI-routed calls to simplify management.

3CX Trunk Configuration

With the Asterisk side ready, switch to your 3CX admin console to create the outbound trunk connection.

Key configuration points:

  • Select "Generic SIP Provider" as the trunk type
  • Use the same authentication credentials as configured in FreePBX
  • Enter the Asterisk server's IP and SIP port
  • Configure appropriate outbound rules to route calls to the trunk

At 4:30 in the video, you can see the trunk successfully registering with a green status indicator in 3CX.

Setting Up Call Routing

The magic happens in the call routing configuration. You'll need to decide which calls should go to the AI agent versus your regular extensions.

Common routing strategies:

  • Route specific DID numbers to AI for after-hours support
  • Send calls that wait too long in queue to AI as fallback
  • Create dedicated AI extensions for specific functions (like 3000 in the demo)

Implementation note: The demo shows a simple extension 3000 routing, but you can create sophisticated rules based on time, caller ID, or IVR selections.

AI Voice Agent Integration

The Asterisk side needs proper dialplan configuration to connect incoming calls to your AI agent. This typically involves:

  • A custom extension with AGI or Stasis application
  • Proper context routing in extensions_custom.conf
  • Integration with your chosen AI platform (Google Gemini in the demo)

Testing the Connection

After completing both sides of the configuration, verify everything works end-to-end:

  1. Check that the trunk shows green/registered in both systems
  2. Make a test call from 3CX to your AI extension
  3. Verify the AI responds correctly to voice prompts
  4. Check call quality and latency
