Step 1: Install Claude Code

Getting Claude Code running takes just one terminal command. Unlike complex AI setups that require GPU servers or cloud configurations, Claude runs locally on your machine with minimal requirements.

At the 0:45 mark in the video, you'll see the exact command to install Claude. Simply copy this into your terminal:

 curl -sSL https://install.claude.ai | bash

Once installed, you can start Claude by typing claude in your terminal. The AI will be ready to accept your n8n workflow instructions immediately after setup.

Step 2: Get Your n8n API Key

For Claude to interact with your n8n instance, you need to provide an API key. This is like giving Claude a secure access card to your automation environment.

Navigate to your n8n settings (shown at 1:15 in the video) and create a new API key. Be sure to:

  • Name the key clearly (e.g., "Claude Integration")
  • Set appropriate permissions (start with full access for testing)
  • Copy the key immediately - it won't be shown again

Security tip: For production use, create API keys with limited scope that only allow access to specific workflow folders. You can always delete and regenerate keys if needed.

Step 3: Connect Claude to n8n

With Claude running and your API key ready, the actual connection takes seconds. At 1:30 in the video, you'll see the simple command structure:

 claude connect-n8n --api-key YOUR_KEY --url YOUR_N8N_URL

Once connected, you can start describing workflows in natural language. For example:

 claude "Create a workflow that checks for new Shopify orders every hour, adds them to Airtable, and sends a Slack notification for orders over $100"

Claude will generate the complete workflow in your n8n instance, complete with all necessary nodes, connections, and error handling.

Security Best Practices

While powerful, AI access to your automation environment requires proper safeguards. Follow these security measures:

  • Use temporary API keys that expire after 24 hours for testing
  • Limit permissions to specific workflow folders when possible
  • Review all generated workflows before activating them
  • Monitor API usage in your n8n instance logs
  • Never share API keys in public forums or chat
