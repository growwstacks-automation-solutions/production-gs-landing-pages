Prerequisites for Excel-n8n Connection

Before connecting Excel to n8n, you'll need three key elements in place. Missing any one will prevent the automation from working properly.

First, your Excel file must be stored in OneDrive or SharePoint - locally stored files won't work with the API. Second, organize your data in proper Excel tables rather than loose cells for more reliable automation. Third, you'll need admin access to a Microsoft 365 tenant to create the necessary app registration.

  • Excel file in OneDrive/SharePoint (not local)
  • Data organized in Excel tables
  • Microsoft 365 admin access

Step 1: Microsoft Azure App Setup

The connection between Excel and n8n requires creating an app registration in Microsoft Azure. This establishes secure OAuth2 authentication between the systems.

Navigate to the Microsoft Azure portal and select "App registrations" from the Azure Active Directory menu. Create a new registration with a descriptive name like "Excel-n8n Automation". Under platform configuration, select "Web" and enter the redirect URI shown in n8n's OAuth2 setup (typically your n8n instance URL plus "/oauth2/callback").

Critical step: After creating the app, you must add API permissions for Microsoft Graph - specifically Files.ReadWrite and Sites.ReadWrite.All. These permissions require admin consent to function properly.

Detailed Azure Setup Steps:

  1. Log in to portal.azure.com
  2. Navigate to Azure Active Directory → App registrations
  3. Click "New registration"
  4. Name your app (e.g., "Excel-n8n Automation")
  5. Select account type (usually "Accounts in this organizational directory only")
  6. Add redirect URI from n8n OAuth2 setup
  7. Register the application
  8. Navigate to API permissions → Add permission → Microsoft Graph
  9. Add delegated permissions for Files.ReadWrite and Sites.ReadWrite.All
  10. Click "Grant admin consent"

Step 2: Configure n8n OAuth2 Credentials

With the Azure app created, you now need to configure n8n to authenticate with Microsoft 365. This involves setting up OAuth2 credentials in n8n using information from your Azure app registration.

In n8n, navigate to Credentials and create a new OAuth2 API credential. You'll need to enter several values from Azure: Tenant ID (found in Azure AD Overview), Client ID (from your app registration), and a Client Secret (created under Certificates & secrets in Azure). The authorization and token URLs follow a standard Microsoft format with your tenant ID inserted.

Key Configuration Values:

  • Grant Type: Authorization Code
  • Authorization URL: https://login.microsoftonline.com/{tenantId}/oauth2/v2.0/authorize
  • Token URL: https://login.microsoftonline.com/{tenantId}/oauth2/v2.0/token
  • Scope: Files.ReadWrite Files.ReadWrite.All Sites.ReadWrite.All
  • Auth URI Query Parameters: response_mode=query

After entering all values, test the connection by clicking "Connect my account" and authenticating with a Microsoft 365 account that has access to your target Excel files.

Step 3: Reading Excel Data in n8n

With authentication configured, you can now create workflows that interact with Excel data. The simplest starting point is reading data from an Excel table.

Create a new workflow in n8n and add an Excel node. Select "Get rows from sheet" as the operation and choose the credential you just created. You'll need to specify the workbook, sheet, and table containing your data. When executed, this node will return all rows from the specified table in JSON format, ready for processing in subsequent workflow steps.

Pro tip: Enable the "Use Schema" option when configuring the Excel node. This automatically detects column data types and ensures consistent formatting when processing the data later in your workflow.

Reading Excel Data Steps:

  1. Create new workflow in n8n
  2. Add Excel node
  3. Select "Get rows from sheet" operation
  4. Choose your OAuth2 credential
  5. Select workbook, sheet, and table
  6. Set return limit (or leave blank for all rows)
  7. Enable "Use Schema" option
  8. Execute node to test connection

Step 4: Updating Excel Sheets Automatically

The real power comes when you start writing data back to Excel automatically. Common use cases include appending new records, updating existing ones, or maintaining dashboards and reports.

Add another Excel node to your workflow, this time selecting "Append rows to table" as the operation. Configure it to target the same workbook and table as your read operation. In the data section, you can either hardcode values for testing or map dynamic data from previous workflow steps. When executed, this will add new rows to your Excel table with the specified data.

Important note: Always test with small data samples first. While n8n handles bulk operations well, it's good practice to verify your workflow with 5-10 records before scaling up to thousands.

Updating Excel Steps:

  1. Add Excel node to existing workflow
  2. Select "Append rows to table" operation
  3. Choose same credential, workbook, and table
  4. Configure data fields (static or dynamic)
  5. Execute node to test updates
  6. Verify changes in Excel online

Common Mistakes to Avoid

When first connecting Excel to n8n, several common pitfalls can derail your automation efforts. Being aware of these upfront saves troubleshooting time later.

The most frequent issues stem from authentication problems - either missing admin consent for permissions or incorrect credential configuration in n8n. Other common mistakes include using locally stored Excel files instead of of OneDrive/SharePoint, or working with unstructured data rather than proper Excel tables.

Troubleshooting tip: If your connection fails, first verify these four elements in order: 1) Admin consent granted in Azure, 2) Correct tenant ID in n8n, 3) File stored in OneDrive/SharePoint, 4) Data in Excel tables.

Top Mistakes List:

  • Missing admin consent for permissions
  • Incorrect tenant ID in n8n configuration
  • Excel file stored locally instead of cloud
  • Working with cell ranges instead of tables
  • Expired client secrets (they last 6 months by default)
