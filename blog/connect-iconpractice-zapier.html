API Key Security Best Practices

IconPractice's API keys provide powerful access to your patient data, making security paramount. These keys act like master passwords that can read and sometimes write data in your system. Treat them with the same care you would your most sensitive credentials.

Before creating your first key, establish clear protocols for your practice:

  • Only super admins should have API key creation privileges
  • Each user needing Zapier access requires their own key
  • Keys should be named descriptively (e.g. "Zapier - Billing Team")
  • Implement a quarterly key rotation schedule

Step 1: Generate Your API Key in IconPractice

Creating your API key is the foundation for all Zapier integrations. Follow these steps carefully:

Step 1.1: Access Super Admin

Navigate to the super admin section of IconPractice. If you don't have access, you'll need to request permissions from your practice owner or IT administrator.

Step 1.2: Create New Key

Go to API Keys → Create New Key. Give it a descriptive name like "Zapier Integration" and optionally add details about its intended use in the description field.

Step 1.3: Assign to User

Select your username from the dropdown (or the appropriate team member's name if creating keys for others). Remember - API keys are user-specific for security auditing.

Pro Tip: Create a dedicated "Automation" user in IconPractice specifically for API connections. This separates automated actions from individual user activity logs.

Step 2: Create the Zapier Connection

With your API key copied from IconPractice, you're ready to establish the Zapier connection:

Step 2.1: Add New Connection

In your Zapier account, navigate to App Connections → Add Connection. Search for "IconPractice" and select the correct app from the dropdown.

Step 2.2: Enter Credentials

Paste your API key into the designated field and enter the username associated with that key. Click "Continue" to authenticate.

Step 2.3: Test Connection

Zapier will verify the connection works. Once confirmed, name your connection (e.g. "IconPractice - Main Office") for easy identification when building zaps.

HIPAA Compliance Considerations

While Zapier offers HIPAA-compliant plans, maintaining compliance requires careful configuration:

Protected Health Information (PHI)

Avoid automating workflows that transmit full medical records or sensitive PHI unless you've signed Zapier's Business Associate Agreement (BAA) and configured appropriate security measures.

Data Minimization

Only share the minimum necessary data in each zap. For example, patient onboarding might only need name, contact info, and appointment date - not full medical history.

Audit Trails

Regularly review zap logs and IconPractice audit trails to ensure data is being handled appropriately. Zapier's activity history provides 30 days of logs by default.

Compliance Tip: Consider using pseudonymized data in automations where possible, replacing names with patient IDs until human review.

Troubleshooting Connection Issues

If your IconPractice-Zapier connection stops working:

First Checks

  • Verify the API key is still active in IconPractice
  • Confirm the associated user account hasn't been deactivated
  • Check Zapier's status page for outages

Reauthentication

Sometimes connections need refreshing. In Zapier:

  1. Go to App Connections
  2. Find your IconPractice connection
  3. Click "Reauthenticate"
  4. Enter credentials again

When to Create New Key

If reauthentication fails:

  1. Generate a new API key in IconPractice
  2. Update the connection in Zapier
  3. Revoke the old key
