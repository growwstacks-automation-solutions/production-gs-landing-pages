What is Model Context Protocol (MCP)?

Model Context Protocol (MCP) is an emerging standard that enables secure communication between AI systems like Claude and external APIs. It solves the challenge of connecting language models to business systems while maintaining proper authentication and data formatting.

Before MCP, connecting AI to custom APIs required extensive custom coding and posed significant security risks. Now, MCP Builder provides a no-code platform that automatically generates compliant MCP servers from your existing API specifications.

Key benefit: MCP handles all the complex translation between Claude's natural language processing and your API's technical requirements, including OAuth 2 authentication flows, request formatting, and response parsing.

Prerequisites for Connecting to Claude

Before starting the integration, you'll need a few key components in place. First, you must have a paid Claude Team plan - the free version doesn't support custom API connections. Second, your API needs proper OAuth 2 configuration with a discovery endpoint.

At 0:45 in the video tutorial, we see the specific requirements: an OpenAPI specification for your API (Swagger works too), valid OAuth 2 credentials from your provider (like Google), and admin access to your Claude organization settings.

  • Claude Team or Enterprise plan
  • API with OpenAPI/Swagger documentation
  • OAuth 2 provider credentials (client ID + secret)
  • MCP Builder account (free tier available)

Step 1: Create Your MCP Server

The first step is building your MCP server using MCP Builder. This server will act as the secure bridge between Claude and your API. As shown at 1:10 in the video, you start by creating a new project and giving it a descriptive name.

MCP Builder's magic comes from its ability to automatically generate tools from your API documentation. Simply upload your OpenAPI specification, and the platform will parse all available endpoints. You then select which operations you want to expose to Claude.

Pro tip: Only expose the minimum endpoints needed. At 1:35, we see how to selectively choose just the pet store API's GET operations while leaving other endpoints protected.

Step 2: Configure OAuth 2 Security

Security is critical when connecting AI to your business systems. At 2:00 in the tutorial, we configure Google OAuth 2 using the discovery URL method that Claude supports. This ensures all API calls will be properly authenticated.

The MCP server automatically detects the authorization requirements from your API spec. You just need to provide the OAuth 2 discovery URL and required scopes. The platform handles generating the proper security definitions and callback endpoints.

  1. Navigate to Security settings in MCP Builder
  2. Select OAuth 2 discovery method
  3. Enter your provider's discovery URL
  4. Add required scopes (separated by spaces)
  5. Save configuration

Step 3: Connect to Claude

With your MCP server running (shown at 2:30), it's time to connect it to Claude. In Claude's settings under Connectors, you'll add a new custom connector. This is where you'll paste your MCP server URL and provide the OAuth client credentials.

The connection process establishes a secure channel between Claude and your MCP server. Claude will use the discovery URL to understand the available endpoints and required authentication flows. Once connected, the integration appears in Claude's list of available tools.

Important: As mentioned at 2:50, Claude requires explicit user confirmation before making any API calls through your MCP server. This safety measure prevents unintended actions.

Step 4: Test Your API Integration

The moment of truth comes at 3:10 when we test the integration by asking Claude for the list of pets. Claude prepares the API call, shows exactly what data it will request, and waits for user confirmation before proceeding.

After approval, Claude executes the call through your MCP server, which handles the OAuth 2 token exchange and API request formatting. The response comes back through the same secure channel, with Claude interpreting and presenting the results naturally.

  • Start with simple read operations
  • Verify Claude shows the correct request structure
  • Check that responses are interpreted accurately
  • Gradually test more complex operations

Security Considerations

While MCP provides a secure connection method, you should still follow API security best practices. Limit the scopes granted to the minimum necessary, and carefully consider which endpoints you expose to Claude.

At 3:40, the tutorial emphasizes that language models can make mistakes, so you should never expose sensitive write operations or endpoints that could cause damage if misused. Implement proper rate limiting on your API and monitor usage patterns.

Security checklist: Use least-privilege OAuth scopes, enable Claude's confirmation requirement, monitor API logs, and regularly review which endpoints are exposed through your MCP server.
