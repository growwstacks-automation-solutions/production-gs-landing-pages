1 billion+ devices can now trigger n8n workflows through PubNub's infrastructure, enabling real-time automations at unprecedented scale.

Setting Up PubNub Credentials

The first step is creating API credentials in n8n to authenticate with PubNub. This allows your workflows to securely communicate with PubNub's global network.

You'll need your existing PubNub publish and subscribe keys, which you can copy from your PubNub dashboard. In n8n, create new credentials for the PubNub API and paste these keys into the appropriate fields.

Pro Tip: Use descriptive names for your credentials (like "Production PubNub" or "Test Environment") to easily identify them when building multiple workflows.

Configuring the Trigger Node

With credentials set up, you can now add a PubNub trigger node to your workflow. This node listens for specific events in your PubNub channels and triggers your automation when they occur.

Key configuration options include:

  • Channel name: The PubNub channel to monitor for messages
  • Filters: Conditions to process only specific messages (like excluding your own messages with meta.n8n ≠ self)
  • Execution settings: How frequently to check for new messages

At the 1:15 mark in the video tutorial, you can see how to test the trigger node by sending a sample message and verifying the execution succeeds.

Adding the Action Node

The PubNub action node lets your workflow publish messages back to PubNub channels. This creates two-way communication between your automations and connected devices.

When configuring the action node:

  1. Select the same credentials used for the trigger node
  2. Specify the target channel for your messages
  3. Define the message content (which can include dynamic data from previous nodes)
  4. Set any optional metadata for the message

As shown at 1:45 in the video, you can test the action node by executing the step and verifying the message appears in your PubNub debug console.

Testing the End-to-End Workflow

The real power comes when connecting trigger and action nodes into a complete workflow. Here's how to test the full integration:

  1. Start the workflow execution (it will wait for incoming messages)
  2. Send a test message from your PubNub debug console or application
  3. Verify the trigger node processes the message
  4. Check that the action node responds with the expected message

At 2:20 in the tutorial, you can see this end-to-end test in action, where a "hello" message triggers a workflow that responds automatically through PubNub.
