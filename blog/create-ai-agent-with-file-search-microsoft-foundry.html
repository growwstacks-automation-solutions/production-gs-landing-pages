Getting Started with Agent Creation

Creating your first AI agent in Microsoft Foundry begins in the Azure AI Foundry portal. Navigate to Build and Customize > Agents to access the agent builder interface. The left-hand menu can be minimized to maximize your workspace, crucial when working with multiple panels.

Selecting "New Agent" initiates the setup process on the right panel. Microsoft Foundry automatically generates a unique agent ID, while you provide:

  • A descriptive agent name (we used "Demo Agent")
  • Model deployment selection (GPT-4-0 in our example)
  • Option to create new deployments from base or fine-tuned models

Pro Tip: If your organization has multiple model deployments available, they'll appear in the dropdown. New deployments can take several minutes to provision.

Defining Your Agent's Personality

The instructions field (also called System Prompt) is where you define both the agent's capabilities and its engagement style. For our surf guide, we specified: "You're a surfer who recommends the best places to surf in California" with added personality cues like "stereotypical surfer personality."

This approach yielded responses like: "Oh dude, you're in Malibu? That's like surf heaven my friend!" - complete with regional slang and enthusiastic tone. The description field, while not affecting behavior, helps organize multiple agents in your project.

Key Insight: Personality instructions work best when specific. Vague prompts like "be friendly" produce generic responses, while "use Southern California surfer slang from the 1990s" creates more distinctive character.

Setting Up File Search Knowledge

The real power comes from connecting your agent to reference documents. Our surf spot guide contained:

  • California regions (Northern, Central, Southern)
  • Skill level recommendations
  • Wave characteristics
  • Consistently structured data

File search works by:

  1. Parsing and chunking uploaded documents
  2. Creating vector embeddings
  3. Storing in a specialized vector database
  4. Enabling both keyword and semantic searches

When users ask questions, the agent retrieves relevant document sections to enhance response accuracy. In our demo, this produced recommendations with direct file references visible as hoverable citations.

Testing and Evaluating Agent Responses

The playground interface lets you test agents in real-time. Our "recommend a spot in Southern California" prompt returned:

"Yo dude, if you're paddling into the surf life, Southern California got some killer spots for you..." with specific references to Surf Rider Beach in Malibu and Huntington Beach.

Each response includes valuable metrics:

  • Generation time (3 seconds in our test)
  • Token counts (82 input, 279 output)
  • AI quality scores (4/5 intent resolution, 3/5 task adherence)

The thread logs provide detailed breakdowns of each interaction, including the exact file search queries performed and content referenced.

Understanding Performance Metrics

Microsoft Foundry's evaluation system assesses responses through:

  • Intent Resolution (1-5): How well the output matches the prompt's intention
  • Task Adherence (1-5): How completely it addresses the requested task

Additional evaluators can be enabled:

  1. Coherence
  2. Relevance
  3. Risk/Safety metrics (self-harm, hate speech, etc.)

Note: These evaluations use AI-assisted scoring powered by another LLM, providing consistent assessment standards across all your agents.

Advanced Features and Customization

Beyond basic configuration, Microsoft Foundry agents support:

  • Actions: Connect to external APIs and services
  • Multi-agent workflows: Chain agents together
  • Voice capabilities: Add text-to-speech functionality
  • Model parameters: Adjust temperature and top_p for creativity control

The code-first approach allows developers to:

  1. View automatically generated Python/C#/JavaScript code
  2. Open in VS Code directly
  3. Authenticate using Entra ID
  4. Extend functionality programmatically

Agents can be deleted from the agents list view when no longer needed, with confirmation to prevent accidental removal.
