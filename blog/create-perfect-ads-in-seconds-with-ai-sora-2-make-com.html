Workflow Overview

This Make.com automation solves the ad creation problem through a four-step process that requires zero manual intervention after setup. At 4:30 in the video, you can see the entire flow visualized in Make.com's scenario builder.

  1. Website Extraction: Pulls all content from a given URL
  2. AI Scripting: Transforms raw HTML into a professional ad script
  3. Video Generation: Converts the script into a Sora 2 video
  4. Delivery System: Automatically saves videos to Google Drive

Time saved: What traditionally took 5-7 days of human work now happens in under 30 minutes, with the ability to scale to hundreds of variations.

Step 1: Website to Prompt

The workflow begins with a simple HTTP request module in Make.com that fetches the HTML content from any website URL. This raw data contains all the messaging, value propositions, and calls-to-action that will form the foundation of your ad.

At 5:45 in the video, you can see exactly how this module is configured - no coding required. The key is preserving all the textual content while stripping unnecessary formatting that might confuse the AI in later steps.

Step 2: Prompt to Script

Here's where the magic happens. The website content feeds into a ChatGPT module with a carefully crafted prompt that transforms dry web copy into an emotionally compelling ad script. The prompt establishes several critical frameworks:

  • Role: Positions the AI as an expert ad director
  • Format: Specifies a 10-second video with 3-second hook
  • Components: Requires visual direction, voice-over, and CTA
  • Tone: Guides the emotional resonance (aspirational, urgent, etc.)

At 7:20 in the video, you can see the exact prompt structure that produces professional-grade scripts automatically.

Step 3: Script to Video

The generated script then moves to the Sora 2 video generation module. This is configured to use the SORA2 Pro model for highest quality output. Important settings include:

  • Resolution: 1080x1080 for square social media ads
  • Duration: 10 seconds (leaving 2-second buffer)
  • Timeout: Set to "No" to avoid API failures

Because Sora 2 videos take 10-20 minutes to render, the workflow doesn't wait for completion. Instead, it triggers a separate monitoring process shown in the next step.

Step 4: Delivery System

A second Make.com scenario handles the video delivery. It runs every 15 minutes to check for completed videos, then:

  1. Lists all pending video jobs
  2. Retrieves completed videos by ID
  3. Creates timestamped folders in Google Drive
  4. Uploads the final video files automatically

At 11:30 in the video, you'll see how this creates an organized library of ad assets ready for your marketing team to deploy.
