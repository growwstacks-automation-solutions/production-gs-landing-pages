Workflow Overview

The complete workflow runs on a daily schedule, triggered by n8n's scheduler. Each morning, it checks a Google Sheet for that day's product information, then executes a series of AI-powered transformations:

  1. Pulls product details (name, image URL, description) from Google Sheets
  2. Generates a dramatic scene description using Gemini 3 Flash
  3. Creates a transformed product image using Nano Banana Pro
  4. Generates a 10-second video from the image using VO3.1
  5. Stores all assets in Google Drive with date-based naming
  6. Updates the Google Sheet with generation status and URLs

Unified experience: The workflow can be easily modified to test different models against the same product, allowing businesses to compare outputs and select their preferred style.

Automated Scene Generation

The first AI step creates a dramatic scene description for the video ad. Using Gemini 3 Flash through Atlas Cloud's chat node, the system generates a cinematic prompt showing the product in an "impossible transformation."

The prompt combines the product name and description from the Google Sheet with instructions to create a 10-second video concept. A system prompt defines the AI's role as a creative director specializing in dramatic product showcases. The resulting scene description is added back to the Google Sheet for future reference and potential manual adjustment.

Image & Video Creation

With the scene description ready, the workflow moves to visual generation. The image-to-image node transforms the product's base image (from the Google Sheet URL) using Nano Banana Pro, applying the dramatic scene description to create a visually striking advertisement still.

This image then feeds into the video generation node using VO3.1, creating a 10-second animated version. Both assets are downloaded via HTTP nodes and stored in Google Drive with clear date-based filenames (like "2026-02-09_image.jpg" and "2026-02-09_video.mp4").

Quality control: The workflow maintains all generated assets in Google Drive, allowing for easy review and selection of the best outputs. The date-based naming ensures perfect organization even when running daily for months.

The Game-Changing Wait-for-Completion Node

Traditional automation workflows struggle with AI generation timing - they either wait too long (wasting time) or not long enough (causing failures). Atlas Cloud's wait-for-completion node solves this by actively monitoring generation status.

When the workflow submits an image or video generation task, this node holds execution until Atlas Cloud confirms completion. The node receives a prediction ID from the submission and polls Atlas Cloud until the asset is ready, then returns the output URL automatically. This eliminates all timing-related failures in the workflow.

Storage & Delivery System

The final workflow components handle asset storage and status tracking. Google Drive nodes upload the generated image and video to a dedicated folder ("Atlas Cloud Ads"), while Google Sheets nodes update the original spreadsheet with:

  • The generated scene description
  • Google Drive URLs for the image and video
  • Generation timestamps

This creates a complete audit trail and makes all assets easily accessible to marketing teams. The system can be extended to automatically post videos to social platforms or send email notifications when new ads are ready.
