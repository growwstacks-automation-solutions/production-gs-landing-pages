What Is Sentiment Analysis?

Every business receives customer feedback - through surveys, emails, reviews, and support tickets. Manually reading and categorizing each message is tedious and inconsistent. Sentiment analysis solves this by automatically detecting emotional tone in text.

This technique interprets customer feedback to classify it as positive ("Your cakes are delicious"), neutral ("Package arrived on time"), or negative ("Wrong flavor delivered"). Modern AI tools like Make.com's sentiment analysis module can detect subtle emotional cues that humans might miss during manual review.

85% of businesses using sentiment analysis report faster response times to customer issues and 30% higher satisfaction scores compared to manual review processes.

How LLMs Analyze Sentiment

Large Language Models (LLMs) perform sentiment analysis in three stages. First, they break down text into tokens (words or phrases). Then they analyze relationships between these tokens to understand context. Finally, they assign probability scores to different sentiment categories.

For example, the feedback "Cakes were delicious but delivery was late" contains both positive and negative elements. The LLM recognizes this mixed sentiment by:

  1. Tokenizing the sentence into components
  2. Identifying "delicious" as positive (+0.8) and "late" as negative (-0.7)
  3. Calculating an overall neutral sentiment score

Make.com's AI toolkit simplifies this process - you just pass the text and receive the sentiment classification without needing to understand the technical details.

Step-by-Step Build

Step 1: Set Up Google Forms Trigger

Create a new scenario in Make.com and add the Google Forms "Watch Responses" trigger. Connect your Google account and select the feedback form. This automatically passes new submissions to the workflow.

Step 2: Add Sentiment Analysis

Drag in the Make AI Toolkit's "Analyze Sentiment" module. Configure it to process the feedback text field from the Google Form. The module will output "positive", "neutral", or "negative".

Step 3: Create Auto-Responses

Add another Make AI Toolkit module ("Ask Anything") to generate customized replies. Use a prompt like: "You are a customer service agent. Respond professionally to this [positive/negative] feedback about our bakery: [customer message]".

Step 4: Route Based on Sentiment

Add a router to split the workflow. One path handles positive feedback by sending a Slack notification. The other path uses a filter (only if sentiment = "negative") to create a Gmail draft for manual follow-up.

Pro Tip: Use Make.com's "Lower" function in your filter to catch all negative variations (Negative, NEGATIVE, etc.)
