Voice AI Solution Architecture

The system combines LiveKit for real-time voice processing, a custom AI agent for conversation logic, and AWS EC2 for reliable hosting. When a patient clicks "Talk to our AI assistant" on the website, here's what happens:

  1. Browser establishes WebRTC connection to LiveKit server
  2. LiveKit streams audio to our EC2 instance
  3. AI agent processes speech, determines intent
  4. System queries calendar API for availability
  5. Response audio streams back through LiveKit

The entire exchange happens in under 1.2 seconds, faster than most human receptionists can locate an appointment book.

AWS EC2 Deployment Process

We chose AWS EC2 for its reliability, scalability, and HIPAA-compliant infrastructure options. The deployment process:

Step 1: EC2 Instance Setup

Launched t3.large instance (2 vCPUs, 8GB RAM) with Ubuntu Server 22.04 LTS. This provides enough processing power for concurrent voice sessions while keeping costs under $30/month.

Step 2: Docker Configuration

Containerized the AI agent using Docker for easy deployment and updates. The compose file includes:

  • LiveKit client service
  • AI conversation engine
  • Calendar API connector
  • NGINX reverse proxy

Step 3: Auto-scaling Rules

Configured CPU-based scaling to handle peak call times (typically Monday mornings). The system can scale to 4 instances during busy periods.

HTTPS Security Implementation

Patient data security was paramount. We implemented:

  1. Let's Encrypt SSL certificate for HTTPS encryption
  2. Strict CORS policies limiting access to the clinic website
  3. Audio stream encryption via LiveKit's built-in DTLS
  4. Regular security patching via unattended-upgrades

The system meets HIPAA requirements for data transmission security while maintaining sub-second response times.

Compliance note: All audio streams are ephemeral - no patient conversations are stored permanently on the server.
