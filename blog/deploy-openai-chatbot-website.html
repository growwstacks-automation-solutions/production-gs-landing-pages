Why Guardrails and References Matter

Unfiltered AI chatbots risk generating harmful, off-brand, or fabricated responses. The workflow shown at 4:30 implements two critical production features most tutorials omit:

1. Pre-processing guardrail screens incoming queries for inappropriate content before they reach the AI agent. This prevents the chatbot from engaging with harmful or off-topic requests.

2. Vector search with references connects to a database of your actual content (help articles, product docs, etc.). Answers cite specific source materials rather than generating generic responses.

These features transform a demo into a business-ready tool. An e-commerce site could deploy this to answer product questions while citing official specs. A law firm could create a legal research assistant that references case files.

What You'll Need Before Starting

Deploying requires four components you may not have handy from typical AI tutorials:

  1. Owned domain (cannot use someone else's—OpenAI requires whitelisting)
  2. OpenAI API key with billing enabled (free tier won't work for production)
  3. Vector database pre-loaded with your content (demo uses markdown files)
  4. Vercel account for hosting the chat interface (free tier sufficient)

The tutorial assumes you've already designed your chatbot's workflow in OpenAI Agent Builder. If not, pause here and complete that step first—we're focusing solely on the deployment process.

Workflow Configuration Steps

At 7:45 in the video, we grab two critical IDs from the published workflow:

  1. Code ID: Unique identifier for your specific agent configuration
  2. Version ID: Tracks iterations when you update the workflow

These IDs plus your OpenAI API key form the deployment trifecta. The tutorial shows how to locate them in the Agent Builder interface—they're not the same as your general OpenAI account credentials.

Protip: Create a new API key specifically for this deployment rather than using your master key. This allows revoking access if needed without disrupting other projects.

Vercel Deployment Process

The 9:10 timestamp shows the actual deployment steps:

  1. Fork the chatkit-starter repository (link in video description)
  2. Connect your GitHub account to Vercel
  3. Import the forked repository as a new Vercel project
  4. Add environment variables (API key, code ID, version ID)
  5. Deploy

This generates a live URL for your chat interface. The entire process takes under 5 minutes if you've prepared the required credentials.

Domain Whitelisting in OpenAI

The most commonly missed step shown at 11:30: whitelisting your domain in OpenAI's settings. Without this, your chatbot will block queries from your live site.

  1. Navigate to OpenAI's content moderation settings
  2. Add your production domain (e.g. yoursite.com)
  3. Purge any CDN cache to apply changes immediately

The tutorial demonstrates this process live, including troubleshooting when changes don't appear instantly due to caching delays.

Website Integration Code

Final step copies the iframe embed code from your Vercel deployment and adds it to your website. The tutorial shows adding to a WordPress site, but the same method works with any platform supporting iframes:

 <iframe    src="https://your-vercel-deployment.vercel.app"    width="100%"    height="500"   frameborder="0" ></iframe>

For better UX, consider wrapping the iframe in a toggle button that expands the chat interface when clicked. The tutorial repository includes sample code for this enhancement.
