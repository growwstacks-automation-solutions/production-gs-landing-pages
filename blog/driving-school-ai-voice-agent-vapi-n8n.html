The 3 Tools That Convert Callers to Students

Your Vapi agent uses three custom tools that systematically guide callers from initial inquiry to booked lesson:

1. Qualify Driving Student

Activates after the caller explains their situation. Captures:

  • Experience level: "I just got my learner's permit last week" vs. "I've been driving for years but need highway practice"
  • License status: Learner's permit, no license yet, full license, or international license
  • Lesson goal: Their actual words like "pass my driving test" or "learn parallel parking"

2. Check Availability

Fires after qualification to:

  • Query your Google Calendar for genuine openings (no double bookings)
  • Present available slots based on their preferred times
  • Handle empty calendar days as "fully available" without crashing

3. Book Appointment

Finalizes the booking by:

  • Creating calendar events with student name and selected time
  • Attaching all qualification data so instructors arrive prepared
  • Sending confirmation in the AI's warm, reassuring tone

Key insight: These tools activate in perfect sequence based on conversation flow - just like your best human receptionist would handle the call naturally.

n8n Backend: Calendar Integration & Booking Logic

The n8n workflow acts as the brains behind the operation, with several critical components:

Webhook Foundation

Receives all data from Vapi in structured JSON containing:

  • Tool call ID and name (qualify, availability, or book)
  • All collected student data (experience level, goals, etc.)
  • Caller's phone number for reminders

Switch Node Traffic Control

Routes each request to the correct processing path based on which tool fired. This ensures qualification happens before availability checks, which happen before final booking.

Google Calendar Integration

Two-way sync that:

  • Checks real instructor availability (no phantom openings)
  • Creates events with complete student context attached
  • Handles empty calendar days gracefully (full availability)
