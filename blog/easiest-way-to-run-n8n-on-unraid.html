Step-by-Step Installation

Installing n8n on Unraid is straightforward using the Community Applications plugin. The entire process takes about 5 minutes and requires no command line work:

Step 1: Find the n8n Container

In your Unraid web interface, navigate to the Apps tab and search for "n8n". Select the official container from the linuxserver.io repository.

Step 2: Configure Basic Settings

Most default settings can remain unchanged. Key configurations to verify:

  • Network Type: Bridge mode
  • WebUI Port: Default is 5678 (ensure this port is available)
  • Time Zone: Set to your local timezone (e.g. America/New_York)

Step 3: Fix Permission Issues

After installation, you may need to run one command to fix permissions:

chmod -R 777 /mnt/user/appdata/n8n

This ensures n8n can write to its data directory. The container should now start successfully.

Configuring Reverse Proxy Access

To securely access n8n from outside your home network, we recommend setting up a reverse proxy using Nginx Proxy Manager (NPM):

Step 1: Create DNS Record

Set up a subdomain (like n8n.yourdomain.com) pointing to your home IP address. Consider using a dynamic DNS service if you don't have a static IP.

Step 2: Add Proxy Host in NPM

In Nginx Proxy Manager:

  1. Click "Add Proxy Host"
  2. Enter your subdomain under Domain Names
  3. Set Forward Hostname/IP to your Unraid server IP
  4. Forward Port to 5678 (or your custom n8n port)

Step 3: Enable SSL

Under the SSL tab in NPM:

  • Request a new Let's Encrypt certificate
  • Enable "Force SSL" and "HTTP/2 Support"
  • Save the configuration

You should now be able to access n8n securely via your subdomain from anywhere.

Creating Your First Workflow

After logging into n8n, you'll be greeted with a blank canvas. Let's create a simple form-to-Slack workflow to demonstrate the power of no-code automation:

Step 1: Add Form Trigger

Click the "+" button and select "On Form Submission" from the triggers list. Configure your form fields (name, email, etc.) and mark required fields.

Step 2: Test the Form

Click "Execute Node" to generate a form URL. Open this URL in another tab and submit test data to verify it works.

Step 3: View Submitted Data

Back in n8n, you'll see the submitted form data displayed. Pin this output to use in the next step.

Pro Tip: The form URL remains active as long as n8n is running. For production use, consider adding basic authentication or IP restrictions.

Slack Integration Example

Now let's connect our form to Slack to automatically notify your team of new submissions:

Step 1: Add Slack Action

Click the "+" after your form node and search for "Slack". Select "Send Message" from the actions list.

Step 2: Configure Slack Connection

You'll need to create a Slack app and obtain an API token. The n8n documentation provides step-by-step instructions for this process.

Step 3: Build Your Message

In the message field, drag form fields from the left panel to include submitted data. For example:

New form submission from {{$node["Form"].json["firstName"]}} ({{$node["Form"].json["email"]}})

Step 4: Test the Workflow

Submit the form again and verify the message appears in your Slack channel. The entire process happens automatically!

Common Issues & Troubleshooting

While n8n on Unraid is generally stable, here are solutions to common problems:

Permission Errors

If n8n fails to start, run the chmod command mentioned earlier to fix directory permissions.

Webhook Issues

External services need to reach your n8n instance. Verify:

  • Reverse proxy is correctly configured
  • Port forwarding is set up on your router
  • No firewall is blocking incoming connections

Credential Problems

When connecting to services like Slack:

  1. Always check the n8n documentation for that service
  2. Verify API keys and permissions
  3. Test connections using the "Test" button in credential setup
