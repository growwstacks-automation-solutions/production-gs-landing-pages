Why This Gmail Connection Matters

Email automation can transform your business operations, but many small businesses hit a roadblock when they discover Make.com's Gmail integration seems to require a paid Workspace account. This creates an unnecessary barrier - especially when you're just starting with automation.

The solution lies in Google's developer tools. By creating a simple project in Google Cloud Console, you can enable API access for your free Gmail account. This approach works because:

Key insight: Google's API access isn't restricted to Workspace accounts - it's available to all Gmail users through the developer console. The 6-minute setup process unlocks the same powerful automation capabilities.

Step 1: Create a Google Cloud Project

Begin by visiting console.cloud.google.com. If this is your first time using Google Cloud Console, you'll need to accept the terms of service.

  1. Click on the project dropdown near the top of the page
  2. Select "New Project"
  3. Name your project (e.g., "Make.com Gmail Connection")
  4. Click "Create"

This project will serve as the container for your Gmail API settings. Don't worry about billing - we're using services that fall under Google's free tier.

Step 2: Enable the Gmail API

With your project created, you'll need to enable the specific API that allows Make.com to communicate with Gmail:

  1. Navigate to "APIs & Services" > "Library"
  2. Search for "Gmail API"
  3. Click on the Gmail API result
  4. Click "Enable"

Note: Enabling the API doesn't immediately grant access to your email. You'll still need to complete the OAuth setup process to establish a secure connection.

Step 3: Configure OAuth Consent Screen

The OAuth consent screen is where you authorize which email accounts can access this connection. This is a crucial security step:

  1. Go to "APIs & Services" > "OAuth consent screen"
  2. Select "External" user type (for personal/non-Workspace accounts)
  3. Add your email address under "Test users"
  4. When prompted for scopes, select all available Gmail scopes
  5. Complete the basic app information (you can use "Make.com Integration" as the app name)

This step ensures that only your specified email account(s) can use this API connection, maintaining security while enabling automation.

Step 4: Create OAuth Credentials

Now you'll generate the credentials that Make.com will use to authenticate with your Gmail account:

  1. Navigate to "APIs & Services" > "Credentials"
  2. Click "Create Credentials" > "OAuth client ID"
  3. Select "Web application" as the application type
  4. Add these exact Authorized Redirect URIs:
    • https://integromat.com
    • https://make.com
  5. Click "Create"

After creation, you'll see your Client ID and Client Secret. Copy both - you'll need them in the next step when connecting to Make.com.

Step 5: Connect to Make.com

The final step is to add your credentials to Make.com:

  1. In Make.com, go to "Connections" > "Add new connection"
  2. Search for and select "Gmail"
  3. Choose "Custom OAuth" as the connection method
  4. Paste your Client ID and Client Secret
  5. Click "Save"

Make.com will redirect you to Google's OAuth screen where you'll authorize the connection. Once completed, you can use this connection in any Make.com scenario to automate email sending, inbox monitoring, and more.
