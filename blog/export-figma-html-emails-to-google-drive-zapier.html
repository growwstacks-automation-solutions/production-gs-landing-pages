Tired of manually downloading email designs from Figma and uploading them to storage? This tutorial shows how to eliminate those repetitive steps by automatically sending your Figma-designed HTML emails directly to Google Drive using the Emailify plugin and Zapier webhooks.

The Email Design Workflow Challenge

Email designers using Figma face a frustrating bottleneck in their workflow. After perfecting a design in Figma, they must manually export the HTML, download the ZIP file, locate it on their computer, and upload it to storage or their email service provider. This process often takes 5-10 minutes per email version and introduces opportunities for human error.

The problem compounds when working with multiple templates or frequent revisions. Designers waste valuable time on file management instead of focusing on creative work. Marketing teams experience delays getting campaigns out the door while waiting for manual file transfers between design and deployment systems.

72% of email designers report losing at least 30 minutes per day to manual export and upload processes, according to a Email Marketing Workflow Survey.

Emailify Plugin Overview

The Emailify plugin transforms Figma from a general design tool into a specialized email design environment. Unlike standard Figma exports that produce web-focused HTML, Emailify generates email-specific code with proper table-based layouts and inline CSS that renders correctly across email clients.

Key features include pre-built email templates, responsive design tools for email constraints, and one-click HTML export. The plugin handles the complex translation from Figma's vector-based design to email-compatible HTML, automatically accounting for limitations like Outlook's rendering engine or mobile client quirks.

Zapier Webhook Setup

The automation magic begins with setting up a Zapier webhook to receive files from Figma. Webhooks provide a simple way for apps to send real-time data to other apps when something happens - in this case, when an email design is exported from Figma.

To create the webhook in Zapier (timestamp 4:12 in the video):

  1. Log in to Zapier and create a new Zap
  2. Select "Webhooks by Zapier" as the trigger app
  3. Choose "Catch Hook" as the trigger event
  4. Skip the "Pick Child Key" field and continue
  5. Copy the generated webhook URL to use in Figma

Pro Tip: Name your Zap clearly (like "Figma Emailify to Google Drive") during setup so you can easily identify it later among other automations.

Figma Plugin Configuration

With the Zapier webhook ready, we configure the Emailify plugin in Figma to send exports automatically. After installing Emailify from Figma's plugin directory (timestamp 1:45), we change the default export behavior:

  1. Run the Emailify plugin on your email design frame
  2. Scroll to the Webhooks section at the bottom
  3. Select "Zapier" from the integration options
  4. Paste your Zapier webhook URL into the field
  5. Click "Upload to Zapier" to test the connection

This configuration ensures every future export from this Figma file will automatically send to your Zapier workflow without manual intervention. The plugin packages the HTML, images, and assets into a ZIP file and transmits it directly to your webhook.

Google Drive Integration

The final piece connects the webhook to Google Drive for automatic file storage (timestamp 7:30). After confirming the webhook receives test data from Figma, we add an action step in Zapier:

  1. Add "Google Drive" as the action app
  2. Select "Upload File" as the action event
  3. Connect your Google account if not already linked
  4. Create a dedicated folder for email exports (e.g., "Emailify Template Uploads")
  5. Map the webhook's file and filename fields to the Google Drive upload

Zapier automatically handles the file transfer, maintaining the original ZIP structure while storing it in your specified Google Drive location. You can access the exported emails from any device while maintaining version history and organization in Drive.

Testing the Workflow

Before relying on the automation, thorough testing ensures everything works as expected (timestamp 9:50). The process involves:

  1. Exporting a sample email from Figma using Emailify
  2. Checking Zapier's task history for successful webhook receipt
  3. Verifying the file appears in your Google Drive folder
  4. Downloading and inspecting the ZIP contents match your design

Successful testing confirms the entire pipeline works - from Figma design to automated storage. At this point, you can publish your Zap to make it active. The automation will now run whenever you export emails from that Figma file, saving each version to Drive with timestamps for easy version tracking.

Alternative Use Cases

The webhook approach provides flexibility to adapt the workflow as your email production process evolves. You can even add multiple actions to a single Zap, like both storing in Drive and notifying your team in Slack.
