Critical Prompt Engineering Rules

The entire system hinges on your prompt structure (timestamp 7:30). Our templates include these key components:

Never break these rules: The AI must always get confirmation before submission, handle corrections naturally, and pronounce numbers clearly ("That'll be twenty dollars" not "20$").

Prompt structure:

  1. Role definition: "You are a friendly but efficient order taker for [RESTAURANT]..."
  2. Menu knowledge: Full menu with prices in JSON format
  3. Flow control: Strict order of operations (greet → items → extras → confirmation)
  4. Error handling: "If the customer requests something not on the menu..."
