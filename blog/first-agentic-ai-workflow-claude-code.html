The Agentic Workflow Revolution

Traditional automation requires manually configuring each step - connecting API calls, handling errors, and maintaining logic flows. This approach works well for predictable, repetitive tasks but struggles with processes requiring judgment calls or research. Agentic workflows flip this model by letting you describe what you want while the AI figures out how to achieve it.

Imagine hiring a talented developer versus explaining code line-by-line to a junior programmer. Agentic AI operates like the experienced developer - you describe the problem and desired outcome, then let the system determine the best approach. This enables automation of complex, variable processes that were previously impossible to systematize.

Key difference: Traditional automation follows your exact instructions. Agentic workflows achieve your goals while handling the implementation details.

Understanding the WAT Framework

The WAT framework structures agentic workflows into three components:

1. Workflows (Instructions)

Written in markdown, workflows describe processes like job descriptions or SOPs. For example, a competitor analysis workflow might instruct the agent to: research businesses → gather competitor data → analyze findings → create PDF report.

2. Agents (Coordinators)

The AI (Claude Code) reads workflows, selects appropriate tools, and sequences tasks. It handles errors by researching solutions and updating workflows - functioning like an automated project manager.

3. Tools (Workers)

Python scripts that perform specific actions (scraping websites, generating PDFs). Tools are modular and automatically fixed/improved by the agent when they fail.

Self-improving system: As you provide feedback ("I liked this but change that"), the agent updates workflows and tools to perform better next time.

Setting Up Claude Code

Getting started requires just a few steps:

  1. Install VS Code: The free editor where Claude Code operates
  2. Add Claude Code extension: Available in the VS Code marketplace
  3. Sign in with Anthropic account: Requires Claude Pro or Max subscription
  4. Create project folder: Organizes your workflows and tools
  5. Add claw.md file: Onboarding document that teaches Claude your preferred structure

The setup process demonstrates agentic principles in action. Rather than manually configuring environments, you describe what you need ("initialize the project") and Claude handles the technical implementation.

Building Your First Workflow

We'll create a competitor research workflow that:

  • Discovers competitors automatically
  • Researches their offerings
  • Generates branded PDF reports

Starting in "plan mode," we describe our goal to Claude: "I want a monthly competitor analysis system that creates branded PDF reports." Claude then:

  1. Asks clarifying questions (target market, analysis focus areas)
  2. Proposes architecture and tools needed
  3. Estimates costs (~$1.50 per report)
  4. Creates folder structure and initial files

No coding required: The entire workflow is built through conversation. Claude generates all necessary Python tools automatically.

Testing and Improving the Workflow

Initial runs reveal areas for improvement - like invisible white logos on PDFs. The agentic approach shines here:

  1. Claude detects the visibility issue automatically
  2. Researches PDF generation best practices
  3. Updates the PDF tool to handle logo contrast
  4. Regenerates reports with proper branding

This self-healing capability means workflows improve over time rather than requiring manual maintenance. Subsequent runs become both cheaper (cached data) and higher quality (applied learnings).

Cost efficiency: Initial research costs ~$1.50 while subsequent monthly updates cost just $0.50 by reusing competitor data.

Deterministic automation follows predictable, repeatable steps with consistent outputs. Non-deterministic automation involves variability where the same input might produce different outputs.

Traditional automation tools excel at deterministic tasks, while agentic workflows handle non-deterministic processes by making judgment calls and adapting to variability. AI is inherently non-deterministic, but agentic workflows make these processes more reliable for business applications.

  • Deterministic = predictable outputs
  • Non-deterministic = variable outputs
  • Agentic workflows bridge the gap
