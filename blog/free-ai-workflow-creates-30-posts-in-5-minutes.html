The Content Repurposing Problem

Content creators waste hours manually repurposing videos into social posts, often creating inconsistent messaging across platforms. At 3:15 in the video, the creator shows how most marketers copy-paste the same content everywhere, hurting engagement.

This automation solves three core problems: time-consuming manual transcription, platform-specific formatting requirements, and maintaining brand voice consistency across channels.

83% of marketers report content repurposing as their biggest time sink, according to HubSpot's Content Marketing Report.

How the Automation Works

The workflow connects four key components: Google Sheets for input, RapidAPI for YouTube transcript scraping, Google Gemini AI for content generation, and Google Docs for organized output.

When you paste a YouTube URL into the Google Sheet, Make.com automatically extracts the transcript, sends it to Gemini with specific formatting instructions for each platform, and saves polished posts to a new Google Doc.

Step 1: Video URL Input

The process begins with a simple Google Sheet containing a single column for YouTube URLs. The creator emphasizes at 1:45 that proper header configuration in Make.com's Google Sheets module is critical for reliable data reading.

Key settings include:

  • Table contains headers: Enabled
  • Limit: Set to 1 to process videos one at a time
  • Range selection: Choose "All" to capture the entire URL column

Step 2: Transcript Extraction

At 4:30 in the video, the creator demonstrates connecting to RapidAPI's YouTube transcript service. The free tier provides sufficient capacity for most small businesses.

The HTTP request module requires:

  • X-RapidAPI-Key header with your unique API key
  • X-RapidAPI-Host header copied exactly from RapidAPI
  • GET method with the video URL passed as a query parameter

The text aggregator module then combines the raw transcript into a single block for AI processing.

Step 3: AI Content Generation

The magic happens when Gemini receives carefully crafted instructions (shown at 6:12) specifying:

  • LinkedIn post structure with H1 headline and key takeaways
  • Twitter thread format with numbered tweets
  • Newsletter-style long-form content

The system prompt includes tone guidelines and character limits for each platform, ensuring ready-to-publish content.

Step 4: Output Organization

Final posts automatically save to Google Docs with descriptive filenames containing the source video URL. At 7:45, the creator shows how this allows easy tracking of which content came from which video.

The Google Docs module settings include:

  • Dynamic document naming with video reference
  • Specific folder destination for organization
  • Clean Markdown formatting preserved from Gemini

Common Mistakes to Avoid

The video highlights three critical errors at 9:15:

  1. Incomplete API keys: Always copy the entire RapidAPI key including prefix/suffix
  2. Vague AI instructions: Specific formatting prompts yield better results
  3. No output naming convention: Include video URLs in document titles

The creator emphasizes that proper configuration of each module prevents 90% of workflow failures.
