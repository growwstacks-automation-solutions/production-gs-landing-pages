How n8n Email Scraping Works

The workflow follows a precise sequence that mimics how a human would manually search for business contacts, but automated at scale:

  1. Trigger: Starts with a list of search queries (e.g., "Bay Area real estate agents")
  2. Google Maps Scrape: HTTP request to Google Maps with the search query
  3. HTML Parsing: Code node extracts business domains from the raw HTML
  4. Domain Filtering: Removes Google-owned URLs and duplicates
  5. Email Extraction: Visits each business site to scrape contact emails
  6. Data Cleaning: Normalizes email formats and removes duplicates
  7. Output: Appends results to Google Sheets or your CRM

Critical design elements include 6-second delays between queries to avoid throttling, comprehensive error handling, and two-stage deduplication to ensure list cleanliness.

Step-by-Step Build Guide

Step 1: Set Up the HTTP Request

Create a new n8n workflow and add an HTTP Request node configured to GET from www.google.com/maps/search/[query]. The query parameter pulls dynamically from your trigger workflow. At 4:32 in the tutorial video, you'll see how to test with a hardcoded query like "California real estate" before connecting the trigger.

Step 2: Parse the HTML Response

Add a Code node after the HTTP request with parsing logic that:

  • Extracts business domains from the HTML
  • Filters out Google infrastructure URLs (usercontent.com, gstatic.com)
  • Outputs a clean array of target business websites

Pro Tip: The tutorial includes a complete code snippet you can copy/paste - no need to write complex regex from scratch.

Step 3: Build the Email Scraping Loop

This is where the magic happens. A Loop node iterates through each business domain and:

  1. Makes an HTTP request to the business site
  2. Scans the HTML for mailto links and contact patterns
  3. Extracts and normalizes any found emails

Critical: Enable "Continue on Error" in the loop settings so one failed request doesn't halt the entire workflow. Expect 30-40% of pages to fail due to timeouts - the system is designed to handle this gracefully.

Troubleshooting Tips

Common issues and solutions:

Workflow Stalls After Few Queries

Usually caused by missing error handling. Ensure:

  • "Continue on Error" enabled in both the loop and execute nodes
  • Error output branches exist to capture failures
  • Timeout settings are sufficiently long (20-30 seconds)

Duplicate Emails in Output

Verify you have both:

  1. Initial domain-level deduplication (filters rescraping the same website)
  2. Final email-level deduplication (catches shared corporate addresses)

Low Email Yield

Try these optimizations:

  • Broaden search queries to more populated areas
  • Increase HTTP timeout to 30 seconds
  • Add residential proxy rotation if scraping at very high volume
