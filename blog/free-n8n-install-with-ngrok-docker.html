Why Run n8n Locally with Public Access?

Many businesses hesitate to adopt workflow automation because of perceived hosting costs and technical complexity. The traditional approach requires setting up cloud servers, configuring DNS, and managing SSL certificates - creating barriers to entry for small teams and individual developers.

This Docker + ngrok solution solves both problems. You get the cost savings and data control of local hosting, combined with the public accessibility needed for API integrations and webhooks. At 2:15 in the video tutorial, you'll see how the ngrok tunnel instantly makes your local n8n instance available worldwide with a secure HTTPS connection.

Cost comparison: Cloud hosting for n8n typically starts at $20/month. This local setup costs $0 while providing identical functionality for testing and small-scale production use.

Prerequisites for Installation

Before beginning, ensure you have these three components ready:

  1. Docker Desktop - Available for Windows, macOS, and Linux
  2. ngrok account - Free tier provides sufficient functionality
  3. Local folder - For persisting n8n configuration and workflows

The entire installation process takes about 15 minutes if you have all prerequisites prepared. At 0:45 in the video, we demonstrate downloading Docker Desktop - the most time-consuming part of setup due to the large installer size.

Setting Up Docker for n8n

Docker simplifies n8n installation by packaging all dependencies into a single container. Follow these steps carefully:

Step 1: Pull the n8n Image

In Docker Desktop, search for the official n8n image (n8nio/n8n) and click pull. This downloads the latest stable version.

Step 2: Configure Container Settings

When creating your container, use these critical settings:

  • Port mapping: 5678:5678 (n8n's default port)
  • Volume mount: Local folder to /home/node/.n8n
  • Environment variables: As shown at 3:20 in the video

Pro tip: The volume mount preserves your workflows and credentials between container restarts. Without it, you'd lose all configuration when updating n8n.

Configuring ngrok for Public Access

ngrok creates a secure tunnel from the public internet to your local n8n instance. Here's how to set it up:

  1. Install the ngrok agent on your machine
  2. Authenticate with your ngrok account (free tier works)
  3. Run the tunnel command matching your n8n port

At 4:50 in the video, we demonstrate the critical moment where the ngrok tunnel becomes active. The free plan provides random subdomains that change each time you restart the tunnel - adequate for testing but consider upgrading for production use.

Final Setup and Verification

With both Docker and ngrok running, complete these final configuration steps:

  1. Set n8n's base URL to your ngrok address
  2. Verify API access by hitting your public endpoint
  3. Test webhook functionality with a sample workflow

The video's climax at 6:15 shows the triumphant moment when the local n8n instance becomes publicly accessible. This setup now allows you to:

  • Trigger workflows via public API calls
  • Receive webhooks from external services
  • Access the n8n editor from any device

Limitations and Considerations

While powerful, this free setup has some constraints to consider:

  • ngrok free tunnels disconnect after 2 hours of inactivity
  • No custom domains without ngrok paid plan
  • Your computer must remain powered on for access
