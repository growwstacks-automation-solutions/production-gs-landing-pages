How to Get $300 in Free Google Cloud Credits

Creating professional videos traditionally requires expensive software, filming equipment, and editing time. Most small businesses and content creators simply don't have the budget or resources to produce videos consistently. That's where Google's Veo 3.1 API changes everything.

Google offers $300 in free credits to every new Google Cloud account, valid for 90 days. These credits can be used for their AI video generation service without ever charging your payment method. At 4:15 in the tutorial video, you'll see exactly how to claim these credits.

Key benefit: This $300 credit typically allows for 300-500 short AI-generated videos, depending on length and resolution. That's enough for daily social media content for an entire quarter.

To get started:

  1. Visit cloud.google.com/freetrial
  2. Click "Get started for free"
  3. Select your Google account
  4. Agree to terms and continue
  5. Enter your contact and payment information (you won't be charged until credits are exhausted)

The payment method is required as a backup, but the system will use your free credits first. You'll receive email confirmation when your $300 credit is activated.

Setting Up Vertex AI API Access

With your Google Cloud account created, the next step is enabling the Vertex AI API. This is the gateway to Google's Veo 3.1 video generation capabilities. Many users get stuck at this point because the interface can be overwhelming.

At 6:30 in the video, you'll see the exact navigation path:

  1. In the Google Cloud Console search bar, type "Vertex AI API"
  2. Click the top result
  3. Click "Enable" (this may take 2-3 minutes)

Important: Make sure you're in the correct project (shown in the blue header bar) before enabling the API. The free credits are tied to this specific project.

Once enabled, you'll see a green checkmark confirming Vertex AI API access. This gives your account permission to make video generation requests through the API.

Creating Your Service Account

For security reasons, you shouldn't use your main Google account credentials to access the API. Instead, you'll create a dedicated service account with limited permissions - a crucial security step many tutorials skip.

At 8:45 in the video, the process is demonstrated:

  1. Click the navigation menu (three lines) → IAM & Admin → Service Accounts
  2. Click "Create Service Account"
  3. Name it (e.g., "veo-video-generator")
  4. Click "Create and Continue"
  5. Assign these two roles:
    • Vertex AI Administrator
    • Storage Admin
  6. Click "Continue" then "Done"

After creating the service account, you'll generate a JSON key file:

  1. Click the three dots next to your new service account
  2. Select "Manage keys"
  3. Click "Add key" → "Create new key"
  4. Choose JSON format and click "Create"

This downloads a private key file to your computer. Never share this file - it contains credentials that could allow others to use your cloud credits.

Configuring the n8n Workflow

Now we'll set up the automation in n8n that will handle the entire video generation process. Even if you've never used n8n before, this visual workflow builder makes it simple with drag-and-drop nodes.

The complete workflow consists of 7 main steps:

  1. Generate access token (using your JSON key file)
  2. Create video prompt (what you want the AI to generate)
  3. Send request to Veo API
  4. Wait for video processing (70 seconds)
  5. Check Google Cloud Storage for completed video
  6. Download the video file
  7. Clean up storage (optional)

At 14:20 in the video, you'll see how to import the pre-built workflow JSON into your n8n instance. This gives you all the nodes pre-configured - you just need to add your specific credentials and parameters.

Time saver: The workflow includes error handling and automatic retries, so you don't have to monitor it constantly. Failed attempts won't waste your credits.

Generating Your Access Token

The first node in your n8n workflow converts your JSON key file into an access token that the API will accept. This is the most technical part, but the video shows exactly how to do it without coding.

At 16:05, you'll see how to:

  1. Right-click in n8n and add a "Code" node
  2. Select JavaScript as the language
  3. Paste in the conversion script (provided in video description)
  4. Run the node to generate your token

The token lasts for one hour, but the workflow automatically generates a new one each time it runs. This is more secure than using a long-lived token.

Important security note: The JavaScript contains your private key information. Never share your workflow file without removing this sensitive data first.

Sending the Video Generation Request

With your access token ready, the workflow next prepares and sends the actual video generation request to Google's Veo 3.1 API. This is where you customize what kind of video you want created.

The key parameters you can control:

  • Prompt: The text description of what you want the video to show
  • Length: Duration in seconds (up to 60)
  • Aspect ratio: Landscape (16:9) or portrait (9:16)
  • Style: You can request specific visual styles
  • Output location: Where to store the generated video

At 19:30 in the video, you'll see how to configure the HTTP Request node that sends this data to Google's servers. The most important parts are:

  1. Set method to "POST"
  2. Enter the correct API endpoint URL (shown in video)
  3. Add authorization headers (using your access token)
  4. Format the JSON body with your parameters

When you run this node, it immediately queues your video for generation. The workflow then automatically moves to the next step.

Retrieving and Managing Generated Videos

After sending the generation request, the workflow waits 70 seconds (adjustable) before checking for your completed video. This processing time is consistent regardless of video length.

The retrieval process involves:

  1. A "Wait" node that pauses execution (set to 70 seconds)
  2. An HTTP Request node that checks your Google Cloud Storage bucket
  3. A JavaScript node that processes the response into usable video data
  4. Another HTTP Request that downloads the actual video file

At 23:45 in the video, you'll see how the workflow automatically handles failed attempts and retries. This is crucial because sometimes the API needs a few extra seconds to complete processing.

Storage tip: By default, the workflow deletes videos from Google Cloud Storage after downloading to prevent storage charges. You can modify this to keep copies in the cloud if needed.

The final output is a video file in standard MP4 format that you can use anywhere - social media, your website, email campaigns, etc.

Automating the Entire Process

The real power comes when you schedule this workflow to run automatically. Instead of generating one video at a time, you can create an entire content calendar's worth of videos with minimal effort.

n8n offers several automation options:

  • Schedule trigger: Run daily/weekly at specific times
  • Webhook trigger: Start when receiving data from another app
  • Manual trigger: Run on-demand when needed

To create multiple videos with different prompts:

  1. Store your prompts in a Google Sheet or Airtable base
  2. Connect it to your workflow using n8n's spreadsheet nodes
  3. Add a loop that processes each row
  4. Set reasonable delays between generations

At 27:10 in the video, you'll see how to add error notifications so you're alerted if anything fails. This lets you run the workflow unattended while still maintaining control.

With this automation in place, you could theoretically generate all 300-500 videos before your $300 credit expires - enough content for a year or more!
