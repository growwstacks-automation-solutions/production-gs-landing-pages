Step 1: Client Research & Brand Extraction

The process begins by selecting a target client and extracting their brand information automatically. At 5:30 in the tutorial, we demonstrate how to use AI to scrape a plumbing company's website for:

  • Brand name and services offered
  • Contact information (especially phone numbers)
  • Color scheme and visual style
  • Key messaging and value propositions

This information is formatted into a clean markdown template that Gemini 3 will use to recreate the brand. If the color detection isn't perfect (as happened at 7:15), you can simply upload a screenshot and have the AI correct it.

Step 2: Website Generation with Voice Agents

With the brand information ready, we move to Gemini 3 to generate the actual website. The key is the prompt structure shown at 9:45:

"Create a one-pager app for a [industry] based on the given brand information below. Include two voice AI agents - one as a front desk and one as an emergency dispatcher - in the above-the-fold section."

Within 2-8 minutes (visible in real-time at 11:20), Gemini 3 generates:

  1. A complete branded website matching the client's style
  2. Two fully functional voice agents with distinct roles
  3. All necessary frontend and backend code
  4. Automatic error correction during generation

Security Considerations for Deployment

At 13:30, we address the most common concern - API key security. When deploying through Google Cloud (recommended):

  • Your Gemini API key is protected by a proxy system
  • The frontend communicates via temporary session tokens
  • No key exposure in client-side code

However, if you download the code to host elsewhere (like GitHub Pages), you'll need to implement backend security measures. For most agency use cases, Google Cloud deployment is the simplest and most secure option.

Voice Agent Customization Options

At 17:50, we show two ways to customize your voice agents:

  1. No-code method: Simply tell the AI "Make Sarah a female voice and Mike a male voice"
  2. Code method: Adjust the JSON configuration (name, voice, role, instructions)

You can change agent behaviors, responses, and even personality traits through natural language prompts. This lets you tailor demos to specific industries or use cases without technical expertise.

Step 3: Deployment & Branding

The final step is deploying your demo to a branded URL. At 20:15, we walk through:

  1. Google Cloud deployment (1-click process)
  2. Adding a custom subdomain (client.yourdomain.com)
  3. DNS configuration for brand authenticity
  4. HTTPS certificate automation

This creates a professional, branded experience that impresses clients. The entire process from research to live demo typically takes under 30 minutes.
