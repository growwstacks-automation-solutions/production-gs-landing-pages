Deployment Options: Google Cloud vs External Hosting

Gemini 3 offers two deployment paths with different security implications:

Google Cloud Hosting (Recommended): Uses a proxy system that protects your Gemini API key. Front-end communicates with a Google-managed back-end where your key resides securely. Deployment takes 1-5 minutes with automatic HTTPS.

External Hosting: Requires additional security measures:

  • Separate back-end service to protect API keys
  • Custom proxy implementation
  • More complex domain configuration

For sales demos and client presentations, Google Cloud hosting provides the fastest, most secure option with minimal setup.

Client Presentation: Custom Domains & White Labeling

To maximize impact, deploy demos on custom subdomains (client.yourdomain.com) using Google Cloud's domain mapping:

  1. In Google Cloud Console, navigate to Domain Mappings
  2. Add a CNAME record pointing your subdomain to ghss.googlehosted.com
  3. Wait for SSL certificate provisioning (5-60 minutes)

This creates white-labeled demos that:

  • Appear as your client's own property
  • Drive traffic back to your main domain
  • Can be customized with client branding extracted earlier

The video shows this process using a plumbing company's name as the subdomain for maximum client relevance.
