How the Closed Loop Voice Stack Works

The closed loop voice stack solves missed calls with three connected components working in sequence: a listener, a thinker, and a speaker. This architecture creates a self-contained system that handles calls intelligently without human intervention.

1. The Listener captures what the caller says using Python's speech recognition libraries. It converts spoken words to text that Gemini can process. Modern libraries achieve 90-95% accuracy for clear English speech.

2. The Thinker is Google Gemini reading the transcribed text and generating intelligent responses. Gemini understands context, remembers conversation history, and operates within boundaries set by your system prompt.

3. The Speaker takes Gemini's text response and delivers it back as natural speech using Python's text-to-speech libraries. The entire loop happens in under 2 seconds - fast enough that callers perceive a natural conversation.

Building the Agent With Google Gemini

Google AI Studio provides free access to Gemini's API - no credit card required. The free tier includes enough capacity for a small business voice agent handling standard inquiries. Here's the step-by-step setup:

Step 1: Get Your Gemini API Key

Visit Google AI Studio and create a new project. Generate an API key specifically for your voice agent - this allows your Python code to send and receive data from Gemini.

Step 2: Set Usage Limits

Configure rate limits to prevent unexpected charges (though the free tier is generous). For a typical business agent, 60 requests per minute covers peak call volume.

Step 3: Test the API Connection

Use Python's requests library to send a test prompt and verify the response. This confirms your key works before building the full voice stack.

Pro Tip: Store your API key securely using environment variables - never hardcode it in your Python scripts. This prevents accidental exposure if sharing code.

Essential Python Libraries for Voice AI

The voice stack requires just two core Python libraries, both free and open-source:

SpeechRecognition handles converting spoken audio to text. It supports multiple engines (Google, Whisper, Sphinx) and works with standard microphone inputs or audio files.

pyttsx3 converts Gemini's text responses back to speech. It offers voice customization (pitch, rate, volume) and works offline without API calls.

Install both with pip:

 pip install SpeechRecognition pyttsx3

The complete voice agent script is under 150 lines of Python. It initializes the microphone, processes audio through speech recognition, sends text to Gemini, and speaks the response - looping continuously until the call ends.

Crafting the Perfect System Prompt

The system prompt is where you give Gemini its identity and instructions. Think of it as an employee handbook for your AI agent. A well-structured prompt includes:

  • Business Details: Name, services, hours, location
  • Call Handling Rules: How to greet, qualify leads, handle objections
  • Information Collection: What details to capture (name, phone, reason for calling)
  • Fallback Protocol: What to say when unsure ("Let me have someone call you back")

Example prompt for a plumbing company:

 "You are the virtual receptionist for City Plumbing. We offer 24/7 emergency plumbing services in the Denver metro area. When calls come in: 1. Greet warmly: 'Thanks for calling City Plumbing, how can we help?' 2. Determine if emergency: 'Is this an emergency with active water leaking?' 3. For emergencies: 'We'll dispatch a technician immediately. What's the address?' 4. For non-emergencies: 'We can schedule a visit tomorrow. What time works best?'"
