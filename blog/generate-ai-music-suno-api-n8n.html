Suno API Overview

Suno.com offers a powerful AI music generation platform, but their web interface requires manual operation. The API (accessed through KeyAI) unlocks automated creation with more customization options than the standard UI.

The API supports two generation modes: Simple (genre/style only) and Custom (full control over lyrics, instrumentation, and vocal style). At 7:30 in the tutorial, you'll see how the API provides parameters missing from the web interface like beat-per-minute control and vocal gender selection.

n8n Workflow Architecture

The complete automation system uses n8n to handle the entire music generation pipeline. The workflow consists of four key components that work together seamlessly:

  1. API Request Node: Sends generation parameters to Suno
  2. Status Checker: Verifies when songs are ready
  3. Download Module: Retrieves finished MP3 files
  4. Storage Integration: Saves tracks to Google Drive

At 12:45 in the video, the creator demonstrates how the workflow handles the asynchronous nature of music generation - songs take 2-3 minutes to create, requiring smart polling rather than immediate download.

Setting Up the API Connection

Connecting to Suno's API requires a KeyAI account and API key. The process involves:

Step-by-step setup:

  1. Create a KeyAI account (Google login supported)
  2. Navigate to Model Market → Text to Music → Suno API
  3. Generate an API key in your account settings
  4. Import the cURL sample into n8n's HTTP Request node
  5. Replace the placeholder token with your actual API key

At 15:20, the tutorial shows how to use n8n's "Import cURL" feature to automatically configure the HTTP request with all required headers and parameters.

Song Generation Process

The core generation workflow accepts parameters that determine the musical output. Key configuration options include:

  • Prompt: Description of desired music (e.g., "chill ambient with piano")
  • Style: Genre or artist inspiration (e.g., "inspired by Hans Zimmer")
  • Custom Mode: Enables advanced parameters when set to true
  • Instrumental: Set to false for songs with vocals
  • Model Version: V5 produces highest quality results

At 18:30, the video demonstrates generating a rap track "inspired by Eminem" that surprisingly delivered female vocals - highlighting how the AI interprets style influences creatively rather than literally.

Status Checking System

Since music generation isn't instantaneous, the workflow needs to poll for completion. The solution uses:

  1. A Wait node (10 second delay between checks)
  2. Switch node evaluating status responses:
    • Pending → Loop back
    • Text Success → Continue checking
    • First Success → Download first track
    • Success → Download both tracks

At 22:10, you'll see how this system handled a generation that took 140 seconds - checking status 8 times before both tracks were ready.

Downloading Finished Tracks

Completed songs are available as MP3s through temporary URLs. The workflow:

  1. Extracts download URLs from the API response
  2. Uses HTTP Request to fetch the audio data
  3. Uploads files to Google Drive with descriptive names
  4. Organizes by genre/date for easy retrieval

At 25:45, the tutorial shows how to configure the Google Drive node to automatically sort tracks into genre-specific folders while preserving all metadata.
