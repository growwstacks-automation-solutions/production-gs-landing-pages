The Great Antigravity Split: IDE vs Agent App

Google's decision to separate Antigravity into two distinct applications left many developers scratching their heads. The once unified platform is now divided between a VS Code-style IDE and a standalone agent chat interface focused on managing autonomous workflows.

This architectural change reflects Google's vision for more specialized tools - the IDE maintains traditional coding features while the new agent app resembles platforms like Co-work and Codeex, designed specifically for orchestrating AI agents. The split allows each component to evolve independently but comes at the cost of increased complexity for users.

Key Insight: Early adopters report that while the separation creates initial friction, the specialized interfaces ultimately provide better experiences for their respective use cases - coding versus agent orchestration.

Dynamic Sub-Agents: Parallel Processing Power

One of Antigravity 2.0's most significant innovations is the introduction of dynamic sub-agents. This feature addresses a fundamental limitation of single-agent systems - context window overload.

When facing complex tasks, your main AI agent can now automatically spin up specialized helper agents to handle different aspects of the work simultaneously. This parallel processing capability mirrors how human teams delegate tasks to specialists, dramatically improving efficiency for multi-faceted projects.

Async Tasks and Scheduled Workflows

Antigravity 2.0 introduces asynchronous task handling inspired by Claude's goal system and Codeex workflows. Agents can now continue working in the background without blocking your main workflow, and can be scheduled to run at specific times using cron-like syntax.

This feature is particularly valuable for:

  • Long-running data processing tasks
  • Regular maintenance operations
  • Batch processing during off-peak hours
  • Periodic monitoring and alerts

The New Project System

Gone are the days of conversations tied to single repositories. Antigravity 2.0 introduces a comprehensive project system that spans multiple folders with custom permissions and workflows.

This change reflects how modern development actually works - across multiple repositories, services, and environments. The new system allows you to:

  • Define project-wide rules and constraints
  • Manage access controls for team members
  • Establish standardized workflows across components
  • Maintain context across related codebases

Essential New Slash Commands

Antigravity 2.0 expands its command repertoire with several powerful new slash commands designed to give developers more control over agent behavior:

/goal - Directs the AI to work continuously until the specified task is complete

/drillme - Forces the agent to ask clarifying questions before starting work

/browser - Enforces web browsing during workflows for up-to-date information

/schedule - Sets up timed agent tasks similar to cron jobs

These commands help bridge the gap between human intent and AI execution, reducing the need for constant supervision and iteration.

Antigravity CLI and SDK

Google has rebranded the Gemini CLI as the Antigravity CLI, aligning it with the platform's new direction. This lightweight terminal interface provides access to the same agent models and workflows available in the desktop app.

More significantly, the new Antigravity SDK gives developers programmatic access to Google's agent harness technology. This enables:

  • Custom autonomous agent systems
  • Deep integration with existing tools
  • Specialized deployment scenarios
  • Tailored workflow automation

Voice Transcription and Other Features

Antigravity 2.0 introduces live voice transcription, allowing developers to speak prompts directly into agent workflows. This hands-free interaction complements traditional text input and can be particularly useful during debugging sessions or when documenting thought processes.

Additional notable features include:

  • Customizable skills for specialized tasks
  • JSON hooks for system integration
  • Improved artifact review workflows
  • Enhanced multi-project support

Getting Started with Antigravity 2.0

To begin using Antigravity 2.0, you'll need to download both the standalone agent app and the separate IDE from Google's website. The installation process will prompt you to install recommended plugins including:

Modern Web Guidance: Keeps coding agents updated with latest web best practices

Chrome DevTools: Provides reliable automation for web browsing tasks

Antigravity SDK: Essential for developers building custom agent systems
