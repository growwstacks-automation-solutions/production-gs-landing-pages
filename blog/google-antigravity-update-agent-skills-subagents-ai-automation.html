Agent Skills Explained

Building AI workflows often means repeating the same instructions across different projects. Whether it's API integrations, testing procedures, or report generation, developers waste countless hours recreating solutions they've built before. Google's new agent skills solve this by packaging knowledge and behavior into reusable modules.

The skills system uses simple markdown and YAML files to define standardized components. Once created, these skills become automatically discoverable and applicable across your projects. As shown in the video at 2:15, a flight tracker demo that previously required re-describing API integration steps can now leverage pre-built skills for consistent, high-quality results.

Key benefit: Skills create verifiable proof artifacts for every run, ensuring transparency and reproducibility in your automated workflows. This is particularly valuable for compliance-heavy industries.

Parallel Subagents Architecture

The update fundamentally changes how Antigravity approaches complex tasks. Instead of writing all code sequentially, the system now orchestrates multiple subagents working simultaneously in the background.

This high-level architecture allows for better task delegation, with subagents handling long-running processes while the main agent focuses on system-level coordination. The video demonstrates this at 4:30, showing how different aspects of a cryptocurrency exchange setup can progress in parallel rather than waiting for each step to complete.

40% faster execution: Early benchmarks show parallel subagents can complete complex workflows nearly twice as fast as traditional sequential approaches for certain types of tasks.

Understanding Usage Limits

The free tier of Antigravity now implements weekly usage caps that refresh every 5 hours, totaling about 5 hours of runtime per week. While generous for casual users, these limits can affect those running intensive workflows.

Google positions these limits as necessary for platform sustainability given the massive demand. As shown at 6:45 in the video, heavy users may find themselves pushed toward the Ultra tier, which remains unlimited. This creates a clear value differentiation between the free and paid offerings.

Enhanced Secure Mode

Security-conscious organizations will appreciate Antigravity's new secure mode, which implements strict controls over agent operations. The system now requires manual approval for terminal commands, JavaScript execution, and certain artifact actions.

Secure mode also restricts website access and locks the workspace to respect gitignore rules. These measures, demonstrated at 7:20 in the video, provide greater controllability and fewer surprises during automated workflows while maintaining security.

Practical Applications

The new features open up numerous real-world applications. One standout example shown at 8:10 is automated Jira ticket management - users can now create and update tickets directly through Antigravity without tab switching.

Other practical uses include rapid prototyping within workflows (no more context switching), automated data manipulation in Google Sheets, and standardized onboarding processes. The video demonstrates how Antigravity can crawl a website to compile brand assets and even draft follow-up emails automatically.

Creating Your Own Skills

Building custom skills is surprisingly straightforward. As shown at 3:45 in the video, you simply add a skills.md file to your workspace - either manually or by instructing Antigravity to generate it for you.

The file contains markdown and YAML descriptions of the skill's purpose, parameters, and execution steps. Once created, these skills become automatically available whenever relevant to your prompts and workflows, significantly reducing repetitive coding.

Pro tip: Start by identifying the most repetitive aspects of your workflows - these are prime candidates for skill creation. Common examples include API wrappers, data validation routines, and report templates.

Flash Mode vs Standard Mode

Antigravity now offers two distinct execution modes: Flash for rapid tasks and Standard for more thorough analysis. Flash mode, highlighted at 9:05 in the video, delivers roughly twice the speed of standard mode for straightforward automation tasks.

While perfect for quick prototyping and basic task execution, Flash mode may lack some depth in analysis compared to standard mode. The choice between them depends on your specific needs - speed versus comprehensiveness.
