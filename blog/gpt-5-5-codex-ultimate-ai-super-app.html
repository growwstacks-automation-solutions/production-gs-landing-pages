Permission Levels and Safety

Given Codex's powerful capabilities, the platform offers three permission levels to balance productivity with safety:

1. Sandbox-Only (Default): Commands run in an isolated environment with no system access.

2. Auto-Review: Elevated requests prompt for user approval before execution.

3. Full Autonomous: No permissions needed - use with caution on trusted projects only.

The recommendation is to create separate projects rather than granting full computer access. This "project container" approach lets you safely experiment while protecting your main system.

Debugging with Codex

One of Codex's most practical features is its debugging capability. Unlike traditional coding assistants that just suggest fixes, Codex can:

1. Test Frontends: Click through apps like a real user to identify UI issues

2. Inspect Console Logs: Analyze network traffic and error messages

3. Propose Fixes: Suggest minimal code changes to resolve issues

4. Implement Solutions: Optionally apply fixes automatically

This creates a complete debugging workflow where Codex can identify, diagnose, and potentially fix issues without constant human oversight.
