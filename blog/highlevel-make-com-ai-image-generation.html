Setting Up the HighLevel to Make.com Webhook

The foundation of this automation is the webhook connection between HighLevel and Make.com. Many marketers waste hours manually downloading content from HighLevel and uploading it to other platforms. This webhook eliminates that friction point completely.

At 0:45 in the video, you'll see the exact steps to create this connection. After logging into Make.com, you'll select "Custom Webhook" from the scenario options. This generates a unique URL that serves as the bridge between platforms.

Key configuration detail: The webhook URL must be copied exactly from Make.com and pasted into the corresponding field in HighLevel without any modifications. Even a single character difference will break the connection.

Once established, this webhook automatically passes any new content created in HighLevel (including text captions and metadata) directly to Make.com for processing. This happens in real-time without any manual intervention required.

Configuring OpenAI for Automatic Image Generation

The magic happens when HighLevel's content reaches the OpenAI node in Make.com. Many businesses don't realize they can automate their entire visual content creation process using AI - they're still paying designers or using stock photos.

At 1:20 in the tutorial, you'll see the OpenAI node configuration. The critical step is selecting "DALL·E 1.5" as your model (the newest version at recording time) and mapping the "caption" field from the incoming webhook data to the image prompt.

Pro tip: You can enhance the AI's output by prepending your prompt with brand-specific instructions like "Minimalist modern style" or "Bright colorful composition" to maintain visual consistency across all generated images.

This setup means every piece of content that comes through HighLevel automatically gets transformed into a unique, on-brand visual asset without any human involvement. The AI handles everything from conceptualization to final rendering.

Converting and Storing Generated Images

Raw AI-generated images need proper formatting before social platforms can use them. The tutorial shows a clever two-step process at 2:10 that many marketers miss - first converting the image format, then properly hosting it.

OpenAI outputs images in PNG format by default, but Instagram prefers JPEG. The Convert Image node handles this transformation automatically. Then Cloudinary provides reliable cloud hosting with a permanent URL that won't break like temporary links often do.

Critical setting: When configuring Cloudinary, ensure you set the resource type to "image" and mime type to "image/jpeg" exactly as shown at 2:35. This guarantees proper handling of the file throughout the workflow.

The public ID field lets you customize how files are named in Cloudinary. Using descriptive names like "instagram-post-[date]" makes it easier to manage and reference images later if needed.

Automating Instagram Posts with Business API

The final piece connects your beautifully generated images back to Instagram automatically. At 3:00 in the video, you'll see the Instagram Business API node configuration that makes this possible.

After authenticating with your Facebook credentials (required for Instagram Business accounts), you'll map two critical fields:

  1. The "secure URL" from Cloudinary for the image
  2. The original "caption" from HighLevel for the post text

This means the entire post - both visual and textual components - maintains complete consistency from the original HighLevel content through to the published Instagram post.

Important note: The Instagram Business API requires your account to have a connected Facebook page. Personal Instagram accounts won't work with this automation.

Measuring Time Savings and Efficiency Gains

Let's quantify the impact of this automation. A typical manual process involves:

  • 15-30 minutes brainstorming/image concepting
  • 60-90 minutes creating the graphic (or searching stock sites)
  • 15 minutes formatting and uploading
  • 10 minutes writing the caption and posting

That's 2-3 hours per post reduced to zero active time with this workflow. For agencies posting daily across multiple client accounts, this represents 60-90 hours saved per month.

Scalability benefit: This workflow maintains consistent quality whether you're posting once a week or ten times daily. The AI doesn't get tired or rushed like human designers might during busy periods.

Scaling This Workflow for Multiple Clients

The tutorial mentions at 4:15 that Make.com's free tier allows approximately 200 posts - enough to demonstrate value before upgrading. But scaling requires some strategic planning.

For agencies managing multiple clients:

  1. Create separate scenarios in Make.com for each client brand
  2. Use client-specific Cloudinary folders for image organization
  3. Set up distinct HighLevel webhooks for different content types

This maintains clean separation while allowing you to manage all client workflows from one Make.com account. The tutorial's mention of Make.com's credit system is important - monitor your usage as you scale.

Pro tip: Consider adding a filter node after the webhook to route different content types (posts vs stories vs reels) to appropriately formatted Instagram post types.
