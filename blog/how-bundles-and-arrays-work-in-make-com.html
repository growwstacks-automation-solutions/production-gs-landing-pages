What Are Bundles in Make.com?

Many Make.com users don't realize they can process multiple records efficiently while using minimal scenario credits. This is where bundles come into play. A bundle represents a single, self-contained record of data that gets processed individually within your scenario.

According to Make.com's documentation, a bundle could be one row from a spreadsheet, one email message, or any other discrete unit of data. When you fetch data from a source like Google Sheets, each row becomes a separate bundle that can be processed while only using one scenario credit for the entire operation.

Key insight: Processing multiple bundles typically uses just one scenario credit, regardless of how many bundles are processed in that operation. This makes bundles an incredibly efficient way to handle batch operations.

Understanding Arrays in Make.com

While bundles represent individual records, arrays give you a way to organize and work with multiple bundles together. An array is a data structure used to store and manage lists of similar items - in this case, your bundles.

At 3:10 in the video, we see the Array Aggregator module in action. This powerful tool collects multiple bundles and organizes them into an array format that you can then work with in subsequent modules. Arrays become particularly useful when you need to access specific records from your data set or perform operations on multiple records simultaneously.

Using the Array Aggregator Module

The Array Aggregator module is your gateway to working with arrays in Make.com. Here's how to use it effectively:

  1. Select your source module: Choose the module that's producing your bundles (like the Google Sheets module in our example)
  2. Choose fields to include: Select which data fields from your bundles should be included in the array
  3. Configure aggregation settings: Decide how you want the array to be structured and whether you need any special processing

When you run your scenario after setting up the Array Aggregator, you'll see your data organized in array format in the module's output. This structured format makes it much easier to work with multiple records in subsequent steps of your workflow.

Accessing Specific Items in an Array

One of the most powerful features of arrays is the ability to access specific items by their position. As shown at 4:30 in the video, array indexes start at 0 - meaning the first item is at position 0, the second at position 1, and so on.

To access a specific item:

  1. Reference the array in your target module
  2. Specify the index position of the item you want
  3. Select the specific field from that item you need to work with

Important note: Make.com's interface sometimes displays positions starting from 1 (like in the video at 5:45), but the underlying array structure still follows the standard programming convention of starting at 0. This can cause confusion, so it's important to test your references.

Credit Efficiency with Bundles

One of the biggest advantages of bundles is their credit efficiency. Here's why they're so valuable:

  • Single credit for multiple records: Processing three rows from a spreadsheet uses one credit, not three
  • Batch operations: You can perform operations on all bundles in a single module execution
  • Reduced scenario complexity: Fewer modules needed to handle multiple records

This efficiency becomes particularly important when working with large data sets or when running scenarios frequently. By understanding bundles, you can design workflows that handle more data without proportionally increasing your credit consumption.

Common Use Cases for Bundles and Arrays

Bundles and arrays shine in several common automation scenarios:

  1. Batch email processing: Send personalized emails to multiple recipients while only using one credit for the entire batch
  2. CRM updates: Update multiple records in your CRM from a spreadsheet import
  3. Order processing: Handle multiple orders from an ecommerce platform in a single scenario run
  4. Data transformations: Process and transform multiple records between systems

In each case, bundles allow you to work with multiple records efficiently, while arrays give you the structure needed to access and manipulate specific items within those groups.
