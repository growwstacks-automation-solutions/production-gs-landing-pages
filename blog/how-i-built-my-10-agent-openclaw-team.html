OpenClaw Agent Architecture Explained

Each OpenClaw agent consists of several markdown files that define its personality, capabilities, and operating parameters:

  • identity.md - Name, emoji, and one-line description
  • soul.md - Personality, communication style, and behavioral guidelines
  • agents.md - Operating instructions and protocols
  • user.md - Everything the agent knows about you
  • tools.md - System access and capabilities
  • memory.md - Long-term curated knowledge
  • heartbeat.md - Autopilot instructions that run every 30 minutes

The heartbeat system is particularly powerful - it allows agents to perform scheduled tasks automatically without your direct input. For example, my research agents scan new publications every 30 minutes while my project managers send daily status updates at 8 AM.
