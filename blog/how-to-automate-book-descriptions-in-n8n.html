n8n Workflow Overview: Form → AI → Google Docs

The complete automation follows a simple but powerful three-step flow that you can adapt to countless author tasks:

  1. Form Trigger: Input your book details and example descriptions
  2. AI Agent: Analyzes patterns and generates custom description
  3. Google Docs: Creates formatted document with the output

At 4:30 in the video tutorial, you'll see how these components connect visually in n8n's workflow editor. The magic happens in the AI agent configuration where we use prompt engineering techniques to ensure high-quality output tailored to your genre.

Setting Up the Form Trigger

The form serves two critical purposes: collecting information about your book and providing examples of successful descriptions to emulate. Here's how to configure it:

  1. Add an "On Form Submission" trigger node
  2. Create two text area fields (both required):
    • "Information about your book" (your manuscript details)
    • "Example of a good book description" (from top sellers in your genre)
  3. Use the pinning feature (shown at 12:45) to lock your test values during development

Pro Tip: Pull example descriptions directly from Amazon bestsellers in your genre (as demonstrated at 9:20). These have been market-tested and contain the hooks that actually convert readers.

Configuring the AI Agent

The AI agent node (added at 6:15) is where the magic happens. Key configuration steps:

  1. Select OpenRouter as your model provider (gives access to multiple AIs)
  2. Choose Gemini or another suitable model for creative writing
  3. Set "Define Below" as the prompt source (not connected chat)
  4. Use expression mode to incorporate form inputs into your prompt

The initial prompt ("analyze the above book description...") at 7:30 provides basic instructions, but the real improvement comes when we add XML tags (at 18:00) to structure the input clearly for the AI:

 <story_info> [Your book details here] </story_info> <book_description_example> [Top-performing example here] </book_description_example>

Prompt Engineering Tips for Better Results

Simple prompt adjustments can dramatically improve output quality:

  • Be specific about genre conventions (e.g., "Use similar emotional hooks as the romance example")
  • Request multiple versions by adding "Generate 3 distinct options" to your prompt
  • Control length with directives like "Keep under 250 words"
  • Include keywords ("Incorporate these phrases naturally: [list]")

At 16:30 in the video, you'll see how the AI initially fails without proper input structuring - and how adding XML tags immediately improves the results by clearly separating your book info from the example description.

Google Docs Integration

The final piece automates document creation (starting at 21:45):

  1. Add a "Create Document" node connected to your Google Drive
  2. Specify the target folder using its ID from the URL
  3. Add an "Update Document" node to insert the AI-generated text
  4. Connect the AI output to the document content field

This creates a ready-to-edit Google Doc with your custom book description - no copying and pasting required. For series authors, you could modify this to create a master document with all book descriptions in one place.

Testing and Pinning Nodes

A crucial n8n feature shown at 12:15 is node pinning, which lets you:

  • Test individual workflow sections without re-running everything
  • Lock in form responses during development
  • Iterate on prompts without resubmitting information

Right-click any node and select "Pin" to freeze its current output. This is especially valuable when tweaking AI prompts - you can modify and re-run just the agent node while keeping your test inputs constant.
