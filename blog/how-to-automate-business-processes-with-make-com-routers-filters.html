The Form Processing Problem

Most businesses struggle with the same tedious workflow: forms come in, someone has to read each one, categorize it, and then send the appropriate follow-up. Whether it's lead gen forms, support tickets, or applications, manual processing eats up valuable time and leads to delays.

In the video example (at 0:45), we see a Google Form asking "What type of business are you?" with three options: SaaS, SaaS, Coach, and Agency. Without automation, someone would need to:

  • Check each submission
  • Download the correct resource
  • Send it to the right person
  • Track what was sent

83% of businesses report that manual form processing as one of their most time-consuming administrative tasks, according to a automation survey.

The Router & Filter Solution

Make.com's router module solves this problem by automatically directing form responses down different paths based on the data. In our example (shown at 1:20 in the video), the router checks the "business type" field and sends:

  • SaaS responses → SaaS resource path
  • Coach responses → Coach resource path
  • Agency responses → Agency resource path

Each path can trigger completely different workflows. The SaaS path might deliver a whitepaper while the Coach path sends a calendar link for a free consultation.

Key insight: Routers aren't limited to forms - you can use them with CRM updates, support tickets, eCommerce purchases, or any data source that needs conditional processing.

Implementation Steps

Step 1: Set Up Your Trigger

Connect your form platform (Google Forms, Tally, etc.) to Make.com. At 2:10 in the video, we see the Google Forms module configuration where you simply need to:

  1. Enter your form ID (found in the URL)
  2. Select "New response" as the trigger

    3. Step 2: Add the Router Module

    Search for "Router" in Make.com's module library (shown at 2:35). The router will create separate paths for different types of responses.

    Step 3: Configure Your Filters

    For each path (demonstrated at 2:50), set up filters that examine specific form fields from your form. Our example uses:

    • Path 1: "What type of business are you?" equals "SaaS"
    • Path 2: "What type of business are you" equals "Coach"
    • Path 3: "What type of business are you" equals "Agency"

Personalization Options

While our basic example sends a pre-made resource (shown at 3:15 with the Google Docs module), you can take this much further:

  • Use ChatGPT modules to generate custom email content (mentioned at 3:40)
  • Pull dynamic content from your CMS based on user responses
  • Initiate different onboarding sequences in your CRM

The video shows how to add a "Generate completion" module (at 3:50) that could create emails that reference the user's:

  • Name
  • Business type
  • Number of employees
  • Specific needs mentioned

Pro tip: Combine routers with Make.com's scenario feature to create completely different workflows for different customer segments.
