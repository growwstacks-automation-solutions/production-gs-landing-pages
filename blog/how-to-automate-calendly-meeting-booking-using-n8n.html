The Calendly API Gap in n8n

If you search for Calendly in n8n today (as of November 2025), you'll find only two trigger nodes - one for when an event is created, and another for when one is cancelled. This leaves businesses needing automated scheduling in a frustrating position. You can react to bookings, but not initiate them.

The breakthrough came when Calendly quietly released their scheduling API about two weeks before this guide was published. While the API documentation exists, n8n hasn't yet incorporated these new endpoints into their native nodes. This is where HTTP requests shine - they let you access new API features immediately.

Key insight: The scheduling API uses Calendly's /invitees endpoint, which isn't yet available in n8n's native nodes. By using HTTP requests, you gain access to this functionality months before it might be officially supported.

Getting Your Access Token

Before making any API calls, you'll need proper authorization. Calendly uses personal access tokens rather than API keys. Here's how to generate yours:

  1. Log in to your Calendly account
  2. Navigate to Integrations → API and Webhooks
  3. Click "Generate New Token"
  4. Name your token (e.g., "n8n Automation")
  5. Complete email verification when prompted
  6. Copy the generated token to a secure location

At the 1:15 mark in the video tutorial, you'll see the exact navigation path and token generation process. This token acts as your authentication for all subsequent API calls, so treat it like a password.

Current User Endpoint

The first API call you'll make retrieves your current user information. This provides critical details needed for subsequent calls, particularly your user URI. Here's the endpoint structure:

Endpoint: https://api.calendly.com/users/me

Method: GET

Headers: Authorization: Bearer [your_token]

When successful, the response includes your user URI in this format: "uri": "https://api.calendly.com/users/ACCOUNT_NUMBER". This URI becomes essential for retrieving your available event types in the next step.

Event Type Identification

With your user URI in hand, you can now query Calendly's event types endpoint to identify which meeting durations are available for scheduling. This is crucial because:

  • Different meeting types have different URIs
  • Your automation needs to target a specific duration
  • The event type URI is required for the booking request

The endpoint format is:

Endpoint: https://api.calendly.com/event_types?user=USER_URI

Note: Replace USER_URI with the uri value from your previous response

The response will list all your available event types. Identify the one you want to automate (e.g., "30 Minute Meeting") and note its URI for the booking step.

Crafting the Booking Request

Now for the main event - actually scheduling a meeting. The booking request requires three key components:

  1. Event Type: The URI of your chosen meeting duration
  2. Start Time: In ISO 8601 format with timezone
  3. Invitee Details: Name, email, and timezone

Here's a sample JSON payload you'd send to the /invitees endpoint:

Sample Request: 

{   "event": "https://api.calendly.com/event_types/EVENT_TYPE_URI",   "start_time": "2025-11-02T09:00:00-05:00",   "invitee": {     "name": "John Doe",     "email": "john@example.com",     "timezone": "America/New_York"   } }

Implementing in n8n

With the API working in Postman, transferring to n8n is straightforward:

  1. Add an HTTP Request node
  2. Set method to POST
  3. Enter the invitees endpoint URL
  4. Add Authorization header with your token
  5. Configure the JSON body with your parameters

The video demonstrates a clever shortcut at 4:30 - using Postman's "Copy as cURL" feature to quickly import the working request into n8n. This eliminates manual configuration errors.

Pro Tip: Store your Calendly token as an n8n credential rather than hardcoding it. This improves security and makes the workflow reusable across your team.
