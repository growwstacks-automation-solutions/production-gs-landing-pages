How Sales Navigator's Saved Searches Work

LinkedIn Sales Navigator's most powerful feature is the ability to save complex searches and be notified when new profiles match your criteria. As shown at 0:58 in the video, each saved search shows how many new results have appeared since your last visit.

The key to automation lies in the search URL parameters - specifically the lastViewedAt timestamp. By modifying this parameter programmatically, we can tell LinkedIn exactly which time period's new leads we want to scrape, making the process completely hands-off.

Workflow Overview

This automation consists of two connected n8n workflows. The first runs on a schedule (typically weekly), checks your Sales Navigator account status, then creates a scraping order through Vayne API. The second workflow receives the scraped data, enriches it with email addresses, and routes qualified leads to your outreach campaigns.

At 3:12 in the video, you can see how the two workflows connect through a webhook, ensuring leads flow automatically from LinkedIn to your CRM without any manual steps in between.

Setting Up the Schedule

The workflow begins with a schedule trigger set to run every Monday morning (or daily if preferred). At 1:45 in the video, the creator explains the importance of calculating "last week's" date rather than using the current date, ensuring you capture all new leads since the last run.

For agencies managing multiple clients, you can create separate workflow instances with different schedules for each client's campaign. The timestamp calculation ensures no leads are missed between runs.

Transforming Dates for LinkedIn

The most technical part of the workflow (shown at 2:18) converts the calculated date into LinkedIn's specific timestamp format. While this might seem complex, the video demonstrates how to generate this code using ChatGPT by providing the Vayne API documentation.

This transformed timestamp becomes the lastViewedAt parameter in your Sales Navigator search URL, telling LinkedIn exactly which new leads to return. The workflow automatically updates this parameter each run to always look back the correct time period.

Checking Account Status

Before creating the scraping order, the workflow checks if your Sales Navigator account is still properly connected to Vayne API (3:45 in the video). If your session has expired, it sends an immediate Slack notification prompting you to update your cookies.

This proactive check prevents failed scrapes due to authentication issues. The notification includes direct links to update your session, minimizing downtime for your lead generation pipeline.

Creating the Scraping Order

With a valid session confirmed, the workflow creates the scraping order through Vayne API (4:20 in the video). It constructs the Sales Navigator URL using your saved search ID and the transformed timestamp parameter.

The workflow then sets up a webhook to receive the scraped data automatically when ready. This eliminates the need to poll for completion status - the results will be pushed directly to your second workflow for processing.

Processing the Results

The second workflow begins with a webhook that receives the scraped lead data (5:05 in the video). It downloads the CSV from AWS, extracts the contact information, then enriches each lead with professional email addresses using services like DropContact.

You can add custom filtering steps to evaluate lead quality before routing to campaigns. The video shows how to filter based on LinkedIn's match confidence scores (6:30) or even connect an AI agent to analyze profile details for better qualification.

87% accuracy: Vayne API's email enrichment typically finds valid professional emails for nearly 9 out of 10 LinkedIn profiles, far higher than most manual lookup methods.
