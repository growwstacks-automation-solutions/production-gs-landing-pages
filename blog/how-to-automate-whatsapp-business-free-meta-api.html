The Problem With Paid WhatsApp Solutions

Businesses worldwide are spending $100-$500+ per month on WhatsApp automation tools that all share the same fatal flaw - they rely on unofficial third-party APIs. These services act as middlemen between you and Meta's servers, creating three major problems:

Second, Meta frequently changes their API rules, causing these third-party services to break unexpectedly. Worst of all, using unofficial APIs violates Meta's terms of service, putting your business at risk of sudden bans with no warning.

Step 1: Clone the GitHub Repository

The entire automation system is available as a free open-source project on GitHub. To get started, simply clone the repository to your local machine:

  1. Visit the GitHub repo (link in video description)
  2. Click "Code" and copy the repository URL
  3. Open terminal and run: git clone [repository-url]
  4. Navigate into the project folder: cd wacm
  5. Install dependencies: npm install

This will download all the necessary files and set up the development environment on your computer. The project uses Node.js for the backend and includes a clean, responsive admin interface for managing your WhatsApp automations.

Step 2: Set Up Supabase Database

The solution uses Supabase (an open-source Firebase alternative) for storing contacts, messages, and automation rules. Here's how to configure it:

  1. Create a free account at supabase.com
  2. Start a new project and note your project URL
  3. Navigate to Project Settings → API to get your public and private keys
  4. Update the .env.local file with your Supabase credentials
  5. Run the database migrations to create all necessary tables

Pro Tip: Make sure to disable email confirmation in Supabase Authentication settings to streamline the signup process for your team members.

Step 3: Configure Meta Developer Portal

Connecting to Meta's official API requires setting up a developer account and configuring several security tokens:

  1. Go to developers.facebook.com and create a new app
  2. Select "Connect with customers through WhatsApp" as your use case
  3. Generate a permanent access token with messaging permissions
  4. Note your WhatsApp Business Account ID and Phone Number ID
  5. Create a system user in Business Settings and assign full permissions

At 8:45 in the video, you'll see exactly how to test your API connection by sending a message directly from the Meta developer portal to verify everything is working correctly before proceeding.

Step 4: Set Up WhatsApp Webhooks

Webhooks allow Meta to notify your system about incoming messages and status updates. Since your local development server isn't publicly accessible, we'll use ngrok to create a secure tunnel:

  1. Install ngrok and authenticate your account
  2. Run ngrok http 3003 to expose your local server
  3. Copy the ngrok HTTPS URL (it will look like https://abc123.ngrok.io)
  4. In Meta Developer Portal, configure your webhook URL as: [ngrok-url]/api/whatsapp/webhook
  5. Set a verify token and save the configuration

Now whenever someone messages your WhatsApp business number, Meta will forward that event to your local development server through the ngrok tunnel, allowing you to test automations in real-time.

Step 5: Create Your First Automation

With everything connected, you can now build powerful automations directly in the web interface:

  1. Log into your local CRM at localhost:3003
  2. Navigate to Automations → Create New
  3. Set trigger condition (e.g. "message contains keyword")
  4. Add actions (send reply, assign conversation, add tag, etc.)
  5. Activate and save the automation

Example: At 12:30 in the video, we create an automation that sends an instant reply when customers message the word "help", demonstrating how to set up keyword-based triggers that work immediately.
