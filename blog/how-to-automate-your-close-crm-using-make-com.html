The Close CRM Manual Update Problem

Most sales teams using Close.io spend 10+ hours weekly manually updating lead statuses and opportunity stages. At 2:15 in the video, you'll see how teams typically juggle between leads and opportunities tabs - often letting one system fall out of sync because maintaining both manually is unsustainable.

The core issue? Close CRM doesn't automatically connect Calendly bookings to opportunity stages. When a lead books, reschedules, or completes a call, someone must manually:

  • Check if the lead exists in Close
  • Create/update their record
  • Adjust the opportunity stage
  • Add relevant notes

Manual CRM updates create three costly problems: 1) Sales reps hate admin work, 2) Pipeline data becomes unreliable, and 3) Managers can't trust their forecasts. Automation solves all three.

Automation Architecture Overview

This Make.com automation creates a closed-loop system between Calendly and Close CRM. At 3:50 in the video, you'll see the trigger watching for three Calendly events:

  1. Call Booked: Creates/updates lead + creates opportunity
  2. Call Rescheduled: Updates existing lead + adjusts opportunity
  3. Call Cancelled: Updates lead status + triggers nurture sequence

A separate automation handles Call Completed events by moving opportunities to the next pipeline stage automatically. The entire system runs on these principles:

Golden Rule: No human should ever manually update lead status or opportunity stage after initial setup. The CRM runs itself.

Call Booked Automation Flow

When someone books through Calendly (4:30 in video), the automation:

  1. Searches Close for existing leads matching the email/phone
  2. Routes logic: If found → update existing lead; If not found → create new lead
  3. Creates opportunity attached to the lead with "Call Booked" stage
  4. Populates custom fields like lead source, meeting type, etc.

This eliminates the most common manual task - creating records for new bookings. Sales reps start calls with complete context without lifting a finger.

Call Rescheduled Handling

At 5:15 in the tutorial, you'll see how reschedules are handled:

  1. Finds the existing lead using Calendly's unique event ID
  2. Updates appointment details (date/time/location)
  3. Adds activity note about the reschedule reason if provided
  4. Maintains opportunity stage (still "Call Booked" just for new time)

This prevents the common issue where rescheduled calls disappear from the pipeline because someone forgot to update the record.

Call Cancelled Scenario

When calls cancel (5:45 timestamp), the automation:

  1. Updates lead status to "Cancelled" or your preferred label
  2. Adds cancellation note with any reason provided
  3. Triggers nurture sequence (optional based on your workflow)
  4. Removes from active pipeline while preserving the record

This automatic cleanup prevents dead opportunities from cluttering your active pipeline while maintaining data for future re-engagement.

Call Completed Automation

A separate but connected automation handles call completions (6:20 in video):

  1. Receives webhook from Calendly when meeting ends
  2. Finds the opportunity attached to the lead
  3. Moves to next stage (e.g., "Call Completed" → "Proposal Sent")
  4. Triggers next steps like sending contracts or scheduling follow-up

This is where most sales processes break down manually - reps complete calls but don't update CRM promptly, causing forecasting errors.

Handling Multiple Acquisition Sources

The system elegantly handles multiple lead sources (7:00 timestamp):

  1. Routes cold email leads through one automation path
  2. Routes landing page leads through a separate path
  3. Updates custom fields to track source accurately
  4. Maintains consistent CRM structure regardless of origin

This allows for accurate ROI tracking across channels while keeping all leads in one unified pipeline.
