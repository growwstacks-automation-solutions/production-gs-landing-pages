The Payment Link Problem

Sales teams using Whop face a frustrating limitation: payment links must be created with fixed amounts in advance. This forces closers to either limit negotiations to preset price points or scramble to create new links manually after each call. The result? Lost deals from rigid pricing and hours wasted on administrative tasks.

Traditional solutions require either pre-generating dozens of possible payment links (creating management chaos) or making clients wait while you manually create their specific link (hurting conversion rates). Neither approach scales for teams closing multiple deals daily.

83% of sales professionals report that flexible pricing increases close rates, yet most payment systems force fixed amounts that limit this flexibility.

How the Automation Works

This Zapier automation solves the problem by generating payment links dynamically when contracts are signed. Here's the breakthrough: instead of creating links beforehand, your closers simply input the negotiated price into the contract. When signed, the system automatically:

  1. Detects the signed contract via webhook
  2. Extracts the agreed payment amount
  3. Identifies if it's recurring or one-time
  4. Generates a Whop payment link at the exact amount
  5. Sends it directly to the client

The entire process happens in seconds with zero manual intervention, giving your team pricing flexibility while eliminating administrative work.

Setting Up the Webhook Trigger

The automation begins with a Zapier webhook trigger. This creates a unique URL you add to your contract system (like PandaDoc) that will notify Zapier whenever a contract is signed. At 1:15 in the video, you can see how simple this setup is:

Key benefit: The webhook passes all contract data to Zapier automatically, including the custom amount your closer negotiated and inputted.

When implementing this, you'll want to test the webhook with sample contract data to ensure all necessary fields (amount, client email, recurring flag) are being passed correctly before connecting the rest of your workflow.

Processing Contract Data

Contract systems often send data in complex formats that need parsing. The demo shows how a "Formatter by Zapier" step breaks down the raw contract data into usable pieces:

  • Splits comma-separated values into individual fields
  • Extracts the specific payment amount
  • Identifies dates and other metadata

This step is crucial because it transforms the messy webhook data into clean variables you can use to generate the payment link. Without it, the automation might fail to locate the exact payment amount in the contract data.

Handling Recurring Payments

The system intelligently handles both one-time and recurring payments through a simple checkbox in the contract. At 2:30 in the video, you'll see how this works:

  1. Closer checks a "Recurring Payment" box during contract creation
  2. This sends a "true" value through the webhook
  3. Zapier filters the automation down the recurring payment path

The recurring path uses slightly different API parameters to ensure proper billing cadence. This elegant solution means your team doesn't need separate contract templates or processes for different payment types.

Generating the Payment Link

The magic happens in the "Code by Zapier" step that calls Whop's API. At 3:15 in the demo, you can see the JavaScript code that:

  • Takes the parsed payment amount
  • Authenticates with your Whop API key
  • Makes the request to generate the link
  • Returns a fully functional payment URL

Testing this step immediately shows the generated link with the exact specified amount. The system even handles decimal amounts perfectly, so $1,999.97 creates a link for that precise total.

Sending to Clients

The final piece automates delivery to clients. Since the contract data includes their email, you can add a Gmail (or other email provider) step to:

  1. Pull the client's email from the contract
  2. Insert the generated payment link
  3. Send a professional payment request

Pro tip: Customize the email template to include contract details and next steps, creating a seamless client experience from signature to payment.
