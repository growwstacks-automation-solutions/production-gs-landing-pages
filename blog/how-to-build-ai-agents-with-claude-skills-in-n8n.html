Document Generation Workflow

The blank page syndrome plagues businesses creating routine documents. Our n8n workflow automates this by combining Claude's generation skills with n8n's scheduling and data handling capabilities. Here's how it works:

Step 1: Configure the API Request

Set up an HTTP request node in n8n with Claude's API endpoint. The magic happens in two special headers that enable skill execution:

  • anthropic-beta: skills-2025102
  • code-execution-2025825

Step 2: Craft Your Prompt

Include both a system prompt (defining the output format) and user prompt (content specifications). Crucially, always specify "create one file only" to avoid excessive token usage.

Step 3: Process the Response

The API returns a file ID which you'll use in a subsequent request to download the generated document. This two-step process ensures clean file handling.

Pro Tip: At 7:42 in the video, we demonstrate how to extract the file ID from the JSON response using n8n's schema viewer - a crucial step many miss when first implementing this workflow.
