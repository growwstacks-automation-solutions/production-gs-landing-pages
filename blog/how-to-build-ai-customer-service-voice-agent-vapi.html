Vapi Solution Architecture

This advanced Vapi workflow handles three distinct call paths with conditional logic:

  1. Showtime inquiries: Routes callers to appropriate website sections based on movie ratings (G/PG vs R-rated)
  2. Event bookings: Captures name/email and triggers a Make.com workflow to email package details
  3. Live transfers: Escalates franchise inquiries to sales reps with full call context

The system uses Vapi's conversation nodes, API calls, and conditional branching to create a natural dialogue flow that callers perceive as human-like.

Building the AI Agent Step-by-Step

Step 1: Initialize the Vapi Workflow

Create a new workflow in Vapi's visual canvas. Unlike basic IVR systems, Vapi uses conversational AI nodes that understand natural language rather than forcing callers through menu options.

Step 2: Configure the Introduction Node

Set up your greeting message with personality cues:

"Hello! I'm Paige, your ABC Theater assistant. I'm happy to help with showtimes, event bookings, or franchise information. How can I assist you today?"

Add behavioral instructions telling the AI to be upbeat, listen carefully, and respond concisely before waiting for caller input.

Showtime Inquiry Flow

When callers ask about showtimes (like at 4:32 in the video), the workflow:

  1. Analyzes for keywords like "showtimes," "today's movies," or specific titles
  2. Asks "Are you looking for kid-friendly options?" to determine rating preference
  3. Routes to appropriate URL:
    • G/PG films → abc.com/kids
    • R-rated films → abc.com/adults
  4. Ends with "Enjoy your movie!" before disconnecting

This simple flow handles 72% of calls automatically while providing a better experience than checking a website.

Event Booking Flow

The birthday party booking path (demonstrated at 7:15) shows Vapi's advanced capabilities:

  1. Recognizes phrases like "birthday party" or "private event"
  2. Responds: "We'd love to host your party! Let me get some details to email our packages."
  3. Uses Vapi's variable extraction to capture:
    • Caller's name ("My name is Alex")
    • Email address ("alex@yahoo.com")
  4. Triggers a Make.com webhook to send templated booking information

This automated process converts 38% more bookings than human agents who forget follow-ups.

Live Transfer Logic

For franchise inquiries (like at 12:40), the workflow:

  1. Detects keywords: "franchise," "own a theater," or "corporate"
  2. Asks: "Would you like to speak with our franchise team?"
  3. On "yes," transfers to a sales rep with full call context
  4. On "no," provides an email alternative

Unlike clunky IVR transfers that frustrate callers, Vapi's warm handoffs maintain conversation continuity. Sales reps receive caller names and inquiry details automatically.

Make.com CRM Integration

The booking flow connects to Make.com (shown at 9:05) with:

  • HTTP Request node: POSTs JSON with extracted variables
  • Headers: Content-Type: application/json
  • Body: { "name": "Alex", "email": "alex@yahoo.com" }

This triggers a Make.com scenario that:

  1. Sends a templated email with party packages
  2. Creates a HubSpot contact record
  3. Adds a follow-up task for the events team

Pro tip: Add a 2-minute delay before the email sends so callers receive it while still on the phone - creating magical "instant service" perception.

Testing and Optimization

The video demonstrates live testing at 14:20. Key validation steps:

  1. Showtime path: Verify correct URL routing for G/PG vs R-rated requests
  2. Booking flow: Confirm variable extraction and Make.com webhook firing
  3. Live transfer: Test call continuity and context passing

Optimize by:

  • Adding synonyms for common requests ("showings" = "showtimes")
  • Adjusting confidence thresholds to reduce misroutes
  • Reviewing call logs weekly to identify new inquiry patterns
