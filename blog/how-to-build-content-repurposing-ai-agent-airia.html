Creating the Content Repurposing Project

Projects organize related agents in Airia. Here's how to set up ours:

  1. Click the "+" icon next to "Project" in the left sidebar
  2. Select an icon (feather used in the demo)
  3. Name it "Content Repurposing Pipeline"
  4. Add a description outlining its purpose
  5. Click "Create" to finalize

The project dashboard will be empty initially. We'll add our first agent in the next section.

Configuring the Input Variables

Input variables define what data your agent expects. For this workflow:

  1. Click the input node at the top of the canvas
  2. Set input type to "Text"
  3. Create a "blog URL" variable (type: URL)
  4. Create a "transcript text" variable (type: string)

These variables allow the agent to accept either web links or raw text, making it flexible for different content sources.

Adding the Web Scraper Tool

The Firecrawl Web Scraper fetches content from URLs:

  1. Navigate to Tools in the Components menu
  2. Search for and select "Firecrawl Web Scraper"
  3. Configure with project-specific scope
  4. Drag the tool onto the canvas between input and output
  5. Set the URL parameter to use our "blog URL" variable

At 12:45 in the video, the presenter explains how this creates a reusable content ingestion point that works with any valid URL.

Implementing the Strategist Model

The strategist (Claude 3.7 Sonnet) analyzes content before repurposing:

  1. Add Claude 3.7 Sonnet from the Models library
  2. Scope it to our content project
  3. Drag the AI model block onto the canvas
  4. Configure with this prompt structure:
    • Role: Senior Content Strategist
    • Task: Analyze provided content
    • Output: Strategy brief (core arguments, key themes, tone)
    • Variables: Transcript text and scraped blog content

This creates the "brain" of our operation that all writers will reference.

Building the Content Writers

We'll create three specialized writers:

  1. LinkedIn Writer: Professional long-form posts with structure
  2. IG Captioner: Snappy, emoji-rich Instagram captions
  3. Viral Hook Generator: 5 attention-grabbing hook ideas

Each writer:

  • References the strategist's output
  • Has platform-specific formatting rules
  • Maintains brand voice consistency

At 22:10 in the tutorial, you can see how each writer transforms the same source material differently.

Connecting the Workflow Components

Airia's visual canvas makes wiring components intuitive:

  1. Connect input to both Firecrawl and strategist
  2. Link Firecrawl output to strategist
  3. Branch strategist to all three writers
  4. Route all writers to the output node

The presenter demonstrates this at 25:40, showing how you can preview valid connection points by hovering over nodes.

Testing and Publishing the Agent

Before going live:

  1. Enter test mode from the upper right corner
  2. Provide sample blog URL and transcript
  3. Trigger with "run" command
  4. Verify all outputs meet quality standards
  5. Click "Publish" to create versioned production agent

At 31:15, the video shows the live interface where you can simply paste a URL and get fully formatted social content in under 90 seconds.
