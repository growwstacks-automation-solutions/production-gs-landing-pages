Why Self-Hosted Voice AI Beats Paid Services

Voice AI services like Bappy charge per-minute fees that quickly add up for businesses making hundreds of calls. At $0.10-$0.30 per minute, a 10-minute sales call costs $1-$3 - multiply that by hundreds of calls per month and the costs become prohibitive.

Self-hosting with LightRAG and LiveKit eliminates these recurring fees. After the initial setup, your only costs are server hosting (typically $20-$100/month) and any SIP trunking fees for phone numbers. The system demonstrated in this guide handles knowledge retrieval, voice synthesis, and call routing entirely on your infrastructure.

Cost comparison: A business making 500 calls/month averaging 8 minutes each would pay $400-$1,200/month with a service like Bappy. The self-hosted solution costs under $100/month for equivalent usage.

System Architecture Overview

This solution combines three powerful open-source technologies into a complete voice AI system:

1. LightRAG handles the Retrieval-Augmented Generation (RAG) component, searching your knowledge base to provide accurate answers. Unlike generic chatbots, it references specific documents you upload to answer questions precisely.

2. LiveKit provides the real-time voice infrastructure, handling call processing, voice synthesis, and SIP trunk integration. It's fully self-hostable and supports both inbound and outbound calling.

3. N8N automates workflows between systems - logging calls to your CRM, triggering follow-ups, or integrating with other business tools. The visual workflow builder makes complex automations accessible without coding.

Key benefit: The web UI included with LightRAG lets non-technical staff manage the knowledge base and monitor calls without needing command line access.

Prerequisites and Setup

Before installing, you'll need a Linux server (Ubuntu recommended) with these components:

Step 1: Server Requirements

  • Ubuntu 22.04 LTS or newer
  • Docker and Docker Compose installed
  • Minimum 4GB RAM (8GB recommended for production)
  • 2+ CPU cores
  • 50GB+ storage (for knowledge base documents)

Step 2: Initial Server Setup

Connect to your server via SSH and run these commands to prepare the system:

 sudo apt update && sudo apt upgrade -y sudo apt install -y docker.io docker-compose sudo systemctl enable docker sudo usermod -aG docker $USER

After running these, log out and back in for the Docker permissions to take effect. The video at 4:12 shows this process in detail.

Installing LightRAG

LightRAG provides the core AI capabilities with a user-friendly web interface. Follow these steps to install:

Step 1: Clone the Repository

 git clone https://github.com/lightrag/lightrag.git cd lightrag

Step 2: Configure Environment Variables

Edit the .env file to set your configuration:

 nano .env

Key settings to configure:

  • WEB_UI_TITLE - Your organization's name
  • API_KEY - Generate a secure key for API access
  • RERANK_MODEL - Set to "colbert" for best results
  • LLM_PROVIDER - Configure for your preferred model

Step 3: Start the Services

 docker-compose up -d

This will pull all required containers and start the system. Initial setup may take 5-10 minutes.

Note: The first time you upload documents to the knowledge base, LightRAG needs to generate embeddings. This process takes longer (up to 30 minutes for large documents) but only happens once per document.

Configuring LiveKit

LiveKit handles the voice communication components of the system. Here's how to set it up:

Step 1: Install LiveKit Server

 curl -sSL https://get.livekit.io | bash livekit-server --dev

Step 2: Configure SIP Trunking

For phone call capabilities, you'll need to:

  1. Purchase a SIP trunk service (Twilio, Flowroute, etc.)
  2. Configure the SIP credentials in LiveKit's settings
  3. Set up call routing rules

The video at 18:45 demonstrates configuring a SIP trunk with a provider and testing inbound/outbound calls.

Step 3: Connect to LightRAG

Edit LightRAG's configuration to point to your LiveKit instance:

 LIVEKIT_URL=ws://your-server:7880 LIVEKIT_API_KEY=your-key LIVEKIT_API_SECRET=your-secret

Production tip: For better voice quality, configure LiveKit to use OPUS codec with 24kHz sampling rate in the audio configuration.

N8N Automation Integration

N8N connects your voice agent to other business systems. Key automation use cases:

Call Logging Workflow

  1. Trigger: Incoming/outgoing call event from LiveKit
  2. Extract: Caller number, duration, transcript
  3. Action: Create CRM record (HubSpot, Salesforce, etc.)

Follow-Up Sequence

  1. Trigger: Call ends with specific intent (e.g. "interested")
  2. Action: Send follow-up email with relevant information
  3. Action: Schedule reminder in calendar

Knowledge Base Updates

  1. Trigger: New document added to Google Drive/Dropbox
  2. Action: Process and upload to LightRAG automatically

Setting Up Your Knowledge Base

Your voice agent's effectiveness depends on the quality of your knowledge base. Follow these best practices:

Document Preparation

  • Convert all materials to text format (PDF, Word, HTML work)
  • Structure documents with clear headings and sections
  • Include FAQs and common customer questions

Upload Process

  1. Log in to LightRAG web UI (http://your-server:3000)
  2. Navigate to "Knowledge Base" → "Upload"
  3. Select files and assign appropriate tags
  4. Wait for embedding process to complete

Pro tip: Start with 5-10 core documents covering your most common inquiries, then expand over time based on call analytics.

Voice and Prompt Customization

Tailor your agent's personality and responses for your brand:

Voice Selection

  • Choose from multiple voice options (male/female, accents)
  • Adjust speech rate and pitch to match your brand
  • Test different voices with sample calls

Prompt Engineering

Edit the system prompt to control:

  • Response tone (professional, friendly, etc.)
  • Length of answers (concise vs detailed)
