Workflow Overview: From Voice to Email

This n8n automation creates a complete pipeline from voice input to email output. The five key components work together seamlessly:

  1. Telegram Trigger - Listens for new messages and detects voice notes
  2. Audio Processor - Identifies and routes voice notes for transcription
  3. OpenAI Whisper - Converts speech to text with 95%+ accuracy
  4. AI Email Composer - Structures the raw text into professional email format
  5. Gmail Sender - Delivers the polished email to the intended recipient

The magic happens in the routing logic that connects these components while handling errors gracefully. Notice at 4:30 in the video how the workflow branches based on whether the input is text or audio.

Step 1: Telegram Trigger Setup

The foundation of any automation is a reliable trigger. For this workflow, we use Telegram's "On Message" trigger because it captures both text and voice notes. Here's how to configure it properly:

Configuration Details:

  • Create a new Telegram bot via @BotFather and note the API token
  • In n8n, add the Telegram trigger node and authenticate with your bot token
  • Enable all message types in the trigger settings (text, audio, voice)
  • Test the connection by sending a message to your bot

At 1:45 in the video, you'll see the critical step of testing the Telegram connection to verify you're receiving the correct chat ID and message data structure.

Pro Tip: Always test your trigger with both text and voice messages to ensure it captures all expected input types before building the rest of the workflow.

Step 2: Audio Processing Logic

Not every Telegram message will be a voice note, so we need smart routing. The Switch node acts as our traffic controller, sending different message types down the appropriate paths:

Routing Rules:

  1. Text Path: Checks if the message contains text (simple string exists check)
  2. Audio Path: Verifies the presence of a voice note file (checks for audio object)
  3. Error Path: Catches any unexpected message formats for handling

The video at 3:20 demonstrates the importance of proper data pathing - notice how the workflow examines the incoming message structure to correctly identify voice notes versus text messages.

Step 3: OpenAI Transcription

With the audio file properly routed, we now convert speech to text using OpenAI's Whisper API. This step requires:

  • Valid OpenAI API credentials (use environment variables for security)
  • Proper file handling from Telegram's binary data format
  • Error handling for transcription failures

At 5:50 in the tutorial, you'll see the moment where the raw audio becomes usable text. The key is configuring the OpenAI node to accept Telegram's file format and output clean transcription text.

Accuracy Note: In our tests, Whisper achieved 92-97% accuracy on clear English voice notes under 2 minutes. Background noise or strong accents may require additional processing.

Step 4: AI Email Generation

Raw transcription text needs polishing before becoming a professional email. Our AI agent node transforms the text using this prompt structure:

 "Create proper email in HTML format. Use this transcribed voice note: {{transcription}}.      Recipient: {{contact_name}}. Maintain professional tone and fix any transcription errors."

The video at 7:30 shows how changing just a few words in the prompt can dramatically alter the email output quality. We recommend:

  • Storing your email template as a variable for easy updates
  • Including brand voice guidelines in the prompt
  • Adding conditional logic for different email types

Step 5: Gmail Delivery System

The final step sends our AI-generated email through Gmail. Configuration requires:

  1. Gmail API credentials with sending permissions
  2. Proper mapping of email fields (to, subject, body)
  3. HTML email support enabled
  4. Error handling for delivery failures

At 9:10 in the video, watch how the workflow maps the AI-generated content to Gmail's required fields. The key is ensuring your variables match exactly - capitalization matters in n8n's JSON paths.

Critical Error Handling

No automation is complete without robust error management. Our workflow includes three safety nets:

1. Input Validation: The initial switch node filters invalid message types before processing

2. Transcription Fallback: If Whisper fails, the workflow notifies the sender to re-record or type their message

3. Email Verification: Before sending, the system checks for required fields and valid email formats

The video at 11:45 demonstrates the error path in action when testing with malformed input. Notice how the workflow fails gracefully rather than crashing.
