What Is Model Context Protocol (MCP)?

Model Context Protocol (MCP) represents a breakthrough in how AI agents interact with WordPress.com. Traditional automation requires writing custom API integrations for every task. MCP changes this by providing a standardized way for agents to understand and execute natural language commands against your WordPress sites.

With MCP, your AI agent becomes a powerful assistant that can search posts, check analytics, manage users, and perform dozens of other administrative tasks - all through conversational prompts. The protocol handles authentication securely through OAuth, so you never need to share passwords or API keys.

Key benefit: MCP reduces the time spent on routine WordPress management by up to 80% according to early adopters. Agents can complete complex multi-step workflows with a single natural language command.

How to Enable MCP on WordPress.com

Enabling MCP access for your WordPress.com account takes less than two minutes. The feature is available on all paid plans and can be toggled on with just a few clicks.

Step 1: Navigate to Your Profile

Visit wordpress.com/me or hover over your name in the top right corner and select "My Profile" from the dropdown menu.

Step 2: Locate MCP Settings

In the sidebar menu, find and click on the MCP settings option. This will display the MCP configuration panel.

Step 3: Enable MCP Access

Toggle the "Enable MCP Tool Access" switch to the on position. You can optionally enable or disable specific tools based on your needs. Changes save automatically.

Pro tip: Start by enabling just a few essential tools, then gradually add more as you become comfortable with MCP. This minimizes potential security exposure while you're learning the system.

Technical Requirements for Connection

Connecting your AI agent requires some basic technical setup. The most important requirement is having Node.js version 22 or newer installed on your computer.

The MCP-WP-COM proxy tool (which handles communication between your agent and WordPress.com) runs on Node.js. You can check your current version by opening a terminal and running node -v. If you need to upgrade, visit nodejs.org for installation packages.

Other requirements include:

  • An MCP-compatible AI agent (most modern frameworks support it)
  • Active WordPress.com paid plan
  • Internet connection for OAuth authentication

Configuring Your AI Agent

Once MCP is enabled on your WordPress.com account, you'll need to configure your AI agent to connect. The process involves creating a JSON configuration file with your specific settings.

Obtaining Your Configuration

From the MCP settings page in your WordPress.com account, click "Configure MCP Client." This displays pre-configured JSON snippets for popular applications like VS Code and Law Desktop.

VS Code Example

For VS Code users, create a file called mcp.json in your project workspace and paste the configuration. VS Code will automatically detect this and add an MCP server connection toolbar.

First connection: When connecting for the first time, a browser window will open for OAuth authentication. Sign into WordPress.com and approve the agent's access when prompted. This establishes the secure connection.

Testing the Connection

After setup, try a simple prompt like "List my sites" to verify everything is working. The agent should return your WordPress.com sites with basic information about each.

Powerful Example Prompts to Try

Once connected, your AI agent can handle a wide variety of WordPress management tasks through natural language. Here are some practical examples to get you started:

Content Management

  • "Show posts modified in the last 3 days"

User Administration

  • "Display traffic sources for our most popular post this month"

Advanced tip: Combine multiple operations in a single prompt like "Find our top performing post this week and email the author a congratulatory message." The agent will execute this as a multi-step workflow.

Security and Access Control

MCP connections are designed with security as a priority. The system uses WordPress.com's OAuth implementation, so your password is never shared with or stored by the AI agent.

Key security features include:

  • Granular tool access control (enable only what you need)
  • Session-based authentication with expiration
  • Centralized access revocation
  • No persistent credentials stored locally

You can review and revoke MCP connections at any time by visiting your WordPress.com account's "Security → Connected Apps" section. Look for the "WordPress MCP Connector" entry.
