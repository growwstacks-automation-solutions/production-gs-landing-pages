Step 1: Setting Up Your Brevo Connection

Begin by logging into your Make.com account and creating a new scenario. Click the "Create Scenario" button in the upper right corner, which will open a blank canvas for building your integration.

Search for "Brevo" in the apps list and select it as your first module. You'll need to authenticate with your Brevo API key, which you can find in your Brevo account settings under "SMTP & API." Name your connection something recognizable like "Brevo Production" and save it.

Choosing Your Trigger

Brevo offers several trigger options in Make.com. For this integration, we recommend using either:

  • New Contact - Triggers when a contact is added to Brevo
  • Updated Contact - Triggers when existing contact data changes

Select the trigger that matches your primary use case. You can always add additional scenarios later for different workflows.

Step 2: Connecting Your GoHighLevel Account

After setting up your Brevo trigger, click the plus icon to add another module. Search for "GoHighLevel" and select it as your action app. Authenticate using your GoHighLevel API key (found in Settings > API & Integrations).

Choose the action you want to perform in GoHighLevel when the Brevo trigger fires. Common options include:

  • Create/Update Contact
  • Add to Workflow
  • Create Task
  • Send SMS/Email

For basic contact syncing, select "Create/Update Contact." This ensures the contact will be added to GoHighLevel if they don't exist, or updated if they do.

Step 3: Mapping Your Contact Fields

The most crucial part of the setup is field mapping - telling Make.com which Brevo fields correspond to which GoHighLevel fields. The platform will automatically suggest common mappings like email to email, but you'll want to review each one carefully.

Pay special attention to:

  • Custom fields that might have different names in each system
  • Tags/labels that help categorize contacts
  • Phone number formats which sometimes need transformation

Pro Tip: Create a test contact in Brevo first and run a single operation to verify all fields map correctly before activating the full integration.

Once your field mapping is complete, save your scenario and toggle it to active. New contacts added to Brevo will now automatically flow into GoHighLevel with all their associated data.
