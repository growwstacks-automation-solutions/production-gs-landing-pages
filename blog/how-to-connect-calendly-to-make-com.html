Step 1: Get Your Calendly API Token

Your Calendly API token acts as a secure key that lets Make.com access your scheduling data. Here's how to generate it:

Step 1.1: Access Calendly's Integration Settings

Log into your Calendly account and navigate to Integrations → Webhooks API. This is where you'll find tools for connecting Calendly to external platforms like Make.com.

Step 1.2: Create a New Token

Click Get a Token Now and follow the prompts. You'll need to:

  • Name your token (e.g., "Make.com Integration")
  • Select the appropriate permissions (choose "Read/Write" for full functionality)
  • Confirm the creation

Step 1.3: Copy Your Token

After creation, Calendly will display your new API token. Copy this immediately - it won't be shown again. Store it securely in a password manager or encrypted note.

Pro Tip: Tokens typically expire after 1 year. Set a calendar reminder to renew it 2 weeks before expiration to avoid workflow interruptions.

Step 2: Create a Make.com Scenario

In Make.com, scenarios are automated workflows that connect apps and services. We'll build one that reacts to Calendly events.

Step 2.1: Start a New Scenario

From your Make.com dashboard, click Create a new scenario. Search for and select Calendly as your trigger app.

Step 2.2: Choose Your Trigger Event

Select When an event is scheduled or canceled as your trigger. This tells Make.com to watch for changes in your Calendly calendar.

Step 2.3: Connect Your Calendly Account

Paste the API token you copied earlier when prompted. Make.com will verify the connection and show your Calendly organization details if successful.

Important: At 1:45 in the video tutorial, we show how to test your connection before proceeding. This quick check can save hours of debugging later.

Step 3: Set Up the Webhook Connection

Webhooks are the communication channels between Calendly and Make.com. Here's how to configure them properly.

Step 3.1: Configure the Webhook Module

In your Make.com scenario, add a Webhook module after the Calendly trigger. Select Custom Webhook as the type.

Step 3.2: Enter Webhook Details

Make.com will generate a unique URL for your webhook. Copy this URL and return to Calendly's Webhooks API page to paste it into the Endpoint URL field.

Step 3.3: Select Event Types

Choose which Calendly events should trigger your workflow. At minimum, select invitee.created (new bookings) and invitee.canceled.

Advanced Option: For team accounts, you can create separate webhooks for different event types or team members, giving you granular control over your automation.

Step 4: Test and Save Your Workflow

Before going live, verify everything works as expected.

Step 4.1: Run a Test Booking

Schedule a test appointment in Calendly using your own email. Watch the Make.com scenario to confirm it triggers correctly.

Step 4.2: Check Data Transfer

In Make.com, use the Run once feature to process your test booking. Verify all appointment details (name, email, date/time) transfer accurately.

Step 4.3: Save and Activate

Once testing succeeds, click Save and toggle your scenario to Active. Your integration is now live!

Monitoring Tip: Make.com's scenario history shows all triggered events. Check it periodically during the first week to catch any issues early.
