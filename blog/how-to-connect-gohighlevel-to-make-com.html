  • Eliminates duplicate data entry - Contacts created in GoHighLevel automatically flow to other systems

Step 1: Set Up Webhook in Make.com

The first step happens in Make.com (formerly Integromat), where we'll create the endpoint that will receive data from GoHighLevel.

Detailed Instructions:

  1. Log in to your Make.com account and create a new scenario
  2. Select "Build from scratch" and close the template selector
  3. Click the big plus button to add your first module
  4. Search for and select "Webhooks" from the app list
  5. Choose "Custom Webhook" as the trigger type
  6. Click "Create Webhook" and name it (e.g., "GoHighLevel Integration")
  7. Save the webhook and copy the generated URL to your clipboard

Pro Tip: Make.com will automatically detect the data structure when you first test the connection, so you don't need to manually map fields at this stage.

Step 2: Configure GoHighLevel Workflow

Now we'll switch to GoHighLevel to set up the automation that will send data to Make.com whenever your specified trigger occurs.

Detailed Instructions:

  1. Log in to your GoHighLevel account and navigate to Automations
  2. Create a new workflow and select "Start from scratch"
  3. Choose your trigger event (e.g., "Contact Created", "Path Stage Updated")
  4. Save the trigger and close the configuration panel
  5. Add a new action step and select "Webhook"
  6. Paste the Make.com webhook URL you copied earlier
  7. Set the method to "POST"
  8. Review the standard data fields that will be sent automatically
  9. Optionally add any custom fields you want to include
  10. Save the action

Step 3: Test and Save Integration

The final step is to test the connection and complete the setup in Make.com.

Detailed Instructions:

  1. In Make.com, click "Save" in the top right corner
  2. In GoHighLevel, click "Test Workflow" and select a contact
  3. Click "Run Test" to send sample data to Make.com
  4. Return to Make.com - you should see the test data appear
  5. Make.com will automatically determine the data structure
  6. Click "Save" in Make.com to complete the setup

Important: Always save your Make.com scenario before running the test in GoHighLevel. This ensures the webhook is active and ready to receive data.

Common Mistakes to Avoid

While the integration process is straightforward, these pitfalls can cause connection failures:

  • Not saving the Make.com scenario before testing - The webhook won't be active until you click Save
  • Using GET instead of POST method - POST is required for sending data from GoHighLevel
  • Forgetting to include key fields - Double-check that all necessary contact data is being sent
  • Not testing with real data - Always verify with an actual contact record

If your connection isn't working, go back through each step to verify the webhook URL is correct and all settings are properly saved in both platforms.
