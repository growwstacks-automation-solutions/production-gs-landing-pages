Manually exporting and importing contacts between Google and other apps wastes hours every month. This n8n integration automatically syncs your Google Contacts with any CRM, email platform, or database - no coding required. Follow these steps to set it up in minutes.

Why Automate Google Contacts

Most businesses store contacts in Google but need those contacts in other systems like CRMs, email platforms, or internal databases. The traditional approach - manual CSV exports and imports - is time-consuming, error-prone, and quickly becomes outdated.

By connecting Google Contacts to n8n, you create a live sync that automatically updates contact information across all your systems. When a customer updates their phone number in Google, that change flows instantly to your CRM, support tools, and marketing platforms.

Time savings: Businesses using this automation save 3-5 hours per week previously spent on manual contact management. More importantly, they eliminate the errors and data inconsistencies that come with manual processes.

Step 1: Create Google Cloud Project

All Google API access requires a Google Cloud project. This is free to set up and only takes a few minutes:

  1. Go to Google Cloud Console
  2. Click "Start Free" and complete account setup (requires credit card for verification but won't charge you)
  3. Click "Create Project" and give it a name like "n8n Google Contacts Integration"

The project creation takes about 30 seconds. Once complete, you'll see your new project in the dashboard. Make sure this project is selected before proceeding to Step 2.

Step 2: Configure OAuth Consent

Google requires you to configure an OAuth consent screen before creating API credentials. This determines what users will see when authorizing access to their contacts.

  1. In Google Cloud Console, navigate to "APIs & Services" > "OAuth consent screen"
  2. Select "External" user type (this allows you to access your own contacts)
  3. Enter your application name (e.g., "My n8n Integration") and support email
  4. Add the required scopes (we'll use just the Google Contacts scope)
  5. Click "Save and Continue" through the remaining screens

Important: After creating the consent screen, you must click "Publish App" to move it from testing to production. This is required for the integration to work properly.

Step 3: Create OAuth Credentials

Now we'll create the actual credentials that n8n will use to authenticate with Google:

  1. In Google Cloud Console, go to "APIs & Services" > "Credentials"
  2. Click "Create Credentials" and select "OAuth client ID"
  3. Choose "Web application" as the application type
  4. Give it a name like "n8n Google Contacts"
  5. Add the authorized redirect URI from n8n (shown when adding the Google node)

After creation, you'll see your Client ID and Client Secret. Keep this page open - we'll need these values in Step 5 when connecting to n8n.

Step 4: Enable Contacts API

Even with credentials created, we need to explicitly enable the Google Contacts API for our project:

  1. In Google Cloud Console, go to "APIs & Services" > "Library"
  2. Search for "Google People API" (this is the current name for Contacts API)
  3. Click on the API and press "Enable"

The API enables instantly. There are no usage limits or quotas to configure for basic contact access. Your project now has permission to access Google Contacts through the API.

Step 5: Connect to n8n

With the Google Cloud setup complete, we can now connect to n8n:

  1. In your n8n workflow, add a "Google Contacts" node
  2. Click "Create New Credentials"
  3. Paste your Client ID and Client Secret from Google Cloud Console
  4. Click "Sign in with Google" and authorize the connection

You may see a warning that the app isn't verified - this is normal for self-configured integrations. Click "Advanced" then "Go to [Your App Name]" to proceed. Once authorized, your Google Contacts will be available in n8n.

Pro tip: Create a test workflow that simply lists contacts to verify the connection works before building more complex automations.
