Why Connect JotForm AI to Box?

Businesses using AI agents often struggle with document management. Conversations disappear into chat histories, and important files get buried in email attachments. By connecting your JotForm AI agent to Box, you create a permanent, organized record of all interactions.

This integration solves three key problems: lost customer documents, disorganized file storage, and manual upload processes. Instead of asking users to email files separately, your AI agent can request and store them directly during the conversation.

Key benefit: The integration automatically creates a dedicated Box folder for each AI agent, keeping all documents organized by conversation type and date.

Step 1: Select Your AI Agent

Start by choosing which JotForm AI agent you want to connect to Box. This could be a customer service bot, document collection agent, or any other AI assistant you've created.

In the JotForm dashboard, navigate to your list of AI agents and select the one you want to configure. For demonstration purposes, we're using a travel planning agent that needs to store booking confirmations and travel documents.

Step 2: Access the Tools Tab

Once you've selected your agent, click on the "Training" tab in the editor. This is where you'll find all the integration options for enhancing your AI agent's capabilities.

Within the Training section, locate and click on the "Tools" tab. This is where JotForm lists all available third-party integrations. Scroll through or search for "Box" in the tools list.

Pro tip: If you don't see Box listed, make sure you're using a JotForm plan that supports third-party integrations. Some basic plans may have limitations.

Step 3: Connect to Box

Click on the Box integration to begin the connection process. You'll be prompted to authenticate with your Box account credentials.

The system will request permission to access your Box account. Make sure you're logged into the correct Box account (personal or business) before granting access. The integration needs permissions to create folders and upload files on your behalf.

Step 4: Configure Folder Settings

After successful authentication, you can customize where files will be stored in Box. By default, the integration creates a folder named after your AI agent.

You can change this to any folder name that makes sense for your workflow. For example, "Customer Documents" or "Travel Bookings". Consider using a naming convention that will help you easily identify the source of files months later.

Step 5: Set Up File Triggers

The most important step is configuring when your AI agent should request and send files to Box. This happens through carefully crafted prompts that tell the agent when to activate the file upload feature.

A good example prompt would be: "When a document is needed, present a file upload box first, then send it to Box." You can make this more specific based on your use case, like "When a customer mentions they have a contract, ask them to upload it and automatically store it in Box."

Best practice: Test your prompts thoroughly to ensure the AI agent only requests files when absolutely necessary, avoiding unnecessary upload requests that might frustrate users.

Testing Your Integration

Always test your Box integration before going live. Use the preview mode to simulate a conversation where a file would be requested.

Upload a test document and verify that it appears in your designated Box folder with the correct name and metadata. Check that the folder structure matches your expectations and that you can easily locate the file later.
