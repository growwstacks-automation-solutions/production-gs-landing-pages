Getting Started in Make.com

To begin, log into your Make.com account or sign up for a free trial. The platform uses a credit-based system, so simple HubSpot automations won't consume many credits. Once logged in:

  1. Click Create a new scenario from your dashboard
  2. Give your scenario a clear name like "HubSpot Contact Sync"
  3. The visual builder opens where you'll add apps and define data flow

At the 1:15 mark in the video tutorial, you'll see how to name your scenario for easy identification later. This becomes crucial as you build multiple automations.

Understanding the HubSpot Connector

Make.com offers a native HubSpot connector designed specifically for working with CRM objects. Unlike generic HTTP modules, this connector:

  • Pre-maps all standard HubSpot fields (contacts, companies, deals, tickets)
  • Supports custom properties you've added in HubSpot
  • Handles associations between objects (contact-to-company, deal-to-contact)
  • Includes built-in error handling for rate limits

To add HubSpot to your scenario, click the plus icon and search for "HubSpot CRM". You'll see two connector options - choose the one marked "HubSpot CRM" (not the legacy version).

Triggers vs Actions: Which to Use When

Make.com workflows begin with either a trigger (something happens) or an action (do something). For HubSpot integrations:

Start with a trigger when you want your automation to react to changes in HubSpot, like new contacts or updated deals. Start with an action when pushing data into HubSpot from another app.

Common HubSpot triggers include "New Contact", "Updated Deal", and "Added to List". Popular actions include "Create Contact", "Update Company", and "Add Deal". At 2:30 in the video, we demonstrate choosing between trigger-first and action-first approaches.

Authenticating Your HubSpot Account

The first time you add a HubSpot module, you'll need to authenticate the connection:

  1. Click "Add a connection" when prompted
  2. A secure window opens to hubspot.com - log in if needed
  3. Authorize Make.com to access your CRM data
  4. Select the correct portal if you manage multiple HubSpot accounts

This connection saves securely in Make.com and can be reused across scenarios. You only need to authenticate once per HubSpot portal.

Configuring HubSpot Modules

After authentication, configure your HubSpot module based on whether it's a trigger or action:

For triggers: Select the object (contact/company/deal) and set filters to specify which records should activate your workflow. For example, only trigger for contacts with "Lead Status = New".

For actions: Choose the operation (create/update/search) and map data from previous modules into HubSpot fields. Required fields are marked with an asterisk.

The video at 3:45 shows how to map a Google Sheets row to HubSpot contact fields, including handling custom properties.

Testing Your Workflow

Before activating your scenario, test it thoroughly:

  1. Click Run once to process a single execution
  2. Check the execution log for errors (red indicators)
  3. Verify the data appears correctly in HubSpot
  4. Adjust field mappings if needed

Pro tip: For triggers, create test records in HubSpot first. For actions, use sample data that matches your real workflow inputs.
