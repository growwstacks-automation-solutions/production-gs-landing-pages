Step 1: Webhook Setup in Make.com

A webhook acts as the digital doorbell between ManyChat and Make.com - when new data arrives from your chatbot, it "rings the bell" to trigger your automation. Here's how to set it up:

Step 1.1: Create a New Scenario

Log into Make.com and create a new scenario. Search for "ManyChat" in the trigger apps and select "Watch Incoming Data." This creates the webhook that will receive information from your chatbot.

Step 1.2: Connect Your ManyChat Account

You'll need your ManyChat API key, found under Settings → API in your ManyChat dashboard. Paste this into Make.com to establish the connection between the two platforms.

Step 1.3: Name Your Webhook

Choose a descriptive name like "Send Leads to MailerLite" so you can easily identify this integration later. The webhook URL will generate automatically once saved.

Pro Tip: Always test your webhook immediately after creation by running the scenario once (click "Run Once") before connecting it to ManyChat.

Step 2: ManyChat Automation Configuration

With your Make.com webhook ready, the next step is telling ManyChat when and what data to send. This happens through a simple automation:

Step 2.1: Create a New Automation

In ManyChat, go to Automations → Create New. Start with a blank canvas and add a trigger based on user interaction (like submitting an email).

Step 2.2: Add the Make.com Action

After your data collection step (like asking for an email), add an action and select "Trigger Make." Choose your webhook from the dropdown (it will appear automatically if correctly connected).

Step 2.3: Set Live and Test

Turn the automation live in ManyChat, then test it by going through the flow as a user would. Watch the data appear in Make.com to confirm everything works.

Common Mistake: Forgetting to click "Run Once" in Make.com after creating your webhook. This primes the system to recognize your ManyChat data structure.

Testing Your Integration

Thorough testing prevents lost leads once your integration goes live. Follow this verification process:

1. Initial Data Flow Test

Run through your ManyChat flow as a user while watching the Make.com scenario. Verify all expected data fields appear correctly.

2. Destination App Test

Add your destination app (like MailerLite) as the next step in Make.com. Test with a fake email to confirm the contact appears where expected.

3. Error Condition Test

Intentionally trigger errors (like an invalid email) to ensure your error handling works as intended.

Essential Error Handling

Without proper error handling, a single failed integration could stop all subsequent automations. Protect your workflow with these safeguards:

Error Handler Setup

Right-click any module in Make.com and select "Add Error Handler." Connect it to a ManyChat flow that notifies you when failures occur.

Notification Automation

Create a separate ManyChat automation that messages you (or your team) when the integration fails, including details about the error and affected contact.

Critical Feature: Error handlers keep your scenario running even if individual contacts fail, preventing lead loss during temporary issues.

Advanced Features: Filters & Routers

Once your basic integration works, these powerful features add precision to your automations:

Filters

Add conditions to control when integrations run. Example: Only add to MailerLite if the contact provided an email.

Routers

Create branching logic to send data different places based on conditions. Example: Route webinar signups to Zoom while product inquiries go to your CRM.
