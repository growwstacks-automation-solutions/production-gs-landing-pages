Preparation Steps Before Connecting

Before connecting n8n to Google Drive, you'll need three essential components: a Google Cloud project with the Drive API enabled, proper authentication credentials, and clear permissions for your target Drive folders.

At the 1:15 mark in the video tutorial, we demonstrate how to create OAuth credentials in Google Cloud Console. This includes setting authorized redirect URIs that must match your n8n instance URL exactly - a common stumbling block for first-time setups.

OAuth vs Service Accounts: Which to Choose

The authentication method you select dramatically impacts what your automations can achieve. OAuth connects as a specific user (ideal for personal workflows), while service accounts authenticate as applications (better for server automations).

For business use cases, service accounts provide more reliable 24/7 access to shared drives without requiring user sign-in. However, they require additional configuration in Google Workspace admin console to access shared team folders.

Rule of thumb: Use OAuth for personal workflows tied to your account, service accounts for business automations accessing shared drives.

Step-by-Step Connection Guide

Step 1: Add Google Drive Node

In your n8n workflow, click "Add Node" and search for "Google Drive". Select the Google Drive node from the results. This provides access to all Drive operations within your workflow.

Step 2: Configure Authentication

Click the credentials dropdown and select "Create New". Choose between OAuth or Service Account authentication based on your use case (see previous section).

Step 3: Enter Credentials

For OAuth: Enter your client ID and secret from Google Cloud Console. Ensure the redirect URI matches your n8n instance URL exactly.

For Service Accounts: Upload the JSON key file downloaded from Google Cloud Console.

Step 4: Complete Authorization

Follow the on-screen prompts to authorize access. For OAuth, you'll need to sign in to Google and grant permissions. Service accounts authenticate automatically with the key file.

Step 5: Select Operation

Choose your desired Drive operation (upload file, list files, create folder etc.) from the node's action dropdown. Each operation has specific configuration fields.

In summary: Add node → Choose auth method → Enter credentials → Authorize → Select operation → Configure fields → Test workflow.

Testing Your First Google Drive Workflow

After connecting, immediately test with a simple operation like listing files in a folder. This verifies your credentials work before building complex workflows.

Use n8n's "Execute Node" feature to test individual steps. Check for proper file permissions if you encounter access errors - service accounts need explicit access to shared drives.

Security Best Practices

Always follow the principle of least privilege when granting permissions. For OAuth, review the scopes being requested. For service accounts, limit Drive access to only necessary folders.

Store credentials securely - never commit them to version control. Consider using n8n's credential encryption or external secret management for production deployments.
