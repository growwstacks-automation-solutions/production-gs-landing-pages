Step 1: Get Your OpenAI API Key

Your OpenAI API key is the secure access token that allows Make.com to communicate with OpenAI's servers. Without it, the integration won't work. Here's how to obtain it:

Detailed Steps:

  1. Log in to your OpenAI account at platform.openai.com
  2. Navigate to the API keys section (usually under your account settings)
  3. Click "Create new secret key"
  4. Give your key a descriptive name (e.g., "Make.com Integration")
  5. Copy the generated key immediately - you won't be able to see it again

Security Tip: Treat your API key like a password. Never share it in emails, messages, or public forums. If compromised, revoke it immediately and generate a new one.

Step 2: Create a Make.com Scenario

Scenarios are the building blocks of automation in Make.com. Think of them as workflows that execute a series of actions based on triggers. Before connecting to OpenAI, you need a scenario to house the integration.

How to Set Up:

  1. Log in to your Make.com account
  2. Click "Create a new scenario"
  3. Choose your trigger (could be a schedule, webhook, or other app connection)
  4. Name your scenario descriptively (e.g., "AI Content Generator")

At the 1:15 mark in the video tutorial, you can see exactly where to locate the scenario creation interface within Make.com's dashboard.

Step 3: Add OpenAI Module

With your scenario created, it's time to add OpenAI's capabilities. Make.com offers dedicated modules for ChatGPT, Whisper, and DALL-E, allowing you to choose the right AI tool for your specific automation needs.

Implementation Steps:

  1. In your scenario, click "Add module"
  2. Search for "OpenAI" in the app directory
  3. Select the specific AI service you need (ChatGPT, Whisper, or DALL-E)
  4. Choose the action you want to perform (e.g., "Create chat completion")

Pro Tip: Start with simple ChatGPT text generation before attempting more complex multi-model workflows. This helps you understand the basic connection before scaling up.

Step 4: Configure API Connection

This is where your OpenAI API key comes into play. The connection configuration tells Make.com how to authenticate with OpenAI's servers and which account to bill for usage.

Configuration Process:

  1. In the OpenAI module settings, locate the connection field
  2. Click "Add" to create a new connection
  3. Paste your copied API key when prompted
  4. Give the connection a recognizable name
  5. Click "Save" to establish the link

Once saved, Make.com will test the connection automatically. If successful, you'll see a confirmation message and can proceed to configure the specific AI parameters for your workflow.

Step 5: Test and Deploy

Before automating your workflow completely, it's crucial to test the integration with real data. This helps catch any configuration errors and ensures the AI outputs meet your quality standards.

Testing Protocol:

  1. Run the scenario once manually using the "Run once" button
  2. Inspect the AI's output in the execution history
  3. Adjust prompts or parameters if needed (common tweaks include temperature and max tokens)
  4. When satisfied, activate the scenario to run automatically

Remember: OpenAI API usage is billed by the token, so extensive testing can incur costs. Use the playground at platform.openai.com for initial prompt refinement before connecting to Make.com.
