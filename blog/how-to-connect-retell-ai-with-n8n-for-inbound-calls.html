The Inbound Call Problem

Imagine this scenario: Your AI voice agent calls a potential candidate for a nursing position. They miss the call but decide to call back later. When they do, your agent has no idea who they are - it can't reference their current job, experience, or the opportunity you wanted to discuss. The conversation becomes generic and ineffective.

This is the inbound call recognition gap that plagues most AI voice agent implementations. While outbound calls can be pre-loaded with caller data, inbound calls traditionally lose all that context. The agent greets the caller with "Am I speaking with user_first_name?" instead of their actual name.

83% of callback opportunities are wasted when AI agents can't recognize returning callers and access their data. This integration solves that problem completely.

How Retell Webhooks Work

Retell AI's inbound webhook feature changes the game. When configured properly, it triggers your n8n workflow before the agent answers the call. This gives you a critical window to:

  1. Capture the caller's phone number
  2. Look up their record in your database
  3. Send their information back to Retell AI

The entire process happens in milliseconds - fast enough that the caller never notices the background lookup. By the time the agent says "Hello," it already knows exactly who it's speaking with and can reference specific details from their record.

Setting Up the Retell Agent

Start by creating your voice agent in Retell AI as you normally would. The key difference comes when you publish it and connect it to a phone number:

Step 1: Publish Your Agent

After building your agent's conversation flow in Retell's interface, publish it to make it active. Ensure you're using dynamic variables in your prompts (like {user_first_name}) that will be replaced with actual data from n8n.

Step 2: Connect to a Phone Number

Assign your published agent to a Twilio number (or other supported provider) through Retell's Phone Numbers tab. This same number will handle both outbound and inbound calls.

Step 3: Enable Inbound Webhook

In the Phone Numbers settings, find the "Add an inbound webhook" option and enable it. Leave the URL field blank for now - we'll come back after setting up the n8n webhook.

Pro Tip: Use the same agent for inbound and outbound calls to maintain consistent branding and conversation flows.

Configuring the n8n Webhook

The n8n workflow serves as the bridge between Retell AI and your database. Here's how to set it up:

Step 1: Create Webhook Trigger

Add a Webhook node to your n8n workflow and configure it with these settings:

  • Method: POST
  • Respond with: Using Respond to Webhook node
  • Production mode: On

Step 2: Copy the Webhook URL

n8n will generate a unique URL for your webhook. Copy this and paste it into the inbound webhook field in your Retell AI phone number settings.

Now when someone calls your Retell number, the sequence will be:

  1. Call comes in to your Retell-connected number
  2. Retell triggers the n8n webhook before answering
  3. n8n processes the caller data
  4. n8n responds with the caller's information
  5. Retell agent answers with full context

Filtering Caller Data

The magic happens when n8n matches the incoming call to your database records. Here's how to set up the filtering:

Step 1: Add Google Sheets Node

Connect a Google Sheets node to retrieve rows from your database. In our example, we're using a recruitment spreadsheet with columns for:

  • User first name
  • Phone number
  • Current job title
  • Job description
  • New opportunity

Step 2: Add Filter Node

Create a Filter node to match the incoming call's phone number (from the webhook data) to your database records. The key fields to match are:

  • Value 1: call_inbound.from_number (from webhook)
  • Value 2: Your database's phone number column

Important: Format phone numbers consistently between your database and the incoming call data. Add country codes if necessary.

Sending Data Back to Retell

The final piece is the Respond to Webhook node that sends the matched data back to Retell AI:

Step 1: Add Respond to Webhook Node

Configure this node to respond with JSON containing all the dynamic variables your Retell agent expects. The structure should match your agent's prompt variables exactly.

Step 2: Map Your Data Fields

Using the expression editor, map your database fields to the JSON response:

{   "user_first_name": "{{$node["Google Sheets"].json["first_name"]}}",   "job_title": "{{$node["Google Sheets"].json["current_job"]}}",   "job_description": "{{$node["Google Sheets"].json["job_description"]}}",   "new_opportunity": "{{$node["Google Sheets"].json["new_position"]}}" }

When complete, your n8n workflow will:

  1. Receive the inbound call notification
  2. Fetch all potential records
  3. Filter to find the matching caller
  4. Return their specific data to Retell

Testing the Integration

To verify everything works:

Step 1: Make a Test Entry

Add your own phone number and test data to your database. Include realistic details your agent would reference in conversation.

Step 2: Call From Your Test Number

Place a call to your Retell-connected number from the test phone. The agent should greet you by name and reference all the details from your database entry.

Step 3: Check n8n Executions

In n8n, inspect the execution to confirm:

  • The webhook received the call notification
  • The filter correctly matched your number
  • The response contained your test data

Success Metric: Your agent should never say placeholder text like "user_first_name" - every variable should be replaced with actual data from your system.
