Getting Started With n8n and Shopify

Before connecting Shopify to n8n, you'll need three things: an active Shopify store, an n8n account (cloud or self-hosted), and admin access to your Shopify backend. The entire setup takes about 15 minutes but saves countless hours in the long run.

At 1:15 in the video tutorial, you'll see the first critical step - selecting the right trigger in n8n. For most stores, the "Order Paid" trigger is ideal as it fires only after successful payment processing. This prevents automation from running on abandoned carts or failed payments.

How to Get Your Shopify API Credentials

The most technical part of the integration is obtaining your Shopify API credentials. These act as secure keys that allow n8n to access your store data. At 2:30 in the video, you'll see exactly where to find these in your Shopify admin.

You'll need to create a custom app in Shopify's developer settings (Settings > Apps and sales channels > Develop apps). When configuring API permissions (scopes), ensure you select:

  • Read and write access for products (to update inventory)
  • Read access for orders (to trigger automations)
  • Read access for customers (for follow-up sequences)

Pro tip: Always copy your API access token immediately when generated - Shopify only shows it once. If lost, you'll need to create new credentials.

Connecting n8n to Your Shopify Store

With your API credentials ready, return to n8n and add a Shopify node. At 3:45 in the video, you'll see where to enter your store's subdomain (yourstore.myshopify.com) and API token. The subdomain must match exactly what's in your Shopify admin, including any custom domains.

After entering credentials, click "Test" to verify the connection. A successful test will return sample order data from your store. If you see errors, double-check:

  1. The subdomain is entered correctly (no http:// or trailing slashes)
  2. The API token was copied completely with no missing characters
  3. The required API scopes are enabled in Shopify's app settings

Common Connection Errors and Fixes

Even with careful setup, you might encounter errors when connecting Shopify to n8n. The most frequent issues include authentication failures (401 errors) and permission errors (403 errors), usually caused by incorrect API scopes or expired tokens.

At 4:20 in the video, you'll see how n8n clearly indicates connection problems with red error banners. For authentication issues, regenerate your Shopify API token. For permission errors, revisit the app settings in Shopify and ensure all required scopes are selected.

Troubleshooting tip: If your workflow suddenly stops working, check if your API token has expired. Shopify tokens don't expire by default but may be revoked if the app is uninstalled or permissions change.
