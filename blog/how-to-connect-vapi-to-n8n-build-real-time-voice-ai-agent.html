Step 1: Setting Up Your Vapi Assistant

The foundation of any voice AI agent is its personality and prompt structure. In the tutorial, we create a "Grumpy Weather Assistant" that demonstrates how to balance functional responses with brand personality - a key differentiator from generic voice assistants.

Begin by creating a new assistant in Vapi's dashboard. Select the blank template for maximum control, then define your opener message. This is the first thing users will hear when initiating the conversation. For weather queries, we use: "Hello. Where would you like to get the weather from today?"

Key Configuration Points:

  • System Prompt: "You're a sarcastic, grumpy weather anchor who hates their job. First give the real weather, but then add a snarky comment like asking why can't humans just look out a window. Keep it under two sentences."
  • Tools: Create a custom "weather tool" that extracts location from voice input and passes it to n8n via webhook
  • Parameters: Mark location as required to ensure the AI always captures this critical data point

Step 2: Connecting n8n Webhooks

Webhooks serve as the bridge between Vapi's voice interface and n8n's automation capabilities. When a user asks "What's the weather in Sydney?", Vapi extracts the location ("Sydney") and sends it to your n8n workflow via HTTP POST request.

In n8n, add a Webhook node and configure it to:

  • Listen for POST requests
  • Capture the incoming location parameter
  • Trigger the subsequent workflow steps

Copy the webhook URL from n8n and paste it into Vapi's tool configuration. This completes the one-way connection from voice input to automation workflow. At this stage (3:22 in the video), you can test that voice queries properly trigger the n8n workflow, though we haven't yet built the weather data response.

Step 3: Building the Weather Logic

The magic happens when we teach n8n to fetch real-time data and format it for voice response. Using OpenWeatherMap's free API, we create a three-step process:

Step 1: AI Interpretation

An AI Agent node in n8n processes the raw location into a properly formatted weather query. The system message instructs it to: "First provide the current temperature and weather condition for the requested city. Keep your response to under two sentences."

Step 2: API Integration

The OpenWeatherMap node makes the actual API call using your free key. Configuration includes:

  • Mapping the location from the webhook to the API query
  • Setting preferred units (Celsius/Fahrenheit)
  • Extracting only the relevant data points

Step 3: Data Processing

The raw API response gets transformed into a conversational format combining factual data ("27° and raining") with the personality-driven commentary defined in Vapi's system prompt.

Step 4: Formatting the API Response

The final technical hurdle is ensuring n8n's response matches Vapi's expected JSON structure. This requires a Respond to Webhook node configured to:

  • Use JSON format
  • Include the original call ID for reference
  • Structure results in Vapi's required format

The response template looks like this:

Successful response format: { "results": [ { "result": "Sydney's got 27.3 deg and some moderate rain. Sounds miserable..." } ] }

When properly configured, Vapi reads this response aloud exactly as formatted, completing the voice interaction loop. The entire process - from spoken question to verbal response - happens in under 2 seconds, creating a seamless conversational experience.
