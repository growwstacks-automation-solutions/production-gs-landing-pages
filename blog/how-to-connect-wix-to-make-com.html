The Wix-Make.com integration solves these problems by creating automated workflows that sync data in real-time. When configured correctly, the connection eliminates manual data entry while reducing errors and improving response times.

Step 1: Create a Custom App in Wix

The foundation of your integration starts in Wix's developer tools. You'll need to create a custom application that grants Make.com the necessary permissions to access your Wix data.

Navigate to your Wix Studio dashboard and locate the Custom App section under Developer Tools. Select "Create New App" and choose the self-hosted option - this ensures you maintain full control over the integration.

Key Configuration Steps:

  1. Give your app a clear, descriptive name (e.g., "Make.com Integration")
  2. Select permissions based on your automation goals (form submissions, products, orders)
  3. Note your app ID and generate an app secret key
  4. Test the app in development mode before finalizing

Pro tip: Always follow the principle of least privilege - only grant permissions your specific workflows require to maintain security.

Step 2: Set Up the Connection in Make.com

With your Wix app credentials ready, switch to Make.com to establish the receiving end of your integration. This is where you'll create the workflows that process data from Wix.

Start a new scenario in Make.com and add the Wix module. When prompted for connection details, enter the app ID and secret key from your Wix custom app. Make.com will authenticate with Wix using these credentials.

Important: Store your app secret securely. If compromised, regenerate it immediately in Wix and update your Make.com connection.

Once authenticated, you can begin building your first workflow. Common starter triggers include "New Form Submission" or "Order Created" depending on your use case.

Step 3: Configure Webhooks

Webhooks serve as the communication bridge between Wix and Make.com. They allow Wix to push data to Make.com in real-time when specific events occur.

In Make.com, create a webhook module within your scenario. Copy the unique URL generated - you'll need this to complete the setup in Wix. Then configure the expected payload structure so Make.com can properly interpret incoming data.

Wix Automation Setup:

  1. In Wix, create a new automation
  2. Select "Webhook Received" as your trigger
  3. Paste your Make.com webhook URL
  4. Map the data fields you want to transfer

Step 4: Test Your Automation

Before relying on your integration for live data, thorough testing is essential. Start by triggering test events in Wix while monitoring the workflow in Make.com.

Create a test form submission or dummy order in Wix and verify that:

  • Data appears in Make.com within seconds
  • All mapped fields contain the expected values
  • Downstream actions in your workflow execute correctly

Testing checklist: Validate with multiple data formats, test error scenarios, and confirm all permissions are working as intended before going live.
