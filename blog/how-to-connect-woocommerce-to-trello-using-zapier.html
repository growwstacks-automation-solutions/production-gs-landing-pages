Zapier Setup Overview

A Zapier automation (called a "Zap") connects two apps - in this case, WooCommerce and Trello. The Zap has two parts: a trigger that starts the automation (new WooCommerce order) and an action that happens afterward (create Trello card).

This simple but powerful structure means every time someone buys from your store, Zapier will create a corresponding Trello card with all the order details pre-filled. No copying, no missed orders, no confusion about order status.

Step 1: WooCommerce Trigger

The first half of your Zap defines what event starts the automation. For order tracking, we want the Zap to trigger whenever a new order comes through WooCommerce.

How to set it up:

  1. In Zapier, click "Create Zap"
  2. Search for and select "WooCommerce" as your trigger app
  3. Choose "New Order" as the trigger event
  4. Connect your WooCommerce store by following the authorization prompts
  5. Test the connection to ensure Zapier can see your orders

Pro tip: If you manage multiple stores, make sure you're connecting the correct WooCommerce account during this step.

Step 2: Trello Action

Now we define what happens after the trigger fires - creating a Trello card with the order details. This is where you'll customize how orders appear on your board.

How to set it up:

  1. Click "Add Action" in your Zap
  2. Search for and select "Trello" as your action app
  3. Choose "Create Card" as the action event
  4. Connect your Trello account and select the target board/list
  5. Map WooCommerce fields to Trello card fields (customer name, order details, etc.)

Customizing Card Fields

The real power comes from customizing which order details appear on your Trello cards. Zapier lets you pull any WooCommerce field into your card template.

Essential fields to include:

  • Customer name and email
  • Order number and date
  • Product names and quantities
  • Order total and payment method
  • Shipping address and method

You can also use Trello's custom fields to surface priority information immediately - like highlighting high-value orders or flagging international shipments.

Testing Your Automation

Before turning your Zap on permanently, test it thoroughly:

  1. Place a test order in WooCommerce
  2. Verify the Trello card appears with all correct details
  3. Check that special cases (discounts, variants, notes) transfer properly
  4. Confirm cards land in the correct Trello list

Scaling to Multiple Stores

If you manage several WooCommerce stores, you can create separate Zaps for each one, routing orders to different Trello boards or lists. This keeps your order tracking organized while maintaining the same automation framework.

Key considerations for multi-store setups:

  • Name your Zaps clearly (e.g., "Store A Orders → Marketing Board")
  • Use Trello labels or custom fields to indicate source store
  • Set up separate testing protocols for each store connection
