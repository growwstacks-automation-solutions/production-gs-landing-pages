Setting Up WordPress Application Passwords

WordPress application passwords provide secure API access without compromising your main admin credentials. These single-use tokens allow specific applications like Codex to interact with your site programmatically.

To create one:

  1. Navigate to your WordPress admin profile
  2. Scroll down to the Application Passwords section
  3. Create a new password with a descriptive name (e.g., "Codex Integration")
  4. Copy the generated password immediately - it won't be shown again

Security note: Treat application passwords like regular passwords. Store them securely and revoke any that may be compromised. Each integration should have its own unique password.

Connecting Codex to Your WordPress Site

With your application password ready, you can now establish the connection in Codex. The AI needs three key pieces of information:

  • Your WordPress site URL
  • Admin username
  • Application password

Provide these details to Codex using a clear prompt format:

"Please establish a connection to my WordPress site at [URL] using username [username] and application password [password]."

Codex will verify the connection and confirm when setup is complete. This process typically takes less than a minute when credentials are correct.

Testing Your First Automated Post

Once connected, you can immediately test the integration by having Codex create and publish content. Start with a simple test article to verify everything works correctly.

For example, prompt Codex:

"Write and publish a 500-word article about [topic] on my WordPress site. Use a friendly, conversational tone and include 3 subheadings."

The AI will generate the content, format it properly for WordPress, and publish it directly to your site. You'll receive a live URL once complete, typically within 2-3 minutes depending on article length.
