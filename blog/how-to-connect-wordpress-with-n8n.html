Getting the Right WordPress Credentials

The biggest mistake beginners make is using their WordPress admin login for API connections. n8n requires specific API credentials that provide limited, secure access to your site's functionality.

At 1:15 in the video tutorial, you'll see the three pieces needed:

  1. Your WordPress username (visible in Users → All Users)
  2. An application password (not your regular password)
  3. Your site URL including https://

These credentials allow n8n to interact with WordPress through its REST API while maintaining security best practices.

The Application Passwords Solution

WordPress introduced application passwords specifically for API access. Here's how to generate one:

Step 1: Navigate to User Profile

Go to Users → All Users in your WordPress dashboard and select your username.

Step 2: Find Application Passwords

Scroll down past your personal options until you see the Application Passwords section.

Step 3: Create New Password

Name it (like "n8n Automation") and click Add New. Copy the generated password immediately - it won't be shown again.

Security tip: Each application password can be revoked individually if compromised, unlike changing your main password.

Common Plugin Conflicts to Watch For

At 2:30 in the video, the creator mentions a critical discovery - some WordPress plugins hide the application passwords section entirely. Through testing, we've identified the most common culprits:

  • Hosting provider plugins (like WP Engine, Kinsta, or SiteGround optimizers)
  • LiteSpeed Cache and other caching tools
  • Security plugins that restrict admin panel features

The solution? Temporarily deactivate these plugins to access the application passwords section, then reactivate them after setup. The connection will continue working even after plugins are back on.

Testing Your Connection

Once you've entered all credentials in n8n's WordPress trigger:

  1. Click "Save" to store the credentials
  2. n8n will automatically test the connection
  3. Look for the "Connection tested successfully" message

If you see errors, double-check:

  • URL includes https:// (required for most modern WordPress sites)
  • Application password was copied completely (no missing characters)
  • No typos in the WordPress username
