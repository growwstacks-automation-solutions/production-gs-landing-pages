What Causes the MCP Configuration Error?

The MCP (Message Control Protocol) error in Cursor AI typically appears when trying to connect to Zapier's automation platform. Most users encounter the frustrating "Unexpected end of input" message, which indicates a formatting issue in the configuration file.

This error occurs because Cursor AI expects a specific JSON structure in the mcp.json file to establish the Zapier connection. When the file is missing, corrupted, or contains invalid syntax, the integration fails to initialize properly.

90% of MCP errors stem from three common issues: Missing file extension (.json), incorrect file location, or malformed JSON structure. The solution involves creating or correcting this configuration file with the proper formatting.

Locating the mcp.json File

Before fixing the error, you need to locate where Cursor AI stores its configuration files. The exact path varies slightly by operating system, but follows this general pattern:

On Windows, navigate to C:/users/cursor/ in your file explorer. Mac users will find it in ~/Library/Application Support/Cursor/ while Linux systems typically use ~/.config/cursor/.

If the mcp.json file doesn't exist in this directory, you'll need to create it. Right-click in the folder and select "New Text Document," then rename it to mcp.json (not mcp.json.txt). The file extension must be exactly .json for Cursor AI to recognize it.

Fixing the JSON Syntax Error

Once you've located or created the mcp.json file, open it with a text editor like Notepad, TextEdit, or VS Code. The file needs to contain valid JSON that specifies your Zapier connection details.

Paste this exact configuration into your mcp.json file:

 {   "servers": [     {       "url": "https://source.actions.zapier.com/mcpsk/k6xgazashukse"     }   ] }

After pasting, double-check that:

  1. All quotation marks are straight ("), not curly (“ ”)
  2. There are no trailing commas after the last item in arrays or objects
  3. The file extension is exactly .json (not hidden as .txt)

Pro Tip: Use a JSON validator like jsonlint.com if you're unsure about the syntax. Just paste your configuration to instantly check for errors before saving.

Restarting Cursor AI Properly

After saving the corrected mcp.json file, you must completely restart Cursor AI for the changes to take effect. Many users make the mistake of simply closing and reopening the application without fully terminating the process.

On Windows, use Task Manager (Ctrl+Shift+Esc) to ensure all Cursor AI processes are ended. On Mac, use Force Quit (Command+Option+Esc). Then restart the application fresh.

When Cursor AI relaunches, it will read the new configuration file during startup. You can verify this worked by navigating to Settings → Tools → MCP section. The red error message should be gone, replaced by an active connection status.

A good starter test is setting up a Zap that sends you a Slack message whenever you use a specific Cursor AI command. If the message arrives in Slack, your connection is working perfectly.
