Clawdbot Overview

Clawdbot (recently rebranded from Openclaw) is an open-source AI assistant exploding in popularity within the smart home community. Unlike generic chatbots, it's specifically designed for home automation with two killer features:

  1. Persistent memory: Remembers your home's context, devices, and preferences across conversations
  2. Multi-channel access: Works through WhatsApp, Telegram, or Discord - any app where you can type a message

The magic happens through its skill system. When you enable the Home Assistant skill during setup, Clawdbot gains the ability to directly modify your automations.yaml file and restart services - all from natural language prompts.

MCP Server Setup

The connection between Clawdbot and Home Assistant relies on the MCP (Message Control Protocol) server. This is the critical bridge that allows external tools to interact with your Home Assistant instance.

To install:

  1. Go to Home Assistant Settings → Devices & Services
  2. Click "Add Integration" and search for "MCP"
  3. Critical: Select "MCP Server" (not client) - this allows external access

Warning: Choosing the MCP client instead of server is the #1 installation mistake. The client won't allow Clawdbot to connect, so double-check you're installing the server component.

Installing Clawdbot

Clawdbot installs with a single curl command on Linux systems. Run this in your user directory (not root):

 curl -sSL https://install.clawdbot.ai | bash

After installation, initialize Clawdbot with:

 clawdbot onboard

The onboarding wizard will guide you through:

  1. Acknowledging security warnings (select "yes")
  2. Choosing quickstart mode
  3. Connecting your AI provider (OpenAI, Anthropic, etc.)
  4. Selecting communication channels (can skip initially)

Configuration Walkthrough

The most crucial configuration step is enabling skills. When prompted:

  1. Select yes to enable skills
  2. Install Homebrew to prevent dependency issues
  3. Choose npm as the installation method
  4. When shown missing dependencies, install MC Porter - this is essential for MCP server communication

After completing the setup, launch Clawdbot using the recommended hatch + TUI method. You'll then need to link it to Home Assistant by providing:

  • Your Home Assistant MCP server URL (usually http://[your-ha-ip]:8123)
  • A long-lived access token (create one in Home Assistant → Profile)

Clawdbot will automatically create a configuration file at /root/cloud/config/mcporter.json with all connection details.

Creating Your First Automation

Here's where many users hit a snag. A natural prompt like "Create an automation to turn off my bedroom plug at 11 PM" often produces disappointing results - the AI might generate YAML code but not deploy it.

The solution? Structured prompting. For reliable deployments, include:

  • The exact file path for automations.yaml
  • Specific automation details (trigger time, entity ID)
  • Instructions to restart Home Assistant after changes

Example effective prompt:

"Write code into /config/automations.yaml to create an automation that turns off entity switch.bedroom_plug every day at 11 PM. The automation should be named 'Bedroom Plug Nightly Off'. After writing the file, execute 'docker restart homeassistant' to apply changes."

This level of specificity yields perfect deployments every time.

Prompt Engineering Tips

Through testing, we discovered several strategies that improve Clawdbot's automation success rate:

  1. Entity verification: Add "First verify the correct entity ID for [device name]" to prevent errors
  2. File management: Specify whether to append or overwrite in automations.yaml
  3. Restart commands: Always include the exact service restart command for your installation type
  4. Error handling: Request that the bot reports back any errors during deployment

With these techniques, you can achieve 10x better results than with vague natural language prompts. The AI becomes incredibly focused, looking up entity IDs, writing perfect YAML, and handling the complete deployment process.
