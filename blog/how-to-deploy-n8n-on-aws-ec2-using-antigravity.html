Antigravity Setup Requirements

Before beginning the deployment, you'll need three key components ready:

  1. Antigravity subscription: The free version works but may have limited capacity. For production deployments, the Ultra subscription (used in the tutorial) provides reliable execution.
  2. AWS account: With EC2 launch permissions and at least $15/month in credits for a t3.small instance.
  3. SSH key pair: Generated through AWS EC2 and downloaded to your local machine (Antigravity will need access to this).

At 3:45 in the video, the tutorial shows how to verify your Antigravity subscription level and connect it to your AWS account. The process is straightforward - you simply provide your AWS credentials and the location of your SSH key file.

AWS EC2 Configuration

The tutorial demonstrates creating a t3.small instance with Amazon Linux 2023 - the ideal configuration for most n8n workloads. Here are the key settings Antigravity automates:

  • Instance type: t3.small (2 vCPUs, 2GB RAM) balances cost and performance
  • Storage: 30GB EBS volume provides ample space for workflows
  • Security groups: Automatically configures HTTP, HTTPS, and custom TCP ports

At 7:20 in the video, you'll see how Antigravity handles the security group configuration - one of the most error-prone parts of manual setup. The AI properly sets up inbound rules while warning about the security implications of leaving HTTP access open.

Connecting Antigravity to AWS

The critical step shown at 12:30 in the tutorial is linking Antigravity to your new EC2 instance. This requires:

  1. The public IP address of your EC2 instance
  2. The file path to your downloaded PEM key
  3. Verification of SSH connectivity

Antigravity tests the connection before proceeding - a valuable safeguard that prevents hours of frustration from misconfigured access. Once connected, the AI begins executing the deployment script automatically.

Pro tip: For production deployments, restrict SSH access to your office IP address rather than leaving it open to "Anywhere" as shown in the tutorial demo.

n8n Configuration & Security

The tutorial highlights an important security consideration at 18:20 - the need to either configure HTTPS or disable secure cookies when running n8n over HTTP. Antigravity:

  • Automatically detects the insecure connection
  • Explains the security implications
  • Provides options for resolution

For testing purposes, the video shows disabling secure cookies. However, for production use, we strongly recommend setting up HTTPS with a proper certificate instead.

Production Environment Considerations

While the tutorial demonstrates a basic deployment, production environments require additional configuration:

Essential upgrades: Monitoring, automated backups, and proper HTTPS setup should all be implemented before using n8n in production. The basic deployment shown is ideal for testing but needs hardening for business-critical workflows.

At 22:10 in the video, the completed n8n instance is accessible via the EC2 public IP. For production use, you'll want to:

  1. Configure a domain name with DNS pointing to your instance
  2. Set up Let's Encrypt or another SSL certificate provider
  3. Implement regular database backups
  4. Configure monitoring and alerts
