Workflow Overview

The n8n workflow solves the problem of manual image creation by automating the entire process. Simply provide a text prompt, select your desired specifications, and receive a professional AI-generated image in seconds.

The workflow consists of several key components: an input form for prompts and settings, Google Cloud authentication, the Nano Banana API connection, and output handling. The magic happens through Vertex AI's integration with Nano Banana, which n8n accesses via HTTP requests.

Two main paths exist in the workflow: text-to-image generation (creating entirely new images from descriptions) and image-to-image modification (altering existing images based on new prompts). The switch node at 6:45 in the video demonstrates how the workflow routes requests appropriately.

Google Cloud Console Setup

The most crucial - and potentially confusing - part of the process is configuring Google Cloud Console. This provides the free credits and infrastructure needed to run the Nano Banana model without cost.

Start by creating a new project in Google Cloud (shown at 4:30 in the video). Name it something recognizable like "n8n-free-images". You'll need to enable billing, but rest assured Google won't charge your card - they just require it for identity verification against their $300 free credit offer.

Important note: The free credits typically last for several months of regular image generation. If you exhaust them, you can create a new Google account to get another $300 credit rather than paying.

After project creation, copy the Project ID from Google Cloud and paste it into the n8n configuration node. This links your workflow to your Google Cloud resources. The other default settings can remain unchanged for basic functionality.

OAuth Configuration

Authentication is handled through Google OAuth, which requires setting up a consent screen and credentials. This ensures secure access to your Google Cloud resources from n8n.

In Google Cloud Console, navigate to APIs & Services > OAuth consent screen (shown at 5:15). Create an external app type, name it (like "n8n-free"), and add your email as a test user. This step is crucial - skipping it will prevent authentication from working.

Create OAuth credentials (web application type) and add n8n's redirect URI. Copy the client ID and secret into the corresponding n8n HTTP request node fields. The scope should be set to "https://www.googleapis.com/auth/cloud-platform" to grant necessary permissions.

Finally, complete the OAuth flow by signing in through n8n's interface. The video at 7:30 shows this process, including how to handle the security warning that appears (it's safe to proceed). Once linked, your workflow will have authenticated access to Google's AI services.

Workflow Configuration

With authentication set up, the workflow needs minimal additional configuration. The key nodes to review are the HTTP request nodes that communicate with Nano Banana through Vertex AI.

The text-to-image path requires no changes - it's pre-configured to send your prompt to Nano Banana's endpoint. The image-to-image path needs the binary data from your uploaded image converted to a base64 string, which happens automatically in the workflow.

A set node handles secret variables and routing, while a switch node determines which generation path to use based on your selection. The final nodes convert the API response into a viewable image file you can download or use directly in other applications.

Testing the Workflow

Before relying on the workflow for production use, test both generation types thoroughly. Start with text-to-image by entering a descriptive prompt like "futuristic cityscape at night, neon lights, cyberpunk style" with 16:9 aspect ratio.

For image-to-image testing, upload any photo and provide a modification prompt like "make it daytime" or "convert to watercolor painting style". The workflow will preserve the original composition while applying your requested changes.

Pro tip: The more specific your prompts, the better the results. Instead of "a dog", try "golden retriever puppy playing in autumn leaves, shallow depth of field, warm sunlight".

If images aren't generating correctly, double-check your Google Cloud project settings and OAuth configuration. The video at 9:45 shows successful generation of both workflow types with the football pitch example.
