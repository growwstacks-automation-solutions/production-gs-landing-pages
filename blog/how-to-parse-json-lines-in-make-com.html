What is JSON Lines Format?

JSON lines (or JSONL) is a convenient format for storing structured data that may be processed one record at a time. Unlike standard JSON which wraps all objects in an array, JSON lines format stores each JSON object on its own line. This makes it ideal for processing large datasets without loading everything into memory at once.

The format is particularly useful in automation workflows where you need to process records sequentially. Each line is a valid JSON object, making it easy to parse individual records without worrying about the complete file structure.

Example JSON lines format: Each line is a complete JSON object that can be parsed independently. This makes the format perfect for streaming processing and handling large datasets efficiently.

Step 1: Create Your Make.com Scenario

Begin by logging into your Make.com account and navigating to the scenarios section. This is where you'll build your JSON parsing automation. Click "Create a new scenario" to start setting up your workflow.

Give your scenario a descriptive name like "Process JSON Lines Data" so you can easily identify it later. The scenario interface provides a canvas where you'll add and connect modules to create your parsing workflow.

Step 2: Split Text into JSON Lines

The first processing step is to split your JSON lines text into individual items. Add a "Text" module to your scenario and select the "Split Text" operation. This will convert your multi-line JSON text into an array where each element is one line.

For the separator, use the newline character ( ). This tells Make.com to split the text at each line break, creating an array item for each JSON object in your source data. The output will be an array you can process in subsequent steps.

Pro Tip: If your JSON lines come from a file, use Make.com's "Download a file" module first to get the content before splitting it.

Step 3: Use an Iterator for Each Line

After splitting your JSON lines into an array, add an "Iterator" module to process each line individually. The iterator takes your array and processes one item at a time, preventing memory issues with large datasets.

Connect the output from your split text module to the iterator's input. Configure the iterator to process all items sequentially. This setup ensures each JSON line gets parsed separately in the next step.

Step 4: Parse Each JSON Line

Inside the iterator, add a "JSON" module and select the "Parse JSON" operation. Map the current item from the iterator as the JSON string to parse. This is where the magic happens - converting each text line into structured data.

Click "Create a data structure" and paste a sample JSON object that matches your data format. For example: {"id":1,"name":"Alice","email":"alice@example.com"}. Make.com will use this sample to generate the proper field mappings for your data.

Step 5: Map Parsed Fields to Applications

With your JSON data now properly parsed, you can map the extracted fields to any connected application. Add modules for your target systems (Google Sheets, Airtable, CRM, etc.) and map the parsed fields to the appropriate columns or properties.

Common destinations include databases for storage, email services for notifications, or CRMs for customer data updates. The parsed JSON fields will be available in all subsequent modules within your iterator.

Automation Benefit: This entire process runs automatically whenever new JSON lines data arrives, eliminating manual parsing work.
