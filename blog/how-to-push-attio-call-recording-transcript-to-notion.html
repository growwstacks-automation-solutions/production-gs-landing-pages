The Zapier Transcript Limit Problem

Sales teams using Attio's call recorder face a critical bottleneck when trying to log conversations in Notion. At 2:43 in the video, we see the moment where Zapier abruptly stops processing because the raw transcript exceeds Notion's 2,000-character field limit — leaving teams with truncated notes and missing deal-critical information.

This limitation forces reps to either manually edit down transcripts (wasting 15–20 minutes per call) or abandon structured note-taking altogether. The result? Lost insights, inconsistent CRM data, and missed follow-up opportunities.

83% of sales teams report losing at least one deal per quarter due to incomplete call documentation, according to Gong's 2024 sales productivity report.

Make.com Workflow Architecture

Unlike Zapier's linear approach, this Make.com solution uses parallel processing to handle large transcripts intelligently. When Attio's webhook fires, the workflow simultaneously:

  1. Stores the full recording in cloud storage (bypassing Notion's size limits)
  2. Splits the transcript into logical sections based on speaker changes
  3. Generates an AI summary of key points and action items

This multi-path approach ensures no data is lost while creating more useful Notion entries than raw transcripts alone could provide.

Configuring Attio Webhooks

The trigger starts with Attio's "call recording created" webhook (shown at 1:12 in the video). In Make.com, we configure a webhook module with:

  • Unique URL generated for the Attio integration
  • Payload structure matching Attio's API documentation
  • Error handling for duplicate events

Test calls validate the connection by simulating completed recordings before going live with production data.

Fetching Transcript Data via API

At 2:18, we see the critical API call that retrieves the full transcript text. The Make.com workflow uses:

  • Bearer token authentication with Attio's API
  • Meeting ID from the initial webhook payload
  • Automatic retry logic for rate-limited requests

A second API call fetches participant details and meeting metadata to enrich the Notion entry with CRM context.

Structuring Notion Database Entries

The solution creates Notion pages with a smart layout that:

  • Stores the recording link in a dedicated media property
  • Distributes transcript sections across multiple text fields
  • Maps Attio CRM fields to corresponding Notion properties

This structure maintains readability while avoiding any single field hitting the character limit.

Adding AI-Powered Meeting Insights

The optional AI module (demonstrated at 4:30) transforms raw transcripts into:

These insights appear in a dedicated Notion section, giving sales managers instant visibility into call outcomes without reading full transcripts.
