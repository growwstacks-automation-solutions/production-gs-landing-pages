Why API Keys Expire (And Why It Matters)

API keys expire as a security measure - services like Google Sheets, Gmail, and CRM platforms automatically revoke access after 30-90 days. When this happens, your Make.com workflows fail silently, potentially disrupting critical business processes without warning.

The average business loses 4-6 hours of productivity per expired API connection while troubleshooting. Worse, some automations handle time-sensitive tasks like lead follow-ups or inventory updates - failures here can directly impact revenue.

Pro Tip: Services using OAuth 2.0 often provide refresh tokens that automatically renew access. Look for "offline access" or "long-lived tokens" in your app's developer settings.

Step 1: Identify the Expired API Key

Log into your Make.com account and navigate to the scenario showing errors. As shown at 0:45 in the video tutorial, disconnected modules display a red warning icon - this visual indicator helps you quickly locate the problem.

For complex scenarios with multiple integrations, check the error logs under "Operations" to pinpoint exactly which connection failed. The error message typically specifies whether it's an authentication issue or expired credentials.

Step 2: Reconnect the Affected Module

Click the module with the warning icon, then select "Connection" from its settings menu. This initiates the reauthorization flow. Make.com will guide you through re-establishing the API handshake with the external service.

Some apps require you to generate a brand new API key before reconnecting. If prompted, carefully copy the new credentials without any leading/trailing spaces that might cause validation errors.

Step 3: Reauthorize With the Connected App

Follow the on-screen prompts to grant Make.com access to the external service again. The exact steps vary by platform, but most will:

  1. Open the service's authentication page
  2. Request necessary permissions
  3. Generate new tokens or API keys
  4. Return you to Make.com automatically

Enterprise systems may require admin approval or additional security verification at this stage. Always check your organization's specific policies.

Step 4: Test and Save Your Scenario

After reconnecting, immediately run your scenario once to verify everything works. Look for green success indicators instead of red errors. Pay special attention to any data mapping that might need updating after the reconnection.

Once confirmed working, save your scenario. Consider running historical operations to process any data that accumulated during the downtime, especially for time-sensitive workflows.

Preventing Future API Key Expirations

While some expiration is inevitable, these strategies reduce disruptions:

  • Enable long-lived tokens where available (Google services offer 7-year refresh tokens)
  • Set calendar reminders 2 weeks before known expiration dates
  • Use Make.com's connection health dashboard to monitor all integrations
  • Build error handling into scenarios to notify you of failures

Critical: For mission-critical automations, implement backup notification systems outside Make.com to alert you of API failures immediately.

Frequently Asked Questions

Common questions about Make.com API reconnections

API keys expire due to security policies from the connected apps (like Google Sheets or Gmail). Most services automatically expire tokens after 30-90 days unless you enable long-lived keys.

Some apps also revoke access when passwords change or when detecting unusual activity. Enterprise systems may have even shorter token lifespans for compliance reasons.

  • Google services: Default to 7-day access unless offline access enabled
  • CRMs: Often expire after 30-60 days
  • Payment processors: May require quarterly reauthorization

Make.com marks modules with expired API keys with a red warning icon. In your scenario editor, look for modules displaying this visual indicator at their top-right corner.

The error message when running the scenario will also specify which connection failed. For complex scenarios, check the "Operations" log to see exactly which step encountered the authentication error.

  • Red triangle icon = Connection issue
  • Check operation history for error details
  • Some services email you about revoked access

Some services offer long-lived API keys or OAuth refresh tokens that automatically renew. These significantly reduce connection issues but aren't available for all integrations.

For Google services, enabling 'offline access' during authorization helps maintain connections longer. Many platforms also allow extending token lifespans in their developer settings.

  • Google: Enable offline access mode
  • Salesforce: Use refresh tokens
  • QuickBooks: Set to 180-day expiration

If standard reauthorization fails, try creating a completely new connection in Make.com. Some apps require generating fresh API credentials when old ones expire.

Also verify you still have proper permissions in the external service. Admin approvals may be needed if your organization changed security policies since the original connection.

  • Create brand new connection
  • Verify app permissions
  • Check for IP restrictions

Critical automations should be monitored weekly. Make.com's connection health dashboard shows expiration dates for all active integrations.

For important workflows, set calendar reminders 1-2 weeks before known expiration dates. Monthly checks suffice for less critical connections, but never go more than 60 days without verification.

  • Weekly for revenue-impacting automations
  • Monthly for others
  • Set expiration reminders

The general reconnection process works for most API-based integrations, including popular services like Google Workspace, Shopify, and HubSpot.

Some enterprise systems (like SAP or Oracle) may require additional authentication steps or VPN access. Always check the specific app's documentation for any special requirements.
