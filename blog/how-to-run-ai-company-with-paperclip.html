What is Paperclip AI?

Most business owners struggle with scaling their operations while maintaining quality and consistency. Paperclip solves this by providing an open-source framework for creating zero-human companies powered by AI agents. Unlike single-purpose AI tools, Paperclip creates an entire organizational structure where AI employees collaborate like a real company.

The platform allows you to define company goals, and the AI CEO will autonomously hire the necessary roles (CTO, CMO, designers) and delegate tasks to achieve those objectives. Each agent has specific skills, tools, and permissions, creating a self-managing system that only requires human input for critical approvals.

Key differentiator: Paperclip isn't just another chatbot - it's a complete organizational simulation where AI agents have distinct roles, decision-making authority, and the ability to hire and manage other agents to accomplish complex business objectives.

Core Concepts of AI Organizations

Understanding these five pillars will help you design effective Paperclip implementations:

1. Organizational Structure

Paperclip mirrors real corporate hierarchies. You start as the board member, appoint a CEO, who then hires other C-level executives and specialists based on company needs. Each role has defined responsibilities and reporting lines.

2. Agent Types & Roles

Different agent types serve specific functions:

  • CEO: Strategic direction, hiring, task delegation
  • CTO: Technical implementation and engineering
  • CMO: Marketing and customer-facing activities
  • UX Designer: Interface and experience design

3. Heartbeat System

Agents don't run continuously but activate periodically (default every 3600 seconds) to check for new tasks. This optimizes resource usage while ensuring timely task completion.

4. Task Lifecycle

Workflows follow a clear path: Board/CEO creates issue → Assigns to appropriate role → Agent completes task → Delivers artifact → Gets approved.

5. Cost Controls

You can set per-agent budgets and monitor token usage across all operations to prevent unexpected expenses.

Local Setup in 5 Minutes

For testing Paperclip's capabilities, you can run it locally with minimal setup. This is ideal for understanding the platform before committing to VPS deployment.

Step 1: Installation

Run this single command in your terminal:

nbx paperclip onboard yes

This installs all dependencies and starts the Paperclip service. First-time setup may take a few minutes as it downloads required components.

Step 2: Initial Configuration

Once installed, access the web interface at localhost:3000 and:

  1. Create your company name and mission
  2. Set up your first agent (typically the CEO)
  3. Choose an LLM adapter (Codex recommended for testing)

Step 3: Authenticate LLM Access

For Codex (OpenAI) users:

  1. Enable device code authorization in ChatGPT account settings
  2. Run codex login --device-off in your Paperclip terminal
  3. Follow the authentication flow using the provided code

Pro Tip: Local setups are great for experimentation, but for production use (especially with multiple agents), a VPS provides better reliability and 24/7 operation.

Building Your AI Company Structure

The real power of Paperclip emerges when you create a complete organizational structure. Here's how to design your AI workforce effectively.

1. Defining Company Goals

Start by setting clear objectives in the Goals section. For example: "Create a weekly newsletter about latest open-source LLMs." The CEO will use this to determine required roles and tasks.

2. CEO Configuration

Your first agent should be the CEO. Configure with:

  • Adapter: Codex (for testing) or your preferred LLM
  • Instructions: Define leadership style and decision-making parameters
  • Permissions: Allow hiring and task delegation
  • Heartbeat: 3600 seconds (1 hour) is a good starting interval

3. Hiring Process

The CEO will autonomously:

  1. Analyze company goals
  2. Determine required roles
  3. Create hiring proposals
  4. Submit for board (your) approval

You'll receive notifications in the Inbox when approval is needed.

4. Role Specialization

Each role should have:

  • Clear responsibilities defined in agent.md
  • Specific tools/skills relevant to their function
  • Appropriate permissions matching their level
  • Budget constraints to control costs

Organization Tip: Start lean - a CEO plus 2-3 key roles. You can always expand later as needs grow. Monitor the Activity tab to see how agents collaborate before adding complexity.

Agent Configuration & Skills

Fine-tuning your agents' capabilities is crucial for optimal performance. Each agent has several configurable components.

1. Instruction Files

Three key files define agent behavior:

  • agent.md: Core responsibilities and operating principles
  • soul.md: Personality traits and communication style
  • tools.md: Capabilities and skills the agent can use

2. Skill Assignment

Paperclip comes with built-in skills you can assign:

  • Paperclip Agent Creation: Allows hiring new roles
  • Plugin Development: For creating custom tools
  • Para Memory: Persistent knowledge retention

You can also create custom skills for specialized tasks.

3. Adapter Selection

Each agent can use different LLMs:

  • Codex: Good for general tasks (free credits available)
  • Claude: Better for strategic thinking
  • Gemini: Strong technical capabilities
  • Cursor: Specialized for coding tasks

4. Monitoring & Optimization

Use these features to maintain performance:

  • Runs tab: View complete activity history
  • Budget controls: Set spending limits per agent
  • Heartbeat adjustment: Increase frequency for critical roles

Configuration Tip: Start with default settings, then refine based on observed performance. The CEO's agent.md is particularly important as it governs overall company direction.

Real Workflow Example: Newsletter Automation

Let's walk through a concrete example showing how Paperclip agents collaborate to complete a business task - creating a weekly newsletter about open-source LLMs.

1. Task Creation

As the board member, you create an issue: "Produce this week's open-source LLM newsletter and share with me." You assign this to the CEO.

2. CEO Delegation

The CEO analyzes the request and:

  1. Determines the CMO is best suited for content creation
  2. Creates subtasks for research, writing, and formatting
  3. Assigns these to the CMO with appropriate deadlines

3. CMO Execution

The CMO agent:

  1. Researches latest open-source LLM developments
  2. Structures content into newsletter format
  3. Submits draft to CEO for review

4. Delivery

Once approved, the newsletter is:

  1. Posted as a comment artifact in the original issue
  2. Available for download in multiple formats
  3. Ready for distribution to your email list

Key Insight: This entire process happens autonomously after the initial request. You only intervene if the CEO flags something for approval, making it highly scalable for recurring tasks.
