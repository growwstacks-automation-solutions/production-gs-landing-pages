By running n8n locally with free open-source AI models, you gain complete ownership of your automation infrastructure. At 4:30 in the video tutorial, we demonstrate how Ollama provides access to powerful models like Llama 3 and Mistral without subscription fees - all running on your own hardware.

Docker Setup for n8n

Docker Desktop provides the simplest way to run n8n locally while maintaining easy access to other services like databases and AI models. The containerized approach ensures all components work together seamlessly.

Here's the step-by-step process shown at 1:15 in the video:

Step 1: Install Docker Desktop

Download and install Docker Desktop from docker.com. Windows users may need to enable WSL (Windows Subsystem for Linux) during installation.

Step 2: Pull the n8n Image

Search for the official n8n image in Docker Desktop and click "Pull" to download it.

Step 3: Configure the Container

When running the image, set these key parameters:

  • Container name: "n8n"
  • Host port: 5678 (or any available port)
  • Volume: Map a local folder to /home/node/.n8n for data persistence

Pro tip: The video at 2:45 shows how to activate your free n8n license key after initial setup - required for accessing premium nodes like AI integrations.

Integrating Free AI Models with Ollama

The real power comes when connecting your local n8n instance to free, open-source AI models. At 6:20 in the tutorial, we demonstrate Ollama - a tool that makes running models like Llama 3 as simple as downloading an app.

Key advantages of using Ollama with n8n:

  • Access to 50+ models including Llama 3, Mistral, and Gemma
  • No API keys or usage limits
  • Complete data privacy - conversations never leave your machine
  • Customizable model sizes (from 7B to 70B parameters)

To connect Ollama to n8n (shown at 8:10):

  1. Download and install Ollama from ollama.com
  2. Download your preferred model (e.g., ollama pull llama3)
  3. In n8n, create new Ollama credentials with base URL http://host.docker.internal:11434

Persistent Database Memory with PostgreSQL

At 10:35 in the video, we encounter a critical limitation - basic AI agents lose conversation history when n8n restarts. The solution? PostgreSQL with pgvector for persistent, searchable memory.

Here's how to set it up:

Step 1: Run PostgreSQL Container

Pull the pgvector image in Docker Desktop and configure with:

  • Host port: 5432
  • Volume for data persistence
  • Environment variables for user/password

Step 2: Connect to n8n

Use the Postgres Chat Memory node with credentials pointing to host.docker.internal and your configured port.

Key benefit: As shown at 13:20, this setup preserves conversation history even after restarting containers - crucial for customer-facing chatbots.

Building a Custom Knowledge Base

The tutorial at 15:50 demonstrates a powerful extension - automatically building a searchable knowledge base from Google Drive documents. Whenever new files are added to a monitored folder, n8n:

  1. Downloads the file
  2. Processes it through an embedding model
  3. Stores it in PostgreSQL with pgvector

This creates an always-updated knowledge source your AI agent can query. At 18:30, we test it by asking "Give me a breakdown of my last few invoices" - the agent successfully retrieves and summarizes documents from the database.

Making Your n8n Instance Public with ngrok

While running locally is great for development, real business use often requires external access. At 20:15, we use ngrok to securely expose the local n8n instance to the internet.

Key steps:

  1. Sign up for a free ngrok account
  2. Install and authenticate the ngrok client
  3. Run ngrok http 5678 (or your n8n port)
  4. Update n8n's webhook URL to use the ngrok address

This allows integration with services like WhatsApp and Telegram while keeping all processing local. At 22:40, we demonstrate accessing the chatbot interface via the public ngrok URL.
