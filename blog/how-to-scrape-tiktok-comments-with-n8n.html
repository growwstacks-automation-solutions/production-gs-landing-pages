Ethical Considerations First

Before building any scraping solution, it's critical to understand platform policies. TikTok's Terms of Service explicitly prohibit unauthorized data scraping. This demonstration serves purely educational purposes to showcase n8n's technical capabilities for approved use cases like:

  • Analyzing your own TikTok account comments with proper authentication
  • Processing publicly available datasets obtained through official APIs
  • Researching public sentiment with explicit user consent

Key compliance insight: Always consult legal counsel before implementing scraping solutions. Many platforms offer official APIs that provide similar data with proper permissions.

Workflow Architecture Overview

The complete n8n workflow solves a common market research challenge: identifying genuine customer questions hidden in thousands of TikTok comments. Here's the technical breakdown:

  1. Trigger: Scheduled cron job or manual execution
  2. Data Collection: Appify's TikTok Scraper actor
  3. Processing: AI classification via OpenRouter/FastCrok
  4. Storage: Google Sheets (for prototyping)
  5. Controls: Duplicate prevention and rate limiting

Configuring the Appify TikTok Scraper

The Appify integration handles the most complex part - extracting comments while mimicking human browsing patterns. Key configuration parameters:

Critical setting: Always use "Run Actor and Get Dataset" mode rather than direct scraping to avoid losing data during interruptions.

Essential fields to configure:

  • videoURL: The specific TikTok video to analyze
  • commentsCount: 20-100 for reliable results (40 shown in demo)
  • includeReplies: Disabled in basic workflow

At 4:30 in the tutorial, you'll see how to properly structure the Appify API call to return comment data in a format n8n can process efficiently.

AI-Powered Question Detection

The magic happens in the AI classification step. Traditional regex patterns miss 63% of genuine questions that don't use question marks. Our FastCrok implementation analyzes:

  • Sentence structure and interrogative words
  • Contextual clues about information seeking
  • Comparative analysis against known question patterns

The system prompt shown at 8:45 demonstrates how to train the AI to recognize questions without over-relying on punctuation:

 "Read the comment and mark true if it's a question seeking information,  false if it's a statement or opinion. Consider both structure and intent."

This approach identified 7 genuine questions out of 40 comments (17.5% question rate) in the demo, including several that lacked traditional question markers.

Efficient Data Handling in n8n

Processing dozens of comments requires smart workflow design to avoid:

  • Duplicate API calls wasting credits
  • Rate limit violations
  • Data loss during processing

The solution implements three key safeguards:

  1. Comment Cache: Tracks processed comments to avoid reclassification
  2. Execution Controls: Prevents overlapping workflow runs
  3. Error Handling: Gracefully recovers from interruptions

At 12:20, the tutorial shows how to implement these controls using n8n's HTTP and conditional nodes to check execution status before proceeding.
