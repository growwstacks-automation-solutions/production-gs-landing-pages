What Makes OpenClaw Different?

Most AI assistants today are glorified chatbots - they answer questions but can't actually do anything. OpenClaw (also called Clawdbot or Moltbot) changes this by turning your messaging apps into powerful action interfaces. Imagine clearing your inbox, finding files, or running computer commands - all through natural conversation in WhatsApp or Telegram.

Unlike cloud-based services, OpenClaw runs locally on your machine. This means no API limits, no subscription fees, and complete privacy for your data. When combined with Ollama's local LLMs, you get an AI assistant that's both powerful and private.

Key differentiator: OpenClaw isn't just an information source - it's an execution engine. Where ChatGPT tells you how to create a file, OpenClaw actually creates it when you ask through WhatsApp.

Step 1: Installing OpenClaw

The installation process is straightforward but requires careful attention to terminal commands. At the 2:15 mark in the video, you'll see the exact installation steps:

  1. Search for "open claw" on Google and navigate to the official website
  2. Scroll down to find the installation command
  3. Copy the command to your clipboard
  4. Open terminal and paste the installation command

The installation typically takes 10-15 minutes depending on your internet speed. Once complete, you'll see the setup screen where you'll need to:

  • Accept the terms and conditions
  • Select "Quick Start" for initial configuration
  • Choose your preferred model (we'll use Google Gemini initially)

Pro Tip: Keep a notepad open to save API keys and tokens during setup. You'll need these for multiple steps.

Step 2: Configuring Google Gemini

While we'll eventually switch to local Ollama models, starting with Gemini provides a reliable baseline. Here's how to get your API key:

  1. Search for "Gemini API key" on Google
  2. Sign in to the Google AI Studio website
  3. Navigate to "Get API Key" in the bottom left
  4. Click "Create API Key" and name it (e.g., "OpenClaw")
  5. Copy the generated key to your clipboard

Back in your terminal during OpenClaw setup:

  1. Paste the Gemini API key when prompted
  2. Select "Gemini 2.5" as your default model
  3. Proceed to channel configuration

Important: The free Gemini tier has usage limits. We'll bypass these later by switching to Ollama, but the initial setup ensures everything works before making it fully local.

Step 3: Creating Your Telegram Bot

Telegram integration turns your messaging app into an AI command center. The setup involves:

  1. Searching for "BotFather" in Telegram
  2. Starting a chat and selecting "/newbot"
  3. Naming your bot (must end with "_bot")
  4. Receiving and saving your unique bot token

In your OpenClaw terminal setup:

  1. Paste the Telegram bot token when prompted
  2. Skip skills and webhooks for now (we'll add these later)
  3. Wait for gateway installation to complete

Once installed, you'll need to:

  1. Open your Telegram bot and send "/start"
  2. Approve the pairing request in your terminal
  3. Begin chatting with your new AI assistant

Security Note: The initial pairing approval is crucial - it prevents unauthorized access to your machine through the bot.

Testing File Commands

At the 7:30 timestamp in the video, you'll see the first real test of OpenClaw's capabilities. Try these commands in your Telegram bot:

  • Create a file named welcome.txt with content "Hello from OpenClaw"
  • List all PDFs downloaded last week
  • Show me the file welcome.txt

Initially, you may encounter permission errors. This is a security feature - OpenClaw requires explicit approval before accessing personal files. To grant permissions:

  1. Prefix commands with command (e.g., command ls Downloads)
  2. Approve the action in your terminal when prompted
  3. Subsequent commands to the same location won't require re-approval

Real-world use case: At 8:45 in the video, the bot successfully creates an index.html file with custom content - proving OpenClaw can handle web development tasks through chat.

Step 4: WhatsApp Integration

Adding WhatsApp follows a similar but distinct process:

  1. Run openclaw onboard in terminal
  2. Accept agreements and select "Quick Start"
  3. Choose "Update values" for config handling
  4. Skip model configuration (we're keeping Gemini for now)
  5. Select WhatsApp from the channel list

The critical WhatsApp-specific steps:

  1. Scan the QR code with your phone (WhatsApp → Settings → Linked Devices)
  2. Use a secondary phone number if possible
  3. Input your number when prompted in terminal
  4. Skip skills configuration for now
  5. Restart the gateway

At the 11:20 mark, you'll see the WhatsApp integration maintaining context from Telegram - proof that OpenClaw creates a unified AI assistant across platforms.

Business application: This allows employees to access company files or data through WhatsApp without installing special software - just chat with your AI assistant.

Step 5: Local AI with Ollama

When you hit API limits with Gemini (shown at 12:40 in the video), Ollama provides the solution:

  1. Search for "ollama download" and get the appropriate version
  2. Install and launch the Ollama application
  3. Select a model (Llama 2, Mistral, etc.) from the interface
  4. Initiate a chat to trigger model download

Key advantages of Ollama:

  • Runs completely locally - no internet required after setup
  • No API limits or costs
  • Multiple model options for different needs
  • Persistent memory across sessions

Performance tip: Smaller models like Mistral 7B work well for most assistant tasks, while larger models are better for complex reasoning. The video shows model selection at 13:15.

Critical Configuration Updates

The most technical but essential step comes at 14:30 - modifying OpenClaw's config.json to work with Ollama:

  1. Navigate to ~/.openclaw directory
  2. Open config.json in a text editor
  3. Add the Ollama provider snippet before the agents section: 
    {   "providers": [     {       "name": "ollama",       "base_url": "http://localhost:11434",       "api_key": "anything",       "models": ["llama2"]     }   ] }
  4. Save the file and rerun openclaw onboard
  5. Select Ollama as your provider when prompted

After restarting the gateway, test with:

  • What models do you have access to?
  • Find all spreadsheets modified yesterday
  • Create a meeting note for tomorrow at 3pm

Success indicator: At 15:50 in the video, the bot successfully lists PDF files using Ollama - now your AI assistant is fully local and private.
