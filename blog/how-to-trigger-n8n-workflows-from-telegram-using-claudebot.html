UGC Generation Example Walkthrough

Our example workflow automates User Generated Content (UGC) video creation - a process that typically requires multiple tools and manual steps. With ClaudeBot integration, you simply send a product photo and description via Telegram, and receive a finished UGC video minutes later.

The magic happens through a coordinated sequence: ClaudeBot updates a Google Sheet with your product details, n8n processes this data through AI image generation (Nano Banana) and video creation (Seedream), then delivers the final MP4 back to your Telegram chat.

Step 1: Send Product Details via Telegram

Message ClaudeBot with the command format: "Append Google sheet product [name] ICP [description] features [details] video setting [context]". Attach a product photo.

Step 2: Trigger the Workflow

ClaudeBot hits your n8n webhook URL, initiating the automation sequence. The rocket emoji confirms successful triggering.

Step 3: Automated Content Creation

The workflow generates an AI product image based on your photo, then creates a 30-second UGC-style video with voiceover matching your description.

Step 4: Delivery to Telegram

n8n converts the final video to Telegram-compatible format and ClaudeBot sends it directly to your chat, typically within 2-3 minutes.

In summary: Send product details → Trigger workflow → AI generates assets → Receive finished video in Telegram. All while you're mobile.

ClaudeBot Integration Setup

Connecting ClaudeBot to your n8n workflows requires careful configuration to ensure smooth operation and security. The integration hinges on ClaudeBot's ability to understand and execute specific commands that trigger your automations.

First, you'll need to train ClaudeBot to recognize your workflow trigger phrases. In our example, we used "/n8nUGC" as the activation command. This training involves providing ClaudeBot with:

  • Your n8n webhook URL
  • Expected input format
  • Authentication credentials
  • Response handling instructions

ClaudeBot's intelligence handles the complex coordination between Telegram's interface and your n8n workflow's requirements. It automatically formats your messages and attachments into the structured data your automation expects.

n8n Webhook Configuration

The heart of this integration is n8n's webhook node, which receives and processes ClaudeBot's triggers. Proper configuration here ensures reliable operation and security.

Key webhook settings include:

  • HTTP Method: POST
  • Authentication: Custom Header with API Key
  • Response Mode: Immediately
  • Path: /claudebot-[workflow-name]

For our UGC example, we added specific data mapping to transform ClaudeBot's message into the format our workflow required. This included extracting:

  • Product name from message text
  • Image URL from attachment
  • Target audience description
  • Product features list

Security note: Always use a randomly generated 32-character API key for authentication. ClaudeBot can help create this secure key and store it properly in its configuration.

Required API Connections

The UGC generation example relies on several API integrations that ClaudeBot coordinates automatically. Each serves a specific purpose in the end-to-end workflow.

Google Sheets API

Stores product details from your Telegram message in a structured format. Requires:

  • Google Cloud Console access
  • Enabled Google Sheets API
  • Service account credentials

IMGBB API

Hosts product images you send via Telegram, providing accessible URLs for the workflow. Free tier available.

RunPod APIs

Power the AI generation components:

  • Nano Banana for product image generation
  • Seedream for video creation

ClaudeBot handles authentication and coordination between these services, significantly simplifying what would otherwise require multiple manual steps.

Security Considerations

While remote workflow triggering offers incredible convenience, it introduces new security considerations that must be addressed.

Our implementation includes three key security layers:

  1. API Key Authentication: ClaudeBot must provide a valid 32-character key with each webhook call
  2. Command Whitelisting: Only pre-approved trigger phrases are accepted
  3. Input Validation: All incoming data is sanitized before processing

Additionally, we recommend:

  • Running ClaudeBot on a separate VPS rather than your main machine
  • Regularly rotating API keys
  • Implementing usage limits on your webhook
  • Monitoring workflow execution logs

These measures ensure your automation remains both accessible and secure when triggered remotely.
