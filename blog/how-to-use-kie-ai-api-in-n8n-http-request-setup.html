How to Connect KIE.ai API to n8n Using HTTP Requests (Step-by-Step Guide)

How KIE.ai Differs From OpenAI in n8n

Many n8n users are familiar with OpenAI's built-in credential type that makes integration seamless. KIE.ai takes a different approach requiring manual HTTP request configuration. This gives you more control but requires precise setup.

The key difference is that instead of selecting "OpenAI" from n8n's credential types, you'll create standard HTTP request nodes and manually add the Authorization header with your KIE.ai API key. This method works identically whether you're using n8n cloud or self-hosted instances.

Important: Unlike OpenAI's native integration, you'll need to paste your KIE.ai API key in every HTTP request node that communicates with their services. There's no central credential storage for KIE.ai in n8n.

Step 1: Getting Your KIE.ai API Key

Before you can connect KIE.ai to n8n, you'll need to generate an API key from your KIE.ai account. The process is straightforward but requires knowing where to look in their interface.

After logging into your KIE.ai account (or creating one if you don't have it yet), navigate to the "API Market" section in the left sidebar. From there, click on "API Keys" and look for the blue "+" button to create a new key. Name it something memorable like "n8n integration" so you can identify it later.

Key Generation Steps:

  1. Log in to your KIE.ai account
  2. Navigate to API Market → API Keys
  3. Click the blue "+ Create new key" button
  4. Enter a descriptive name (e.g., "n8n workflow")
  5. Click "Create" and immediately copy your new API key

Security Tip: Treat your KIE.ai API key like a password. If it's compromised, anyone can use your credits. Never share it publicly or commit it to version control.

Step 2: Properly Formatting the Bearer Token

The most common mistake when setting up KIE.ai in n8n is incorrect bearer token formatting. Unlike some APIs that accept the key directly, KIE.ai requires proper "Bearer" prefix formatting.

In each HTTP request node, you'll need to add an Authorization header with the value formatted as "Bearer YOUR_API_KEY" - with a single space between "Bearer" and your actual key. The capitalization of "Bearer" matters, and the space is critical for the API to recognize your credentials.

Correct Format: Bearer abc123def456ghi789 (replace with your actual key)

If you're using n8n's UI, you'll add this as a header parameter with "Authorization" as the name and your properly formatted bearer token as the value. In code mode, you would add it to your headers object as { "Authorization": "Bearer YOUR_API_KEY" }.

Step 3: Configuring the HTTP Request Node in n8n

With your API key copied and properly formatted, it's time to set up the HTTP request node in n8n. This is where you'll actually connect to KIE.ai's services.

Add an HTTP Request node to your workflow and configure it as follows:

Configuration Steps:

  1. Set the request method to POST (for most KIE.ai endpoints)
  2. Enter the appropriate KIE.ai API URL for the service you're using
  3. Add headers: Content-Type → application/json
  4. Add headers: Authorization → Bearer YOUR_API_KEY
  5. Configure your request body with the required parameters

At 1:32 in the video tutorial, you can see exactly where to paste your API key in the HTTP request node's Authorization header field. The space after "Bearer" is clearly visible in the demonstration.

Step 4: Testing Your API Connection

Before building complex workflows, always test your basic API connection to verify everything is configured correctly. n8n's workflow emulation makes this easy.

Click the "Execute Workflow" button to test your HTTP request node. If configured correctly, you should receive a valid response from KIE.ai's API. Check for:

  • HTTP status code 200 (success)
  • Expected response data structure
  • No authentication errors in the response

Troubleshooting Tip: If you get 401 errors, double-check your bearer token formatting. The space after "Bearer" is the most common oversight.

Using One API Key Across Multiple Workflows

While you need to include your KIE.ai API key in every HTTP request node, you can use the same key across all your n8n workflows. This simplifies management but requires attention to security.

For teams or complex automation systems, consider these best practices:

  • Create separate API keys for different departments or purposes
  • Monitor usage in KIE.ai's dashboard to detect anomalies
  • Rotate keys periodically (especially if team members leave)
  • Store keys securely and never in plain text in shared documents

Remember that each time you modify a workflow or add new KIE.ai integrations, you'll need to paste your API key (or a relevant key) into the new HTTP request nodes' Authorization headers.
