Step 1: Installing Ollama Locally

The foundation of your private AI system is Ollama — an open-source framework that manages large language models on your hardware. Installation takes under 5 minutes:

For macOS:

  1. Download the installer from ollama.com
  2. Double-click the .pkg file and follow prompts
  3. Verify installation by running ollama --version in Terminal

For Windows:

  1. Get the .exe from ollama.com
  2. Run the installer as Administrator
  3. Check it works via PowerShell with ollama list

Pro Tip: If you see "command not found" errors, add Ollama to your system PATH or restart your terminal session.

Step 2: Setting Up Llama 3

With Ollama running, we'll deploy Meta's powerful Llama 3 model — completely offline:

  1. Open your terminal (macOS/Linux) or PowerShell (Windows)
  2. Enter ollama pull llama3 — this downloads ~4GB model files
  3. Test with ollama run llama3 — ask a simple question to confirm responses

The first run may take 1-2 minutes as the model loads into memory. Subsequent queries will respond in 1-3 seconds on most modern laptops.

Note: For weaker hardware, try llama3:8b-instruct-q4_0 — a quantized version that runs on 8GB RAM.

Step 3: Configuring n8n's AI Agent

Now we'll connect n8n to your local Ollama server — creating an automation powerhouse:

  1. In n8n, create a new workflow and add a Manual Trigger
  2. Add an "AI Agent" node from the right panel
  3. Configure with these settings:
    • Chat Model: Ollama
    • Base URL: http://localhost:11434
    • Model Name: llama3 (exact match)
    • System Prompt: "You are an AI assistant. Answer clearly and concisely."

This setup tells n8n to route all AI requests to your local Ollama instance instead of cloud providers.

Step 4: Testing Your First AI Workflow

Let's validate everything works end-to-end (timestamp 2:45 in video):

  1. Click "Execute Workflow" in n8n
  2. Enter a test question like "Summarize this in one sentence: [paste text]"
  3. Watch for the response in n8n's execution panel

If you get connection errors:

  • Confirm Ollama is running (ollama list should work)
  • Check n8n's Base URL matches Ollama's port (default 11434)
  • Verify model name spelling matches exactly

First-run tip: Initial responses may be slow as the model warms up — subsequent calls will be faster.
