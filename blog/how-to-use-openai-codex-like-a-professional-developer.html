Essential CLI Commands Every Developer Should Know

The Codex CLI provides powerful control over AI-assisted development directly from your terminal. Beginners often struggle with the array of available commands, but mastering just a few fundamentals unlocks most of Codex's potential.

Starting with the basics, the slash (/) command brings up all available options. From here, you can change models (we recommend always using the latest), adjust reasoning levels (medium to high for most tasks), and check your usage status. The fast mode command toggles between standard and accelerated response times - useful when iterating quickly but consumes tokens faster.

Critical CLI Commands:

  • /model - Switch between available AI models
  • /reasoning - Adjust how deeply the AI thinks about problems
  • /fast - Toggle faster response times (uses more tokens)
  • /status - View your current usage and limits
  • /new - Start a fresh conversation context
  • /resume - Return to previous conversations

Unlocking the Power of the Codex Desktop App

While the CLI offers precision control, the Codex desktop application provides a visual interface that dramatically improves multitasking across projects. The left panel organizes all your active projects, while the main window shows code changes, conversations, and execution results.

One standout feature is the visual diff tool that highlights exactly what changes Codex proposes before you accept them. Pro tip: Use Command+B to toggle the sidebar and Command+Option+B for the right panel - these shortcuts help maximize screen real estate when focusing on complex tasks.

Configuring Permissions for Optimal Security and Workflow

Codex's permission system balances security with developer productivity, but the options can seem overwhelming at first. The three key settings - sandbox mode, network access, and approval policy - determine what the AI can do on your system and when it needs your explicit consent.

For most development work, the workspace-right sandbox mode is ideal. This allows Codex to read any file (necessary for understanding your project) but only write to your current working directory. Pair this with approval-on-request to maintain control over sensitive operations while avoiding constant interruptions.

These settings are stored in ~/.codex/config.toml and apply globally across both CLI and desktop app. Advanced users working in trusted environments might opt for full-access sandbox with never-ask approval policy for completely uninterrupted workflows.

Supercharging Productivity with Skills and Plugins

Skills represent one of Codex's most powerful yet underutilized features. These standardized operating procedures (defined in markdown files) teach the AI how to perform specific tasks in your preferred way. Common examples include Git workflows, code review standards, and testing procedures.

Plugins extend this concept further by integrating external tools like project management systems. For instance, the Linear plugin connects Codex to your issue tracker, allowing the AI to create tickets, update statuses, and reference tasks directly in your code commits. The combination of skills and plugins creates a customized AI assistant tailored to your exact workflow.

Implementation tip: Start with a basic agents.md file in your project root describing your core workflow. Keep it minimal - only include what the AI can't infer from your code structure. Add skills gradually as you identify repetitive tasks.

Integrating Codex with Your Git Workflow

Codex revolutionizes version control by understanding and participating in your Git workflow. The desktop app visually tracks all changes the AI makes, allowing you to review diffs before committing. For team projects, Codex can automatically include ticket IDs in commit messages and update corresponding issues in your project management system.

The Git work tree feature is particularly valuable when running multiple tasks concurrently. It creates isolated copies of your repository so different AI agents can work simultaneously without conflicts. When you're ready, changes from work trees merge back into your main branch cleanly.

Creating Scheduled Automations for Recurring Tasks

Codex's automation feature transforms tedious recurring tasks into scheduled background processes. Common examples include dependency updates, test suite runs, and data processing jobs. These automations can run locally or in isolated work trees to prevent environment pollution.

Setting up an automation is as simple as describing the task and schedule in plain English. The AI generates the necessary configuration, which you can refine through conversation. The desktop app provides visual feedback on automation status and results, making it easy to monitor without digging through logs.
