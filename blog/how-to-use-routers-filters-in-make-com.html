What Are Routers in Make.com?

Routers are Make.com's powerful flow control tools that transform linear automations into dynamic decision trees. Where traditional automations follow a single path from start to finish, routers create multiple parallel paths that can process data differently based on your business rules.

Imagine your sales team growing from 2 to 10 members - manually assigning leads becomes exponentially more complex. Routers solve this by automatically splitting your workflow into branches, each handling a specific scenario. At 1:15 in the video, you'll see how clicking the router icon instantly creates multiple paths from a single trigger point.

Key benefit: Routers reduce complex if-then logic into visual branches that are easier to build, understand, and modify as your team evolves.

Understanding Filters as Gatekeepers

While routers create multiple paths, filters determine which data travels down each branch. These digital gatekeepers inspect every piece of information and only allow it to proceed if it meets specific conditions - like a bouncer checking IDs at a club entrance.

In our lead routing example at 2:30, filters check which team member is assigned to each lead. One filter looks for "Shazam" in the assignment field, while another searches for "Juned". The lead only continues down the branch where the filter condition matches, ensuring perfect routing accuracy.

Pro tip: Using "contains" rather than "equals" in your filter conditions makes your automation more resilient to minor variations in team member names.

Practical Example: Lead Routing Automation

Let's walk through the complete automation shown in the video that eliminates manual lead assignment. The scenario: your Google Sheet receives new leads that need immediate attention from either Shazam or Juned based on their assignment field.

The automation begins with a Google Sheets trigger watching for new rows. Each new lead passes through a router that splits the flow into two branches. Filters then examine the assignment field and route the lead to the appropriate team member's notification path.

Time saved: What typically takes 5-10 minutes per lead assignment now happens instantly, saving sales teams 5+ hours weekly on administrative work.

Step-by-Step: Setting Up the Router

Step 1: Add the Router Module

After your trigger (Google Sheets in this example), click the "+" button and select "Router" from the Flow Control section. You'll see the familiar branching icon appear in your scenario.

Step 2: Create Multiple Paths

By default, routers create two paths. You can add more by clicking the small "+" on the router module. For our lead routing, we need two paths - one for each team member.

Step 3: Name Each Branch

Double-click each path's label to rename them descriptively (e.g., "Shazam Leads" and "Juned Leads"). This makes your automation self-documenting and easier to maintain.

In summary: Add router after trigger, create needed branches, and label them clearly before adding filters.

Configuring Filters for Team Members

Step 1: Add Filter to First Branch

Click the three dots between modules on the first path and select "Add a filter". Set the condition to check if the "Team Assignee" field contains "Shazam".

Step 2: Repeat for Second Branch

Add another filter to the second path checking for "Juned" in the same field. These filters ensure leads only proceed down their assigned team member's path.

Step 3: Test Filter Logic

Use Make.com's "Run Once" feature with sample data to verify filters route leads correctly before adding notification actions.

Remember: Filters silently stop unmatched data - add error handling if you need to catch unassigned leads.

Notification Setup for Each Branch

With routing logic complete, we'll add personalized notifications. At 4:45 in the video, you'll see how to configure Gmail actions for each team member, but you could use Slack, SMS, or other channels.

Each branch gets its own notification module with team-specific details. The Shazam branch sends to his email with his name in the greeting, while the Juned branch uses his contact information and name.

Customization tip: Use merge tags to include lead details like name and contact information in notifications for complete context.

Testing and Verifying the Automation

Before running live, test with sample data that includes both team assignments. At 5:30 in the video, you'll see the test run processing four leads - two for each team member.

Verify that emails (or other notifications) contain the correct information and only go to the intended recipients. Check your spam folder if notifications don't appear, and consider using a more reliable channel like Slack for critical alerts.

Monitoring: Set up a separate notification for yourself to track automation performance without interfering with the team workflow.

Routers in Make.com are flow control tools that allow you to split a single automation path into multiple branches. They transform linear workflows into dynamic decision trees where data can take different paths based on your business rules.

In lead routing scenarios, routers enable you to send different leads to different team members or departments automatically. Each branch can have its own series of actions and filters tailored to specific processing needs.

  • Create parallel processing paths from a single trigger
  • Visually organize complex decision logic
  • Easily add new branches as your needs evolve

Filters act as conditional gatekeepers between modules in your automation. They inspect each piece of data and only allow it to proceed if it meets specified criteria. Filters silently stop unmatched data unless you configure error handling.

In our lead routing example, filters check the assigned team member field and route the lead accordingly. You can create complex conditions using multiple fields and AND/OR logic to handle sophisticated routing rules.

  • Evaluate data against multiple conditions
  • Support text, numbers, dates and other field types
  • Can be added at any point in your workflow

Make.com filters support a wide range of condition types to handle various routing scenarios. Basic operators include equals, contains, starts with, ends with, greater than, and less than. You can combine conditions using AND/OR logic for complex rules.

For lead routing, text comparison operators like "contains" work well because they accommodate minor variations in team member names. For numerical or date-based routing, you might use greater/less than comparisons to prioritize leads.

  • Text comparisons (contains, equals, etc.)
  • Numerical comparisons
  • Date-based conditions
  • Combination logic with AND/OR
