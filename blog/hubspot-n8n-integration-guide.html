Why HubSpot Authentication Changed

If you've recently found your HubSpot API keys suddenly failing in n8n, you're experiencing HubSpot's ongoing security modernization. The company began phasing out traditional API keys in in favor of more secure OAuth 2.0 and app token authentication methods.

This shift catches many businesses off guard - workflows that ran perfectly for years suddenly break with cryptic permission errors. The change reflects HubSpot's commitment to enterprise-grade security, but creates immediate headaches for automation teams.

Key impact: By , all HubSpot portals will require OAuth 2.0 or app tokens for API access. Traditional API keys will completely stop working, breaking any existing n8n workflows still using them.

Step-by-Step Connection Guide

Connecting HubSpot to n8n takes just minutes when you use the right authentication method. Here's the foolproof process we use for all client implementations:

Step 1: Add HubSpot Node

In your n8n workspace, create a new workflow or open an existing one. Click "Add first step" or the "+" button between nodes, then search for "HubSpot" in the node picker. Select the official HubSpot node.

Step 2: Choose Authentication Method

In the node's credentials dropdown, click "Create new credential". You'll see three options: OAuth 2.0 (recommended), App Token, or API Key (legacy). For most users, OAuth 2.0 provides the best balance of security and convenience.

Step 3: Complete OAuth Flow

After selecting OAuth 2.0, click "Connect my account". A HubSpot window will open asking you to select the correct portal and approve permissions. If you have multiple HubSpot accounts, ensure you're authenticating with the right one.

Step 4: Name and Save Credential

Once authenticated, give your credential a descriptive name like "HubSpot Production" so you can easily identify it later. Verify the credential shows as "Connected" before proceeding.

OAuth vs. App Tokens: Which to Choose

While OAuth 2.0 works for most use cases, HubSpot app tokens provide specific advantages in certain scenarios. Here's how to decide which authentication method fits your needs:

OAuth 2.0 is best when: You need quick setup without developer involvement, want automatic token refresh, or are working with standard HubSpot features. It authenticates as your user account with all your existing permissions.

App tokens shine when: You need granular permission control, are building system-to-system integrations, or require credentials that aren't tied to an individual user's account. They're ideal for production environments where you want to minimize permission creep.

The key difference comes down to accountability - OAuth actions are attributed to your user account in HubSpot's audit logs, while app token actions show the private app as the actor. Choose based on your compliance and reporting requirements.

Solving Common Connection Errors

Even with proper authentication, HubSpot-n8n connections can fail for several reasons. These are the most frequent issues we encounter and how to resolve them:

"Invalid OAuth access token" errors typically mean your token expired before n8n could refresh it. The solution is to reconnect your credential - the new token will have a fresh expiration date. Consider switching to app tokens if this happens frequently.

Permission denied errors indicate your OAuth scopes or app token permissions don't match what the workflow requires. Check the specific API endpoint being called against your credential's permissions. You may need to reconnect with expanded scopes.

Critical note: If you see "API key is no longer supported" errors, your HubSpot portal has completely disabled legacy authentication. You must switch to OAuth 2.0 or app tokens immediately - there's no workaround.

Security Best Practices

While HubSpot's modern authentication methods are secure by design, these additional measures will protect your data and prevent unauthorized access:

Credential naming: Always give your n8n HubSpot credentials descriptive names indicating their purpose and environment (e.g., "HubSpot Production - Marketing Automation"). This prevents accidental misuse in workflows.

Permission minimization: When using app tokens, grant only the specific API permissions your workflow requires. HubSpot's developer portal lets you select individual CRUD operations for each object type.

Regular audits: Review connected apps in your HubSpot portal quarterly. Remove any unused or outdated n8n connections to reduce your attack surface.

Advanced Configurations

Once you've mastered basic HubSpot-n8n connections, these advanced techniques can unlock additional capabilities:

Multiple account management: Create separate credentials for different HubSpot portals (e.g., production vs. sandbox). Use n8n's credential sharing features to safely distribute access across your team.

Custom object integration: HubSpot Enterprise plans support custom objects. Configure your app token or OAuth scopes to include these, then use n8n's "Custom Object" resource type in the HubSpot node.

Webhook verification: When receiving HubSpot webhooks in n8n, implement signature verification using your client secret to ensure payloads haven't been tampered with in transit.
