The Problem With Standard Hunter Integration

Sales teams using Hunter.io through Zapier hit an immediate roadblock: the integration requires both first and last names to search for emails. As shown in the video at 0:45, attempting to search zapier.com without specific names returns zero results. This limitation makes the tool useless for prospecting when you only have company domains.

The frustration compounds when you realize Hunter.io's own website offers domain-wide searches without name requirements. Their API actually supports this functionality - it's just not exposed in the standard Zapier integration. This creates a false belief that automation isn't possible without complete contact details.

Key insight: Hunter.io maintains two separate search methods - personal verification (name+domain) and domain scraping (domain-only). The Zapier integration only implements the first method, forcing users into manual workarounds.

API Solution Overview

At 2:10 in the tutorial, we see the breakthrough: Hunter.io's domain search API endpoint doesn't require names. Instead of using their pre-built Zapier action, we'll make a direct API call to api.hunter.io/v2/domain-search with just three parameters:

  1. domain (required): The company domain you're targeting
  2. api_key (required): Your Hunter.io API key from account settings
  3. seniority (optional): Filter for "executive" level contacts

This approach mirrors what sales teams actually need - the ability to find all executive emails at a target company, not just verify known contacts. The API returns a structured JSON response with emails, positions, and verification data we can parse in Zapier.

Step-by-Step Zapier Setup

Step 1: Create a New Zap

Start with a blank Zap and add a "Webhooks by Zapier" trigger. Select "Custom Request" as shown at 3:15 in the video. This gives us full control over the API call structure.

Step 2: Configure the API Call

Set the method to GET and build the URL following this exact format:

 https://api.hunter.io/v2/domain-search?domain=COMPANY_DOMAIN&api_key=YOUR_KEY&seniority=executive&limit=3

Replace COMPANY_DOMAIN with either a hardcoded value or a dynamic field from a previous step. Your API key comes from Hunter.io's account settings (shown at 2:45).

Step 3: Test the Connection

At 4:30, we see the test return three executive contacts from zapier.com including names, positions, and LinkedIn URLs. The response structure automatically maps to Zapier's output fields for use in subsequent steps.

Pro tip: Add error handling by creating a filter step that checks if data.emails exists in the response. Route empty results to a notification or retry path.

Filtering Results for Quality

Not all emails Hunter.io finds are equally valuable. At 5:50 in the video, we examine the confidence scores and verification status included in each result:

  • Confidence scores (0-100): Filter for scores above 70 to avoid spam traps
  • Verification status: Prefer "valid" over "accept_all" domains
  • Seniority levels: Combine with department filters (e.g., "finance") for targeted outreach

Add a Zapier filter step after the API call to enforce these quality controls. For example, only pass through emails where:

 confidence > 70 AND verification = "valid" AND seniority = "executive"

This ensures your CRM only receives high-probability executive contacts worth your sales team's time.

Always include error handling steps in your Zapier workflow to manage these scenarios. For example, add a delay step between requests and create a separate path for contacts that fail validation checks.
