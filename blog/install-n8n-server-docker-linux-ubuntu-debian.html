Prerequisites for n8n Docker Installation

Before installing n8n, your Ubuntu or Debian server needs Docker properly configured. At the 0:45 mark in the video, we see Docker must be installed and running - if not, you'll need to install it first using the official Docker documentation.

Portainer is highly recommended for visual container management. Unlike command-line Docker tools, Portainer provides a web interface that makes monitoring and controlling your n8n instance significantly easier, especially for teams with multiple users.

Critical requirement: You'll need a valid email address for n8n account registration. This becomes your admin login and is where the free license key gets sent for activating advanced features.

Setting Up Persistent Storage Folders

The biggest mistake when deploying n8n with Docker is neglecting persistent storage. Without proper volume mapping, all your workflows disappear when the container restarts. At 2:15 in the tutorial, we create dedicated folders for configuration and data.

Create two directories: one for n8n configurations and another for persistent data storage. The data folder must have correct permissions (user 1000 with read/write/execute) to ensure n8n can access it properly within the container environment.

Permission command: sudo chown -R 1000:1000 ./n8n-data && sudo chmod -R 755 ./n8n-data ensures the container can read/write your workflow data.

Environment Variable Configuration

The .env file contains critical settings that control how n8n operates. At 4:30 in the video, we configure the host to 0.0.0.0 (allowing external access), port 5678 (n8n's default), and production mode for optimal performance.

Time zone configuration is often overlooked but crucial for proper scheduling of automated workflows. Set TZ to your local timezone (like America/New_York) so workflow triggers execute at the expected times.

Docker Compose File Setup

The docker-compose.yml file defines the entire container configuration. At 6:10, we specify the latest n8n image, container name, restart policy (always), port mapping, and volume binding to our persistent storage folder.

Key elements include the user: "1000" setting (matching our folder permissions) and linking the .env file we created earlier. This compose file makes deployment reproducible - you can destroy and recreate the container with identical settings.

Portainer Management Interface

While Docker CLI works, Portainer (shown at 8:45) provides visual container management. It shows real-time logs, resource usage, and lets you start/stop containers with one click - invaluable when troubleshooting workflow issues.

Portainer also simplifies updates - when a new n8n version releases, you can recreate the container with the updated image through Portainer's web interface without touching the command line.

Firewall and Network Access

If your server firewall is active (like UFW), you must allow port 5678 for n8n access. At 10:20, we run sudo ufw allow 5678/tcp - without this, your browser can't connect to the n8n interface.

For production use, consider adding Nginx as a reverse proxy with SSL termination. This provides HTTPS encryption and lets you use a standard port (443) instead of 5678 for cleaner URLs and better security.

First Launch and Account Registration

When first accessing n8n at your-server-ip:5678 (11:50 in video), you'll complete initial setup. This includes email registration, basic preferences, and answering usage questions that help n8n optimize its interface for your needs.

The email you register becomes your admin account - choose one you control long-term. This same email will receive your free license key for unlocking advanced features covered in the next section.

License Key Activation for Advanced Features

n8n's free license (activated at 13:30) unlocks powerful capabilities: workflow history lets you audit past executions, advanced debugging tools help troubleshoot issues, and folders organize complex automation setups.

Check your email for the license key, then paste it into n8n's activation screen. These professional features are essential for business use - without them, you're limited to basic functionality that quickly becomes inadequate.

Pro tip: The license ties to your email, not your server. If you rebuild your n8n instance, you can reuse the same license key by registering with the original email.
