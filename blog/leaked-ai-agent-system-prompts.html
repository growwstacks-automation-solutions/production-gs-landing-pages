The 137,000-Star Repository No One Took Down

For 14 months, a GitHub repository has publicly hosted the complete system prompts of 32 AI coding tools—including industry leaders like Cursor and Claude Code—without a single takedown notice. Unlike model weights which are fiercely protected, these operational manuals have remained open despite containing proprietary workflow rules and behavioral constraints.

The repository reveals that Anthropic's Sonnet prompt alone spans 99,000 bytes (roughly 80 printed pages) of instructions before user interaction begins. These aren't simple priming statements but full operational manuals governing everything from file editing priorities to framework restrictions.

Key insight: The same maintainer who curates these leaks also runs Zero Leaks—a paid service helping AI startups prevent prompt extraction. This creates a unique ecosystem where one entity both exposes and protects proprietary prompts.

Confidence Boosts and Stack Locks

Devon's prompt begins with something unexpected—a confidence-building pep talk: "You are Devon, a software engineer using a real computer operating system. You are a real code wiz. Few programmers are as talented as you at understanding code bases." This isn't documentation; it's startup confidence injected directly into the model.

Lovable takes a different approach with explicit framework prohibitions: "Lovable projects are built on top of React, Vit, Tailwind CSS, and TypeScript. Therefore, it is not possible for lovable to support other frameworks like Angular, Vue, Spelt, or Next." The stack lock exists in the prompt itself, overriding any user requests for prohibited technologies.

Tool Use Schemas as Differentiators

Cursor's agent prompt 2.0 reveals four distinct search paths with specific rules: "Exact text matches use gp. Reading a known file use read file." Claude Code opens with a system reminder block: "Always prefer editing an existing file to creating a new one. Never proactively create documentation files."

These constraints create dramatically different behaviors from the same underlying models. Devon includes specialized commands like "report environment issue" and specific rules about test files—features completely absent from Claude Code's prompt architecture.

Implementation insight: When building AI tools, the real product differentiation comes from the tool use schema—the specific permissions, constraints, and workflow rules wrapped around the base model.

The Identical Rules Across Competing Tools

Searching the repository for one phrase—"Never assume that a given library is available"—returns hits from Devon, Cursor, and Anthropic's prompts. Three competing tools, one identical rule. This pattern repeats across the collection: be concise, maximize efficiency, don't make assumptions about the environment.

The boring parts all sound the same because they solve universal challenges in AI tooling. The interesting differentiators live in the control panels—Cursor's search paths, Claude Code's file discipline, Devon's test rules—that make each product feel unique despite similar foundation models.

Why Prompt Engineering Is the Real Moat

The leaks demonstrate that leading AI tools are workflow engines wrapped around frontier models—not just chat interfaces with better prompts. The fine-tune is barely the moat; the real competitive advantage comes from tool use schemas, permissions, and constraints.

This explains why companies like Anthropic invest heavily in prompt engineering despite using similar underlying technology to competitors. Their Sonnet prompt's 80 pages of instructions create behaviors and safeguards that can't be replicated by simply using the same model family.

The Legal Paradox of Leaked Prompts

The repository's longevity highlights a legal gray area—while model weights are protected intellectual property, system prompts exist in a more ambiguous space. The same maintainer operates both this exposure hub and Zero Leaks (a prompt protection service), creating a self-sustaining ecosystem around prompt secrecy.

This duality suggests that as AI products mature, we'll see more sophisticated prompt protection measures—not just legal threats but technical safeguards against extraction. The current leaks may represent a brief window before stricter controls emerge.
