How the Workflow Works

This Make.com automation has two main components working together seamlessly. The first part handles content discovery and preparation, while the second manages publishing.

At 2:15 in the video, you can see how these two parts connect through Google Sheets, creating a complete system that runs on autopilot once configured.

System overview: Content scraping → Duplicate check → AI filtering → Brand voice rewriting → Automatic scheduling → Timed publishing

Part One: Content Discovery

The content discovery module does the heavy lifting of finding and preparing posts for publication:

Step 1: Content Scraping

An Appify webhook triggers whenever new content is scraped from LinkedIn, pulling in fresh posts from your target sources.

Step 2: Duplicate Check

The system compares new posts against your existing database, dropping any duplicates to maintain content originality.

Step 3: Content Classification

An OpenAI filter analyzes each post, categorizing it as either personal (to discard) or evergreen (to keep and repurpose).

Step 4: Brand Voice Rewriting

Kept posts are rewritten in your specified tone and style using another OpenAI node, then stored in your generated posts database.

Step 5: Automatic Scheduling

Each new post is scheduled to publish exactly one hour after the last scheduled post, creating perfect posting cadence.

In summary: Scrape → Filter → Rewrite → Schedule. This process runs automatically whenever new content is available.

Part Two: Content Publishing

The publishing module is simpler but equally important — it handles the actual posting to LinkedIn:

Step 1: Schedule Check

At each interval (default hourly), the system checks your scheduled posts database for content due to publish.

Step 2: Content Retrieval

If a post is scheduled for the current time window, the workflow retrieves the complete post content and media.

Step 3: LinkedIn Publishing

Using Make.com's LinkedIn connector, the post is published automatically with all formatting intact.

Key advantage: The publishing module can run independently on a schedule, while the content discovery module runs whenever new content is available.

Customization Options

While the core workflow is powerful as-is, several customization options are available:

Brand Voice Parameters
