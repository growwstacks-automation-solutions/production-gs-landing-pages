Key benefit: A manufacturing client reduced equipment check-in time by 73% when technicians could verbally trigger n8n workflows instead of using tablets.

System Architecture Overview

The modular design uses Docker containers for each component, making it adaptable to different business environments. At its core, LiveKit handles the real-time voice pipeline, connecting to speech-to-text models like Whisper and text-to-speech systems like Kukoro or Piper.

Ollama serves as the AI brain, using Model Context Protocol (MCP) to dynamically discover available tools from your n8n instance. This architecture means every new workflow you create in n8n automatically becomes a voice-commandable function - no manual integration needed.

Step-by-Step Installation Guide

Step 1: Environment Setup

Begin by cloning the repository and configuring the .env file with your local IP address. The video at 5:30 shows the critical network settings that ensure components can communicate properly.

Step 2: Docker Configuration

Modify the docker-compose.yml to set network_mode: host and update LiveKit URLs to localhost. These changes (shown at 7:45 in the video) resolve common connection issues between components.

Step 3: Model Selection

Choose between GPU-accelerated models like Kukoro for best performance or CPU-optimized options like Piper for lightweight deployment. The web interface at port 3000 lets you test different configurations.

Pro tip: For business deployments, allocate at least 8GB RAM and 4 CPU cores for smooth operation with multiple concurrent users.

n8n Workflow Auto-Discovery

The magic happens through Model Context Protocol (MCP), which continuously scans your n8n instance for new workflows. Each discovered workflow becomes available through natural language commands like "Run the inventory check" or "Update the CRM with new leads."

At 9:20 in the video, we see how the system automatically surfaces workflow parameters and converts them into conversational prompts. This eliminates the need to pre-program specific voice commands for each automation.
