What Is the Get Function?

Many automation builders hit a wall when they need to extract specific data from complex nested structures. The Get function in Make.com solves this by letting you precisely target and retrieve individual elements from arrays and collections.

Unlike modules that process entire data sets, the Get function acts like a surgical tool - extracting only what you need. This becomes invaluable when working with API responses, eCommerce orders, or CRM data where you might only need specific fields from deeply nested objects.

Key insight: The Get function is most powerful when combined with other functions. While it can retrieve standalone items, its true value emerges when used with Sort, Map, or conditional logic to create sophisticated data processing workflows.

Function Structure Explained

The Get function follows a specific syntax that must be precisely followed:

 get(array_or_collection; path_to_value)

The first parameter identifies the source data, while the second specifies exactly what to extract. For arrays, you'll typically use an index (like 0 for the first item), while for collections you'll use dot notation to navigate nested properties.

Remember: Every character matters in the function syntax. Missing a semicolon or using incorrect capitalization will cause the function to fail. Always test your Get functions with "Run once" before deploying them in live scenarios.

Mastering Dot Notation

Dot notation is the key to unlocking nested data structures. It works by chaining property names with dots to "drill down" into objects:

 customer.address.city

This example would extract just the city from a customer's address object. The tutorial demonstrates this at 7:35 with a product example - using boots.color to get just the color property from a boots product object.

When working with arrays of collections, you'll often combine array indices with dot notation:

 get(orders; "0.product.name")

This would get the product name from the first order in an orders array.

The Importance of Raw Names

One of the most common pitfalls when using the Get function is mismatching property names. Make.com uses "raw names" behind the scenes that may differ from what's displayed in the UI.

To find a property's raw name:

  1. Hover over the item in the mapping panel
  2. Look for the "raw:" label in the tooltip
  3. Use that exact spelling and capitalization in your function

At 9:20 in the video, we see a critical example where "item_color" in the UI actually has a raw name of "itemcolor". Using the displayed version would fail, while the raw name works perfectly.

Common Mistakes to Avoid

After implementing hundreds of Get function scenarios, we've identified these frequent pitfalls:

  • Case sensitivity: Raw names are exact - "itemname" ≠ "itemName"
  • Incorrect paths: Verify each step in your dot notation path exists
  • Missing data: Always include error handling for cases where expected data isn't present
  • Overuse: Sometimes a simple mapping is better than multiple Get functions
