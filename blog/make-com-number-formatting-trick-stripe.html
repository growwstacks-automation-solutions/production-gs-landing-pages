Why Stripe Numbers Show Up Wrong

If you've connected Stripe to Make.com and seen payment amounts displaying as 15000 instead of $150, you've encountered a common frustration. This isn't a bug - it's how payment processors avoid decimal calculation errors. Stripe and others process all amounts in cents, so $150 becomes 15000 in their systems.

While this approach prevents rounding errors in financial calculations, it creates display issues when the data reaches Make.com. Your reports, notifications, and CRM updates show numbers that are 100x larger than they should be, confusing both your team and customers.

Key insight: Payment processors use cents (15000) instead of dollars (150.00) because floating-point math can introduce tiny rounding errors that accumulate over thousands of transactions. Working in whole numbers eliminates this risk.

The Simple Make.com Fix

The solution to Stripe's number formatting issue requires just one module in Make.com - the Set Variable tool. By dividing the Stripe amount by 100, you convert cents back to dollars while maintaining perfect accuracy.

This approach works because it's performing integer division on an integer value. Unlike floating-point division which can create repeating decimals, dividing cents by 100 always produces clean dollar amounts when displayed. Step-by-Step Implementation

Here's exactly how to implement the number formatting fix in your Make.com workflows:

Step 1: Locate Your Stripe Amount

In your Make.com scenario, identify the module where Stripe's payment amount appears. This is typically in a "Charge Created" or "Payment Succeeded" webhook.

Step 2: Add a Set Variable Module

Drag a Set Variable module after your Stripe trigger. Name it something clear like "Format Currency Amount".

Step 3: Create the Division Formula

In the variable value field, use this formula: {{amount}}/100 (replace "amount" with your actual field name from Stripe). The brackets tell Make.com to use the dynamic value.

Step 4: Test Your Workflow

Use Make.com's Run Once feature to test with real Stripe data. Verify the output shows the correct dollar amount without extra zeros.

In summary: 1) Find Stripe amount → 2) Add Set Variable → 3) Divide by 100 → 4) Test output. This four-step process fixes number formatting in minutes.

Handling Currency Symbols

Currency symbols introduce another layer of complexity. The video shows what happens when both Stripe and your formatting add dollar signs - you get duplicate symbols ($150).

The best practice is to let Stripe provide the symbol in its currency field rather than adding it manually in Make.com. This ensures proper localization for different currencies and prevents symbol duplication in your outputs.

Testing Your Formatting

Testing is crucial when working with financial data. Make.com's Run Once feature lets you verify your formatting without processing real transactions.

Test with various amount types: whole dollar amounts ($100), amounts with cents ($19.99), and large values ($10,000). Check how the formatted numbers appear in subsequent modules like Google Sheets or email notifications.
