What Is Make.com? (Formerly Integromat)

Make.com (previously known as Integromat) is a no-code automation platform that connects your apps and services to create powerful workflows. Unlike basic automation tools that simply move data from point A to B, Make.com provides a visual builder where you can create complex, multi-step automations called "scenarios."

Every scenario starts with a trigger - an event like a new form submission, incoming email, or database update. Once triggered, your scenario can perform multiple actions across different apps: adding spreadsheet rows, sending messages, updating CRMs, and more. The real power comes from chaining these actions together with logic and conditions.

Key difference: While tools like Zapier focus on simple 1:1 app connections, Make.com specializes in complex workflows with multiple steps, branches, and transformations between apps.

The 3 Types of Make.com Automations

Most beginners don't realize Make.com supports three distinct types of automations, each with different capabilities:

1. Classic Automations

These are Make.com's foundational workflows - deterministic "if this, then that" sequences. Example: When a YouTube video is posted (trigger), share it to LinkedIn (action). Classic automations excel at repetitive tasks where the steps never change.

2. AI Automations

These enhance classic automations with artificial intelligence. Using the YouTube example, an AI automation might summarize the video before posting to LinkedIn. The key difference is the inclusion of at least one AI module that processes or enhances the data mid-flow.

3. AI Agents

The most advanced option, AI agents can make decisions and act autonomously based on inputs. Unlike classic automations with fixed logic, agents evaluate situations and choose appropriate responses. For example, an agent might analyze customer support tickets and either: 1) Answer directly, 2) Route to human support, or 3) Trigger a refund - all based on the ticket content.

For beginners: Start with classic automations, then progress to AI automations once comfortable. AI agents require more advanced understanding and are covered in our Make.com automation service.

Building Your First Automation

Let's walk through creating a practical automation from scratch - sending Slack alerts for new Google Form submissions. This demonstrates core Make.com concepts you'll use in nearly every workflow.

Step 1: Set Up the Trigger

All automations start with a trigger. For our example, we'll use "Watch Responses" from Google Forms - this fires whenever someone submits your form. After selecting Google Forms:

  1. Connect your Google account (one-time authorization)
  2. Paste your form's unique ID (found in the form URL)
  3. Set the limit to 1 (process only the newest submission each run)

Step 2: Add an Action

Next, we'll add Slack to send our alert. Select "Create a Message" and:

  1. Connect your Slack workspace
  2. Choose the channel for notifications
  3. Compose the message using data from the form (name, email, etc.)

Pro Tip: Use the "Run Once" feature to test your automation. Each module shows a bubble with its output - click these to inspect exactly what data is being passed between steps.

Adding AI Modules to Your Workflow

Now let's enhance our form-to-Slack automation with AI. We'll analyze the sentiment of feedback submissions and highlight negative responses.

Option 1: Make's Built-in AI Modules

Make offers pre-configured AI modules for common tasks. For sentiment analysis:

  1. Add the "Analyze Sentiment" module between your trigger and Slack action
  2. Map the form's feedback field as the text to analyze
  3. Configure to return both sentiment (positive/negative) and a description
  4. Update your Slack message to include the analysis

Option 2: Third-Party AI (OpenAI, etc.)

For more customization, connect services like OpenAI:

  1. Add an "OpenAI: Generate Response" module
  2. Write a prompt like: "Analyze this customer feedback. Return sentiment and key points:"
  3. Map the form feedback as the input
  4. Process the AI's response in your Slack message

Which to choose? Make's modules are easier for beginners (no prompt engineering), while third-party AI offers more flexibility. At 12:45 in the video, we show both approaches side-by-side.

Debugging & Troubleshooting

Even well-built automations sometimes fail. Here's how to diagnose common issues:

Problem: Incorrect Data Mapping

Symptom: Your Slack message shows raw field names instead of values.
Fix: In the Slack module, ensure you're selecting the actual answer values (under "answers") not the question objects.

Problem: Authorization Errors

Symptom: Modules fail with connection errors.
Fix: Reconnect the affected service in Make.com's connections panel. Tokens sometimes expire.

Problem: Unexpected AI Output

Symptom: AI modules return irrelevant or poorly formatted results.
Fix: For third-party AI, refine your prompt. For Make's modules, verify you're passing the correct data field.

Debugging workflow: 1) Run the scenario once, 2) Inspect each module's output bubble, 3) Identify where data diverges from expectations, 4) Correct the mapping in the next module.

Advanced Features: Filters & Routers

Once comfortable with basic automations, these powerful features unlock new possibilities:

Filters

Add conditions to control when actions execute. Right-click between modules to add a filter. Example: Only send Slack alerts for negative feedback (sentiment = negative).

Routers

Branch your workflow based on conditions. Right-click to add a router, then create multiple paths. Example: Route positive feedback to a "Happy Customers" Slack channel, negative to "Support Alerts."

Understanding Pricing & Credits

Make.com uses a credit-based system:

  • Free plan: 1,000 credits/month (~30-50 automation runs/day)
  • Pro plan: 10,000 credits + 1-minute execution cycles
  • Teams plan: 50,000 credits + collaboration features

Credit usage: Each module run consumes 1 credit. Our form-to-Slack example uses 2 credits per execution (Google Forms trigger + Slack action). Adding AI analysis would make it 3 credits.

Execution cycles: Free accounts can only run automations every 15+ minutes. Paid plans allow 1-minute intervals - crucial for time-sensitive workflows.
