The Agent Management Problem

As AI agents have become more capable, they've created a new challenge for developers and teams. No longer are we babysitting single chat sessions - we're now managing multiple agents working simultaneously on different tasks. This leads to terminal chaos with multiple windows open, constant context switching, and difficulty tracking which agents need input.

The community responded with various solutions like Vibe Kanban and T-Mux, but these were unofficial workarounds. The new Claude Code agent view provides a native solution built directly into the platform, offering a centralized dashboard to manage all your active agents.

90% completion rate: Modern AI agents can now complete most tasks from a single good prompt, making multi-agent management essential for productivity rather than just a convenience.

Getting Started with Agent View

To access the agent view, you'll need to be on Claude Code version 2.1.139 or higher. Check your current version by running claude --version in your terminal. If you need to update, simply run claude-update to get the latest version.

Once updated, launch the agent view by running claude agents. This opens the summary view - your central dashboard for all agent activity. The interface shows all active sessions organized by status by default, giving you immediate visibility into which agents are running, waiting for input, or completed.

Migrating Existing Sessions

If you have existing terminal sessions running when you start the agent view, they won't automatically appear in the dashboard. To migrate them, go to each active terminal session and type /bg (for "background"). This command continues the session in the background while freeing up your terminal.

As shown at the 1:45 mark in the video, backgrounded sessions immediately appear in your agent view dashboard. You can then safely close the original terminal windows - the sessions continue running in the agent view. This migration process works for any number of existing sessions.

No data loss: Backgrounding sessions preserves all conversation history and context, just like keeping the terminal window open.

Dashboard Features

The agent view dashboard offers several powerful features for managing multiple agents. By default, it sorts sessions by status (active, waiting for input, completed), but you can press Ctrl+S to toggle sorting by repository or folder structure instead.

Key navigation features include:

  • Shift+Up/Down arrows to reorder sessions manually
  • Ctrl+T to pin important sessions to the top
  • Spacebar to select and interact with agents directly from the dashboard
  • Arrow keys to jump into detailed session views when needed

Each session card shows the task description, last output snippet, and how long it's been waiting for input if applicable. This at-a-glance information helps you prioritize which agents need attention first.

Working with Multiple Repos

For developers working across multiple repositories, the repo sorting feature (Ctrl+S) becomes invaluable. It groups all agents by their working directory, letting you see all tasks related to a specific project at once.

Currently, the sorting doesn't drill down to subfolder levels - all sessions in a repository appear together regardless of which subfolder they're working in. As mentioned at 3:20 in the video, this limitation affects workflows that use subfolders for different clients or projects in an Agentic OS setup.

Despite this limitation, the repo-level sorting still provides significant organization benefits over managing separate terminal windows. Future updates may add subfolder-level sorting to better support complex Agentic OS architectures.

Interacting with Agents

The agent view allows two levels of interaction with your AI agents. From the dashboard view, you can use the spacebar to select any agent and type responses directly - perfect for quick approvals or simple follow-ups.

For more complex interactions, press the right arrow key to jump into the full session view. This shows the complete conversation history and gives you all the functionality of a regular terminal session. Press left arrow to return to the dashboard when done.

Terminal-only for now: Currently the agent view is only available in the terminal interface, but a desktop app version with similar functionality is likely coming in future updates.

Pairing with Agentic OS

The agent view works seamlessly with an Agentic Operating System setup. Since your Agentic OS is essentially a folder structure that injects context at the right time, each conversation in the agent view automatically benefits from your existing:

  • Brand voice and style guidelines
  • Client-specific contexts and preferences
  • Scheduled jobs and recurring tasks
  • Project-specific knowledge bases

This means you get all the organizational benefits of your Agentic OS combined with the multi-agent management capabilities of the new view. You can operate multiple client projects simultaneously without losing any of the context that makes your Agentic OS powerful.
