n8n Overview: Cloud vs Self-Hosted

Businesses choosing automation tools often face a dilemma: convenience versus control. n8n offers both options through its cloud and self-hosted versions. The cloud version provides quick setup with a 14-day free trial, ideal for testing workflows before committing. The self-hosted option gives complete data control and is free forever - perfect for regulated industries.

During setup, you'll notice key differences. Cloud users access n8n through app.n8n.cloud with subscription plans starting at $20/month. Self-hosted installations run on your infrastructure via Docker, requiring technical setup but offering unlimited workflows at no ongoing cost. Both versions share the same powerful workflow builder interface.

Pro Tip: Start with the cloud trial to learn n8n, then migrate to self-hosted for production workflows. This combines easy onboarding with long-term cost savings.

Workflow Breakdown

The demo workflow solves a common business problem: automating multi-step approvals with AI assistance. When a user sends a chat message (like requesting information), the AI agent processes it, remembers the conversation context, sends an email summary for approval, then takes action based on the response.

This creates a complete automation loop:

  1. Chat trigger receives user message
  2. AI agent processes request using OpenAI
  3. Simple memory node retains conversation history
  4. Gmail node sends approval email with response options
  5. IF condition routes workflow based on approval/rejection

The entire process runs in seconds, eliminating manual back-and-forth while maintaining human oversight.

Setting Up the AI Agent

Configuring the AI agent requires just three steps in n8n. First, add a chat trigger node to receive incoming messages. This creates the entry point for user interactions. Next, connect it to an AI agent node configured with your preferred language model (OpenAI in this demo).

The magic happens in the agent configuration. By connecting the chat input to the agent's prompt source, you create a seamless conversation flow. The agent automatically processes each new message through the AI model, returning thoughtful responses. For businesses using multiple AI services, n8n supports switching models with just a few clicks.

Implementation Note: The demo uses n8n's built-in OpenAI integration, but you can connect any AI service through HTTP requests or specialized nodes.

Implementing Conversation Memory

Traditional chatbots frustrate users by forgetting previous interactions. The simple memory node solves this by retaining conversation history. Configured to remember the last 10 messages, it provides context for each new exchange.

Adding memory transforms the workflow from transactional to conversational. When integrated with the AI agent, the system can reference past discussions naturally. This works particularly well for approval processes where context matters - like tracking why a request was previously rejected.

The memory implementation requires just one node between the chat trigger and AI agent. No additional coding or complex configuration needed - just set your desired message retention count and connect the nodes.

Email Approvals with Conditional Logic

The approval system demonstrates n8n's powerful conditional logic. After the AI generates a response, the workflow emails it for human review. The Gmail node's "send and wait for response" option pauses execution until the recipient acts.

Configuration options let you customize approval requests:

  • Approval buttons (Approve/Reject or custom labels)
  • Free-text response fields
  • Custom forms with multiple questions

The IF node then routes the workflow based on the response. Approved requests continue to fulfillment steps, while rejections loop back to the AI agent for refinement. This creates a self-correcting system that improves with each iteration.
