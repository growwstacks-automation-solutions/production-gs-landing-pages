What Is the Aggregate Node?

When preparing reports or sending batched API payloads, manually combining data from multiple sources is time-consuming and error-prone. The n8n Aggregate node solves this by automatically merging items based on your specified criteria. Think of it as your digital data organizer - transforming scattered information into structured, actionable outputs.

Unlike simple arithmetic operations, the Aggregate node specializes in restructuring data relationships. At 2:15 in the video tutorial, you'll see how it serves as the opposite of a Split node - instead of breaking data apart, it intelligently combines related items while preserving their contextual relationships.

Key capability: The Aggregate node can combine entire items or just specific fields, group by unique values (like customer IDs), and perform calculations on grouped data - all in one step.

Example 1: Aggregating Individual Fields

Many workflows need to combine specific fields from multiple records while leaving others separate. For instance, you might want to merge order amounts from three separate transactions while keeping customer details distinct.

Here's how to set this up:

Step 1: Configure Your Trigger

Start with a Manual Trigger node or connect to your data source (like Google Sheets as shown at 3:40 in the video). The key is ensuring your input data contains the fields you want to aggregate.

Step 2: Add the Aggregate Node

Search for "Aggregate" in the node panel and add it to your workflow. Select "Aggregate Individual Fields" as your operation type.

Step 3: Map Your Fields

In the node configuration, specify which fields to combine. For order amounts, you'd select the "amount" field from your input data. The node will output a new array containing your aggregated values.

Pro tip: Rename your output fields for clarity. Instead of "aggregatedField1", use descriptive names like "totalOrderAmount" that make sense in your workflow context.

Example 2: Grouping by Field Value

One of the most powerful features is grouping data by unique field values. This lets you organize records by customer, date, category, or any other shared attribute - exactly what you need for customer purchase histories or daily sales reports.

At 5:20 in the tutorial, you'll see how to:

  1. Select "Group by Field Value" as your operation type
  2. Choose your grouping field (like "customerID")
  3. Define which fields to include in each group

The output creates structured JSON with all related items nested under their group identifier. This format is perfect for feeding into reports or customer communication workflows.

Example 3: Include/Exclude Fields

Not all data needs to pass through aggregation. The Include/Exclude functionality (demonstrated at 7:05) lets you control exactly which fields get processed and which remain unchanged.

For a customer order workflow, you might:

  • Include: order amounts, product IDs, and timestamps for aggregation
  • Exclude: customer names and addresses that should remain as-is

This selective approach maintains data integrity while still allowing powerful transformations where you need them most.

Example 4: Mathematical Operations

While primarily a data organization tool, the Aggregate node also handles basic calculations on grouped data. After organizing records by your chosen field, you can:

  • Sum numeric values (like total sales per customer)
  • Calculate averages (like mean order value)
  • Find minimum/maximum values in each group

At 9:30 in the video, you'll see how these operations automatically apply to each group created in previous steps - no additional nodes required. The results can then feed directly into reports, dashboards, or notification systems.

Common Use Cases

The Aggregate node shines in several business scenarios that traditionally require manual data manipulation:

Daily/Monthly Reports

Combine sales, support tickets, or other metrics by time period for automated reporting. Instead of manually compiling spreadsheets, your workflow generates ready-to-share summaries on schedule.

Customer Activity Summaries

Group orders, support interactions, and engagement metrics by customer for personalized communications or account reviews.

Batched API Payloads

Many APIs limit request rates or charge per call. Aggregating data lets you send fewer, more efficient requests - cutting costs and improving performance.

Real-world impact: One ecommerce client reduced their monthly reporting time from 8 hours to 15 minutes by automating data aggregation with n8n.

Best Practices

To get the most from the Aggregate node while avoiding common pitfalls:

1. Test with Small Datasets First

Verify your field mappings and operations work as expected before processing large volumes. The "Execute Node" button lets you test each step independently.

2. Document Your Field Structure

Use clear, consistent naming for both input and output fields. This becomes crucial when chaining multiple Aggregate nodes together.

3. Add Error Handling

Include a Function or Error Trigger node after aggregation to catch and manage any unexpected data format issues.

4. Consider Performance

For very large datasets, break processing into smaller batches or use the Split node first to parallelize operations.

Grouping and aggregating are related but distinct operations in data processing. Grouping organizes records by shared characteristics, while aggregating performs operations on those groups.

In n8n's Aggregate node: 1) Grouping happens first, creating collections of related items, 2) Aggregating then applies operations (sum, average, etc.) to each group, and 3) The node outputs structured results that maintain these relationships for downstream use.

  • Grouping creates collections
  • Aggregating performs operations on collections
  • Both happen sequentially in the node
