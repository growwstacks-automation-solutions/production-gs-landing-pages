How to Connect Monday.com to Make.com with Webhooks (2026 Tutorial)

Tired of manually updating multiple systems when tasks change status in Monday.com? This webhook integration automatically syncs status changes to Make.com, where you can trigger Slack notifications, update other boards, or start approval workflows - all without lifting a finger.

The Problem With Manual Updates

Most teams using Monday.com waste hours each week on manual updates. When a task moves to "Done," someone has to remember to notify stakeholders in Slack, update related records in other systems, or move the item to an archive board. These repetitive tasks are:

  • Time-consuming: Takes 2-3 minutes per task × dozens of tasks weekly
  • Error-prone: Easy to forget steps when juggling multiple projects
  • Demoralizing: Drains team energy on administrative work instead of meaningful progress

Teams using manual updates spend 15-20% of their week on administrative coordination instead of actual work. Automation recaptures this lost productivity.

Setting Up Your Monday.com Boards

The webhook integration requires two Monday.com boards:

  1. Active Projects Board: Where your team manages current work with columns for Task, Owner, Status, and Due Date
  2. Completed Projects Board: A separate board for historical record-keeping when tasks are finished

This separation keeps your active workspace clean while preserving all completed work data. At the 2:15 mark in the video, you'll see how to create these boards with the right column structure.

Creating the Make.com Webhook

The magic starts in Make.com (formerly Integromat):

  1. Create a new scenario and select "Build from scratch"
  2. Search for and add the "Webhook" module
  3. Choose "Custom webhook" and name it descriptively (like "Monday Task Status Webhook")

Make.com generates a unique URL that acts like a phone number Monday.com will call with status updates. This URL is the critical connection point between the two platforms.

Connecting Monday.com to the Webhook

With your Make.com webhook URL copied:

  1. In Monday.com, open your Active Projects board
  2. Click "Automations" and search for "webhook"
  3. Select "When status changes to something, send a webhook"
  4. Paste your Make.com webhook URL and connect
  5. Specify which status change should trigger the webhook (like moving to "Done")

At this point, Monday.com is ready to send data - but Make.com hasn't received anything yet. You need to trigger the first update manually to establish the connection.

Testing the Integration

To activate the connection:

  1. Change a task status in Monday.com (like from "Working" to "Done")
  2. Return to Make.com - you should see the webhook captured the data
  3. Inspect the payload to see all available fields (item ID, board info, column values)

This test confirms Monday.com is successfully sending data to Make.com whenever the specified status change occurs. Now you can build powerful automations using this real-time data.

Expanding the Workflow

With the basic connection working, you can now add actions in Make.com that trigger when status changes occur:

  • Slack Notifications: Alert relevant team members when tasks complete
  • Board Updates: Automatically move completed tasks to your archive board
  • CRM Updates: Sync task completion status to customer records
  • Approval Workflows: Kick off review processes when tasks hit certain statuses
