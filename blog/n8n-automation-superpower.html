The Server Crash Solution That Runs Itself

Every 4-5 days, my UniFi OS server would lock up - requiring manual service restarts that often happened when I wasn't available. This meant network downtime until I could physically access the server. The solution? An n8n workflow that automatically detects and fixes the issue.

Here's how it works: Uptime Robot monitors the server and triggers a webhook to n8n when downtime is detected. The workflow then:

Key automation: The entire process runs autonomously using SSH with a restricted service account that only has permission to restart UniFi services - no full server access required. Each step is logged to Google Sheets for audit history.

  1. Connects via SSH using key authentication
  2. Stops the UniFi OS services
  3. Waits 1 minute
  4. Restarts the services
  5. Verifies successful restart
  6. Logs all actions to Google Sheets

This simple automation eliminated 3-5 hours monthly of manual server troubleshooting and reduced network downtime by 80%. The entire workflow took just 30 minutes to build.

AI Hiring Assistant That Filters 50+ Resumes

Hiring for my support team meant wading through 50+ resumes for each position - a 10+ hour monthly time sink. My n8n-powered hiring assistant now handles the first round automatically:

Time saved: Reduced resume review time from 10 hours to just 2 hours per hiring round by automatically filtering out unqualified candidates and scoring the rest.

The workflow:

  1. Imports applicants from Google Sheets
  2. Immediately filters out international candidates (we don't hire internationally)
  3. Sends remaining resumes to Claude AI with a custom scoring rubric evaluating:
    • Customer service capability (30% weight)
    • Technical knowledge (25% weight)
    • Experience relevance (25% weight)
    • Availability (20% weight)
  4. Generates a summarized Google Doc for each candidate scoring 81+
  5. Optionally sends interview scheduling emails

This 2-hour automation project now saves 8+ hours per hiring cycle while actually improving candidate quality by applying consistent evaluation criteria.

Personalized Tech News Delivered Automatically

Staying current with cybersecurity, networking, VoIP, and AI news required checking dozens of RSS feeds daily. My n8n solution now delivers only the most relevant articles directly to Discord:

Efficiency gain: Replaced 5+ hours monthly of manual news scanning with automated delivery of only the 10-15 most relevant articles weekly.

The two-part workflow runs every 6 hours:

  1. Parent workflow:
    • Pulls RSS feed URLs from Google Sheets (easy to add/remove sources)
    • Sends each feed to a child workflow for processing
  2. Child workflow:
    • Analyzes each article with Anthropic's Claude
    • Scores relevance (1-5) for my key interest areas
    • For scores 4-5, sends to Discord with:
      • Article category
      • Title and link
      • 3-5 bullet point summary

This complex automation took about 4 hours to build but now delivers better results than my manual scanning ever did.

Getting Started With n8n

Ready to try n8n? Here's what you need to know about setup and learning:

Quick start: Most users can have n8n running via Docker in under 15 minutes, with basic workflows built in the first hour using template examples.

Installation options:
