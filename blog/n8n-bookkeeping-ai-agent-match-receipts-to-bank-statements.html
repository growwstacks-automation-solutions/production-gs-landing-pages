Step-by-Step: OCR Processing Workflow

Step 1: File Type Detection

The workflow first checks whether each receipt is an image (JPG/PNG) or PDF. Images follow a direct processing path while PDFs undergo page splitting for multi-page receipts. This ensures consistent handling regardless of how vendors issue documents.

Step 2: Google OCR Extraction

Google's Document AI parses each file into structured text while preserving spatial relationships. This captures not just amounts and dates, but also identifies which numbers represent subtotals, taxes, or tips based on their position relative to labels.

Step 3: Gemini Data Structuring

The raw OCR output feeds into Gemini 2.5 Flash which extracts 15+ standardized fields including vendor, date, payment method (last 4 digits), line items, and jurisdictional tax breakdowns. The model handles variations like "GST" vs "HST" or "VAT" automatically.

In summary: 1) Detect file type → 2) Extract text with Google OCR → 3) Structure data with Gemini → 4) Output to Google Sheets → 5) Move to Processed folder

The Smart Matching Algorithm Explained

The secret sauce lies in the 130-point scoring system that matches receipts to bank transactions. While most solutions only match amounts exactly, this workflow accounts for real-world banking behaviors:

  • Base 100 points for amount matches (within $0.01 tolerance)
  • +20 points when card last-4 digits match
  • +10 points for vendor name correlation
  • Date proximity scaling (max 5-day window)

This handles common scenarios like Costco purchases that post 3 days later with a different merchant descriptor ("COSTCO WHSE #1234" vs receipt's "Costco Vancouver"). At 12:30 in the video, you'll see how the algorithm correctly matches transactions despite these discrepancies.

Multi-Currency & Tax Calculation Handling

For businesses with international expenses, the workflow automatically converts foreign amounts using real-time exchange rates. It stores both original and converted values while calculating accurate tax allocations based on jurisdictional rules.
