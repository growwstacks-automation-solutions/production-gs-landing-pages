Sticky Notes for Workflow Documentation

Documentation is the first casualty in fast-moving automation projects. You build a complex workflow, it works perfectly, and six months later you can't remember why you made certain design decisions. n8n's sticky notes solve this by letting you embed documentation directly into your workflow canvas.

These digital notes serve multiple purposes: explaining complex logic sections, noting why certain nodes are configured a specific way, or simply dividing a large workflow into logical sections. At 1:15 in the video, you'll see how color-coding different note types can create a visual documentation system.

Best practice: Use one sticky note per workflow section, with a brief description of what that group of nodes accomplishes. Green for input nodes, yellow for processing, red for outputs or alerts.

Canvas Navigation Tools

As workflows grow, navigating them becomes challenging. n8n provides several tools to manage this complexity. The zoom-to-fit button (shown at 2:30 in the video) instantly scales your entire workflow to fit the visible canvas - perfect when you've built something sprawling that extends beyond view.

Zoom in/out controls let you focus on specific areas when working on detailed node configurations. The execution button (top-left) runs the entire workflow, while the stop button halts any in-progress executions. These might seem basic, but they're essential for testing and debugging complex automations.

The Tidy Up Feature

Ever built a workflow that became a tangled mess of nodes and connections? The tidy up button (demonstrated at 3:45) is n8n's secret weapon for visual organization. With one click, it automatically rearranges your nodes into a clean, logical layout while maintaining all connections.

This doesn't affect functionality - it's purely about readability. The feature is especially valuable when collaborating, as it creates a standardized visual structure that anyone can follow. For maximum impact, use tidy up after completing each major workflow section.

Pro tip: Combine tidy up with sticky notes to create self-documenting workflows that are easy to maintain and modify months later.

Workflow Execution History

n8n maintains a complete history of every workflow run, visible through the execution panel (shown at 4:20). Each entry shows the timestamp, duration, and whether the run succeeded or failed. Clicking an execution reveals detailed input/output data for every node.

This historical record is invaluable for troubleshooting. When something breaks, you can compare recent failed executions with older successful ones to identify what changed. The data persists until you manually clear it, creating an audit trail for critical business automations.

Workflow Settings Menu

The three-dot menu (demonstrated at 5:00) contains essential workflow management tools. The edit description field lets you document the workflow's overall purpose - crucial for team environments where multiple people might need to understand or modify the automation.

Duplicate creates a copy for experimentation without risking the original. The import/export options let you share workflows between instances or team members. Most importantly, the settings option (covered next) contains critical configuration for error handling and scheduling.

Error Workflow Setup

At 6:15 in the video, we see the most important setting many users overlook - the error workflow. This allows you to designate a separate workflow that runs whenever the main one fails. Typically, this would send an email alert, create a ticket, or notify a chat channel.

Without an error workflow, failed automations might go unnoticed for days or weeks. The video shows how to configure this safety net - a 5-minute setup that can save hours of troubleshooting down the line. Every production workflow should have one.

Timezone Configuration

The final critical setting is timezone (shown at 7:30). While n8n defaults to UTC, setting your local timezone ensures scheduled workflows run at the intended local times. This is especially important for time-sensitive automations like daily reports or marketing emails.

The video demonstrates searching for and selecting your timezone - a simple change that prevents confusing time conversions. Remember to check this setting when duplicating workflows between different geographic locations.

Remember: n8n doesn't auto-save. The video emphasizes frequent manual saves to prevent losing work to browser crashes or power outages.

You should save your n8n workflows frequently, as there's no auto-save feature. A good practice is to save after every significant change or every 5-10 minutes of work.
