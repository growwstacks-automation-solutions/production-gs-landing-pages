The End of Custom GPT Limitations

Custom GPTs served businesses well in early , but they've become increasingly limited as workflow automation needs have grown more complex. The fundamental problem? Custom GPTs exist in isolation from your actual business processes and automation workflows.

n8n's Chat Hub solves this by integrating workflow execution directly into chat conversations. Where custom GPTs require complex API connections to trigger external workflows, the Chat Hub provides native integration with your existing n8n automations.

Key advantage: The Chat Hub eliminates the need to maintain separate systems for AI conversations and workflow execution. You can now have a continuous conversation-action loop within a single interface.

Multi-Model Chat Interface

The Chat Hub isn't limited to a single AI provider. It supports conversations with multiple models including OpenAI, Anthropic, Google Gemini, AWS Bedrock, open-source models via OpenRouter, and local models through Ollama.

This multi-model approach means you can:

  • Compare responses from different providers side-by-side
  • Use specialized models for different tasks without switching interfaces
  • Fall back to alternative models if your primary provider experiences issues

The interface includes familiar chat features like web search, voice control, and image attachments, making it a complete replacement for standalone chat interfaces.

Seamless Workflow Integration

The real power of the Chat Hub comes from its ability to execute your existing n8n workflows directly from chat conversations. Any workflow can become a "chat agent" that you invoke with natural language commands.

For example, you could:

  • Ask your web scraper agent to collect information from a website
  • Have your meeting notes processor extract action items from a transcript
  • Trigger your lead qualification workflow with a simple chat command

Implementation note: Workflows appear in the Chat Hub when they include a Chat Trigger node with the "Make available in any chat" option enabled. This requires no additional API configuration compared to custom GPT solutions.

Simple Setup Process

Converting an existing workflow for Chat Hub use requires just three steps:

Step 1: Add a Chat Trigger

Drag a Chat Trigger node into your workflow and enable the "Make available in any chat" toggle. This makes the workflow appear as an agent option in your Chat Hub.

Step 2: Name and Describe Your Agent

Give your agent a clear, descriptive name and explanation of its function. This helps users know when to invoke it during conversations.

Step 3: Publish Your Workflow

Ensure your workflow is published and functioning correctly. The Chat Hub will only show published workflows with active Chat Triggers.

Pro tip: For workflows with existing triggers (like webhooks), you can add a Chat Trigger alongside them to maintain all current functionality while adding Chat Hub access.

Troubleshooting Tips

If your workflows aren't appearing in the Chat Hub, try these solutions:

  • Update n8n: Ensure you're running the latest version that supports Chat Hub functionality
  • Recreate Chat Triggers: For imported workflows, remove and re-add the Chat Trigger node
  • Check publishing status: Only published workflows with active Chat Triggers appear
  • Rebuild if needed: As a last resort, recreate the workflow in a new canvas

Remember that workflow executions triggered through the Chat Hub appear in your normal execution history with a special identifier, making it easy to track chat-initiated runs separately from other triggers.
