What Makes ChatHub Different?

Most automation platforms force you to choose between power and accessibility. You either hunt through dozens of disconnected workflows (losing precious time) or simplify your setup (sacrificing functionality). n8n's ChatHub changes this equation by giving you centralized access to all your specialized AI agents through a single conversational interface.

Imagine having ChatGPT's simplicity combined with the power of your entire automation stack. That's ChatHub - where you can call any custom agent naturally within one continuous conversation. No more copying/pasting between tools or rebuilding context with each switch.

Key differentiator: ChatHub maintains full conversation history when switching between agents, unlike Telegram bots or published workflows that require starting fresh with each interaction.

The Problems ChatHub Solves

Before ChatHub, using multiple AI agents in n8n created three major friction points:

1. Workflow hunting: Finding specific agents among hundreds of poorly named workflows ("My Workflow 146") wasted countless hours.

2. Context loss: External solutions like Telegram required separate conversations per agent, forcing teams to constantly re-explain situations when switching tools.

3. Limited access: Published workflows created security concerns, while self-hosted bots couldn't scale across team members efficiently.

ChatHub addresses all three by providing:

  • Unified agent directory
  • Persistent conversation history
  • Controlled access management

1-Minute Setup Guide

Enabling ChatHub for your workflows couldn't be simpler. As shown at the 4:30 mark in the video:

  1. Create a new workflow or open an existing one
  2. Add a Chat Trigger node - this is required for ChatHub integration
  3. Enable "Available in n8n chat" in the trigger settings
  4. Name your agent clearly (avoid generic names)
  5. Add a description for team members (optional but recommended)
  6. Enable file uploads if your agent needs document handling
  7. Save and activate the workflow

After refreshing ChatHub, your new agent appears immediately in the interface. That's all it takes to make any workflow accessible through conversational AI.

Two Types of AI Agents

ChatHub supports two distinct agent categories, each serving different purposes:

1. Tool-based Agents (shown in the lead management demo):

  • Connected to specific workflows and external tools
  • Can access databases, CRMs, email systems, etc.
  • Perform concrete actions like updating records or sending messages

2. Chat-based Agents (like the Decision Maker example):

  • Specialized for specific cognitive tasks
  • No direct workflow connections

Team Access Considerations

While ChatHub simplifies agent access, permissions still follow n8n's standard sharing model. Key points from the 6:45 timestamp:

Cloud Version:

  • Share workflows directly with team members
  • Or create shared folders for group access

Self-Hosted:

  • Requires Starter or Pro plan for sharing features
  • Same folder-based permission system applies

    • Pro Tip: Create a "Team Agents" folder with commonly used workflows to simplify onboarding and ensure everyone accesses the same tools.
