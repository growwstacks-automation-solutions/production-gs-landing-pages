n8n Core Concepts

Understanding these fundamentals will accelerate your n8n learning curve:

Nodes: The Building Blocks

Each step in your workflow is a node - small pre-built pieces of logic that do one job. Common node types:

  • Triggers - Start workflows (schedule, webhook, email arrival)
  • Actions - Perform tasks (send email, update spreadsheet, call API)
  • Logic - Control flow (if/else, merge, loops)
  • Data - Transform information (filter, sort, aggregate)

Data Flow

Nodes pass JSON data between them. Click any node to inspect exactly what it received and sent - this visibility makes troubleshooting straightforward.

Error Handling

n8n shows exactly where failures occur with detailed error messages. You can configure workflows to retry or follow alternative paths when errors happen.

Building Your First Workflow

Let's create a practical automation that fetches weather data and sends personalized alerts (full build starts at 14:30 in the video):

Step 1: Setup Trigger

Add a Schedule Trigger node to run daily at 7 AM. This replaces manual testing while developing.

Step 2: Get Location Data

Use an HTTP Request node to call ip-api.com - returns coordinates based on server location.

Step 3: Fetch Weather

Another HTTP Request calls Open-Meteo's API with the coordinates, returning temperature forecasts.

Step 4: Add Logic

An If node checks if temperature exceeds your threshold (e.g. 25°C/77°F) and branches workflow accordingly.

Step 5: Send Email

Connect Gmail node to send appropriate clothing advice based on the temperature check.
