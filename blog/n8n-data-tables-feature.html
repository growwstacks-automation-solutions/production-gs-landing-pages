What Are n8n Data Tables?

Every automation workflow eventually needs to store data - whether it's tracking processed records, maintaining configuration settings, or building simple CRM functionality. Traditionally, this required connecting to external databases like PostgreSQL or Airtable, adding complexity and potential performance bottlenecks.

n8n's new Data Tables feature changes this paradigm by bringing simplified database functionality directly into your workflows. Think of it as a lightweight, low-code database designed specifically for automation use cases.

Key benefit: Data Tables complete operations in just 19 milliseconds compared to 8900 milliseconds with external databases - making them ideal for workflows where speed matters.

Step-by-Step: Creating Your First Data Table

Getting started with Data Tables requires just a few clicks in your n8n instance. Here's how to create your first table:

Step 1: Access the Tables Interface

After updating to the latest n8n version (1.10+), you'll see a new "Tables" tab in the top navigation. Click this to access the table management interface.

Step 2: Create a New Table

Click "New Table" and give it a descriptive name (e.g., "Companies"). n8n will automatically create an "id" column as the primary key.

Step 3: Add Custom Columns

Click "Add Column" to define your data structure. Currently supported types:

  • String: For text data (company names, descriptions)
  • Number: For numerical values (prices, quantities)
  • Boolean: True/false flags (active/inactive status)
  • Time: Date/time values (created dates)

Pro Tip: Column order matters for the default display. Arrange your most frequently accessed columns first for better workflow readability.

Inserting and Managing Records

With your table structure defined, adding records is straightforward using n8n's "Table" node. Here's how it works:

Inserting New Records

Add a "Table" node to your workflow and configure it to:

  1. Select your table from the dropdown
  2. Choose "Insert" as the operation
  3. Map your input data to the appropriate columns

The node automatically maps fields when property names match column names exactly. For mismatched names, you'll need to manually configure the mapping.

Updating Existing Records

The same "Table" node can perform updates by:

  1. Selecting "Update" as the operation
  2. Specifying which record to update (typically by ID)
  3. Mapping the new values

Important: Data Tables currently don't support complex queries or joins. For advanced filtering, consider processing data in your workflow before/after table operations.

Using Data Tables as Workflow Triggers

One of Data Tables' most powerful features is the ability to trigger workflows on data changes. This enables real-time automation scenarios without polling external databases.

To set up a trigger:

  1. Add a "Table" node at the start of your workflow
  2. Select "Trigger" as the operation type
  3. Choose which events should trigger the workflow (insert, update, delete)

When triggered, the workflow receives the changed record as input, allowing you to:

  • Send notifications for new records
  • Process updates in real-time
  • Maintain audit logs of changes

Current Limitations and Workarounds

While powerful, Data Tables are still in active development. Here are the current limitations to consider:

Missing Features

  • No JSON column type yet (coming soon)
  • Limited query capabilities (basic filtering only)
  • No joins between tables

Workarounds

  • For complex data, use string columns with JSON.stringify()
  • Implement filtering in your workflow logic
  • Combine with external databases for advanced query needs

Despite these limitations, Data Tables already solve many common workflow data storage needs with unmatched speed and simplicity.
