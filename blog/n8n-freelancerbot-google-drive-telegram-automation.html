Google Drive API Setup

Most freelancers struggle with client file management - documents arrive in random emails, messages, and cloud folders. The first step to automation is centralizing everything in Google Drive with proper API access.

Create a new project in Google Cloud Console (2:15 in the video) and enable the Google Drive API. This grants your n8n workflow permission to watch folders and process files automatically.

Critical security note: Your client ID can be public, but the client secret must remain confidential. Never expose this in your n8n workflow or share it publicly.

Step-by-Step Credential Setup:

  1. Navigate to Google Cloud Console > APIs & Services
  2. Search for and enable "Google Drive API"
  3. Create OAuth credentials (Client ID and Client Secret)
  4. Configure authorized redirect URIs for your n8n instance

Project Folder Configuration

Without a dedicated folder structure, automated file processing becomes chaotic. The FreelancerBot needs a clean starting point to monitor for new client submissions.

Create a new folder in Google Drive specifically for incoming projects (7:45 in the video). This becomes your automation's "inbox" where clients or team members can drop files that need processing.

Pro tip: Name your folder clearly (like "FreelancerBot_Projects") and share edit access only with necessary team members to maintain security.

Telegram Notification Setup

Real-time alerts transform automation from background process to active assistant. Telegram's bot API provides instant notifications when new projects arrive.

Create your bot through Telegram's BotFather (10:30 in the video) to get your access token. You'll need separate tokens for development and production environments.

Key Components:

  • Bot token from @BotFather
  • Your personal chat ID (found via /my_id command)
  • Message templates for different notification types

Automation Trigger Configuration

The magic happens when Google Drive changes trigger your n8n workflow. Configure the "Watch for file created" trigger to monitor your designated folder.

At 14:20 in the video, you'll see how to connect the trigger to subsequent processing steps. This creates a complete pipeline from file upload to notification.

Schedule optimization: Balance frequency with resource usage - checking every 15 minutes catches projects quickly without overloading your system.

Going Live with Your Bot

Transitioning from testing to production requires careful validation. The final steps (16:50 in the video) cover live deployment considerations.

Document your configuration thoroughly - including API credentials, folder IDs, and Telegram settings. This ensures smooth recovery if you need to rebuild your environment.

Deployment Checklist:

  1. Verify all credentials in production environment
  2. Test with sample files in your Google Drive folder
  3. Confirm Telegram notifications arrive as expected
