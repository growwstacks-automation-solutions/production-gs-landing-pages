How to Configure OpenAI and RAG for Your n8n FreelancerBot

Freelancers waste hours manually matching client requirements with their skills. This n8n workflow automates the entire process using OpenAI's API and RAG (Retrieval-Augmented Generation) to intelligently match projects with your capabilities and generate personalized responses.

OpenAI API Configuration

The foundation of your FreelancerBot's intelligence comes from OpenAI's API. Many freelancers assume AI integration requires complex coding, but n8n makes it accessible through simple configuration.

At 2:15 in the video, we walk through the exact steps to generate your API key. You'll need to create an OpenAI account if you don't have one, then navigate to the API section to generate a new security key.

Pro tip: Add at least $10 credit to your OpenAI account before starting. This gives you enough testing capacity to ensure your workflow functions properly before scaling up.

Step 1: Create Your API Key

Log in to your OpenAI account and go to the API keys section. Click "Create new secret key" and copy it immediately - you won't be able to see it again.

Step 2: Configure n8n OpenAI Node

In your n8n workflow, locate the OpenAI node. Paste your API key in the credentials section and select "Apply to all OpenAI nodes in workflow" to automatically configure any other AI nodes.

Skills Customization

Generic responses turn off potential clients. The power of this workflow comes from its ability to match specific client needs with your exact skillset.

At 4:30 in the tutorial, we modify the skills array that drives the compatibility filter. This is where you'll replace the sample skills with your actual expertise.

Important: Be specific with your skills. Instead of "web design," list "WordPress development" or "Shopify theme customization" for better matching.

Step 1: Locate the Skills Array

Find the JavaScript code block containing the "mySkills" array. This is typically in a Function node early in the workflow.

Step 2: Update with Your Skills

Replace the placeholder skills with your actual expertise, keeping the array format. Use between 5-15 key skills for optimal matching.

Superbase Database Setup

Your FreelancerBot needs a database to store client requirements and match them with your skills. Superbase provides a free tier perfect for this purpose.

The tutorial shows how to create a new Superbase project and configure the database table that will store all your client interactions and matches.

Step 1: Create Superbase Project

Sign up at superbase.com and create a new project. The free tier includes everything you need for this workflow.

Step 2: Run the SQL Schema

Navigate to the SQL editor and run the provided schema to create your freelancer_matches table. This creates the structure for storing all client interactions.

Step 3: Get Your API Keys

Under Project Settings → API, copy your Superbase URL and service role key. You'll need these to configure the n8n Superbase nodes.

RAG Integration

Retrieval-Augmented Generation combines the power of OpenAI's language model with your specific knowledge base for more accurate responses.

At 8:45 in the video, we configure the RAG components that allow your FreelancerBot to reference past client interactions when generating responses.

Why this matters: RAG prevents generic responses by incorporating context from your database, making each reply uniquely tailored to the client's needs.

Step 1: Configure Retrieval Node

Set up the node that queries your Superbase database for relevant past interactions before generating a response.

Step 2: Adjust Context Window

Modify how much historical data gets included in each response. More context creates better responses but may increase API costs.

Testing Your Workflow

Before connecting real clients, you need to verify your FreelancerBot responds accurately to different scenarios.

The tutorial demonstrates testing with sample client requests to ensure the skills matching and response generation work as intended.

Step 1: Manual Trigger Test

Use n8n's manual trigger to simulate client requests covering various skill matches and mismatches.

Step 2: Review Database Entries

Check your Superbase table to confirm interactions are being logged correctly with proper skill matching.

Step 3: Refine Responses

Tweak the prompt engineering in your OpenAI nodes based on test results to improve response quality.
