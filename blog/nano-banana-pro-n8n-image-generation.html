Getting Started with Nano Banana Pro API

Accessing Nano Banana Pro requires setting up API credentials through Google AI Studio (since Google developed the model). The process is straightforward:

Step 1: Create API Key

Visit aistudio.google.com and navigate to the API key section. Create a new key and associate it with a Google Cloud project (you'll get $100 in free credits when you first sign up).

Step 2: Configure Billing

While free credits are available, you'll need to set up billing for ongoing usage. Google's pricing is transparent and based on API calls/resolution levels.

Step 3: Access n8n

You can use n8n Cloud (starting at €24/month) or self-host using solutions like Hostinger (as low as $5/month). The workflow works identically in both environments.

Pro tip: Always go directly to Google AI Studio rather than third-party providers to avoid markup fees and get the most current API features.

Basic Image Generation Workflow

The simplest way to integrate Nano Banana Pro into n8n uses the Gemini node (though we'll improve this in the next section). Here's the basic setup:

Step 1: Add Gemini Node

Search for "Google Gemini" in n8n and select the "Generate an Image" operation. Create new credentials using your API key.

Step 2: Configure Model

Since Nano Banana Pro isn't listed by default, select "By ID" and enter: models/Gemini3ProImagePreview

Step 3: Enter Prompt

Describe your desired image. For example: "A woman holding a coffee cup with a coffee shop in the background. She's wearing a dress and a hat."

Limitation: This basic approach doesn't allow control over resolution or aspect ratio - which we'll address next.

Advanced Control Over Image Generation

For professional results, we need finer control over image generation parameters. This requires using the HTTP Request node instead of the Gemini node.

Step 1: HTTP Request Setup

Configure a POST request to Google's API endpoint with:

  • Authentication header: x-goog-api-key with your API key
  • Content-Type: application/json

Step 2: JSON Body Configuration

The body includes:

  • Prompt text
  • Generation config specifying resolution (1K, 2K, or 4K)
  • Aspect ratio controls

Step 3: Convert Base64 to File

The API returns images in base64 format. Add a "Convert to File" node to make the image usable in your workflow.

4K capability: This method unlocks Nano Banana Pro's full 4K resolution potential - perfect for print materials and high-definition displays.

Editing Existing Images

Nano Banana Pro's most powerful feature is its ability to edit existing images while maintaining consistency. Here's how the workflow differs from generation:

Step 1: Upload Original Image

Use an n8n form to accept file uploads. The workflow will convert these to base64 for the API.

Step 2: Include Edit Instructions

Along with the image, provide text instructions like: "Replace the coffee cup with ice cream and the background with the ocean and beach."

Step 3: Process Through API

The JSON body now includes both the original image (base64) and the edit instructions. The API handles the rest.

Consistency breakthrough: Edited images maintain the original character's facial features, clothing details, and accessories - perfect for product placement tests.

Combining Multiple Images

For complex compositions, Nano Banana Pro can combine elements from up to 12 reference images into a single generation.

Step 1: Upload Reference Images

The workflow accepts multiple file uploads, converting each to base64 separately to prevent overwrites.

Step 2: Specify Composition

The prompt describes how to combine elements: "Take the woman from image one and dress her in the hat from image two."

Step 3: Generate Composite

The JSON body includes all reference images and the composition instructions. The API seamlessly merges the elements.

Product placement: This feature is ideal for testing how products look when held by models or placed in different environments.
