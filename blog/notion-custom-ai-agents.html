What Are Custom AI Agents?

Notion's standard AI feature responds to one-off prompts - ask a question, get an answer. Custom AI Agents are fundamentally different. They're persistent automation workers that operate based on triggers you define, executing complex workflows without manual intervention.

During our month-long testing, we identified three game-changing capabilities:

1. Scheduled Execution: Agents run daily, weekly, or at specific times (like 12am task processing). Unlike manual AI queries, they work while you sleep.

2. Database Awareness: They understand Notion's relational data model, allowing actions like "when any task is marked complete, check if it's recurring and duplicate it for tomorrow."

3. Multi-Step Workflows: A single agent can analyze yesterday's habits, generate a progress report, notify you via Slack, and update multiple databases - all from one trigger.

How Triggers & Scheduling Work

Custom AI Agents shine because they activate automatically. During testing, we utilized four trigger types:

Database Triggers (Most Powerful): Agents spring into action when specific database events occur - new pages added, properties changed, or items removed. Our task management agent triggers when any task is marked complete.

Scheduled Triggers: Set daily, weekly, or custom schedules. Our daily reporter runs every morning at 7am, while the task cleaner activates at midnight.

Mention Triggers: Type @AgentName anywhere in Notion to manually trigger execution. Useful for on-demand reports.

External Triggers: Limited Slack, Calendar, and Email integrations exist. When fully developed, these will enable cross-platform workflows like "when sales email received, create CRM record."
