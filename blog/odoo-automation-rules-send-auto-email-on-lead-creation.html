Odoo Automation Rules Explained

Odoo's automation rules (formerly called Server Actions) let you define "if this, then that" workflows that run automatically when specified conditions are met. For lead response emails, we configure rules that:

  • Trigger when a new lead is created (from any source)
  • Filter by specific criteria if needed (e.g., only for leads from your website)
  • Execute actions like sending emails, updating fields, or creating follow-up tasks

The system handles everything in the background, requiring zero manual intervention once configured. At 2:15 in the video tutorial, you can see how the automation rule instantly sends an email when we create a test lead.

Step-by-Step Configuration

Step 1: Access Automation Rules

Navigate to Settings → Technical → Automation → Automation Rules. Click "Create" to make a new rule.

Step 2: Configure Basic Settings

Name your rule descriptively (e.g., "Send Welcome Email on Lead Creation"). Select "Lead/Opportunity" as the model and set the trigger to "On Creation".

Step 3: Add Email Action

Under Actions, click "Add" and choose "Send Email". Select or create your email template (we'll cover templates next). Set the recipient to the lead's contact email (${object.email_from}).

Step 4: Save and Activate

Save your rule and toggle it to active. Odoo will now automatically execute this workflow for all new leads.

Pro tip: Set a small delay (1-5 minutes) if you want to give sales reps time to review leads before the auto-response goes out.

Email Template Best Practices

Your automated emails should feel personal and helpful - not robotic. Follow these guidelines when creating templates:

  • Use dynamic fields like ${object.name} to personalize each message
  • Include a clear CTA (e.g., "Schedule a call" or "Reply to this email")
  • Keep it concise - 3-5 sentences maximum for the initial response
  • Brand consistently with your other communications

At 3:45 in the video, you can see a well-structured template that includes the lead's name, reference number, and next steps - all populated automatically from the lead record.

Testing Your Automation

Before going live, thoroughly test your automation rule:

  1. Create test leads from different sources (website, email, manual entry)
  2. Verify emails are sending to the correct addresses
  3. Check that all dynamic fields populate correctly
  4. Review spam scores using tools like Mail-Tester.com

Odoo's "Run Manually" option lets you test rules without creating actual leads. The video demonstrates this at 4:30 by triggering the rule manually to preview the email.

Advanced Scenarios

Once you've mastered basic lead response, consider these powerful extensions:

  • Conditional responses: Send different emails based on lead source or other criteria
  • Multi-step sequences: Follow up with additional emails after set intervals
  • SMS integration: Send text messages for urgent inquiries
  • Task creation: Automatically assign leads to sales reps with follow-up tasks

These advanced workflows require no coding - just additional automation rules configured in the same interface.

Performance Considerations

While automation rules are lightweight, follow these guidelines for optimal performance:

  • Limit active rules: Archive unused rules to reduce system load
  • Batch operations: For large lead volumes, process in batches overnight
  • Monitor logs: Check the Automation → Rules menu for failed executions
  • Use filters: Narrow rule applicability to prevent unnecessary processing

Most businesses find that even dozens of automation rules have negligible impact on Odoo's performance when configured properly.
