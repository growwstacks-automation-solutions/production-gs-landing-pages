What is OpenClaw? (Formerly Cloudbot)

OpenClaw represents the next evolution of private AI assistants - a rebranded version of the previously known Cloudbot framework. At its core, it's a self-hosted solution that bridges the gap between local large language models (LLMs) and your everyday messaging platforms.

The concept is revolutionary yet simple: instead of relying on cloud APIs that store your conversations, OpenClaw runs entirely on your hardware. Whether you're using Telegram, WhatsApp, or Discord, your messages stay private while still benefiting from AI assistance.

Key differentiator: Unlike cloud-based alternatives, OpenClaw doesn't require internet access to function once configured. All processing happens locally through your Ollama models, making it ideal for sensitive communications or offline use cases.

System Requirements and Setup

Before diving into OpenClaw installation, you'll need to prepare your system. The requirements vary significantly based on your chosen Ollama model size and expected usage patterns.

For testing purposes with smaller models like GLM-4.7B or GPTQ-8B, a system with 16GB RAM will suffice. However, for production use with larger models (especially the recommended GPTQ-120B), you'll need serious hardware:

  • GPU: Nvidia RTX 6000 with 48GB VRAM (or equivalent)
  • RAM: 64GB+ for comfortable operation
  • OS: Ubuntu/Linux preferred (Windows/MacOS also supported)
  • Software: Node.js v18+ and Python 3.10+

Installing OpenClaw Step-by-Step

The installation process has been streamlined in the OpenClaw rebrand, with an improved onboarding script that handles most configuration automatically. Here's the condensed version:

Step 1: Install Node.js

OpenClaw requires Node.js as its runtime environment. Install it using nvm for easiest version management:

 curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.5/install.sh | bash source ~/.bashrc nvm install --lts

Step 2: Install OpenClaw Globally

With Node.js in place, install OpenClaw as a global package:

 npm install -g openclaw

Step 3: Run Onboarding Script

The new onboarding wizard handles core configuration:

 openclaw onboard

During setup (shown at 7:15 in the video), you'll be prompted to:

  • Choose quick start or advanced configuration
  • Select your model provider (skip cloud options for local setup)
  • Configure your preferred messaging platform (Telegram in our example)

Pro Tip: The onboarding process installs OpenClaw as a systemd service (daemon), ensuring your gateway runs continuously in the background. This is crucial for maintaining active connections to your messaging apps.

Configuring Ollama Model Integration

The magic of OpenClaw lies in its ability to connect with local Ollama models. Unlike cloud APIs, this integration keeps all processing on your hardware.

After completing the basic OpenClaw setup, you'll need to:

1. Install Ollama

If you haven't already, download and install Ollama from their official website. The Linux installation is straightforward:

 curl -fsSL https://ollama.ai/install.sh | sh

2. Download Your Preferred Model

Pull your chosen model (GLM-4.7B flash used in demo):

 ollama pull glm-4.7b-flash

3. Configure OpenClaw for Ollama

Modify the OpenClaw config file to point to your local Ollama instance:

 {   "provider": "ollama",   "endpoint": "http://localhost:11434",   "model": "glm-4.7b-flash" }

At 9:45 in the video, you can see this configuration being applied. The key is ensuring the endpoint matches your Ollama server address and the model name exactly matches what you've downloaded.

Telegram Bot Setup and Pairing

Connecting OpenClaw to Telegram transforms your messaging app into a private AI assistant. Here's how to create and pair your bot:

1. Create Your Telegram Bot

Search for @BotFather in Telegram and follow these steps:

  • Send /newbot command
  • Choose a name for your bot (e.g., "MyPrivateAI")
  • Receive and securely store your API token

2. Configure OpenClaw with Telegram Token

During onboarding or in config files, add your Telegram token:

 "telegram": {   "enabled": true,   "token": "YOUR_TELEGRAM_BOT_TOKEN" }

3. Pair Your Account

After configuration (shown at 12:30 in video):

  1. Message your bot in Telegram (any message)
  2. Copy the pairing code from bot response
  3. Run in terminal: openclaw pairing approve telegram YOUR_CODE

This two-factor pairing ensures only authorized users can access your AI assistant, adding a crucial security layer.

Production Considerations

While OpenClaw shows tremendous promise, the current version (as demonstrated at 16:20 in the video) has some stability challenges you should prepare for:

Crash Frequency: The gateway service crashes regularly, especially under heavy load. Implement these mitigations:

  • Create a cron job to restart the service every 30 minutes
  • Configure systemd to auto-restart on failure
  • Regularly backup your session data

Performance Monitoring: Keep an eye on:

  • GPU memory usage (nvidia-smi for Nvidia cards)
  • Response latency (adjust model size as needed)
  • Conversation history storage

The OpenClaw team is actively improving stability, but until then, these workarounds will help maintain reliable operation.

Security and Privacy Concerns

While local AI offers unparalleled privacy, OpenClaw's current implementation requires careful security consideration:

Critical warning: You're granting an AI permission to execute commands and send messages on your behalf. Never connect sensitive accounts or information without proper safeguards.

Recommended security practices:

  • Run OpenClaw in a sandboxed environment or VM
  • Never integrate banking, medical, or other sensitive accounts
  • Regularly rotate your Telegram bot token
  • Monitor all outbound messages for unexpected behavior

At 18:00 in the video, the presenter emphasizes these concerns - the vision of private AI is compelling, but requires responsible implementation.
