Step 1: Private Networking with Tailscale

The first critical security layer involves removing your OpenClaw instance from the public internet entirely. Tailscale creates a private network between your devices using WireGuard VPN technology, meaning your bot will only be accessible to your authorized devices.

Installation is straightforward:

  1. Run the Tailscale install command on your VPS
  2. Authenticate using your preferred identity provider (Google, GitHub, etc.)
  3. Install Tailscale on your local machine and connect to the same account
  4. Verify connectivity between devices

Pro tip: Avoid installing Tailscale from app stores - download it directly from their website for the most reliable performance. The mobile app is particularly useful for accessing your OpenClaw instance from anywhere while maintaining security.

Step 2: Configuring the UFW Firewall

Even with Tailscale's private network, we need to ensure no public access to our OpenClaw instance. The Uncomplicated Firewall (UFW) provides simple yet powerful protection:

 sudo ufw allow 22/tcp sudo ufw allow 41641/udp  # Tailscale port sudo ufw enable

This configuration:

  • Allows SSH access (port 22) for administration
  • Permits Tailscale traffic (port 41641)
  • Blocks all other incoming connections by default

After enabling UFW, test that you can still access your VPS via SSH and that Tailscale connectivity remains intact. These rules create an essential second layer of defense should any part of your private network configuration fail.

Step 3: Implementing Token Authentication

Token authentication ensures that even if someone discovers your OpenClaw endpoint, they can't use it without the proper credentials. The process involves:

  1. Finding your bot's port using Docker commands
  2. Generating a secure access token
  3. Configuring your access URL with the token parameter

Your final access URL will look like:

 https://[your-tailscale-ip]:[port]?token=[your-secure-token]

Security note: Treat this token like a password - anyone with it can access your OpenClaw instance. Consider rotating tokens periodically and never share them in public forums or unsecured channels.

Step 4: Adding Fail2Ban SSH Protection

Fail2Ban monitors your SSH logs for repeated failed login attempts, automatically banning suspicious IP addresses. This prevents brute force attacks against your VPS:

 sudo apt install fail2ban sudo systemctl enable fail2ban sudo systemctl start fail2ban

Once installed, verify it's running:

 sudo systemctl status fail2ban

You should see "active (running)" in the output. Fail2Ban works silently in the background, adding another layer of protection against automated attacks that target Linux servers.

Step 5: Telegram Integration & Allow Lists

The final security layer restricts which users can interact with your OpenClaw instance through Telegram:

  1. Create a new bot via BotFather
  2. Configure OpenClaw with your bot token
  3. Pair your Telegram account with the bot
  4. Add your user ID to the allow list

This ensures that even if someone discovers your bot's Telegram handle, they can't use it unless explicitly added to your allow list. For businesses, you can maintain tight control over who has access to your AI agent while still benefiting from convenient messaging integration.

Flexibility: While this guide demonstrates Telegram integration, the same principles apply to other platforms like WhatsApp or Discord. The key is implementing proper access controls regardless of your chosen messaging platform.
