Automating Baptism Dates (No More Spreadsheets)

Every baptism Sunday creates a new administrative headache - manually recording dates for each participant in spreadsheets or individual profiles. This process often gets delayed, leading to incomplete records and frustrated staff trying to reconstruct events weeks later.

The breakthrough comes when you connect three Planning Center features: recurring check-in events, smart lists, and profile field automations. At 2:15 in the video tutorial, you'll see how setting up a weekly "Baptism" check-in event creates the perfect trigger point.

Key workflow: When someone checks into the baptism event (either self-service or admin-assisted), a list detects their attendance and triggers an automation that writes the date to their profile's custom baptism field - with no manual data entry required.

Critical implementation details:

  • Create a dedicated custom date field under People → Settings → Custom Fields
  • Set the automation to use "day before automation runs" to ensure correct dating for Sunday events
  • Configure the list to include both regulars and guests in case of misclassification

Class Completion Tracking That Works While You Sleep

Growth track classes and discipleship programs present another manual tracking nightmare. Staff often waste hours cross-referencing sign-up sheets with attendance records to determine who actually completed requirements.

The automation solution leverages the same check-in → list → automation pattern but with yes/no fields instead of dates. At 7:45 in the video, the presenter demonstrates how checking into a "Growth Track Class 101" event can automatically mark completion in the participant's profile.

Pro tip: Use recurring check-in events rather than one-time registrations for ongoing classes. This creates a permanent automation pathway that doesn't require reconfiguration each semester.

Implementation checklist:

  1. Create yes/no custom fields for each class milestone
  2. Set up check-in events matching your class schedule
  3. Build lists watching each class check-in event
  4. Configure automations to update profile fields when list conditions are met

Pro Tips for Maintaining Reliable Automations

Like any system, Planning Center automations require occasional maintenance. The tutorial's presenter shares several best practices at 16:50 that can prevent issues:

  • Naming conventions: Use clear names like "Automation - Baptism Date" rather than generic labels
  • Organization: Create an "Automations" category to group related workflows
  • Monitoring: Check automation history monthly via the three-dot menu
  • Alerts: Ensure notification emails route to multiple staff members

Common pitfalls to avoid:

  1. Changing check-in event names without updating corresponding lists
  2. Editing custom field types after automations are established
  3. Overlooking the "day before" vs "day of" automation timing setting
