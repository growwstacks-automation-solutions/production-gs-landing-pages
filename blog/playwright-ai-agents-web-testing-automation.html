What Are Playwright AI Agents?

Manual web testing is one of the most time-consuming parts of software development. Teams typically spend 30-40% of their development cycle writing and maintaining tests, only to have them break with every UI change. Playwright's new AI agents (introduced in version 1.56) solve this by automating the entire testing workflow.

The system consists of three specialized agents that work together:

90% reduction in test maintenance: The Healer agent automatically updates broken locators when your UI changes, eliminating the most common reason tests fail during development.

  • Planner Agent: Analyzes your web application and creates comprehensive test plans with scenarios and success criteria
  • Generator Agent: Converts test plans into executable Playwright test scripts
  • Healer Agent: Monitors test runs and automatically fixes broken locators when UI changes occur

Step-by-Step Installation

Getting started with Playwright Agents requires just three commands, but each serves a specific purpose in setting up the complete AI testing environment.

Step 1: Update VS Code

Ensure you're running the latest version of VS Code (Help → Check for Updates). The agents rely on recent VS Code APIs for seamless integration.

Step 2: Install/Update Playwright

For new projects: npm init playwright@latest
For existing projects: npm install -D @playwright/test@latest

Step 3: Install Agents

Run npx playwright init --agents --loop=vs-code
This creates a .github/chat-modes folder containing the three agents.

Configuration tip: If tools appear highlighted in yellow in VS Code, select "Playwright Test" from the MCP server options to ensure all agents are properly recognized.

Planner Agent: Creating Test Plans

The Planner agent acts as your testing strategist. When given a target page (like a login form), it analyzes the UI and proposes comprehensive test scenarios covering all critical paths.

In our example at 4:32 in the video, we asked the Planner to "create three test cases for the login page at alex-usa-base.com". It automatically generated:

  1. Successful login with valid credentials
  2. Login failure with invalid credentials
  3. Login attempt with empty password field

Each test case included detailed instructions, test steps, and success criteria - essentially creating a complete test specification without any manual input.

Generator Agent: Writing Test Code

Where the Planner creates the blueprint, the Generator builds the house. Switching to the Generator agent (or opening a new chat with it) automatically converts the test plan into executable Playwright code.

The Generator:

  • Writes all necessary page interactions (navigations, clicks, inputs)
  • Adds appropriate waits and timeouts
  • Includes assertions to verify expected outcomes

At 7:15 in the tutorial, we see the Generator produce complete test code for all three login scenarios, including verifying error messages for invalid attempts and checking UI state changes after successful login.

Healer Agent: Fixing Broken Tests

The real magic happens when tests break. Traditional testing frameworks leave you debugging failed locators, but Playwright's Healer agent automatically:

  1. Detects which element locators failed
  2. Analyzes the current page structure
  3. Updates the test files with new, working locators

At 12:40 in the video, we see a powerful example where the initial test missed verifying a password validation popup. By simply describing the issue to the Healer ("We're not capturing the visible pop-up"), it:

Zero manual fixes: The Healer added the missing assertion expect(validationMessage).toContain('Please fill out this field') and updated the test to properly verify the popup behavior.

Running Agents in Headless Mode

While the interactive chat interface is great for development, you'll want to run agents automatically in CI/CD pipelines. This requires just one configuration change:

Add "--headless" to your .vscode/mcp.json file. This allows the agents to:

  • Run tests without opening a browser UI
  • Still perform all healing and maintenance functions
  • Integrate seamlessly with GitHub Actions, CircleCI, etc.

At 16:20 in the tutorial, we demonstrate headless execution where all three test cases run automatically with the agents monitoring and adjusting as needed.
