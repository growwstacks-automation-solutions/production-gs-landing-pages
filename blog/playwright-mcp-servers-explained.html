Playwright MCP Servers Explained: Which One Should You Use for Automation and Testing?

Most developers using Playwright don't realize there are two different MCP servers available - one for general automation and one specialized for testing. Understanding the difference can dramatically improve your workflow efficiency and test coverage.

What Are Playwright MCP Servers?

Playwright's Model-Context-Protocol (MCP) servers are powerful tools that enable AI-assisted browser automation and testing. Many developers struggle with repetitive testing tasks and maintaining test coverage, especially as applications grow in complexity. The MCP servers provide a solution by automating these processes.

There are two distinct MCP servers available: the general Playwright MCP server for broad automation tasks, and the Playwright Test MCP server specifically designed for testing workflows. Each serves different purposes and comes with its own set of capabilities.

Key insight: The general MCP server can perform any browser automation task, while the test server includes specialized tools for generating, running, and debugging tests - making it significantly more efficient for testing purposes.

The General Automation Server

The Playwright MCP server is the official tool for general browser automation tasks. It provides capabilities like navigating websites, filling forms, clicking elements, taking screenshots, and more. Developers often use it for scraping data, automating repetitive tasks, or interacting with web applications.

This server includes tools for basic browser interactions: browser click, close, console messages, drag, evaluate, file upload, fill form, handle dialogue, hover, navigate back, press key, resize, select, snapshots, tabs, take screenshots, type, and wait for elements. These tools give the LLM access to perform nearly any action you could do manually in a browser.

The Specialized Testing Server

The Playwright Test MCP server comes bundled with three specialized agents when installed: a test generator, test healer, and test planner. These agents work together to create comprehensive test plans, generate tests, and debug failing tests automatically.

This server includes all the general automation tools plus testing-specific capabilities like verifying element visibility, checking text content, running test suites, debugging failing tests, generating locators, and tracing network requests. The healer agent can autonomously fix broken tests without human intervention.

Important note: As of , the test server is only available for TypeScript projects. For other languages, you'll need to use the general automation server.

Key Differences Between the Two Servers

The primary difference lies in their specialized tools and installation methods. The general MCP server can be installed globally across all projects, while the test server is installed per-project and comes with pre-configured agents.

The test server includes additional capabilities specifically for testing workflows: network request monitoring, test debugging tools, automatic test generation, and specialized verification methods. In one real-world example shown in the video, 70-75% of tests (75 out of 107) were created automatically by the test server's agents.

When to Use Each Server

Use the general Playwright MCP server when you need broad browser automation capabilities across multiple languages and projects. It's ideal for tasks like form filling, data extraction, or general web interaction automation.

The Playwright Test MCP server should be your choice when working on TypeScript projects requiring comprehensive test coverage. The specialized agents can dramatically reduce the time spent creating and maintaining tests. In the demo, the healer agent autonomously fixed nine failing tests without any human input.

Installation Comparison

The general MCP server can be installed globally or per-project, while the test server is specifically designed for per-project installation. The test server gets added when running the Playwright test initialization command (npx playwright init).

Both servers can be customized by modifying their configuration files, but changes to the test server may be overwritten if you re-run the initialization command. The general server's configuration persists unless explicitly uninstalled.
