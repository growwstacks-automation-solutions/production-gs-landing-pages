What Is Power Automate UI Automation?

Manual data entry and web scraping are among the most time-consuming digital tasks in any business. Employees waste hours copying information from websites into spreadsheets or entering data into legacy systems that lack APIs. Power Automate's UI automation capabilities solve this by recording and replaying your interactions with applications and browsers.

Unlike traditional automation that requires APIs or coding skills, UI automation works by capturing mouse movements, clicks, and keyboard inputs. This makes it ideal for automating tasks with older systems or websites where no API exists. The desktop flow recorder creates automation scripts visually, without requiring any programming knowledge.

Key benefit: Power Automate UI automation can reduce manual data entry time by 80-90% for repetitive tasks. A process that takes an employee 4 hours daily can be automated to run in minutes with perfect accuracy every time.

Setup Requirements for Desktop Flows

Before creating your first UI automation, you'll need to set up the Power Automate Desktop application. This free tool from Microsoft provides the recording and playback engine for desktop automation. The setup process takes about 10 minutes:

  1. Download Power Automate Desktop from Microsoft's official site
  2. Run the installer with administrator privileges
  3. Sign in with your Microsoft account (work or school account for business use)
  4. Configure the runtime environment for attended or unattended automation

For web scraping automation specifically, you'll want to use a modern browser like Microsoft Edge or Google Chrome. The tutorial in the video demonstrates scraping course data from a website using Edge, but the same principles apply to any browser.

Creating Your First UI Automation Flow

Starting a new UI automation in Power Automate is straightforward. From the Power Automate portal, create a new "Instant cloud flow" and add the "Run built Power Automate desktop" action. This connects your cloud flow to the desktop automation engine.

At the 2:15 mark in the tutorial video, you can see how to:

  1. Name your flow (e.g., "Web Scraping Demo")
  2. Select the "Desktop flows" connector
  3. Choose between attended or unattended execution
  4. Map input/output variables between cloud and desktop

Pro tip: Always test your UI automation flows in attended mode first to verify they work correctly before setting them up to run unattended. This helps catch any issues with element selectors or timing.

Web Scraping Example: Extracting Course Data

The tutorial demonstrates a practical web scraping example - extracting course titles and URLs from an education website. Here's how the automation works:

  1. The flow launches Edge browser and navigates to the target URL
  2. It identifies and extracts specific page elements (course links)
  3. The scraped data is stored in variables for processing
  4. Finally, the results are saved to an Excel file

What makes this powerful is the element recognition system. As shown at 5:30 in the video, Power Automate can learn to identify specific page elements by their properties (like href attributes for links). This makes the automation resilient to minor page layout changes.

Saving Results to Excel Automatically

The final step in any data extraction workflow is saving your results. Power Automate Desktop provides several options:

  • Save directly to Excel (as demonstrated in the tutorial)
  • Write to a CSV or text file
  • Insert into a database
  • Send to a cloud service like SharePoint or OneDrive

In our example flow, the scraped course data gets saved to an Excel file at a specified path. The automation handles opening Excel, creating the worksheet if needed, formatting the columns, and entering all the data - exactly as a human would, but faster and error-free.

Running Flows in the Cloud vs Desktop

Power Automate offers two execution environments for UI automation:

  1. Attended mode: Runs on your local machine with you present. Ideal for testing and personal productivity automation.
  2. Unattended mode: Runs on a designated machine without human interaction. Required for scheduled automation and server-side execution.

The tutorial focuses on attended execution, but at 8:45 it briefly covers the additional setup needed for unattended flows, including machine registration and licensing requirements. Unattended automation is where the real time savings happen, as flows can run overnight or on a schedule.

Licensing Options and Costs

Power Automate Desktop is free for personal use with basic attended automation. For business scenarios, you'll need:

  • Power Automate per user plan: $15/user/month - includes attended automation
  • Unattended RPA add-on: $150/month per bot - enables scheduled/server-side execution
  • Windows Pro/Enterprise: Required for some advanced features

Compared to enterprise RPA tools that can cost thousands per month, Power Automate provides an affordable entry point into UI automation. The tight integration with Microsoft 365 makes it particularly valuable for businesses already using SharePoint, Teams, or Dynamics.
