What is Flu AI Framework?

Flu is an open-source framework for building AI agents created by the team behind Astro. Unlike traditional agent frameworks that require extensive setup and configuration, Flu provides a complete harness for creating fully-featured agents with just a few lines of TypeScript code.

The framework was originally developed to run AI workflows within Astro's GitHub repository, but its potential became clear when an engineer from Amplitude tested it. The team realized they had created something that could benefit any developer building AI agents.

Key insight: Flu takes what Claude Code does as a harness and makes it 100% programmable, with built-in support for skills, tools, and sandboxes out of the box.

Key Features That Make Flu Powerful

Flu stands out from other AI frameworks through several innovative features designed to simplify agent development while maintaining flexibility:

1. Built-in Sandbox Support

Every Flu agent runs in an isolated sandbox by default, providing security without additional configuration. The sandbox uses Vercel's just-bash implementation for lightweight operation.

2. Skill and Tool Integration

Flu makes it easy to create reusable skills and tools that can be shared across agents. The framework handles the plumbing so developers can focus on business logic.

3. Flexible Deployment Options

Agents can be deployed as Node.js HTTP servers or Cloudflare Workers with durable object persistence, allowing you to choose the right infrastructure for your use case.

Innovation: Flu's architecture enables both interactive agents (requiring human input) and fully autonomous workflows within the same framework.

Flu's Sandbox Innovation

One of Flu's most impressive technical achievements is its sandbox implementation. Unlike traditional approaches that spin up full containers, Flu uses Vercel's just-bash to run commands in memory.

This provides several key benefits:

  • Cost efficiency: No need to boot complete Linux environments for simple operations
  • Performance: Commands execute immediately without container startup delays
  • Scalability: Thousands of agents can run simultaneously at minimal cost

For tasks requiring more isolation, developers can opt into full container sandboxes while keeping the majority of operations lightweight.

Getting Started With Flu

Setting up Flu requires just a few simple steps (shown at 3:15 in the video):

Step 1: Install the Runtime and CLI

The Flu runtime powers your agents while the CLI handles compilation and serving:

 bun add @flu-runtime/node bun add @flu-runtime/cli -g

Step 2: Configure Your Project

Initialize your project with a target (Node or Cloudflare):

 flu init --target node

Step 3: Set Up API Keys

Configure your preferred AI provider (Anthropic, OpenAI, etc.) in the generated config file.

Pro tip: The CLI creates a Flu config file that tells the system how to package your project for your chosen target.

Building Your First Agent

Creating a basic Flu agent is remarkably simple. At minimum, you need to:

  1. Define your model (Claude, GPT, etc.)
  2. Add system instructions
  3. Export the agent configuration

Here's the complete code for a simple "hello world" agent (shown at 4:30 in the video):

 import { agent } from '@flu-runtime/node'; export default agent({   model: 'claude-3',   instructions: 'You are a helpful assistant' });

To run the agent, use:

 flu connect hello-world --instance-id test123

The instance ID allows you to manage multiple agent instances when deploying to Cloudflare.

Creating a Content Workflow

Flu really shines when building more complex workflows. For example, creating a YouTube title generator (shown at 6:45 in the video):

  1. Create a skill directory with Python scoring logic
  2. Import the skill into your workflow
  3. Define instructions for studying scripts and generating titles
  4. Run the workflow with your content as input

The workflow automatically:

  • Processes your script
  • Generates 10 potential titles
  • Scores each using your custom Python skill
  • Returns ranked results

Security note: For local file access, explicitly opt-in using the local runtime module rather than granting blanket permissions.

Deployment Options

Flu offers flexible deployment to match your infrastructure needs:

Node.js HTTP Server

Simple to set up and ideal for development:

 flu build --target node node server.mjs

Cloudflare Worker

Production-ready with durable object persistence:

 flu build --target cloudflare wrangler publish

For HTTP-triggered workflows, simply add the root middleware:

 import { root } from '@flu-runtime/node/middleware';

This enables both REST API and WebSocket interfaces for your agents.

How Flu Compares to Other Frameworks

Flu takes a different approach than frameworks like Mastra or Vercel AI SDK:

Feature Flu Traditional Frameworks
Architecture Harness-first Granular components
Setup Complexity Minimal Extensive
Sandboxing Built-in (just-bash) Requires configuration
Deployment Node or Cloudflare Varies by framework

The key difference is that Flu assumes you'll need sessions, memory, sandboxing and tool loading - building these in from the start rather than making you assemble them piece by piece.

  • Node.js HTTP server

Flu is primarily designed for TypeScript/JavaScript development, leveraging Bun runtime for fast execution. The framework itself is written in TypeScript.

Through its sandbox and tool system, you can integrate code written in other languages like Python for specific tasks. The framework uses Valibot for parameter validation in custom tools.

  • Primary language: TypeScript
  • Bun runtime for performance
  • Python integration via tools

By default Flu runs agents in isolated bash sandboxes with no file system access. This provides strong security out of the box.

For workflows requiring local file access, developers must explicitly opt-in using the local runtime module. Alternatively, sensitive operations can be encapsulated as custom tools with validated parameters.

  • Default sandbox isolation
  • Explicit opt-in for local access
  • Tool encapsulation pattern
