What is Real-Time Speech-to-Speech?

Traditional voice AI systems process conversations in discrete chunks - you speak, wait for processing, then hear a response. This creates unnatural pauses that break conversation flow. Real-time speech-to-speech (STS) technology changes everything.

With STS, your speech gets processed continuously as you talk. The AI generates responses with almost zero latency, creating fluid, human-like interactions. At the 2:15 mark in the video, you can hear how naturally the conversation flows - no awkward pauses, just instant responses.

Key advantage: STS reduces response latency from 1-2 seconds (typical for chunked processing) to under 200 milliseconds - faster than human reaction time in conversation.

Docker Configuration

The Docker setup handles all the complex orchestration behind the scenes. After cloning the repository (shown at 0:45 in the video), you'll see the project includes configurations for multiple providers - we're focusing on the real-time STS setup today.

Key containers in the stack:

Pro tip: The docker-compose.yml file specifies all networking requirements. Containers communicate over internal networks on ports 5001 (core), 10000 (SIP), and 6030 (STS WebSocket).

Getting Your OpenAI API Key

At 1:20 in the tutorial, you'll see how to obtain and configure your OpenAI API key. This is essential for connecting to their speech models.

Step-by-step:

  1. Log into platform.openai.com
  2. Navigate to API keys in your dashboard
  3. Click "Create new secret key"
  4. Name it (like "AVR demo")
  5. Copy the key immediately (you won't see it again)
  6. Add it to your environment variables file

The key gets added to your .env file as OPENAI_API_KEY=[your_key_here]. This allows secure connection to OpenAI's API without exposing credentials in your code.

Telephony Configuration

The demo uses Asterisk to handle telephony connections. At 3:10, you'll see the SIP configuration details:

  • Default SIP username: 1000
  • Default password: 1000
  • Port: 10000
  • Transport: TCP

For MacOS users, we recommend configuring your softphone to connect to localhost port 10000. Crucial settings:

Must disable: "Update EP address" flag in your SIP client settings. This prevents registration issues with the local Asterisk instance.

You can customize these credentials in the sip.conf and extensions.conf files if needed for your production deployment.
