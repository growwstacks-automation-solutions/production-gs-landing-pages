How Voice AI Solves the Manual Order Entry Challenge

Retell Voice AI agents act as your 24/7 order takers that never call in sick. When integrated with Make.com, they transform the order workflow:

  1. Customer calls your business number
  2. Voice AI greets them and collects order details conversationally
  3. The agent structures the order into a JSON format
  4. Make.com receives the order via webhook
  5. Automated workflow posts it directly to your order dashboard

This eliminates the error-prone middle step where staff manually transcribe calls. The system achieves 98% order accuracy compared to 80-85% with human entry.

Connecting Retell Voice AI to Make.com

The integration requires just three core components:

1. A Retell Voice AI agent configured for order taking

2. A Make.com scenario with a custom webhook

3. A JSON schema defining your order structure

The magic happens in the webhook connection. When a customer places an order through the Voice AI, it sends the structured data to Make.com's waiting webhook. Make then processes this data and posts it to your order management system.

Step-by-Step Webhook Setup

Setting up the webhook connection takes under 10 minutes:

Step 1: Create a new Make.com scenario

Start with a blank scenario and add the "Webhooks" module. Select "Custom Webhook" since we're connecting to Retell's API.

Step 2: Configure the webhook

Make.com will generate a unique URL for your webhook. Copy this - you'll need it to configure your Retell Voice AI agent.

Step 3: Test the connection

Use Make.com's testing tool to verify the webhook is listening. At this point, it won't process data yet - we're just confirming the connection works.

Pro Tip: Create separate webhooks for different order types (delivery vs pickup) to simplify your workflow logic.

Creating the Order JSON Schema

The JSON schema defines how order data is structured between systems. For our restaurant example, we need three core fields:

 {   "order_details": "string",   "phone_number": "string",   "customer_name": "string" }

This schema acts as a contract between your Voice AI agent and Make.com workflow. Additional fields like order_time, special_instructions, or delivery_address can be added as needed.

The schema ensures consistent data formatting regardless of how the customer phrases their order during the call.

Testing and Validating the Workflow

Before going live, thoroughly test the integration:

  1. Place test orders through your Voice AI agent
  2. Verify Make.com receives the webhook data
  3. Check that orders appear correctly in your dashboard
  4. Test edge cases (special instructions, modifications)

Common testing scenarios include:

  • Orders with multiple modifications ("no onions, extra sauce")
  • Large group orders with many items
  • Callers interrupting or changing their order mid-call

Implementation Tip: Run 20-30 test orders before going live. Capture any errors and refine your Voice AI prompts or JSON schema as needed.
