Setting Up Docker Model Runner

The setup process begins with Docker Desktop, which provides a unified interface for managing containers. At 2:45 in the video, you'll see the crucial step of enabling host-side TCP support in Docker's AI settings - this allows external applications like n8n to communicate with your local models.

Step 1: Install Docker Desktop

Download and install Docker Desktop for your operating system. The free version includes all features needed for model running.

Step 2: Configure AI Settings

Navigate to Settings → AI and enable both Docker Model Runner and host-side TCP support. Note the port number (default is 11434) as you'll need this for n8n configuration.

Pro tip: If you're running n8n in Docker (recommended), use the internal Docker network address model-runner.docker.internal instead of localhost for optimal performance.

Connecting n8n to Local Models

The magic happens when you connect n8n's OpenAI node to your local Docker Model Runner. At 4:30 in the video, watch how we:

Step 1: Create New OpenAI Credential

Use any placeholder text for the API key (it won't be used) and set the base URL to your Docker Model Runner endpoint.

Step 2: Select Local Models

Once connected, n8n will display all available Docker models in the model dropdown, just like it shows OpenAI options.

Important: The exact base URL differs depending on whether n8n runs inside Docker (model-runner.docker.internal) or outside (localhost:11434).

Performance Optimization Tips

While Docker Model Runner works on modest hardware, these optimizations can dramatically improve response times:

1. GPU Acceleration

Enable CUDA support in Docker settings to utilize your NVIDIA GPU. This can speed up inference by 5-10x compared to CPU-only operation.

2. Model Quantization

Some models offer quantized versions (4-bit or 8-bit) that maintain quality while using less VRAM.

3. Batch Processing

Design n8n workflows to process multiple items in batches rather than single requests.

Benchmark: On an RTX 3060 GPU, LLaMA 3-7B produces ~25 tokens/second - fast enough for most business automation needs.
