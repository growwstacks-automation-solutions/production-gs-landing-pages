How the Automation Works

The solution uses Make.com's AI agent capability to bridge Claude and Dropbox. Instead of manual copying, you simply type commands like "save this to Dropbox as African elephant research" directly in Claude. The AI agent interprets your natural language and triggers the Make scenario to handle the file creation.

Key components of the system:

  1. Claude AI - Where conversations and content generation happen
  2. Make.com AI Agent - Interprets natural language commands
  3. Make Scenario - Handles the Dropbox file creation logic
  4. Dropbox Connection - Where files are automatically saved

The magic happens through carefully crafted system prompts that teach the AI agent how to format data for the Make scenario, and a simple but powerful Make workflow that receives the data and saves it to your specified Dropbox folder.

Setting Up the Make Scenario

The core of the automation is an on-demand Make scenario that accepts two parameters from Claude: the file name and document content. Here's how to configure it:

Step 1: Create a New Scenario

In Make.com, create a new scenario with an "On Demand" trigger. This ensures it only runs when called by the AI agent, not on a schedule.

Step 2: Define Input Parameters

Set up two required text inputs:

  • file_name - The name you want for the saved file
  • document_data - The actual content to save (multi-line)

Step 3: Add Dropbox Module

Connect your Dropbox account and configure the "Upload a File" action. Map the inputs from Claude to the Dropbox fields:

  • File path: Your chosen Dropbox folder + file_name + .md extension
  • File content: The document_data input

Pro Tip: Add a date prefix to filenames by using Make's formatDate function combined with your file_name input for better organization.

Configuring the AI Agent

The AI agent requires careful system prompt configuration to properly format data for the Make scenario. After initial challenges, we developed an effective prompt structure:

Key Components of the System Prompt

  • Tool Definition: Clearly describes the Dropbox save functionality
  • Parameter Requirements: Specifies exactly what data Make expects
  • Output Formatting: Ensures Claude structures responses correctly
  • Error Handling: Provides guidance for when things go wrong

The prompt essentially teaches Claude how to "speak Make's language" when sending data to the scenario. This includes:

  • Converting conversation content to clean Markdown
  • Generating appropriate filenames when none are provided
  • Structuring the JSON payload correctly

Note: The complete system prompt is available in the GitHub repository linked in the video description at 7:32.

Connecting Claude to Make

With the scenario built and AI agent configured, the final step is establishing the connection between Claude and Make:

Step 1: Get Your MCP Server URL

From your Make API settings, copy your MCP server URL. This is the endpoint Claude will use to communicate with Make.

Step 2: Create Custom Connector in Claude

In Claude's settings under Connectors, add a new custom connector with:

  • Name: Descriptive name for the functionality (e.g., "Save to Dropbox")
  • URL: Your MCP server URL with the specific scenario ID appended

Step 3: Test the Connection

Start a new chat in Claude, enable the connector, and test with a simple save command. You should see:

  1. Claude preparing the data to send
  2. Make executing the scenario
  3. Confirmation message with the saved file details

Testing the Workflow

Once everything is connected, testing is straightforward:

Basic Test Commands

  • "Save this to Dropbox as [filename]"
  • "Save our conversation about [topic]"
  • "Store this research in Dropbox" (let Claude generate the filename)

Expected outcomes:

  • File appears in your specified Dropbox folder
  • Content is properly formatted as Markdown
  • Filename matches your request or is sensibly generated

Troubleshooting Tips

If you encounter issues:

  1. Verify all scenario inputs match the AI agent's expected parameters
  2. Check the system prompt for proper formatting instructions
  3. Ensure your MCP server URL includes the correct scenario ID
  4. Test with simple commands before complex ones

Remember: You can choose "Allow once" for each save operation until you're confident in the automation's reliability.
