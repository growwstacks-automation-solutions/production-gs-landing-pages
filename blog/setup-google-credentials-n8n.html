Why Google Credentials Matter

Google's API security requires proper authentication for any application accessing its services. Without correctly configured credentials, your n8n workflows will fail when trying to interact with Google Sheets, Drive or Gmail.

Many users waste hours troubleshooting connection issues because they skipped steps in the credential setup process. Following this guide ensures you configure everything properly the first time.

Key benefit: Proper credentials let you automate Google workflows like updating spreadsheets, managing Drive files, and sending emails - all from within n8n.

Step 1: Create a Google Cloud Project

Every API connection starts with a Google Cloud project. This organizes your credentials and tracks API usage.

  1. Go to console.cloud.google.com
  2. Click the projects dropdown and select "New Project"
  3. Name your project (e.g., "n8n Credentials")
  4. Click "Create"

Make sure your new project appears selected in the top navigation bar before proceeding.

Step 2: Enable Required APIs

Google requires explicit enablement for each API service. We'll activate Sheets, Drive and Gmail APIs:

  1. Search "Google Sheets API" in the search bar
  2. Click "Enable" on the API page
  3. Repeat for "Gmail API" and "Google Drive API"

Pro tip: Enable all three APIs even if you only need one now. This prevents future interruptions when expanding your automations.

Step 3: Configure OAuth Consent Screen

The OAuth consent screen determines what users see when authorizing your app:

  1. Navigate to "OAuth consent screen"
  2. Select "External" user type
  3. Enter "n8n" as the app name
  4. Add your email as both support and developer contact
  5. Save and continue through all steps

This creates the authorization framework for your credentials.

Step 4: Create API Credentials

Now we'll generate the actual credentials n8n will use:

  1. Go to "Credentials" in the left menu
  2. Click "Create Credentials" > "OAuth client ID"
  3. Select "Web application" as the type
  4. Name it "n8n Client"
  5. Add authorized redirect URI from n8n (shown in next step)
  6. Click "Create"

You'll now see your Client ID and Client Secret - keep these handy.

Step 5: Connect Credentials to n8n

With credentials created, we'll link them to n8n:

  1. In n8n, go to "Credentials"
  2. Create new credential for "Google Sheets OAuth2 API"
  3. Paste your Client ID and Client Secret
  4. Click "Save" then authenticate with Google
  5. Repeat for Google Drive and Gmail

Important: Google Sheets requires Google Drive credentials to work properly since Sheets files are stored in Drive.

Once authenticated, your credentials will appear in n8n ready for use in workflows.
