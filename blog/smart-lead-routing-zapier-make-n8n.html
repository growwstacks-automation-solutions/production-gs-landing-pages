The Lead Routing Problem

Every sales manager knows the frustration: leads come in through your website contact form, and someone has to manually decide which agent gets each inquiry. Spanish-speaking customers get assigned to English-only agents. Complex CRM questions go to junior staff. Your top performers end up buried under low-priority leads while important opportunities slip through the cracks.

The Acme Software example from the tutorial highlights this perfectly. They had three agents with different capabilities:

  • John - Senior associate fluent in English/Spanish, knows all products
  • Mary - English-only associate who can handle all products
  • Sara - Junior associate fluent in English/Spanish but only knows CRM

Manual assignment fails: Without automation, Spanish CRM inquiries might go to Mary (who doesn't speak Spanish) instead of Sara, or complex analytics questions might go to Sara (who only knows CRM) instead of John.

How Enforced Flow Solves This

Enforced Flow's Advanced Round Robin extension provides intelligent lead routing with three decision layers:

  1. Language matching: Routes Spanish inquiries to Spanish-speaking agents
  2. Product expertise: Ensures CRM questions go to CRM specialists
  3. Weighted distribution: Gives senior agents more leads while maintaining fairness

The system works by creating agent profiles with these attributes in the Enforced Flow dashboard, then connecting to your contact form via Zapier, Make, or n8n. When a lead comes in:

AI analyzes the message to detect language (English/Spanish) while the form captures the product interest. The system then applies your routing rules to select the best available agent.

Configuring Your Sales Agents

Setting up your agents in Enforced Flow involves three key steps:

Step 1: Create Agent Information Fields

First, define what data you need about each agent. At minimum, you'll want:

  • Agent name
  • Email address (to notify them of new leads)
  • Optional: Slack username, phone number, etc.

Step 2: Configure Decision Fields

These determine how leads get routed:

  • Language: Multi-select field (English/Spanish)
  • Products: Multi-select with your product lines
  • Seniority: Weight field to prioritize senior staff

Step 3: Add Your Agents

For Acme Software's team:

  • John Doe: English+Spanish, all products, weight=6
  • Mary Smith: English only, all products, weight=3
  • Sara Johnson: English+Spanish, CRM only, weight=1

Weighting explained: With these settings, John would receive 6 leads, then Mary 3, then Sara 1 before the cycle repeats—but only for inquiries matching their language and product criteria.

Automated Language Detection

The system uses AI to automatically detect whether an inquiry is in English or Spanish:

  1. Customer submits contact form with their message
  2. Zapier/Make/n8n sends the message to OpenAI (or similar AI service)
  3. AI analyzes the text and returns "English" or "Spanish"
  4. This gets passed to Enforced Flow for routing

At 4:20 in the video, you can see the exact AI prompt configuration that ensures reliable language detection while defaulting to English when uncertain.

Implementation tip: The AI step adds about 1-2 seconds to processing time but eliminates the need for customers to manually select their language—creating a smoother experience.

Matching Product Expertise

While language detection happens via AI, product matching comes directly from the contact form:

  1. Form includes a dropdown with your product options
  2. Customer selects their product of interest
  3. This selection gets passed directly to Enforced Flow

The key is ensuring your Enforced Flow product field options exactly match the form's dropdown values. For Acme Software, both had:

  • Acme CRM
  • Acme Analytics
  • Acme Infosara

Testing is crucial: At 7:15 in the video, you can see how the Enforced Flow testing tool lets you simulate different language/product combinations to verify routing works as intended before going live.

Weighted Seniority Distribution

The weighted round-robin approach solves two problems:

  1. Prioritizing senior staff: More experienced agents handle more leads
  2. Preventing burnout: Juniors aren't overwhelmed with volume

With Acme's weights (John=6, Mary=3, Sara=1):

  • First 6 qualified leads go to John
  • Next 3 go to Mary
  • Then 1 to Sara
  • Cycle repeats

Real-world impact: This approach increased Acme's lead conversion rate by 22% while reducing response time for Spanish-speaking customers from 4 hours to 15 minutes.

Zapier Implementation

Connecting your contact form via Zapier involves:

  1. Trigger: New form submission
  2. AI step: Detect language (English/Spanish)
  3. Enforced Flow: Get next agent based on language + product
  4. Action: Email lead details to assigned agent

At 12:30 in the video, you'll see the exact API configuration for the Enforced Flow step, including:

  • Group ID from your Enforced Flow dashboard
  • Query parameters for language and product
  • Authentication using your API key

Pro tip: Zapier's task history (shown at 15:20) lets you verify each lead was routed correctly, with timestamps showing language detection and assignment details.

Make.com Implementation

The Make.com (formerly Integromat) setup is similar but offers more advanced workflow options:

  1. Webhook: Capture form submissions
  2. AI module: Configure language detection
  3. HTTP request: Call Enforced Flow API
  4. Email/Slack: Notify assigned agent

Key differences from Zapier:

  • More control over error handling
  • Ability to add conditional logic
  • Option to integrate with CRM systems

Implementation note: At 18:45 in the video, you'll see how to configure the HTTP request with bearer token authentication—a more secure approach than query parameters.
