Smartling Zapier Triggers and Actions Explained

Smartling's Zapier integration provides two-way automation between your translation platform and other business tools:

Triggers (Events that start workflows)

  • Translation job completed
  • New issue created
  • File uploaded
  • Project status changed

Actions (Tasks Smartling can perform)

  • Create translation jobs
  • Upload files for translation
  • Push new content strings
  • Update project metadata

This combination means you can both receive Smartling updates in other apps and send content to Smartling automatically - creating complete workflow loops without manual intervention.

Implementation Steps for Non-Technical Users

Setting up the example notification workflow requires no coding - just follow these steps:

Step 1: Connect Your Accounts

Link Smartling and Gmail (or your preferred notification tool) to Zapier using OAuth. This takes about 30 seconds per service.

Step 2: Choose Your Trigger

Select "Issue Created" from Smartling's trigger options. Zapier will automatically detect your available Smartling projects.

Step 3: Set Up the Action

Choose "Send Email" in Gmail (or "Send Channel Message" for Slack). Map the Smartling fields to your message template as shown at 2:15 in the video.

Step 4: Test and Activate

Zapier will walk you through a test run. Once confirmed, publish the zap - it will run automatically moving forward.

Pro tip: Add a filter step to only notify for high-priority issues, reducing inbox clutter for your team.
