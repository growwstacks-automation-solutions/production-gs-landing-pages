The Knowledge Crisis in Global Manufacturing

Simmons Energy, a global leader in energy products with operations in 90+ countries, faced a critical knowledge management challenge. Each of their highly customized products - from transformers to storage systems - required unique documentation stored across thousands of files in multiple languages. Factory workers often spent 20+ minutes per request searching for specifications and procedures.

The problem intensified with their global workforce. Training a skilled winder took four years because critical knowledge wasn't systematically accessible. "We had workers who were using a computer mouse for the first time," recalls Anabietris, Digitalization Expert at Simmons Energy. "Yet they needed to navigate complex documentation systems daily."

Key insight: Traditional knowledge management systems fail when information is highly customized, multilingual, and accessed by non-technical users. AI chatbots must bridge the gap between complex documentation and real-world usability.

From Summit Prototype to Scalable Product

The journey began with a proof-of-concept showcased at Snowflake Summit 2024, which won the Data Innovator Award. However, the initial Streamlit implementation revealed critical adoption barriers:

  • Factory workers struggled with Snowflake login processes
  • Document processing couldn't handle multiple languages effectively
  • Source document verification required switching to SharePoint

The team rebuilt the solution with three key improvements:

  1. Mendix Frontend: Replaced Streamlit with simpler interfaces accessible via single sign-on
  2. Automated Pipelines: Implemented daily Cortex Search updates instead of manual processing
  3. Unified Endpoint: Created a single Cortex Q&A function callable from multiple systems

As Product Owner Janus Kalp explains, "Adoption skyrocketed when we stopped forcing users to adapt to our technology and instead adapted our technology to users' existing workflows."

Snowflake Cortex Chatbot Architecture

The production architecture solves several enterprise-scale challenges through Snowflake Cortex capabilities:

Core Components: Document staging → Power Document processing → Vector database → Cortex Search → LLM response generation → Mendix presentation layer

The workflow begins when documents are uploaded to a Snowflake stage. The Power Document functionality extracts and vectorizes text, storing it in a database. A daily Cortex Search update maintains current vectors without manual intervention.

When a worker asks a question through Mendix:

  1. The interface sends the query (with chat history context) via REST API
  2. Snowflake executes Cortex Search to find relevant document passages
  3. An LLM synthesizes the answer while citing exact document locations
  4. Results return to Mendix with source document links

This architecture supports multiple factories and languages through centralized processing with localized interfaces.

Solving Multilingual Challenges

With factories in Brazil, Germany, China and beyond, language support was non-negotiable. The solution handles:

  • Document Processing: Automatically ingests English, Portuguese, Spanish and German documents without translation
  • Query Understanding: Interprets questions in any supported language, regardless of document language
  • Response Generation: Answers in the user's language while maintaining technical accuracy

During the demo (timestamp 18:45), the chatbot seamlessly switched from answering English questions about oven conditions to Portuguese queries about bushing assembly - all while citing the correct source documents.

Critical Feature: Workers can click through to view exact document passages, building trust in the AI's accuracy across languages.

Overcoming User Adoption Barriers

The technical solution was only half the battle. Many factory workers had spent 30+ years working the same way and distrusted new technologies. The team implemented several change management strategies:

  • Persona Development: Created "Tina" (Transformer Intelligence and New Assistant) as a friendly, approachable identity
  • Verification Culture: Encouraged workers to check source documents, positioning Tina as a helper rather than replacement
  • Digital Literacy: Provided basic computer training for workers unfamiliar with technology
  • Local Champions: Identified respected factory workers to demonstrate and advocate for the solution

The results speak for themselves: after switching to Mendix, user engagement increased 300%, proving that interface simplicity drives adoption more than technical sophistication.

Key Takeaways

The Simmons Energy case demonstrates how enterprises can transform AI prototypes into scalable solutions that deliver real business value. Three critical lessons emerge:

  1. User experience drives adoption: The switch from Streamlit to Mendix tripled usage by meeting workers where they were
  2. Trust through transparency: Showing source documents built confidence in AI-generated answers
  3. Language is no barrier: Properly implemented, LLMs can bridge knowledge gaps across global organizations

In summary: Snowflake Cortex LLMs enabled Simmons Energy to turn fragmented, multilingual documentation into an accessible knowledge base that saves 20 minutes per query while supporting workers across 90+ countries - proving AI's potential to transform enterprise operations at scale.
