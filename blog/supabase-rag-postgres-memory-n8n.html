The Problem With Traditional Database Queries

Business teams constantly need answers from company databases - which department someone works in, when they joined, or what projects they're on. But without SQL knowledge or direct database access, they're stuck waiting for IT or data teams to run queries. Even simple questions can take hours to resolve.

The breakthrough comes when we realize modern AI agents can understand natural language questions and automatically determine how to query the database. By connecting n8n to Supabase, we eliminate the SQL bottleneck while maintaining data accuracy.

Key insight: In tests, non-technical users asked 3-5x more data questions when they could use natural language instead of SQL, leading to better-informed decisions across departments.

Supabase AI Agent Setup

Connecting n8n to Supabase starts with creating a new project in Supabase Cloud (the free tier works fine for testing). After setting up your HR schema tables (department, employee, salary), you'll configure the n8n Supabase node with:

  • Supabase project URL from Settings → API
  • Service role key (not the public anon key)
  • Table permissions configured in Supabase

The magic happens when you add an AI agent node with tools pointing to your Supabase tables. Without any SQL knowledge, the AI can answer questions like "Which department does Jones work in?" by:

  1. Analyzing table relationships
  2. Determining the correct join logic
  3. Returning the accurate answer

Implementation tip: Name your tools descriptively (like "employee_department_query") to help the AI understand when to use each one.

Postgres Chat Memory Implementation

Chat interfaces feel broken when they don't remember previous conversations. Using Supabase's underlying Postgres database, we can implement persistent chat memory by:

  1. Adding a Postgres chat memory node in n8n
  2. Configuring it with your Supabase connection details
  3. Setting a unique session ID for each conversation thread

All messages are stored as JSON in a dedicated table, allowing the AI to maintain context across conversations. At 12:35 in the video, you'll see how this enables follow-up questions like "What about Kim?" after initially asking about Jones' department.

Running Complex Queries via HTTP

While Supabase's native n8n node handles basic CRUD operations, complex queries require the HTTP node. This approach lets you:

  • Execute any SQL query Supabase supports
  • Join multiple tables with precise control
  • Handle advanced filtering and sorting

The key steps are:

  1. Get your REST endpoint from Supabase settings
  2. Format queries as URL parameters
  3. Include your anon/public key for authentication

Performance note: HTTP queries are about 15% slower than native database connections but enable far more complex operations when needed.

Document Upload and Vector Store

Retrieval-Augmented Generation (RAG) brings document knowledge into your AI agent. To set this up in Supabase:

  1. Create a documents table with vector embeddings column
  2. Add a match_documents function for similarity search
  3. Configure n8n's document loader for file uploads

The admin interface (shown at 18:20 in the video) lets users upload PDFs or Word docs through a simple form. Behind the scenes:

  • Documents are split into chunks
  • Each chunk gets vector embeddings
  • Content is stored in Supabase with metadata

RAG Agent Configuration

The final piece is the RAG agent that answers questions based on uploaded documents. Configuration involves:

  1. Adding a separate AI agent node
  2. Connecting it to the Supabase vector store
  3. Setting up a new chat history table

When asked "What was the last company John worked at?", the agent:

  1. Converts the question to a vector
  2. Finds the most relevant document chunks
  3. Generates an accurate answer from the context

Accuracy tip: Use the same embedding model for queries that you used when storing documents to ensure optimal matching.
