How the Supabase-to-Sheets Sync Works

Most businesses using Supabase for their database still rely on manual exports to Google Sheets for reporting, analysis, or sharing data with non-technical teams. This creates version control issues and wastes hours each week on repetitive data transfers.

The automation works through Supabase's webhook functionality. When any change occurs in your specified database table (insert, update, or delete), Supabase sends the changed data to n8n via a webhook. n8n then processes this data and appends it to your Google Sheet automatically.

Key benefit: Your Google Sheet becomes a real-time mirror of your Supabase table without any manual intervention, saving 2-5 hours per week on average for teams regularly exporting data.

Setting Up Supabase Webhooks

The first step is configuring Supabase to send webhook notifications when your database changes. This requires creating a PostgreSQL function that triggers on table modifications.

In the Supabase SQL editor, you'll create a function called notify_new_row that executes whenever data changes in your target table. This function needs to be configured with your n8n webhook URL to send the data payload.

Important: After creating the function, you must run it manually the first time to activate the trigger. The video tutorial at 1:15 shows exactly how to execute this setup step.

Configuring the n8n Workflow

The n8n workflow serves as the bridge between Supabase and Google Sheets. When designing this automation, we need to handle three key functions: receiving webhook data, processing the payload, and writing to Google Sheets.

The workflow starts with a Webhook node that listens for Supabase notifications. You'll then add function nodes to extract the relevant data fields (ID, name, price, etc.) from the webhook payload before passing them to the Google Sheets node.

Pro tip: Add data validation steps in n8n to ensure only properly formatted records get written to your sheet, preventing corruption of your spreadsheet data.

Google Sheets Integration

The final piece connects your processed data to Google Sheets. Using n8n's Google Sheets node, you'll authenticate with your Google account and specify the target spreadsheet and worksheet.

The node can append new rows or update existing ones based on a unique identifier (like the Supabase record ID). For most use cases, we recommend appending new rows to maintain a complete change history in your sheet.
