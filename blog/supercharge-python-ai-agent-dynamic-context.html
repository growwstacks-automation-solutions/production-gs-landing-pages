Static vs. Dynamic Context: Why It Matters

Traditional AI agents receive their context only once - at initialization. This creates a fundamental limitation: any information that changes over time (like the current time, user session data, or file contents) becomes immediately outdated.

Dynamic context injection solves this by regenerating relevant information before processing each request. At 3:15 in the tutorial, the instructor demonstrates this perfectly by showing how asking "What time is it?" returns different answers throughout the day because the time context refreshes with each query.

Key benefit: Your agent becomes truly stateful without complex session management. Each request contains fresh, accurate context tailored to that moment.

The Decorator Pattern for Context Injection

The implementation uses Python's decorator pattern to register context functions. These are simple Python functions that return strings of contextual information (like the current time or user details).

The @context decorator marks these functions for automatic execution before each LLM request. Their outputs get formatted into XML-style context blocks and prepended as system messages. This elegant pattern makes it easy to add or remove context sources without modifying core agent logic.

Creating Effective Context Functions

Well-designed context functions follow three principles:

  1. Focused: Each function should return just one type of information (time, user details, etc.)
  2. Descriptive naming: Function names become context keys in the LLM prompt
  3. Fast execution: They run before every request, so avoid slow operations

The tutorial includes examples for getting the current time, system user, and even injects humor by having the agent end responses with "yo mama" jokes (visible at 9:15 in the video).

Real-World Applications

This technique unlocks several practical use cases:

  • Time-sensitive assistants: Agents that always know the current date/time for scheduling
  • Personalized responses: Incorporating user details or preferences into answers
  • File-aware agents: Injecting document contents that may change between requests
  • API-connected bots: Pulling fresh data from external services before responding

Pro tip: Start with 1-2 essential context items (like time), then expand based on what proves most useful in your application.

Current Limitations

While powerful, this approach has some constraints:

  1. Text-only: The agent can't take actions (send emails, modify files, etc.)
  2. Context overload: Too much injected context can confuse the LLM
  3. Prompt engineering: Still need careful prompt design for accurate responses

At 10:30 in the video, the instructor shows how the agent can miscalculate times if not properly prompted, demonstrating that while context helps, it doesn't eliminate the need for good prompt design.
