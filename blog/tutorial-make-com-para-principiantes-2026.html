El Problema: Gestión Manual de Leads

Jo, diseñador web independiente, recibía 20-30 consultas semanales través de su formulario de contacto. Cada lead requería:

  • Copiar manualmente datos a una hoja de cálculo
  • Investigar el sitio web del cliente
  • Redactar una respuesta personalizada
  • Programar una reunión en calendario
  • Enviar confirmación por correo

Este proceso tomaba 25-30 minutos por lead, sumando 12+ horas semanales en tareas repetitivas. Peor aún, los leads esperaban horas (a veces días) por una respuesta.

Dato clave: 68% de los clientes esperan respuesta en 1 hora tras enviar unormulario, pero el 24% abandona si no la recibe.

Formulario Automatizado con Tally

Tally es un constructor de formularios gratuito que se integra perfectamente con Make.com. Configuramos un formulario de contacto con:

  • Campos básicos (nombre, correo, teléfono)
  • Enlace al sitio web/red social
  • Selector de fecha y hora preferida

El disparador en Make.com se configura para activarse con cada nuevo envío:

Pro tip: Usa "Watch New Response" en Tally para activaciones instantáneas sin sondeo.

Al enviar el formulario, Make.com recibe los datos via webhook y comienza el flujo de automatización.

Gestión de Datos en Google Sheets

El segundo paso es almacenar sistemáticamente los leads en Google Sheets:

  1. Configurar hoja con columnas para cada campo
  2. Mapear datos del formulario a las columnas correspondientes
  3. Agregar campos técnicos (ID de evento, timestamp)

Esta hoja servirá como base de datos central para:

  • Histórico de leads
  • Análisis posteriores
  • Integración con otros sistemas

Truco clave: Usa "Actualizar fila" en lugar de "Agregar fila" para evitar duplicados al reprocesar datos.

Filtros y Lógica Condicional

El módulo "Router" permite bifurcar la automatización según condiciones:

  • Ruta 1: URLs bien formadas (con http://)
  • Ruta 2: URLs mal formadas (sin prefijo)

Para la ruta 2, añadimos lógica para:

  1. Verificar si la URL incluye "http" o "https"
  2. Agregar el prefijo faltante si es necesario
  3. Continuar con el procesamiento

Error común: 40% de los usuarios ingresan URLs incompletas. Esta validación previene fallos en pasos posteriores.

Extracción Automática de Datos Web

Browsera (antes Diffbot) analiza automáticamente el sitio web del lead:

  • Extrae texto principal
  • Captura screenshot
  • Proporciona HTML estructurado

Estos datos permiten:

  1. Entender el negocio del lead
  2. Identificar necesidades específicas
  3. Personalizar respuestas

Nota: Configuramos un segundo escenario que espera cuando Browsera completa el análisis, evitando tiempos de espera.

Agente de IA para Respuestas Personalizadas

El agente de IA de Make.com (antes Integromat AI) realiza tres acciones clave:

  1. 1. Programa reuniones: Revisa disponibilidad en calendario y agenda
  2. 2. Redacta respuestas: Analiza el sitio web y escribe correos personalizados
  3. 3. Envía confirmaciones: Usa Gmail para comunicarse con el lead

    4. Prompt del sistema (configuración del agente):

    "Eres Jo de Jo's Web Design. Analiza el sitio web del lead usando el HTML proporcionado. Programa reunión en su horario preferido si está disponible, o propón alternativas. Redacta respuesta profesional que demuestre comprensión de su negocio."

    Resultado: Respuestas en 2 minutos (vs 24 horas manuales) con mayor calidad y personalización.
