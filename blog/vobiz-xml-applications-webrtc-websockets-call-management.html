Vobiz XML vs SIP Signaling: Key Differences

Traditional voice applications rely on SIP (Session Initiation Protocol) to establish and manage calls through signaling layers. This adds complexity and latency as audio must pass through multiple routing points before reaching your AI agents.

Vobiz XML applications take a different approach. Instead of using SIP signaling, they instruct the Vobiz platform to answer calls directly and stream the raw audio over WebSocket connections to your server. This creates a direct pipeline between the caller and your AI processing logic.

75% faster audio delivery: By eliminating SIP signaling hops, Vobiz XML applications reduce audio latency by approximately 300-500ms per call compared to traditional SIP routing. This creates more natural conversations with AI agents.

Configuring Inbound Call Handling

Setting up inbound call handling with Vobiz requires creating an application that specifies your answer URL containing XML instructions. This tells Vobiz how to process calls for specific phone numbers.

At 1:45 in the tutorial video, you can see the exact steps for creating a new application. The key components are:

  1. Answer URL: Contains your XML instructions for call handling
  2. Hang-up URL: Receives notifications when calls end
  3. Fallback URL: Used if the answer URL becomes unavailable

Once configured, you link phone numbers to the application. All calls to those numbers will follow your XML-defined workflow.

XML Application Features & Capabilities

Vobiz XML applications support sophisticated call management features that would normally require expensive PBX systems. The XML instructions can implement:

  • Machine detection (distinguishing between humans and answering machines)
  • Call transfers between agents or departments
  • Custom hang-up conditions based on call analytics
  • Live call retrieval for quality monitoring
  • Call recording with configurable storage options

These features are implemented through simple XML tags that Vobiz interprets in real-time during calls. The platform handles all the underlying telephony complexity.

WebRTC Endpoint Management

Vobiz endpoints enable browser-based calling through WebRTC technology. Each endpoint represents a user or agent who can make and receive calls directly from their web browser.

Creating an endpoint (shown at 4:20 in the video) requires:

  1. A unique identifier (username)
  2. Authentication password
  3. Optional metadata for tracking

Endpoints integrate with your existing web applications through JavaScript APIs. This allows building custom calling interfaces without requiring users to install softphone applications.

Call Logs & Quality Analytics

Vobiz provides comprehensive call analytics through its call logs interface. For every call, you can view:

  • Call status and duration
  • Packet loss percentage (network quality indicator)
  • MOS (Mean Opinion Score) for audio quality
  • Codec used (PCMU, Opus, etc.)
  • Call flow diagram showing interaction points

These metrics help identify quality issues and optimize your XML configurations. The platform maintains historical data for trend analysis and reporting.

Multiple Number Support per Application

A powerful Vobiz feature is the ability to associate multiple phone numbers with a single application. This means:

  • Different numbers can share the same call handling logic
  • You can segment calls by number while maintaining centralized control
  • Number changes don't require XML modifications

At 3:15 in the video, you can see how to attach additional numbers to an existing application. This flexibility simplifies management for businesses with multiple departments, locations, or marketing campaigns.

Real-Time Application Updates

Vobiz applications can be modified at any time with changes taking effect immediately. This enables:

  • A/B testing of different XML configurations
  • Emergency updates during system issues
  • Seasonal or promotional workflow changes

The video demonstrates updating an application's answer URL at 2:50. The modification was applied to live calls within seconds, showing the platform's real-time capabilities.
