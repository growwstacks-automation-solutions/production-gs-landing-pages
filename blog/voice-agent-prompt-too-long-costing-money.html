The Hidden Cost of Long Prompts

Voice agents often start failing right when businesses need them most. A cancellation function stops working. The agent begins giving inconsistent responses. The natural solution seems to be adding more instructions - but this approach backfires spectacularly.

What most teams don't realize is that long prompts create three simultaneous problems: they increase LLM costs, reduce reliability, and make troubleshooting nearly impossible. At 5500 tokens, agents make 20-30% more mistakes than at 3500 tokens, while costing significantly more to operate.

The paradox of prompt engineering: Adding more instructions to fix problems often creates conflicting guidance that makes the agent less reliable. Clear, concise prompts outperform verbose ones every time.

Live Optimization: 5500 → 3500 Tokens

During a live build session (shown at 4:12 in the video), we analyzed a real voice agent prompt that had grown to 5500 tokens. The agent was struggling with function reliability - cancellation and rescheduling functions weren't triggering consistently.

Using three specific techniques, we reduced the prompt to 3500 tokens while actually improving function reliability:

  1. Consolidated redundant sections: The cancellation handling appeared three times with overlapping content
  2. Reduced example interactions: From 10 scenarios to 3-4 composite examples covering key paths
  3. Streamlined function guidance: Used the "Agent SES" format to explicitly show function usage

The result? 20% lower costs from staying under Retell's 3500 token threshold, plus more reliable function calls despite the shorter prompt.

3 Proven Prompt Shortening Techniques

Through hundreds of voice agent implementations, we've identified three techniques that reliably shorten prompts while improving performance:

1. Remove Redundant Instructions

Most long prompts say the same thing multiple ways. Cancellation handling appeared three times in our example. Consolidating these into one clear section saved 400+ tokens while making the guidance clearer.

2. Consolidate Examples

While examples are powerful teachers, 10 scenarios is excessive. We combined them into 3-4 composite examples covering: happy path, objection handling, and time zone scenarios.

3. Function-Specific Formatting

The "Agent SES" format (Say, Execute, Say) shows exactly how the LLM should handle functions:

Agent SES format example:
"No worries let me check availability again
[use check_availability function]
Here are three available times that work..."

This mirrors how LLMs actually process function calls, improving reliability by 30-40%.

The Research Behind Prompt Length

Academic research (shown at 12:45 in the video) demonstrates a clear relationship between prompt length and LLM accuracy. Across multiple models, performance declines as prompts grow longer.

While the exact threshold varies by model, the pattern is consistent: shorter prompts yield more reliable agents. Retell's 3500 token cost threshold aligns remarkably well with observed reliability breakpoints.

Key finding: At 1000+ tokens, accuracy begins declining. By 5500 tokens, error rates are 20-30% higher than at 3500 tokens. This explains why shortened prompts often perform better.

The implication is clear - focused, concise prompts create more reliable voice agents than comprehensive ones that try to cover every edge case.

How Prompt Length Affects Your Costs

Retell implements a cost surcharge when prompts exceed 3500 tokens. Here's how it works:

  • Base LLM cost (e.g., $0.05/minute) applies to first 3500 tokens
  • Each additional 3500 token chunk incurs another full LLM cost
  • A 5500 token prompt costs 20% more than a 3500 token one

For our example agent:

Cost breakdown:
Base cost: $0.12/minute
Tokens: 5200 (1700 over 3500)
Surcharge: $0.024/minute
Total cost: $0.144/minute (20% increase)

By reducing to 3500 tokens, we eliminated the surcharge while improving reliability - a rare win-win in engineering.

Improving Function Reliability

Function calls are particularly sensitive to prompt length. Long prompts often cause functions to:

  • Trigger at wrong times
  • Miss required parameters
  • Return inconsistent formats

The solution combines prompt shortening with specific function training:

  1. Include one example interaction per function type
  2. Show the complete Agent SES flow (Say, Execute, Say)
  3. Keep function descriptions concise but explicit

This approach reduced function errors by 40% in our testing, while simultaneously cutting prompt length by 35%.
