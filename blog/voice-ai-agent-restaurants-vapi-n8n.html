How the Voice AI Agent Works

VAPI serves as the conversational interface while n8n handles the backend logic and CRM integrations. The system uses GPT-4 as its "brain" to interpret requests and select appropriate tools from its workflow playbook.

Key components include:

  • Deepgram Aura 2 for ultra-low latency voice transcription
  • Customizable system prompts that define the agent's personality and capabilities
  • Variable replacement where curly bracket placeholders insert dynamic data
  • Five core tools for table availability, reservations, rescheduling, guest lookup, and cancellations

n8n Workflow Integration

The magic happens when VAPI webhooks trigger n8n workflows. This connects the voice conversation to practical business actions:

  1. VAPI captures call details (party size, date/time, special requests)
  2. n8n checks the reservation system for availability
  3. Upon confirmation, n8n updates the calendar
  4. Twillio sends SMS confirmations to guests
  5. Guest details save to the CRM (Google Sheets, Notion, Airtable, or Salesforce)

Implementation tip: Start with Google Sheets integration before moving to more complex CRMs. This allows quick testing without disrupting existing systems.
