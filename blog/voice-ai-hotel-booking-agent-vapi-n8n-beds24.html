Configuring Your Vapi AI Agent

The foundation is creating a Vapi agent that sounds professional yet approachable. At 4:32 in the video, you'll see the critical system prompt configuration that makes Sophia (our example agent) so effective:

  1. Choose OpenAI's GPT-4-turbo model for optimal balance of speed and intelligence
  2. Set latency mode to "Low" (costs $0.09/minute)
  3. Use ElevenLabs multilingual V2 voice with "Sarah" as the base
  4. Configure Deepgram as the speech-to-text transcriber

Pro tip: The system prompt should establish the agent's role ("Sophia, friendly receptionist at Grand Budapest Hotel"), core directives (efficient booking while being warm), and common phrases ("One moment please" when checking availability).

Beds24 PMS Integration

Connecting to Beds24's API allows the AI to check real-time availability and make reservations directly in your system. The integration requires:

  • Beds24 API credentials (found under Settings → API)
  • Property ID for the specific location
  • Configured webhook endpoints for availability checks and booking creation

At 12:15 in the video, you'll see the exact JSON structure needed for the booking creation call, including parameters for check-in/out dates, guest count, and special requests.

Building the n8n Automation Workflow

n8n acts as the bridge between Vapi and Beds24, handling the logic flow:

  1. Vapi triggers n8n when a caller wants to check availability
  2. n8n calls Beds24's API with the requested dates
  3. Available rooms and prices are formatted for the AI to present
  4. Once guest confirms, n8n creates the reservation in Beds24
  5. Confirmation details are sent back to Vapi to read to the guest

Critical node: The HTTP Request node at 17:40 shows how to properly authenticate with Beds24's API while protecting sensitive credentials. Always use environment variables for API keys.

Testing and Optimization

Before going live, rigorously test with:

  • Different accent callers (ensure 90%+ speech recognition accuracy)
  • Edge cases like fully booked dates or special requests
  • Payment processing failures
  • API timeout scenarios

Monitor the first 100 calls closely, tracking where human intervention is needed. These become training opportunities to improve your agent's success rate.
