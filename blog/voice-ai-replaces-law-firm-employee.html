How the Voice AI Intake System Works

This isn't a simple IVR tree or basic chatbot. The architecture combines three components:

1. Vapi's conversational AI handles natural dialogue using GPT-4 with legal-specific prompt engineering (we'll share the exact template)

2. Specialized data extraction nodes parse free-form responses about injuries, liability, and damages into structured fields

3. n8n workflows route completed intakes to the right attorney based on case type and urgency

Step-by-Step Vapi Configuration

Starting at 11:05 in the video, we configure the AI agent:

Step 1: Create Your Assistant

Select GPT-4 as the base model with these critical settings:

  • Voice: "Hermes" (natural legal tone)
  • Background noise: Off
  • Speech recognition: Enhanced for medical/legal terms

Step 2: Craft the Legal Intake Prompt

The prompt template shown at 12:30 includes four essential sections most implementations miss:

  1. Firm Identity: "You are the intake specialist for [Firm Name] specializing in [Practice Areas]"
  2. Style Guidelines: "Speak at 85 words per minute. Pause 1.2 seconds after questions."
  3. Task Flow: Must collect name → contact → incident → injury → follow-up
  4. Error Handling: "If caller mentions suicide or child abuse, immediately transfer to [Human Number]"

Pro Tip: At 14:20 we add a webhook safety net - when Vapi's API occasionally delays, this prevents lost calls by having n8n poll for recent interactions every 30 minutes.

Building the n8n Automation Workflow

The magic happens in the n8n workflow shown at 16:00:

Step 1: Call Processing Trigger

Configure the HTTP node to receive Vapi webhooks, then filter for only completed calls (status = "finished")

Step 2: Data Extraction

Specialized function nodes parse the transcription for:

  • Named entities (people, places, dates)
  • Injury types and body parts
  • Liability indicators ("ran red light", "wet floor")

Step 3: Case Routing Logic

Based on extracted data, the workflow:

  1. Scores case urgency (1-5 scale)
  2. Matches to attorney specialties
  3. Checks for conflicts
  4. Routes to appropriate team member

Specialized Data Extraction Techniques

Generic LLMs miss critical legal details. At 20:10, we implement three extraction methods:

1. Predefined Schemas: For dates, locations, and contact info where formats are predictable

2. Contextual Analysis: Identifying "soft injuries" like emotional distress that clients might not volunteer

3. Legal Taxonomy Matching: Mapping colloquial terms ("broken arm") to medical codes (S42.301A)
