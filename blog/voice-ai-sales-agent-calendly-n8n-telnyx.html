n8n + Telnyx Workflow Overview

This automation connects three powerful systems: your Calendly scheduler, n8n's workflow engine, and Telnyx's voice AI platform. When combined, they create a seamless experience where the AI calls exactly when the customer expects.

The workflow has three key components we'll walk through:

  1. Telnyx API credentials setup
  2. Calendly trigger configuration
  3. AI call scheduling logic

Step 1: Setting Up Telnyx Credentials

Begin by creating API credentials in your Telnyx account (2:15 in the video). Navigate to API Keys and generate a new key specifically for n8n. This ensures secure access without exposing your main account credentials.

In n8n, add a new Telnyx node and paste your API key. The "Create Scheduled Event" operation lets you programmatically trigger calls through your AI assistant. You'll need your Assistant ID from Telnyx's AI portal - this identifies which agent script to use.

Pro tip: Create separate assistants for different products or sales stages. The demo uses a general sales agent, but you might have specialized agents for onboarding, upgrades, etc.

Step 2: Configuring the Calendly Trigger

Replace the manual trigger with n8n's Calendly node (4:30 in the video). This listens for new bookings in real-time. The critical configuration is mapping the event's location field (where users enter their phone number) to the call recipient.

Time zone handling is crucial here. Ensure your Calendly account, n8n workflow, and Telnyx assistant all use the same time zone reference. The demo shows how to extract and format the start time correctly for the Telnyx API.

Step 3: AI Call Scheduling Logic

With the data flowing from Calendly, the Telnyx node schedules the call for the exact booked time (6:45 in the video). The AI agent's phone number (Agent Target) stays constant, while the end user's number comes from the booking.

When executed, this creates a scheduled call in Telnyx's system. At the appointed time, your AI agent calls the prospect with a natural conversation flow. The entire process happens without any human intervention after the initial booking.

Implementation note: For high-volume sales teams, add a delay node to space calls 2-3 minutes apart. This prevents call quality issues from back-to-back connections.
