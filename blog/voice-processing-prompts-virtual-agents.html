The Silence Problem in AI Calls

When AI-powered virtual agents process requests, there's often a 3-5 second delay while fetching data or generating responses. Unlike human agents who might say "Let me check that for you," most AI systems simply go silent.

This creates an uncomfortable experience where callers wonder: Did the call drop? Is the system frozen? Should I repeat my question? Research shows 68% of customers perceive these silent pauses as system failures, even when responses eventually come through.

Key insight: Silence during AI processing isn't neutral - it actively damages customer trust. Voice processing prompts solve this by providing auditory feedback that maintains engagement.

How Voice Processing Prompts Work

The November 2025 enhancement introduces configurable audio feedback during processing delays. When the system detects a response delay exceeding 1 second, it triggers your chosen audio prompt until the full response is ready.

This works seamlessly with existing LLM-powered virtual agents. The feature doesn't change response times - it simply fills what would otherwise be dead air with brand-appropriate audio signals. Administrators can set minimum/maximum delay thresholds to optimize when prompts activate.

Two Configuration Options

Voice processing prompts offer flexibility through two configuration approaches:

1. Global Flow Settings

Configured under Actions, these settings apply to all call guides in your flow. Ideal for maintaining consistent audio branding across all interactions. Changes here automatically propagate unless overridden at the guide level.

2. Per-Guide Customization

Each call guide in the architect botflow can override global settings with guide-specific prompts. This allows tailored experiences - like technical status messages for IT helpdesk vs conversational hold music for customer service.

Implementation tip: Start with global settings for consistency, then add guide-level customizations only where they significantly improve context-specific experiences.

Prompt & Music Customization Types

Three audio customization options create natural-sounding processing feedback:

Text-to-Speech Messages

Custom phrases like "Checking your details now" or "One moment while I verify this" signal activity. These work with your existing TTS voice for consistency.

Background Music

Brand-appropriate hold music maintains engagement without repetitive verbal messages. Ideal for longer processing times.

Combined Approach

Play a brief verbal message followed by music. This provides immediate feedback then transitions to non-intrusive audio.
