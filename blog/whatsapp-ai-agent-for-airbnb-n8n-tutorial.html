Step 1: n8n Setup and WhatsApp Trigger

The foundation is n8n's WhatsApp Business Cloud node. This triggers the workflow whenever a new message arrives. Configuration requires:

Step 1.1: WhatsApp Business API Connection

Create a new credential in n8n with your WhatsApp Business API client ID and secret. The setup process involves:

  • Registering your phone number with Facebook's WhatsApp Business platform
  • Generating API credentials in the WhatsApp Manager
  • Adding these to n8n's credential manager

Step 1.2: Message Trigger Configuration

Set the trigger to activate on all incoming messages. The key output is the message body and sender's WhatsApp ID - these become the foundation for all subsequent automation.

Pro tip: Test your connection by sending a message to your WhatsApp number and verifying it appears in n8n's execution log before proceeding.

Step 2: Building the AI Agent Core

The brain of the system is n8n's AI Agent node configured with OpenAI's GPT model. Three critical components make it effective:

Step 2.1: System Prompt Engineering

The system message defines the AI's role and capabilities. Ours includes:

  • Identity: "You are the Pearl District Loft AI concierge"
  • Purpose: "Make each guest stay effortless through communication"
  • Data access instructions for guest verification
  • Property details (Wi-Fi codes, check-in/out times, etc.)
  • Rules for different guest statuses (current, upcoming, past)

Step 2.2: Model Selection

We use OpenAI's GPT-3.5-turbo for its balance of cost and performance. Key settings:

  • Temperature: 0.7 (for slightly creative but consistent responses)
  • Max tokens: 256 (keeps responses concise)
  • Frequency penalty: 0.5 (reduces repetition)

Step 2.3: Conversation Memory

A Simple Memory node stores chat history using the guest's WhatsApp ID as the key. This enables contextual conversations where the AI remembers previous questions and preferences.

Step 3: Guest Verification with Google Sheets

Security is critical - we can't share property details with just anyone. The solution integrates n8n's Google Sheets node to verify guests against a reservations spreadsheet.

Step 3.1: Google Sheets Setup

Create a sheet with columns for:

  • Guest name
  • WhatsApp number (in international format)
  • Check-in date
  • Check-out date

Step 3.2: Real-Time Verification

The workflow:

  1. Extracts sender's WhatsApp number from trigger
  2. Queries Google Sheets for matching number
  3. Checks if current date falls between reservation dates
  4. Passes verification status to AI agent

Security note: The AI automatically withholds sensitive information like door codes if the guest's stay has ended or hasn't begun yet, while still providing polite assistance.

Step 4: Adding Local Recommendations via Perplexity

Guests often ask about local attractions and dining. Instead of maintaining a static list, we connect Perplexity AI for real-time web searches.

Step 4.1: Perplexity Tool Integration

Configure the AI Agent to use Perplexity for:

  • Restaurant/cafe recommendations
  • Attraction information
  • Activity suggestions

Step 4.2: Location Sharing

When guests ask "Where is [place]?", the system:

  1. Uses Perplexity to find the address
  2. Converts address to latitude/longitude
  3. Sends a WhatsApp location pin

This creates a seamless experience where "What's the best brunch spot?" yields not just names, but direct map links to top recommendations.

Step 5: Emergency Handling and Owner Alerts

Some situations require human intervention. The system intelligently escalates issues while providing immediate guest support.

Step 5.1: Emergency Detection

The AI scans messages for keywords like:

  • "Locked out"
  • "No power"
  • "Flood"
  • "Emergency"

Step 5.2: Dual Response System

When emergencies are detected:

  1. The owner receives an email with all guest details and the exact issue
  2. The guest gets immediate instructions ("Try backup key in lockbox")
  3. Both parties receive ETA for resolution

Real-world result: One host reported resolving a lockout in 12 minutes (vs. 47 minutes previously) because the system immediately provided temporary access while alerting them.

Step 6: Testing and Deployment

Before going live, rigorous testing ensures reliable performance across scenarios.

Step 6.1: Test Cases

Verify the system handles:

  • Current guest inquiries (full access)
  • Upcoming guest questions (pre-arrival info only)
  • Past guest requests (limited response)
  • Unknown numbers (polite deflection)

Step 6.2: Going Live

Final steps:

  1. Activate the n8n workflow
  2. Set up error monitoring
  3. Create owner notification protocols
  4. Inform guests about the AI assistant
