Step 3: Install Evolution API

With your server ready, installing Evolution API is straightforward:

  1. SSH into your server (use Putty on Windows or Terminal on Mac)
  2. Run the installation command: npm install -g @evolution/api
  3. Create a new instance with evolution create-instance
  4. Name your instance (e.g., "MyBusiness")
  5. Note the API token that's generated

Once installed, you'll need to connect your WhatsApp account by scanning a QR code from your phone. This links your dedicated number to the API instance.

Important: Keep your API token secure - this is essentially the password to your WhatsApp automation. Never share it publicly or commit it to GitHub.

Step 4: Configure n8n Integration

Now we'll connect Evolution API to n8n to build powerful automations:

  1. In n8n, install the Evolution API community node
  2. Create new credentials with your instance URL and API token
  3. Test the connection by sending a message to yourself
  4. Set up a webhook to receive incoming messages

The video shows exactly how to configure the webhook at 4:32. This is crucial for building two-way communication flows where your automation can respond to customer messages.

Once connected, you'll see green indicators in n8n confirming everything is working. Now you're ready to build your first automation workflow.

Step 5: Build Your First Automation

With everything connected, you can now create powerful WhatsApp automations. Here's a simple lead qualification bot to get started:

  1. Trigger: Webhook receives new message
  2. Check if message contains keywords like "price" or "quote"
  3. If yes, send automated response with pricing options
  4. If no, route to human agent or send default reply
  5. Log all interactions to Google Sheets or your CRM

Dynamic text variables let you personalize messages with the customer's name, order details, or other data from your systems - making the automation feel human while saving hours of manual messaging.

Best Practices to Avoid Bans

While Evolution API is reliable, you should follow these guidelines to minimize risk:

  • Limit to 300 messages/day - Higher volumes may trigger flags
  • Add delays between messages - 2-5 seconds to appear human
  • Warm up new accounts - Gradually increase activity over 10 days
  • Avoid spammy content - No bulk marketing messages
  • Monitor delivery reports - Watch for failed messages

By keeping your usage reasonable and focused on legitimate business communication, your automation can run indefinitely without issues. The dedicated number acts as a buffer, protecting your primary business contacts.
