Twilio WhatsApp Sandbox Setup

While you could use WhatsApp Business API directly, Twilio's sandbox environment eliminates the complex Meta verification process. Here's how to connect it:

Step 1: Create Twilio Account

Sign up at Twilio.com and navigate to Messaging → WhatsApp. The sandbox provides free testing with one approved phone number.

Step 2: Connect Your WhatsApp Number

Under Sandbox Settings, find the "join [code]" message. Send this exact message from your phone to Twilio's number (+1 415-523-8886). This pairs your device for testing.

Step 3: Configure Webhook URL

In the "When a message comes in" field, you'll paste your n8n webhook URL after we set it up in the next section.

Production note: For live deployment, you'll upgrade from the sandbox to Twilio's paid WhatsApp API, which costs $0.005 per message but handles higher volume with delivery guarantees.

n8n Webhook Configuration

The webhook acts as your WhatsApp message receiver. At 4:31 in the video, you'll see the exact setup:

Step 1: Create Webhook Node

Add a Webhook node in n8n and set the method to POST. Rename the path to something like "/whatsapp-incoming".

Step 2: Connect to Twilio

Copy the webhook's Production URL (not Test URL) and paste it into Twilio's "When a message comes in" field we prepared earlier.

Step 3: Add Twilio Send Node

This node replies to customers. Configure it to use the SMS resource (Twilio handles WhatsApp through this) and toggle "Send to WhatsApp".

The critical configuration is the phone number parsing. Customer messages arrive in format "whatsapp:+1234567890" - you'll extract just the number part using:

 {{$node["Webhook"].json["body"]["From"].split(":")[1]}}

Building the AI Agent Core

The AI Agent node (shown at 6:18 in the video) is the brain of the operation. Key configuration points:

System Prompt Engineering

The template includes a battle-tested prompt that:

  • Injects context (customer number, current date, booking ID)
  • Mandates tool usage before confirming bookings
  • Sets polite but concise response style
  • Prevents hallucinations with strict instructions

Model Selection

Using OpenRouter as middleware lets you choose between Gemini, Claude, or GPT models. Gemini 2.5 Pro offers the best balance of cost and performance for this use case.

Memory Configuration

Window Buffer Memory keeps conversation context using the customer's phone number as session key. This maintains separate threads for different customers.

Pro tip: During testing, append "+20" to phone numbers to easily reset memory while iterating on your prompt.

Connecting Inventory Tools

The four tools (demonstrated at 8:47) give the AI real business capabilities:

1. Fleet Inventory Tool

Google Sheets node configured to "Get Rows" from your inventory sheet. Returns available cars/items with IDs and daily prices.

2. Availability Check Tool

Another Sheets node querying your bookings sheet to verify requested dates are open.

3. Booking Management Tool

Configured to "Append or Update Row", this handles both new bookings and modifications using a unique Booking ID system.

4. Calculator Tool

Automatically called when the AI needs to multiply daily rate by rental duration.
