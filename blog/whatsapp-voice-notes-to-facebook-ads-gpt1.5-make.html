WhatsAppable Integration Setup

The first critical component is connecting your WhatsApp account to Make.com. While you could build your own WhatsApp Business API integration, WhatsAppable provides a ready-made solution that works with personal or business accounts.

At the 1:45 mark in the video, you'll see the two WhatsAppable options: Their bot solution (using their WhatsApp number) or the Notifier system (using your own number). The Notifier system requires installing a module in Make.com and authorizing the connection.

Key configuration: When setting up the trigger in Make.com, ensure you select the correct WhatsAppable module (Notifier Bot vs Notifier System) based on which product you're using.

Voice-to-Text Transcription

Once the WhatsApp voice note is received, the workflow downloads the audio file via HTTP request. The raw audio then feeds into OpenAI's Whisper model for transcription through Make.com's "Generate a Transcription" module.

At 3:20 in the tutorial, you'll see the critical detail: The audio file must be named with an .ogg extension for Whisper to process it correctly. The transcription output becomes the raw material for the next step - prompt engineering.

Prompt Engineering for GPT-1.5

Raw voice transcriptions rarely make good image generation prompts. The workflow includes a crucial refinement step using another OpenAI module ("Generate a Response") to transform the rough transcript into an optimized DALL·E prompt.

The example at 4:15 shows how a casual voice note about a "skate park with paella stand" becomes a detailed prompt specifying "lively beachside skate park with palm trees, vibrant paella stand in foreground, sunset lighting, digital art style."

Pro tip: You can customize the prompt engineering instructions to match your brand's visual style - specifying particular color schemes, compositions, or art directions.

GPT-1.5 Image Generation Workaround

Here's where things get interesting. While Make.com has a dedicated "Generate Image" module, it hadn't been updated for GPT-1.5 at recording time (5:30 in the video). The solution? Using the generic OpenAI API module with a custom JSON payload.

The workflow creates a JSON structure specifying the GPT-1.5 model, your refined prompt, quality settings, and output format. This gets sent via API call to OpenAI's image generation endpoint, returning the base64-encoded image data.

Google Drive Storage Setup

The base64 image data needs decoding before storage. At 7:50, you'll see the formula required to properly format the image for Google Drive upload. The workflow saves each generated ad to a specified folder with a timestamped filename.

This creates an organized library of all your generated ads. The web content link for each image then becomes available for the final step - sending the completed ad back to WhatsApp.

WhatsApp Completion Notification

The circle completes when the workflow sends the generated image back to your WhatsApp (8:40 in the video). Using the same WhatsAppable module, it attaches the Google Drive image URL and includes any caption text you've specified.

This creates a seamless loop: Voice note in → polished ad out. The entire process typically completes in 2-3 minutes, letting you review and approve ads from your phone without ever touching design software.

Advanced option: You can extend this workflow to automatically create Facebook Ads Manager campaigns using the generated images and transcribed text as ad copy.
