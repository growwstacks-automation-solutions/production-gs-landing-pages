The Hidden Cost of Zoom Cloud Storage

Every Zoom meeting host knows the frustration: you finish an important client call or team meeting, hit "Stop Recording," and immediately get the dreaded storage warning. Despite paying $15-20/month for your Zoom Pro account, you're constantly deleting old recordings or facing expensive upgrade prompts.

The math is brutal: a single hour-long HD meeting recording consumes 200-500MB. With most paid plans including just 1-10GB of cloud storage, you're looking at 20-50 hours of recordings before hitting capacity. Many businesses don't realize they've hit their limit until they urgently need to record an important meeting.

The average business wastes $480/year on unnecessary Zoom storage upgrades when free alternatives like Google Drive (15GB free) or Dropbox (2GB free) could handle their recordings indefinitely.

How the Make.com Automation Works

The solution is surprisingly simple: automatically transfer recordings to cheaper cloud storage while keeping your Zoom account clean. Here's what happens behind the scenes:

  1. The workflow checks Zoom every hour for new cloud recordings
  2. When a recording is found, it verifies the download URL exists
  3. It downloads the recording file to a temporary location
  4. The system uploads it to your preferred cloud storage (Google Drive shown)
  5. Finally, it deletes the original from Zoom to free up space

What makes this particularly powerful is the smart filtering built into the workflow. At 2:15 in the video, you'll see how we automatically ignore recordings under 5MB (accidental starts) and route different meeting types to specific folders based on their titles.

Key Components of the Workflow

1. The Watch Recordings Module

This acts as the trigger, polling Zoom's API for new cloud recordings. It's configured to check every hour, but you can adjust this frequency based on your meeting volume.

2. Download URL Verification

As shown at 3:42 in the video, you need to navigate to payload → recording → recording files → download URL to properly access the recording. This step ensures the file is ready before proceeding.

3. Size Filter (5MB Threshold)

The workflow checks if recordings are above 5MB (about 1 minute of video) to avoid processing accidental starts. This simple filter prevents wasted processing on recordings you don't actually need.

4. Google Drive Upload

Files are named using the meeting date and topic for easy searching. Critical detail: always append .mp4 to the filename (4:56 in video) so Google can properly preview the video.

Smart Filtering for Different Meeting Types

The real power comes from automatically routing recordings to the right locations based on meeting titles. Here's how it works:

Example: Weekly team meetings titled "Marketing Sync" automatically go to your team folder, while client calls titled "Acme Corp Q2 Review" go to the client's dedicated folder.

The workflow uses Make.com's router module with multiple filters (shown at 5:30 in the video). Each filter checks the meeting title against your predefined patterns:

  • Contains "Team Meeting" → /Team Recordings/
  • Contains "Client" → /Client Calls/
  • Default → /Other Recordings/

This eliminates manual sorting while ensuring the right people always have access to the right recordings.

Implementation Steps

Step 1: Set Up Zoom API Connection

Create a JWT app in your Zoom developer console and note the API key and secret. These credentials allow Make.com to access your recordings.

Step 2: Configure Google Drive Integration

Connect your Google account in Make.com and create the target folder structure (Team, Client, Other etc.). Note the folder IDs for the workflow.

Step 3: Build the Workflow

Follow the video tutorial starting at 3:10 to recreate the exact module sequence: Watch → Filter → Download → Upload → Delete.

Step 4: Test with Sample Recordings

Create test meetings with different titles to verify the routing logic works before enabling auto-delete from Zoom.

Step 5: Schedule Regular Runs

Set the workflow to run hourly (or more frequently if needed) to process new recordings promptly.
