What are AI Agents?

AI Agents bring reasoning and adaptability into your automation setup in Make. Instead of manually building out every step, Agents let you describe what you need in natural language, then work out how to make it happen using the 2,500+ apps and tools already available in Make.

Think of them as intelligent operators that sit inside Make scenarios. They're able to interpret fuzzy or unpredictable input (even with spelling errors), decide which existing sub-scenarios, apps or other Agents to call on, and handle back-and-forth interactions or multi-step requests.

Pro tip: AI Agents excel at handling requests that would require complex conditional logic in traditional automation, like interpreting natural language commands or adapting to incomplete information.

Do AI Agents replace traditional automation?

In a word, no. There's still a time and place for classic or deterministic automation, with and without AI added in. As Benjamin Potet explains, "AI Agents are not meant to replace what you've built in Make - they are simply another tool in your automation kit."

Traditional automation remains ideal for structured, repetitive tasks where the inputs and outputs are predictable. AI Agents complement these workflows by handling the more complex, interpretive tasks that would be cumbersome to build manually.

What's the difference between AI agents and traditional workflow automation?

Classic automation is deterministic, or rule-based - you define every step and create routes that your bundles of data will follow every time. It's ideal for structured, repetitive, and most of all predictable tasks. It succeeds at dealing with well-defined inputs and outputs, but scenarios can get big fast and take time to maintain.

Agentic automation, on the other hand, is goal-based. You tell the agent or agents what you want, your agent assesses the resources - usually other scenarios or subscenarios - you've made available, and figures out how to accomplish the task you've outlined. Scenarios stay smaller and updates are easier because the agent handles the majority of the logic for you, even if your explanation is in "messy" language.

Agentic vs. classic AI in action

To show the difference between classic and agentic automation, the demo walks through the same use case built two ways. The example task is managing contacts in a database, using Telegram as a trigger for receiving user input. It includes adding a contact, updating or deleting a contact, searching contacts, and requesting follow-up tasks like filtering or exporting to CSV.

The traditional approach

Tackling the task using a classic Make scenario with fixed logic involves several specific steps:

  • Sub-scenarios are built to create, update, delete, and search contacts
  • Manual routing is set up, with an AI module to detect intent and route to the right sub-scenario
  • Follow-up logic is set to detect incomplete input and to send follow-up questions

This approach handles simple commands well but struggles with complex requests that haven't been explicitly programmed, like "list contacts in China and send a CSV".

The Make MCP server architecture that powers AI Agent functionality

The AI Agents approach

The same task built with AI Agents uses the same sub-scenarios but with key differences:

  • Instead of building routes or filters, a Make AI Agent module is added
  • Each sub-scenario is given a short description so the agent understands when to use it
  • A Telegram message triggers the agent, which handles everything from there

This approach requires no routing logic. The AI agent interprets intent and handles follow-ups automatically, adapting to complex or unexpected inputs that the traditional approach couldn't manage.

No, AI Agents don't replace traditional automation. There's still a place for classic, rule-based automation - especially for structured, repetitive tasks where predictability is crucial.

AI Agents complement traditional automation by handling more complex, interpretive tasks. Many businesses use both approaches together, with traditional automation handling the predictable parts of a workflow and AI Agents managing the more variable components.

Traditional automation is deterministic - you define every step and data route in advance. AI Agents are goal-based - you describe what you want to accomplish and the agent figures out how to achieve it using available resources.

This fundamental difference means AI Agents can handle requests you haven't explicitly programmed solutions for, as long as they have the necessary sub-scenarios available and properly described.

Use traditional automation for predictable, repetitive tasks with clear inputs and outputs - things like data synchronization, scheduled reports, or straightforward data processing.

AI Agents shine when you need flexibility - handling unpredictable inputs, complex multi-step requests, or situations requiring interpretation of natural language commands. They're particularly valuable for customer-facing interfaces or processes with many potential variations.

AI Agents can interpret and execute complex requests by analyzing the intent behind the command and determining which available sub-scenarios can fulfill it. For example, a request like "list contacts in China and send a CSV" would be broken down into components.

The agent would first identify this requires a search/filter operation followed by a data export. It would then determine which sub-scenarios can perform these functions, execute them in sequence, and return the combined result - all without explicit programming for this specific combination of actions.

AI Agents may not be as fast as traditional automation for simple, repetitive tasks since they need to interpret each request. They also require well-described sub-scenarios to function optimally - poorly described components can lead to suboptimal agent behavior.

For mission-critical processes where absolute predictability is required (like financial transactions), traditional automation might be preferable. AI Agents also currently work best with English-language inputs, though support for other languages is improving.
