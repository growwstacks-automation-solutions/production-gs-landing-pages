What is an AI sales agent?

A Sales AI Agent is a goal-driven assistant that automates parts of your outreach process. Instead of manually switching between tools, searching for lead information, or copying data into CRMs, the agent handles those tasks based on your instructions.

In Make, you create a Sales AI Agent by assigning it a clear goal (such as sending personalized follow-ups or enriching leads) and a set of tools it can utilize. You then connect the agent to an input source, such as an email inbox or form submissions. Whenever the agent receives a new lead, it decides which tools to use to complete the task and returns a response.

What AI sales agent does (and why it's useful)

Sales teams often lose time manually responding to inbound leads, enriching data, and logging contacts into their CRM. This AI Agent solves that by automating the entire workflow from first reply to optional meeting scheduling.

Here's what this Sales Outreach Agent is designed to do:

  • Monitor a dedicated sales inbox for new leads
  • Send personalized first replies with qualifying questions
  • Enrich company data using the lead's email domain
  • Create CRM records with all interaction details
  • Follow up automatically on missing information
  • Schedule meetings when leads are qualified

Pro tip: Start with 2-3 core functions like email replies and CRM logging, then expand to more advanced features like meeting scheduling once your basic workflow is stable.

Apps used in the example

This implementation uses several common sales tools, but all can be swapped for alternatives:

1. Gmail

Monitors incoming sales emails and sends replies. Can be replaced with Outlook, Front, or any email service with IMAP support. The agent watches for messages sent to addresses like sales@yourcompany.com and triggers when new potential leads arrive.

2. ZoomInfo

Enriches company details using the lead's domain. Alternatives include Clearbit, Apollo, or any enrichment API. This automatically pulls data like company size, industry, and funding to help prioritize and personalize follow-ups.

3. HubSpot

Logs leads as contacts and companies. Other CRMs like Salesforce, Pipedrive or Copper work equally well. The agent creates complete records including notes from the interaction and enriched company data.

4. OnceSub

Sends scheduling links when leads are ready to talk. Replace with Calendly, SavvyCal, or your preferred booking tool. This optional component helps convert qualified leads into meetings automatically.

Scenarios your agent will use

Your AI agent will execute several specialized scenarios (mini-workflows) to handle different parts of the sales process:

1. Send Initial Reply

Crafts a friendly, professional email that thanks the sender, introduces your product in 2-3 sentences, and asks key qualifying questions. The tone and structure follow your brand guidelines while leaving room for personalization based on lead details.

This scenario includes fallback logic to handle incomplete responses. If a lead doesn't answer certain questions, the agent can send a polite follow-up after a set period asking for the missing information.

2. Enrich Company Info

Takes the lead's email domain and queries enrichment services for firmographic data. Returns details like company name, industry, employee count, location, and recent funding rounds. This information helps tailor follow-ups and prioritize high-value accounts.

The scenario handles cases where enrichment data is incomplete or unavailable. It can either proceed with partial data or trigger a request for manual review depending on your preferences.

3. Log Contact in CRM

Creates or updates contact and company records in your CRM. Includes all available details from the initial interaction and data enrichment. Can also set lead scores, tags, or custom fields based on your sales process.

This scenario checks for existing records to prevent duplicates and can associate new contacts with existing accounts when appropriate.

Pro tip: Name your scenarios clearly (like "Send Initial Reply" or "Enrich Company Info") so your AI agent can easily understand when to use each one based on the context.

Step-by-step: How to build the Sales outreach AI agent

Now let's walk through the actual construction process in Make.com:

Step 1: Set up email monitoring

Create a new scenario that watches your sales inbox. Use the "Watch Emails" module for your email provider (Gmail, Outlook, etc.). Configure filters to only trigger for messages sent to your sales alias (like sales@company.com) that appear to be potential leads.

Add conditions to exclude automated messages, out-of-office replies, and other non-lead emails. You can filter by subject line keywords, sender domains, or message content patterns.

Step 2: Add the AI agent

Insert the "Run an agent" module into your scenario. Map the incoming email content into the Message field, including the sender's details, email body, and any relevant metadata. This provides the context your agent needs to make decisions.

Configure the agent's goal to align with your sales process, such as "Qualify inbound leads and schedule meetings with promising opportunities." The clearer your goal, the better the agent will perform.

Step 3: Build the tool scenarios

Create separate scenarios for each function your agent will use (initial reply, data enrichment, CRM update, etc.). Each should be self-contained with clear inputs and outputs.

End each tool scenario with a "Return output" module that provides structured data back to the main agent. Test each scenario independently before connecting them to ensure they work as intended.

Step 4: Connect tools to the agent

In your agent's settings, add the scenarios you built as available tools. Give each one a descriptive name and explanation so the agent understands when to use it.

For example, label your initial reply scenario as "Send first response to new lead" with the description "Use when a new inquiry arrives to send our standard qualifying questions."

Step 5: Map the agent's response

Return to your main email scenario. Take the output from the agent (which will include the appropriate reply and any follow-up actions) and use it to send the response back to the lead.

Include the original thread ID to maintain conversation context. You can also add tracking parameters to monitor open and click rates on your automated emails.

Step 6: Test the flow

Send test emails from different addresses that mimic real leads. Verify that the agent responds appropriately, enriches data correctly, creates proper CRM records, and follows up when needed.

Use Make's scenario logs to review each step and identify any issues. Start with simple test cases before moving to more complex scenarios.

Prompt design tips

Your agent's effectiveness depends heavily on how you structure its prompts. Follow these best practices:

Be specific about tone and structure

Define exactly how messages should sound and what format they should follow. For example: "Write a professional reply that thanks the sender, explains our product in 2 concise sentences, and asks the 3 qualifying questions below in bullet points."

Provide examples of good responses and specify any required elements like disclaimers, compliance notices, or call-to-actions that must appear in every message.

Include fallback instructions

Tell the agent how to handle missing or unclear information. Example: "If the lead doesn't provide their company name, search for it using their email domain. If still unavailable, ask politely in the reply: 'Could you share what company you're with?'"

Build in escalation paths for when the agent encounters situations it can't handle, like complex technical questions or pricing negotiations that should go to a human.

Format for clarity

Request responses with proper formatting to improve readability. Example: "Use short paragraphs (2-3 sentences max) with line breaks between them. Include bullet points for lists of features or questions. Bold important details like meeting times."

Specify any formatting constraints for your CRM or other systems, like character limits for certain fields or required date formats.

Real-time walkthrough: how it all works together

Here's the complete sequence when a new lead comes in:

  1. Email arrives: The scenario triggers when someone emails your sales address
  2. Agent analyzes: The message goes to your AI agent which evaluates content and context
  3. Initial reply sent: Agent uses the "Send Initial Reply" tool to craft and send response
  4. Data enriched: In parallel, the "Enrich Company Info" tool gathers firmographic data
  5. CRM updated: All details are logged in your CRM with appropriate tags and scores
  6. Follow-up if needed: If key info is missing, the agent sends a polite reminder after 24-48 hours
  7. Meeting booked: For qualified leads, the agent includes a scheduling link in its reply

Customizing the workflow

The beauty of this approach is its flexibility. Here are common customizations:

  • Swap email providers: Use Outlook, Front, or IMAP instead of Gmail
  • Change enrichment services: Try Clearbit, Apollo, or other data providers
  • Connect different CRMs: Works with Salesforce, Pipedrive, HubSpot, etc.
  • Modify input sources: Trigger from web forms, chatbots, or Slack messages
  • Add routing logic: Send different lead types to appropriate teams/agents
  • Localize messaging: Customize replies based on region or language
