AI Sentiment Analysis for Product Satisfaction Surveys: A Tutorial

In this tutorial, we'll show you how to build a Make scenario that automatically performs accurate sentiment analysis for product satisfaction surveys using ChatGPT. Moreover, this solution is also available as a pre-built template.

  2. Step 2: Add the OpenAI module

  4. Step 4: Add the Airtable module

Step 1: Create a new scenario and add the Typeform module

After logging into Make, click Create a new scenario from your dashboard, which will land you on the visual canvas - the place where we'll build this solution. We'll be using Typeform to collect survey responses. Click the plus symbol at the center of the visual canvas, and search for the Typeform app.

Note that Make supports dozens of online survey tools, so feel free to use another one if you prefer. After searching for the Typeform app, select the Typeform > Watch responses module. This module uses a webhook to instantly receive responses as they are submitted, processing them in Make without delay.

Creating a new scenario in Make's visual builder

Step 2: Add the OpenAI module

Now that the Typeform module is configured, it's time to add AI analysis. Click the Add another module button next to the Typeform module and search for the OpenAI (DALL-E & ChatGPT) app. Select the Create a Completion module.

When configuring the OpenAI module, you'll need to connect your OpenAI account using an API key. The module should be set to use the gpt-3.5-turbo-0301 model, which offers the best balance of cost and performance for this use case. The key part is crafting the right prompt to analyze the survey responses.

Configuring the OpenAI module for sentiment analysis

Step 3: Add a variable module to your scenario

ChatGPT outputs the analysis as a single block of text, so we need to parse it into separate components. Add a Set variable module from the Tools section after the ChatGPT module. This will split the response into individual lines for sentiment, usefulness, and reasoning.

Configure the module to use the split function with ChatGPT's output and the newline keyword. This separates the different analysis components onto their own lines, making them easier to map in subsequent steps. The split function can be found under Text and binary functions in Make.

Pro tip: When setting up the variable module, name your variables clearly (like "sentiment_result") to make the workflow easier to understand and maintain later.

Step 4: Add the Airtable module

Now we'll store the processed data in Airtable. Add the Airtable > Create a record module to your scenario. Connect your Airtable account and select the base and table where you want to store the survey analysis.

Map the data items carefully: the original survey responses from Typeform, plus the parsed sentiment, usefulness, and reasoning from the variable module. Remember that the variable module output is numbered by line, so map each line to the appropriate field in Airtable.

Mapping parsed sentiment data to Airtable fields

Step 5: Add the final module to your scenario

As a final step, consider adding a notification module like Slack to alert your team about new survey responses. Add the Slack > Create a Message module after Airtable. Configure it to post to your preferred channel with key details from the analysis.

Once all modules are connected and configured, save your scenario and turn it on. You now have an automated AI-powered sentiment analysis system for your product surveys!
