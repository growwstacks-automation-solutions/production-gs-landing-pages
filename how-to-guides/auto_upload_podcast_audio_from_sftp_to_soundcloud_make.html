Auto Upload Podcast Audio from SFTP to SoundCloud

Are you a podcaster, producer, or musician? This use case will show you how to overcome a cumbersome part of your job: uploading files to SoundCloud.

In This Guide

  1. The problem
  2. The solution

The problem

Many podcasters and audio producers face a tedious manual process when publishing episodes to SoundCloud. After completing production work, they must download audio files from remote SFTP directories and manually upload them to SoundCloud.

This workflow typically involves two time-consuming steps: First, downloading the audio file from an SFTP server, which can be slow depending on file size and connection speeds. Second, uploading the file to SoundCloud while manually entering metadata like episode titles and descriptions.

The entire process often takes 15 minutes or more per episode, with most of that time spent waiting for file transfers rather than productive work. This inefficiency becomes particularly problematic for studios producing multiple episodes weekly.

Initial automation steps watching SFTP folder and processing files

The solution

We implemented an "Automation Sandwich" approach that combines automated processes with necessary human input. This hybrid workflow maintains efficiency while allowing for creative input where needed.

The automation sandwich begins with fully automated steps: monitoring the SFTP folder, processing new files, and storing them in Dropbox. The middle "filling" involves human input for metadata via a Slack-integrated form. The workflow concludes with automated uploading to SoundCloud.

Pro tip: Always include error handling steps in your automation to notify teams of any SFTP connection issues or file processing failures.

Step 1 of the SoundCloud automation sandwich

The first automation module watches a designated SFTP folder for new podcast audio files. When a new file appears, the system immediately begins processing it through several steps:

First, it cleans and formats the filename using regular expressions to remove special characters and standardize naming conventions. The file then gets temporarily stored in Dropbox, creating a cloud-based copy that's accessible to the next stages of the workflow.

Slack notification prompting for podcast metadata input

Step 2 of the SoundCloud automation sandwich

The system sends a Slack message to the production team containing a pre-filled form link. This form collects essential podcast metadata including episode title, description, guest information, and topic details.

The form serves as the human "filling" in our automation sandwich. Team members can review the automatically generated filename and make adjustments if needed. All form fields related to file locations are pre-populated using query parameters to prevent manual entry errors.

Custom form for collecting podcast episode details

Step 3 of the SoundCloud automation sandwich

When the form is submitted, it triggers the final automated steps. The system retrieves the audio file from Dropbox and uploads it to SoundCloud along with all collected metadata.

A confirmation message is sent back to Slack, notifying the team that the episode is live on SoundCloud. This completes the automation sandwich - starting with automation, inserting human input, and concluding with more automation.
