Why Alternative Data Matters for Company Scoring

Traditional datasets answer basic firmographic questions like company size, location, and industry. While useful, these static attributes tell you little about what's happening inside an organization right now.

Alternative data provides real-time signals about operational momentum:

  • Is the company hiring aggressively?
  • Are they investing in leadership roles?
  • What technologies are they adopting?
  • Are there significant news events?

Real-World Use Cases: Who It Is For

This scoring system benefits multiple business functions:

Investment Teams: Screen for high-momentum companies before financials reflect growth. Identify turnaround candidates through hiring spikes.

Corporate Development: Prioritize acquisition targets showing operational strength. Avoid companies with leadership churn.

Pro tip: Combine hiring signals with technology adoption data to identify companies modernizing their tech stack - often a precursor to scaling.

What Makes a Scoring System "Investment-Grade"

Effective scoring systems share four key attributes:

Time-based: Focuses on recent activity (last 30-90 days) rather than static snapshots. Momentum matters more than absolute size.

Explainable: Every score component traces back to specific signals. No black-box machine learning.

Workflow Architecture Overview

The system follows a clear pattern:

  1. Pull company list from Google Sheets
  2. Process each company independently
  3. Fetch alternative data signals
  4. Calculate scores
  5. Write back to same row in Sheets

This atomic processing prevents race conditions and ensures explainability.

Step 1: Set Up Your Company List in Google Sheets

Create a Google Sheet with these columns:

  • domain (primary key)
  • jobs_90d (90-day hiring)
  • jobs_30d (30-day hiring)
  • senior_roles_90d (leadership hiring)
  • score (composite score)
  • last_scored_at (timestamp)

Step 2: Create a New Make Scenario

In Make.com:

  1. Click "Create new scenario"
  2. Add "Google Sheets → Get rows" as first module
  3. Select your company list sheet
  4. Set pagination to return all rows

Step 3: Iterate Through Companies

Add "Tools → Iterator" to process companies one by one:

  • Map rows from Google Sheets
  • Set "domain" as the iterator value
  • All subsequent steps run per company
Sample hiring data output from PredictLeads

Step 4: Count Job Openings (90 Days)

Add "PredictLeads → List Job Openings":

  • Domain = current company domain
  • first_seen_at_from = now - 90 days
  • Returns one bundle per job opening

Use "Array Aggregator" to count total openings, then "Set variable" to store as jobs_90d.

Step 5: Identify Senior Hiring Activity

Repeat the 90-day query but filter for leadership roles:

  • Add seniority filter: manager, director, VP, executive
  • Count results as senior_roles_90d
  • Leadership hiring often precedes growth initiatives

Step 6: Add Short-Term Momentum (Optional)

Repeat the process for 30-day hiring:

  • Same filters as 90-day query
  • Set first_seen_at_from = now - 30 days
  • Store as jobs_30d
  • Helps identify accelerating hiring
Final scoring output in Google Sheets

Step 7: Update Google Sheets

At scenario end, add "Google Sheets → Update row":

  • Match rows by domain
  • Update all metrics in one operation
  • Include jobs_90d, jobs_30d, senior_roles_90d
  • Set last_scored_at timestamp
