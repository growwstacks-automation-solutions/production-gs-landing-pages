Can You Tweet From Notion? Yes, and Here's How to Do It [Tutorial]

Learn how to schedule and post tweets automatically from a Notion database with this step-by-step Integromat tutorial.

Step 1: Configuring Notion

The first step in automating tweets from Notion is setting up your database correctly. You'll need to create a dedicated database in Notion that will serve as your tweet scheduler. This database requires specific properties to track tweet content, scheduling times, and posting status.

Start by creating a new database in Notion and naming it something like "Tweet Scheduler." The default Title property should be renamed to "Tweet text" - this will contain the actual content of your tweets. Next, add a Date property called "Scheduled for" to specify when each tweet should be posted.

Example of Notion database setup for tweet scheduling

Two additional properties are crucial for the automation to work properly. Add a "Last edited time" property so Make can detect when new tweets are added or modified. Finally, create a Checkbox property called "Tweeted" that will be automatically checked when a tweet is successfully posted, preventing duplicate posts.

Pro tip: Consider adding a "Ready" checkbox property to mark tweets as ready for scheduling. This gives you more control over when drafts become active in your automation.

Step 2: Creating the first Make scenario

With your Notion database prepared, it's time to build the automation in Make (formerly Integromat). The first scenario will monitor your Notion database for new or updated entries and transfer them to a temporary storage system for scheduling.

Begin by creating a new scenario in Make and adding the Notion "Watch database items" module. This module needs to be configured to connect to your Notion account and select the specific database you created. Set it to trigger on changes to the "By updated time" property and adjust the retrieval limit to handle multiple scheduled tweets efficiently.

Creating a new scenario in Make dashboard

The next component is a Data Store module that will temporarily hold tweets until their scheduled posting time. Configure this module to create a new data store called "Scheduled tweets" with properties matching your Notion database fields. Importantly, set it to overwrite existing records to handle updates to scheduled tweets.

Between the Notion and Data Store modules, add a filter to ensure only ready-to-post tweets are processed. This filter should check that the "Tweeted" checkbox is false and the "Ready" checkbox is true (if you added this property). This prevents duplicate posting and gives you control over which tweets enter the scheduling queue.

Filter setup to prevent duplicate tweets

Step 3: Creating the second Make scenario

The second scenario handles the actual posting of tweets at their scheduled times. Create a new scenario in Make starting with a "Search Records" module that queries your Data Store for tweets where the "Scheduled for" time has passed.

Configure this module to find records where the scheduled time is "Earlier than or equal to" the current time. This ensures tweets are posted at or slightly after their scheduled time, accounting for any delays in the automation's checking frequency.

Make scenario setup for executing scheduled tweets

Add the Twitter "Create a Tweet" module next, connecting it to your Twitter account. Map the tweet content from your Data Store record to this module. For safety during testing, consider using a throwaway Twitter account before switching to your main account.

After successfully posting a tweet, the automation should update the original Notion database to mark the tweet as posted (checking the "Tweeted" box) and remove it from the Data Store. This prevents the same tweet from being posted multiple times.

Pro tip: Add a character count filter before the Twitter module to ensure your tweets don't exceed Twitter's limits. This prevents failed posts due to length restrictions.

Finally, schedule how often this scenario runs. Every 30 minutes is a good balance between timely posting and efficient use of your Make task allocation. Test thoroughly with different scheduling times and tweet states before relying on the automation for important posts.
