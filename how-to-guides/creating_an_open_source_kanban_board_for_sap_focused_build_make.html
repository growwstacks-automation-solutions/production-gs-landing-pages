Creating an Open Source Kanban Board for SAP - Focused Build

Project Overview

SAP Focused Build provides robust project tracking capabilities but lacks visual workflow management tools that many agile teams rely on. This creates a gap between SAP's structured environment and the flexible Kanban approach preferred by development teams.

The solution connects SAP Focused Build with Taiga, an open-source project management platform featuring Kanban boards. This integration gives teams the best of both worlds: SAP's traceability combined with Taiga's visual workflow management.

Integration Setup

The integration uses Make (formerly Integromat) to bridge SAP Focused Build and Taiga. Make listens for SAP requests via webhooks, processes the data, and syncs it to Taiga as Kanban cards. This creates a real-time connection between the two systems.

The setup involves configuring webhooks in SAP, creating API connections to Taiga, and mapping SAP fields to Taiga card properties. The entire process is handled visually in Make without requiring any coding knowledge.

Make scenario flow connecting SAP Focused Build to Taiga

Webhook Configuration

SAP Focused Build triggers webhooks whenever work items are created or updated. These webhooks contain all relevant work item data in JSON format. Make captures these webhooks and parses the JSON to extract key information.

The webhook setup requires administrative access to SAP Focused Build to configure the outgoing notifications. You'll need to specify which events should trigger notifications and define the payload structure.

Data Transformation

Make transforms the SAP data into Taiga's expected format. This includes mapping SAP statuses to Taiga columns, converting assignee information, and formatting any custom fields. The JSON module in Make handles this transformation.

You can customize exactly which SAP fields appear on the Taiga cards. Common mappings include work item ID, title, description, priority, and due dates. The transformation logic is completely flexible to match your team's workflow.

Taiga Configuration

Taiga requires initial setup before receiving data from SAP. This includes creating a project, defining the Kanban board structure, and configuring user permissions. The Taiga API then allows Make to create and update cards programmatically.

The Kanban columns in Taiga should mirror your team's workflow stages. Common setups include columns for Backlog, To Do, In Progress, Review, and Done. You can customize these based on your development process.

SAP work items displayed as cards in Taiga's Kanban interface

Card Customization

Taiga allows extensive customization of Kanban cards. You can configure which fields appear on cards, color coding by priority or type, and card sizing based on work item complexity. This visual customization helps teams quickly assess work status.

The integration maintains a link back to the original SAP work item, allowing team members to jump directly to SAP for detailed information when needed. This preserves SAP as the system of record while providing Taiga for daily workflow management.

Pro tip: Use Taiga's tagging system to categorize work items by project, team, or work type. This makes filtering and reporting much easier when dealing with multiple concurrent projects.

Key Benefits

This integration provides several significant advantages over using SAP Focused Build alone or purchasing commercial Kanban solutions:

Visual Workflow Management

The Kanban board gives teams an immediate visual representation of work status across all projects. Bottlenecks become obvious when columns become overloaded, allowing for quick rebalancing of resources.

Unlike SAP's list-based interface, the Kanban board shows work distribution at a glance. This is particularly valuable for Scrum Masters and team leads managing multiple concurrent initiatives.

Real-Time Synchronization

Changes in either system sync automatically to the other. Updates in SAP appear on Taiga cards within seconds, and status changes in Taiga flow back to SAP. This eliminates duplicate data entry and ensures both systems stay current.

The real-time connection means teams can work in Taiga while maintaining all the compliance and traceability benefits of SAP Focused Build behind the scenes.

Cost Savings

Taiga is completely free and open-source, unlike commercial Kanban solutions that charge per user. The Make integration replaces expensive plugins or custom development that would otherwise be needed to add Kanban to SAP.

For organizations already using SAP Focused Build, this solution provides Kanban capabilities without additional software licensing costs beyond the Make subscription.
