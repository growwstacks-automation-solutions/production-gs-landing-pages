How the Google Calendar-Outlook Calendar integration works

Syncing Google Calendar and Outlook Calendar automatically solves a common problem for professionals who need to maintain events across both platforms. While it sounds straightforward, neither platform offers built-in functionality for this specific integration.

The automation we'll build uses Make to monitor your Google Calendar for new, updated, or deleted events. When changes are detected, Make will mirror those changes in your Outlook Calendar. The key to making this work reliably is using unique event IDs to track corresponding events between the two platforms.

Example of an Outlook Calendar event ID that will be used for synchronization

Pro tip: Before starting, ensure you have admin access to both calendars and that API connections are enabled if using work/school accounts.

1. Create a new Make scenario that watches for new Google Calendar events

Begin by logging into your Make account and creating a new scenario. Add the Google Calendar > Watch Events module as your trigger. This module will monitor your specified Google Calendar for any changes to events.

Configure the Watch Events module with these settings:

  • Select your target Google Calendar from the dropdown
  • Set Watch Events to "By Updated Date" to catch both new and modified events
  • Enable "Show deleted events" to sync deletions
  • Set an appropriate Limit based on your typical event volume
Configuration settings for the Google Calendar Watch Events module

2. Add a module to create new Outlook Calendar events for new Google Calendar ones

Next, add the Microsoft 365 Calendar > Create an Event module. This will handle creating new events in Outlook whenever they're added to Google Calendar. Connect this module to your Microsoft account and select the target Outlook Calendar.

Map the relevant fields from your Google Calendar event to the Outlook event fields:

  • Summary → Subject
  • Start → Start Date
  • End → End Date
  • Description → Body Content
  • Location → Location
Mapping Google Calendar fields to Outlook Calendar event creation

3. Add a module to update the original Google Calendar event with its corresponding Outlook Calendar event's ID

After creating the Outlook event, we need to store its unique ID back in the original Google Calendar event. Add a Google Calendar > Update an Event module connected to the same calendar.

Configure this module to:

  • Select the same Google Calendar
  • Map the original event ID
  • Add a custom function to the Description field that preserves existing content while appending the Outlook event ID
Updating Google Calendar event with Outlook event ID

4. Add a text parser module to identify a Google Calendar event's corresponding Outlook Calendar event ID

Insert a Text parser > Match pattern module between your Watch Events and Create Event modules. This will search for existing Outlook event IDs in Google Calendar event descriptions.

Configure it with:

  • Pattern: \(Outlook Calendar Event ID: (.*)\)
  • Set to continue execution even if no match is found
  • Map the Google Calendar event Description as the text to parse
Text parser configuration for extracting Outlook event IDs

5. Add a module to match existing Outlook Calendar events with updated Google Calendar events

Add a Microsoft 365 Calendar > Update an Event module connected via a router. This will handle updates to existing Outlook events when their Google counterparts change.

Configure it to:

  • Use the parsed Outlook event ID from the text parser
  • Map all relevant fields from the Google event to the Outlook event
  • Update all fields that might change (title, times, description, etc.)
Connecting the Update Event module via router

6. Add a module to delete Outlook Calendar events when they're deleted in Google Calendar

Add a Microsoft 365 Calendar > Delete an Event module on a separate route from the router. This will remove Outlook events when their Google counterparts are deleted.

Configure it to:

  • Use the parsed Outlook event ID
  • Only trigger when the Google event status is "cancelled"
Setting up event deletion handling

7. Add route filters that determine whether to create a new Outlook Calendar event or update or delete an existing one

Add filters to each route from the router to control the workflow:

  1. Create route: Only proceed if no Outlook event ID exists
  2. Update route: Only proceed if an Outlook ID exists AND the event isn't cancelled
  3. Delete route: Only proceed if an Outlook ID exists AND the event is cancelled
Final scenario structure with all routes and filters

8. Turn on and run your scenario to start syncing your Google and Outlook calendars

Before activating your scenario:

  1. Choose whether to sync historical events (set a start date) or only new events ("From now on")
  2. Set your preferred schedule frequency
  3. Run a test to verify everything works
  4. Turn on the scenario

Pro tip: For initial sync of existing events, temporarily increase your Watch Events limit to process more events per run.

Common questions about syncing Google Calendar with Outlook Calendar

Many professionals need to maintain calendars across both platforms when their organization uses Outlook but they personally prefer Google Calendar. This integration ensures events stay synchronized without manual updates.

This is particularly valuable for people who work in organizations that mandate Outlook but prefer Google Calendar's interface and features for their personal scheduling. The automation maintains consistency while respecting your preferred tools.

The tutorial focuses on syncing from Google to Outlook, but similar principles can be applied to create a two-way sync by reversing the module connections and mappings.

For true bidirectional sync, you would need to create a separate scenario that watches Outlook Calendar and updates Google Calendar, using similar ID tracking techniques. The two scenarios can run independently to maintain synchronization in both directions.

The sync frequency depends on your Make scenario schedule settings. You can set it to run as frequently as every 15 minutes or just a few times per day.

More frequent syncs provide near real-time updates but consume more operations in your Make plan. For most users, syncing every 30-60 minutes provides a good balance between timeliness and resource usage.

You can sync historical events by setting the "Since specific date" option in the Watch Events module to a date before your earliest event.

For large calendars with many historical events, you may need to run the scenario multiple times or temporarily increase the event limit setting to process all events efficiently.

The automation will detect deleted events (when Show deleted events is set to Yes) and remove the corresponding Outlook Calendar event.

This maintains consistency between both calendars. The deletion is handled through the dedicated delete route in your scenario, ensuring proper cleanup of cancelled events.
