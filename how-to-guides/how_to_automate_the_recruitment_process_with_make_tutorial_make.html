The recruitment automation process

In today's competitive hiring landscape, automating recruitment processes can give companies a significant advantage. This tutorial demonstrates how to streamline hiring using Make (formerly Integromat) combined with familiar apps like Typeform, Airtable, and Google Meet.

The solution we'll build handles the complete recruitment lifecycle automatically:

  • Processing incoming applications from candidates
  • Scheduling interviews with selected applicants
  • Notifying both candidates and interviewers about upcoming meetings
  • Managing acceptance, rejection, or second interview decisions

This automation eliminates repetitive administrative tasks while ensuring consistent communication with candidates. HR teams can focus on evaluating applicants rather than manual coordination.

Pro tip: Start with automating one recruitment stage at a time. Begin with application processing, then add interview scheduling, before implementing the complete workflow.

Scenario 1: Automatically process job applications

The first automation scenario connects Typeform (for application collection) with Airtable (for candidate management). When a candidate submits an application through your Typeform, their information automatically flows into your Airtable recruitment database.

Typeform provides an intuitive, mobile-friendly way for candidates to apply. You can include:

  • Job role selection dropdowns
  • File upload fields for resumes/CVs
  • Work experience descriptions
  • Contact information collection
Example Typeform application form with job role selection

Airtable serves as your recruitment command center, storing all candidate information with status tracking. The Make automation creates a new record for each applicant, setting their initial status to "Application Submitted".

CV upload field in the recruitment form

The Make scenario requires just two modules:

  1. Typeform's "Watch Responses" trigger
  2. Airtable's "Create a Record" action

Configuration involves mapping each Typeform field to corresponding Airtable columns. The status field automatically sets to "Application Submitted" for new candidates.

Work experience question in the application form

For the Airtable base, create these essential columns:

  • Candidate Name
  • Job Role
  • Status (dropdown with application stages)
  • Email
  • Experience
  • CV (attachment field)
  • Interview Date
  • Interviewer
Contact information section of the application form

The second table in Airtable ("Interviewers") contains your hiring team details:

  • Interviewer Name
  • Email
  • Department
  • Available Time Slots
Creating a new scenario in Make

Once configured, this automation runs instantly whenever a new application arrives. Recruiters see candidate information appear in Airtable in real-time, ready for review.

Pro tip: Add a "Last Modified" field in Airtable to track when each application was updated. This helps manage your recruitment pipeline effectively.
