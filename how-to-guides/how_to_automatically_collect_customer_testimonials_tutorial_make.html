Tracking testimonials on Capterra: Solution overview

Testimonials are a valuable asset for companies, as they reflect the experiences of customers who interacted with your products or services. These first-hand experiences provide validation of your offerings and valuable insights to improve your products. Furthermore, this feedback helps potential customers make better decisions before purchasing.

The most popular software review platforms include Capterra, G2, Trustpilot, and TrustRadius. Monitoring testimonials allows you to:

  • Use positive testimonials in marketing materials
  • Respond promptly to customer feedback
  • Identify product or service issues quickly

In this tutorial, we'll show you how to use Make to:

  • Automatically monitor a company profile on Capterra
  • Store new testimonials in an Airtable database
  • Receive Slack notifications for new reviews
The Make.com dashboard interface with the create scenario button highlighted

Pro tip: While we're focusing on Capterra in this guide, the same principles apply to other review platforms. The main difference will be the specific API endpoints and authentication methods required.

Step 1: Creating the Make scenario and adding the HTTP app

Begin by creating a new scenario in your Make dashboard. Click the "Create a new scenario" button which will take you to the scenario builder interface. This is where you'll construct your automation workflow.

Start by adding the HTTP app to your scenario. Search for "HTTP" in the app selection menu and choose the "Make a request" module. This module will handle the API calls to Capterra's review endpoint.

The Make scenario builder interface with HTTP app selected

You'll need to configure the HTTP module with Capterra's API endpoint, which you can find in their documentation. Be sure to enable the "Parse response" option so Make can properly handle the returned data. The endpoint will return up to 50 reviews at a time, which we'll process in subsequent steps.

Step 2: Adding the first Airtable module

After setting up the HTTP module, we need to process the returned reviews. Since the API returns an array of testimonials, we'll use Make's Iterator tool to handle each review individually. Add the Iterator from the Tools menu and connect it to the HTTP module.

Configure the Iterator to process the reviews array from the HTTP response. This ensures each testimonial gets processed separately rather than trying to handle them all at once.

Configuring the HTTP module to fetch reviews from Capterra's API

Next, add the Airtable "Search records" module to check if each review already exists in your database. This prevents duplicate entries. Configure it to search using the testimonial's unique URL as the identifier.

Step 3: Adding the second Airtable module

For reviews that don't exist in your Airtable base (as determined by the search module), we'll add them using the "Create a record" module. Connect this module to the Iterator and configure it with the same base and table as your search module.

Map the relevant fields from the review data to your Airtable columns. Typical fields might include reviewer name, rating, review text, date, and the product or service being reviewed.

Mapping review data to Airtable fields in the create record module

Between the search and create modules, add a filter that only allows new reviews (those not found in the search) to proceed. Configure the filter to check if the "Total number of bundles" from the search equals zero, indicating no matching record was found.

Step 4: Sending the Slack notification

To be notified of new testimonials, we'll add Slack integration. First, use Make's Text Aggregator tool to group the new review data into a single message. This prevents getting multiple notifications if several new reviews come in at once.

Add the Slack "Create a message" module and configure it with your desired channel and message format. Include key details like the reviewer's name, rating, and a link to the full review. You can also include the review text or a summary.

Setting up the Slack notification with aggregated review information

Pro tip: Customize your Slack notification format to include the most important information at a glance. Consider adding emoji indicators for positive/negative reviews or buttons to quickly access the full review.

Step 5: Testing the scenario

Before setting your automation to run on a schedule, test it thoroughly. Use the "Run once" button in Make to execute the scenario manually and verify that:

  • Reviews are being fetched correctly from Capterra
  • Only new reviews are added to Airtable
  • Notifications are sent to Slack with the expected format
Testing the complete scenario execution in Make

Once testing is complete, set your preferred schedule for the automation. For most businesses, checking for new testimonials once per day provides a good balance between timeliness and system load.
