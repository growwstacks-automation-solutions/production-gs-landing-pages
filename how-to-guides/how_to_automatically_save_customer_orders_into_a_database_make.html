How to Automatically Save Customer Orders into a Database

Learn how to automatically export and store ecommerce orders in a database using Make.com with this step-by-step guide.

Whether you're starting an ecommerce business or bringing a legacy one into digital readiness, saving orders to a database is one of those things you wouldn't want to forget about. Automating this process provides numerous benefits that can significantly improve your operations.

There are multiple reasons why you'd want to automate order storage:

  • Instant access to every order without manual searching
  • Improved customer response times when troubleshooting order issues
  • Better understanding of business performance through demand analysis
  • Elimination of manual data entry errors
  • Always up-to-date records for reporting and analytics

Pro tip: Before starting, ensure you have a Make.com Core plan or higher, an ecommerce platform account (we're using WooCommerce), and a database solution (we're using Airtable).

Step 1: Create a new Make scenario

Begin by logging into your Make.com account and navigating to the Scenarios tab. Click the "Create a new scenario" button located in the upper right corner of the dashboard. This action opens the visual scenario builder where you'll construct your automation workflow.

In the scenario builder, search for and select the WooCommerce app. This will be the foundation of your automation, connecting your store to Make.com. The first module you'll set up will detect new orders as they come in.

The Make.com scenario builder interface where you'll create your automation

Step 2: Configuring the first WooCommerce module

Select the "New Event" module from WooCommerce's available actions. This module uses a webhook to instantly detect when new orders are placed in your store. Webhooks provide real-time notifications compared to periodic polling, ensuring no delays in your data processing.

The first configuration step is connecting your WooCommerce store to Make.com. Click "Add" and enter your store's API credentials. While naming your webhook is optional, it's good practice for organization. Most importantly, select "Order Created" as the topic to ensure the webhook captures all new orders.

Configuring the WooCommerce New Event module in Make

Step 3: Configuring the second WooCommerce module

When the webhook receives a new order, it provides an Object ID. You'll need a second WooCommerce module to retrieve the complete order details using this ID. Add a "Get an Order" module by clicking the plus sign next to your first module.

Make automatically uses the WooCommerce connection you established earlier. The only configuration needed is mapping the Object ID from the webhook to the Order ID field in this module. This connection ensures all relevant order information is fetched for processing.

Setting up the Get an Order module to retrieve complete order details

Step 4: Adding an Iterator to handle multiple items

Many orders contain multiple products that need individual database entries. Make's Iterator tool solves this by separating each line item. Add an Iterator from the Tools menu at the bottom of the scenario builder and connect it to your Get an Order module.

Configure the Iterator by mapping the "Line Items []" array from the order data to the Iterator's Array field. This tells the Iterator to process each product in the order separately while maintaining the connection to the original order through shared fields like order ID and customer information.

Configuring the Iterator to process multiple order items

Step 5: Adding the database module

The final piece is adding your database module. We're using Airtable's "Create a record" action, but you could substitute Google Sheets, MySQL, or another supported database. First, ensure your database table has columns for all the order data you want to store.

After connecting your Airtable account, select the appropriate base and table. Then map the relevant fields from your WooCommerce and Iterator modules to your database columns. Essential fields typically include customer name, email, product details, quantity, and price.

Mapping order data to Airtable database fields

Showtime: See how the scenario works

With everything configured, your automation is ready to test. The scenario will now automatically create a database entry for each product in every new order. You can watch the execution in real-time through Make's interface to verify everything works correctly.

Remember to save and activate your scenario once testing is complete. From this point forward, your database will automatically update with each new order, providing you with organized, searchable records of all customer purchases without any manual intervention.

The Iterator separates each line item from orders containing multiple products. This ensures every purchased item gets its own database entry while maintaining the connection to the original order through shared fields like order ID and customer email.

Without the Iterator, an order with five products would create one database entry with all items combined. The Iterator processes each product individually, which is essential for accurate inventory tracking, product performance analysis, and customer purchase history.

Make allows you to add data transformation steps between modules. You can use functions to reformat dates, calculate values, filter information, or add custom fields before the data reaches your database.

Common transformations include converting currency formats, splitting customer names into first/last fields, calculating discounts or taxes, and adding internal product codes. These modifications ensure your stored data matches your preferred format and business requirements.

The automation is highly reliable when properly configured. Make provides error handling options and you can set up notifications for failed operations. The webhook-based trigger ensures no orders are missed, even during system downtime.

Make.com maintains a history of all scenario executions, allowing you to review any errors and reprocess failed operations. For critical business functions, you can implement redundancy by sending order notifications to multiple systems or adding manual review steps for high-value orders.
