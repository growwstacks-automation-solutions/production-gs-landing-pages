How to Automatically Transfer Email Purchase Orders to Google Sheets and Create Invoices with Xero

Stop wasting time on manual data entry! This workflow automates the entire process of receiving email purchase orders, logging them in Google Sheets, and generating invoices in Xero.

What This Workflow Does

This powerful automation eliminates the tedious and error-prone task of manually processing purchase orders received via email. Imagine an online store receiving dozens, or even hundreds, of orders daily. Without automation, each email would require someone to open it, extract customer details, order items, quantities, and prices, then manually input this data into a spreadsheet and create an invoice in their accounting software. This workflow completely transforms that process.

It acts as a digital assistant, constantly monitoring your inbox for new purchase orders. Once an order email arrives, it intelligently reads and understands the content, pulls out all the critical information, logs it neatly into a designated Google Sheet for tracking, and then automatically generates a professional invoice in Xero. This not only saves countless hours of administrative work but also drastically reduces the risk of human error, ensuring accuracy in your sales records and financial reporting.

How It Works

This automation is built on a series of interconnected steps within Make.com, designed to handle each part of the purchase order process seamlessly.

1. Capture Email Purchase Orders

The workflow begins with a Webhooks > Custom mailhook module. This unique email address acts as a dedicated inbox for your purchase orders. You simply configure your email client (like Gmail) to automatically forward any incoming purchase order emails to this mailhook. This ensures that every new order instantly triggers the automation.

Screenshot of Gmail forwarding settings, showing how to add the custom mailhook address.

2. Filter and Parse Email Content

After receiving an email, a filter ensures that only relevant emails (e.g., those with "Order" in the subject line) proceed. The core of the data extraction is handled by the Text Parser > Match pattern module. This module uses regular expressions to identify and pull out specific pieces of information from the email body, such as customer name, product details, quantities, and prices. This intelligent parsing ensures accurate data capture.

Configuration of the Text Parser module, demonstrating how regular expressions are used to extract data from email text.
A filter configured in Make.com to process only emails containing "Order" in the subject.

3. Log Data in Google Sheets

The extracted data is then immediately sent to a Google Sheets > Add a row module. This module populates a pre-defined Google Sheet with all the order details, creating a real-time log of every purchase. This centralizes your order information, making it easy to track, analyze, and report on sales performance.

Settings for the Google Sheets module, showing the selection of the target spreadsheet and sheet.
Data mapping from the Text Parser output to the corresponding columns in Google Sheets.
Using the 'formatdate' function to ensure consistent date formatting in Google Sheets.

4. Manage Customer Contacts in Xero

A Xero > Search for contacts module checks if the customer already exists in your Xero accounting system. A router then intelligently directs the workflow: if the contact is new, a Xero > Create a contact module is triggered to add the new customer. If the contact already exists, the workflow proceeds directly to invoice creation.

Configuration of the Xero 'Search for contacts' module, searching by customer name.
A filter that checks if a Contact ID exists, routing new contacts for creation.

5. Create Invoice in Xero

Finally, a Xero > Create an invoice module generates a sales invoice. For new contacts, the Contact ID from the newly created contact is used. For existing contacts, the ID found in the search step is used. All relevant details extracted from the email are mapped to the invoice fields, ensuring a complete and accurate invoice is created automatically.

Quick Setup Guide

Follow these steps to quickly set up your email purchase order automation:

  1. Create the Scenario: In your Make.com account, create a new scenario and add the following modules: Webhooks > Custom mailhook, Text Parser > Match pattern, Google Sheets > Add a row, Router, Xero > Search for contacts, Xero > Create a contact, and Xero > Create an invoice (two instances, one for each route).
  2. Configure Mailhook: Copy the generated mailhook URL from the Webhooks module. In your email client (e.g., Gmail), go to settings, find the forwarding options, and add this mailhook URL as a forwarding address. Ensure emails with purchase orders are forwarded.
  3. Set up Text Parser: In the Text Parser module, map the 'Text' element from the mailhook. Create a regular expression to extract key data points (customer name, items, quantities, prices) from your typical purchase order email format. Enable 'Global match' and 'Multiline'.
  4. Add Email Filter: Between the mailhook and Text Parser, add a filter condition: "Subject" contains "Order" (case insensitive) to process only relevant emails.
  5. Map to Google Sheets: Connect the Text Parser to the Google Sheets 'Add a row' module. Select your target spreadsheet and sheet. Map the extracted data from the Text Parser to the corresponding columns in your Google Sheet. Use the `formatdate` function for date fields if needed.
  6. Configure Xero Contact Search: Connect the Google Sheets module to the Xero 'Search for contacts' module. Search by 'Name' using the customer name extracted by the Text Parser.
  7. Set up Router for Contacts: Add a Router after the Xero 'Search for contacts' module. Create two routes:
    • Route 1 (New Contact): Add a filter where 'Contact ID' (from Xero Search) 'Does not exist'. Connect this to a Xero 'Create a contact' module, mapping the customer name.
    • Route 2 (Existing Contact): Add a filter where 'Contact ID' (from Xero Search) 'Exists'.
  8. Create Xero Invoices: Add a Xero 'Create an invoice' module to both routes. For Route 1, map the 'Contact ID' from the 'Create a contact' module. For Route 2, map the 'Contact ID' from the 'Search for contacts' module. Map the extracted order details (description, quantity, unit amount, item) to the invoice line items.
  9. Test and Activate: Run the scenario once with a sample purchase order email to ensure all steps execute correctly and data is transferred accurately. Once confirmed, activate your scenario.

Pro tip: Regularly review your Google Sheet and Xero account after initial setup to ensure data is flowing as expected. Adjust your regular expressions in the Text Parser if your email formats change or if you need to capture additional data points.
