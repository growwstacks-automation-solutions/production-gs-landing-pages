What do you need to build a custom app in Make?

Creating custom apps in Make requires just two essential components: a Make account and access to the API you want to integrate. The platform's flexibility allows you to connect virtually any application that provides an API, significantly expanding Make's already extensive library of 1,300+ native integrations.

For this tutorial, we'll be using Geocodify's API as our example. Geocodify provides location services including geocoding (converting addresses to coordinates) and reverse geocoding. Since it's not natively integrated with Make but offers a free plan, it serves as an ideal candidate for demonstrating custom app creation.

Pro tip: Before starting, ensure you have admin privileges in your Make organization if you plan to share the custom app with team members. The creator of a custom app automatically becomes its administrator.

Step 1: Get the API Key from the Geocodify website

The first step in building our custom Geocodify app is obtaining the API key. After creating your Geocodify account, you'll land on the dashboard where your API key is prominently displayed. This key serves as your authentication token for all API requests.

Copy this key to your clipboard or save it in a secure note - we'll need it throughout the tutorial. The API key acts like a password, so treat it with the same level of security. Never share it in public forums or commit it to version control systems.

Geocodify dashboard showing API key location

Next, open Geocodify's API documentation in a new tab. This documentation is crucial as it details all available endpoints, required parameters, authentication methods, and response formats. Familiarize yourself with the base URL (https://api.geocodify.com/v2) and the /geocode endpoint we'll be using.

Step 2 (optional): Test the API using Postman

While optional, testing the API in Postman first provides several advantages. It helps verify that your API key works correctly, allows you to understand the request/response structure, and serves as a reference when configuring the custom app in Make.

To test the Geocodify API in Postman:

  • Create a new GET request
  • Set the URL to https://api.geocodify.com/v2/geocode
  • Add query parameters:
    • api_key: Your Geocodify API key
    • q: A test address (e.g., "3041 30th St, San Diego, CA")
  • Send the request and verify the response
Testing Geocodify API in Postman

Successful testing confirms that our API key is valid and we understand how to structure requests. This knowledge directly translates to how we'll configure our custom app in Make, particularly the communication settings between Make and Geocodify's API.

Step 3: Create a new custom app

Now we'll transition to Make to begin building our custom app. Navigate to your Make dashboard and locate the "My Apps" section in the left sidebar (you may need to click "More" to reveal it). This section manages all your custom app creations.

Click "Create a new app" in the top-right corner. You'll be presented with a form to define basic app properties including:

  • Label: The display name (we're using "Geocodify")
  • Description: Briefly explain the app's purpose
  • Theme: Select a color (#6f42c1 for our example)
  • App icon: Upload a 512x512px image under 100KB
Make's My Apps section where custom apps are created

These details help users identify your app in Make's module selector. Choose clear, descriptive values that communicate the app's purpose at a glance. After saving, you'll be redirected to your new app's dashboard where we'll configure the technical components.

Step 4: Create a new connection

Connections define how users authenticate with your custom app. Since Geocodify uses API key authentication, we'll configure our connection accordingly. Navigate to the "Connections" tab in your custom app dashboard and click "Create a new connection".

Name your connection thoughtfully - this will be visible to users when they authenticate. Select "API Key" as the connection type. Make will provide default templates that we'll customize for Geocodify's specific requirements.

Connection configuration form in Make

In the Parameters section, replace the default code with configuration that prompts users for their Geocodify API key. This ensures each user authenticates with their own credentials while maintaining a consistent interface. Remember to click Save after making changes.

The Communication tab defines how Make tests the connection. We'll configure it to make a sample request to Geocodify's API using the provided key, verifying that authentication works before allowing users to proceed.

Step 5: Configure the base URL and the common query parameters

The Base section establishes foundational settings applied to all modules in your custom app. Here we'll define Geocodify's base API URL (https://api.geocodify.com/v2) and common parameters like the API key that should be included in every request.

Configure the Base URL field with Geocodify's endpoint. Then set up the common query parameters to automatically include the api_key parameter with each request, using the connection's authentication value. This prevents users from having to manually add authentication to every module.

Pro tip: Use the sanitize command for sensitive data like API keys. This prevents them from appearing in logs while still allowing the connection to function properly.

The Base configuration serves as the template for all modules you'll create. Any settings defined here (like authentication) will be inherited by default, reducing repetitive configuration and ensuring consistency across your app's functionality.

Step 6: Create a new module

Modules represent specific actions your custom app can perform. We'll create a "Search" module for the /geocode endpoint we tested earlier. In your custom app dashboard, navigate to the Modules tab and click "Create a new Module".

Select "Search" as the module type since we're querying geocoding data. Name it descriptively ("geocode") and provide clear labels and descriptions that will help users understand its purpose when browsing Make's module library.

Module creation interface in Make

Configure the module's Mappable Parameters to define what inputs users must provide (in our case, an address to geocode). Then set up the Communication settings to specify exactly how Make should call the /geocode endpoint, including how to handle the response data.

These module configurations transform raw API calls into user-friendly building blocks that can be easily incorporated into Make scenarios. Well-designed modules abstract away technical complexity while maintaining flexibility.

Step 7: Test the new Custom App

With our custom Geocodify app complete, it's time to test it in a real scenario. Create a new scenario in Make and search for your Geocodify app in the module library. You should see the "geocode" module we created available for use.

Add the module to your scenario and configure the connection using your Geocodify API key. Then provide a test address as input and run the scenario. Verify that the module returns the expected geocoding data in the correct format.

Testing the custom Geocodify app in a Make scenario

Successful testing confirms your custom app works as intended. You can now share it with team members or publish it for broader use. Make provides options to control visibility and permissions, allowing you to manage who can use or modify your custom app.
