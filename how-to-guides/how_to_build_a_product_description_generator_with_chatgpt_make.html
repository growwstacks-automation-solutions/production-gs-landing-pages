Generating product descriptions automatically with Make and ChatGPT

Anyone who's run an ecommerce business knows how time-consuming writing product descriptions can be. Each product and variant requires unique, compelling copy that converts visitors while maintaining SEO value. Traditionally, this meant either spending hours writing manually or outsourcing to expensive copywriters.

With the combination of Make and ChatGPT, you can automate this entire process. The AI generates high-quality, SEO-optimized descriptions in seconds, while Make handles the workflow automation between your ecommerce platform and OpenAI. The result is a system that produces better descriptions than most human writers at a fraction of the cost.

Creating a new scenario in Make's visual automation builder

Step 1: Create the Make scenario and add the first WooCommerce module

Begin by creating a new scenario in Make (formerly Integromat). Click the "Create a new scenario" button in the top right corner of your dashboard. This opens the visual workflow builder where you'll construct your automation.

Add the "WooCommerce > Search for a Product" module as your first step. This module retrieves products from your store based on specific criteria you define. Configure it to connect to your WooCommerce store by entering your store URL and API credentials.

Pro tip: Create a dedicated WooCommerce API key for Make with read/write permissions to keep your automation secure while maintaining necessary access.

Step 2: Add the OpenAI module

Connect the "OpenAI > Create a Completion" module to your WooCommerce module. This is where the AI magic happens. You'll need an OpenAI API key - this requires a paid account as free tiers don't provide API access.

Configure the module to use the "Create a Chat Completion" method with the "gpt-3.5-turbo" model. While more advanced models exist, this provides excellent results at lower cost. In the Messages section, craft your prompt carefully to generate the type of descriptions you want.

Configuring the OpenAI module with a detailed prompt

Step 3: Add the Markdown module

The OpenAI module outputs descriptions in Markdown format, but WooCommerce requires HTML. Add Make's native "Markdown > Markdown To HTML" module to handle this conversion automatically.

This step ensures your product descriptions maintain proper formatting with paragraphs, lists, and other HTML elements. The module requires no special configuration - simply map the Markdown content from the OpenAI module to the input field.

Step 4: Add the final WooCommerce module

Complete your workflow by adding a "WooCommerce > Update a Product" module. This takes the HTML-formatted description from the previous step and updates the corresponding product in your store.

Map the Product ID from your initial search module to ensure each description gets applied to the correct product. You can configure the scenario to run on a schedule (daily, weekly) or trigger it manually when adding new products.
