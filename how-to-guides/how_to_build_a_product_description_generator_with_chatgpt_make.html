Generating product descriptions automatically with Make and ChatGPT

Ecommerce businesses know the pain of writing unique product descriptions for every item in their catalog. Many resort to outsourcing this time-consuming task, sacrificing both quality and budget. With Make and ChatGPT, you can automate this process while maintaining high-quality, SEO-friendly content.

The solution we'll build retrieves products from your store, generates professional descriptions using AI, converts them to proper HTML formatting, and updates your product listings automatically. You'll need:

  • A paid OpenAI account (API access required)
  • A Make account (Core plan or higher)
  • An ecommerce platform (we use WooCommerce but others work too)

Pro tip: Create a dedicated "Update Description" tag in WooCommerce to mark products needing refreshed content without affecting your entire catalog.

Step 1: Create the Make scenario and add the first WooCommerce module

Begin by creating a new scenario in Make. Click the "Create a new scenario" button in your dashboard to access the visual workflow builder. This is where we'll construct our automation from modular components.

Add the "WooCommerce > Search for a Product" module as your starting point. This module will retrieve products matching your criteria from your online store. Configure it by first establishing a connection to your WooCommerce installation using your store's API credentials.

Creating a new scenario in Make's visual workflow builder

Set filters to identify which products need descriptions. You might filter by category, price range, or a custom tag you create specifically for this purpose. The tag approach works well for updating existing products while allowing new products to be processed automatically.

Step 2: Add the OpenAI module

Connect the "OpenAI > Create a Completion" module to process each product. This is where ChatGPT generates your descriptions. Configure it by selecting "Create a Chat Completion" method and the "gpt-3.5-turbo" model (or newer if available).

The key to quality output is crafting an effective prompt. Use this structure:

You are an ecommerce specialist writing SEO-friendly product descriptions. Describe [Product Name] with these features: [Key Features]. Use a [Tone] tone, highlight [Unique Selling Points], include [Keywords], and limit to [Word Count]. Format with clear paragraphs and bullet points where appropriate.

Map product data from WooCommerce to the prompt variables. Set temperature to 0.7 for balanced creativity, and max tokens to 300-400 for adequate length without verbosity.

Configuring the OpenAI module with product-specific prompts

Step 3: Add the Markdown module

ChatGPT outputs descriptions in Markdown format, but WooCommerce requires HTML. Add Make's native "Markdown > Markdown To HTML" module to convert the AI-generated content.

Simply connect the OpenAI module's output to the Markdown module's input. No additional configuration is needed - Make handles the conversion automatically while preserving your formatting like headings, lists, and emphasis.

This step ensures your product pages display properly formatted content with correct paragraph spacing, bullet points, and other HTML elements that enhance readability.

Converting markdown to HTML for proper ecommerce display

Step 4: Add the final WooCommerce module

Complete the workflow by adding a "WooCommerce > Update a Product" module. This pushes your AI-generated descriptions back to your store.

Configure it to:

  • Match products by ID from the initial search
  • Map the HTML content from the Markdown module
  • Optionally preserve other product fields

Set the scenario to run on a schedule (daily works well) or trigger manually when you add new products. Test with a few products first to refine your prompt and verify output quality before scaling to your entire catalog.
