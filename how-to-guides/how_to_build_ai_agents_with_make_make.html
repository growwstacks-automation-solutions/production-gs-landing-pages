What are AI agents in Make?

Make AI Agents are goal-driven automations powered by large language models (LLMs). Unlike traditional scenarios that follow fixed paths, agents receive tasks, analyze them using AI reasoning, select appropriate tools from those you've provided, and return structured responses.

The key difference lies in flexibility. With regular automation, you define every step in advance. With agents, you define the goal and available tools, then let the AI determine how to accomplish it based on the incoming request.

Pro tip: Think of agents as virtual employees. You train them by defining their role (description) and tools (scenarios), then let them figure out how to handle each specific request.

Example: shop inventory agent

To illustrate how agents work, let's build a Shop Inventory Assistant. This agent will help manage stock by answering questions like "Do we have blue hoodies?" and handling requests like "Order 10 more coffee bags."

Our agent will need three core capabilities:

  • Check current inventory levels
  • Place orders for new stock
  • Communicate through Slack
The Make AI Agents interface where you define your agent's behavior and connect tools

Step 1: Create your AI agent

Creating your agent is the foundation of the automation. This is where you define what the agent does and connect it to an AI model for reasoning capabilities.

1. Open the AI Agents section

In your Make dashboard, navigate to the AI Agents section from the left sidebar. Click "Create agent" to begin setting up your new automation assistant.

2. Name and describe the agent

Give your agent a clear, descriptive name like "Inventory Assistant." Then write a detailed description of its role and limitations. For our example:

"You help manage store inventory. You can check stock levels or place restock orders using the tools provided. Only respond based on tool outputs and available data. Do not order more than 100 units of any product at once."

3. Connect a language model

Select your preferred LLM provider (like OpenAI) and enter your API key. This gives your agent the ability to understand requests and determine appropriate actions.

4. Save the agent

Click "Create agent" to save your setup. Your agent is now ready to receive tools, which we'll add in the next step.

Step 2: Add tools (scenarios)

Tools are the capabilities your agent can use to accomplish tasks. In Make, each tool is a separate scenario that performs a specific function.

1. Create your tool scenarios

For our inventory agent, we'll create two main tools:

Tool 1: List Inventory - This scenario connects to your inventory system (like Google Sheets or Airtable), accepts optional product name filters, retrieves stock data, and returns it using the Return output module.

Tool 2: Order Stock - This scenario creates purchase orders or sends restock requests to your suppliers. It should accept product names and quantities, process the order, and return a confirmation message.

Pro tip: Each tool scenario must end with a Return output module (found under Scenarios tools). This sends structured data back to the agent for processing into a final response.

2. Add tools to your agent

Return to your agent's settings and navigate to the Agent-level tools section. Click "Add" and select the scenarios you created. The agent will now consider these when determining how to handle requests.

Selecting scenarios to add as tools for your AI agent

Step 3: Connect the agent to inputs and outputs

With your agent and tools ready, you need a way to send it requests and receive responses. We'll use Slack as our interface in this example.

1. Set up the Slack trigger

Create a new scenario with the Slack "Watch messages" module. Configure it to monitor the channel where users will interact with your agent. Add filters to ignore messages from your bot to prevent loops.

2. Add the Run an agent module

After the trigger, add the "Run an agent" module from Make's AI Agents section. Select your Inventory Assistant and map the Slack message text to the agent's Message field.

3. Reply in Slack

Add the Slack "Create a message" module to send the agent's response back to the user. Map the agent's Response field to the message content and use the original message's Thread Timestamp to keep conversations organized.

4. Use a custom Slack app (optional)

For better control over your agent's identity in Slack, connect using a custom app. This lets you set the bot's name, avatar, and other details that help users recognize it.

Step 4: Test and refine the agent

Thorough testing ensures your agent behaves as expected before deploying it to users.

1. Try simple requests

Start with straightforward queries like "What's our hoodie inventory?" Check that the agent selects the List Inventory tool and returns accurate stock data.

2. Test edge cases

Try less direct prompts like "We're running low on espresso" to ensure the agent understands the intent to check stock levels rather than immediately placing orders.

3. Check thread behavior

Verify that responses stay in the correct Slack threads, especially when handling multiple simultaneous conversations.

4. Review the logs

Examine scenario run histories to see which tools were used, what inputs they received, and what outputs they returned. This helps identify where refinements are needed.

5. Adjust as needed

Based on test results, refine your agent's description or tool configurations. You might need to clarify instructions, add example interactions, or adjust tool descriptions.

Best practices

Follow these guidelines to create effective, reliable AI agents:

Keep the scope focused

Start with a specific task like inventory management. Once that works well, you can expand to related functions like supplier communication or sales reporting.

Write clear descriptions

Your agent's description guides its decisions. Be explicit about what it should do, which tools to use, and how to behave. Include limitations like "Only use the tools provided" or "Ask for clarification if requests are unclear."

Use descriptive tool names

Name tools based on how users might phrase requests. "List Inventory" is clearer than "Get Info" because the agent matches requests to tool names.

Always return results

Every tool scenario must end with a Return output module. Without this, the agent can't generate helpful responses based on the tool's actions.

Start with essential tools

Begin with just the tools your agent absolutely needs. Too many options can confuse its decision-making. You can always add more capabilities later.
