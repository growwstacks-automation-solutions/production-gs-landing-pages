How It Works

This automation orchestrates a series of actions across multiple applications to create a fully intelligent project intake system. Here's a step-by-step breakdown:

1. Watch for New Project Requests (JotForm)

The workflow begins with a trigger in JotForm. Whenever a new project request form is submitted, Make.com instantly captures all the essential details. This includes the project name, description, business goals, deadlines, priority levels, and involved stakeholders. This initial data collection forms the foundation for all subsequent automated steps.

JotForm module set to watch for new project submissions.

2. Analyze with AI Intelligence (Make AI Toolkit)

Once the form data is received, Make.com sends these project details to the Make AI Toolkit module. The AI then performs an intelligent review of the submission. It generates critical insights such as potential risks, identifies any missing information that needs clarification, and highlights key assumptions that should be confirmed before the project proceeds. This AI-powered analysis transforms a raw request into an informed starting point, proactively identifying potential issues.

Make AI Toolkit module processing project data for insights.

3. Create the Notion Project Brief

Next, Make.com automatically generates a comprehensive project brief in Notion. This brief combines the original data from the JotForm submission with the AI-generated insights. Notion becomes the central reference hub for the project, ensuring that all relevant information, including risks and assumptions, is organized and accessible in one place for the team's review and planning.

Notion module automatically generating a project brief.

4. Set Up Project Tracking in Asana

Simultaneously, a new task is created in the team's Asana project review board. This task includes all essential project details and a direct link back to the newly created Notion brief, providing immediate context. This step ensures that the project is tracked within your existing project management system, allowing for seamless integration into your team's workflow.

Asana module creating a new task for project review.

5. Notify the Team in Slack

To keep everyone in the loop, the main team Slack channel receives an instant alert. This message contains key information such as the project title, the requester's name, the deadline, the project's priority, and direct links to both the Asana task and the Notion brief. This ensures that all relevant stakeholders are immediately informed about the new project and can access all necessary documentation.

Slack module notifying the team about a new project request.

6. Confirm with the Requester (Slack)

Finally, the person who submitted the project request receives a personalized Slack message confirming that their submission has been received and is now under review. This provides immediate feedback to the requester, assuring them that their request is being processed and keeping them informed about the status of their submission.

Slack module sending a confirmation to the project requester.

Quick Setup Guide

Setting up this powerful AI-powered project intake automation is straightforward. Follow these steps to get your workflow running:

  1. Create Your JotForm: Design a JotForm with all the necessary fields for project requests (e.g., Project Name, Description, Deadline, Priority, Requester Email, etc.).
  2. Set up Make.com Scenario: Log in to your Make.com account and create a new scenario.
  3. Add JotForm Watch Module: Select JotForm as the first module and configure it to "Watch New Submissions" from your newly created form.
  4. Integrate Make AI Toolkit: Add the Make AI Toolkit module. Map the relevant fields from your JotForm submission (e.g., Project Name, Description) to the AI module, prompting it to analyze for risks, missing info, and assumptions.
  5. Configure Notion Module: Add a Notion module to "Create a Database Item." Map the original JotForm data and the AI-generated insights to create a comprehensive project brief in your Notion database.
  6. Set up Asana Task: Add an Asana module to "Create a Task." Populate the task details with information from JotForm and Notion, including a link to the Notion brief. Assign it to the appropriate project review board.
  7. Add Slack Notification Modules: Include two Slack modules. One to "Send a Message" to your team's main project channel with key project details and links. Another to "Send a Direct Message" to the requester, confirming receipt and status.
  8. Test and Activate: Run a test submission through your JotForm to ensure all modules execute correctly and data flows as expected. Once verified, activate your Make.com scenario.
