The last business glossary your company will ever need

Corporate jargon is ubiquitous in the business world. From "synergy" and "paradigm shift" to countless acronyms, these terms can leave employees confused and disconnected from their work. However, company buzzwords also serve as shorthand for complex ideas, effectively streamlining communication when everyone understands them.

The challenge lies in providing this knowledge to employees without requiring them to ask colleagues, which many avoid due to fear of appearing uninformed. Traditional static glossaries often go unused because they're inconvenient to access and maintain.

The common workplace dilemma this automation solves

This automated solution solves these problems by combining:

  • Slack for convenient access through a familiar interface
  • Google Sheets for maintaining internal company terminology
  • ChatGPT for general business term definitions
  • Make.com to orchestrate the workflow between these services

Pro tip: Create a dedicated Slack user like "Mr. Glossary" that employees can DM privately. This maintains privacy while making the glossary easily accessible.

Step 1: Create the scenario and add the first Slack module

Begin by creating a new scenario in Make.com. Search for the Slack app in the scenario builder and select the "New Event" module. This webhook-based module allows processing multiple requests simultaneously, which is crucial for supporting many employees querying the glossary at once.

Configure the module by clicking "Add" to create a new webhook, then select "New IM Message" as the event type. Leave the channel field blank to receive messages from all users. Connect your Slack account and save the configuration.

Configuring the Slack New Event module in Make

Step 2: Create the spreadsheet with definitions

For storing internal company terms, create a Google Sheets spreadsheet with two columns: "Business Terms" (column A) and "Definitions" (column B). Populate this with your organization's specific jargon, acronyms, and terminology.

Back in Make, add a "Search rows" module from Google Sheets after the Slack trigger. Connect your Google account, select your glossary spreadsheet, and configure the filter to use "Text operators: Contains (case insensitive)" on the Business Terms column.

Setting up the Google Sheets search functionality

Step 3: Add a router for term lookup

The scenario needs to handle two paths: when a term is found in your glossary (Google Sheets) and when it's not. Add a Router tool from Make's tools menu to create these separate paths.

The router will check the output from the Google Sheets search. If the term is found (outputs 1 bundle), it will follow the top route to return the internal definition. If not found (0 bundles), it will follow the bottom route to query ChatGPT.

Router configuration for handling found/not found terms

Step 4: Complete the top route (Term Found)

For the top route (term found in Google Sheets), add a Slack "Create a Message" module. Configure it to reply to the original requester with the definition from your spreadsheet. Map the user field from the initial Slack trigger to ensure the reply goes to the right person.

Add a filter between the router and Slack module that only allows bundles where the Google Sheets search returned results (Total number of bundles > 0). This ensures the path only activates for terms in your glossary.

Configuring the Slack reply for terms found in Google Sheets

Step 5: Complete the bottom route (Term Not Found)

For the bottom route (term not found), first add a Slack module that informs the user their term wasn't in the internal glossary but ChatGPT is being consulted. Include the word "Oops!" in this message - we'll use this later to prevent loops.

Add a filter between the router and this Slack module that only allows bundles when the Google Sheets search returned no results (Total number of bundles = 0). This ensures the path only activates for terms not in your glossary.

Configuring the Slack reply for terms not found internally

Step 6: Integrate ChatGPT

After the "term not found" Slack message, add OpenAI's "Create a Chat Completion" module. Configure it to use GPT-4 and set the role to "System". Add the user's original query as the message content.

Finally, add another Slack "Create a Message" module to send ChatGPT's response back to the user. Include "ChatGPT Definition" in this message - we'll use this text to prevent reply loops in the next step.

Setting up the ChatGPT integration for undefined terms

Step 7: Add filter to prevent loops

The final critical step is adding a filter after the initial Slack trigger that prevents the scenario from retriggering on its own replies. Click the connection between the Slack trigger and Google Sheets search, then create a filter named "Definition requests only".

Configure it to ignore messages containing "Oops!" or "Definition" - the keywords we included in our reply messages. This ensures the scenario only processes original employee queries, not its own automated replies.

Pro tip: Test your scenario thoroughly by querying terms both in and not in your glossary. Verify replies come through private DMs and that the system doesn't get stuck in loops.
