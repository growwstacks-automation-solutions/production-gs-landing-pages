How to Delete Google Drive Files Automatically: Two Methods

Learn how to automatically clean up old Google Drive files using Make to free up storage space and maintain an organized drive

In This Guide

  1. Method 1: Automatically delete old Google Drive files
  2. Method 2: Automatically send old Google Drive files to Trash

Managing Google Drive storage can become challenging as files accumulate over time. Without regular cleanup, you may find yourself hitting Google's storage limits or struggling to locate important files among outdated ones. This guide presents two automated solutions using Make (formerly Integromat) to maintain an organized and efficient Google Drive.

Pro tip: Before automating deletions, consider backing up important files or using Google's native "Storage manager" tool to review large files manually first.

Method 1: Automatically delete old Google Drive files

This direct approach permanently removes files that meet your specified age criteria. It's ideal when you're confident about deleting certain types of old files and want to immediately reclaim storage space.

1. Set up the Make scenario and the first Google Drive module

Begin by creating a new scenario in Make and adding the "Google Drive > Search for Files/Folders" module. Connect it to your Google account and configure the search parameters to identify files based on their last modified date.

Key settings include selecting your target Drive (like "My Drive"), specifying folders to search (or leaving blank for entire Drive), and setting the custom query to find files older than your chosen timeframe (like 6 months). The query syntax is critical - use single straight quotes and the correct date function format.

Configuring the Google Drive search module in Make

2. Delete your old Google Drive files

Add the "Google Drive > Delete a File/Folder" module to your scenario. After connecting it to your Google account, map the File ID from the previous search module to this deletion module. This creates the automation link between finding and deleting matching files.

While the module specifies "File" type, it will still process folders if they appear in the search results. The deletion is permanent, so ensure your search criteria are accurate before running the scenario in production.

Setting up the deletion module in Make

3. Test and activate your scenario

Before running the full automation, test it carefully to avoid accidental deletions. Run the search module alone first to review the files it identifies. Then modify your query to target just one test file you're certain can be deleted.

After successful testing, remove the single-file restriction and schedule the scenario to run automatically at your preferred interval (like monthly). Make allows you to set this schedule directly in the scenario settings.

Pro tip: Consider running the scenario manually the first few times to monitor its behavior before enabling full automation.

Method 2: Automatically send old Google Drive files to Trash

This safer alternative moves files to Google Drive's Trash instead of deleting them immediately, giving you a 30-day recovery window. It also includes notifications about moved files.

1. Create a new Make scenario and add the first Google Drive module

Set up this scenario similarly to Method 1, using the same "Google Drive > Search for Files/Folders" module with your desired age criteria. The initial configuration remains identical - the difference comes in how we handle the found files.

Initial search module setup for the Trash method

2. Add the second module to move your old Google Drive files to Trash

Instead of a deletion module, add "Google Drive > Move a File/Folder to Trash". Connect it to your Google account and map the File ID from the search module. This creates the automated movement of old files to Trash rather than permanent deletion.

Google will automatically empty the Trash after 30 days, but until then, you can restore any files if needed. This provides a valuable safety net compared to immediate deletion.

Configuring the move to Trash module

3. Compile a list of the Google Drive files that have been moved to Trash

Add a "Text aggregator" module to create a readable list of all files moved to Trash. Configure it to pull filenames from the move operation and format them clearly, with each file on a new line.

This list serves as your audit trail and will be used for notifications. Set the aggregator to stop processing if no files are found to avoid empty notifications.

Setting up the text aggregator for file listings

4. Add a filter in between the two Google Drive modules

Insert a filter between the search and move modules to ensure the scenario only proceeds when files are actually found. Configure it to check that the File ID exists from the search results.

This prevents unnecessary operations and potential errors when no old files are found. It's particularly important for the notification steps that follow.

Creating a filter to check for found files

5. Send yourself the list of the Google Drive files that have been moved to Trash

Add a notification module (like Slack or email) to receive the aggregated list of moved files. This creates a record of what was cleaned up and allows you to spot-check if any files were moved unexpectedly.

Configure the notification with clear wording about these files being moved to Trash (not permanently deleted yet). Include the date and any other relevant context in the message.

Setting up notifications for moved files
