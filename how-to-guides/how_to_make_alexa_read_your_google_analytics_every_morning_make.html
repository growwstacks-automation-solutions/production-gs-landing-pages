This innovative workflow empowers you to start your day with critical business insights, delivered directly by your Alexa device. Instead of manually checking dashboards or reports, you can have your Google Analytics data read aloud as part of your morning flash briefing. This automation connects Google Analytics, processes the data, and makes it accessible to Alexa, providing a seamless and hands-free way to stay informed about your website's performance.

Beyond just Google Analytics, the underlying method is highly versatile. It can be adapted to broadcast information from various sources like Reddit, RSS feeds, or any API, transforming how you consume daily updates. How It Works

Step 1: Fetch Data from Google Analytics

The process begins in Make.com, where a scheduled scenario is set up to connect to your Google Analytics account. This module is configured to retrieve specific metrics, such as daily website sessions, from the previous day. You have the flexibility to choose any metric that is most relevant to your business needs, ensuring you get the most impactful data in your briefing.

Step 2: Structure Data into JSON Format

Once the data is fetched, the workflow uses a "Tools - Set variable" module to transform this raw data into a structured JSON (JavaScript Object Notation) format. This is a critical step because Alexa's Flash Briefing skill requires data to be presented in a specific JSON structure. The JSON includes a unique identifier, the update date, a title for the briefing, and the main text that Alexa will read, dynamically embedding your Google Analytics metrics.

Configuring the 'Set variable' module to format data into JSON.

Step 3: Upload JSON to Dropbox

The formatted JSON file is then uploaded to a designated folder in your Dropbox account using the "Dropbox - Upload a file" module. This step is crucial for making the data accessible to Alexa. The workflow is configured to overwrite the existing file daily, ensuring your flash briefing always contains the most up-to-date information. After the file is uploaded, a public direct-download link is created for this JSON file.

Dropbox module settings for uploading the JSON file.

Step 4: Configure Alexa Flash Briefing Skill

In the Alexa Developer Console, you create a new Flash Briefing skill. This skill is configured to use the public direct-download link from Dropbox as its data source. You'll specify a preamble (what Alexa says before the data), a name for the briefing, and set the content type to "text" with a daily update frequency. This tells Alexa where to find your daily website performance summary.

Creating a new skill in the Alexa Developer Console.

Step 5: Test and Enjoy Your Daily Briefing

Finally, you enable the newly created skill for testing within the Alexa Developer Console. Once enabled, you can open your Alexa app or speak to your Alexa device, saying "Alexa, play news" or "Alexa, what's my flash briefing?" to hear your customized Google Analytics report. This completes the loop, providing you with an effortless, audible summary of your website's performance every morning.

Quick Setup Guide

Setting up this workflow is straightforward. Follow these steps to get your daily Google Analytics briefing from Alexa:

  1. Create a Make.com Scenario: Log into your Make.com account and create a new scenario. Add a "Google Analytics - Get a report" module and connect it to your Google Analytics account. Select the specific metric (e.g., sessions, page views) you want Alexa to report for the previous day.
  2. Format Data for Alexa: Add a "Tools - Set variable" module. In the variable value field, paste the provided JSON template and replace "HERE-YOUR-VARIABLE" with the output from your Google Analytics module (e.g., the 'sessions' variable).
  3. Upload to Dropbox: Add a "Dropbox - Upload a file" module. Connect your Dropbox account, specify a folder, and set the file name (e.g., myAnalyticsBriefing.json). Crucially, enable the "Overwrite an existing file" option in advanced settings.
  4. Make File Public: Go to your Dropbox account, find the uploaded JSON file, and create a shareable link. Change the dl=0 parameter in the URL to dl=1 to ensure it's a direct download link. Test this link in your browser to confirm it downloads the file directly.
  5. Configure Alexa Skill: Log into the Alexa Developer Console. Create a new skill, choosing "Flash Briefing" as the model. Add a new feed, provide a preamble (e.g., "From Google Analytics"), a name (e.g., "My Daily Analytics"), set content type to "text" and frequency to "Daily". Paste your direct Dropbox JSON link into the "Feed" field.
  6. Enable Testing: In the Alexa Developer Console, navigate to the "Test" tab and change the skill status dropdown to "Development" to enable the skill for your account.
  7. Enjoy Your Briefing: Open your Alexa App or speak to your Alexa device, saying "Alexa, play news" or "Alexa, what's my flash briefing?" to hear your personalized Google Analytics update!
