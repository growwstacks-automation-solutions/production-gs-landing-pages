How to Send New WooCommerce Orders to Shopify Using Make

Scenario Summary

One of the most common challenges in ecommerce is integrating WooCommerce and Shopify. Many store owners use multiple platforms to sell products but prefer Shopify's superior order management capabilities. Existing solutions often fall short - they're either too expensive or lack necessary features.

This guide provides a comprehensive solution using Make (formerly Integromat) to automatically transfer multi-product orders from WooCommerce to Shopify. Certified Make Partner Andy O'Neil walks through each step to create this advanced automation.

Prerequisites

  • Make account (free tier available)
  • Active Shopify store
  • WooCommerce installation
  • Access to order data in both platforms

Module Overview

The integration uses eight key Make modules to transform and transfer order data:

1. WooCommerce » Watch Orders [Trigger]

This module initiates the scenario whenever a new order is placed in WooCommerce. You can configure filters to only process paid orders or specific order types.

2. WooCommerce » Get an Order [Action]

Retrieves complete order details including all line items. This ensures you have access to the full product data needed to recreate the order in Shopify.

3. Flow » Iterator [Action]

Processes the line items array from WooCommerce, separating bundled products into individual items for Shopify. If an order contains 5 products, it creates 5 separate data bundles.

4. JSON » Parse JSON [Action]

Structures the individual line items from the iterator into a format that can be transformed for Shopify's requirements.

5. JSON » Aggregate to JSON [Action]

Transforms WooCommerce product data into Shopify's specific JSON structure, including required fields like product_id, variant_id, quantity, and price.

6. JSON » Parse JSON [Action]

Processes the newly formatted Shopify JSON to prepare it for order creation.

7. Flow » Array Aggregator [Action]

Combines all line items back into a single array that Shopify can process as a complete order.

8. Shopify » Create an Order [Action]

The final module that creates the order in Shopify with all transferred products and customer information.

Pro tip: Before building the full scenario, test each module individually using real order data to ensure proper field mapping between platforms.

Building the Scenario

Now let's dive into the step-by-step process of creating this automation in Make. Follow each section carefully to ensure proper data transformation between platforms.

1. Setting Up the WooCommerce Trigger

The foundation of your automation begins with the WooCommerce "Watch Orders" module. This acts as your scenario trigger, initiating the workflow whenever a new order is placed.

When configuring this module:

  • Connect your WooCommerce store using API credentials
  • Set the trigger to "Order Created"
  • Consider adding filters for order status (e.g., only process "Completed" or "Processing" orders)
  • Set an appropriate polling interval based on your order volume

2. Retrieving Complete Order Details

While the trigger captures basic order information, you'll need the "Get an Order" module to retrieve all necessary details:

  1. Add the WooCommerce "Get an Order" module after your trigger
  2. Map the Order ID from the trigger module to this module's Order ID field
  3. Configure to retrieve all available fields including line items, customer data, and shipping information

3. Processing Line Items with Iterator

WooCommerce provides line items as a complex array structure. The Iterator module breaks this down:

  • Add a Flow Control Iterator module
  • Set the source array to the line items from your "Get an Order" module
  • Configure to output individual bundles for each product

This transformation is crucial because Shopify requires each line item to be processed individually before being combined into the final order.

4. Parsing WooCommerce Product Data

The first JSON Parse module structures the raw WooCommerce product data:

  1. Add a JSON Parse module after the Iterator
  2. Map the "Value" output from the Iterator to the JSON string field
  3. No need to define a schema - we'll handle that in the transformation step

5. Transforming Data for Shopify

The most complex part involves reformatting the data for Shopify's API requirements:

  1. Add an "Aggregate to JSON" module
  2. Set source module to your JSON Parse module
  3. Create a new data structure named "ShopifyLineItem"
  4. Use the generator with this sample JSON structure:
{
  "product_id": "",
  "variant_id": "",
  "title": "",
  "quantity": "",
  "price": "",
  "sku": "",
  "tax_lines": [{
    "title": "",
    "price": "",
    "rate": ""
  }],
  "requires_shipping": ""
}

6. Preparing the Shopify Payload

After transforming the data, parse it again to ensure proper formatting:

  • Add another JSON Parse module
  • Map the output from your Aggregate module
  • This creates clean, validated data ready for Shopify

7. Aggregating Line Items

Before creating the Shopify order, combine all line items:

  1. Add an Array Aggregator module
  2. Set source module to your first WooCommerce trigger
  3. Select all fields from the JSON Parse module to include
  4. This creates the complete line items array Shopify expects

8. Creating the Shopify Order

The final step implements the order creation in Shopify:

  1. Add a Shopify "Create Order" module
  2. Connect your Shopify store using API credentials
  3. Enable the "map" option for line items
  4. Map the array from your aggregator to Shopify's line items field
  5. Include customer details, shipping information, and any order notes

Pro tip: Run your scenario in test mode first with a real WooCommerce order to verify all data transfers correctly before activating the automation.
