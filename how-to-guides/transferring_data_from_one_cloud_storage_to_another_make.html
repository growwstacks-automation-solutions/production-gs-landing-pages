Configuring the scenario

This scenario is designed to automatically monitor your Dropbox for new file uploads and seamlessly transfer them to Google Drive, while intelligently avoiding duplicate files. The automation ensures your files are synchronized between cloud storage platforms without manual intervention.

The process begins with the Dropbox "Watch Uploads" trigger module, which detects when new files are added to your specified folder. When triggered, the scenario downloads the file using the "Get a File" module before checking Google Drive for existing duplicates.

Configuration interface for the Dropbox Watch Uploads trigger module

The Google Drive "Search for files/folders" module plays a crucial role in preventing duplicates. It searches your specified Google Drive folder for files matching the name of the newly uploaded Dropbox file. The module uses the file name from the Dropbox trigger as the search query parameter.

Pro tip: For optimal performance, ensure your Google Drive search folder is properly indexed by keeping the folder structure organized and avoiding overly complex naming conventions.

A filter is implemented between the search and upload steps to ensure only new files are transferred. The filter checks if the Google Drive search returned any results - if no matching files are found (indicating the file is new), the scenario proceeds to upload the file to Google Drive.

Filter configuration that prevents duplicate file uploads

The final step uses the Google Drive "Upload a file" module to transfer the file from Dropbox to the designated Google Drive folder. The module connects to the previously downloaded file from Dropbox and places it in your specified target location.

Scenario execution 1:

When a new file is uploaded to Dropbox, the scenario successfully retrieves it and searches Google Drive. Finding no matching file, the filter allows the transfer to proceed, resulting in the file being uploaded to Google Drive.

Execution path when transferring a new file successfully

Scenario execution 2:

If the same file is manually selected for reprocessing (perhaps during testing), the scenario will detect that the file already exists in Google Drive. The filter prevents the upload module from executing, effectively stopping duplicate files from being created.

Execution halted when detecting a duplicate file
