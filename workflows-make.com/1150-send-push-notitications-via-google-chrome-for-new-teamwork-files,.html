What This Workflow Does

This automation solves the common problem of team members missing important file updates in Teamwork projects. Instead of requiring constant manual checks or relying on easily overlooked email notifications, it delivers instant Chrome push notifications directly to team members' desktops whenever new files are added to monitored projects.

The workflow monitors specified Teamwork projects in real-time, triggering alerts the moment any team member uploads documents, images, spreadsheets, or other files. Each notification includes key details like the file name, project name, and upload timestamp, helping recipients quickly assess the importance of each update.

How It Works

1. Teamwork Project Monitoring

The workflow continuously watches designated Teamwork projects for new file uploads using Make.com's Teamwork integration. You can configure it to monitor specific projects or all projects in your account.

2. Instant Event Detection

When Teamwork records a new file upload, Make.com immediately captures the event details including file name, type, size, uploader information, and project context.

3. Notification Processing

The system formats a clear notification message with all relevant file details and prepares it for delivery through Chrome's push notification system.

4. Desktop Alert Delivery

Chrome displays the notification on recipients' desktops, even when they're not actively using Teamwork or their browser. Notifications remain visible until dismissed.
