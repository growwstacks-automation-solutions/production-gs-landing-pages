What This Workflow Does

If your business uses Smith.ai for call handling, you know the value of having a professional receptionist service. But manually transferring call data—caller details, duration, purpose, and outcomes—into spreadsheets for analysis is a tedious, error-prone task that wastes valuable time. This automation solves that problem by creating a seamless bridge between Smith.ai and Google Sheets.

The workflow automatically detects new calls in your Smith.ai account and creates corresponding rows in your Google Sheets spreadsheet. Each row contains structured data like timestamp, caller information, call duration, status, and any notes from the virtual receptionist. This transforms raw call logs into organized data ready for reporting, analysis, and strategic decision-making.

By implementing this automation, you eliminate the gap between call reception and data analysis. Your team gets immediate access to call metrics without manual intervention, enabling faster response times, better customer insights, and more accurate performance tracking for your sales and support teams.

How It Works

This Make.com scenario acts as an intelligent data pipeline, moving information from Smith.ai to Google Sheets on a scheduled basis.

Step 1: Scheduled Trigger

The workflow begins with a Schedule module set to run automatically every 15 minutes. This acts as the heartbeat of the automation, regularly checking Smith.ai for new call activity without any manual prompting.

Step 2: Fetch New Calls from Smith.ai

When triggered, the scenario connects to your Smith.ai account via API and retrieves all calls that have occurred since the last check. It filters for new calls based on timestamps, ensuring no duplicate entries are created in your spreadsheet.

Step 3: Data Mapping and Transformation

Each call's data is extracted and mapped to specific spreadsheet columns. The automation can handle various data points including caller name, phone number, call duration (in seconds), call status (answered, missed, voicemail), start/end times, and any custom notes or tags added by the Smith.ai receptionist.

Step 4: Create Rows in Google Sheets

The formatted call data is then sent to your designated Google Sheets spreadsheet where it creates a new row for each call. The automation appends data to the next available row, maintaining a clean, chronological log of all call activity.

Pro tip: Before activating the automation, set up your Google Sheets spreadsheet with clear column headers that match the data points you want to capture (e.g., Date, Time, Caller Name, Phone, Duration, Status, Notes). This ensures clean, organized data from day one.

  2. Connect Smith.ai: In the first module, authenticate your Smith.ai account by entering your API credentials when prompted.
