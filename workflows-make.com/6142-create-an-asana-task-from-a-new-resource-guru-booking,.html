What This Workflow Does

For project managers and operations teams, one of the biggest time sinks is manually transferring information between systems. When you book a team member on a project in Resource Guru, you then have to go into Asana (or another task manager) to create the corresponding task, assign it, set a due date, and add details. This disconnect between resource scheduling and task execution leads to missed deadlines, double-booked team members, and a lack of real-time visibility.

This free Make.com template solves that problem by creating a seamless, automated bridge between Resource Guru and Asana. Every time a new booking is created for a project, this workflow triggers instantly. It captures all the critical booking details—like the project name, assigned person, dates, and notes—and uses them to automatically generate a fully-formed task in the correct Asana project. This turns a scheduling action into an immediate, actionable work item, ensuring nothing falls through the cracks.

The business value is clear: you reclaim hours each week previously spent on manual data entry, eliminate human error in task creation, and create a single, synchronized source of truth for who is doing what and when. Your team can focus on executing the work, not administrating it.

How It Works

The automation follows a clean, logical sequence to transform a booking into a task.

Step 1: Watch for New Bookings

The workflow starts with a Make.com "Watch Resource" module monitoring your Resource Guru account for new bookings. The instant a booking is created for a project, client, or specific resource, Make captures the event and all its associated data.

Step 2: Map Booking Data to Task Fields

Make then processes the incoming data. It extracts key pieces of information: the booking name becomes the task title, the start/end dates inform the task due date, the assigned resource's name or email is identified, and any booking notes are prepared for the task description.

Pro tip: Use this mapping stage to add custom logic. For example, you can set the task priority in Asana based on the project type in Resource Guru, or automatically tag tasks with a specific label.

Step 3: Create the Task in Asana

Finally, the workflow uses the Asana module within Make to create a new task. It places the task in the pre-defined Asana project that corresponds to the work, assigns it to the correct team member by matching their email, populates the description, and sets the due date. The result is a complete, assigned task that appears in your team's Asana workspace moments after the booking was made.

What You'll Need

To implement this template, you'll need just a few things set up in advance:

  1. A paid Make.com account with available operations.
  2. An active Resource Guru subscription with admin or API access.
  3. An active Asana account with projects already set up.
  4. The email addresses of team members must be consistent across both Resource Guru and Asana for smooth assignment.
  5. Decide which Asana project(s) will receive the automated tasks.

Quick Setup Guide

You can have this automation running in under 30 minutes.

  1. Get the Template: Click the "Get This Workflow" button above to clone the template into your Make.com account.
  2. Connect Your Apps: In the Make scenario, authorize the connections to your specific Resource Guru and Asana accounts.
