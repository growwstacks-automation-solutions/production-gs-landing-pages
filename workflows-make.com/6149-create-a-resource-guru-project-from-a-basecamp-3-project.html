What This Workflow Does

For agencies and professional service teams, the gap between winning a project and effectively scheduling your team to deliver it is where profit leaks and stress builds. A project gets the green light in Basecamp, but then someone must manually duplicate all that information into Resource Guru—a process that takes 15-30 minutes, is prone to errors, and often gets deprioritized. This delay means your most valuable assets—your people—aren't formally allocated, leading to overbooking, last-minute scrambles, and inaccurate forecasting.

This automation solves that by creating an instant, seamless bridge. The moment a project is created or marked as 'Active' in Basecamp 3, this workflow automatically generates a corresponding project in Resource Guru. It maps key fields like project name, client, dates, and manager, creating a ready-made container for detailed resource scheduling. This turns a multi-step, error-prone administrative task into a zero-touch background process, ensuring your resource plan is always a real-time reflection of your active project portfolio.

Pro tip: Use this automation to enforce a "no schedule, no start" policy. Configure it to only create the Resource Guru project when a Basecamp project reaches a specific status (like "Approved"). This guarantees resource planning is a formal gate before work commences.

How It Works

The workflow is a straightforward, reliable sequence that moves data on your behalf.

1. Monitor for New Basecamp Projects

The scenario watches your specified Basecamp 3 account for new projects or for updates to existing projects (like a status change to "Active"). It triggers instantly, ensuring no lag between project creation and resource setup.

2. Extract and Structure Key Project Data

It pulls the essential information needed for scheduling: the project name, description, start and due dates, the associated client/company from Basecamp, and the project manager. The automation can be tailored to include custom fields like project code, budget ID, or priority level.

3. Create the Corresponding Resource Guru Project

Using the structured data, the scenario logs into your Resource Guru account and creates a new project. It populates the project name, client, dates, and notes, effectively mirroring the Basecamp project shell. The new project is immediately visible to your resource manager for detailed team assignment.

4. (Optional) Notify the Team

The workflow can be extended to send a notification to a Slack channel, Microsoft Teams, or via email to the project manager and resource planner, confirming the project is now "schedulable." This closes the communication loop automatically.

Quick Setup Guide

You can have this automation running in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" to copy the ready-made scenario into your Make.com account.
  2. Connect Your Apps: In the Make scenario, authorize the connections to your specific Basecamp 3 and Resource Guru accounts. This is a one-time, secure OAuth process.
  3. Configure the Trigger: Point the Basecamp module to the specific Basecamp account and choose whether to watch for "New Project" or "Updated Project" events.
  4. Map the Data Fields: Review the data mapping between Basecamp and Resource Guru. Adjust if your field names differ (e.g., map "Company" in Basecamp to "Client" in Resource Guru).
  5. Test with a Dummy Project: Create a test project in Basecamp (e.g., "TEST - Automation Setup") and run the scenario once to verify it creates the project correctly in Resource Guru.
  6. Activate & Monitor: Turn the scenario on. It will now run automatically. Check the scenario history for the first few executions to ensure smooth operation.
