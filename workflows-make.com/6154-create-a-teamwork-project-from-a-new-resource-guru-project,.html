Automatically Create Teamwork Projects from Resource Guru

What This Workflow Does

For service-based businesses, agencies, and internal IT teams, the gap between planning and execution is where projects stumble. You schedule your team's time in a resource planning tool like Resource Guru, but then your project managers must manually recreate that plan in a project management tool like Teamwork. This double handling is not just tedious—it introduces errors, causes delays, and creates version control nightmares.

This automated workflow solves that by creating a seamless bridge between planning and doing. The moment a new project is created and staffed in Resource Guru, this Make.com scenario triggers. It captures all the essential project details—name, client, dates, assigned personnel—and uses them to automatically generate a fully-fledged project in Teamwork. This ensures your project delivery team has an actionable workspace ready to go, perfectly aligned with the resource plan, from the very first day.

The business value is clear: faster project mobilization, eliminated administrative waste, and guaranteed consistency between what was sold (and scheduled) and what is being delivered. It turns a multi-step, error-prone process into a reliable, hands-off operation.

How It Works

The automation acts as a intelligent connector, translating a resource booking into a project blueprint.

Step 1: Monitor New Resource Guru Projects

The workflow starts by watching for new projects in your Resource Guru account. Using Make.com's instant watch feature or scheduled checks, it detects the moment a project is created, capturing its unique ID and all associated data.

Step 2: Map and Prepare Project Data

Make.com then processes the incoming data. It extracts the core information needed for Teamwork: the project title, start and end dates, client name, and any custom fields or notes. This step often involves formatting dates and text to match Teamwork's expected structure.

Pro tip: Use this mapping step to enrich the data. You can append the client name to the project title or set the project owner in Teamwork based on the primary resource assigned in Resource Guru.

Step 3: Create the Teamwork Project

With the data prepared, the workflow makes an API call to Teamwork to create a new project. It passes all the mapped details, setting up the project with the correct name, dates, and privacy settings. You can configure it to use a specific project template in Teamwork for consistency.

Step 4: Add Team Members and Initial Tasks

Optionally, the automation can go further. It can add the team members assigned in Resource Guru to the new Teamwork project and even create initial task lists or milestones based on the project's scope or type, making it truly "ready-to-work."

What You'll Need

  1. A Make.com account (Free or paid plan).
  2. An active Resource Guru account with API access enabled (typically available in paid plans).
  3. An active Teamwork account with administrator or owner permissions to create projects via API.

  5. A clear understanding of which project fields in Resource Guru (e.g., custom fields for client ID) should map to which fields in Teamwork.

Quick Setup Guide

Follow these steps to deploy this automation in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect Your Apps: In the Make.com scenario, authorize the connections to your Resource Guru and Teamwork accounts using OAuth or API keys.
  3. Configure the Trigger: Set the Resource Guru module to watch the specific "Projects" resource or the correct portfolio/account.
  4. Map the Data Fields: Review the data mapping between the "Create a project" module in Teamwork and the data coming from Resource Guru. Adjust field mappings to fit your project naming conventions.
  5. Test with a Dummy Project: Create a test project in Resource Guru and run the scenario once to verify a project is created correctly in Teamwork.
  6. Activate & Monitor: Turn on the scenario schedule and monitor the operations log for the first few projects to ensure smooth running.
