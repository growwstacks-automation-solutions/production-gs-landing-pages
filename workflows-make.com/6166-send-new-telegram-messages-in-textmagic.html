What This Workflow Does

In today's fast-paced business environment, communication happens across multiple platforms. Your sales leads might come into a Telegram group, system alerts might ping a monitoring bot, and urgent support requests might arrive in a private chat. The problem is that critical information can get buried if the right person isn't actively checking Telegram at that moment. This delay can mean a lost sale, a prolonged system outage, or a frustrated customer.

This free Make.com workflow template solves that by creating an automatic bridge between Telegram and TextMagic's SMS service. Every time a new message arrives in a specified Telegram chat or channel, the workflow triggers and sends that message content directly to a designated mobile number via SMS. It transforms a potentially slow, app-dependent notification into a universal, immediate mobile alert that anyone can receive, ensuring time-sensitive information never waits.

How It Works

The automation runs on a simple but powerful "when this, then that" logic, orchestrated by Make.com's visual automation platform.

Step 1: Monitor Telegram for New Messages

The workflow starts with a Telegram module set to "Watch Messages" in a specific chat, group, or channel you configure. It acts as a silent sentinel, instantly detecting new text messages as they arrive.

Step 2: Process and Format the Message

Make.com captures the message content, along with useful metadata like the sender's name and timestamp. You can add filters here to only proceed if the message contains certain keywords, comes from a specific user, or meets other criteria you define, preventing unnecessary SMS alerts.

Pro tip: Use filters to forward only messages tagged as #urgent or containing words like "help," "down," or "order." This saves SMS costs and prevents alert fatigue for your team.

Step 3: Send SMS via TextMagic

The core action: the formatted message is passed to TextMagic's "Send SMS" module. You configure this with your TextMagic API credentials and the recipient's phone number. Within seconds, the Telegram message is delivered as a standard SMS to the target mobile device.

Step 4: Handle Errors and Logs

The workflow includes built-in error handling. If the SMS fails to send (e.g., due to an invalid number or API issue), Make.com can route the error to a log, send an alert to an admin, or even retry the operation, ensuring robustness.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps.

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Telegram: In the first module, authorize Make.com to access your Telegram account and select the specific chat you want to monitor.
  3. Connect TextMagic: In the SMS module, add your TextMagic API credentials (found in your TextMagic account settings).
  4. Set the Recipient: Enter the destination phone number for the SMS alerts, using the international format (e.g., +1234567890).
  5. Test and Activate: Send a test message to your monitored Telegram chat. Within moments, you should receive it as an SMS. Once confirmed, turn the scenario "ON" to run continuously.
