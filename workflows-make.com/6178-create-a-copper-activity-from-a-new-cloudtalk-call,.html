What This Workflow Does

For sales teams using CloudTalk for phone calls and Copper for CRM, a critical gap often exists: manual call logging. After a productive conversation, reps must switch contexts, open Copper, find the contact, and manually type notes, duration, and outcome. This process is not only time-consuming but prone to errors and omissions, leading to incomplete customer histories and missed follow-up opportunities.

This automation bridges that gap seamlessly. It acts as a digital assistant that listens for completed calls in CloudTalk and instantly creates a corresponding, detailed activity in the correct contact's record within Copper. The result is a perfectly synchronized system where your CRM always reflects the latest customer interaction, without any manual effort from your team.

How It Works

The workflow is a straightforward, reliable pipeline that moves data from your telephony system to your CRM the moment a call ends.

Step 1: Trigger on New CloudTalk Call

The automation is initiated by the "Call Finished" event in CloudTalk. Make.com watches for this trigger, capturing the call's payload which includes essential data like the caller's phone number, call duration, timestamps, direction (inbound/outbound), and any notes or tags added by the agent during the call.

Step 2: Find the Matching Contact in Copper

Using the phone number or contact details from the call, the workflow searches your Copper database for the corresponding Person record. This step is crucial—it ensures the activity is logged against the right contact. The template assumes the contact already exists, creating a clean, linked activity history.

Step 3: Create a Detailed Activity in Copper

Finally, the automation creates a new "Activity" in the found Copper person's record. It populates fields like the activity type (e.g., "Call"), a descriptive note summarizing the call, the date and time, duration, and any custom fields you wish to map. This creates a complete audit trail directly within the sales pipeline.

Pro tip: Use CloudTalk's tagging feature during calls (e.g., "Interested", "Demo Scheduled", "Callback Needed"). The automation can pass these tags to Copper, allowing you to trigger further automated actions based on call outcome, like sending a follow-up email or creating a task.

Quick Setup Guide

You can have this automation running in under 15 minutes.

  1. Clone the template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect your apps: Authorize Make.com to access both your CloudTalk and Copper accounts using API keys or OAuth.
  3. Map the data: Review the module settings to ensure key fields like phone number and call notes are correctly mapped from CloudTalk to the Copper activity fields.
  4. Test with a real call: Make a test call in CloudTalk, let it finish, and verify that a new activity appears in the corresponding Copper contact record.
  5. Activate: Turn the scenario on. It will now run automatically for every completed call.
