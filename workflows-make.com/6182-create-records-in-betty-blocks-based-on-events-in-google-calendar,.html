What This Workflow Does

Businesses that rely on meetings—client consultations, project syncs, team stand-ups—often struggle to capture the valuable data generated in those interactions. Manually transferring details from Google Calendar into a database like Betty Blocks is tedious, error-prone, and creates data silos. This disconnect means you can't easily report on meeting trends, automate follow-ups, or connect calendar activity to other business processes.

This automation solves that by creating a live bridge between Google Calendar and Betty Blocks. Whenever a new event is created or starts in your calendar, this workflow automatically captures all relevant details—who, what, when—and creates a corresponding, structured record in your Betty Blocks application. This transforms your calendar from a simple scheduling tool into a powerful data source that feeds your custom business software.

How It Works

The workflow runs on a schedule (default: hourly) to check for new activity, ensuring your Betty Blocks database is always up-to-date.

Step 1: Monitor Google Calendar for New Events

The automation is triggered by the "Watch Events" module in Make.com, which polls your specified Google Calendar for new events that have started. It captures the event in real-time as it begins.

Step 2: Extract and Structure the Event Data

The workflow parses the raw calendar event and extracts the key properties needed for your database: the event creator, the list of attendees, the start and end times, the meeting summary/title, and its status (e.g., "confirmed").

Step 3: Create a Record in Betty Blocks

Using the Betty Blocks API module, the workflow takes the structured data and creates a new record in your specified Betty Blocks data model. Each piece of calendar information is mapped to its corresponding property field in Betty Blocks, building a complete, queryable record of the meeting.

Pro tip: Use the "Status" field from Google Calendar to filter records in Betty Blocks. For instance, you could build a view that only shows records for "cancelled" meetings to analyze scheduling conflicts.

What You'll Need

  1. A Make.com account (free tier available).
  2. A Google Cloud Project with the Calendar API enabled and OAuth 2.0 credentials to connect your Google Calendar.
  3. A Betty Blocks account with an existing application and data model.
  4. Your Betty Blocks data model must include properties to receive the calendar data. The template expects properties like: Creator (Text), Attendees (Long Text), Meeting Start (Date time), Meeting End (Date time), Summary (Text), Status (Text).
  5. Your Betty Blocks API Token for authentication.

Quick Setup Guide

  1. Get the Template: Click "Get This Workflow" to copy the template to your Make.com account.
  2. Connect Google Calendar: In the first module, authenticate Make.com with your Google account and select the specific calendar to watch.
  3. Connect Betty Blocks: In the "Create a Record" module, add your Betty Blocks API token, select your application, and choose the correct data model (e.g., "Meetings" or "Calendar Events").
  4. Map the Data Fields: Ensure the data from Google Calendar (creator, attendees, etc.) is correctly mapped to the corresponding property names in your Betty Blocks data model. Double-check date/time formats.
  5. Set the Schedule: The scenario is set to run every hour. You can adjust this interval in the scheduler module to run more or less frequently based on your needs.
  6. Test & Activate: Run the scenario once manually to test. Create a test event in your Google Calendar, then check if a corresponding record appears in Betty Blocks. Once confirmed, activate the scenario to run automatically.
