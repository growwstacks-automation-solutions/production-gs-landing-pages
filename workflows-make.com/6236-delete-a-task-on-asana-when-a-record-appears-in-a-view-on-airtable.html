What This Workflow Does

In modern project management, teams often use Airtable as a flexible database to track work items and Asana for day-to-day task execution. A common headache occurs when a project or item is completed: someone must remember to go into Asana and manually delete or archive the corresponding task. This leads to cluttered task lists, outdated information, and wasted administrative time.

This automation solves that problem by creating a seamless bridge between the two systems. When you move a record into a specific "Completed" or "Archive" view in Airtable, the workflow automatically finds and deletes the matching task in Asana. It turns a multi-step, error-prone manual process into a reliable, hands-off operation. The result is a cleaner Asana workspace that always reflects the true state of work in your Airtable base.

Beyond simple deletion, this workflow embodies a principle of single-source truth. Your Airtable becomes the system of record for project status, and Asana becomes the dynamic execution layer that automatically adjusts. This eliminates version confusion and ensures everyone is looking at the same, up-to-date information.

How It Works

The workflow follows a logical, event-driven sequence. It waits for a change in Airtable, processes the data, and takes action in Asana.

Step 1: Monitor the Airtable View

The automation starts by watching a specific view in your Airtable base. Make.com polls this view at regular intervals (e.g., every 15 minutes) or uses a webhook for instant triggering. When a new record appears in that view—typically because you changed a status field or manually moved it—the workflow captures all the data from that record.

Pro tip: Use a dedicated "Workflow: Completed" view in Airtable with a filter like Status = "Done". This gives you clear control over what triggers the automation and allows for easy testing.

Step 2: Retrieve the Asana Task ID

For the automation to delete the correct task, it needs to identify it uniquely. This step involves searching Asana using criteria mapped from the Airtable record. Commonly, this could be the task name, a custom ID field you store in both systems, or the project and assignee combination. The workflow uses Asana's search API to find the matching task and retrieve its unique Asana Task ID.

Step 3: Delete the Task in Asana

With the correct Task ID in hand, the workflow sends a delete command to Asana's API. The task is permanently removed from the Asana project. Optionally, you can add a module beforehand to post a final comment to the task for audit purposes, or to send a notification to Slack confirming the action was taken.

What You'll Need

To implement this automation successfully, you should have the following set up and ready:

  1. An active Airtable account with a base containing the records you want to track.
  2. A specific view in that Airtable base that will serve as the trigger (e.g., "Completed Items").
  3. An active Asana account with projects and tasks already created.
  4. A consistent way to match Airtable records to Asana tasks (e.g., a shared unique ID, identical task names, or matching project fields).

Quick Setup Guide

Follow these steps to configure the template in your Make.com account:

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com workspace.
  2. Connect Your Apps: In the first module, authorize the connection to your Airtable account and select the specific base and view you want to monitor.
  3. Map the Data: In the Asana search module, configure the search criteria. Map the Airtable field (like "Task Name" or "Asana ID") to the corresponding search parameter in Asana.
