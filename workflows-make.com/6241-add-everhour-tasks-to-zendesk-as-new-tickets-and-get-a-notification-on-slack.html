What This Workflow Does

In many businesses, critical client issues or follow-up actions are first identified within project management tools like Everhour. A developer might log a bug, a project manager might note a client change request, or a designer might flag a dependency. However, getting that issue into the official support system (Zendesk) often requires manual, repetitive work—copying details, creating a ticket, and notifying the team. This creates delays, information silos, and a high risk of tasks falling through the cracks.

This automation solves that disconnect. It acts as a intelligent bridge between your project execution and customer support teams. Whenever a new, relevant task is created in Everhour, this workflow automatically generates a corresponding, fully-detailed ticket in Zendesk and posts a concise notification to a designated Slack channel. It transforms project notes into actionable support items in seconds, ensuring nothing gets lost in translation and your team can respond to clients faster.

How It Works

The workflow runs on a scheduled trigger in Make.com, constantly monitoring for new tasks and orchestrating the data flow between your apps.

Step 1: Monitor for New Everhour Tasks

The scenario starts by periodically checking your Everhour account for newly created tasks. You can filter this to specific projects, clients, or tags to ensure only support-related tasks trigger the automation.

Step 2: Format & Create a Zendesk Ticket

For each new task, the workflow extracts key information: task name, description, project, assignee, and estimated time. It then structures this data and uses the Zendesk module to create a new ticket. The ticket subject, description, requester (often mapped from the project client), and tags are automatically populated.

Pro tip: Use custom fields in Zendesk to map the Everhour "Estimated Hours" or "Project Code." This gives your support team immediate context on the scope of the issue.

Step 3: Post a Notification to Slack

Immediately after the ticket is created, the workflow sends a formatted message to a Slack channel of your choice. This alert includes a link to the new Zendesk ticket, the task summary, and the assignee, allowing for instant team awareness and facilitating quick assignment or discussion.

Step 4: Handle Errors & Logs

The scenario includes error handling routes. If a ticket fails to create in Zendesk or the Slack message fails, the workflow can send an alert to an admin or retry the operation, ensuring reliability.

Quick Setup Guide

You can have this workflow live in under 30 minutes by following these steps:

  1. Clone the Template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect Your Apps: In the Make scenario, establish connections (OAuth or API key) for Everhour, Zendesk, and Slack modules.
  3. Configure the Trigger: Set the Everhour module to watch the correct project(s) or apply filters (e.g., only tasks with a "Support" tag).
  4. Map Ticket Fields: Adjust the data mapping in the Zendesk module. Define which Everhour field goes into the Zendesk ticket subject, description, and requester email.
  5. Set Up Slack: Paste your Slack webhook URL and customize the notification message format.
  6. Test & Activate: Run a single test cycle by creating a dummy task in Everhour. Verify the ticket appears in Zendesk and the alert posts to Slack. Then schedule the scenario to run automatically (e.g., every 15 minutes).
