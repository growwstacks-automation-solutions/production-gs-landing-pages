What This Workflow Does

For marketing, product, and content teams, launching a new project often involves a frustrating, manual handoff. A project manager defines scope and deadlines in Asana, then someone must duplicate that effort by manually creating a corresponding project in GatherContent for the content team. This process is slow, error-prone, and creates silos between planning and execution.

This automation solves that by creating a seamless bridge between Asana and GatherContent. The moment a new project is created in Asana—whether it's a new marketing campaign, product launch, or website redesign—this workflow automatically triggers and creates a fully configured project in GatherContent. It transfers key details like the project name, description, due date, and assignees, ensuring your content team has immediate access to a structured workspace with all the context they need to start producing.

The result is a unified workflow where project initiation in one tool automatically sets up the necessary environment in another, eliminating duplicate data entry, reducing setup time from 30 minutes to 30 seconds, and ensuring both teams are aligned from day one.

How It Works

The automation follows a simple but powerful sequence, acting as a digital project coordinator between your two essential platforms.

Step 1: Monitor New Asana Projects

The workflow uses Make.com's Asana module to watch a specific Asana workspace or team for newly created projects. You can filter this to trigger only for projects with certain tags, in specific portfolios, or created by particular team members, giving you precise control over which initiatives kick off the content process.

Step 2: Extract Project Details

Once a new project is detected, the automation captures all relevant data: the project name, notes/description, start and due dates, custom fields (like "Content Lead" or "Campaign Code"), and even the list of project members. This information forms the blueprint for the new GatherContent project.

Step 3: Create the GatherContent Project

Make.com then connects to your GatherContent account via API. It uses the data from Asana to create a new project in your designated GatherContent account. The Asana project name becomes the GatherContent project name, the description populates the project brief, and dates are carried over to set timelines.

Pro tip: Use Asana custom fields to specify the GatherContent project template or folder structure you want to use. This allows the automation to create not just a blank project, but one pre-organized with your standard content types and workflows.

Step 4: Invite Team Members & Set Status

Optionally, the workflow can go further. It can invite the Asana project members to the new GatherContent project, assigning them appropriate roles. It can also set the initial project status in GatherContent to "In Planning" or "Ready for Brief," providing a clear starting point for the content team.

Quick Setup Guide

Follow these steps to deploy this automation in your own environment:

  1. Clone the Template: Click "Get This Workflow" to open the template in your Make.com account and create a copy (scenario).
  2. Connect Your Apps: In the Make.com scenario editor, establish connections to your specific Asana and GatherContent accounts using the API credentials.
  3. Configure the Asana Trigger: Set the module to watch the correct Asana workspace, team, or project section. Apply any filters (e.g., only projects with a "Content Required" tag) to avoid unnecessary triggers.
  4. Map the Data Fields: In the GatherContent "Create a Project" module, map the Asana data (project name, description, etc.) to the corresponding fields in GatherContent. Define your default account, project type, and status.
  5. Test with a Dummy Project: Run the scenario once manually and create a test project in Asana to verify a project is correctly created in GatherContent.
  6. Activate & Monitor: Turn on the scenario's scheduler. It will now run automatically in the background. Monitor the operations history for the first few days to ensure it's working smoothly.
