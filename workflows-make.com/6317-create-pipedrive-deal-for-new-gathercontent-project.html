Automatically Create Pipedrive Deals from New GatherContent Projects

Bridge the gap between content creation and sales. Every new content project in GatherContent instantly creates a tracked deal in your Pipedrive pipeline.

What This Workflow Does

For agencies, marketing teams, and content creators, a new project in GatherContent signifies the start of billable work and a potential revenue stream. However, this information often remains siloed within the content team, leaving the sales team in the dark about new opportunities entering the production phase. This disconnect leads to missed follow-ups, inaccurate pipeline forecasting, and a lack of alignment between project delivery and client management.

This automation solves that by creating an instant, bi-directional link between your content operations and your sales CRM. The moment a project manager or content lead creates a new project in GatherContent, this workflow automatically generates a corresponding deal in Pipedrive. The deal is populated with key project details—client name, project title, estimated value, and start date—ensuring your sales pipeline always reflects current work and future revenue.

How It Works

The workflow acts as a intelligent connector, listening for new projects and translating them into sales-ready deals.

Step 1: Monitor for New GatherContent Projects

The automation uses Make.com's GatherContent module to watch for newly created projects. It can be set to trigger instantly or at scheduled intervals, capturing all relevant project data like name, description, account, and custom fields.

Step 2: Map Project Data to Deal Fields

Make.com then transforms the GatherContent project information into the format Pipedrive expects. The project name becomes the deal title, the associated client becomes the organization or person in Pipedrive, and the project description adds context as deal notes. You can map custom fields for budget, timeline, or project type.

Pro tip: Use a formula in Make.com to calculate the initial deal value based on the number of content items or a predefined pricing tier stored in GatherContent custom fields.

Step 3: Create the Deal in Pipedrive

The workflow uses the Pipedrive module to create a new deal in the correct pipeline and stage. You can define rules to set the deal stage based on the project's status (e.g., "Briefing" stage for "Planning" projects). The sales team now has immediate visibility without a single manual entry.

Step 4: (Optional) Notify Teams and Set Follow-ups

As a final step, the automation can send a Slack or email notification to the sales lead, and even schedule a follow-up task in Pipedrive for the account manager to check in with the client after project kick-off.

Quick Setup Guide

You can have this automation running in under 20 minutes.

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com account.
  2. Connect Your Apps: In the Make.com scenario, add your Pipedrive and GatherContent accounts using the respective modules. Authorize the connections.
  3. Configure the Trigger: In the GatherContent "Watch Projects" module, select the specific account or workspace you want to monitor for new projects.
  4. Map Your Data: In the Pipedrive "Create a Deal" module, map the incoming GatherContent data to your desired Pipedrive deal fields (Title, Organization, Value, Stage, etc.).
  5. Test and Activate: Create a test project in GatherContent and run the scenario once to ensure a deal appears correctly in your Pipedrive. Then, turn the scenario on "Schedule" to run automatically.
