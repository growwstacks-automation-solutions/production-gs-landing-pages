What This Workflow Does

Businesses use Typeform to collect critical information—from customer inquiries and job applications to feedback and registrations. However, the real work begins after the form is submitted. Manually checking for new responses, copying data, and initiating the next steps in tools like Pneumatic is a slow, error-prone process that creates bottlenecks.

This automation solves that by creating a seamless bridge between data collection and process execution. Every time a new Typeform response is received, this Make.com scenario instantly acts as a relay, taking the submitted data and using it to launch a predefined process in Pneumatic. This turns a static form into a dynamic starting point for quotes, onboarding, support tickets, or any multi-step workflow you've built in Pneumatic.

The value is immediate: operations accelerate from hours to seconds, data accuracy improves because there's no human middleman, and your team can focus on high-value work instead of administrative data shuffling.

How It Works

The workflow is an elegant, single-purpose automation designed for reliability. Here is the step-by-step flow:

Step 1: Monitor Typeform for New Responses

The scenario starts with a Typeform module set to watch a specific form. It uses a webhook or the "Watch Responses" trigger to detect submissions in real-time. As soon as someone clicks "Submit," the entire response packet—including all answers, hidden fields, and metadata—is captured by Make.com.

Pro tip: Use Typeform's hidden fields to pass internal data like campaign IDs or user segments, which can then be used within your Pneumatic process for routing or personalization.

Step 2: Launch the Pneumatic Process

Make.com immediately takes the captured data and passes it to the Pneumatic module. This module is configured to call a specific process within your Pneumatic workspace. The form response data is mapped to the input variables that the Pneumatic process expects, ensuring the launched process has all the context it needs to run correctly.

Step 3: Process Execution & Outcome

Once triggered, the Pneumatic process runs independently. This could involve generating documents, creating records in a database, assigning tasks to team members, sending notifications, or executing complex business logic. The Make.com scenario can be extended to handle the success or failure of this launch, optionally logging the result or alerting your team if something goes wrong.

Quick Setup Guide

Follow these steps to configure the template for your own use:

  1. Clone the Template: Use the "Get This Workflow" button to open the template on Make.com and create a copy in your own workspace.
  2. Connect Your Apps: In the Make.com scenario, update the connections for both the Typeform and Pneumatic modules to use your own accounts. Authorize the integrations if prompted.
  3. Select Your Typeform: In the Typeform module, choose the specific form you want to monitor from the dropdown list.
  4. Configure the Pneumatic Launch: In the Pneumatic module, select the specific process you wish to trigger. Carefully map the fields from the Typeform response (e.g., `answers.email`) to the corresponding input variables expected by your Pneumatic process (e.g., `customerEmail`).
  5. Test and Activate: Run a test by submitting your Typeform. Check the Make.com execution history to confirm the data was passed and that the Pneumatic process was launched successfully. Once verified, activate the scenario.
