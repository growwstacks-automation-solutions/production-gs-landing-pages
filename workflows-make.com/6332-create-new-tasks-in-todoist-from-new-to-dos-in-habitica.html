What This Workflow Does

Many productivity enthusiasts face a common dilemma: they love the motivational, gamified experience of Habitica for building habits and tracking personal goals, but they rely on the robust project and task management features of Todoist for their serious work. This forces them to manually duplicate tasks between two systems, leading to wasted time, inconsistency, and the risk of items falling through the cracks.

This automation solves that problem by creating a seamless, one-way bridge from Habitica to Todoist. Every time you create a new to-do in Habitica—whether it's a daily habit, a personal goal, or a one-off task—this workflow automatically creates a corresponding task in your designated Todoist project. It eliminates the manual copy-paste, ensuring your commitment in your fun, motivational app instantly becomes an actionable item in your primary productivity system.

The template is designed for simplicity and reliability. It focuses on core data transfer: the task title, notes, and priority. It creates one-off tasks in Todoist by default, providing a clean foundation that you can easily extend to include due dates, labels, or even recurrence logic based on your specific Habitica setup.

How It Works

The workflow runs on Make.com, listening for new events in Habitica and pushing structured data to Todoist. Here’s the step-by-step process:

Step 1: Trigger on New Habitica To-Do

The automation is triggered instantly whenever you (or a team member) create a new to-do item within your Habitica account. Make.com watches the Habitica API for this specific event, capturing the new task's details like its title, description, difficulty, and tags as soon as it's saved.

Step 2: Map and Format the Data

The captured data from Habitica is then processed. This step involves mapping fields between the two apps. For example, Habitica's "Difficulty" (Trivial, Easy, Medium, Hard) can be intelligently mapped to Todoist's "Priority" levels. The task title and notes are formatted to ensure they appear correctly in Todoist.

Pro tip: Use this mapping step to add a consistent prefix like "[Habitica]" to all synced tasks in Todoist. This makes it easy to identify their origin at a glance and apply filters or labels.

Step 3: Create the Task in Todoist

Finally, the formatted data is sent to Todoist's API via a "Create Task" module. The automation specifies which Todoist project the task should belong to and populates it with the title, description, and priority. The task appears in your Todoist inbox or specified project immediately, ready for scheduling or action.

What You'll Need

  1. A Make.com account (free tier is sufficient to run this workflow).
  2. An active Habitica account with to-dos you want to sync.
  3. A Todoist account (any plan, including free).
  4. Your Habitica API key (found in Habitica Settings > API).
  5. Your Todoist API token (found in Todoist Settings > Integrations).
  6. A designated Todoist project where you want the synced tasks to appear (e.g., "Personal," "Habits," or "Inbox").

Quick Setup Guide

You can have this automation running in under 10 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button to open the template on Make.com and create a copy in your own account.
  2. Connect Habitica: In the first module, add your Habitica connection using your User ID and API Key from your Habitica account settings.
  3. Connect Todoist: In the "Create Task" module, add your Todoist connection using your API token from Todoist settings.
  4. Set the Target Project: Specify the exact Todoist project name (e.g., "Inbox") where new tasks should be created.
  5. Test and Activate: Turn on the scenario and create a test to-do in Habitica. Within seconds, check your specified Todoist project to confirm the new task appears correctly.
