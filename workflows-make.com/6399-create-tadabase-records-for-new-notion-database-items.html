What This Workflow Does

Many businesses use Notion as a flexible workspace for planning, collaboration, and capturing ideas—from sales leads and project briefs to content calendars. However, when it's time to act on that data, you often need it in a more structured, operational system like Tadabase, which powers customer portals, internal apps, and databases.

This automation solves the frustrating and error-prone process of manually copying data from Notion to Tadabase. It creates a seamless, real-time bridge between the two platforms. Whenever a new item is logged in your designated Notion database, this Make.com workflow triggers instantly, reads all the relevant details, and creates a perfectly formatted record in your Tadabase application. It turns a multi-step manual task into a zero-touch background process.

How It Works

The workflow operates on a simple but powerful trigger-action principle, ensuring data flows in one direction: from Notion (the source) to Tadabase (the destination).

Step 1: Monitoring the Notion Database

The workflow starts by watching a specific Notion database for new entries. Make.com checks for updates at regular intervals or can use webhooks for instant triggering. When a new row appears—whether added manually, via a form, or through an integration—the workflow captures the entire row's data, including all its properties.

Step 2: Processing and Mapping Data

Make.com then processes the captured information. It maps each Notion property (like "Name," "Status," "Date," "Assignee") to the corresponding field in your Tadabase table. This step ensures data types align correctly—text stays as text, dates as dates, and person fields can be mapped to email or name fields in Tadabase.

Pro tip: Use Make's filter module to ignore test entries or specific statuses. For example, you can set a rule to only proceed if the Notion row's "Status" is not "Draft," preventing incomplete data from syncing.

Step 3: Creating the Tadabase Record

Finally, the workflow uses the Tadabase module to create a new record. It sends the mapped data to your specified Tadabase application and table. Upon success, you can optionally log the action, send a confirmation to Slack, or even update the original Notion item with a "Synced" status, closing the loop.

What You'll Need

  1. A Make.com account (free tier sufficient for basic sync).
  2. A Notion account with a database you want to monitor. You'll need the database ID and an integration token.
  3. A Tadabase account with an existing application and a table ready to receive records. You'll need your API credentials.
  4. Clear understanding of which fields/properties in your Notion database should map to which fields in your Tadabase table.

Quick Setup Guide

  1. Clone the Template: Click "Get This Workflow" to open the template in your Make.com account and create a copy.
  2. Connect Notion: In the first module, authorize Make to access your Notion workspace and select the specific database to watch.
  3. Connect Tadabase: In the Tadabase module, authorize Make with your Tadabase API key and select the target application and table.
  4. Map the Fields: Carefully map each Notion property output to the correct Tadabase field. Pay attention to field types (text, number, date).
  5. Test and Activate: Add a test row to your Notion database. Run the scenario once to verify a record appears correctly in Tadabase. Then turn on the schedule for automatic operation.
