Sync Notion Databases to Google Calendar

What This Workflow Does

If you manage meetings and appointments in a Notion database, you likely face the tedious task of manually copying each one into Google Calendar. This double entry is not only time-consuming but also error-prone. A missed meeting or an incorrect time can damage client relationships and disrupt team productivity.

This automation solves that problem by creating a live, two-way sync between your Notion database and Google Calendar. Whenever a new meeting is added to your Notion database, the workflow instantly creates a corresponding event in the calendar. If the meeting details change—like the time, title, or attendees—the calendar event is updated automatically. And if a meeting is cancelled or deleted in Notion, the calendar event is removed, keeping everyone's schedule clean and accurate.

The result is a single source of truth for all scheduling. Your team can continue planning in the flexible environment of Notion, while enjoying the reliability and notifications of Google Calendar, without any manual work in between.

How It Works

The workflow runs on Make.com, acting as a smart bridge between Notion and Google Calendar. It periodically checks your database for changes and performs the necessary actions to keep both systems in sync.

Step 1: Watch for New or Updated Database Items

The automation starts by monitoring your specified Notion database. Using the "Watch Database Items" module, it triggers whenever a new page is created or an existing page is updated. This ensures the workflow catches every change in real-time.

Step 2: Extract and Map Meeting Details

For each triggered item, the workflow reads the relevant properties from the Notion page. Key details like the meeting title, date, start/end time, description, and attendee emails are extracted. These values are then formatted to match Google Calendar's expected input structure.

Pro tip: Structure your Notion database with clear property names like "Meeting Date", "Start Time", and "Attendee Emails" to make the field mapping process simple and reliable.

Step 3: Create or Update the Calendar Event

The workflow checks if a Google Calendar event already exists for this Notion item (often using a unique ID stored in a Notion property). If not, it creates a new event. If an event does exist, it updates the existing event with the new details. This step handles all the API calls to Google Calendar.

Step 4: Handle Deletions and Cancellations

If a meeting page is deleted in Notion or a status property is set to "Cancelled", the workflow identifies this change and proceeds to delete the corresponding event from Google Calendar. This prevents ghost events from cluttering your schedule.

Quick Setup Guide

Follow these steps to deploy this automation in under 15 minutes.

  1. Get the Template: Click "Get This Workflow" above to copy the template to your Make.com account.
  2. Connect Notion: In the first module, authenticate your Notion account and select the specific database you want to sync.
  3. Connect Google Calendar: Authenticate your Google account in the Calendar modules and select the target calendar (e.g., "Primary").
  4. Map Your Fields: Adjust the router and mapper modules to match your Notion property names to the correct Google Calendar event fields (summary, start, end, description, attendees).
  5. Set a Unique ID: Configure a property (like a "Calendar Event ID") in your Notion database for the workflow to track and update events correctly.
  6. Test & Activate: Run the scenario once to test with a sample meeting. If it works, schedule it to run every 15-30 minutes for near real-time sync.
