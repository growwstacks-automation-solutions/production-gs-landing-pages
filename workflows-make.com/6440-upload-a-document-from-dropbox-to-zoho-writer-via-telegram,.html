What This Workflow Does

For writers, content managers, and remote teams, the process of moving documents from cloud storage to an editing suite is a repetitive, time-consuming chore. You finish a draft in Dropbox, then must manually download it, navigate to Zoho Writer, upload the file, and organize it—stealing focus from actual creative work. This friction disrupts workflow and introduces the risk of uploading the wrong version or losing files in the shuffle.

This automation solves that by creating a seamless bridge between your storage, your communication tool, and your writing platform. With this Make.com workflow, you simply send a Telegram message containing the file path. The automation instantly locates the document in your Dropbox, retrieves it, and uploads it directly into your specified Zoho Writer account. It turns a multi-step, minute-long task into a single command that executes in seconds, reliably and without error.

How It Works

The workflow is an elegant three-step process that mimics the action you would take manually, but with perfect consistency and speed.

Step 1: Trigger via Telegram Message

The automation sits idle until it receives a specific command in a designated Telegram chat or from an authorized user. You send a message like “/upload /ClientProjects/Q3Report.docx”. The workflow instantly captures this text, parsing it to identify the command and the crucial file path within your Dropbox.

Step 2: Fetch the File from Dropbox

Using the parsed file path, Make.com securely connects to your Dropbox account via API. It locates the exact document you referenced, downloads its content, and prepares it for the next step. This happens in the background, with no need for you to open Dropbox or manually share a link.

Step 3: Upload & Create in Zoho Writer

The final module takes the file data and uses Zoho Writer's API to create a new document directly in your Zoho workspace. It uploads the content, sets the document title (often based on the original filename), and places it in your desired folder. You receive a confirmation, and the document is immediately ready for editing, collaboration, or publishing.

Pro tip: Use a dedicated private Telegram channel or bot for this workflow. This keeps your automation commands separate from personal or team chats, adding security and reducing the chance of accidental triggers.

Quick Setup Guide

Getting started is straightforward. Follow these steps to configure your automated pipeline.

  1. Clone the Template: Click the "Get This Workflow" button above to open the template in your Make.com account and create a copy.
  2. Connect Your Apps: In the Make.com scenario editor, authorize the three modules. Connect your Telegram account (via a bot), your Dropbox account, and your Zoho Writer account by following the on-screen OAuth steps.
  3. Configure the Trigger: Set up the Telegram module to watch for messages in a specific chat or from your user ID. Define the command keyword (e.g., "/upload").
  4. Map the Data Path: Ensure the module that parses your Telegram message correctly extracts the Dropbox file path from your message text.
  5. Set the Destination: In the Zoho Writer module, specify the default folder where new documents should be created. You can also map the document title from the original filename.
  6. Test and Activate: Send a test Telegram message with a safe file path. Turn on the scenario and verify a document appears in your Zoho Writer. Your automation is now live!
