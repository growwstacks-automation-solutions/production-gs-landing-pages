What This Workflow Does

For teams using Smartsheet for project planning and JIRA for development tracking, a major bottleneck exists in the handoff process. Product managers, project coordinators, or clients add new tasks, features, or bug reports to a shared Smartsheet. Someone then has to manually log into JIRA, create a new issue, and copy-paste all the relevant details—title, description, priority, assignee, and due dates. This process is slow, prone to human error, and distracts technical teams from their core work.

This automation solves that by creating a live bridge between the two platforms. Whenever a new row is added to a specified Smartsheet, this workflow automatically triggers. It reads the data from that row, formats it according to your JIRA project's requirements, and creates a fully-fledged JIRA issue—all within seconds. This ensures that nothing falls through the cracks and that your development backlog is always up-to-date with the latest business requirements.

Pro tip: Use this automation not just for task creation, but to enforce process. For example, only rows marked "Approved" or "Ready for Dev" in a specific Smartsheet column can trigger JIRA creation, adding a layer of quality control.

How It Works

The workflow is a straightforward, reliable sequence that operates silently in the background.

Step 1: Monitor Your Smartsheet

The automation starts by watching a specific Smartsheet you designate. It uses the "Watch Rows" module in Make.com to detect whenever a new row is added. This trigger is instant and reliable, ensuring no new task is missed.

Step 2: Extract and Map Row Data

Once a new row is detected, the workflow captures all the data from its columns. You then map these Smartsheet columns to the corresponding fields in JIRA. For example, the "Task Title" column becomes the JIRA "Summary," the "Detailed Notes" column becomes the "Description," and the "Assignee Email" column is used to set the JIRA assignee.

Step 3: Create the JIRA Issue

The workflow authenticates with your JIRA instance using an API key or OAuth. It then sends a structured request to create a new issue in your chosen project (e.g., Software Project XYZ), with the issue type set as "Story," "Bug," or "Task" based on your mapping. All the mapped data is populated into the new ticket.

Step 4: Confirm and Log (Optional)

For auditing and error handling, you can add a step to log the successful creation—including the new JIRA issue key (e.g., PROJ-123)—back to the original Smartsheet row. This creates a two-way reference link, making it easy to track the ticket's origin.

Quick Setup Guide

You can have this automation running in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com account.
  2. Connect Your Apps: In the first module, authorize the connection to your Smartsheet and select the specific sheet to watch. In the JIRA module, authorize the connection to your JIRA instance.
  3. Configure the Trigger: Set the Smartsheet module to "Watch Rows" and ensure it's pointing to the correct sheet. Test it by adding a dummy row.
  4. Map Your Data: In the JIRA "Create Issue" module, select your target Project and Issue Type. Then, map the data from the previous Smartsheet step to each JIRA field (Summary, Description, etc.).
  5. Test and Activate: Run the scenario once in test mode. Add a new row to your Smartsheet and verify a JIRA issue is created correctly. Once confirmed, schedule the scenario to run in real-time and turn it on.
