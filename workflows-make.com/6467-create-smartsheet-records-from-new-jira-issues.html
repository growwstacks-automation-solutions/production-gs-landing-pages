What This Workflow Does

For teams using JIRA for development tracking and Smartsheet for project planning, a significant operational gap often exists. Project managers and stakeholders need up-to-date information on issues, tasks, and bugs to manage timelines and resources effectively, but manually transferring this data is slow, error-prone, and creates information lag.

This automation solves that by creating a live, one-way bridge from JIRA to Smartsheet. Every time a new issue is created in your specified JIRA project—whether it's a bug, story, or task—the workflow automatically captures key details and adds them as a new, formatted row in your designated Smartsheet. This ensures your project plans, dashboards, and Gantt charts reflect the true state of work without any manual intervention.

The business value is clear: it eliminates duplicate data entry, reduces the risk of reporting errors, and gives non-technical team members real-time visibility into development progress directly within a tool they already use for planning.

How It Works

The workflow runs on a scheduled trigger or watches for webhook events, ensuring no new issue is missed. Here’s the step-by-step process:

Step 1: Monitor for New JIRA Issues

The automation is configured to watch a specific JIRA project, board, or filter. It uses the JIRA module in Make.com to poll for newly created issues at regular intervals or listens instantaneously via webhook.

Step 2: Extract and Map Key Issue Data

For each new issue found, the workflow extracts relevant fields. This typically includes the issue key (e.g., PROJ-123), summary, description, assignee, status, priority, due date, and any custom fields you specify.

Pro tip: Use filters in this step to only sync specific issue types (like 'Story' or 'Bug') to avoid cluttering your Smartsheet with internal tasks or subtasks.

Step 3: Format Data for Smartsheet

The raw data from JIRA is often formatted or transformed to fit your Smartsheet column structure. For example, it might convert a JIRA assignee's name into an email address for Smartsheet's contact column, or parse dates into the correct format.

Step 4: Create a New Row in Smartsheet

The formatted data is then sent to Smartsheet via the Make.com module. It creates a new row in your specified sheet, populating each column with the corresponding JIRA field. The row can include hyperlinks back to the original JIRA issue for quick access.

What You'll Need

  1. A Make.com account (free or paid plan).
  2. Admin or user access to a JIRA Cloud or JIRA Server/Data Center instance with API credentials (API token or OAuth).
  3. A Smartsheet account with a sheet ready to receive the JIRA data. You'll need the Sheet ID and appropriate column structure.
  4. Basic understanding of which JIRA project and issue types you want to sync.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Get the Template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect JIRA: In the first module, authorize Make.com to access your JIRA account using your API token or OAuth.
  3. Connect Smartsheet: In the Smartsheet module, authorize the connection using your Smartsheet API access token.
  4. Configure Triggers: Set the JIRA module to watch the correct project, board, or use a JQL filter to target specific new issues.
  5. Map Your Fields: In the Smartsheet "Add Row" module, map the JIRA data (like `issue.key`, `issue.fields.summary`) to the correct column IDs in your Smartsheet.
  6. Test & Activate: Run a test by creating a dummy issue in JIRA. Check if a row appears in Smartsheet. Once confirmed, activate the scenario.
