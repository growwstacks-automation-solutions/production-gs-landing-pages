What This Workflow Does

Manually adding new customers or leads to your live chat support system is a repetitive, error-prone task that steals time from your support team. When a new user signs up via a form, is added by sales, or comes from a webinar list, someone has to copy their details from a spreadsheet into Crisp. This delay means your support agents start conversations without crucial context, leading to slower responses and a poorer customer experience.

This free Make.com template solves that by creating a seamless, automatic bridge between Google Sheets and Crisp. Whenever a new row is added to your designated spreadsheet—whether it's a list of new trial users, paid customers, or support contacts—the workflow instantly triggers. It reads the data, formats it correctly, and uses the Crisp API to create a fully-formed user profile. Your support team gains immediate visibility, and you maintain a perfectly synchronized customer database without lifting a finger.

Pro tip: Use this workflow not just for new signups, but also to backfill your Crisp account with existing customers from your CRM or billing system by exporting them to a Google Sheet first.

How It Works

The automation follows a simple, reliable sequence to ensure data flows accurately from your spreadsheet to Crisp.

Step 1: Monitor Google Sheets for New Rows

The workflow starts by watching a specific Google Sheets spreadsheet. It is configured to trigger instantly whenever a new row is added. You can define which sheet and column range to monitor, giving you full control over the data source.

Step 2: Extract and Map User Data

Make.com reads the values from the new row. It maps spreadsheet columns—like 'Email', 'Full Name', 'Phone', and 'Company'—to the corresponding user profile fields required by Crisp. You can also include custom data attributes to segment users or provide additional context to your agents.

Step 3: Create the User in Crisp

The workflow sends a secure API call to your Crisp workspace with the compiled user data. Crisp then creates a new user profile with all the provided information. The user is now immediately visible in your Crisp dashboard, ready for any incoming chats or for proactive outreach.

Step 4: Handle Outcomes and Errors

The template includes basic error handling. If the user creation fails (e.g., due to a duplicate email or an API issue), the workflow can log the error or send a notification. For advanced setups, you can add steps to update the original spreadsheet row with a 'Status' like 'Created' or 'Failed'.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps.

  1. Get the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Google Sheets: In the first module, authenticate and select your spreadsheet and the specific worksheet you want to monitor for new rows.
  3. Connect Crisp: In the Crisp module, enter your API credentials (identifier and key) to link your workspace.
  4. Map Your Data: Match the columns from your Google Sheet (e.g., Column B for email, Column C for name) to the required fields in the Crisp "Create a Person" action.
  5. Test and Activate: Add a test row to your Google Sheet and run the scenario once to verify a user appears in your Crisp dashboard. Then, turn the scenario on to run automatically.

Pro tip: Create a dedicated "Automation Log" column in your Google Sheet. Use a simple Make.com step to mark rows as "Processed" or log a timestamp after successful creation. This gives you a clear audit trail.
