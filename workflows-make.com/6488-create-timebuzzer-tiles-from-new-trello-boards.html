What This Workflow Does

For agencies, consultants, and project teams, starting a new project involves multiple setup steps across different tools. One critical yet often manual task is creating a dedicated time tracking entry for the project. When you create a new Trello board to plan client work, you then have to switch to timeBuzzer (or another time tracker) to manually create a tile, copy project details, and assign team members. This process is repetitive, prone to being forgotten, and creates disconnect between project planning and actual time expenditure.

This automated workflow solves that problem by creating a seamless bridge between Trello and timeBuzzer. Every time a new board is created in Trello—whether for a new client, a new phase, or an internal initiative—Make.com automatically triggers and creates a corresponding tile in timeBuzzer. This ensures time tracking is ready before any work begins, linking project scope directly to billable hours from the very first minute.

The business value is clear: you eliminate administrative overhead, ensure 100% of billable work is tracked, and create a unified system where project management and time tracking are intrinsically connected. This leads to more accurate invoicing, better project profitability analysis, and hours saved each week that can be redirected to client work.

How It Works

The automation follows a simple but powerful sequence, transforming a project creation event into an actionable time tracking asset.

Step 1: Monitor New Trello Boards

The workflow starts with a Trello trigger module in Make.com that watches for new boards in your specified workspace or organization. You can filter this to specific teams or types of boards (e.g., only boards with a "Client Project" label). The moment a board is created, the workflow captures all relevant data like board name, description, ID, and the creator.

Step 2: Map Data to timeBuzzer Fields

Make.com then processes the Trello data, mapping it to the required fields for creating a timeBuzzer tile. The board name becomes the tile title, the description can be added as notes, and you can set default values for billing rates, project codes, or assignees based on your business rules. This step is where you customize the automation to fit your operational needs.

Step 3: Create the Tile in timeBuzzer

Finally, the workflow uses the timeBuzzer API module in Make.com to create a new tile. It sends the formatted data, and within seconds, a new time tracking tile is active in your timeBuzzer account, ready for your team to start logging hours against that specific Trello project.

Pro tip: Extend this workflow by adding a step that posts a comment back to the new Trello board with a link to the created timeBuzzer tile. This gives your entire team immediate visibility into where to track their time for that project.

What You'll Need

To implement this workflow, you'll need a few prerequisites set up. Don't worry—they're straightforward and most teams already have them.

  1. A Make.com account with an active plan (the free tier may suffice for lower volume).
  2. A Trello account with boards you use for project management.
  3. A timeBuzzer account for time tracking.
  4. API access (keys/tokens) for both Trello and timeBuzzer. These are obtained from each service's settings and are used to securely connect Make.com to your accounts.
  5. Basic understanding of your project naming conventions and what data (like client codes or default hourly rates) you want passed from Trello to timeBuzzer.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps.

  1. Get the Template: Click "Get This Workflow" to open the template in your Make.com account and create a copy.
  2. Connect Trello: In the first module, authenticate your Trello account and select the workspace or team where you want to monitor for new boards.
  3. Connect timeBuzzer: In the tile creation module, authenticate your timeBuzzer account using your API key.
  4. Map Your Data: Review the data mapping. Customize which Trello board fields (name, description) go into which timeBuzzer tile fields. You can add static values like a default project type or billing code.
  5. Test & Activate: Create a test Trello board to trigger the workflow. Verify that a tile appears in your timeBuzzer account. Once confirmed, turn the scenario on. Your automation is now live.
