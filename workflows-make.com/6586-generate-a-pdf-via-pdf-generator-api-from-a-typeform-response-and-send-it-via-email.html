What This Workflow Does

Manually creating PDFs from form submissions is a tedious, error-prone task that consumes valuable employee time. Whether you're processing applications, generating quotes, or creating certificates, the process of copying data, formatting a document, and sending it out is ripe for automation.

This Make.com workflow solves that problem completely. It automatically monitors your Typeform for new responses. When someone submits a form, the workflow captures their data, uses the PDF Generator API to create a polished, branded PDF document, and then emails that PDF directly to the respondent—or to your team—all within minutes. It turns a multi-step manual process into a seamless, hands-off operation.

The business value is clear: eliminate repetitive administrative work, ensure consistent and professional document output for every single submission, and dramatically speed up your response time to customers or applicants. This automation reliably handles the task 24/7, freeing your team to focus on higher-value activities.

How It Works

The automation follows a logical, reliable sequence triggered by each new form entry.

Step 1: Trigger on New Typeform Responses

The workflow is set to check your Typeform account periodically (e.g., every 15 minutes) for new submissions. You can also configure it to use instant webhooks for real-time processing. It securely retrieves all the answer data from the completed form.

Step 2: Prepare Data and Call PDF Generator API

The relevant answers from Typeform (like name, email, and responses) are mapped and sent to the PDF Generator API. Here, you use a pre-designed template—featuring your logo, branding, and layout—which the API populates with the submitted data to create a unique PDF file.

Step 3: Send the Generated PDF via Email

Once the PDF is created, the workflow attaches the file to an email. It can send this email to the person who filled out the form (using the email address they provided), to a designated internal team member, or to both. The email can be fully customized with a subject line and body text.

Pro tip: Use this workflow to automatically file a copy of each generated PDF in a Google Drive or Dropbox folder. Simply add a module after the PDF is created to upload the file. This creates a perfect, searchable audit trail.

Quick Setup Guide

Follow these steps to configure the template for your use case:

  1. Clone the Template: Click "Get This Workflow" to copy the scenario into your Make.com account.
  2. Connect Your Apps: In the first module, authorize the connection to your Typeform account and select the specific form you want to monitor.
  3. Configure the PDF API: In the PDF Generator API module, input your API key and select or design your PDF template. Map the Typeform response fields (e.g., `answer_1`, `answer_2`) to the variables in your PDF template.
  4. Set Up the Email: In the email module, connect your email service. Customize the recipient (use the respondent's email from Typeform), subject line, and body. Ensure the generated PDF is attached correctly.
  5. Test and Activate: Run a test by submitting a dummy response to your Typeform. Verify that a correctly formatted PDF is generated and received in the inbox. Once confirmed, activate the scenario's schedule.
