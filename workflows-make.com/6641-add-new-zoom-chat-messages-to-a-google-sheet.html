What This Workflow Does

In today's remote and hybrid work environments, Zoom meetings are central to collaboration. The chat feature is a vital channel for questions, feedback, shared links, and action items. However, once the meeting ends, that valuable chat history disappears or becomes difficult to access and analyze. Manually copying messages is tedious, error-prone, and often forgotten.

This automation solves that problem by creating a seamless bridge between Zoom and Google Sheets. It listens for new chat messages as they happen and instantly logs them into a structured spreadsheet. This turns transient conversation into permanent, searchable data. Business owners, team leaders, trainers, and compliance officers can now have an automatic record of every discussion point, enabling better follow-up, accountability, and insights.

The workflow uses a webhook to capture events from Zoom in real-time. When a participant sends a chat message during a meeting, the automation triggers, extracts the message details, and adds a new row to your designated Google Sheet. This happens automatically, without any manual intervention, ensuring no important comment is lost.

How It Works

The process is designed for reliability and simplicity, leveraging Make.com's powerful integration platform.

Step 1: Webhook Trigger from Zoom

The workflow starts with a webhook module configured to receive events from Zoom. When you set up the Zoom integration in Make.com, you specify which meeting or account to monitor. As chat messages are sent, Zoom pushes an event notification to this webhook, containing the message data, sender information, and timestamp.

Step 2: Data Parsing and Filtering

Make.com receives the webhook data and parses it. You can add optional filter modules here. For instance, you might choose to only log messages from specific participants, or messages that contain certain keywords like "urgent" or "question." This step ensures your sheet contains only the relevant data you need.

Pro tip: Use a filter to ignore system messages (like "xxx joined the meeting") to keep your log focused on human conversation.

Step 3: Formatting for Google Sheets

The parsed data is then mapped into a format suitable for Google Sheets. The workflow defines which pieces of data go into which columns: typically Message Text, Sender Name, Timestamp, and Meeting ID. You can customize this to include additional fields like sender email or message type.

Step 4: Adding a Row to Google Sheets

The final module is the Google Sheets "Add a Row" action. It connects to your specified spreadsheet and worksheet, and inserts the formatted data as a new row. Each new chat message creates a new entry, building a chronological log of the entire meeting's chat history.

Quick Setup Guide

Follow these steps to get your automation running quickly using the provided template.

  1. Clone the Template: Click "Get This Workflow" to open the template on Make.com. Use the "Copy" function to create your own scenario.
  2. Connect Your Zoom: In your Make.com scenario, add your Zoom account connection. Follow the prompts to authenticate and set up the webhook trigger for chat events.
  3. Connect Your Google Sheets: Add your Google account and select the specific spreadsheet and worksheet where messages should be logged.
  4. Map the Data Fields: In the Google Sheets module, map the incoming data from Zoom (message, sender, time) to the corresponding columns in your sheet.
  5. Test and Activate: Run a test by sending a chat message in a Zoom meeting. Check if a new row appears in your sheet. Once confirmed, activate the scenario to run automatically.

Pro tip: Schedule your scenario to run "instantaneously" so it processes messages as they arrive, ensuring real-time logging.
