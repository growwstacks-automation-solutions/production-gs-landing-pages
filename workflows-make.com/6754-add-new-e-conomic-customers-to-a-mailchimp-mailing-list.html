What This Workflow Does

For businesses using E-conomic for accounting and Mailchimp for email marketing, a significant operational gap often exists. New customers are added to your financial system, but they remain invisible to your marketing team until someone manually copies their details into Mailchimp. This delay can be days or weeks, causing missed opportunities for welcome emails, promotions, and nurturing sequences.

This free Make.com workflow template solves that problem by creating a seamless, real-time bridge between E-conomic and Mailchimp. The moment a new customer record is created in E-conomic, the automation triggers. It collects the customer's contact information and instantly adds them as a subscriber to a designated Mailchimp audience list. This ensures your marketing funnel is always populated with your latest clients, enabling timely and relevant communication.

The automation not only saves countless hours of administrative work but also enhances data accuracy by eliminating human error from manual entry. It turns a reactive, tedious task into a proactive, value-adding process that strengthens customer relationships from the very first interaction.

How It Works

The workflow operates on a simple but powerful "watch and act" principle, orchestrated by Make.com.

Step 1: Monitoring E-conomic for New Customers

The scenario starts with a Make.com module that watches your E-conomic account for new customer creations. This can be set to check at regular intervals (e.g., every hour) or can be triggered via a webhook if your E-conomic plan supports it. When a new customer is detected, the workflow captures the complete dataset, including name, email, company, and any other relevant fields you wish to sync.

Step 2: Formatting and Preparing the Data

Before sending data to Mailchimp, a router or filter module can be used to ensure data quality. For instance, you can set a rule to only proceed if the contact has a valid email address. The data is then mapped into the specific format required by Mailchimp's API, ensuring a smooth transfer.

Step 3: Adding the Contact to Mailchimp

The core action module connects to your Mailchimp account and adds the new contact to your specified audience list. You can define their subscription status (e.g., subscribed, pending) and assign them to specific tags or groups based on data from E-conomic, like "Customer" or "High-Value Client."

Pro tip: Use this automation to trigger a Mailchimp automation series immediately. Set the contact's status to "subscribed" and have them instantly receive a welcome email or a special offer for new customers, dramatically improving engagement rates.

What You'll Need

To clone and use this template, you will need the following accounts and permissions:

  1. A Make.com account (free or paid plan).
  2. An active E-conomic subscription with API access enabled. You'll need your E-conomic API credentials (Agreement Grant Token and App Secret Token).
  3. A Mailchimp account with at least one audience list created. You'll need a Mailchimp API key.
  4. Basic understanding of where to find API settings in both E-conomic and Mailchimp (the template guide will walk you through this).

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button above. This will open the template in your Make.com account and create a copy for you to edit.
  2. Connect E-conomic: In the first module, click to add a connection. You will be prompted to log into your E-conomic account and authorize Make.com to access your customer data.
  3. Connect Mailchimp: In the Mailchimp module, similarly add a connection using your Mailchimp API key. Select the specific audience list where new contacts should be added.
  4. Map the Data Fields: Review the data mapping between E-conomic and Mailchimp. Ensure key fields like Email Address, First Name, and Last Name are correctly linked. You can add more fields like Company or Phone Number if desired.
  5. Test and Activate: Use the "Run once" function in Make.com to test the scenario. Create a test customer in E-conomic and verify they appear in your Mailchimp list. Once confirmed, turn the scenario ON to run automatically.
