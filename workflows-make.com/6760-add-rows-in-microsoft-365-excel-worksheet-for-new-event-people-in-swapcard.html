Automatically Add Swapcard Event Attendees to Excel

What This Workflow Does

Managing event registrations involves constant data transfer between your event platform and your internal spreadsheets. Manually copying each new attendee's details—name, email, company, ticket type—from Swapcard into Microsoft Excel is a repetitive, error-prone task that consumes valuable team time. This automation solves that problem completely.

The workflow acts as a real-time bridge between Swapcard and Excel. Whenever a new "event person" (attendee) is added in Swapcard, the automation instantly captures that data and appends it as a new row in your designated Excel worksheet. This ensures your master attendee list is always up-to-date, accurate, and readily available for analysis, reporting, or follow-up actions without any manual intervention.

Beyond simple data entry, this setup transforms your Excel sheet into a dynamic, living database of your event audience. It enables proactive event management, allowing you to react to registration trends, segment attendees for targeted communications, and maintain a single source of truth for all stakeholder reporting.

How It Works

The automation follows a clean, trigger-action sequence powered by Make.com. Here’s a step-by-step breakdown of the process.

Step 1: The Trigger – Monitoring Swapcard

The workflow starts by watching the Swapcard module for a specific event. It is configured to monitor the "Event People" object. Any time a new record is created—meaning a new attendee registers or is added to the event—Swapcard sends a notification to Make.com, triggering the entire automation sequence.

Step 2: Data Mapping & Transformation

Make.com receives the raw data from Swapcard, which includes fields like first name, last name, email address, company, registration date, and any custom attributes. The workflow maps these fields to the corresponding columns in your Excel spreadsheet. You can customize this mapping to fit your exact sheet structure, ensuring data lands in the right place.

Pro tip: Before activating the workflow, prepare your Excel sheet with column headers matching the data you want to capture (e.g., "Full Name", "Email", "Company", "Registration Date"). This alignment guarantees seamless, error-free data insertion.

Step 3: The Action – Adding a Row to Excel

Finally, the Make.com scenario uses the Microsoft 365 Excel module. It connects to your specific workbook and worksheet, and executes an "Add a row" operation. The mapped data from Swapcard is inserted into the new row, completing the cycle. The entire process happens within seconds, providing near-instantaneous synchronization.

Quick Setup Guide

Follow these steps to configure the template for your own event and spreadsheet.

  1. Clone the Template: Use the "Get This Workflow" button to copy the scenario into your Make.com account.
  2. Connect Swapcard: In the Swapcard module, authenticate your account and select the specific event you want to monitor for new attendees.
  3. Connect Microsoft 365 Excel: Authenticate your Microsoft account in the Excel module and select the exact workbook and worksheet where rows should be added.
  4. Map the Data Fields: In the Excel "Add a row" action, map each Swapcard output field (like `firstName`, `email`) to the corresponding column in your Excel sheet. Ensure the column order matches your sheet's headers.
  5. Test the Flow: Add a test attendee in Swapcard or use the Make.com testing tool to simulate a new registration. Verify that a row appears correctly in your Excel sheet.
  6. Activate & Monitor: Turn the scenario on. Your automation is now live. Monitor the logs initially to ensure it runs smoothly for each new registration.
