What This Workflow Does

For ecommerce businesses using WooCommerce for sales and ScanOrders for warehouse management, a critical gap often exists: when an order is picked and packed in the warehouse, the online store still shows it as "processing." This disconnect forces staff to manually log into WooCommerce to update each order status—a tedious, error-prone task that wastes valuable time and delays customer notifications.

This free Make.com template solves that problem by creating a seamless, automated bridge between ScanOrders and WooCommerce. The moment an order is marked as fulfilled within your ScanOrders system, this workflow instantly triggers and updates the corresponding order in WooCommerce to "completed." It can also add tracking information and send the standard WooCommerce fulfillment email to your customer, all without any human intervention.

The result is a unified operation where your front-end store and back-end warehouse are always in perfect sync. You eliminate double-data entry, drastically reduce the risk of human error, and provide a superior customer experience with instant, accurate shipping updates.

How It Works

The automation follows a simple yet powerful sequence, acting as a real-time translator between your warehouse and your store.

Step 1: Monitor ScanOrders for Fulfillment Events

The workflow is set to watch your ScanOrders account for specific triggers, such as an order status changing to "Fulfilled," "Shipped," or "Picked." Make.com polls the ScanOrders API at regular intervals to catch these updates as soon as they happen.

Step 2: Identify the Matching WooCommerce Order

Once a fulfillment event is detected, the automation extracts the unique order identifier (like the order number or a custom field). It then searches your WooCommerce store via its API to find the exact matching order. Robust error handling ensures that if an order can't be found, the process halts and can notify an admin.

Pro tip: Ensure your order numbers or a custom ID (like a "ScanOrders Reference") are consistent between both systems. This matching key is the foundation of a reliable automation.

Step 3: Update WooCommerce Order Status & Details

With the correct WooCommerce order located, the workflow executes the update. It changes the order status to "Completed" and can optionally push additional data from ScanOrders, such as the tracking number, shipping carrier, and the actual fulfillment date. This enriches the WooCommerce order history with precise warehouse data.

Step 4: Trigger Customer Notification & Complete the Loop

The final step leverages WooCommerce's native functionality. Updating the order status to "Completed" automatically triggers the system's "Order completed" email to the customer, providing them with immediate confirmation and any tracking details. The loop is closed: warehouse action leads to instant store update and customer communication.

What You'll Need

  1. A Make.com account (Free tier or higher).
  2. An active WooCommerce store with API access enabled. You'll need the store URL, a Consumer Key, and a Consumer Secret.
  3. A ScanOrders account with API credentials available.
  4. A consistent identifier (like an order number) that exists in both ScanOrders and WooCommerce for each order.

Quick Setup Guide

You can have this automation running in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect ScanOrders: In the first module, add your ScanOrders API credentials and configure the trigger to watch for "Order Fulfilled" events.
  3. Connect WooCommerce: In the update module, add your WooCommerce store URL, Consumer Key, and Consumer Secret.
  4. Map the Data: Instruct the workflow how to find the matching order (e.g., map the ScanOrders order number to the WooCommerce order number field).
  5. Test with a Live Order: Run the scenario once manually with a recent fulfilled order to verify the connection and data mapping work correctly.
  6. Activate & Monitor: Turn on the scenario scheduler and let it run. Monitor the operations history for the first few days to ensure smooth running.
