What This Workflow Does

This automation solves the critical challenge of ensuring your engineering team is immediately aware of production deployments. When new code is deployed to monitored environments in New Relic, the system automatically sends SMS alerts via Twilio to designated team members.

The business value comes from reducing mean time to awareness (MTTA) for deployment-related incidents. According to DevOps Research, teams that implement real-time deployment notifications experience 43% faster incident response times compared to those relying on manual monitoring or email alerts.

How It Works

1. New Relic Deployment Detection

The workflow monitors New Relic's deployment API for new events. It can filter deployments by environment, application, or other metadata to ensure only relevant changes trigger alerts.

2. Alert Message Composition

Key deployment details are extracted and formatted into a concise SMS message, including application name, deployment timestamp, and a direct link to the New Relic deployment details.

3. Twilio SMS Delivery

The formatted message is sent via Twilio's reliable SMS gateway to pre-configured recipient lists. The workflow supports dynamic recipient routing based on deployment characteristics.
