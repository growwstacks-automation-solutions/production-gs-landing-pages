New Relic to Slack Deployment Notifications

Automatically notify your team in Slack whenever a new deployment happens in New Relic

Get This Workflow → Make.com · DevOps · Free Template

What This Workflow Does

This automation solves the common problem of deployment visibility across engineering teams. When developers push new code, other team members often remain unaware until issues arise. The workflow automatically detects deployment events in New Relic and posts detailed notifications to a designated Slack channel.

By connecting New Relic and Slack through Make.com, you eliminate manual status updates and create a single source of truth for deployment tracking. The automation captures essential details like deployment timestamps, version numbers, and associated metadata, then formats them into clear Slack messages that keep everyone informed.

How It Works

1. New Relic deployment detection

The workflow constantly monitors New Relic for new deployment markers. These markers are automatically created when your CI/CD pipeline pushes new code or when developers manually mark deployments in New Relic.

2. Deployment data extraction

Make.com retrieves all relevant deployment information including the deployment version, timestamp, description, and any custom attributes you've configured in New Relic. This ensures the notification contains all context your team needs.

3. Slack message formatting

The workflow transforms the raw deployment data into a well-structured Slack message. You can customize the format to include emojis, @mentions for specific teams, and links to related documentation or tickets.

4. Automated notification delivery

The formatted message is automatically posted to your chosen Slack channel. The workflow can be configured to notify specific team members or entire channels, ensuring the right people see deployment updates immediately.
