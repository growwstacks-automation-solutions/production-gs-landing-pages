New Relic Deployment Email Notifications

Automatically send email alerts when new code deployments are detected in New Relic

What This Workflow Does

This automation solves the common challenge of keeping teams informed about code deployments in real-time. When engineering teams push new code, stakeholders across the organization often remain unaware until issues arise or someone manually shares the update.

The workflow automatically detects new deployment markers in New Relic and sends customized email notifications to designated recipients. This ensures product managers, QA teams, customer support, and other stakeholders are immediately aware of production changes without requiring developers to send manual updates.

How It Works

1. New Relic Deployment Detection

The workflow monitors New Relic for deployment markers, which are automatically created when new code is deployed to your environments.

2. Deployment Data Extraction

Key details like deployment timestamp, version number, commit messages, and affected services are extracted from New Relic's API.

3. Email Template Processing

The workflow populates a pre-designed email template with the deployment details, creating a professional notification.

4. Recipient Management

Emails are sent to predefined distribution lists that can vary based on deployment type, environment, or other criteria.
