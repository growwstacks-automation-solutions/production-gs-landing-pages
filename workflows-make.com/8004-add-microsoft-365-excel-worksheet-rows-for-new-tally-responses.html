What This Workflow Does

This automation solves the tedious and error-prone process of manually copying data from Tally form submissions into Excel spreadsheets. Every time someone submits your Tally form, the workflow automatically adds their responses as a new row in your specified Microsoft 365 Excel worksheet.

The system runs continuously in the background, checking for new responses at regular intervals you configure. It handles all the data mapping between form fields and spreadsheet columns, maintaining perfect consistency without any human intervention. Whether you receive 5 or 500 responses daily, your Excel file stays perfectly updated.

How It Works

1. Trigger on New Tally Responses

The workflow monitors your Tally form for new submissions using Make.com's Tally connector. You specify which form to watch and how frequently to check for updates (typically every 15-60 minutes).

2. Process Response Data

Each new submission's data is extracted and formatted appropriately for Excel. The workflow can handle all Tally field types including text, numbers, dates, multiple choice, and file uploads.

3. Add Row to Excel Worksheet

Using Microsoft 365's API, the workflow adds the processed data as a new row in your specified Excel file. You configure exactly which form fields map to which spreadsheet columns during setup.

Pro tip: Set up a dedicated "Raw Data" worksheet to receive all submissions, then use Excel formulas to create summary views in separate tabs. This preserves original responses while enabling analysis.
