What This Workflow Does

This automation solves a critical challenge for sales teams using both Sellsy and Corymbus - the manual duplication of opportunity data between systems. Every time your team creates a new sales opportunity in Sellsy, this workflow automatically creates a corresponding record in Corymbus with all relevant details.

The integration ensures both CRMs stay perfectly synchronized without requiring sales reps to manually enter data twice. This eliminates errors from duplicate entry, prevents missed opportunities from slipping through the cracks, and gives your entire team complete visibility into the sales pipeline across both platforms.

How It Works

1. New opportunity detection

The workflow continuously monitors your Sellsy account for newly created opportunities using Make.com's polling functionality. When a new record appears, it triggers the automation.

2. Data extraction

The system retrieves all relevant opportunity details from Sellsy including contact information, deal value, expected close date, probability percentage, and any custom fields you've configured.

3. Field mapping

Make.com translates the Sellsy data structure into Corymbus' format, ensuring all fields are properly matched between the different CRM systems.

4. Opportunity creation

The workflow creates a new opportunity in Corymbus with all the mapped data from Sellsy, maintaining consistency between both systems.

5. Error handling

If any step fails, the workflow retries automatically and notifies administrators if persistent issues occur, ensuring no opportunities are lost.

Quick Setup Guide

  1. Clone the template in your Make.com account
  2. Connect your Sellsy account using OAuth or API keys
  3. Connect your Corymbus account with appropriate credentials
  4. Map the Sellsy opportunity fields to corresponding Corymbus fields
  5. Set your preferred polling interval for checking new opportunities
  6. Test with sample data to verify proper synchronization
  7. Activate the workflow for live operation

Pro tip: Start with a small subset of fields to sync, then expand once you've verified the basic integration works. This minimizes potential mapping errors.
