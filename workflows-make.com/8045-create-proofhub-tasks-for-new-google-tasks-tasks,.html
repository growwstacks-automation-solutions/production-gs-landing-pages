What This Workflow Does

This automation solves the common challenge of managing tasks across multiple platforms. Many teams use Google Tasks for quick personal task management but need ProofHub for formal project tracking. Manually copying tasks between systems wastes valuable time and introduces errors.

The workflow automatically monitors your Google Tasks at regular intervals (configurable from 1 minute to 24 hours). When it detects new tasks, it creates corresponding tasks in your specified ProofHub projects with all relevant details - including titles, descriptions, due dates, and priority levels.

How It Works

1. Monitoring Google Tasks

The workflow starts by connecting to your Google account and checking your specified task lists for new items. You can configure which lists to monitor and how frequently checks occur.

2. Processing New Tasks

When new tasks are detected, the system extracts all relevant details including the task name, notes, due date, and completion status. Optional filters can exclude certain tasks based on criteria you define.

3. Creating ProofHub Tasks

Each qualifying Google Task gets transformed into a ProofHub task in your designated project. The workflow maps fields appropriately and can assign tasks to specific team members if configured.

Pro tip: Set up separate workflows for different Google Task lists to automatically organize tasks into corresponding ProofHub projects.
