Share new Checkvist item on Discord

Automatically post new tasks from Checkvist to your team's Discord channel

Get This Workflow → Make.com · Free Template

What This Workflow Does

This automation bridges the gap between task creation in Checkvist and team communication in Discord. Whenever a new item gets added to your Checkvist list—whether it's a project milestone, daily task, or meeting action item—the workflow instantly shares it in your designated Discord channel.

Teams waste countless hours manually copying tasks between apps or sending "don't forget" messages. This solution eliminates that friction while ensuring critical items never slip through the cracks. Remote teams particularly benefit from having tasks appear where conversations already happen.

How It Works

1. New Item Detection

The workflow monitors your specified Checkvist list for new items using Make.com's instant trigger. No polling delays—new tasks get processed within seconds.

2. Data Formatting

Make.com extracts relevant task details (name, due date, assignee etc.) and structures them into a clean Discord message format you configure.

3. Discord Delivery

The formatted message posts to your pre-configured Discord channel via webhook, complete with @mentions for assignees and color-coded priority flags.

This automation delivers the most value for:

What You'll Need

  1. A Checkvist account with admin access to your task lists
  2. A Discord server where you have webhook creation permissions
  3. A Make.com free account (no credit card required)

Quick Setup Guide

  1. Copy this template to your Make.com account
  2. Connect your Checkvist account in the "Trigger" module
  3. Create a Discord webhook in your desired channel settings
  4. Paste the webhook URL into Make.com's Discord module
  5. Customize the message format (we recommend including due dates)
  6. Turn on the scenario and test with a new Checkvist item
