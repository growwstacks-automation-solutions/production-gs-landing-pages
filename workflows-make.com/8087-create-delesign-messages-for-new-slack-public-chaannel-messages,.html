What This Workflow Does

This automation solves the common problem of important discussions getting lost in Slack's fast-moving public channels. It automatically monitors specified Slack channels at regular intervals and creates corresponding project messages in Delesign, ensuring critical conversations are properly documented where your team can reference them later.

Teams using both Slack for communication and Delesign for project management often struggle with manually transferring context between platforms. This workflow eliminates that friction by automatically creating Delesign messages containing the Slack message content, author information, and timestamps.

How It Works

1. Slack Message Monitoring

The workflow checks your specified Slack public channels at configured intervals (typically every 15-60 minutes) for new messages.

2. Message Processing

Each new message is processed to extract the content, sender information, timestamp, and any relevant metadata.

3. Delesign Message Creation

The workflow then creates a corresponding message in your Delesign project, preserving the original context while formatting it appropriately for project documentation.

Pro tip: Configure filters to only capture messages containing specific keywords or from certain team members to avoid information overload.
