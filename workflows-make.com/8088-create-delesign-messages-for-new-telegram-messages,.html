What This Workflow Does

This automation solves the common problem of fragmented communication channels. Many businesses receive customer messages on Telegram but need those conversations documented in their Delesign account for team collaboration and record-keeping. Manually copying messages between platforms is time-consuming and prone to errors.

The workflow automatically detects new messages in your Telegram bot and forwards them to your specified Delesign account. It maintains message context, preserves timestamps, and can handle various media types. This creates a seamless bridge between your customer-facing Telegram channel and internal Delesign communication system.

How It Works

1. Telegram Message Trigger

The workflow monitors your Telegram bot for new incoming messages. When a message arrives, Make.com captures its content, sender information, and any attachments.

2. Message Processing

The system formats the Telegram message into Delesign-compatible structure, preserving key metadata. Optional filters can be applied to only forward messages meeting specific criteria.

3. Delesign Delivery

Make.com sends the processed message to your designated Delesign account through its API. The message appears in Delesign with all original content intact.

Pro tip: Add a processing delay of 2-3 seconds if forwarding high volumes of messages to avoid hitting API rate limits.
