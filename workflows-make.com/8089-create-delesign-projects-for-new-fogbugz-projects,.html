What This Workflow Does

This automation bridges the gap between your development and design teams by automatically creating Delesign projects whenever new projects are added in FogBugz. It eliminates the manual process of setting up duplicate projects across different platforms, ensuring both teams have immediate access to properly configured workspaces.

The workflow monitors your FogBugz instance at regular intervals, detects newly created projects, and replicates them in Delesign with all relevant project details. This creates a seamless handoff between development planning and design execution, reducing setup delays and miscommunication between teams.

How It Works

1. FogBugz Project Monitoring

The workflow checks your FogBugz account at scheduled intervals (typically every 15-60 minutes) for newly created projects. It can monitor specific project categories or all new projects based on your configuration.

2. Data Extraction

For each new project detected, the workflow extracts key information including project name, description, assignees, deadlines, and any custom fields you've configured to sync.

3. Delesign Project Creation

Using the extracted data, the automation creates a corresponding project in Delesign with matching attributes. You can configure template mappings to ensure different FogBugz project types create appropriately structured Delesign projects.

4. Notification & Verification

Optionally, the workflow can send Slack or email notifications to relevant team members when new linked projects are created, or verify successful project creation before proceeding.

Quick Setup Guide

  1. Install the template from Make.com and open it in your scenario editor
  2. Connect your FogBugz account by entering API credentials
  3. Connect your Delesign account using OAuth or API keys
  4. Configure the polling interval for checking new FogBugz projects
  5. Map FogBugz project fields to corresponding Delesign project fields
  6. Test with a sample FogBugz project to verify proper Delesign creation
  7. Activate the scenario and monitor initial executions
