What This Workflow Does

This automation solves the common problem of maintaining duplicate projects across Hubstaff (for time tracking) and Delesign (for creative collaboration). Every time your team creates a new project in Hubstaff, this workflow automatically generates a matching project in Delesign with the same name, description, and relevant details.

For creative agencies and design teams, this eliminates the manual back-and-forth between time tracking and creative platforms. No more forgetting to create the Delesign counterpart, no more mismatched project names, and no more wasted time on duplicate administrative setup.

How It Works

1. Monitoring Hubstaff for new projects

The workflow checks your Hubstaff account at regular intervals (default: every 15 minutes) for newly created projects. It identifies projects that don't yet have corresponding Delesign entries.

2. Extracting project details

For each new Hubstaff project, the automation pulls key information including project name, description, client details, start date, and any custom fields you've configured to sync.

3. Creating Delesign projects

The system then uses Delesign's API to create a new project with identical parameters. It maintains all naming conventions and organizational structures you've established across both platforms.

4. Error handling and notifications

If any step fails (like API rate limits or invalid data), the workflow automatically retries up to 3 times and notifies your team via email if manual intervention is needed.
