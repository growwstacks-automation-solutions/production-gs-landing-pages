What This Workflow Does

This automation solves a critical pain point for sales teams using Qwilr for proposals - the tedious manual process of tracking which proposals get accepted and logging them in spreadsheets. Every time a client accepts a Qwilr proposal, the workflow automatically creates a new row in your designated Excel worksheet with all relevant details.

The system captures essential proposal data like client name, proposal amount, acceptance date, and any custom fields you've set up in Qwilr. This creates a centralized, searchable record of all won deals without requiring your team to manually transfer information between systems.

How It Works

1. Trigger: New Qwilr Page Accepted

The workflow activates whenever a client accepts a proposal in your Qwilr account. Make.com monitors your Qwilr activity through a secure API connection.

2. Data Extraction

The system retrieves all relevant proposal details including client information, pricing, custom fields, and metadata. This data is structured for clean Excel import.

3. Excel Worksheet Update

Using your Microsoft 365 credentials, the workflow adds a new row to your specified Excel file. You can map Qwilr fields to specific Excel columns for perfect formatting.

4. Error Handling

If any step fails (like Excel being unavailable), the workflow retries or sends an alert notification so you never lose proposal data.
