What This Workflow Does

This automation bridges the gap between time tracking in Zistemo and project management in Asana. Whenever a new task is created in Zistemo (typically for client billing or time tracking purposes), the workflow automatically generates a corresponding task in Asana where your team manages project execution.

The integration eliminates manual data entry between systems, ensuring all tasks requiring both time tracking and project management attention are properly accounted for in both platforms. This is particularly valuable for agencies, consultants, and professional service teams who need to track billable hours in Zistemo while managing project workflows in Asana.

How It Works

1. Monitoring Zistemo for new tasks

The workflow continuously checks your Zistemo account for newly created tasks using Make.com's monitoring capabilities. You can configure which projects or task types should trigger the automation.

2. Extracting task details

When a new task is detected, the workflow captures key information including the task name, description, due date, and any custom fields you want to transfer between systems.

3. Creating Asana tasks

The system then creates a corresponding task in your specified Asana project, mapping the Zistemo task details to appropriate Asana fields. You can configure default assignees, projects, and tags in Asana.

4. Error handling and notifications

If any task fails to create in Asana, the workflow automatically retries and notifies you of the issue. This ensures no tasks fall through the cracks due to temporary API issues or configuration errors.
