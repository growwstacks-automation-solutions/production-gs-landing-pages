What This Workflow Does

This automation bridges the gap between time tracking and project management by automatically creating Jira Software issues whenever new tasks are logged in Zistemo. It eliminates the manual copy-paste between systems that often leads to errors, missed tasks, or outdated information.

When creative teams track time in Zistemo and development teams work in Jira, this workflow ensures both sides stay aligned. Project managers get visibility into development progress without leaving their preferred time tracking tool, while developers receive properly formatted tickets with all necessary details.

How It Works

1. Task Creation in Zistemo

The workflow monitors your specified Zistemo projects for new tasks. You can configure it to watch specific clients, projects, or task types based on your team's needs.

2. Data Mapping

Key task details like title, description, due date, and assignee are extracted from Zistemo and mapped to corresponding Jira fields. Custom fields can be included to maintain specific workflows.

3. Jira Issue Creation

The system creates a new Jira issue in your designated project with all mapped information. You control which Jira project, issue type, and status each Zistemo task becomes.

4. Status Synchronization

Optional extensions can update Jira issues when Zistemo tasks change, keeping both systems in sync throughout the project lifecycle.
