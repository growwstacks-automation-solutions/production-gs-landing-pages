What This Workflow Does

This automation solves the tedious manual process of creating shortened URLs for marketing campaigns and social media posts. Every time you add a new long URL to your designated Airtable base, the workflow automatically generates a clean, trackable shortened version using dlvr.it and updates your record with the new link.

Marketing teams waste countless hours copying links between tools and maintaining spreadsheets. This workflow eliminates those repetitive tasks while ensuring all your shortened links are properly documented in Airtable with timestamps and metadata. The system maintains perfect synchronization between your URL shortener and database.

How It Works

1. New URL Detection

The workflow monitors your specified Airtable base for new records containing long URLs. It can be configured to watch specific fields or tables based on your structure.

2. URL Shortening Process

When a new URL is detected, the system sends it to dlvr.it's API to generate a shortened version. You can customize the short link alias or let dlvr.it generate one automatically.

3. Database Update

The shortened URL is then written back to the original Airtable record, along with any additional metadata like creation timestamp or click analytics. This creates a complete audit trail.

Pro tip: Add a "Status" field in Airtable to implement approval workflows before URLs get shortened, giving managers control over link creation.
