What This Workflow Does

For businesses selling both online via WooCommerce and in-person through Clover POS, managing separate customer databases is a major operational headache. When a new customer shops on your website, their profile is trapped in WooCommerce, invisible to your store staff. This forces manual copying of details, leads to errors, and creates a fragmented view of your customer base, hurting marketing, loyalty, and service quality.

This automation template solves that by creating a seamless, one-way bridge from your WooCommerce store to your Clover POS system. It automatically detects new customer registrations and purchases on your website and instantly creates a corresponding, accurate customer record in Clover. This means your sales team immediately has access to the customer's name, contact information, and order history, enabling personalized service from the very first in-store visit.

How It Works

The workflow runs on a scheduled basis, acting as a diligent assistant that never misses a new online customer.

Step 1: Watch for New WooCommerce Customers

The automation is triggered at regular intervals you set (e.g., every 15 minutes, hourly). It queries your WooCommerce store via its API, searching for any customers created since the last check.

Step 2: Extract and Format Customer Data

For each new customer found, the workflow extracts key details like first name, last name, email address, billing address, and phone number. It then structures this data into the specific format required by the Clover POS API to create a new customer entry.

Pro tip: You can easily customize this step to include additional data points from WooCommerce, such as the customer's first purchased product, which can be added as a note in their Clover profile for personalized service.

Step 3: Create the Customer in Clover POS

The formatted data is sent securely to your Clover POS account via its API. A new customer profile is created in your Clover customer list, complete with all the imported details. The workflow can be configured to log each successful sync for your records.

What You'll Need

  1. A live WooCommerce store with administrative access to generate API keys (Consumer Key and Consumer Secret).
  2. A Clover POS account with merchant-level access to generate API tokens.
  3. A Make.com account (free tier available) to deploy and run the automation scenario.
  4. Basic information about your store, such as your WooCommerce store URL.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps:

  1. Get the Template: Click "Get This Workflow" to copy the ready-made scenario into your Make.com account.
  2. Connect WooCommerce: In the Make scenario, create a connection to your WooCommerce store using the API credentials from your WooCommerce settings.
  3. Connect Clover POS: Similarly, create a connection to your Clover POS account using the merchant ID and API token from your Clover developer dashboard.
  4. Configure the Schedule: Set how often you want the automation to check for new customers (e.g., every hour).
  5. Test and Activate: Run the scenario once manually to test with a recent customer. Once confirmed, activate the schedule to let it run automatically.
