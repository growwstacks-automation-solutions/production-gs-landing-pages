What This Workflow Does

Field teams using ArcGIS Field Maps collect critical data—asset inspections, environmental samples, incident reports, customer locations—but that valuable information often gets stuck in silos. Manually transferring data from field apps to spreadsheets is slow, error-prone, and creates reporting delays. This automation solves that by creating a seamless bridge between your field operations and your central data repository in Microsoft Excel.

Whenever a field worker creates a new feature (like logging a damaged utility pole or recording a water sample location), this Make.com workflow instantly captures that event. It extracts all relevant data—timestamp, user name, details, contact information, and even geographic coordinates—and appends it as a perfectly formatted new row in your designated Excel worksheet. Your reports, dashboards, and analysis are always current without anyone lifting a finger.

How It Works

The workflow acts as a real-time data pipeline, ensuring no field observation is missed or delayed.

Step 1: Monitor ArcGIS Field Maps

The scenario starts by watching a specific ArcGIS layer or map for "Create Feature" events. Using ArcGIS's webhooks or polling, it detects the moment a new entry is submitted from a mobile device in the field, even in areas with intermittent connectivity.

Step 2: Extract and Structure the Data

Make.com parses the incoming feature data. It maps each field from the ArcGIS form (like "type of debris," "details," "phone," "email") into separate variables. It also handles geometry data, extracting the X and Y coordinates for precise location tracking.

Step 3: Connect to Microsoft 365 Excel

The workflow authenticates with your Microsoft 365 account and accesses the specified Excel workbook and worksheet. It prepares the data in the exact column order your sheet requires.

Step 4: Add the New Row

Finally, the scenario performs the core action: adding a new row to the Excel worksheet. Each piece of field data is placed into its correct column, creating a complete, timestamped record. The process completes in seconds, providing immediate data availability for your team.

Pro tip: Use this workflow as a trigger for further actions. Once the data is in Excel, you can set up additional automations to notify managers via email, create tasks in project tools, or update dashboards in Power BI.

Quick Setup Guide

  1. Clone the Template: Click "Get This Workflow" to open the template in your Make.com account and create a copy.
  2. Connect ArcGIS: In the first module, add your ArcGIS connection and select the specific map or feature layer you want to monitor for new creations.
  3. Connect Microsoft 365: In the Excel module, authenticate with your Microsoft 365 account and browse to select the exact Excel file and worksheet.
  4. Map Your Columns: Review the sample data flow. In the "Add a Row" action, map each output from ArcGIS (datetime, name, type, details, etc.) to the corresponding column in your Excel sheet. Adjust if your columns differ from the sample.
  5. Test & Activate: Run a test by creating a dummy feature in your ArcGIS Field Maps app. Verify the new row appears correctly in Excel. Then, activate the scenario to run automatically.
