What This Workflow Does

For businesses that rely on contracts, the process after signing is often chaotic. Executed documents get emailed, downloaded to random folders, or worse—lost in inboxes. This creates compliance risks, slows down operations, and wastes time searching for the final version.

This automation solves that by creating a seamless bridge between your e-signature platform, Signi, and your cloud storage, Dropbox. Every time a document is signed and meets your defined criteria, the workflow instantly triggers. It captures the final, executed contract and saves it to a new, neatly labeled Dropbox folder—often using the current date for perfect chronological organization. This turns a manual, error-prone task into a zero-touch, reliable process.

The result is a single source of truth for all signed agreements. Your team gains immediate access to any contract, auditors can easily trace documents, and you eliminate the risk of misplacing critical legal paperwork. The automation works silently in the background, ensuring your document management is as professional and efficient as the rest of your business.

How It Works

The workflow is a straightforward, three-step process built on Make.com's visual automation platform.

Step 1: Monitor for Signed Documents in Signi

The scenario starts by watching your Signi account for specific events. You can configure it to trigger when a document's status changes to "signed," "completed," or when it meets other custom conditions you set, like a specific tag or template name.

Step 2: Retrieve the Final Contract File

Once the trigger fires, Make.com communicates with Signi's API to fetch the final, signed PDF of the document. It securely pulls all relevant data, including the document name, signing parties, and the date of completion.

Pro tip: Use Signi's custom fields or tags to add metadata (like Client ID or Project Code) to your documents. The automation can read this data and use it to create even smarter folder names and file organization in Dropbox.

Step 3: Create Folder and Save to Dropbox

This is where the magic of organization happens. The workflow uses the date the scenario runs (or the signing date) to create a new folder in your designated Dropbox directory—for example, "Signed Contracts/2024-11-26". It then uploads the PDF into that new folder. This creates a clean, date-based archive automatically.

Quick Setup Guide

You can have this automation running in under 15 minutes.

  1. Clone the Template: Click the "Get This Workflow" button to open the template in your Make.com account and create a copy.
  2. Connect Your Apps: In the Make.com scenario editor, click on the Signi module and authenticate your account. Repeat the process for the Dropbox module.
  3. Configure the Trigger: Set the Signi module to watch for the specific document status that indicates a completed signing (e.g., "completed").
  4. Set the Dropbox Path: In the Dropbox module, specify the main parent folder (e.g., "/Company/Signed Contracts/") and confirm the folder naming convention (using the date is default).
  5. Activate & Test: Turn the scenario on and sign a test document in Signi. Check your Dropbox in a minute to see the new dated folder and your contract safely inside.
