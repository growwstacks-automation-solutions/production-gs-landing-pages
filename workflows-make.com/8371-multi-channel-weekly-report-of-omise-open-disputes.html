What This Workflow Does

Payment disputes cost businesses time and money. This automation solves the visibility problem by automatically gathering all open Omise disputes each week and distributing them to your team through multiple channels. Instead of manually checking the Omise dashboard or waiting for email notifications, your finance, customer service, and management teams receive consistent, formatted reports in their preferred communication platforms.

The workflow pulls dispute details including transaction amounts, customer information, dispute reasons, and resolution deadlines. It compiles this data into an easy-to-scan format and delivers it simultaneously to Gmail inboxes, Slack channels, and Telegram groups. This ensures all stakeholders have the same information at the same time, reducing miscommunication and delayed responses.

How It Works

1. Scheduled Weekly Trigger

The workflow activates automatically every Monday morning, querying the Omise API for all open disputes from the previous week.

2. Data Collection & Formatting

It retrieves key dispute details including transaction IDs, amounts, customer names, dispute reasons, and response deadlines. The system organizes this data into a clean, consistent format.

3. Multi-Channel Distribution

The formatted report gets sent simultaneously to pre-configured recipients:

  • Gmail: Detailed HTML email with dispute summaries
  • Slack: Concise message with key metrics and links
  • Telegram: Mobile-friendly alert with priority disputes

Pro tip: Configure different report formats for different teams - finance may need full transaction details while customer service benefits from customer contact information.

What You'll Need

  1. An active Omise account with API access
  2. Make.com account (formerly Integromat)
  3. Destination channels configured (Gmail, Slack, and/or Telegram)
  4. List of email addresses/Slack channels/Telegram groups to receive reports

Quick Setup Guide

  1. Clone the template in your Make.com account
  2. Connect your Omise API credentials
  3. Configure your output channels (Gmail, Slack, Telegram)
  4. Set your preferred report day/time (default is Monday 9AM)
  5. Test with sample data to verify formatting
  6. Activate the scenario
