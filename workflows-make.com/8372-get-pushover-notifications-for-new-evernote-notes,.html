What This Workflow Does

This automation solves the problem of missing important Evernote notes among the flood of daily notifications. Many professionals rely on Evernote for critical information but don't always check the app regularly. By routing new note alerts to Pushover, you get immediate, prioritized notifications with custom sounds/vibrations that stand out from other alerts.

The workflow monitors your Evernote account for new notes and automatically pushes formatted notifications to your Pushover account. You'll see the note title, preview text, and can configure additional metadata like tags or notebook names. This creates a seamless bridge between your note-taking system and notification management.

How It Works

1. Evernote triggers on new notes

The workflow starts when Evernote detects a new note creation in your account. Make.com polls Evernote's API at regular intervals to check for updates.

2. Note data is processed

The workflow extracts key details from the note including title, content preview, creation time, and any tags or notebook associations. You can customize which fields to include.

3. Notification is formatted for Pushover

The system structures the note information into a clean notification format with prioritized fields and optional emoji indicators for quick scanning.

4. Alert is delivered to Pushover

Pushover receives the formatted notification and delivers it to all your registered devices with your preferred sound, vibration, and priority settings.

Pro tip: Add a direct link back to the Evernote note in your Pushover notification by including the note's shareable URL in the message.

What You'll Need

  1. An active Evernote account (free or paid)
  2. A Pushover account with registered devices
  3. A Make.com account (formerly Integromat)
  4. API credentials for both services (provided in setup guide)

Quick Setup Guide

  1. Clone the template in your Make.com account
  2. Connect your Evernote account in the first module
  3. Add your Pushover API key and user key in the notification module
  4. Customize the notification message format if desired
  5. Set your preferred polling interval (default is every 15 minutes)
  6. Activate the scenario and test with a new Evernote note
