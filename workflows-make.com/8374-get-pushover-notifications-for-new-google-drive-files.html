What This Workflow Does

This automation solves the problem of missing critical file uploads in busy Google Drive accounts. When multiple team members or clients upload files to shared folders, important documents can easily get overlooked without proper notification systems in place.

The workflow monitors your Google Drive in real-time, detecting every new file upload across selected folders. Instead of requiring manual checks or relying on unreliable email notifications, it instantly pushes mobile alerts through Pushover - complete with file details, upload timestamps, and direct access links.

How It Works

1. Real-Time File Detection

The Make.com scenario continuously monitors your Google Drive account through API connections, checking for new file uploads at configurable intervals (as frequent as every minute).

2. Smart Filtering

Optional filters let you specify which folders to monitor or file types to alert about, preventing notification overload from less important uploads.

3. Notification Assembly

For each qualifying file, the workflow compiles key details including filename, file type, uploader information (if available), folder location, and direct access links.

4. Pushover Delivery

The formatted alert gets sent to your Pushover account, appearing instantly on all connected mobile devices with configurable priority levels and notification sounds.

Pro tip: Set different notification sounds for different file types or folders - like a distinct alert for contracts versus routine documents.

Quick Setup Guide

  1. Clone the template in your Make.com account
  2. Connect your Google Drive account in the first module
  3. Configure which folders to monitor (entire Drive by default)
  4. Add your Pushover API keys and user/group tokens
  5. Customize the notification message format if desired
  6. Set the polling interval (1-15 minutes recommended)
  7. Activate the scenario and test with a sample upload
