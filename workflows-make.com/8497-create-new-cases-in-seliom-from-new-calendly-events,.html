What This Workflow Does

This automation solves a common efficiency problem for service businesses using Calendly for scheduling and Seliom for case management. Without automation, staff must manually create new case files for every appointment scheduled - a time-consuming process prone to delays and errors.

The workflow automatically detects new Calendly events and creates corresponding cases in Seliom with all relevant appointment details. This ensures your team has complete case information immediately available, eliminating manual data entry and reducing the risk of missed cases.

How It Works

1. Calendly Event Trigger

The workflow monitors your Calendly account for new scheduled events. When someone books an appointment, it instantly captures all event details including client information, meeting time, and any custom questions answered during booking.

2. Data Transformation

The system maps Calendly event fields to Seliom case fields, formatting the data appropriately for your case management system. This includes converting date formats, combining name fields, and extracting key details from the event description.

3. Seliom Case Creation

Using Seliom's API, the workflow creates a new case with all the transformed data. The case is automatically categorized based on your business rules and assigned the appropriate status (typically "Scheduled" or "New").

4. Confirmation & Logging

After successful case creation, the system logs the transaction and can optionally send confirmation notifications to relevant team members or sync the information to other connected systems.
