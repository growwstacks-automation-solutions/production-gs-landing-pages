What This Workflow Does

This automation solves the common problem of manually transferring information from spreadsheets to case management systems like Seliom. Teams often collect customer requests, bug reports, or service tickets in Google Sheets through forms or shared documents, then waste time copying this data into Seliom.

The workflow automatically checks your specified Google Sheet for new rows at regular intervals (default every 15 minutes). When new entries are detected, it creates corresponding cases in Seliom with all relevant details mapped from spreadsheet columns to Seliom case fields. This eliminates duplicate data entry while ensuring no requests get overlooked.

How It Works

1. Scheduled Google Sheets Check

The workflow runs on a schedule you configure (default: every 15 minutes) to scan your designated Google Sheet for new rows added since the last check.

2. Data Extraction

For each new row found, the system extracts specified column values that will populate the Seliom case fields like title, description, priority level, and custom fields.

3. Case Creation in Seliom

Using Seliom's API, the workflow creates a new case for each qualifying row, mapping spreadsheet data to the appropriate case fields according to your configuration.

Pro tip: Add a "Status" column in your sheet to mark rows as "Processed" after case creation, preventing duplicates while maintaining an audit trail.
