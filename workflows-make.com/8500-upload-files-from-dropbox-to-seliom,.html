What This Workflow Does

Managing digital assets across different platforms is a common yet tedious task for creative teams, marketing agencies, and project managers. You might store final designs, client briefs, or project resources in Dropbox for easy sharing, but your team collaborates and manages tasks within Seliom. Manually downloading files from Dropbox and re-uploading them to Seliom is not only time-consuming but also prone to errors, version mix-ups, and delays.

This automation solves that problem completely. It creates a seamless, one-way bridge from a designated Dropbox folder to your Seliom account. Whenever a new file is added to your watched Dropbox folder, the workflow triggers automatically. It fetches the file and uploads it directly to a specified location in Seliom, ensuring your project resources are always synchronized without any manual intervention.

The template is pre-configured to check for new files every 15 minutes, but you can easily adjust this schedule to match your workflow's urgency—whether you need near-instant updates or daily batches.

How It Works

The workflow operates on a simple "watch and move" principle, leveraging Make.com's robust automation engine to connect two powerful cloud services.

Step 1: Monitoring Dropbox for Changes

The scenario starts with a Dropbox module set to "Watch files" in a specific folder. It polls the folder at your chosen interval (e.g., every 15 minutes) and detects any newly added files. This trigger is the starting point for the entire automation.

Step 2: Retrieving the File Data

Once a new file is detected, the workflow retrieves its essential details: the file name, path, size, and a secure download link. Make.com temporarily holds this file data in memory to pass it to the next step.

Pro tip: You can add a filter after the trigger to only process certain file types (like .jpg, .pdf, .mp4) or files containing specific keywords in their names. This prevents unwanted files from being transferred.

Step 3: Uploading to Seliom

The core action happens here. The workflow uses the Seliom module to "Upload a file." It takes the file data from the previous step and sends it to a predefined destination within your Seliom account, such as a specific project, task, or general file library.

Step 4: Completion & Error Handling

After a successful upload, the workflow can be extended to send a confirmation notification to a Slack channel, update a Google Sheet log, or send an email. If an upload fails (e.g., due to a network error or file size limit), the scenario can route the error to a separate path for alerting and retry logic.

Quick Setup Guide

You can have this automation running in under 10 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button to open the template on Make.com. Click "Copy" to create your own version of the scenario.
  2. Connect Your Apps: In your new scenario, click on the Dropbox and Seliom modules. Authorize Make.com to access your accounts by logging in and granting the necessary permissions.
  3. Configure the Trigger: In the Dropbox "Watch files" module, select the specific folder you want to monitor from your connected account.
  4. Set the Destination: In the Seliom "Upload a file" module, choose the target project, task, or library where you want the files to land. You can map the file name and content directly from the Dropbox module.
  5. Adjust the Schedule: By default, the scenario runs every 15 minutes. You can change this to hourly, daily, or even in real-time with a webhook (advanced) based on your needs.
  6. Test and Activate: Turn on the "Run once" option to test. Upload a sample file to your Dropbox folder. Wait for the schedule cycle (or run it manually) and verify the file appears in Seliom. Once confirmed, activate the scenario.
