What This Workflow Does

Manually transferring files from cloud storage to project management or client portals is a repetitive, time-consuming task that plagues many businesses. It creates bottlenecks, leads to version confusion, and risks human error where critical documents are forgotten or misplaced. This workflow solves that by creating a seamless, automatic bridge between Google Drive and Seliom.

Whenever a new file is added to a designated Google Drive folder, this Make.com automation instantly detects it and uploads it directly to your specified location in Seliom. It runs on a schedule you control (default is every 15 minutes), ensuring a near real-time sync. This means your sales proposals, design assets, contracts, or reports are always available where your team or clients need them, without any manual intervention.

How It Works

The automation follows a simple but powerful sequence, triggered by the presence of a new file.

Step 1: Monitor Google Drive Folder

The workflow starts by periodically checking a specific folder you choose in your Google Drive. Using Make.com's Google Drive module, it searches for any new files that have appeared since the last check.

Step 2: Retrieve File Details

For each new file found, the automation gathers essential information: the file name, its unique ID in Google Drive, its MIME type (e.g., PDF, JPEG), and a secure download link. This data is passed to the next step.

Pro tip: Use a dedicated "To-Seliom" folder in Google Drive for maximum control. This keeps the automation focused and prevents unrelated personal files from being synced accidentally.

Step 3: Upload to Seliom

Make.com then takes the file data and uses Seliom's API to perform an upload. It sends the file directly from Google's servers to Seliom, attaching it to the correct project, task, or client space as you've configured. The process maintains file integrity and naming.

Step 4: (Optional) Log & Notify

You can extend the template to log each successful transfer in a Google Sheet or send a confirmation notification to a Slack channel. This provides visibility and an audit trail for your automated file movements.

Quick Setup Guide

You can have this automation live in minutes by following these steps:

  1. Get the Template: Click the "Get This Workflow" button to copy the template into your Make.com account.
  2. Authorize Connections: In the Make.com scenario editor, click on the Google Drive and Seliom modules to connect your accounts. This is a secure, one-time login process.
  3. Configure the Trigger: In the first Google Drive module, specify the exact folder you want to monitor for new files.
  4. Set the Destination: In the Seliom module, map where the files should go. This might require selecting a project ID or specifying a folder path within Seliom.
  5. Adjust the Schedule: The template is set to run every 15 minutes. You can change this to hourly, daily, or even trigger instantly using webhooks if your plan supports it.
  6. Turn it On: Activate the scenario. Drop a test file into your designated Google Drive folder and watch it appear in Seliom automatically.
