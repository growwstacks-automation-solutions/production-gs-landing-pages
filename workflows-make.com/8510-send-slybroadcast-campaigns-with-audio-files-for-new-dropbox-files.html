What This Workflow Does

This automation solves the tedious process of manually uploading audio files to Slybroadcast and launching voice campaigns. Many businesses waste hours each week downloading recordings from Dropbox, logging into Slybroadcast, and configuring campaigns - especially when sending time-sensitive announcements or recurring reminders.

The workflow automatically detects new audio files in your specified Dropbox folder, extracts the necessary file details, and triggers a Slybroadcast campaign with your pre-configured settings. This ensures your voice messages go out immediately when new content is ready, without any manual intervention.

How It Works

1. Dropbox File Monitoring

The workflow continuously checks your designated Dropbox folder for new audio files at regular intervals (default: every 15 minutes). When a new file appears, it captures the file URL and metadata.

2. Audio File Processing

The system verifies the file format (supports MP3, WAV, M4A) and prepares it for broadcasting by confirming it meets Slybroadcast's technical requirements.

3. Slybroadcast Campaign Launch

Using your pre-configured Slybroadcast API credentials, the workflow automatically creates a new voice campaign with your audio file, scheduled according to your default settings or specific timing rules you've established.

4. Confirmation & Logging

The system records each campaign launch in your Make.com scenario history and can optionally send you a confirmation email or Slack notification with campaign details.

Pro tip: Create different Dropbox subfolders for various campaign types (e.g., /promotions, /reminders) and set up separate workflows with different Slybroadcast settings for each.
