What This Workflow Does

This automation solves a common pain point for development teams using GitLab for issue tracking. When developers create issues containing code snippets (like bug reports or feature implementations), team members often need to share these code segments externally for debugging or collaboration purposes.

The workflow automatically detects new GitLab issues, extracts any code blocks or relevant content, and creates corresponding Pastebin pastes. It intelligently handles privacy settings - creating private pastes for confidential issues and public ones for regular issues. This eliminates manual copying while maintaining proper access controls.

How It Works

1. New GitLab Issue Detection

The workflow monitors your GitLab repository for new issues using webhooks or API polling. When a new issue is created, Make.com receives the complete issue details including description, labels, and confidentiality status.

2. Content Extraction & Formatting

The system parses the issue content to identify code blocks (marked with ) or other specified content patterns. It formats this content with appropriate syntax highlighting based on the language specified in the code block.

3. Pastebin Creation

Using Pastebin's API, the workflow creates a new paste with the extracted content. The paste's privacy setting automatically matches the GitLab issue's confidentiality status. The paste includes reference back to the original GitLab issue for traceability.

4. Notification & Logging

The system can optionally notify team members via Slack, email, or GitLab comments about the created paste. It also logs all actions for audit purposes, including paste URLs and creation timestamps.

Pro tip: Configure the workflow to add your team's standard header/footer to pastes, including disclaimer text, contact information, or expiration dates for sensitive code.
