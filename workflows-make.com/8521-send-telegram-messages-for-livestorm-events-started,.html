What This Workflow Does

Hosting a successful online event requires perfect timing. A delayed host or unprepared moderator can frustrate attendees and harm your brand's credibility. Manually monitoring the Livestorm dashboard for event start times is inefficient and prone to error, especially when managing multiple events or teams across different time zones.

This automation solves that problem by creating a direct, real-time bridge between Livestorm and Telegram. Every time a scheduled event's status changes to "started" or "live" in Livestorm, this workflow triggers instantly. It then crafts a customized notification and sends it directly to a specified Telegram chat, group, or channel. This ensures that everyone who needs to know—hosts, co-hosts, technical support, or community managers—is alerted the second the event begins, enabling them to join promptly and deliver a seamless experience.

How It Works

The workflow is built on Make.com and operates on a simple trigger-action principle, ensuring reliability and speed.

Step 1: Monitoring Livestorm for Event Start

The automation constantly watches your Livestorm account using a webhook or scheduled API check. The moment Livestorm signals that an event has transitioned from "scheduled" to "live," it captures this trigger. The system pulls key details about the event, such as its title, start time, direct join link, and host information.

Step 2: Processing and Formatting the Alert

Make.com receives the trigger data and prepares it for notification. This step allows you to customize the message. You can include dynamic variables like the event name, a clickable join link for quick access, and the current time. The message is formatted to be clear and actionable, ready for delivery.

Pro tip: Use Telegram's formatting options in your message. You can add bold text for the event title (**Event Title**) and include a prominent "Join Now" button using the direct event URL to maximize click-throughs.

Step 3: Sending the Instant Telegram Notification

Finally, the workflow uses the Telegram Bot API to send the formatted message. The alert arrives instantly in the pre-defined Telegram destination—whether it's a private group for your production team, a channel for all moderators, or a direct message to the main host. The entire process from event start to notification delivery takes mere seconds.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button above to copy the ready-made scenario to your Make.com account.
  2. Connect Livestorm: In the first module, authorize Make.com to access your Livestorm account by logging in with your Livestorm credentials when prompted.
  3. Configure Telegram: In the Telegram module, paste your Bot Token and the target Chat ID. Send a test message to verify the connection.
  4. Customize Your Message: Edit the message template module. Insert the dynamic data from Livestorm (like `Event Title` and `Join Link`) to create a helpful alert.
  5. Activate and Test: Turn the scenario on. Start a test event in Livestorm or wait for your next scheduled event to begin. You should receive the Telegram notification instantly.
