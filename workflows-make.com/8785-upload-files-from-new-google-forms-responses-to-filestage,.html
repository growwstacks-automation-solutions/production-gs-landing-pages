What This Workflow Does

Creative teams, marketing agencies, and HR departments often use Google Forms to collect files—like design portfolios, video submissions, or campaign assets—from clients, candidates, or internal teams. The traditional process involves manually downloading each attachment from the form responses and then uploading it to a review platform like Filestage. This is repetitive, time-consuming, and prone to errors, leading to lost files and delayed feedback cycles.

This automation solves that problem entirely. It acts as a digital bridge between Google Forms and Filestage. Whenever a respondent submits a form with a file attachment, the workflow instantly detects the new submission, retrieves the uploaded file, and pushes it directly into a specified Filestage project. The result is a fully automated, hands-off pipeline that ensures every submitted file is immediately available for review by the right stakeholders.

The business value is clear: you reclaim hours of administrative work each week, eliminate the risk of human error, and significantly accelerate your project timelines. Feedback can start minutes after submission, not hours or days later, improving client satisfaction and team productivity.

How It Works

The workflow is built on Make.com and follows a simple, reliable sequence. Once set up, it runs automatically in the background, requiring zero ongoing maintenance.

Step 1: Trigger on New Form Response

The scenario starts by watching your specific Google Form for new responses. The Make.com module connects to your form and polls for new entries. The moment a submission comes in, the workflow is triggered, capturing all the response data, including any file URLs attached to the form.

Step 2: Retrieve the Submitted File

Next, the workflow accesses Google Drive (where Forms stores uploaded files) using the file URL from the response. It securely fetches the file and prepares it for transfer. This step handles the file download process automatically, so you never have to visit Google Drive yourself.

Step 3: Upload to Filestage Project

The core action: the retrieved file is uploaded to your pre-configured Filestage project. You specify the project and, optionally, the folder within Filestage where all submissions should go. The file appears in Filestage with its original filename, ready for your team to add comments, annotations, and approvals.

Pro tip: You can extend this workflow to rename the file in Filestage using data from the form, like "Portfolio_JohnDoe_20250415.pdf", making it instantly identifiable for your review team.

What You'll Need

To use this template, you'll need a few accounts and permissions set up beforehand:

  1. A Make.com account (free tier is sufficient to start).
  2. A Google Workspace or standard Google account with access to the form you want to automate.
  3. The Google Form must have a File Upload question type enabled to collect attachments.
  4. A Filestage account with an active project where you want the files to be sent.
  5. API connection access (Make.com will guide you through authorizing Google and Filestage with OAuth).

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button above. This will copy the pre-built scenario into your Make.com account.
  2. Connect Your Apps: In the Make.com scenario editor, authorize the connections to your Google account (for the Forms trigger) and your Filestage account (for the upload action).
  3. Configure the Trigger: Select the specific Google Form you want to monitor from the dropdown list in the first module.
  4. Set the Destination: In the Filestage module, choose your target project and folder from the connected account.
  5. Activate & Test: Turn the scenario ON and submit a test file to your Google Form. Check your Filestage project within moments to confirm the file arrived automatically.

Pro tip: Before going live, run a few test submissions with your team to ensure files are landing in the correct Filestage folder and that notifications are working as expected.
