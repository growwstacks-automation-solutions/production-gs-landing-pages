What This Workflow Does

For creative teams, marketing agencies, and production houses, managing the flow of assets from creation to client review is a major operational headache. You track deliverables in a spreadsheet, but then someone has to manually find each file and upload it to a review platform like Filestage. This process is repetitive, error-prone, and steals valuable hours from your team every week.

This automation solves that by creating a seamless bridge between your project tracker (Google Sheets) and your review platform (Filestage). Every time a new row is added to your designated spreadsheet—signaling a new asset is ready for review—the workflow springs into action. It reads the file information, fetches the asset, and uploads it directly to the correct Filestage project, all without human intervention.

The result is a dramatic acceleration of your review cycles. Files move from "ready" to "in review" instantly, eliminating the lag of manual handling. Project managers gain back time, creatives get faster feedback, and clients receive a more professional, responsive experience.

How It Works

The workflow is a smart sequence that connects two critical business tools. Here’s the step-by-step process:

Step 1: Monitor Google Sheets for New Rows

The automation continuously watches your specified Google Sheets spreadsheet. It's triggered the moment a new row is added. This row should contain essential data like the file name, a link to the file (e.g., a Google Drive or Dropbox URL), and potentially the target Filestage project name or ID.

Step 2: Parse and Validate File Data

Make.com reads the data from the new row. It extracts the file URL and any metadata (like version number, due date, or client name). The system validates that the URL is accessible and that the necessary information to proceed is present.

Pro tip: Structure your Google Sheets with consistent columns like 'File URL', 'Project', and 'Asset Type'. This makes the automation more reliable and easier to maintain.

Step 3: Fetch the File from Cloud Storage

Using the provided URL, the workflow accesses the file from your cloud storage (Google Drive, Dropbox, etc.). It downloads the file into the automation's memory, preparing it for the next step. This step handles the secure transfer from your storage to the automation platform.

Step 4: Upload to the Correct Filestage Project

This is the core action. The automation connects to your Filestage account, identifies the correct project (based on data from the Sheets row), and uploads the file. It can also set custom parameters, such as adding a description from the spreadsheet or assigning specific reviewers.

Step 5: Confirm and Log the Action

Finally, the workflow can send a confirmation—for example, logging a timestamp back in the Google Sheets row, sending a notification to a Slack channel, or triggering an email to the project manager. This closes the loop, providing visibility and ensuring nothing falls through the cracks.

Quick Setup Guide

Follow these steps to configure the template for your business:

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Your Apps: Authorize the Google Sheets and Filestage modules within Make.com, granting the necessary permissions.
  3. Configure the Google Sheets Trigger: Point the trigger to your specific spreadsheet and worksheet. Define which column addition should start the process.
  4. Map Your Data Fields: Inside the workflow, map the columns from your Sheets row (e.g., Column B = File URL, Column C = Project Name) to the corresponding fields in the Filestage upload module.
  5. Set Up Filestage Destination: Configure the Filestage module to find the correct project using the data from Sheets, and set any default upload options.
  6. Test with a Dummy Row: Add a test row to your Sheets with a sample file. Run the scenario once to ensure the file uploads to the right Filestage project.
  7. Activate and Monitor: Turn on the scenario. It will now run automatically. Monitor the initial few uploads and set up a simple error notification (like an email) for peace of mind.
