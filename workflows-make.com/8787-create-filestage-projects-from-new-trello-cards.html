What This Workflow Does

For marketing, design, and content teams, the gap between project planning and creative review is where deadlines often slip. A designer finishes a mockup, but then someone must manually create a Filestage project, upload files, add reviewers, and send notifications. This workflow eliminates that friction entirely.

By connecting Trello and Filestage, you create a seamless pipeline. The moment a task card is created in Trello—whether it's for a new blog graphic, a website update, or a video ad—a corresponding review project is automatically spun up in Filestage. The card's title becomes the project name, its description becomes the brief, and assigned team members can be added as reviewers. This ensures the feedback process begins the instant the work is ready, not after a manual setup delay.

How It Works

The automation acts as a intelligent bridge, translating project management actions into review-ready environments.

Step 1: Monitor Your Trello Board

The workflow watches a specific Trello board (or list) for new cards. You can filter this to only trigger for cards with certain labels, in specific lists like "Ready for Review," or assigned to particular members.

Step 2: Extract Card Details

When a new card is detected, Make.com captures key information: the card name, description, due date, members, attachments, and checklist items. This data forms the foundation of the Filestage project brief.

Pro tip: Use Trello's custom fields or checklist titles like "Target Audience" or "Brand Guidelines Link" to pass structured brief information directly into the Filestage project description.

Step 3: Create the Filestage Project

Using the Filestage API, the workflow creates a new project. It maps the Trello card name to the project title and populates the description. You can configure it to set a due date based on the Trello card's deadline and even add the card's attachments as the first files for review.

Step 4: Notify and Sync

Optionally, the automation can then post a comment back on the Trello card with the direct link to the new Filestage project, or send a notification to a Slack channel. This closes the loop, giving everyone immediate access to the review environment.
