What This Workflow Does

For businesses using Thankster for address management and Shopify for e-commerce, maintaining separate customer lists is a major operational headache. Every time a new address or contact is added in Thankster, someone must manually copy that information into Shopify to create a customer record. This process is slow, prone to typos, and leads to inconsistent data across your systems.

This automation solves that by creating a live bridge between Thankster and Shopify. Whenever a new address is created in your Thankster account, this Make.com workflow instantly triggers and creates a corresponding, fully-formed customer profile in your Shopify store. It transfers key details like name, email, phone number, and shipping address automatically, ensuring your sales platform always has your most up-to-date customer information without any manual effort.

The result is a unified customer database that improves marketing accuracy, streamlines order fulfillment, and provides a single source of truth for customer interactions. It turns two separate tools into one cohesive system.

How It Works

The workflow operates on a simple trigger-action principle, managed entirely in the cloud by Make.com.

Step 1: Monitoring Thankster for New Addresses

The workflow starts by watching your Thankster account for new events. Using Thankster's API, Make.com polls for newly created addresses at regular intervals you define (e.g., every 15 minutes, hourly, or in real-time via webhook if supported). When a new address record is detected, the workflow captures all its associated data.

Step 2: Parsing and Structuring the Customer Data

Make.com then processes the raw data from Thankster. It maps the address fields (like first name, last name, company, street, city, zip code, country) to the corresponding fields required by Shopify's customer creation API. This step can include data cleaning, like standardizing country codes or splitting full names into first and last names.

Pro tip: Use Make.com's data transformation functions here to set default values, like adding a "Thankster Import" tag to these customers for easy segmentation later in Shopify.

Step 3: Creating the Customer in Shopify

With the data correctly formatted, the workflow authenticates with your Shopify store via API and executes the "Create Customer" command. It sends the parsed information, and Shopify creates the new customer record in your admin panel. The workflow can be configured to send only the data you need, respecting Shopify's field requirements.

Step 4: Error Handling and Logging

A robust automation includes error management. If the creation fails (e.g., due to a duplicate email, invalid data, or API limit), the workflow can be set to retry, send an alert to your team via email or Slack, and log the issue for review. This ensures no customer falls through the cracks due to a technical hiccup.

Quick Setup Guide

Follow these steps to configure the template for your business:

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Thankster: In the first module, authenticate Make.com with your Thankster account using API keys.
  3. Connect Shopify: In the Shopify module, create a private app in your Shopify admin, copy the API key and secret, and paste them into Make.com.
  4. Map Your Data Fields: Review the data mapping between Thankster and Shopify modules. Adjust field mappings to match your specific Thankster address field names.
  5. Set the Schedule: Configure how often Make.com should check Thankster for new addresses (start with hourly for testing).
  6. Test with a Live Address: Create a test address in Thankster and run the scenario once to verify a customer is created correctly in Shopify.
  7. Activate & Monitor: Turn the scenario on. Monitor the operations history for the first few days to ensure it runs smoothly.
