What This Workflow Does

For e-commerce businesses using Thankster for personalized cards and gifts, customer data often lives in isolation. When a new customer places an order and provides their shipping address, that valuable information isn't automatically available to your customer support team in Intercom. This creates a disjointed experience where support agents lack context, leading to slower response times and frustrated customers who have to repeat their information.

This free Make.com template solves that problem by building a real-time bridge between Thankster and Intercom. Every time a new address is created in your Thankster account—whether from a first-time buyer or a returning customer updating their details—the workflow instantly triggers. It captures the new address data and uses it to create or update a corresponding user profile in your Intercom workspace. This automation ensures your support, success, and sales teams operate from a single, unified source of customer truth.

How It Works

The automation runs on a simple but powerful trigger-action sequence, requiring zero manual intervention once set up.

Step 1: Monitor for New Thankster Addresses

The workflow uses Make.com's Thankster module to watch for new "address created" events. This can be configured to run instantly via webhook or on a regular schedule, checking your Thankster account for any fresh customer contact information.

Step 2: Map and Prepare the Customer Data

When a new address is detected, the workflow extracts key fields like the customer's name, email address, and the full physical address. It then structures this data into the exact format required by Intercom's API for creating or updating a user. This step often includes data cleaning, like ensuring email formats are correct.

Pro tip: Use this step to enrich the data. You could add a custom attribute like "Thankster_Customer_Since" with the current date, or segment users by country right from the start.

Step 3: Create or Update the User in Intercom

The prepared data packet is sent to Intercom. The workflow uses the customer's email as a unique identifier. If a user with that email already exists in Intercom, their profile is updated with the new address information. If they are a brand new user, a complete new profile is created, instantly making them visible to your support team.

Who This Is For

This automation is ideal for any business that sells physical products or personalized items through Thankster and uses Intercom for customer communication. It's particularly valuable for growing e-commerce brands, subscription box companies, and direct-to-consumer retailers where timely, informed customer support is a key competitive advantage. If your team wastes time manually copying customer data between systems or struggles with outdated contact information, this workflow is for you.

What You'll Need

  1. A Make.com account (free tier available).
  2. An active Thankster account with API access or webhook capabilities.
  3. An active Intercom workspace with admin permissions to generate API keys.
  4. The customer address data you wish to sync (name, email, physical address) must be collected in Thankster.

Quick Setup Guide

  1. Clone the Template: Click "Get This Workflow" and duplicate the scenario into your Make.com account.
  2. Connect Thankster: In the first module, authenticate your Thankster account by entering your API credentials.
  3. Connect Intercom: In the Intercom module, authenticate using an API key from your Intercom workspace settings.
  4. Test the Flow: Add a test address in your Thankster account and run the scenario once to ensure data flows correctly into Intercom.
  5. Activate: Turn the scenario on. It will now run automatically based on your chosen schedule (e.g., instantly or every 15 minutes).
