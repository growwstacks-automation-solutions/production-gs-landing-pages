What This Workflow Does

For e-commerce businesses, the gap between printing a shipping label and getting the order out the door is where delays and errors happen. Manually transferring order details from your shipping platform like Easyship to a project management tool like Trello is a repetitive, time-consuming task that steals focus from value-added work like customer service and growth.

This automated workflow solves that problem instantly. It acts as a digital bridge between Easyship and Trello. Every time a new shipping label is generated in Easyship, this system automatically captures the key details—tracking number, destination, service type—and creates a corresponding, fully-formed card on your designated Trello board. This card can be assigned to team members, moved through custom lists like "To Pack," "Shipped," and "Delivered," and used as a central hub for fulfillment communication.

The business value is clear: elimination of manual data entry, a single source of truth for order status, and a dramatic reduction in the time between label creation and warehouse action. This leads to faster shipping times, fewer missed orders, and a more coordinated, efficient team.

How It Works

The automation is built on Make.com (formerly Integromat), a visual workflow platform that connects apps without coding. Here’s the step-by-step process:

Step 1: Trigger on New Label

The workflow is initiated by the Easyship module, which watches for the event "New Shipping Label Created." This is a real-time trigger, meaning the moment a label is generated in your Easyship account, the automation starts.

Step 2: Extract Label Data

Make.com receives a package of data from Easyship. This includes crucial information such as the tracking number, recipient name and address, parcel dimensions, shipping service selected, and the label's creation timestamp.

Step 3: Format Data for Trello

Before sending data to Trello, the workflow can transform it. It can combine fields (e.g., "Customer: [Name] - [City]"), format dates, or apply logic—like adding a "RUSH" label if the service is express. This step ensures the Trello card is informative and actionable.

Step 4: Create the Trello Card

The formatted data is sent to Trello's "Create a Card" module. You specify the target Board and List (e.g., "Fulfillment Board" > "New Orders"). The card is created with a title (often the tracking # or order ID), a description containing full details, and can include attachments like the label PDF URL, members assigned, due dates, and custom labels.

Pro tip: Use Trello's custom fields to add structured data like "Estimated Delivery Date" or "Package Weight." This makes filtering and reporting on your board much more powerful.

Quick Setup Guide

You can have this workflow running in under 15 minutes.

  1. Clone the Template: Click "Get This Workflow" above to open the template on Make.com and create a copy in your own workspace.
  2. Connect Easyship: In the Make.com scenario, click on the Easyship module. Authenticate by entering your Easyship API key (found in your Easyship account settings under "API & Integrations").
  3. Connect Trello: Click on the Trello module. Authenticate with your Trello account. You will be asked to allow Make.com to access your boards.
  4. Configure the Trello Card: Select the specific Board and List where you want new shipping label cards to appear. Map the data from Easyship (like tracking number, address) to the Trello card's title and description fields.
  5. Test & Activate: Run a single test execution by creating a dummy shipping label in Easyship. Verify that a card appears correctly in your Trello board. Once confirmed, turn on the scenario scheduler to run automatically.
