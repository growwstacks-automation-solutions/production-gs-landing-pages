What This Workflow Does

If your team collects files through Jotform—like design submissions, client briefs, or document approvals—you know the manual drill: download the attachment from the form notification email, figure out which project it belongs to, log into Filestage, navigate to the correct project, and finally upload the file. This process is repetitive, error-prone, and steals valuable time from creative work.

This automation solves that by creating a seamless bridge between Jotform and Filestage. Every time a new form submission is received, the workflow automatically extracts the attached PDF (or other file) and uploads it directly to a specified project in Filestage. The file is instantly ready for review, with stakeholders notified, turning a multi-step manual task into a zero-touch background process. It ensures no submission is ever forgotten and dramatically accelerates the feedback cycle.

The business value is clear: reduced administrative overhead, faster project kick-offs, and a more professional experience for both submitters and reviewers. By automating this transfer, you eliminate a common bottleneck in creative and operational workflows, allowing your team to focus on higher-value tasks like providing feedback and moving projects forward.

How It Works

The workflow acts as an intelligent middleman between your form and your review platform. It listens, processes, and delivers files without human intervention.

Step 1: Trigger on New Jotform Submission

The automation is triggered the moment a new submission hits your Jotform. Make.com watches your form and instantly captures all submission data, including the uploaded file URL, form answers, and submitter details.

Step 2: Process the File Data

The workflow parses the submission to identify the uploaded file. It ensures the file is accessible and prepares it for transfer. You can also map form fields (like project name or client ID) to use as metadata in the next step.

Step 3: Upload to the Correct Filestage Project

Using the Filestage module, the automation authenticates and connects to your account. It locates the target project (which can be static or dynamically chosen based on a form field) and uploads the file directly from Jotform's storage. The file is now live in Filestage, ready for review and annotation.

Pro tip: Use a hidden field in your Jotform to let submitters select the target Filestage project. The automation can read this value and route the file accordingly, making a single form serve multiple teams or clients.

What You'll Need

To set up this automation, you'll need a few accounts and basic information ready:

  1. A Make.com account (free or paid plan).
  2. An active Jotform account with a form that includes a file upload field.
  3. A Filestage account with at least one project created to receive files.
  4. Your Filestage API key (found in your Filestage account settings under Integrations).
  5. The specific Jotform form ID and, optionally, the Filestage project ID you want to target.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button above to open the template in your Make.com account and create a copy.
  2. Connect Jotform: In the first module, authorize Make.com to access your Jotform account and select the form you want to use as the trigger.
  3. Connect Filestage: In the Filestage module, add your API key to authorize the connection. Then, select your target workspace and project from the dropdown list.
  4. Map the File Field: Ensure the "File" field in the Filestage module is mapped to the file URL from the Jotform submission. This is usually done automatically.
  5. Test & Activate: Run a test by submitting your Jotform. Check that the file appears in the correct Filestage project. Once confirmed, turn the scenario on. It will now run automatically for every new submission.
