What This Workflow Does

For businesses using Zendesk for customer support and JIRA Cloud for project management, a critical disconnect often exists. Valuable customer-reported bugs, feature requests, and technical issues get stuck in the support queue, requiring manual effort to re-create as tasks for the development team. This process is slow, error-prone, and vital context like screenshots or log files can be lost.

This automated workflow solves that problem entirely. It acts as a real-time bridge between your support and engineering departments. Whenever a new ticket is created in Zendesk—for instance, one tagged as a "bug" or assigned to a "Development" group—the automation instantly triggers. It captures all ticket details, downloads any attachments, and creates a corresponding, fully-formed issue in your JIRA Cloud project. The attachments are neatly added to the first comment, giving developers immediate access to the evidence they need.

The result is a seamless handoff. Support agents no longer waste time copying information. Developers receive clear, actionable tasks directly from the source, dramatically reducing the "time-to-ticket" and ensuring customer feedback flows directly into the product development cycle.

How It Works

The automation follows a precise, reliable sequence to ensure data integrity and completeness.

Step 1: Monitoring New Zendesk Tickets

The workflow starts with a Zendesk module configured to watch for new tickets. You can set filters based on ticket tags, group, priority, or custom fields to ensure only relevant tickets (like bugs or feature requests) trigger the process. This prevents JIRA from being flooded with non-technical support inquiries.

Step 2: Extracting Ticket Data & Attachments

Once a matching ticket is detected, the automation pulls all crucial information: the ticket subject, description, requester details, priority, and any tags. Simultaneously, it checks for attachments. If attachments exist, it securely downloads them from Zendesk, preparing them for transfer.

Pro tip: For this to work, you must disable the "secure downloads" option in your Zendesk ticket settings (under Attachments). This allows the automation to access the files.

Step 3: Creating the JIRA Issue

The core action is creating a new issue in your specified JIRA Cloud project. The workflow maps the Zendesk data to JIRA fields: subject becomes the summary, description becomes the description, and you can map the requester, set the issue type (Bug, Story), assignee, and priority. This creates a ready-to-work task for your development team.

Step 4: Attaching Files & Adding Context

Finally, the downloaded attachments are uploaded and attached to the first comment on the newly created JIRA issue. This comment can also include a link back to the original Zendesk ticket for full context. The issue is now live in JIRA with all necessary information and evidence, enabling developers to start work immediately.

Quick Setup Guide

Follow these steps to configure the template for your business:

  1. Clone the Template: Use the "Get This Workflow" button to copy the template into your Make.com account.
  2. Connect Your Apps: In Make.com, authorize the Zendesk and JIRA Cloud modules using your admin credentials or API tokens.
  3. Configure the Trigger: Set the Zendesk module to watch for "New Ticket" events. Apply filters (e.g., ticket tags contain "bug") to target only relevant tickets.
  4. Map Your Fields: In the JIRA "Create Issue" module, map the Zendesk data (subject, description) to the corresponding JIRA fields. Define your project, issue type, and any default assignees.
  5. Adjust Attachment Settings: In your Zendesk Admin settings, navigate to Settings > Tickets > Attachments and disable "Secure downloads."
  6. Test the Flow: Create a test ticket in Zendesk with a dummy attachment and verify a corresponding issue appears in your JIRA project with the file attached.
  7. Activate: Turn on the scenario in Make.com. Your automated bridge between support and development is now live.
