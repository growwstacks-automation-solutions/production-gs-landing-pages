What This Workflow Does

For support teams using Freshdesk, manually transferring ticket due dates to a shared calendar is a repetitive, error-prone task. It leads to missed deadlines, double-booked agents, and a lack of visibility into team workload. This disconnect between your support ticketing system and your scheduling calendar creates operational blind spots and reduces accountability.

This automation workflow solves that by creating a live, two-way sync between Freshdesk and Microsoft 365 Calendar. Whenever a due date is set or updated on a Freshdesk ticket, the workflow automatically creates a corresponding event in your designated Microsoft 365 Calendar. If the due date changes later, the calendar event is updated instantly. This ensures your team's calendar is always an accurate, real-time reflection of support commitments, eliminating manual entry and preventing oversights.

How It Works

The workflow acts as an intelligent bridge, monitoring Freshdesk for changes and managing your Microsoft 365 Calendar accordingly.

Step 1: Monitor Freshdesk for Ticket Updates

The automation continuously watches your Freshdesk instance for any tickets that are updated. It specifically filters for tickets where a "due date" field has been added or modified, ensuring it only acts on relevant changes.

Step 2: Create or Find the Calendar Event

For each updated ticket, the workflow checks your Microsoft 365 Calendar to see if an event for that ticket already exists (typically using the ticket ID in the event title). If it doesn't find one, it proceeds to create a new event.

Step 3: Build the Calendar Event Details

The system constructs a detailed calendar event using data from the Freshdesk ticket. The event title includes the ticket ID and subject for easy identification. The start and end times are set based on the ticket's due date and time. Additional details like the ticket description, priority, and a direct link back to the ticket in Freshdesk are added to the event description.

Step 4: Update or Create the Event

If an existing event was found in Step 2, the workflow updates it with the new due date and any other changed information. If no event was found, it creates a brand new one in the calendar. This step ensures the calendar always has the most current information.

Pro tip: Configure the workflow to add the assigned agent's email as an attendee. This automatically sends them a calendar invitation, ensuring they get a direct notification of the deadline.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps:

  1. Clone the Template: Click "Get This Workflow" and duplicate the provided Make.com scenario into your own account.
  2. Connect Freshdesk: In the first module, authorize Make.com to access your Freshdesk account using an API key or OAuth. Select the specific Freshdesk event you want to watch (e.g., "Updated Ticket").
  3. Connect Microsoft 365: Authorize the connection to your Microsoft 365 account, granting permissions to read and write to Calendars.
  4. Map the Data: Configure the modules to map the correct Freshdesk fields (ticket ID, subject, due date, description) to the corresponding Microsoft Calendar event fields (title, start/end time, description).
  5. Set the Calendar: Specify the exact Microsoft 365 Calendar ID (usually the email address of the calendar) where events should be created or updated.
  6. Test & Activate: Run a test by updating a ticket's due date in Freshdesk. Check your Microsoft 365 Calendar to confirm the event appears correctly. Once verified, activate the scenario to run automatically.
