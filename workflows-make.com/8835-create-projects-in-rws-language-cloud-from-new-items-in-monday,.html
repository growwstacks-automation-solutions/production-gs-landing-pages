What This Workflow Does

Managing translation projects manually is a time-consuming and error-prone process. Project managers often waste hours copying information from a task management tool like Monday.com into a translation management platform like RWS Language Cloud. This manual handoff leads to delays, miscommunication about deadlines, and data entry mistakes that can derail a project before it even starts.

This automated workflow solves that problem by creating a seamless bridge between your project planning and execution phases. Whenever a new item is added to a specific Monday.com board—signaling a new translation request—this Make.com scenario automatically triggers. It captures all the crucial details (like source file, target languages, and deadline) and uses them to create a corresponding project in RWS Language Cloud. The result is a frictionless pipeline where approved requests instantly become active translation jobs.

The automation ensures nothing falls through the cracks. It provides a reliable, hands-off system that guarantees every request is processed consistently and immediately, giving your linguists more time to focus on quality and dramatically improving project turnaround times for your clients or internal stakeholders.

How It Works

The workflow operates on a simple but powerful trigger-action principle, orchestrating data between two critical business platforms.

Step 1: Monitor Monday.com for New Items

The scenario starts by periodically checking a designated Monday.com board. You can configure it to watch a specific board and column, such as a "Status" column changing to "Ready for Translation." This ensures the automation only acts on finalized, approved requests.

Step 2: Retrieve and Map Item Data

Once a new qualifying item is found, the workflow retrieves all its relevant data. This includes the item name (which often becomes the project name), custom field values for source/target languages, deadlines, and links to source documents stored in Google Drive, Dropbox, or another integrated cloud service.

Step 3: Create Project in RWS Language Cloud

Using the RWS Language Cloud API module in Make.com, the workflow sends a structured request to create a new translation project. It maps the Monday.com data to the required fields in RWS, such as project name, client, language pair, due date, and attaches the source file from the provided link.

Step 4: Confirm and Update Records

After successfully creating the project, the workflow can capture the new RWS project ID and update the original Monday.com item with this reference. This creates a two-way link, allowing teams to track the translation project's progress directly from their Monday.com dashboard.

Pro tip: Use Monday.com's "Status" column as your trigger gate. Set the automation to run only when status is "Approved" or "Ready for Translation." This prevents accidental project creation from draft or pending requests.

What You'll Need

To implement this template successfully, you will need the following accounts and setup:

  1. A Make.com account with an active subscription plan that allows for multiple API operations.
  2. Admin or member access to a Monday.com workspace with a board dedicated to translation requests.
  3. An RWS Language Cloud account with API access enabled. You'll need your API credentials (client ID and secret).
  4. Your Monday.com board should have consistent columns for key data: Source Language, Target Language, Deadline, and Source File Link.
  5. (Recommended) The source files for translation should be accessible via a cloud storage link (e.g., Google Drive, Dropbox) added to the Monday item.

Quick Setup Guide

Follow these steps to configure the template for your own use:

  1. Clone the Template: Click "Get This Workflow" to copy the scenario into your Make.com account.
  2. Connect Your Apps: In the first module, reconnect the Monday.com module to your own account. Specify the exact Board and Column you want to monitor for new items.
  3. Configure RWS Language Cloud: In the RWS module, enter your API credentials. Test the connection to ensure Make.com can communicate with your RWS account.
  4. Map Your Data Fields: Adjust the data mapping between Monday.com columns and RWS project fields. Ensure the source language, target language, and deadline information are pulled from the correct Monday columns.
  5. Set the Schedule: Configure how often the scenario should check Monday.com for new items (e.g., every 15 minutes, hourly).
  6. Test with a Dummy Item: Turn on the scenario and create a test item in your Monday board. Verify that a corresponding test project is created in your RWS Language Cloud sandbox or a test folder.
  7. Go Live: Once testing is successful, your automation is ready. New translation requests added to Monday will now automatically spawn projects in RWS.
