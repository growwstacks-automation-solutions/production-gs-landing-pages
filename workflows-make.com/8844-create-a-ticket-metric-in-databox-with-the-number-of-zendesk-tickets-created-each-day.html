What This Workflow Does

Customer support teams often struggle to translate daily ticket volume into actionable business intelligence. Manually counting tickets and updating dashboards is time-consuming, error-prone, and creates lag in decision-making. This automation solves that by seamlessly connecting Zendesk, your support hub, with Databox, your visualization platform.

The workflow automatically queries your Zendesk instance, counts the number of tickets created each day, and pushes that metric directly into your designated Databox dashboard. It processes all historical tickets in a one-time run to establish a baseline, after which you can use a companion daily update workflow to keep the metric current. This provides leadership with an always-accurate, visual representation of support demand trends.

How It Works

The automation follows a logical sequence to ensure data accuracy and seamless integration between the two platforms.

Step 1: Trigger & Date Range Setup

The scenario can be triggered manually for the initial historical import. You define the date range for which you want to process tickets, allowing you to backfill data for weeks, months, or even years.

Step 2: Fetch Tickets from Zendesk

Using the Zendesk API module, the workflow retrieves all tickets created within the specified date period. It handles pagination automatically to ensure no ticket is missed, regardless of volume.

Step 3: Aggregate & Count by Day

The core logic of the workflow groups the retrieved tickets by their creation date. It then counts how many tickets were created on each individual day within your selected range.

Step 4: Push Metric to Databox

For each unique day, the workflow sends the corresponding ticket count to Databox via its API. It creates or updates a specific metric in your connected Databox account, tagging each data point with the correct date.

Pro tip: Run this template once to establish your historical baseline. Then, schedule the separate "daily update" template to run automatically each morning, ensuring your dashboard reflects yesterday's volume without any manual intervention.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps.

  1. Get the Template: Click the "Get This Workflow" button above to open the template in your Make.com account and create a copy.
  2. Connect Zendesk: In the first module, click to add a connection. Use your Zendesk subdomain and API token to authenticate the app.
  3. Connect Databox: Similarly, authenticate the Databox module with your account credentials or API key from the Databox developer settings.
  4. Configure the Metric: Specify the exact name for your new metric in Databox (e.g., "Daily Zendesk Tickets").
  5. Set the Date Range: For the initial run, define the start and end dates for your historical data import.
  6. Test Run: Run the scenario once manually to verify it fetches tickets and posts data to Databox correctly.
  7. Schedule Updates (Optional): Once confirmed, set up the separate daily update template on a schedule to run every 24 hours automatically.
