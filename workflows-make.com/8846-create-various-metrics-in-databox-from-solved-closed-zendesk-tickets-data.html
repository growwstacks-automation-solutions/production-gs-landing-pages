What This Workflow Does

For support teams using Zendesk, valuable data is locked away in every closed ticket—resolution time, requester details, channel, and more. Manually extracting this data to create reports is a tedious, weekly chore that consumes hours better spent helping customers. This automation solves that by instantly sending data from every solved or closed Zendesk ticket directly to your Databox account.

The workflow automatically creates and updates key metrics in Databox, turning raw ticket information into actionable visualizations. You can track trends in support volume, monitor average resolution times, and measure team performance without ever opening a spreadsheet. This means your dashboards are always current, providing leadership with real-time insights to make data-driven decisions about resource allocation, training needs, and customer satisfaction.

Pro tip: Use this automation to create a live "Support Wallboard" displayed on a monitor in your team's area. Seeing real-time metrics like tickets solved today or average resolution time fosters healthy competition and keeps performance goals top of mind.

How It Works

The process is a seamless, hands-off pipeline from ticket closure to business intelligence.

Step 1: Trigger on Ticket Closure

The workflow is activated whenever a ticket's status in Zendesk changes to "Solved" or "Closed." Make.com watches for this specific event, ensuring no manually resolved ticket is missed.

Step 2: Extract Key Ticket Data

Once triggered, the workflow pulls the relevant data from the ticket. This typically includes the ticket ID, the time it was solved, the assignee (agent), the requester, and any custom fields you've defined, such as priority or category.

Step 3: Transform & Prepare Data for Databox

The raw data is then formatted to match the specific metric requirements in Databox. For example, the time difference between ticket creation and solution is calculated to populate an "Average Resolution Time" metric. The data is structured into the precise JSON format Databox expects via its API.

Step 4: Push Metrics to Your Databox Dashboard

Finally, the formatted data is sent to Databox using its API connection. It creates or updates the corresponding metrics in your designated Databox account. Your dashboard widgets—like line charts for ticket volume or gauges for SLA compliance—immediately reflect the new data point.

What You'll Need

  1. A Make.com Account: Free tier is sufficient to run this automation.
  2. An Active Zendesk Subscription: With administrator access to create an API token and set up the connection.
  3. A Databox Account: With a project created where you want the metrics to appear.
  4. API Credentials: Your Zendesk subdomain and API token, plus your Databox API key. These are found in the respective admin settings of each platform.
  5. Clear KPIs in Mind: Decide which 2-5 metrics are most critical for your team to track from the start (e.g., Tickets Solved Today, Avg. First Reply Time).

Quick Setup Guide

You can have this automation running in under 30 minutes.

  1. Clone the Template: Click "Get This Workflow" to copy the template into your Make.com account.
  2. Connect Zendesk: In the first module, add your Zendesk connection using your subdomain and API token. Test the connection to ensure it works.
  3. Connect Databox: In the Databox module, add your connection using your Databox API key. Specify the project name where metrics should be sent.
  4. Map Your Data: Review the data mapping. The template is pre-configured for common fields like ticket ID and solved time. You can add custom Zendesk fields if needed (e.g., "Priority" or "CSAT Score").
  5. Activate & Test: Turn the scenario on. Solve a test ticket in Zendesk and verify that a new data point appears in your Databox dashboard within a minute.

Pro tip: Start simple. Begin by tracking just two core metrics like "Daily Solved Tickets" and "Average Handle Time." Once you see the value, you can expand the workflow to include more complex calculations or split data by agent or channel.
