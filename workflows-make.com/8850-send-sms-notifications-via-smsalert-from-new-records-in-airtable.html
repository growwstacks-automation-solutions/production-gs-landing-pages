What This Workflow Does

Manual communication is a major bottleneck for growing businesses. When a new customer signs up, an appointment is booked, or an order is placed, someone has to stop their work, find the contact details, and send a message. This process is slow, prone to errors, and doesn't scale.

This automation solves that by creating a direct bridge between your data in Airtable and your customers' phones via SMSAlert. The moment a new record with a name and phone number is added to your specified Airtable table, the workflow springs into action. It formats a personalized message and sends it as an SMS, all without any human intervention. This ensures timely, reliable, and professional communication for every new entry.

Whether you're confirming appointments, notifying teams of new leads, or alerting customers about order status, this workflow turns your Airtable base into a powerful notification hub. It runs on a schedule you control (default every 15 minutes), guaranteeing that no new record is ever missed.

How It Works

The workflow is a simple yet powerful sequence on Make.com, acting as the conductor between your data and your messaging.

Step 1: Scheduled Check for New Records

A "Schedule" module is set to trigger the workflow at regular intervals. By default, it checks your Airtable base every 15 minutes. This module kicks off the entire process, ensuring the automation runs consistently and reliably.

Step 2: Fetch New Data from Airtable

Immediately after the schedule triggers, an "Airtable" module searches your specified table for new records. It is configured to look for records that haven't been processed yet—often by checking a "Status" field or by identifying the most recently created entries. It retrieves the crucial data: the recipient's name and their phone number.

Step 3: Format and Send the SMS via SMSAlert

The workflow then takes the retrieved data and passes it to an "SMSAlert" module. Here, you define your message template. You can personalize it by inserting the customer's name or other details from Airtable (e.g., "Hi [Name], your appointment is confirmed for tomorrow at 3 PM."). The module uses your SMSAlert API credentials to send the formatted message to the provided phone number instantly.

Step 4: Logging and Error Handling (Optional)

For robustness, you can extend the workflow to log successful sends back to Airtable (e.g., updating a "Notification Sent" field) or to handle errors. If an SMS fails to send—perhaps due to an invalid number—the workflow can route that record to an error log or send an alert to your team, ensuring you can fix issues promptly.

Pro tip: Use a dedicated "Status" field in your Airtable (like "SMS Sent") and have the workflow update it after successful sending. This prevents duplicate messages if the same record is fetched again on the next run.

Quick Setup Guide

You can have this workflow live in under 15 minutes by following these steps:

  1. Clone the Template: Click "Get This Workflow" to copy the template into your own Make.com account.
  2. Connect Airtable: In the first Airtable module, click to add a connection. Authorize Make.com to access your Airtable workspace and select the correct base and table.
  3. Configure the Search: Set the module to search for records in a specific view. You may want to add a filter, like "Where Status is empty," to only fetch unprocessed records.
  4. Connect SMSAlert: In the SMSAlert module, add your API credentials from your SMSAlert account dashboard.
  5. Craft Your Message: In the SMSAlert module, write your message template. Use the mapping panel to insert data from Airtable (e.g., the customer's name) by clicking on the fields from the previous module.
  6. Set the Schedule: Adjust the Schedule module's interval from 15 minutes to whatever suits your needs—instant, hourly, or daily.
  7. Test and Activate: Run the scenario once to test it. Add a test record to your Airtable and see if the SMS comes through. Once confirmed, activate the scenario to run automatically.
