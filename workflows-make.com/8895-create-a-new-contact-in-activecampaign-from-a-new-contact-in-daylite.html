What This Workflow Does

For businesses using Daylite for sales pipeline management and ActiveCampaign for marketing automation, a major operational bottleneck is manually transferring new contact information between the two systems. Sales teams add promising leads to Daylite, but marketing can't engage them until someone copies the data over—a process prone to delays, errors, and missed opportunities.

This automated workflow solves that problem completely. It acts as a real-time bridge between your CRMs. Every time a salesperson creates a new contact in Daylite, this Make.com scenario instantly and accurately creates a corresponding contact in ActiveCampaign. Beyond just basic details, it ensures the contact is added to the correct marketing list (like "New Leads" or "Sales Qualified") and even passes along the unique Daylite Contact ID into a custom field in ActiveCampaign. This creates a permanent, searchable link between the contact record in both platforms.

The result is a seamless, synchronized system where marketing can begin nurturing a lead the moment sales identifies them, dramatically improving conversion rates and team alignment while saving hours of administrative work each week.

How It Works

The automation follows a clean, reliable sequence to ensure data integrity and useful outcomes.

Step 1: Trigger on New Daylite Contact

The workflow is initiated the instant a new contact is created within your Daylite account. Make.com watches for this event using the Daylite module's "Watch Contacts" trigger, capturing the full contact data payload.

Step 2: Map and Prepare Contact Data

The captured data—name, email, phone, company, and any custom fields—is then mapped to the corresponding fields in ActiveCampaign. You can customize this mapping to include only the data you need, ensuring a clean transfer.

Pro tip: Use this step to also assign a default "Lead Source" value in ActiveCampaign, like "Daylite Sales," to track where these high-quality contacts originated.

Step 3: Create Contact in ActiveCampaign

Make.com uses the ActiveCampaign module to create a new contact with the mapped data. The action is configured to add the contact to one or more specific lists you define, immediately enrolling them in relevant email sequences or tagging them for sales follow-up.

Step 4: Link Records with Custom Field

A critical final step is updating the newly created ActiveCampaign contact with a custom field value—the original Daylite Contact ID. This creates a two-way reference, allowing anyone to quickly find the linked record in the other system, which is invaluable for support and detailed reporting.

What You'll Need

To implement this automation, ensure you have the following set up and ready:

  1. A paid Make.com account with available operations.
  2. An active Daylite account with API access enabled.
  3. An active ActiveCampaign account with API credentials.
  4. Administrator access or the necessary permissions in both Daylite and ActiveCampaign to create API connections.
  5. A clear understanding of which ActiveCampaign list you want new contacts added to.
  6. (Optional) A custom field created in ActiveCampaign to store the Daylite Contact ID.

Quick Setup Guide

Follow these steps to configure the template for your business:

  1. Clone the Template: Click "Get This Workflow" to copy the scenario to your Make.com account.
  2. Connect Your Apps: In the Make.com scenario, update the Daylite and ActiveCampaign modules with your own API credentials. Test the connections.
  3. Configure the List: In the "Create a contact" ActiveCampaign module, specify the exact list ID or name where new contacts should be added.
  4. Map Your Fields: Review the data mapping between Daylite and ActiveCampaign. Adjust field mappings to match your custom field names in both systems.
  5. Set the Custom Field: Ensure the "Update contact" module is correctly pointed to the custom field you created in ActiveCampaign for the Daylite ID.
  6. Test with a Dummy Contact: Run the scenario once manually by creating a test contact in Daylite. Verify it appears correctly in the designated ActiveCampaign list with the ID field populated.
  7. Activate: Turn on the scenario's scheduler. It will now run automatically for every new Daylite contact.
