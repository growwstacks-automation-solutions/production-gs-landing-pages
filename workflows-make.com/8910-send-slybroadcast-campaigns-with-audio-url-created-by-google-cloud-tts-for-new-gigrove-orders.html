Automated Voice Campaigns for Gigrove Orders

Automatically convert new order details into personalized voice messages and send them as broadcast campaigns via Slybroadcast.

Get This Workflow → Make.com · Gigrove · Free Template

What This Workflow Does

For businesses using Gigrove or similar order management systems, communicating order details to customers or staff manually is time-consuming and prone to delays. This automation solves that by instantly turning raw order data—like customer names, order totals, or delivery times—into clear, natural-sounding voice messages and delivering them directly to a list of phone numbers via Slybroadcast.

The workflow watches for new Gigrove orders at regular intervals. When an order appears, it extracts the specific details you want to communicate, uses Google Cloud Text-to-Speech to generate a high-quality audio file, securely stores that file in Google Drive, and then launches a Slybroadcast campaign to send the voice message. This creates a seamless, hands-off communication channel that ensures timely, personalized outreach without any manual intervention.

Pro tip: This isn't just for order confirmations. You can adapt it for appointment reminders, delivery status updates, urgent alerts, or internal team notifications—any scenario where a voice call is more effective than text.

How It Works

Step 1: Monitoring New Orders

The Make.com scenario is configured to watch your Gigrove account for new orders. It can poll at intervals you set (e.g., every 15 minutes) or trigger instantly via a webhook if Gigrove supports it. Each new order record is captured, and the relevant data fields (like customer phone number, order ID, items, total) are mapped for the next steps.

Step 2: Generating the Voice Message

Using Google Cloud Text-to-Speech (TTS), the workflow constructs a spoken message from the order data. You define a template—for example, "Hello [Customer Name], your order #[Order ID] for [Item] totaling [Amount] has been confirmed." Google TTS synthesizes this text into a natural, studio-quality audio file in your chosen language and voice style.

Step 3: Storing & Linking the Audio

The generated audio file is automatically uploaded to a designated folder in your Google Drive. The workflow then creates a shareable link for the file. This link is used by Slybroadcast to access and play the audio, ensuring the voice message is always available and auditable.

Step 4: Launching the Voice Campaign

Finally, the scenario sends the Google Drive audio link, along with the target phone number(s) from the order, to Slybroadcast. Slybroadcast initiates a voice campaign, dialing the number(s) and playing the personalized message. You can configure it to send to a single customer, a group of staff members, or any list you define.

Quick Setup Guide

Once you have the template, follow these steps to get it live:

  1. Clone the template: Click "Get This Workflow" to copy the scenario into your Make.com account.
  2. Connect your apps: In each module (Gigrove, Google Cloud TTS, Google Drive, Slybroadcast), replace the placeholder connections with your actual API keys and account credentials.
  3. Configure the trigger: Set the Gigrove module to watch for "New Orders" at an interval that suits your business pace.
  4. Customize the message template: Edit the text that is sent to Google TTS. Include the dynamic fields from Gigrove (like customer name, order ID) that you want voiced.
  5. Set your target phone list: Define where Slybroadcast gets the recipient phone numbers—from the Gigrove order data or from a static list you manage.
  6. Test with a live order: Create a test order in Gigrove and run the scenario once to verify the audio is generated, uploaded, and the call is sent successfully.
  7. Schedule and activate: Set the scenario to run automatically on your chosen schedule and activate it.
