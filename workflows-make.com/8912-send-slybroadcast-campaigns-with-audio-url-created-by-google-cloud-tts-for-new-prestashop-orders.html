Automate Audio Order Notifications from PrestaShop

Convert new order details into personalized audio messages and send SMS/voice campaigns instantly via Slybroadcast—no manual work required.

What This Workflow Does

For e-commerce businesses using PrestaShop, manually confirming orders and updating customers is a repetitive, time-consuming task. This automation solves that by instantly converting new order data into a personalized audio message and broadcasting it to your customers via SMS or voice call.

The workflow watches your PrestaShop store for new orders. When an order is placed, it extracts key details like the customer name, order number, total amount, and items. Using Google Cloud Text-to-Speech, it synthesizes a natural-sounding audio file summarizing this information. That file is uploaded to your Google Drive, a shareable link is created, and finally, Slybroadcast sends the audio as a campaign to a designated list of phone numbers.

This creates a seamless, automated communication channel that enhances customer experience with timely, professional updates while freeing your team from manual notification duties.

How It Works

Step 1: Monitor New PrestaShop Orders

The Make.com scenario triggers on a schedule (e.g., every 5 minutes) to check your PrestaShop store for new orders. It fetches the order object containing all relevant details you wish to communicate.

Step 2: Generate Audio with Google Cloud TTS

The workflow passes specific order values (customizable by you) to Google Cloud Text-to-Speech. The service converts the text into a high-quality, spoken audio file in your chosen language and voice profile.

Step 3: Upload & Share via Google Drive

The generated audio file is automatically uploaded to a folder in your Google Drive. A shareable public link is created for the file, which will be used by Slybroadcast to access the audio content.

Step 4: Broadcast Campaign via Slybroadcast

Using the audio file URL from Drive, Slybroadcast initiates a campaign. It sends the audio message as an SMS with a playable link or as a direct voice call to the list of recipients—typically the customer's phone number from the order.

Pro tip: You can customize the audio script to include promotional messages, delivery estimates, or thank-you notes, turning a simple notification into a marketing touchpoint.

What You'll Need

  1. A PrestaShop store with API access or webhook capability to trigger on new orders.
  2. A Make.com account to build and run the automation scenario.
  3. A Google Cloud account with Text-to-Speech API enabled and credentials.
  4. A Google Drive account for storing the generated audio files.
  5. A Slybroadcast account with campaign sending capabilities.
  6. The customer phone numbers you intend to notify (usually from the order data).

Quick Setup Guide

1. Import the template into your Make.com account and connect your PrestaShop module.

2. Configure the Google Cloud TTS module: select your preferred voice, language, and speaking rate.

3. Set up the Google Drive upload path and sharing permissions.

4. Connect your Slybroadcast account and define the recipient list (you can map it directly from the order's phone field).

5. Customize the text template that will be converted to speech—include order number, customer name, total, and a friendly message.

6. Test the flow with a dummy order and verify the audio campaign is sent correctly.

7. Schedule the scenario to run at your desired frequency (real-time or batch).
