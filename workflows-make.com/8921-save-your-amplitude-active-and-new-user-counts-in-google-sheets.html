What This Workflow Does

For product managers, growth marketers, and data analysts, manually checking Amplitude and updating spreadsheets is a repetitive, time-consuming task. This process is prone to errors and creates delays in reporting, leaving stakeholders without the latest user engagement insights when they need them.

This automation solves that by creating a hands-free pipeline between your Amplitude analytics and Google Sheets. It acts as a scheduled data collector, fetching your defined active and new user counts at regular intervals you set (e.g., daily, hourly) and appending them as new rows in your designated spreadsheet. This transforms your Google Sheet into a living, historical record of your user base growth.

How It Works

The workflow runs on a simple, reliable schedule, ensuring your data is always current.

Step 1: Scheduled Trigger

The automation is initiated by a Schedule module in Make.com. You configure it to run at your desired frequency—whether that's every morning at 9 AM for a daily report or every 6 hours for more frequent monitoring.

Step 2: Fetch Data from Amplitude

Once triggered, the scenario uses the Amplitude module to call the Amplitude API. It authenticates using your API key, specifies your project, and requests the metrics for "active users" and "new users" for the previous time period (e.g., last 24 hours).

Step 3: Format and Prepare Data

The raw numbers from Amplitude are then structured into a clean format. The automation typically adds a timestamp for the data point and organizes the metrics into distinct columns, ready for insertion into your spreadsheet.

Step 4: Update Google Sheets

Finally, the workflow connects to your specified Google Sheets file via the Google Sheets module. It adds a new row to your sheet, populating the columns with the date, active user count, and new user count. This creates a perfect log over time.

Pro tip: Use the timestamp column created by this automation to build time-series charts directly in Google Sheets or in connected BI tools like Looker Studio, giving you instant visual trends of user growth.

What You'll Need

To get this automation running in under 10 minutes, ensure you have the following:

  1. A Make.com account (free tier is sufficient to start).
  2. An Amplitude account with access to the project containing the user metrics you want to track.
  3. Your Amplitude API Key and Secret Key (found in your Amplitude project settings).
  4. A Google Sheets spreadsheet set up with column headers (e.g., Date, Active Users, New Users).
  5. Editor access to that Google Sheet to allow Make.com to add rows.

Quick Setup Guide

Follow these steps to configure your template after importing it to Make.com:

  1. Connect Amplitude: In the first Amplitude module, click to create a connection. Enter your API Key and Secret Key from your Amplitude project settings.
  2. Set Your Project & Metrics: In the same module, select your Amplitude Project ID and ensure the metrics are set to "active_users" and "new_users".
  3. Connect Google Sheets: In the Google Sheets module, authorize Make.com to access your Google account and select the specific spreadsheet and worksheet where you want the data to land.
  4. Configure the Schedule: Set the Schedule module to run at your preferred interval (e.g., daily at 8 AM UTC).
  5. Test and Activate: Run the scenario once manually to ensure data flows correctly into your sheet. Then, turn the scenario "ON" to let it run automatically.
