What This Workflow Does

For freelancers, consultants, and agencies, tracking time accurately is the lifeline of the business. You might start your day logging hours in Toggl for its simplicity and powerful tracking, but your invoicing, reporting, or project management happens in Timing. Manually copying time entries from one app to another is a tedious, error-prone task that wastes valuable hours each month. This workflow solves that problem permanently.

This free Make.com template creates a seamless, automated bridge between Toggl and Timing. Whenever you or your team logs time in Toggl, this automation detects the new entry and creates a corresponding, detailed time entry in Timing automatically. It transfers all the crucial information—description, duration, project, and tags—ensuring your data is consistent, accurate, and ready for action in both systems. It turns two separate tools into one cohesive time-tracking powerhouse.

The business value is clear: reclaim lost administrative time, eliminate revenue leakage from unbilled hours, and gain a single source of truth for all your time data. This automation isn't just about convenience; it's about protecting your profitability and scaling your operations efficiently.

How It Works

The automation runs on a simple, reliable trigger-action model powered by Make.com. Here’s a step-by-step breakdown of the process:

Step 1: Monitoring Toggl for New Entries

The scenario starts by polling your Toggl account at regular intervals you define (e.g., every 15 minutes). It uses the Toggl API to check for any new time entries that have been created since the last check. This is the "trigger" that kicks off the entire workflow.

Step 2: Extracting and Structuring the Data

When a new entry is found, Make.com retrieves all its details. This includes the entry's description, start and end timestamps, calculated duration, associated project name, client name, tags, and any notes. The workflow structures this data, handling any formatting differences between the two platforms.

Step 3: Creating the Entry in Timing

Using the prepared data, the scenario then calls the Timing API. It creates a new time entry in your specified Timing workspace or project. The entry in Timing will mirror the one from Toggl, containing all the transferred metadata so it's immediately usable for reporting, client invoicing, or internal analysis.

Pro tip: You can add a filter module between Step 2 and Step 3 to only sync entries that are billable, tagged with a specific client, or over a certain duration. This keeps your Timing workspace clean and focused.

To implement this workflow, you'll need a few things set up beforehand:

  1. A Make.com account (free tier available). This is the platform that will host and run your automation.

Quick Setup Guide

Getting this automation live is straightforward. Follow these steps after cloning the template in Make.com:

  1. Connect Your Apps: In the Make.com scenario editor, click on the Toggl module and add your connection using your API token. Repeat the process for the Timing module with your Timing API key.
  2. Configure the Trigger: Set the polling schedule for the Toggl module. For most users, checking every 30 minutes or hourly is sufficient and stays within API limits.
  3. Map the Data Fields: The template pre-maps common fields like description and duration. Review the mapping between the Toggl data and the Timing "Create Time Entry" module to ensure project names and tags transfer correctly for your use case.
  4. Add Optional Filters (Recommended): Insert a filter module to only sync entries that are "billable" or tagged with a specific client. This prevents syncing internal or break time.
  5. Test and Activate: Run the scenario once manually to test. Create a new time entry in Toggl and see if it appears in Timing correctly. Once confirmed, turn the scenario "ON" to let it run automatically.
