What This Workflow Does

For any business using Sendinblue (now Brevo) for email marketing, the quality of your contact list is everything. Invalid, risky, or spam trap email addresses lead to high bounce rates, poor deliverability, wasted budget, and damaged sender reputation. Manually checking each new sign-up, lead, or subscriber is impractical at scale.

This automation solves that problem. It connects Sendinblue with MailerCheck, a dedicated email verification service. Every 15 minutes (or on your custom schedule), the workflow checks for new contacts added to your Sendinblue account. It sends each new email address to MailerCheck for validation, receives a clear status (like Valid, Risky, Invalid), and then updates a custom contact attribute in Sendinblue—named "MAILERCHECK"—with that result. This creates a live, verified segmentation layer within your CRM without any manual work.

The result is a perpetually clean list. You can confidently send campaigns only to verified addresses, tailor sequences for risky contacts, and exclude invalid ones automatically. It transforms list hygiene from a periodic, painful audit into a seamless, real-time process.

How It Works

Step 1: Trigger on New Contacts

The workflow is triggered by a schedule module, running automatically at intervals you set (default: every 15 minutes). It then uses the Sendinblue module to search for contacts created since the last run. This ensures every new addition is captured promptly.

Step 2: Extract & Send Emails for Verification

For each new contact retrieved, the workflow extracts the email address and passes it to the MailerCheck module. MailerCheck performs a multi-layer validation: checking syntax, domain existence, mailbox availability, and spam trap indicators.

Step 3: Receive & Process Validation Status

MailerCheck returns a definitive status for each email. The workflow maps this result (e.g., "valid", "risky", "invalid", "unknown") into a standardized value ready for Sendinblue.

Step 4: Update Sendinblue Contact Attribute

The final step uses Sendinblue again to update the specific contact. It adds or modifies the custom attribute "MAILERCHECK" with the verification status. This attribute is now available for segmentation, filtering, and personalization within all your Sendinblue campaigns.

Pro tip: You can extend this workflow easily. Add a step to move invalid contacts to a separate list, send an internal alert for risky addresses, or trigger a different onboarding flow based on the MAILERCHECK status.

What You'll Need

To set up this template successfully, ensure you have the following prerequisites:

  1. A Make.com account (free or paid) to host and run the automation.
  2. A Sendinblue (Brevo) account with an API key. You'll need access to your contact lists and the ability to create custom attributes.
  3. A MailerCheck account and API key. MailerCheck offers pay-as-you-go verification credits.
  4. Basic understanding of how to connect apps in Make.com using API keys or OAuth.
  5. The custom attribute "MAILERCHECK" created in your Sendinblue account (or you can modify the workflow to use a different attribute name).

Quick Setup Guide

Follow these steps to deploy the workflow in under 10 minutes:

  1. Clone the template: Use the "Get This Workflow" button above to open the template on Make.com and create a copy in your own account.
  2. Connect Sendinblue: In the first Sendinblue module, add your Sendinblue API key. Test the connection by retrieving a sample contact.
  3. Connect MailerCheck: In the MailerCheck module, add your MailerCheck API key. You may need to purchase verification credits if you haven't already.
  4. Configure the schedule: The trigger runs every 15 minutes. Adjust this interval based on your contact volume—more frequent for high-volume lists, less frequent for smaller lists.
  5. Run a test: Add a new contact to your Sendinblue list manually. Run the scenario once to see it fetch, verify, and update the attribute. Check the Sendinblue contact profile to confirm the MAILERCHECK attribute is populated.
  6. Activate: Turn the scenario on. It will now run automatically, keeping your list verified continuously.
