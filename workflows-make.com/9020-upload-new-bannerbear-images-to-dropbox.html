What This Workflow Does

For marketing teams, designers, and content creators, Bannerbear is a powerful tool for generating on-brand visuals at scale. However, managing the output—downloading images, renaming files, and organizing them in cloud storage—creates a significant manual bottleneck. This repetitive task is prone to error, leading to lost assets, disorganized folders, and wasted creative time.

This automation solves that problem completely. It acts as a silent, reliable bridge between Bannerbear and Dropbox. Every time a new image is generated in your Bannerbear account, the workflow is triggered. It fetches the image file and automatically uploads it to a pre-defined folder in your Dropbox. The result is a fully automated digital asset pipeline that ensures backup, organization, and instant accessibility for your entire team.

How It Works

The workflow is built on Make.com and operates on a simple "watch and move" principle. Here’s the step-by-step process:

Step 1: Trigger on New Image Creation

The scenario starts with a Bannerbear module set to "Watch Images." It polls your Bannerbear account at regular intervals. The moment a new image creation event is detected (like a completed social media post or product graphic), the workflow is activated, capturing all relevant data about that image.

Step 2: Retrieve the Image File

Next, the workflow uses the unique image ID from the trigger to call the Bannerbear API again. This step retrieves the actual image file URL or binary data, preparing it for transfer. It ensures the automation has the correct, high-quality asset ready for storage.

Pro tip: You can add a filter after the trigger to only process images from specific templates or projects, giving you precise control over what gets saved to Dropbox.

Step 3: Upload to Your Dropbox Folder

The final step uses the Dropbox module. The workflow takes the image file and uploads it directly to a folder path you specify (e.g., /Marketing-Assets/Bannerbear-Output/). You can configure the file naming convention to include dates, project names, or template IDs for perfect organization from day one.

  3. A Dropbox account (Business or Personal).

Quick Setup Guide

You can have this automation running in under 10 minutes by following these steps:

  1. Clone the Template: Click the "Get This Workflow" button above to open the template on Make.com and create a copy in your own account.
  2. Connect Bannerbear: In the first module, click to add a connection. Enter your Bannerbear API key when prompted to authorize Make.com to access your account.
  3. Connect Dropbox: In the Dropbox upload module, click to add a connection. Follow the OAuth process to link your Dropbox account securely.
  4. Set Your Destination Folder: In the Dropbox module settings, specify the exact folder path (e.g., /Apps/MyMarketing/Graphics). The folder will be created if it doesn't exist.
  5. Test and Activate: Run the scenario once manually to test. Create a new image in Bannerbear and watch as it appears in your Dropbox. Once confirmed, schedule the scenario to run automatically (e.g., every 15 minutes).
