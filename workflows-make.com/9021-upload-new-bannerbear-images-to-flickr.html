What This Workflow Does

For marketing teams, designers, and content creators, the process of moving assets from creation tools to storage or sharing platforms is a repetitive, time-consuming chore. You design a perfect social media graphic in Bannerbear, then you must manually download it, navigate to Flickr, find the right album, and upload it. This workflow breaks that bottleneck.

This automation acts as a digital courier between Bannerbear and Flickr. The moment a new image is generated or finalized in your Bannerbear account, the workflow is triggered. It captures the image file and all its relevant data, then seamlessly uploads it to a pre-defined photoset in your Flickr account. The result is a fully synchronized asset library where your latest creative work is always immediately accessible for your team, clients, or public sharing.

Pro tip: Use this automation to maintain a "live" portfolio or client gallery. Every time you create a new project mockup or marketing asset in Bannerbear, it automatically populates a public or private Flickr gallery, keeping it always up-to-date without any extra effort.

How It Works

The workflow is built on a simple but powerful "watch-and-act" principle using Make.com's automation platform.

Step 1: Trigger on New Bannerbear Image

The scenario starts by monitoring your Bannerbear account for new image creation events. This could be when a template is rendered, a new design is saved, or an image is added to a specific project. Make.com connects to Bannerbear's API and waits for this signal.

Step 2: Retrieve the Image File & Details

Once triggered, the workflow fetches the newly created image. It doesn't just get the image URL; it can also pull metadata like the image name, template ID, creation date, and any custom variables you passed during its creation in Bannerbear. This data can be used for organization later.

Step 3: Upload to Your Flickr Photoset

The core action: the workflow takes the image file and uploads it directly to your connected Flickr account. You specify the destination photoset during setup. The automation handles the authentication, file transfer, and confirmation, ensuring the image lands in the correct digital folder every single time.

What You'll Need

To set up this workflow, you'll need just a few key accounts and permissions:

  1. A Make.com account (free tier available). This is the automation platform that will run the workflow.
  2. A Bannerbear account with API access. You'll need an API key from your Bannerbear dashboard to allow Make.com to connect.
  3. A Flickr account. You'll need to authenticate Make.com with Flickr, typically via OAuth, granting permission to upload photos to your account.
  4. Clear destination: Decide which Flickr photoset you want to use as the default upload destination. You can create a new one specifically for automated uploads.

Quick Setup Guide

You can have this automation running in under 15 minutes by following these steps:

  1. Get the Template: Click the "Get This Workflow" button above to open the pre-built template in your Make.com account.
  2. Connect Bannerbear: In the first module, click to add your Bannerbear connection. You'll be prompted to enter your Bannerbear API key, which you can find in your account settings.
  3. Connect Flickr: In the upload module, click to add your Flickr connection. You'll go through a standard OAuth flow to log in to Flickr and grant Make.com the necessary permissions.
  4. Configure the Destination: Select the specific Flickr photoset where you want all new Bannerbear images to be sent. You can find the photoset ID in its Flickr URL.
  5. Test & Activate: Run the scenario once manually to test. Create a simple image in Bannerbear and watch as it automatically appears in your Flickr photoset. Once confirmed, turn the scenario to "ON" to let it run automatically.
