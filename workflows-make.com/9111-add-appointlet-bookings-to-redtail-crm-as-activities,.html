What This Workflow Does

For financial advisors, insurance agents, and service-based businesses, every client interaction is valuable. When a client books a meeting through your Appointlet calendar, that interaction needs to be captured in your client relationship management (CRM) system, Redtail. Manually copying appointment details—names, times, notes—is a repetitive, error-prone task that wastes precious time you could spend with clients.

This automation solves that problem completely. It acts as a silent bridge between your scheduling tool and your CRM. Every time a new booking is confirmed in Appointlet, this Make.com workflow instantly triggers and creates a corresponding "Activity" in Redtail CRM. The activity is populated with all the relevant details from the booking, providing a complete and timely record of client engagement without you lifting a finger.

The result is a CRM that is always current, giving you a reliable source of truth for client history. This enables better preparation for meetings, more accurate follow-up, and ultimately, stronger, more informed client relationships.

How It Works

The workflow follows a simple, reliable sequence to transfer data from point A (Appointlet) to point B (Redtail CRM).

Step 1: Trigger on New Appointlet Booking

The workflow is activated the moment a new appointment is booked and confirmed in your connected Appointlet account. Make.com watches for this event in real-time, capturing the entire booking payload.

Step 2: Parse and Format Booking Data

The raw data from Appointlet (like client name, email, appointment date/time, service type, and any custom answers) is extracted and formatted to match the expected fields in Redtail CRM's activity module.

Step 3: Create Activity in Redtail CRM

Using the formatted data, the workflow makes an API call to Redtail CRM to create a new activity. It associates the activity with the correct contact (if they exist in Redtail) and logs all the details, such as setting the subject, date, time, duration, and adding notes from the appointment.

Pro tip: Enhance this workflow by adding a step to search for the client in Redtail first. If they aren't found, the automation can create a new contact record before logging the activity, ensuring no booking is ever lost.

Quick Setup Guide

You can have this automation running in less than 30 minutes by following these steps:

  1. Clone the Template: Click "Get This Workflow" and duplicate the provided Make.com scenario into your own account.
  2. Connect Appointlet: In the first module, authorize Make.com to access your Appointlet account using your API key (found in Appointlet Settings > API).
  3. Connect Redtail CRM: In the Redtail CRM module, input your API Key and Database ID from your Redtail account settings.
  4. Map the Data Fields: Review the data mapping. Ensure key Appointlet fields (like start_time, invitee_name, invitee_email) are correctly linked to the corresponding Redtail activity fields (Date, Subject, Linked Contact).
  5. Test & Activate: Run a single test cycle by creating a dummy booking in Appointlet. Verify that a test activity appears in Redtail. Once confirmed, activate the scenario to run automatically.
