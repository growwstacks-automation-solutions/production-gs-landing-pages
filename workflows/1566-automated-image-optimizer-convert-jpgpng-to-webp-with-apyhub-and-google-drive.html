What This Workflow Does

This automation solves the tedious, time-consuming process of manually converting and optimizing images for web use. It automatically retrieves JPG/PNG files from Google Drive, converts them to WebP format using APYHub's API, and saves the optimized versions back to your storage.

For e-commerce sites, blogs, and marketing teams handling hundreds of images weekly, this workflow eliminates hours of repetitive work while ensuring consistent optimization. The WebP format typically reduces file sizes by 25-35% without noticeable quality loss, significantly improving page load times and SEO performance.

Visual comparison showing original JPG (left) and optimized WebP (right) with 32% smaller file size

How It Works

1. Retrieves image URLs from Google Drive

The workflow scans specified Google Drive folders for new JPG/PNG uploads, creating a list of files needing conversion. It handles authentication and pagination automatically for large directories.

2. Processes images through APYHub API

Each image gets sent to APYHub's conversion endpoint, which applies optimal WebP compression settings. The API maintains visual quality while aggressively reducing file size.

Technical diagram of the automated conversion workflow steps

3. Saves optimized WebP files

Converted images are saved to a designated output folder with proper naming conventions. The workflow maintains your original folder structure while adding "_webp" suffixes to distinguish optimized versions.

Pro tip: Set up a separate Google Drive folder for converted images to maintain organization and avoid processing the same files repeatedly.
