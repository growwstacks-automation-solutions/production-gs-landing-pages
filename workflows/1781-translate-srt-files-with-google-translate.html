What This Workflow Does

This n8n workflow automates the translation of SRT subtitle files using Google Translate's powerful machine translation capabilities. It solves the time-consuming manual process of converting subtitles between languages, which typically involves copying text segments, translating them individually, and reformatting the timing data.

The workflow preserves all original timing information while translating the actual subtitle text, ensuring your videos maintain perfect synchronization across different language versions. This is particularly valuable for content creators, educators, and businesses expanding their reach to international audiences.

Example showing translated subtitles maintaining original timing format

How It Works

1. Input SRT File Processing

The workflow begins by accepting an SRT file, which contains numbered subtitles with precise timing information. It parses the file to separate the timing data from the actual text content that needs translation.

2. Text Extraction and Preparation

Each subtitle segment's text is extracted while preserving the original sequence numbers and timecodes. The text is formatted for optimal translation quality, handling line breaks and special characters appropriately.

3. Google Translate Integration

The extracted text segments are sent to Google Translate's API, where they're translated to the target language while maintaining context. The workflow handles API calls efficiently, respecting rate limits and managing large files through batch processing.

Google Translate API integration within the n8n workflow

4. Reassembly and Output

After translation, the workflow reassembles the translated text with the original timing information, generating a new SRT file in the target language. The output maintains all formatting standards required by video players and editing software.
