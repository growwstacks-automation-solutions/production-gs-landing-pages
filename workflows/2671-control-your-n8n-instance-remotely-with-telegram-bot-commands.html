What This Workflow Does

This automation transforms your Telegram chat into a powerful control panel for your self-hosted n8n instance. It enables you to execute commands, monitor workflows, and manage automations remotely through simple Telegram messages. Perfect for system administrators, developers, and teams who need to maintain automation systems while on-the-go or working remotely.

The workflow solves the problem of being tethered to your n8n dashboard for routine operations. Instead of requiring direct access to your n8n interface, you can now perform common tasks through Telegram commands from your smartphone or any device with Telegram installed. This is particularly valuable for distributed teams, on-call engineers, or when you need to trigger time-sensitive actions while away from your computer.

How It Works

1. Telegram Bot Setup

The workflow begins with configuring your Telegram bot through BotFather. Once created, the bot receives a unique token that authenticates communication between Telegram and your n8n instance.

2. Command Processing

When you send a command to your bot (like "/status" or "/restart_workflow"), the message is processed by n8n through a webhook connection. The workflow validates the command and sender before proceeding.

3. n8n API Interaction

Valid commands trigger specific API calls to your n8n instance. These might include checking active workflows, restarting failed executions, or triggering new workflow runs with parameters.

4. Response Generation

The workflow formats the n8n API response into a human-readable Telegram message, providing you with immediate feedback about the command's execution and results.

Quick Setup Guide

  1. Download the template JSON file
  2. Import it into your self-hosted n8n instance
  3. Create your Telegram bot via BotFather and note the token
  4. Configure the Telegram node in the workflow with your bot token

  7. Deploy the workflow and start controlling n8n via Telegram
