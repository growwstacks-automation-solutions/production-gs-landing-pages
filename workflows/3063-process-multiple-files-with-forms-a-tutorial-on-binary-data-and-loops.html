How It Works

1. Form Submission Trigger

The workflow begins when users submit files through a web form. The Form Trigger node captures all uploaded files along with any additional form data, maintaining the original file names and formats.

2. Binary Data Processing

Each file is handled as binary data, preserving the exact original content. The workflow includes proper MIME type detection and file size validation to ensure only permitted documents proceed through processing.

3. Automated File Looping

A Loop node automatically processes each uploaded file individually, applying the same set of actions to every document. This maintains consistency while allowing for conditional logic based on file types or content.

Visual representation of the file processing loop in n8n

4. Conditional Processing Paths

Based on file types or content analysis, documents can be routed through different processing branches. PDFs might go through text extraction while images undergo resizing, all within the same automated workflow.

5. Output and Notifications

Processed files are saved to designated locations (cloud storage, databases, etc.) and confirmation notifications are sent to both users and administrators, completing the automated document handling cycle.
