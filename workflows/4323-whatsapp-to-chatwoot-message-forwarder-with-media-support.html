What This Workflow Does

This automation solves the challenge of managing customer support conversations that originate on WhatsApp while using Chatwoot as your primary helpdesk platform. Many businesses struggle with manually copying messages between these platforms, leading to delayed responses and potential miscommunication.

The workflow automatically forwards incoming WhatsApp messages (including all media attachments) to your Chatwoot inbox, creating a seamless bridge between these communication channels. This ensures your support team can manage all customer interactions from Chatwoot's unified interface while maintaining WhatsApp's convenience for customers.

WhatsApp messages with media appearing in Chatwoot's conversation view

How It Works

1. WhatsApp message detection

The workflow monitors your connected WhatsApp number via Evolution API, detecting new incoming messages in real-time. It captures both text content and any attached media files.

2. Message processing

Each message is parsed to extract the sender information, message content, timestamps, and media attachments. The system prepares this data for forwarding while maintaining conversation context.

3. Chatwoot integration

The processed messages are sent to your Chatwoot instance through its API. The workflow creates new conversations or adds to existing threads based on the WhatsApp contact, ensuring continuity.

4. Media handling

Attachments are downloaded from WhatsApp, temporarily stored, and then uploaded to Chatwoot with proper file type recognition. The system preserves original file names and formats.
