What This Workflow Does

This n8n workflow solves the critical problem of website downtime by providing real-time monitoring with automatic alerts and status updates. Instead of relying on manual checks or discovering outages from customer complaints, this system proactively monitors your website's availability around the clock.

When downtime is detected, the workflow immediately sends email alerts to your team and automatically updates your GitHub status page to keep users informed. This creates a complete monitoring solution that both detects issues and initiates your response protocol without human intervention.

Visualization of uptime monitoring data and alert notifications

How It Works

1. Continuous Website Monitoring

The workflow regularly checks your website's HTTP status code and response time from multiple locations. It verifies not just that the server responds, but that critical pages load correctly with the expected content.

2. Downtime Detection

When the workflow detects an HTTP error (like 500 or 503) or timeout, it immediately triggers the alert sequence. The system includes verification steps to prevent false positives from temporary network glitches.

3. Alert Notification

Email alerts are sent to designated team members with details about the outage - when it started, error details, and affected pages. The alert includes severity levels based on outage duration.

4. Status Page Update

The workflow automatically posts an incident report to your GitHub-hosted status page, keeping users informed. When the site recovers, it updates the status accordingly.

What You'll Need

  1. An n8n instance (cloud or self-hosted)
  2. SMTP credentials for sending email alerts
  3. GitHub repository for status page updates
  4. Personal access token with repo permissions
  5. List of website URLs to monitor

Pro tip: For mission-critical sites, set up monitoring from at least 3 different geographic locations to distinguish between local network issues and actual downtime.

Quick Setup Guide

  1. Download the JSON template file
  2. Import into your n8n instance
  3. Configure the HTTP Request nodes with your website URLs
  4. Set up your SMTP email credentials
  5. Add your GitHub repository details and access token
  6. Test with intentional downtime to verify alerts
  7. Schedule the workflow to run every 1-5 minutes
