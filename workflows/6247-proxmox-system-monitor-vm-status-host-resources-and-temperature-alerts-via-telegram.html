What This Workflow Does

This n8n workflow provides comprehensive monitoring for your Proxmox Virtual Environment (VE) server, delivering automated status reports and critical alerts directly to Telegram. It eliminates the need for manual server checks while ensuring you're immediately notified of any potential issues that could impact your virtual infrastructure.

The solution tracks vital metrics including VM/container status changes, CPU/memory utilization, storage capacity thresholds, and system temperatures. By automating these monitoring tasks, IT teams can focus on higher-value work while maintaining complete visibility into their virtualization environment.

Sample Telegram notification showing Proxmox resource utilization metrics

How It Works

1. Proxmox API Data Collection

The workflow connects to your Proxmox server's API to gather real-time performance data. It retrieves comprehensive metrics about host resources, virtual machine status, and system health indicators.

2. Threshold Analysis

Collected metrics are compared against configurable thresholds to identify potential issues. The system checks for CPU overload, memory pressure, storage capacity limits, and abnormal temperature readings.

3. Alert Generation

When thresholds are exceeded or VMs change state, the workflow formats detailed alert messages with severity indicators. Critical issues trigger immediate notifications while regular status updates follow the scheduled interval.

4. Telegram Delivery

Formatted messages are sent to specified Telegram chats or channels using Telegram's bot API. The workflow supports rich formatting with emoji indicators for quick status recognition.

What You'll Need

  1. A running Proxmox VE server (version 6.0 or newer)
  2. API access credentials with appropriate permissions
  3. A Telegram account and bot token
  4. n8n instance (self-hosted or cloud)
  5. Cron or scheduler to run the workflow regularly

Pro tip: Create a dedicated Telegram group for your IT team to receive all monitoring alerts in one place, with different members getting notification permissions based on their roles.

Quick Setup Guide

  1. Import the JSON template into your n8n instance
  2. Configure Proxmox API connection details
  3. Set up your Telegram bot credentials
  4. Adjust monitoring thresholds to match your hardware
  5. Test with manual execution before scheduling
  6. Set the workflow to run every 15 minutes (or your preferred interval)

Active monitoring (like this workflow) polls systems proactively to detect issues before users notice. Passive monitoring relies on system-generated alerts. Active approaches catch more potential problems earlier but require more resources. This template implements smart active monitoring that minimizes overhead while maximizing early detection capabilities.

For instance, active monitoring might detect a slowly filling disk at 85% capacity, while passive monitoring would only alert when the disk actually becomes full. The active approach allows planned maintenance during business hours rather than emergency response.

  • Active: Proactive, comprehensive, more resource-intensive
  • Passive: Reactive, event-driven, less overhead
  • Hybrid approaches often work best
