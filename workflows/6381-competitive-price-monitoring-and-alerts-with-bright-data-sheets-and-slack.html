Competitive price monitoring & alerts with Bright Data, Sheets & Slack

Automate competitor price tracking and get instant Slack notifications when key price changes occur

What This Workflow Does

This automated workflow solves the critical challenge of staying competitive in dynamic ecommerce markets by continuously monitoring competitor pricing. It eliminates manual price checking by automatically scraping competitor websites using Bright Data's enterprise-grade web scraping API, storing the data in Google Sheets for historical analysis, and sending real-time Slack alerts when significant price changes occur.

For retail and ecommerce teams, this means always knowing when competitors adjust prices without spending hours manually checking websites. The system can track hundreds of products across multiple competitors daily, detecting price drops, promotions, and out-of-stock situations that impact your pricing strategy.

The workflow structure in n8n showing Bright Data scraping, Google Sheets integration, and Slack alert steps

How It Works

1. Scheduled price scraping

The workflow triggers on a schedule (default daily) to scrape competitor websites using Bright Data's API. This handles complex sites with JavaScript rendering, bypasses anti-bot measures, and extracts clean pricing data even from dynamic ecommerce platforms.

2. Data processing and comparison

Scraped prices are compared against your reference prices stored in Google Sheets. The system calculates percentage differences and identifies which changes exceed your configured thresholds for alerts.

Historical price tracking in Google Sheets showing competitor comparisons over time

3. Alert generation

When significant price changes are detected, the workflow formats detailed Slack messages showing the product, old price, new price, percentage change, and competitor name. These appear in designated channels with @mentions for quick team response.

Pro tip: Configure different alert thresholds for different product categories - you might want immediate alerts for bestsellers but weekly summaries for slow-moving items.

4. Historical data storage

All scraped prices are appended to Google Sheets with timestamps, creating a searchable price history database. This enables trend analysis and helps identify seasonal pricing patterns.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. Bright Data account with web scraping API access
  3. Google Sheets with your product catalog and reference prices
  4. Slack workspace with appropriate channel permissions
  5. List of competitor URLs and CSS selectors for price elements

Quick Setup Guide

  1. Download the JSON template file
  2. Import into your n8n instance
  3. Configure Bright Data credentials in the HTTP Request node
  4. Update the Google Sheets node with your spreadsheet ID
  5. Set up Slack webhook for notifications
  6. Adjust scraping schedule and price change thresholds as needed
  7. Test with a few competitor products before full rollout
