What This Workflow Does

This automation solves the time-consuming process of manually optimizing images for web and marketing use. It automatically compresses new images uploaded to specified Google Drive folders using the Tinify API (powering TinyPNG), then generates detailed reports about the compression results.

The workflow monitors designated Google Drive folders, processes new images through Tinify's compression algorithm, saves the optimized versions, and emails a summary report with before/after file sizes and compression ratios. This ensures all your digital assets are consistently optimized for web performance without manual intervention.

Visual comparison showing typical file size reductions from 50-80% while maintaining visual quality

How It Works

1. Monitoring Google Drive for new images

The workflow continuously checks specified Google Drive folders for newly uploaded images. It can be configured to monitor multiple folders simultaneously, making it ideal for teams with different departments or clients.

2. Processing through Tinify API

When new images are detected, they're automatically sent to Tinify's compression API. The service uses smart lossy compression techniques to reduce file sizes while preserving visual quality better than most alternatives.

How Tinify's compression algorithm works while maintaining image quality

3. Saving optimized versions

The compressed images are saved back to Google Drive with clear naming conventions (original_filename_optimized.ext). The workflow maintains folder structure and can optionally archive originals in a separate folder.

4. Generating email reports

After processing, the system compiles a detailed email report showing before/after file sizes, percentage reductions, and total bandwidth savings. Reports can be sent to multiple recipients on customizable schedules.

Pro tip: Configure the workflow to run during off-peak hours to avoid API rate limits and ensure fastest processing when your team needs the images.
