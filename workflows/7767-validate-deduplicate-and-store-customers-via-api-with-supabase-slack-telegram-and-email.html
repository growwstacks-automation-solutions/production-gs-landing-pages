What This Workflow Does

This workflow solves the common retail problem of dirty customer data by providing an automated API-first solution to validate, clean, deduplicate and store customer information. It ensures your customer database remains accurate and up-to-date while reducing manual data entry errors.

The system accepts customer data via API, validates each field (email format, phone number validity, address completeness), checks for duplicates in your Supabase database, then stores clean records while sending notifications via Slack, Telegram and email about new customer additions or potential duplicates.

Example of customer data validation steps in the workflow

How It Works

1. API Endpoint Receives Customer Data

The workflow starts with an API endpoint that accepts customer information from your website, app, or other sources. The endpoint validates the request format and authentication before processing.

2. Data Validation and Cleaning

Each customer record undergoes rigorous validation: email format checking, phone number normalization, address standardization, and required field verification. Invalid records trigger immediate error responses.

3. Duplicate Detection

The workflow queries your Supabase database to check for existing customers with matching email, phone, or other identifiers. Potential duplicates are flagged for review while unique records proceed.

Duplicate detection process showing potential matches

4. Database Storage

Validated customer records are stored in your Supabase database with proper timestamps and metadata. The workflow handles all database operations securely.

5. Multi-Channel Notifications

Successful additions trigger notifications via Slack (team channel), Telegram (admin alerts), and email (customer confirmation). Duplicate alerts go to your operations team.

Pro tip: Configure different notification channels for different customer segments (VIPs, first-time buyers, returning customers) by adding conditional logic to the workflow.
