AAVE Portfolio AI Agent: Telegram & Email DeFi Monitor

What This Workflow Does

Managing DeFi positions across multiple wallets is a full-time job. Between checking health factors, monitoring liquidation risks, and tracking yields, investors spend hours each week manually reviewing data. This automation solves that problem by creating an AI-powered assistant that continuously monitors your Aave V3 positions.

The workflow pulls real-time data from the blockchain via Moralis API, processes it through GPT-4o to generate human-readable insights, and delivers those insights through both instant Telegram alerts and comprehensive email reports. It transforms complex on-chain metrics into simple, actionable intelligence that helps you protect your assets and optimize your strategy without constant screen time.

Whether you're managing personal investments, a DAO treasury, or client portfolios, this system provides professional-grade monitoring that scales with your needs. The template handles everything from data collection to intelligent analysis to multi-channel delivery, giving you back the time you'd spend on manual tracking while providing better, more consistent oversight.

How It Works

1. Scheduled Trigger & Wallet Loading

The workflow starts on a schedule you define—daily, hourly, or weekly. It first loads all wallet addresses from a Google Sheet where you maintain your monitored portfolio list. This centralized approach lets you easily add or remove wallets without modifying the automation itself.

2. Real-Time Blockchain Data Fetching

For each wallet, the system queries the Moralis API to retrieve current Aave V3 positions. This includes supply amounts, borrowed assets, collateral ratios, health factors, and liquidation thresholds. The data is collected in real-time, ensuring you're working with the most current state of your positions.

3. AI-Powered Analysis & Summarization

Raw blockchain data gets fed into GPT-4o (via OpenAI Chat Model node) with carefully crafted prompts. The AI analyzes the numbers, identifies risks and opportunities, and generates clear, concise summaries in plain English. It explains what the numbers mean, not just what they are.

4. Multi-Channel Report Delivery

Critical alerts and brief summaries get sent immediately to your Telegram chat for mobile notification. Simultaneously, comprehensive HTML reports with full details, historical context, and formatted tables get emailed to your inbox for deeper analysis and record-keeping.

What You'll Need

  1. n8n Instance: Self-hosted or cloud version of n8n to run the automation.
  2. Moralis API Key: Free tier available for basic monitoring, paid tiers for higher volume.
  3. OpenAI API Key: Access to GPT-4o or GPT-4o-mini models for AI analysis.
  4. Telegram Bot Token: Created via @BotFather on Telegram for instant alerts.
  5. Gmail OAuth2 Credentials: Or SMTP setup for email delivery of detailed reports.
  6. Google Sheet: With a "wallet_address" column containing the Ethereum addresses to monitor.

Pro tip: Start with just 2-3 wallets during testing. This lets you verify the system works correctly before scaling to monitoring dozens of addresses. Pay special attention to the alert thresholds—set them conservatively at first to avoid notification fatigue.

Quick Setup Guide

Step 1: Download the JSON template file and import it into your n8n instance via the "Import from File" option in the workflow editor.

Step 2: Create credentials for each service: Moralis (API Key), OpenAI (API Key), Telegram (Bot Token), and Gmail (OAuth2). Test each connection individually before proceeding.

Step 3: Create your Google Sheet with wallet addresses. Share it with your n8n service account or use API access. Update the Google Sheets node in the workflow with your specific spreadsheet ID.

Step 4: Configure the Telegram node with your chat ID and the Gmail node with your recipient email address. Customize the AI prompt in the OpenAI node if you want different analysis focus.

Step 5: Set the schedule trigger to your preferred frequency (start with daily). Run the workflow manually once to verify everything works, then activate it and monitor the first few reports for accuracy.
