What This Workflow Does

This automation template solves a critical gap for AI-driven climate analysis: connecting AI agents directly to authoritative CO2 measurement data. The CarbonDoomsDay API provides historical atmospheric CO2 levels from the Mauna Loa observatory—a gold standard dataset used by scientists worldwide. However, AI agents cannot natively call this API.

This n8n workflow bridges that gap by creating a fully functional MCP (Model Context Protocol) server. It exposes two key API operations—fetching CO2 measurements by date and retrieving general CO2 data—as standardized tools that AI agents like Claude Desktop, Cursor, or custom AI applications can use immediately. Instead of manual data scraping or complex integration coding, your AI agents can now query CO2 trends, generate climate reports, and analyze emissions data autonomously.

The workflow handles authentication, parameter mapping, error management, and data formatting, returning clean JSON responses that AI agents understand. It turns a technical API into an AI-ready toolset, enabling automated sustainability research, ESG reporting, and real-time climate monitoring.

How It Works

Step 1: MCP Server Trigger

The workflow starts with an MCP Server Trigger node, which creates a webhook endpoint. This endpoint listens for incoming requests from AI agents. When an AI agent sends a request (e.g., "Get CO2 data for January 2024"), the trigger captures the query and passes it to the next nodes.

Step 2: Parameter Extraction with AI Expressions

Using n8n's $fromAI() expressions, the workflow automatically extracts parameters from the AI agent's request. Dates, filters, and identifiers are parsed without manual configuration. This allows the AI agent to provide natural-language queries like "Show CO2 levels from last week" which are converted into API-ready parameters.

Step 3: HTTP Request to CarbonDoomsDay API

The workflow sends an authenticated HTTP request to the CarbonDoomsDay API (https://api.carbondoomsday.com/api). It supports two operations: GET /co2/ for general measurements and GET /co2/{date}/ for date-specific data. The API returns structured CO2 readings, including daily and weekly averages dating back to 1958.

Step 4: Data Transformation & Response

Raw API responses are transformed into clean JSON format optimized for AI consumption. The workflow ensures data consistency, handles missing values gracefully, and structures the output so AI agents can easily interpret trends, averages, and anomalies.

Step 5: Return to AI Agent

The final step sends the processed CO2 data back to the requesting AI agent through the MCP server response channel. The AI receives a complete, actionable dataset ready for analysis, reporting, or further automation.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n workspace.
  2. Add API Credentials: Create a credential for CarbonDoomsDay API in n8n's credential management.
  3. Activate the Workflow: Turn on the workflow to start the MCP server.
  4. Copy the Webhook URL: From the MCP Trigger node, copy the generated webhook endpoint URL.
  5. Connect Your AI Agent: Add the URL to your AI agent's MCP server configuration (Claude Desktop settings, Cursor config, etc.).
  6. Test the Integration: Ask your AI agent to "Get CO2 measurements for March 2023" and verify the response.

Pro tip: Use n8n's schedule trigger to automatically fetch CO2 data daily and store it in a database (like PostgreSQL or Airtable) for historical trend analysis.
