What This Workflow Does

This automation solves the common problem of losing potential sales leads when developers show interest in your GitHub repositories. Many technical companies miss valuable business opportunities because they don't systematically track who forks their code.

The workflow automatically creates a new lead in Pipedrive whenever someone forks your GitHub repository. It captures the developer's GitHub profile information and connects their technical engagement with your sales pipeline, ensuring your team can follow up with relevant, timely outreach.

How It Works

1. GitHub fork event trigger

The workflow monitors your specified GitHub repositories using webhooks. When a new fork event occurs, n8n receives the payload containing details about who forked the repository and when.

2. Profile data enrichment

The system then makes additional API calls to GitHub to retrieve the forker's public profile information, including their name, company, location, and other available details that can help qualify the lead.

3. Pipedrive lead creation

Using the enriched data, the workflow creates a new lead in Pipedrive with all relevant GitHub activity details. The lead record includes custom fields tracking which repository was forked and the exact timestamp.

Pro tip: Configure custom Pipedrive fields to capture technical details like programming languages used in the forked repository, which can help prioritize leads.
