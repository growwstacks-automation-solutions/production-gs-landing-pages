What This Workflow Does

This workflow solves a common problem for website owners, marketers, and SEO professionals: getting quick, actionable insights from Google Search Console data. Instead of navigating complex reports and exporting spreadsheets, you can now have a conversation with your data.

The AI agent acts as an intelligent interface between you and your Search Console. You ask questions in plain English like "What were my top 10 pages by clicks last week?" or "Show me queries with high impressions but low CTR." The agent fetches the relevant data, processes it through OpenAI's language model, and returns a clear, summarized answer. It stores historical data in Postgres, enabling trend analysis and more sophisticated queries over time.

This automation turns a reactive reporting tool into a proactive insights engine, saving hours of manual analysis each week and helping you make faster, data-driven SEO decisions.

Correct OAuth scope configuration is critical for stable API connections.

How It Works

The workflow is built on n8n and connects three core systems: Google Search Console (data source), OpenAI (intelligence layer), and Postgres (memory and history).

Step 1: User Query via Webhook

A user submits a question through a secure webhook interface (with authentication). The question is captured and passed to the workflow for processing.

Step 2: Data Retrieval & Processing

The workflow parses the question, determines what data is needed from Search Console (e.g., performance report by page, query, country), and makes the appropriate API calls. The raw data is fetched and structured.

Detailed API credential setup ensures reliable data access.

Step 3: AI Analysis & Answer Generation

The structured data and the original question are sent to an OpenAI chat model (like GPT-4). A carefully engineered system prompt instructs the AI to act as an SEO analyst, interpret the data, and generate a helpful, concise answer.

Step 4: Response Delivery & Storage

The AI's answer is sent back to the user. Simultaneously, the question, answer, and relevant data points are stored in a Postgres database. This creates a searchable history and enriches the dataset for future, more complex queries.

What You'll Need

  1. A Google Cloud Platform project with the Search Console API enabled and OAuth 2.0 credentials (Client ID & Secret) configured with the correct scopes.
  2. An OpenAI API key with access to a suitable chat model (e.g., gpt-4o-mini).
  3. A Postgres database (can be local, cloud-based like AWS RDS, or a service like Supabase/Neon).
  4. An n8n instance (self-hosted or cloud) to import and run the workflow.
  5. Basic understanding of webhook security to set up authentication (like Basic Auth) for the public-facing interface.

Pro tip: Pay close attention to the OAuth scope configuration. Using the wrong scope (like .../auth/webmasters.readonly) can cause persistent refresh token issues, forcing frequent re-authentication. The template includes guidance on the correct scopes to use.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure credentials: Set up the Google OAuth, OpenAI, and Postgres credentials in n8n's credentials management.
  3. Adjust the webhook: Set a unique path for your webhook and enable authentication (Basic Auth is a good start).
  4. Test the connection: Trigger the webhook with a simple question like "What is my total clicks for the last 7 days?"
  5. Review and customize: Check the database tables are being populated and adjust the AI system prompt if needed to better suit your tone or reporting style.
