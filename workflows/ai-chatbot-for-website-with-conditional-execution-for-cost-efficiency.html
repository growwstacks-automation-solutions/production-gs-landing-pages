AI Chatbot for Website with Conditional Execution

What This Workflow Does

This n8n workflow solves a common and expensive problem: AI chatbots that run continuously, consuming API tokens and generating costs even when no one is chatting. It implements an intelligent, conditional AI chatbot named "Sophia" that only activates when a genuine website visitor initiates a conversation.

The automation serves as a 24/7 interactive assistant for service businesses, collecting user details (name, email, request type) in a structured conversation flow. It helps with inquiries about guest posting, YouTube reviews, IT consultancy, and online courses while ensuring every interaction is logged and confirmed.

By introducing a gatekeeping conditional step, this template prevents unnecessary AI API calls, potentially reducing your monthly AI costs by 60–80% compared to traditional always-on chatbots. It's perfect for businesses that want AI-powered engagement without the unpredictable expense.

How It Works

The workflow uses a logical sequence of checks and responses to maximize efficiency while maintaining a natural user experience.

1. Chat Message Trigger

The process begins only when a webhook receives a chat message from your website. This ensures the automation doesn't run on empty or automated page loads.

2. Conditional Input Check

An "If" node immediately checks whether the chat input contains actual text. If the input is empty, the workflow stops here—consuming zero AI tokens. This is the core cost-saving mechanism.

3. AI-Powered Conversation

If a valid message exists, the Google Gemini AI node generates a personalized response as "Sophia." The AI is instructed to collect user information systematically: first the full name, then email, then specific request type.

4. Context-Aware Memory

A Window Buffer Memory node stores the conversation history, allowing the chatbot to remember previously provided information and avoid asking repetitive questions in multi-turn dialogues.

5. Structured Data Collection

The workflow guides users through a natural but structured qualification process, ensuring you capture complete lead information rather than fragmented messages.

6. Confirmation & Routing

Once all details are collected, Sophia sends a confirmation message summarizing the request. The structured data is then ready to be sent to your CRM, email system, or internal database.

What You'll Need

  1. An n8n instance (cloud or self-hosted) to run the workflow.
  2. Google Gemini API key or alternative AI provider credentials.
  3. Website integration capability to send chat messages via webhook (most chat widgets support this).
  4. Destination system ready to receive qualified leads (CRM, spreadsheet, email, etc.).

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Configure the Webhook node with a unique URL and connect it to your website's chat widget.
  3. Add your AI credentials in the Google Gemini node (or swap for your preferred AI model).
  4. Test the conditional logic by sending empty and valid chat messages to verify the cost-saving gate works.
  5. Connect the output to your lead management system (HubSpot, Google Sheets, etc.).
  6. Activate the workflow and monitor initial conversations to refine the AI instructions if needed.

Pro tip: Start with a conservative AI model (like Gemini Nano) for testing to minimize costs. Once conversation flow is validated, you can upgrade to more capable models for complex inquiries.
