What This Workflow Does

This n8n workflow solves a common but time-consuming problem: turning raw, scattered web data into polished, publish-ready charts. Instead of manually researching, copying data into a spreadsheet, designing a chart, exporting an image, and uploading it to your CMS, this automation does it all in one seamless process.

You provide a simple prompt—like “Generate a graph of Apple's market share in Q1 2025”—and the workflow uses GPT-4o to browse the web, find the latest figures, structure the data, choose the best chart type, render a professional image via QuickChart.io, and upload it directly to your WordPress media library. The result is a live chart embedded in your content, driven by the most current data available.

It’s designed for marketers, analysts, bloggers, and business owners who need to produce data-driven content regularly but want to eliminate the manual grunt work and potential for human error.

How It Works

The workflow is a smart pipeline that connects AI intelligence with practical publishing tools.

1. Prompt & Web Search

The process starts when you trigger the workflow with a data query. The OpenAI node (GPT-4o with browsing enabled) interprets your prompt, searches the web for relevant and recent information, and extracts the key figures into a clean markdown table. It also returns citation URLs for transparency.

2. AI-Powered Chart Design

A LangChain Agent analyzes the extracted table. It decides the most appropriate chart type—bar chart for comparisons, line chart for trends, doughnut for proportions—and constructs a valid Chart.js configuration JSON object. This step ensures the visualization matches the data's story.

3. Image Rendering

The Chart.js configuration is sent to the QuickChart.io API, a free service that renders chart images on-demand. It returns a high-quality PNG image of your chart, ready for publication.

4. Automated WordPress Upload

Finally, an HTTP Request node posts the PNG image directly to your WordPress site's REST API media endpoint. It handles authentication, sets the correct filename and MIME type, and returns the new media item's URL. Your chart is now live in your WordPress library, ready to be inserted into any post or page.

Pro tip: You can schedule this workflow to run daily or weekly, ensuring your site always displays charts with the latest data without any manual intervention.

What You'll Need

  1. An n8n instance (cloud or self-hosted).
  2. OpenAI API credentials with access to GPT-4o (or GPT-4o-mini). The browsing capability is required for live web data.
  3. WordPress site credentials with REST API access and permission to upload media (Application Password or User credentials).
  4. A QuickChart.io account (free tier is sufficient for most use cases).

Quick Setup Guide

Import and configure this workflow in under 10 minutes.

  1. Download the template using the button above and import it into your n8n instance.
  2. Connect your OpenAI credentials to the “Message a model” node. Ensure the model is set to “gpt-4o” with browsing enabled.
  3. Configure the WordPress HTTP Request node with your site's base URL (e.g., https://yoursite.com/wp-json/wp/v2) and add your authentication headers (Username/Application Password).
  4. Test the trigger by running the workflow manually with a sample prompt like “US inflation rate last 6 months.”
  5. Check your WordPress Media Library for the newly uploaded chart image and verify the data looks correct.

Once tested, you can change the trigger to a schedule (e.g., weekly) or connect it to other workflows, like automatically creating a new blog post draft with the chart included.

Pro tip: For consistent branding, modify the Chart.js configuration in the AI Agent node to use your company's color palette and font styles.
