What This Workflow Does

This workflow solves the constant pressure of content creation by automating the entire "research-to-draft" process. It acts as your autonomous content factory, monitoring trending topics through RSS feeds, filtering them for relevance using AI, and generating polished drafts for multiple platforms—all while keeping human approval at the center.

Instead of spending hours each day searching for topics and writing initial drafts, this system delivers ready-to-review content directly to your Slack channel. You maintain complete quality control with a simple approve/reject decision, while the automation handles the repetitive heavy lifting. Approved content automatically organizes into Google Sheets, creating a structured content calendar.

How It Works

The workflow follows a sophisticated pipeline that mimics an editorial team's workflow but executes it automatically.

1. Trend Monitoring & Filtering

The system starts by fetching the latest articles from your configured RSS feeds (Google News, industry blogs, or Google Trends). An AI agent acts as your "editor-in-chief," analyzing each item for viral potential and relevance to your specific niche before passing only the most promising topics forward.

2. Multi-Platform Content Generation

For each approved topic, three specialized AI agents work in parallel to create platform-optimized content: an SEO-structured blog post outline, a 5-slide Instagram carousel plan, and a short-form video script for TikTok or Reels. This parallel processing ensures consistency across channels while respecting each platform's unique requirements.

3. Human-in-the-Loop Approval

All generated content bundles into a formatted Slack message with interactive buttons. The workflow pauses here, waiting for your team's decision. This critical step ensures brand alignment and quality before anything gets saved, combining AI efficiency with human judgment.

4. Final Storage & Organization

When you click "Approve" in Slack, the workflow resumes and appends the complete content package—blog outline, Instagram plan, and video script—to your designated Google Sheet. Each piece gets timestamped and categorized, building a searchable content library ready for your publishing team.

  1. n8n Instance (version 1.19.0 or higher) with access to AI Agent nodes
  2. OpenAI API Key (the workflow is optimized for cost-effective gpt-4o-mini)
  3. Slack Workspace where you can install a custom app for receiving approval requests
  4. Google Sheets Document with columns prepared for Blog, Instagram, and Script content
  5. RSS Feed URL for your topic source (Google News, industry publications, or curated feed)

Pro tip: Start with a low-cost AI model like gpt-4o-mini for draft generation. The human approval layer ensures quality, allowing you to optimize for cost during the initial creation phase.

Quick Setup Guide

Follow these steps to implement this content factory in your own environment:

  1. Import the Template: Download the JSON file and import it into your n8n instance using the workflow import function.
  2. Configure Credentials: Set up credentials for OpenAI, Slack, and Google Sheets in n8n's credentials management.
  3. Create Slack App: In your Slack workspace, create a new app with "Interactivity" enabled. Set the Request URL to your n8n production webhook URL and add the chat:write scope before installing.
  4. Prepare Google Sheet: Create a new sheet with "Blog," "Instagram," and "Script" as column headers in row 1. Note the sheet ID for configuration.
  5. Customize RSS Source: Update the RSS node with your preferred feed URL. Test with a known-reliable source initially.
  6. Test Approval Flow: Trigger the workflow manually once to ensure the Slack message arrives correctly and approval triggers the Google Sheets append.
