What This Workflow Does

Manual email triage is a silent productivity killer for customer support teams. Agents waste hours each day reading, categorizing, and forwarding emails before any real problem-solving begins. This creates delayed responses, misrouted issues, and frustrated customers.

This automation solves that by creating an intelligent email router. It connects your Gmail inbox to AI (OpenAI), your team communication (Slack), and a database (Supabase). Every new customer email is instantly read, understood, categorized (e.g., "Billing," "Technical Support," "Sales Inquiry"), posted to the correct Slack channel for the assigned team, and logged—all before a human even needs to check their inbox. The email is then marked as read to prevent duplicate processing.

The result is a support operation that reacts in seconds, not hours. Urgent issues are flagged immediately, teams only see relevant requests, and every interaction is tracked from the first touchpoint.

How It Works

The workflow follows a logical, step-by-step process to ensure no email is missed or mishandled.

1. Trigger & Duplicate Check

The workflow is triggered by a new, unread email arriving in a specified Gmail inbox. Its first action is to check your Supabase database using unique identifiers (like the Gmail thread ID) to see if this email thread has already been processed. This prevents infinite loops and duplicate alerts.

2. AI Classification

If the email is new, its content (subject and body) is sent to the OpenAI API. A carefully crafted prompt instructs the AI to analyze the email's intent and classify it into predefined categories relevant to your business, such as "Support," "Billing," "Feature Request," or "Urgent."

3. Intelligent Routing

The AI's classification decision is passed to a Switch node. This node acts as a digital traffic controller, routing the email's details (sender, subject, snippet) to a pre-configured Slack channel based on the category. For example, a "Billing" email goes to the #finance-team channel, while a "Bug Report" goes to #engineering.

4. Logging & Completion

Once routed, all relevant details—sender, subject, classification, timestamps, and internal IDs—are saved as a new record in your Supabase database. This creates a searchable audit trail. Finally, the original Gmail message is marked as "read," signaling that the automated triage process is complete.

  1. A Gmail or Google Workspace Account: With API access enabled (OAuth 2.0 credentials) for the inbox you want to monitor.
  2. An OpenAI API Account: With an API key and a small credit balance. Classification costs are minimal (fractions of a cent per email).
  3. A Slack Workspace: Where you have permission to create or use channels (e.g., #support-billing, #support-technical) and create incoming webhooks or install an app.
  4. A Supabase Project (or Alternative Database): A free Supabase project is perfect for this. You'll need the project URL and anon/public API key to create a simple table for logging emails.
  5. n8n Instance: Either a self-hosted n8n setup or a n8n.cloud account to import and run the workflow.

Pro tip: Start with just 3-4 broad email categories (e.g., Support, Sales, Billing, Other). You can always refine and add more later as you see how the AI performs and how your team is structured.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps.

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure Credentials: In the imported workflow, set up the credentials for each node: Gmail (OAuth2), OpenAI (API Key), Slack (Webhook or OAuth2), and Supabase (Project Details).
  3. Customize Categories & Channels: In the OpenAI node, edit the system prompt to define your specific email categories. In the Switch node, update the branch conditions and connect each branch to the correct Slack channel webhook URL.
  4. Set Up Your Database: In Supabase, create a simple table with columns for `id`, `email_subject`, `sender`, `category`, `gmail_thread_id`, and `created_at`. Update the Supabase nodes in the workflow with your table name.
  5. Test & Activate: Turn on the workflow. Send a test email to your monitored Gmail address. Watch as it gets classified, posted to Slack, and logged in Supabase within seconds. Tweak as needed, then let it run automatically.
