What This Workflow Does

This automation solves the critical problem of dirty, inconsistent lead data from large-scale scraping operations. When you scrape thousands of leads from platforms like Apollo or via Apify, you get raw, unverified information filled with formatting errors, missing fields, and duplicates. Manually cleaning this data takes hours and still results in quality issues that waste sales efforts.

The workflow creates a fully automated pipeline that fetches raw leads, passes them through Google Gemini for intelligent validation and cleansing, checks for duplicates, and delivers clean, standardized records directly to Google Sheets. It transforms messy scraped data into sales-ready leads with zero manual intervention, ensuring your team only engages with properly formatted, complete contacts.

How It Works

1. Data Collection & Trigger

The workflow starts when new scraped lead data arrives, typically via a webhook from Apify or scheduled scraping jobs. It authenticates with the data source API and retrieves the complete dataset of raw leads.

2. Batch Processing Setup

To handle large volumes efficiently, the system automatically splits leads into manageable batches (usually 1000 records each). This prevents memory issues and allows for graceful error handling if any batch encounters problems.

3. AI-Powered Validation with Gemini

Each lead passes through Google Gemini, which acts as an intelligent data quality gatekeeper. Unlike simple validation that just checks for empty fields, Gemini understands context—it corrects formatting inconsistencies, infers missing company details from email domains, and flags implausible data.

4. Deduplication & ID Generation

Before adding any lead to your sheet, the system cross-references the email address against all existing records to prevent duplicates. Validated leads receive a unique sequential ID (format: AP-DDMMYY-XXXX) for tracking and CRM integration.

5. Google Sheets Integration

Clean, unique leads append as new rows to your designated Google Sheet with all fields standardized. The workflow respects Google Sheets API rate limits with built-in delays between operations.

6. Notification & Monitoring

After each batch completes, a concise summary sends to your Telegram channel detailing success counts, warnings, and errors. This provides real-time operational visibility without needing to check the sheet manually.

  4. Telegram Bot Token & Chat ID – For completion notifications
