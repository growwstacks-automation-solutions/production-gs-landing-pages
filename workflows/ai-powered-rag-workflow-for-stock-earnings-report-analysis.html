What This Workflow Does

This automation transforms how financial analysts and investors process quarterly earnings reports. Instead of manually reading through hundreds of pages of PDF documents, this workflow uses AI to extract, analyze, and summarize key financial information automatically.

The system creates a financial intelligence tool that can answer complex questions about a company's performance, trends, risks, and opportunities. By combining Retrieval-Augmented Generation (RAG) with powerful AI models, it provides accurate, document-grounded analysis that saves dozens of hours per company researched.

Whether you're an investment firm covering multiple stocks, a financial advisor preparing client reports, or a corporate finance team analyzing competitors, this automation delivers actionable insights faster and more consistently than manual methods.

How It Works

The workflow follows a sophisticated three-stage process that mimics how expert analysts work, but at machine speed and scale.

1. Data Loading and Indexing

The system starts by fetching links to PDF earnings documents from a Google Sheet containing your watchlist. It downloads these PDFs from Google Drive, parses the text, and splits it into meaningful chunks. Using Google's text-embedding-004 model, it generates embeddings that capture the semantic meaning of the financial content.

These embeddings and their corresponding text chunks are stored in a Pinecone vector database, creating a searchable knowledge base of the company's financial history. This setup enables instant semantic search across all uploaded documents.

2. Report Generation with AI Agent

An AI Agent node orchestrates the analysis with a specifically crafted system prompt designed for financial analysis. The agent uses a Vector Store Tool to access and retrieve relevant information from the Pinecone database based on your questions or analysis requirements.

The AI (using OpenAI GPT-4o-mini or Google Gemini) synthesizes information from multiple documents, identifies trends, calculates metrics, and generates coherent analysis with proper financial context and terminology.

3. Report Delivery and Storage

The generated report is saved as a structured Google Doc in your specified Google Drive location, complete with formatting, headings, and clear sections. The system can also trigger notifications, update databases, or integrate with other financial systems as needed.

Pro tip: Start with 2-3 companies in your initial setup to validate the analysis quality. Once you're confident in the outputs, scale to cover your entire watchlist. The system handles volume effortlessly once configured.

What You'll Need

  1. Google Cloud Project with Vertex AI API enabled
  2. Google AI API key from Google AI Studio
  3. Pinecone account with API key and an index named "company-earnings"
  4. Google Drive with earnings PDFs and a Google Sheet containing file URLs
  5. n8n instance (cloud or self-hosted) with credentials configured for Google Sheets, Drive, Docs, Gemini API, and Pinecone
  6. Earnings reports in PDF format from company investor relations pages

Quick Setup Guide

Follow these steps to implement this financial analysis automation in your environment:

  1. Download and import the workflow JSON into your n8n instance
  2. Configure credentials for all required services in n8n's credentials management
  3. Update the Google Sheet node to point to your sheet containing PDF URLs
  4. Set the Google Drive folder path where reports should be saved
  5. Test with a single company to verify the end-to-end process works correctly
  6. Scale up by adding more companies to your tracking sheet

Pro tip: Create a standardized naming convention for your PDF files and Google Sheet columns. Consistency here makes the automation more reliable and easier to maintain as you add more companies over time.
