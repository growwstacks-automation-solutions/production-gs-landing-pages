AI-powered ServiceNow Chat Triage with GPT-4

Automatically classify incoming IT support messages as incidents or requests, create ServiceNow tickets, and summarize updates—reducing manual triage by 80%.

What This Workflow Does

IT service desks are overwhelmed with incoming messages through chat, email, and portals. Manually reading each message, determining if it's an incident (something broken) or a request (something needed), and creating the correct ServiceNow ticket consumes valuable agent time and delays resolution.

This AI-powered workflow solves that by automatically processing incoming chat messages. It uses GPT-4 to intelligently classify the message into one of five categories: Incident, Request, Follow-up, Update Action, or General Query. Based on the classification, it either creates a new incident or service request in ServiceNow, updates an existing ticket with new information, or searches knowledge bases for answers.

The result is instant ticket creation, accurate routing, and consistent data entry—freeing your IT team to focus on solving problems rather than administrative triage.

How It Works

1. Message Reception & Classification

The workflow is triggered by an incoming webhook from your chat platform (Slack, Teams, etc.). The message content is sent to an OpenAI classifier node that analyzes the language and intent, returning a structured classification with confidence scores.

2. Incident Creation & Routing

If classified as an incident, the workflow uses the ServiceNow node to create a new incident ticket with automatically populated fields: short description, category, urgency, and assignment group based on keywords. The system can be configured with your specific incident routing rules.

3. Service Request Handling

For requests, the workflow calls the ServiceNow Service Catalog API to create a formal request item. It maps the user's needs to predefined catalog items, ensuring proper fulfillment workflows are initiated automatically.

4. Ticket Updates & Summarization

When users provide updates to existing tickets, the workflow retrieves the ticket, appends new work notes, and uses AI to generate a concise summary of the conversation history for quick agent review.

5. Response & Notification

A confirmation message is sent back to the user through the original chat channel, providing their ticket number, expected resolution time, and next steps—creating a seamless closed-loop experience.

What You'll Need

  1. ServiceNow Instance with API access (Basic Authentication credentials)
  2. OpenAI API Key for GPT-4 classification and summarization capabilities
  3. n8n Instance (cloud or self-hosted) to run the workflow
  4. Webhook Source from your chat platform (Slack, Microsoft Teams, etc.)
  5. Optional: SerpAPI Key if you want general queries to trigger web searches

Pro tip: Before going live, train your AI classifier with 50-100 examples of your actual ticket descriptions. This improves accuracy by teaching it your organization's specific terminology and problem categories.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for ServiceNow (instance URL, username, password) and OpenAI (API key).
  3. Adjust classification prompts in the OpenAI nodes to match your incident/request definitions.
  4. Update ServiceNow field mappings to align with your specific ticket forms and workflows.
  5. Test with sample messages to verify accurate classification and ticket creation.
  6. Connect your chat webhook to the workflow trigger and deploy for production use.
