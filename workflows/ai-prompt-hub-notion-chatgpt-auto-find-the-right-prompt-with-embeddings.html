AI Prompt Hub: Automatically Find the Right Prompt with Notion & ChatGPT

Build your own semantic prompt search system. Store prompts in Notion, use HuggingFace embeddings, and let ChatGPT pick the perfect prompt based on meaning—not keywords.

What This Workflow Does

Managing dozens of AI prompts across different projects becomes chaotic. Developers waste time searching through text files or spreadsheets to find the “right” prompt for each situation. Keyword searches fail because prompts are conceptually similar but use different wording.

This workflow solves that by building your own AI Prompt Hub inside n8n. It connects your Notion prompt database with HuggingFace’s embedding models to perform semantic search. When a user sends any message to ChatGPT, the workflow automatically finds the most relevant prompt based on meaning—not exact keywords—and feeds it to ChatGPT. This ensures consistent, high-quality AI outputs while eliminating manual prompt hunting.

The system also automatically regenerates embeddings when prompts change in Notion, keeping your search index always up-to-date. It’s a complete, self-maintaining prompt management system for teams and developers.

How It Works

Step 1: Store Prompts in Notion

You maintain a Notion database with columns for Prompt (text), Embeddings (text), and Checksum (text). This becomes your central, collaborative prompt repository. Any team member can add or edit prompts with proper categorization.

Step 2: Automatic Embedding Generation

When a prompt is created or updated in Notion, the workflow triggers. It sends the prompt text to HuggingFace, which converts it into a vector embedding—a numerical representation capturing semantic meaning. This embedding is stored back in Notion alongside the prompt.

Step 3: Semantic Search Activation

A user interacts with ChatGPT through a chat interface. Instead of using a fixed prompt, ChatGPT calls this workflow as an AI Agent. The user’s query is converted into an embedding via HuggingFace.

Step 4: Similarity Matching & Prompt Selection

The workflow calculates similarity scores between the query embedding and all stored prompt embeddings. It returns the prompt with the highest similarity score—the one that conceptually matches the user’s intent.

Step 5: AI Execution with Optimal Prompt

ChatGPT receives the selected prompt and generates a response using it. The user gets AI output tailored with the most appropriate prompt, without ever knowing the complex search happened behind the scenes.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n instance.
  2. Set Credentials: Configure credentials for Notion, OpenAI, and HuggingFace in n8n.
  3. Prepare Notion Database: Create your database with the required columns and note its ID.
  4. Update Database IDs: Paste your Notion database ID into four nodes: “Get All Prompts,” “On Page Update,” “On Page Create,” and “Get All Prompts for Search.”
  5. Enable & Test: Activate the workflow and open the URL from “When chat message received.” Start chatting—the system will automatically search for matching prompts.

Pro tip: Start with a small set of 10–15 prompts in Notion to verify the semantic search works correctly. Once confirmed, scale up to hundreds. Use categories or tags in Notion for better organization.
