What This Workflow Does

This n8n workflow transforms your standard Telegram bot into an intelligent personal assistant capable of understanding and remembering conversations. Unlike basic chatbots that respond and forget, this assistant maintains long-term memory using Baserow as a database, allowing it to recall past interactions, save important notes, and provide contextually relevant responses over time.

The system processes multiple message types: text for direct communication, voice messages through AI transcription, and photos with caption analysis. This multimodal approach means users can interact naturally—sending voice memos, sharing images with instructions, or typing messages—all while the assistant intelligently determines what to remember, what to act upon, and how to respond appropriately.

How It Works

1. Message Reception & Routing

When a message arrives via Telegram, the workflow immediately categorizes it by type. A switch node examines whether it's text, voice, or photo content and routes it to the appropriate processing pipeline. This ensures each format receives specialized handling while maintaining a unified conversation thread.

2. Content Processing & Analysis

Voice messages are converted to text using AI transcription. Photos are analyzed for visual content and combined with any captions. Text messages proceed directly to interpretation. The system extracts key information, intent, and actionable items from each message type, preparing them for memory evaluation.

3. AI-Powered Decision Making

The core AI agent (using GPT-4o-mini) evaluates the processed message against conversation history stored in PostgreSQL memory buffers. It decides whether the information should be saved as long-term memory in Baserow, stored as a quick note, or used for immediate response generation based on predefined rules and context.

4. Memory Management & Response

Important details are saved to Baserow tables—personal preferences as "memories" and actionable items as "notes." The AI then crafts a personalized response that references relevant past interactions when appropriate, creating a natural, continuous conversation flow that feels genuinely helpful rather than robotic.

Quick Setup Guide

Follow these steps to implement this Telegram AI assistant in under 30 minutes:

  1. Create your Telegram bot via @BotFather and copy the API token.
  2. Set up Baserow tables with "Memory" and "Notes" tables following the template structure.
  3. Import the workflow JSON into your n8n instance and activate the webhook trigger.
  4. Configure credentials for Telegram, Baserow, OpenAI, and your database connections.
  5. Test with sample messages to verify memory storage and response quality before going live.
