What This Workflow Does

Running a bakery involves constant juggling: tracking daily sales, managing inventory, predicting demand, and spotting trends. Traditionally, this means hours spent manually reviewing spreadsheets, calculating totals, and trying to interpret numbers. This free n8n workflow template eliminates that manual grind.

It automates the entire process of analyzing bakery sales and stock data stored in Google Sheets. You interact with an AI assistant via a simple chat interface—ask questions like “What was our best-selling pastry last week?” or “Are we low on flour?”—and the system fetches the relevant data, performs calculations, and delivers clear, concise insights in plain English. It maintains conversation context so you can ask follow-up questions naturally, turning your spreadsheet into a conversational business advisor.

The workflow bridges Google Sheets (your data source) with Azure GPT Chat Assistant (your analysis engine), creating a seamless loop where data queries become actionable insights without any coding or complex reporting tools.

How It Works

Step 1: Trigger via Chat Message

The workflow starts when you send a question through a chat interface (like a webhook, Slack, or a custom frontend). This trigger captures your natural language query—for example, “Show me sales trends for croissants over the past month.”

Step 2: AI Agent Interprets the Query

The AI Agent node acts as the central reasoning engine. It interprets your intent, decides what data needs to be fetched, and structures the request for the Google Sheets node. It’s programmed to provide professional, concise responses and avoid making assumptions beyond the available data.

Step 3: Retrieve Data from Google Sheets

The workflow connects to your bakery’s Google Sheets dataset using OAuth2 credentials. It fetches the relevant rows and columns—sales figures, inventory counts, dates, product names—based on the AI’s parsed request. The data is cleaned and formatted for analysis.

Step 4: Simple Memory Maintains Context

A Simple Memory node stores short-term conversation history. This allows the AI to remember previous questions and answers, enabling natural follow-ups like “Compare that to last month” without you re-specifying the context.

Step 5: Azure OpenAI Generates the Insight

The Azure OpenAI Chat Model node processes the retrieved data and the user’s original query. It generates a human-readable summary, highlights key figures, and may present breakdowns in bullet points or mini-tables. The response is then sent back to you via the same chat interface.

Step 6: Deliver Actionable Output

The final output is a compact, conversational insight delivered directly to you. It might say: “Last week, croissants sold 320 units, generating $1,280. Sales peaked on Friday. Current flour stock is at 15kg—consider ordering more if weekly usage exceeds 20kg.”

Pro tip: Configure your Google Sheets with clear columns (Date, Product, Quantity Sold, Price, Stock Remaining) to ensure the AI can parse data accurately. Use consistent naming for products.

Quick Setup Guide

Follow these steps to deploy this template in your n8n environment:

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure Google Sheets credentials in n8n’s credentials section, linking to your bakery data sheet.
  3. Set up Azure OpenAI credentials with your API key and endpoint.
  4. Adjust the AI Agent instructions if needed to match your specific reporting style.
  5. Test with a simple query like “What were total sales yesterday?” to verify data retrieval and response.
  6. Connect your preferred chat channel (Slack, webhook, etc.) to the trigger node.
  7. Run the workflow and start asking questions about your bakery performance.

Pro tip: Start with a small, clean dataset in Google Sheets. Ensure date formats are consistent and product names are unambiguous to improve AI accuracy.
