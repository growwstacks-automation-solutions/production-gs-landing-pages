What This Workflow Does

Security teams are overwhelmed with suspicious emails that require manual analysis. This creates alert fatigue, slows response times, and increases the risk of missing critical threats. The manual process of checking emails, extracting indicators, and creating investigation cases in tools like TheHive consumes hours of analyst time each day.

This n8n workflow automates the entire email security analysis pipeline. It monitors your IMAP inbox for suspicious emails, automatically parses their content, extracts potential threats (URLs, attachments, sender information), and creates standardized investigation cases in TheHive. It can also trigger Cortex analyzers to examine attachments and provide threat intelligence context, turning raw email alerts into actionable security incidents.

How It Works

1. Email Monitoring & Trigger

The workflow starts by monitoring your designated IMAP email folder (like a "Suspicious" or "Phishing" inbox). It uses filters to identify emails that match your security criteria—specific senders, subjects containing threat keywords, or emails with attachments.

2. Content Extraction & Parsing

Once a suspicious email is detected, the workflow extracts all relevant information: email headers (From, To, Subject), body content (HTML and plain text), attachments, and embedded URLs. It cleans and structures this data for analysis.

3. Threat Analysis & Enrichment

The extracted data is analyzed using conditional logic. URLs are checked against threat intelligence, attachments can be sent to Cortex analyzers (like VirusTotal, Hybrid Analysis), and email content is scanned for patterns indicating phishing or malware distribution.

4. Case Creation in TheHive

Based on the analysis results, the workflow automatically creates a new case in TheHive with all extracted evidence. It sets appropriate severity levels, adds observables (IPs, domains, file hashes), and assigns the case to the relevant security team or analyst.

5. Notification & Reporting

Finally, the workflow sends notifications to your security team via Slack, Microsoft Teams, or email with a summary of the detected threat and a link to the newly created TheHive case for immediate investigation.

Who This Is For

This automation is ideal for security operations centers (SOCs), managed security service providers (MSSPs), IT security teams, and compliance officers. It's particularly valuable for organizations that receive regular suspicious email reports from employees, have dedicated security email addresses (like abuse@ or security@), or need to automate phishing analysis workflows.

Companies in regulated industries (finance, healthcare, government) benefit from the consistent, auditable process this automation provides. Even small businesses without dedicated security staff can use this to triage suspicious emails before they cause damage.

What You'll Need

  1. n8n instance (self-hosted or cloud)
  2. IMAP-enabled email account for receiving suspicious emails
  3. TheHive instance (v3 or v4) with API access
  4. Cortex instance (optional but recommended for advanced analysis)
  5. API credentials for any threat intelligence services you want to integrate
  6. Notification channel (Slack, Teams, or email) for alerts

Pro tip: Start with a dedicated email address like "security-alerts@yourcompany.com" that employees can forward suspicious emails to. This keeps your automation separate from regular business email and provides a clear reporting channel.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure the IMAP node with your email server details and folder to monitor
  3. Set up TheHive credentials in the respective nodes with your instance URL and API key
  4. Adjust filtering logic to match your security criteria (keywords, senders, attachment types)
  5. Test with a sample email to ensure the workflow triggers and creates a case correctly
  6. Activate the workflow and monitor the initial executions for any configuration issues
