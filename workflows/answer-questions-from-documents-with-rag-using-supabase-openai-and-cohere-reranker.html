What This Workflow Does

This comprehensive RAG (Retrieval-Augmented Generation) workflow enables your AI agents to answer user questions with contextual knowledge pulled from your own documents. It uses metadata-rich embeddings stored in Supabase for efficient document retrieval, OpenAI for generating answers, and Cohere's reranker to improve response quality.

The system handles the complete pipeline from document ingestion to question answering. It automatically processes uploaded documents, extracts their content, generates vector embeddings with metadata, stores them in Supabase, and provides an interface for querying this knowledge base with natural language questions.

How It Works

1. Document Processing

The workflow first extracts text from uploaded documents (PDFs, Word files, etc.) while preserving structural elements and metadata. This content is then chunked into manageable segments for embedding.

2. Vector Embedding

Each document chunk is processed through OpenAI's embedding model (like text-embedding-3-small) to create a numerical representation that captures semantic meaning. These embeddings are stored in Supabase along with their metadata.

3. Query Processing

When a question is received, the system first generates an embedding for the query itself. It then searches Supabase for document chunks with the most similar embeddings to the question.

4. Reranking

Cohere's reranker analyzes the initially retrieved documents and reorders them by contextual relevance to the question, ensuring the most appropriate content is prioritized.

5. Answer Generation

The top relevant document chunks are fed to OpenAI's language model (GPT-3.5 or GPT-4) along with the original question to generate a natural language answer grounded in your documents.
