What This Workflow Does

This automation template provides a complete solution for querying data through GraphQL APIs and processing the responses. It's designed to receive country codes via HTTP requests, fetch corresponding data from a GraphQL endpoint, convert the response to JSON format, and construct a return string with the processed information.

The workflow solves common challenges in API integration by handling the entire data flow from input to processed output. It demonstrates how to structure GraphQL queries in an automation context and process the nested responses that GraphQL often returns.

How It Works

1. Receiving the Input

The workflow starts by receiving a country code through an incoming HTTP request. Example URL structure: https://yourdomain.com/webhook/1/webhook/webhook?code=DE

2. Querying GraphQL

The country code is passed to a GraphQL query that retrieves specific data related to that country. The template shows how to structure GraphQL queries with variables.

3. Data Conversion

The GraphQL response is converted to JSON format, making it easier to work with in subsequent steps and compatible with most systems.

4. Response Construction

The workflow constructs a return string with the processed data, ready to be sent back to the requesting system or passed to another workflow step.
