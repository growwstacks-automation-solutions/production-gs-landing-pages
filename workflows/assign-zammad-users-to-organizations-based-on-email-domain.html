What This Workflow Does

Manual user assignment in customer support systems creates bottlenecks, misrouted tickets, and administrative headaches. When users sign up or are imported into Zammad, they often remain unassigned to organizations, causing confusion about which support team should handle their tickets.

This automation solves that problem by automatically analyzing every user's email domain and matching it against your configured organizations. If a user has an @acmecorp.com email, they're automatically assigned to the "Acme Corporation" organization. This happens instantly, whether for new signups or when cleaning up legacy user data.

The result is a perfectly organized support system where tickets are automatically routed to the correct teams, reporting accurately reflects organizational relationships, and your support staff can focus on solving problems instead of sorting users.

How It Works

Step 1: Fetch All Users

The workflow starts by retrieving all users from your Zammad instance through its secure API. It gathers essential information including user IDs, email addresses, and current organization assignments.

Step 2: Extract Email Domains

Each user's email address is parsed to extract the domain portion (everything after the @ symbol). This domain becomes the key for matching users to organizations.

Step 3: Match Domains to Organizations

The workflow compares extracted domains against your configured Zammad organizations. Organizations must have domain-based assignment enabled in Zammad for this to work seamlessly.

Step 4: Update User Assignments

For each successful match, the workflow updates the user's organization assignment in Zammad. The system logs all changes and can be configured to handle unmatched domains according to your business rules.

Pro tip: Run this workflow on a schedule (daily or weekly) to catch any users who might have been missed during initial setup or new signups.

What You'll Need

  1. Active Zammad Instance: A working Zammad installation with API access enabled.
  2. API Credentials: Zammad API token with permissions to read users and update organization assignments.
  3. Configured Organizations: Organizations in Zammad must be set up with domain-based assignment enabled.
  4. n8n Access: Either n8n.cloud account or self-hosted n8n instance to run the workflow.
  5. User Data: Existing users in Zammad with valid email addresses for processing.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure Zammad credentials in n8n by creating a new Zammad node connection with your API token.
  3. Review organization settings in Zammad to ensure domain-based assignment is enabled for relevant organizations.
  4. Test with a small subset of users first by modifying the workflow to limit processing to 5-10 users.
  5. Schedule execution once testing is successful—set it to run daily or weekly based on your user volume.

Implementation note: For large user bases (1000+ users), consider running the workflow during off-hours and implementing batch processing to avoid API rate limits.
