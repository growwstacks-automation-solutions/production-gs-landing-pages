Auto-assign JIRA Support Tickets with AI

Prevent tickets from falling through the cracks by automatically assigning stale issues to the most relevant team member using AI and historical data.

What This Workflow Does

Support tickets that go unassigned for days or weeks create frustrated customers and internal chaos. Manual ticket triage is time-consuming and often results in assignments based on who's available rather than who's best suited. This automation solves both problems by ensuring no JIRA issue remains unassigned for more than a week.

The workflow uses AI to analyze ticket content and compare it against a historical database of resolved issues stored in Supabase. It identifies which team member has the most relevant expertise based on past work, then automatically assigns the ticket while balancing current workload across the team. A comment is added to notify the assignee they've been auto-assigned due to the ticket's age.

This creates a self-correcting system where tickets never get lost in the backlog, response times improve, and team members work on issues that match their expertise—all without manual intervention.

How It Works

The automation consists of two coordinated flows that run continuously via schedule triggers.

1. Historical Data Collection

Daily, the workflow pulls recently resolved JIRA issues and stores them in a Supabase vector database. It extracts key metadata including issue descriptions, resolution notes, assignee information, and categories. This creates a searchable knowledge base of past work.

2. Stale Ticket Detection

Another scheduled flow scans for unassigned JIRA issues older than your configured threshold (default: 7 days). For each stale ticket, it generates an AI embedding of the issue description and searches the vector database for similar resolved tickets.

3. Intelligent Assignment

The system identifies team members who have successfully handled similar issues in the past. It then checks each candidate's current workload (number of assigned, in-progress tickets) and assigns the ticket to the member with both relevant expertise and available capacity.

4. Notification & Documentation

Once assigned, the workflow adds a comment to the JIRA ticket explaining the auto-assignment logic and notifying the team member. All assignment decisions are logged for transparency and future optimization.

What You'll Need

  1. JIRA account with API access to read issues and update assignments
  2. OpenAI API key for generating embeddings and understanding ticket content
  3. Supabase account with vector extension enabled for storing historical issue data
  4. n8n instance (cloud or self-hosted) to run the automation
  5. Team member list with their JIRA usernames for assignment

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Configure credentials for JIRA, OpenAI, and Supabase in n8n's credential management
  3. Set your JIRA project key and adjust the "stale threshold" (default: 7 days)
  4. Define your team members by updating the list of JIRA usernames in the workflow
  5. Test with a small project first to verify assignment logic works as expected
  6. Schedule both flows to run at appropriate intervals (daily for data collection, hourly for assignment)
  7. Monitor for a week and adjust assignment criteria based on results

Pro tip: Start with a longer stale threshold (10-14 days) during testing to avoid overwhelming your team. Gradually reduce it as confidence in the automation grows.
