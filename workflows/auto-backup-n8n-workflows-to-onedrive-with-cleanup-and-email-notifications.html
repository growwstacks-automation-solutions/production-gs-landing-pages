What This Workflow Does

This automation solves a critical problem for businesses using self-hosted n8n instances: workflow backup reliability. Manual exports are forgotten, inconsistent, and vulnerable to server failures. This workflow automatically exports all your n8n workflows as individual JSON files, uploads them to Microsoft OneDrive with timestamped versioning, cleans up old backups to prevent storage bloat, and sends email notifications for confirmation and monitoring.

It transforms a manual administrative task into a fully automated system that runs on schedule (daily, weekly, or custom), ensuring your automation investments are protected against data loss. The workflow creates a complete disaster recovery solution with audit trails and version history accessible from anywhere via cloud storage.

How It Works

Step 1: Fetch All Workflows

The workflow starts by calling your n8n instance's REST API to retrieve all existing workflows. It uses HTTP Request nodes with authentication to securely access your workflow data without exposing credentials.

Step 2: Process Each Workflow

A SplitInBatches node iterates through each workflow, converting them to properly formatted JSON files with metadata including workflow names, IDs, and export timestamps.

Step 3: Upload to OneDrive

Each JSON file is uploaded to a designated OneDrive folder using OAuth2 authentication. Files are organized with clear naming conventions for easy identification and recovery.

Step 4: Cleanup Old Backups

The workflow automatically scans the backup folder and deletes files older than your configured retention period (e.g., 30, 60, or 90 days). This prevents unlimited storage consumption while maintaining recent backup history.

Step 5: Send Notification Email

Upon completion, an email is sent via SMTP or Microsoft Outlook to designated recipients with backup summary statistics: number of workflows backed up, file sizes, and any errors encountered.

What You'll Need

  1. A self-hosted n8n instance with REST API access
  2. Microsoft OneDrive account with OAuth2 credentials configured
  3. Email service credentials (SMTP or Microsoft Outlook)
  4. Basic understanding of n8n workflow import process
  5. Decided backup retention period (how long to keep old backups)

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n instance.
  2. Configure the HTTP Request node with your n8n base URL and authentication details.
  3. Set up Microsoft OneDrive OAuth2 credentials in the OneDrive nodes.
  4. Configure the Email node with your SMTP or Outlook credentials.
  5. Adjust the retention logic in the cleanup section to match your backup policy.
  6. Set the Schedule Trigger node to your desired backup frequency (daily recommended).
  7. Test the workflow manually before enabling automated scheduling.

Pro tip: Start with weekly backups while testing, then move to daily once confirmed stable. Always verify you can restore workflows from backup files before relying on the system.
