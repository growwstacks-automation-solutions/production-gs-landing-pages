What This Workflow Does

Email overload is a real productivity killer for freelancers, business professionals, and teams managing high-volume communication. Manually sorting through dozens or hundreds of emails daily consumes valuable time that could be spent on revenue-generating activities. This workflow solves that problem by automating the entire categorisation process.

Using AI-powered analysis, the system automatically reads incoming Outlook emails, understands their content and context, and sorts them into appropriate categories like Action Required, Junk, Business Correspondence, SaaS Notifications, or Receipts. It then moves emails to designated folders and can even flag urgent items, creating an organised inbox that helps you focus on what matters most.

How It Works

The automation follows a logical sequence that mimics how an efficient assistant would process your emails, but at scale and without human error.

1. Email Retrieval & Initial Filtering

The workflow connects to your Microsoft Outlook account via secure API credentials and retrieves new, unread emails. It first filters out already-flagged messages and previously categorised emails to avoid duplicate processing, ensuring only fresh communications are analysed.

2. Content Preparation & Cleaning

Each email's content is extracted and converted into a clean, structured format using Markdown. This step removes HTML formatting, signatures, and disclaimers that could confuse the AI model, presenting only the core message for accurate analysis.

3. AI-Powered Categorisation

The prepared content is sent to an AI model (configurable with Ollama API or similar services) that analyses the text against predefined categories. The AI considers context, intent, sender patterns, and content keywords to assign the most appropriate category with a confidence score.

4. Automated Folder Management

Based on the AI's classification, the workflow updates each email's metadata in Outlook and moves it to corresponding folders. For example, receipts go to an accounting folder, junk gets archived, and action items remain prominently visible in your primary inbox or a dedicated action folder.

5. Error Handling & Quality Control

The system includes conditional checks to handle edge cases and errors gracefully. If categorisation confidence is below a threshold, emails can be flagged for manual review rather than automatically sorted, ensuring important messages are never misplaced.

What You'll Need

  1. Microsoft Outlook Account: Office 365 or Outlook.com with API access enabled.
  2. n8n Instance: Self-hosted n8n or n8n.cloud account to run the workflow.
  3. AI Model Access: Ollama API (local or cloud) or alternative AI service with text classification capabilities.
  4. Outlook Folder Structure: Pre-created folders in Outlook matching your desired categories (Action, Junk, Business, Receipts, etc.).
  5. Basic Automation Understanding: Familiarity with connecting APIs and adjusting workflow settings.

Pro tip: Start with broad categories (3-5 main folders) and refine as the AI learns your preferences. Over-categorisation can defeat the purpose of simplification.

Quick Setup Guide

Getting started takes about 15-20 minutes if you have your credentials ready.

  1. Import the Template: Download the JSON file and import it into your n8n instance using the "Import from File" option.
  2. Connect Outlook: Set up the Microsoft Outlook node with your account credentials. Test the connection by fetching a sample email.
  3. Configure AI Service: Connect to your preferred AI model (Ollama recommended for privacy). Adjust the prompt to match your desired categories.
  4. Map Your Folders: Update the folder names in the workflow to match your actual Outlook folder structure.
  5. Test with Sample Data: Run the workflow manually with a few test emails to verify categorisation accuracy before enabling automation.
  6. Schedule Automation: Set the workflow to run every 15-30 minutes or trigger it when new emails arrive, depending on your n8n plan.
