What This Workflow Does

Manually categorizing blog posts is a tedious, time-consuming task that becomes overwhelming as your content library grows. This workflow eliminates that manual process by using artificial intelligence to automatically analyze your WordPress post titles and assign them to the most relevant categories.

The automation connects to your WordPress site via its REST API, retrieves all uncategorized posts, sends the titles to an AI service for analysis, and then updates each post with the appropriate category ID. What typically takes hours of manual review can be completed in minutes, with consistent, intelligent categorization that improves your site's organization and SEO.

Beyond basic categorization, this system maintains a consistent taxonomy across your entire content library, ensures new posts are properly organized from day one, and can be scheduled to run automatically—freeing up your editorial team to focus on creating content rather than organizing it.

How It Works

The workflow follows a logical sequence that mimics how a human editor would categorize content, but with AI-powered precision and consistency.

1. Trigger & WordPress Connection

The workflow begins by connecting to your WordPress site using secure API credentials. It queries for posts that lack category assignments, filtering specifically for content that needs organization. This initial step ensures the automation only processes uncategorized content, avoiding redundant updates.

2. AI Analysis & Classification

Each post title is sent to an AI model (like OpenAI's GPT or Claude) with a carefully crafted prompt that includes your category definitions. The AI analyzes the semantic meaning, context, and subject matter to determine the most appropriate category based on your predefined taxonomy.

3. Category Mapping & Assignment

The AI's category recommendation is matched against your WordPress category IDs. The workflow includes error handling to ensure only valid category IDs are applied, with fallback options for ambiguous cases. This mapping process maintains consistency with your existing category structure.

4. WordPress Update & Logging

Finally, each post is updated via the WordPress REST API with its new category assignment. The workflow logs all changes for review and includes error reporting for any posts that couldn't be categorized, ensuring complete transparency and control over the automation process.

Pro tip: Before running this on your entire blog, test it with a small batch of posts (5-10) to verify the AI's categorization accuracy matches your expectations. Adjust your category descriptions in the prompt if needed.

What You'll Need

  1. WordPress Admin Access: You'll need administrator credentials or an application password with write permissions to your WordPress site.
  2. Predefined Categories: Your WordPress site must have categories already created. Note down the category IDs you want the AI to use.
  3. AI API Key: An API key from OpenAI, Anthropic, or another AI provider that offers text classification capabilities.
  4. n8n Instance: Either n8n.cloud account or self-hosted n8n installation with internet access to your WordPress site and AI provider.
  5. Category Definitions: Clear descriptions of what each category represents to guide the AI's classification decisions.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the Template: Download the JSON file above and import it into your n8n instance using the "Import from File" option.
  2. Configure WordPress Credentials: Set up the WordPress node with your site URL and admin API credentials. Test the connection by retrieving a sample post.
  3. Set Up AI Integration: Add your AI provider credentials (OpenAI, Claude, etc.) and update the prompt with your specific category names and IDs.
  4. Define Your Categories: In the AI prompt node, replace the example categories with your actual WordPress category names and their corresponding IDs.
  5. Test with Sample Data: Run the workflow manually with a small batch of posts to verify categorization accuracy before automating the entire process.
  6. Schedule Automation: Once tested, add a schedule trigger to run this workflow daily, weekly, or whenever new posts are published.

Important: Always backup your WordPress database before running categorization automation for the first time. While the workflow includes safeguards, having a restore point ensures you can easily revert any incorrect categorizations.
