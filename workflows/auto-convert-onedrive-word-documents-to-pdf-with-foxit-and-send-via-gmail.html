What This Workflow Does

This automation solves the tedious, error-prone process of manually converting Word documents to PDF and emailing them. When documents are added to a specific OneDrive folder, the workflow automatically detects them, converts them to PDF using Foxit's conversion API, and emails the PDF version to designated reviewers or recipients via Gmail.

Businesses waste countless hours on manual document processing—downloading files, opening conversion tools, saving in different formats, and sending emails. This workflow eliminates those repetitive tasks, ensuring documents are processed consistently and delivered promptly without human intervention. It's particularly valuable for legal firms, accounting practices, HR departments, and any business that regularly processes client documents or internal reports.

How It Works

1. Monitor OneDrive for New Documents

The workflow continuously monitors a specified OneDrive folder for new Word documents. When a document is added, it triggers the automation process immediately, ensuring no documents are missed or delayed in processing.

2. Download and Validate Document

The system downloads the Word document and validates it's in a supported format. This step ensures only appropriate documents proceed through the conversion process, preventing errors from incompatible file types.

3. Convert to PDF Using Foxit API

The document is securely uploaded to Foxit's conversion service, which transforms it into a high-quality PDF while preserving formatting, images, and document structure. The conversion happens in the cloud without requiring local software installation.

4. Email PDF via Gmail

Once converted, the PDF is automatically attached to an email sent through Gmail. The email can include custom subject lines, body text, and be directed to specific recipients based on the document type or source folder.

What You'll Need

  1. Microsoft OneDrive account with documents stored in cloud storage
  2. Foxit developer account for document conversion API access
  3. Gmail account or Google Workspace for sending emails
  4. n8n instance (cloud or self-hosted) to run the workflow
  5. API credentials for all connected services

Pro tip: Start with a test folder in OneDrive and a personal Gmail account to validate the workflow before implementing it for critical business documents. This allows you to fine-tune the automation without risking important files.

Quick Setup Guide

1. Download the template using the button above and import it into your n8n instance.

2. Configure the OneDrive trigger node with your folder ID and authentication credentials.

3. Set up the Foxit node with your API key and configure conversion settings as needed.

4. Connect the Gmail node with your account credentials and customize the email template.

5. Test the workflow by uploading a Word document to your monitored OneDrive folder.

6. Once verified, activate the workflow to run automatically on a schedule or in real-time.
