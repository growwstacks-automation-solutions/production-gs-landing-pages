What This Workflow Does

Documenting complex workflows is one of the most tedious yet critical tasks in business automation. Without proper documentation, team members waste hours trying to understand existing automations, knowledge is lost when employees leave, and scaling becomes nearly impossible. This workflow solves that problem by automatically generating comprehensive documentation for your n8n workflows using AI.

The template takes any workflow JSON file, analyzes each node's function, and creates clear, concise sticky notes that explain what each part does. It also generates an overall workflow overview that describes the automation's purpose, data flow, key parameters, and potential gotchas. Everything is arranged neatly on your canvas, transforming complex technical workflows into easily understandable visual blueprints.

This isn't just about saving time—it's about creating living documentation that evolves with your workflows. When you modify an automation, you can re-run this documentation process to keep everything current. This ensures your entire team stays aligned and can work with any automation, regardless of who originally built it.

How It Works

The workflow follows a systematic process to analyze and document your automations with minimal manual intervention.

Step 1: Load and Parse Your Workflow

The process begins by loading your existing n8n workflow JSON file. The system parses through all nodes, identifying their types, configurations, and connections while filtering out any existing sticky notes to avoid duplication.

Step 2: Generate Node-Level Documentation

Using GPT-4o-mini, the workflow analyzes each node individually. For every trigger, action, and transformation node, it creates a concise sticky note explaining what that specific component does, what data it processes, and how it contributes to the overall automation.

Step 3: Create Overall Workflow Overview

The AI then examines the complete workflow structure to generate a comprehensive overview. This includes the automation's primary goal, the step-by-step flow of data, important configuration parameters, and any potential issues or considerations users should be aware of.

Step 4: Arrange Documentation Visually

All generated sticky notes are automatically positioned on your canvas using a right-to-left layout algorithm. Each node is paired with its documentation note placed directly below it, creating a clean, readable documentation layer that doesn't interfere with the workflow's functionality.

Step 5: Save and Share

The final documented workflow is saved as a new JSON file, ready to be imported into n8n, shared with team members, or submitted to marketplaces. The original workflow remains untouched, giving you both versions for flexibility.

  4. Basic understanding of how to import workflows and configure API credentials in n8n

Pro tip: Start with your most complex or frequently modified workflows first. The time savings from having clear documentation will be most noticeable there, and you'll quickly see the value this automation brings to your team.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance using the workflow import feature.
  2. Configure the file path in the "Load Workflow" node to point to your existing workflow JSON that needs documentation.
  3. Add your OpenAI API key in both the "Node Sticky Notes" and "Overall Sticky Note" nodes within their credential settings.
  4. Set the output path in the "Save Documented Workflow" node where you want the documented version saved.
  5. Execute the workflow manually to test. Once confirmed working, you can schedule it or trigger it whenever workflows are updated.
