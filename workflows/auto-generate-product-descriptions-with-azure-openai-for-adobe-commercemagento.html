What This Workflow Does

This n8n automation template solves a common but time‑consuming e‑commerce problem: blank or poorly written product descriptions. For Magento/Adobe Commerce stores with large catalogs, manually writing unique, SEO‑optimized descriptions for every product is impractical. This workflow automatically generates professional, on‑brand product descriptions using Azure OpenAI (or any LLM) and pushes them directly back into your Magento store.

The system intelligently fetches products missing descriptions, enriches raw attribute data (turning numeric IDs into readable labels like “Red” instead of “color_id: 23”), and uses a customizable AI prompt to create compelling copy. It ensures every product has a complete, sales‑ready listing without manual intervention, improving both customer experience and search engine visibility.

Beyond filling gaps, this automation maintains consistency across your entire catalog, adapts tone for different product categories, and can be scheduled to run continuously—keeping your store’s content fresh and comprehensive as new products are added.

How It Works

The workflow operates in three clear stages: data retrieval, AI processing, and store update.

1. Fetch Products from Magento

The automation connects to your Adobe Commerce instance via its REST API and queries for products where the description field is empty or below a certain quality threshold. It retrieves all relevant product data, including SKU, name, price, categories, and custom attributes.

2. Enrich Data & Generate Description

Raw attribute values (like dropdown IDs) are mapped to their human‑readable labels. This structured, enriched data is then sent to Azure OpenAI with a tailored prompt that instructs the model on brand voice, key features to highlight, SEO keywords, and desired length. The AI returns a unique, engaging product description.

3. Update Magento Product

The generated description is formatted and posted back to Magento via API, updating the product record instantly. The workflow can include quality‑control steps, like checking for appropriate length or flagging for human review if confidence is low, before final submission.

What You'll Need

  1. An n8n instance (cloud or self‑hosted).
  2. Adobe Commerce (Magento 2) store with API access (Admin token with product write permissions).
  3. Azure OpenAI API credentials (or credentials for OpenAI, Anthropic, Gemini, etc.).
  4. Basic understanding of your product attribute structure in Magento.
  5. Optional: A style guide or example descriptions to fine‑tune the AI prompt.

Pro tip: Before running at scale, test the workflow on a few sample products. Adjust the AI prompt to match your brand’s tone—whether it’s technical, luxurious, or conversational—and include key SEO phrases you want to target.

Quick Setup Guide

Import and configure this template in under 15 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Magento node with your store’s base URL and an admin API token.
  3. Set up the Azure OpenAI node (or your chosen LLM) with your API endpoint and key.
  4. Customize the prompt in the AI node to reflect your brand voice and any specific instructions (word count, keyword inclusion, etc.).
  5. Test with a single product by temporarily limiting the search query, then review the output.
  6. Schedule the workflow to run daily or weekly, or trigger it via webhook when new products are added.
