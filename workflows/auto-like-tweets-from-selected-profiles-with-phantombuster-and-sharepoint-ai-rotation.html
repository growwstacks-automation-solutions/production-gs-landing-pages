Auto-like Tweets from Selected Profiles with Phantombuster & SharePoint AI Rotation

Automate Twitter engagement by intelligently liking posts from target profiles. This workflow rotates sessions, prevents duplicates, and maintains authentic activity patterns.

What This Workflow Does

Manual Twitter engagement is time-consuming and inconsistent. Growth hackers, community managers, and marketers struggle to maintain active presence by liking relevant content from key profiles—influencers, customers, or industry leaders. This automation solves that by systematically engaging with selected accounts while appearing completely organic.

The workflow fetches recent tweets from your target profiles, selects ones you haven't liked before, rotates between different Twitter sessions to avoid detection, and executes likes through Phantombuster's automation agent. All activity is logged in SharePoint to prevent duplicates and provide an audit trail. The result is consistent, strategic engagement that builds relationships and increases your visibility without manual effort.

How It Works

1. Scheduled Trigger & Profile Processing

The workflow activates on a schedule (configurable from hourly to daily). It reads your list of target Twitter profiles from a SharePoint CSV file, then uses Phantombuster's Profile Post Extractor to fetch up to 20 recent tweets from each profile.

2. Session Rotation & Duplicate Prevention

Before liking anything, the workflow selects a Twitter session cookie from a rotating pool stored in SharePoint. This mimics different user sessions, reducing platform detection risk. It then checks each candidate tweet against a "already liked" CSV to ensure you never engage with the same post twice.

3. Intelligent Selection & Execution

The system randomly selects a qualifying tweet (avoiding patterns), builds a clean CSV of posts to like, and uploads it back to SharePoint. Phantombuster's Autolike Agent then executes the like action using the selected session cookie.

4. Logging & Maintenance

Finally, the liked tweet URL is appended to the "already liked" CSV, creating a permanent record. Error handling ensures failed attempts are logged for review without breaking the entire workflow.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution permissions.
  2. Phantombuster account with Twitter Autolike Agent configured and API credentials.
  3. Microsoft SharePoint access with a dedicated "Phantombuster" folder for storing CSV files and session cookies.
  4. Twitter (X) accounts with valid session cookies exported to text files for rotation.
  5. Target profile list in CSV format with a "profileUrl" column containing direct Twitter profile links.

Pro tip: Use multiple Twitter accounts with different session cookies in your rotation pool. Space out automation runs to 2-4 hour intervals to mimic human browsing patterns and avoid triggering rate limits.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for Phantombuster and SharePoint in n8n's credentials management.
  3. Prepare SharePoint folder named "Phantombuster" with three files: "twitter_session_cookies.txt" (one cookie per line), "twitter_posts_already_liked.csv" (with "postUrl" header), and "profiles_twitter.csv" (with "profileUrl" header and target profiles).
  4. Update workflow nodes with your specific SharePoint folder paths and Phantombuster agent IDs.
  5. Test with one profile by running the workflow manually before enabling the schedule trigger.
  6. Activate and monitor the schedule, checking SharePoint logs weekly to ensure proper duplicate prevention.
