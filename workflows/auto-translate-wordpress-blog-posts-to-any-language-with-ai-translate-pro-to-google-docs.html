What This Workflow Does

This n8n workflow solves the challenge of creating multilingual versions of WordPress content without manual translation work. When you publish a new blog post, the system automatically detects it, sends the content to AI Translate Pro for conversion into your target language(s), then saves the translated versions as organized Google Docs.

The automation eliminates hours of copying/pasting content between translation tools and your CMS. It ensures consistent formatting across all language versions while maintaining a searchable archive of translations for future updates or reference.

How It Works

1. WordPress Post Detection

The workflow monitors your WordPress site for new or updated posts using the REST API. When content is published, it extracts the post title, body, and metadata for processing.

2. AI Translation Processing

The system sends the content to AI Translate Pro with your specified target languages. The translation service uses neural machine learning to produce accurate, context-aware translations that preserve the original meaning and tone.

3. Google Docs Creation

For each translated version, the workflow creates a new Google Doc with standardized naming (e.g., "[Post Title] - [Language Code]"). The documents include the translated content with original formatting preserved, ready for review or direct publishing.

What You'll Need

  1. An n8n instance (cloud or self-hosted)
  2. WordPress admin access with REST API enabled
  3. AI Translate Pro account with API credentials
  4. Google Workspace account with Docs API access
  5. List of target languages for translation

Quick Setup Guide

  1. Download the JSON template file
  2. Import into your n8n dashboard
  3. Configure WordPress API connection with your site URL and credentials
  4. Set up AI Translate Pro node with your API key and target languages
  5. Connect Google Docs node with your Google account
  6. Test with a sample post and activate the workflow
