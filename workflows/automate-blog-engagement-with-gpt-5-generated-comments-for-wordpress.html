What This Workflow Does

Maintaining an active comment section is one of the biggest challenges for blog owners. Empty comment sections signal to readers that your content isn't engaging, while manually creating comments is time-consuming and unsustainable. This creates a vicious cycle where great content goes unnoticed because it lacks social proof.

This n8n workflow solves this problem by automatically generating realistic, context-aware comments for your WordPress articles using AI. It fetches your published posts, analyzes their content, creates thoughtful comments that match the article's tone, and posts them back to your WordPress site. The result is an engaged-looking blog that encourages real readers to participate while improving your SEO through increased on-page engagement signals.

Pro tip: Use this workflow to seed initial discussions on new articles. Once real comments start flowing, you can reduce AI frequency or use it only for older posts that need revival.

How It Works

The automation follows a logical sequence that mimics how a thoughtful reader would engage with your content, but at scale and automatically.

Step 1: Fetch Recent WordPress Articles

The workflow connects to your WordPress REST API and retrieves recently published articles. It can filter by category, date range, or specific tags to target the most relevant content for comment generation.

Step 2: Analyze Content and Generate AI Prompt

For each article, the system extracts the title, excerpt, and key content sections. It then constructs a tailored prompt for the AI that includes instructions to generate natural-sounding comments—some positive, some with questions, some sharing personal experiences related to the topic.

Step 3: Generate Context-Aware Comments

Using OpenAI's API (compatible with GPT-3.5, GPT-4, or other models), the workflow creates unique comments that reference specific points from the article. The AI varies comment length, tone, and perspective to avoid patterns that might appear artificial.

Step 4: Format and Post to WordPress

Each generated comment is assigned a random but plausible name and email address, then posted back to your WordPress site through the comments API. The workflow can be configured to space out comments naturally rather than posting them all at once.

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance using the "Import from File" option.
  2. Configure credentials: Add your OpenAI API key and WordPress credentials (username + application password) in n8n's credential management.
  3. Update WordPress URL: Replace the sample WordPress URL in the HTTP Request node with your actual site address.
  4. Test with one article: Execute the workflow manually first to ensure comments post correctly and appear as intended.
  5. Schedule automation: Set the workflow to run daily or weekly, or trigger it whenever new articles are published.

Pro tip: Start with conservative settings—generate 1-2 comments per article initially. Monitor how they look on your site and adjust the AI prompting before scaling up.
