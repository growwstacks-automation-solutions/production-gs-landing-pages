What This Workflow Does

Manual candidate creation is a time-consuming bottleneck in recruitment. When applicants submit their information through a career page, someone has to manually copy their details into your Applicant Tracking System (ATS), match them to the correct job opening, and upload their resume—a process that can take 10-15 minutes per candidate and is prone to human error.

This n8n workflow completely automates that entire process. It captures candidate submissions from an online form, uses AI to intelligently match them to the appropriate vacancy in Recrutei based on the job title they selected, extracts text from their uploaded PDF resume, and creates a fully populated candidate profile with all attachments and notes—all within seconds of application.

The system ensures candidates are immediately routed to the correct hiring pipeline, their resume content becomes searchable within Recrutei, and your recruitment team can focus on evaluating qualified candidates rather than data entry. It's particularly valuable for companies receiving high volumes of applications or those with multiple simultaneous openings.

How It Works

The automation follows a logical sequence that mimics what a human recruiter would do, but with perfect consistency and instant execution.

Step 1: Form Submission Trigger

The workflow activates when a candidate submits your n8n-powered application form. It captures their name, email, phone number, selected vacancy name (like "Senior Frontend Developer"), and their uploaded PDF resume.

Step 2: AI-Powered Vacancy Matching

Here's where the magic happens. The vacancy name from the form is sent to an OpenAI GPT-4 agent equipped with Recrutei's Model Context Protocol (MCP) tool. The AI intelligently searches your Recrutei instance to find the exact vacancy ID that corresponds to the job title the applicant selected.

Step 3: Candidate Profile Creation

With the correct vacancy ID identified, the workflow fetches the available pipeline stages for that job and creates a new "Prospect" candidate in Recrutei. All form data is mapped to the appropriate fields in the ATS, and the candidate is placed in the first available stage of the hiring process.

Step 4: Resume Text Extraction & Attachment

The uploaded PDF resume is processed to extract all textual content. This extracted text is then added as an internal observation/note to the candidate's profile (using observation type ID 11 in Recrutei), making their qualifications instantly searchable within the system without needing to open the PDF.

What You'll Need

  1. Recrutei ATS account with API access enabled
  2. Recrutei API key for authentication
  3. OpenAI API key (GPT-4 or GPT-3.5-turbo) for the AI matching component
  4. Recrutei MCP Server endpoint URL (provided by Recrutei or configured separately)
  5. n8n instance (cloud or self-hosted) with HTTP Request and Form Trigger nodes available

Pro tip: Before implementing, ensure your Recrutei vacancy names are consistent and descriptive. The AI matching works best when job titles in your ATS match what candidates would naturally select on your application form.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the template using the download button above
  2. Create a Header Auth credential named "Recrutei API" with Authorization: Bearer YOUR_API_KEY
  3. Configure the OpenAI node with your API key in the "Get Vacancy ID from AI" step
  4. Update the Recrutei MCP Tool node with your actual MCP server endpoint URL
  5. Test with a sample application to verify vacancy matching and profile creation work correctly
  6. Embed the n8n form on your career page or link it from your existing application system

The workflow includes error handling and fallback logic. If the AI agent cannot confidently match an applicant to a specific vacancy, the system can be configured to either: 1) Place the candidate in a 'General Applications' pipeline, 2) Send an alert to a recruiter for manual review, or 3) Trigger a follow-up email to the applicant requesting clarification on their role preference.

This ensures no candidate falls through the cracks. You can set confidence thresholds for the AI matching, and when below threshold, implement human-in-the-loop workflows that still save time while ensuring quality control for edge cases.

Modern PDF text extraction tools handle most standard PDF formats with high accuracy, including text-based PDFs and those generated from Word documents. However, scanned PDFs (image-based resumes) require OCR technology. This template focuses on text-based PDFs. For scanned resumes, you would need to add an OCR node to convert images to text before parsing.

Most professional applicants submit text-based PDFs, but if you receive many scanned resumes, consider adding a step like Tesseract OCR or using a service like Adobe PDF Extract. The extracted text quality directly impacts how searchable candidate information becomes within your ATS.
