What This Workflow Does

This automation solves the tedious, time-consuming process of manually researching companies. Instead of spending hours browsing Crunchbase, LinkedIn, and traffic estimation tools, this workflow automatically gathers comprehensive company intelligence with just a domain name.

It connects to the ProspectLens API—a powerful data source that aggregates information from multiple business intelligence platforms—and systematically extracts key metrics like estimated monthly traffic, founding year, funding rounds, employee count, and executive team details. The structured data is then automatically logged into a Google Sheets spreadsheet, creating a searchable, sortable company database.

The workflow is particularly valuable for sales teams building prospect lists, investors screening opportunities, marketers analyzing competitors, or recruiters sourcing target companies. It transforms what was traditionally a days-long research project into a minutes-long automated process.

How It Works

Step 1: Input Company Domains

The workflow begins with a list of company domains you want to research. This can come from a manual trigger, a scheduled run, or be fed from another system like your CRM or lead list.

Step 2: Query ProspectLens API

For each domain, the workflow sends a request to the ProspectLens API with your authentication key. The API returns structured JSON containing dozens of data points about the company.

Step 3: Parse and Transform Data

The raw API response is processed to extract the most valuable information. This includes cleaning up data formats, converting timestamps, and calculating derived metrics like company age or funding per employee.

Step 4: Write to Google Sheets

The cleaned data is appended as a new row in your designated Google Sheets spreadsheet. Each column corresponds to a specific data point (company name, domain, traffic, funding, etc.), creating a structured database.

Step 5: Error Handling and Logging

The workflow includes robust error handling for cases where companies aren't found in the database or API limits are reached. Failed attempts are logged separately for follow-up.

What You'll Need

  1. ProspectLens API Key: Available from APIRoad marketplace (free tier available with limited queries).
  2. Google Sheets Account: With a spreadsheet prepared with column headers for the data you want to collect.
  3. n8n Instance: Either self-hosted n8n or n8n.cloud account (free tier sufficient for moderate usage).
  4. Company Domain List: A list of domains you want to research (can start with just a few for testing).
  5. Basic Understanding of APIs: Enough to configure API keys and understand rate limits.

Quick Setup Guide

  1. Download and Import: Download the JSON template file and import it into your n8n instance.
  2. Configure Credentials: Set up HTTP Header credential with your ProspectLens API key (x-apiroad-key).
  3. Connect Google Sheets: Authenticate n8n with your Google account and select your target spreadsheet.
  4. Test with Sample Data: Run the workflow with 2-3 test domains to verify data flows correctly.
  5. Schedule Automation: Set the workflow to run on a schedule (daily/weekly) or trigger it manually as needed.
  6. Customize Output: Modify the Google Sheets columns or add additional data processing based on your needs.

Pro tip: Start with a small batch of companies (5-10) to test the workflow and verify data quality before scaling to hundreds of domains. This helps you identify any formatting issues or missing data points early.
