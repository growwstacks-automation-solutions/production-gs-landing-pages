What This Workflow Does

Manually reviewing CVs from Telegram job applicants is slow, error-prone, and often inconsistent. This workflow automates the entire front-end of your recruitment pipeline. It listens for CVs sent via a Telegram bot, validates they are proper PDFs, uploads them securely to Google Drive, uses Google's Gemini AI to extract key candidate information, and logs everything into a structured Google Sheet.

For HR teams and recruiters, this means no more downloading, renaming, and manually entering data from hundreds of resumes. The system filters out invalid submissions, standardizes data extraction, and creates a searchable candidate database instantly. You get a clean, automated pipeline from applicant submission to shortlisted candidate, saving 5–10 minutes per CV and ensuring no qualified applicant gets lost in an inbox.

How It Works

The automation connects three powerful platforms to create a seamless hiring assistant.

Step 1: CV Collection via Telegram Bot

A Telegram bot is set up to receive messages and document uploads from candidates. The workflow triggers whenever a new message arrives in the designated chat. It first checks if the message contains a document and validates that it's a PDF file, rejecting images or other formats immediately.

Step 2: Secure Storage & Text Extraction

Valid PDFs are uploaded to a specific folder in Google Drive with a consistent naming convention. The workflow then extracts the raw text from the PDF, preparing it for AI analysis. This step ensures the CV is preserved and accessible while making the content machine-readable.

Step 3: AI-Powered CV Parsing with Gemini

The extracted text is sent to Google's Gemini AI with a carefully crafted prompt. The AI identifies and structures key information: candidate name, contact details, skills, years of experience, education, and current job title. It outputs this as clean JSON data, ready for database entry.

Step 4: Data Logging & Deduplication

The structured data is appended to a Google Sheet, creating a real-time candidate tracker. The workflow includes logic to check for existing entries based on name or email, preventing duplicate records. HR teams now have a live, sortable database of all applicants with direct links to their stored CVs.

Pro tip: Customize the AI prompt to extract specific skills or qualifications relevant to your roles. You can add scoring logic to automatically rank candidates based on keyword matches.

What You'll Need

  1. A Telegram Bot Token (created via @BotFather)
  2. Google Cloud Project with Drive and Sheets APIs enabled
  3. Service Account credentials for Google Workspace
  4. Google Gemini API key (free tier available)
  5. An n8n instance (cloud or self-hosted)
  6. A Google Sheet prepared with column headers for candidate data
  7. A dedicated Google Drive folder for CV storage

Quick Setup Guide

Get this workflow running in your n8n instance in under 30 minutes.

  1. Import the template: Download the JSON file above and import it into your n8n workspace.
  2. Configure Telegram: Replace the placeholder bot token with your actual Telegram Bot API credentials in the Telegram Trigger node.
  3. Set up Google Services: Create and connect Google Drive and Google Sheets nodes using your service account credentials. Update the folder and spreadsheet IDs.
  4. Connect Gemini AI: Add your Gemini API key to the AI node and adjust the extraction prompt if needed.
  5. Test the flow: Send a sample PDF CV to your Telegram bot and verify the data appears correctly in your Google Sheet.
  6. Activate: Turn on the workflow and share your bot link with candidates or post it in job channels.
