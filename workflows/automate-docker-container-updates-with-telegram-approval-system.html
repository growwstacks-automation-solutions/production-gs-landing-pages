Automate Docker Container Updates with Telegram Approval

Free n8n workflow that checks for Docker image updates, requests human approval via Telegram, and deploys updates securely with audit trails.

What This Workflow Does

Manually checking and updating Docker containers is a repetitive, time-consuming task that often gets neglected, leaving systems vulnerable to security patches and missing feature updates. This automation solves that problem by creating a systematic, approval-based update process that ensures your containers stay current without compromising stability.

The workflow automatically monitors Docker image repositories for new versions, compares them with your running containers, and sends approval requests to your team via Telegram. Once approved, it securely executes the update commands, restarts containers with the new images, and confirms completion—all while maintaining a clear audit trail of who approved what and when.

How It Works

1. Version Detection & Comparison

The workflow starts by fetching the latest available Docker image version from your specified registry (Docker Hub, GitHub Container Registry, etc.). It then connects to your server via SSH to check the currently running version of your target container.

2. Approval Request via Telegram

If a newer version is available, the workflow sends a detailed Telegram message to your configured chat or group. This message includes the current version, new version, release notes link, and interactive buttons for approval or rejection.

3. Secure Update Execution

Upon receiving approval, the workflow establishes a secure SSH connection to your server, pulls the new Docker image, stops the old container, starts the new one with the same configuration, and verifies the service is running correctly.

4. Confirmation & Logging

After successful update, a confirmation message is sent back to Telegram with deployment details. All actions—approvals, rejections, update attempts, and outcomes—are logged for compliance and troubleshooting.

This automation is ideal for DevOps engineers, system administrators, and development teams managing Docker-based applications in production. It's particularly valuable for small to medium teams without dedicated SRE staff, startups running lean infrastructure, and any organization needing to maintain update discipline across multiple containers.

If you're tired of manually checking for updates, worrying about missing security patches, or need a clear approval process for production changes, this workflow provides the structure and automation you need.

What You'll Need

  1. n8n instance (self-hosted or cloud) with SSH and Telegram nodes installed
  2. Telegram Bot Token created via BotFather
  3. Telegram Chat ID for the person/group receiving approval requests
  4. SSH access to your Docker host with appropriate permissions
  5. Docker container name you want to monitor and update
  6. Image registry credentials if using private repositories

Pro tip: Create a dedicated "Deployment Bot" Telegram account and add it to a team channel rather than using personal chats. This creates a transparent approval history everyone can see.

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance via the workflow import function.
  2. Configure Telegram: Replace the placeholder bot token and chat ID with your actual Telegram credentials in the Telegram node settings.
  3. Set up SSH credentials: Add your server's SSH connection details (host, username, private key) in the SSH node credentials.
  4. Customize container details: Update the "Set" node with your specific Docker container name and working directory path.
  5. Test with manual trigger: Use the manual trigger node to run the workflow once and verify all connections work correctly.
  6. Schedule regular checks: Replace the manual trigger with a Schedule Trigger node to run daily or weekly checks automatically.
