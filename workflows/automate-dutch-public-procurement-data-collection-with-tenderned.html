What This Workflow Does

Manually monitoring TenderNed for new public procurement opportunities is a tedious, time-consuming task that often leads to missed deadlines and overlooked contracts. This automation workflow solves that problem by continuously scanning the official Dutch tender platform, collecting relevant data, and delivering filtered opportunities directly to your team.

The system connects to the TenderNed API to fetch the latest tender publications, retrieves detailed information in both XML and JSON formats, parses key data like organization names, contract descriptions, and reference numbers, applies your custom filtering criteria, and stores everything in a database for easy analysis. It transforms hours of daily manual searching into a fully automated process that runs 24/7.

Whether you're a construction company, IT service provider, consultant, or any business seeking government contracts in the Netherlands, this workflow ensures you never miss a relevant opportunity. It gives you the competitive advantage of being first to know about new tenders that match your capabilities.

How It Works

Step 1: Trigger & Fetch Publications

The workflow begins either on a scheduled basis (daily/hourly) or manually. It makes authenticated API calls to TenderNed to retrieve the latest tender publications, handling pagination to collect comprehensive data.

Step 2: Process & Split Data

Raw publication data is extracted and split into individual tender items. Each tender gets processed independently, allowing for parallel handling and ensuring no single failure stops the entire workflow.

Step 3: Retrieve Detailed Information

For each tender, the system makes parallel API calls to fetch both XML documentation (complete tender details) and JSON metadata (reference numbers, keywords, dates). This dual approach ensures you get both comprehensive documentation and structured metadata.

Step 4: Parse & Merge Data

The XML and JSON data are parsed, cleaned, and merged into a unified data structure. Key fields like publication dates, organization names, tender titles, descriptions, estimated values, and deadlines are extracted and standardized.

Step 5: Apply Custom Filters

Your business-specific criteria are applied: contract value thresholds, relevant CPV codes, specific government agencies, regions, or keywords. Only tenders matching your parameters proceed, eliminating noise.

Step 6: Store & Notify

Filtered opportunities are stored in your database with timestamps and source references. Notifications can be configured to alert your team via email, Slack, or other channels when new matching tenders are found.

What You'll Need

  1. TenderNed API credentials – Register for access at tenderned.nl
  2. n8n instance – Self-hosted or n8n.cloud account
  3. Database – PostgreSQL, MySQL, or similar for storing tender data
  4. Notification channel – Email, Slack, Teams, or other alert system
  5. Filter criteria – Your business requirements for relevant tenders

Pro tip: Start with broad filters and refine over time. Capture more data than you initially need—historical tender data becomes valuable for predicting future opportunities and understanding procurement patterns.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure TenderNed credentials in the HTTP Request nodes with your API username and password.
  3. Set up your database connection to store the processed tender data.
  4. Customize the filter node with your specific criteria (minimum contract value, relevant CPV codes, preferred agencies).
  5. Test with a manual trigger to verify data flows correctly from TenderNed through filtering to storage.
  6. Schedule the workflow to run automatically (daily recommended) and add notification nodes for alerts.
