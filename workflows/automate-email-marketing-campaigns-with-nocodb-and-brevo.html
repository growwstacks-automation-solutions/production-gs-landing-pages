What This Workflow Does

This n8n workflow is a complete marketing automation engine that bridges the gap between your customer data and your email marketing. It solves the common problem of disconnected systems: customer information stuck in a database (like NocoDB) and email campaigns managed manually in tools like Brevo (formerly Sendinblue).

The automation intelligently selects which marketing flow to send to each user based on rules you define, pulls their personal data, and dispatches a personalized email—all while logging every action for performance tracking. It transforms static customer lists into dynamic, triggered communication streams, saving marketing teams 10–15 hours per week on manual segmentation and sending.

How It Works

The workflow operates on a two‑phase system: preparation and execution.

1. User Selection & Data Preparation

A scheduled trigger (e.g., every 30 minutes) queries your NocoDB CDP for users who match specific criteria—like new signups, inactive customers, or post‑purchase segments. For each user, it inserts a record into a transaction table with a “processing” status, ensuring no duplicate sends.

2. Flow Logic & Personalization

The workflow reads the intended “flow_id” from a separate templates table in NocoDB. This determines which email template and logic to apply (welcome series, cart abandonment, re‑engagement). User‑specific fields (first name, last purchase, location) are merged into the Brevo template variables.

3. Email Delivery & Tracking

Each personalized email is sent via Brevo’s API. The workflow then updates the transaction record with the send result, timestamp, and status (“sent”, “failed”, “skipped”). This creates a full audit trail for every communication, enabling precise analytics and list hygiene.

Pro tip: Use the built‑in filter to block disposable email domains (like “tempmail.com”). This improves deliverability and protects your sender reputation.

What You'll Need

  1. An n8n instance (cloud or self‑hosted).
  2. A NocoDB project with at least three tables: a customer table, a templates/flow table, and a transactions log table.
  3. A Brevo account with API access and at least one email template configured.
  4. Basic familiarity with connecting APIs in n8n (credentials setup).

Quick Setup Guide

Import and configure this workflow in under 15 minutes:

  1. Download the template using the button above and import it into your n8n workspace.
  2. Set up credentials for NocoDB and Brevo in the n8n credentials panel.
  3. Update the node configurations with your specific NocoDB base ID, table names, and Brevo template ID.
  4. Adjust the schedule trigger to match your desired sending frequency (default is every 30 minutes).
  5. Test with a single user by temporarily modifying the filter, then activate the workflow.

The workflow includes a deduplication step that checks a transaction log (in NocoDB) before sending. Each user is assigned a unique ID and flow status. If a user is already marked as 'processing' or 'sent' for a specific campaign, they're skipped. This prevents spammy overlaps and ensures each contact receives the right message at the right time, maintaining list health and sender reputation.

This is crucial for automated triggers. Without deduplication, a user who triggers the same condition multiple times (like visiting a page daily) would get spammed. The log acts as a sent ledger, so you can also report on exactly who received what and when.
