What This Workflow Does

This automation solves the tedious, error-prone process of manually checking GitHub's trending repositories. Instead of spending hours each week copying and pasting data, this workflow automatically collects, structures, and stores trending repository information across daily, weekly, and monthly timeframes.

The system extracts key project details including repository names, URLs, descriptions, primary programming languages, and star counts. It then organizes this data with proper timestamps and category labels before inserting everything into a Supabase database for long-term storage and analysis.

By automating this process, you gain reliable historical data that can power dashboards, inform technical decisions, track technology trends, and provide insights that would be impossible to gather manually over time.

How It Works

Step 1: Scheduled Trigger & Timeframe Selection

The workflow begins on a schedule you define (daily, weekly, etc.). It then determines which GitHub trending pages to scrape based on your configuration—daily, weekly, and monthly trending repositories can all be collected simultaneously or separately.

Step 2: Web Scraping with FireCrawl

FireCrawl navigates to GitHub's trending pages and extracts the raw HTML content. This approach is more reliable than traditional scraping because it handles JavaScript rendering and can adapt to minor layout changes on GitHub's site.

Step 3: Data Parsing & Structuring

The raw HTML is processed to identify and extract individual repository entries. Each repository's name, URL, description, primary language, and current star count are parsed into clean, structured data fields ready for database insertion.

Step 4: Database Preparation & Insertion

The structured data is formatted for Supabase insertion, with proper data types, timestamps, and category labels. The workflow then performs batch inserts into your Supabase table, ensuring efficient data storage with proper error handling.

Step 5: Validation & Notification

After insertion, the workflow validates that data was stored correctly and can send notifications (email, Slack, etc.) about the collection status, including counts of new repositories added and any errors encountered.

What You'll Need

  1. An n8n instance (cloud or self-hosted)
  2. FireCrawl API credentials for web scraping
  3. Supabase account with a prepared database table
  4. GitHub account (for understanding the data structure)
  5. Basic understanding of database concepts and API authentication

Pro tip: Before running the workflow, create your Supabase table with appropriate indexes on timestamp and language fields. This will significantly improve query performance when building dashboards later.

Quick Setup Guide

  1. Download the template JSON file using the button above
  2. Import it into your n8n instance (Settings → Import Workflow)
  3. Create credentials in n8n for FireCrawl and Supabase
  4. Update the workflow nodes with your specific Supabase table name
  5. Configure the schedule trigger to run at your preferred frequency
  6. Test the workflow with a manual execution to verify data collection
  7. Enable the workflow and monitor the first few automated runs
