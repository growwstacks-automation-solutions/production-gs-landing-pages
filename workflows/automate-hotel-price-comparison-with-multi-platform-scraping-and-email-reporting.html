What This Workflow Does

This production-ready automation solves a critical problem for travel businesses, corporate travel managers, and frequent travelers: manually checking hotel prices across multiple booking platforms is time-consuming, error-prone, and inefficient. The workflow automatically scrapes prices from major booking sites, compares them in real-time, identifies the best deals, and delivers professional email reports to users.

Unlike basic building blocks, this is a complete end-to-end solution that handles natural language queries via webhook, processes data from multiple sources simultaneously, formats beautiful HTML reports, and provides real-time feedback. It transforms what would typically take 30-60 minutes of manual research into an automated process that completes in seconds.

How It Works

Step 1: Receive Natural Language Query

The workflow starts with a webhook that accepts natural language requests like "Hilton Hotel in Dubai from 20th December to 23rd December." It uses parsing logic to extract the hotel name, city, check-in, and check-out dates automatically.

Step 2: Parallel Platform Scraping

Once the query is parsed, the workflow simultaneously checks prices across multiple booking platforms. This parallel processing ensures results are gathered quickly rather than waiting for each platform sequentially.

Step 3: Data Aggregation & Comparison

All collected prices are aggregated into a unified format. The system identifies the best available deal, calculates savings compared to other platforms, and prepares the data for reporting.

Step 4: Email Report Generation

A professional HTML email template is populated with the comparison results, including visual elements like price comparison charts, direct booking links, and clear best-value recommendations.

Step 5: Analytics Tracking

Every search is logged to Google Sheets for analytics, allowing businesses to track search volume, popular destinations, average savings, and platform performance over time.

What You'll Need

  1. n8n instance (cloud or self-hosted) to run the workflow
  2. SMTP email credentials for sending reports (Gmail, SendGrid, etc.)
  3. Google Sheets access for analytics tracking (optional)
  4. Scraping service or API for accessing booking platform data
  5. Basic understanding of webhooks and API integrations

Quick Setup Guide

  1. Download the template JSON file using the button above
  2. Import it into your n8n instance via Workflows → Import from File
  3. Configure your SMTP credentials in the Settings → Credentials section
  4. Set up your scraping service endpoints in the HTTP Request nodes
  5. Activate the webhook and test with a sample query
  6. Toggle the workflow to Active and start receiving requests

Pro tip: Start with a single booking platform to verify your scraping setup works correctly before adding additional platforms. This makes troubleshooting much easier.
