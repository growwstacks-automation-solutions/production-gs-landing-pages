What This Workflow Does

This automation solves the tedious and error-prone process of manually classifying products with Harmonized System (HS) codes for international shipments. Every product crossing borders requires a specific tariff classification code that determines duties, taxes, and compliance requirements.

The workflow automatically looks up and validates HS codes by connecting your product data in Google Sheets with customs classification APIs. It eliminates hours of manual research while reducing classification errors that can lead to customs delays or penalties.

Example of customs documentation requiring accurate HS codes

How It Works

1. Product data extraction

The workflow pulls product descriptions, materials, and other classification-relevant data from your Google Sheets inventory or order management system.

2. API classification request

Each product's details are sent to customs classification APIs that analyze the product attributes and return the most likely HS codes along with confidence scores.

3. Data enrichment

The workflow updates your Google Sheets with the recommended HS codes, duty rates, and any required documentation for each product. Optional human review steps can be added for validation.

Frequently Asked Questions

Common questions about HS code classification and automation

HS (Harmonized System) codes are standardized numerical classifications for traded products used by customs authorities worldwide. They determine import duties, taxes, and regulatory requirements. Accurate classification is legally required and affects shipping costs, delivery times, and compliance. Misclassification can lead to penalties, shipment delays, or seizure of goods.

For example, a clothing manufacturer shipping jackets needs different HS codes for wool vs synthetic materials, with duty rates varying from 5-20%. Automation ensures consistent application of the correct codes across all shipments and markets.

  • Used by 200+ countries for customs declarations
  • 6-10 digit codes with increasing specificity
  • Updated every 5 years (current version: HS2022)

Modern classification APIs achieve 85-95% accuracy for common products when provided with complete descriptions and material details. They combine machine learning with official customs databases and rulings. The best systems provide confidence scores and alternative code suggestions for human review.

A kitchenware company using automation reduced classification errors from 15% to under 3% while cutting processing time from 45 to 3 minutes per product. The system flags borderline cases for expert review while handling straightforward classifications automatically.

  • Accuracy improves with detailed product attributes
  • Always validate codes for high-value shipments
  • Maintain an audit trail of classifications
