What This Workflow Does

This template solves the tedious, error-prone task of manually copying data from invoice PDFs into spreadsheets—a common bottleneck for small businesses, accounting teams, and operations departments. It watches a designated Google Drive folder, uses OCR technology to read text from uploaded invoices, leverages AI to intelligently extract and structure key information (like dates, amounts, and vendor details), and automatically appends a clean, organized row to a Google Sheet. The result is a centralized, searchable invoice log that eliminates manual typing, prevents missed payments, and provides an audit trail with links back to the original files.

Beyond simple data entry, this automation brings consistency and speed to your financial processes. It handles varied invoice formats, reduces human error to near zero, and ensures your financial data is always up-to-date and ready for reporting or reconciliation. By automating this foundational task, you free up valuable employee time for analysis and strategic work instead of data copying.

How It Works

The workflow is built on a logical, step-by-step process that mimics—and improves upon—the manual method.

Step 1: Scheduled Trigger & Folder Scan

A schedule trigger initiates the workflow at regular intervals (e.g., hourly or daily). It first scans your configured Google Drive parent folder to identify the correct monthly subfolder (like "2025年10月分"), ensuring it only processes invoices from the current period.

Step 2: File Retrieval & OCR Processing

The workflow downloads any new PDF or image files from the target folder. Each file is sent to the OCR.Space API, which converts the visual document into raw, machine-readable text, regardless of the original layout or formatting.

Step 3: AI-Powered Data Structuring

The raw OCR text is passed to an AI model (like OpenAI's GPT). A carefully crafted prompt instructs the AI to act as a data extraction specialist, identifying and returning structured JSON for fields such as invoice number, issue date, due date, vendor name, line items, subtotals, taxes, and total amount.

Step 4: Data Validation & Sheet Update

The extracted JSON data is mapped to specific columns in your Google Sheet. The workflow appends a new row for each successfully processed invoice, including a hyperlink back to the original file in Google Drive for easy verification and audit purposes.

What You'll Need

  1. An n8n Instance: Either a self-hosted n8n setup or a n8n.cloud account to import and run the workflow.
  2. Google Drive & Sheets Access: A Google account with Drive and Sheets. You'll need to create the destination spreadsheet and the folder structure for invoices.
  3. OCR.Space API Key: A free or paid API key from OCR.Space to power the text extraction from PDFs and images.
  4. OpenAI API Credentials: An API key from OpenAI (or a compatible LLM provider) to enable the AI data structuring step.
  5. Organized Invoice Storage: Your invoice PDFs should be saved in Google Drive, ideally within monthly subfolders (e.g., "Invoices/2025-10/") for the workflow to process them correctly.

Quick Setup Guide

Get your automation running in under 15 minutes by following these steps.

  1. Import the Template: Download the JSON file from this page and import it into your n8n instance using the "Import from File" option.
  2. Configure Google Connections: Set up credentials for Google Drive and Google Sheets within n8n. Update the workflow nodes with your specific Google Drive folder ID and Google Sheet ID.
  3. Add API Keys: In the HTTP Request node for OCR.Space, replace the placeholder with your actual OCR.Space API key. Select your configured OpenAI credentials in the AI node.
  4. Test with a Sample Invoice: Place a test invoice PDF in your designated Google Drive folder. Execute the workflow once manually to verify data extraction and sheet population work correctly.
  5. Activate the Schedule: Once testing is successful, enable the Schedule Trigger node to run the workflow automatically at your desired frequency (e.g., every 6 hours).

Pro tip: Start by processing a small batch of historical invoices. This not only tests the workflow but also populates your sheet with initial data, giving you an instant historical record.
