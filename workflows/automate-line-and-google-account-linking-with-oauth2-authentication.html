What This Workflow Does

This automation solves a common bottleneck for businesses using LINE for customer communication: manually linking new users to their Google accounts for data enrichment and personalized follow-up. When a user adds your LINE Official Account as a friend, the workflow instantly triggers, sending them a secure OAuth2 link to authorize Google account access.

After authorization, the workflow fetches the user's Google profile information (name, email, etc.) and sends a confirmation message back through LINE, completing the linking loop. This eliminates hours of manual data entry, ensures consistent data collection, and enables immediate personalized marketing or onboarding sequences.

The entire process is secure, using Google's official OAuth2 protocol, and scalable—handling thousands of users without additional staff. It transforms a tedious, error-prone manual task into a seamless, automated experience that builds richer customer profiles from the first interaction.

How It Works

Step 1: LINE Webhook Trigger

When a new user adds your LINE Official Account as a friend, LINE sends a webhook event to n8n containing the user's LINE ID and profile details. This triggers the workflow automatically, ensuring real-time response.

Step 2: Generate Google OAuth2 Link

The workflow constructs a secure Google OAuth2 authorization URL with your client ID and configured scopes. This link is personalized and includes state parameters to prevent CSRF attacks.

Step 3: Send Authentication Message via LINE

The generated OAuth2 link is sent back to the user through LINE's Messaging API as a friendly, explanatory message. The user clicks this link to authorize Google account access.

Step 4: Handle Google OAuth2 Callback

After the user authorizes access, Google redirects back to your n8n callback webhook with an authorization code. The workflow exchanges this code for an access token using your client secret.

Step 5: Fetch Google Profile Data

Using the obtained access token, the workflow calls Google's People API to fetch the user's profile information—name, email, profile picture, and other consented data.

Step 6: Send Completion Confirmation

The workflow sends a confirmation message back through LINE, thanking the user and optionally providing next steps. The fetched Google data is now available for your CRM or marketing systems.

What You'll Need

  1. A running n8n instance (cloud or self-hosted) with webhook capabilities.
  2. A LINE Developers account with a Messaging API channel and a long-lived Channel Access Token.
  3. A Google Cloud Platform project with OAuth2 consent screen configured and OAuth2 client credentials (Client ID & Secret).
  4. Basic understanding of webhook URLs and OAuth2 redirect URIs for configuration.
  5. Your LINE Official Account ID to test the flow with real users.

Pro tip: Before going live, add your own Google account as a test user in the GCP OAuth consent screen. This lets you test the full flow without publishing the app.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n canvas.
  2. Create Header Auth credentials in n8n for LINE (with your Channel Access Token) and Google (placeholder token).
  3. Update the "Create Google Auth URL" node with your Google Client ID and n8n callback webhook URL.
  4. Update the "Get Google Access Token" node with your Client ID, Client Secret, and callback URL.
  5. Configure the "Get Google User Info" node to use the dynamic token from the previous step.
  6. Set your LINE Official Account ID in the "Redirect to LINE OA" node.
  7. Activate the workflow and test by adding your LINE account as a friend.
