What This Workflow Does

This automation bridges the gap between bug reporting and IT incident management by automatically converting Marker.io issues into fully contextual ServiceNow incidents. When testers or users report bugs through Marker.io's visual feedback tools, the workflow captures all technical details and creates a structured incident in ServiceNow with attachments, environment data, and proper categorization.

The solution eliminates manual ticket creation that often loses critical context, ensuring support teams have everything they need to diagnose and resolve issues without back-and-forth communication. It handles priority setting, team assignment, and data mapping according to your existing ITIL processes.

How It Works

1. Marker.io Trigger

The workflow starts when a new issue is created in Marker.io, whether through browser extensions, embedded widgets, or mobile apps. It immediately captures the full report including annotated screenshots, console logs, and custom metadata.

2. Context Enrichment

The automation parses the Marker.io payload to extract technical details like browser version, OS, screen resolution, network conditions, and custom tags. It structures this data for ServiceNow's incident fields.

3. ServiceNow Incident Creation

Using ServiceNow's API, the workflow creates a new incident with all contextual information properly mapped to relevant fields. Attachments are uploaded, priority is set based on severity, and the ticket is assigned to the appropriate team.
