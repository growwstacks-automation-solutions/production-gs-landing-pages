What This Workflow Does

Managing multiple websites means logging into numerous Google Analytics accounts daily, copying data into spreadsheets, and trying to make sense of disparate metrics. This manual process consumes 5-10 hours weekly and often misses important trends because you're looking at numbers instead of insights.

This n8n workflow solves that problem completely. It automatically extracts performance data from all your Google Analytics properties simultaneously, processes the raw metrics through OpenAI's GPT models to generate intelligent analysis, and delivers concise, actionable reports directly to your email and Slack channels. Instead of spending mornings on data collection, you start your day with clear insights about what's working, what needs attention, and specific recommendations for improvement.

The automation centralizes your entire analytics monitoring into a single system that runs on schedule, ensuring consistent reporting regardless of team availability. It transforms complex web metrics into business intelligence that anyone on your team can understand and act upon immediately.

How It Works

Step 1: Multi-Website Data Collection

The workflow connects to each configured Google Analytics property simultaneously using the Google Analytics API. It extracts key performance metrics including sessions, conversion rates, top pages, traffic sources, and user engagement data for the specified time period (typically previous day).

Step 2: AI-Powered Analysis & Summarization

Raw metrics from each website are sent to OpenAI's GPT models. The AI analyzes patterns, identifies significant changes, explains performance drivers in plain language, and generates actionable insights. This transforms numbers into business intelligence with specific recommendations.

Step 3: Report Compilation & Formatting

The workflow compiles individual website analyses into a unified report structure. It highlights comparative performance across properties, flags anomalies, and creates executive summaries alongside detailed metrics. The report is formatted for both human readability and data visualization.

Step 4: Multi-Channel Distribution

Completed reports are distributed through configured channels: comprehensive HTML email reports to stakeholders and managers, while concise Slack notifications with key insights go to operational teams for immediate discussion and action.

What You'll Need

  1. Google Analytics API access with read-only credentials for each property you want to monitor
  2. OpenAI API key for GPT processing (any model that supports analysis tasks)
  3. Email SMTP credentials or connected email service (Gmail, Outlook, etc.) for report distribution
  4. Slack bot token with permissions to post messages to your chosen channels
  5. n8n instance (cloud or self-hosted) to run the workflow

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Configure Google Analytics credentials in n8n with OAuth2 access to each property you monitor.
  3. Update website nodes with your actual Google Analytics property IDs (replace placeholder URLs).
  4. Set up OpenAI, email, and Slack credentials in n8n's credential management.
  5. Test with a single website first to verify data flow and report formatting.
  6. Schedule the workflow to run daily at your preferred time (morning is ideal).
  7. Review delivered reports and adjust AI prompt parameters if you want different insight focus.

Pro tip: Start with 2-3 key websites to validate the setup before scaling to all properties. Configure different report detail levels for different recipients—executives get summaries, marketing teams get detailed analysis.
