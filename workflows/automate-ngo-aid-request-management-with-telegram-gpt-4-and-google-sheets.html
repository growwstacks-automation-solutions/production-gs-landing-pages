What This Workflow Does

Humanitarian organizations face immense challenges managing aid requests—messages arrive through multiple channels, in different languages, with varying urgency levels, and manual tracking leads to missed cases and reporting gaps. This automation solves these problems by creating a seamless, 24/7 intake system that works directly through Telegram, a platform already used by millions in crisis regions.

The workflow automatically receives text and voice messages from beneficiaries, transcribes and translates them using AI, categorizes requests by type and priority, logs everything to a centralized Google Sheets database, and sends immediate acknowledgments back to users. It transforms chaotic, manual request management into a structured, auditable, and scalable operation.

Beyond just logging requests, the system provides field teams with AI-generated action recommendations, tracks response times, and generates performance analytics—turning raw messages into actionable intelligence for more effective humanitarian response.

How It Works

1. Multi-Channel Request Reception

A Telegram bot receives messages 24/7 from beneficiaries via text or voice recordings. The bot welcomes users, explains the process in simple terms, and begins collecting information immediately, eliminating time zone barriers and staff availability constraints.

2. Voice-to-Text Transcription

For beneficiaries who prefer speaking or face literacy challenges, voice messages are automatically transcribed using OpenAI's Whisper model. The system handles multiple languages and dialects, ensuring no one is excluded due to communication preferences.

3. AI-Powered Categorization

GPT-4 analyzes each request to identify the type of aid needed (food, medical, shelter, logistics), urgency level, location details, and specific requirements. It extracts key information and assigns standardized categories for consistent tracking and reporting.

4. Actionable Response Generation

Based on the categorized request, the AI suggests specific next steps for field teams—whether it's preparing a food package, scheduling a medical visit, or coordinating logistics. These recommendations are generated in the local language and include practical considerations for implementation.

5. Centralized Logging & Tracking

Every request, along with its AI analysis and metadata (timestamp, user ID, location), is automatically logged to Google Sheets. This creates a searchable, filterable database that serves as both an operational tool and an audit trail for donor reporting.

6. Instant Acknowledgment & Follow-up

The system sends immediate confirmation messages back to beneficiaries, letting them know their request has been received and providing estimated response times. This builds trust and reduces anxiety during crisis situations.

What You'll Need

  1. Telegram Bot Token: Free from @BotFather on Telegram
  2. OpenAI API Key: For Whisper transcription and GPT-4 categorization
  3. Google Sheets API Credentials: Service account credentials from Google Cloud Console
  4. Google Spreadsheet: Pre-formatted with columns for request data
  5. n8n Instance: Self-hosted or cloud version
  6. Sample Request Messages: Real or simulated messages in your target languages

Quick Setup Guide

  1. Create Your Telegram Bot: Message @BotFather on Telegram, create a new bot, and copy the API token.
  2. Set Up OpenAI Access: Create an account at platform.openai.com, generate an API key, and ensure you have GPT-4 access.
  3. Configure Google Sheets API: Create a service account in Google Cloud Console, enable Sheets API, and download the JSON credentials.
  4. Prepare Your Spreadsheet: Create a Google Sheet with columns for Timestamp, User ID, Request Text, Category, Urgency, Suggested Actions, and Status.
  5. Import the Template: Download the JSON file above, import it into your n8n instance, and configure the credential nodes with your API keys.
  6. Customize AI Prompts: Adjust the system messages in the OpenAI nodes to match your specific aid categories and response protocols.
  7. Test Thoroughly: Send test messages via Telegram, verify transcription accuracy, check categorization logic, and ensure data logs correctly to Sheets.

Implementation note: Include sticky notes within the workflow with Arabic language examples and TPM-specific categorization logic. These are invaluable for field teams adapting the system to local contexts.
