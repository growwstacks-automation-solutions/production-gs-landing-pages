What This Workflow Does

Manual payment receipt handling is a time-consuming, error-prone task that frustrates customers and burdens finance teams. This automation solves that by creating a seamless bridge between your payment processor (Stripe) and your productivity suite (Google Workspace).

Whenever a customer payment is confirmed in Stripe, this workflow automatically retrieves the invoice, generates a professional receipt email with the PDF attached, saves a copy to your designated Google Drive folder for permanent archiving, and logs all transaction details into a Google Sheet ledger. It also includes intelligent error handling to ensure failed items are captured for review without stopping the entire process.

How It Works

1. Trigger & Data Fetch

The workflow can be triggered on a schedule (e.g., every hour) or manually. It authenticates with the Stripe API and retrieves the most recent invoices, including all relevant customer and payment details.

2. Validation & Routing

It checks the API response for validity and then routes each invoice based on its payment status. Only paid invoices proceed to the receipting stage, while unpaid or failed ones are logged appropriately.

3. Receipt Generation & Delivery

For each paid invoice, the system downloads the official Stripe PDF, composes a personalized email to the customer using Gmail, and sends the receipt with the attached invoice.

4. Archiving & Logging

The invoice PDF is uploaded to a specific Google Drive folder, named systematically for easy retrieval. Simultaneously, key metadata—date, invoice number, amount, customer email, and Drive file link—is appended to a master Google Sheet, creating a real-time transaction ledger.

5. Error Management

Any errors encountered during API calls, email sending, or file uploads are captured and written to a separate error-logging sheet, providing full visibility for troubleshooting.

Who This Is For

This automation is ideal for SaaS companies, subscription businesses, freelancers, agencies, and any organization that uses Stripe for payments and wants to professionalize and scale their financial operations. It's particularly valuable for finance teams drowning in manual receipt tasks, solopreneurs wearing multiple hats, and businesses aiming for audit-ready financial records without the administrative overhead.

What You'll Need

  1. A Stripe account with API access (secret key with invoice read permissions).
  2. A Google Workspace/Gmail account with OAuth2 configured for sending emails.
  3. A Google Drive folder set up for storing archived invoice PDFs.
  4. A Google Sheet with two tabs: one for the transaction ledger and one for error logging.
  5. An n8n instance (cloud or self-hosted) to run the workflow.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure credentials for Stripe, Gmail, Google Drive, and Google Sheets within n8n's credential management.
  3. Update resource IDs in the workflow nodes: specify your target Google Drive folder ID and the ID of your Google Sheet.
  4. Test with a single invoice by using the manual trigger. Verify the email is sent, the PDF is saved to Drive, and the sheet is updated.
  5. Activate the workflow and set it to run on a schedule that matches your billing frequency (e.g., hourly).

Pro tip: Before going live, run the workflow in "test" mode by limiting the Stripe node to fetch just one invoice. This allows you to verify the entire process end-to-end without affecting all your customers.
