What This Workflow Does

This automation solves the tedious, time-consuming process of manually removing backgrounds from product photos. Ecommerce businesses, photographers, and marketing teams often spend hours in Photoshop cleaning up images. The workflow automatically processes new images uploaded to Google Drive using Photoroom's AI-powered background removal API, then saves the transparent-background versions back to Drive.

By connecting Photoroom's advanced image processing with Google Drive's cloud storage, you create a seamless pipeline for product image preparation. The workflow handles the entire process without manual intervention - detecting new uploads, sending to Photoroom, processing with AI, and organizing the results. This eliminates bottlenecks in product listing workflows and ensures consistent, professional-looking images across all your sales channels.

Example showing original product photo and AI-processed version with transparent background

How It Works

1. New image detection in Google Drive

The workflow monitors a specific Google Drive folder for new image uploads. When a new product photo appears, it triggers the automation sequence. You can configure the watched folder based on your existing image management system.

2. Image processing via Photoroom API

Each detected image gets sent to Photoroom's background removal API. The AI analyzes the photo, identifies the product versus background, and generates a clean cutout with transparent background. The API handles various image formats and product types.

Diagram showing the image processing steps within Photoroom's API

3. Quality check and error handling

The workflow includes validation steps to ensure successful processing. If the API encounters issues (like low-quality source images), it can trigger notifications or route problematic files for manual review.

4. Organized storage of processed images

Finished images automatically save to a designated output folder in Google Drive, with consistent naming conventions. The workflow can create dated subfolders or organize by product categories based on your needs.

Pro tip: Set up a separate Google Drive folder for "problem" images that need manual review to maintain automation efficiency while catching edge cases.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. Photoroom API key (available from their developer portal)
  3. Google Drive account with API access enabled
  4. Product images in JPG or PNG format (minimum 500px width recommended)
  5. Designated Google Drive folders for source images and processed outputs

Quick Setup Guide

  1. Download and import the JSON workflow file into your n8n instance
  2. Configure the Google Drive node with your credentials and folder paths
  3. Add your Photoroom API key to the respective node
  4. Test with sample images to verify folder locations and processing quality
  5. Activate the workflow and upload new product images to your watched folder
