How It Works

The workflow follows a sophisticated multi-step verification process to ensure you only receive actionable, accurate alerts.

1. Social Media Scraping with Apify

The workflow triggers (manually or on a schedule) and uses Apify's Tweet Scraper to search X (Twitter) for Japanese keywords like "ポケモンカード" (Pokemon Card), "販売情報" (Sales Info), and "ポケカ" (Pokeca). It captures recent posts that might indicate new releases, restocks, or store availability.

2. Data Processing & Batching

Scraped tweets are split into manageable batches for individual processing. This ensures the system can handle large volumes of data without overwhelming downstream AI analysis and maintains performance even during peak announcement periods.

3. AI Verification & Retailer Identification

An AI Agent powered by OpenRouter analyzes each tweet's content to determine if it contains legitimate sales or stock information versus rumors, discussions, or unrelated content. It simultaneously uses HTTP requests to access the official Pokemon Card Map (map.pokemon-card.com) to identify and verify mentioned retailers.

4. Official Stock Availability Check

A second AI Agent takes the filtered information and performs direct checks against official sources. It browses the Pokemon Card Product page and Pokemon Center Online store through HTTP requests to verify actual purchase availability, filtering out announcements about future releases or out-of-stock items.

5. Notification & Reporting

Finally, the workflow compiles verified findings into a structured "Sales Flash Report" and sends it to your designated Slack channel. The report includes card details, retailer information, availability status, and direct links where applicable.

Pro tip: Replace the manual trigger with a Schedule Trigger node to run this workflow automatically every 30-60 minutes. This creates a fully autonomous monitoring bot that works around the clock while you focus on other aspects of your business.

Quick Setup Guide

  1. Import the Workflow: Copy the downloaded JSON and paste it into your n8n editor's import function.
  2. Configure Credentials: Add your Apify API token, OpenRouter key, and Slack credentials to n8n's credential store.
  3. Adjust Search Terms: Modify the searchTerms array in the Apify node to target specific card sets or retailers.
  4. Set Slack Destination: Open the "Send a message" node and select your target channel.
  5. Test & Deploy: Run the workflow manually to verify all connections, then add a Schedule Trigger for automation.

Customization idea: Add a Discord or Telegram node alongside Slack to create multi-channel alerts for team coordination. You can also integrate with Google Sheets to maintain a historical log of all detected opportunities for analysis.
