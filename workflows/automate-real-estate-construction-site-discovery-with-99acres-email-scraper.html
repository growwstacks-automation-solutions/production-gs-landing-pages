What This Workflow Does

This automation solves a critical problem for real estate investors, agents, and developers: manually searching for new construction projects is incredibly time-consuming and inefficient. You might spend hours each week browsing 99acres, MagicBricks, and other property websites, only to miss opportunities because you didn't check at the right time or overlooked listings that didn't match your simple search terms.

The workflow transforms this manual process into an automated discovery system. Users simply send an email with their location preferences (like "Gota, Ahmedabad" or "3 BHK in Bangalore"), and within minutes receive a professionally formatted list of all matching construction projects with detailed information including pricing, area, possession dates, construction status, and location specifics.

Beyond basic listing aggregation, the system handles data variations across different platforms, ensures real-time information through web scraping, and provides structured output that makes comparison and decision-making straightforward. It's like having a dedicated research assistant working 24/7 to find your next investment opportunity.

How It Works

1. Email Trigger & Location Extraction

The workflow monitors a dedicated email inbox for property search requests. When an email arrives, it automatically extracts the location preferences using natural language processing. The system can identify specific areas, cities, and even optional criteria like BHK configuration or budget ranges mentioned in the email body.

2. Multi-Platform Web Scraping

Based on the extracted location, the workflow simultaneously scrapes 99acres and other major real estate platforms. It uses proper headers and user agents to avoid blocking, implements fallback mechanisms for website structure changes, and captures all available project details including those that might be missed in manual searches.

3. Data Parsing & Standardization

Raw HTML data from different websites gets transformed into structured project entries. The system handles variations in how different platforms present information—converting area measurements to consistent units, standardizing date formats, and categorizing construction status consistently across all sources.

4. Results Formatting & Delivery

All parsed projects are compiled into an email-ready format with clear organization, visual indicators, and actionable information. The output includes summary statistics, individual project details with standardized fields, and next-step recommendations. This formatted report gets delivered directly to the original requester.

What You'll Need

  1. An n8n instance (cloud or self-hosted) with web scraping capabilities enabled
  2. Email account with IMAP/SMTP access for automated communication (Gmail, Outlook, or custom domain)
  3. Reliable internet connection for real-time property data scraping
  4. Basic understanding of n8n workflows for initial configuration and testing
  5. Target website access (99acres, MagicBricks, or other regional property platforms you want to monitor)

Pro tip: Start with monitoring 2-3 locations you're most familiar with. This helps you verify the accuracy of scraped data and refine your filtering criteria before scaling to additional markets.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the workflow into your n8n instance using the downloaded JSON file
  2. Configure email credentials in the Email Trigger and Send Email nodes with your IMAP/SMTP settings
  3. Test web scraping with a sample location to ensure proper headers and user agents are configured
  4. Customize filtering criteria if you want to add specific requirements like minimum area or maximum price
  5. Send a test email to your configured address with a sample location request
  6. Verify the output matches your expectations and adjust parsing logic if needed
  7. Deploy the workflow and start receiving automated property listings
