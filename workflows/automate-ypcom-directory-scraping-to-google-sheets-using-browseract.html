What This Workflow Does

Manual lead generation from online directories is a time-consuming, repetitive task that slows down sales teams. Searching YP.com for local businesses, copying contact details, and pasting them into spreadsheets can take hours each week. This process is prone to errors, inconsistent formatting, and doesn't scale.

This n8n automation workflow solves that problem by automatically scraping business listings from YP.com based on your chosen category and location. It extracts company names, addresses, phone numbers, and other details, then saves them directly to Google Sheets in a clean, structured format. The workflow runs on demand, giving you fresh lead lists whenever you need them without manual effort.

By automating this process, you transform a days-long manual research task into a minutes-long automated operation. Sales teams can focus on outreach and closing deals rather than data entry, while marketing teams gain reliable prospect lists for campaigns.

How It Works

The automation follows a logical sequence to ensure reliable data collection and storage.

1. Manual Trigger & Configuration

You start the workflow manually and specify your search parameters: business category (like "restaurants" or "plumbers") and city location. These inputs tell the scraper exactly what type of businesses to look for and where.

2. BrowserAct Web Scraping

The workflow sends these parameters to BrowserAct, a specialized web scraping service. BrowserAct navigates YP.com, searches for businesses matching your criteria, and extracts the relevant contact information from each listing.

3. Data Processing & Parsing

Once BrowserAct completes the scraping job, the raw data (a JSON string) is passed to a Code node. This node parses the information, splitting it into individual items for each business and structuring it into consistent fields like company name, address, and phone number.

4. Google Sheets Integration

The processed data flows to Google Sheets, where each business becomes a new row in your spreadsheet. The workflow checks for existing entries by company name to prevent duplicates, ensuring your lead list stays clean and organized.

Pro tip: Schedule this workflow to run weekly to keep your lead database fresh. New businesses open regularly, and contact details change—automated updates ensure you never miss opportunities.

What You'll Need

  1. BrowserAct API account for web scraping (sign up for their service)
  2. BrowserAct "Online Directory Lead Scraper (YP.com)" Template configured in your account
  3. BrowserAct n8n Community Node installed in your n8n instance
  4. Google Sheets credentials connected to n8n for saving the leads
  5. Self-hosted n8n instance (this workflow uses community nodes not available on cloud)

Quick Setup Guide

Follow these steps to get the automation running in your environment:

  1. Download and import the template JSON file into your n8n instance using the import function.
  2. Configure BrowserAct credentials in n8n by adding your API key and workflow ID from your BrowserAct account.
  3. Connect Google Sheets by setting up OAuth authentication for n8n to access your spreadsheet.
  4. Test the workflow manually with a simple search (like "coffee shops" in "Austin") to verify data flows correctly.
  5. Customize the output by modifying the Google Sheets node to match your preferred column structure.

Important: This workflow is designed for self-hosted n8n instances only due to its use of community nodes. It won't work on n8n.cloud.
