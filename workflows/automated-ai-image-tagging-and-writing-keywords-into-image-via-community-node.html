What This Workflow Does

Manually tagging images with descriptive keywords is a tedious, time-consuming task that plagues marketing teams, e-commerce managers, and content creators. This automated workflow solves that problem by using AI to analyze image content and write relevant keywords directly into the image file's metadata.

Whenever a new image is added to a specific Google Drive folder, the workflow automatically downloads it, sends it to an AI vision service (like OpenAI, Anthropic, Google Vision, or local Ollama), generates descriptive tags based on the visual content, and embeds those keywords back into the image file using EXIF/IPTC metadata standards. The tagged image then replaces the original in Google Drive, creating a fully organized, searchable media library without manual intervention.

This automation transforms chaotic image collections into structured assets that are instantly searchable by content, improving workflow efficiency for teams managing product catalogs, marketing materials, user-generated content, or archival photography.

How It Works

This intelligent workflow follows a logical six-step process to transform raw images into tagged, organized assets.

1. Google Drive Trigger

The workflow monitors a designated Google Drive folder for new image uploads. When a new JPG, PNG, or other image file is detected, the automation immediately starts processing it. This trigger-based approach ensures real-time tagging as soon as assets are added to your storage.

2. File Download & Preparation

The identified image is downloaded from Google Drive to the n8n server for processing. The workflow extracts file information and prepares it for AI analysis, handling various formats and sizes automatically.

3. AI Content Analysis

The image is sent to your chosen AI vision service (configurable for OpenAI, Anthropic, Google Vision, or local Ollama). The AI analyzes the visual content and generates descriptive keywords—identifying objects, scenes, colors, themes, and contextual elements present in the image.

4. Metadata Merging

The workflow merges the original image file with the AI-generated keywords, preparing them for insertion into the standard metadata fields that image files use for organization and search.

5. Keyword Writing to Image

Using the n8n-nodes-exif-data community node (required for self-hosted instances), the generated keywords are written directly into the image's EXIF/IPTC metadata under the "dc:subject" or "keywords" field. This embeds the tags permanently within the file itself.

6. File Update in Google Drive

The newly tagged image file is uploaded back to Google Drive, replacing the original or creating a new version. The updated file now contains searchable metadata that works across platforms and systems.

What You'll Need

  1. Self-hosted n8n instance – This workflow requires the n8n-nodes-exif-data community node which only works with self-hosted installations
  2. Google Drive access – With permissions to read from and write to your target folder
  3. AI API credentials – From OpenAI, Anthropic, Google Vision, or a local Ollama instance
  4. n8n-nodes-exif-data community node – Must be installed on your n8n instance for metadata writing functionality
  5. Dedicated Google Drive folder – Where new images will be monitored and processed

Important Note: This workflow specifically requires a self-hosted n8n instance because it uses the community node for writing EXIF data. Cloud n8n instances cannot install community nodes.

Quick Setup Guide

Follow these steps to implement this automated image tagging system in your workflow:

  1. Install the community node – Run npm install n8n-nodes-exif-data in your self-hosted n8n environment and restart the service
  2. Import the template – Download the JSON file from this page and import it into your n8n instance
  3. Configure Google Drive – Set up the Google Drive trigger node with your folder path and authentication
  4. Set up AI service – Connect your preferred AI vision provider (OpenAI, Anthropic, Google, or Ollama) with API credentials
  5. Adjust keyword settings – Modify the AI prompt to generate tags relevant to your specific use case and image types
  6. Test with sample images – Upload test images to your monitored folder and verify the tagging results
  7. Activate the workflow – Turn on the automation and begin processing your image library automatically
