What This Workflow Does

This automated workflow provides immediate response capabilities when AWS IAM access keys are compromised. It continuously monitors AWS CloudTrail logs for suspicious activity patterns that indicate potential key compromise, such as unusual API calls, geographic anomalies, or abnormal usage spikes.

When a potential compromise is detected, the system automatically initiates a multi-step response: revoking the compromised credentials, notifying security teams via Slack with detailed incident information, and generating AI-powered analysis through Claude to assess the threat level and recommend next steps. This reduces response time from hours to minutes while providing human-readable context for security teams.

How It Works

1. Continuous AWS Activity Monitoring

The workflow polls AWS CloudTrail logs at regular intervals, analyzing API call patterns against your configured baseline of normal activity. Machine learning algorithms help identify deviations that may indicate compromised credentials.

2. Compromise Detection Logic

Custom detection rules evaluate multiple risk factors including geographic location changes, unusual service access patterns, and request volume anomalies. The system scores each event to determine when automated response should trigger.

3. Automated Credential Rotation

When a high-confidence compromise is detected, the workflow immediately deactivates the suspicious IAM keys and generates replacement credentials if configured to do so. All actions are logged for audit purposes.

4. Slack Alerting with Context

Security teams receive rich Slack notifications containing the compromised key details, affected resources, timeline of suspicious activity, and interactive buttons to approve additional response actions.

5. Claude AI Threat Analysis

The workflow sends event data to Claude AI which returns natural language analysis explaining the likely attack vector, potential impact, and recommended containment steps - all formatted for easy sharing with stakeholders.
