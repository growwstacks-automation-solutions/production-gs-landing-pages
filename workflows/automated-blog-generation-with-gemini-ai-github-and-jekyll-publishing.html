What This Workflow Does

This n8n workflow solves the time-consuming challenge of creating and publishing regular technical blog content. Instead of spending hours researching, writing, and formatting posts, this automation handles everything from topic selection to live publication.

The system reads blog topics from a Google Sheet, conducts research using Tavily and Wikipedia, generates well-structured Markdown articles with Gemini AI, commits them to a GitHub repository, and automatically updates your Jekyll-powered blog. It's perfect for developers, technical content teams, or anyone maintaining a documentation site who wants consistent content without the manual effort.

How It Works

1. Topic Selection & Research

The workflow starts by checking your Google Sheet for topics marked as "pending." It extracts the next topic, then uses Tavily Search to gather current information and Wikipedia for foundational knowledge. This creates a comprehensive research base for the AI.

2. AI Content Generation

Gemini AI (Google's language model) receives the research materials and generates a complete blog post with proper YAML front matter, headings, code examples where relevant, and SEO-friendly structure. The AI follows your brand guidelines and technical depth requirements.

3. GitHub Integration & Publishing

The generated Markdown file is automatically committed to your GitHub repository in the correct _posts/ directory. The commit triggers your Jekyll build process (via GitHub Actions or similar), making the post live on your website without any manual deployment steps.

4. Status Tracking & Completion

Once successfully published, the workflow updates the Google Sheet to mark the topic as "completed" and logs the publication date. This creates a clear audit trail and prevents duplicate content generation.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. Tavily API key for web research
  3. Google Sheets with topic list (Title, Status columns)
  4. Gemini AI API key (Google AI Studio)
  5. GitHub repository with Jekyll setup
  6. GitHub OAuth2 credentials for n8n
  7. Google OAuth2 credentials for Sheets access

Pro tip: Start with a private GitHub repository for testing. Once the workflow runs smoothly, switch to your production repo. This prevents accidental publication of test content.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials for Tavily, Google Sheets, Gemini AI, and GitHub in n8n's credential management.
  3. Prepare your Google Sheet with columns: Title, Status, Row_Number. Add 5-10 blog topics with Status blank.
  4. Update node configurations with your GitHub repo path, Jekyll post directory, and desired AI tone/style.
  5. Test with one topic by manually triggering the workflow and verifying the post appears in your GitHub repo.
  6. Schedule automation to run daily or weekly based on your content calendar.
