Automated Digital Certificate Creator & Validator

A complete n8n workflow to create, send, and verify digital certificates with PDF generation—fully automated for courses, events, and training.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manually creating and sending digital certificates is a time-consuming, error-prone task that scales poorly. For every course completion, training session, or event, administrators must design certificates, enter participant data, generate PDFs, email them individually, and maintain a verification system. This process can take 5–10 hours per batch and is vulnerable to human error.

This n8n workflow automates the entire digital certificate lifecycle. It receives participant data via webhook, generates a unique certificate ID, creates a professional PDF, emails it to the recipient, stores all records in a searchable database, and provides a public verification endpoint. The system runs 24/7, ensuring instant, accurate, and verifiable certificate delivery for any scale of operation.

How It Works

1. Webhook Trigger & Data Collection

The workflow starts when your website, LMS, or form sends a POST webhook with participant details: name, surname, course name, and email address. This trigger can come from any system that can make an HTTP request.

2. Unique ID Generation & Data Storage

A Code node creates a random, unique alphanumeric Certification ID. The system checks for duplicates and generates a new one if needed. All data—name, surname, email, course, and the unique ID—is saved to an n8n Data Table, creating a permanent, searchable registry.

3. Professional PDF Generation

Using the PDF Generator API, the workflow builds a branded certificate from a customizable HTML template. It dynamically inserts the participant's name, course title, date, and unique ID, producing a high-quality PDF ready for delivery.

4. Automated Email Delivery

The PDF certificate is attached to a personalized email sent via Gmail (OAuth2). The subject, body, and sender are fully configurable. The system logs the delivery and can handle retries if needed.

5. Public Verification Endpoint

A separate webhook endpoint (/certificationscheck) allows anyone to verify a certificate's authenticity by submitting the unique ID. The workflow queries the Data Table and returns the participant's name and surname if valid, or a "not found" response.

What You'll Need

  1. An n8n instance (cloud or self-hosted).
  2. An n8n Data Table with fields: Name (string), Surname (string), CertificationID (string).
  3. PDF Generator API account (credentials configured in n8n).
  4. Gmail OAuth2 credentials (or another email service like SendGrid).
  5. A way to trigger the workflow (website form, LMS webhook, Zapier/Make integration).

Pro tip: Use a branded HTML template for your PDF certificates. Include your logo, brand colors, and official signatures to enhance perceived value and professionalism.

Quick Setup Guide

  1. Import the workflow: In your n8n dashboard, go to Workflows → Import and paste the downloaded JSON.
  2. Configure the Data Table: Update the three Data Table nodes (Insert_Certificaton, Find_Certification_By_ID, Find_Certification_By_ID1) to point to your own Data Table with the required fields.
  3. Set up credentials: Add your PDF Generator API credentials to the Generate_PDF node and your Gmail OAuth2 credentials to the Email_Certification node.
  4. Customize the email and PDF template: Adjust the HTML in the PDF node and the email subject/body to match your branding.
  5. Activate and test: Turn on the workflow and send a test webhook to verify the entire process from creation to verification.
