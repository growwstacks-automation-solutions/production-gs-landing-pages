Automated n8n Execution Cleanup System

Keep your n8n instance fast and clean by automatically deleting old workflow executions with custom retention rules.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Every time a workflow runs in n8n, it creates an execution log. Over time, these logs accumulate—slowing down your database, consuming storage, and making the n8n interface sluggish. This automation solves that problem by systematically cleaning old executions while preserving recent ones you actually need for debugging.

The workflow connects to your n8n instance via its own API, retrieves execution history, applies your custom retention rules (like "keep the last 10 executions per workflow"), and safely deletes everything older. It works for both n8n Cloud and self-hosted setups without requiring direct database access.

Think of it as automated janitorial service for your automation platform—removing digital clutter so your system stays responsive and efficient.

How It Works

1. Schedule Trigger

The workflow starts on a schedule you set—daily, weekly, or monthly. This ensures cleanup happens automatically without manual intervention.

2. Fetch Executions via n8n API

Using n8n's "Get Many Executions" node with your personal API key, the workflow retrieves up to 250 recent executions across all workflows. The API handles authentication and pagination.

3. Apply Retention Rules

A "Set" node lets you configure how many recent executions to keep per workflow (e.g., 10). You can set this to zero to delete all historical executions if needed.

4. Filter & Group Executions

A Code node processes the data, grouping executions by workflow ID and identifying which ones exceed your retention limit based on their timestamps.

5. Delete Old Executions

The "Delete Executions" node removes all identified old executions in batches, regardless of their status (success, error, or waiting).

Pro tip: Start with a conservative retention setting like 20 executions per workflow. Monitor for a week, then adjust based on your debugging needs. High-error workflows might need more history.

Quick Setup Guide

Follow these steps to implement this cleanup system in under 10 minutes:

  1. Download the template using the button above and save the JSON file.
  2. Create API credentials in your n8n instance: Settings → API Keys → Generate new key.
  3. Import the workflow into your n8n instance via the import function.
  4. Configure the n8n API nodes with your base URL (e.g., https://your-instance.n8n.cloud/api/v1) and the API key.
  5. Set your retention number in the "Set Executions to Keep" node (default is 10).
  6. Test with manual trigger first, then switch to scheduled execution.

Security note: This workflow deletes data permanently. Always test first on a staging environment or with a small subset of workflows. Consider backing up critical execution logs before widespread cleanup.
