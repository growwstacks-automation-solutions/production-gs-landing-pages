Automated GLPI Ticket Deadline Alerts via Microsoft Teams

Prevent SLA breaches with proactive notifications. This free n8n workflow monitors GLPI tickets and alerts technicians in Microsoft Teams before deadlines expire.

What This Workflow Does

Missed ticket deadlines damage customer trust and strain IT teams. Manually monitoring hundreds of GLPI tickets for approaching SLAs is time-consuming and error-prone. This automation solves that by creating a proactive notification system that watches your GLPI instance and sends targeted Microsoft Teams alerts to technicians when their assigned tickets are nearing expiration.

The workflow runs on a daily schedule, queries GLPI for tickets expiring within the next 48 hours, filters them by assigned technician, and delivers personalized messages with ticket details directly to the right Teams channels. It handles authentication, session management, and error handling automatically, ensuring reliable operation without manual intervention.

How It Works

1. Scheduled Trigger & GLPI Session Initiation

The workflow activates daily at 9:00 AM (configurable). It first establishes a secure session with your GLPI REST API using application token authentication, ensuring proper access to ticket data.

2. Ticket Query & Deadline Filtering

It queries GLPI for all active tickets, then applies intelligent filtering to identify only those with due dates falling within the next two days. This focused approach prevents alert overload while catching critical deadlines.

3. Technician Assignment & Data Processing

Tickets are grouped by assigned technician ID. The workflow extracts key information including ticket title, ID, priority, and calculates remaining time, formatting it for clear communication.

4. Conditional Routing & Teams Notification

Using conditional logic, the workflow routes each ticket batch to the appropriate Microsoft Teams channel based on technician assignment. It sends structured messages with actionable details and direct ticket links.

5. Session Cleanup & Error Handling

After processing, the workflow properly closes the GLPI session and includes error handling to manage API failures, missing assignments, or Teams connectivity issues gracefully.

It's particularly valuable for teams managing strict SLAs, handling high ticket volumes, or supporting multiple technicians who need timely notifications about their workload.

What You'll Need

  1. A running GLPI instance (version 9.x or 10.x) with REST API enabled
  2. GLPI application token from a user with technician/admin privileges
  3. Microsoft Teams account with permissions to create incoming webhooks
  4. n8n instance (self-hosted or cloud) with network access to both systems
  5. Technician user IDs from GLPI for proper assignment mapping
  6. Microsoft Teams channel webhook URLs for each technician/team

Pro tip: Before deploying, test with a small subset of tickets and a dedicated test Teams channel. Verify the alert timing matches your SLA definitions (e.g., 24-hour vs 48-hour warnings).

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance via the workflow import function.
  2. Configure GLPI credentials: Update the HTTP Request nodes with your GLPI URL and application token in the credentials manager.
  3. Set up Teams webhooks: Create incoming webhooks in Microsoft Teams for each channel, then paste the URLs into the appropriate webhook nodes.
  4. Map technician IDs: Replace the placeholder technician IDs in the IF conditions with actual GLPI user IDs from your system.
  5. Adjust scheduling: Modify the Schedule Trigger node to match your preferred monitoring frequency and timezone.
  6. Test and activate: Run the workflow manually with test data, verify Teams notifications arrive correctly, then activate the schedule.
