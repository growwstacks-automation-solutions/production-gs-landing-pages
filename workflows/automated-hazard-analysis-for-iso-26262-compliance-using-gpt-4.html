What This Workflow Does

This n8n workflow automates the Hazard Analysis and Risk Assessment (HARA) process required by ISO 26262, the international functional safety standard for road vehicles. It transforms weeks of manual documentation into a streamlined, AI-assisted process that identifies potential hazards, assesses risk levels (ASIL ratings), and generates preliminary safety reports.

For automotive engineers and safety managers, compliance isn't optional—it's mandatory for bringing vehicles to market. Traditional HARA requires specialized teams to manually analyze hundreds of potential failure modes across complex electrical/electronic systems. This workflow uses AI to process system descriptions, suggest hazard scenarios, calculate Automotive Safety Integrity Levels (ASIL), and create audit-ready documentation templates, cutting analysis time by 80% while maintaining rigorous compliance standards.

How It Works

The workflow combines AI reasoning with structured safety engineering methodologies to create a repeatable, traceable analysis process.

1. System Description Input

You provide a detailed description of the automotive system or component (e.g., "electro-hydraulic braking system with redundancy"). The workflow accepts text files, structured data from requirements management tools, or manual input through forms.

2. AI-Powered Hazard Identification

Using GPT-4 or similar LLMs, the workflow analyzes the system description to identify potential hazardous events, failure modes, and operational scenarios. It considers vehicle states, environmental conditions, and human interactions that could lead to safety violations.

3. Automated Risk Assessment

The AI suggests severity (S), exposure (E), and controllability (C) ratings based on ISO 26262 guidelines, then calculates preliminary ASIL levels (QM, A, B, C, D). These suggestions are presented for human validation with reasoning and references to the standard.

4. Safety Goal Generation

For each identified hazard with ASIL rating, the workflow drafts corresponding safety goals—clear, verifiable requirements that specify what the system must do (or not do) to prevent the hazardous event.

5. Report Compilation

All analysis is compiled into structured documents (CSV, PDF, or Word formats) including hazard tables, ASIL classifications, safety goals, and traceability matrices—ready for review by your safety team and certification bodies.
