What This Workflow Does

Manual event research is time-consuming, inconsistent, and you inevitably miss opportunities. This automation solves that by continuously monitoring platforms like Eventbrite, Meetup, and Facebook Events for activities matching your business criteria.

The workflow scrapes event data using Bright Data's reliable infrastructure, filters by location, date range, and keywords, then uses OpenAI to intelligently categorize each event (conference, workshop, networking), extract key details (venue, organizers, ticket price), and assess relevance. Enriched events are posted to Slack for immediate team awareness and archived in Airtable for long-term reference and analysis.

This creates a systematic competitive intelligence pipeline that operates 24/7, transforming scattered public information into structured, actionable business insights.

How It Works

1. Scheduled Trigger & Data Collection

A schedule node runs daily (or more frequently) to initiate the monitoring cycle. Bright Data nodes then scrape multiple event platforms simultaneously, handling anti-bot measures and extracting raw event listings including titles, descriptions, dates, locations, and URLs.

2. Intelligent Filtering & Enrichment

Raw events pass through filtering nodes that exclude irrelevant content based on your configured criteria (city, radius, keywords). The Set node allows easy customization of these parameters without technical expertise.

3. AI-Powered Analysis & Categorization

Each qualifying event is sent to OpenAI's models for natural language processing. The AI classifies event type, extracts entities (organizer names, venues, prices), assesses potential value, and generates a concise summary—replicating hours of human analysis in seconds.

4. Notification & Data Storage

Analyzed events are formatted into digestible Slack messages with priority indicators and direct links. Simultaneously, complete event records with all extracted metadata are saved to Airtable, creating a searchable historical database for trend analysis and reporting.

What You'll Need

  1. n8n instance (self-hosted version required for community nodes)
  2. Bright Data account for reliable web scraping
  3. OpenAI API key for AI analysis and categorization
  4. Slack workspace with webhook access for notifications
  5. Airtable base or Google Sheets for data storage
  6. Basic understanding of your target locations and event keywords

Pro tip: Start with narrow filters (one city, 2-3 specific keywords) to validate the workflow before expanding. This prevents information overload and helps refine your criteria based on initial results.

Quick Setup Guide

  1. Import the workflow into your n8n instance using the downloaded JSON file.
  2. Configure Bright Data nodes with your account credentials and adjust target URLs if needed.
  3. Insert your OpenAI API key in the AI analysis nodes and optionally adjust the categorization prompts.
  4. Connect Slack by creating an incoming webhook and pasting the URL into the workflow.
  5. Set up Airtable connection with your base ID and table name for event storage.
  6. Customize filters in the initial Set node—update city, radius (miles/km), date range, and industry keywords.
  7. Test with a manual trigger before enabling the schedule to verify everything works correctly.
