Automated Order Confirmations with First-Buyer Rewards

Trigger instant email confirmations, Slack notifications, and personalized coupon rewards for first-time customers when a payment succeeds.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the chaotic manual process that follows a successful customer payment. Without automation, teams must manually send confirmation emails, notify internal channels, and track customer history to offer rewards—a time-consuming and error-prone routine.

The workflow listens for a successful payment webhook from Abacate Pay (a popular Brazilian payment gateway similar to Stripe). Upon trigger, it instantly sends a professional HTML confirmation email to the customer, posts a detailed alert to your Slack channel, and checks if the buyer is a first-time customer. If they are, it automatically generates a 10% discount coupon for their next purchase and includes it in the communication.

This creates a seamless, immediate post-purchase experience that builds customer loyalty, reduces support inquiries, and turns one-time buyers into repeat customers.

How It Works

Step 1: Payment Webhook Trigger

The workflow starts when Abacate Pay sends a "payment successful" webhook event. This secure trigger ensures real-time reaction without polling or manual checks.

Step 2: Customer History Check

The system queries the payment gateway's API to retrieve the customer's past orders. This determines whether the purchase is their first, enabling personalized reward logic.

Step 3: Conditional Coupon Generation

If the customer is a first-time buyer, the workflow calls the payment gateway's coupon API to create a unique, single-use 10% discount code. This reward is stored and passed forward.

Step 4: Dynamic Communication

The workflow assembles email and Slack messages with dynamic content. First-time buyers receive the coupon details; returning customers get standard confirmations. All communications include order specifics and company branding.

Step 5: Multi-Channel Delivery

A responsive HTML email is sent to the customer via your email service (SMTP, SendGrid, Gmail). Simultaneously, a formatted Slack notification using Block Kit posts to your designated team channel for internal awareness.

What You'll Need

  1. Abacate Pay account (or similar payment gateway) with API access for webhooks and coupon creation.
  2. Email service credentials (SMTP, SendGrid, Gmail, etc.) to send customer confirmations.
  3. Slack workspace with app permissions to post notifications to a chosen channel.
  4. n8n instance (cloud or self-hosted) to import and run the workflow.
  5. Basic configuration data like company name, website, and support email for personalization.

Quick Setup Guide

  1. Import the template into your n8n workspace using the downloaded JSON file.
  2. Configure the webhook by copying the URL from the Webhook node and adding it to your Abacate Pay dashboard for the "payment successful" event.
  3. Update the Configs node with your company details (name, site, email) and the webhook validation token.
  4. Set API credentials in the HTTP Request nodes for Abacate Pay order history and coupon creation.
  5. Connect email and Slack by selecting your credentials in the Send Email and Slack nodes, choosing your destination channel.
  6. Activate the workflow and test with a simulated payment to ensure emails and Slack messages deliver correctly.

Pro tip: Customize the email HTML template to match your brand's design. Use inline CSS for responsiveness and include clear call-to-action buttons for post-purchase steps.
