What This Workflow Does

Manual SEO performance tracking is a repetitive, time-consuming task that often gets deprioritized. Marketing teams and analysts waste hours each week logging into Google Search Console, exporting CSV files, and manually updating spreadsheets or databases. This process is not only inefficient but also prone to human error, leading to inconsistent data and missed insights.

This n8n workflow solves that problem by fully automating the collection of critical SEO metrics. It connects directly to the Google Search Console API, extracts performance data for your specified domain, and structures it into a NocoDB table. You get automated, scheduled updates of clicks, impressions, CTR, average position, and device breakdown for every query and page—without ever opening a spreadsheet.

The result is a centralized, queryable database of your SEO performance over time. This enables advanced analysis, trend identification, and seamless integration with business intelligence tools like Metabase or Superset for live dashboards that stakeholders can actually use.

How It Works

The automation follows a logical, step-by-step process to transform raw search data into actionable business intelligence.

1. Schedule & Trigger

The workflow begins with a Schedule Trigger node, which you can set to run weekly, daily, or monthly. This ensures your SEO data is collected consistently without manual intervention, creating a reliable historical record.

2. Fetch from Google Search Console

Using the Google Search Console node, the workflow authenticates via OAuth2 and requests performance data for your configured property. You can specify date ranges, dimensions (query, page, device, country), and row limits to capture exactly the metrics you need.

3. Transform & Structure Data

Raw API responses are processed through n8n's Function and Set nodes. This step cleans the data, calculates derived metrics (like CTR from clicks/impressions), and formats it to match your NocoDB table schema—ensuring clean, consistent records every time.

4. Write to NocoDB

The structured data is sent to NocoDB via its API node. Each search performance record (query, page, device combination) is created or updated as a new row in your designated table, building a searchable historical dataset.

5. Error Handling & Logging

The workflow includes error-catching logic and optional notification nodes (like Slack or Email). If the API call fails or credentials expire, you're alerted immediately—preventing silent data gaps in your reporting.

What You'll Need

  1. Google Search Console Access: Owner or full user permissions to the property (website) you want to track. API access must be enabled.
  2. NocoDB Instance: A self-hosted NocoDB installation or a NocoDB Cloud account. You'll need API credentials (token) and a pre-created table with the correct schema.
  3. n8n Instance: A running n8n environment (cloud or self-hosted) where you can import and activate the workflow.
  4. Basic API Knowledge: Ability to create OAuth2 credentials in Google Cloud Console and generate an API token in NocoDB.
  5. Defined NocoDB Schema: A table with columns for: query (Text), page (URL), device (Text), clicks (Number), impressions (Number), ctr (Number), position (Number), and date (DateTime).

Quick Setup Guide

Get your automated SEO pipeline running in under 30 minutes by following these steps.

  1. Import the Template: Download the JSON file using the button above and import it into your n8n instance.
  2. Configure Google OAuth2: In the Google Search Console node, set up OAuth2 credentials. Use a service account or standard OAuth with the https://www.googleapis.com/auth/webmasters.readonly scope.
  3. Set Up NocoDB Credentials: Create an API token in your NocoDB project settings. Add it to the NocoDB node in the workflow, along with your base URL and table ID.
  4. Adjust the Query Parameters: In the Google Search Console node, specify your target domain (e.g., `sc-domain:example.com` or exact URL), date range (e.g., last 7 days), and dimensions (query, page, device).
  5. Test & Schedule: Execute the workflow once manually to verify data flows into NocoDB correctly. Then, set the Schedule Trigger to your desired frequency (e.g., every Monday at 9 AM).
  6. Connect to Dashboards: Use NocoDB's REST API or built-in views to connect your new SEO table to BI tools like Metabase, Google Looker Studio, or Tableau for visualization.
