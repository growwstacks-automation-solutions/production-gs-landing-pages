Automated Server Log Cleanup via Email Alerts with SSH

Free n8n workflow that automatically cleans Nginx, Docker, and system logs when disk usage alerts arrive—preventing downtime and saving hours of manual work.

What This Workflow Does

Server downtime due to filled disks is a common but preventable problem. When log files accumulate, they can consume all available disk space, causing applications to crash, databases to stop, and websites to go offline. Traditionally, system administrators must manually monitor alerts, log into each server, and run cleanup commands—a reactive process that wastes time and often happens during emergencies.

This n8n workflow automates the entire process. It monitors a dedicated email inbox for disk utilization alerts (typically triggered at 80%+ usage), extracts the affected server's IP address from the alert email, establishes a secure SSH connection, and executes targeted cleanup commands to purge old logs from Nginx, PM2, Docker containers, and system log directories. The result: disk space is automatically reclaimed before it becomes critical, preventing outages without manual intervention.

How It Works

1. Email Monitoring & Parsing

The workflow starts with an IMAP email trigger node that watches a designated inbox (like alerts@yourcompany.com). When a disk usage alert arrives from your monitoring system, the workflow parses the email subject and body using regex patterns to extract the server IP address and the specific disk threshold that was exceeded.

2. Server Identification & Credential Selection

Using the extracted IP, the workflow matches it against a stored list of server credentials (securely stored in n8n credentials). It selects the appropriate SSH key or password for that specific server, ensuring the automation only attempts to connect to authorized systems.

3. Secure SSH Connection & Command Execution

The workflow establishes an SSH connection to the target server and executes a series of cleanup commands. These typically include removing old compressed log files, truncating active logs, cleaning Docker container logs, and rotating application-specific logs. Each command includes safety checks to prevent deletion of critical files.

4. Verification & Notification

After cleanup, the workflow runs df -h to verify freed disk space, compares the before/after usage, and sends a summary notification via Slack, email, or your preferred channel. If cleanup fails or disk usage remains critical, it escalates to human administrators.

Pro tip: Configure your monitoring system to send alerts in a consistent format (like "CRITICAL: Disk usage on 192.168.1.10 is 85%") to make parsing more reliable. Include the server IP in the subject line for easiest extraction.

What You'll Need

  1. n8n instance (self-hosted or cloud) with network access to your servers
  2. Monitoring system configured to send email alerts on disk usage (Nagios, Zabbix, CloudWatch, etc.)
  3. Dedicated email inbox for receiving alerts (Gmail, Office 365, or IMAP-compatible)
  4. SSH access to target servers with key-based authentication (recommended) or password
  5. Server permissions to delete/rotate log files in /var/log/ and other log directories
  6. Notification channel (Slack, Teams, email) for receiving cleanup reports

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure the IMAP email trigger with your alerts inbox credentials
  3. Set up SSH credentials in n8n for each server you want to automate
  4. Customize the regex patterns to match your alert email format
  5. Adjust cleanup commands for your specific log directories and retention policies
  6. Test with a simulated alert by sending a test email to the alerts inbox
  7. Monitor the first few runs and adjust thresholds as needed

Security note: Always use SSH keys instead of passwords, restrict the automation user's sudo permissions to only necessary commands, and consider implementing a jump host or VPN for accessing production servers.
