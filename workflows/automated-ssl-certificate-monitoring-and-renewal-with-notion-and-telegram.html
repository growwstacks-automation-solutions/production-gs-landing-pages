Automated SSL Certificate Monitoring and Renewal

What This Workflow Does

Expired SSL certificates are a silent killer for websites and online services. They cause immediate downtime, trigger browser security warnings that scare away customers, and can lead to data breaches. Manual tracking of certificate expiration dates across multiple domains is error-prone and time-consuming for IT teams.

This automated workflow solves these problems by creating a proactive monitoring and renewal system. It fetches your domain list from Notion, checks each SSL certificate's validity through SSL-Checker, sends instant Telegram alerts when certificates are nearing expiration, and can even trigger automatic renewals via SSH commands to your servers. The system prevents unexpected service interruptions by acting before certificates expire.

How It Works

The automation follows a logical sequence that ensures no certificate slips through the cracks while preventing unnecessary alerts for healthy certificates.

Step 1: Domain Inventory from Notion

The workflow begins by fetching all your registered domains from a Notion database. This centralizes domain management and ensures the system checks every website you own. Each domain entry can include additional metadata like owner, priority level, and renewal contacts.

Step 2: SSL Certificate Validation

For each domain, the system sends a request to SSL-Checker API to verify the current certificate's validity period, issuer, and encryption strength. It calculates exactly how many days remain until expiration and categorizes certificates as "Healthy," "Warning" (14-30 days), or "Critical" (less than 14 days).

Step 3: Intelligent Alerting via Telegram

Based on the validation results, the workflow sends tailored Telegram messages to your designated IT channels. Critical alerts include immediate action items, while warning notifications provide advance notice for planning. Successful renewals also generate confirmation messages.

Step 4: Automated Renewal Trigger

For certificates in the critical zone, the workflow can execute SSH commands on your web servers to trigger Certbot or similar renewal tools. This happens before human intervention is needed, though the system waits for confirmation before proceeding if you prefer manual approval.

Step 5: Notion Status Update

Finally, the workflow updates the original Notion database with the latest certificate status, expiration date, and last check timestamp. This creates an audit trail and gives your entire team visibility into certificate health without manual reporting.

This automation is ideal for IT managers, web agencies, SaaS companies, and any business managing multiple websites. If you have more than 3 domains, manual tracking becomes unreliable. Agencies managing client websites particularly benefit from centralized monitoring that prevents embarrassing client downtime. What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Notion account with a database containing your domain list (URL, notes, owner fields)
  3. Telegram account and a bot token (free to create via BotFather)
  4. SSH access to web servers for automated renewals (optional but recommended)
  5. SSL-Checker API access (free tier available for moderate usage)

Pro tip: Start with monitoring only for the first week to verify the system works correctly, then enable automated renewals once you're confident in the alerts. Always keep manual renewal as a backup option for critical production domains.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure Notion integration by adding your database ID and API key to the Notion node
  3. Set up Telegram bot by creating a bot via @BotFather and adding the token to the Telegram node
  4. Test SSL checking with a single domain to verify the SSL-Checker connection works
  5. Schedule the workflow to run daily or weekly based on your risk tolerance
  6. Configure alert thresholds (default is 14 days for critical, 30 days for warning)
  7. Optional: Set up SSH renewal by adding server credentials and testing Certbot commands
