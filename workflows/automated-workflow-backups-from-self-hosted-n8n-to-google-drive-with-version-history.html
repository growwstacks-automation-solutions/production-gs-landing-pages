Protect your workflows with versioned backups and easy restoration

What This Workflow Does

If you're using a self-hosted n8n instance, there's no built-in version history or undo for your workflows. This creates significant risk - if a workflow is accidentally modified, corrupted, or deleted, there's no simple way to recover it. This backup automation solves that problem by automatically syncing your workflows to Google Drive with full version history.

The workflow creates timestamped backups of each workflow, organized in dedicated folders. It intelligently detects changes to avoid duplicate backups while ensuring you always have recent versions available. Should disaster strike, you can easily restore any workflow version directly from Google Drive.

How It Works

1. Scheduled Trigger

The workflow begins on a timed schedule (configurable from hourly to weekly) using n8n's Cron node. This ensures backups happen automatically without manual intervention.

2. Fetch All Workflows

Using n8n's API endpoint, the workflow retrieves a complete list of all your existing workflows including their metadata and current versions.

3. Change Detection

For each workflow, the system compares the last modified timestamp against the most recent backup in Google Drive. This prevents unnecessary duplicate backups while catching all genuine changes.

4. Organized Storage

Each workflow gets its own folder in Google Drive, with JSON files named by timestamp (YYYY-MM-DD-HH-mm-ss). This creates a clear version history that's easy to navigate when restoration is needed.
