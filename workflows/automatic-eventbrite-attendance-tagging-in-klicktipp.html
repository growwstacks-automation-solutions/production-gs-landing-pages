What This Workflow Does

This automation solves a common pain point for event marketers: manually exporting attendance lists from Eventbrite and updating email lists in KlickTipp. The workflow automatically checks Eventbrite for attendee check-in status every 15 minutes and applies corresponding tags in KlickTipp.

When someone checks into your event, they're tagged as "Eventbrite | Participated." If they registered but didn't attend, they're tagged as "Eventbrite | Not participated." This real-time segmentation enables immediate, personalized follow-up campaigns without any manual data handling.

The system works for both virtual and in-person events, providing accurate attendance data that fuels your post-event marketing strategy. You can trigger different email sequences, measure event ROI more precisely, and build richer contact profiles based on actual participation.

How It Works

The automation follows a logical sequence to ensure reliable data synchronization between your event platform and email marketing system.

1. Scheduled Trigger

A schedule node runs every 15 minutes (configurable) to check for new attendance data. This frequency ensures timely updates without overwhelming either platform's API limits.

2. Fetch Eventbrite Attendees

The workflow connects to your Eventbrite account via OAuth2 and retrieves the latest attendee list for your specified event. It pulls key data including registration details, ticket information, and most importantly—check-in status.

3. Process Each Registrant

A Split Out node processes each attendee record individually. This ensures that if one record encounters an error, the rest continue processing without interruption.

4. Attendance Check & Tagging Logic

A Switch node evaluates the "checked_in" field for each attendee. If true, the contact receives the "Eventbrite | Participated" tag in KlickTipp. If false or null, they receive the "Eventbrite | Not participated" tag.

5. Update KlickTipp Contacts

Using KlickTipp community nodes (requires self-hosted n8n), the workflow applies the appropriate tags to each contact. The system matches contacts by email address, ensuring accurate updates to your existing email list.

6. Error Handling & Logging

The workflow includes basic error handling to catch API issues, missing contacts, or rate limiting. Failed operations can be configured to retry or send notifications for manual review.

What You'll Need

  1. Self-hosted n8n instance (community nodes like KlickTipp aren't available on n8n.cloud)
  2. Eventbrite organizer account with API access (OAuth2 credentials)
  3. KlickTipp account with API access enabled
  4. Pre-existing tags in KlickTipp: "Eventbrite | Participated" and "Eventbrite | Not participated"
  5. Event ID from your Eventbrite event URL
  6. Check-in method configured in Eventbrite (Organizer App, barcode scanning, or manual check-in)

Quick Setup Guide

Follow these steps to implement this automation in your n8n instance:

  1. Import the template into your n8n workspace using the downloaded JSON file.
  2. Connect your Eventbrite account via OAuth2 in the Eventbrite node settings.
  3. Update the Event ID in the Eventbrite node URL to match your specific event.
  4. Connect your KlickTipp account using API credentials from your KlickTipp settings.
  5. Replace the tag IDs in the KlickTipp nodes with your actual KlickTipp tag IDs.
  6. Test the workflow manually with a few registrants to ensure tags apply correctly.
  7. Activate the workflow and let it run on the scheduled trigger.

Pro tip: For high-attendance events, consider adjusting the schedule to run every 5 minutes during peak check-in times, then revert to 15-minute intervals afterward to balance timeliness with API limits.
