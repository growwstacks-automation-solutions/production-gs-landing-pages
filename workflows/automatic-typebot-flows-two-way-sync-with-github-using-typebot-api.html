What This Workflow Does

This automation solves a critical problem for businesses using Typebot for conversational AI: the risk of losing valuable chatbot configurations, flows, and logic. Without proper backups, hours of work designing sophisticated chatbot conversations can disappear due to server failures, accidental deletions, or migration issues.

The workflow automatically creates a two-way sync between your Typebot workspace and GitHub repositories. It exports all your Typebot flows using the Typebot API, checks existing files in GitHub, and intelligently updates, creates, or removes files to maintain perfect synchronization. This gives you version control, disaster recovery, and safe migration capabilities for your chatbot investments.

How It Works

Step 1: Fetch Typebot Flows

The workflow connects to the Typebot API using your credentials and retrieves all flows from your workspace. It extracts the complete configuration including blocks, options, variables, and settings in JSON format.

Step 2: Check GitHub Repository

For each Typebot flow, the automation checks your specified GitHub repository to see if a corresponding file already exists. It compares the flow ID and current configuration to determine what action is needed.

Step 3: Intelligent Sync Decisions

The workflow makes three types of decisions: update existing GitHub files when Typebot flows change, create new files for new flows, or ignore files that haven't changed to optimize performance.

Step 4: Cleanup Deleted Flows

A critical feature detects when flows have been deleted from your Typebot workspace and automatically removes the corresponding files from GitHub, keeping your repository clean and accurately reflecting your current chatbot setup.

What You'll Need

  1. Typebot Account: Access to a Typebot workspace with flows you want to backup
  2. Typebot API Credentials: Authentication tokens to connect to the Typebot API
  3. GitHub Account: A GitHub account with repository creation permissions
  4. GitHub Personal Access Token: API token with repo scope for repository access
  5. n8n Instance: Self-hosted n8n or n8n.cloud account to run the workflow

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure the Typebot credentials node with your API authentication details
  3. Set up the GitHub credentials with your personal access token
  4. Specify your GitHub repository details (owner, repo name, branch)
  5. Test the workflow with a single flow before enabling full synchronization
  6. Schedule the workflow to run automatically (daily, weekly, or on-demand)

Pro tip: Start by syncing to a private test repository before moving to your main backup repo. This lets you verify the process works correctly without affecting important backups.
