Google Drive Image Optimization n8n Automation

Automatically optimise images added to a Google drive folder

What This Workflow Does

This automation solves the common problem of unoptimized images accumulating in Google Drive folders. Many businesses and content creators upload images directly to Drive without proper compression, resulting in unnecessarily large files that slow down websites and waste storage space.

The workflow automatically detects new image uploads to specified Google Drive folders, processes them through TinyPNG's optimization API, and replaces the original files with compressed versions. This happens seamlessly in the background without any manual intervention required from your team.

How It Works

1. Monitoring Google Drive

The workflow continuously monitors your designated Google Drive folder for new image uploads. It can watch multiple folders simultaneously and handles all common image formats including JPG, PNG, and WebP.

2. Processing Through TinyPNG

When a new image is detected, the workflow sends it to TinyPNG's compression API. This service uses smart lossy compression techniques to reduce file size by an average of 60-80% while maintaining visual quality.

3. Replacing Original Files

The optimized image automatically replaces the original file in Google Drive. The workflow preserves all metadata and can maintain version history if configured to do so. You can also set it to store optimized copies in a separate folder if needed.

Pro tip: For maximum efficiency, set up separate workflows for different types of images (product photos vs. blog images) with customized compression levels for each.

Who This Is For

This automation is ideal for e-commerce businesses with large product image libraries, marketing teams producing visual content, bloggers and publishers, photographers storing client work, and any organization that regularly uploads images to Google Drive for web use.
