What This Workflow Does

Manual sales reporting is a time-consuming, error-prone task that distracts from running your business. This automation solves that by connecting directly to your Square account, pulling yesterday's complete sales data, and emailing a professional summary report to your management team every morning.

The workflow generates a CSV file that exactly matches Square's built-in Sales Summary report—including gross sales, taxes, tips, discounts, and payment methods—ensuring financial consistency. It runs automatically at 4:00 AM daily, so your team starts each day with accurate numbers without lifting a finger.

Beyond basic reporting, this template creates a reliable system for financial transparency, simplifies bookkeeping, and provides stakeholders with consistent data for better decision-making. It's particularly valuable for multi-location businesses, franchises, or any operation where timely sales data impacts daily operations.

How It Works

1. Daily Trigger & Location Fetching

The workflow activates automatically each morning via a schedule trigger. It first connects to the Square API to retrieve all business locations associated with your account, ensuring comprehensive coverage whether you have one store or twenty.

2. Order Data Collection

For each active location, the system pulls completed orders from the previous business day. It filters for finalized transactions only, excluding voids or test orders, to ensure report accuracy matches your actual revenue.

3. Data Processing & Aggregation

A custom code node processes the raw order data, calculating totals that mirror Square's official Sales Summary: net sales, gross sales, tax collected, tips, discounts, and payment method breakdowns. Locations with no sales are automatically filtered out.

4. Report Generation & Delivery

The processed data converts into a clean CSV file with proper formatting. The workflow then sends this file via Gmail to your designated recipients—managers, accountants, or partners—with a customizable subject line and message body.

What You'll Need

  1. Square API credentials – A Square access token with appropriate permissions to read location and order data.
  2. Gmail account – Or any email service compatible with n8n for sending reports.
  3. n8n instance – Self-hosted or cloud version of n8n to run the workflow.
  4. Recipient email addresses – The team members who should receive daily reports.
  5. Basic n8n familiarity – To configure credentials and activate the workflow.

Pro tip: Test the workflow with a single location first before enabling all locations. This helps verify data accuracy and email formatting before full deployment.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Configure Square credentials as Header Auth with name "Authorization" and value "Bearer YOUR_ACCESS_TOKEN".
  3. Set up Gmail credentials in n8n with appropriate sending permissions.
  4. Update email settings – Enter recipient addresses, customize subject line, and adjust message body as needed.
  5. Test and activate – Run the workflow manually once to verify, then set to active for daily automation.
