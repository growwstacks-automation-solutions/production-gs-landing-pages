What This Workflow Does

If you use YNAB (You Need A Budget) and shop frequently on Amazon, you know the frustration: bank imports show generic "Amazon" transactions with no details about what you actually bought. Manually updating each transaction memo with itemized purchases is tedious and time-consuming.

This automation solves that problem completely. It automatically scans your YNAB for unapproved Amazon transactions with empty memos, searches your Gmail for matching Amazon order confirmations, uses AI to extract the specific purchase details, and updates your YNAB transaction memos with clean, concise item lists. The result is a perfectly detailed budget without any manual work.

The workflow runs on a schedule you set (daily, weekly, or manually), processes each transaction individually to ensure accuracy, and includes smart error handling. If no matching email is found, it adds a fallback note instead of leaving the field blank.

How It Works

1. Trigger & Transaction Retrieval

The workflow starts on a schedule trigger (or manually/webhook). It connects to your YNAB account via API and retrieves all unapproved transactions from your specified budget. It filters these to find Amazon purchases with empty memo fields—these are the transactions needing details.

2. Email Matching & AI Extraction

For each Amazon transaction, the workflow searches your Gmail for Amazon order confirmation emails within ±5 days of the transaction date. When a matching email is found, the email content (HTML or text) is sent to an AI agent (OpenAI or compatible model). The AI is instructed to extract product names, quantities, and prices, then format a concise memo line under 499 characters.

3. YNAB Update & Error Handling

The generated memo is then used to update the specific YNAB transaction via API. A built-in 5-second delay between transactions prevents API rate limiting. If no valid purchase info is extracted (or no matching email is found), the workflow adds a fallback message like "Amazon purchase - details not found" instead of leaving the memo empty.

Pro tip: For even better accuracy, you can customize the email search date range or add HTML-to-Markdown conversion before sending to AI. This reduces token usage and can improve extraction precision for complex order emails.

What You'll Need

  1. YNAB account with API access (personal access token from YNAB Developer Settings)
  2. Gmail account where Amazon order confirmations are delivered (with OAuth2 access enabled)
  3. OpenAI API key or access to another compatible LLM (Gemini, Claude, etc.)
  4. n8n instance (self-hosted or n8n.cloud) to run the workflow
  5. Basic understanding of how to import JSON workflows and configure API credentials in n8n

Quick Setup Guide

  1. Download the template using the button above and save the JSON file to your computer.
  2. Import into n8n by creating a new workflow and using the "Import from File" option.
  3. Configure credentials for YNAB (personal access token), Gmail (OAuth2), and OpenAI (API key) in n8n's credential management.
  4. Set your YNAB Budget ID in the YNAB node—this tells the workflow which budget to check for transactions.
  5. Adjust the schedule trigger to run daily, weekly, or manually based on your preference.
  6. Test with a single transaction by using the manual trigger and checking that memos update correctly.
  7. Activate the workflow and let it run automatically according to your schedule.

Security note: Never store API keys or personal email data directly in workflow nodes. Always use n8n's credential system to manage sensitive information securely. The workflow uses official API connections with proper authentication.
