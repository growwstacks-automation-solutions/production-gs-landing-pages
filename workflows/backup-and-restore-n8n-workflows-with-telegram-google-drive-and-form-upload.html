What This Workflow Does

Losing your n8n workflows means losing hours of automation design, complex logic, and critical business processes. This template solves that risk by creating a fully automated backup and restore system for your n8n instance. It runs scheduled backups, sending compressed workflow files directly to Telegram for instant team access, while providing two flexible restoration methods: downloading from Google Drive or uploading a file directly through a form.

The system intelligently handles your entire workflow library, ensuring you can recover from accidental deletions, server failures, or migration errors within minutes rather than days. It's like insurance for your automation infrastructure—quietly working in the background until you need it most.

How It Works

1. Scheduled Backup Process

A schedule trigger initiates the backup every 3 days (configurable). The workflow calls the n8n API to fetch all workflows from your instance, aggregates them into a single JSON array, converts this to a text file, and sends it as a document to a designated Telegram chat. This creates an off-site, timestamped backup that's immediately accessible to your team.

2. Dual Restoration Pathways

The restore functionality offers two entry points. The first path downloads a backup file from a specified Google Drive link—perfect for shared team repositories. The second path accepts a direct file upload through an n8n form trigger, ideal for quick local file recovery. Both paths converge to parse the JSON and prepare workflows for restoration.

3. Intelligent Create-or-Update Logic

Before creating new workflows, the system checks if a workflow with the same name already exists in your n8n instance. If it exists, it updates the existing workflow; if not, it creates a new one. This prevents duplicates and maintains your workflow IDs where possible, preserving any external references or triggers.

4. Batch Processing with Error Handling

Using the "Split in Batches" node, workflows are processed in manageable groups to respect API rate limits. If any individual workflow fails to restore (due to invalid JSON or API issues), the system continues with the remaining items and logs the error, ensuring partial failures don't block the entire restoration process.

  3. A Google Drive account (optional, only needed for Drive-based restores).
  4. Basic understanding of n8n credential setup for n8n API, Telegram API, and Google Drive OAuth2.
  5. Permissions to create and modify workflows in your n8n instance.

Pro tip: Create a dedicated Telegram group for backup notifications and add all relevant team members. This creates a transparent audit trail of when backups occurred and who accessed them.

Quick Setup Guide

Follow these steps to implement this backup system in under 15 minutes:

  1. Import the template: Download the JSON file above and import it into your n8n instance via the "Import from File" option.
  2. Configure credentials: Set up three credentials in n8n: n8n API (your instance's API key), Telegram API (bot token), and Google Drive OAuth2 (authorized).
  3. Update configuration nodes: In the "Send Backup to Telegram" node, replace the placeholder chat ID with your actual Telegram chat ID. In the "Download Backup from Drive" node, update the File ID with your Google Drive file ID.
  4. Test the backup: Manually execute the Schedule trigger node to run an immediate backup and verify the file arrives in Telegram.
  5. Test restoration: Use the manual trigger to test Google Drive restoration, or activate the form trigger to test file upload restoration.
  6. Activate and monitor: Enable the workflow and check the next scheduled run completes successfully. Consider adding error notifications to a separate channel.
