What This Workflow Does

This intelligent AI agent system solves the problem of handling diverse business requests through a single interface. Instead of creating separate workflows for different task types, you build a manager agent that intelligently routes incoming requests to specialized sub-agents.

The system automatically determines whether a user's request requires email composition (like outreach or follow-up messages) or data analysis (like summaries or metrics), then delegates to the appropriate specialized agent. This creates a scalable automation foundation that grows with your business needs without requiring manual intervention for task classification.

By implementing this multi-agent architecture, you save hours previously spent manually routing tasks between team members or systems, while ensuring each request gets handled by the most qualified "virtual employee" for that specific type of work.

How It Works

Step 1: Manager Agent Analysis

The manager agent receives user input and analyzes it against predefined criteria. It evaluates whether the request involves communication tasks (emails, messages) or data tasks (analysis, summaries, metrics). This decision-making is powered by clear system prompts that define each sub-agent's domain.

Step 2: Intelligent Task Routing

Based on the analysis, the manager agent routes the task to either the email agent or data agent. The routing includes the original request plus any contextual information needed for the sub-agent to perform its work effectively.

Step 3: Specialized Task Execution

The email agent focuses on crafting professional communications with appropriate tone and structure, while the data agent specializes in analyzing information, identifying patterns, and presenting insights. Each sub-agent has its own memory system to maintain context within its domain.

Step 4: Response Delivery

The completed work is returned through the manager agent to the original requestor, creating a seamless experience where users interact with a single intelligent system rather than multiple specialized tools.

What You'll Need

  1. An n8n instance (cloud or self-hosted)
  2. OpenAI API key with access to GPT-4 or GPT-4o models
  3. Basic understanding of n8n's interface and node system
  4. The @n8n/n8n-nodes-langchain package installed in your n8n instance
  5. Clear definitions of what constitutes "email tasks" vs "data tasks" for your business

Pro tip: Before implementing, document 20-30 example requests your business receives and categorize them as email or data tasks. This will help you create more accurate system prompts for the manager agent's decision-making.
