What This Workflow Does

Manual card validation is error-prone and slows down payment processing. This automation solves that by instantly checking the Bank Identification Number (BIN)—the first 6-8 digits of any payment card—against a global database. It fetches critical details like the issuing bank, card brand (Visa, Mastercard), type (credit, debit, prepaid), and country of origin, all in real-time.

The workflow acts as a fraud prevention layer, enriching transaction data before it hits your payment gateway. It helps e-commerce stores, SaaS platforms, and financial services reduce chargebacks, improve authorization rates, and segment customers based on their payment method, turning a simple validation step into a strategic business intelligence tool.

How It Works

The automation connects your systems to apilayer's BIN Check API through a simple, reliable n8n workflow.

Step 1: Trigger & Input

The workflow can be triggered manually for testing or automatically by a webhook when a new transaction or customer record appears in your system. It receives the BIN code as input.

Step 2: API Request & Validation

n8n sends a secure request to the BIN Check API with the provided code. The API queries its global database and returns a structured JSON response containing all available card details.

Step 3: Data Processing & Output

The workflow parses the API response, extracts key fields (issuer, country, card type), and can format the data for your needs. The enriched information is then passed to the next step in your process—like updating a CRM, approving/declining a transaction, or logging to a database.

Pro tip: Use the returned 'country' field to dynamically adjust shipping options or display local currency. For example, flag cards from high-risk countries for manual review before processing.

Quick Setup Guide

Get this automation running in your n8n environment in under 10 minutes.

  1. Download & Import: Click the download button above to get the JSON file. In your n8n dashboard, go to Workflows > Import from File and select the downloaded template.
  2. Configure API Credentials: In the imported workflow, find the HTTP Request node. Replace the placeholder text in the 'apikey' header with your actual apilayer API key.
  3. Test with a Sample BIN: The workflow includes a sample BIN code. Run the workflow once manually to ensure the API connection works and you see a successful response with card details.
  4. Connect Your Data Source: Replace the Manual Trigger or sample data node with your actual trigger—like a webhook from your website or a scheduled check of new records in your database.
  5. Route the Output: Add nodes after the workflow to use the enriched data. Send it to your CRM, update a transaction log, or trigger a conditional approval process in your payment system.

Pro tip: For production use, add an error handling node after the API call. If the API is unavailable or returns an error, the workflow can retry, log the issue, and route the transaction for manual review instead of failing completely.
