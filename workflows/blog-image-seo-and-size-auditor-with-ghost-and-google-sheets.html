What This Workflow Does

Bloggers and content marketers often overlook image SEO, leaving alt text empty, filenames cryptic, and file sizes bloated. This hurts search rankings and slows page loads. This automation solves that by auditing every image in your Ghost CMS blog automatically.

The workflow extracts all published posts, scans HTML for images, checks alt text length and presence, measures file sizes, evaluates filename SEO quality, and logs everything into a Google Sheet. You get a clean, actionable report showing which images need fixing—without manual review.

It turns a tedious, error-prone manual process into a scheduled, reliable audit that ensures your visual content is fully optimized for search engines and user experience.

The workflow connects Ghost CMS, HTTP Request nodes for image downloads, and Google Sheets for reporting.

How It Works

Step 1: Fetch Blog Posts from Ghost CMS

The workflow starts by calling the Ghost Content API to retrieve all published posts. It pulls the HTML content of each article, ready for parsing.

Step 2: Parse Images from HTML

A Code node or HTML parser extracts all <img> tags from each post's content. It captures the src URL, alt text (if present), and any other attributes.

Step 3: Initial Log to Google Sheets

Basic metadata (post title, image URL, alt text status) is written to a Google Sheet. This creates the initial audit log before deeper analysis.

Step 4: Download & Analyze Each Image

For each image URL, an HTTP Request node downloads the file. It extracts the file size (in KB), extension, and checks filename SEO rules (lowercase, hyphens, no special chars).

Step 5: Update Sheet with Full Insights

The workflow updates the Google Sheet row with the detailed analysis: file size, whether it's under recommended limits, filename quality score, and overall compliance status.

Step 6: Schedule or Trigger

You can run this manually or set a schedule (weekly/monthly) to keep your blog's image SEO continuously monitored.
