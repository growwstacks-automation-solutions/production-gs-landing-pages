What This Workflow Does

This automation transforms a book idea into a fully researched, structured manuscript automatically. Instead of spending weeks manually outlining, researching, and drafting, you trigger the process from Airtable. The workflow uses AI agents to generate a compelling book description, create chapter outlines, conduct deep research for each chapter, and write complete chapter content.

It's designed for authors, ghostwriting agencies, and content teams who need to produce high-quality books quickly. The system connects your idea database (Airtable) with AI models (Google Gemini, OpenAI) and research tools (Perplexity) to automate the entire book creation pipeline. You get a structured manuscript ready for editing, saving 80–90% of the initial drafting time.

The workflow handles four key stages: book details generation, chapter outline creation, chapter‑specific research, and chapter content writing. Each stage updates your Airtable base, so you can track progress and intervene where needed. This turns book writing from a manual creative process into a scalable, automated production system.

The workflow structure: webhook trigger, action routing, AI agents for research and writing, and Airtable updates.

How It Works

1. Trigger from Airtable

When you add a book idea to Airtable and set the "Action" field (e.g., "Generate Book Details"), an Airtable automation sends a webhook to n8n. The webhook includes the record ID and the desired action, telling the workflow which stage to execute.

2. Action Routing

A Switch node reads the action parameter and routes the workflow to the appropriate branch: book details generation, chapter creation, chapter research, or chapter content writing. This ensures the right AI process runs for each request.

3. Data Retrieval

The workflow fetches the book or chapter record from Airtable using the provided ID. It pulls the idea, existing research, and any metadata needed for the AI agents.

4. AI Processing & Research

Depending on the action, different AI agents activate. For book details, an agent researches the topic using Perplexity and crafts a description and target audience. For chapters, an LLM generates 7–10 titles and descriptions. For research, an agent searches for stories, case studies, and expert perspectives. For content, an LLM writes the full chapter using the gathered research.

5. Airtable Updates

Each stage saves its output back to Airtable—research briefs, chapter lists, research notes, or full chapter text. The "Action" field is reset to "Idle," ready for the next step.

Quick Setup Guide

  1. Copy the Airtable base from the blueprint link in the template description.
  2. Create four Airtable automations (two for Book table, two for Chapter table) that trigger when the "Action" field changes and run a webhook script.
  3. Import the JSON workflow into your n8n instance.
  4. Configure all Airtable nodes with your base and table IDs.
  5. Connect your AI service accounts (Google Gemini, OpenAI, Perplexity) in the LLM and Agent nodes.
  6. Activate the workflow and test by creating a book record in Airtable and setting the Action field.
