What This Workflow Does

This template solves a critical business problem: expensive AI agent costs for simple knowledge retrieval. Traditional AI agents use powerful (and costly) models for every interaction, even basic greetings like "Hello." This creates unnecessary token waste and ballooning API bills.

The workflow provides a high-performance, cost-optimized alternative by implementing a modular "Routing & Specialized Steps" architecture. Instead of one expensive model deciding everything, it intelligently routes queries: simple conversations go to small, cheap models; only knowledge-based questions trigger the full RAG (Retrieval-Augmented Generation) pipeline with Lookio and GPT-4.1.

By leveraging Lookio as the core RAG platform, you connect your documentation (PDFs, Docs, webpages) to a chat interface without managing vector databases or custom chunking strategies. The result is a chatbot that delivers answers up to 50% faster and 3x more cost-efficiently than standard AI agents.

How It Works

Step 1: Memory & Intent Routing

The workflow first fetches past conversation history and uses a specialized Text Classifier (powered by a small, efficient model) to determine the user's intent. Is this a simple greeting or social interaction, or is it a knowledge-based question requiring information from your documents?

Step 2: Path A – Simple Response Handling

If the classifier detects a basic conversation (greetings, thanks, casual chat), the query is routed to a "nano" model. This small, inexpensive model generates an appropriate response instantly, saving the cost of involving larger models.

Step 3: Path B – Knowledge Retrieval Preparation

If information is needed, the workflow moves to a specialized LLM step that crafts a clean, optimized search query specifically for Lookio. This query refinement ensures the RAG system receives the most effective search terms for your knowledge base.

Step 4: RAG Execution via Lookio

The Lookio API retrieves exact insights from your connected documents. Lookio handles the entire retrieval pipeline—searching vector embeddings, ranking results, and extracting relevant passages—returning the most pertinent information for the query.

Step 5: Final Generation & Response

A larger model (like GPT-4.1) synthesizes the specific Lookio results with conversation history into a final, fact-based response. This step only occurs when deep knowledge is actually required, maximizing cost efficiency.

  2. API Credentials: Your Lookio API key and assistant ID for the RAG node configuration.
  3. AI Provider Access: OpenAI or alternative provider credentials for the small, medium, and large model nodes (supports multiple providers).

  5. Document Repository: PDFs, Word documents, webpages, or other structured content you want the chatbot to reference.

Quick Setup Guide

  1. Set up Lookio: Create an account at Lookio.app, upload your documents, and create an assistant. Note your API key and assistant ID.
  2. Configure API Key: In the "RAG via Lookio" node, replace <YOUR-API-KEY> in the header and paste your assistant_id in the request body.
  3. Add AI Credentials: Connect your OpenAI (or preferred provider) credentials to the "Very small model," "Mini model," and "Large model" nodes in n8n.
  4. Import & Activate: Import the downloaded JSON template into your n8n instance and activate the workflow.
  5. Test & Deploy: Test the chatbot via n8n's chat interface, then connect it to your preferred deployment channel (website, Slack, etc.).

Pro tip: Start with a small set of well-structured documents (like product FAQs) before scaling to larger repositories. This helps you refine the query routing and response quality with manageable complexity.

RAG (Retrieval-Augmented Generation) is an AI technique that combines information retrieval with text generation. It allows AI models to answer questions based on your specific company documents, not just general knowledge.
