What This Workflow Does

This automation solves the problem of information silos in businesses. When company knowledge is locked away in documents, employees waste time searching for answers or make decisions without complete information. This RAG (Retrieval-Augmented Generation) system creates a single source of truth that anyone can query naturally.

The workflow automatically processes documents uploaded to Google Drive, converts them into searchable chunks, stores them in a Qdrant vector database, and provides a chat interface powered by OpenAI. When someone asks a question, the system searches your documents first, then uses that context to generate accurate, source-cited answers.

Unlike generic chatbots, this system provides answers specific to your company's documentation—whether it's HR policies, technical specifications, compliance requirements, or internal procedures. It maintains conversation context and improves over time as you add more documents.

How It Works

1. Document Detection & Processing

The workflow monitors a designated Google Drive folder. When new files are added (Google Docs, PDFs, Word documents), it automatically downloads and converts them to markdown format while preserving structure.

2. Chunking & Vector Embedding

Documents are split into logical chunks using a recursive text splitter. Each chunk is converted into a vector embedding using OpenAI's embedding model, capturing the semantic meaning of the text.

3. Vector Storage in Qdrant

Embeddings are stored in a Qdrant vector database with metadata about the source document. This creates a searchable knowledge base that can find relevant information based on semantic similarity, not just keyword matching.

4. Query Processing & Response Generation

When a user asks a question, the system converts it to an embedding, searches Qdrant for the most relevant document chunks, then sends those chunks plus the question to OpenAI's chat model to generate a contextual answer with source citations.

What You'll Need

  1. n8n instance (self-hosted or cloud)
  2. Google Cloud credentials with Drive API enabled
  3. Qdrant account and API key (cloud or self-hosted)
  4. OpenAI API key with access to embedding and chat models
  5. Google Drive folder where documents will be uploaded
  6. Basic understanding of API key management

Pro tip: Start with a small test folder containing 5-10 representative documents. This lets you verify the system works correctly before scaling to your entire document library.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure Google Drive credentials in the "Detect New Files" trigger node following n8n's OAuth guide.
  3. Set up Qdrant credentials by creating an account at qdrant.tech and adding your API key to the "Insert into Vector Store" node.
  4. Add your OpenAI API key to both the embedding and chat model nodes.
  5. Specify your Google Drive folder ID in the trigger node settings.
  6. Activate the workflow and upload test documents to your Drive folder.
  7. Start chatting with your documents using the provided chat interface.
