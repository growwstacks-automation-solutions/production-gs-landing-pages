What This Workflow Does

This automation creates a Retrieval-Augmented Generation (RAG) chatbot that answers questions by referencing documents stored in your Google Drive. Unlike standard chatbots that rely on pre-trained knowledge, this solution taps into your specific PDFs, Word docs, and text files to provide accurate, context-aware responses.

The workflow solves the common problem of employees or customers needing quick answers from documentation without manually searching through files. It automatically indexes your Google Drive content, processes queries through OpenAI's language model, and returns precise answers with source references—reducing support tickets and improving knowledge accessibility.

How It Works

1. Document Processing

The workflow monitors your specified Google Drive folder for new or updated files. When detected, it extracts text content and breaks documents into manageable chunks for AI processing.

2. Vector Embedding Creation

Each text chunk is converted into vector embeddings using OpenAI's API. These numerical representations capture semantic meaning and are stored in Supabase's vector database for efficient retrieval.

3. Query Handling

When a user submits a question, the system searches the vector database for the most relevant document chunks, then constructs a prompt combining these references with the original query for OpenAI to generate a natural language response.

  3. OpenAI API key (GPT-3.5 or GPT-4)
  4. Supabase account for vector storage
  5. A webhook endpoint to receive user queries (Slack, website, etc.)

Pro tip: Start with a small set of high-quality documents rather than your entire Drive. This helps refine the system before scaling up.
