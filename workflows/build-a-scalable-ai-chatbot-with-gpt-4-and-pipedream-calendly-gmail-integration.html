What This Workflow Does

This automation solves a common business problem: handling repetitive customer inquiries and scheduling manually. Support teams spend hours answering FAQs, coordinating meetings, and sending confirmations. This workflow replaces that manual process with a scalable AI chatbot.

The chatbot uses GPT-4 to understand customer messages, accesses Pipedream's MCP server to securely connect with external tools, and can perform actions like scheduling a Calendly event or sending a Gmail confirmation. It acts as a virtual assistant that works 24/7, reducing response times from hours to seconds.

Beyond basic answers, it maintains conversation memory, asks clarifying questions, and executes multi‑step workflows—like collecting details, checking availability, booking a slot, and following up. This turns a simple chatbot into a full‑fledged support agent.

How It Works

Step 1: Chat Trigger

A customer message arrives via a chat platform (like Slack, Telegram, or a web widget). The workflow is triggered, capturing the user's query and context.

Step 2: AI Agent Processing

The message passes to an AI Agent node, which decides what action to take based on the intent. It evaluates whether the user wants information, scheduling, or another service.

Step 3: Language Model & Memory

GPT-4 processes the query, generating a natural response. A Simple Memory node retains the conversation history, allowing the chatbot to reference previous messages and maintain coherence across multiple turns.

Step 4: Tool Execution via Pipedream

If action is required (like scheduling), the agent calls Pipedream's MCP server, which securely bridges to Calendly or Gmail. The chatbot sends the necessary API requests without exposing credentials.

Step 5: Response Delivery

The final answer—whether informational or a confirmation of action—is sent back to the user, completing the interaction.

What You'll Need

  1. n8n Instance: Self‑hosted n8n (community nodes require self‑hosted version).
  2. Pipedream Account: Sign up at Pipedream.com to set up your MCP server.
  3. OpenAI API Key: Credentials for GPT‑4 access (any tier works).
  4. Calendly & Gmail Accounts: Connected in Pipedream with SSE endpoints generated.
  5. Chat Platform: A source for customer messages (Slack, Discord, webhook, etc.).

Quick Setup Guide

  1. Import the Template: Download the JSON file and import into your n8n workspace.
  2. Configure Pipedream: Create your MCP server, connect Calendly and Gmail, and copy the SSE endpoint URLs.
  3. Update Node Endpoints: Replace the placeholder SSE URLs in the Calendly and Gmail nodes with your live endpoints.
  4. Set OpenAI Credentials: Add your API key in the OpenAI Chat Model node settings.
  5. Activate Trigger: Enable the “When chat message received” node and connect it to your chat source.
  6. Test & Deploy: Send a test query, verify the chatbot responds and schedules correctly, then activate the workflow.

Pro tip: Start with a limited set of tools (just Calendly and Gmail) to validate the flow. Once working, gradually add more integrations via Pipedream—like Slack for internal alerts or Stripe for payments—to expand the chatbot’s capabilities.
