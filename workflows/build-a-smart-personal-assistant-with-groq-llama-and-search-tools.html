How It Works

The workflow follows a structured reasoning process that mimics how a human expert would approach a complex query.

1. Query Analysis & Planning

When a question arrives, the GROQ-powered LLaMA model first analyzes what's being asked. It determines if the query requires factual data (triggering a web search), mathematical computation (activating the calculator), or can be answered directly from its knowledge base.

2. Tool Selection & Execution

Based on its analysis, the assistant selects and uses the appropriate tools. For research questions, it calls SerpAPI to fetch current information from Google. For numerical problems, it uses the calculator node. The Think Tool helps break down complex multi-step problems into manageable pieces.

3. Memory Integration

A sliding window memory buffer maintains context from the last 5-10 messages. This allows the assistant to reference previous parts of the conversation, creating coherent dialogues rather than isolated Q&A sessions. The memory resets after inactivity to maintain privacy.

4. Response Synthesis

The assistant combines information from tools, its own reasoning, and conversation history to craft a comprehensive, accurate response. It cites sources for factual claims and shows its work for calculations, building trust through transparency.

5. Output Delivery

The final response is formatted for clarity and delivered through your chosen channel—whether that's a chat interface, email, document, or integrated directly into another business application.

Pro tip: Start with simple queries to test each tool independently before combining them. This helps you understand how the assistant reasons through different types of questions.

Quick Setup Guide

Follow these steps to get your intelligent assistant running in under 15 minutes:

  1. Import the Template: Download the JSON file above and import it into your n8n instance via the "Import from File" option.
  2. Configure Credentials: Set up your GROQ and SerpAPI API keys in the respective credential nodes.
  3. Test Basic Queries: Start with simple questions like "What's the weather in London?" to verify web search functionality.
  4. Test Calculations: Ask mathematical questions like "What is 15% of $247?" to ensure the calculator tool works.
  5. Test Memory: Ask follow-up questions that reference previous answers to confirm the memory buffer is functioning.
  6. Connect Your Interface: Replace the manual trigger with your preferred input method—Telegram bot, Slack app, webhook, etc.
  7. Customize Prompts: Adjust the system prompts in the AI nodes to match your preferred tone and response style.

Pro tip: Create a testing checklist for each tool before going live. Verify search results are current, calculations are accurate, and memory persists correctly across conversations.
