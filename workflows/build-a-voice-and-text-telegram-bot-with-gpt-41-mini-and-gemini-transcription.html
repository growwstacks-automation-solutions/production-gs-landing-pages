What This Workflow Does

This automation creates a sophisticated Telegram bot that can handle both voice and text messages from users. When a customer sends a voice message, the bot automatically transcribes it using Google Gemini's speech-to-text capabilities. The transcribed text (or direct text messages) is then processed by OpenAI's GPT-4.1-Mini model to generate intelligent, context-aware responses.

The bot maintains the original communication format—if a user sends a voice message, they receive a voice reply; if they send text, they get a text response. This creates a natural, conversational experience that feels personal and responsive, perfect for customer support, FAQ handling, or personal assistant applications.

By automating this entire interaction chain, businesses can provide 24/7 support without human intervention, reduce response times from hours to seconds, and handle multiple conversations simultaneously while maintaining consistent quality.

How It Works

Step 1: Message Reception

The Telegram Trigger node listens for incoming messages in your Telegram bot. It captures both text messages and voice notes, along with user information and chat context.

Step 2: Voice Transcription (If Applicable)

If the message contains a voice note, the workflow routes it to the Google Gemini node, which converts speech to text with high accuracy. The transcribed text is then passed forward for processing.

Step 3: AI Processing

The message text (either direct or transcribed) is sent to the OpenAI Chat Model node (GPT-4.1-Mini). The AI analyzes the query, considers conversation history if available, and generates an appropriate, helpful response.

Step 4: Response Delivery

The workflow checks the original message format and delivers the response accordingly—text replies for text messages, or text-to-speech conversion and voice message sending for original voice inputs.

Step 5: Context Management

Optional memory nodes can maintain conversation context across multiple exchanges, allowing the bot to remember previous interactions and provide more coherent, personalized assistance over time.

What You'll Need

  1. Telegram Bot Token: Create a bot via BotFather on Telegram and obtain your API key.
  2. OpenAI API Key: Sign up at platform.openai.com and generate an API key with appropriate credits.
  3. Google Gemini API Key: Get a free API key from Google AI Studio for speech-to-text transcription.
  4. n8n Instance: Either n8n.cloud account or self-hosted n8n installation.
  5. Basic Understanding: Familiarity with copying API keys into n8n credential fields.

Pro tip: Start with the free tiers of both OpenAI and Gemini to test the workflow. Both services offer initial credits that are sufficient for prototyping and moderate usage before scaling to paid plans.

Quick Setup Guide

  1. Download and Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows → Import from File and select the downloaded template.
  2. Configure Telegram: In the Telegram Trigger node, add your bot token from BotFather. Set up the webhook if using n8n self-hosted.
  3. Add API Keys: Create credentials in n8n for OpenAI and Google Gemini. Paste your respective API keys into each credential.
  4. Test Basic Flow: Activate the workflow and send a text message to your Telegram bot. You should receive an AI-generated response.
  5. Test Voice Functionality: Send a voice message to verify transcription and voice reply work correctly.
  6. Customize Prompts: Modify the system prompt in the OpenAI node to tailor responses to your specific use case and tone.
  7. Monitor and Iterate: Check the execution logs in n8n to see how messages are processed. Adjust parameters based on performance.
