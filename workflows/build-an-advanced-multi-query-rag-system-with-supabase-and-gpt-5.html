What This Workflow Does

This advanced template solves the critical limitations of basic Retrieval-Augmented Generation (RAG) systems. While simple RAG setups can answer straightforward questions, they often fail with complex queries and can be polluted by irrelevant information. This workflow introduces a sophisticated architecture that empowers your AI agent to think and act like a true research assistant.

By decoupling the agent from the knowledge base with a smart sub-workflow, this template enables multi-query decomposition, relevance-based filtering, and an intermediate reasoning step. The result is an AI agent that can handle complex questions, filter out noise, and synthesize high-quality, comprehensive answers based on your data in Supabase. This represents a production-ready architectural pattern that dramatically outperforms basic RAG implementations.

How It Works

This template consists of a main AI agent workflow and a smart sub-workflow that handles knowledge retrieval, creating a robust, decoupled system.

Step 1: Multi-Query Decomposition

When you ask the AI Agent a complex question, its system prompt instructs it to first break it down into an array of multiple, simpler sub-queries. For example, "How does natural selection work at molecular, organismal, and population levels?" becomes three targeted queries.

Step 2: Decoupled Knowledge Retrieval

The agent doesn't have direct access to the vector store. Instead, it calls a "Query knowledge base" tool, which is a sub-workflow. It sends the entire array of sub-queries to this sub-workflow in a single tool call, maintaining clean separation between reasoning and data retrieval.

Step 3: Iterative Retrieval & Filtering

The sub-workflow loops through each sub-query, querying your Supabase Vector Store for each one. It then checks the similarity score of returned data chunks and uses a Filter node to discard any that are not highly relevant (default score > 0.4), ensuring only quality information reaches the agent.

Step 4: Intermediate Reasoning & Synthesis

The sub-workflow returns all filtered information to the main agent. The agent then uses its Think tool to review this information, synthesize key points, and structure a plan for its final, comprehensive answer before delivering the response.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Connect your Supabase account in the sub-workflow ("RAG sub-workflow") to the Supabase Vector Store node and select your table.
  3. Connect your OpenAI account in two places: to the Embeddings OpenAI node (in the sub-workflow) and to the OpenAI Chat Model node (in the main workflow).
  4. Customize the agent's purpose by editing the AI Agent's system prompt in the main workflow. Change the context from "biology course" to your specific knowledge domain.
  5. Adjust relevance filtering by modifying the 0.4 threshold in the Filter node within the sub-workflow to match your quality requirements.
  6. Activate the workflow and start asking complex questions to test the system.

Pro tip: Start with a small subset of your data to test the relevance filtering threshold before scaling to your entire knowledge base. This helps you fine-tune the similarity score cutoff for optimal results.
