What This Workflow Does

This template solves the challenge of making business tools dynamically available to AI agents. Instead of hard-coding integrations, you build MCP (Model Context Protocol) servers that expose Google Calendar and custom functions as discoverable tools. AI agents can then automatically understand how to use these tools without manual configuration.

The workflow creates two separate MCP servers—one for native Google Calendar operations (check availability, create events) and another for custom business logic. This demonstrates how to blend standard SaaS integrations with proprietary functions, giving your AI agents both breadth and depth of capability.

Businesses using this approach eliminate repetitive scheduling tasks, enable AI to execute complex workflows, and future-proof their automation stack as new AI models emerge.

How It Works

Step 1: MCP Server Initialization

The workflow starts by setting up MCP Trigger nodes that act as servers. Each generates a unique URL endpoint that exposes its available tools with descriptions, parameters, and usage instructions.

Step 2: Google Calendar Tool Definitions

The Google Calendar server defines tools for checking free/busy status, creating events, updating existing appointments, and sending meeting invitations. Each tool includes proper authentication handling and error management.

Step 3: Custom Function Tool Creation

The second server exposes custom JavaScript functions that can perform calculations, transform data, or integrate with internal systems. These functions become immediately available to any connected AI agent.

Step 4: MCP Client Configuration

An MCP Client node connects to both server URLs, automatically discovering all available tools. The AI agent (configured separately) uses this client to understand what tools exist and how to call them.

Step 5: Dynamic Tool Usage

When an AI agent needs to schedule a meeting or run a calculation, it queries the MCP client, receives tool specifications, and makes appropriate API calls—all without human intervention.

Pro tip: Start with the Google Calendar server first, as it provides immediate value. Once comfortable, add custom functions for your specific business logic.

Quick Setup Guide

Follow these steps to get your MCP servers running:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure Google Calendar credentials in the respective node with OAuth.
  3. Review the custom function nodes and modify the JavaScript to match your needs.
  4. Activate both MCP Trigger nodes and note their generated URLs.
  5. Configure your AI agent's MCP client with the two server URLs.
  6. Test with simple prompts like "Check my calendar availability tomorrow" or "Calculate quarterly growth based on these numbers."

Important: The MCP servers run continuously while the workflow is active. Keep them running for persistent tool availability to your AI agents.
