What This Workflow Does

Many websites suffer from the same problem: visitors have questions but can't find answers quickly. They leave frustrated, increasing bounce rates and missing conversion opportunities. This creates unnecessary support tickets and lost business.

This workflow solves that by transforming any website into an intelligent, 24/7 AI chatbot. Using Retrieval-Augmented Generation (RAG) technology, it automatically extracts your website content, converts it into searchable vectors, and provides accurate, instant answers to visitor questions. The system learns from your specific content—not generic AI training data—ensuring brand-appropriate, up-to-date responses that actually help users.

The automation handles everything from content ingestion to real-time Q&A. When a visitor asks a question, it searches through your website's information, finds the most relevant sections, and generates a helpful answer using OpenAI's GPT-4o-mini. The result is a professional AI assistant that reduces support workload while improving user experience.

How It Works

The workflow follows a sophisticated pipeline that mimics how a knowledgeable human support agent would find and deliver information.

1. Content Ingestion & Processing

You submit website URLs through a simple form. The workflow fetches the HTML content, cleans it (removing navigation, ads, scripts), and parses it into logical sections. These chunks are optimized for both search relevance and AI comprehension.

2. Vector Embedding Creation

Each content chunk is converted into a mathematical vector using OpenAI's embedding model. These vectors capture the semantic meaning of the text, allowing the system to find conceptually similar content even when keywords don't match exactly.

3. Vector Database Storage

The embeddings, along with their source content and metadata, are stored in Supabase's vector database. Supabase provides a scalable, Postgres-based solution with pgvector extension, enabling fast similarity searches across thousands of documents.

4. Real-time Query Processing

When a visitor asks a question, the workflow converts that question into an embedding, searches Supabase for the most similar content chunks, retrieves the relevant source material, and sends both the question and context to OpenAI for answer generation.

5. Response Delivery

The AI generates a natural-language answer based specifically on your website content, cites its sources, and delivers it through a chat interface embedded on your site. The entire process takes seconds, providing instant value to visitors.

What You'll Need

  1. OpenAI API key for both embeddings and chat generation (GPT-4o-mini offers excellent performance at lower cost)
  2. Supabase account with a project where you've enabled the vector extension
  3. n8n instance (cloud or self-hosted) to run the workflow
  4. Website URLs you want to index (start with 5-10 key pages)
  5. Basic understanding of API keys and webhook configuration

Quick Setup Guide

Follow these steps to deploy your AI chatbot in under an hour:

  1. Download and import the template JSON file into your n8n instance
  2. Configure credentials for OpenAI and Supabase in n8n's credential management
  3. Set up Supabase by creating the required tables and enabling the vector extension
  4. Test the ingestion by submitting a single website URL through the form trigger
  5. Embed the chat widget on your website using the provided JavaScript snippet
  6. Monitor and refine by reviewing unanswered questions to identify content gaps
