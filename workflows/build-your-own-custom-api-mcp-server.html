Build Your Own Custom API MCP Server

A free n8n template to create secure MCP servers that let AI assistants interact with your internal APIs and data.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This template solves a common productivity bottleneck: employees needing to switch between multiple systems to find or update basic information. Instead of training everyone on complex database interfaces, you build a secure MCP (Model Context Protocol) server that lets AI assistants like Claude Desktop interact with your internal APIs through natural language.

The workflow demonstrates how to expose a sample HR system (PayCaptain) through three controlled tools: search employees, get employee details by ID, and update employee fields. It strips sensitive data, validates update permissions, and logs every interaction for audit purposes. This creates a safe bridge between conversational AI and your operational data.

How It Works

Step 1: MCP Server Trigger Setup

The workflow starts with an MCP Server Trigger node that listens for incoming requests from connected AI clients. This node handles the protocol communication, converting natural language queries into structured API calls.

Step 2: Custom Tool Definitions

Three custom Workflow Tool nodes are configured: Search Employee, Get Employee by ID, and Update Employee. Each tool maps to specific API endpoints in your internal system, with field filtering rules to exclude confidential information.

Step 3: API Integration & Data Sanitization

HTTP Request nodes call your actual backend APIs (like PayCaptain). Response data is processed to remove sensitive fields before being returned to the AI assistant. Update operations validate which fields can be modified to prevent unauthorized changes.

Step 4: Audit Logging

A Google Sheets node records every MCP interaction—who asked what, what data was returned, and any updates made. This creates a compliance trail and helps monitor usage patterns.

What You'll Need

  1. A running n8n instance (cloud or self-hosted)
  2. Access to an internal API you want to expose (like an HR system, CRM, or database)
  3. API credentials or developer keys for that system
  4. A Google Sheets spreadsheet for audit logging (optional but recommended)
  5. An MCP-compatible AI client (Claude Desktop, Cline, Windsurf, or custom agent)

Pro tip: Start with a non-critical system like an internal knowledge base or employee directory. This lets you test security and usability before connecting to sensitive financial or customer data.

Quick Setup Guide

  1. Download and import the JSON template into your n8n workspace.
  2. Replace the sample PayCaptain API calls with your own internal API endpoints.
  3. Configure field filtering in the Workflow Tool nodes—define which data fields should be exposed or hidden.
  4. Set up audit logging by connecting the Google Sheets node to your spreadsheet.
  5. Connect your MCP client (like Claude Desktop) using the n8n documentation guidelines.
  6. Test with simple queries like "Find employees who started last month" or "Update department for John Smith."
