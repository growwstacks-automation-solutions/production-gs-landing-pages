What This Workflow Does

Manually sharing dozens or hundreds of files from Google Drive is a tedious, error-prone task. This automation solves that by turning a multi-hour manual process into a one-click operation. It connects to your Google Drive via secure OAuth2, scans a specified folder, generates direct download links for every file, and updates their sharing permissions to be publicly accessible.

Built for businesses that regularly distribute digital assets—like marketing agencies sending brand kits, educators sharing course materials, or companies providing client deliverables—this workflow eliminates the risk of forgetting to share a file or sending incorrect links. It ensures every file is shared consistently and instantly.

The output is a clean list of direct download URLs you can import into a spreadsheet, send via email automation, or post to a client portal. It’s been tested on folders with over 4,000 files, proving its reliability for large-scale operations.

How It Works

The workflow orchestrates a series of steps to securely access, process, and share your files without manual intervention.

1. Trigger & Folder Setup

The process starts with a manual trigger, allowing you to run it on-demand or schedule it. You specify the Google Drive Folder ID containing the files you want to share. This ID is the only input required.

2. File Listing & Authentication

The Google Drive node authenticates using OAuth2 (no passwords stored) and lists all files within your target folder. It retrieves metadata like file names, IDs, and current permissions.

3. Batch Processing Loop

A "Split in Batches" node divides the file list into manageable groups (e.g., 50 files per batch). This prevents API rate limits and ensures smooth processing of large volumes.

4. Link Generation & Permission Update

For each file, the workflow generates a direct download link (not just a preview link) and updates the file's sharing settings to "anyone with the link can view." This creates a true public download URL.

5. Data Consolidation & Output

A "Merge" node combines all processed file data into a single, clean output. Each item includes the file name, direct download link, permission status, and other metadata, ready for use in your systems.

Pro tip: Use the "Set" node to add custom metadata—like client names or project codes—to each file record before output. This makes it easy to filter and distribute links later.

Quick Setup Guide

  1. Download the Template: Click the "Download Template" button above to get the JSON file.
  2. Import into n8n: In your n8n dashboard, go to "Workflows" > "Import from File" and select the downloaded JSON.
  3. Configure Google Drive Credentials: Click the Google Drive node, create a new OAuth2 connection, and follow the prompts to authorize n8n.
  4. Set the Folder ID: In the "Set Folder ID" node, paste the ID of your target Google Drive folder.
  5. Test Run: Execute the workflow manually with a few files first to verify links are generated correctly.
  6. Schedule or Trigger: Set the workflow to run on a schedule (e.g., daily) or trigger it via webhook when new files are added.

Pro tip: Use n8n's "Error Trigger" node to catch and notify you of any failed file operations, ensuring 100% reliability.
