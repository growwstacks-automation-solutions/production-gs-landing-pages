Bulk Create Shopify Products from Google Sheets with Inventory Sync

What This Workflow Does

This automation solves a critical pain point for e-commerce businesses: manually adding and updating products in Shopify. Instead of logging into your admin panel for every new item or stock change, you manage everything in a familiar Google Sheet. The workflow reads your spreadsheet, creates new products in Shopify with all their details (title, description, price, variants), and automatically sets inventory quantities at your store's default location.

It's particularly valuable for businesses launching new collections, syncing inventory from suppliers, or maintaining test/staging environments. The automation intelligently checks for existing products using SKU or handle matching, preventing duplicates while ensuring your catalog stays current with your master spreadsheet data.

Beyond basic product creation, this template establishes a reliable two-way synchronization foundation. While this version focuses on pushing data from Sheets to Shopify, it can be extended to pull inventory data back from Shopify, creating a complete inventory management system that keeps your spreadsheet as the single source of truth.

How It Works

The workflow connects your Google Sheets data directly to Shopify's Admin API, transforming spreadsheet rows into fully-configured products with proper inventory tracking.

Step 1: Google Sheets Trigger

The automation begins by reading your designated Google Sheet. It processes each row as a potential product, expecting specific columns like title, description, price, SKU, and stock_on_hand. You can schedule this to run automatically (daily, hourly) or trigger it manually when you update your product data.

Step 2: Data Validation & Transformation

Before sending data to Shopify, the workflow validates and formats it appropriately. It ensures prices are numbers, handles (slugs) are URL-friendly, and required fields are present. This preprocessing prevents API errors and maintains data consistency across your store.

Step 3: Shopify Product Creation

Using Shopify's GraphQL Admin API, the workflow creates products with all specified attributes. It sets product status (ACTIVE, DRAFT, or ARCHIVED), assigns categories, adds descriptions, and configures pricing including compare_at_price for sales. Each product receives a unique handle for SEO-friendly URLs.

Step 4: Inventory Management

After product creation, the workflow updates inventory levels at your store's default location. It connects the product's inventory item with the quantity specified in your spreadsheet, ensuring your available stock is accurately reflected on the storefront to prevent overselling.

Step 5: Error Handling & Logging

The automation includes error checking to handle API limits, missing data, or connectivity issues. Failed items are logged for review, while successful creations are marked to prevent reprocessing. This ensures reliable operation even with large product catalogs.

What You'll Need

  1. A Shopify store with admin access to generate API credentials
  2. Google Sheets access with edit permissions for your product spreadsheet
  3. n8n instance (cloud or self-hosted) to run the workflow
  4. Product spreadsheet with columns: title, description, price, compare_at_price, SKU, stock_on_hand, status, and handle/slug
  5. Shopify Store URL to configure the GraphQL API endpoints

The workflow uses Shopify's 2025-04 GraphQL Admin API, so ensure your store supports this version. Your Google Sheet should have the column headers in the first row, with each subsequent row representing one product variant.

Pro tip: Before running the automation on your live store, test it with a small subset of products or on a development store. Use DRAFT status for initial imports to review products before making them active.

Quick Setup Guide

Follow these steps to implement this automation in your own n8n environment:

  1. Download the template using the button above and import it into your n8n instance
  2. Configure Google Sheets node with your spreadsheet ID and worksheet name
  3. Update all GraphQL nodes with your Shopify store URL (replace placeholder with your actual store.myshopify.com domain)
  4. Set up Shopify credentials using an API access token with products and inventory write permissions
  5. Test with a few rows using product status "DRAFT" to verify everything works correctly
  6. Schedule the workflow to run automatically based on your update frequency needs

The template includes comprehensive error handling and will skip existing products if the slug matches an existing product handle. This means you can safely re-run the automation as your spreadsheet evolves without creating duplicates.
