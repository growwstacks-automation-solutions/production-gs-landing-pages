What This Workflow Does

Manually deleting hundreds or thousands of outdated, invalid, or unsubscribed contacts from HubSpot is a tedious, error-prone task that wastes valuable time. This automation solves that problem by letting you upload an Excel or CSV file containing email addresses, and it automatically finds and deletes those contacts from your HubSpot CRM.

The workflow acts as a self-service portal for your marketing, sales, or operations teams. Instead of requesting deletions from IT or navigating complex HubSpot interfaces, team members simply upload a spreadsheet. The system handles the rest—searching, verifying, and deleting each contact—while maintaining a clean, compliant database.

This is particularly valuable for GDPR compliance, cleaning bounced email lists, removing unsubscribed contacts, or pruning old leads that no longer fit your ideal customer profile. It transforms a manual, hour-long process into a 30-second task.

How It Works

The automation follows a logical sequence to ensure safe, accurate contact deletion while handling errors gracefully.

1. File Upload & Trigger

The workflow starts when a team member uploads an Excel (.xlsx) or CSV file via a webhook endpoint. This creates a secure, controlled entry point for your deletion requests without requiring direct HubSpot access.

2. Data Extraction & Normalization

The system reads the uploaded file, extracts the email addresses from the specified column (default "emails"), and converts them into structured data items. It normalizes the data, ensuring consistent formatting for downstream processing.

3. Batch Processing Loop

Each email address is processed individually through a batch loop. This approach maintains control over execution speed, manages API rate limits, and allows for detailed error tracking per contact.

4. HubSpot Contact Search

For each email, the workflow queries HubSpot's Contact API to find the corresponding contact record. It retrieves the internal HubSpot contact ID needed for deletion.

5. Existence Verification

A conditional check determines if the contact was found in HubSpot. If found, the workflow proceeds to deletion. If not found (or already deleted), it skips to the next email, preventing errors.

6. Contact Deletion

Using the retrieved contact ID, the workflow calls HubSpot's deletion endpoint to permanently remove the contact from your database. Each successful deletion is logged for reporting.

7. Optional Post-Processing

A placeholder node allows you to extend the workflow with custom logic—such as sending confirmation emails, updating internal dashboards, or logging results to Google Sheets for audit purposes.

What You'll Need

  1. HubSpot Account: A HubSpot account with API access and permissions to search and delete contacts.
  2. HubSpot App Token: An API token configured in n8n with sufficient permissions.
  3. Excel/CSV File: Spreadsheet containing email addresses in a dedicated column (default column name: "emails").
  4. n8n Instance: Either n8n.cloud or self-hosted n8n with webhook capability enabled.
  5. Basic Understanding: Familiarity with n8n node configuration for potential customizations.

Pro tip: Test the workflow with a small sample file (5-10 emails) before running it on your full contact list. This helps verify your configuration and prevents accidental mass deletions.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the Template: Download the JSON file and import it into your n8n instance via the workflow import function.
  2. Configure HubSpot Credentials: Add your HubSpot App Token to the HubSpot node credentials in n8n.
  3. Set Up Webhook: Note the webhook URL generated by n8n—this is where you'll POST your Excel files.
  4. Prepare Your Spreadsheet: Ensure your Excel/CSV file has a column named "emails" containing the addresses to delete.
  5. Test the Flow: Upload a test file with a few known email addresses and verify the deletion occurs correctly in HubSpot.
  6. Deploy & Monitor: Activate the workflow and set up basic monitoring or notifications for completion alerts.
