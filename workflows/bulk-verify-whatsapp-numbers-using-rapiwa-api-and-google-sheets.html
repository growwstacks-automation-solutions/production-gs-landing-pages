What This Workflow Does

This automation solves a critical problem for businesses using WhatsApp for marketing, support, or customer communication: wasted effort on invalid phone numbers. Manually checking hundreds or thousands of contacts for WhatsApp registration is time-consuming and error-prone.

The workflow automatically processes contact lists from Google Sheets, verifies each number through the Rapiwa API, and updates your spreadsheet with verification status. It identifies which contacts are active WhatsApp users before you invest in campaign creation or messaging, ensuring your communications reach real people.

By automating this verification process, businesses can maintain cleaner contact databases, improve campaign success rates, and allocate resources more effectively. The system works continuously, processing new contacts as they're added to your sheets and providing real-time verification status.

How It Works

The automation follows a logical sequence to ensure reliable verification while managing API limits and maintaining data integrity.

Step 1: Trigger & Data Collection

The workflow activates on a schedule (configurable from every 5 minutes to daily) and reads pending verification requests from your Google Sheet. It identifies rows where the status column is marked "pending" and extracts phone numbers for processing.

Step 2: Phone Number Cleaning

Before verification, the workflow standardizes phone number formats by removing non-digit characters, country code inconsistencies, and formatting variations. This ensures the Rapiwa API receives properly formatted numbers for accurate verification.

Step 3: API Verification

Each cleaned number is sent to the Rapiwa verification endpoint. The API checks whether the number is registered with WhatsApp and returns a verified/unverified status. The workflow handles API responses, including rate limit management and error recovery.

Step 4: Status Update & Logging

Verified numbers are marked as "verified" in your Google Sheet, while unverified numbers receive "unverified" status. The system logs timestamps, API responses, and any errors for audit purposes. Optional message sending can be triggered for verified contacts.

Step 5: Batch Management & Throttling

The workflow processes contacts in configurable batches with built-in delays between API calls to prevent rate limiting. You can control how many numbers are verified per run and adjust timing based on your API plan limits.

What You'll Need

  1. n8n instance (self-hosted or cloud) with access to Google Sheets and HTTP Request nodes
  2. Google Sheets account with a spreadsheet containing your contact list (columns for phone numbers, names, and status)
  3. Rapiwa API account and authentication token (Bearer token for API access)
  4. Basic understanding of Google Sheets formatting and n8n workflow import
  5. WhatsApp-enabled phone number for testing (if extending to message sending)

Pro tip: Start with a small test batch of 10-20 numbers to verify your setup works correctly before processing large contact lists. This helps identify formatting issues and ensures your API credentials are properly configured.

Quick Setup Guide

Follow these steps to implement the WhatsApp verification automation in your environment:

  1. Download and import the template JSON file into your n8n instance using the workflow import function.
  2. Duplicate the sample Google Sheet or prepare your own with columns for: WhatsApp Number, Name, Message, Verification Status, and General Status.
  3. Configure Google Sheets authentication in n8n by creating OAuth2 credentials and connecting to your spreadsheet.
  4. Add your Rapiwa API token as an HTTP Bearer credential in n8n's credential management system.
  5. Adjust workflow settings including trigger frequency, batch size, and delay between API calls based on your volume needs.
  6. Test with sample data by adding a few test contacts with "pending" status and running the workflow manually.
  7. Monitor initial runs and verify that status columns update correctly before enabling scheduled automation.
