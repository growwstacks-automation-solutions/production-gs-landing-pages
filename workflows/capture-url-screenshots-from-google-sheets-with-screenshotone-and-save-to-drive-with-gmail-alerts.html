Capture URL Screenshots from Google Sheets with ScreenshotOne & Save to Drive

Automatically capture website screenshots from URLs in Google Sheets, save to Google Drive, and receive email notifications.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the tedious, time-consuming task of manually capturing screenshots from multiple websites. Instead of opening each URL individually, taking screenshots, saving files, and organizing them, this workflow handles everything automatically. When you add a Google Sheets spreadsheet to a designated Google Drive folder, the workflow extracts URLs from the "Url" column, sends each URL to ScreenshotOne for high-quality screenshot capture, saves the resulting images back to Google Drive, and sends a Gmail notification with a link to the completed folder.

The automation eliminates repetitive manual work that can consume hours each week, especially for teams monitoring multiple websites. It ensures consistency in screenshot quality, naming conventions, and organization while providing immediate notifications when the process completes. This transforms what was previously a manual, error-prone task into a reliable, hands-off process.

How It Works

Step 1: Google Drive Trigger

The workflow monitors a specific Google Drive folder for new spreadsheet additions. When a spreadsheet is added, the workflow automatically starts processing. This trigger-based approach means you simply upload your URL list to the right folder, and everything else happens automatically.

Step 2: Google Sheets Processing

The workflow reads the spreadsheet and extracts all URLs from the column named "Url." It can handle hundreds of URLs in a single batch, processing each one sequentially. The Google Sheets integration ensures that any team member can update the URL list using familiar spreadsheet tools.

Step 3: Screenshot Capture via ScreenshotOne

Each extracted URL is sent to the ScreenshotOne API, which captures high-quality, full-page screenshots. The API handles JavaScript-heavy websites, provides device emulation options, and ensures consistent results regardless of website complexity. Screenshots are captured with appropriate delays to ensure page loading completes.

Step 4: Save to Google Drive & Send Notifications

Captured screenshots are automatically saved to the same Google Drive folder with descriptive filenames. Once all screenshots are processed, a Gmail notification is sent to designated recipients with a direct link to the folder containing all images. This completes the automated workflow from URL list to delivered screenshots.

What You'll Need

  1. Google Drive access with permissions to create and access folders
  2. Google Sheets integration configured in n8n
  3. Gmail account for sending notifications
  4. ScreenshotOne account (free or paid plan) with an access key
  5. n8n instance (cloud or self-hosted) to run the workflow

Pro tip: Start with ScreenshotOne's free tier to test the workflow with a small number of URLs before scaling up. Their API playground is excellent for testing screenshot configurations before implementing them in your automation.

Quick Setup Guide

  1. Download the template using the button above
  2. Import the JSON file into your n8n instance
  3. Configure the Google Drive trigger node to monitor your target folder
  4. Set up Google Sheets credentials to access your spreadsheets
  5. Add your ScreenshotOne access key to the HTTP Request node
  6. Configure Gmail credentials for notification emails
  7. Test with a spreadsheet containing 2-3 URLs to verify everything works
  8. Deploy the workflow and start uploading URL lists
