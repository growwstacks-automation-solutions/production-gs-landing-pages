What This Workflow Does

Manual expense tracking from Revolut statements consumes valuable hours each month and is prone to human error. This automation solves that by transforming raw Revolut CSV exports into intelligently categorized financial data ready for analysis.

The workflow downloads your Revolut transaction statements, normalizes the data, uses AI to understand and categorize each expense, then stores everything in a Supabase database. What used to be a monthly chore becomes an automated process that provides real-time visibility into spending patterns.

How It Works

1. Data Extraction & Normalization

The workflow begins by fetching Revolut transaction data, typically from exported CSV files. It cleans and structures the raw data, handling different currencies, date formats, and transaction types to create a standardized dataset.

2. AI-Powered Categorization

Each transaction is analyzed by GPT-4, which examines merchant names, descriptions, amounts, and context to assign accurate categories. The AI references your custom category taxonomy stored in Supabase, learning from previous categorizations to improve accuracy over time.

3. Database Storage & Enrichment

Categorized transactions are stored in Supabase with rich metadata including category confidence scores, AI reasoning, and timestamps. This creates a queryable financial database that serves as the foundation for reports, dashboards, and integrations.

4. Optional Reporting & Notifications

The workflow can be extended to generate summary reports, send Slack notifications for large expenses, or trigger reconciliation processes in accounting software like QuickBooks or Xero.

What You'll Need

  1. Revolut Business Account with transaction export capabilities (CSV exports)
  2. Supabase Project with tables for transactions and categories
  3. OpenAI API Key or compatible LLM provider access
  4. n8n Instance (self-hosted or cloud)
  5. Storage for CSV files (Google Drive, Dropbox, or local storage)

Pro tip: Start with a small set of historical transactions to train the AI on your specific expense categories before processing current data. This significantly improves initial accuracy.

Quick Setup Guide

  1. Download the JSON template file using the button above.
  2. Import it into your n8n instance via the workflow import function.
  3. Configure the Revolut node with your account credentials or CSV file source.
  4. Set up your Supabase database with the required transaction and category tables.
  5. Add your OpenAI API key to the AI categorization node.
  6. Define your expense categories in the Supabase categories table.
  7. Test with a small batch of transactions, then schedule the workflow to run daily or weekly.
