Free workflow template to validate file hashes against VirusTotal and send real-time Slack alerts for security threats.

What This Workflow Does

This security automation workflow solves the critical problem of manual file hash analysis, which consumes valuable security team time and introduces human error. When security analysts encounter suspicious files, they need to quickly determine if they're malicious, but checking hashes across multiple platforms is slow and inconsistent.

The workflow automates this entire process by accepting file hashes (MD5, SHA1, or SHA256) via HTTP webhook or Slack slash command, querying VirusTotal's comprehensive threat intelligence database, analyzing detection statistics, and delivering instant verdicts. It categorizes files as Malicious, Suspicious, Clean, or Unknown, then sends formatted alerts to Slack channels with severity-based formatting while returning structured JSON responses for integration with other security tools.

Pro tip: Use this workflow as the foundation for a security orchestration platform. You can extend it to automatically quarantine malicious files, create incident tickets, or trigger deeper forensic analysis based on the VirusTotal verdict.

How It Works

1. Hash Submission & Normalization

The workflow accepts file hashes through two channels: an HTTP POST endpoint for API-based integrations from security tools, or a Slack slash command (/hash-check) for quick analyst-driven checks. It normalizes the input by trimming whitespace and converting to lowercase, then validates the hash format against MD5, SHA1, and SHA256 patterns.

2. VirusTotal API Query

Using your VirusTotal API credentials, the workflow queries the file hash reputation endpoint. VirusTotal aggregates data from 70+ antivirus engines and threat intelligence sources, providing comprehensive detection statistics, file metadata, and community analysis.

3. Threat Analysis & Verdict Calculation

The workflow analyzes VirusTotal's detection ratios and additional threat indicators to calculate a security verdict. It applies configurable thresholds to determine if a file is Malicious (high detection rate), Suspicious (moderate detection or suspicious behaviors), Clean (no detections), or Unknown (no data available).

4. Alert Delivery & Response

For all verdicts, the workflow sends formatted messages to designated Slack channels. Malicious results trigger alert-style formatting with priority indicators. Simultaneously, it returns a structured JSON response containing the hash, verdict, detection statistics, and analysis timestamp for integration with SIEM systems or ticketing platforms.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n instance via the workflow import function.
  2. Configure VirusTotal Credentials: Create a new credential in n8n using your VirusTotal API key. Test the connection with a known hash.
  3. Set Up Slack Integration: Create a Slack App in your workspace, enable slash commands, and configure the Request URL to point to your n8n webhook endpoint. Add the Slack credentials to n8n.
  4. Customize Alert Channels: Update the Slack node to send messages to your preferred security channel. Configure message formatting based on your team's preferences.
  5. Test & Activate: Test with sample hashes using both HTTP POST and Slack slash command methods. Once verified, activate the workflow.
  6. Integrate with Security Tools: Share the HTTP webhook URL with your existing security tools (EDR, SIEM, email gateways) to enable automated hash checking.

The workflow supports MD5, SHA1, and SHA256 hash formats—the three most common cryptographic hashes used in security analysis. These cover 95% of file identification needs in security operations, from malware analysis to forensic investigations.

MD5 is fastest but has collision vulnerabilities, SHA1 is widely supported, and SHA256 provides the strongest security. The workflow automatically detects which format you're using and processes it appropriately with VirusTotal's API.

  • MD5: 32-character hexadecimal (fast, legacy systems)
  • SHA1: 40-character hexadecimal (balanced speed/security)
  • SHA256: 64-character hexadecimal (most secure standard)
