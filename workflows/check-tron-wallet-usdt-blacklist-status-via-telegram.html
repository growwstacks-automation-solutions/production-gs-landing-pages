Check Tron Wallet USDT Blacklist Status via Telegram

Automatically verify if a Tron wallet address is blacklisted on the USDT contract using a Telegram bot. Streamline crypto compliance checks for your team.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a critical compliance challenge for crypto businesses, exchanges, and investment platforms. When dealing with Tron-based USDT transactions, you need to ensure you're not interacting with blacklisted wallets that could be involved in fraud, sanctions violations, or illegal activities. Manual checking is slow, error-prone, and doesn't scale.

The workflow creates a Telegram bot interface that allows any team member to instantly check a wallet's blacklist status by simply sending the address. It queries the Tronscan API in real-time, processes the response, and delivers a clear result back to the requester—all within seconds. This turns a complex compliance task into a simple chat command.

How It Works

Step 1: Telegram Command Trigger

The workflow listens for a specific command (like /checkwallet) followed by a Tron wallet address in your Telegram group or direct message with the bot. When a user sends this command, the automation captures the message and extracts the wallet address for processing.

Step 2: API Query to Tronscan

The extracted wallet address is sent to the Tronscan API endpoint that checks the USDT contract blacklist. The HTTP request node performs this query, retrieving the official blacklist status data directly from the blockchain explorer's reliable source.

Step 3: Response Processing

A function node analyzes the API response. If the wallet appears on the blacklist, it formats an alert message with the blacklisted address details. If the wallet is clean, it creates a confirmation message. This logic handles various response scenarios and error conditions.

Step 4: Result Delivery

The formatted result is sent back to the original requester via Telegram, providing immediate feedback. The message includes clear visual indicators (like warning emojis for blacklisted addresses) and relevant details from the API response for documentation purposes.

Who This Is For

This automation is ideal for crypto exchanges needing quick compliance checks during customer withdrawals, OTC trading desks verifying counterparty addresses, investment platforms conducting due diligence, compliance teams monitoring transaction patterns, and crypto businesses that need to demonstrate regulatory diligence. It's particularly valuable for teams where multiple staff members need to perform checks without specialized blockchain expertise.

  4. Telegram group or channel where the bot will operate
  5. Team members trained on the simple command syntax

Quick Setup Guide

  1. Import the downloaded JSON template into your n8n instance
  2. Add your Telegram bot credentials to the Telegram trigger and send message nodes
  3. Configure the command prefix (default is /sorgu but you can customize it)
  4. Test with a known Tron wallet address to verify the API connection
  5. Share the bot with your team and demonstrate the simple command format
  6. Monitor the workflow execution for the first few days to ensure reliability
