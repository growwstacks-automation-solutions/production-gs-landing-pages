What This Workflow Does

For hotels, vacation rentals, and hospitality businesses, every guest enquiry is a potential booking. But manually reading, categorizing, and responding to dozens of emails daily is slow, error-prone, and leaves guests waiting. This automation solves that.

This n8n workflow acts as an intelligent front-desk assistant. It captures guest enquiries from your website, booking platform, or contact form, uses GPT-4o to understand the guest's intent instantly, generates a polite and personalized acknowledgement email, assigns the enquiry to the correct internal team with a clear SLA, and posts a full summary to a Slack channel for visibility and follow-up.

The result? Guests receive an immediate, professional response 24/7. Your sales, front-desk, or reservations team gets pre-sorted, actionable tickets instead of a chaotic inbox. You gain full oversight of enquiry volume and response performance, all while dramatically reducing manual workload.

How It Works

The workflow is a linear, logical pipeline that transforms a raw guest message into a structured, routed task.

1. Webhook Trigger – Enquiry Intake

The workflow starts with a webhook node, waiting for a POST request from your website form, booking engine, or other system. It receives the guest's name, email, message, and any other relevant details (dates, party size, preferences). This is your universal entry point for all guest communications.

2. AI Intent Classification & Reply Drafting

The core of the automation: the OpenAI (GPT-4o) node. The guest's message is sent to the AI with a custom prompt instructing it to: a) Classify the intent (Booking, Pricing, Availability, Policy/FAQ, Other), b) Determine the appropriate internal team (Sales, Front Desk, Reservations, Management), and c) Generate a warm, professional acknowledgement email draft that can be sent immediately.

3. Data Extraction & Routing

A Code node or Function node parses the AI's structured response, extracting the detected category and assigned team. A Switch node then routes the enquiry down the appropriate path based on this category, ensuring it reaches the right people.

4. Team Assignment & SLA Setting

Based on the routed category, the workflow sets a Service Level Agreement (SLA)—for example, "Booking" enquiries must be answered within 1 hour, "Pricing" within 4 hours. This SLA is attached to the enquiry data for tracking and accountability.

5. Automated Guest Acknowledgement (Conditional)

If the guest's contact preference is email, the Gmail node sends the AI-generated acknowledgement instantly. This step can be made conditional, so you only auto-reply when appropriate, maintaining a human touch where needed.

6. Internal Notification & Logging

A Slack node posts a rich, formatted summary of the enquiry to a designated channel (e.g., #guest-enquiries). This summary includes the guest's details, the AI's classification, the assigned team, the SLA, and a link to the full data. Another Slack node logs the exact AI-generated reply sent to the guest for transparency and quality assurance.

7. Error Handling & Alerts

An Error Trigger node monitors the entire workflow. If any step fails—API error, connection issue, etc.—it immediately sends a detailed alert to a separate Slack channel (#automation-alerts), so your tech team can intervene before any guest enquiry is lost.

What You'll Need

  1. An n8n instance (cloud or self-hosted).
  2. An OpenAI API key (or Azure OpenAI endpoint) for GPT-4o or GPT-4o-mini access.
  3. A Gmail account with OAuth2 credentials set up in n8n to send automated replies.
  4. A Slack workspace and a Slack app with a Bot User OAuth Token (with chat:write permissions) to post to your channels.
  5. A webhook source, such as a website contact form (using a service like Formspree, Tally, or a custom endpoint) or a booking platform that can send POST requests.

Pro tip: Test the webhook trigger first using a tool like Postman or Insomnia. Send a sample JSON payload matching your form data structure to the n8n webhook URL to ensure the workflow receives data correctly before configuring the AI steps.

Quick Setup Guide

Get this automation running in your n8n environment in under 30 minutes.

  1. Import the Template: Download the JSON file using the button above. In your n8n dashboard, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure Credentials: In the n8n credentials section, set up your OpenAI, Gmail (OAuth2), and Slack Bot Token credentials. Name them appropriately (e.g., "OpenAI Production", "Company Gmail").
  3. Update the Webhook URL: Activate the workflow and copy its unique Webhook URL. Paste this URL into your website form's "Webhook" or "Zapier" integration settings, or configure your booking system to send data to this endpoint.
  4. Customize the AI Prompt: Open the OpenAI node. Review and tweak the system prompt to match your brand's tone, your specific enquiry categories, and your team names.
  5. Set Your Slack Channels: Update the Slack node URLs to point to your actual team channels (e.g., #guest-enquiries, #automation-logs, #alerts).
  6. Test & Activate: Send a test enquiry through your form. Watch the workflow run, check the Slack channel for the summary, and verify the test email is sent. Once confirmed, activate the workflow.
