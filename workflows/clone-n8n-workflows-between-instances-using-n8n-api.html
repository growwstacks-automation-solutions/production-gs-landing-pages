Clone n8n Workflows Between Instances

Automatically copy, sync, and migrate workflows across development, staging, and production environments using n8n's API.

What This Workflow Does

Managing multiple n8n instances across different environments (development, staging, production) or teams creates a significant operational challenge. Manually exporting and importing workflows is time-consuming, error-prone, and doesn't scale. This automation solves that problem by programmatically cloning workflows between n8n instances using n8n's own API.

The workflow acts as a bridge between your source and destination n8n environments. It fetches all workflows from your source instance, compares them with what exists in the destination, and only copies the missing or updated workflows. This ensures consistency across your automation infrastructure while preventing accidental overwrites or duplication.

Beyond simple copying, this solution enables proper DevOps practices for automation—treating workflows as code that can be versioned, tested in isolation, and safely promoted through environments. It transforms workflow management from a manual administrative task into a reliable, automated process.

How It Works

1. API Authentication Setup

The workflow begins by authenticating with both source and destination n8n instances using API credentials. These credentials are securely stored as variables, ensuring they're not hardcoded into the workflow logic itself.

2. Source Workflow Retrieval

Using n8n's REST API, the automation fetches all workflows from your source instance. This includes workflow names, IDs, and the complete JSON definitions that contain all nodes, connections, and settings.

3. Destination Inventory Check

Simultaneously, the workflow queries the destination instance to get a complete inventory of existing workflows. This creates a baseline for comparison to determine what needs to be copied.

4. Differential Analysis

A comparison logic identifies which workflows exist in the source but not in the destination, or which have been updated since the last sync. This differential approach prevents unnecessary transfers and preserves existing configurations.

5. Batch Creation Process

Using n8n's "Loop Over Items" node, the workflow processes each qualifying workflow individually, creating it in the destination instance with the exact same configuration. Error handling ensures failed transfers don't stop the entire process.

6. Logging and Notification

Each transfer is logged with success/failure status, and optional notifications can be sent to Slack, email, or other channels to keep your team informed of the sync results.

What You'll Need

  1. Two or more n8n instances (self-hosted or cloud) with network accessibility between them
  2. API credentials for both source and destination instances with appropriate read/write permissions
  3. Basic understanding of n8n workflows to configure the authentication and potentially modify the destination project settings
  4. Network access allowing HTTP requests between your n8n instances (or through a central orchestrator)
  5. Optional: A notification channel (Slack, email, etc.) for receiving sync reports

Pro tip: Before running this in production, test with a single non-critical workflow to verify your API credentials and network connectivity work correctly. Consider implementing a "dry run" mode that logs what would be transferred without actually making changes.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance (this will be your "orchestrator" instance).
  2. Configure API credentials for both source and destination n8n instances in the HTTP Request nodes.
  3. Test the connection by running the first few nodes manually to ensure they can fetch workflows from both instances.
  4. Adjust the filter logic if you want to transfer workflows to a specific project or exclude certain workflows.
  5. Set up a schedule trigger (daily, weekly, or on-demand) based on your synchronization needs.
  6. Implement error notifications so you're alerted if the sync fails for any reason.
  7. Document the process for your team, especially if multiple people will be managing the workflow.
