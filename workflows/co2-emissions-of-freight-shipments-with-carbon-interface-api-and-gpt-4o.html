What This Workflow Does

This automation solves a critical problem for logistics and supply chain businesses: manually tracking the carbon footprint of every shipment is time-consuming, error-prone, and unsustainable at scale. When a carrier sends a shipment confirmation email, employees must manually extract details like pickup/delivery addresses, cargo weight, and distance—then calculate emissions using external tools or spreadsheets.

This n8n workflow automates the entire process. It listens for new shipment emails, uses an AI agent to parse unstructured text into structured data, calls the Carbon Interface API to compute CO2 emissions based on weight and distance, and logs everything—including timestamps and calculated metrics—into a Google Sheet. The result is a real-time, auditable carbon ledger for your logistics operations.

Beyond saving hours of manual work each week, this system enables accurate sustainability reporting, helps meet ESG (Environmental, Social, Governance) goals, and provides data for customer transparency. It’s designed for logistics coordinators, transportation planners, and sustainability officers who need to quantify environmental impact without adding administrative burden.

How It Works

Step 1: Trigger on New Shipment Email

The workflow starts when a Gmail trigger detects a new shipment confirmation email from your carrier. This could be an automated notification from FedEx, UPS, DHL, or any logistics provider. The email contains pickup/delivery addresses, cargo weight, estimated distance, and delivery window.

Step 2: AI Agent Parses Shipment Details

An AI Agent (GPT‑4o) reads the email body and extracts key fields into a structured JSON object. It identifies addresses, converts weight units, estimates distance if not provided, and normalizes dates. This replaces manual copy‑pasting and ensures consistent formatting for the carbon API.

Step 3: Store Metadata in Google Sheets

The extracted shipment metadata—carrier, pickup/delivery locations, weight, distance, scheduled time—is appended as a new row in a Google Sheet. This creates a master log of all shipments before emissions are calculated.

Step 4: Call Carbon Interface API

Using the weight and distance data, the workflow calls the Carbon Interface API—a specialized service that returns CO2 emissions estimates based on transport mode (road, air, sea) and regional emission factors. The API returns kilograms or tons of CO2 equivalent for that shipment.

Step 5: Append Emissions to the Sheet

The calculated CO2 value is added to the same row in Google Sheets, completing the record. You now have a timestamped entry with all shipment details and its carbon footprint, ready for reporting or dashboard integration.

What You'll Need

  1. A Gmail account (or Google Workspace) that receives shipment confirmation emails from your carriers.
  2. A Google Sheet prepared with columns for shipment metadata and CO2 results.
  3. A free Carbon Interface API key from carboninterface.com.
  4. Access to OpenAI (or another LLM) for the AI Agent node.
  5. A few sample shipment emails to test the parsing logic.
  6. An n8n instance (cloud or self‑hosted) to run the workflow.

Pro tip: Start with one carrier’s email format. Once the AI Agent is tuned to extract data correctly, you can add additional triggers for other carriers.

Quick Setup Guide

  1. Import the JSON template into your n8n workspace.
  2. Configure the Gmail trigger node with your account credentials.
  3. Set up the AI Agent node with your OpenAI API key and adjust the prompt to match your email format.
  4. Add your Carbon Interface API key to the HTTP Request node.
  5. Connect the Google Sheets node to your spreadsheet and map the columns.
  6. Test with a real shipment email—check that extraction, calculation, and logging work correctly.
  7. Activate the workflow and let it run automatically for all incoming shipment notifications.
