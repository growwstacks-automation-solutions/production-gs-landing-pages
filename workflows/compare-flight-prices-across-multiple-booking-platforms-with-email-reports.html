What This Workflow Does

Manually comparing flight prices across multiple booking platforms is time-consuming and inefficient. Business travelers and travel managers often spend hours searching Kayak, Skyscanner, Expedia, and Google Flights to find the best deals, only to miss price drops that happen after they've booked.

This automation workflow solves that problem by running simultaneous price checks across all major platforms, analyzing the results, and delivering a comprehensive email report with the best available options. It accepts natural language requests like "Flight from NYC to London on March 25" or structured data, making it accessible for both technical and non-technical users.

The system doesn't just find the cheapest flight—it identifies the best value by considering price, duration, stops, and departure times, then presents everything in an easy-to-read format that supports quick decision making.

How It Works

1. Receive & Parse Flight Request

The workflow starts with a webhook that accepts flight requests in natural language or structured JSON. It uses natural language processing to extract departure city, destination, dates, passenger count, and travel class, converting city names to airport codes automatically.

2. Parallel Platform Scraping

Once the request is validated, the workflow simultaneously queries Kayak, Skyscanner, Expedia, and Google Flights through SSH nodes running Python scraping scripts. This parallel approach is 4x faster than checking platforms sequentially.

3. Data Aggregation & Analysis

All flight data from different platforms is collected, normalized, and analyzed. The system identifies the overall best deal, the best non-stop option, calculates average prices, and sorts all results by price and value.

4. Email Report Generation & Delivery

A professional email report is formatted with route details, the top recommendations, complete price breakdown, and direct booking links. This is sent via SMTP to the requester while the workflow simultaneously returns a JSON response for API users.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow import capabilities
  2. SMTP credentials for sending email reports (Gmail, SendGrid, etc.)
  3. SSH access to a server for running Python scraping scripts
  4. Python environment with Selenium and ChromeDriver for web scraping
  5. Basic understanding of n8n nodes and credential management

Pro tip: Use a dedicated email address for sending flight reports to avoid deliverability issues. Services like SendGrid or Mailgun offer better reliability for automated emails than personal Gmail accounts.

Quick Setup Guide

Follow these steps to get your flight price comparison automation running in under 30 minutes:

  1. Download and import the JSON workflow file into your n8n instance using the "Import from File" option.
  2. Configure SMTP credentials in n8n for your email service provider to enable report delivery.
  3. Set up SSH credentials pointing to your server with the Python scraping environment.
  4. Deploy the Python scraper on your server and test it with sample flight searches.
  5. Activate the workflow and test with a simple flight request to verify everything works end-to-end.
