Compare Lists & Identify Common Items & Differences Using Custom Keys

Automatically find duplicates, unique entries, and overlaps between two datasets using a custom identifier like email, ID, or domain.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manual list comparison is a tedious, error-prone task that wastes valuable business hours. Whether you're reconciling customer databases, checking inventory discrepancies, or merging marketing lists, manually scanning spreadsheets for matches and differences is inefficient and risky.

This automation workflow solves that by programmatically comparing two lists of objects (List A and List B) using a user‑specified key—such as email, customer ID, SKU code, or domain name. It returns three clean outputs: items common to both lists (matches), items only in List A, and items only in List B. The process is fully automated, consistent, and scalable.

Instead of spending hours cross‑referencing spreadsheets, you get instant, accurate results that can be fed into downstream actions—like updating a CRM, sending follow‑up emails to new contacts, or flagging discrepancies for review.

How It Works

Step 1: Input Configuration

The workflow accepts a JSON input containing your two lists and the comparison key. You define the key field (e.g., “email”) that exists in both datasets. The lists can come from any source—Google Sheets, Airtable, a CRM export, or an API response.

Step 2: Key‑Based Matching

The automation engine extracts the specified key from each item in both lists and performs a field‑based comparison. It ignores other fields (like names or dates) and focuses solely on the identifier you choose, ensuring precise matching.

Step 3: Output Segmentation

The workflow outputs three structured arrays: common (items with matching keys), onlyInA (items unique to List A), and onlyInB (items unique to List B). Each output retains the full object data, so you have complete records for further processing.

Step 4: Integration & Action

The results can be sent to any connected app—added to a Slack channel, inserted into a database, appended to a spreadsheet, or used to trigger follow‑up workflows (e.g., sending welcome emails to new leads).
