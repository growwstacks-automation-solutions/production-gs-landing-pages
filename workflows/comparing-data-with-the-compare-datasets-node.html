Compare Datasets Automatically with n8n

Free workflow template to identify matches, differences, and unique entries between two datasets—no manual spreadsheet work required.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manually comparing spreadsheets or database exports is tedious, error-prone, and wastes valuable time. This automated workflow solves that by programmatically comparing two datasets based on a common field (like "fruit" in the example, or "customer_id," "product_sku," "email" in real use). It instantly identifies which items exist in both sets, which are unique to Dataset 1, and which are unique to Dataset 2.

Think of reconciling daily sales reports from Shopify and QuickBooks, merging new newsletter signups with your existing CRM contacts, or checking inventory lists between warehouses. This template provides the foundational logic to automate those comparisons, turning hours of cross-referencing into a button-click operation.

The business value is clear: reduce manual effort, eliminate human error in data reconciliation, and free your team to focus on analysis and action rather than data janitorial work.

How It Works

The workflow uses n8n's powerful "Compare Datasets" node to perform the heavy lifting.

Step 1: Trigger & Data Input

The workflow begins with a manual trigger, simulating a user action like "Run comparison." Two separate nodes then generate or fetch the datasets to compare. These could be connected to actual data sources like a Google Sheet, an Airtable base, a MySQL query, or an API response in your live implementation.

Step 2: Standardization & Preparation

Before comparison, data is often cleaned or transformed. The workflow ensures both datasets have a consistent field to match on (e.g., a standardized email column, a product ID). This step is crucial for accurate matching.

Step 3: The Core Comparison

The "Compare Datasets" node receives both prepared datasets. You configure it to match rows based on your chosen key field. It then outputs separate streams of data: matches (items in both), differences (items only in Dataset 1), and unique entries (items only in Dataset 2).

Step 4: Output & Action

The results are structured and can be sent to various destinations. You could automatically add new unique entries to a master database, send an alert for mismatched prices, or generate a summary report in a Google Doc.

What You'll Need

  1. An n8n instance (either n8n.cloud account or self-hosted).
  2. Two data sources you want to compare (e.g., CSVs, Google Sheets, Airtable bases, database tables).
  3. A common identifier field present in both datasets (like email, user ID, SKU, transaction ID).
  4. Basic understanding of how to import a JSON workflow into n8n.

Pro tip: Start with small, clean sample datasets to test your matching logic before running on production data. Ensure your key field is consistently formatted (e.g., all lowercase emails) to avoid false mismatches.

Quick Setup Guide

Follow these steps to get your data comparison running in minutes.

  1. Download the template using the button above and save the JSON file.
  2. In your n8n dashboard, go to Workflows > Import from File and select the downloaded JSON.
  3. Replace the sample "Dataset" nodes with nodes that connect to your actual data sources (e.g., Google Sheets, MySQL, Airtable).
  4. Configure the "Compare Datasets" node: Set the "Join Field" to your common identifier (e.g., "email").
  5. Test the workflow with a small dataset using the "Execute Workflow" button. Check the output branches for matches and differences.
  6. Add output actions based on the results—like sending matches to a CRM or logging differences to a file.
