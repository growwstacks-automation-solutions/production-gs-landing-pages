What This Workflow Does

If you use Airtable to manage content like blog posts, product descriptions, or email templates, you've likely encountered the challenge of rich text fields storing content in Markdown format. While Markdown is excellent for writing, most publishing platforms, websites, and email systems require HTML. Manually converting dozens or hundreds of records is tedious, error-prone, and doesn't scale.

This automation workflow solves that problem by automatically converting Airtable rich text fields from Markdown to clean, standards-compliant HTML. Whether you need to process a single updated record or sync an entire table, this template handles the conversion seamlessly. It preserves all your formatting—headings, lists, links, images, and code blocks—while transforming them into proper HTML elements ready for any web application.

The workflow operates in two modes: trigger-based for real-time conversion of new or updated records, and batch processing for migrating entire tables. This flexibility makes it ideal for content teams, marketers, and developers who manage structured content in Airtable but need HTML output for their websites, newsletters, or applications.

How It Works

The workflow connects to your Airtable base, processes rich text fields through a Markdown-to-HTML converter, and outputs the formatted content to your chosen destination.

Step 1: Airtable Trigger or Schedule

The workflow begins by either watching for new/updated records in your Airtable base or running on a schedule to process records in batches. You can configure it to monitor specific tables and fields, ensuring only relevant content gets converted.

Step 2: Markdown Extraction

For each record, the workflow extracts the content from your designated rich text field. It handles Airtable's specific Markdown formatting, including any embedded media or special characters that might be present in your content.

Step 3: HTML Conversion

A dedicated conversion node transforms the Markdown into clean HTML. This process maintains semantic structure—headings become proper h1-h6 tags, lists become ul/ol elements, links keep their href attributes, and images get appropriate img tags with alt text.

Step 4: Output & Delivery

The converted HTML can be sent to multiple destinations: written back to another Airtable field, posted to a CMS via API, inserted into a database, attached to an email, or stored in a file. You can also add conditional logic to route content based on specific criteria.

Pro tip: Add a "Last Converted" timestamp field in Airtable to track when records were processed. This helps prevent duplicate conversions and makes it easy to identify records that need updating.

Quick Setup Guide

Follow these steps to implement this automation in your n8n instance:

  1. Import the template: Download the JSON file above and import it into your n8n instance via the workflow import function.
  2. Configure Airtable connection: Add your Airtable API key and select the base and table containing your rich text fields.
  3. Set field mappings: Specify which field contains Markdown content and where the HTML should be stored (either in a new Airtable field or external destination).
  4. Test with one record: Run the workflow manually on a single record to verify the conversion works correctly with your specific Markdown formatting.
  5. Configure triggers: Set up the trigger to run on schedule (daily/hourly) or watch for record updates, depending on your needs.
  6. Monitor and refine: Check the first few conversions, adjust any formatting preferences, then activate the workflow for full automation.

Important: Always test with a copy of your data first. Create a test table or duplicate records to ensure the conversion handles your specific Markdown patterns correctly before applying to production data.
