What This Workflow Does

Network documentation is a critical but tedious task for IT teams. Keeping visual diagrams updated as devices change, links are added, or configurations evolve is a manual, error-prone process. This workflow solves that by automating the entire diagram creation pipeline.

It connects to your AWX (Ansible) automation platform, runs Cisco CDP commands to gather real-time neighbor data, parses the output, and uses Google's Gemini AI to generate a structured prompt specifically formatted for Lucidchart's AI diagram generator. You get an accurate, ready-to-use prompt that Lucidchart can instantly turn into a visual network map.

The result is a living documentation system: your diagrams automatically reflect the current state of your network, saving hours of manual drawing and reducing risk of outdated or incorrect visual records.

How It Works

Step 1: Trigger AWX Job Execution

The workflow starts by calling your AWX Job Template. This template contains the network commands (like show cdp neighbors detail) that collect topology data from your Cisco switches and routers. AWX executes these commands securely across your infrastructure.

Step 2: Monitor Job Completion & Fetch Results

n8n polls AWX to wait for the job to finish, then retrieves the stdout—the raw text output of the CDP commands. This data contains all neighbor relationships, interface details, and device identifiers.

Step 3: Parse & Structure Network Data

A JavaScript node processes the raw output, extracting device names, connected interfaces, and link information into a clean JSON structure. This step normalizes the data, removing inconsistencies and preparing it for AI processing.

Step 4: Generate Lucidchart AI Prompt

The structured data is sent to Google Gemini AI with instructions to create a Lucidchart-specific prompt. The AI formats the topology into a descriptive text that Lucidchart's diagram AI can understand, including node placement, connection lines, and labels.

Step 5: Output Ready-to-Use Prompt

The final step delivers a clean text prompt you can copy-paste directly into Lucidchart. Lucidchart's AI then generates the visual diagram automatically, completing the automation loop from live network data to updated visual documentation.

Pro tip: You can modify the AWX Job Template to include additional commands like show inventory or show version to enrich the diagram with device models and software versions.

What You'll Need

  1. AWX or Ansible Tower instance with API access and a Job Template configured to run CDP commands.
  2. Google Gemini API credentials (or another LLM like OpenAI) configured in n8n.
  3. n8n instance (cloud or self-hosted) with HTTP Request, Code, and AI nodes available.
  4. Basic understanding of your network device IPs and credentials stored in AWX.
  5. Lucidchart account (free or paid) to use the generated prompts.

Quick Setup Guide

  1. Download and import the JSON template into your n8n workspace.
  2. Configure AWX credentials in n8n's HTTP Request nodes: add your base URL, username, and password/token.
  3. Update the Job Template ID in the "Launch Job" node to match your AWX template that runs CDP commands.
  4. Add Gemini AI credentials in the LLM node, selecting the Gemini model.
  5. Test the workflow with a manual trigger. Verify the output prompt is well-structured.
  6. Copy the final prompt into Lucidchart's AI diagram tool to generate your network map.

Pro tip: Schedule this workflow to run weekly or after any network change. This keeps your diagrams perpetually updated without manual intervention.
