What This Workflow Does

This automation solves the tedious and error-prone process of manually converting business documents into Markdown format. It handles PDFs, Word documents, and PowerPoint files - extracting text while preserving structure, tables, and formatting elements.

The workflow combines MinerU API's advanced document processing capabilities with GPT-4o-mini's ability to clean and structure content. It's particularly valuable for teams maintaining documentation in version control systems, knowledge bases, or content management systems where Markdown is the preferred format.

How It Works

1. Document Upload

The workflow begins when a user uploads a document through the n8n chat interface. Supported formats include PDF, DOCX, PPTX, and more.

2. MinerU API Processing

The document is sent to MinerU API which performs OCR (for scanned PDFs), extracts text content, identifies document structure, and processes tables/formulas.

3. AI Enhancement

GPT-4o-mini cleans and structures the raw output, ensuring proper Markdown formatting for headings, lists, tables, and other elements.

4. Output Generation

The final Markdown is saved to a specified location and can be automatically pushed to repositories, CMS systems, or other destinations.

What You'll Need

  4. Storage location for processed files

Quick Setup Guide

  1. Import the workflow JSON into your n8n instance
  2. Configure your OpenAI API key in the GPT node
  3. Set your MinerU MCP server address (default: localhost:8000)
  4. Specify output directory in the "Write Files" node
  5. Test with sample documents
