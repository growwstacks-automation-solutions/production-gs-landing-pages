What This Workflow Does

If you're building AI-powered Telegram bots or notification services, you've likely faced a common headache: Large Language Models (LLMs) generate long, unstructured text mixed with media links, but Telegram has strict limits and expects formatted messages. Manually parsing this output is tedious and error-prone.

This automation solves that. It acts as a post‑processing layer for your AI assistant. You feed it raw LLM output—text with embedded image, video, or audio URLs—and it automatically transforms that into Telegram‑friendly messages. Media links become native previews (images show inline, videos/audio get players), and long text is intelligently split into sequential chunks that respect Telegram's character limits. The entire process happens in a single n8n execution, ensuring your bot delivers a polished, professional user experience without manual intervention.

Essentially, it bridges the gap between raw AI creativity and Telegram's messaging constraints, turning clunky outputs into engaging conversations.

How It Works

The workflow is triggered by a parent process (like your main AI bot workflow) via the Execute Workflow node. It receives the chat ID and the raw LLM output string.

Step 1: Extract & Classify Media Links

A JavaScript Code node scans the input string for HTTP/HTTPS URLs using a regex pattern. It deduplicates links and classifies each by file extension (.jpg, .png, .mp4, .mp3, etc.). Extracted links are stored separately, and the original text is cleaned of those URLs.

Step 2: Route Media to Correct Telegram Methods

A Switch node evaluates each link's type. Image URLs are sent via the Telegram node's "Send Photo" method, video links via "Send Video", and audio links via "Send Audio". This ensures Telegram renders native previews and players—users don't just see a plain link.

Step 3: Smart Text Chunking & Sending

If the cleaned text exceeds 1,000 characters, another Code node splits it at line breaks or sentence boundaries to preserve readability. A Split in Batches node then sends each chunk as a separate Telegram message, in order, preventing truncation.

Step 4: Convergence & Completion

All branches—media sends and text sends—converge. The workflow completes, logging success. Your parent workflow receives confirmation that the rich message was delivered.

Pro tip: You can adjust the character limit in the "If text too long" node's expression to match Telegram's exact limits or your preferred chunk size.
