Convert PDF to HTML using PDF.co and Google Drive

Automatically convert newly uploaded PDFs in Google Drive to HTML files and save them back—no manual work required.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the tedious, error-prone task of manually converting PDF documents to HTML format. When businesses receive PDFs—whether they're reports, invoices, manuals, or forms—they often need that content in web-friendly HTML for publishing, archiving, or further processing.

The workflow monitors a specific Google Drive folder for new PDF uploads. When a PDF appears, it automatically sends it to the PDF.co API for conversion, receives the HTML output, and saves it back to Google Drive in an organized location. This eliminates hours of manual copy-pasting, formatting cleanup, and file management.

Beyond basic conversion, the workflow includes smart validation—it checks if the uploaded file is actually a PDF before processing, preventing errors. This creates a completely hands-off pipeline for document transformation that works 24/7.

How It Works

Step 1: Google Drive Trigger Watches for New Files

The workflow starts with a Google Drive trigger node that monitors a designated folder. Whenever any new file is added to this folder, the automation activates. This setup means you can simply drag-and-drop PDFs into a specific Google Drive location to initiate the entire conversion process.

Step 2: Validation Check Ensures Only PDFs Process

An "IF" node examines the uploaded file's extension. If it's a PDF (.pdf), the workflow continues. If not (like an image or Word document), the process stops gracefully. This prevents wasted API calls and errors from trying to convert incompatible files.

Step 3: PDF.co API Converts Document to HTML

The PDF file is sent to PDF.co's conversion API via an HTTP Request node. This professional conversion service preserves formatting, tables, images, and structure far better than free online tools. The API returns clean, structured HTML ready for web use.

Step 4: Response Processing and File Preparation

A Function node processes the API response, transforming the HTML data into a proper binary file format that Google Drive can store. This step ensures the converted content is packaged correctly with appropriate metadata and file naming.

Step 5: Converted HTML Saves Back to Google Drive

The final Google Drive node uploads the newly created HTML file to a destination folder of your choice. You can organize converted files by date, project, or type—keeping your digital assets neatly managed without manual intervention.

Quick Setup Guide

Importing and configuring this workflow takes about 15 minutes:

  1. Download and Import: Download the template file and import it into your n8n instance via the workflow import function.
  2. Configure Google Drive Connections: Authenticate both the trigger and upload nodes with your Google account, selecting your designated input and output folders.
  3. Add PDF.co API Key: Insert your PDF.co API key into the HTTP Request node headers where indicated in the workflow.
  4. Test with Sample PDF: Upload a test PDF to your Google Drive input folder and trigger the workflow manually to verify everything works.
  5. Activate and Monitor: Turn on the workflow and monitor the first few conversions to ensure files are processed correctly and saved to the right location.
