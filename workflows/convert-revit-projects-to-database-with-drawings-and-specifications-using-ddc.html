What This Workflow Does

Construction and engineering firms spend countless hours manually extracting data from Revit BIM files for cost analysis, quantity tracking, and project planning. This repetitive process is error-prone, time-consuming, and limits scalability across multiple projects.

This automation workflow solves that problem by automatically converting .rvt files into open, analysis-ready Excel databases (XLSX) with optional DAE/PDF outputs. Using the RvtExporter.exe from the DDC Revit toolkit, it transforms complex BIM data into structured formats that can be immediately used for estimation, reporting, and integration with other construction management systems.

The workflow provides a simple Form UI for setting paths and export flags, then executes the conversion with safe defaults. It handles multiple export modes (basic, standard, complete) and optional features like bounding box extraction, Revit schedules, or PDF drawings generation.

How It Works

Step 1: Form Configuration

A user-friendly form interface allows project managers or engineers to specify the Revit converter path, select the .rvt file, choose export mode, and set optional conversion flags. This eliminates command-line complexity and makes the tool accessible to non-technical team members.

Step 2: Automated Execution

The workflow launches RvtExporter.exe with the configured parameters, executing the conversion process automatically. It handles Windows command execution, monitors for completion, and manages any errors that occur during the conversion process.

Step 3: Output Generation

Upon successful conversion, the workflow generates *_rvt.xlsx and *_rvt.dae files in the specified output folder. These structured outputs are immediately ready for analysis, reporting, or integration with other construction software systems.

What You'll Need

  1. A Windows host (the DDC Revit toolkit is a Windows executable)
  2. DDC Revit toolkit installed (typically at C:\DDC_Converter_Revit\datadrivenlibs\RvtExporter.exe)
  3. Local .rvt files accessible to the workflow
  4. Basic understanding of Revit file structure and export requirements
  5. n8n instance (self-hosted or cloud) to run the automation

Pro tip: Test the conversion with a small sample .rvt file first to verify output formats match your analysis requirements. This ensures the automation delivers exactly what your team needs before scaling across all projects.

Quick Setup Guide

  1. Import the downloaded JSON workflow into your n8n instance
  2. Open the Form UI trigger (ensure pop-ups are allowed in your browser)
  3. Fill in the converter path: C:\DDC_Converter_Revit\datadrivenlibs\RvtExporter.exe
  4. Specify your Revit file path (e.g., C:\Sample_Projects\your_project.rvt)
  5. Select export mode: basic (309 categories), standard (724), or complete (1209)
  6. Choose optional conversion flags: bbox, schedule, sheets2pdf, -no-xlsx, -no-collada
  7. Set output folder if different from default
  8. Click Submit – the workflow automatically executes the conversion
