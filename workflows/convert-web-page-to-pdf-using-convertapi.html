Convert Web Page to PDF using ConvertAPI

Automatically convert any webpage to a PDF document and store it locally—eliminate manual saving and ensure consistent formatting every time.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manually saving web pages as PDFs is a tedious, error-prone task that wastes valuable employee time. This workflow automates the entire process: it takes a webpage URL, converts it to a high-quality PDF using the ConvertAPI service, and saves the resulting file to your local file system or cloud storage.

The automation solves several business problems: it ensures every PDF is generated with consistent formatting (margins, page size, headers), creates reliable archives for compliance or record-keeping, and eliminates the risk of human error when saving important documents like invoices, reports, or confirmation pages.

Whether you need to archive weekly reports, generate printable versions of dashboards, or save customer receipts from web portals, this workflow handles it automatically on a schedule or trigger.

How It Works

Step 1: Trigger the Conversion

The workflow can be triggered manually, on a schedule (e.g., daily at 9 AM), or by an external event like a new database entry. You provide the target webpage URL.

Step 2: Call ConvertAPI

An HTTP Request node sends the URL to ConvertAPI's conversion endpoint with your API token for authentication. You can customize parameters like paper size, margins, and conversion delay for JavaScript-heavy pages.

Step 3: Receive and Process the PDF

ConvertAPI returns the PDF file as binary data. The workflow extracts this data and prepares it for storage.

Step 4: Save the File

A Write File node saves the PDF to a specified local directory with a filename based on the URL and timestamp. You can easily modify this to save to cloud storage like Google Drive or Dropbox.

What You'll Need

  1. A running n8n instance (cloud or self-hosted).
  2. A ConvertAPI account (free tier available) and API token.
  3. Access to a file system or cloud storage destination for saving PDFs.
  4. The URLs of webpages you want to convert (public or authenticated).

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Configure the HTTP Request node with your ConvertAPI token (add as Bearer token in Authorization header).
  3. Set the target URL parameter to the webpage you want to convert.
  4. Adjust output settings like paper size, margins, or orientation if needed.
  5. Set the save location in the Write File node to your desired directory.
  6. Test and activate the workflow to generate your first automated PDF.

Pro tip: Use n8n's schedule trigger to run this workflow daily or weekly, automatically archiving important web pages like pricing pages, policy updates, or dashboard exports without any manual intervention.
