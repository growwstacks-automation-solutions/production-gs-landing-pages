What This Workflow Does

Manual scheduling creates bottlenecks. Clients email to ask "When are you available?", you check your calendar, they suggest times that don't work, and the back-and-forth wastes valuable time. This free n8n template solves that by turning your Calendly schedule into a programmable API endpoint.

The workflow creates a RESTful API that returns real-time availability data from your Calendly account. Send a simple POST request with optional parameters, and receive structured JSON showing all available time slots, next available appointment, and booking URLs—perfect for integrating into custom applications.

Instead of forcing clients to use Calendly's embedded forms, you can now build booking experiences that match your brand, integrate scheduling into chatbots, display availability on your website, or check slots programmatically before sending meeting invitations.

How It Works

The automation follows a logical sequence to fetch, process, and return availability data from Calendly's API.

1. Webhook Trigger & Configuration

The workflow starts with a webhook node that listens for incoming POST requests. It accepts optional parameters like event_type_uri (to check specific event types) and days_ahead (how far into the future to check). Default values are set if parameters aren't provided.

2. Calendly Authentication

Using OAuth2 credentials, the workflow authenticates with Calendly's API to access your scheduling data. This secure connection ensures only your account's availability is checked and returned.

3. User & Event Type Discovery

The system retrieves your Calendly user information and lists all active event types (15-minute consult, 30-minute demo, etc.). If a specific event type URI was provided in the request, it uses that; otherwise, it selects the first available event type.

4. Availability Check

The workflow queries Calendly's availability API for the specified date range (default: next 7 days). It calculates start and end times, formats the request properly, and fetches all available slots for the selected event type.

5. Data Processing & Formatting

Raw availability data is transformed into a clean, structured format. Time slots are converted to human-readable formats, grouped by day, and enriched with direct booking URLs. The system also calculates summary statistics like total available slots and the next available appointment.

6. JSON Response Delivery

A comprehensive JSON response is returned containing user information, event type details, availability summary, all time slots (with formatted times and booking URLs), and metadata about when the check was performed.

What You'll Need

  1. A Calendly account with at least one active event type configured
  2. n8n instance (self-hosted or n8n.cloud) with webhook access
  3. Calendly OAuth2 API credentials (created in Calendly's integration settings)
  4. Basic understanding of APIs to integrate the endpoint into your applications
  5. Optional: Custom domain if you want to expose the API publicly with your own branding

Pro tip: Before setting up the OAuth credentials in Calendly, decide which scopes you need. For just checking availability, you typically need read:event_types and read:availability permissions. If you'll also create bookings through the API later, include write:event scope too.

Quick Setup Guide

Get your Calendly availability API running in under 15 minutes:

  1. Download the template using the button above and import it into your n8n instance
  2. Create Calendly OAuth credentials at calendly.com/integrations → API & Webhooks
  3. Add the Calendly credential in n8n and authenticate with your account
  4. Activate the workflow to generate your unique webhook URL
  5. Test the endpoint using Postman, curl, or directly from your application with a POST request
  6. Customize parameters like default days ahead or event type selection logic if needed

The webhook URL generated by n8n becomes your API endpoint. You can call it from any application that can make HTTP requests.
