How It Works

The workflow acts as a centralized authentication service that your applications communicate with via HTTP requests.

1. Webhook Trigger & Routing

A webhook node listens for POST requests at your designated endpoint. Each request must include an "action" parameter (signup, login, or forgot) that determines which authentication path to follow. The workflow validates incoming data and routes requests to the appropriate authentication logic.

2. User Registration (Signup)

When a new user registers, the workflow validates their email format, checks for existing users with the same email (case-insensitive), and securely hashes their password using bcrypt before storing it in PostgreSQL. The system generates a unique user ID and timestamp, returning the new user's details (excluding password) in the response.

3. Secure Login Verification

For login attempts, the workflow queries the database for the user's email (case-insensitive comparison), retrieves the stored password hash, and uses bcrypt to verify the provided password matches. If credentials are valid, user information is returned; if not, appropriate error messages help users understand what went wrong.

4. Password Reset Flow

When users forget their password, the workflow generates a cryptographically secure temporary password, hashes it, updates the user's record in the database, and returns the temporary password in the response. In production, you would extend this to email the temporary password securely rather than returning it directly.

5. Response Standardization

All authentication outcomes return consistent JSON responses with status codes, clear messages, and relevant user data. This standardization makes integration with frontend applications predictable and error handling straightforward across your entire application ecosystem.

  2. PostgreSQL database – Any PostgreSQL instance (local, cloud, or Supabase)
  3. Database extensions – uuid-ossp and pgcrypto enabled in your PostgreSQL database
  4. Users table – Create the required table structure (SQL provided in setup)

Quick Setup Guide

Follow these steps to implement this authentication system in your project:

  1. Prepare your database – Run the provided SQL script to create the users table with proper constraints and indexes.
  2. Configure PostgreSQL credentials – Add your database connection details to the n8n PostgreSQL nodes in the workflow.
  3. Import the workflow – Download the JSON file and import it into your n8n instance.
  4. Activate the webhook – Enable the workflow and note your unique webhook URL.
  5. Test the endpoints – Use Postman or curl to send test requests for signup, login, and password reset.
  6. Integrate with your app – Replace your current authentication calls with requests to your new webhook endpoint.
  7. Extend as needed – Add email verification, social login, or multi-factor authentication based on your requirements.
