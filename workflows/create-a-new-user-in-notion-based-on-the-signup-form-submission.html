What This Workflow Does

This automation solves the tedious manual process of adding new users to Notion whenever you receive form submissions. Whether you're onboarding employees, registering students, or managing client access, manually creating each user account wastes valuable time and risks data entry errors.

The workflow automatically converts form submissions into properly configured Notion user accounts. It captures essential details like name and email from the form, checks for existing users to prevent duplicates, and creates new accounts with the correct permissions. For educational institutions, it can also associate users with the current semester or course cohort.

How It Works

1. Form Submission Trigger

The workflow activates when your form platform (Typeform, Google Forms, etc.) sends submission data via webhook. This captures all form responses in real-time without polling.

2. Data Validation

Essential fields are verified—ensuring valid email formats, required name fields, and any other critical information needed for user creation.

3. Notion User Check

The system queries your Notion workspace to determine if the submitted email already exists, preventing duplicate account creation attempts.

4. User Creation

For new users, the workflow creates a Notion account with the provided details, assigning appropriate workspace permissions and properties.

5. Semester Tagging (Education Use)

For academic applications, the workflow can automatically tag new users with the current semester or academic period for easy cohort management.

Notion identifies users by their email address, preventing true duplicates. However, automation workflows should include logic to check for existing users before attempting creation to avoid unnecessary API calls. The template includes this verification step, only creating accounts for genuinely new email addresses while updating existing records if needed.

This is particularly important for forms that might be resubmitted or for users who register for multiple programs. The workflow intelligently handles these cases by first checking the Notion API for existing users matching the submitted email, then either creating a new account or updating the existing one with any new information from the form.

  • Prevents redundant API operations
  • Maintains single source of truth
  • Updates existing records with new data
