What This Workflow Does

This automation solves the challenge of organizing and searching through documents received via messaging apps. Instead of manually saving, renaming, and cataloging each file, the workflow automatically processes documents sent to a Telegram chat or channel.

The system uploads original files to Google Drive for secure storage, uses optical character recognition (OCR) to extract text from images and PDFs, then creates searchable records in Airtable with all document metadata. This creates a centralized, queryable archive without manual data entry.

How It Works

1. Document Capture

When a file is sent to your designated Telegram chat, the workflow triggers automatically. It supports multiple file types including PDFs, images (JPG/PNG), and office documents.

2. Secure Storage

The original file uploads to a specified Google Drive folder with timestamped naming. Folder structure can be customized by date, project, or other parameters.

3. Text Extraction

For image-based files, OCR technology converts the visual text into machine-readable content. PDFs are processed to extract both text and metadata.

4. Database Creation

Extracted text along with document details (source, date, file type) are added to Airtable as a searchable record, with a link back to the original Drive file.
