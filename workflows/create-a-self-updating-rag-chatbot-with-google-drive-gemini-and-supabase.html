What This Workflow Does

This automation template solves a common business problem: employees or customers asking repetitive questions about company policies, technical documentation, or project details that are buried in documents. Instead of manually searching through Google Drive folders or relying on human support, this workflow creates an intelligent AI agent that answers questions instantly using your actual documents as its source.

The system is "self-updating"—when you add a new file to a designated Google Drive folder, the workflow automatically processes it, extracts the text, converts it into AI-readable embeddings, and stores them in Supabase. When a file is deleted, the system cleans up the corresponding data. This ensures your chatbot's knowledge base is always current without any manual intervention.

Beyond simple question-answering, this RAG (Retrieval-Augmented Generation) setup provides context-aware, accurate responses grounded in your company's specific information, reducing misinformation and improving decision-making speed.

How It Works

The workflow is built as three interconnected systems that operate seamlessly.

1. The RAG Agent (Chatbot Core)

When a user submits a question, the workflow queries the Supabase vector store to find the most relevant document snippets from your Google Drive content. It uses a Cohere reranker to improve accuracy, then passes the retrieved context along with the question to Google Gemini AI, which generates a final, nuanced answer. Conversation history is maintained in a Postgres database, allowing for contextual follow-ups.

2. The Ingestion Pipeline (Auto-Learning)

This system monitors your Google Drive folder for new or updated files. Upon detection, it determines the file type (Google Doc or PDF), extracts the text content, splits it into manageable chunks, generates embeddings via Gemini, and stores them in Supabase. This process happens automatically, turning static documents into dynamic AI knowledge.

3. The Cleanup System (Knowledge Hygiene)

A scheduled task runs periodically to compare the list of files in Google Drive with the stored embeddings in Supabase. Any embeddings corresponding to deleted files are removed, preventing the chatbot from referencing outdated or removed information. This maintains a clean, accurate knowledge base.

What You'll Need

  1. n8n Instance: A running n8n environment (cloud or self-hosted).
  2. Google Drive Account: With OAuth2 credentials configured in n8n, and a specific folder designated as the knowledge source.
  3. Supabase Project: A Supabase instance with a vector-enabled PostgreSQL database and API key.
  4. Postgres Database: For storing conversation history metadata (can be within Supabase or separate).
  5. Google Gemini API Key: From Google AI Studio, for generating embeddings and answering questions.
  6. Cohere API Key: For reranking retrieved snippets to improve answer relevance.

Quick Setup Guide

Import and configure this template in under 30 minutes.

  1. Download & Import: Download the JSON template file and import it into your n8n workspace.
  2. Configure Credentials: In n8n's credential manager, set up connections for Google Drive (OAuth2), Supabase (API Key), Postgres, Google Gemini, and Cohere.
  3. Set Your Drive Folder: In the "Search files and folders" node, replace the placeholder folder ID with the ID of your target Google Drive folder.
  4. Prepare Databases: Ensure your Supabase and Postgres instances have the required tables (documents for vectors, document_metadata for tracking).
  5. Activate Workflows: Turn on the schedule trigger for the cleanup system and the webhook/trigger for the RAG agent. Test with a sample question.
