What This Workflow Does

This automation solves a common problem for product teams and developers: capturing events from external services into your analytics platform without writing custom code. When you use third-party tools like payment processors, CRMs, or marketing platforms, they often send webhook notifications about important events. These events contain valuable data about user behavior, but they typically get lost in server logs instead of being tracked in your analytics.

The workflow automatically converts any incoming webhook request into a structured PostHog event. This means you can track Stripe payments, Typeform submissions, Slack interactions, or any other webhook-triggered action as part of your product analytics. You get a complete picture of user journeys across all touchpoints, not just within your own application.

By connecting webhooks directly to PostHog, you eliminate manual data entry, ensure no events are missed, and create a unified analytics pipeline. This is particularly valuable for SaaS companies, e-commerce platforms, and any business that needs to understand how users interact with multiple systems.

How It Works

The workflow follows a simple but powerful sequence that transforms raw webhook data into actionable analytics.

Step 1: Webhook Trigger

The n8n Webhook node creates a unique URL endpoint that listens for incoming HTTP requests. When any service sends data to this URL (like when a payment succeeds or a form is submitted), the workflow automatically triggers. The node captures the entire request payload, headers, and metadata.

Step 2: Data Parsing & Transformation

Before sending data to PostHog, the workflow can optionally transform the webhook payload. This might include extracting specific fields, converting data formats, adding timestamps, or enriching events with user identifiers. You can use n8n's built-in functions to manipulate JSON, parse dates, or merge data from other sources.

Step 3: PostHog Event Creation

The PostHog node takes the processed data and creates a new event in your PostHog project. You can specify the event name (like "payment_received" or "form_submitted"), set user properties, add custom metadata, and include the full webhook payload for debugging. The event appears in PostHog's dashboard immediately, ready for analysis.

Step 4: Error Handling & Logging

The workflow includes error handling to ensure reliability. If PostHog is unavailable or returns an error, the workflow can retry, send notifications, or log the failure. This prevents data loss and provides visibility into the automation's health.

Quick Setup Guide

Follow these steps to implement this automation in under 10 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Webhook node by copying the unique URL it generates.
  3. Set up your PostHog credentials in n8n using your project API key and host URL.
  4. Test the connection by sending a sample webhook to your n8n endpoint.
  5. Connect your actual service by pasting the webhook URL into your third-party tool's settings.
  6. Customize event naming in the PostHog node to match your analytics taxonomy.
  7. Activate the workflow and verify events appear in your PostHog dashboard.

Pro tip: Use distinct event names for different webhook sources. For example, "stripe_payment_succeeded" and "typeform_submission_completed" instead of generic names. This makes segmentation and analysis much easier in PostHog.
