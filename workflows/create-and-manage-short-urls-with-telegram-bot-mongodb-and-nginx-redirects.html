What This Workflow Does

This automation solves the problem of cumbersome URL management by providing a complete self-hosted URL shortening system controlled through Telegram. Instead of relying on third-party services with limitations, branding issues, or privacy concerns, you can create branded short links instantly via chat.

The workflow enables team members to send any long URL to your Telegram bot, which then generates a custom short code (or suggests one), stores the mapping in your MongoDB database, and makes the shortened link immediately available. When users click the short URL, Nginx handles the redirection at lightning speed while the system tracks analytics.

This is particularly valuable for marketing teams needing branded tracking links, support teams sharing documentation, or any business that frequently shares URLs internally or externally and wants full control over the data and user experience.

How It Works

1. Telegram Bot Interaction

Users send a long URL to your Telegram bot. The bot responds asking if they want a custom short code or an auto-generated one. This conversational interface makes URL shortening accessible from any device without logging into a dashboard.

2. MongoDB Storage

The workflow validates the URL, checks for duplicates, and stores the long URL-short code mapping in MongoDB. Each document includes metadata like creation timestamp, creator ID, and initializes click counters for analytics.

3. Short Link Generation

The system generates the complete short URL using your custom domain (e.g., s.yourdomain.com/abc123) and sends it back through Telegram. The link becomes active immediately as Nginx is configured to forward requests to your n8n webhook.

4. Nginx Redirection

When someone accesses the short URL, Nginx proxies the request to a dedicated n8n webhook. The workflow looks up the short code in MongoDB, retrieves the original URL, and returns a 301 redirect. For high performance, frequently accessed links can be cached at the Nginx level.

Pro tip: Set up Nginx caching for your most popular short links to reduce database load and improve redirect speed to under 10ms.

Quick Setup Guide

Follow these steps to deploy your custom URL shortener:

  1. Import the workflow: Download the JSON template and import it into your n8n instance.
  2. Configure credentials: Add your Telegram bot token and MongoDB connection details in n8n credentials.
  3. Set environment variable: Create a SHORTENER_DOMAIN variable (e.g., "https://s.yourdomain.com").
  4. Prepare MongoDB: Create a database and collection for storing URL mappings.
  5. Configure Nginx: Set up your domain to proxy requests to the n8n webhook endpoint.
  6. Activate workflows: Enable both the Telegram creator and redirect handler workflows.
  7. Test the system: Send a URL to your Telegram bot and verify the short link works.

Important: This workflow contains community nodes that are only compatible with self-hosted n8n. Cloud n8n users will need to adapt the nodes or use the self-hosted version.
