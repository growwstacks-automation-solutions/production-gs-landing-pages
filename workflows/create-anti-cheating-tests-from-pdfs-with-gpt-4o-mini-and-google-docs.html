Create anti-cheating tests from PDFs with GPT-4o-mini and Google Docs

What This Workflow Does

This automation solves the time-consuming challenge of creating multiple test versions to prevent cheating in educational and training environments. Traditional test creation requires instructors to manually develop equivalent questions, a process that can take hours per assessment. The workflow automates this by leveraging AI to generate distinct but equally challenging test versions from your source materials.

The system extracts content from uploaded PDFs (textbooks, study guides, or lecture notes), processes the material through GPT-4o-mini to create two different test versions, formats them professionally in Google Docs, and automatically distributes them to students via email. This ensures fair testing conditions while saving educators significant preparation time.

How It Works

1. PDF Content Extraction

The workflow begins by extracting text and structured content from uploaded PDF files. It identifies key concepts, definitions, and important information that typically form the basis of test questions.

2. AI-Powered Question Generation

GPT-4o-mini analyzes the extracted content to generate two sets of questions covering the same material but phrased differently. The AI ensures both versions maintain equivalent difficulty levels and comprehensive coverage of the source material.

3. Google Docs Formatting

The generated questions are automatically formatted into professional test documents using Google Docs templates. This includes consistent styling, numbering, spacing, and section headers for a polished appearance.

4. Automated Distribution

Completed tests are delivered via email with secure links to the Google Docs. The system can randomize version assignments or distribute specific versions based on class lists or other criteria.
