What This Workflow Does

Content teams often struggle with the manual process of transferring content from planning tools like Airtable to publishing platforms like WordPress. This repetitive work consumes valuable time that could be spent on content creation and strategy. The process typically involves copying and pasting text, manually uploading images, setting metadata, and scheduling posts—tasks that are prone to human error and inconsistency.

This automation workflow solves this problem by creating a seamless bridge between Airtable (where you plan and organize content) and WordPress (where you publish it). When you add or update content in your Airtable dashboard, the workflow automatically converts markdown to HTML, publishes the post to WordPress with proper formatting, uploads featured images, and sets all necessary metadata—all without manual intervention.

The result is a streamlined publishing process that reduces administrative overhead by 80% or more. Content teams can focus on creating great content rather than managing the mechanics of publishing, while ensuring every post meets your brand standards and technical requirements.

How It Works

The workflow follows a logical sequence that mirrors how content teams think about publishing, but executes it automatically in the background.

Step 1: Content Retrieval from Airtable

The workflow monitors your Airtable base for new or updated records. When content is marked as "ready for publishing" or reaches a specific status, it retrieves the title, body content (in markdown format), metadata, and image URLs. This step ensures your editorial workflow in Airtable drives the automation process.

Step 2: Content Transformation

Since Airtable typically stores content in markdown for flexibility, the workflow converts this to HTML format suitable for WordPress publishing. It applies your standard formatting rules, handles code blocks, lists, and other structural elements consistently every time.

Step 3: WordPress Post Creation

The transformed content is published to WordPress via the REST API. The workflow sets the post title, content, author, categories, tags, excerpt, and publication status (draft, published, or scheduled). It handles all the API authentication and error checking automatically.

Step 4: Image Processing and Attachment

Featured images and inline media are downloaded from their source URLs (which could be in Airtable attachments, cloud storage, or other locations), uploaded to the WordPress media library, and properly attached to the post. The workflow sets the featured image and can resize or optimize images based on your requirements.

Step 5: Confirmation and Logging

Once the post is successfully published, the workflow updates the Airtable record with the WordPress post URL, publication date, and status. This creates a complete audit trail and keeps your content planning system synchronized with what's actually published.

Pro tip: Use Airtable's collaborative features for content review and approval before automation publishes to WordPress. This maintains editorial control while automating the technical publishing steps.

What You'll Need

  1. Airtable Account: With a base containing your content (title, body, metadata, image URLs).
  2. WordPress Website: With the REST API enabled and appropriate authentication credentials (Application Password or similar).
  3. n8n Instance: Either self-hosted or using n8n.cloud, with webhook capabilities.
  4. Content Structure: Consistent field naming in Airtable that maps to WordPress post fields.
  5. Image Hosting: Images accessible via URL (could be in Airtable attachments, cloud storage, or CDN).

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download and Import: Download the template file and import it into your n8n instance using the workflow import feature.
  2. Configure Airtable Connection: Add your Airtable API key, base ID, and table name to the Airtable node. Test that it can read your content records.
  3. Set Up WordPress Credentials: Configure the WordPress node with your website URL and authentication details. Create an Application Password in WordPress if needed.
  4. Map Your Fields: Adjust the field mappings between Airtable columns and WordPress post fields. Ensure titles, content, categories, and tags map correctly.
  5. Test with a Draft: Run the workflow once with a test record, publishing as a draft first to verify formatting and image handling work correctly.
  6. Activate the Trigger: Set the schedule trigger (how often to check for new content) or configure a webhook from Airtable to trigger the workflow automatically.
  7. Monitor and Refine: Check the first few automated publications, then refine field mappings or formatting rules as needed for your specific content.

Pro tip: Start with a small subset of content and publish as drafts. Review the results, adjust your mappings, then scale up to full production publishing.
