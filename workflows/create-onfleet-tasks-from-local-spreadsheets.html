What This Workflow Does

This automation solves a common pain point for delivery businesses: manually transferring order data from spreadsheets into Onfleet. Dispatchers often waste hours each week copying customer addresses, delivery instructions, and contact information—a process prone to errors that lead to failed deliveries and customer complaints.

The workflow automatically reads delivery data from your local spreadsheet files (Excel, CSV, Google Sheets exports) and creates corresponding tasks in Onfleet with all necessary details. It handles address formatting, phone number validation, and can be scheduled to run daily or triggered whenever new orders arrive.

Beyond basic import, this template includes error handling for invalid data, logging of successful/failed task creation, and the ability to map custom spreadsheet columns to Onfleet's task fields. It transforms what was a 30-minute manual process into a 30-second automated operation.

How It Works

Step 1: Read Spreadsheet Data

The workflow starts by accessing your local spreadsheet file. It reads all rows containing delivery information, including recipient details, addresses, delivery windows, and special instructions. The template supports multiple file formats and can be configured to process specific sheets or ranges.

Step 2: Validate and Transform Data

Each row of data undergoes validation checks. Addresses are formatted for Onfleet's requirements, phone numbers are standardized, and required fields are verified. Missing or problematic data can be flagged for manual review or automatically corrected using mapping rules you define.

Step 3: Create Onfleet Tasks

Validated delivery records are sent to Onfleet via its API. For each row, a task is created with the complete delivery specification. The workflow handles API rate limits, authentication, and error responses, ensuring reliable task creation even with large datasets.

Step 4: Log Results and Handle Errors

Successful task creations are logged with their Onfleet task IDs for future reference. Failed creations (due to invalid addresses, API issues, etc.) are captured separately with error details, allowing for easy troubleshooting without losing track of problematic orders.

What You'll Need

  1. Onfleet Account: An active Onfleet subscription with API access enabled. You'll need your API key from the Onfleet dashboard.
  2. Spreadsheet Files: Your delivery data in CSV, Excel, or Google Sheets format with consistent column structure.
  3. n8n Instance: Either n8n.cloud account or self-hosted n8n installation (version 0.200+ recommended).
  4. Basic Technical Access: Ability to configure API credentials in n8n and modify file path locations if using local spreadsheets.
  5. Data Structure Knowledge: Understanding of your spreadsheet columns and how they map to Onfleet task fields (recipient, address, notes, etc.).

Pro tip: Before automating your entire delivery pipeline, test with a small sample spreadsheet (5-10 deliveries) to verify your column mappings and ensure addresses format correctly for Onfleet's geocoding system.

Quick Setup Guide

  1. Download and Import: Download the template file and import it into your n8n instance via the workflow import function.
  2. Configure File Source: Update the "Read Binary File" node with the path to your spreadsheet file. For cloud spreadsheets, replace this with the appropriate node (Google Sheets, Airtable, etc.).
  3. Set Up Onfleet Credentials: Add your Onfleet API key to the Onfleet node. Test the connection to ensure proper authentication.
  4. Map Your Data: Adjust the mapping between your spreadsheet columns and Onfleet task fields in the "Set" or "Function" nodes. Match recipient name, address, phone, and delivery notes.
  5. Test with Sample Data: Run the workflow with a small test spreadsheet to verify task creation works correctly before processing your full delivery list.
  6. Schedule or Trigger: Set up a schedule (daily, hourly) or trigger (when new spreadsheet is uploaded) based on your delivery workflow needs.
