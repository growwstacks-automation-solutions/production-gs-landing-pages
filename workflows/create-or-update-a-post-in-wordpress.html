What This Workflow Does

Manually creating and updating WordPress posts is a repetitive, time-consuming task that distracts from content strategy and creation. This automation template solves that by connecting your content sources directly to your WordPress site.

The workflow automatically creates new posts or updates existing ones based on triggers from other applications. Whether you're importing articles from a spreadsheet, syndicating content from another platform, or updating product information, this automation handles the entire publishing process without manual intervention.

Beyond basic posting, it can manage categories, tags, featured images, custom fields, and publication status. This turns WordPress from a manual publishing platform into an automated content hub that integrates seamlessly with your entire tech stack.

How It Works

The automation follows a logical sequence to ensure reliable content publishing:

1. Trigger & Data Collection

The workflow starts when new data arrives from your chosen trigger source. This could be a new row in Google Sheets, a form submission, a webhook from another application, or a scheduled time. The automation captures all necessary content elements: title, body, excerpt, metadata, and image URLs.

2. Content Processing & Formatting

Before sending to WordPress, the content is processed and formatted. This includes converting markdown to HTML, resizing images to optimal dimensions, extracting metadata, and applying your brand's formatting rules. The automation can also enrich content by pulling additional information from other sources if needed.

3. WordPress API Communication

The processed content is sent to WordPress via the REST API. The workflow checks if a post with the same title or custom identifier already exists. If it does, it updates the existing post; if not, it creates a new one. All API authentication is handled securely in the background.

4. Media Handling & Attachment

If the content includes images, the automation downloads them, uploads them to the WordPress media library, and attaches them as featured images or inserts them into the post content at specified locations. This ensures all media is properly hosted on your site.

5. Status Management & Notification

Finally, the workflow sets the post status (draft, publish, or scheduled for future publication) and sends notifications to your team via Slack, email, or other channels. Error handling ensures any issues are immediately flagged for review.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure your WordPress credentials in the WordPress node. Generate an Application Password in your WordPress admin under Users → Your Profile.
  3. Set up your trigger based on where your content originates. This could be a Google Sheets trigger, webhook, or schedule.
  4. Map your data fields to match your WordPress post structure. Ensure titles, content, categories, and tags are correctly mapped.
  5. Test with a single item using the "Execute Workflow" function. Check that the post appears correctly in WordPress.
  6. Activate the workflow and set it to run automatically based on your preferred schedule or trigger.
