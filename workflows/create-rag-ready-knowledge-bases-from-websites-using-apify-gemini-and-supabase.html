Create RAG-ready Knowledge Bases from Websites

Automatically scrape, clean, and embed website content into a searchable vector database for AI chatbots and semantic search.

What This Workflow Does

Building AI chatbots that provide accurate, up-to-date answers requires feeding them your own knowledge base. Manually converting websites into structured, searchable data is tedious and error-prone. This automation solves that by transforming any website into a ready-to-use vector database for RAG (Retrieval-Augmented Generation) systems.

Simply submit a URL and choose your scraping scope—single page, limited site crawl, or full website exploration. The workflow handles the entire pipeline: extracting content with Apify, cleaning out navigation and ads, intelligently chunking text, generating semantic embeddings with Google Gemini, and storing everything in Supabase with vector search capabilities. What used to take days of manual work now happens automatically in minutes.

How It Works

1. URL Submission & Scoping

You provide the target website URL and select one of three scraping modes. The workflow triggers Apify actors optimized for different use cases, balancing depth with resource efficiency.

2. Content Extraction & Cleaning

Apify scrapes the website, following links based on your scope selection. The raw HTML is processed to remove boilerplate content—navigation menus, footers, ads, cookie banners—leaving only the substantive text you want in your knowledge base.

3. Intelligent Text Chunking

The cleaned content is split into optimal chunks (around 800 characters) using markdown-aware segmentation. This preserves logical structure while creating pieces small enough for effective retrieval but large enough to maintain context.

4. Vector Embedding Generation

Each text chunk is sent to Google Gemini's embedding model, converting the semantic meaning into 768-dimensional vectors. These numerical representations capture the conceptual relationships between different pieces of content.

5. Vector Database Storage

The embeddings, along with their source text and metadata, are stored in a Supabase table with pgvector extension enabled. This creates a fully searchable knowledge base where AI systems can find the most relevant information for any query.

Pro tip: Start with single-page or limited-scope scraping to test content quality before running full website crawls. Review the chunks stored in Supabase to ensure your Apify filters are capturing the right content before scaling up.

What You'll Need

  1. Apify account with an API token for web scraping services
  2. Supabase project with PostgreSQL and pgvector extension enabled
  3. Google Gemini API key for generating text embeddings
  4. n8n instance (cloud or self-hosted) to run the workflow
  5. Target websites you have permission to scrape and include in your knowledge base

Quick Setup Guide

  1. Prepare your Supabase database: Create a `documents` table with an embedding column (vector 768). Run the pgvector setup SQL in your Supabase project.
  2. Import the workflow: Download the template and import it into your n8n instance.
  3. Configure credentials: Add your Apify API token to all three "Run Apify Scraper" nodes, then add Supabase and Gemini credentials to their respective nodes.
  4. Test with a small site: Start with a 5-10 page website or single URL to verify content quality and embedding results.
  5. Connect to your AI application: Use the Supabase vector store with LangChain, OpenAI assistants, or custom chatbot implementations.
