What This Workflow Does

This automation solves a common problem for creators, developers, and professionals: maintaining up-to-date profile statistics across platforms. Manually updating your GitHub profile, portfolio website, or README files with current stats (workflows created, repositories starred, followers gained) is time-consuming and easily forgotten.

The workflow automatically fetches your latest creator statistics from various sources, processes the data, generates a visually appealing SVG badge, and commits it to your GitHub repository. The result is a dynamically updated profile badge that always displays your current achievements, enhancing your professional credibility without manual effort.

Whether you're showcasing your n8n workflow count, GitHub contributions, or other creator metrics, this automation ensures your digital presence remains current and impressive to visitors, collaborators, and potential clients.

How It Works

The automation follows a logical sequence to transform raw data into a beautiful, shareable SVG badge.

1. Scheduled Trigger & Configuration

The workflow begins on a schedule (daily, weekly, or custom interval). A configuration node sets up your creator username, preferred color schemes for the SVG, and aesthetic options like icon colors, border styles, and card backgrounds.

2. Data Collection & Processing

The system fetches your workflows from the n8n API and user details from relevant platforms. It calculates various statistics including total workflows, complex workflows, approved workflows, pending workflows, total nodes used, and total views. This data processing ensures only relevant, accurate metrics are included in your badge.

3. Image Preparation & SVG Generation

Your profile image is downloaded and prepared for inclusion. The workflow then generates a complete SVG file with your updated stats, formatted according to your configuration. The SVG includes visual elements like icons, progress bars, or numerical displays based on your preferences.

4. GitHub Commit & Deployment

The generated SVG is committed to your specified GitHub repository with a descriptive commit message. Once in your repository, the SVG can be served via GitHub's raw content URL or through a CDN, making it embeddable anywhere that accepts images.

Quick Setup Guide

Follow these steps to implement this automation in your own environment:

  1. Download & Import the template JSON file into your n8n instance using the import functionality.
  2. Configure GitHub Credentials in your n8n instance settings to enable repository commits.
  3. Adjust Schedule Trigger to set your preferred execution frequency (daily is recommended for fresh stats).
  4. Customize Configuration Node with your GitHub username, color preferences, and aesthetic options.
  5. Test Execution manually first to ensure the SVG generates correctly and commits to your repository.
  6. Embed Your Badge using the raw GitHub URL or CDN link in your README, portfolio, or profile.

The template includes detailed comments at each configuration point to guide you through the customization process.
