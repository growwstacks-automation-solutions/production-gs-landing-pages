What This Workflow Does

This automation template solves the problem of generic, hard-to-parse notifications from media servers like Radarr, Sonarr, and Bazarr. By default, these services send basic alerts that lack context and can clutter your Discord channels.

The workflow transforms these notifications into clean, actionable messages with proper formatting, relevant details, and appropriate channel routing. You'll get clear alerts about download completions, failed imports, missing subtitles, and other important events - all formatted for easy reading in Discord.

How It Works

1. Webhook Trigger

The workflow starts when your media server sends a webhook notification to the n8n endpoint. This happens automatically for any configured event in Radarr, Sonarr or Bazarr.

2. Event Processing

The workflow identifies which server sent the notification (Radarr/Sonarr/Bazarr) and categorizes the event type (download complete, import failed, etc.).

3. Message Formatting

Based on the event type, the workflow constructs a clean Discord message with only the most relevant information, using proper formatting and emoji indicators.

4. Channel Routing

Different notification types get routed to appropriate Discord channels - downloads in one channel, failures in another, etc.
