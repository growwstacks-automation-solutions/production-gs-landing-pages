What This Workflow Does

This automation solves the problem of information overload in AI research. Every day, hundreds of new papers are published on arXiv, making it impossible for professionals to stay current manually. This workflow acts as your personal research assistant, systematically fetching, processing, and distributing the latest AI research.

It transforms raw academic papers into actionable intelligence by extracting key insights, storing them in an organized knowledge base, and delivering timely notifications. This eliminates hours of manual reading while ensuring you never miss groundbreaking developments in artificial intelligence, machine learning, and related fields.

How It Works

1. Scheduled Paper Fetching

Every morning at 8:00 AM, the workflow queries arXiv's API for the latest AI research papers. It filters by your specified categories (like cs.AI, cs.LG, cs.CV) and retrieves metadata including titles, abstracts, authors, and PDF links.

2. Deduplication & Filtering

The system checks for previously processed papers and removes duplicates. A time-window filter ensures you only receive recent publications (typically from the last 24 hours), preventing information overload from older papers.

3. Notion Database Creation

For each new paper, a structured page is created in your Notion database. This includes all metadata fields, direct PDF links, and formatted content ready for AI processing. Your research becomes searchable, taggable, and collaborative.

4. AI-Powered Analysis

Gemini AI reads each PDF and generates a comprehensive "Deep Research Summary." This includes key contributions, methodology overview, results significance, and practical implications—transforming technical papers into understandable insights.

5. Telegram Alert Distribution

A formatted message is sent to your Telegram channel with the paper title, brief abstract, PDF link, and Notion page link. This creates an instant mobile notification system for your research feed.

What You'll Need

  1. n8n Instance: Self-hosted or cloud version of n8n
  2. Notion Account: With a database created for research papers
  3. Notion Integration: API token with access to your database
  4. Google Gemini API Key: For AI summarization capabilities
  5. Telegram Bot: Created via BotFather with channel posting permissions
  6. arXiv API Access: No authentication required for basic queries

Pro tip: Start with a narrow arXiv category filter (like just cs.AI) for the first week. Once the workflow is stable, expand to additional categories based on what you find most valuable.

Quick Setup Guide

  1. Import the Template: Download and import the JSON file into your n8n instance
  2. Configure Notion: Create your research database and connect the integration in the "Register to Notion Database" node
  3. Add Gemini API: Insert your Google Gemini API key and select your preferred model in the "Analyze doc" node
  4. Setup Telegram: Add your bot token and chat_id in the "Send a text message" node
  5. Customize arXiv Query: Adjust the HTTP Request node to focus on your specific AI subfields
  6. Test & Schedule: Run a manual test, then enable the scheduled trigger for daily automation
