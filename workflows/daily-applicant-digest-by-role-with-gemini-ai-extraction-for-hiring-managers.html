Daily Applicant Digest by Role with Gemini AI Extraction

Automatically collect, analyze, and summarize job applications by role—delivering clean daily digests to hiring managers.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Hiring managers often drown in a flood of unstructured application emails, spending hours manually reading, extracting details, and grouping candidates by role. This workflow solves that by automating the entire process.

It connects to your Gmail, scans all new job application emails labeled 'applicants' in the last 24 hours, uses Google Gemini AI to extract structured candidate details (name, email, phone, role, experience, skills, location, notice period, summary), groups applicants by their target role, and emails a clean HTML digest table to each hiring manager daily at 6 PM IST. This turns chaos into clarity, giving managers everything they need at a glance.

How It Works

Step 1: Scheduled Trigger

The workflow runs automatically every day at 6:00 PM (Asia/Kolkata time). You can adjust this schedule to match your team's review rhythm—morning, midday, or end-of-day.

Step 2: Fetch Applicant Emails

It queries your Gmail account via API, searching for emails with the label 'applicants' received within the past 24 hours. This ensures only fresh applications are processed.

Step 3: Prepare Email Text for AI

Each email's content is converted to plain text, stripping away HTML formatting and attachments to provide clean input for the AI model, improving extraction accuracy.

Step 4: AI Extraction of Candidate Details

Google Gemini (or OpenAI) analyzes the text and returns structured JSON with key fields: candidate name, contact info, applied role, years of experience, key skills, location, notice period, and a brief AI-generated summary.

Step 5: Assign to Hiring Managers

A mapping table routes each applicant to the correct hiring manager based on the extracted 'role' field. If no mapping exists, a fallback manager receives the digest.

Step 6: Build Role-Specific Digest Tables

Applicants are grouped by manager and role. For each group, a clean HTML table is generated, presenting all new candidates in an easy-to-read format.

Step 7: Send Daily Digest Emails

Each hiring manager receives one consolidated email containing the HTML table summary for their roles. No multiple threads—just one actionable digest.

Pro tip: Centralize your configuration (label name, fallback email, AI model, schedule) in a single 'Set' node at the start of the workflow. This makes future edits and scaling much easier.

What You'll Need

  1. A Gmail account with OAuth2 enabled and a label named 'applicants' created. Set up filters to automatically label incoming job application emails.
  2. A Google Gemini API key (or OpenAI API key) for structured AI extraction. Gemini offers strong performance for text parsing at competitive costs.
  3. An n8n instance—self-hosted or cloud. The workflow is compatible with any n8n deployment.
  4. SMTP credentials (optional) if you prefer to send digest emails via a direct email node instead of the Gmail node.
  5. A mapping of job roles to hiring manager email addresses. Even a simple fallback email ensures digests are delivered.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n workspace.
  2. Configure Credentials: Set up connections for Gmail (OAuth2) and Google Gemini (API key) in the credential management section.
  3. Edit Role Mapping: In the "Assign Manager Emails" node, edit the JavaScript code to map your specific job roles to manager emails.
  4. Adjust Schedule: Change the Cron trigger time if 6 PM IST doesn't fit your workflow. Consider multiple triggers for different regions.
  5. Test with a Mock Application: Send yourself a test email, label it 'applicants', and run the workflow manually to verify extraction and delivery.
  6. Monitor & Optimize: Check workflow logs for any errors, refine the AI prompt if extraction accuracy is low, and expand role mappings as needed.

Pro tip: Add a node to convert PDF/DOCX attachments to text before AI extraction. Many applicants send resumes as attachments only.
