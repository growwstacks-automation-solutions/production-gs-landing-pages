What This Workflow Does

Manually moving contacts from your internal database to your email marketing platform is a repetitive, error-prone task that wastes valuable time. This automation solves that by creating a seamless, daily pipeline between NocoDB (your data hub) and Brevo (your email platform).

The workflow intelligently fetches only new or unimported records, performs essential validation checks, filters out invalid data, and then creates clean, verified contacts in your Brevo lists. It also updates the status of each record in NocoDB as it progresses, providing full transparency and preventing duplicate processing.

By automating this flow, you ensure your marketing campaigns always target an up-to-date audience, improve data hygiene, and free your team to focus on strategy rather than data entry.

How It Works

The workflow operates on a scheduled trigger and processes contacts through a logical, step-by-step pipeline.

1. Scheduled Trigger & Data Fetch

A Schedule Trigger node initiates the workflow daily. It connects to your NocoDB instance via API and queries for records where the status field is marked as "not imported" or similar. This ensures only new contacts are processed.

2. Data Validation & Cleansing

Each contact record passes through validation nodes. These check for required fields (like email and name), validate email format, and can integrate with services to filter out disposable or temporary email addresses. Invalid records are flagged and their status updated in NocoDB accordingly.

3. Contact Creation in Brevo

Validated contacts are sent to Brevo's API. The workflow maps fields from your NocoDB table (e.g., `first_name`, `email`, `company`) to the corresponding attributes in Brevo. Contacts are added to specified lists or segments based on your configuration.

4. Status Update & Logging

Immediately after a successful (or failed) import, the workflow updates the record's status in NocoDB. This creates an audit trail and prevents the same record from being processed again. Error handling ensures any API issues are caught and logged for review.

Pro tip: Use the "Split in Batches" node to process contacts in small groups. This respects API rate limits and makes debugging easier if an issue arises.

What You'll Need

  1. A n8n instance: Either self-hosted or using n8n.cloud.
  2. NocoDB Access: API token and the Project ID & Table ID for your contacts table.
  3. Brevo Account: An API key from your Brevo account with permissions to manage contacts.
  4. Prepared NocoDB Table: Your table should have columns for at least: id, email, first_name, last_name, and a status field (defaulting to "0" for new records).

Quick Setup Guide

Get this automation running in your n8n instance in under 15 minutes.

  1. Download & Import: Click the "Download Template" button above and import the JSON file into your n8n workspace.
  2. Configure Credentials: Set up the NocoDB and Brevo nodes with your respective API keys. The template guides you on where to find these in each platform.
  3. Map Your Fields: In the "Brevo: Create Contact" node, map the field names from your NocoDB table to the expected Brevo contact attributes.
  4. Set Your Schedule: Adjust the Schedule Trigger node to run at your preferred time (e.g., daily at 9 AM).
  5. Test with Sample Data: Activate the workflow and add a test record with a status of "0" to your NocoDB table. Verify it appears in Brevo and the status updates to "imported".
