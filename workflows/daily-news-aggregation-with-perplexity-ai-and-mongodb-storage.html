Daily News Aggregation with Perplexity AI & MongoDB Storage

Automatically collect, summarize, and store daily news with AI—perfect for market intelligence, content monitoring, and competitive analysis.

What This Workflow Does

This automation solves the time-consuming problem of manual news monitoring by creating a fully automated pipeline that collects, processes, and stores daily news articles. Instead of spending hours scanning multiple sources, this workflow runs silently in the background, delivering structured, analyzed content directly to your database.

The system uses Perplexity AI to fetch and intelligently summarize the latest global news based on your specified topics. It then cleans and formats this data before storing it in MongoDB with proper categorization. Finally, it sends a confirmation email with a daily digest—giving you complete visibility without any manual effort.

How It Works

1. Scheduled Trigger

Every morning at 8:00 AM UTC, the workflow automatically activates. This consistent timing ensures you receive fresh news at the start of each business day, making it part of your daily routine without requiring any manual initiation.

2. AI-Powered News Fetching

The Perplexity AI node queries the latest global news using your configured topics and search parameters. It returns not just links but structured information including headlines, summaries, sources, publication dates, and relevance scores—all formatted for immediate processing.

3. Data Transformation

A custom JavaScript function parses and cleans the AI response, converting it into a standardized JSON array. Each news item becomes a separate object with consistent fields, ready for database insertion and future analysis.

4. Database Storage

The workflow iterates through each news article, inserting it into your MongoDB "daily_news" collection. It stores comprehensive metadata including categories, sentiment indicators, and source credibility ratings, creating a searchable historical archive.

5. Aggregation & Notification

After all items are processed, the system aggregates results and sends a summary email via Gmail. This provides immediate confirmation of successful operation and a quick overview of what was collected—maintaining complete operational transparency.

What You'll Need

  1. n8n instance (self-hosted version required for community nodes)
  2. Perplexity AI API access with appropriate credits for daily queries
  3. MongoDB database (cloud or self-hosted) with connection credentials
  4. Gmail account or SMTP service for notification emails
  5. Basic understanding of JSON data structures and API authentication

Pro tip: Start with 2-3 highly specific topics rather than broad categories. "AI regulation in healthcare" yields better results than just "technology news." This reduces noise and increases actionable intelligence from day one.

Quick Setup Guide

  1. Download the template and import it into your n8n instance
  2. Configure the Schedule Trigger node to your preferred timezone
  3. Add your Perplexity API credentials to the AI node
  4. Update the MongoDB connection string with your database details
  5. Set up Gmail authentication for the notification email
  6. Test with a manual trigger before enabling the daily schedule
  7. Review initial results and adjust search parameters as needed
