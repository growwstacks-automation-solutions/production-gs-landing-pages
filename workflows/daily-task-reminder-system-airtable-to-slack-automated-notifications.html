What This Workflow Does

Manual task reminders are a productivity killer. Managers waste valuable morning hours compiling lists, chasing team members, and sending follow-up messages. This creates friction, delays projects, and creates unnecessary stress for everyone involved.

This automated workflow solves that problem by connecting your Airtable task database directly to Slack. Every morning at 9 AM (or your chosen time), it automatically scans your Airtable base for active tasks, filters by status (like "In Progress"), and sends personalized reminders to the appropriate Slack channel. Team members receive clear notifications about what they need to work on, complete with deadlines and priorities.

The system ensures nothing falls through the cracks while freeing up leadership time. Instead of being a taskmaster, you become an enabler—your team gets the information they need automatically, and you gain visibility into what's happening without constant check-ins.

How It Works

The workflow follows a simple but powerful three-step process that runs automatically on a schedule you define.

1. Scheduled Trigger

The automation begins with a Schedule Trigger node configured to run daily at your preferred time (typically 9 AM). This acts as your digital assistant that never forgets to check for pending tasks.

2. Query Airtable for Active Tasks

The workflow connects to your Airtable base using a secure API token and searches for records matching specific criteria. You can filter by status fields (like "In Progress"), due dates, assignees, or project categories.

Configuring the Airtable node to filter tasks by status and other criteria.

3. Format and Send Slack Messages

Each task retrieved from Airtable is formatted into a clear, actionable Slack message. The automation can include task titles, assignees, due dates, priority levels, and even links back to the Airtable record for quick access.

Pro tip: Add conditional logic to only send reminders for tasks due within the next 48 hours, or to escalate overdue tasks to a manager's channel. This prevents notification overload while ensuring critical items get attention.

What You'll Need

  1. Airtable Base: With a table containing tasks, including fields for Title, Assignee, Status, and Due Date at minimum.
  2. Airtable API Token: Generated from your Airtable account with read access to your base.
  3. Slack Workspace: With permissions to create a bot or use an existing app with chat:write permissions.
  4. Slack Channel: Designated for task notifications (e.g., #daily-tasks, #team-updates).
  5. n8n Instance: Either self-hosted or using n8n.cloud with webhook capabilities.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Create Airtable credentials in n8n using your Personal Access Token with data.records:read scope.
  3. Update the Airtable node with your Base ID, Table name, and filter formula (e.g., {Status} = 'In Progress').
  4. Configure Slack credentials using a Bot User OAuth Token with chat:write scope.
  5. Set your Slack channel in the Slack node (format: #channel-name).
  6. Adjust the schedule trigger to your preferred reminder time (default is 9 AM daily).
  7. Test the workflow with a single execution to verify messages appear correctly in Slack.
  8. Activate the workflow and let it run automatically.
