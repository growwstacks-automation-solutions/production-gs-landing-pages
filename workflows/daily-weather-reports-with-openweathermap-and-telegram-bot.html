What This Workflow Does

This automation solves the problem of manually checking weather apps multiple times a day. It connects OpenWeatherMap's comprehensive weather data API with Telegram's messaging platform to deliver scheduled, formatted weather reports directly to your preferred chat.

Businesses and individuals use this workflow for consistent weather monitoring without manual effort. Whether you're planning outdoor work, managing logistics, coordinating events, or simply wanting your morning weather briefing delivered with your coffee, this automation ensures you have the right information at the right time.

The workflow is completely customizable—you can adjust the schedule, location, data points included, and message format. It runs reliably in the background, fetching fresh data and delivering it exactly when needed.

How It Works

The automation follows a clean, three-step process that runs on your specified schedule.

1. Scheduled Trigger

A schedule node initiates the workflow at your chosen intervals—daily at 8 AM, every 6 hours, or whatever cadence fits your needs. This trigger ensures the automation runs consistently without manual intervention.

2. Weather Data Fetch

The workflow calls the OpenWeatherMap API with your configured location parameters. It retrieves current conditions, temperature, humidity, wind speed, precipitation probability, and sunrise/sunset times—all the data points you specify.

3. Message Formatting & Delivery

A function node processes the raw weather data into a human-readable format with emojis and clear organization. This formatted message is then sent via your Telegram bot to your specified chat ID, appearing as a clean, informative update.

Pro tip: Add conditional logic to only send alerts when specific conditions are met—like only notifying about rain if probability exceeds 50%, saving notification noise on clear days.

What You'll Need

  1. OpenWeatherMap API Key: Free tier provides 1,000 calls/day—more than enough for personal use.
  2. Telegram Bot Token: Created via @BotFather in Telegram—takes 2 minutes to set up.
  3. Telegram Chat ID: Your personal or group chat identifier where messages will be sent.
  4. n8n Instance: Either n8n.cloud (hosted) or self-hosted n8n installation.
  5. Location Details: City name, optionally with country/state code for precise forecasting.

Quick Setup Guide

Follow these steps to implement this weather automation in under 15 minutes:

  1. Import the template into your n8n workspace using the download button above.
  2. Create credentials for OpenWeatherMap (sign up at openweathermap.org/api) and Telegram (via @BotFather).
  3. Configure the schedule node with your preferred timing using cron syntax (default: daily at 8 AM).
  4. Set your location in the OpenWeatherMap node using "City,Country" format.
  5. Add your Telegram chat ID to the send message node (find it via @userinfobot).
  6. Test the workflow manually once, then activate it to run automatically.

Pro tip: Start with the free OpenWeatherMap tier to test, then upgrade only if you need extended forecasts or more frequent API calls for business use.
