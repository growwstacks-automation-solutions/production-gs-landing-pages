What This Workflow Does

Manual web scraping is time-consuming, fragile, and doesn't scale. This automation solves that by combining Firecrawl's AI-powered extraction with n8n's workflow engine to create a reliable, scheduled data gathering system.

The workflow automatically scrapes specific structured data from target websites every day at your chosen time. It uses Firecrawl's API to intelligently extract information based on your schema and prompts, then formats and delivers the results directly to your Telegram channel or group. If data isn't immediately available, it includes retry logic to ensure completeness.

This transforms what was previously a manual, error-prone task into a fully automated intelligence pipeline that runs while you focus on higher-value work.

How It Works

1. Schedule Trigger

The workflow starts automatically every day at your configured time (default 6 PM). This scheduled trigger ensures consistent, hands-off execution without manual intervention.

2. Firecrawl POST Request

n8n sends a custom extraction request to Firecrawl's API with your target URLs, extraction schema, and natural language prompts. Firecrawl's AI engine processes the pages to extract structured data according to your specifications.

3. Processing Wait & Result Retrieval

The workflow waits 30 seconds for Firecrawl to complete processing, then fetches the extraction results using the returned request ID. This asynchronous approach handles variable processing times reliably.

4. Conditional Retry Logic

An IF node checks whether data was returned. If not, it waits another 15 seconds and retries the fetch. This simple retry mechanism handles temporary processing delays without failing the entire workflow.

5. Data Formatting

A Set node cleans and formats the extracted results into a readable message structure, preparing it for clear presentation in the notification.

6. Telegram Delivery

The formatted data is sent directly to your configured Telegram chat, channel, or group, providing instant visibility to your team without checking emails or dashboards.

What You'll Need

  1. Firecrawl API Key – For AI-powered web extraction (header authentication)
  2. Telegram Bot Token – Created via @BotFather on Telegram
  3. Telegram Chat ID – The channel, group, or chat where notifications should be sent
  4. n8n Instance – Self-hosted or cloud version of n8n
  5. Target URLs – The websites you want to monitor daily
  6. Extraction Schema – Define what data to extract (product names, prices, dates, etc.)

Pro tip: Start with 2-3 critical websites to refine your extraction schema before scaling to dozens of targets. Test different prompt variations with Firecrawl to get the cleanest data structure.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Schedule Trigger to run at your preferred time (daily, hourly, etc.).
  3. Add your Firecrawl API key in the HTTP Request node's headers.
  4. Update the target URLs and extraction schema in the Firecrawl request body.
  5. Set up your Telegram bot and add the token and chat ID to the Telegram node.
  6. Test with a manual trigger to verify data extraction and delivery work correctly.
  7. Activate the workflow and let it run automatically on schedule.
