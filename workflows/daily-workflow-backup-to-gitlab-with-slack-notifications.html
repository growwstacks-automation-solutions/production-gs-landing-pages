What This Workflow Does

This automation solves the critical problem of losing valuable workflow configurations in n8n. Manual backups are easily forgotten, leaving businesses vulnerable to data loss from accidental deletions, system failures, or human errors. The workflow automatically creates daily backups of your n8n workflows and stores them securely in GitLab.

The solution goes beyond simple backup by adding Slack notifications that confirm successful backups and alert you immediately if any issues occur. This creates a reliable safety net for your automation infrastructure while keeping your team informed about backup status without manual checking.

How It Works

1. Scheduled Daily Trigger

The workflow activates automatically at noon each day (configurable to any time). This scheduled trigger ensures backups happen consistently without requiring manual initiation.

2. n8n Workflow Export

The system connects to your n8n instance via API and exports all workflow configurations in JSON format. This captures the complete state of your automations including nodes, connections, and settings.

3. GitLab Repository Update

Each exported workflow gets committed to your designated GitLab repository with a timestamped filename. GitLab's version control maintains a complete history of all changes, allowing rollback to any previous version if needed.

4. Slack Notification

After each successful backup, the workflow sends a confirmation message to your Slack channel with details about the backup. If any errors occur during the process, immediate alerts notify your team for quick resolution.
