What This Workflow Does

This n8n workflow automates the deployment and management of Docker InfluxDB instances for WHMCS or WISECP hosting platforms. It serves as an API backend that receives commands from your billing system, connects via SSH to target servers with Docker installed, and executes predefined actions like container creation, configuration, and maintenance.

For hosting businesses offering InfluxDB monitoring as a service, manual deployment is time-consuming and error-prone. This automation eliminates repetitive SSH sessions, ensures consistent configurations across customer environments, and enables self-service provisioning through your existing billing interface. The workflow handles the entire lifecycle from initial Docker Compose generation to nginx proxy setup and ongoing container management.

How It Works

The automation follows a structured pipeline from API trigger to server execution.

1. API Webhook Receives Commands

WHMCS/WISECP modules send authenticated API requests to the n8n webhook endpoint. These requests contain action parameters (create, update, delete) along with customer-specific configuration like domain names and storage paths.

2. Parameter Processing & Validation

The workflow validates incoming parameters, sets default values for optional settings, and prepares environment-specific variables. Critical parameters include server domain, client directory paths, and mount points for persistent storage.

3. Docker Compose Generation

Based on the action type, n8n generates tailored Docker Compose configuration files. These include InfluxDB version, port mappings, volume mounts, and environment variables. The templates are dynamic, adapting to different customer requirements while maintaining security standards.

4. Secure SSH Execution

Using credential management, the workflow establishes SSH connections to target servers. It executes bash scripts to deploy Docker Compose files, manage containers, configure nginx reverse proxies, and set up firewall rules.

5. Response & Logging

After execution, the workflow captures command outputs, formats JSON responses for the calling system, and logs all activities for auditing. Success/failure notifications can be routed to monitoring systems or support teams.

What You'll Need

  1. n8n Instance: Self-hosted n8n server or n8n cloud account with webhook access
  2. Target Servers: Linux servers with Docker and Docker Compose installed, accessible via SSH
  3. SSH Credentials: Key-based authentication with appropriate sudo privileges for Docker commands
  4. WHMCS/WISECP Module: Custom module or existing integration capable of making API calls
  5. Basic Auth or API Keys: For securing the webhook endpoint between your billing system and n8n

Pro tip: Start with a staging server to test the automation before production deployment. Use separate SSH keys with limited permissions specifically for automation purposes, not root access.

Quick Setup Guide

Follow these steps to implement the automation in your environment.

  1. Import the Template: Download the JSON file and import it into your n8n instance via the workflow import function.
  2. Configure Credentials: Set up SSH credentials for your target servers and Basic Auth credentials for the webhook node.
  3. Adjust Parameters: Update the parameters node with your server domains, directory paths, and any custom Docker or nginx settings.
  4. Test Webhook: Use a tool like Postman or curl to send test API calls to your n8n webhook URL with sample payloads.
  5. Integrate with Billing: Modify your WHMCS/WISECP module to call the n8n API endpoint when services are provisioned or modified.
  6. Monitor Execution: Check n8n execution logs and set up alerts for failed deployments to ensure reliability.
