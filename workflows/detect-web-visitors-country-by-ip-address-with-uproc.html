What This Workflow Does

This automation solves the challenge of not knowing where your website visitors are located geographically. Many businesses operate in multiple countries or need to comply with regional regulations like GDPR, but lack visibility into visitor origins. Without this data, you can't personalize experiences, optimize marketing, or ensure compliance.

The workflow automatically detects a visitor's country (and optionally city) by their IP address using uProc's geolocation API. It then enriches your lead data with this geographic information, which can be used for personalization, analytics, or compliance purposes. The process happens in real-time without requiring any action from the visitor.

How It Works

1. Capture visitor IP address

The workflow starts by capturing the visitor's IP address from your website tracking system, form submission, or analytics tool. This can be triggered by page visits, lead submissions, or other engagement events.

2. Query uProc geolocation API

The IP address is sent to uProc's Location By IP address API, which returns detailed geographic data including country, region, city, timezone, and connection type (mobile, ISP, etc.).

3. Enrich lead records

The geographic data is then appended to your CRM, marketing automation system, or database records. This creates a complete profile with location context for each visitor or lead.

4. Trigger location-based actions

Based on the location data, the workflow can trigger personalized follow-ups, compliance measures, or marketing automation sequences tailored to the visitor's region.

Pro tip: Combine IP geolocation with timezone detection to schedule follow-ups at optimal local times for each lead.

IP geolocation estimates location based on internet infrastructure, while GPS uses satellite signals from devices. IP location works for all web visitors without requiring permissions, but has lower accuracy. GPS provides precise coordinates but requires mobile apps and user consent. Each serves different business needs.

For website analytics and marketing, IP geolocation is typically sufficient. GPS tracking is more appropriate for delivery apps, field service management, or other use cases needing precise real-time location data from mobile users who have opted in.

  • IP works for all web traffic automatically
  • GPS requires app installation and permissions
  • Choose based on your accuracy requirements
