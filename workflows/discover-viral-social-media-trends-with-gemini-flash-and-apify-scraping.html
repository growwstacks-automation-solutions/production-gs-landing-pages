What This Workflow Does

This workflow solves the critical business problem of disconnected market intelligence. Many companies monitor Google Trends for search interest but lack a systematic way to validate if those trends are actually gaining traction on social media. This gap leads to missed opportunities and misallocated marketing resources.

The automation bridges search interest with social engagement by polling Google Trends RSS daily for rising regional queries, then concurrently scraping TikTok, Instagram, and X (Twitter) via Apify actors. It cleans and merges the data, uses Gemini Flash AI to analyze core topics and sentiment, and delivers structured reports directly to Discord. This provides a complete, automated view of viral potential before trends peak.

How It Works

Step 1: Trend Detection

The workflow starts by polling the Google Trends RSS feed for your target country. It extracts rising search queries that show increased interest over the past 24 hours. This provides the initial signal of what people are searching for.

Step 2: Parallel Social Scraping

For each detected query, the workflow triggers three Apify actors simultaneously to scrape TikTok, Instagram, and X (Twitter). These industrial-grade scrapers avoid account bans and extract post counts, engagement metrics, and recent content without risking your personal accounts.

Step 3: Data Aggregation & Cleaning

Custom JavaScript logic merges the disparate data points from each platform, removing duplicates and formatting the information optimally for LLM processing. This step ensures the AI receives clean, structured input.

Step 4: AI Analysis & Insight Generation

Google Gemini Flash analyzes the aggregated data to identify the core topic behind the trend, assess sentiment (positive, negative, neutral), and calculate trend strength based on engagement velocity. It generates a concise report explaining why the trend is emerging.

Step 5: Granular Report Delivery

Individual structured reports for each identified trend are sent via Discord webhooks to your designated channel. Each report includes the search query, social platform metrics, AI analysis summary, and recommended action.

What You'll Need

  2. Google Gemini API Key – Needed for the AI analysis step (available via Google AI Studio).
  3. Discord Server – To receive the automated reports via webhooks.
  4. n8n Instance – Self-hosted or cloud version to run the workflow.
  5. Target Country Code – Configure your regional focus (e.g., US, JP, ID).

Quick Setup Guide

  1. Import the Template – Download the JSON file and import it into your n8n instance.
  2. Configure API Credentials – Add your Apify API Token and Google Gemini API Key in the respective nodes.
  3. Set Up Discord Webhook – Create a webhook in your Discord server and paste the URL into the Discord node.
  4. Adjust Regional Settings – Edit the “Edit Fields” node at the start to set your target country code.
  5. Enable Error Handling – Ensure all scraper nodes are set to “Continue on Fail” to maintain workflow resilience.
  6. Activate Schedule – Set the schedule trigger to run daily (or adjust frequency as needed).
  7. Test & Deploy – Run a manual test to verify data flow, then activate the workflow.

Pro tip: Use the “Continue on Fail” setting on all scraper nodes. If one platform’s scraper temporarily fails (due to rate limits), the workflow will still proceed with data from the other platforms, ensuring you never miss a trend.
