Transform your Google Drive documents into an intelligent, searchable knowledge base with AI-powered chat capabilities

What This Workflow Does

This n8n workflow transforms your static Google Drive documents into a dynamic, intelligent knowledge base using Retrieval-Augmented Generation (RAG) technology. It automatically processes documents as they're added to a designated folder, extracts text (including OCR for images and PDFs), converts the content into searchable vectors using Qdrant, and enables natural language queries through an AI chat interface.

The system solves the common problem of "document silos" where valuable company knowledge gets buried in files. Instead of manually searching through folders, users can ask questions in natural language and receive accurate answers with source references. This is particularly valuable for customer support teams, legal departments, and any business that relies on extensive documentation.

How It Works

1. Document Monitoring & Processing

The workflow monitors a specified Google Drive folder for new or modified documents. When changes are detected, it triggers the processing pipeline. The system handles various file types including PDFs, Word documents, PowerPoint files, and images containing text.

2. Text Extraction & OCR

For native text documents, the content is extracted directly. For image-based files (scanned documents, PDFs with images), the Mistral OCR component converts visual text into machine-readable content. This ensures all document types become searchable.

3. Vector Embedding Generation

The extracted text is processed by an AI model that converts the semantic meaning into numerical vectors (embeddings). These embeddings capture the conceptual relationships between words and phrases, enabling semantic search beyond simple keyword matching.

4. Qdrant Vector Database Storage

The generated embeddings are stored in Qdrant, a high-performance vector search engine. Qdrant organizes the vectors for efficient retrieval, allowing the system to quickly find the most relevant document sections for any query.

5. Natural Language Query Processing

When a user submits a question, the system converts the query into embeddings, searches Qdrant for the most relevant document sections, and uses AI to generate a natural language response that combines information from multiple sources when needed.
