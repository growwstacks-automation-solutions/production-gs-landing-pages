What This Workflow Does

Customer support teams spend hours daily drafting initial responses to common inquiries. This repetitive task delays resolutions and creates inconsistency. This workflow automates the first line of defense by intelligently drafting personalized email replies using your company's actual knowledge base.

It bridges your CRM (Supabase), technical documentation (vector store), and inbox (Outlook) to create context-aware drafts in seconds. When a support email arrives, the system identifies the sender, researches their question against your documentation, and generates a professional, accurate reply ready for agent review.

How It Works

1. Trigger & Contact Enrichment

The workflow monitors your Microsoft Outlook inbox for new messages. When an email arrives, it immediately queries your Supabase CRM to identify the sender—retrieving their name, organization, department, and any relevant history.

2. Context Construction & AI Analysis

A custom code node prepares a comprehensive "chat input" containing the email body, sender metadata, and your specific business rules. An AI Agent (GPT-4o) analyzes the request to determine intent, urgency, and required information.

3. Knowledge Retrieval (RAG)

If technical details are needed, the agent uses Supabase Vector Store to semantically search your product manuals, FAQs, and documentation. It retrieves the most relevant passages to ensure the response is accurate and specific to your products.

4. Draft Creation & Logging

The AI generates an HTML-formatted reply with personalized greetings, accurate solutions, and proper brand tone. This draft is saved to Outlook for manual review and sending. Simultaneously, the interaction is logged to Google Sheets for performance tracking and analytics.

What You'll Need

  1. Microsoft Outlook Business/Office 365 account with API access
  2. Supabase project with:
    • A contacts table for customer lookup
    • Vector-enabled database with document embeddings (for RAG)
  3. OpenAI API key for GPT-4o model access
  4. Google Sheets for execution logging (optional but recommended)
  5. n8n instance (cloud or self-hosted) to run the workflow

Pro tip: Before implementing, audit your most common support questions and ensure your Supabase knowledge base contains clear, up-to-date answers. The AI's quality depends directly on your documentation's quality.

Quick Setup Guide

  1. Import the template by pasting the downloaded JSON into your n8n canvas
  2. Configure credentials for Outlook, OpenAI, Supabase, and Google Sheets in n8n
  3. Update placeholder values:
    • Set the Supabase table name to your actual CRM contacts table
    • Configure the vector store query to match your document embeddings setup
    • Replace the Google Sheets document ID and sheet name with your tracking spreadsheet
  4. Customize business rules in the "Build AI Prompt" code node—replace [YOUR_COMPANY] and [YOUR_NAME] with your actual brand details
  5. Test with sample emails to verify contact lookup, knowledge retrieval, and draft quality before going live
