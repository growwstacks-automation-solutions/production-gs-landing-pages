Dynamic AI Model Selector with GDPR Compliance

Automatically route AI requests through European infrastructure, dynamically select optimal models, and maintain full audit trails—all while staying compliant.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Businesses using AI face two major challenges: complying with data protection regulations like GDPR, and efficiently managing multiple AI models for different tasks. This workflow solves both by creating an intelligent routing system that automatically selects the best AI model for each request while ensuring all data processing stays within GDPR-compliant jurisdictions.

Instead of hardcoding a single AI provider, this automation dynamically queries available models from Requesty's European routing service, presents options to users or systems, logs selections in Google Sheets for audit purposes, and processes requests through the chosen model—all while maintaining full data privacy compliance. It turns AI integration from a technical headache into a strategic advantage.

How It Works

1. Model Discovery & Availability Check

The workflow starts by querying Requesty's API to get real-time availability of AI models. It checks which models are online, their current latency, and any rate limits, creating a dynamic dropdown of options.

2. User/System Selection & Logging

When a request comes in (via webhook, form, or scheduled trigger), the system presents available models. The selection is immediately logged in Google Sheets with timestamp, user context, and request metadata for compliance auditing.

3. GDPR-Compliant Routing

The actual AI request is routed through Requesty's European infrastructure, ensuring data never leaves GDPR-approved jurisdictions. This happens transparently—your workflow sends requests as usual, but they're automatically redirected through compliant channels.

4. Response Processing & Delivery

AI responses are processed, formatted, and delivered back to the requesting system. The workflow can handle retries, fallback models, and error recovery, ensuring reliable operation even if primary models experience issues.

Quick Setup Guide

1. Import the template into your n8n instance using the downloaded JSON file.

2. Configure Requesty credentials in n8n's credential store with your API key from app.requesty.ai.

3. Set up Google Sheets integration by creating a new sheet and sharing it with n8n's service account.

4. Test the model discovery by manually triggering the workflow and verifying available models appear.

5. Connect your trigger—replace the manual trigger with your actual webhook, form, or schedule.

6. Run a compliance check by processing sample data and verifying it appears in your audit sheet.
