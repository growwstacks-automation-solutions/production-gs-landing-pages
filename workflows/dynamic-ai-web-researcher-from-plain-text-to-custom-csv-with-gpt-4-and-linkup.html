What This Workflow Does

This workflow revolutionizes how businesses conduct web research by combining AI-powered natural language processing with automated data collection. Instead of requiring technical expertise to configure web scrapers or manually search for information, users simply describe what they need in plain English.

The system intelligently interprets research requests, identifies relevant data sources, extracts the required information, and formats it into a structured CSV file ready for analysis. This eliminates hours of manual research while ensuring consistent, high-quality data collection.

How It Works

1. Natural Language Processing

GPT-4 analyzes the plain text research request to identify key parameters, data requirements, and optimal sources. It determines what information needs to be collected and how it should be structured.

2. Automated Web Research

The workflow uses Linkup's job search API and other data sources to find relevant web pages containing the requested information. It navigates through search results and identifies the most authoritative sources.

3. Data Extraction & Structuring

Key information is extracted from web pages and processed by GPT-4 to ensure accuracy and consistency. The AI validates data points, removes duplicates, and formats everything according to the original request.

4. CSV Generation

The final output is generated as a well-structured CSV file with appropriate column headers and properly formatted data. This file can be automatically delivered via email or saved to cloud storage.

What You'll Need

  1. An n8n instance (cloud or self-hosted)
  2. OpenAI API access with GPT-4 capability
  3. Linkup API credentials (optional but recommended)
  4. A destination for output files (email, Google Drive, etc.)

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure your OpenAI API credentials in the GPT-4 node
  3. Set up your Linkup API key if using that data source
  4. Define your output destination (email, cloud storage, etc.)
  5. Test with sample research requests and refine as needed
