What This Workflow Does

This automation solves the critical business problem of manual competitor price monitoring, which consumes hours of staff time each week and often misses important price changes. The workflow automatically scrapes competitor product pages using AI-powered technology, extracts current prices and product details, stores this historical data in a Baserow database for trend analysis, and sends real-time Slack alerts when significant price changes occur.

For e-commerce businesses, maintaining competitive pricing is essential for sales and profitability. This template provides a systematic approach to price intelligence, enabling data-driven pricing decisions rather than guesswork. By automating what was previously a manual, error-prone process, businesses can reallocate human resources to strategic activities while gaining superior market visibility.

How It Works

1. Scheduled Trigger & URL Preparation

The workflow begins on a schedule you define (daily, weekly, etc.). A code node loads your list of competitor product URLs or SKUs to monitor, preparing them for processing in manageable batches to avoid overwhelming target websites.

2. AI-Powered Web Scraping

ScrapeGraphAI nodes process each product URL, using artificial intelligence to navigate complex e-commerce pages, handle dynamic content, and extract precise price information along with product names and other relevant details.

3. Data Validation & Processing

Conditional nodes check if valid price data was retrieved, filtering out failed scrapes. The successful data is then structured and prepared for database storage, ensuring clean, consistent records.

4. Historical Storage in Baserow

Each price point is upserted into a Baserow table with timestamp, creating a searchable historical record. This database becomes your single source of truth for price trends over time.

5. Trend Analysis & Alert Generation

The system analyzes recent price movements, calculating percentage changes and identifying significant drops or increases that warrant immediate attention based on your configured thresholds.

6. Real-Time Slack Notifications

When important price changes are detected, formatted alerts are sent to designated Slack channels, enabling your team to respond quickly to market movements with appropriate pricing adjustments.

What You'll Need

  1. n8n instance (self-hosted or n8n.cloud)
  2. ScrapeGraphAI community node installed in your n8n environment
  3. Baserow account with API access and a prepared table for price data
  4. Slack workspace with webhook access for alerts
  5. List of competitor product URLs or SKUs you need to monitor
  6. API credentials for ScrapeGraphAI, Baserow, and Slack configured in n8n

Pro tip: Start with monitoring 5-10 key competitor products that directly impact your sales. Once the workflow is proven, expand to broader product categories. Use environment variables for all API keys to keep your workflow secure and portable.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Install ScrapeGraphAI via the Community Nodes panel if not already available.
  3. Configure credentials in n8n for ScrapeGraphAI, Baserow, and Slack.
  4. Prepare your Baserow table with fields for product_name, product_url, current_price, and scrape_date.
  5. Edit the URL list in the "Prepare URLs" code node with your actual competitor product links.
  6. Adjust the schedule trigger to match your desired monitoring frequency.
  7. Test with a single product to verify the complete data flow from scraping to Slack.
  8. Activate the workflow and monitor initial runs before scaling to full product lists.
