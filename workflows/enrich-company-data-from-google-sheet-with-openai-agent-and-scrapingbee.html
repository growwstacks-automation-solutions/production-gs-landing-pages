What This Workflow Does

Manually researching companies to understand their business, target market, and value proposition is a massive time sink for sales, marketing, and business development teams. This automation solves that problem by turning a simple spreadsheet of company names and websites into a rich, actionable database.

The workflow connects to your Google Sheet, visits each company's website using a scraping service, uses an OpenAI Agent to intelligently analyze the content, and writes back structured insights like business area, offer, value proposition, business model, and ideal customer profile (ICP). It transforms hours of manual work into a fully automated, scalable process.

Beyond just data collection, it includes quality checks—flagging insufficient information or mismatched content—so you can trust the output. This is perfect for building targeted lead lists, market research, competitor analysis, or enriching your CRM data.

How It Works

The workflow is built in n8n and operates in a clear, sequential manner to ensure reliable data enrichment.

Step 1: Trigger & Data Input

The process starts with a Webhook node, but you can easily swap this for a Schedule Trigger to run daily or weekly, or a Google Sheets Trigger for when a new row is added. It fetches your list of companies from a predefined Google Sheet, which must at minimum contain "Company" and "Website" columns.

Step 2: Looping & Scraping

A Loop Over Items node ensures each company is processed individually. For each row, it calls a sub-workflow (the "Scraper Agent") which uses the ScrapingBee node to fetch the HTML content of the company's website. This HTML is then converted into clean Markdown to reduce token usage for the AI analysis.

Step 3: AI-Powered Analysis

The core of the workflow is an AI Agent node. It sends the scraped website content, along with the company name, to an OpenAI Chat Model (like GPT-4). A carefully crafted prompt instructs the AI to extract specific business attributes: Business Area, Offer, Value Proposition, Business Model, and ICP. A Structured Output Parser formats the AI's response into clean, consistent JSON.

Step 4: Data Validation & Output

The AI also assesses the quality of its own analysis, tagging entries as "Sufficient" or "Insufficient" and providing notes if information is missing or the page content seems mismatched. Finally, an Update Row node writes all this enriched data back into the original Google Sheet, completing the loop and creating a powerful, self-updating company intelligence system.

What You'll Need

  1. A Google Sheet with at least two columns: "Company" and "Website".
  2. An n8n instance (cloud or self-hosted).
  3. An OpenAI API key with access to a model like GPT-4.
  4. A ScrapingBee API key (or an alternative web scraping service configured in n8n).
  5. Google Sheets credentials set up in n8n to read from and write to your spreadsheet.

Pro tip: Before running at scale, test the workflow with 2-3 companies to ensure your AI prompt and scraper are yielding the desired results. Adjust the prompt in the AI Agent node to focus on the specific insights most valuable to your business.

Quick Setup Guide

  1. Download & Import: Click the "Download Template" button above and import the JSON file into your n8n instance.
  2. Configure Credentials: In n8n, set up credentials for Google Sheets, OpenAI, and ScrapingBee (or your chosen scraper).
  3. Connect Your Sheet: In the "Get Rows from Google Sheet" node, select your spreadsheet and worksheet.
  4. Adjust the AI Prompt: Review the prompt in the "AI Agent" node to ensure it asks for the business information you need.
  5. Test Run: Use the "Webhook" node's test button or a Manual Trigger to run the workflow for one company and verify the output in your sheet.
  6. Schedule or Trigger: Replace the Webhook with a Schedule Trigger to run automatically (e.g., nightly) or a Google Sheets Trigger to run when new rows are added.
