What This Workflow Does

This automation transforms raw IP addresses into actionable security intelligence. When your systems detect suspicious activity—whether from firewall logs, authentication attempts, or application access—this workflow automatically enriches those IPs with geographic and network data, filters out internal traffic, and delivers contextual alerts to your security team.

Instead of security analysts manually looking up IP addresses in various databases, this n8n workflow automates the entire process. It validates IP formats, checks for private ranges (like 192.168.x.x or 10.x.x.x), enriches public IPs using IP intelligence services, and sends structured notifications to Slack with country, ISP, and severity information.

The workflow serves as a lightweight threat intelligence layer that can be triggered via webhook from any security tool or log source. It's particularly valuable for small to mid-sized security teams who need to prioritize alerts and understand attack patterns without investing in expensive SIEM platforms.

How It Works

1. IP Input & Validation

The workflow receives IP addresses via webhook—either from security tools, API calls, or Slack slash commands. It first validates that the input is a properly formatted IP address and rejects malformed data immediately.

2. Private IP Filtering

Before making external API calls, the workflow checks if the IP falls within private ranges (RFC 1918). Private IPs are logged but not enriched, saving API credits and focusing attention on external threats.

3. IP Enrichment via External Service

Public IPs are sent to an IP intelligence service (like IPinfo or similar providers) that returns geographic data (country, region, city), network information (ISP, organization, ASN), and sometimes threat reputation scores.

4. Data Normalization & Severity Assignment

The enrichment data is structured into a consistent format. The workflow can apply simple rules—like flagging IPs from certain countries as higher severity—to help prioritize investigations.

5. Slack Notification Delivery

Formatted alerts are sent to designated Slack channels with emoji indicators, clickable links to view more details, and all relevant context for immediate security team awareness.

6. Structured Response Return

The workflow returns a clean JSON response that can be consumed by other systems, creating an audit trail and enabling further automation downstream.

Quick Setup Guide

Follow these steps to implement this IP enrichment automation in under 30 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Configure the webhook trigger and note the unique URL provided by n8n
  3. Set up Slack credentials in n8n's credentials management and update the Slack node with your channel name
  4. Obtain an IPinfo API key (free tier offers 50,000 lookups/month) and add it to the HTTP Request node
  5. Test with sample IPs like "8.8.8.8" using curl or Postman to verify the full flow works
  6. Connect your data source by pointing your security tool or logging system to the webhook URL
  7. Activate the workflow and monitor initial alerts to fine-tune formatting and severity rules

Pro tip: Start with the free tier of IPinfo to validate the workflow, then upgrade based on volume. Consider adding a simple database node to log all enriched IPs for trend analysis over time.
