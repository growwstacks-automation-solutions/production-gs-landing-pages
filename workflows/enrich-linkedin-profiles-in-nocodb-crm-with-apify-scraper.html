What This Workflow Does

Manual LinkedIn research is a major bottleneck for sales, recruitment, and marketing teams. Copy-pasting data from profiles into your CRM is slow, error-prone, and leads to inconsistent, outdated records. This n8n automation solves that by connecting Apify's LinkedIn scraper directly to your NocoDB database.

The workflow automatically finds contacts in your NocoDB table that have a LinkedIn URL but are missing enriched profile data. It sends each URL to Apify, which extracts professional details like current role, company, bio, skills, and experience. The cleaned and structured data is then written back into the corresponding NocoDB record, creating a complete, actionable profile.

You move from a process that takes 10-15 minutes per profile manually to one that processes hundreds automatically on a schedule, ensuring your team always works with the latest, highest-quality data.

How It Works

The automation follows a logical, error-resistant sequence to ensure reliable data enrichment.

1. Trigger & Find Records

A Schedule Trigger initiates the workflow monthly (or on your chosen frequency). The first NocoDB node queries your table, finding all records where a LinkedIn URL field is populated but key enrichment fields (like headline or current company) are empty. This targets only records that need updating.

2. Scrape Profile Data

Each LinkedIn URL is sent to the Apify actor via an HTTP Request node. Apify handles the complex scraping, navigating the profile and returning a rich dataset including name, headline, bio, skills, experience, and more, all as structured JSON.

3. Transform & Sanitize Data

A Function or Code node processes the raw Apify output. It cleans the data—removing emojis, standardizing date formats, handling missing values—and maps it to the specific field names in your NocoDB table (e.g., linkedin_headline, linkedin_current_company).

4. Update NocoDB & Handle Errors

The transformed data is sent back to NocoDB via an Update Record node, populating the contact's record. The workflow includes error handling: if a profile is private or a scrape fails, it logs the error reason and updates a scrape_status field, preventing the process from halting.

Pro tip: Set the Schedule Trigger to run weekly or bi-weekly. This keeps your contact data fresh without overloading the scraper or hitting API limits, ensuring your team always has recent information.

Quick Setup Guide

Get this automation running in your environment in under 15 minutes.

  1. Import the Template: In your n8n dashboard, create a new workflow and use the "Import from File" option to upload the downloaded JSON template.
  2. Connect NocoDB: Click on any NocoDB node, add new credentials, and enter your NocoDB API Token. Update the 'Project ID' and 'Table Name' parameters to point to your specific base and table.
  3. Configure Apify: In the HTTP Request node, replace the placeholder API token with your own from Apify.com. Ensure the actor/endpoint is correctly set for LinkedIn profile scraping.
  4. Map Your Fields: Review the 'Transform Data' node. Adjust the field mapping (the JavaScript code) to match the exact column names in your NocoDB table where you want the LinkedIn data saved.
  5. Test & Activate: Click "Execute Workflow" to test with one record. Verify the data flows into NocoDB correctly. Then, toggle the workflow to 'Active' and let the Schedule Trigger handle the rest.
