AI Construction Cost Estimator: Telegram + OpenAI + DDC CWICR

Convert natural-language work descriptions into detailed cost estimates instantly. Free n8n workflow template for contractors, estimators, and construction managers.

Download Template JSON · n8n compatible · Free

What This Workflow Does

For contractors, project managers, and estimators, getting quick, accurate cost figures from verbal or text-based work descriptions is a daily challenge. Traditional methods involve manual lookups in rate books, spreadsheets, and guesswork—a process that can take hours and is prone to errors.

This automation solves that by turning Telegram into an intelligent estimation assistant. Simply send a message like “Living room renovation: laminate flooring 25 m², wall painting 60 m², 3 electrical outlets” and receive a fully broken-down cost estimate in seconds. It uses AI to parse the text, searches a comprehensive construction database (DDC CWICR) via vector similarity, and calculates labor, materials, and machine costs—supporting 9 languages for global teams.

The business value is clear: reduce quotation time from hours to minutes, minimize manual entry mistakes, and empower field teams with instant, data‑backed figures during client meetings or site visits.

How It Works

Step 1: Natural Language Input via Telegram

A user sends a work description to a Telegram bot. The message can be a list, paragraph, or even a voice note converted to text. The bot supports 9 languages, making it accessible for international projects.

Step 2: AI Parsing & Structured Extraction

OpenAI (or Claude/Gemini) analyzes the text, identifying individual work items, quantities, units, and room/area context. It transforms “laminate flooring 25 m²” into a structured object ready for database lookup.

Step 3: Vector Search Against DDC CWICR

The parsed query is converted into a vector embedding and searched against a Qdrant vector database containing the DDC CWICR rate library (55,000+ construction items). Semantic search finds the closest matching rates, not just keyword matches.

Step 4: AI Reranking & Cost Calculation

Top matches are reranked by AI for contextual accuracy. The system then applies quantities, sums labor, materials, and machine costs, and builds a detailed line‑item breakdown.

Step 5: Report Delivery & Export

A formatted estimate is sent back to Telegram. Users can request Excel or PDF exports for professional reporting, or continue the conversation to adjust quantities or add items.

What You'll Need

  1. n8n instance (v1.30+ for AI node support).
  2. Telegram Bot Token from @BotFather.
  3. OpenAI API key for embeddings and text parsing (Anthropic/Google Gemini optional).
  4. Qdrant vector database (self‑hosted or cloud) with DDC CWICR collections loaded.
  5. DDC CWICR data (open‑source construction rate database) imported into Qdrant.

Pro tip: Start with one language collection (e.g., EN_construction_rates) to test the workflow before loading all nine. Ensure your Qdrant instance has sufficient memory for vector similarity searches.

Quick Setup Guide

  1. Download & Import the JSON template into your n8n instance.
  2. Configure Credentials in n8n settings: add your OpenAI API key and Telegram bot token.
  3. Set Up Qdrant by loading DDC CWICR collections for your target languages (follow the GitHub guide).
  4. Update Configuration Node with your Qdrant URL and API key (if using Qdrant Cloud).
  5. Activate Workflow—the Telegram trigger will auto‑register the webhook.
  6. Test by sending /start to your bot, then a work description.
