What This Workflow Does

Construction, engineering, and architecture firms face a persistent data bottleneck: manually extracting information from Revit models into spreadsheets for analysis, costing, and reporting. This process eats up hours of skilled labor, introduces errors, and delays downstream decisions.

This automation workflow solves that by automating the entire extraction pipeline. It runs a local converter tool (RvtExporter.exe) on your Revit (.rvt) file, checks for successful conversion, locates the generated Excel file, reads it from disk, and parses the workbook into clean, structured data items within n8n. The output is ready for immediate transformation and loading into your ERP, project management software, or analytics dashboards.

By eliminating manual steps, you turn a multi‑hour, error‑prone task into a reliable, repeatable process that runs in minutes.

How It Works

Step 1: Setup & Path Definition

The workflow begins by defining two critical paths: the location of your Revit converter executable and the target Revit file. This setup node ensures the automation knows exactly where to find both the tool and the source data.

Step 2: Execute Converter

A command execution node runs the RvtExporter.exe with the provided file paths. It triggers the local conversion process that writes a structured Excel file (typically named *_rvt.xlsx) adjacent to the original Revit model.

Step 3: Success Validation

The workflow checks the converter's output for errors. If the conversion fails, the flow branches to handle the error appropriately—logging the issue and stopping further processing. On success, it proceeds to the next step.

Step 4: File Detection & Reading

The automation intelligently derives the expected Excel filename from the original Revit file name, locates it on disk, and reads the binary file content into the workflow.

Step 5: Data Parsing & Structuring

A spreadsheet node processes the Excel workbook, converting rows and columns into individual data items. Each item represents a structured piece of information from the Revit model—ready for mapping, transformation, or direct integration with other systems.

What You'll Need

  1. A Windows environment (the converter runs locally).
  2. The DDC Revit toolkit installed, with RvtExporter.exe available at a known path (e.g., C:\DDC_Converter_Revit\datadrivenlibs\RvtExporter.exe).
  3. A Revit (.rvt) file accessible to the workflow.
  4. Write permissions in the project folder for the converter to output the Excel file.
  5. An n8n instance (cloud or self‑hosted) to import and run the workflow.

Quick Setup Guide

  1. Import the template – Download the JSON file and import it into your n8n workspace.
  2. Configure paths – Open the “Setup – Define file paths” node and set the path_to_revit_converter and revit_file to match your local environment.
  3. Test with a sample – Run the workflow manually using a sample Revit file to verify the conversion and parsing works correctly.
  4. Add downstream steps – Connect the output to your own transformation nodes (data mapping, filtering) or integration nodes (to push data to your CRM, database, or project management tool).
  5. Schedule or trigger – Set the workflow to run on a schedule (e.g., nightly) or trigger it via webhook when new Revit models are uploaded to a shared folder.
