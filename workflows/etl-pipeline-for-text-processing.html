What This Workflow Does

This automated ETL (Extract, Transform, Load) pipeline collects tweets, analyzes their sentiment using Google Cloud Natural Language, stores raw data in MongoDB, and inserts processed results into Postgres. Positive tweets are automatically posted to Slack for engagement.

By automating this text processing pipeline, businesses can gain real-time insights from social media conversations without manual data collection and analysis. The workflow handles the entire process from data extraction to actionable insights.

How It Works

Step 1: Scheduled Data Collection

The Cron node triggers the workflow on your schedule (daily by default). The Twitter node then collects tweets based on your search parameters.

Step 2: Raw Data Storage

Tweets are stored in MongoDB as unstructured JSON data, preserving all original metadata for future analysis.

Step 3: Sentiment Analysis

Google Cloud Natural Language API analyzes each tweet's sentiment, providing a score (-1 to 1) and magnitude indicating emotional intensity.

Step 4: Processed Data Storage

The Set node extracts key sentiment metrics which are then stored in Postgres with the original tweet content for structured querying.

Step 5: Actionable Alerts

Positive tweets (score > 0.25) are automatically posted to a Slack channel for engagement opportunities, while negative ones are filtered out.
