What This Workflow Does

This automation solves a critical challenge for businesses deploying AI assistants: how to efficiently evaluate whether AI responses actually answer user questions correctly. Manual review of thousands of conversations is time-consuming and inconsistent.

The workflow automatically calculates a "Relevance" score for each AI agent response by comparing the original user question with a question generated from the AI's answer. Using OpenAI and cosine similarity, it provides objective, scalable quality assessment that helps identify when AI agents go off-topic, hallucinate, or provide irrelevant information.

Originally adapted from the RAGAS evaluation framework, this template brings enterprise-grade AI monitoring to any business using chatbots, customer support AI, or internal knowledge assistants.

How It Works

Step 1: Input Processing

The workflow receives pairs of user questions and AI agent responses, typically from chat logs, CRM systems, or conversation databases. Each pair is prepared for analysis with proper formatting and context preservation.

Step 2: Question Generation from Answers

Using OpenAI's language model, the system analyzes the AI's response and generates a question that this answer would appropriately address. This reverse-engineering step is key to measuring relevance.

Step 3: Embedding Creation

Both the original user question and the generated question are converted into numerical vector embeddings using OpenAI's embedding model. These embeddings capture semantic meaning in a format suitable for mathematical comparison.

Step 4: Cosine Similarity Calculation

The workflow calculates the cosine similarity between the two embedding vectors. This produces a score from 0 to 1, where higher values indicate greater relevance between what was asked and what was answered.

Step 5: Scoring and Output

Each interaction receives a relevance score, with results categorized (excellent, good, poor). The workflow outputs detailed reports, identifies low-performing conversations for review, and can trigger alerts when scores fall below thresholds.

What You'll Need

  1. n8n instance (version 1.94 or higher) either self-hosted or cloud
  2. OpenAI API key with access to embedding and chat completion models
  3. Source data containing user questions and AI responses (CSV, database, or API connections)
  4. Basic understanding of your AI agent's conversation structure and data format
  5. Optional: Destination for evaluation results (Google Sheets, database, or notification system)

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure OpenAI node with your API key and select appropriate models
  3. Connect your data source – adjust the input node to match your conversation data format
  4. Set scoring thresholds – customize what scores classify as excellent (0.8+), good (0.6-0.8), or poor (<0.6)
  5. Test with sample data using the provided Google Sheets template or your own examples
  6. Configure outputs to send results to your preferred destination (reports, alerts, dashboards)
  7. Schedule execution for regular evaluation (daily, weekly, or real-time as conversations occur)
