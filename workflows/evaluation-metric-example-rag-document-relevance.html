What This Workflow Does

This template solves a critical problem in AI implementation: knowing whether your Retrieval-Augmented Generation (RAG) system is actually retrieving relevant documents. Many businesses deploy AI assistants that sound convincing but provide inaccurate information because the underlying document retrieval is flawed.

The workflow automatically evaluates your RAG pipeline by comparing retrieved documents against user questions, calculating relevance scores, and identifying patterns where your system struggles. Instead of manually testing dozens of queries, this automation provides systematic, quantitative feedback about your AI's retrieval performance.

By implementing this evaluation system, you gain confidence that your AI is working with appropriate source material before generating answers. This prevents "hallucinations" caused by irrelevant context and ensures your AI assistant remains reliable as you add new documents or change your knowledge base structure.

How It Works

The workflow uses n8n's evaluation feature to systematically test your RAG system with real user questions and measure retrieval quality.

1. Evaluation Dataset Processing

The workflow begins by loading your test dataset—real questions users ask your AI system. These questions serve as the benchmark for measuring retrieval performance across different query types and difficulty levels.

2. Parallel Trigger Configuration

Dual triggers allow the same workflow to handle both evaluation mode and production mode. The evaluation trigger processes test questions with metrics calculation, while the regular trigger handles live user queries without evaluation overhead.

3. Document Retrieval & Agent Execution

Your RAG agent retrieves documents from your vector database and generates answers as usual. The workflow captures both the retrieved documents and the agent's tool usage data for analysis.

4. Relevance Metric Calculation

When in evaluation mode, the workflow uses AI to compare each retrieved document against the original question, scoring relevance on a standardized scale. This happens automatically without manual intervention.

5. Results Aggregation & Reporting

The system aggregates scores across all test questions, identifies patterns (which question types perform poorly), and prepares actionable insights for improving your RAG pipeline.

What You'll Need

  1. n8n instance (cloud or self-hosted) with AI nodes configured
  2. Vector database (Pinecone, Weaviate, Qdrant, or similar) with your documents indexed
  3. AI model access (OpenAI, Anthropic, or open-source) for both your RAG system and evaluation
  4. Test dataset of real user questions (20-50 questions minimum for meaningful evaluation)
  5. Existing RAG workflow or willingness to build one using n8n's agent nodes

Pro tip: Start with a small but diverse test dataset—include easy, medium, and difficult questions. Include edge cases and questions you know your current system struggles with. This gives you the most actionable improvement insights from day one.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Configure your AI credentials for both the RAG agent and evaluation nodes
  3. Connect your vector database and test document retrieval with sample questions
  4. Prepare your test dataset as a CSV or JSON file with question and expected answer pairs
  5. Run initial evaluation to establish baseline performance metrics
  6. Analyze results and identify top improvement opportunities for your RAG system
  7. Implement changes based on insights and re-evaluate to measure improvement
