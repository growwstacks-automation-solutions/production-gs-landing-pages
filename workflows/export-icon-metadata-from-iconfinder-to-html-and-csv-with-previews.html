What This Workflow Does

If you're a designer, icon creator, or agency managing an Iconfinder portfolio, you know how frustrating it is to manually track and document your icon collections. Iconfinder doesn't provide a built-in export feature for contributors, leaving you with scattered assets and no centralized catalog.

This automation solves that problem by connecting to the Iconfinder API, fetching all your iconsets and icons, then generating two professional outputs: an HTML file with visual previews in a clean table format, and a CSV file containing all metadata including names, tags, iconset information, and preview URLs. What typically takes hours of manual screenshotting and spreadsheet work now happens automatically in minutes.

The workflow handles pagination automatically (important for large collections), extracts consistent preview image sizes for easy viewing, and structures the data in formats that are immediately useful for client presentations, team documentation, or integration with other design tools.

How It Works

The automation follows a logical sequence to ensure complete and accurate data extraction from your Iconfinder account.

1. API Authentication & User ID Retrieval

The workflow begins by securely connecting to the Iconfinder API using your API key. A dedicated node fetches your user ID to ensure it accesses the correct contributor account. This setup node passes authentication to all subsequent requests, eliminating repetitive credential entry.

2. Iconsets Discovery & Pagination Handling

The system retrieves all your iconsets, automatically managing API pagination if you have extensive collections. This ensures no iconset is missed, regardless of how many you've created over time.

3. Individual Icon Processing

For each iconset, the workflow loops through every icon, extracting comprehensive metadata including the icon name, associated tags, preview image URLs at consistent sizes (typically 128x128), and the parent iconset name. This structured data forms the foundation for both output formats.

4. HTML & CSV Generation

The processed data feeds into two parallel output streams. The HTML generator creates a visually organized table with thumbnail previews, names, and key details—perfect for quick browsing. Simultaneously, the CSV generator produces a spreadsheet-friendly format with all metadata fields, ideal for filtering, analysis, or import into other systems.

5. Output Delivery

Both files are packaged and made available for download. You can trigger this workflow manually when needed, or schedule it to run periodically, ensuring your icon catalog stays current as you add new designs.

Pro tip: Schedule this workflow to run monthly. This creates versioned catalogs of your growing portfolio, making it easy to track your creative output over time and demonstrate progress to clients or stakeholders.

What You'll Need

  1. Iconfinder Contributor Account: You must have an active Iconfinder account with uploaded icons.
  2. Iconfinder API Key: Generate this from your Iconfinder account settings under the API section.
  3. n8n Instance: Either self-hosted n8n or n8n.cloud account to run the workflow.
  4. Basic API Understanding: Comfort with copying API keys into configuration fields (no coding required).

Quick Setup Guide

Getting this automation running takes about 10 minutes if you have your API key ready.

  1. Download and Import: Click the download button above to get the JSON file, then import it into your n8n instance via the workflow import function.
  2. Configure API Access: Locate the "Iconfinder Setup" node in the workflow and paste your Iconfinder API key into the designated field.
  3. Test User ID Retrieval: Execute the initial nodes to fetch your Iconfinder user ID—this confirms your API connection is working correctly.
  4. Adjust Output Preferences (Optional): Modify the HTML template or CSV fields if you want to customize column names, image sizes, or included metadata.
  5. Execute and Download: Run the workflow once to generate your initial HTML and CSV files, then download them from n8n's output panel.
  6. Schedule Automation (Optional): Set the workflow to trigger on a schedule (weekly/monthly) to keep your catalog automatically updated.

Pro tip: Before your first full run, test with a small iconset to verify output formatting. Once satisfied, run against your entire collection.
