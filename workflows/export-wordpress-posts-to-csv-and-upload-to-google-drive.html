What This Workflow Does

Manually exporting your WordPress posts for backups, SEO audits, or content migration is a repetitive and time-consuming task. This automation eliminates that manual work by connecting your WordPress site directly to Google Drive.

The workflow fetches all your published posts, extracts key details like title, URL, and content, formats this data into a clean CSV file, and automatically uploads it to a designated folder in your Google Drive. This creates a living, searchable archive of your site's content that updates automatically.

Beyond simple backup, this automated export is a powerful tool for content strategy. Marketing teams can analyze post performance, SEO specialists can audit on-page elements at scale, and site administrators can maintain reliable records for compliance or migration projects—all without lifting a finger after setup.

How It Works

The automation follows a logical, step-by-step process to transform your live WordPress content into a structured data file in the cloud.

1. Trigger the Process

The workflow can be initiated manually within n8n for an on-demand export, or set to run on a schedule (e.g., weekly, monthly) to ensure your backup is always current.

2. Fetch Posts from WordPress

Using the WordPress REST API, the workflow retrieves a list of all published posts. It can be configured to pull specific data points: post ID, title, slug, published date, author, categories, tags, featured image URL, excerpt, and the full content.

3. Structure & Format the Data

The raw post data is processed and mapped into a clean, column-based structure ideal for spreadsheets. This step ensures dates are formatted correctly, HTML is stripped from content if needed, and all fields are organized for easy analysis.

4. Convert Data to CSV File

The formatted data is passed to a node that converts the JSON array into a standard CSV file. You can customize the filename to include timestamps (e.g., `wordpress-posts-2025-03-25.csv`) for versioning.

5. Upload to Google Drive

The newly created CSV file is automatically uploaded to a specified folder in your Google Drive. The workflow handles authentication and file management, so the CSV appears in your Drive just as if you had dragged and dropped it there manually.

What You'll Need

  1. A self-hosted or WordPress.com site with the REST API enabled (standard for most WordPress installations).
  2. Your WordPress site's URL and, if required, API authentication credentials (often a username and application password).
  3. A Google Cloud Project with the Google Drive API enabled.
  4. Service account credentials or OAuth 2.0 setup to allow n8n to upload files to your Google Drive.
  5. An n8n instance (cloud or self-hosted) to import and run this workflow.

Pro tip: For WordPress.com sites, you'll typically use an "Application Password" generated in your user profile. For self-hosted sites, ensure the REST API is not blocked by security plugins.

Quick Setup Guide

Get this automation running in your n8n instance in just a few minutes.

  1. Download the Template: Click the "Download Template" button on this page to get the JSON file.
  2. Import into n8n: In your n8n dashboard, go to "Workflows" > "Import from file" and select the downloaded JSON.
  3. Configure WordPress Credentials: In the "WordPress" node, enter your site's URL and API authentication details.
  4. Set Up Google Drive Connection: In the "Google Drive" node, authenticate n8n with your Google account and specify the target folder ID.
  5. Test the Workflow: Execute the workflow once manually to confirm it fetches posts and uploads the CSV correctly.
  6. Schedule It (Optional): Add a "Schedule Trigger" node to the start to run the export automatically daily, weekly, or monthly.
