What This Workflow Does

Managing a WordPress site often means your valuable content is locked inside the database. Manually exporting posts for analysis, backup, or reporting is time‑consuming and error‑prone. This automation solves that by connecting your WordPress site directly to a spreadsheet.

The workflow fetches posts (or any custom post type) from your WordPress REST API, extracts key data like title, content, author, categories, and publish date, then formats it into a clean CSV file. You can run it on demand or schedule it weekly to maintain an always‑current external content archive.

Beyond simple backup, this enables content audits, SEO analysis, editorial planning, and migration preparation—all without touching phpMyAdmin or clunky export plugins.

How It Works

1. Trigger & Authentication

The workflow starts either manually or on a schedule. It authenticates with your WordPress site using application passwords or JWT tokens via the REST API, ensuring secure access without exposing admin credentials.

2. Fetch Posts from WordPress

It queries the WordPress REST API to retrieve posts. You can configure filters: by date range, category, status (published/draft), author, or custom post type. The workflow handles pagination to fetch all posts, not just the first page.

3. Data Transformation & Cleaning

Each post’s data is parsed and cleaned. HTML tags are stripped from content, featured image URLs are extracted, custom fields are included, and dates are formatted consistently. This ensures the output is analysis‑ready.

4. Generate CSV File

The cleaned data is assembled into a structured CSV format. Columns are defined (Title, URL, Author, Publish Date, Excerpt, etc.), and special characters are encoded to prevent spreadsheet corruption.

5. Deliver the File

The final CSV file is saved to your local machine, uploaded to Google Drive, sent via email, or stored in a cloud storage service. You can also append data to an existing Google Sheet for historical tracking.

What You'll Need

  1. A self‑hosted WordPress site (or WordPress.com with plugin support).
  2. WordPress REST API enabled (standard on all modern installs).
  3. An application password or JWT token for API authentication.
  4. A running n8n instance (cloud, self‑hosted, or desktop).
  5. Optional: Google Drive, Dropbox, or email account for file delivery.

Pro tip: Use WordPress’s “application passwords” feature (under User Profile) instead of your admin password for API access. This is more secure and can be revoked anytime.

Quick Setup Guide

Import and configure this workflow in under 10 minutes:

  1. Download the template using the button above and open it in your n8n instance.
  2. Configure the WordPress node with your site URL and application password.
  3. Adjust the query parameters to filter posts by date, status, or category as needed.
  4. Map the data fields you want to export (title, content, author, etc.).
  5. Set the output destination—local file, Google Drive, or email attachment.
  6. Test the workflow with a single post to verify the format, then run it fully.
  7. Schedule it (optional) for weekly or monthly automatic backups.
