Extract and Store Text from Telegram Chat Images Using AWS S3

Automatically extract text from images sent in Telegram, upload originals to AWS S3, and log extracted data to Airtable—fully automated.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Businesses and communities often share important information through images in chat apps—receipts, screenshots, documents, or handwritten notes. Manually extracting and storing this text is time-consuming and error-prone. This automation solves that by creating a seamless pipeline from chat to cloud storage to structured data.

When an image is sent in a Telegram channel or group, this workflow instantly captures it, uploads the original file to AWS S3 for secure archival, uses AWS Textract (AI-powered OCR) to extract all readable text, and then logs both the image metadata and extracted content into an Airtable base. The result is a searchable, organized record of visual information without any manual steps.

How It Works

Step 1: Telegram Trigger Listens for Images

The workflow starts with a Telegram Trigger node configured to monitor a specific chat, channel, or group. When any participant sends an image, the node captures the file, sender details, timestamp, and any accompanying caption.

Step 2: AWS S3 Stores the Original Image

The image file is immediately uploaded to a designated AWS S3 bucket. This ensures durable storage, version control, and easy retrieval if needed later. The S3 node can apply organization with folder structures based on date, sender, or chat ID.

Step 3: AWS Textract Extracts Text

The workflow passes the image to AWS Textract, Amazon's advanced OCR service. Textract detects and extracts printed text, handwriting, tables, and forms—returning structured data with confidence scores. It handles various image qualities and layouts better than basic OCR tools.

Step 4: Airtable Logs Everything

Finally, all data—image URL in S3, extracted text, sender information, timestamp, and Textract confidence metrics—is written to an Airtable table. This creates a centralized, queryable database for reporting, analysis, or integration with other business systems.

Who This Is For

This automation is ideal for customer support teams receiving screenshot-based issues, community managers moderating image submissions, finance departments digitizing receipt images, logistics companies processing shipping label photos, or any team that needs to transform visual information into actionable, searchable data. It's particularly valuable for remote teams using Telegram for collaboration and AWS for scalable infrastructure.

What You'll Need

  1. A Telegram bot token (created via @BotFather) with access to the relevant chat.
  2. AWS account with S3 bucket created and Textract service enabled.
  3. AWS IAM credentials (access key and secret) with permissions for S3 and Textract.
  4. Airtable account with a base and table prepared to receive the data.
  5. n8n instance (cloud or self-hosted) to run the workflow.

Pro tip: Start with a test Telegram group and a dedicated S3 bucket folder. Use low-resolution images initially to validate the text extraction accuracy before scaling to production volumes.

Quick Setup Guide

  1. Download the template and import it into your n8n workspace.
  2. In the Telegram Trigger node, replace the placeholder bot token with your actual token and specify the chat ID to monitor.
  3. Configure the AWS S3 node with your bucket name, region, and folder path preference.
  4. Update the AWS Textract node with the same region and ensure IAM permissions allow Textract access.
  5. Set up the Airtable node with your base ID, table name, and map the fields correctly.
  6. Activate the workflow and test by sending an image to your Telegram chat.
