Extract Business Contact Leads from Google Maps

Automate lead generation by extracting business contact data from Google Maps via RapidAPI and storing it in Google Sheets.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Manual lead generation from Google Maps is time-consuming and inconsistent. Sales teams spend hours searching for businesses, copying contact details, and organizing data—often missing opportunities and creating duplicate entries.

This automation solves that problem by programmatically extracting comprehensive business information from Google Maps using RapidAPI's Local Business Data service. It searches for any business type in any location, gathers contact details, social media profiles, ratings, and verification status, then stores everything in Google Sheets with intelligent duplicate prevention.

The workflow enables scalable lead generation for B2B sales, market research, and competitive analysis—processing multiple cities or countries simultaneously while optimizing API costs through smart filtering.

How It Works

Step 1: Load Search Criteria

The workflow reads your search queries from a Google Sheet where you've marked locations with "X". Each query includes keywords (like "Restaurants Madrid"), latitude/longitude coordinates, and country codes.

Step 2: Check Existing Data

It retrieves previously extracted business data from your storage sheet to identify duplicates. This prevents wasting API calls on already-processed searches and ensures each business appears only once.

Step 3: Filter New Searches

A smart merge operation compares new query+location combinations against historical data, filtering out anything already processed. This saves 50-80% on API costs.

Step 4: Call RapidAPI

For each new search, the workflow sends parameters to RapidAPI's Local Business Data endpoint, requesting business listings matching your keywords within the specified geographic radius.

Step 5: Parse and Structure Data

The API response contains raw business information. The workflow extracts and structures key fields: business names, phone numbers, addresses, websites, emails (when available), social media links, ratings, and verification status.

Step 6: Save to Google Sheets

Each extracted lead receives a unique business ID and is appended to your Google Sheets database. The sheet automatically organizes columns for easy filtering and analysis.

Step 7: Rate Limiting and Loop

A 10-second delay between requests ensures API compliance and reliability. The workflow loops until all new searches are processed, then completes.

What You'll Need

  1. RapidAPI account with subscription to "Local Business Data" API (Basic plan supports 1000 requests/month)
  2. Google account for Google Sheets integration
  3. n8n instance (cloud or self-hosted)
  4. Google Sheet with two tabs: "keyword_searches" for queries and "stores_data" for extracted leads
  5. API credentials: RapidAPI key and Google Sheets OAuth2 setup

Quick Setup Guide

  1. Import the workflow: Copy the provided JSON into your n8n instance via Import from JSON.
  2. Configure credentials: Create HTTP Header Auth credential for RapidAPI with your X-RapidAPI-Key and host. Set up Google Sheets OAuth2 credential.
  3. Prepare your Google Sheet: Create sheets with columns as described in the template documentation.
  4. Update placeholder values: Replace YOUR_GOOGLE_SHEET_ID with your actual sheet ID in the workflow nodes.
  5. Add search queries: Populate your keyword_searches sheet with locations marked "X", including queries, coordinates, and country codes.
  6. Test execution: Run the workflow once to verify data extraction and storage.
  7. Schedule automation: Set the workflow to run weekly or monthly for continuous lead updates.

Pro tip: Start with a small geographic area (one city) and limited queries (5-10) to test API response quality and cost before scaling to hundreds of locations.
