What This Workflow Does

This automation solves the tedious problem of manual data entry from physical documents, scanned PDFs, or images. Instead of typing out text from receipts, invoices, forms, or meeting notes, users simply send these files to a Telegram bot. The workflow automatically detects the file type, extracts all text using Mistral AI's advanced OCR capabilities, and returns clean, formatted Markdown text that's ready for editing, storage, or integration with other business systems.

The template includes intelligent file validation (enforcing Telegram's 25MB limit), optional user whitelisting for security, and a clever self-referencing download mechanism that delivers processed files back through Telegram. It's designed for both individual productivity and team collaboration, allowing anyone with Telegram access to digitize documents instantly from their mobile device or desktop.

Beyond basic text extraction, this workflow represents a complete document processing pipeline that can be extended with additional automation steps—like categorizing documents, extracting specific data fields, or triggering follow-up actions in your CRM or accounting software.

How It Works

The automation follows a logical sequence that balances user convenience with robust processing.

1. Telegram Bot Setup & Webhook Configuration

First, you create a Telegram bot via BotFather and obtain an API token. The template includes utility flows to help set up webhooks for both development and production environments, ensuring your bot can receive file uploads from users. This foundational step establishes the communication channel between Telegram users and your n8n instance.

2. File Reception & Validation

When a user sends a document to your bot, the workflow immediately validates it. It checks file type (PNG, JPEG, or PDF), verifies the size is under 25MB (Telegram's limit), and optionally validates the user against a whitelist. Invalid files or unauthorized users receive immediate feedback, while valid requests proceed to processing.

3. AI-Powered OCR Processing

The core of the workflow uses Mistral AI's OCR capabilities to extract text from the document. Unlike basic OCR that struggles with complex layouts or poor quality scans, Mistral's AI understands context, handles various fonts and languages, and maintains formatting cues. The system processes multi-page PDFs and image files with equal effectiveness.

4. Markdown Conversion & Delivery

Extracted text is converted into clean Markdown format with proper headings, lists, and paragraph breaks. The workflow then generates a downloadable .md file and creates a temporary download link by making a GET request to itself. This link is sent back to the user via Telegram, allowing them to download the processed text immediately.

Quick Setup Guide

Follow these steps to deploy your automated document processing system:

  1. Create Your Telegram Bot: Message @BotFather on Telegram, create a new bot, and save the API token provided.
  2. Import the Template: Download and import the JSON file into your n8n instance using the workflow import feature.
  3. Configure Credentials: Add your Telegram bot token and Mistral AI API key to n8n's credentials manager.
  4. Set Up Webhooks: Use the included utility flows to configure webhooks pointing to your n8n instance URL.
  5. Customize Settings: Adjust the whitelist, file size limits, and output formatting to match your requirements.
  6. Test & Deploy: Send a test image or PDF to your bot, verify the Markdown output, then activate the workflow for team use.

Pro tip: Start with a small whitelist of test users before opening the bot to your entire team. This lets you refine the processing and ensure quality output before scaling.
