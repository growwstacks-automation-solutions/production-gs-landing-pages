What This Workflow Does

This n8n workflow automates the retrieval of Bible scriptures from the GetBible API, transforming manual verse lookups into a seamless, programmable service. It accepts structured JSON input containing book, chapter, and verse references, then returns clean, formatted scripture text ready for integration into your applications.

For ministries, content creators, and developers, this eliminates hours of manual copying and pasting. Whether you're building a daily devotional app, enhancing your church website with dynamic verses, or creating automated social media posts with scripture, this workflow handles the complex API interactions so you can focus on delivering value to your audience.

The template is production-ready and modular—simply call it from other workflows or applications whenever you need scripture content. It supports multiple translations, verse ranges, and complex reference formats while maintaining data integrity and proper error handling.

How It Works

The workflow acts as an intelligent middleware between your applications and the GetBible API, processing requests and returning structured responses.

1. Accept Structured Input

The workflow begins by receiving a JSON payload containing scripture references, translation preference, and API version. It validates the input format and prepares it for processing, ensuring all required parameters are present before proceeding.

2. Parse Scripture References

Advanced parsing logic breaks down complex references into manageable components. It handles single verses (John 3:16), comma-separated lists (John 3:16,18), and verse ranges (John 3:16-18) with intelligent error checking for malformed inputs.

3. Query GetBible API

For each valid reference, the workflow makes optimized HTTP requests to the GetBible API. It manages rate limiting, handles network timeouts, and implements retry logic for failed requests to ensure reliable scripture retrieval even during temporary API issues.

4. Format and Return Results

Raw API responses are transformed into a consistent JSON structure with standardized field names, proper formatting, and error messages where applicable. The output maintains the original reference context while providing clean, ready-to-use scripture text.
