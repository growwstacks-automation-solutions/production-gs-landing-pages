Fetch the Public IP Addresses of Your n8n Instance

Automatically discover and list all public IP addresses for your n8n instance. Essential for firewall whitelisting and secure API connections.

What This Workflow Does

If you use n8n to connect to external service providers—like payment gateways, CRM platforms, or enterprise databases—many of these services require you to whitelist specific IP addresses in their firewall for security. This means your automations can fail if the receiving service doesn't recognize the IP address your n8n instance is using to make outbound calls.

This workflow solves that problem by automatically discovering and listing all public IP addresses associated with your n8n instance. It queries an external service multiple times to capture all potential IPs (useful if your setup uses multiple gateways for redundancy) and returns a clean, deduplicated list. You can then use this list to update firewall rules, ensuring your automations run without interruption.

It's a foundational security and connectivity tool for any business running n8n in a production environment, especially when integrating with strict, security-conscious third-party APIs.

How It Works

The workflow is triggered via a webhook, making it easy to run on-demand or on a schedule. It's built with reliability and accuracy in mind.

1. Webhook Trigger

The workflow starts with a Webhook node, waiting for an HTTP request. This allows you to trigger the IP check manually via a simple cURL command or automatically on a schedule using n8n's scheduler.

2. Multiple IP Lookup Requests

An HTTP Request node queries the public service api.ipify.org in JSON format. The request is repeated multiple times (e.g., 10 iterations) to account for network configurations where traffic might exit through different gateways or load balancers, each with a distinct public IP.

Pro tip: Repeating the request helps identify all possible public IPs in environments with high availability or load-balanced setups, preventing "IP not whitelisted" errors.

3. Data Aggregation

The results from all requests are collected. An Aggregate node bundles the individual IP responses into a single array for processing.

4. IP Extraction & Deduplication

A Code node (or Function node) uses JavaScript to pluck the 'ip' value from each result object and then deduplicate the list, ensuring you get a clean array of unique public IP addresses.

5. Secure Response

The final, deduplicated array of IP addresses is returned as the webhook response. You can configure the webhook to require an API key in the header for security, ensuring only authorized users or systems can trigger the IP discovery.

Quick Setup Guide

Get this workflow running in your n8n instance in under 5 minutes.

  1. Download the template using the button above and save the JSON file.
  2. Import into n8n: In your n8n dashboard, go to Workflows > Import from File and select the downloaded JSON.
  3. Configure the Webhook: Open the Webhook node. Note the unique webhook URL. For security, we recommend adding a header authentication credential (like an API key) in the "Authentication" dropdown and selecting it.
  4. Update the workflow to use your new credential in the HTTP Request node if you added header auth.
  5. Activate the workflow and test it by calling the webhook URL with a tool like cURL or Postman.

Example cURL test command:
curl -H "api-key: your-super-secret-token" https://your-n8n-domain.com/webhook/your-unique-path
You should receive a JSON response like: ["203.0.113.10", "198.51.100.25"]
