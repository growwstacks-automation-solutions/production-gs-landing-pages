What This Workflow Does

This n8n workflow solves a critical security gap in AI agent deployments: the inability to verify file safety before processing. When AI agents handle files from emails, uploads, or external sources, they risk executing or analyzing malware. This template bridges that gap by creating an MCP (Model Context Protocol) server that exposes the CIRCL hashlookup API's 11 security operations directly to your AI agents.

The workflow transforms complex security API calls into simple, AI-accessible tools. Your AI agents can now check MD5, SHA1, and SHA256 hashes against a database of 40+ billion known file signatures, get database statistics, perform bulk lookups, and maintain search sessions—all through natural language requests. This turns security verification from a manual, technical process into an automated, integrated part of your AI operations.

By deploying this template, you're not just adding a security check; you're building a security layer that grows with your AI capabilities. As your agents handle more files and make more autonomous decisions, this verification system ensures every file is vetted before processing, preventing potential breaches and maintaining operational integrity.

How It Works

The workflow acts as a bridge between your AI agents and the CIRCL security database, handling all the technical complexity so your agents can focus on their primary tasks.

1. MCP Server Activation

When you activate the workflow, it creates an MCP server endpoint that listens for requests from connected AI agents. This server understands the MCP protocol, meaning agents like Claude Desktop or Cursor can communicate with it natively without custom integration code.

2. AI Agent Request Handling

When an AI agent needs to verify a file hash, it sends a request to your MCP server. The workflow automatically extracts parameters using $fromAI() expressions, which intelligently parse the agent's natural language requests into structured API calls.

3. CIRCL API Integration

The workflow makes authenticated HTTP requests to the CIRCL hashlookup API, supporting all 11 available operations including individual hash lookups, bulk searches, session management, and database statistics. Error handling is built-in, with proper response formatting for the AI agent.

4. Security Response Delivery

Results from the CIRCL database are formatted and returned to the AI agent in a structured way. The agent receives clear information about whether a hash is known-clean, potentially malicious, or unknown, along with metadata about the file when available.

Quick Setup Guide

You can have this security layer operational in under 10 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Add your CIRCL credentials in the HTTP Request nodes (register at hashlookup.circl.lu if needed)
  3. Activate the workflow to start your MCP server
  4. Copy the webhook URL from the MCP trigger node configuration
  5. Configure your AI agents to use the MCP server URL (Claude Desktop: settings → Developer → MCP Servers)
  6. Test with a sample hash like asking your agent "Check this MD5 hash: d41d8cd98f00b204e9800998ecf8427e"

Common questions about AI agent security and file hash verification

File hash verification is the process of checking a file's unique digital fingerprint (hash) against a database of known malicious or safe files. For AI agents, it's critical because they often process files from external sources. Without verification, an AI could unknowingly execute or analyze malware, compromising security. This workflow gives AI agents the ability to perform these checks in real-time before acting on file data.

Think of it as a bouncer for your AI's file processing club. Every file that wants attention gets its ID (hash) checked against a known list of troublemakers before being allowed in. This prevents your AI from accidentally spreading malware or processing compromised documents.

The CIRCL hashlookup API is a free, public service provided by the Computer Incident Response Center Luxembourg. It contains over 40 billion known file hashes from clean software sources, helping security teams identify known-good files and potentially spot unknown or malicious ones. It's a valuable resource for security automation without the cost of commercial threat intelligence feeds.

The database includes hashes from operating systems, software packages, and known clean files across multiple industries. When your AI agent checks a hash against this database, it's essentially asking "Has this file been seen before in legitimate software?" A match means the file is likely safe; no match warrants further investigation.

MCP (Model Context Protocol) provides a standardized way for AI agents to access external tools and data sources.
