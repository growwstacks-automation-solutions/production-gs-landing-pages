Transform raw flight data into actionable insights with automated chart generation and instant Telegram delivery

What This Workflow Does

This intelligent automation solves the common business problem of data-rich but insight-poor operations. Many companies collect extensive flight or travel data but struggle to extract meaningful patterns quickly. Manual analysis in spreadsheets is time-consuming, while traditional BI tools require technical expertise and separate logins.

The workflow transforms this challenge into an opportunity by creating an accessible data visualization system. It automatically processes CSV flight data, generates four types of professional charts using Chart.js, and delivers them instantly through a Telegram bot interface. This means stakeholders can request specific visualizations—like airline performance comparisons or price trend analysis—and receive them directly on their phones within seconds.

The solution bridges the gap between complex data analysis and practical business decision-making. Instead of waiting for weekly reports or struggling with dashboard software, team members get on-demand insights through a familiar messaging platform, enabling faster responses to market changes and operational opportunities.

How It Works

1. User Interaction via Telegram

The workflow begins when a user sends a command to your Telegram bot. The bot presents a menu of visualization options, making the system accessible to non-technical users who simply select what type of chart they want to see.

2. Data Processing Pipeline

Once a selection is made, the automation reads your flight data CSV file, parses the information, and performs calculations. It aggregates metrics like flight counts per airline, categorizes durations, segments price ranges, and identifies trends—all automatically.

3. Dynamic Chart Generation

Using Chart.js configurations, the workflow creates professional visualizations tailored to the user's request. Each chart includes calculated percentages, averages, and key insights automatically added as annotations, transforming raw numbers into understandable business intelligence.

4. Image Rendering & Delivery

The chart configurations are sent to QuickChart API, which generates high-quality PNG images. These images are then delivered back through the Telegram bot, appearing directly in the user's chat with accompanying insights and recommendations.

What You'll Need

  1. n8n instance (self-hosted or cloud version)
  2. Telegram Bot Token from @BotFather
  3. Flight data CSV file with columns for airline, flight, source city, departure time, arrival time, duration, price, class, destination city, and stops
  4. Internet connectivity for QuickChart API access
  5. Basic understanding of n8n interface for configuration

Pro tip: Start with a sample dataset of 100-200 flights to test the visualization quality before processing larger files. This helps optimize chart configurations for your specific data characteristics.

Quick Setup Guide

Follow these steps to implement this automation in your business:

  1. Download the template using the button above and import it into your n8n instance
  2. Create a Telegram bot through @BotFather and copy your bot token
  3. Configure the Telegram nodes in the workflow with your bot token
  4. Upload your flight data CSV to the designated location in your n8n file system
  5. Test the /start command in Telegram to ensure the bot responds correctly
  6. Activate the workflow and begin requesting visualizations from your team
