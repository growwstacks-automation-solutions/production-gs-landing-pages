Gate Deployments on WAF Scan Results with WAFtester

Automate security validation in your CI/CD pipeline. Block deployments automatically when WAF protection is insufficient.

What This Workflow Does

This n8n workflow creates an automated security gate for your CI/CD pipeline that blocks deployments when Web Application Firewall (WAF) protection is insufficient. Your deployment pipeline sends a simple webhook with the target URL, the workflow automatically runs comprehensive WAFtester security scans, and returns a pass/fail HTTP response that your pipeline can use to gate the deployment.

Instead of relying on manual security reviews or hoping your WAF configuration is adequate, this automation provides objective, automated validation before any code reaches production. It transforms security from a bottleneck into an integrated, automated quality check that runs alongside your other tests.

The workflow is particularly valuable for organizations practicing continuous deployment where security validation must keep pace with rapid release cycles. It ensures that security doesn't get sacrificed for speed, giving both development and security teams confidence in the deployment process.

How It Works

The workflow consists of seven intelligent nodes that work together to provide comprehensive security validation:

1. Webhook Trigger

Your CI/CD pipeline sends a POST request with the target URL and requested attack categories (like SQLi, XSS, SSRF). This triggers the security validation process automatically as part of your deployment workflow.

2. WAF Detection

The workflow calls WAFtester's detect_waf tool to automatically fingerprint the WAF vendor and configuration. This ensures the subsequent tests are tailored to your specific security infrastructure.

3. Security Scan Initiation

An asynchronous scan task launches with the specific attack categories requested. The workflow can test across 18 different vulnerability categories with 2,800+ attack payloads.

4. Intelligent Waiting

The workflow pauses appropriately to allow the comprehensive security scan to complete. This ensures thorough testing without unnecessary delays in your pipeline.

5. Results Polling

The system calls get_task_status to retrieve completed scan results, including detection rates, security scores, and detailed vulnerability information.

6. Security Evaluation

The workflow compares the detection rate against your configured WAF_PASS_THRESHOLD to determine if security is sufficient for deployment.

7. Automated Response

Based on the evaluation, the workflow returns either HTTP 200 (pass - deploy allowed) or HTTP 422 (fail - deploy blocked) with detailed bypass information for security teams to investigate.

Pro tip: Start with a conservative threshold (like 95% detection rate) and adjust based on your risk tolerance. Consider running this in staging environments first to establish baseline performance before enforcing in production pipelines.

Quick Setup Guide

Follow these steps to implement this security automation in your environment:

  1. Download and import the template JSON file into your n8n instance
  2. Configure the webhook node with a secure path for your CI/CD pipeline to call
  3. Set up WAFtester on your n8n server or ensure it's accessible via API
  4. Configure security thresholds in the Evaluate node based on your risk profile
  5. Test the integration by sending sample requests from your CI/CD tool
  6. Integrate into pipeline by adding the webhook call before deployment steps
  7. Monitor and refine based on initial results and security team feedback
