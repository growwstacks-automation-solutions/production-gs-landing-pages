What This Workflow Does

This automation solves the time-consuming, manual process of creating and publishing video content. It provides a complete, hands-off pipeline that turns a simple text prompt into a published YouTube video. The workflow leverages cutting-edge AI for both video generation (Google Veo3) and title optimization (GPT-4o), while ensuring reliable storage and seamless publishing.

Businesses and creators can use this to scale their video output for marketing, education, or social media without increasing production overhead. It eliminates the need for video editing software, manual uploads, and creative bottlenecks, turning a complex multi-step process into a single automated operation triggered from a Google Sheet.

How It Works

The workflow follows a logical sequence, moving data between services with built-in error handling and status checks.

1. Trigger & Data Fetch

The automation starts either on a schedule or manually. It reads new video requests from a designated Google Sheet, looking for rows where the video column is empty. Each row contains the text prompt and desired duration.

2. AI Video Generation

The prompt and duration are sent to the Google Veo3 API via Fal.run. The workflow then enters a polling loop, checking the generation status every 60 seconds until the video is ready. This ensures reliability even if generation takes longer than expected.

3. Title Creation & File Processing

Once the video is ready, the system fetches the video URL and simultaneously uses GPT-4o to generate an SEO-optimized, engaging YouTube title based on the original prompt. The video file is then downloaded to the workflow's temporary memory.

4. Storage & Upload

The video is first saved to a specified Google Drive folder, creating a secure backup and central asset library. Immediately after, it's uploaded to YouTube via the Upload-Post API with the AI-generated title and relevant metadata.

5. Tracking & Completion

Finally, the Google Sheet is updated with the video's Drive URL and the published YouTube link, providing full traceability and closing the loop.

What You'll Need

  1. A running n8n instance (Cloud, Self-Hosted, or Desktop).
  2. A Google Cloud Project with the Google Sheets and Google Drive APIs enabled, and corresponding service account credentials.
  3. A Fal.run account and API key for accessing Google Veo3.
  4. An Upload-Post account and API key for YouTube uploads (or direct YouTube API credentials with appropriate upload permissions).
  5. An OpenAI API key (or alternative) for GPT-4o title generation.
  6. A Google Sheet formatted with columns: PROMPT, DURATION, VIDEO, YOUTUBE_URL.

Pro tip: Start with a low-frequency schedule (e.g., once per day) to monitor costs and output quality before scaling. Video generation APIs can have variable latency and cost per second.

Quick Setup Guide

  1. Import the Template: Download the JSON file and import it into your n8n instance.
  2. Configure Credentials: Set up the required credentials for Google Sheets, Google Drive, Fal.run, Upload-Post, and OpenAI in n8n's credentials management.
  3. Update Node Parameters: In the "Get new video" node, insert the URL of your Google Sheet and the sheet name. In the "HTTP Request" node for YouTube uploads, replace "YOUR_USERNAME" with your Upload-Post profile.
  4. Set API Keys: Add your Fal.run API key to the "Create Video" node's Authorization header, and your Upload-Post key to the corresponding "HTTP Request" node.
  5. Test with a Single Row: Add one prompt to your Google Sheet and trigger the workflow manually to verify the entire pipeline works.
  6. Activate the Schedule: Enable the "Schedule Trigger" node to run at your desired interval (e.g., every 6 hours).
