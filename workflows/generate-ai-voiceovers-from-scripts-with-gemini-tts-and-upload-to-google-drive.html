What This Workflow Does

This automation solves a critical bottleneck in content production: turning written scripts into professional voiceovers. Instead of hiring voice actors or recording audio manually, this workflow uses Google's advanced Gemini Text-to-Speech API to generate natural-sounding audio from any text input.

The system connects to your Google Sheets containing video scripts, processes each script through AI voice generation, converts the output to standard audio formats using FFmpeg, and automatically organizes the finished files in your Google Drive. It then updates your tracking sheet with direct links to each audio file, creating a complete, hands-free pipeline from text to ready-to-use audio assets.

This is particularly valuable for businesses producing video content at scale—marketing agencies, e-learning platforms, social media teams, and content creators who need to produce consistent voiceovers quickly without compromising quality.

How It Works

1. Script Retrieval

The workflow begins by connecting to your Google Sheets document where video scripts are stored. It identifies new or unprocessed scripts based on status columns, ensuring only fresh content gets voiceovers generated.

2. AI Voice Generation

Each script is sent to Google's Gemini TTS API with configurable parameters for voice type, language, speaking rate, and pitch. The API returns high-quality audio in raw format that captures natural intonation and expression.

3. Audio Processing

The raw audio is processed locally using FFmpeg to convert it into standard WAV format with optimal compression settings. This ensures compatibility with video editing software and maintains audio quality throughout the pipeline.

4. Cloud Storage

Processed audio files are uploaded to a designated Google Drive folder with organized naming conventions. The workflow creates a structured hierarchy based on project, date, or content type for easy retrieval.

5. Tracking & Completion

Finally, the original Google Sheet is updated with the Drive link to each generated audio file and marked as complete. This creates an audit trail and prevents duplicate processing of the same scripts.

What You'll Need

  1. A self-hosted n8n instance (this workflow uses Execute Command and Read/Write Files nodes)
  2. FFmpeg installed on your n8n server for audio conversion
  3. Google Cloud account with Gemini API access enabled
  4. Google Sheets containing your video scripts
  5. Google Drive for storing generated audio files
  6. Basic understanding of API keys and service account setup

Important: This is an advanced workflow requiring local FFmpeg installation and a self-hosted n8n instance. It will not work on n8n Cloud due to the Execute Command node requirements.

Quick Setup Guide

  1. Import the template into your n8n instance and activate the workflow
  2. Connect your Google Sheets node to the spreadsheet containing your video scripts
  3. Configure the HTTP Request node with your Gemini API key and preferred voice parameters
  4. Set the local file path in the Read/Write Files node where temporary audio will be stored
  5. Connect Google Drive and specify the destination folder for final audio files
  6. Test with a single script to verify the complete pipeline works end-to-end
  7. Schedule or trigger the workflow to run automatically as new scripts are added
