Generate & Email Custom NDA Documents from Web Forms

Automate the creation and delivery of personalized Non-Disclosure Agreements. Capture client details via a web form, generate a professional Word document, and email it instantly—no manual work required.

What This Workflow Does

Manually creating and sending Non-Disclosure Agreements (NDAs) is a repetitive, time-consuming task that slows down business operations. Every new client, freelancer, or partner requires a customized document with their specific details inserted—a process prone to errors and delays.

This automation solves that by turning a web form submission into a finished, professional NDA delivered via email in minutes. When someone fills out your online NDA request form, the workflow automatically populates a legal document template with their information, converts it to a Microsoft Word (.docx) file, and emails it directly to the recipient. It eliminates copy-pasting, ensures consistency, and provides instant service.

The system handles the entire lifecycle: from serving the landing page with the form, processing the submission, generating the document with proper clauses (confidentiality definitions, obligations, exclusions, duration, return of information, and governing law), to final delivery. It’s ideal for law firms, agencies, startups, and any business that regularly exchanges NDAs.

How It Works

The workflow is a seamless, multi-step pipeline that transforms a simple form fill into a delivered legal document.

1. Form Submission & Data Capture

The process begins with a user-friendly landing page served by n8n. The visitor enters their details—first name, last name, address, city, country, etc.—into a web form. Upon submission, this data is sent as structured JSON to a dedicated webhook endpoint within the workflow.

2. Data Processing & Template Assignment

The workflow receives the form data and maps each field to corresponding variables. It then passes this information to an HTML template node, which acts as your NDA document blueprint. The template is pre-written with all standard legal clauses, with placeholders for the recipient's personal and company details.

3. Document Generation (HTML to DOCX)

This is the core technical step. The populated HTML template is sent to a document conversion service (like the CustomJS HTML to Docx node). This service renders the HTML with all the formatted text, styling, and inserted data, and outputs a fully formatted Microsoft Word (.docx) file. This file is ready for signing and is universally accessible.

4. Automated Email Delivery

The generated .docx file is attached to a pre-configured email. The workflow uses an SMTP node (compatible with Gmail, Outlook, SendGrid, etc.) to send the NDA directly to the requester or a designated internal contact. The email includes a polite message confirming the document is attached and ready for review.

5. Confirmation & Logging

After successful sending, the workflow can log the transaction, send an internal notification to your team, or even trigger a follow-up task in a project management tool. This creates a full audit trail for every NDA generated.

What You'll Need

  1. A self-hosted n8n instance or n8n.cloud account to run the workflow.
  2. Access to the CustomJS "HTML to Docx" node, which requires a CustomJS API key from customjs.space (for self-hosted n8n).
  3. SMTP credentials for sending email (e.g., from Gmail, Outlook, Amazon SES, or SendGrid).
  4. A finalized NDA template in HTML format that you want to auto-populate. The template should have clear placeholder fields for the data you collect.
  5. A public-facing URL for your n8n instance if you want to expose the form landing page directly (alternatively, you can trigger the workflow from other form tools like Typeform or Google Forms).

Quick Setup Guide

Get this automation running in your environment in under 30 minutes.

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure Webhook URLs: The workflow contains two Webhook nodes. Activate them to generate unique, public URLs. Note the URL for the "FormData Endpoint."
  3. Update the HTML Template: Open the "HTML for Landingpage" node. Locate the form's `action` attribute and replace the placeholder URL with your actual "FormData Endpoint" webhook URL from step 2.
  4. Set Up Document Conversion: In the "HTML to Docx" node, configure the CustomJS credential with your API key from customjs.space. Ensure your self-hosted n8n has the CustomJS node installed.
  5. Configure Email Settings: In the "Send email" node, set up your SMTP credentials. Update the "From" and "To" email addresses, subject line, and email body text to match your needs.
  6. Test the Flow: Activate the workflow. Open the "Landingpage Endpoint" URL in your browser, fill out the test form, and submit. Check your inbox for the automatically generated and sent NDA document.
