What This Workflow Does

This automation solves a common content accessibility problem: many readers prefer audio content but creating it manually is time-consuming. The workflow automatically processes WordPress articles, generates either summaries or full transcriptions using AI, converts them to high-quality speech, and uploads the audio files back to your website.

By adding audio versions to your content, you cater to auditory learners, increase engagement metrics, and provide additional value without proportional increases in production time. The system handles everything from content extraction to final publication, ensuring consistent quality across all your audio content.

The automation is particularly valuable for content teams, publishers, and educational websites looking to repurpose written material across multiple formats efficiently.

How It Works

The workflow follows a logical sequence that transforms text content into published audio files.

1. Article Retrieval

The system connects to your WordPress REST API using secure credentials and fetches the specified article by its post ID. It extracts the complete content, including title, body text, and metadata, preparing it for processing.

2. Content Processing

An AI language model (GPT-4o-mini) analyzes the article and generates either a concise summary or a full transcription, depending on your configuration. The summary focuses on key takeaways for time-pressed audiences, while the transcription serves users who want complete content access.

3. Audio Generation

The processed text is sent to Eleven Labs' text-to-speech API, which converts it into natural-sounding audio. You can customize voice characteristics, speaking speed, and emotional tone to match your brand identity.

4. File Upload & Integration

The generated MP3 file is uploaded to your WordPress media library. The workflow then updates the original article to include an embedded audio player, making the content immediately accessible to visitors.

What You'll Need

  1. WordPress REST API Access: Admin credentials with permission to read and update posts via the WordPress REST API.
  2. Eleven Labs API Key: An account with Eleven Labs for text-to-speech conversion (they offer a free tier for testing).
  3. OpenAI API Access: For the AI summarization/transcription step (GPT-4o-mini offers excellent quality at reasonable cost).
  4. n8n Instance: Either self-hosted n8n or n8n.cloud account to run the workflow.
  5. Content Strategy Decision: Determine whether your audience prefers summaries or full transcriptions for audio content.

Pro tip: Start with summaries for most articles—they're faster to generate, cheaper to produce, and often preferred by time-constrained audiences. Reserve full transcriptions for cornerstone content or in-depth guides.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download & Import: Download the template file and import it into your n8n instance through the workflow import function.
  2. Configure WordPress Credentials: Set up the WordPress node with your site URL, username, and application password for API access.
  3. Connect AI Services: Add your OpenAI and Eleven Labs API keys to their respective nodes in the workflow.
  4. Customize Processing Logic: Adjust the AI prompt to match your preferred summary length, tone, and focus areas.
  5. Test with Sample Content: Run the workflow on a test article to verify audio quality and integration before automating your entire catalog.
  6. Schedule Automation: Set up triggers to automatically process new articles as they're published or batch-process existing content.
