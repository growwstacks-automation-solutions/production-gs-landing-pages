What This Workflow Does

Managing Azure virtual machines manually is time-consuming and error-prone. IT teams struggle to correlate VM state changes with performance issues, cost spikes, and security events across multiple subscriptions. This creates reactive firefighting instead of proactive management.

This AI-powered automation solves that by providing an intelligent chat assistant that understands natural language questions about your Azure environment. Ask "What happened to our production VMs last night?" or "Show me cost anomalies from last week," and the AI gathers data from Azure APIs, analyzes patterns, and generates comprehensive timeline reports.

The workflow transforms raw Azure Monitor metrics, Activity Log events, and VM configurations into actionable business intelligence. Instead of digging through separate portals, you get consolidated narratives that explain what changed, why it mattered, and what to do next—all through a simple chat interface.

How It Works

Step 1: Natural Language Query Processing

The workflow starts with a chat trigger that accepts user questions in plain English. Google Gemini AI parses the intent and determines which Azure data sources to query based on the question context.

Step 2: Multi-Source Azure Data Collection

The AI agent uses six specialized tools to gather comprehensive VM data: resource group listings, VM instance views, performance metrics (CPU, memory, disk I/O), network statistics, activity log events, and cost management data.

Step 3: Timeline Analysis and Correlation

Google Gemini analyzes the collected data, identifying patterns, anomalies, and correlations across time. It connects VM state changes with performance impacts, cost implications, and related security or deployment events.

Step 4: Human-Readable Report Generation

The AI synthesizes findings into detailed timeline reports with executive summaries, technical details, and actionable recommendations. Reports can be delivered via chat, email, or saved to shared storage.

What You'll Need

  1. n8n instance (cloud or self-hosted version 1.0+)
  2. Azure subscription with virtual machines to monitor
  3. Microsoft Azure Monitor OAuth2 API credentials with read permissions
  4. Google Gemini API credentials (Gemini Pro or higher recommended)
  5. Proper Azure RBAC permissions (Reader role on subscription/resource groups)
  6. Network access from n8n to Azure API endpoints

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure credentials: Add Azure Monitor OAuth2 and Google Gemini API credentials in n8n's credential store.
  3. Update workflow parameters: In the "Set Common Variables" node, replace the placeholder with your actual Azure subscription ID.
  4. Test the chat trigger: The workflow automatically generates a webhook URL for receiving chat messages.
  5. Validate data access: Send a test query like "List my VMs" to ensure Azure API connectivity works correctly.
  6. Customize reporting: Adjust the AI prompt templates to match your organization's terminology and reporting standards.

Security note: Always use Azure Managed Identities where possible instead of service principals. Implement least-privilege access—the workflow only needs read permissions. Store all credentials in n8n's encrypted credential store, never in plaintext.
