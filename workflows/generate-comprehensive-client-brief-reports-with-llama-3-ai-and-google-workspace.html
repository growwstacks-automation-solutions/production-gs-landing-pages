What This Workflow Does

This automation solves the time-consuming process of manually reviewing client documents and creating structured brief reports. Professionals in consulting, legal, and marketing agencies often spend hours extracting key information from client submissions and formatting it into usable briefs.

The workflow automatically processes any PDF, DOCX, or TXT files uploaded to a designated Google Drive folder, analyzes the content using Llama 3 AI, and generates a comprehensive client brief with key insights, summaries, and action items - saving 2-3 hours per client engagement.

How It Works

1. Document Upload Trigger

The workflow activates when a new file is added to your specified Google Drive folder for client briefs. It supports common document formats including PDFs, Word documents, and plain text files.

2. Content Extraction

The system extracts all readable text from the uploaded document, handling OCR for scanned PDFs and preserving formatting from structured documents.

3. AI Analysis with Llama 3

Llama 3 processes the extracted text to identify key themes, requirements, pain points, and action items. The AI is configured to focus on business-relevant information.

Pro tip: For best results, train your team to include clear section headings in client submissions. The AI uses these to better structure the output brief.

4. Report Generation

The system compiles the AI analysis into a professionally formatted Google Doc with sections for executive summary, key findings, recommended actions, and appendices with source references.

Quick Setup Guide

  1. Download and import the JSON workflow file into your n8n instance

  3. Configure your Llama 3 API credentials (or alternative AI service)
  4. Customize the Google Docs template to match your preferred brief format
  5. Test with sample documents and refine the AI prompt as needed
