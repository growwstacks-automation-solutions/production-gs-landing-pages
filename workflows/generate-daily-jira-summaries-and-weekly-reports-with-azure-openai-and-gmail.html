What This Workflow Does

This automation solves the time-consuming manual process of creating daily standup reports and weekly sprint summaries from Jira. Instead of managers spending hours compiling status updates, the workflow automatically extracts completed tasks, identifies blockers, and generates human-readable summaries using Azure OpenAI's advanced natural language capabilities.

The system runs twice daily - once for end-of-day summaries and once for comprehensive weekly reports. It first queries Jira for updated tickets, then processes the data through Azure OpenAI (GPT-4o-mini) to generate insightful analysis. Finally, it delivers formatted Excel reports to designated stakeholders via Gmail, keeping everyone aligned without manual effort.

How It Works

1. Jira Data Extraction

The workflow starts by executing JQL queries to fetch updated issues from your specified projects. It filters for tickets modified in the last 24 hours (for daily reports) or last 7 days (for weekly summaries). The system captures key fields like status changes, assignees, comments, and time tracking data.

2. AI Analysis & Summarization

Raw Jira data gets sent to Azure OpenAI where GPT-4o-mini analyzes patterns and generates natural language summaries. The AI identifies completed work, new blockers, priority shifts, and potential risks. For weekly reports, it adds velocity calculations and sprint health assessments.

3. Excel Report Generation

The workflow compiles both raw data and AI insights into formatted Excel sheets. Daily reports focus on immediate action items while weekly versions include trend charts, burn-down analysis, and predictive metrics. All reports maintain consistent branding and structure.

4. Email Delivery

Finally, the system attaches reports to Gmail messages sent to predefined distribution lists. Daily summaries go to team members for standup preparation, while weekly reports reach managers and stakeholders. The template includes options for Slack/Teams notifications as well.
