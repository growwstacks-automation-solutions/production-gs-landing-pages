What This Workflow Does

Creating consistent, high-quality LinkedIn content is time-consuming. Reading articles, extracting key points, crafting engaging posts, and getting team approval can take hours each week. This automation solves that by turning any webpage URL into a polished LinkedIn-ready post in minutes.

The workflow automatically scrapes web content, uses GPT-4 to generate LinkedIn-optimized posts, sends them to Slack for human review, and handles revision requests. It's perfect for content teams, marketers, and founders who want to share industry insights without the manual work.

How It Works

1. Input & Web Scraping

Users submit a webpage URL and optional instructions (like tone or key points to emphasize). The workflow uses Airtop to scrape and clean the page content, extracting the main text and headlines.

2. AI Post Generation

GPT-4 analyzes the scraped content and generates a LinkedIn post following best practices: educational tone, storytelling elements, relevant hashtags, and a clear call-to-action. It can create multiple variations based on instructions.

3. Slack Review & Collaboration

The generated post is sent to a designated Slack channel with interactive buttons for "Approve," "Request Revisions," or "Decline." Team members can review and provide feedback directly in Slack.

4. Revision Handling

If revisions are requested, the workflow captures the feedback, reprocesses the content with updated instructions, and resubmits for approval—all without manual intervention.

5. Final Delivery

Once approved, the post is delivered back to Slack, ready to be copied and published on LinkedIn. The workflow can be extended to auto-publish or save to a content calendar.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. Airtop API key (free tier available for web scraping)
  3. OpenAI API credentials with GPT-4 access
  4. Slack workspace with OAuth app configured
  5. Designated Slack channel for post reviews

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance.
  2. Configure credentials for Airtop, OpenAI, and Slack in n8n's credentials management.
  3. Update the Slack channel ID in the workflow to match your team's review channel.
  4. Test with a sample URL like your latest blog post to verify the scraping and generation work correctly.
  5. Customize the GPT-4 prompt if you want to adjust the post style, length, or formatting rules.
  6. Activate the workflow and start submitting URLs through the webhook or form trigger.
