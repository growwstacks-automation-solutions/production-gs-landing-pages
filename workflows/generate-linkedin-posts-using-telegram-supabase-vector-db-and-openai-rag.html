What This Workflow Does

Creating consistent, high-performing LinkedIn content is time-consuming and unpredictable. Most businesses struggle with content fatigue, inconsistent posting, and guessing what will resonate with their audience. This intelligent automation solves that by building a self-learning content system.

The workflow has two interconnected modules. First, a Telegram-based scraper that continuously collects viral LinkedIn posts and stores them in a Supabase vector database with OpenAI embeddings. Second, a web form that uses Retrieval-Augmented Generation (RAG) to analyze your saved content and generate new posts that match proven successful patterns.

Instead of starting from scratch each time, you're creating content informed by actual performance data. The system learns what hooks work in your industry, which structures get engagement, and what CTAs drive action—then applies those patterns to generate optimized posts automatically.

How It Works

1. Content Collection via Telegram

When you find a LinkedIn post that's performing well, simply send the URL to your Telegram bot. The workflow validates the link, scrapes the public content (respecting LinkedIn's terms), converts it to embeddings, and stores it in your Supabase vector database with metadata about engagement patterns and content structure.

2. Multi-Agent AI Analysis

When you need to create new content, you provide a hook or topic via the web form. Three specialized AI agents work sequentially: the Hook Analyzer extracts key elements, the Structure Agent creates a 5-section outline, and the Post Generator uses RAG to retrieve similar viral posts and apply their successful patterns to your new content.

3. Pattern-Based Generation

The system queries your vector database for the 5 most similar successful posts, analyzes their hooks, storytelling techniques, formatting, and CTAs, then generates new content that follows those proven patterns while being original and tailored to your specific topic and voice.

4. Publication or Export

The final post can be automatically published to LinkedIn via API or delivered as formatted text for manual review and posting. The workflow maintains your brand voice while optimizing for engagement based on real performance data from your industry.

  6. Basic understanding of API key management

Pro tip: Start by collecting 20-30 high-performing posts in your niche before generating new content. This gives the RAG system enough quality examples to identify meaningful patterns rather than surface-level trends.

Quick Setup Guide

  1. Import the template into your n8n instance using the download button above.
  2. Create your Supabase project and enable the vector extension with: CREATE EXTENSION vector;
  3. Set up your Telegram bot via @BotFather and add the token to the "On Telegram Message" trigger node.
  4. Configure OpenAI credentials in both embedding nodes and the language model nodes.
  5. Test the Telegram scraper by sending a LinkedIn post URL to your bot—you should get a confirmation message.
  6. Access the web form from the "LinkedIn Form" webhook node URL and start generating posts.
  7. Optional: Add LinkedIn OAuth2 credentials to the "Create a post" node for automatic publishing.
