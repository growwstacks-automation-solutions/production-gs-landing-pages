What This Workflow Does

For travel agencies and tour operators, responding to customer inquiries with personalized itinerary options is time-consuming and repetitive. This workflow automates that entire process. When a potential customer emails asking for tour recommendations—mentioning preferences like destination, budget, travel dates, and interests—the system automatically reads the email, understands the request using AI, searches your database of tour packages using semantic matching, and replies with the top 3 most relevant options.

It transforms a manual, hours-long research and proposal process into a fully automated, seconds-long operation. The AI doesn't just match keywords; it understands context, nuance, and intent, ensuring recommendations feel personal and consultative, not robotic. This allows your team to focus on closing sales and managing complex bookings, while the AI handles the initial qualification and suggestion phase.

How It Works

The automation follows a logical, multi-step process that mimics a skilled travel agent's workflow.

1. Email Trigger & Classification

The workflow starts by monitoring a dedicated inbox (like tours@yourcompany.com) using the IMAP Email Trigger. When a new email arrives, it's immediately captured. A Text Classifier node then analyzes the email content to determine if it's a genuine tour request, a follow-up, or other correspondence, ensuring only relevant inquiries proceed.

2. Data Extraction & Structuring

A Code node parses the email to extract key details: the sender's email address, subject line, and the body text. It structures this data into a clean format, isolating the customer's stated needs—destination, number of travelers, budget range, preferred activities, and travel dates.

3. AI Agent & Prompt Engineering

The structured request is passed to a custom "Tour Recommendation AI Agent." This agent, powered by OpenAI's GPT-4o or OpenRouter, is pre-configured with your company's tone, brand voice, and sales guidelines. It formulates an intelligent search query based on the extracted data, ready to query your tour database.

4. Vector Database Search

The AI-generated query is converted into a numerical vector (embedding) using OpenAI's Embeddings node. This vector is then sent to Pinecone, a vector database where all your tour packages are stored as similar embeddings. Pinecone performs a lightning-fast semantic search, returning the most conceptually similar tour packages, not just those with matching keywords.

5. Recommendation Generation & Email Dispatch

The AI agent receives the top matching packages from Pinecone. It then crafts a friendly, detailed, and personalized email response. It explains why the selected packages are a good fit, highlights key features, and includes clear next-step instructions. Finally, the Send Email node delivers this tailored proposal directly back to the inquiring customer.

Pro tip: Use a "Simple Memory" node within the workflow to maintain context if a customer replies to the automated email. This allows the AI to reference the previous conversation, creating a continuous, intelligent dialogue.

What You'll Need

  1. An n8n instance: Either n8n.cloud or a self-hosted setup.
  2. Email Credentials: IMAP/SMTP details for the inbox you want to monitor and send from.
  3. OpenAI API Key: For accessing GPT-4o to understand requests and generate responses.
  4. Pinecone API Key & Index: A Pinecone account with a pre-populated index containing your tour package data converted into vector embeddings.
  5. Tour Package Database: Your catalog of tours, including descriptions, pricing, destinations, and inclusions, prepared for vectorization.

Quick Setup Guide

Import and configure this workflow in under 30 minutes to start automating your tour inquiries.

  1. Download & Import: Click the "Download Template" button above and import the JSON file into your n8n instance.
  2. Configure Email Trigger: In the "Email Trigger (IMAP)" node, enter your email server credentials (host, port, user, password).
  3. Set Up AI Services: In the OpenAI and OpenRouter nodes, add your respective API keys. Choose your preferred model (GPT-4o is recommended).
  4. Connect Pinecone: In the Pinecone Vector Store and Embeddings nodes, input your API key, environment, and index name where your tour data is stored.
  5. Customize the AI Agent: Edit the "Tour Recommendation AI Agent" node to reflect your company's brand voice and any specific sales scripts or disclaimers.
  6. Test & Activate: Send a test email to your monitored inbox. The workflow should trigger, process the request, and send a reply. Tweak the prompt in the AI agent based on the output quality.
