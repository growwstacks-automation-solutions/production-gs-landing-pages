Generate PII-safe Helpdocs from Crisp Support chats

Automatically create help articles from resolved Crisp chats using AI, while protecting customer privacy.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Customer support teams spend countless hours answering repetitive questions in chat platforms like Crisp. Once a chat is resolved, that valuable solution disappears—unless someone manually writes it into a help article. This creates documentation gaps, inconsistent knowledge bases, and wasted effort.

This automation solves that by automatically transforming resolved Crisp chats into structured help articles. It listens for chat completion events, extracts the conversation, formats it into clear Q&A pairs, uses AI to generate a polished article while removing personal information (PII), and saves it to your documentation system for review. Your knowledge base grows automatically from actual customer interactions.

How It Works

Step 1: Capture Chat Events

A webhook trigger listens for Crisp events—when messages are sent or received, and especially when a chat is marked "resolved." This ensures the workflow only processes completed conversations with actual solutions.

Step 2: Store Conversation History

Each message is stored in a data table organized by session ID. This creates a complete thread of the conversation, preserving both customer questions and support agent answers.

Step 3: Process on Resolution

When a chat reaches "state:resolved," the workflow fetches the full thread from storage. It then passes the raw conversation to a formatting node that structures messages into "Q:" (customer question) and "A:" (agent answer) pairs.

Step 4: AI Article Generation

The formatted Q&A pairs are sent to an AI model (GPT-4.1-mini) configured to create a professional help article. The AI is instructed to redact personal information—names, emails, specific identifiers—while preserving the core problem and solution.

Step 5: Save for Review

The generated article is saved to another data table with a "publish = false" flag. This creates a draft repository where your team can review, edit, and approve articles before they go live in your help documentation.

What You'll Need

  1. A Crisp workspace with webhook access (Settings → Advanced → Webhooks)
  2. An n8n instance (cloud or self-hosted) with Data Tables enabled
  3. OpenAI API credentials or access to another LLM service
  4. A documentation destination (Helpdocs, internal wiki, or similar)

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the Crisp webhook node with your Crisp workspace webhook URL.
  3. Set up your Data Tables in n8n for storing messages and generated articles.
  4. Connect the AI node with your OpenAI API key or preferred LLM service.
  5. Test by resolving a Crisp chat and verifying the article appears in your draft table.
  6. Adjust the AI prompt if needed to match your documentation style and redaction rules.

Pro tip: Add a Slack or email notification node after the "store-doc" step to alert your team when new articles are ready for review. This creates a smooth approval workflow.
