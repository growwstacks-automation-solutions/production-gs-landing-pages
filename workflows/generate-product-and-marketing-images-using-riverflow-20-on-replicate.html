What This Workflow Does

This automation solves a critical bottleneck for e-commerce brands, marketing teams, and creative agencies: producing high-quality visual content at scale. Instead of manually designing each product image, social media graphic, or ad banner, this workflow connects directly to the Riverflow 2.0 AI model through the Replicate API to generate professional images automatically.

You provide simple text prompts and parameters (like product description, desired style, dimensions), and the system returns ready-to-use images. It handles both generation from scratch and editing existing images. The workflow is built to run multiple variations in parallel, enabling A/B testing and bulk asset creation in minutes rather than days.

How It Works

1. Input Collection & Sanitization

A user-friendly input form captures all required parameters for the Riverflow 2.0 model: product description, style instructions, resolution, aspect ratio, number of outputs, and optional reference images. These inputs are cleaned and formatted for the API.

2. Parallel API Processing

The workflow triggers multiple sub-processes that call the Replicate API simultaneously. Each sub-process sends a POST request to start image generation, then polls with GET requests until completion. This parallel architecture dramatically speeds up batch creation.

3. Output Management & Delivery

Generated image URLs are stored in a central n8n data table. The main workflow monitors this table, collects all outputs, and makes the final images available for download or direct integration into other systems like Shopify, Google Drive, or social media schedulers.
