What This Workflow Does

This automation solves the challenge of consistently producing high-quality, SEO-optimized blog content at scale. Content marketers and bloggers often struggle with research time, maintaining SEO standards, and publishing frequency. This workflow automates the entire process from keyword research to publishing.

The system pulls your content strategy from Google Sheets, conducts in-depth research via Perplexity AI, generates structured articles using GPT-4, performs final edits, converts to WordPress-compatible HTML, and publishes as drafts for review. It maintains SEO best practices throughout while saving 80-90% of the time typically spent on manual content creation.

How It Works

1. Keyword Input

Pulls primary keywords, search intent, and related terms from your Google Sheets content calendar using OAuth2 authentication.

2. AI Research

Uses Perplexity AI to gather authoritative sources, statistics, and current information about each topic to inform content creation.

3. Content Generation

GPT-4 creates structured articles with title variants, introductions, key takeaways, detailed sections, conclusions, and FAQs based on your brand voice guidelines.

4. Quality Refinement

Performs automated edits for readability, inserts internal/external links, and ensures proper keyword placement before final assembly.

5. WordPress Publishing

Converts the polished content to HTML and publishes it to your WordPress site as drafts via the REST API for final review.

Pro tip: Use the Set node to customize the AI prompts for your specific industry voice and content standards before deploying.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. OpenAI API key (for GPT-4 access)
  3. Perplexity API credentials
  4. Google Sheets with your content calendar
  5. WordPress installation with REST API enabled

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure the Google Sheets node with your content calendar
  3. Add your OpenAI and Perplexity API keys
  4. Set up WordPress credentials in the HTTP Request nodes
  5. Test with a single keyword before running the full workflow
