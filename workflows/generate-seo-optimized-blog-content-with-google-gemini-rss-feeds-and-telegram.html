What This Workflow Does

This n8n workflow solves the time-consuming challenge of consistent content creation by automating the entire pipeline. It monitors your specified RSS feeds (like industry news or competitor blogs) for new articles, then uses Google Gemini AI to generate comprehensive, SEO-optimized blog posts inspired by that content. The system creates unique cover images, assembles everything into a structured format, and delivers the final content package to stakeholders via Telegram—all without manual intervention.

For marketing teams, content agencies, and solo entrepreneurs, this automation turns content creation from a sporadic, labor-intensive task into a reliable, scalable system. You maintain a steady stream of fresh, relevant content while freeing up creative resources for strategy and refinement rather than repetitive writing tasks.

How It Works

1. RSS Feed Monitoring

The workflow starts by periodically checking configured RSS feeds from sources like Search Engine Journal, TechCrunch, or your industry-specific blogs. When new articles are detected, they're queued for processing.

2. Duplicate Prevention

Each article URL is checked against an AWS DynamoDB database to ensure it hasn't been processed before. If it's a duplicate, the workflow skips it and logs the event, preventing redundant content generation.

3. AI Content Generation

New articles are sent to Google Gemini with a detailed prompt that instructs the AI to create a complete blog post including title, meta description, SEO slug, image generation prompt, and full markdown body with introduction, subheadings, conclusion, and FAQ section.

4. Content Processing & Image Creation

The AI's raw response is cleaned and parsed into valid JSON. The image generation prompt is then sent to Google Gemini's image model to create a 1200x630 cover image, which is uploaded to cloud storage (like Cloudflare R2).

5. Final Assembly & Delivery

The text content and image URL are merged into a complete content package, converted to a JSON file named with the SEO slug, and sent as a document to designated Telegram recipients for review or immediate publishing.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Google Gemini API key for both text and image generation
  3. AWS DynamoDB table for duplicate tracking (or alternative database)
  4. Cloud storage like Cloudflare R2, AWS S3, or similar for image hosting
  5. Telegram bot token and chat ID for content delivery
  6. RSS feed URLs from your target content sources

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance.
  2. Configure credentials: Set up credentials for Google Gemini, AWS DynamoDB, your cloud storage, and Telegram in n8n's credentials management.
  3. Update RSS feeds: Replace the sample RSS feed URLs in the trigger nodes with your target sources.
  4. Adjust AI prompts: Review and customize the Google Gemini prompts to match your brand voice and content requirements.
  5. Set Telegram recipients: Update the Telegram node with your target chat IDs or channel.
  6. Test execution: Run the workflow manually with a test RSS item to verify all connections work.
  7. Schedule automation: Set the RSS trigger to run at your desired frequency (hourly, daily, etc.).

Pro tip: Start with 2-3 high-quality RSS feeds rather than many low-quality sources. Monitor the AI's output for the first week and refine your prompts based on what generates the best content for your audience.
