What This Workflow Does

Documentation rot is a silent productivity killer. Teams waste hours searching for accurate information only to find outdated Confluence pages that haven't been touched in months or years. This creates confusion, leads to mistakes, and makes onboarding new team members unnecessarily difficult.

This n8n workflow solves that problem by automatically scanning your Confluence spaces for stale pages—documents that haven't been updated within your defined timeframe. It generates a comprehensive report showing exactly which pages need attention, how long they've been outdated, and provides direct links for quick cleanup.

The template supports both Confluence REST API v1 (using CQL search) and the newer v2 API (space-by-space scanning), making it compatible with various Confluence configurations. Whether you're managing technical documentation, company policies, or project wikis, this automation brings visibility to your documentation health.

How It Works

The workflow follows a logical sequence to identify, process, and report on stale Confluence pages.

Step 1: Configuration & Setup

The process begins with a Manual Trigger or Schedule Trigger. A Set node configures all necessary variables: your Atlassian domain, API credentials, space keys to scan, and the cutoff period (e.g., 90 days). You can toggle between API v1 and v2 depending on your Confluence setup.

Step 2: Data Retrieval

For API v2, the workflow first retrieves all specified spaces, then fetches pages from each space. For API v1, it uses Confluence Query Language to directly search for pages modified before your cutoff date. HTTP Request nodes handle the API calls with proper authentication.

Step 3: Processing & Filtering

Each retrieved page is checked against your cutoff date. The workflow calculates exactly how many days overdue each page is and formats the data consistently. A Switch node handles different API response structures between v1 and v2.

Step 4: Report Generation

All identified stale pages are aggregated into a single, clean dataset. The workflow outputs a structured report containing page titles, statuses, last updated dates, days overdue, and direct URLs. This data is ready for notification or export.

Pro tip: Set this workflow to run monthly on a schedule trigger. This creates consistent documentation hygiene without manual intervention, ensuring your team always knows which pages need updating.

What You'll Need

  1. An n8n instance (cloud or self-hosted) with HTTP Request node capabilities
  2. Confluence Cloud or Server/Data Center access with administrative or appropriate read permissions
  3. Atlassian API token with read access to the spaces you want to audit
  4. Space keys for the Confluence spaces you want to monitor (e.g., "DOCS", "ENG", "HR")
  5. A decision on cutoff period—typically 90 days for fast-moving projects, 180 days for stable documentation

Quick Setup Guide

Follow these steps to implement this automation in under 15 minutes:

  1. Download the template using the button above and import it into your n8n instance
  2. Create an Atlassian API token in your account settings with read permissions for Confluence
  3. Configure the Set node with your domain, space keys, cutoff days, and API version preference
  4. Add your API credentials to all HTTP Request nodes using HTTP Basic Auth
  5. Test with a single space first to verify the connection and data retrieval works correctly
  6. Add your notification method—connect to Email, Slack, Microsoft Teams, or Google Sheets to receive the reports
  7. Set a schedule (optional) to run monthly or quarterly for ongoing documentation hygiene
