What This Workflow Does

This automation solves the classic "what should we do today?" problem by generating a smart, weather-aware date plan in minutes. Instead of manually searching for ideas, checking the forecast, and planning a route, this workflow does it all automatically.

It starts with a location, finds highly-rated nearby spots using Google Places, fetches the current weather, uses AI to match an emoji to the conditions, crafts a friendly bilingual itinerary (English & Japanese), and posts the complete plan with a Google Maps walking link directly to a Slack channel. It turns scattered data into a cohesive, actionable plan.

The business value is clear: it saves hours of manual research, ensures suggestions are practical given the weather, and delivers a polished, personalized output that can be used for client services, personal productivity, or as a unique engagement tool.

How It Works

The workflow is a sequenced pipeline that transforms a simple location input into a rich, shared itinerary.

Step 1: Manual Trigger & Configuration

The workflow is initiated manually (or can be scheduled). You provide the starting coordinates, search radius, and total time budget (e.g., 120 minutes). This configures the entire search scope.

Step 2: Discover Local Spots

Using the Google Places API, the workflow searches for nearby points of interest—like cafes, parks, museums, or restaurants—within the specified radius. It filters and ranks results based on user ratings and relevance.

Step 3: Fetch Real-Time Weather

Simultaneously, it calls the OpenWeatherMap API for the current conditions at the location. This data is crucial for making appropriate activity recommendations (e.g., avoid outdoor dining if it's raining).

Step 4: AI-Powered Emoji & Itinerary Generation

An AI model (via OpenRouter) performs two key tasks. First, it analyzes the weather data to assign a fitting emoji (🌤️, ☔, etc.). Second, it takes the top-ranked places and the time budget to write a charming, step-by-step itinerary in both English and Japanese, adding context and narrative.

Step 5: Compose & Share

The workflow compiles everything: the itinerary, weather emoji, and a generated Google Maps walking route link. It then formats this into a clean message and posts it to a designated Slack channel, notifying the relevant team or person.

What You'll Need

  1. An n8n instance (Cloud or self-hosted).
  2. Google Cloud Platform account with the Places API enabled and credentials (API key).
  3. OpenWeatherMap API key for current weather data.
  4. Slack workspace with a bot token (with chat:write scope) installed in your desired channel.
  5. An LLM provider API key (OpenRouter, OpenAI, etc.) for generating the itinerary and emoji.
  6. Optional: A DeepL API key if you wish to use a dedicated translation service instead of the LLM for language conversion.

Pro tip: Use n8n's built-in credential management to securely store your API keys. This keeps them out of your workflow code and makes the template portable across your team.

Quick Setup Guide

Follow these steps to import and run this automation in your own n8n environment.

  1. Download the template using the button above. You'll get a .json file.
  2. In your n8n dashboard, go to Workflows and click Import from file. Select the downloaded JSON.
  3. Configure the Set node labeled "Config". Enter your test coordinates, search radius (in meters), and total duration (in minutes).
  4. Set up credentials for each service node (Google Places, OpenWeatherMap, Slack, LLM). Use n8n's credential modals to add your API keys securely.
  5. Test the trigger by executing the workflow once manually. Check the Slack node output to see the generated itinerary in your channel.
  6. Activate the workflow and consider adding a schedule trigger (e.g., weekly) or a webhook to make it fully automated.
