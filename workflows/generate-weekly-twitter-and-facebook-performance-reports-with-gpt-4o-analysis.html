What This Workflow Does

Manual social media reporting is a time-consuming, error-prone task that distracts marketing teams from strategic work. This automation solves that by creating a complete, hands-off system for weekly performance analysis across Twitter (X) and Facebook.

The workflow retrieves real-time metrics from both platforms' APIs, validates the data quality, merges everything into a unified dataset, then uses GPT-4o to generate a professional HTML report with actionable insights. It stores historical data in Notion for trend tracking, emails the report to your team, and posts key highlights to Slack—eliminating hours of manual work every week.

How It Works

1. Data Collection & Validation

The workflow starts by fetching the latest performance metrics from the Twitter (X) API and Facebook Graph API simultaneously. It checks for data completeness—ensuring follower counts, engagement rates, and impression data are present—and logs any missing or invalid entries to a Google Sheets error log for transparency.

2. Data Processing & Normalization

Raw API responses from different platforms have varying formats. A custom JavaScript node normalizes this data into a clean, consistent JSON structure, mapping metrics to standardized field names (like "engagement_rate" and "impressions") so the AI can analyze them effectively.

3. AI-Powered Analysis & Report Generation

The normalized data is sent to Azure OpenAI's GPT-4o model with a carefully crafted prompt. The AI analyzes week-over-week changes, identifies top-performing content, compares platform performance, and generates a visually formatted HTML report with charts, insights, and recommendations in your brand's tone.

4. Distribution & Storage

The completed HTML report is emailed to designated team members via Gmail. Key metrics and insights are also posted to a Slack channel for immediate visibility. Meanwhile, all structured data is appended to a Notion database, creating a searchable historical record of your social media performance.

What You'll Need

  1. API Credentials: Twitter OAuth2 keys and Facebook Graph API token for data access.
  2. AI Service: Azure OpenAI credentials with GPT-4o model access.
  3. Communication Tools: Slack bot token (with chat:write permission) and Gmail OAuth2 credentials.
  4. Data Storage: Notion API integration with a "Growth Chart" database ready.
  5. Error Logging: Google Sheets setup for tracking data validation issues.
  6. n8n Instance: A running n8n environment (cloud or self-hosted) to import and execute the workflow.

Quick Setup Guide

  1. Download & Import: Click the download button above to get the JSON file, then import it into your n8n instance.
  2. Configure Credentials: Set up all required API credentials (Twitter, Facebook, Azure OpenAI, Notion, Gmail, Slack, Google Sheets) as environment variables or resource credentials in n8n.
  3. Customize Platform IDs: Update the HTTP Request nodes with your specific Twitter username and Facebook Page ID.
  4. Test Data Flow: Run the workflow manually once to verify data collection from both platforms works correctly.
  5. Adjust AI Prompt: Review and modify the GPT-4o prompt in the Azure OpenAI node to match your preferred reporting style and focus areas.
  6. Set Recipients: Add your team's email addresses in the Gmail node and specify your Slack channel ID.
  7. Schedule Execution: Once tested, set the workflow to trigger automatically every Monday morning for weekly reporting.

Pro tip: Start with just the data collection and validation steps. Once you confirm clean data flows from both platforms, then enable the AI analysis and distribution steps. This phased approach helps troubleshoot any API issues before the full automation runs.
