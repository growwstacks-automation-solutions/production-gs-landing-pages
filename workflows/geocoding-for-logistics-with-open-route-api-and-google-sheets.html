What This Workflow Does

Manual address-to-coordinate conversion is a major bottleneck in logistics, transportation, and market analysis. Teams waste hours copying addresses between spreadsheets and mapping tools, introducing errors and delaying critical decisions. This free n8n automation solves that by connecting Google Sheets directly to the OpenRouteService API.

The workflow automatically reads address lists from your spreadsheet, sends each address to the geocoding service, extracts precise latitude/longitude coordinates, and writes the results back to your sheet. It also captures neighborhood and city data for deeper location intelligence. What used to take a full day of manual work now happens in minutes with perfect accuracy.

Beyond simple coordinate conversion, this automation creates a foundation for advanced logistics planning. You can now calculate actual driving distances, optimize delivery routes, analyze service area coverage, and visualize your entire supply chain network—all from data that updates automatically as your address list changes.

How It Works

The workflow follows a clean, logical sequence that ensures reliable geocoding even for large address datasets.

Step 1: Read Address Data from Google Sheets

The automation starts by connecting to your Google Sheet where addresses are stored. It reads each row containing address information and country codes, preparing the data for processing. The workflow handles authentication securely and can process hundreds or thousands of addresses in a single run.

Step 2: Loop Through Each Address Record

Using n8n's powerful iteration capabilities, the workflow processes each address individually. This ensures that if one address fails (due to formatting issues or API limits), the rest continue processing. Error handling is built in to log issues without stopping the entire workflow.

Step 3: Query OpenRouteService API

Each formatted address is sent to the OpenRouteService geocoding endpoint. The API returns comprehensive location data including precise coordinates, bounding boxes, and location metadata. The workflow includes proper rate limiting and retry logic to handle API constraints gracefully.

Step 4: Extract and Transform Location Data

The raw API response is parsed to extract the most valuable information: latitude, longitude, neighborhood, city, and country. The workflow transforms this data into a clean, structured format ready for writing back to your spreadsheet or feeding into other systems.

Step 5: Update Google Sheets with Results

Finally, the enriched data is written back to your Google Sheet in new columns. The workflow maintains data integrity by matching results to the original rows and provides clear status indicators for each processed address.

Pro tip: Add a "Last Updated" timestamp column to track when each address was geocoded. This helps identify stale data that might need refreshing if locations have changed.

What You'll Need

  1. A Google Sheet with addresses you need to geocode (include columns for address and country code)
  2. Free OpenRouteService API key (sign up at openrouteservice.org - generous free tier)
  3. n8n instance (cloud or self-hosted) with Google Sheets and HTTP Request nodes available
  4. Basic spreadsheet skills to format your address data correctly
  5. 10 minutes to import and configure the workflow

Quick Setup Guide

Get this automation running in your environment with these simple steps:

  1. Download the template using the button above and import it into your n8n instance
  2. Create your OpenRouteService account and generate an API key (free tier is sufficient for most use cases)
  3. Prepare your Google Sheet with addresses in one column and two-letter country codes in another
  4. Connect the Google Sheets node to your spreadsheet by authenticating with Google
  5. Add your API key to the HTTP Request node configuration
  6. Test with a few addresses first, then run the full automation
  7. Schedule regular runs to keep your location data current as addresses change
