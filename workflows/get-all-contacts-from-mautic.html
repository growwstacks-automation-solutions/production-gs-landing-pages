What This Workflow Does

Manually exporting contact lists from Mautic is a repetitive, time-consuming task that distracts marketing and sales teams from higher-value work. This automation solves that by connecting directly to the Mautic API to retrieve all your contacts automatically.

The workflow extracts contact details—including names, email addresses, segment tags, and custom field data—and prepares them for use in other systems. You can schedule it to run daily, weekly, or in real-time, ensuring your downstream tools always have the latest prospect and customer information without manual intervention.

Beyond simple extraction, this template serves as a foundation for more advanced marketing operations. It enables reliable data backup, seamless CRM synchronization, and creates a single source of truth for your lead database, improving campaign targeting and sales follow-up accuracy.

How It Works

This n8n workflow uses a straightforward yet powerful sequence to connect to Mautic and process your contact data.

Step 1: Authenticate with Mautic

The workflow begins by establishing a secure connection to your Mautic instance using API credentials. This ensures that only authorized systems can access your contact data, maintaining security and compliance.

Step 2: Fetch Contact Data

It queries the Mautic API to retrieve all contacts, including their core details and any custom fields you've defined. The workflow can be configured to pull specific segments or filter by date ranges, activity, or lead score.

Step 3: Transform and Structure

Raw API data is cleaned and formatted. Duplicates are identified, empty fields are handled, and the data is structured into a consistent format (like a table) ready for export or integration.

Step 4: Output to Your Chosen Destination

The processed contact list is sent to your selected destination. This could be a Google Sheets spreadsheet for analysis, a CSV file saved to cloud storage, or directly into another application like a CRM, email marketing platform, or database.

  1. A running Mautic instance (cloud or self-hosted) with administrative access.
  2. API credentials from Mautic (Client ID, Client Secret, and your instance URL).
  3. Access to an n8n instance (self-hosted or n8n.cloud).
  4. A destination for the data (e.g., a Google Sheet, a folder in Google Drive/Dropbox, or API access to another platform like HubSpot or Airtable).
  5. Basic understanding of how to create and manage API keys in Mautic.

Pro tip: Before going live, test the workflow with a small segment of contacts. This helps verify field mappings and ensures your data output matches expectations without processing your entire database during setup.

Quick Setup Guide

Follow these steps to import and configure this automation in your n8n environment.

  1. Download the template using the button above and save the JSON file to your computer.
  2. In your n8n dashboard, go to Workflows and click Import from File. Select the downloaded JSON file.
  3. Locate the Mautic node in the imported workflow. Click on it to open its credentials settings.
  4. Add a new Mautic credential using your instance URL, Client ID, and Client Secret. (Find these in Mautic under Configuration > API Credentials).
  5. Configure the output node (e.g., Google Sheets, HTTP Request, etc.) with your destination details.
  6. Click Execute Workflow to test. Check that contacts appear correctly in your chosen destination.
  7. Finally, activate the workflow trigger (like a schedule) to run it automatically.
