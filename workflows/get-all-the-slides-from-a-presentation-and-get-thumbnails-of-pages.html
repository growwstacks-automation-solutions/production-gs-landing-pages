What This Workflow Does

Manually extracting slides and creating thumbnails from Google Slides presentations is a tedious, time-consuming task that content teams, educators, and businesses face regularly. Whether you're archiving quarterly reviews, creating training materials, or documenting client presentations, the process of screenshotting, cropping, naming, and organizing slides eats into productive work time.

This automation solves that problem completely. It connects to your Google Slides account, retrieves every slide from specified presentations, generates clean thumbnails of each page, and outputs them in an organized format. What used to take 15-20 minutes per presentation now happens automatically in under 2 minutes, with perfect consistency every time.

The workflow creates a visual archive of your presentations that's searchable, shareable, and ready for integration with other tools. It's particularly valuable for teams that need to repurpose slide content for different channels or maintain compliance records of important presentations.

How It Works

Step 1: Authentication & Presentation Selection

The workflow begins by authenticating with your Google account using OAuth 2.0. You specify which presentation to process—either by URL, presentation ID, or by searching your Google Drive. The connection uses secure, read-only permissions to ensure your data remains protected while allowing the automation to access slide content.

Step 2: Slide Content Extraction

The Google Slides node retrieves every slide from the selected presentation, including all text, images, shapes, and formatting information. It processes the presentation structure to understand slide order, layout, and content hierarchy. This step captures the complete visual and textual content of each slide for further processing.

Step 3: Thumbnail Generation

A second Google Slides node generates high-quality thumbnails for each extracted slide. You can configure thumbnail size, resolution, and format preferences. The automation creates consistent, properly cropped images that serve as perfect visual references for catalogs, documentation, or quick preview systems.

Step 4: Output & Integration

The extracted slides and generated thumbnails are output in structured formats. You can configure the workflow to save files to Google Drive, upload to Dropbox, send to other applications, or trigger notifications. Each output includes metadata like slide numbers, presentation titles, and processing timestamps for easy organization.

Pro tip: Schedule this workflow to run automatically when new presentations are added to specific Google Drive folders. This creates a real-time visual archive without any manual intervention.

What You'll Need

  1. A Google Workspace account or personal Google account with presentations to process
  2. n8n instance (cloud or self-hosted) with the Google Slides node installed
  3. Google Cloud Platform project with Slides API enabled (for service account authentication)
  4. OAuth 2.0 credentials configured in n8n for Google Slides access
  5. Destination for output files (Google Drive, Dropbox, or other storage service)
  6. Basic understanding of presentation URLs or IDs for targeting specific slides

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Download & Import: Download the template file and import it into your n8n instance using the workflow import function.
  2. Configure Google Connection: Set up the Google Slides node with your authentication credentials. Use OAuth 2.0 for user-specific access or a service account for automated processing.
  3. Specify Presentation: Enter the URL or ID of the Google Slides presentation you want to process. Test with a small presentation first.
  4. Adjust Thumbnail Settings: Configure the thumbnail size and quality in the second Google Slides node based on your needs.
  5. Set Up Output Destination: Connect nodes to save extracted content to your preferred storage service (Google Drive, Dropbox, etc.).
  6. Test & Schedule: Run the workflow once to verify everything works, then set up scheduling or triggers for automatic execution.

Pro tip: Create a dedicated Google Drive folder for processed presentations and configure the workflow to automatically organize outputs by date, presentation type, or department for easy retrieval.
