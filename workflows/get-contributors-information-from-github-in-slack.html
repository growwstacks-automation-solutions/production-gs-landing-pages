What This Workflow Does

This automation solves the common challenge of tracking developer contributions across GitHub repositories. Instead of manually checking GitHub profiles or running reports, team members can simply type a Slack command like /contributor @username to instantly retrieve relevant GitHub activity.

The workflow connects your Slack workspace to GitHub's API, fetching contributor information including commit history, pull requests, issues contributed to, and repository activity. It formats this data into a clean Slack message with relevant links back to GitHub.

How It Works

1. Slack Command Trigger

The workflow starts when a user enters a slash command in Slack. The command includes parameters like the GitHub username to look up.

2. GitHub API Query

The workflow makes authenticated requests to GitHub's API to retrieve the specified user's contribution data across configured repositories.

3. Data Processing

Raw API responses are transformed into a more digestible format, filtering unnecessary details and calculating meaningful metrics.

4. Slack Message Generation

The processed data is formatted into an attractive Slack message with interactive elements like buttons and links.

What You'll Need

  1. A Slack workspace with permissions to create slash commands
  2. GitHub account with API access
  3. n8n instance (cloud or self-hosted)
  4. Basic understanding of API authentication

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure your GitHub API credentials in the HTTP Request nodes
  3. Set up the Slack webhook URL for responses
  4. Create a Slack slash command pointing to your n8n webhook
  5. Test with /contributor yourusername in Slack
