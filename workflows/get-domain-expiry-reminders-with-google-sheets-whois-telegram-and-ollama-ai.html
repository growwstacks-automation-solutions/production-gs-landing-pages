What This Workflow Does

Forgetting to renew a domain is a costly mistake that leads to immediate website downtime, lost email functionality, and damage to brand reputation. This automation solves that by creating a hands-off monitoring system for your entire domain portfolio.

The workflow reads your domain list from Google Sheets, queries public WHOIS data for each domain's expiration date, uses AI to extract the key information reliably, and sends you a Telegram alert when a renewal is approaching. It runs daily, so you're always informed with plenty of lead time.

This is especially valuable for agencies managing client domains, IT departments overseeing corporate assets, or entrepreneurs with multiple projects. It transforms a reactive, stressful task into a proactive, managed process.

How It Works

Step 1: Scheduled Trigger & Data Fetch

A scheduler node (like Cron) triggers the workflow daily at a set time. It then connects to your designated Google Sheet, which acts as the master list of all domains you need to monitor.

Step 2: WHOIS Lookup for Each Domain

For each domain in your sheet, the workflow performs an HTTP request to a WHOIS service or API. This fetches the raw registration data, which contains the expiration date, registrar, and other details.

Step 3: AI-Powered Data Extraction

WHOIS responses are unstructured text that varies by registrar. An AI node (like Ollama or another LLM) is prompted to extract the expiration date, domain status, and registrant name from this text, outputting clean, structured data.

Step 4: Logic & Alert Decision

The workflow calculates how many days are left until expiration. If a domain is within your configured alert window (e.g., 90 days), it proceeds to create a notification message. It also checks a log to avoid sending duplicate alerts.

Step 5: Sending the Telegram Alert

A formatted message is sent to your specified Telegram chat or channel. The message includes the domain name, days until expiry, and often a direct link to renew. The notification date is then recorded back to your sheet.

What You'll Need

  1. A running n8n instance (cloud or self-hosted).
  2. A Google Sheets document with your domain list. Ensure you have columns for domain name and (optionally) last alert date.
  3. Google Service Account credentials configured in n8n to access your sheet.
  4. A Telegram Bot Token and Chat ID to receive the alerts.
  5. Access to an LLM/ AI node like Ollama (local), OpenAI, or Anthropic for information extraction.

Pro tip: For maximum reliability with large portfolios, consider using a paid WHOIS API instead of free public lookups. They offer higher rate limits and more consistent data formatting, which makes the AI extraction step even more robust.

Quick Setup Guide

  1. Download and Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded template.
  2. Configure Google Sheets: In the "Google Sheets" node, replace the placeholder Spreadsheet ID with your own. Set the correct sheet name and range (e.g., `A2:B100`).
  3. Set Up Telegram: In the "Telegram" node, add your Bot Token (created via @BotFather) and the destination Chat ID.
  4. Configure the AI Node: Connect your preferred LLM provider (Ollama, OpenAI, etc.) in the "Extract Information" node. The prompt is pre-configured, but you can tweak it.
  5. Adjust the Schedule: In the "Cron" or scheduler node, set your preferred run time (e.g., daily at 9 AM).
  6. Test and Activate: Run the workflow once manually with a test domain to ensure all connections work. Then toggle the workflow to "Active" to let it run automatically.
