What This Workflow Does

This n8n workflow template solves a common challenge in IT operations and development: extracting valuable data from command line tools and making it usable in automated processes. Many system administration tools, monitoring utilities, and DevOps scripts output data in text format that's difficult to integrate with modern applications and databases.

The template demonstrates how to execute shell commands programmatically, capture their output, and transform that data into structured JSON format. This transformation bridges the gap between legacy command-line interfaces and modern API-driven automation, enabling you to incorporate system data into your business workflows without manual intervention.

Whether you're monitoring server health, tracking deployment status, extracting log information, or gathering system metrics, this workflow provides the foundation for turning command-line data into actionable, structured information that can trigger notifications, update databases, or feed into dashboards.

How It Works

The workflow follows a logical sequence to capture, process, and utilize command line data within your automation ecosystem.

Step 1: Command Execution

The Execute Command node runs your specified shell command or script. This could be anything from system monitoring commands like df -h for disk usage, git log for version control data, or custom scripts that output system information. The node captures both standard output and error streams.

Step 2: Data Parsing and Transformation

Raw command output often contains headers, formatting characters, and inconsistent spacing. The workflow uses n8n's data transformation capabilities to clean this output—removing unnecessary lines, extracting key values, and structuring the information into a consistent format.

Step 3: JSON Conversion

Once cleaned, the data passes through a Function node or Set node that converts it into proper JSON structure. This might involve creating key-value pairs from colon-separated output, parsing tabular data into arrays of objects, or extracting specific patterns using regular expressions.

Step 4: Integration and Action

The resulting JSON data can then be used throughout your automation. Examples include sending system alerts via Slack or email when thresholds are exceeded, updating a Google Sheet with server metrics, creating tickets in Jira for issues detected, or storing historical data in a database for trend analysis.

Pro tip: For commands that already output JSON (like many modern CLI tools), you can skip the parsing step and use the output directly. Tools like jq can also be used within the command itself to pre-format output as JSON before n8n processes it.

Quick Setup Guide

Follow these steps to implement this command-line automation in your n8n instance:

  1. Import the template by downloading the JSON file and importing it into your n8n instance via the workflow import function.
  2. Configure the Execute Command node with the specific command you want to run. Test it manually first to ensure it produces the expected output.
  3. Adjust the parsing logic in the Function or Set nodes to match your command's output format. You may need to modify regex patterns or splitting logic.
  4. Set up your destination nodes to receive the JSON data—connect to your database, notification system, or API endpoints.
  5. Add error handling to manage situations where the command fails or returns unexpected output.
  6. Test the complete workflow with various scenarios to ensure reliability before putting it into production.
  7. Add scheduling using n8n's Schedule Trigger node if you need to run the command periodically.

Security note: Be cautious about which commands you automate and with what permissions. Consider creating limited-service accounts for n8n and using command whitelists where possible.
