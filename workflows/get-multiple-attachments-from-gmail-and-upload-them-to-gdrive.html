What This Workflow Does

Manually downloading email attachments and uploading them to cloud storage is a repetitive, time-consuming task that plagues businesses of all sizes. Employees waste hours each week saving invoices, contracts, client documents, and project files from their inboxes to organized locations. This workflow eliminates that manual process entirely.

When a new email arrives in Gmail with attachments, this automation instantly detects it, extracts all attached files (even multiple attachments per email), and uploads them directly to your specified Google Drive folder. The process happens in seconds, ensuring files are never lost in crowded inboxes and are immediately available to your team in organized cloud storage.

Beyond simple file transfer, this template demonstrates how to handle multiple attachments as iterable outputs—a fundamental automation pattern that can be extended to include file renaming, categorization, notification alerts, and integration with other business systems.

How It Works

Step 1: Gmail Trigger Detection

The workflow starts with a Gmail Trigger node that monitors your inbox for new emails containing attachments. You can configure it to watch specific labels, filter by sender, or look for keywords in subject lines to target only business-critical emails.

Step 2: Attachment Extraction

A Code node processes the email data, extracting all attachments as binary files and preparing them for upload. This step handles multiple attachments efficiently, creating separate data items for each file while maintaining their relationship to the original email.

Step 3: Google Drive Upload

Each extracted attachment is uploaded to your specified Google Drive folder. The template includes configuration options for folder organization, file naming conventions, and duplicate handling to ensure a clean, organized file structure.

Pro tip: Extend this workflow by adding a step to rename files based on email metadata (like "Invoice_ClientName_Date.pdf") or organize them into dated subfolders automatically.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Gmail account with API access enabled (Google Cloud Console setup)
  3. Google Drive account with storage space for uploaded files
  4. OAuth 2.0 credentials configured for both Gmail and Google Drive APIs
  5. Basic understanding of n8n node configuration (credentials and folder paths)

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure Gmail credentials by creating OAuth 2.0 credentials in Google Cloud Console and adding them to n8n
  3. Set up Google Drive credentials similarly with appropriate folder access permissions
  4. Adjust the Gmail Trigger node to specify which emails to monitor (all emails, specific labels, or filtered queries)
  5. Configure the Google Drive node with your target folder path and preferred file naming conventions
  6. Test the workflow by sending a test email with attachments to your monitored inbox
  7. Activate the workflow and let it run automatically for continuous attachment processing
