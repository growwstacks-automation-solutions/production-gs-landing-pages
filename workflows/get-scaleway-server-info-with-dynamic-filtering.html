What This Workflow Does

Managing cloud infrastructure manually is time-consuming and error-prone, especially when you need to quickly find specific servers across multiple zones or projects. This n8n workflow automates the retrieval and intelligent filtering of your Scaleway server inventory.

It connects to the Scaleway API to fetch data from both instances and baremetal servers across all your configured zones. You can then filter results dynamically based on criteria like server tags, name patterns, public IP addresses, or specific zones—all through a simple webhook request. This turns what would be a manual console-hopping session into a single API call that returns exactly the server information you need.

The workflow is particularly valuable for DevOps teams, system administrators, and platform engineers who need to maintain accurate server inventories, perform quick audits, trigger automated actions based on server states, or integrate server data with other monitoring and management systems.

How It Works

The workflow follows a logical sequence to gather, process, and return your server data efficiently.

1. Webhook Trigger with Authentication

The workflow starts with a secure webhook node that accepts POST requests. You can configure basic authentication for added security. The webhook expects a JSON payload containing your search criteria—specifically what to search by (tags, name, public_ip, or zone) and the actual search value.

2. API Data Collection

Once triggered, the workflow makes parallel API calls to Scaleway to fetch server data from both the instance and baremetal services. It iterates through your predefined zones (which you can customize) to ensure comprehensive coverage of your infrastructure.

3. Data Processing & Normalization

A Code node processes the raw API responses, extracting and standardizing the most relevant server attributes: name, ID, tags, public IP, type, state, zone, and creation date. This creates a clean, uniform dataset regardless of whether the server is an instance or baremetal.

4. Dynamic Filtering Logic

Based on the "search_by" parameter from your webhook request, the workflow routes the data through specialized filtering logic. It can match servers by exact tag values, partial name matches, specific IP addresses, or particular zones, returning only the servers that meet your criteria.

5. Structured Response

The filtered results are formatted into a clean JSON array and returned through the webhook response node. If no servers match your criteria, you receive an appropriate message instead of an empty array.

Pro tip: Extend this workflow by connecting the output to notification channels like Slack or email, or use it to trigger subsequent automation based on the filtered server list—like restarting services or applying updates.

What You'll Need

  1. A Scaleway account with active servers (instances or baremetal).
  2. Scaleway API token with appropriate read permissions for your servers.
  3. n8n instance (cloud or self-hosted) to import and run the workflow.
  4. Webhook access to trigger the workflow from your tools or scripts.

Quick Setup Guide

Get this automation running in your environment in under 10 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the "Edit Fields" node by replacing the placeholder text with your actual Scaleway API token.
  3. Review zone lists in the workflow (ZONE_INSTANCE and ZONE_BAREMETAL) and update them to match your Scaleway regions if needed.
  4. Test the webhook by sending a POST request with JSON like: {"search_by": "tags", "search": "production"}
  5. Integrate with your tools by calling the webhook URL from your monitoring systems, scripts, or other workflows.

Note: For production use, consider adding rate limiting, error handling for API failures, and logging of all queries for audit purposes.
