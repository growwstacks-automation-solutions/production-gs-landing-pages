Get Job Details Using the Cortex Node

Automatically fetch and process security job data from Cortex XSOAR to streamline incident response and security operations.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Security operations teams spend valuable time manually checking Cortex XSOAR for job statuses, incident details, and investigation progress. This repetitive administrative work distracts from actual threat analysis and slows down incident response times.

This n8n workflow automates the retrieval of job details from Cortex XSOAR, fetching critical security operation data and routing it to the right systems and teams. It transforms manual security monitoring into an automated, consistent process that ensures nothing gets missed and analysts receive timely updates.

The automation connects Cortex XSOAR with your existing security ecosystem, creating a seamless flow of information between your SOAR platform, ticketing systems, communication channels, and reporting dashboards.

How It Works

1. Trigger & Authentication

The workflow begins by authenticating with your Cortex XSOAR instance using API credentials. It can be triggered on a schedule (e.g., every 15 minutes), by a webhook from another system, or manually when needed.

2. Job Data Retrieval

The Cortex node queries XSOAR for specific job details based on your criteria—this could be by incident ID, time range, status, or priority level. It retrieves comprehensive data including investigation progress, analyst assignments, and timeline information.

3. Data Processing & Enrichment

Retrieved data is processed to extract the most relevant information. The workflow can filter results, add contextual information from other sources, format the data for different destinations, and apply business logic to determine next steps.

4. Distribution & Notification

Processed job details are automatically sent to designated systems. This might include creating tickets in ServiceNow, posting updates to Slack channels, updating security dashboards, or sending email alerts to relevant team members.

5. Logging & Audit Trail

Every execution is logged with timestamps, data retrieved, and actions taken. This creates a complete audit trail for compliance purposes and provides visibility into your automated security operations.

Who This Is For

This workflow is ideal for security operations centers (SOCs), IT security teams, managed security service providers (MSSPs), and any organization using Cortex XSOAR for security orchestration. It's particularly valuable for:

Security Analysts who need timely updates on investigations without constantly checking multiple systems. Security Managers who require visibility into team workload and incident status. IT Operations Teams that coordinate with security on incident response. Compliance Officers who need audit trails of security activities.

If your team handles more than a few security incidents per week or manages complex investigations spanning multiple systems, this automation will significantly reduce manual overhead.

What You'll Need

  1. Cortex XSOAR Instance with API access enabled and appropriate credentials
  2. n8n Instance (cloud or self-hosted) with the Cortex node installed
  3. Destination Systems configured and accessible (Slack, ServiceNow, email, etc.)
  4. API Credentials for any systems you want to send data to
  5. Clear Understanding of what job data you need to retrieve and where it should go

Pro tip: Start by automating retrieval for a single, common type of security job before expanding to more complex workflows. This lets you validate the process and demonstrate value quickly.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance
  2. Configure the Cortex node with your XSOAR instance URL and API credentials
  3. Set your query parameters to specify which jobs to retrieve (by ID, time, status, etc.)
  4. Connect your destination nodes like Slack, email, or ticketing systems
  5. Test with a single job to ensure data flows correctly end-to-end
  6. Set the trigger schedule or webhook based on your operational needs
  7. Monitor initial executions and adjust filters or formatting as needed
