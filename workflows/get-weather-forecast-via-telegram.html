How It Works

The automation follows a clean three-step process that transforms a simple message into actionable weather intelligence.

1. Telegram Trigger Listens for Commands

The workflow starts with a Telegram Trigger node configured to monitor a specific bot. When a user sends a message (like "weather London" or "forecast 90210"), the trigger captures it, extracts the text and user details, and passes this data to the next node.

2. OpenWeatherMap Fetches Forecast Data

The user's location query is sent to the OpenWeatherMap API. The node retrieves current conditions, temperature, humidity, wind speed, and a short forecast. The API handles location parsing—whether it's a city name, ZIP code, or coordinates—and returns structured JSON data.

3. Telegram Delivers Formatted Response

The raw weather data is formatted into a readable message (e.g., "🌤️ London: 18°C, 65% humidity, 12 km/h wind"). The Telegram node sends this response back to the same user or chat, completing the loop within seconds.

Pro tip: Extend this workflow by adding conditional logic—for example, only send alerts if rain probability exceeds 70%, or log all queries to a Google Sheet for analysis.

Quick Setup Guide

Follow these steps to get your weather bot running in under 10 minutes:

  1. Download the template using the button above and save the JSON file.
  2. Import into n8n: In your n8n dashboard, click "Import from File" and select the downloaded JSON.
  3. Configure Telegram credentials: Add your bot token from @BotFather to the Telegram Trigger and Telegram nodes.
  4. Add OpenWeatherMap API key: Create a credential with your free API key and connect it to the OpenWeatherMap node.
  5. Activate the workflow: Toggle the workflow on, message your bot with a location, and receive your first automated forecast.

Pro tip: Test with your city first, then try variations like "weather tomorrow in Paris" or "forecast 94016" to see how the API handles different queries.
