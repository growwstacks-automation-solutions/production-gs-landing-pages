Gong Tool MCP Server

What This Workflow Does

This workflow solves the problem of manually integrating Gong data with AI agents and automation systems. Gong provides valuable conversation intelligence for sales teams, but accessing this data programmatically requires API knowledge and custom coding. This MCP server template eliminates that complexity by providing a ready-to-use interface that exposes all Gong Tool operations to AI agents.

The workflow acts as a bridge between Gong's API and AI systems like Claude Desktop, custom AI applications, or other n8n workflows. It automatically handles authentication, parameter mapping, and error handling, allowing AI agents to retrieve call recordings, user information, and analytics without any configuration work.

Pro tip: This template uses $fromAI() expressions that automatically populate parameters from AI agent requests, making it truly zero-configuration.

How It Works

Step 1: MCP Trigger Setup

The workflow starts with an MCP Server Trigger node that creates a webhook endpoint. This endpoint serves as the standardized interface for AI agents to send requests. When activated, it generates a unique URL that AI systems can use to communicate with the Gong operations.

Step 2: Gong Tool Operations

Four pre-configured Gong Tool nodes handle all available operations: Get Call, Get Many Calls, Get User, and Get Many Users. Each node is configured with $fromAI() placeholders that automatically receive parameters from incoming AI requests.

Step 3: AI Integration

AI agents send requests to the MCP server URL with specific parameters (like call IDs or user identifiers). The workflow processes these requests, executes the appropriate Gong operation, and returns structured data back to the AI agent in native Gong API format.

Step 4: Error Handling & Logging

Built-in n8n error management ensures failed requests are logged and retried appropriately. The workflow includes comprehensive error handling for authentication failures, invalid parameters, and Gong API limitations.

  2. Gong API credentials (access token)
  3. Basic understanding of MCP protocol (optional – the template handles everything)
  4. An AI agent or system that supports MCP integration (Claude Desktop, custom apps, etc.)
  5. Network access between your n8n instance and AI systems

Quick Setup Guide

  1. Import the template into your n8n instance via the import functionality.
  2. Activate the workflow to start the MCP server and generate the webhook URL.
  3. Copy the MCP URL from the MCP Server Trigger node configuration.
  4. Configure your Gong credentials in the Gong Tool nodes (add your API token).
  5. Connect AI agents by adding the MCP URL to their configuration.
  6. Test with a simple request like "Get call with ID X" to verify functionality.

Pro tip: The template includes all four Gong operations, so you don't need to modify anything. Just add your credentials and start using it.
