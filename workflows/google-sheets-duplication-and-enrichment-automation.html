What This Workflow Does

Manually duplicating complex Google Spreadsheets with multiple tabs is a tedious, error-prone task that consumes valuable business hours. This automation solves that by programmatically reading a source spreadsheet, creating an exact replica, and optionally enriching the data with information from other systems.

The workflow handles the entire process: fetching sheet metadata, creating a new document, replicating each tab's structure, reading source data, and writing it into the corresponding new sheets. It's ideal for financial reporting, project template creation, inventory management, and any process requiring consistent spreadsheet duplication.

How It Works

The automation follows a logical sequence to ensure accurate replication and data integrity.

1. Trigger & Spreadsheet Creation

The workflow starts manually or on a schedule. It first creates a new, blank Google Spreadsheet that will serve as the destination for the duplicated content.

2. Source Metadata Retrieval

It connects to your source spreadsheet via the Google Sheets API and retrieves all sheet names, IDs, and structural details. This metadata dictates what needs to be replicated.

3. Sheet Structure Replication

Using a loop, the workflow iterates through each sheet from the source. It deletes the default "Sheet1" in the new document and creates new sheets with matching titles and properties.

4. Data Reading & Enrichment

For each sheet, the workflow reads all data rows and columns from the source. At this stage, you can integrate other apps (like a CRM or database) to add additional data points, calculations, or status flags to each row.

5. Data Writing & Completion

Finally, the enriched (or plain) data is written into the newly created sheets in the destination spreadsheet. The process repeats for all sheets, resulting in a perfect, populated copy.

What You'll Need

  1. A Google Workspace account with access to Google Sheets and Drive.
  2. n8n instance (cloud or self-hosted) with the Google Sheets node installed and authenticated.
  3. Source Google Spreadsheet with the sheet structure you want to duplicate.
  4. Optional: Credentials for any additional apps you want to use for data enrichment (e.g., HTTP Request for APIs).

Pro tip: Before running this automation on critical data, test it with a simple spreadsheet containing 2-3 sheets. Verify that all columns, data validation rules, and basic formatting are copied correctly.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment.

  1. Download and Import: Click the download button above to get the JSON file. In your n8n dashboard, go to Workflows → Import from file and select the downloaded JSON.
  2. Configure Google Sheets Connection: Click the Google Sheets nodes and authenticate with your Google account. Ensure the connection has read/write permissions to the relevant spreadsheets.
  3. Set Your Spreadsheet IDs: In the "HTTP Request" and "Read Spreadsheet" nodes, replace the placeholder spreadsheet ID with the ID of your actual source spreadsheet. You can find this ID in your spreadsheet's URL.
  4. Test the Trigger: Click "Execute Workflow" on the Manual Trigger node. Monitor the execution to see if a new spreadsheet is created and populated correctly.
  5. Add Enrichment (Optional): Between the "Read Spreadsheet" and "Write Sheet" nodes, you can insert other n8n nodes to fetch additional data from your CRM, database, or API to enrich each row before it's written.
  6. Schedule or Webhook: Once tested, replace the Manual Trigger with a Schedule Trigger (for regular duplication) or a Webhook (to run when an event occurs in another app).
