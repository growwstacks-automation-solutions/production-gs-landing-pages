What This Workflow Does

This automation solves a common yet tedious business problem: manually copying and pasting data from a Google Sheet into an HTML format for websites, reports, or dashboards. Every time your spreadsheet updates, you'd normally have to reformat the data, write the table code, and redeploy it—a process prone to errors and wasted time.

This n8n workflow automates the entire conversion. It connects directly to your Google Sheet, reads the data (including headers), structures it into proper HTML table tags (<table>, <tr>, <td>), and outputs a clean, ready-to-use HTML snippet. You can then embed this snippet anywhere—a company intranet, a client portal, a static website, or even an email template.

The business value is immediate: data consistency, zero manual effort after setup, and a single source of truth. Your marketing team can manage a client list in Sheets, and your website displays it live. Your operations team can track project status in Sheets, and the team dashboard updates automatically.

How It Works

The workflow follows a logical, step-by-step process to transform raw spreadsheet data into a structured web format.

Step 1: Trigger & Authentication

The workflow can be triggered manually, on a schedule, or via a webhook. It first authenticates with Google Sheets using OAuth, granting secure, read-only access to your specified spreadsheet.

Step 2: Data Extraction

The Google Sheets node fetches all rows and columns from your chosen sheet. It intelligently handles the first row as table headers (<th> tags) and subsequent rows as data cells (<td> tags). You can configure it to pull a specific range or the entire sheet.

Configuring the Google Sheets node to connect to your data source.

Step 3: HTML Generation

n8n's built-in functions and a Code node transform the raw cell data into valid HTML. The workflow constructs the full table structure, ensuring proper nesting and escaping of special characters. You can customize the table's CSS classes for styling.

Finding your Google Sheet ID from the URL is a key setup step.

Step 4: Output & Delivery

The final HTML is output. You can send it via a Webhook node to a waiting endpoint (like a static site generator), save it as a file, or even email it as part of a report. The workflow ensures the output is clean, valid HTML ready for use.

Pro tip: Pair this workflow with a static site deployment tool (like Netlify or Vercel) or a CMS webhook to create a fully automated, live data publishing pipeline. Every time your sheet updates, your website updates.

Quick Setup Guide

Follow these steps to import and run this automation in under 10 minutes.

  1. Download the Template: Click the "Download Template" button on this page to get the JSON file.
  2. Import into n8n: In your n8n dashboard, go to Workflows > Import from File and select the downloaded JSON.
  3. Connect Google Sheets: In the Google Sheets node, click "Add Credential" and follow the OAuth flow to grant n8n access.
  4. Configure the Sheet: Paste your Google Sheet ID into the node settings. The ID is the long string in your sheet's URL: https://docs.google.com/spreadsheets/d/[THIS_IS_YOUR_SHEET_ID]/edit.
  5. Set the Range: Specify the sheet name and cell range (e.g., Sheet1!A1:E100) or leave it blank to fetch all data.
  6. Test the Workflow: Click "Execute Workflow" to run it once. Check the output of the HTML node to see your generated table.
  7. Activate & Schedule: Turn on the workflow and set a schedule (e.g., every hour) or connect a Webhook trigger for real-time updates.
The workflow canvas in n8n showing the connection between Google Sheets and the HTML output.

Pro tip: Use the "Split In Batches" node if you have a large dataset to avoid hitting API rate limits. You can also add a "Set" node to inject custom CSS classes into the HTML for immediate styling.
