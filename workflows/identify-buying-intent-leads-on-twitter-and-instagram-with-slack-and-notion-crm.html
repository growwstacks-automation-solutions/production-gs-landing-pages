Automate Social Media Lead Discovery with AI

What This Workflow Does

Manual social media listening is time-consuming and inconsistent. Sales and growth teams spend hours scrolling through Twitter and Instagram, trying to spot potential customers who are actively asking for help, tools, or recommendations. Most signals get missed, and even when found, leads aren't qualified or logged systematically.

This automation solves that by turning social platforms into a consistent, high-quality lead source. It listens for buying-intent signals—phrases like “looking for a tool to…”, “recommendations for…”, or “struggling with…”. An AI agent then filters out noise, classifies the lead's intent level, and creates structured records for immediate follow-up.

The result is a fully automated pipeline that discovers, qualifies, and distributes warm leads to your team, ensuring no opportunity slips through the cracks and your salespeople spend time selling, not searching.

How It Works

1. Receive a Lead Discovery Query

You or a team member submits a natural-language query via a chat interface (like Slack). For example, “Find people looking for project management tools for remote teams.” The workflow captures this and maintains short-term memory for context if you ask a follow-up question.

2. Search Social Platforms

The system searches Twitter and Instagram using the query parameters. It looks for recent posts where users are explicitly asking for solutions, sharing frustrations, or seeking recommendations—clear indicators of buying intent.

3. AI Qualification & Filtering

Each discovered post is analyzed by an AI agent. It classifies the intent as Low, Medium, or High, summarizes the core problem, and filters out spam, promotions, or irrelevant content. Only genuine, qualified leads proceed.

4. Create Outputs for Slack & Notion

The AI generates two outputs: a concise, human-readable summary for Slack that your team can skim in seconds, and a structured, CRM-ready record for Notion with all extracted details (username, problem, intent score, link).

5. Deliver & Log

The Slack summary is posted to a designated channel for immediate visibility. Simultaneously, the full lead record is created in your Notion database, building a searchable history of all discovered leads for tracking and follow-up.

6. Error Handling

If any step fails, an error handler triggers an email alert to your technical team, ensuring the pipeline's reliability and quick issue resolution.

What You'll Need

  1. A self-hosted n8n instance (this workflow uses features not supported on n8n Cloud).
  2. Azure OpenAI API credentials (or another compatible LLM provider) for the AI qualification step.
  3. MCP bearer authentication for the external social search tool.
  4. Slack API credentials with permissions to post messages to a channel.
  5. Notion API credentials and a pre-configured database to receive the lead records.

Quick Setup Guide

  1. Download & Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows → Import from File and select the downloaded template.
  2. Configure Credentials: Set up the required credentials (Azure OpenAI, MCP, Slack, Notion) in n8n's credentials management. Ensure each node in the workflow is connected to the correct credential.
  3. Adjust Search Parameters: In the “Discover Buying-Intent Leads” node, modify the default search query to match your target audience and industry keywords.
  4. Set Output Destinations: Update the Slack channel ID and the Notion database ID in their respective nodes to point to your team's workspace.
  5. Test & Activate: Run a manual test with a sample query like “find people needing email marketing tools.” Check that leads appear in Slack and Notion correctly, then activate the workflow.
