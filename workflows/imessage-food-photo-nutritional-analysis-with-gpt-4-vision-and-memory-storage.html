What This Workflow Does

This automation solves the tedious problem of manual food logging for health-conscious individuals and professionals. Instead of searching databases or typing ingredients, users simply send a photo of their meal via iMessage. The system automatically analyzes the image using GPT-4 Vision, identifies food items, estimates portion sizes, and calculates nutritional values including calories, macros, fiber, sugar, and sodium.

The workflow then stores this analysis in a Postgres database for historical tracking and sends back a friendly, conversational report directly to the user's iMessage conversation. Over time, the system builds a comprehensive food journal without any manual data entry, enabling weekly summaries, trend analysis, and personalized insights based on accumulated dietary patterns.

Beyond personal use, this template serves health coaches, fitness apps, meal delivery services, and corporate wellness programs by providing automated nutrition tracking that integrates seamlessly into users' existing communication habits, dramatically increasing compliance and engagement compared to standalone tracking apps.

How It Works

1. Photo Submission Trigger

The workflow activates when a user sends an iMessage with a food photo attachment. Blooio captures the message and image, triggering the webhook that starts the automation process. This eliminates the need for users to open a separate app or navigate complex interfaces.

2. Image Processing & Validation

The system checks for valid image attachments, extracts image URLs, and prepares them for AI analysis. Conditional logic ensures only food-related images proceed, filtering out irrelevant content and maintaining processing efficiency.

3. AI-Powered Nutritional Analysis

GPT-4 Vision examines the food photo using a specialized prompt that identifies meal components, estimates portion sizes, and calculates nutritional values. The AI provides confidence scores and health ratings while generating a conversational summary suitable for messaging interfaces.

4. Memory Storage & Historical Tracking

Each analysis gets stored in a Postgres database with timestamps and user identifiers. This creates a searchable food journal that enables trend analysis, weekly summaries, and personalized recommendations based on historical eating patterns.

5. Response Generation & Delivery

The system formats the nutritional analysis into a friendly message optimized for mobile viewing and sends it back through Blooio's iMessage integration. Users receive immediate feedback without leaving their messaging app.

What You'll Need

  1. OpenAI API Key with GPT-4 Vision access for image analysis capabilities
  2. Blooio Account configured for iMessage sending/receiving with webhook delivery enabled
  3. PostgreSQL Database (or compatible alternative) for storing meal history and user data
  4. n8n Instance either self-hosted or cloud-based to run the workflow
  5. Webhook Endpoint publicly accessible for receiving iMessage triggers

Pro tip: Start with a free PostgreSQL database from services like Neon or Supabase to minimize initial costs while testing the workflow functionality.

Quick Setup Guide

  1. Import the Template: Download the JSON file above and import it into your n8n instance via the workflow import function.
  2. Configure Credentials: Set up your OpenAI API key, Blooio credentials, and PostgreSQL connection in n8n's credentials manager.
  3. Test the Webhook: Use the generated webhook URL in your Blooio project settings to connect iMessage triggers to your workflow.
  4. Customize Prompts: Adjust the GPT-4 Vision system prompt to match your specific nutritional focus (keto, diabetic, athletic, etc.).
  5. Deploy & Monitor: Activate the workflow and test with sample food photos to ensure proper analysis and response delivery.
