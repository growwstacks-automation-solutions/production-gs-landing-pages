What This Workflow Does

Moving automation workflows between different n8n instances—from development to staging to production—is a common but tedious challenge for businesses scaling their automation infrastructure. Each time you export a workflow, you face the manual headache of reconfiguring every single credential: API keys, database connections, webhook secrets, and authentication tokens.

This template solves that problem completely. It provides an automated multi-form interface that guides you through importing workflows from either JSON files or directly from remote n8n instances, then systematically maps all credentials to their counterparts in your target environment. What typically takes hours of manual work becomes a 5-minute guided process.

The business value is substantial: reduced deployment errors, faster iteration cycles, consistent automation across environments, and eliminated security risks from manual credential handling. Teams can confidently promote tested workflows to production without worrying about broken connections or misconfigured integrations.

How It Works

The workflow uses n8n's Form Trigger node to create an interactive multi-step interface, then processes the migration through a series of intelligent steps.

Step 1: Choose Your Import Source

The form begins by asking whether you want to upload a workflow JSON file or connect to a remote n8n instance. For remote connections, you select from pre-configured instances defined in a Settings node—each with its API key and base URL.

Step 2: Retrieve Available Workflows

If connecting remotely, the workflow uses n8n's API nodes to fetch a list of all workflows from the selected instance. You then choose which specific workflow to import from a dropdown menu.

Step 3: Analyze Credential Requirements

The imported workflow is parsed to identify every credential reference. Simultaneously, the template queries your current n8n instance (using Execute Command nodes) to list all available credentials.

Step 4: Map Credentials Visually

A dynamic form page displays each credential from the source workflow alongside dropdowns showing matching credential types from your current instance. For each credential, you can either select an existing one or choose to create a new placeholder credential.

Step 5: Create Missing Credentials

For any credentials marked for creation, the workflow automatically generates empty credential records in your current n8n instance using the API. These are tagged with emoji indicators (like ⚙️) to show they need configuration.

Step 6: Finalize and Import

The workflow updates all credential IDs in the imported workflow JSON, then creates the complete workflow in your current instance. You receive immediate confirmation of success or detailed error messages if anything goes wrong.

Pro tip: Use this workflow as part of your CI/CD pipeline for n8n. Schedule regular exports from development environments and automated imports to staging, creating a seamless deployment process for your automations.

Quick Setup Guide

Follow these steps to implement this workflow migration system:

  1. Download and import the template JSON file into your n8n instance
  2. Configure the Settings node with your remote n8n instance details (name, baseUrl, apiKey)
  3. Test with a simple workflow containing 2-3 different credential types first
  4. Access the Form Trigger URL from the first node to launch the migration interface
  5. Follow the guided form to import your first workflow and map its credentials
  6. Verify the imported workflow runs correctly with the newly mapped credentials

Security note: This workflow decrypts and processes credentials during migration. Use it primarily for transferring workflows between trusted testing environments. For production deployments between untrusted environments, consider additional security measures or use n8n Enterprise Edition's deployment features.
