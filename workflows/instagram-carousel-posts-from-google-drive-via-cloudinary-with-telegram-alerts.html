What This Workflow Does

This n8n workflow solves a common pain point for social media managers and content creators: the tedious, manual process of creating and publishing Instagram carousel posts. Instead of downloading images from Google Drive, manually uploading them to a hosting service, formatting them for Instagram, writing captions, and finally publishing—this automation handles everything from start to finish.

The workflow downloads specified images from Google Drive, uploads them to Cloudinary for optimized hosting, creates an Instagram media container via Facebook's Graph API, assembles the carousel with your caption, publishes it to your connected Instagram Business account, and sends a confirmation alert to your Telegram channel. What typically takes 20-30 minutes of manual work happens automatically in the background.

Beyond just saving time, this automation ensures consistency in your posting schedule, reduces human error (like forgetting to tag products or using wrong hashtags), and provides reliable notifications so you always know when content goes live. It's particularly valuable for e-commerce brands, agencies managing multiple clients, or anyone posting regular visual content to Instagram.

How It Works

The automation follows a logical sequence that mirrors the manual process but executes it programmatically.

1. Data Preparation & Trigger

The workflow begins by receiving structured data containing your carousel details. This can come from a Google Sheet, Airtable, webhook, or scheduled trigger. The data includes Instagram caption text, target account ID, and Google Drive links for each image in the carousel (typically 2-10 images).

2. Image Processing Pipeline

Each Google Drive link is processed individually. The workflow downloads the image file, then uploads it to your Cloudinary account where it's automatically optimized for web delivery. Cloudinary generates a public URL that Facebook's API can access directly.

3. Instagram Media Creation

For each successfully uploaded image, the workflow calls Facebook's Graph API to create an Instagram Media Container. This step registers each image with Instagram's system and returns a unique media ID. The workflow collects all these IDs for the next step.

4. Carousel Assembly & Publishing

Once all media IDs are collected, the workflow makes another API call to create the actual carousel post, combining all images with your caption, hashtags, and location tags. After a brief wait (for Instagram's processing), it publishes the carousel to your feed.

5. Notification & Error Handling

Finally, the workflow sends a Telegram message confirming successful publication with a link to the post. If any step fails (like an invalid Drive link), the error is logged and the workflow continues with remaining images, ensuring one problem doesn't stop your entire content schedule.

What You'll Need

  1. Instagram Business Account connected to a Facebook Page (required for Graph API access)
  2. Facebook Developer App with Instagram Basic Display permissions and a valid access token
  3. Google Drive account with images organized and sharing permissions set correctly
  4. Cloudinary account (free tier available) for image hosting and optimization
  5. Telegram bot token and chat ID for notifications (optional but recommended)
  6. n8n instance (cloud or self-hosted) with the HTTP Request, Google Drive, and Cloudinary nodes installed

Pro tip: Before automating at scale, test with 1-2 images to ensure your Facebook app has all necessary permissions. Instagram's API requires specific review processes for publishing capabilities.

Quick Setup Guide

Follow these steps to implement this automation in your n8n instance:

  1. Import the template using the download button above. The JSON file contains the complete workflow structure.
  2. Configure credentials for Facebook Graph API, Google Drive, Cloudinary, and Telegram in n8n's credentials management.
  3. Update node parameters with your specific Instagram Business Account ID, Cloudinary folder path, and Telegram chat ID.
  4. Test with sample data using the Manual Trigger node. Provide a JSON object with sample image URLs and caption.
  5. Set up your trigger—replace the Manual Trigger with a Schedule Trigger for regular posting, or connect to Google Sheets for batch processing.
  6. Activate the workflow and monitor the first few executions in n8n's execution list to ensure everything works correctly.
