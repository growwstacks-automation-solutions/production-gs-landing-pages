Keyboard Interest Checker: GeekHack Forum to Discord Notifications

Automatically monitor GeekHack for new keyboard Interest Checks & Group Buys and get instant Discord alerts with images and details.

What This Workflow Does

For mechanical keyboard enthusiasts, vendors, and community managers, staying on top of new Interest Check (IC) and Group Buy (GB) announcements on GeekHack is crucial but time-consuming. Manually checking the forum multiple times daily leads to missed opportunities and delayed reactions to limited-run keyboard projects.

This n8n automation solves that problem by acting as your 24/7 forum monitor. It automatically scans GeekHack's RSS feeds every hour, identifies new thread posts (not replies), extracts the full content and images, and delivers beautifully formatted notifications to your Discord server. The system includes duplicate prevention through PostgreSQL tracking and supports multiple Discord webhooks for cross-posting to different communities.

Beyond simple notifications, this workflow serves as a foundation for keyboard community management automation. You can extend it to log announcements to a database, trigger follow-up actions, or integrate with other tools like Google Sheets for tracking interest levels across different projects.

How It Works

The automation follows a sophisticated multi-step process to ensure accurate, reliable notifications without spam.

1. Scheduled Monitoring

A schedule trigger runs the workflow hourly, checking two GeekHack RSS feeds simultaneously: one for Interest Checks and another for Group Buys. Each feed retrieves up to 50 recent posts to ensure comprehensive coverage.

2. Thread Filtering

The workflow intelligently filters out reply posts by checking for "Re:" prefixes in thread titles. Only original thread starters are processed, preventing notification spam from discussion threads.

3. Duplicate Prevention

Each thread's unique identifier is checked against a PostgreSQL database of previously processed posts. If the thread already exists in the database, it's skipped entirely, ensuring each announcement is only sent once.

4. Content Extraction

For new threads, the workflow fetches the complete HTML page and uses CSS selectors to extract the original post content. It also employs regex patterns to identify and extract all embedded images from the post.

5. Discord Embed Creation

The post data is transformed into a rich Discord embed with proper formatting: thread title as the embed title, author and date information, truncated content preview, and up to 4 embedded images displayed as attachments.

6. Multi-Webhook Distribution

The workflow queries the database for all registered Discord webhook URLs and sends the notification to each one simultaneously. This allows a single announcement to reach multiple community servers or channels.

7. Processing Logging

Finally, the thread is recorded in the PostgreSQL database with its timestamp, preventing future duplicate processing while maintaining a searchable history of all announcements.

Pro tip: The included web form for managing webhooks means community moderators can add new notification channels without touching the database or n8n interface—just share the form URL with trusted team members.

Quick Setup Guide

Follow these steps to deploy this automation in under 30 minutes:

  1. Database Preparation: Create the two required PostgreSQL tables using the SQL commands provided in the workflow documentation.
  2. n8n Import: Download the JSON template and import it into your n8n instance via the workflow import feature.
  3. Credential Configuration: Set up your PostgreSQL credentials in n8n and assign them to all database nodes in the workflow.
  4. Discord Webhook Setup: Create a webhook in your Discord server channel and add it to the database via the provided web form or direct SQL insert.
  5. Schedule Adjustment (optional): Modify the schedule trigger if you prefer different check intervals than the default hourly frequency.
  6. Test Execution: Manually trigger the workflow once to verify database connections and Discord notifications work correctly.
  7. Activation: Enable the workflow and let it run automatically. Monitor the first few executions to ensure everything processes smoothly.

Pro tip: Start with a test Discord channel and a single webhook. Once verified, add your main community channels. The webhook management form makes it easy to expand notification destinations later.
