What This Workflow Does

This template is an interactive, step-by-step tutorial designed to teach you the most important skill in n8n: using expressions to access and manipulate data. If you've ever wondered how to pull specific information from one node and use it in another—like extracting a customer's email from CRM data to send a personalized message—this workflow provides the foundational knowledge you need.

Expressions are what transform n8n from a simple connector between apps into a powerful automation engine. They allow you to work with dynamic data rather than static values, enabling workflows that adapt to changing information. This tutorial starts with a single "Source Data" node that acts as your filing cabinet, then walks you through eight practical lessons, each demonstrating a new technique for retrieving and transforming that data.

By the end, you'll understand how to navigate JSON structures, use built-in selectors, apply JavaScript functions, and combine multiple data operations—skills that will help you build more sophisticated, reliable automations that save hours of manual work.

How It Works

The tutorial follows a logical progression from simple to complex concepts, with each lesson building on the previous one. You'll execute the workflow and examine how each node uses expressions to achieve specific outcomes.

Lesson 1: Accessing Simple Values

Learn to reference basic data from previous nodes using the expression editor. This foundational skill lets you pull single values like names, dates, or IDs from your data streams.

Lesson 2: Using Built-in Selectors

Discover n8n's powerful selectors like .first() and .last() to target specific items in your data. These are essential for working with arrays and ensuring you're always referencing the correct piece of information.

Lesson 3: Working with Arrays

Master accessing specific items from lists. Whether you need the third item in a product catalog or a filtered subset of records, array navigation is crucial for processing collections of data.

Lesson 4: Drilling into Nested Objects

Learn to navigate complex JSON structures with multiple levels of nesting. This skill is vital when working with APIs that return deeply structured data like customer profiles or multi-level categories.

Lesson 5: Combining Techniques for Complex Data

Apply what you've learned to access data within arrays of objects—a common pattern in real-world automation. This lesson shows you how to chain selectors and property accessors to get exactly what you need.

Lesson 6: Applying JavaScript Functions

Go beyond simple retrieval by using JavaScript functions to transform data. Learn to format text, perform calculations, and manipulate values directly within your expressions.

Lesson 7: Data Inspection Utilities

Use functions like Object.keys() and JSON.stringify() to understand your data structure and debug your workflows. These tools help you work with unfamiliar data formats.

Lesson 8: Summarizing Multiple Items

Learn to process entire datasets using .all() and arrow functions. This advanced technique lets you perform operations on every item in a collection, like calculating totals or extracting specific fields.

Pro tip: Keep the Expression Editor documentation open in another tab while working through this tutorial. The autocomplete feature and built-in examples will accelerate your learning.

Hardcoded values are static and never change, while expressions are dynamic references that pull live data from your workflow. For example, instead of typing 'hello@company.com' in every email, an expression can fetch the recipient's email from your CRM data, making your automation adaptable and scalable.

This difference becomes crucial as your automation grows. Hardcoded values require manual updates when data changes, while expressions automatically adjust. A sales follow-up workflow using expressions will always send to the correct contact, even as your team adds new leads daily.

Expressions can include fallback values using the || operator. For example, $json.customer.email || 'default@example.com' will use the customer email if available, otherwise use the default. You can also use conditional logic with ternary operators to handle different data scenarios gracefully.
