LinkedIn Lead Generation: Auto DM System with Comment Triggers

What This Workflow Does

Manual LinkedIn lead generation is time-consuming and inconsistent. You post valuable content, people comment with interest, but following up individually takes hours you don't have. This results in missed opportunities and stalled pipeline growth.

This automation solves that by transforming LinkedIn from a broadcasting platform into a lead capture engine. It continuously monitors your posts for comments containing specific trigger words (like "interested," "how," or "tell me more"), identifies whether commenters are connections, and automatically engages them with personalized direct messages or connection requests. The system tracks all interactions in a database to prevent duplicate outreach, ensuring every engaged user receives timely, relevant follow-up.

How It Works

1. Comment Monitoring & Trigger Detection

The workflow starts by fetching all comments from a specified LinkedIn post using Unipile. It scans each comment for customizable trigger words that indicate buying intent or interest. When a match is found, the commenter's profile data is extracted for the next steps.

2. Connection Status Verification

Before engaging, the system checks whether the commenter is already a 1st-degree connection. This is crucial because LinkedIn's messaging rules differ for connections versus non-connections. The workflow uses Unipile's API to determine network distance accurately.

3. Smart Engagement Routing

Based on connection status, the workflow takes different paths. For existing connections, it sends a personalized direct message with a lead magnet link or valuable resource. For non-connections, it replies to their comment with a connection request invitation, starting the relationship-building process.

4. Database Logging & Duplicate Prevention

Every interaction—whether DM sent or connection request made—is recorded in a NocoDB database. This includes the user's LinkedIn ID, name, headline, profile URL, and interaction type. Before any new action, the workflow checks this database to ensure the same user isn't contacted twice, preventing spammy behavior.

5. Batch Processing with Delays

When multiple trigger comments are found, the workflow processes them in batches with built-in delays (8-12 seconds between actions). This mimics human behavior, reduces the risk of LinkedIn restrictions, and ensures sustainable, long-term automation.

What You'll Need

  1. Unipile API Key: For LinkedIn API access to retrieve comments, check connections, and send messages.
  2. NocoDB API Token: For database operations to track interactions and prevent duplicates.
  3. NocoDB Database: Set up with a "leads" table containing fields for LinkedIn ID, name, headline, URL, date, post ID, connection status, and DM status.
  4. n8n Instance: Self-hosted or cloud version of n8n to run the workflow.
  5. LinkedIn Account: Professional account with posting privileges and established network.

Pro tip: Start with a low-volume post to test your trigger words and message templates before scaling to high-engagement content. Monitor response rates and adjust your approach based on what generates the best conversations.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Create credentials for Unipile and NocoDB in n8n's settings.
  3. Set up your NocoDB database with the required table structure.
  4. Configure environment variables for Unipile root URL and your LinkedIn account ID.
  5. Test the workflow with a sample post ID, trigger word, and lead magnet link.
  6. Schedule the workflow to run periodically (every 30 minutes works well).
  7. Monitor the NocoDB database to see captured leads and refine your approach.
