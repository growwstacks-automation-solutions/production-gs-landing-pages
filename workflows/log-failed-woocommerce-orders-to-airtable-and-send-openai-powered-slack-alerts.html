What This Workflow Does

Every failed WooCommerce order represents potential lost revenue and a frustrated customer. Manually checking for these failures is time-consuming and error-prone. This automation solves that problem by continuously monitoring your store for failed payments, creating a reliable audit trail, and alerting your team instantly.

The workflow runs on a schedule you control (every 5, 15, or 60 minutes), fetches all failed orders from WooCommerce, checks Airtable to prevent duplicate entries, logs new failures with complete details, and sends intelligent Slack notifications powered by OpenAI. This creates a closed-loop system where nothing slips through the cracks.

How It Works

1. Scheduled Monitoring

The workflow begins with a scheduler node that triggers at your chosen interval. This replaces manual checking with automated, consistent monitoring that never misses a beat, even outside business hours.

2. Fetch Failed Orders

Using WooCommerce's REST API, the workflow retrieves all orders marked as "failed" since the last check. It handles authentication securely and can process multiple orders in a single run.

3. Duplicate Prevention Check

Each order is checked against your Airtable base using the unique order ID. If the order already exists in your log, the workflow skips it to maintain clean, non-duplicated data.

4. Data Enrichment & Storage

New failed orders are formatted with customer details, product information, failure reasons, and actionable links (admin and retry URLs). This enriched data is then saved to Airtable for permanent record-keeping and analysis.

5. Intelligent Alert Generation

OpenAI processes the order data to generate concise, human-readable summaries that highlight key information. These are then sent to your designated Slack channel, giving your team context-rich notifications instead of raw data dumps.

Pro tip: Set the scheduler to run every 15 minutes during peak business hours and every hour during off-hours. This balances timely detection with API rate limit considerations.

Quick Setup Guide

Follow these steps to get this automation running in under 15 minutes:

  1. Download and import the template JSON file into your n8n instance.
  2. Configure the scheduler node to your preferred monitoring frequency.
  3. Add your WooCommerce credentials in the HTTP Request node (domain, consumer key, and secret).
  4. Connect your Airtable base in both the Search and Create nodes, mapping the fields as shown.
  5. Add your OpenAI API key if using AI summaries, or adjust the Slack message format if not.
  6. Connect your Slack workspace and select the channel for alerts.
  7. Test with a manual trigger, then activate the workflow to run automatically.

Pro tip: Create a test failed order in WooCommerce (use a test payment method) to verify the entire flow works before relying on it for production monitoring.
