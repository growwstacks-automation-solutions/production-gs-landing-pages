What This Workflow Does

Flutterflow is fantastic for building beautiful mobile app interfaces quickly, but it often needs a backend to handle data operations, business logic, or integrations. Writing and maintaining a full backend API is a major hurdle for many low‑code developers.

This n8n workflow solves that by acting as a ready‑to‑use backend API. It listens for HTTP requests from your Flutterflow app, connects to your data source (like a database, spreadsheet, or CRM), processes the information, and returns clean, structured JSON. You get a fully functional API endpoint in minutes, without writing a single line of server‑side code.

The template demonstrates a common pattern: a Flutterflow app triggers a GET request, n8n fetches all records from a datastore, aggregates the data into a usable format, and sends it back. This pattern can be adapted for user authentication, data filtering, CRUD operations, and connecting to hundreds of other apps.

How It Works

The workflow is a straight‑through pipeline designed for reliability and easy modification.

1. Webhook Trigger

The flow starts when your Flutterflow app makes an HTTP GET call to the unique webhook URL generated by n8n. This webhook acts as your API endpoint. You can configure it to accept query parameters (like user IDs or filters) sent from the app.

2. Database Query

Once triggered, the workflow connects to your data source. The template uses a Customer Datastore node, but you can replace this with any n8n database node (PostgreSQL, MySQL, Google Sheets, Airtable, etc.). It executes a "Get All" operation to retrieve the necessary records.

3. Data Processing & Aggregation

The raw data from the database is passed through Set nodes to map fields and ensure consistent formatting. An Aggregate node then structures the data—grouping, counting, or summarizing as needed—preparing it for consumption by your Flutterflow interface.

4. API Response

A final Webhook Response node sends the processed data back to your Flutterflow app as a clean JSON payload. You can set HTTP status codes, headers, and error messages here, making it a fully compliant REST API endpoint.

What You'll Need

  1. A running n8n instance (Cloud, Self‑Hosted, or Desktop).
  2. A Flutterflow app with the ability to make HTTP requests (using the API Call action).
  3. Access to your data source (database credentials, API keys for services like Airtable, Google Sheets, etc.).
  4. The webhook URL from this workflow to add to your Flutterflow project.

Quick Setup Guide

Import and configure this backend in under 10 minutes.

  1. Download the template using the button above and import it into your n8n workspace.
  2. Configure the Webhook node to activate the trigger and copy your unique webhook URL.
  3. Replace the Customer Datastore node with your actual database connection (PostgreSQL, MySQL, etc.) and adjust the query to match your data model.
  4. Test the endpoint by opening the webhook URL in a browser or using a tool like Postman. You should see a JSON response with your data.
  5. Connect Flutterflow by adding an API Call action in your Flutterflow workflow, pasting the webhook URL, and mapping the response to your app's widgets.

Pro tip: Use query parameters in your webhook URL (like ?user_id=123) to filter data dynamically. Add an 'IF' node after the webhook to check for these parameters and modify the database query accordingly.
