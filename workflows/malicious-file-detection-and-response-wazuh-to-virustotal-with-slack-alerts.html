Malicious File Detection & Response: Wazuh to VirusTotal with Slack Alerts

What This Workflow Does

Security Operations Center (SOC) teams face constant alerts about suspicious files detected on endpoints. Manual validation of each file hash against threat intelligence databases consumes hours of analyst time, delays response, and increases risk of missing genuine threats.

This workflow automates the entire detection-to-response pipeline. When Wazuh identifies a suspicious file, the automation instantly extracts its hash, queries VirusTotal for reputation scoring, generates a human-readable threat summary, and routes alerts based on risk level. High-risk files trigger Slack notifications and incident tickets, while lower-risk alerts generate summary emails for review.

The result: SOC teams gain immediate context for every file alert, reduce false positive investigations, and accelerate genuine threat response from hours to minutes.

How It Works

Step 1: Alert Ingestion

A webhook node receives Wazuh alerts containing file hash (SHA256/MD5), filename, detection time, and endpoint details. The workflow parses this JSON payload to extract key indicators of compromise (IOCs).

Step 2: Threat Intelligence Validation

The extracted file hash is sent to VirusTotal's API via an HTTP Request node. VirusTotal returns reputation scores, detection engines that flagged the file, and community analysis. This automated validation replaces manual researcher checks.

Step 3: Risk Scoring & Decision Routing

A Switch node evaluates the VirusTotal response. Files with high malicious scores (>60% detection) route to high-priority channels. Moderate-risk files route for analyst review. Clean files are logged without alerting.

Step 4: Notification & Action

High-risk files trigger Slack alerts to security team channels with enriched context. Simultaneously, an incident creation node opens a ticket in your ticketing system (ServiceNow, Jira, etc.). All files generate formatted email summaries via Gmail for record-keeping.

What You'll Need

  1. Wazuh deployment with alert forwarding configured to send webhook notifications.
  2. VirusTotal API key (free tier available, premium recommended for production).
  3. Slack workspace with a channel dedicated to security alerts.
  4. Gmail account or alternative email service for summary reports.
  5. Incident management system (ServiceNow, Jira, etc.) if creating automated tickets.
  6. n8n instance (self-hosted or cloud) to run the workflow.

Quick Setup Guide

  1. Download the template JSON file and import into your n8n instance.
  2. Configure the Webhook node with your Wazuh alert endpoint URL.
  3. Add your VirusTotal API key to the HTTP Request node authentication settings.
  4. Connect the Slack node to your workspace and specify the alert channel.
  5. Set up Gmail credentials for email summaries.
  6. Adjust risk thresholds in the Switch node based on your organization's risk tolerance.
  7. Test with sample Wazuh alerts to validate the entire pipeline.

Pro tip: Create a separate "low-risk review" Slack channel for files with moderate VirusTotal scores. This allows analysts to quickly approve or escalate without interrupting high-priority investigations.
