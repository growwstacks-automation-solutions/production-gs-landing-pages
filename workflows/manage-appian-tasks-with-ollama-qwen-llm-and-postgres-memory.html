What This Workflow Does

This workflow creates an AI-powered interface for managing Appian tasks by combining a local LLM (Ollama Qwen) with Postgres for persistent conversation memory. It demonstrates how to build an intelligent agent that can understand natural language requests about Appian tasks and take appropriate actions through API calls.

The system handles the complete lifecycle from receiving user queries through webhooks, processing them with the LLM while maintaining context in Postgres, executing Appian API actions when needed, and returning understandable responses. This pattern eliminates the need for users to navigate Appian interfaces directly for common task management operations.

How It Works

The workflow follows a clear sequence to process requests while maintaining context:

1. User Input Handling

Receives natural language queries through webhook triggers, extracting key parameters like user ID and session context.

2. Context Retrieval

Queries Postgres for previous conversation history related to the current session to maintain continuity.

3. LLM Processing

Ollama Qwen LLM analyzes the query and context to determine needed Appian actions or appropriate responses.

4. Appian API Execution

When task actions are required, makes authenticated calls to Appian's APIs to list, create, or update tasks.

5. Response Generation

Formats results into natural language responses while updating Postgres with new conversation context.

Pro tip: For production use, add rate limiting to the webhook trigger and implement proper error handling for Appian API failures.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure the Postgres connection with your database credentials

  5. Test with sample queries through the webhook URL
