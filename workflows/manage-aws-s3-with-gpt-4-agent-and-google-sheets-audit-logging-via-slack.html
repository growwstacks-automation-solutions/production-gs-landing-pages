What This Workflow Does

This automation solves a critical problem for teams managing cloud storage: the friction between needing quick access to files and maintaining proper security compliance. Instead of requiring team members to navigate AWS consoles or memorize CLI commands, they can simply ask in Slack: "Copy the Q3 report from dev to prod bucket" or "List all files in the marketing folder."

The AI agent interprets these natural language requests, executes the appropriate AWS S3 operations, and automatically logs every action to Google Sheets with full context—who requested it, what parameters were used, the AI's reasoning, and the outcome. This creates a searchable audit trail for compliance while dramatically reducing the time spent on routine storage tasks.

How It Works

1. Natural Language Input via Slack

A team member sends a message in Slack (or Telegram/Discord) requesting an S3 operation. The workflow triggers via webhook, capturing the message and user identity for audit purposes.

2. AI Interpretation & Intent Mapping

The GPT-4 agent analyzes the request, determines which of six S3 tools to use (ListBuckets, ListObjects, CopyObject, DeleteObject, ListFolders, CreateFolder), and extracts relevant parameters like bucket names, file paths, or destination locations.

3. AWS S3 Operation Execution

Using configured AWS credentials with appropriate IAM permissions, the workflow executes the precise S3 API call. The operation respects AWS security policies while performing the requested action efficiently.

4. Automated Audit Logging to Google Sheets

Immediately after the S3 operation, the workflow appends a detailed record to a designated Google Sheet. Each log entry includes timestamp, tool used, parameters, original prompt, AI reasoning, user information, and operation status.

Pro tip: Configure your Google Sheet with data validation rules and conditional formatting to automatically highlight sensitive operations (like deletions) or unusual access patterns for security review.

What You'll Need

  1. AWS IAM User/Role with appropriate S3 permissions (List, Read, Write based on your needs)
  2. OpenAI API Key for GPT-4 or GPT-3.5-turbo access
  3. Slack Workspace with webhook permissions (or Telegram/Discord equivalent)
  4. Google Sheet with columns: timestamp, tool, status, chat_prompt, parameters, user_name, tool_call_reasoning
  5. n8n instance (cloud or self-hosted) with AI Agent capabilities

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download and import the JSON template into your n8n instance
  2. Configure AWS credentials in the S3 nodes with least-privilege IAM permissions
  3. Set up OpenAI connection with your API key in the AI Agent node
  4. Create your Google Sheet with the required column structure
  5. Connect your chat platform webhook to the trigger node
  6. Test with simple commands like "list buckets" before moving to production

Pro tip: Start with read-only permissions (ListBuckets, ListObjects) during testing. Only add write/delete capabilities after validating the audit logging works correctly and your team is comfortable with the workflow.
