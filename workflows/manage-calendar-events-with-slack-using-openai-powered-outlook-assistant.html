What This Workflow Does

This automation transforms how your team interacts with Outlook Calendar by bringing intelligent scheduling directly into Slack. Instead of constantly switching between apps to check availability, schedule meetings, or find open slots, team members can simply ask questions in natural language.

The AI-powered assistant understands queries like "When is the product demo scheduled?" or "Find 30 minutes where both Sarah and Mark are free tomorrow" and provides instant answers by accessing real-time calendar data. It eliminates the back-and-forth of manual scheduling and reduces meeting coordination time by up to 80%.

Beyond simple queries, the system can summarize weekly meetings, identify scheduling conflicts, and even suggest optimal meeting times based on team patterns—all without leaving your team's primary communication channel.

How It Works

1. Slack Trigger & Message Parsing

The workflow activates when someone @mentions your bot in a designated Slack channel. The trigger captures the complete message and user context, then passes it to an Edit Fields node that extracts the core query while filtering out noise like emojis or tangential conversation.

2. AI Agent Processing with Calendar Tools

An OpenAI-powered agent receives the parsed query and determines intent. Equipped with Outlook Calendar API tools, it can search events, read calendar details, check availability, and understand date/time references. The AI interprets natural language like "next Thursday" or "this afternoon" into specific time ranges.

3. Outlook Calendar Integration

The agent uses Microsoft Graph API permissions to securely access authorized calendars. It can query multiple calendars simultaneously, check free/busy statuses, read event details (title, participants, location), and understand recurring meeting patterns—all while maintaining proper authentication and data privacy.

4. Response Generation & Delivery

After gathering the relevant calendar data, the AI formulates a human-friendly response. Instead of raw API data, it returns conversational answers like "Sarah has a client call from 2-3 PM but is free after 3:30" or "The team sprint review is scheduled for Friday at 10 AM in Conference Room B." This response is then posted back to the Slack thread.

Pro tip: Start with read-only queries before enabling write capabilities. Let your team get comfortable asking questions before allowing the AI to actually create or modify calendar events.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow permissions
  2. Slack workspace with admin access to create a bot/app
  3. Outlook/Microsoft 365 business account with calendar access
  4. OpenAI API key (or compatible LLM provider)
  5. Slack channel where the bot will be invited and used
  6. Microsoft Azure App Registration with Calendar.Read permissions

Quick Setup Guide

1. Download and import the JSON template into your n8n instance.

2. Configure Slack credentials in the trigger node using OAuth or bot token.

3. Set up Microsoft Outlook connection with delegated calendar permissions.

4. Add your OpenAI API key to the AI agent node configuration.

5. Test with simple queries like "What's on my calendar today?" before expanding to team calendars.

6. Deploy to production and invite your Slack bot to relevant channels.

Important: For production use, switch the workflow to active mode and use the production webhook URL for Slack event subscriptions. Test thoroughly in a dedicated channel before rolling out to the entire organization.
