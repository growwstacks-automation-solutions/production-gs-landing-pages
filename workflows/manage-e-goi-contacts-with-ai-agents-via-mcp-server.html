Manage E-goi Contacts with AI Agents via MCP Server

What This Workflow Does

This n8n workflow solves the disconnect between conversational AI agents and actionable marketing automation. When you use AI chatbots, support agents, or lead qualification tools, valuable contact data and intent signals often get stuck in chat logs instead of flowing into your email marketing platform.

The template creates a complete MCP (Model Context Protocol) server that exposes all E-goi contact operations—create, read, update, and list—as tools your AI agents can directly use. This means your AI can add new leads to specific lists, update contact tags based on conversations, segment audiences in real-time, and trigger personalized nurture sequences—all without human intervention.

Instead of manually copying emails from chat transcripts or CSV imports, your AI becomes an active member of your marketing team, managing your E-goi contacts with context-aware intelligence 24/7.

How It Works

The workflow is built as a standalone n8n webhook server that follows the MCP specification, making it compatible with any AI agent platform that supports the protocol.

Step 1: MCP Server Endpoint

The workflow starts with an HTTP webhook trigger configured as an MCP server endpoint. When activated in n8n, it generates a unique URL that AI agents can connect to. This endpoint accepts structured requests from AI tools and routes them to the appropriate E-goi operations.

Step 2: Operation Routing & Validation

Incoming requests are validated against the MCP specification, checking for required parameters and authentication. The workflow then routes the request to one of four pre-configured E-goi tool nodes based on the requested operation type.

Step 3: E-goi API Execution

Each tool node connects to E-goi's API using your stored credentials. The template includes all contact operations: creating new members, retrieving contact details, listing members with filters, and updating existing contact information. AI-provided parameters (like email, name, tags) are automatically mapped to E-goi's expected fields.

Step 4: Response Formatting

After executing the E-goi operation, the workflow formats the response according to MCP standards and returns it to the AI agent. This includes success/failure status, any returned data (like contact IDs), and error messages if applicable—all in a structure the AI can understand and act upon.

Quick Setup Guide

You can have this MCP server running and connected to your AI agents in under 10 minutes:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Add your E-goi credentials to the credential manager in n8n. The workflow nodes are pre-configured to use these stored credentials.
  3. Activate the workflow to start the webhook server. Copy the generated webhook URL from the MCP trigger node.
  4. Configure your AI agent to connect to the MCP server URL. In Claude Desktop, this goes in the `claude_desktop_config.json`; in other platforms, consult their MCP documentation.
  5. Test the connection by asking your AI to perform a simple E-goi operation, like "Add test@example.com to our newsletter list."
  6. Monitor executions in n8n's workflow history to see exactly what data your AI is sending and how E-goi is responding.
