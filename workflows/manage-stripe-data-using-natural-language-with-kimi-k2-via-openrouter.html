What This Workflow Does

Manually navigating Stripe dashboards, writing API queries, or creating discount codes consumes valuable time that could be spent on growth. Financial data stays locked away from non-technical team members, and simple questions like "Who are our top customers this month?" require jumping through multiple screens.

This workflow transforms Stripe from a technical platform into a conversational business partner. It connects an AI agent (Kimi K2 via OpenRouter) to your Stripe account, allowing you and your team to ask questions in plain English and receive summarized, secure answers. More than just data retrieval, it can also create coupons with explicit user confirmation, ensuring business rules are followed while eliminating manual entry errors.

The system maintains strict privacy controls—sensitive information is masked, no data is stored, and any modification to your Stripe account requires explicit approval. It's like having a knowledgeable financial assistant available 24/7, but one that never forgets security protocols.

How It Works

1. Natural Language Input

A team member asks a question through a chat interface: "Show me last week's transactions over $500" or "Create a 15% discount for email subscribers." The AI agent interprets the intent, determining whether it's a data query or an action request.

2. Intent Analysis & Security Check

The AI agent classifies the request and checks against permitted operations. Unclear or unsupported requests receive helpful explanations about what can be done, maintaining a conversational tone while enforcing boundaries.

3. Secure Data Retrieval

For data queries, the workflow fetches information from Stripe but applies privacy filters automatically. Customer names might be partially masked, card numbers truncated, and only relevant summary information returned—never raw sensitive data.

4. Action Confirmation & Execution

If the request involves creating a coupon or modifying Stripe data, the AI clearly summarizes what will happen and requires explicit user confirmation. Only after approval does it execute the action, providing a clean success message with safe details.

What You'll Need

  1. Active Stripe account with API keys (read/write access for full functionality)
  2. n8n instance (cloud or self-hosted) to run the workflow
  3. OpenRouter account with credits and API key for Kimi K2 model access
  4. Basic understanding of how to set up API credentials in n8n

Pro tip: Start with read-only permissions during testing. Once comfortable with the data retrieval functionality, enable write permissions for coupon creation features.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Configure credentials for Stripe and OpenRouter in n8n's credential management.
  3. Test with simple queries like "What's my current balance?" to verify connections.
  4. Adjust privacy settings if needed—the default masks sensitive data appropriately for most businesses.
  5. Share with your team and demonstrate sample questions to get them started.
