What This Workflow Does

This automation continuously monitors the health and performance of critical travel supplier APIs like Amadeus and Booking.com, sending instant WhatsApp alerts to your team when response times exceed SLA thresholds or services go down. It solves the costly problem of API failures going unnoticed until customers report issues - by which time revenue and reputation damage has already occurred.

The workflow performs synthetic transactions to measure real API performance from your infrastructure's perspective, not just the supplier's status page. It tracks uptime percentages, response times, error rates, and timeout frequency against your contractual SLAs. When thresholds are breached, technical teams receive rich notifications with diagnostic details to accelerate troubleshooting.

How It Works

1. API Health Checks

The workflow executes predefined API calls to critical endpoints (like hotel search or booking confirmation) at regular intervals. Each request measures response time, status code, and payload completeness.

2. SLA Threshold Evaluation

Response metrics are compared against your configured SLA thresholds. The system distinguishes between warning-level breaches (approaching thresholds) and critical failures (complete outages or severe SLA violations).

3. Alert Triggering

When thresholds are breached, the workflow composes detailed alert messages including API name, error details, severity level, and duration of the issue. It suppresses duplicate alerts for ongoing incidents.

4. WhatsApp Notification

Alerts are sent via WhatsApp Business API to designated on-call phones. Messages include actionable buttons for acknowledging alerts and links to diagnostic dashboards.

5. Incident Logging

All incidents are logged with timestamps and metrics for historical reporting. This data helps identify problematic APIs and supports SLA renegotiations with suppliers.
