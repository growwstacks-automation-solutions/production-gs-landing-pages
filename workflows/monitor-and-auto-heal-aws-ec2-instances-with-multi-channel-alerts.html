How It Works

The workflow operates on a continuous monitoring loop with intelligent response mechanisms.

1. Scheduled Health Check Trigger

A schedule node initiates the workflow at configurable intervals (default: every 5 minutes). This ensures regular, proactive monitoring without overwhelming AWS APIs or your systems.

2. EC2 Instance Discovery

The workflow queries AWS for all EC2 instances tagged for production monitoring. It filters by environment tags (like Environment=Production) to focus only on critical infrastructure, ignoring development or test instances.

3. Individual Instance Analysis

Each discovered instance undergoes parallel health assessment. The system retrieves detailed metrics including CPU utilization, memory pressure, disk I/O, network performance, and AWS status checks.

4. Health Evaluation & Decision Logic

Custom logic nodes evaluate metrics against your predefined thresholds. If an instance shows sustained high CPU (>90% for 5+ minutes), failed system checks, or is in a stopped state, it's flagged for intervention.

5. Automated Healing Actions

For unhealthy instances, the workflow automatically executes AWS API commands to restart or replace the instance. This happens within seconds of detection, dramatically reducing downtime compared to manual intervention.

6. Multi-Channel Notification System

Simultaneously, alert messages deploy through WhatsApp for urgent notifications, detailed email reports generate for team review, and all actions log to Google Sheets for audit trails and historical analysis.

What You'll Need

  1. AWS Account with IAM permissions for DescribeInstances, DescribeInstanceStatus, and RebootInstances actions
  2. Production EC2 instances tagged consistently (e.g., Environment=Production) for proper filtering
  3. Twilio account with WhatsApp sandbox or approved business number for instant alerts
  4. SMTP email service (Gmail, Outlook, AWS SES, etc.) with app-specific passwords enabled
  5. Google account for Sheets integration (Workspace or personal)
  6. n8n instance (cloud or self-hosted) with AWS, Twilio, SMTP, and Google Sheets nodes installed

Pro tip: Start with a dry-run mode by disabling the actual restart actions while testing. Monitor the Google Sheets log to verify detection logic works correctly before enabling auto-healing in production.

Quick Setup Guide

  1. Import the workflow into your n8n instance using the downloaded JSON file
  2. Configure the Schedule Trigger to run every 5 minutes (adjust based on your needs)
  3. Set up AWS credentials in the EC2 nodes with appropriate IAM permissions
  4. Connect Twilio for WhatsApp alerts using your SID, auth token, and phone numbers
  5. Configure SMTP settings for email reports with your preferred email service
  6. Link Google Sheets by providing spreadsheet ID and OAuth credentials
  7. Customize health thresholds in the decision nodes based on your application requirements
  8. Test with a single instance first, then gradually expand to your full production fleet
