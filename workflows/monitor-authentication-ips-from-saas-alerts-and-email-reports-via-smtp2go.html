Monitor Authentication IPs from SaaS Alerts & Email Reports via SMTP2Go

Automate daily security monitoring by collecting sign-in IP addresses and sending formatted email alerts to your team.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a critical security visibility problem for modern businesses. As teams use dozens of SaaS applications, tracking who logs in from where becomes nearly impossible manually. This workflow automatically collects all authentication IP addresses from your SaaS Alerts platform over the past 24 hours, processes the data, and sends a clean, actionable email report via SMTP2Go to your security team.

Instead of requiring IT staff to log into multiple dashboards each morning, this system delivers a consolidated security briefing directly to their inbox. It identifies unusual login patterns, highlights potential threats, and creates an audit trail for compliance purposes—all without manual intervention.

How It Works

Step 1: Scheduled Trigger

The workflow activates daily on a schedule you configure (typically early morning). It can also be triggered manually for immediate security reviews after suspected incidents.

Step 2: Fetch Authentication Logs

Using the SaaS Alerts API, the system retrieves all sign-in events from the past 24 hours. This includes user information, timestamps, IP addresses, geographic locations, and authentication success/failure status.

Step 3: Filter & Process Data

The workflow applies your custom filtering rules—such as flagging logins from unfamiliar countries, multiple failed attempts, or privileged account activity. It organizes data by user, application, and risk level.

Step 4: Format Report

Raw log data is transformed into a human-readable format with clear sections, highlighting critical alerts at the top. The system calculates statistics like total logins, unique IPs, and anomaly counts.

Step 5: Send Email Notification

Via SMTP2Go's reliable email delivery, the formatted security report is sent to designated recipients. The email includes both summary information and detailed data for investigation.

Pro tip: Configure different recipient lists for different alert levels. Send high-risk anomalies to your security team immediately, while daily summaries go to IT managers.

Companies with 20+ employees using multiple SaaS applications will see immediate value. What You'll Need

  1. SaaS Alerts account with API access enabled
  2. SMTP2Go account for sending email notifications
  3. n8n instance (cloud or self-hosted) with internet access
  4. List of recipient email addresses for security alerts
  5. Basic understanding of your organization's normal login patterns

Quick Setup Guide

  1. Download the template and import it into your n8n instance
  2. Create credentials for SaaS Alerts API in n8n's settings
  3. Configure your SMTP2Go API key and sender email address
  4. Update the recipient list with your security team's emails
  5. Adjust filtering thresholds based on your security policies
  6. Test with a manual trigger, then schedule for daily execution
