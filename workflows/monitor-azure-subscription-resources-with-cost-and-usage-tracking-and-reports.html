What This Workflow Does

This automation solution connects to your Azure subscription to track resource utilization and associated costs. It solves the common challenge of unexpected cloud bills by providing regular, detailed reports on your Azure spending patterns.

The workflow automatically retrieves all Azure resources, analyzes their costs, and generates formatted reports highlighting total spending, top expensive resources, and cost breakdowns by resource type. This gives IT teams and finance departments clear visibility into cloud expenditures without manual reporting.

How It Works

1. Azure Authentication

The workflow uses OAuth2 Service Principal authentication to securely connect to your Azure subscription with appropriate permissions.

2. Resource Discovery

It queries the Azure Resource Graph API to retrieve all resources across your subscription, including metadata like resource groups, locations, and tags.

3. Cost Data Retrieval

The workflow fetches detailed cost data from Azure Cost Management API, matching expenses to specific resources and time periods.

4. Data Processing

Resource information and cost data are merged to create comprehensive records showing exactly what each Azure component costs to operate.

5. Report Generation

Formatted reports are created in multiple formats (text, HTML) with key metrics, visual breakdowns, and actionable insights.

What You'll Need

  1. An active Azure subscription with appropriate permissions
  2. Ability to create a Service Principal with Reader and Cost Management Reader roles
  3. n8n instance (cloud or self-hosted version 1.0+)
  4. OAuth2 credentials capability in n8n

Quick Setup Guide

  1. Create an Azure Service Principal with required permissions
  2. Configure OAuth2 credentials in n8n
  3. Update the workflow configuration with your subscription details
  4. Set your preferred time range for cost reporting
  5. Assign credentials to HTTP nodes
  6. Enable optional output formats (Excel, Power BI, or API)
