What This Workflow Does

This workflow solves the tedious problem of manually hunting for valuable deals on eBay. Instead of spending hours browsing through hundreds of listings—many of which are low‑quality or misleading—this automation continuously scans eBay’s deals pages, applies intelligent AI scoring to evaluate each product’s true bargain potential, and sends instant Telegram alerts only when genuinely high‑quality deals are detected.

It combines robust web scraping (handling eBay’s anti‑bot protections), smart JavaScript preprocessing to extract clean data, AI‑based classification and scoring, and rule‑based filtering. The result is a streamlined deal‑discovery pipeline that saves time, increases accuracy, and ensures you never miss a worthwhile bargain.

How It Works

Step 1: Scheduled or Manual Trigger

The workflow can run on a schedule (e.g., every hour) or be triggered manually. This flexibility allows you to match scanning frequency to deal volatility—more frequent during peak sales events, less often for general monitoring.

Step 2: eBay Deals Page Scraping via Decodo

A specialized web‑scraping node (Decodo) accesses eBay’s deals page, handling proxies and anti‑bot measures automatically. This ensures reliable data extraction without IP blocks or access failures.

Step 3: JavaScript Data Preprocessing

Raw HTML is processed with JavaScript to extract only key product fields: ID, title, price, URL, and image. This step prevents sending bulky HTML to the AI model, reducing cost and improving analysis speed.

Step 4: AI Agent Classification & Scoring

Each product is sent to an AI agent (e.g., GPT‑4, Gemini) that classifies its category, evaluates seller credibility, assesses discount depth, and assigns a deal‑quality score (0–10). The AI considers factors beyond simple price, such as brand reputation and listing authenticity.

Step 5: Rule‑Based Filtering

An IF node applies configurable thresholds: minimum deal score (e.g., ≥7) and maximum price limit (e.g., ≤$500). Only deals meeting both criteria proceed to the final alert stage.

Step 6: Telegram Alert Delivery

Filtered deals are formatted into a concise alert message and sent to your Telegram chat via a bot. The message includes product title, price, score, and direct link for quick action.

What You'll Need

  1. n8n instance (cloud or self‑hosted) with basic workflow import capability.
  2. Decodo API credentials for web‑scraping (sign up at decodo.com).
  3. LLM API access (OpenAI, Google Gemini, etc.) for the AI scoring node.
  4. Telegram Bot Token and your chat_id (create via BotFather).
  5. Basic understanding of n8n node configuration (credentials, field mapping).

Pro tip: Start with a low scanning frequency (once per day) to test the workflow’s stability and alert accuracy before increasing to hourly runs.

Quick Setup Guide

  1. Import the template into your n8n workspace via the downloaded JSON file.
  2. Configure Decodo node with your API credentials; optionally adjust the target eBay URL if you want to monitor a specific deals category.
  3. Set up AI agent node with your LLM credentials; review the prompt to ensure it matches your scoring priorities (you can tweak category emphasis).
  4. Add Telegram bot token and chat_id in the Telegram node; customize the alert message format if desired.
  5. Adjust filtering rules in the IF node: set your minimum deal score and maximum price threshold.
  6. Test manually by executing the workflow once; verify that alerts arrive correctly and contain relevant deals.
  7. Enable scheduling if you want continuous monitoring; set an appropriate interval based on your needs.
