What This Workflow Does

This automation solves a critical problem for businesses operating in the cryptocurrency space: the lack of visibility into large, potentially suspicious, on-chain transactions. Manually scanning the Ethereum blockchain for high-value USDT transfers is inefficient, error-prone, and doesn't scale.

This n8n workflow automatically monitors the Ethereum mainnet for USDT (Tether) transfer events. It connects to a node provider (like Alchemy), fetches the latest block data, extracts all USDT transfers, filters for transactions above a configurable threshold (e.g., $1M+), logs the structured data into an Airtable base for permanent record-keeping, and sends a consolidated daily summary alert to a designated Slack channel.

You move from reactive, manual checking to a proactive, automated surveillance system. It provides a clear audit trail, ensures no large movement goes unnoticed, and keeps your team informed without notification fatigue.

How It Works

The workflow is triggered on a daily schedule, but can be adjusted to run hourly or in real-time.

1. Fetch the Latest Block

The workflow starts by calling an Ethereum JSON-RPC endpoint (via Alchemy or Infura) to get the most recent block number. This ensures you're always analyzing the newest data.

2. Query USDT Transfer Events

Using the standard ERC-20 Transfer event signature, the workflow queries the blockchain logs for the USDT contract address. It retrieves all transfer events from the latest block, capturing sender, receiver, and amount data in raw hexadecimal format.

3. Decode and Transform Data

A Function node converts the hex-encoded values into human-readable formats. Wallet addresses are converted to checksum format, and token amounts are adjusted for decimals (USDT uses 6 decimals). This creates clean, analyzable data.

4. Filter for High-Value Transactions

An IF node applies your business logic, filtering out transfers below your defined threshold. You can easily set this to monitor for "whale" movements (e.g., >1,000,000 USDT) or any value relevant to your risk model.

5. Log to Airtable

Each qualifying transaction is written as a new record in your Airtable base. Fields include transaction hash, block number, timestamp, from/to addresses, amount, and a direct link to Etherscan for further investigation.

6. Generate & Send Slack Alert

The workflow aggregates the day's results, counting total large transfers and identifying the single largest transaction. A clean, formatted message is then sent to your chosen Slack channel, providing instant, actionable insight without spamming the team.

What You'll Need

  1. An n8n Instance: Either n8n.cloud (hosted) or a self-hosted n8n setup.
  2. Ethereum Node Access: An API key from a node provider like Alchemy, Infura, or QuickNode for the Ethereum mainnet.
  3. Airtable Base: A pre-configured Airtable base with a table to receive the transaction data. You'll need a Personal Access Token from your Airtable account.
  4. Slack Workspace: Access to a Slack workspace where you can create a webhook or install an app to post messages to a channel.
  5. Basic Web3 Knowledge: Understanding of concepts like smart contracts, transaction hashes, and block numbers is helpful but not required to run the workflow.

Pro tip: For the most reliable data, use a paid tier from Alchemy or Infura. Free tiers have rate limits and lower reliability, which could cause you to miss critical transactions.

Quick Setup Guide

Follow these steps to import and configure this workflow in your n8n instance.

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n editor, use the "Import from File" option to load the workflow.
  2. Configure Blockchain Connection: Open the two "HTTP Request" nodes. Replace the placeholder URL with your own Alchemy or Infura HTTPS endpoint (e.g., https://eth-mainnet.g.alchemy.com/v2/YOUR_API_KEY).
  3. Connect Airtable: Create a new Airtable resource in n8n or use an existing one. Map the node fields to your Airtable table columns: contract, from, to, value, blockNumber, txHash.
  4. Set Up Slack: Use the Slack node and authenticate with your workspace. Specify the channel where you want daily alerts to be posted.
  5. Adjust the Threshold: Open the "IF" node. Change the value in the condition to match your monitoring needs (e.g., item.json.value > 1000000000000 for 1,000,000 USDT).
  6. Test & Activate: Click "Execute Workflow" to test. Check Airtable for new records and Slack for the alert. Once verified, activate the "Schedule Trigger" node to run daily.
