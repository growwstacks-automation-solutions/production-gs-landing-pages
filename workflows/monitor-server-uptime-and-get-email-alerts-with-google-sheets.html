Monitor Server Uptime & Get Email Alerts with Google Sheets

Automate server uptime monitoring with this free n8n workflow. Ping servers, log status in Google Sheets, and get instant email alerts when a server goes down.

What This Workflow Does

This automation solves a critical problem for businesses and IT teams: maintaining visibility over server uptime without manual effort. Instead of relying on expensive monitoring tools or sporadic checks, this workflow continuously pings your web servers, logs their status in Google Sheets, and sends immediate email alerts when any server becomes unreachable.

It transforms reactive troubleshooting into proactive maintenance. You no longer wait for users to report issues—you know instantly when something fails. The system creates an audit‑ready historical log, enabling uptime reporting, SLA compliance tracking, and trend analysis. Whether you manage one critical server or dozens across different platforms, this workflow gives you peace of mind and actionable data.

The beauty lies in its simplicity. Adding or removing servers requires only a spreadsheet update—no code changes. The workflow runs silently in the background, checking every minute, logging every result, and alerting only when necessary. It’s a foundational automation that prevents downtime‑related revenue loss, customer frustration, and reputation damage.

How It Works

1. Scheduled Trigger

The workflow activates automatically every minute (or any interval you set). This ensures real‑time monitoring without manual initiation. The schedule node is the heartbeat of the automation.

2. Fetch Server List from Google Sheets

It reads a Google Sheet named “Server_List” that contains your server URLs or IP addresses. Each row represents one server to monitor. This central list makes management effortless—just edit the sheet to scale.

3. HTTP Request to Ping Each Server

For each server in the list, the workflow sends an HTTP GET request. If the server responds successfully, the workflow continues to the “Alive” log. If the request fails (server down, timeout, error), it triggers the error path automatically via “continueOnFail” logic.

4. Log Alive Servers in Google Sheets

Successful pings are recorded in a “Server_Status_Alive” sheet with timestamp, server address, and status “Alive”. This creates a positive uptime log for reporting and historical analysis.

5. Send Email Alert via Gmail

When a server fails, the Gmail node sends an alert to your designated admin email. The message includes the server address, failure timestamp, and suggested action—ensuring immediate awareness.

6. Log Down Servers in Google Sheets

Failed pings are logged separately in a “Server_Status_Down” sheet with timestamp and server details. This provides a failure history for debugging, SLA tracking, and post‑incident review.

What You'll Need

  1. A Google Sheet with a column named “Server” containing your server URLs/IPs.
  2. Google Sheets and Gmail accounts connected to n8n via OAuth2.
  3. A running n8n instance (cloud or self‑hosted) to execute the workflow regularly.
  4. Basic understanding of your server addresses and expected response behavior.

Quick Setup Guide

1. Download the template JSON file and import it into your n8n workspace.

2. Connect your Google Sheets account to the “Server_List”, “Alive”, and “Down” sheet nodes.

3. Connect your Gmail account to the “Server Down Notification” node and set the recipient email.

4. Update the “Server_List” sheet with your actual server addresses (e.g., http://your-server.com).

5. Adjust the schedule trigger interval if needed (default is 1 minute).

6. Test with one live server and one dummy/unreachable address to verify alive logging, down logging, and email alerting.

7. Activate the workflow. It will now run automatically, monitor your servers, and alert you of any failures.

Pro tip: For critical production servers, consider adding a secondary “watchdog” workflow that monitors this primary one, ensuring the monitoring system itself doesn’t fail silently.
