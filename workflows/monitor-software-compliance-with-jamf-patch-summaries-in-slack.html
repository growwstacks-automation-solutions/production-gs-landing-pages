What This Workflow Does

This automation solves the challenge of keeping IT and security teams informed about software patch compliance across all managed devices. Instead of manually checking Jamf or waiting for scheduled reports, this workflow automatically posts summarized patch compliance data to designated Slack channels.

The solution helps organizations maintain better security posture by providing real-time visibility into outdated software installations. IT teams can quickly identify and address vulnerabilities without switching between multiple systems.

How It Works

1. Fetch software titles from Jamf

The workflow begins by retrieving a list of all software titles managed in Jamf Pro through its API.

2. Filter target applications

Focuses on specific software you want to monitor (like Chrome, Edge, or critical security tools) by filtering the complete list.

3. Retrieve patch compliance data

For each filtered application, the workflow pulls detailed patch summary information including version distribution across devices.

4. Format for Slack

Transforms the raw compliance data into visually clear Slack messages using Block Kit formatting for optimal readability.

5. Post to Slack channel

Delivers the formatted compliance report to specified Slack channels where IT teams can take immediate action.

Who This Is For

This automation is ideal for:

  • IT operations teams managing macOS/iOS devices
  • Security teams monitoring patch compliance
  • MSPs managing multiple client environments
  • Organizations with Jamf and Slack already in use

What You'll Need

  1. Jamf Pro administrator access
  2. API credentials with permissions to read software titles and patch summaries
  3. Slack workspace with webhook or app integration permissions
  4. n8n instance (cloud or self-hosted)

Quick Setup Guide

  1. Import the JSON template into your n8n instance
  2. Configure Jamf API credentials in the HTTP Request nodes
  3. Set your target software IDs in the Filter node
  4. Add your Slack webhook URL or OAuth token
  5. Test with a manual trigger before scheduling

Pro tip: Start with daily reports and adjust frequency based on your patch management cycle. Critical applications may warrant more frequent updates.
