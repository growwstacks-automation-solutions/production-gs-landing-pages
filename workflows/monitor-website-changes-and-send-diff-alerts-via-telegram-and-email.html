What This Workflow Does

This automation solves the tedious problem of manually checking websites for updates. Whether you're tracking competitor pricing changes, monitoring job postings for recruitment, watching regulatory policy updates, or following product launches, this workflow does the heavy lifting for you.

It automatically monitors a list of URLs on a schedule, fetches each page, extracts clean text content, compares it against previous snapshots stored in Google Sheets, generates detailed line-by-line diffs with change percentages and severity ratings, and sends instant alerts via Telegram and email when changes are detected. No more missed updates or hours wasted on manual checking.

How It Works

1. Scheduled Monitoring

A schedule trigger checks your target URLs at configurable intervals—from every 15 minutes for critical pages to daily for low-priority sites. The workflow runs automatically 24/7 without manual intervention.

2. Content Fetching & Cleaning

HTTP requests fetch current HTML content with browser-like headers. A content extractor then strips HTML tags, scripts, styles, and navigation elements to get clean, readable text while preserving page titles and meta descriptions.

3. Change Detection Engine

The system loads previous snapshots from Google Sheets and performs line-by-line comparisons. It calculates precise change percentages, assigns severity levels (low/medium/high/critical), and generates human-readable diff summaries showing exactly what was added or removed.

4. Smart Filtering & Storage

Only pages with actual changes pass through the filter—eliminating noise and false alerts. New versions are automatically saved to Google Sheets, creating a complete historical record for trend analysis and audit trails.

5. Multi-Channel Alerting

Formatted notifications with diff summaries, change percentages, severity ratings, and direct links are sent instantly via Telegram for mobile alerts and email for formal documentation. Teams get notified exactly when changes occur.

What You'll Need

  1. Google Sheets account with OAuth credentials for storing snapshots and change history
  2. Telegram bot created via @BotFather with bot token and chat ID for instant alerts
  3. SMTP or Gmail credentials for email notifications (optional but recommended)
  4. n8n instance (cloud or self-hosted) to run the automation
  5. List of target URLs you want to monitor regularly

Pro tip: Start with monitoring 3-5 critical URLs to validate the system works for your use case before scaling to dozens of pages. This helps you fine-tune detection sensitivity and alert formatting.

Quick Setup Guide

  1. Add your URLs: Edit the "URL List" code node and add target URLs with friendly names and optional CSS selectors for specific page sections.
  2. Configure Google Sheets: Create a spreadsheet with the required columns and connect your OAuth credentials in both Sheet nodes.
  3. Set up Telegram: Create a bot via @BotFather, get your token and chat ID, and connect the Telegram credential.
  4. Configure email (optional): Connect your SMTP or Gmail credential and set from/to addresses in the Email Alert node.
  5. Adjust schedule: Set the cron interval based on your monitoring needs—more frequent for critical pages, less for informational sites.
  6. Run initial baseline: Execute the workflow once manually to save baseline snapshots. Change detection begins from the second run onward.
