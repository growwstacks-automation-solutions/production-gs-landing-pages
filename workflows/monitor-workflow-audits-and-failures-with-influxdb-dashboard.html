What This Workflow Does

This automation solves a critical visibility gap for teams running self-hosted n8n instances: the lack of centralized monitoring for workflow execution health, failure patterns, and security audits. Without proper monitoring, automation failures can go unnoticed for days, leading to broken business processes, data inconsistencies, and security vulnerabilities.

The workflow automatically collects execution data from your n8n instance, transforms it into time-series metrics compatible with InfluxDB, and pushes this data to create real-time dashboards. This gives DevOps teams and business owners immediate visibility into automation performance, failure rates, and audit trails—all within the monitoring tools they already use.

Pro tip: This monitoring approach is especially valuable for regulated industries where audit trails are mandatory. By automating data collection, you create immutable logs of all automation activity for compliance reporting.

How It Works

The automation follows a straightforward pipeline that transforms raw n8n execution data into actionable monitoring insights.

Step 1: Data Collection from n8n API

The workflow uses n8n's native HTTP Request nodes to query its own execution API, gathering data on recent workflow runs, success/failure statuses, execution times, and error messages. This self-monitoring capability is built right into n8n's architecture.

Step 2: Data Transformation for Time-Series

Raw execution data is processed through Set nodes to structure it according to InfluxDB's line protocol format. This includes adding precise timestamps, tagging data by workflow name, error type, and environment, and converting execution metrics into numerical fields that InfluxDB can efficiently store and query.

Step 3: Delivery to InfluxDB

Transformed data is sent to your InfluxDB instance via HTTP POST requests. The workflow handles authentication, batch processing for efficiency, and error recovery if the database is temporarily unavailable. Failed deliveries are queued for retry to ensure no monitoring data is lost.

Step 4: Dashboard Visualization

Once in InfluxDB, the data becomes available for visualization in tools like Grafana or Chronograf. You can create dashboards showing success rates over time, failure hotspots, execution duration trends, and custom alerts when thresholds are breached.

What You'll Need

  1. A self-hosted n8n instance with API access enabled and an API key created for programmatic access
  2. An InfluxDB instance (cloud or self-hosted) with a bucket created to receive the monitoring data
  3. Your InfluxDB organization ID and bucket name for proper data routing
  4. Basic familiarity with n8n to import and configure the workflow template
  5. Optional: Grafana or similar visualization tool to create dashboards from the InfluxDB data

Pro tip: If you're new to InfluxDB, start with their cloud offering which provides a free tier sufficient for monitoring a single n8n instance. The setup is straightforward and eliminates infrastructure management.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance using the "Import from File" option
  2. Configure the three required variables: Your InfluxDB URL, organization ID, and bucket name in the workflow settings
  3. Add your n8n API key to the HTTP Request node that queries execution data (use n8n's credential system for security)
  4. Test the workflow manually to verify data flows correctly to your InfluxDB instance
  5. Schedule the workflow to run daily (or more frequently for critical systems) using n8n's schedule trigger
  6. Create your first dashboard in Grafana or Chronograf using the newly available n8n metrics
