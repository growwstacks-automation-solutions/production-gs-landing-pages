What This Workflow Does

Manually checking multiple YouTube channels for new content is a time-consuming task that often gets overlooked. This automation solves that by providing a daily, consolidated summary of new videos from your monitored channels directly in your Discord server.

Unlike complex YouTube API integrations that require Google Cloud projects and managing API quotas, this workflow uses the simple, reliable RSS feeds that YouTube provides for every public channel. It runs on a daily schedule, filters for videos published in the last 24 hours, and formats a clean notification with titles, links, and publish dates.

The business value is clear: save 1-2 hours per week of manual checking, ensure your team never misses important content from competitors, industry leaders, or favorite creators, and centralize updates in a collaborative platform where discussions can happen immediately.

How It Works

The workflow follows a logical sequence to gather, filter, and deliver content efficiently.

1. Define Channels to Monitor

You provide a list of YouTube Channel IDs (the "UC..." identifiers). The template includes example channels but you replace these with your own targets—competitors, influencers, or educational creators relevant to your business.

2. Fetch RSS Feeds

For each channel ID, the workflow retrieves the public RSS feed. This feed contains the 15 most recent videos with basic metadata: title, link, description snippet, and publication date.

3. Apply Date Filter

A strict filter passes only videos published within the last 24 hours. This prevents sending old content and ensures your daily summary is truly "daily."

4. Format & Send to Discord

Filtered videos are formatted into an attractive Discord message using webhooks. The notification includes channel name, video title as a clickable link, and publish time, all delivered to your specified Discord channel.

Pro tip: Use this for competitive intelligence by monitoring competitor channels. Get notified the moment they release new product demos, tutorials, or announcements without visiting YouTube.

Quick Setup Guide

Get this automation running in under 10 minutes with these simple steps.

  1. Download and import the JSON template into your n8n instance.
  2. Edit the "Define Channel IDs" node to replace example IDs with your target YouTube channels.
  3. Create a Discord webhook in your server (Settings → Integrations → Webhooks).
  4. Add the webhook URL to the "Discord Notification" node credentials in n8n.
  5. Activate the workflow and test with the "Execute Workflow" button.
  6. Schedule it daily (default 8:30 AM) or adjust the schedule trigger to your preferred time.

Pro tip: Start with 3-5 channels to test, then expand. The workflow can handle dozens of channels efficiently since RSS calls are lightweight and don't hit API rate limits.
