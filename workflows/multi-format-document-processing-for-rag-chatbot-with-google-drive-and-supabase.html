What This Workflow Does

This automation solves a critical bottleneck in implementing AI chatbots for businesses: getting your documents into the system. Most companies have valuable information trapped in PDFs, Word documents, Excel spreadsheets, and Google Docs, but manually processing these for AI consumption is time-consuming and error-prone.

The workflow automatically monitors a designated Google Drive folder for new files, intelligently processes them based on their format, extracts the text content, and prepares it for your RAG (Retrieval-Augmented Generation) chatbot. This creates a self-updating knowledge base where your AI assistant can answer questions based on your actual business documents, not just general knowledge.

How It Works

1. Document Detection & Routing

The workflow starts when a new file appears in your specified Google Drive folder. It immediately identifies the file type (PDF, Word, Excel, Google Doc) and routes it to the appropriate processing path using intelligent switching logic.

2. Multi-Format Text Extraction

Each document type follows an optimized extraction path. PDFs have their text extracted directly. Google Docs are downloaded and converted to plain text. Excel files have their data aggregated into coherent text blocks. Word documents are automatically converted to Google Docs format for consistent processing.

3. Intelligent Chunking & Enrichment

The extracted text is split into manageable chunks (around 2000 characters) while preserving semantic boundaries. Each chunk is enriched with metadata from the original file—file name, creator, creation date—so the AI understands the context of the information it's retrieving.

4. Vectorization & Storage

Finally, the workflow uses OpenAI's embeddings to create vector representations of the text chunks and stores them in a Supabase vector database. This creates a searchable knowledge base that your RAG chatbot can query to provide accurate, document-based answers.

What You'll Need

  1. A Google Drive account with documents to process
  2. A Supabase account with vector extension enabled
  3. OpenAI API key for embeddings (or alternative embedding service)
  4. n8n instance (cloud or self-hosted)
  5. A designated Google Drive folder for document ingestion

Quick Setup Guide

1. Download the template using the button above and import it into your n8n instance.

2. Configure the Google Drive trigger node with your folder ID and set up authentication.

3. Connect your Supabase credentials in the vector store node and ensure the vector extension is enabled.

4. Add your OpenAI API key to the embeddings node (or configure an alternative embedding service).

5. Test the workflow by adding a document to your monitored Google Drive folder and verify it appears in your Supabase vector store.

Pro tip: Start with a test folder containing a few sample documents of different types before connecting your main document repository. This lets you verify processing works correctly without affecting production data.
