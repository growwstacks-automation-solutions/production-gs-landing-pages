Multi-service Terminal with Gemini AI

Access Gmail, X, Telegram & news via retro console using AI agent

Download Template JSON · n8n compatible · Free

What This Workflow Does

This innovative workflow transforms any retro console or simple terminal into a powerful multi-service access point. Using Gemini AI as the brain, it interprets plain English commands to interact with Gmail, X (Twitter), Telegram, RSS feeds, and even execute Linux commands.

The system is designed for minimalists and productivity enthusiasts who want to reduce digital clutter. Instead of switching between multiple apps and browsers, you can check emails, read news, send messages, and more through a single retro-style interface.

How It Works

1. Command Input

Users type natural language commands into a retro console interface (running on an old notebook or netbook). The Python script sends these commands to the workflow via webhook.

2. AI Processing

Gemini AI analyzes the command to determine which services need to be accessed and what actions to take. It can handle complex multi-service requests like "Check my unread emails and summarize the top tech news."

3. Service Integration

The workflow connects to each required service using their respective APIs (Gmail, X, Telegram, RSS) with proper authentication. It executes the necessary actions while maintaining security.

4. Response Delivery

Results are formatted and returned to the console interface, keeping the retro aesthetic while delivering modern functionality.

What You'll Need

  1. Gemini API key from Google
  2. Google Cloud credentials for Gmail and Calendar access
  3. X API credentials (if accessing Twitter functionality)
  4. Telegram bot token
  5. RSS feed URLs you want to monitor
  6. A Linux machine or old notebook to run the console interface

Quick Setup Guide

  1. Download and import the JSON workflow into your n8n instance
  2. Configure all credential nodes with your API keys
  3. Set up the webhook URL that your console script will use
  4. Copy the provided Python script to your Linux machine
  5. Update the script with your webhook URL
  6. Run the script and start issuing commands

Pro tip: For security, create dedicated API keys and OAuth credentials just for this automation rather than using your personal credentials.
