What This Workflow Does

This n8n workflow solves a common but tedious problem: manually processing URLs for sharing, tracking, or analysis. When you have a list of links—whether from blog posts, product pages, or campaign assets—you need to extract their metadata (title, description, image), possibly capture a screenshot, and create a branded short link for clean sharing. Doing this manually for dozens or hundreds of URLs is slow and error-prone.

The workflow automates the entire pipeline. It takes raw URLs, parses them to pull OpenGraph data, fetches favicons, can capture screenshots, and then creates or updates shortened links via Switchy.io. The result is a ready‑to‑use shortened URL enriched with metadata, hosted images (via GitHub), and organized for your campaigns.

This is especially valuable for marketing teams, content creators, and developers who manage link sharing at scale. It turns a multi‑step manual task into a single automated process, saving hours per week and ensuring consistency.

How It Works

Step 1: Batch Input & API Auth

The workflow starts with a “Split In Batches” node, allowing you to process URLs in groups. An “API Auth” node configures credentials for Switchy.io, OpenGraph, GitHub, and other services, ensuring secure access.

Step 2: URL Metadata Extraction

Each URL passes through a series of extraction nodes: “Get Headers,” “OpenGraph API,” and “Meta Tags Scraper.” These pull the page title, description, OpenGraph image, and favicon. Conditional nodes (“IF OpenGraph Invalid,” “If – Enable ScreenShots”) handle edge cases and optional screenshot capture via Microlink or pxl.to.

Step 3: Data Aggregation & Image Hosting

A “Method 1 – META” node aggregates all extracted metadata. If screenshots are enabled, they are uploaded to a personal GitHub repository via the “GitHub” node, making them accessible via CDN (Statically). This ensures all visual assets are stored and linked properly.

Step 4: Shortened Link Creation

The final step uses the “Switchy API” node to create or update a shortened link. The new link includes all the extracted metadata and, if available, a screenshot URL. The output is a clean, branded short URL ready for sharing or tracking.

Pro tip: Configure the GitHub node with your repository details before running the workflow. This allows the workflow to host screenshots and images directly in your account, giving you full control over the assets.

What You'll Need

  1. A running n8n instance (cloud or self‑hosted).
  2. Switchy.io API credentials (for creating/updating short links).
  3. GitHub API token (if you want to host screenshots/images).
  4. Optional: Microlink or pxl.to API keys for screenshot capture.
  5. A list of URLs to process (from a spreadsheet, database, or incoming webhook).

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Configure the “API Auth” node with your Switchy.io, GitHub, and optional screenshot service credentials.
  3. Adjust the batch size in “Split In Batches” based on your URL volume.
  4. Connect your URL source—could be a Google Sheets node, a database trigger, or a manual input.
  5. Test with a single URL to verify metadata extraction and link creation.
  6. Activate the workflow and let it process your URL list automatically.
