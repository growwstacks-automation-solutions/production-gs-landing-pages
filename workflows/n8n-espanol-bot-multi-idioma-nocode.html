What This Workflow Does

This automation solves a critical problem for businesses serving international customers: language barriers in real-time communication. The workflow creates a Telegram bot that automatically detects, translates, and responds to messages in multiple languages.

When a customer messages your Telegram bot in Spanish, French, German, or any other language, the system instantly translates their query into your team's preferred language. Your support agent can then respond in English (or any language), and the bot automatically translates the reply back to the customer's language before sending it. This creates a seamless bilingual conversation where neither party needs to know the other's language.

The template eliminates the need for bilingual staff or manual copy-pasting into translation tools. It's particularly valuable for e-commerce stores, SaaS companies with global users, travel businesses, and any organization receiving customer inquiries in multiple languages through messaging platforms.

How It Works

The workflow connects Telegram with translation services through n8n's visual automation platform. Here's the step-by-step process:

1. Message Reception & Language Detection

The Telegram trigger node listens for new messages in your bot. When a message arrives, the workflow captures the text content and user information. An initial language detection step identifies the source language using AI or simple pattern matching.

2. Translation Processing

The core message text is sent to a translation service (like Google Translate, DeepL, or Azure Translator). The workflow can be configured to translate between any language pair—common setups include Spanish↔English, French↔English, German↔English, or multi-language routing where messages are translated to a central language for team handling.

3. Response Management

Translated messages are formatted and routed. They can be sent directly to a support agent with context about the original language, stored in a database for tracking, or processed through additional automation rules. When your team responds, the workflow reverses the process—translating your reply back to the customer's language before delivering it through Telegram.

4. Delivery & Logging

The final translated response is sent back to the user through the Telegram Send Message node. Simultaneously, the entire conversation—original message, translation, response, and timestamps—can be logged to Google Sheets, Airtable, or your CRM for quality assurance and analytics.

What You'll Need

  1. A Telegram Bot Token: Create a bot via @BotFather on Telegram and save the API token.
  2. Translation API Access: Sign up for a free tier of Google Cloud Translation, DeepL, or Microsoft Translator.
  3. n8n Instance: Either self-hosted n8n or a n8n.cloud account.
  4. Basic Webhook Understanding: Know how to set up webhooks (the template guides you through this).
  5. Target Languages Defined: Decide which languages you need to support (the template is easily adjustable).

Pro tip: Start with just 2-3 key languages your customers use most frequently. You can always add more languages later as your international business grows.

Quick Setup Guide

Follow these steps to implement this multilingual bot in under 30 minutes:

  1. Import the template: Download the JSON file above and import it into your n8n instance.
  2. Configure Telegram: In the Telegram Trigger node, paste your bot token from @BotFather.
  3. Set up translation: Add your Google Translate/DeepL API credentials to the HTTP Request or dedicated translation node.
  4. Define language pairs: Adjust the workflow to specify which languages to translate from/to (default is Spanish↔English).
  5. Test thoroughly: Send test messages in different languages to verify translation accuracy and response flow.
  6. Deploy: Activate the workflow and share your bot's link with customers or add it to your website/communications.
