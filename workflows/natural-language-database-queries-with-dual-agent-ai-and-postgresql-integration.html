What This Workflow Does

This workflow creates a smart database assistant that safely handles natural language queries without crashing your system. It solves the problem of non-technical team members needing to access database insights but lacking SQL skills, while preventing dangerous queries that could overload your database.

Instead of writing complex SQL code, users can ask questions like "Show me top 10 popular products" or "What were last month's sales by region?" The AI interprets these requests, generates optimized PostgreSQL queries with built-in safety limits, executes them, and returns formatted results. The dual-agent architecture ensures both accuracy and system protection.

How It Works

Step 1: Natural Language Input

A user submits a question in plain English through a webhook, chat interface, or form. The workflow captures this input and prepares it for AI processing.

Step 2: Main AI Agent Interpretation

The first AI agent analyzes the request, identifies the intent, and extracts key parameters. It ensures the query includes safety limits (default 10 results, maximum 50) and validates that the request is appropriate for database access.

Step 3: SQL Sub-Agent Generation

A specialized SQL agent converts the interpreted request into precise PostgreSQL syntax. This agent understands your database schema, handles table joins, WHERE clauses, and ORDER BY statements, while maintaining the safety limits set by the main agent.

Step 4: Database Execution & Results

The generated SQL query executes against your PostgreSQL database. Results are formatted for readability, with clear column headers and appropriate data presentation, then returned to the user through the original interface.

Quick Setup Guide

1. Download the template and import it into your n8n instance.

2. Configure your PostgreSQL credentials in the database node.

3. Set up your AI provider credentials in both agent nodes.

4. Update the table names and field references in the SQL templates to match your schema.

5. Test with sample queries like "Show me 5 recent orders" to verify the flow works correctly.

6. Connect your preferred input method (webhook, form, chat interface) to trigger the workflow.

Pro tip: Start with a read-only database user for maximum safety. Test all query types before giving business users access, and monitor the first few days of usage to catch any unexpected query patterns.
