What This Workflow Does

This intelligent automation system solves a critical business problem: data trapped in spreadsheets that only technical users can analyze. Most companies have valuable information in Google Sheets but lack the SQL skills to extract meaningful insights. Team members waste hours manually filtering, sorting, and creating pivot tables instead of getting instant answers.

The workflow bridges this gap by converting natural language questions into Google Sheets SQL queries using OpenAI's GPT-4o model. When a user asks "What were our top 5 products by revenue last quarter?" the AI generates the proper query syntax, executes it via HTTP requests, and returns formatted results. This eliminates the need for complex formulas, pivot tables, or manual data manipulation.

Beyond simple queries, the system maintains conversation context for follow-up questions, automatically maps column names to their corresponding letters (A, B, C), and bypasses API limits by using Google Sheets' built-in query functionality. The result is a conversational data analyst available 24/7 to anyone in your organization.

How It Works

Step 1: Natural Language Processing

The workflow begins when a user submits a question in plain English through a chat interface or webhook. The OpenAI GPT-4o model analyzes the question, understands the intent, and identifies the required data points, filters, and calculations needed.

Step 2: Schema Understanding & Column Mapping

The system connects to your Google Sheet to understand its structure. It reads column metadata and creates a mapping between human-readable column names (like "Monthly Revenue") and their corresponding spreadsheet letters (like "Column D"). This mapping ensures the generated SQL references the correct data.

Step 3: SQL Query Generation

Using the understood intent and schema, GPT-4o generates a properly formatted Google Sheets SQL query. This includes SELECT statements, WHERE filters, ORDER BY clauses, and aggregation functions like SUM, AVERAGE, or COUNT. The AI considers business context to apply appropriate logic.

Step 4: Query Execution & Data Retrieval

The generated query is converted into a Google Sheets query URL and executed via HTTP request. This approach bypasses traditional API limits and provides direct access to your spreadsheet data. The system retrieves live results without manual intervention.

Step 5: Result Formatting & Presentation

A secondary GPT-4.1 Mini model formats the raw data into human-readable insights. It creates clear summaries, highlights key findings, and presents the information in tables or bullet points. The formatted answer is then delivered back to the user through the original interface.

Step 6: Conversation Memory Management

A memory buffer maintains context throughout the conversation. When users ask follow-up questions like "Now show me that by region," the system remembers previous queries and builds upon them, creating a natural analytical dialogue rather than isolated question-answer pairs.

What You'll Need

  1. An n8n instance (cloud or self-hosted version)
  2. OpenAI API account with billing enabled and GPT-4o access
  3. Google account with access to Google Sheets
  4. A Google Sheet with data you want to analyze (set to "Anyone with link can view" for HTTP access)
  5. Google Cloud Console project with Sheets API enabled and OAuth2 credentials
  6. Basic understanding of your spreadsheet structure and column names

Quick Setup Guide

  1. Import the workflow: Download the JSON template and import it into your n8n instance via Workflows → Import from JSON.
  2. Configure OpenAI: Create an OpenAI API key and add it to both AI model nodes in the workflow.
  3. Set up Google Sheets: Create OAuth2 credentials in Google Cloud Console, enable Sheets API, and connect your account in the "Get Column Info" node.
  4. Prepare your spreadsheet: Ensure your Google Sheet has clear column headers and is shared with "Anyone with link can view" permissions.
  5. Update sheet references: Enter your Google Sheet ID and sheet name in the configuration nodes.
  6. Test the system: Activate the workflow and ask a simple question like "How many rows are in the spreadsheet?" to verify connectivity.
  7. Deploy for team use: Set up a webhook or chat interface for team members to submit questions directly.
