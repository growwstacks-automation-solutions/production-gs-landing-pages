What This Workflow Does

This automated workflow creates a rapid response system for email security incidents. When Sublime Security quarantines a suspicious email, it immediately notifies the intended recipient via Slack with details about the potential threat. If the user accidentally opens the quarantined email despite the warning, the system automatically creates a detailed Jira ticket to document the security incident.

The solution bridges the gap between email security systems and team collaboration platforms, ensuring threats are both prevented and properly documented. It reduces reliance on manual IT processes while maintaining compliance with security protocols through automated audit trails.

Example Slack notification showing quarantined email details

How It Works

1. Email Quarantine Detection

The workflow triggers when Sublime Security identifies and quarantines a suspicious email. It extracts key details like sender, subject, recipient, and threat type.

2. Slack Notification

An interactive Slack message is sent to the recipient with the quarantine details and instructions. The message includes buttons to request release or confirm quarantine.

3. Open Email Monitoring

If the recipient opens the quarantined email despite the warning, the workflow detects this through Sublime Security's activity logs.

4. Jira Ticket Creation

The system automatically creates a Jira ticket in your security project with all incident details, including timestamps, user information, and email content metadata.
