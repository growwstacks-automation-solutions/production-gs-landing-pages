What This Workflow Does

This automation solves a common problem for content creators, educators, and knowledge workers: you have valuable written content in Obsidian, but limited time to repurpose it into audio formats. Manual podcast production requires recording, editing, and publishing—a process that can take hours per episode.

The workflow automatically converts your Obsidian notes into professionally narrated podcast episodes using AI voice generation. When you publish or update a note, it triggers an automated process that converts text to speech, generates episode descriptions, stores audio files, and updates a podcast RSS feed compatible with all major platforms.

Beyond simple text-to-speech conversion, the system maintains your content structure, adds appropriate metadata, and creates a sustainable content pipeline that turns your note-taking habit into a regular podcast without additional effort.

How It Works

1. Note Trigger from Obsidian

Using the Obsidian Post Webhook plugin, the workflow detects when you create or update notes. The plugin sends the note content, title, and metadata to n8n via a webhook, initiating the audio conversion process.

2. AI Voice Generation with OpenAI

The workflow sends your note content to OpenAI's text-to-speech API, which converts it into natural-sounding audio. You can configure voice type, speed, and tone to match your brand or content style. Simultaneously, AI generates episode descriptions and summaries.

3. Audio Storage in Cloudinary

Generated audio files are uploaded to Cloudinary for reliable hosting and delivery. Cloudinary provides optimized streaming, CDN distribution, and automatic format conversion for different devices and bandwidth conditions.

4. Podcast Feed Generation

The workflow updates a Google Sheet (or similar database) with episode metadata, then generates a standards-compliant RSS feed. This feed includes all necessary elements for podcast platforms: episode titles, descriptions, publication dates, audio URLs, and duration.

5. Note Enhancement in Obsidian

Finally, the workflow adds the generated audio file link back to your original Obsidian note, creating a connected knowledge system where written and audio versions coexist and reference each other.

What You'll Need

  1. Obsidian with Post Webhook Plugin: The free community plugin that enables webhook triggers from your notes.
  2. OpenAI API Access: An account with credits for text-to-speech and optional description generation.
  3. Cloudinary Account: Free tier available for audio storage and delivery.
  4. n8n Instance: Self-hosted n8n or n8n.cloud account to run the workflow.
  5. Basic Podcast Metadata: Title, author name, cover art URL, and description for your podcast feed.

Pro tip: Start with a small subset of notes to test voice quality and pacing before automating your entire vault. Different content types (technical vs. narrative) may benefit from different AI voice settings.

Quick Setup Guide

  1. Download and Import: Download the template file and import it into your n8n instance.
  2. Configure Credentials: Set up API credentials for OpenAI and Cloudinary in n8n's credentials management.
  3. Install Obsidian Plugin: Add the Post Webhook plugin to Obsidian and configure it to point to your n8n webhook URL.
  4. Customize Podcast Metadata: Update the workflow nodes with your podcast title, author, description, and cover image URL.
  5. Test with Sample Note: Create a test note in Obsidian to trigger the workflow and verify all steps complete successfully.
  6. Submit RSS Feed: Take the generated RSS feed URL and submit it to your preferred podcast platforms (Apple, Spotify, etc.).
