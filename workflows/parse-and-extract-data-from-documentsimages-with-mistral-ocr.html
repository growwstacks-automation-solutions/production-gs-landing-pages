What This Workflow Does

Manual data entry from documents is one of the most time-consuming and error-prone tasks in business operations. Employees waste hours each week copying information from invoices, receipts, bank statements, and contracts into various systems. This workflow solves that problem by automating document data extraction using Mistral OCR, an AI-powered optical character recognition service.

The template processes multi-page PDFs and image files, extracts structured data like tables, dates, amounts, and text, and converts everything into usable formats like JSON or markdown. What makes Mistral OCR special is its ability to perform document page splitting and markdown conversion in a single API call, reducing dependencies on multiple tools. This means you can take a complex bank statement PDF and automatically extract all transaction data without touching a single spreadsheet.

Beyond simple text extraction, this workflow demonstrates three powerful approaches: direct file processing, secure cloud storage integration, and document understanding through question-answering. You can ask the AI specific questions about your documents, like "How many deposits are shown in this bank statement?" and get precise answers, transforming passive documents into interactive data sources.

How It Works

The workflow is built around Mistral Cloud's OCR API and demonstrates multiple implementation patterns for different security and use case requirements.

1. Direct Document Processing

The workflow first accesses the Mistral OCR API through HTTP request nodes. It accepts two file types: PDF documents and image files (PNG, JPG). For each document, it sends the file to Mistral's servers where AI models analyze the content, identify text regions, tables, and structural elements, then return the extracted data in structured formats.

2. Secure Cloud Storage Integration

For sensitive documents, the workflow demonstrates a more secure approach by first uploading files to Mistral's cloud storage, generating signed URLs, and then processing them through OCR. This ensures documents aren't publicly accessible and protects against unauthorized access while still leveraging the powerful extraction capabilities.

3. Document Understanding & Question Answering

Beyond raw extraction, the workflow shows how to use Mistral's document understanding capabilities. Instead of just extracting all text, you can ask specific questions about the document content. The AI analyzes the document and provides precise answers, enabling intelligent querying of your document archives without manual searching.

4. Data Transformation & Output

Once data is extracted, the workflow uses Set nodes to transform and structure the output. Tables are converted to CSV-compatible formats, text is cleaned and organized, and the final data is formatted for integration with downstream systems like databases, spreadsheets, or business applications.

What You'll Need

  1. Mistral Cloud Account: You'll need an API key from Mistral AI with credits available on your account (OCR usage is billed per page).
  2. n8n Instance: Either n8n.cloud (hosted) or a self-hosted n8n installation (version 1.0+ recommended).
  3. Documents in Accessible Locations: PDFs or images either publicly accessible via URL or uploaded to Mistral cloud storage/S3/R2 for secure processing.
  4. Basic API Knowledge: Understanding of how to configure API keys and HTTP requests in n8n.
  5. Target System: Where extracted data should go (Google Sheets, database, CRM, etc.) for complete automation.

Pro tip: Start with a pilot project focusing on one document type (like invoices) before expanding to multiple formats. This lets you refine extraction accuracy and validation rules specific to your use case before scaling.

Quick Setup Guide

  1. Download and Import: Click the download button above to get the JSON file, then import it into your n8n instance via the workflow import function.
  2. Configure Mistral API: Replace the placeholder API key in the HTTP Request nodes with your actual Mistral Cloud API key from your account dashboard.
  3. Test with Sample Documents: Use the provided test paths or replace them with URLs to your own PDFs/images. Run the workflow once to ensure connections work.
  4. Adjust Extraction Parameters: Modify the OCR request parameters based on your document types—enable table extraction for financial statements, text-only for contracts, etc.
  5. Connect Output Systems: Replace the final nodes with connections to your actual systems (database inserts, Google Sheets updates, webhook notifications).
  6. Implement Validation: Add data validation steps to check extraction accuracy before sending to critical systems, with human review for low-confidence extractions.
  7. Schedule or Trigger: Set up the workflow to run on a schedule (daily invoice processing) or trigger via webhook (when documents arrive in a specific folder).
