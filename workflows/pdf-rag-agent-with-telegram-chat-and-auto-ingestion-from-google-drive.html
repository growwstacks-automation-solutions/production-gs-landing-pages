What This Workflow Does

This automation solves the problem of information silos in documents. Teams waste hours searching through PDFs, manuals, and policy documents to find specific answers. This workflow creates "Mookie"—an intelligent document assistant that provides instant, accurate answers based solely on your uploaded materials.

The system automatically processes PDFs from Google Drive, converts them into searchable vectors, and enables natural language questioning through Telegram. Employees get instant answers without interrupting subject matter experts, while maintaining conversation context for follow-up questions. It's like having a 24/7 expert who's read every document in your organization.

How It Works

1. Document Ingestion & Processing

The workflow monitors a designated Google Drive folder for new PDFs. When detected, files are automatically downloaded, text is extracted, and converted into vector embeddings using Mistral AI. These embeddings are stored in a PostgreSQL database with PGVector extension, creating a searchable knowledge base.

2. Intelligent Question Handling

When users ask questions via Telegram, the system converts the query into vectors and performs similarity searches against the stored document embeddings. It retrieves the most relevant document chunks, providing context to the AI model while maintaining conversation history for each user.

3. Context-Aware Response Generation

Using GPT-4 (or your preferred LLM), the system generates answers based exclusively on the retrieved document context. If the question isn't covered in the documents, the assistant politely declines to answer, preventing hallucinations. Responses are delivered back through Telegram with source citations.

4. Continuous Learning & Management

The system includes approval mechanisms for document ingestion, handles bulk processing from historical Google Drive files, and provides web interfaces for managing the knowledge base. Conversation logs help identify knowledge gaps for future document updates.

What You'll Need

  1. PostgreSQL database with PGVector extension installed (for vector storage)
  2. API credentials for Azure OpenAI (GPT-4) or alternative LLM provider
  3. Mistral Cloud API key for embeddings (or compatible alternative)
  4. Google Service Account with Drive API access to your target folder
  5. Telegram Bot Token created via @BotFather for chat interface
  6. n8n instance (cloud or self-hosted) with webhook capabilities

Pro tip: Start with a small set of well-structured documents (like an employee handbook) to validate the system before scaling to thousands of files. This helps identify formatting issues and optimize chunking strategies early.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Configure credentials for PostgreSQL, AI services, Google Drive, and Telegram in n8n's credentials management
  3. Update webhook URLs throughout the workflow to match your n8n instance's public URL
  4. Test the ingestion pipeline by placing a sample PDF in your Google Drive folder and verifying it appears in the database
  5. Validate the chat interface by sending a test question to your Telegram bot and checking the response accuracy
  6. Deploy the workflow and train your team on how to ask effective questions to get the best results
