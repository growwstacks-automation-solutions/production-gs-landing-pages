How It Works

The workflow follows a simple but powerful sequence, triggered automatically on a schedule you define.

Step 1: Schedule Trigger

A Cron or Schedule Trigger node initiates the workflow at your chosen frequency—every minute, hour, day, or custom interval. This is the automation’s heartbeat.

Step 2: HTTP Request

The workflow calls an external API, website endpoint, or data service using the HTTP Request node. You configure the URL, method (GET/POST), headers, and parameters to fetch the exact data you need.

Step 3: Data Processing

Optional Function or Set nodes can filter, transform, or enrich the retrieved data. You might extract specific fields, calculate values, or format the output for readability.

Step 4: Telegram Delivery

The processed data is sent to a Telegram chat or channel via the Telegram node. You can send plain text, formatted messages, or even images—all automatically.

What You'll Need

  1. A running n8n instance – either self‑hosted or n8n.cloud.
  2. A Telegram Bot Token – created via @BotFather on Telegram.
  3. The Telegram Chat ID – where messages should be sent (channel, group, or private chat).
  4. Access to your target API/URL – with any required authentication keys or parameters.
  5. Basic understanding of the data structure you want to fetch and send.

Quick Setup Guide

Importing and configuring this template takes about 10 minutes.

  1. Download the template using the button above and save the JSON file.
  2. In your n8n dashboard, go to Workflows → Import from File and select the downloaded JSON.
  3. Configure the Schedule Trigger node to set your desired frequency (e.g., every day at 9 AM).
  4. Update the HTTP Request node with your API URL, method, and any authentication headers.
  5. Edit the Telegram node with your Bot Token and Chat ID (find these in Telegram).
  6. Test the workflow by clicking “Execute Workflow” and verify the message appears in Telegram.
  7. Activate the workflow to let it run automatically on your schedule.

Pro tip: Start with a short test interval (like every 5 minutes) to confirm everything works, then switch to your production schedule.
