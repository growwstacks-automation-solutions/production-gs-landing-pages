What This Workflow Does

Manual candidate communication is one of the most time-consuming tasks in HR. Recruiters spend hours drafting emails, reviewing resumes for feedback, and tracking responses—often leading to generic messages or silent rejections that damage employer branding.

This workflow solves that by automating personalized communication for every applicant. It fetches candidate details from Google Sheets, downloads their resumes from Google Drive, analyzes them with GPT-4o AI, and sends tailored emails through Gmail. Shortlisted candidates receive congratulatory messages with onboarding plans, while rejected candidates get constructive feedback with learning resources.

The system ensures no candidate falls through the cracks, provides genuinely helpful feedback based on actual resume content, and maintains a professional, human touch at scale—freeing recruiters to focus on strategic hiring rather than administrative tasks.

How It Works

Step 1: Data Collection & Validation

The workflow starts by pulling structured candidate data from your Google Sheets database, including names, emails, resume links, skills, job information, and application status. It then downloads resumes from Google Drive links and validates each file exists and is accessible.

Step 2: AI-Powered Analysis

Valid resumes are processed through a PDF-to-text extractor, then combined with the candidate data into a comprehensive profile. GPT-4o analyzes each candidate's background, identifies skill gaps relative to the job requirements, and prepares personalized feedback.

Step 3: Branching Communication Paths

The workflow splits based on candidate status. Shortlisted candidates receive congratulatory HTML emails with identified skill gaps, recommended online courses (Coursera, Udemy, LinkedIn Learning), and clear next onboarding steps. Rejected candidates receive empathetic rejection emails with constructive feedback and specific learning resources to improve for future opportunities.

Step 4: Delivery & Error Handling

Personalized emails are sent directly via Gmail with professional HTML formatting. Any failed or missing resumes are logged in a dedicated Google Sheets error log for audit and follow-up, ensuring system resilience.

What You'll Need

  1. Google Sheets – A structured candidate database with columns for name, email, resume link, skills, job info, and status.
  2. Google Drive – Storage for candidate resumes with accessible sharing links.
  3. Gmail API access – To send candidate emails directly from your HR domain.
  4. Azure OpenAI (GPT-4o-mini) – API access for AI-powered feedback generation.
  5. n8n instance – Self-hosted or cloud version to run the workflow.

Quick Setup Guide

  1. Download the template – Click the download button above to get the JSON file.
  2. Import into n8n – In your n8n instance, go to Workflows → Import from file and select the downloaded JSON.
  3. Configure credentials – Set up connections for Google Sheets, Google Drive, Gmail, and Azure OpenAI in n8n's credentials management.
  4. Update sheet IDs – Replace the placeholder Google Sheet IDs in the workflow with your actual candidate database and error log sheet IDs.
  5. Test with one candidate – Run the workflow manually with a single test candidate to verify all connections work.
  6. Schedule or trigger – Set the workflow to run on a schedule (daily/weekly) or trigger it manually when new candidates need processing.

Pro tip: Start with a small batch of 5-10 candidates to test email templates and AI feedback quality. Adjust the GPT-4o prompts in the LLM Chain nodes to match your company's voice and specific feedback preferences before scaling to full volume.
