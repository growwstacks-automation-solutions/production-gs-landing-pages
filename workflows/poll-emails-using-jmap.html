Poll Emails Using JMAP

Automate email polling from any provider using JMAP. Retrieve unread counts & message details without Gmail/Outlook limitations.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Many automation platforms only offer email triggers for Gmail or Outlook, leaving businesses using other email providers stuck with manual processes. This workflow solves that by leveraging JMAP—a modern, open standard for email, calendar, and contact access.

It polls your JMAP-compatible email server to retrieve the total number of unread messages and fetches details for the first few emails. You can adapt it to search for specific messages, filter by sender or subject, and trigger actions based on inbox status—all without relying on vendor-specific integrations.

Whether you're monitoring support tickets, tracking invoice emails, or automating customer inquiry routing, this template gives you programmatic access to any JMAP-supported email service, breaking free from platform limitations.

How It Works

Step 1: HTTP Request to JMAP Server

The workflow starts with an HTTP Request node configured to call your JMAP server's API endpoint. It uses Bearer token authentication (or OAuth2) to securely fetch mailbox state and email records.

Step 2: Parse & Filter Email Data

The response is parsed to extract the total unread count and the first 3 email details (sender, subject, date). You can easily adjust the filter to return more emails or apply custom criteria like "only emails from specific domains."

Step 3: Transform & Route Output

Using Set nodes, the workflow structures the data into a clean format. You can then route this information to other systems—like logging unread counts to a dashboard, forwarding urgent emails to a team channel, or creating tickets in your helpdesk software.

Step 4: Schedule & Automate

The workflow is designed to run on a schedule (e.g., every 15 minutes) using n8n's scheduler. This creates a continuous monitoring loop that keeps your team informed without manual inbox checks.

What You'll Need

  1. A JMAP‑compatible email service (Fastmail, Stalwart Mail Server, or another provider that supports JMAP).
  2. API credentials (Bearer token or OAuth2) for authenticating with the JMAP server.
  3. n8n instance (cloud or self‑hosted) version 1.20 or later (older versions can be adapted).
  4. Basic understanding of HTTP requests and JSON responses.
  5. A destination for the polled data (Slack, database, CRM, etc.).

Pro tip: If your email provider doesn't yet support JMAP, contact them and ask about JMAP adoption. The standard is gaining traction because it's more efficient and developer‑friendly than IMAP.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n workspace.
  2. Replace the JMAP endpoint URL with your provider's JMAP API URL.
  3. Configure authentication credentials (Header Auth with Bearer token is typical for Fastmail).
  4. Adjust the filter parameters if you want more than 3 emails or different search criteria.
  5. Connect output nodes to your desired destinations (Slack, Google Sheets, Airtable, etc.).
  6. Set a schedule trigger to run the workflow automatically at your preferred interval.
  7. Test with a small batch of emails to verify data extraction and routing.

JMAP is a modern email protocol designed for efficiency and simplicity. Unlike IMAP, JMAP uses a single HTTP request to fetch multiple emails, supports real-time updates via push notifications, and handles calendars and contacts too. For automation, JMAP is faster, more reliable, and easier to integrate because it's built on RESTful principles—making it ideal for workflows that need to poll email data without vendor-specific nodes.

IMAP requires multiple commands and persistent connections, which are harder to manage in automated workflows. JMAP's JSON‑based API is cleaner and more predictable, reducing complexity and error rates.
