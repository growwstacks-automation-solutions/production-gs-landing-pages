What This Workflow Does

This automation solves a critical problem in business operations: duplicate data processing. When you're handling hundreds of client projects, customer records, or transactions in automated loops, a single error can cause the same item to be processed multiple times. This creates data messes, wastes resources, and requires manual cleanup.

The Redis State Tracking workflow provides a simple yet powerful solution. It acts as a digital memory system for your automations, tracking each item's status in real-time. Before processing any item, it checks if work has already started or completed. This prevents duplicates, ensures data integrity, and allows your workflows to recover gracefully from failures.

Originally built to manage weekly client project updates, this pattern has proven invaluable for any business dealing with batch processing, scheduled jobs, or distributed automation systems where reliability matters more than speed.

How It Works

The system follows a clean three-component architecture that separates concerns while maintaining simplicity.

1. Item Status Check

Before processing any item (like a project ID or customer record), the workflow queries Redis using a structured key format. This key combines the item group, unique identifier, and status suffix. If Redis returns an "in-progress" or "completed" status, the workflow skips that item entirely.

2. Status Update Protocol

When an item is ready for processing, the workflow immediately sets an "in-progress" status in Redis with a configurable timeout. This temporary lock prevents other processes or retries from picking up the same item. Once processing completes successfully, the status updates to "completed."

3. Failure Recovery

If processing fails, the "in-progress" status eventually expires based on the timeout. The system can then safely retry that specific item without affecting others. This targeted recovery eliminates the need to restart entire batches from scratch.

Pro tip: Set your Redis timeout slightly longer than your average processing time. This prevents premature expiration during legitimate long-running tasks while ensuring failed items don't stay locked indefinitely.

Quick Setup Guide

Implementing this pattern takes about 30 minutes if you have Redis ready. Follow these steps:

  1. Import the template – Download and import the JSON file into your n8n instance.
  2. Configure Redis connection – Add your Redis host, port, and credentials to the Redis nodes.
  3. Define your key structure – Modify the key templates to match your data (e.g., "projects:{id}:status").
  4. Set timeouts – Adjust Redis expiration times based on your processing duration.
  5. Integrate your logic – Replace the example processing steps with your actual business operations.
  6. Test with sample data – Run the workflow with a small dataset to verify status tracking works.
  7. Monitor and adjust – Watch for expired items and adjust timeouts as needed.

Pro tip: Start with a conservative timeout (like 1 hour) and monitor your "in-progress" items. If many expire before completion, increase the timeout. If items stay locked too long after failures, decrease it.
