What This Workflow Does

This automation solves the critical business problem of information silos in document repositories. Companies accumulate thousands of documents in Google Drive—contracts, reports, manuals, research papers—but finding specific information requires manual searching through each file. This workflow transforms passive documents into an active, queryable knowledge base using Retrieval-Augmented Generation (RAG).

The system automatically ingests new documents from Google Drive, intelligently chunks them while preserving context, generates AI embeddings, and stores them for instant retrieval. When users ask questions, it finds the most relevant document segments and uses AI to generate precise, context-aware answers. This eliminates hours of manual research and ensures consistent information delivery across your organization.

How It Works

1. Document Ingestion & Processing

A Google Drive Trigger monitors specified folders for new files. When a document is added, the workflow extracts metadata (file name, type, modification date) and downloads the content. A Switch node routes files to appropriate extractors based on file type—PDFs go through PDF parsing, while text documents use simpler extraction.

2. Content Transformation & Chunking

The extracted text passes through a Recursive Splitter that understands document structure. Unlike basic splitting that might cut sentences in half, this method preserves logical boundaries like paragraphs and sections. It creates overlapping chunks that maintain context between segments, crucial for accurate retrieval later.

3. Embedding & Storage

Each text chunk is processed through OpenAI's embedding model, converting semantic meaning into numerical vectors. These embeddings, along with metadata and the original text, are stored in a vector database (Supabase in this template). The system also generates document summaries for quick overviews.

4. Query Processing & Retrieval

When a user submits a question, the workflow converts it to embeddings and performs a similarity search against stored vectors. The most relevant chunks are retrieved and fed to Google Gemini (or OpenAI) with the original question, instructing the AI to answer based only on the provided context. This prevents hallucinations and ensures accuracy.

What You'll Need

  1. Google Drive account with documents to process (PDF, DOCX, TXT formats supported)
  2. OpenAI API key for embeddings generation (GPT-3.5-turbo or GPT-4)
  3. Google Gemini API key for response generation (or additional OpenAI credits)
  4. Supabase account for vector storage (free tier available) or alternative vector database
  5. n8n instance (self-hosted or cloud) with workflow execution permissions
  6. Basic folder structure in Google Drive for organized document ingestion

Pro tip: Start with a small, well-organized folder of documents to test the pipeline. Ensure your documents have clear headings and structure—this significantly improves the recursive chunking quality and final retrieval accuracy.

Quick Setup Guide

Follow these steps to implement this document RAG system in your environment:

  1. Download and import the JSON template into your n8n instance using the import workflow feature.
  2. Configure credentials for Google Drive, OpenAI, Google Gemini, and Supabase in n8n's credentials management.
  3. Set the Google Drive folder ID in the trigger node to monitor your specific document repository.
  4. Adjust chunking parameters in the Recursive Splitter node based on your document types—legal documents may need smaller chunks than technical manuals.
  5. Test with sample queries using the Manual Trigger node to verify retrieval accuracy before enabling automatic processing.
  6. Deploy the workflow and add sample documents to your monitored folder to begin building your knowledge base.
