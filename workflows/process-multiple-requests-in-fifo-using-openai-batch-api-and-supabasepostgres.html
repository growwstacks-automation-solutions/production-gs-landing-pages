What This Workflow Does

This automation solves the challenge of processing large volumes of AI prompts efficiently and cost-effectively. Instead of making expensive individual API calls for each prompt, this workflow implements a First-In-First-Out (FIFO) queue system that batches up to 50,000 prompts into a single OpenAI Batch API request.

The system tracks each batch job in a Supabase database table, automatically polls for completion every 5 minutes, and stores the decoded results back in your database. This creates a complete, auditable pipeline for AI processing that eliminates manual intervention while reducing API costs by up to 50% compared to real-time processing.

How It Works

1. Prompt Queue Management

New prompts are added to a Supabase table with a "pending" status. The workflow checks for unprocessed prompts and groups them into batches based on your configured size limits.

2. Batch API Submission

The workflow creates a single OpenAI Batch API request containing all queued prompts along with your system instructions. This batch is submitted asynchronously, and the batch ID is stored in Supabase for tracking.

3. Automated Polling & Status Checking

A scheduled trigger polls the OpenAI Batch API every 5 minutes to check completion status. Once a batch is marked as completed, the workflow retrieves the results file from OpenAI's servers.

4. Result Processing & Storage

The workflow decodes the batch results, matches each response to its original prompt in the database, and updates the Supabase records with the AI-generated content and metadata.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the template: Download the JSON file and import it into your n8n instance using the "Import from File" option.
  2. Configure credentials: Add your OpenAI API key and Supabase/PostgreSQL connection credentials to the credential manager.
  3. Set up database: Create the required openai_batches table in Supabase using the schema provided in the sticky note node.
  4. Customize prompts: Replace the sample prompt data with your actual prompts and system instructions.
  5. Test with small batch: Run the workflow with 5-10 test prompts to verify the complete pipeline works end-to-end.
  6. Schedule execution: Adjust the schedule trigger based on your processing needs (hourly, daily, or on-demand).

Pro tip: Start with small batches (10-50 prompts) to test the workflow before scaling to thousands. Monitor your OpenAI usage dashboard to track costs and optimize batch sizes based on your budget.
