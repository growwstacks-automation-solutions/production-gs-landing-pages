What This Workflow Does

This automation workflow solves the challenge of extracting valuable insights from YouTube videos by processing their transcripts and making them searchable. It connects Apify's YouTube transcript scraper with OpenAI's embedding technology and Pinecone's vector database to create a powerful content indexing system.

The workflow automatically fetches video URLs from Airtable, extracts transcripts with timestamps, processes the text into meaningful chunks, generates AI-powered embeddings, and stores them in Pinecone for semantic search. This enables users to find specific moments in videos based on meaning rather than just keywords.

How It Works

1. Fetch Video Records

The workflow begins by retrieving YouTube video URLs and metadata from Airtable. Each record contains essential information like video title, description, and URL.

2. Scrape Transcripts

Using Apify's YouTube Transcript Scraper, the workflow extracts transcripts with timestamps for each video. This includes both the spoken content and the exact timecodes.

3. Process Transcripts

The raw transcripts are parsed and grouped into logical chunks based on content and timing. Each chunk is enriched with metadata including video title, description, and direct links to specific moments.

4. Generate Embeddings

OpenAI's embedding model converts each transcript chunk into a vector representation that captures semantic meaning. This enables searches based on concepts rather than exact wording.

5. Index in Pinecone

The vector embeddings are stored in Pinecone along with their metadata. Pinecone's efficient indexing allows for fast semantic searches across large video libraries.
