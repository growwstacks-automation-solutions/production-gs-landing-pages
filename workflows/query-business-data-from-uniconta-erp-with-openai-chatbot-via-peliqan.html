What This Workflow Does

This automation creates an intelligent chatbot interface for your Uniconta ERP system, allowing employees to ask business questions in natural language and receive accurate answers by querying live ERP data. Instead of running complex reports or navigating multiple screens in Uniconta, users get instant answers about inventory levels, financial KPIs, order statuses, and other operational data.

The workflow connects three powerful technologies: Uniconta (your ERP system), Peliqan (the data integration platform), and OpenAI (the AI processing engine). When a question comes in, the system intelligently determines which Uniconta data tables to query, retrieves the relevant information through Peliqan's API, and uses AI to formulate a clear, conversational response.

How It Works

1. User submits a business question

The workflow begins when an employee or manager asks a question through your preferred interface (Slack, Teams, web portal, etc.). Example questions might include "What's our current inventory level for product X?" or "Which customers have overdue invoices?"

2. AI analyzes the question intent

OpenAI's natural language processing determines what data needs to be retrieved from Uniconta. The AI identifies entities (like product names or date ranges) and maps them to specific Uniconta data tables and fields through Peliqan's integration layer.

3. Peliqan queries Uniconta ERP

Peliqan securely connects to your Uniconta instance and retrieves only the relevant data needed to answer the question. This happens through Uniconta's API with proper authentication and access controls.

4. AI formulates the response

The raw data from Uniconta is processed by OpenAI to create a natural language answer that's easy to understand. The AI adds context where helpful, like comparing current numbers to historical averages when discussing sales figures.

5. Response delivered to user

The final answer is returned through the original interface where the question was asked, completing the conversational loop. The system can optionally log all queries and responses for auditing purposes.
