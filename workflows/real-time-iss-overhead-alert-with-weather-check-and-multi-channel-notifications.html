What This Workflow Does

This n8n workflow solves the frustration of missing International Space Station (ISS) sightings due to weather conditions or lack of timely notifications. It automatically tracks the ISS position relative to your location, checks local weather conditions, and only sends alerts when visibility is good.

The system combines real-time data from NASA's ISS tracking API with weather data from services like OpenWeatherMap. When both conditions are met (ISS overhead + clear skies), it triggers multi-channel notifications via email, SMS, or messaging apps so you never miss a visible pass.

How It Works

1. ISS Position Tracking

The workflow regularly checks the current position of the International Space Station using NASA's API. It calculates whether the ISS will pass within visible range of your specified location.

2. Weather Condition Check

Before sending any alerts, the system verifies current weather conditions at your location. It checks for cloud cover percentage, precipitation, and other factors that would affect visibility.

3. Notification Logic

Only when both conditions are met (ISS overhead + clear weather) does the system proceed to send notifications. This prevents unnecessary alerts when viewing conditions are poor.

4. Multi-channel Alerts

The workflow can deliver notifications through multiple channels simultaneously - email, SMS (via Twilio), Slack, Telegram, or other messaging platforms based on your configuration.

Frequently Asked Questions

Common questions about ISS tracking and notification automation

NASA's ISS tracking data is extremely accurate, with position updates every few seconds. The orbital predictions are typically accurate to within a few kilometers, which is more than sufficient for visibility alerts. The ISS orbits Earth every 90 minutes, but visibility depends on your location and daylight conditions.

Automating the tracking process ensures you get alerts for all potential visible passes, not just the most prominent ones. The system accounts for factors like the ISS's altitude (400km) and speed (28,000 km/h) to calculate precise overhead times.

Clear or mostly clear skies with minimal cloud cover provide the best ISS viewing conditions. The workflow typically checks for cloud cover below 30% and no precipitation. Nighttime passes are most visible, but some daylight passes can be seen if the ISS is sufficiently illuminated by the sun.

The automation can be customized to your preferred weather thresholds. For example, some users might accept slightly higher cloud cover if they're using binoculars, while photographers might require completely clear skies.

We recommend running checks every 5-10 minutes for optimal results. This frequency balances API usage with timely notifications. Since the ISS completes an orbit every 90 minutes, more frequent checks aren't necessary unless you're tracking multiple satellites.

The workflow is designed to be efficient with API calls. It only checks weather conditions when an ISS pass is imminent, reducing unnecessary API requests that could lead to rate limiting or additional costs.

While this template is configured for the ISS, the same principles can be adapted for other bright satellites like Hubble or the Chinese space station. You would need to modify the tracking API endpoint and potentially adjust the visibility parameters since different satellites have different brightness levels.

Many satellite tracking APIs provide similar data to NASA's ISS API. The weather check and notification logic would remain largely the same, making it relatively straightforward to expand the system's capabilities.

The workflow supports all notification methods available in n8n, including email (via SMTP), SMS (via Twilio or similar services), Slack, Telegram, Discord, and more. You can configure multiple channels simultaneously so you receive alerts wherever you prefer.

Advanced users can even add voice calls or smart home announcements through integrations with services like Amazon Alexa or Google Home. The flexibility of n8n means you're not limited to basic notifications.

NASA's ISS tracking API is completely free with registration. Weather APIs like OpenWeatherMap offer free tiers with limited calls (usually sufficient for personal use). Notification services vary - email is often free, while SMS services like Twilio have pay-as-you-go pricing (typically $0.01-$0.05 per message).

For most individual users, the total monthly cost would be under $5, and potentially free if staying within free tier limits. The workflow is optimized to minimize API calls and associated costs.
