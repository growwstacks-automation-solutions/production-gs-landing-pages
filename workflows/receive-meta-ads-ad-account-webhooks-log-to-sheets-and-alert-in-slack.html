Meta Ads Webhook Automation: Log Events & Get Slack Alerts

What This Workflow Does

Managing Meta Ads campaigns involves constant monitoring for performance changes, account issues, and optimization opportunities. Manually checking for creative fatigue, ad recommendations, or account problems is time-consuming and reactive. This automation solves that by creating a centralized system that listens for Meta's real-time webhook notifications, processes them intelligently, and ensures your team is immediately informed.

The workflow acts as a production-ready dispatcher for Meta Ads webhooks specifically for the Ad Account object. It receives webhook deliveries from Meta's platform, validates them, routes different event types to appropriate handling, logs everything to Google Sheets for historical analysis, and sends concise Slack notifications to keep your team informed without inbox overload. This creates a complete audit trail while enabling rapid response to advertising performance signals.

How It Works

The automation is organized into logical blocks that handle the complete webhook lifecycle from reception to notification.

1. Webhook Reception & Verification

A Webhook trigger node acts as the endpoint that Meta calls. It first detects verification requests (when you initially subscribe your webhook in Facebook Developers) and validates the verify token, responding with the correct challenge value to complete setup. For actual webhook deliveries, it parses the incoming JSON payload containing ad account events.

2. Event Routing & Acknowledgment

The workflow examines the webhook payload to identify the specific event type (creative_fatigue, ad_recommendations, account issues, etc.). It immediately sends back an acknowledgment response to Meta to confirm successful delivery, then routes execution to the appropriate processing branch based on the event field. This ensures Meta doesn't retry deliveries while your processing continues.

3. Data Normalization & Logging

Each event branch processes the webhook data, splitting array payloads (Meta can send multiple entries and changes in one delivery). The normalized data is then appended to a dedicated Google Sheets tab specific to that event type. One master spreadsheet contains separate sheets for creative fatigue events, ad recommendations, account issues, and other webhook types, with the raw webhook body preserved for debugging.

4. Notification & Summary

After all events are logged, the workflow aggregates counts by event type and sends a compact Slack notification summarizing what was received. This provides immediate visibility without overwhelming your team with individual alerts for every webhook delivery, striking the right balance between awareness and notification fatigue.

What You'll Need

  1. A Meta Developer Account with app creation permissions to set up webhook subscriptions for your ad accounts.
  2. Google Sheets access with edit permissions for a spreadsheet where events will be logged (create one or use an existing).
  3. Slack workspace access with permissions to create incoming webhooks or install apps in your desired notification channel.
  4. n8n instance (cloud or self-hosted) with internet-accessible webhook capabilities to receive calls from Meta's servers.
  5. Meta Ads account with sufficient permissions to configure webhooks for the ad account object.

Pro tip: Before going live, test your webhook setup using the open-source meta-ads-webhook-tester tool available on GitHub. This lets you simulate webhook deliveries without affecting actual ad accounts.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download and import the template JSON file into your n8n instance using the import workflow feature.
  2. Configure the Webhook node with a unique path and note your n8n webhook URL (you'll need this for Meta setup).
  3. Create a Meta app in Facebook Developers, add the Webhooks product, subscribe to the Ad Account object, and enter your n8n webhook URL as the callback with the verify token from the workflow.
  4. Connect Google Sheets credentials in all Sheets nodes, specifying your spreadsheet ID and ensuring tabs exist for each event type (creative_fatigue, ad_recommendations, etc.).
  5. Set up Slack integration by creating an incoming webhook in your Slack workspace and adding the URL to the Slack node, selecting your notification channel.
  6. Activate the workflow and trigger the verification request from Facebook Developers to confirm everything is connected correctly.
