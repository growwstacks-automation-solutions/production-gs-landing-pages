What This Workflow Does

This automation solves a common challenge in data engineering and IoT projects: collecting time‑stamped location data from a public API and storing it reliably for analysis. Manually polling an API and updating a database is repetitive, error‑prone, and doesn't scale.

The workflow automatically queries the Open‑Notify ISS Position API at scheduled intervals, extracts the timestamp, latitude, and longitude, and inserts each record into a TimescaleDB table. This creates a historical dataset of the International Space Station's orbit—ready for mapping, trend analysis, or triggering location‑based alerts.

Beyond space tracking, this template is a blueprint for any use case that involves periodic API calls and time‑series data storage, such as weather monitoring, stock prices, sensor readings, or social media metrics.

How It Works

The workflow uses four core nodes in a clean, linear pipeline.

1. Cron Node: Scheduled Trigger

The Cron node acts as the workflow's scheduler. You configure it to run at your desired frequency—every minute, every hour, or on a custom cron expression. This eliminates the need for manual execution and ensures your dataset grows automatically over time.

2. HTTP Request Node: Fetch Live Data

This node sends a GET request to the Open‑Notify API endpoint that returns the ISS's current position. The response is a JSON object containing timestamp, latitude, longitude, and velocity. The node handles the HTTP connection, error checking, and passes the raw data downstream.

3. Set Node: Transform & Map Fields

The Set node extracts the specific fields you need from the API response and formats them for database insertion. You define the output structure—typically timestamp, lat, lon—and can add computed fields if needed. This step ensures clean, consistent data enters your database.

4. TimescaleDB Node: Persistent Storage

The final node inserts the transformed data into a TimescaleDB table. TimescaleDB is a PostgreSQL extension optimized for time‑series data, offering superior performance for timestamp‑based queries. The node manages the database connection and executes the INSERT operation.

Quick Setup Guide

Follow these steps to import and configure the workflow in your n8n environment.

  1. Download the template using the button above and save the JSON file.
  2. In your n8n dashboard, go to Workflows → Import from File and select the downloaded JSON.
  3. Configure the Cron node to your desired schedule (e.g., every 5 minutes).
  4. Set up the TimescaleDB node with your database credentials, table name, and column mapping.
  5. Test the workflow manually once to verify the API call and database insert work correctly.
  6. Activate the workflow and let it run automatically on your schedule.

Your database will begin populating with timestamped ISS coordinates, ready for querying and analysis.
