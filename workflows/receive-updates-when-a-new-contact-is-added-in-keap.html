How It Works

The automation follows a simple but powerful trigger-action sequence. You set it up once, and it runs autonomously.

Step 1: The Keap Trigger

The workflow is initiated by the Keap Trigger node. It continuously monitors your specified Keap account for the "Contact Added" event. This is a webhook-based trigger, meaning it listens for real-time updates from Keap, not periodic polling, so there's virtually no delay.

Step 2: Data Extraction & Shaping

Once triggered, the workflow captures the complete new contact data—name, email, phone, custom fields, tags, and source. This data is parsed and formatted into a clean, structured JSON object. This step is crucial because it standardizes the information, making it ready for use in any subsequent application.

Step 3: Conditional Routing (Optional)

Before taking action, you can add logic. For example, you might check if the contact has an email address, or if they came from a specific campaign. Based on these conditions, the workflow can branch to different actions, ensuring relevant follow-ups for different lead types.

Step 4: Execute Connected Actions

This is where the magic happens. The workflow uses the structured contact data to perform one or many actions simultaneously. Common actions include sending a Slack/Teams message, adding a row to Google Sheets, creating a card in Trello, or triggering an email via SendGrid.

Step 5: Logging & Error Handling

The final part of the workflow ensures reliability. It logs each execution (success or failure) and can be configured to send an alert if something fails, so you're never left wondering if the automation is working.

Quick Setup Guide

You can have this automation running in under 10 minutes.

  1. Download the template using the button above. You'll get a .json file.
  2. Open your n8n instance and create a new workflow.
  3. Import the JSON file using the import option in the n8n editor.
  4. Configure the Keap Trigger node: Click on it, add your Keap credentials, and select the event "Contact Added".
  5. Configure your action nodes: For each node (like Slack or Google Sheets), add the necessary credentials and map the incoming contact data to the required fields.
  6. Test the workflow: Activate it and add a test contact in Keap. Check if the desired action (e.g., a Slack message) is triggered correctly.
  7. Go live: Once testing is successful, toggle the workflow to "Active". It will now run automatically for every new contact.
