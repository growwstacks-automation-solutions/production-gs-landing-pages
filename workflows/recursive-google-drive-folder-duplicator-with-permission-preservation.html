What This Workflow Does

Manually copying complex Google Drive folder structures with multiple nested subfolders, hundreds of files, and specific sharing permissions is a time-consuming and error-prone task. This recursive Google Drive folder duplicator automation solves that problem completely.

The workflow creates an exact replica of any source folder in your desired destination, maintaining the complete hierarchical structure, all file metadata, and critically, every sharing permission setting. Whether you're setting up project templates for new clients, migrating department drives, or creating standardized folder structures for team onboarding, this automation handles the tedious work while ensuring nothing gets lost in translation.

Unlike manual copying or basic Google Drive operations, this n8n workflow preserves the intricate permission settings that define who can view, comment on, or edit each file and folder—a crucial requirement for maintaining security and collaboration protocols during organizational changes.

How It Works

The automation follows a systematic approach to ensure complete and accurate duplication of your Google Drive content.

1. Initialization & Setup

The workflow begins by authenticating with Google Drive using OAuth2 credentials and identifying the source folder to duplicate. You specify the target location where the duplicated structure will be created, and the automation establishes the main destination folder with proper naming conventions.

2. Recursive Folder Scanning

The system scans through each level of the folder hierarchy, intelligently separating files from subfolders at each directory level. This recursive processing handles unlimited nesting depth, ensuring even the most complex organizational structures are mapped completely before duplication begins.

3. File Duplication with Metadata Preservation

Each file from the source is copied to the corresponding location in the destination, maintaining original filenames, creation timestamps, and other metadata. The workflow applies the exact same sharing permissions from the original files to the copies, preserving viewer, commenter, and editor access for all users, groups, and domains.

4. Folder Creation & Permission Application

Empty folders are created in the target location mirroring the source structure. Each new folder receives the identical permission settings of its corresponding source folder, maintaining the collaborative environment and access controls essential for team workflows.

5. Self-Recursion for Nested Processing

For each subfolder discovered, the workflow calls itself recursively to process that folder's contents. This elegant self-referential design allows the automation to handle unlimited nested levels without complex programming, ensuring complete duplication regardless of organizational depth.

Quick Setup Guide

Follow these steps to implement this Google Drive folder duplicator in your n8n environment:

  1. Download the template using the button above and import it into your n8n instance
  2. Configure Google Drive OAuth credentials in the first node, ensuring you have drive.file and drive.readonly scopes
  3. Set your source folder ID in the workflow trigger—find this in your Google Drive URL after "/folders/"
  4. Specify your destination folder ID where the duplicate should be created
  5. Test with a small folder first to verify the duplication works correctly
  6. Schedule or trigger the workflow as needed for your automation requirements
