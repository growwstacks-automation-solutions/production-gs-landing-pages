What This Workflow Does

Manually checking remote job boards is time-consuming and inefficient. This automation pipeline solves that by creating a fully automated system that fetches the latest remote job listings from RemoteOK's API, processes and cleans the data, stores it in an organized Airtable database, and broadcasts relevant opportunities to your Telegram community or team channel.

The workflow eliminates duplicate entries, extracts key information like salary ranges, formats job descriptions for readability, and ensures your team or community gets real-time notifications about new opportunities. It transforms hours of manual monitoring into a fully automated process that runs on schedule.

How It Works

1. Fetch Jobs from RemoteOK API

The workflow starts by making an HTTP request to the RemoteOK API endpoint. It retrieves the latest job listings in JSON format, including details like job title, company, description, salary, location, and posting date.

2. Clean & Normalize Data

A Code node processes the raw data: filtering out irrelevant metadata, removing HTML tags from descriptions, handling special characters, and standardizing date formats. This ensures clean, consistent data ready for storage.

3. Extract Salary Information

The workflow parses salary ranges from various formats (hourly, yearly, contract) and converts them to standardized ranges for easy comparison and filtering in your database.

4. Upsert to Airtable

Each job is checked against existing records in Airtable using the job ID. New jobs are added as records, while duplicates are skipped. This creates a growing, organized database of remote opportunities.

5. Format Telegram Post

The workflow formats the job information into a clean, readable message with emojis, clear sections, and clickable links. It structures the information for optimal readability in Telegram channels or groups.

6. Send to Telegram Bot

A Telegram bot node sends the formatted job post to your specified channel or group. Your community gets instant notifications about new remote opportunities without any manual intervention.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Airtable account with a base and table ready to receive job data
  3. Airtable Personal Access Token for API authentication
  4. Telegram bot token created via @BotFather
  5. Telegram channel or group ID where the bot has posting permissions
  6. Basic understanding of API endpoints and JSON data structure

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance.
  2. Configure the HTTP Request node with the RemoteOK API endpoint URL.
  3. Set up Airtable credentials in n8n using your Personal Access Token.
  4. Update the Airtable node with your base ID, table name, and field mappings.
  5. Configure the Telegram node with your bot token and channel/group ID.
  6. Test the workflow with a manual trigger to ensure all connections work.
  7. Schedule the workflow to run hourly or daily for automatic updates.

Pro tip: Add a filter node after the data cleaning step to only forward jobs matching specific criteria like minimum salary, job type, or technology stack. This prevents notification overload for your community.
