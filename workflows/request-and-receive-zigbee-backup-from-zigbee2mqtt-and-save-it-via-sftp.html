What This Workflow Does

This n8n workflow solves a critical smart home maintenance problem: automatically backing up your Zigbee device configurations from zigbee2mqtt/Home Assistant. Smart home setups can take dozens of hours to configure, with device pairings, network keys, and coordinator settings that are painful to recreate after a system failure.

The automation provides a reliable, scheduled backup system that requests configuration snapshots from zigbee2mqtt, receives the backup files via MQTT, and securely stores them on an SFTP server with timestamped filenames. This ensures you always have a recent recovery point, transforming what could be a multi-hour restoration process into a simple file restore.

How It Works

Flow 1: Scheduled Backup Trigger

A schedule trigger node runs at your configured interval (daily, weekly, etc.) and sends an MQTT message to zigbee2mqtt requesting a new backup. This replaces manual backup initiation with reliable automation.

Flow 2: Backup Reception & Storage

An MQTT trigger node listens for the backup completion message from zigbee2mqtt. When received, it extracts the ZIP backup file from the message, adds a timestamp to the filename for organization, and uploads it to your configured SFTP server for secure storage.

What You'll Need

  1. A running n8n instance (cloud or self-hosted)
  2. zigbee2mqtt installation with backup functionality enabled
  3. MQTT broker access (like Mosquitto) with credentials
  4. SFTP server for backup storage (could be NAS, cloud storage, or local server)
  5. Basic understanding of your MQTT topics and SFTP connection details

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Create an MQTT connection named "MQTT Account" with your broker details
  3. Create an SFTP connection named "SFTP Zigbee Backups" with server credentials
  4. Adjust the schedule trigger to your preferred backup frequency
  5. Test the workflow manually, then activate it for automated operation

Pro tip: Store your backups in at least two locations—consider adding a second SFTP node to copy backups to cloud storage like AWS S3 or Backblaze for geographic redundancy.
