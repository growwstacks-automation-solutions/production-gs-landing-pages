What This Workflow Does

When your n8n instance crashes, gets migrated, or experiences credential loss, manually recreating dozens of API connections can take hours and introduce errors. This workflow automates credential recovery from a Google Drive backup file, ensuring your automations come back online quickly and correctly.

The template intelligently checks for existing credentials to prevent accidental overwrites, making it safe for both disaster recovery and instance migrations. It's the companion piece to a backup workflow, completing the circle of protection for your business-critical automations.

By automating what's typically a manual, error-prone process, you reduce recovery time from hours to minutes while eliminating configuration mistakes that could break integrations with services like CRM platforms, payment gateways, and communication tools.

How It Works

The restoration process follows a logical sequence that ensures safety and completeness:

1. Inventory Existing Credentials

The workflow first inventories all credentials currently in your n8n instance using the n8n CLI. This creates a baseline for duplicate checking, ensuring nothing gets overwritten unintentionally during restoration.

2. Retrieve Backup from Google Drive

It connects to your Google Drive account, searches for the designated backup file (typically n8n_backup_credentials.json), and downloads it securely. This step assumes you have a companion backup workflow regularly saving credentials to Drive.

3. Process and Validate Backup Data

The downloaded file gets converted from JSON format into individual credential entries. Each entry is validated for required fields and structure before proceeding to restoration.

4. Restore with Duplicate Protection

Each credential from the backup is checked against your current inventory. Only new credentials (those not already present) get restored. The workflow uses n8n's API to create each credential with its original name, type, and encrypted data.

5. Completion and Logging

After processing all credentials, the workflow provides a summary of what was restored versus what was skipped due to duplicates. Built-in error handling manages API rate limits and connection issues gracefully.

What You'll Need

  1. Self-hosted n8n instance with CLI access for credential inventory
  2. Google Drive API credentials with access to your backup folder
  3. n8n API key with permissions to create credentials
  4. Existing credential backup file in Google Drive (created by a companion backup workflow)
  5. Basic n8n knowledge to configure nodes and test the restoration

Pro tip: Always test the restore process in a development environment first. Create a test backup with non-critical credentials, then run the restoration to verify everything works before relying on it for production recovery.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure Google Drive nodes with your OAuth credentials and specify your backup file name/path
  3. Set up the n8n API node with an API key that has credential management permissions
  4. Test the workflow manually with a small backup file to verify connections work
  5. Schedule or trigger the restoration as needed for migrations or disaster recovery scenarios
