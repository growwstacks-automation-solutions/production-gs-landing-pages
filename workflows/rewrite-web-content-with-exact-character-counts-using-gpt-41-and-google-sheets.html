AI Content Rewriter with Exact Character Counts

Automatically rewrite web content while preserving exact character counts for SEO. Uses GPT-4.1, Google Sheets, and HTTP requests.

What This Workflow Does

This workflow solves a critical SEO challenge: rewriting web content while preserving exact character counts on every line. When businesses need to update competitor analysis, refresh their own website copy, or create multiple content variations for A/B testing, they often break existing layouts because the new text doesn't match the original character counts.

The automation fetches content from any reference URL, converts it to clean Markdown, and uses GPT-4.1 to intelligently rewrite marketing text, headings, and CTAs while maintaining the exact same character count as the original—down to the letter. It automatically classifies content into three categories: KEEP (form labels unchanged), SKIP (URLs and footers excluded), and REWRITE (marketing content transformed).

A second AI agent then compares the original and rewritten versions line-by-line, verifying character counts match perfectly and logging everything to Google Sheets for quality control. This creates a complete audit trail while ensuring the rewritten content fits perfectly into existing page designs and CSS frameworks.

How It Works

1. Submit Rewriting Request

Users fill a simple form with Client ID, target keyword, specific instructions, and the reference URL they want rewritten. This form-based interface requires no coding knowledge, making it accessible to marketing teams and content managers.

2. Fetch and Convert Content

The workflow sends an HTTP request to fetch the webpage's HTML content, then converts it to clean Markdown format. This removes unnecessary tags while preserving structure, headings, lists, and text content that the AI needs to process effectively.

3. AI-Powered Rewriting with Character Matching

GPT-4.1 processes the content with ultra-strict rules: it counts every character (letters, spaces, numbers, symbols), maintains exact counts, and intelligently decides what to keep, skip, or rewrite. If a line is too short, the AI adds words; if too long, it uses shorter synonyms—all while preserving meaning and SEO value.

4. Quality Verification and Logging

A second AI agent compares original vs. rewritten content, creates structured JSON showing each change, verifies character and word counts match, and logs everything to Google Sheets. This creates columns for Old Text, AI Suggested Text, and both character counts for manual review.

What You'll Need

  1. An active n8n instance (self-hosted or n8n Cloud)
  2. Google Sheets with OAuth access for comparison tracking
  3. OpenAI API key with GPT-4.1 and GPT-4o-mini access
  4. Target website URLs to rewrite
  5. Basic understanding of your website's content structure

Quick Setup Guide

  1. Connect Google Sheets OAuth2 API credentials in n8n
  2. Add your OpenAI API key and configure the three GPT model nodes
  3. Copy the form URL and share it with your team for submissions
  4. Customize the AI rewriting rules in the system message if needed
  5. Test with a sample URL and verify results in Google Sheets
  6. Review the comparison log and adjust prompts for your specific use case

Pro tip: Start with non-critical pages to fine-tune the AI's classification rules before applying to important website sections. The system learns from your feedback in Google Sheets.
