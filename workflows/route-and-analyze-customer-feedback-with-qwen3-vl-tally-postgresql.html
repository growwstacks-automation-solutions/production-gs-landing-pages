Automate Customer Feedback Analysis with Qwen3-VL AI & Tally

Capture, analyze, and route customer feedback automatically using local multimodal AI, PostgreSQL storage, and real-time Discord alerts.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This workflow solves the critical business problem of manually processing high volumes of customer feedback. Instead of having support teams read every submission, it automatically captures feedback from Tally forms, analyzes sentiment and category using local Qwen3-VL AI, stores structured data in PostgreSQL, and routes appropriate alerts to Discord channels.

The system provides complete data privacy by processing everything locally—no customer data ever leaves your infrastructure. It transforms unstructured feedback into actionable insights, enabling teams to respond to urgent issues within minutes while building a searchable database of customer input for product development.

How It Works

1. Feedback Collection via Tally

The workflow triggers when a new submission arrives from your Tally form. It captures the customer's name, email, message, and any attached images. This provides a user-friendly front-end that customers already know how to use.

2. Multimodal AI Analysis with Qwen3-VL

The local AI model (hosted via LM Studio) performs three layers of analysis simultaneously: sentiment detection (positive/negative/neutral), zero-shot classification into your custom categories, and image analysis if attachments are present. All processing happens on your hardware.

3. Structured Storage in PostgreSQL

Every submission, along with AI-generated metadata, is logged in a PostgreSQL database. This creates a historical record for reporting, trend analysis, and customer relationship management. The structured format enables SQL queries and dashboard integrations.

4. Intelligent Routing to Discord

Based on the AI analysis, the workflow determines the appropriate destination: urgent bugs go to #support, feature requests to #product, positive testimonials to #marketing, and general feedback to appropriate team channels. Each alert includes key details for immediate action.

Who This Is For

This automation is ideal for customer success managers, product teams, community leads, and support departments in SaaS companies, digital products, or any business receiving regular customer feedback. It's particularly valuable for organizations handling sensitive data that cannot use cloud AI APIs, teams spending 10+ hours weekly on manual feedback review, and companies wanting to implement data-driven customer insight processes.

Pro tip: Start with 3-5 clear feedback categories (bug, feature request, praise, question, complaint) for best AI classification accuracy. You can expand categories as the system learns your specific terminology.

What You'll Need

  1. LM Studio running locally with the Qwen3-VL-4B model loaded
  2. PostgreSQL instance with a table configured for feedback data
  3. Discord bot token and channel IDs for your team notifications
  4. Tally form configured to send webhook data to n8n
  5. n8n instance (self-hosted or cloud) to run the workflow

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure credentials for PostgreSQL, Discord, and Tally in n8n
  3. Set up LM Studio on your local machine or server, ensuring CORS is enabled
  4. Update the workflow nodes with your specific Discord channel IDs
  5. Test with sample feedback submissions and verify data flows correctly
  6. Activate the workflow and connect your live Tally form

Pro tip: Run the workflow in test mode for 24-48 hours with a small group of beta users before full deployment. This helps refine category definitions and routing logic based on real feedback patterns.
