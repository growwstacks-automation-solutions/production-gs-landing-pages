What This Workflow Does

This automation solves a common frustration for podcast listeners: platform lock-in. ARD Audiothek, a major German public broadcasting service, hosts thousands of podcasts but doesn't always provide standard RSS feeds. This means you're forced to use their proprietary app to listen.

Our n8n workflow automatically scrapes podcast episode data from ARD Audiothek and generates a valid, standards-compliant RSS feed. This feed can be added to any podcast app—like Pocket Casts, Overcast, Apple Podcasts, or AntennaPod—giving you complete control over your listening experience.

The template is configured for the popular "Kalk & Welk" podcast but can be easily adapted for any ARD Audiothek podcast by simply changing the source URL. Once deployed, it creates a persistent webhook URL that serves your personalized RSS feed, updating automatically whenever new episodes are published.

Pro tip: Use this workflow as a foundation to liberate podcasts from other proprietary platforms too. The pattern of scraping episode data and converting it to RSS is applicable to many content platforms that don't offer proper syndication.

How It Works

The workflow follows a logical sequence to transform raw webpage data into a polished RSS feed.

Step 1: Fetch Podcast Overview Page

The workflow starts with an HTTP Request node that retrieves the ARD Audiothek podcast page. It extracts the raw HTML containing episode listings, metadata, and media URLs.

Step 2: Parse and Structure Episode Data

Using n8n's HTML Extract and Function nodes, the workflow parses the page content to identify individual episodes. It extracts titles, publication dates, descriptions, durations, and direct audio file URLs.

Step 3: Build RSS XML Structure

The core transformation happens here. The workflow constructs properly formatted RSS XML with all required elements: channel metadata, item entries for each episode, enclosure tags for audio files, and proper date formatting.

Step 4: Serve via Webhook

A Webhook node exposes the generated RSS feed at a persistent URL. When your podcast app polls this URL, it receives the latest XML with all available episodes, just like any standard podcast feed.

Step 5: Validation and Testing

The workflow includes validation against W3C RSS standards and provides testing endpoints to verify the feed works correctly with popular podcast apps before you commit to using it daily.

Quick Setup Guide

Get your personalized podcast feed running in under 10 minutes:

  1. Download the template using the button above and import it into your n8n instance
  2. Locate the HTTP Request node labeled "Get overview page" and update the URL to your desired ARD Audiothek podcast
  3. Activate the workflow and note the Webhook URL generated by the "Feed Webhook" node
  4. Test the feed by opening the Webhook URL in a browser—you should see valid RSS XML
  5. Add to your podcast app by pasting the Webhook URL as a new podcast subscription
  6. Verify episodes appear in your app and configure download/playback preferences as needed

Important: When adding the feed to apps like Miniflux, select "Private" to avoid exposing your n8n instance publicly. For personal use, keep the webhook secured or add basic authentication.
