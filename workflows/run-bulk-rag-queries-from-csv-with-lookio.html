What This Workflow Does

Manually answering dozens or hundreds of questions from your company documents is time-consuming and inconsistent. Whether you're responding to RFPs, handling support tickets, or generating knowledge-base content, researching each query across multiple documents slows down operations and leads to missed information.

This automation solves that by connecting a CSV of questions directly to your Lookio AI assistant. The workflow processes each row, retrieves document-backed answers using Retrieval-Augmented Generation (RAG), and delivers a complete CSV with both questions and authoritative responses. It transforms days of manual research into minutes of automated processing.

The system ensures every answer is grounded in your actual documentation—reducing AI hallucinations while maintaining the speed and scalability of automation. It's particularly valuable for compliance-heavy industries, technical support teams, and sales organizations that need accurate, document-verified responses at scale.

How It Works

1. CSV Upload & Validation

The workflow starts with a form trigger that accepts CSV file uploads. It validates that the file contains a "Query" column and extracts all rows for processing. This simple interface means even non-technical team members can submit batches of questions without touching the automation backend.

2. Document Retrieval via Lookio

Each question is sent to your configured Lookio assistant via API. Lookio handles the complex RAG pipeline—searching your uploaded documents, retrieving relevant sections, and generating answers that reference specific source material. This managed service approach eliminates the need to build and maintain your own vector databases and embedding models.

3. Response Compilation & Output

The AI-generated responses are appended to each row in a new "Response" column. The workflow then reconstructs a complete CSV file with both original questions and their document-backed answers, ready for download or further processing in other business systems.

What You'll Need

  1. An n8n instance (cloud or self-hosted) with form trigger capabilities
  2. A Lookio account with API access and at least one assistant configured
  3. Your documents uploaded to Lookio (PDFs, text files, or other supported formats)
  4. CSV files containing a "Query" column with your questions
  5. Basic familiarity with n8n workflow configuration (or willingness to follow our setup guide)

Pro tip: Start with a small test CSV (5-10 questions) to verify your Lookio assistant is retrieving from the right documents before processing large batches. This helps identify any document formatting or chunking issues early.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure the Lookio HTTP Request node with your API key and assistant ID from your Lookio account
  3. Enable the Form Trigger node and note the generated form URL
  4. Prepare a test CSV with a "Query" column containing sample questions
  5. Upload via the form and verify the enriched CSV output contains accurate responses
  6. Adjust rate limiting if processing large batches to respect API limits
