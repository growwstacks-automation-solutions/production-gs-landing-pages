Safely Update n8n with Version Checks & Telegram Notifications

What This Workflow Does

For system administrators managing self-hosted n8n instances, keeping the platform updated is crucial for security, bug fixes, and new features. However, manual updates are risky—they can interrupt running workflows, cause data loss, or break integrations if not handled carefully.

This automation solves that problem by creating a fully automated update pipeline. It regularly checks for new n8n versions, verifies that no workflows are currently executing, safely applies updates during appropriate windows, and sends Telegram notifications at every step. You get peace of mind knowing your automation platform stays current without risking business operations.

How It Works

The workflow follows a logical safety-first approach to ensure updates never disrupt your automation.

1. Scheduled Version Check

The workflow triggers daily (configurable) or on n8n startup. It first executes a command to retrieve your current n8n version, then queries the official n8n GitHub API to fetch the latest stable release version.

2. Version Comparison & Decision Logic

An "If" node compares the current version against the latest. If they match, it sends a "Already up to date" Telegram message and stops. If an update is available, it proceeds to safety checks.

3. Safety Verification Phase

Before any update, the workflow queries the n8n API to check for currently running executions. If any are found, it sends an alert and waits (or schedules for later). Only when the system is idle does it proceed.

4. Update Execution with Monitoring

The workflow executes the update command (npm update n8n -g or docker pull) and monitors the output. It captures success/failure status and any relevant logs for notification.

5. Comprehensive Notification

Telegram messages are sent at each critical juncture: update available alert, safety check passed, update started, update completed successfully (or failed with error details).

What You'll Need

  1. Self-hosted n8n instance with SSH/command-line access
  2. Telegram Bot Token (create via @BotFather) and Chat ID

  4. Basic understanding of your n8n installation method (npm, Docker, etc.)
  5. Permissions to execute update commands on your server

Quick Setup Guide

Import and configure this workflow in under 10 minutes:

  1. Download and import the JSON template into your n8n instance
  2. Configure Telegram credentials in the Telegram nodes with your bot token and chat ID
  3. Update the command nodes to match your n8n installation method (adjust npm/docker commands as needed)
  4. Set the schedule trigger to your preferred time (default: daily at 9 AM)
  5. Test in a staging environment first, then activate the workflow
