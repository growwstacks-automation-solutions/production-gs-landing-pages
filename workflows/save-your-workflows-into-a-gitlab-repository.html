What This Workflow Does

This automation solves a critical problem for businesses using n8n or similar automation platforms: the risk of losing valuable workflow logic. When you spend hours or days building complex automations, a platform failure, accidental deletion, or configuration error can wipe out your work instantly. This template automatically retrieves all workflows from your n8n instance and saves them to a GitLab repository with proper version control.

The solution intelligently compares existing files with current workflows, only committing changes when modifications occur. This prevents repository bloat while maintaining a complete history of every iteration. It's like having an automatic backup system for your automation logic that integrates seamlessly with modern DevOps practices.

How It Works

The workflow follows a systematic approach to ensure reliable backups without unnecessary overhead:

1. Configuration Setup

First, the workflow sets custom parameters including repository owner, name, branch, and path. These settings determine where and how your workflows will be stored in GitLab, allowing customization for different environments or teams.

2. Workflow Retrieval

The automation connects to your n8n instance and retrieves all existing workflows. It handles authentication and pagination to ensure no workflows are missed, regardless of how many automations you've built.

3. Intelligent Comparison

For each workflow, the system checks if a file already exists in GitLab. It compares the current workflow configuration with the stored version, ignoring transient fields like timestamps to focus on meaningful changes.

4. Conditional Committing

Based on the comparison results, the workflow either creates new files, updates existing ones, or skips unchanged workflows. This incremental approach saves time and storage while maintaining accuracy.

5. Result Reporting

Finally, the automation generates a comprehensive report showing the status of each workflow backup. Errors don't stop the entire process—they're logged for later investigation while other workflows continue backing up.

Quick Setup Guide

Follow these steps to implement workflow backups in under 30 minutes:

  1. Download and import the template into your n8n instance
  2. Configure credentials for n8n and GitLab in their respective nodes
  3. Set repository details in the "Globals" node (owner, name, branch, path)
  4. Test with a single workflow before backing up everything
  5. Schedule automatic execution using n8n's scheduler or external triggers
  6. Verify backups by checking your GitLab repository
  7. Set up notifications for backup failures or significant changes
