What This Workflow Does

This automation solves a critical business problem: manual file backups are time-consuming, error-prone, and often neglected. The workflow automatically transfers files from any FTP or SFTP server (including WordPress hosting, legacy systems, or remote file servers) to MinIO object storage on a scheduled basis.

It preserves the complete folder structure during transfer, ensuring your backup mirrors the original organization. This is essential for quick recovery, compliance audits, and maintaining data relationships. The automation runs unattended, handles failures gracefully, and provides audit trails of all backup activities.

For businesses, this means eliminating 5-10 hours of weekly manual backup work, reducing human error in critical data protection processes, and ensuring consistent backup schedules are actually followed. The MinIO destination provides cost-effective, S3-compatible storage that can scale with your data growth.

How It Works

1. Schedule Trigger

The workflow begins with a schedule trigger that runs at your specified intervals—daily, weekly, or custom timing. This ensures backups happen consistently without manual initiation.

2. FTP/SFTP Connection & File Listing

The automation connects to your source FTP or SFTP server using secure credentials. It lists all files in the specified directory, including subfolders, to understand what needs to be backed up.

3. File Download & Structure Preservation

Each file is downloaded while maintaining its relative path. The workflow creates the same folder hierarchy in memory, ensuring the backup structure matches the source exactly.

4. MinIO S3 Upload

Files are uploaded to your MinIO bucket using S3-compatible API calls. The workflow handles large files efficiently and includes error handling for network interruptions.

5. Logging & Notification

Each backup operation is logged with details: files transferred, sizes, timestamps, and any errors. Optional notifications can be sent to Slack, email, or other channels.

What You'll Need

  1. FTP or SFTP server credentials (hostname, username, password, port)
  2. Source directory path to backup (can be root or specific folder)
  3. MinIO instance running locally, in a container, or on a cloud provider
  4. MinIO access keys (access key and secret key)
  5. Bucket name created in your MinIO instance
  6. n8n instance (self-hosted or cloud) to run the workflow

Pro tip: For WordPress backups, target the /wp-content/uploads/ directory for media files and consider adding database backup integration for complete site protection.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Schedule Trigger node with your preferred backup frequency (daily at 2 AM is common).
  3. Set up the FTP/SFTP node with your server credentials and source directory path.
  4. Configure the MinIO (S3) node with your endpoint URL, access keys, and bucket name.
  5. Test with a small directory first to verify connectivity and folder structure preservation.
  6. Monitor the first full run and check logs for any permission or network issues.
  7. Set up notifications (optional) to get alerts about backup successes or failures.
