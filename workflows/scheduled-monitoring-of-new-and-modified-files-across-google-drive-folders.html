What This Workflow Does

Manually checking Google Drive folders for new or updated files is a time-consuming task that often leads to missed updates and delayed responses. This automation solves that problem by systematically monitoring your designated folders and all their subfolders on a scheduled basis.

The workflow detects every file that has been created or modified since the last check, providing you with a reliable audit trail of document activity. Unlike Google Drive's native notifications, which only work on specific files or immediate folder changes, this solution recursively scans entire directory structures, making it perfect for team collaboration spaces, client submission folders, or document management systems.

Once files are detected, the automation can trigger numerous downstream actions—from simple notifications to complex document processing workflows—ensuring your team stays informed and responsive without manual oversight.

How It Works

1. Scheduled Trigger Activation

The workflow begins on a predefined schedule (configurable from minutes to days). A timestamp is recorded to track when the monitoring cycle started, ensuring only files changed since the last execution are identified.

2. Recursive Folder Scanning

The system identifies your target root folder, then recursively discovers all subfolders within it. For each folder found, it executes a sub-workflow that lists all contained files along with their metadata including modification dates.

3. Change Detection Logic

Each file's "created time" and "modified time" are compared against the last execution timestamp. Files that are newer than the previous check are flagged as new or modified, while unchanged files are filtered out.

4. Output and Action Trigger

The workflow outputs a clean list of only the changed files, organized by folder. This output becomes the trigger point for your custom actions—whether that's sending notifications, updating databases, or initiating document processing.

Pro tip: Connect this workflow to a loop node to process each detected file individually. This allows you to apply different actions based on file type, size, or location within your folder structure.

What You'll Need

  1. n8n instance (cloud or self-hosted) with workflow execution capabilities
  2. Google Drive credentials configured in n8n with appropriate folder access permissions
  3. Target folder ID from Google Drive that you want to monitor
  4. Destination system credentials for where you want to send notifications or data (Slack, email, database, etc.)
  5. Basic understanding of your folder structure and what constitutes a "new" or "modified" file in your context

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance using the import workflow function.
  2. Configure Google Drive credentials in all four Google Drive nodes within the workflow.
  3. Set your target folder ID in the two nodes marked within the red box in the workflow interface.
  4. Adjust the schedule trigger to your preferred monitoring frequency (default is every 60 minutes).
  5. Connect your action nodes after the output node (marked in yellow) to process detected files.
  6. Test with a manual execution to verify folder access and output format before activating the schedule.

Important: The persistent timestamp variable is only set during scheduled executions, not manual runs. This ensures accurate change detection between automated cycles.
