What This Workflow Does

Manual YouTube research is slow, inconsistent, and impossible to scale. This automation solves that by creating a complete pipeline that discovers educational or relevant videos, evaluates their quality using your criteria, organizes the best candidates in a spreadsheet, and can even download them for offline archives or analysis.

The workflow connects YouTube's vast content library directly to Google Sheets, turning a chaotic search process into a structured, repeatable system. It's designed for content curators, market researchers, educators, and anyone who needs to systematically track video content without manual effort.

How It Works

The automation follows a logical, multi-stage process to ensure only high-quality, relevant videos are captured and stored.

1. Search & Discovery

The workflow starts by querying the YouTube Data API using your predefined search terms (e.g., "digital marketing tutorials 2024"). It fetches a broad set of videos based on relevance, date, or view count.

2. Quality Filtering

Raw results are filtered through multiple gates. It checks video duration to exclude shorts, ensures a minimum view count, verifies a positive like ratio, and filters by publish date. The ISO-8601 duration format is converted to seconds for precise numeric comparisons.

3. Scoring & Deduplication

Each passing video receives a custom relevance score based on engagement metrics (views, likes) and metadata. Duplicates are removed using the unique YouTube video ID. The top-scoring videos are then selected for the final output.

4. Storage in Google Sheets

Selected video metadata—title, URL, channel, views, likes, publish date, and your custom score—is written to a designated Google Sheet. The video ID acts as a primary key, allowing the workflow to update rows if information changes later.

5. Optional Asynchronous Download

For videos marked for download, the workflow submits a request to the FetchMedia.io API. It then polls the service at intervals, checking the status (pending, success, failed). Upon completion, the download URL or error message is written back to the corresponding row in the sheet.

Pro tip: Start by using the workflow without the download step to build your curated video database. The download functionality is ideal for creating offline video libraries for training or compliance purposes.

What You'll Need

  1. YouTube Data API v3 Credentials: A Google Cloud project with the YouTube Data API enabled and OAuth 2.0 credentials set up.
  2. Google Sheets API Access: OAuth credentials to read from and write to your Google Sheets.
  3. FetchMedia.io API Key (Optional): Required only if you plan to use the video download functionality. Available via their free tier.
  4. n8n Instance: A self-hosted n8n setup or a n8n.cloud account to import and run the workflow.
  5. A Target Google Sheet: A prepared spreadsheet with appropriate column headers for the video data.

Quick Setup Guide

  1. Download & Import: Click the download button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure Credentials: Set up the "YouTube" and "Google Sheets" node credentials in n8n using your OAuth client IDs and secrets.
  3. Set Environment Variable: If using downloads, create an environment variable named FETCHMEDIA_API_KEY and add your API key.
  4. Customize Search Queries: In the "YouTube" node, edit the "q" parameter to reflect your target search terms (e.g., "project management basics").
  5. Update Sheet ID: In the "Google Sheets" node, paste the ID of your target spreadsheet and specify the worksheet name.
  6. Test & Activate: Click "Execute Workflow" to test. Once confirmed, add a Schedule Trigger node to run the workflow daily or weekly.
