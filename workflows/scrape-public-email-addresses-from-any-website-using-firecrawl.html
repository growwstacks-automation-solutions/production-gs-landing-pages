What This Workflow Does

Manual email hunting is one of the most time-consuming tasks in sales and business development. Professionals spend hours visiting company websites, searching through "About Us" pages, team directories, and contact sections to find the right people to reach out to. This process is not only tedious but also inconsistent—you might miss key contacts or waste time on outdated information.

This automation solves that problem by systematically extracting publicly available email addresses from any website. It uses Firecrawl's intelligent mapping API to identify relevant pages (about, contact, team, authors), then batch-scrapes those pages to extract email addresses while handling common obfuscations like "(at)" and "(dot)". The result is a clean, deduplicated list of valid email addresses ready for your outreach campaigns.

The workflow is designed for reliability, with built-in rate limiting, retry logic for failed requests, and intelligent filtering to ensure you get quality, actionable contact information. It transforms what used to be a days-long research process into a minutes-long automated task.

How It Works

1. Website URL Input

The workflow starts with a simple form where you enter the target website URL. This could be a company homepage, a directory page, or any site where you suspect contact information might be published.

2. Intelligent Page Discovery

Firecrawl's mapping API analyzes the website structure to find pages most likely to contain email addresses. It searches for common patterns like "/about", "/contact", "/team", "/company", and "/authors" pages.

3. Batch Email Extraction

The system processes multiple pages simultaneously, using pattern recognition and AI to identify email addresses even when they're partially obfuscated or formatted differently.

4. Validation and Deduplication

Extracted emails are validated against standard email patterns, duplicates are removed, and the results are formatted into a clean dataset ready for export or integration with your CRM.

5. Error Handling and Retry Logic

If a request fails due to rate limiting or temporary server issues, the workflow automatically retries with exponential backoff, ensuring maximum success rates.

What You'll Need

  1. Firecrawl API Access: Sign up for a free account at firecrawl.dev to obtain your API key
  2. n8n Instance: Either self-hosted n8n or n8n.cloud account
  3. Target Website URLs: A list of company websites you want to extract emails from
  4. Basic Understanding of Web Scraping Ethics: Knowledge of robots.txt, rate limiting, and data usage best practices

Pro tip: Start with 5-10 test websites to verify the quality of extracted emails before scaling to larger lists. This helps you fine-tune the search parameters for your specific use case.

Quick Setup Guide

  1. Download the template JSON file using the button above
  2. Import it into your n8n instance (Settings → Workflows → Import from file)
  3. Create an HTTP Header Auth credential named "Firecrawl" with your API key
  4. Configure the search parameters in the "map_website" node if needed
  5. Test the workflow with a sample website URL
  6. Once verified, connect the output to your CRM or export the data
