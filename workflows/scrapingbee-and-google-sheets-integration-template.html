What This Workflow Does

This automation solves the tedious problem of manually collecting website URLs for market research, competitive analysis, or content auditing. Instead of spending hours copying links from sitemaps or crawling websites page by page, this workflow automatically extracts every URL from a website's sitemap and organizes them neatly in Google Sheets.

The system intelligently navigates website structures by first checking robots.txt files, then locating sitemap.xml files, and recursively processing any nested sitemaps. It handles various file formats including compressed XML files, ensuring comprehensive data extraction regardless of how the website organizes its content.

For businesses, this means you can automatically build complete website directories, monitor competitor content updates, create backlink research databases, or compile product catalogs—all without manual data entry. The automation runs on demand via webhook, making it easy to trigger fresh data collection whenever needed.

How It Works

1. Webhook Trigger & Domain Input

The workflow starts with a webhook that accepts a domain parameter (like "example.com"). This can be triggered manually via browser, scheduled, or integrated with other systems. The domain is then prepared for sitemap discovery.

2. Sitemap Discovery & Parsing

The system first attempts to locate the website's robots.txt file to find sitemap references. If unavailable, it directly checks for sitemap.xml. Using ScrapingBee's API, it bypasses anti-bot protections and renders JavaScript if needed.

3. Recursive XML Processing

When XML files are found, the workflow intelligently determines if they're regular sitemaps, compressed files, or binary data. It decompresses when necessary, extracts text content, and parses for both regular website links and additional XML references.

4. Data Organization & Output

Extracted website URLs are immediately appended to a designated Google Sheet. Any discovered XML links trigger recursive processing until all nested sitemaps are exhausted, ensuring complete website coverage.
