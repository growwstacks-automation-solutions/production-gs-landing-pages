What This Workflow Does

This automation solves a common productivity bottleneck: constantly switching between messaging apps and web browsers to look up information. Whether you're answering customer questions, researching during a team chat, or gathering quick facts, this workflow brings the power of Brave Search directly into your Telegram conversations.

By simply sending a command like /brave what are the latest AI trends, users receive search results instantly within the chat. This eliminates the need to open a separate browser tab, navigate to a search engine, type the query, and then copy-paste results back into the conversation. The workflow handles the entire process automatically, saving minutes per search that add up to hours of recovered productivity each week.

The automation is particularly valuable for remote teams, customer support agents, researchers, and anyone who needs quick access to web information while maintaining conversation flow.

How It Works

The workflow follows a logical sequence to process search requests and deliver results efficiently.

1. Telegram Command Detection

The workflow starts with a Telegram Trigger node that monitors your specified chat or group for new messages. When a message arrives, it checks if it begins with the /brave command. This filtering ensures only search requests are processed, ignoring regular conversation.

2. Query Extraction and Cleaning

Once a valid command is detected, the workflow extracts the message text and removes the /brave prefix, leaving only the search query. Additional cleaning removes extra spaces and special characters that might interfere with the search API.

3. Brave Search API Call

The cleaned query is sent to the Brave Search API via an HTTP Request node. The workflow includes proper authentication using your Brave API key and formats the request according to Brave's documentation. The API returns structured search results including titles, URLs, and snippets.

4. Result Formatting and Delivery

The raw API response is parsed and formatted into a readable message. The workflow typically selects the top 3-5 most relevant results, formats them with clear headings and brief descriptions, and sends this formatted response back to the Telegram chat where the request originated.

Pro tip: Configure the workflow to include "Instant Answers" from Brave when available. For factual queries like weather, definitions, or calculations, Brave can return direct answers that make responses even more useful.

What You'll Need

  1. A Telegram Bot Token: Create a bot via @BotFather on Telegram and save the authentication token.
  2. Brave Search API Key: Sign up for a free account at brave.com/search/api to get your API key.
  3. n8n Instance: Either self-hosted n8n or n8n.cloud account with the HTTP Request and Telegram nodes available.
  4. Basic n8n Knowledge: Understanding how to import workflows and configure credentials.
  5. Telegram Chat ID: The ID of the chat or group where you want the bot to operate (can be obtained from the workflow during setup).

Quick Setup Guide

Follow these steps to get the automation running in under 15 minutes:

  1. Download and Import: Download the template file and import it into your n8n instance via the "Import from File" option.
  2. Configure Telegram Credentials: In the Telegram Trigger node, add your bot token credentials. Test the connection by sending a message to your bot.
  3. Set Up Brave API: Add your Brave Search API key to the HTTP Request node's authentication settings. You may need to adjust parameters like region or safe search based on your needs.
  4. Adjust Result Formatting: Customize how many results to return and how they're formatted in the "Edit Fields" or "Code" nodes if desired.
  5. Activate and Test: Turn on the workflow and test by sending /brave test query to your Telegram bot. Verify results appear correctly formatted.

Pro tip: Start with a private chat with the bot for testing before adding it to groups. This lets you debug without disrupting team conversations.
