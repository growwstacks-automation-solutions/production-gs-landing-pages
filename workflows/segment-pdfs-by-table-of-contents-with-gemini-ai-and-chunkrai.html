Automatically Segment PDFs by Table of Contents with AI

Transform unstructured PDF documents into organized, searchable sections using AI-powered table of contents generation and intelligent content segmentation.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the critical problem of working with unstructured PDF documents. Instead of treating a 100-page PDF as a single, unwieldy block of text, this workflow intelligently identifies or generates a hierarchical table of contents and then segments the entire document based on these headings. It transforms monolithic documents into organized, manageable sections that can be processed individually.

The system combines OCR technology with AI analysis to handle both scanned and digital PDFs, even when they lack explicit table of contents. This creates a structured data format where each section includes its heading, hierarchical level, and content in multiple formats (text, HTML, and markdown). The result is document intelligence that enables precise content retrieval and analysis.

How It Works

1. Document Ingestion & Advanced Parsing

The workflow accepts a PDF URL and uses Chunkr.ai to perform Optical Character Recognition (OCR) and parse the document into detailed structural elements. This extracts text, HTML, and Markdown for each segment while preserving the document's visual layout and formatting.

2. AI-Powered Table of Contents Generation

Google Gemini AI analyzes the initial pages where tables of contents typically reside, along with section headers extracted during parsing. The AI constructs an accurate, hierarchical table of contents in structured JSON format, even for documents without explicit ToC or with poor formatting.

3. Precise Content Segmentation

Custom code meticulously maps the AI-generated headings to their corresponding content within the parsed document. The system intelligently determines the precise start and end of each section, handling edge cases like nested subsections and content that spans multiple pages.

4. Structured & Flexible Output

The workflow outputs each identified section as an individual item with heading text, hierarchical level, and content in three formats. Alternatively, it can reconstruct the entire document into a single, navigable HTML or Markdown file with proper section tagging.

What You'll Need

  1. Input PDF: A URL pointing to the PDF document when triggered by another workflow, or a pre-configured sample for manual testing
  2. Chunkr.ai API Key: Required for the initial parsing and OCR of PDF documents (insert into HTTP Request nodes)
  3. Google Gemini API Credentials: Necessary for the AI model to generate the table of contents (configure in Google Gemini nodes)
  4. n8n Instance: Self-hosted or cloud version of n8n to run the workflow
  5. Basic Automation Understanding: Familiarity with connecting APIs and configuring workflow triggers

Pro tip: Start with a simple PDF that has a clear table of contents to verify the setup works correctly before processing complex documents with inconsistent formatting.

Quick Setup Guide

  1. Download and Import: Download the template file and import it into your n8n instance via the workflow import function
  2. Configure API Credentials: Add your Chunkr.ai API key to the HTTP Request nodes and your Google Gemini credentials to the AI model nodes

  4. Test with Sample: Execute the workflow manually with the provided sample PDF to verify all connections work
  5. Connect Triggers: Set up webhook, schedule, or file watch triggers based on your use case requirements
