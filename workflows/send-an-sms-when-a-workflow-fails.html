Send SMS Alerts When Your Workflows Fail

Get instant phone notifications when critical automations break—ensure zero downtime for your business processes.

What This Workflow Does

When your business automations fail silently, you might not discover broken processes until customers complain or revenue drops. This n8n template solves that by sending immediate SMS alerts to your phone whenever a workflow encounters a critical error.

The automation connects n8n's Error Trigger node with Twilio's SMS capabilities, creating a real-time monitoring system for your most important business processes. Whether it's a payment processing failure, CRM sync issue, or data pipeline breakage, you'll know within seconds—not hours or days.

This is particularly valuable for small teams without dedicated DevOps staff, e-commerce businesses with time-sensitive operations, or any company where automation downtime directly impacts customer experience or revenue.

How It Works

1. Error Detection

The workflow uses n8n's Error Trigger node to monitor your other workflows. When any configured workflow fails, the Error Trigger captures the error details including timestamp, error message, and workflow name.

2. Alert Processing

The captured error information is processed to determine severity and format an actionable alert message. The system filters out minor warnings if configured, focusing only on critical failures that require immediate attention.

3. SMS Delivery

Using Twilio's SMS node, the formatted alert is sent directly to your mobile phone. The message includes essential details: which workflow failed, when it happened, and a brief description of the error.

4. Optional Escalation

For teams, the template can be extended to send alerts to multiple phone numbers, create escalation paths if the first responder doesn't acknowledge, or integrate with Slack/Teams for broader team notification.

What You'll Need

  1. A running n8n instance (cloud or self-hosted)
  2. Twilio account with SMS capabilities enabled
  3. Twilio phone number purchased and configured
  4. Your mobile phone number for receiving alerts
  5. Workflows you want to monitor (payment processing, data sync, etc.)

Quick Setup Guide

1. Download the template JSON file using the button above.

2. In your n8n instance, go to Workflows → Import from file and select the downloaded JSON.

3. Configure the Twilio node with your Account SID, Auth Token, and From number (your Twilio phone number).

4. Set the To field to your mobile number where alerts should be sent.

5. Connect the Error Trigger node to the workflows you want to monitor by referencing their IDs or names.

6. Test by intentionally causing a failure in a monitored workflow—you should receive an SMS within seconds.

7. Adjust alert messaging and severity filters based on your needs.

Pro tip: Start by monitoring only your most critical 2-3 workflows to avoid alert fatigue. Once tuned properly, expand to additional automations.
