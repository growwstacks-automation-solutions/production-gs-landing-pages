Automate Malaysian Weather Alerts with AI & Telegram

What This Workflow Does

This automation solves a critical problem for businesses and communities in Malaysia: staying ahead of severe weather. Manual monitoring of multiple government websites and news sources is time-consuming and error-prone. This n8n template automatically fetches official weather warnings from data.gov.my, uses Perplexity AI to search for recent related news articles from Malaysian outlets, scrapes the content with Firecrawl, summarizes everything with OpenAI, and delivers a concise, actionable report directly to a Telegram channel.

It transforms hours of daily research into a fully automated, reliable system. You get early warnings for floods, storms, and other hazardous conditions, complete with context from local news, ensuring your team or community can take proactive measures. The workflow intelligently filters out "No Advisory" alerts to avoid spam and focuses on delivering only relevant, urgent information.

Beyond personal safety, this automation is a business continuity tool. Logistics companies can reroute deliveries, event planners can make timely cancellations, and agricultural operations can protect assets—all based on automated, data-driven insights.

How It Works

The workflow is built for reliability and efficiency, handling data from multiple sources and delivering a polished final report.

1. Scheduled Data Fetching

A Schedule Trigger initiates the workflow daily at 9 AM (configurable). It first calls the official Malaysian government weather API (data.gov.my) to retrieve all active warnings. A JavaScript node then processes this raw data, extracting key details: warning type (e.g., "continuous rain," "strong wind"), severity level, and the specific affected states or regions.

2. AI-Powered News Discovery

The extracted location and warning data is used to construct intelligent search queries. The Perplexity AI Agent (via OpenRouter) then scours the web for recent news (within the last 3 days) from major Malaysian sources like Utusan, Harian Metro, Berita Harian, and Kosmo. This step finds on-the-ground reports that the official API might not include.

3. Content Scraping & Summarization

Each discovered news article URL is passed through a "Split in Batches" node to manage API rate limits, then sent to Firecrawl. Firecrawl scrapes the article and extracts the core content. All article summaries and source links are aggregated into a single dataset.

4. Intelligent Report Generation

The combined data—official warnings and news summaries—is sent to OpenAI (GPT-4.1 mini). A custom prompt instructs the AI to generate a clear, English-language report suitable for a Telegram message, highlighting the most critical information and actionable advice.

5. Final Notification Delivery

The polished report is sent via the Telegram node to your specified channel or group. The workflow includes error handling to catch API failures and ensure you are notified of any issues with the automation itself.

What You'll Need

  1. n8n instance (Cloud or self-hosted).
  2. OpenAI API key for the final summarization step.
  3. OpenRouter API key with access to Perplexity Sonar Pro for news searching.
  4. Firecrawl API key (free tier available) for web scraping.
  5. Telegram Bot Token and Chat ID for the destination channel or group.

Pro tip: When setting up the Perplexity AI Agent, specify a focus on Malaysian news domains for more relevant results. The free tier of Firecrawl has hourly limits, so the built-in 3-second delay between requests helps stay within them.

Quick Setup Guide

Import and configure this template in under 10 minutes.

  1. Download & Import: Click the "Download Template" button above and import the JSON file into your n8n instance.
  2. Configure Credentials: In the n8n workflow, set up the credential nodes for OpenAI, OpenRouter (Perplexity), Firecrawl, and Telegram with your respective API keys and tokens.
  3. Set Your Chat ID: Replace the placeholder Telegram Chat ID in the "Send Message" node with your actual channel or group ID.
  4. Adjust Schedule (Optional): The Schedule Trigger is set for 9 AM daily. Modify this in the trigger node if you prefer a different time or frequency.
  5. Test & Activate: Click "Execute Workflow" once to test. If successful, activate the workflow to run automatically on your chosen schedule.
