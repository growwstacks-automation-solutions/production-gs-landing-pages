What This Workflow Does

Manually checking the weather every morning is a repetitive task that consumes valuable time. For businesses that rely on weather conditions—like logistics companies, event planners, or agricultural operations—missing a forecast can lead to costly disruptions.

This automation solves that problem by delivering a personalized weather report directly to your email inbox at a scheduled time each day. It connects to the OpenWeatherMap API to fetch accurate, real-time data for your specified city, formats it into a clear, readable summary, and sends it via Gmail. You get consistent, actionable information without lifting a finger.

The workflow is designed for reliability and simplicity, turning a manual checking habit into a seamless, automated process that keeps you informed and prepared.

How It Works

The automation follows a logical four-step process, moving data from an external API to your personal inbox.

1. Scheduled Trigger

A scheduler node (like Cron) activates the workflow at your chosen time—for example, every weekday at 7:00 AM. This sets the entire process in motion automatically, ensuring you receive your report consistently.

2. Fetch Live Weather Data

The workflow sends a secure request to the OpenWeatherMap API. It passes your configured city name and API key, retrieving a comprehensive data packet containing temperature, humidity, wind speed, conditions description, and more.

3. Format the Report

A processing node takes the raw API data and transforms it into a human-friendly email body. It extracts key metrics, structures them into clear sentences or bullet points, and prepares a subject line that includes the location and date.

4. Send the Email

Finally, the workflow uses your authenticated Gmail connection to dispatch the formatted report. It lands in your inbox (or any specified recipient list) as a regular email, ready to read at the start of your day.

Quick Setup Guide

Get this automation running in your environment in just a few minutes.

  1. Download the template using the button above and import it into your n8n workspace.
  2. Sign up at OpenWeatherMap, navigate to your API keys section, and copy your free key.
  3. In the workflow, locate the HTTP Request node and paste your API key into the designated field. Set your desired city name.
  4. Create a Gmail credential in n8n by clicking "Add Credential" and following the OAuth2 authentication flow with Google.
  5. Configure the schedule in the Cron node to your preferred time (e.g., "0 7 * * 1-5" for 7 AM on weekdays).
  6. Activate the workflow and test it by triggering it manually once. Check your inbox for the first report.
