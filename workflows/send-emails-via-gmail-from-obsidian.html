Send Emails from Obsidian Notes via Gmail

Automatically send emails directly from your Obsidian notes. Connect your knowledge base to your email client with this free n8n workflow template.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves a common productivity bottleneck: manually copying content from your notes into emails. If you use Obsidian for research, meeting notes, or project documentation, you often need to share that information via email. Switching apps, copying text, formatting, and attaching files wastes valuable time and introduces errors.

The workflow bridges Obsidian and Gmail seamlessly. When you create or update a note with specific metadata, it automatically triggers an email sent via Gmail. The note's content becomes the email body, attachments are processed, and recipients are defined within the note itself. This turns your knowledge base into an actionable communication tool.

Beyond simple sending, the template includes validation and error handling. It ensures emails are formatted correctly, attachments are encoded properly, and delivery status is confirmed. This reliability means you can trust the automation for important communications, saving 10–15 minutes per email and reducing the risk of missed or incorrect information sharing.

How It Works

1. Trigger from Obsidian

The workflow starts when you save a note in Obsidian configured with the Post Webhook plugin. The plugin sends the note's content and metadata (written in YAML frontmatter) to a webhook URL hosted by your n8n instance. This acts as the automation trigger, capturing the note data instantly.

2. Parse Note Metadata

The incoming webhook data is parsed to extract email parameters. The YAML frontmatter defines recipients (to, cc, bcc), subject line, and any tags for routing. The note body becomes the email content. This structured approach lets you control the email entirely from within Obsidian, using your familiar editing interface.

3. Process Attachments

If your note references or includes files, the workflow processes them. It encodes attachments into a format Gmail can accept, ensuring images, documents, or other files are included in the email automatically. This eliminates the manual step of downloading and attaching files separately.

4. Send via Gmail

Using your authenticated Gmail account, the workflow constructs and sends the email. It applies the subject, body, recipients, and attachments from the parsed note data. The Gmail node handles the actual sending, leveraging Google's reliable email infrastructure.

5. Confirm & Log

After sending, the workflow can confirm success back to Obsidian or log the activity in another system (like a database or spreadsheet). This provides visibility and ensures you know the email was dispatched, completing the loop from note creation to confirmed delivery.

  2. The Obsidian Post Webhook plugin installed in your Obsidian vault.
  3. A Gmail account with API access enabled (OAuth credentials).
  4. Basic understanding of YAML frontmatter in Obsidian notes to define email parameters.
  5. A webhook URL generated by n8n to receive data from Obsidian.

Quick Setup Guide

Follow these steps to implement this automation in your own environment:

  1. Import the template: Download the JSON file and import it into your n8n workspace.
  2. Configure the Webhook node: Set up the Webhook node in n8n to receive POST requests. Note the generated URL.
  3. Install Obsidian plugin: Install the "Post Webhook" plugin in Obsidian and configure it to send note data to your n8n webhook URL.
  4. Set up Gmail credentials: Authenticate the Gmail node in n8n using OAuth. Ensure you have sending permissions.
  5. Define note structure: Create a note template in Obsidian with YAML frontmatter for recipients, subject, etc.
  6. Test the flow: Create a test note in Obsidian and verify the email is sent correctly via Gmail.
  7. Activate: Enable the workflow in n8n and start automating your note-to-email process.

Pro tip: Use tags in your note's frontmatter to route emails to different recipient groups automatically. For example, tag: "client" sends to client list, tag: "team" sends internally.
