What This Workflow Does

This automation solves a common business problem: sending professional emails that don't look like they're coming from a personal Gmail account. The native Gmail integration in automation platforms often lacks support for custom sender aliases—those professional email addresses like support@yourcompany.com or billing@yourbrand.com that you've set up through Gmail.

This workflow bridges that gap by using Gmail's API directly, allowing you to send emails with custom aliases while attaching files from URLs. It transforms your Gmail account into a professional email sending service that can be triggered by other business events—new customer signups, completed purchases, support ticket updates, or scheduled reports.

Whether you're automating invoice delivery, sending personalized marketing campaigns, or distributing internal reports, this template ensures your emails maintain brand consistency while eliminating manual sending tasks that consume valuable staff time.

How It Works

1. Trigger & Input Collection

The workflow begins with a webhook trigger that accepts a structured JSON payload containing all email details: sender name, from email (your custom alias), recipient address, subject line, HTML body content, and optional attachment URLs.

2. Email Message Construction

A Code node formats the email according to Gmail API specifications, constructing the proper MIME message with headers, HTML content, and encoded attachments. This is where the custom alias gets properly configured as the sender identity.

3. Authentication & API Call

The workflow uses your Gmail OAuth credentials to authenticate with Google's API, then sends the properly formatted message through an HTTP Request node to Gmail's messages.send endpoint. This bypasses the limitations of standard email nodes.

4. Attachment Processing

If attachment URLs are provided, the workflow fetches each file, encodes it in base64, and attaches it to the email before sending. This happens automatically without manual file handling.

5. Response Handling

The workflow captures the API response, checks for success or errors, and can route notifications to other systems (like Slack or your database) to confirm delivery or alert about failures.

What You'll Need

  1. A Gmail or Google Workspace account with sending privileges
  2. Custom email aliases already configured in your Gmail settings
  3. n8n instance (cloud or self-hosted) with Gmail OAuth credentials set up
  4. Basic understanding of JSON payload structure for API calls
  5. Attachment files hosted at publicly accessible URLs (if using file attachments)

Pro tip: Before using this workflow, ensure your custom alias is properly verified in Gmail settings. Unverified aliases may be rejected or marked as spam by recipient servers.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n instance
  2. Configure your Gmail OAuth credentials in n8n's credential management
  3. Add the custom alias you want to use as a sender in your Gmail account settings
  4. Test the webhook with a simple payload containing required fields (fromEmail, toEmail, subject, htmlBody)
  5. Connect the workflow trigger to your business events—form submissions, database updates, or scheduled triggers
  6. Monitor initial sends to ensure proper deliverability and adjust spam settings if needed
