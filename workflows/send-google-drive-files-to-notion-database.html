What This Workflow Does

This automation solves the common problem of disconnected file management between Google Drive and Notion. When teams store files in Google Drive but organize projects in Notion, they often waste time manually copying file links, updating databases, and ensuring everyone has access to the latest versions.

The workflow automatically detects when a new file is added to a specific Google Drive folder and creates a corresponding entry in your Notion database. Each entry includes the file name, direct link, creation date, and any custom properties you configure. This creates a living document library that stays synchronized without manual intervention.

Beyond simple tracking, this automation enables better collaboration, version control, and document discovery. Team members can quickly find files through Notion's powerful search and filtering capabilities, while maintaining the security and storage benefits of Google Drive.

How It Works

The workflow follows a clean, reliable process to ensure accurate file synchronization between platforms.

1. Monitor Google Drive Folder

The automation continuously watches a designated Google Drive folder for new file uploads. It can be configured to trigger on specific file types, from certain users, or with particular naming patterns.

2. Capture File Metadata

When a new file is detected, the workflow extracts key information including file name, size, type, owner, creation date, and the shareable link. This metadata forms the basis of your Notion database entry.

3. Prepare Notion Database Entry

The workflow structures the file information according to your Notion database schema. You can map Google Drive properties to specific Notion field types, add custom tags, or include additional context about the file's purpose.

4. Create Database Page in Notion

The automation creates a new page in your specified Notion database with all the file information. The page includes the direct Google Drive link, making the file accessible with one click from within Notion.

5. Optional: Notify Team Members

You can extend the workflow to send notifications via Slack, email, or other channels when important files are added, ensuring team awareness of new resources.

What You'll Need

  1. A Google account with access to Google Drive
  2. A Notion account with create/edit permissions for databases
  3. A Notion database prepared with columns for file information (Name, Link, Type, Date Added)
  4. n8n instance (cloud or self-hosted) to run the workflow
  5. Google Drive API credentials configured in n8n
  6. Notion integration token from your Notion workspace settings

Quick Setup Guide

Follow these steps to implement this automation in your workflow:

  1. Import the template: Download the JSON file and import it into your n8n instance through the workflow import function.
  2. Configure Google Drive connection: Add your Google Drive credentials to n8n and specify which folder to monitor for new files.
  3. Set up Notion integration: Create an internal integration in Notion, share your database with it, and add the integration token to n8n.
  4. Map database properties: Match the Google Drive file properties to the corresponding fields in your Notion database schema.
  5. Test the workflow: Upload a test file to your monitored Google Drive folder and verify it appears correctly in your Notion database.
  6. Activate the workflow: Turn on the workflow in n8n to begin automatic synchronization.
