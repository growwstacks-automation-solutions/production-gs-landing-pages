What This Workflow Does

This automation solution solves the challenge of reliably streaming real-time data from external APIs to cloud message queues. Instead of building custom integration code that requires constant maintenance, this n8n workflow automatically fetches International Space Station position data every minute and delivers it to AWS Simple Queue Service (SQS).

The workflow creates a fully automated data pipeline that transforms raw API responses into structured messages ready for downstream processing. This eliminates manual data collection, ensures data consistency, and provides a scalable foundation for real-time analytics, monitoring systems, or event-driven applications.

How It Works

The workflow follows a clean, four-step process that demonstrates best practices for API-to-queue integrations.

1. Scheduled Trigger

A Cron node initiates the workflow every minute, providing reliable scheduling without external dependencies. This time-based trigger ensures consistent data collection intervals, which is crucial for time-series analysis and real-time monitoring applications.

2. API Data Fetching

The HTTP Request node calls the Where the ISS At API (api.wheretheiss.at) to retrieve current satellite position data. This public API provides latitude, longitude, altitude, velocity, and visibility status—perfect for demonstrating real-time data streaming patterns.

3. Data Transformation

A Set node structures the raw API response into a clean, standardized format. This step filters unnecessary fields, renames properties for clarity, and adds metadata like timestamp and data source identifiers. Proper data transformation ensures downstream systems receive consistent, well-formed messages.

4. Queue Delivery

The AWS SQS node delivers the transformed data to your specified message queue. This provides guaranteed delivery, message persistence, and enables multiple consumer applications to process the data independently. The queue acts as a buffer during high-load periods or consumer downtime.

Pro tip: Use this workflow as a template for any API-to-queue integration. Simply replace the ISS API endpoint with your data source and configure the SQS node for your AWS environment.

What You'll Need

  1. A running n8n instance (cloud or self-hosted) with network access to public APIs
  2. AWS account with SQS permissions and a configured queue
  3. AWS credentials (Access Key ID and Secret Access Key) with SQS send message permissions
  4. Basic understanding of message queues and API data formats
  5. Optional: Downstream service ready to consume messages from your SQS queue

Quick Setup Guide

Get this workflow running in your environment in under 15 minutes with these simple steps.

  1. Download and import the template JSON file into your n8n instance
  2. Configure AWS credentials in the SQS node using your IAM access keys
  3. Update the queue URL in the SQS node to point to your AWS SQS queue
  4. Test the workflow manually to verify API connectivity and queue delivery
  5. Activate the workflow to begin automatic minute-by-minute execution
  6. Monitor message delivery in AWS CloudWatch or your queue monitoring tool

Pro tip: Before going live, test with a dedicated test queue. Once verified, switch to your production queue and consider adding error handling nodes for production resilience.
