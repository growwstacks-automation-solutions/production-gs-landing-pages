What This Workflow Does

This automation solves the challenge of extracting meaningful insights from Matomo website analytics data. Instead of manually reviewing visitor statistics, this workflow automatically processes your data through AI analysis and stores the results in Baserow for easy reference and reporting.

The system gathers Matomo analytics for the past 7 days, identifies visitors with multiple sessions, and uses AI to generate SEO recommendations and content opportunities based on visitor behavior patterns. This saves hours of manual analysis while providing actionable insights to improve your website's performance.

Matomo analytics data being prepared for AI processing

How It Works

Step 1: Gather Matomo Analytics

The workflow connects to your Matomo instance and retrieves visitor data for the specified time period, focusing on users with multiple visits.

Step 2: Process Data

Raw analytics data is cleaned and formatted for AI analysis, removing any personally identifiable information while preserving behavioral patterns.

Step 3: AI Analysis

The formatted data is sent to Openrouter where an AI model identifies trends, content opportunities, and visitor segments worth targeting.

Step 4: Store Results

AI-generated insights are automatically saved to your Baserow database with timestamps for trend analysis over time.

What You'll Need

  1. Matomo analytics account with API access
  2. Openrouter API key (free tier available)
  3. Baserow account with a database table prepared
  4. n8n instance (self-hosted or cloud)

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure the Matomo node with your site ID and credentials
  3. Add your Openrouter API key to the AI processing step
  4. Connect the Baserow node to your database
  5. Test with a small date range before running the full workflow

Pro tip: Schedule this workflow to run weekly right after your team meeting to always have fresh insights for discussion.
