Send Multi-City Weather Forecasts with AI-Enhanced Formatting

Automate weather intelligence delivery for field teams. Collect city names, fetch 5-day forecasts, generate AI-formatted emails with safety alerts, and send via Gmail.

What This Workflow Does

Manual weather checking for multiple locations is time-consuming and error-prone. Field teams, event planners, and logistics managers need timely, accurate weather intelligence to make safe operational decisions. This automation solves that problem by transforming raw weather data into actionable, professionally formatted briefings.

The workflow collects up to three city names via a simple form, fetches detailed 5-day forecasts from OpenWeatherMap's professional API, processes the data to calculate key metrics like temperature ranges and precipitation probability, then uses GPT-4 to generate human-readable HTML emails with condition-based color coding and safety warnings.

Instead of teams spending hours checking different weather sources, they receive a consolidated, AI-enhanced briefing that highlights risks, suggests precautions, and presents information in an easily digestible format—all automatically delivered to their inbox.

How It Works

Step 1: City Collection & Geocoding

A form trigger captures city names from users. The workflow validates input and uses OpenWeatherMap's geocoding API to convert city names into precise coordinates, ensuring accurate forecast retrieval for the exact locations.

Step 2: Forecast Data Retrieval

The system fetches comprehensive 5-day weather forecasts for each location, including temperature highs/lows, precipitation levels, wind speeds, humidity, and condition descriptions. Data is structured for easy processing in subsequent steps.

Step 3: Data Processing & Analysis

JavaScript nodes calculate daily summaries, identify weather patterns, and flag potential risks like extreme heat, heavy rain, or high winds. This processed data forms the foundation for the AI-generated report.

Step 4: AI-Enhanced Report Generation

GPT-4 transforms the processed weather data into professionally formatted HTML emails. The AI adds contextual safety warnings, color-codes conditions for quick scanning, and creates executive summaries that highlight the most critical information for decision-makers.

Step 5: Validation & Delivery

A validation node ensures proper JSON formatting before Gmail sends the final briefing. The system handles authentication, email composition, and delivery, providing reliable communication to designated recipients.

What You'll Need

  1. OpenWeatherMap API credentials: Sign up for a free or paid account to access their weather data API.
  2. OpenAI API key: Required for GPT-4 to generate formatted reports with safety insights.
  3. Gmail OAuth2 authentication: Set up in n8n to send emails from your Gmail account.
  4. n8n instance: Self-hosted or cloud version of n8n to run the automation.
  5. Basic understanding of API keys: Knowledge of how to obtain and configure API credentials in n8n.

Quick Setup Guide

  1. Download the template file using the button above.
  2. Import the JSON file into your n8n instance via the workflow import function.
  3. Configure the OpenWeatherMap node with your API key from their developer portal.
  4. Set up the OpenAI node with your API key and select GPT-4 as the model.
  5. Authenticate the Gmail node using OAuth2 with your Google account.
  6. Test the workflow with sample city names to ensure all connections work properly.
  7. Activate the workflow and share the form link with your team members.

Pro tip: Customize the AI prompt in the OpenAI node to match your industry's specific terminology and risk thresholds. For construction, emphasize safety warnings for high winds. For agriculture, highlight precipitation patterns.
