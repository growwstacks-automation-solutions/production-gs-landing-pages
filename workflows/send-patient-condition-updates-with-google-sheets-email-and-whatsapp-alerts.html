Automate Patient Condition Updates with Google Sheets, Email & WhatsApp

Free n8n workflow to send daily patient reports, alert doctors instantly, and notify staff for critical cases—all automated.

What This Workflow Does

Manual patient updates are a major bottleneck in healthcare. Nurses and administrative staff spend hours compiling reports, calling doctors, and tracking down contact information. This process is slow, prone to errors, and delays critical care.

This automated n8n workflow solves that by creating a seamless communication bridge. It reads patient data from a Google Sheet every morning, processes the information, and instantly sends formatted reports via email and WhatsApp to the assigned doctors. For patients marked as "critical," it simultaneously alerts hospital staff via a separate email channel. Every action is logged back into the sheet for full auditability.

Imagine a world where the morning handover is already in every doctor's inbox and phone by 8:05 AM, and the charge nurse is instantly aware of any deteriorating patient—without anyone lifting a finger. That's the efficiency this template delivers.

How It Works

The workflow is built on a logical sequence that ensures reliability and clarity. Here’s the step-by-step process:

1. Scheduled Trigger

A Cron node acts as the alarm clock, triggering the entire workflow daily at 8:00 AM. You can easily adjust this schedule to hourly, twice-daily, or any other interval that fits your ward rounds.

2. Data Fetching

The Google Sheets node connects to your "Patient Roster" sheet and pulls the latest data for all active patients. This includes vital signs, current condition, medication, and the contact details of the responsible doctor.

3. Data Processing & Filtering

A Filter node first ensures only patients with an "Active" status are processed. Then, a Code node takes this raw data and transforms it into a clear, readable report. It also evaluates conditions (e.g., high fever, unstable BP) to flag patients as "critical."

4. Multi-Channel Communication

The workflow branches. For every active patient, a personalised report is sent via the Gmail/Email node to the doctor's email. In parallel, the same report is sent via an HTTP Request node to the WhatsApp Business API, delivering an instant message to the doctor's phone.

5. Critical Alerting

A second Filter node identifies patients flagged as "critical." For these cases, a separate, urgent email is sent to a designated staff distribution list (e.g., charge nurse, department head) to ensure immediate awareness and action.

6. Activity Logging

Finally, a Google Sheets node writes a record of all sent reports—including patient ID, timestamp, and status—to a "Reports_Log" sheet. This creates a perfect audit trail for compliance and review.

Pro tip: Use the Code node to customise report templates. You can add hospital branding, specific clinical guidelines, or links to the full patient chart in your EHR system for deeper context.

What You'll Need

To run this workflow, you'll need to set up a few accounts and access keys. Gather these prerequisites before you start:

  1. A Google Cloud Service Account with access to Google Sheets and Drive. This allows n8n to read and write to your sheets securely.
  2. A Google Sheet structured with columns for Patient ID, Name, Condition, Vitals, Doctor Email, Doctor WhatsApp Number, and Status.
  3. A Gmail account with Two-Factor Authentication (2FA) enabled. You'll need to generate an "App Password" for n8n to send emails on your behalf.
  4. Access to the WhatsApp Business API (via providers like Twilio or MessageBird) or a service that offers a WhatsApp webhook. You'll need an API token and phone number ID.
  5. An n8n instance, either self-hosted or using n8n.cloud, where you can import and activate the workflow.

Quick Setup Guide

Follow these steps to get your patient update system live in under 30 minutes:

  1. Download & Import: Click the "Download Template" button above. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON file.
  2. Configure Google Sheets: In the "Google Sheets" nodes, replace the placeholder Sheet ID with your own. Paste the credentials JSON from your Google Service Account into the authentication fields.
  3. Set Up Email: In the "Send Email" node, enter your Gmail address and the app password you generated. Update the "To" field to map to the 'Doctor Email' column from your sheet.
  4. Connect WhatsApp: In the "HTTP Request" node for WhatsApp, input your API endpoint URL, bearer token, and format the message payload using data from the previous nodes.
  5. Test & Activate: Use n8n's "Execute Workflow" button to test with a few sample rows. Check that emails and WhatsApp messages are delivered correctly. Once verified, toggle the workflow to "Active." It will now run automatically on its schedule.

Pro tip: Start with a test sheet and a small list of internal team phone numbers/emails. Run the workflow manually several times to perfect the message format and routing logic before going live with real patient data.
