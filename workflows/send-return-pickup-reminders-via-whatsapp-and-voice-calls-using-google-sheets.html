Send Return Pickup Reminders via WhatsApp & Voice Calls Using Google Sheets

Automate customer notifications for return pickups to reduce missed appointments and improve logistics efficiency.

Download Template JSON · n8n compatible · Free

What This Workflow Does

Missed return pickups are a major pain point for e-commerce and logistics teams. They create delays, require manual follow-ups, and lead to frustrated customers. This workflow solves this problem by automating the entire reminder process. It runs on a daily schedule, reads pickup data from Google Sheets, and sends personalized reminders via WhatsApp messages and automated voice calls.

By identifying customers with pending pickups scheduled for today, the workflow ensures timely notifications. It updates the status in Google Sheets to prevent duplicate reminders and provides clear tracking for operations teams. This eliminates manual work, reduces missed pickups, and improves customer satisfaction.

How It Works

Step 1: Daily Schedule Trigger

The workflow activates automatically at a set time each day, such as 9 AM. This ensures reminders are sent at a consistent time, allowing customers to prepare their packages for pickup.

Step 2: Read Data from Google Sheets

It connects to your Google Sheet containing return pickup information. The workflow searches for rows where the pickup date matches today's date and the status is marked as "Pending." This filters the list to only the pickups that need attention.

Step 3: Loop Through Each Pickup

Each matching row is processed individually. This allows for personalized messages that include the customer's name, product details, pickup address, and reason for return.

Step 4: Generate Personalized Messages

Using a Code node, the workflow creates a custom WhatsApp message and a voice call script. The messages are tailored to each customer, making the reminder feel more personal and increasing the likelihood of compliance.

Step 5: Send WhatsApp Reminder

The personalized WhatsApp message is sent via the Twilio API. This ensures the customer receives a clear, written reminder directly on their phone, which they can refer to later.

Step 6: Place Voice Call Reminder

If required, the workflow also places an automated voice call using Twilio's text-to-speech capabilities. This provides an additional layer of notification for customers who may not check their messages immediately.

Step 7: Update Status in Google Sheets

Once the reminders are sent, the workflow updates the status column in Google Sheets to "Reminder Sent." This prevents the same pickup from being processed again and provides a clear audit trail for operations teams.

What You'll Need

  1. A Google Sheets document with columns for Order ID, Customer Name, Phone Number, Product, Pickup Date, Address, Return Reason, and Status.
  2. Twilio account credentials for WhatsApp and Voice APIs.
  3. An n8n instance (self-hosted or cloud) to import and run the workflow.
  4. Basic understanding of how to connect Google Sheets and Twilio nodes in n8n.
