What This Workflow Does

This automation transforms how you handle workflow errors by implementing intelligent, severity-based alerting. Instead of treating all failures equally, it automatically classifies issues and routes notifications appropriately:

Critical errors trigger immediate email alerts to on-call staff while also logging details to Google Sheets for audit purposes. High priority issues generate Telegram notifications for the operations team, while normal alerts go to a general monitoring channel.

How It Works

1. Error Detection

The workflow activates when a connected n8n workflow fails, capturing the execution context and error details.

2. Severity Classification

A code node analyzes the error type, frequency, and business impact to assign one of three severity levels.

3. Alert Routing

Based on severity, alerts are routed to appropriate channels with relevant context for quick diagnosis.

4. Audit Logging

All critical errors are logged to Google Sheets with timestamps, error details, and resolution status.

Pro tip: Customize severity thresholds based on your SLA requirements and team structure.
