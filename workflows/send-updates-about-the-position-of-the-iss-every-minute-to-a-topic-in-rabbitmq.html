What This Workflow Does

This automation template solves the challenge of continuously monitoring and distributing real-time data from external APIs to internal systems. It automatically fetches the current position of the International Space Station every minute, processes the location data, and publishes it to a specified topic in RabbitMQ.

Businesses dealing with IoT devices, live tracking systems, or real-time analytics can adapt this workflow to stream any API data to message queues. Instead of building custom polling scripts and queue management systems, this template provides a production-ready solution that handles scheduling, error recovery, data transformation, and reliable message delivery out of the box.

The workflow ensures data consistency and availability by implementing proper error handling, retry logic, and connection management. It transforms raw API responses into structured messages that downstream systems can easily consume, creating a reliable data pipeline without manual intervention.

How It Works

1. Scheduled Trigger

A cron node initiates the workflow every minute, ensuring regular data collection without manual triggers. This scheduled execution forms the foundation of real-time data streaming pipelines.

2. API Data Fetching

The workflow makes an HTTP request to the Open Notify ISS Position API to retrieve current latitude, longitude, velocity, and timestamp data. This demonstrates how to integrate with external REST APIs in automation workflows.

3. Data Processing & Transformation

Using the Set node, the raw API response is transformed into a structured message format suitable for queue publishing. This includes extracting relevant fields, adding metadata, and formatting the payload according to consumer requirements.

4. Message Queue Publishing

The processed data is published to a specified RabbitMQ topic using the RabbitMQ node. The workflow handles connection management, message serialization, and delivery confirmation to ensure reliable data transmission.

5. Error Handling & Logging

Comprehensive error handling nodes catch and process failures at each stage, with appropriate retry logic and notification mechanisms. This ensures the pipeline remains robust even during API outages or network issues.

Quick Setup Guide

Follow these steps to implement this real-time data streaming workflow in your environment:

  1. Download and import the JSON template into your n8n instance using the import workflow function.
  2. Configure RabbitMQ connection in the RabbitMQ node with your server details, credentials, and target topic/exchange.
  3. Adjust the schedule if needed – the default is every minute, but you can modify the cron expression for different intervals.
  4. Test the API connection by manually executing the workflow once to verify data flows from API to queue.
  5. Activate the workflow and monitor initial executions to ensure messages are being published successfully.
  6. Set up monitoring for queue depth and error notifications to maintain pipeline health.

Pro tip: Before going live, test with a dedicated "test" topic and implement a dead letter exchange in RabbitMQ to capture failed messages for analysis and reprocessing.

RabbitMQ is a robust, open-source message broker that excels in reliability and complex routing scenarios. Compared to alternatives like Apache Kafka or AWS SQS, RabbitMQ offers flexible routing patterns, strong delivery guarantees, and excellent management tools. For automation workflows, it provides durable message queuing, ensuring no data is lost even if systems restart, making it ideal for critical business processes where message delivery must be guaranteed.

RabbitMQ's acknowledgment system, dead letter exchanges, and flexible exchange types (direct, topic, fanout, headers) make it particularly suitable for workflow automation where messages might need to be rerouted based on content or where failed messages require special handling.
