What This Workflow Does

This automation solves the tedious, time-consuming task of manually compiling weekly sales reports from WooCommerce. Instead of spending hours each Monday pulling data, calculating metrics, and formatting reports, this workflow runs automatically to deliver key performance indicators directly to your team's Slack channel and a organized Google Sheets spreadsheet.

The system connects to your WooCommerce store's API, retrieves all orders and refunds from the previous week, performs calculations for revenue, average order value, refund rates, and top products, then formats this information into an easily digestible report. This ensures your entire team—from operations to leadership—has consistent, accurate sales data at the same time every week without manual intervention.

How It Works

1. Weekly Schedule Trigger

The workflow activates automatically every Monday morning (configurable to any day/time). It begins by calculating the exact date range for the previous complete week to ensure accurate period-over-period comparisons.

2. WooCommerce Data Retrieval

Using HTTP requests with your store's API credentials, the system fetches all completed and processing orders from the specified date range, along with any refund data. This raw transaction data forms the foundation for all subsequent calculations.

3. KPI Calculation Engine

Multiple code nodes process the raw data to compute essential metrics: total orders, gross revenue, net revenue (after refunds), average order value, refund count and amount, and identification of top-performing products by revenue or quantity sold.

4. Data Consolidation & Formatting

All calculated KPIs are merged into a clean, standardized dataset. Unnecessary fields are removed, and the data is structured for both human-readable reporting and systematic record-keeping.

5. Dual Output Delivery

The formatted report is simultaneously sent to your designated Slack channel for immediate team visibility and appended as a new row in a Google Sheets spreadsheet for historical tracking and trend analysis.

What You'll Need

  1. WooCommerce Store with REST API access enabled
  2. Consumer Key & Secret from WooCommerce Settings > Advanced > REST API
  3. n8n Instance (cloud or self-hosted) with workflow scheduling capabilities
  4. Slack Workspace where you have permission to post to a channel
  5. Google Sheets spreadsheet with write access for the service account
  6. Basic understanding of connecting APIs (credentials setup)

Pro tip: Create a dedicated "#weekly-metrics" Slack channel for these automated reports. This keeps sales discussions focused and makes historical reports easily searchable for your entire team.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure WooCommerce credentials in all HTTP Request nodes with your store's domain, consumer key, and secret.
  3. Set your Slack channel in the Slack node where reports should be posted.
  4. Connect Google Sheets by specifying the spreadsheet ID and worksheet name.
  5. Adjust the schedule if needed (default is Monday 9 AM).
  6. Test with a manual trigger to verify all connections work correctly.
  7. Activate the workflow and let it run automatically each week.
