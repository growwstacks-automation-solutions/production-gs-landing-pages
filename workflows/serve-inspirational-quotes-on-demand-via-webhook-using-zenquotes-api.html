What This Workflow Does

This automation solves a common need for developers and content creators: delivering fresh, inspirational content on demand without building a custom backend. Instead of maintaining a static list of quotes or manually updating content, this n8n workflow taps into the free ZenQuotes API to serve randomized, high-quality quotes instantly via a simple webhook call.

Whenever an external application—like a website, chatbot, or internal dashboard—makes an HTTP request to your n8n webhook URL, the workflow triggers. It fetches a configurable number of quotes, formats them into a clean JSON structure, and returns them immediately. This turns a simple API into a scalable content delivery system that can power motivational features across your digital products.

The business value is clear: you save development time, eliminate manual content updates, and create a more engaging user experience. Whether you're building a Slack bot for team morale, a daily quote email campaign, or a dynamic widget for your website, this template provides the reliable, automated backbone.

How It Works

The workflow is a straightforward yet powerful pipeline designed for reliability and ease of use.

1. Webhook Trigger

The process starts with a Webhook node configured to listen on a specific path (e.g., /inspire). This node acts as the public endpoint. Any GET or POST request to this URL initiates the workflow. You can secure this endpoint within n8n if needed.

2. Fetch Quotes from ZenQuotes API

An HTTP Request node calls the ZenQuotes API (https://zenquotes.io/api/random?count=5). The count parameter is adjustable, letting you control how many quotes are retrieved per request. The API returns an array of quote objects, each containing the quote text and author.

3. Data Transformation

A Set node (or Function node) processes the raw API response. It maps each item, often combining the q (quote) and a (author) fields into a single, nicely formatted string (e.g., "“The journey of a thousand miles begins with one step.” – Lao Tzu"). This step ensures the output is clean and ready for consumption by your application.

4. Respond with Structured JSON

A final Respond to Webhook node sends the processed array of quotes back to the original caller. The response is a standard JSON array, making it incredibly easy to parse and use in any frontend application, chatbot, or other integrated service.

Pro tip: You can easily extend this workflow. Add a Code node to filter quotes by keyword, integrate with a database to log requests, or connect it to a scheduling app like Cron to send daily quotes via email or Slack automatically.

Quick Setup Guide

Get this automation running in your n8n instance in under 5 minutes.

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n dashboard, go to Workflows > Import from File and select the downloaded JSON.
  2. Activate the Webhook: Open the imported workflow. Click the "Webhook" node and note the unique URL it generates (e.g., https://your-n8n.com/webhook/1234). Toggle the workflow to "Active".
  3. Test the Endpoint: Open a new browser tab or use a tool like Postman/curl to send a GET request to your webhook URL. You should immediately receive a JSON response with inspirational quotes.
  4. Integrate: Use this URL in your application code. For example, in a website's JavaScript, you could fetch the quotes and display them dynamically.
  5. Customize (Optional): Edit the HTTP Request node to change the number of quotes (count parameter) or modify the Set node to alter the response format to suit your needs.
