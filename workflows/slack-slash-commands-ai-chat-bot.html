How It Works

1. Slack Command Trigger

When a user types a slash command (like /ask "How do I request time off?") in any Slack channel, Slack sends this request to a webhook URL configured in your n8n workflow. The command includes the user's question, channel information, and user details for context-aware responses.

2. Request Processing & Routing

The workflow receives the webhook payload and uses a Switch node to route different commands to appropriate handlers. This allows you to support multiple slash commands (/ask, /docs, /help) from the same automation, each with specialized functionality.

3. AI Response Generation

For /ask commands, the question is sent to an AI language model (like OpenAI's GPT) with proper context and instructions. The AI generates a helpful, accurate response based on your configured knowledge base, tone guidelines, and response parameters.

4. Channel Response Delivery

The generated response is formatted and sent back to the exact Slack channel where the command originated. The workflow uses Slack's API to post messages with proper formatting, threading, and user attribution, making the interaction feel natural and integrated.

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance using the "Import from File" option in the workflow editor.
  2. Configure Slack app: Create a new Slack app in your workspace, add the chat:write permission scope, and create a slash command pointing to your n8n webhook URL.
  3. Set up AI integration: Connect your AI service (like OpenAI) credentials in the appropriate nodes within the workflow.
  4. Customize responses: Adjust the prompt templates and response parameters to match your company's tone and knowledge requirements.
  5. Test and deploy: Install the Slack app to your workspace, test the slash commands in a private channel, then roll out to appropriate teams.

Pro tip: Start with a single team or channel for initial testing. Create a dedicated #ai-assistant channel where users can experiment with commands before rolling out organization-wide. This allows you to refine responses and build confidence in the system.
