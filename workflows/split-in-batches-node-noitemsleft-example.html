What This Workflow Does

This workflow demonstrates how to use the noItemsLeft parameter to check if there are items remaining to be processed by the SplitInBatches node. It provides a complete solution for managing batch processing workflows where you need to track completion status and implement conditional logic based on whether batches have been fully processed.

The template includes practical examples of batch processing with mock data generation, batch splitting, conditional checking, and completion handling - all essential components for building robust batch processing automations.

How It Works

1. Data Generation

The Function node generates mock data for demonstration purposes. In a real implementation, you would replace this with your actual data source node.

2. Batch Splitting

The SplitInBatches node divides the incoming data into manageable chunks. The batch size is configurable based on your specific requirements and API limitations.

3. Completion Checking

The IF node evaluates the noItemsLeft parameter to determine if all items have been processed. This enables conditional routing in your workflow based on batch completion status.

4. Completion Handling

The Set node executes when all batches are complete, allowing you to implement any post-processing logic needed after batch completion.
