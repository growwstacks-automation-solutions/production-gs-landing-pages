What This Workflow Does

This n8n template builds Stacey, an intelligent AI assistant that operates entirely within Telegram. Instead of switching between multiple apps to manage emails, schedule meetings, or create content, users can simply message Stacey in natural language. The assistant understands the request, determines the appropriate action using MCP (Model Context Protocol) logic, and executes it through connected tools like Gmail, Google Calendar, or a content writer.

The workflow solves the problem of fragmented business tools by creating a single, conversational interface for common operational tasks. Business owners, support teams, and entrepreneurs can delegate routine work to Stacey, reclaiming hours each week previously spent on manual administrative work. It transforms Telegram from a simple messaging app into a powerful command center for your business operations.

Beyond basic automation, Stacey demonstrates how AI can act as an intelligent router—not just executing fixed commands, but understanding intent and selecting the right tool for each job. This pattern can be extended to virtually any business process, making it a foundational template for building more sophisticated AI-powered operations.

How It Works

The workflow follows a clear pipeline from user request to executed action and confirmation.

Step 1: Message Reception & Processing

The Telegram Trigger node listens for incoming messages (both text and voice notes) in your designated bot. Voice messages are automatically transcribed using Whisper AI, converting speech to text for processing. The system captures the complete user request along with sender information for personalized responses.

Step 2: AI Intent Recognition

The transcribed or text message is sent to Google Gemini (or optionally GPT-4o) where Stacey's system prompt analyzes the request. The AI determines the user's intent—whether they want to send an email, schedule a meeting, write content, search the web, or perform calculations—and extracts relevant parameters like dates, recipients, and subject matter.

Step 3: MCP Tool Routing

Based on the identified intent, the workflow routes the request to the appropriate tool via the MCP Server. This is where the magic happens: instead of hard-coded "if-then" logic, the AI dynamically selects the right tool for the job. The MCP Map Server contains all available tools and their capabilities, allowing for flexible expansion.

Step 4: Action Execution

The selected tool executes the requested action. For emails, it connects to Gmail API with proper OAuth authentication. For calendar events, it interacts with Google Calendar. For content creation, it triggers a sub-workflow that generates blog posts or emails based on the topic and tone specified. Each tool handles its specific API calls and error management.

Step 5: Response & Confirmation

Once the action is completed (or if clarification is needed), Stacey sends a natural language response back through Telegram. The response confirms what was done, provides any relevant output (like generated content or search results), and maintains a conversational tone that makes users feel like they're interacting with a helpful assistant rather than a robotic system.

Pro tip: For best results, use GPT-4o instead of Gemini for the AI agent node. While Gemini works well for basic tasks, GPT-4o provides significantly better reasoning for complex, multi-step requests and handles nuanced business language more effectively.

Quick Setup Guide

Follow these steps to get your Telegram AI assistant running in under 30 minutes:

  1. Create your Telegram bot by messaging @BotFather on Telegram, using the /newbot command, and saving the provided token.
  2. Set up Google Cloud credentials by creating a project in Google Cloud Console, enabling Gmail and Calendar APIs, and creating OAuth2 credentials with the appropriate scopes.
  3. Import the workflow into your n8n instance by downloading the JSON template and using the "Import from File" option in the n8n editor.
  4. Connect your credentials to each node: Telegram Bot token to the trigger and sender nodes, Google OAuth to Gmail and Calendar nodes, and your AI API keys to the respective nodes.
  5. Test basic functionality by messaging your bot commands like "Schedule a meeting tomorrow at 2pm" or "Send an email to client@example.com about the project update."
  6. Customize the system prompt in the AI Agent node to match your business tone, add custom tools, or adjust response styles.
  7. Expand with additional tools by duplicating existing tool patterns and connecting them to the MCP Map Server for new capabilities like CRM updates or document generation.

Important: Before going live, test the bot thoroughly with different phrasings of the same request to ensure the AI correctly interprets various user expressions. Set up error handling for API failures and consider implementing a fallback response for unrecognized commands.
