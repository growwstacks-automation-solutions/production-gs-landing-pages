What This Workflow Does

This automation monitors stock prices using Alpha Vantage's API to detect key technical analysis patterns - specifically the Golden Cross (bullish signal) and Death Cross (bearish signal) using simple moving averages (SMAs). When these patterns are identified, it sends formatted alerts to a Discord channel via webhook.

The workflow runs automatically after market close each weekday, analyzing the latest price data without requiring manual intervention. It maintains historical price data in PostgreSQL for accurate moving average calculations and can monitor multiple stock symbols simultaneously.

How It Works

1. Data Collection

The workflow starts by fetching daily stock price data from Alpha Vantage's API for each configured ticker symbol. This includes open, high, low, close prices and volume.

2. Data Storage

New price data is stored in a PostgreSQL database to maintain historical records needed for moving average calculations. The database preserves 121 days of history for accurate SMA computations.

3. Moving Average Calculation

A Code node computes two simple moving averages (SMAs) for each stock - a short-term (60-day) and long-term (120-day) average. These are calculated for both the current and previous day.

4. Signal Detection

If nodes compare the SMAs to identify when the short-term average crosses above (Golden Cross) or below (Death Cross) the long-term average, indicating potential trend changes.

5. Alert Delivery

Formatted messages are sent to Discord via webhook when signals are detected, including the stock symbol, signal type, and relevant price information.

  4. n8n instance (self-hosted or cloud)

Quick Setup Guide

  1. Import the JSON template into your n8n instance
  2. Configure PostgreSQL credentials in both database nodes
  3. Add your Alpha Vantage API key to the HTTP Request node
  4. Update the stock tickers in the Set node
  5. Paste your Discord webhook URL in the final HTTP Request node
  6. Test the workflow manually before enabling the schedule
