What This Workflow Does

Manually tracking daily orders and compiling reports is a repetitive, error-prone task that steals valuable time from business growth. This automation solves that by creating a seamless, hands-off pipeline for order management.

The workflow listens for new orders via a webhook—whether from your e-commerce platform, payment processor, or custom app—and instantly records each one into a designated Airtable base. Then, on a schedule you set (like every evening at 7 PM), it queries Airtable for all orders from that day, formats them into a clear HTML table, and emails the summary to you or your team.

This turns a fragmented, manual process into a reliable system. You get a permanent, searchable record of every transaction and a timely digest without lifting a finger, ensuring nothing slips through the cracks.

How It Works

The automation follows a logical three-step process: capture, store, and report.

1. Capture: Webhook Trigger

The workflow is activated by an incoming HTTP POST request (a webhook). When a new order is created in your system, you configure it to send the order data (like ID, price, timestamp) to a unique URL generated by n8n. This happens in real-time, the moment the order is placed.

2. Store: Airtable Integration

The data from the webhook is mapped and sent to your Airtable base. The workflow uses your Airtable access token to securely create a new record in your specified table, logging the order details. This builds a comprehensive, queryable database of all transactions.

3. Report: Scheduled Email Summary

A schedule trigger runs once per day. It connects to the same Airtable base, fetches all records created that day, and structures the data. An HTML node formats the orders into a readable table, which is then sent via Gmail (or your email service) to your specified recipients, providing a clean end-of-day report.

Pro tip: Use the "Set" node in the template to rename or transform incoming data fields before they hit Airtable. This ensures your database stays clean and consistent, even if your source system uses different field names.

What You'll Need

  1. An n8n instance: You can use n8n.cloud (free tier available) or self-hosted n8n.
  2. An Airtable account: Free plan is sufficient. You'll need to create a base and a table with fields for orderID (Number), orderPrice (Number), and time (Date).
  3. Airtable Access Token: Created in your Airtable account with data.records:read, data.records:write, and schema.bases:read scopes for your base.
  4. Gmail/Email Credentials: OAuth2 credentials for Gmail or SMTP settings for another email service to send the daily summary.
  5. A source that can send webhooks: Your e-commerce platform, payment gateway, or custom application must be able to send a POST request with JSON order data to the n8n webhook URL.

Quick Setup Guide

Follow these steps to import and configure this automation in under 10 minutes.

  1. Import the Template: Click the "Download Template" button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded JSON.
  2. Configure Airtable: Create your Airtable base and table as described. Generate a personal access token and create a new Airtable credential in n8n using the "Access Token" method. Enter this credential in both the "Store Order" and "Airtable Get Today's Orders" nodes.
  3. Set Up the Webhook: Activate the workflow and copy the unique "Production URL" from the Webhook node. Configure your order system (e.g., Shopify webhook, Zapier, Make) to send a POST request to this URL with a JSON body containing orderID and orderPrice.
  4. Configure the Email: Set up your Gmail (or other email) credentials in the "Send to Gmail" node. Update the "To" email address and customize the email subject/body in the HTML node if desired.
  5. Test the Flow: Use the provided curl command in the template description to send a test order. Check Airtable for the new record and wait for the scheduled email to confirm the full cycle works.
  6. Activate & Schedule: Turn on the workflow. The schedule trigger is set to run daily at 7 PM by default—adjust this time in the "Schedule Trigger" node to match your needs.
