Automated URL Submission for SEO

This workflow automatically submits sitemap URLs from Oncrawl to both Google's Indexing API and IndexNow, ensuring faster discovery and indexing of your most important pages. Save hours of manual submission while improving search visibility.

Download Template JSON · Zapier compatible · Free

What This Workflow Does

This automation solves a critical SEO challenge: getting important website pages indexed quickly by search engines. Traditional methods rely on passive discovery through sitemaps, which can take days or weeks. This workflow actively pushes URLs to both Google and multiple search engines via IndexNow, dramatically reducing indexing time.

For businesses, this means new products, updated content, and seasonal pages appear in search results faster. The workflow intelligently filters Oncrawl data to submit only high-priority URLs, maximizing the impact of your API quotas while minimizing manual oversight.

How It Works

Step 1: Extract URLs from Oncrawl

The workflow begins by pulling newly discovered or updated URLs from your Oncrawl project. It can filter by crawl priority, content type, or modification date to focus on the most important pages.

Step 2: Format for API Submission

URLs are formatted according to Google Indexing API specifications, with proper metadata including update type (URL_UPDATED or URL_DELETED). The same URLs are prepared for IndexNow's simpler format.

Step 3: Parallel API Submission

The workflow simultaneously submits to both Google's Indexing API (for Google Search) and IndexNow (for Bing, Yandex, and other supporting search engines). This dual approach ensures maximum coverage.

Pro tip: Configure the workflow to run after your regular Oncrawl crawls to ensure you're always submitting the freshest URL data.
