What This Workflow Does

This automation solves the universal business problem of information overload. Teams drown in lengthy reports, contracts, proposals, and research documents that nobody has time to read thoroughly. Important decisions, action items, and risks get buried in pages of text.

The workflow automatically monitors designated Google Drive folders for new Google Docs or PDFs. When a document appears, it extracts the text, sends it to OpenAI's GPT-4 for intelligent analysis, and delivers a structured summary to your team via Slack or email. It identifies key points, suggested actions, sentiment, and language—transforming hours of reading into minutes of actionable insights.

How It Works

1. Folder Monitoring & Detection

The workflow starts by watching your specified Google Drive folder for new files. It immediately detects whether each file is a native Google Doc or a PDF, routing them through different extraction paths.

2. Text Extraction

For Google Docs, it uses the Google Docs API to pull clean, structured text. For PDFs, it employs text extraction tools that handle both digital PDFs and scanned documents (using OCR when needed).

3. AI Analysis & Summarization

The extracted text goes to GPT-4 with a carefully crafted prompt that instructs the AI to generate: a concise title, executive summary, 3-5 key points, suggested action items with owners, language detection, and sentiment analysis (positive/neutral/negative).

4. Notification Delivery

The structured output is formatted into clean messages and sent to your chosen Slack channel and/or email recipients. The format is customizable—you can send full details to project teams and executive summaries to leadership.

Pro tip: Customize the AI prompt to match your industry terminology. For legal documents, emphasize clauses and obligations. For financial reports, highlight metrics and projections. For research papers, focus on methodology and findings.

What You'll Need

  1. Google Workspace Account: With access to Google Drive and Google Docs APIs (standard business or enterprise accounts work).
  2. OpenAI API Key: Access to GPT-4 or GPT-3.5-turbo models (available through OpenAI's platform).
  3. Slack Workspace: With permissions to create incoming webhooks or use OAuth for posting to channels.
  4. Gmail Account (Optional): For email notifications if you prefer that over Slack.
  5. n8n Instance: Self-hosted or cloud version of n8n to run the workflow.

Quick Setup Guide

Follow these steps to get your document summarization system running in under 30 minutes:

  1. Download & Import: Download the template JSON file and import it into your n8n instance.
  2. Configure Google Drive: Set up Google OAuth credentials and select the folder you want to monitor for new documents.
  3. Connect OpenAI: Add your OpenAI API key and select your preferred model (GPT-4 for best results, GPT-3.5-turbo for cost efficiency).
  4. Set Up Notifications: Connect either Slack (via webhook or OAuth) or Gmail for delivering summaries.
  5. Test & Activate: Upload a test document to your monitored folder, verify the summary arrives correctly, then activate the workflow.

Pro tip: Start with a dedicated "AI Processing" folder rather than monitoring critical business folders directly. This gives you control over which documents get summarized and prevents accidental processing of sensitive files.
