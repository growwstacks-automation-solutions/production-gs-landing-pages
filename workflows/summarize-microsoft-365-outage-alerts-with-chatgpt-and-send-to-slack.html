What This Workflow Does

IT administrators and managed service providers (MSPs) are often flooded with verbose, technical email alerts from Microsoft 365 whenever there's a service incident. These emails are crucial but time-consuming to parse, especially during critical outages. This automation solves that problem by acting as your intelligent alert filter.

The workflow automatically scans a designated Outlook mailbox for Microsoft 365 service health notifications every minute. When an alert arrives, it uses OpenAI's GPT-4o-mini model to extract the essential details: what service is affected, the severity, estimated resolution time, and—critically—whether your specific user regions are impacted. It then formats this into a clean, actionable Slack message and posts it to a dedicated channel (like #it-alerts or #m365-status). Finally, it cleans up by deleting the processed email, keeping the inbox tidy.

This transforms a reactive, manual process into a proactive, automated communication system, ensuring your team is informed instantly without sifting through emails.

How It Works

The automation follows a logical, step-by-step process to ensure reliable and intelligent alerting.

1. Monitor & Trigger

The workflow is triggered on a schedule (every 1 minute) to check a specific Outlook mailbox or shared mailbox for new emails. It filters for emails from Microsoft 365 service notifications, typically identified by sender address or subject line keywords like "Service health incident".

2. Parse & Filter

When a matching email is found, the workflow extracts the full HTML or plain text body. A Code node can be used to perform initial checks, such as scanning for mentions of specific countries or user regions you define, to determine immediate relevance to your organization.

3. AI Summarization

The raw email content is sent to the OpenAI node (using the cost-effective GPT-4o-mini model). A system prompt instructs the AI to summarize the incident concisely: identify the service, impact, root cause, Microsoft's action, ETA for fix, and—based on the region filter—state if your users are affected. This turns paragraphs of technical jargon into three clear bullet points.

4. Slack Notification

The AI-generated summary is formatted into a Slack "block" message. This includes a clear title, the summary, a direct link to the official Microsoft 365 admin center incident page for details, and an appropriate status emoji (e.g., 🔴 for major outage). This message is posted to your pre-configured Slack channel.

5. Cleanup

After successful posting to Slack, the workflow uses the Outlook node to move the original alert email to a processed folder or delete it entirely. This prevents duplicate processing and keeps the monitoring mailbox clean.

Pro tip: Use OpenAI's "Projects" feature to assign a specific monthly budget to this workflow. This isolates costs and prevents unexpected charges, though usage is typically just pennies per month.

What You'll Need

  1. An n8n instance: You can use n8n.cloud (quickest) or self-host n8n on your own server.
  2. Microsoft 365/Outlook Credentials: An account (user or shared mailbox) that receives the Microsoft 365 Service Health notifications. You'll need to create an OAuth 2.0 app credential in n8n.
  3. Slack Credentials: A Slack bot token with the chat:write scope, added to the channel you want to post alerts to.
  4. OpenAI API Key: An API key from OpenAI, preferably assigned to a "Project" with a small monthly budget for cost control.
  5. 5 minutes for configuration: To set the target Slack channel and customize the region check in the system prompt.

Quick Setup Guide

Get this automation running in your environment in just a few steps:

  1. Download & Import: Click the "Download Template" button above to get the JSON file. In your n8n instance, go to Workflows > Import from File and select the downloaded file.
  2. Configure Outlook: In the first "Outlook" node, click the credential dropdown and set up a new OAuth 2.0 connection. Authenticate with the Microsoft 365 account that receives the service alerts.
  3. Customize the AI Prompt: Open the "OpenAI" node. In the system prompt field, update the example countries ("U.S. and Australia") to the primary regions where your users are located. This ensures accurate impact statements.
  4. Set Your Slack Channel: In the "Slack" node, configure the credential with your bot token. Then, in the node parameters, specify the exact Slack channel ID (e.g., C1234567890) where alerts should be posted.
  5. Activate & Test: Turn the workflow ON. You can test it by sending a simulated alert email to your monitored mailbox or wait for the next real Microsoft notification.
Example Slack output: A clean, formatted message showing service impact, summary, and a direct link for more details.
