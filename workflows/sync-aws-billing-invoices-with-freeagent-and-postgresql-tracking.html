What This Workflow Does

Managing cloud expenses manually is a time-consuming and error-prone task for businesses. Every month, finance teams or business owners must download AWS invoices, manually enter them into their accounting software (like FreeAgent), mark them as paid, and ensure everything reconciles. This workflow automates that entire process.

It connects directly to the AWS Invoicing API to fetch invoice data for the previous month, creates corresponding bills in FreeAgent with accurate dates and amounts, marks them as paid with a bank transaction explanation, and logs each processed invoice in a PostgreSQL database to prevent duplicates. This turns a 30–60 minute monthly chore into a fully automated, reliable process that runs in the background.

The result is flawless financial records, saved administrative hours, and complete visibility into your cloud spending—all without manual intervention.

How It Works

The automation follows a logical, step-by-step process to ensure accuracy and reliability.

Step 1: Schedule & Fetch

A scheduled trigger (like a monthly cron job) initiates the workflow. It first calculates the date range for the previous month's invoices, then calls the AWS Invoicing API using signed requests to securely retrieve all invoice summaries for that period.

Step 2: Filter & Validate

The workflow filters the retrieved invoices by issue date to ensure only new, unprocessed invoices move forward. It then checks each invoice ID against a PostgreSQL database table that tracks previously processed invoices. If an invoice already exists in the database, it's skipped to prevent duplication.

Step 3: Transform & Create

For each new invoice, the workflow maps AWS data fields (invoice ID, billing period, total amount, currency, issue date, due date) to the corresponding FreeAgent bill fields. It handles currency conversion if needed and applies the correct VAT treatment (default 20%, customizable). A bill is then created in FreeAgent via its API.

Step 4: Mark Paid & Log

Immediately after creating the bill, the workflow creates a bank transaction explanation in FreeAgent to mark the bill as paid, using the invoice total and date. Finally, it inserts a record into the PostgreSQL `aws_invoices_processed` table, storing the invoice ID, FreeAgent bill URL, amount, currency, and timestamp for permanent audit tracking.

What You'll Need

  1. AWS Account with billing access and an IAM user configured with the invoicing:ListInvoiceSummaries permission.
  2. FreeAgent Account with API access (OAuth2 credentials).
  3. PostgreSQL Database (or a compatible alternative) to track processed invoices. A simple table schema is provided.
  4. n8n or Zapier Instance to host and run the workflow.
  5. Your AWS Account ID (12-digit number), FreeAgent Contact ID for AWS, and relevant category/bank account IDs from FreeAgent.

Pro tip: You can modify this workflow to use n8n's built-in "Data Table" node instead of PostgreSQL if you prefer a simpler, no-database setup. However, PostgreSQL provides better persistence and querying capabilities for audit purposes.

Quick Setup Guide

Follow these steps to get this automation running in under 30 minutes:

  1. Download & Import the JSON template into your n8n or Zapier instance.
  2. Create PostgreSQL Table using the provided SQL schema to track processed invoices.
  3. Configure Credentials in the workflow: Add your PostgreSQL database connection, FreeAgent OAuth2 credentials, and AWS access keys.
  4. Update Placeholder Values in the code nodes: Replace YOUR_AWS_ACCOUNT_ID, YOUR_CONTACT_ID, YOUR_CATEGORY_ID, and YOUR_BANK_ACCOUNT_ID with your actual IDs from FreeAgent.
  5. Test with One Invoice by temporarily modifying the date filter to catch a recent invoice, then run the workflow manually to verify everything works.
  6. Schedule the Workflow to run monthly (e.g., on the 5th of each month) to automatically process last month's AWS invoices.
