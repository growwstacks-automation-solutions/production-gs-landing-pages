Sync Calendly Bookings to Easy Redmine CRM Automatically

What This Workflow Does

This automation solves a common but critical problem for sales and customer success teams: the manual, error-prone process of transferring meeting details from scheduling tools to CRM systems. When someone books a meeting through Calendly, this workflow automatically finds their contact in Easy Redmine CRM, logs the meeting as a sales activity, adds a detailed comment with the meeting context, and sends an Outlook email notification to the account manager.

The solution eliminates the 5–10 minutes per meeting that sales reps typically spend copying information between systems. More importantly, it ensures no meeting goes unlogged, provides complete visibility into prospect engagement, and creates a consistent audit trail for every customer interaction. This is particularly valuable for teams managing high volumes of discovery calls, demos, or check-in meetings where tracking every touchpoint directly impacts conversion rates and customer satisfaction.

How It Works

The workflow follows a logical sequence that mirrors how a sales rep would manually process a booking, but does it instantly and consistently.

1. Calendly Trigger

The workflow starts when someone books a meeting through your Calendly link. The Calendly Trigger node listens for the "invitee.created" event, capturing all booking details including the attendee's name, email, meeting time, and any custom questions they answered during scheduling.

2. Find Lead in Easy Redmine

Using the attendee's email address, the workflow queries your Easy Redmine CRM to find the matching lead or contact. This step ensures the meeting is attached to the correct record in your sales pipeline. If no match is found, the workflow can be extended to create a new lead automatically.

3. Add CRM Comment

Once the lead is identified, the workflow posts a detailed comment to their CRM record. This comment includes the meeting purpose, scheduled time, and any notes from the booking form—creating immediate context for anyone viewing the lead's history.

4. Create Sales Activity

A new sales activity is created in Easy Redmine, categorized appropriately (like "Discovery Call" or "Demo Scheduled"). This activity appears in the lead's timeline and can trigger CRM workflows for follow-up tasks or pipeline stage changes.

5. Send Outlook Notification

Finally, the workflow sends an email through Microsoft Outlook to the assigned account manager. This notification includes all meeting details, ensuring the rep is prepared and can add the event to their calendar with one click.

Quick Setup Guide

Follow these steps to implement this automation in your n8n environment:

  1. Import the template into your n8n editor using the JSON file downloaded from this page
  2. Configure Calendly credentials using OAuth2 and set the webhook to listen for "invitee.created" events
  3. Set up Easy Redmine connection using API token authentication with appropriate permissions
  4. Connect Microsoft Outlook using OAuth2 for the account that should receive notifications
  5. Test with a real booking using a test Calendly link to verify the entire flow works end-to-end
  6. Activate the workflow and connect it to your production Calendly event types
