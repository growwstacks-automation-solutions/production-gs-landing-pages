What This Workflow Does

This automation solves a critical problem for businesses using Salesforce for CRM and KlickTipp for email marketing: data silos. When contact information, consent status, and segmentation tags live in separate systems, your sales and marketing teams work with outdated or conflicting data. This leads to missed opportunities, compliance risks, and inefficient campaigns.

The workflow establishes a reliable, automated two-way bridge between Salesforce Contacts and KlickTipp subscribers. It ensures that when a contact's details are updated in Salesforce, KlickTipp reflects those changes, and vice-versa. Crucially, it syncs marketing consent status, ensuring you never email someone who has opted out, protecting you from GDPR violations and preserving sender reputation.

Beyond basic contact info, it intelligently maps KlickTipp tags to Salesforce Topics (and back), keeping segmentation consistent. It also includes automated cleanup processes, removing contacts deleted in one system from the other, maintaining a lean and accurate database.

How It Works

The workflow operates through two primary synchronized pathways, triggered by changes in either system.

1. KlickTipp to Salesforce Sync

When a subscriber is tagged or updated in KlickTipp, the workflow fetches their full profile. It first checks if a Salesforce Contact ID is already linked. If not, it creates a new Contact record; if it exists, it updates the existing one. Key fields like name, email, and custom fields are mapped. The Salesforce ID is then written back to a custom KlickTipp field to permanently link the records.

2. Salesforce to KlickTipp Sync

Triggered by a new or updated Salesforce Contact, this path normalizes data formats (like dates) and checks for an existing KlickTipp subscriber ID. New contacts are added via a Single Opt-In process, while existing subscribers are updated. The KlickTipp ID and marketing status are written back to Salesforce, and relevant Topic Assignments are converted into KlickTipp tags.

3. Automated Data Hygiene

A scheduled trigger runs a daily cleanup, identifying contacts deleted in Salesforce within the last 24 hours and removing them from KlickTipp. It also listens for GDPR deletion requests from KlickTipp, triggering the removal of the corresponding Salesforce record if it exists.

Pro tip: For maximum reliability, use a unique, immutable identifier like email address as the primary matching key between systems, supplemented by the stored internal IDs for speed.

What You'll Need

  1. A self-hosted n8n instance (required for the KlickTipp community node).
  2. Active accounts with API access for both Salesforce and KlickTipp.
  3. Salesforce OAuth2 credentials configured in n8n.
  4. KlickTipp API credentials (username and API key).
  5. A custom field in KlickTipp (e.g., field227883) to store the Salesforce Contact ID.
  6. Custom fields in Salesforce for KlickTipp_ID__c and KlickTipp_marketing_status__c.
  7. Basic understanding of your Salesforce instance URL and object/field names.

Quick Setup Guide

Follow these steps to deploy this automation in your n8n environment.

  1. Import the Template: Download the JSON file above and import it into your n8n instance.
  2. Configure Credentials: Set up your Salesforce (OAuth2) and KlickTipp (API Key) credentials in n8n.
  3. Update Field Mappings: In the "Edit Fields (Set)" nodes, verify the field mappings match your Salesforce Contact and KlickTipp subscriber field names.
  4. Map Your Segments: In the "Segment Matcher" nodes, replace the example Tag IDs and Salesforce Topic IDs with your own.
  5. Set Instance URLs: Update all "HTTP Request" nodes pointing to Salesforce with your specific Salesforce instance base URL.
  6. Test with One Contact: Activate the workflow and test the sync in both directions using a single test contact to verify data flows correctly.
  7. Activate & Monitor: Once tested, activate the workflow and monitor the executions for the first few days to ensure smooth operation.

Important: This workflow uses the KlickTipp community node, which is only available for self-hosted n8n instances. It will not work on n8n.cloud.
