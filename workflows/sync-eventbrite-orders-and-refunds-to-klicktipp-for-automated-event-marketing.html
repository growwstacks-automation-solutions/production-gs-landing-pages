Sync Eventbrite Orders & Refunds to KlickTipp

Automate event marketing by syncing Eventbrite orders and refunds directly to KlickTipp. Free n8n workflow template for self-hosted instances.

What This Workflow Does

Event organizers and marketers often struggle with manual data transfer between registration platforms and email marketing tools. Every new Eventbrite order requires copying attendee details into KlickTipp, tagging them correctly, and updating their status if they refund. This workflow automates that entire process.

It listens for new Eventbrite orders and refunds, then automatically updates the contact in KlickTipp with event details, adds appropriate tags (like “Buyer” or “Refundee”), and stores key event information in custom fields. This ensures your email lists are always accurate and segmented, ready for targeted follow-up campaigns.

The automation eliminates hours of manual work per event, reduces errors, and enables real-time marketing reactions—like sending a welcome email immediately after registration or a survey after the event ends.

How It Works

1. Eventbrite Trigger

The workflow starts with an Eventbrite trigger node that monitors your event for two key actions: “order.placed” (new registration) and “order.refunded” (cancelation). Each time one of these events occurs, the workflow is activated and pulls the attendee’s details.

2. Switch Logic Routing

A Switch node examines the event type. If it’s an order placement, the workflow proceeds to add the contact to KlickTipp and tag them as a buyer. If it’s a refund, it routes to a separate path that updates the contact with a refundee tag and removes the buyer tag.

3. Data Enrichment & Field Mapping

For new orders, the workflow fetches additional event data (name, start date, page URL) from Eventbrite and maps it to predefined custom fields in KlickTipp. This creates a rich contact profile with event context.

4. Fee Check & Segmentation

The workflow checks if the order included a fee (paid ticket) versus free registration. This allows for further segmentation—you could tag paid attendees differently or trigger specific follow-up sequences for premium ticket holders.

5. KlickTipp Contact Update

Finally, the contact is added or updated in KlickTipp with all the mapped data and tags. The process is fully automated; no manual import or CSV upload is needed.

What You'll Need

  1. A self-hosted n8n instance (community nodes required).
  2. Eventbrite account with OAuth2 access enabled.
  3. KlickTipp account with API credentials.
  4. KlickTipp custom fields created: “Eventbrite | Event name”, “Eventbrite | Start timestamp”, “Eventbrite | Event page URL”.
  5. KlickTipp tags created: “Eventbrite | Buyer”, “Eventbrite | Refundee”, “Eventbrite | Registrant”.
  6. Basic understanding of n8n node configuration.

Pro tip: Enable “auto tag removal” in KlickTipp settings so the “Buyer” tag is automatically removed when the “Refundee” tag is applied. This keeps your lists clean.

Quick Setup Guide

  1. Import the JSON template into your self-hosted n8n instance.
  2. Configure the Eventbrite trigger node with your OAuth2 credentials and select the event you want to monitor.
  3. Set up the KlickTipp node with your API key and ensure the custom fields and tags exist in your KlickTipp account.
  4. Replace the placeholder tag IDs in the workflow with your actual KlickTipp tag IDs.
  5. Test the flow by placing a test order in Eventbrite and verifying the contact appears correctly tagged in KlickTipp.
  6. Activate the workflow and let it run automatically for all future orders and refunds.
