What This Workflow Does

This automation solves the common problem of teams using Jira for technical project management while other departments rely on Notion for documentation and collaboration. Without synchronization, information becomes siloed, leading to duplicate work, outdated references, and communication gaps between technical and non-technical teams.

The workflow automatically creates, updates, and deletes Notion database pages whenever corresponding changes occur in Jira. It maintains a complete two-way sync including all subsequent comments on issues, ensuring discussions remain accessible to all stakeholders regardless of which platform they prefer.

Configuration settings for mapping Jira fields to Notion properties

How It Works

1. Trigger: Jira Webhook Activation

The workflow starts when Jira sends a webhook notification about an issue event (creation, update, or deletion). The webhook is configured to capture all relevant changes including comments.

2. Data Extraction

The workflow extracts key issue details from the Jira payload including title, description, status, assignee, and any new comments. It intelligently handles different types of updates to determine what needs synchronization.

3. Notion Database Update

Based on the Jira event type, the workflow either creates a new Notion page, updates an existing one, or deletes the corresponding page. All operations maintain referential integrity between the systems.

4. Comment Synchronization

For issue updates containing new comments, the workflow appends these to the Notion page while preserving the original author and timestamp information from Jira.
