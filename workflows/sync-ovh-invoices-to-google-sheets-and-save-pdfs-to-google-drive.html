What This Workflow Does

This automation solves a common pain point for businesses using OVH cloud services: manually downloading invoices from the OVH portal and organizing them for accounting. Currently, OVH sends invoice notifications as links, requiring you to log in (often with 2FA) each time you need a copy.

The workflow automatically connects to the OVH API using OAuth2, retrieves your latest invoices, extracts key data like invoice numbers, dates, amounts, and services, then writes this information to a Google Sheets spreadsheet. Simultaneously, it downloads the PDF invoices and saves them to organized yearly folders in your Google Drive.

You get a complete, automated invoice management system that runs on schedule, ensuring you always have up-to-date financial records without manual intervention.

How It Works

Step 1: Authenticate & Fetch Invoices

The workflow starts by authenticating with the OVH API using OAuth2 tokens. It queries for invoices within your specified date range (configurable from daily to all-time). The API returns invoice metadata including IDs, dates, totals, and PDF download URLs.

Step 2: Process Invoice Data

Each invoice's data is parsed and structured into a consistent format. Key fields like invoice number, issue date, due date, amount with tax, VAT amount, and service descriptions are extracted for reporting.

Step 3: Update Google Sheets

The structured data is appended to your Google Sheets spreadsheet. The template includes columns for all relevant invoice information, creating a searchable database of your OVH expenses that you can sort, filter, and analyze.

Step 4: Archive PDFs to Google Drive

The workflow downloads each invoice PDF from OVH and saves it to a Google Drive folder. By default, it organizes invoices by year (e.g., "OVH Invoices/2025/"), making document retrieval simple. File naming includes invoice numbers and dates.

Step 5: Error Handling & Logging

Built-in error handling manages API rate limits, authentication issues, and connectivity problems. Failed operations are logged for review, and the workflow can continue processing other invoices if one fails.

What You'll Need

  1. OVH Account with active services (domains, hosting, cloud, etc.)
  2. OVH API Credentials from the OVH API console with invoice read permissions
  3. Google Account with access to Google Sheets and Google Drive
  4. n8n Instance (cloud or self-hosted) to run the workflow
  5. Google Sheets Template (provided) or your own spreadsheet with appropriate columns

Pro tip: Set up a dedicated Google Service Account for this automation rather than using personal credentials. This improves security and allows you to control folder permissions precisely.

Quick Setup Guide

1. Download the template using the button above and import it into your n8n instance.

2. Duplicate the Google Sheets template from the provided link in the workflow description to your Google Drive.

3. Get OVH API credentials by logging into the OVH API console, authorizing access, and copying your OAuth2 token.

4. Configure the HTTP Request nodes with your OVH authorization header (Bearer token).

5. Set up Google Sheets and Drive connections in n8n using OAuth2.

6. Adjust the date range in the "Query Latest OVH Invoices" node to match your needs (start with last 30 days).

7. Test the workflow manually to ensure all connections work, then schedule it to run daily or weekly.
