What This Workflow Does

This automation solves a critical bottleneck in product development: the manual handoff between product planning and engineering execution. When product managers create new features in Productboard, engineering teams often don't see them until the next planning meeting, causing delays and misalignment.

The workflow automatically detects new Productboard features, transforms them into properly formatted Linear issues, and sends instant Telegram notifications to your team. This creates a seamless pipeline where product decisions immediately become actionable engineering tickets, eliminating copy-paste work and ensuring nothing gets lost in translation.

Beyond simple sync, the workflow includes intelligent filtering to prevent duplicates, data cleaning to ensure consistent formatting, and fail-safe notifications so your team knows exactly when new work arrives. It turns what was typically a weekly manual process into a real-time automated system.

How It Works

The automation follows a logical five-step process that mimics how a product manager would manually handle this task, but with perfect consistency and instant execution.

1. Scheduled Trigger & Data Fetch

The workflow starts on a schedule you define (hourly, daily, or real-time). It connects to the Productboard API using secure authentication and retrieves all recent features. This replaces manual checking of Productboard for updates.

2. Data Transformation & Cleaning

Raw Productboard data often contains HTML formatting, inconsistent date formats, and nested information. A custom code node strips unnecessary formatting, extracts key fields (name, description, priority, owner), and structures everything for Linear's requirements.

3. Intelligent Filtering

An "If" node compares creation dates with your threshold (today, this week, or custom). Only genuinely new features proceed, preventing duplicate tickets in Linear. This logic mimics how a human would decide what's actually new.

4. Linear Issue Creation

The cleaned data creates a new Linear issue with proper title, description, labels, assignee (mapped from Productboard owner), and priority. The workflow maintains a link back to the original Productboard item for full traceability.

5. Team Notification

Finally, a Telegram message sends to your designated channel or group confirming the sync completed successfully, listing the new features created. If anything fails, error notifications go to a separate alert channel.

What You'll Need

  1. Productboard account with API access enabled (Admin permissions required)
  2. Linear workspace with API key generated (Team settings → API)
  3. Telegram bot token and channel/group ID (create via @BotFather)
  4. n8n instance (cloud or self-hosted) running version 1.0+
  5. Basic understanding of how to import JSON workflows and configure credentials in n8n

Pro tip: Before going live, test with a dedicated "Sandbox" Linear project and Telegram test channel. This lets you verify field mappings and formatting without affecting your production workflow.

Quick Setup Guide

Follow these steps to implement this automation in under 15 minutes:

  1. Download the template using the button above and save the JSON file locally.
  2. Import into n8n by clicking "+ Workflow" → "Import from file" and selecting the downloaded JSON.
  3. Configure credentials for Productboard, Linear, and Telegram in n8n's credentials management.
  4. Adjust the schedule trigger to match your team's review frequency (daily at 9 AM works for most).
  5. Map field values between Productboard statuses and Linear labels/priorities if needed.
  6. Test with one feature by temporarily modifying the filter to include older items, then run manually.
  7. Activate the workflow once testing confirms proper sync and notifications.
