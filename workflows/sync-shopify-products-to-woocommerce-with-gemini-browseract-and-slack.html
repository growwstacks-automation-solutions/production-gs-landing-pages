What This Workflow Does

Managing products across multiple e-commerce platforms is a time-consuming and error-prone task. This automation solves that by creating a seamless, one-way sync from your Shopify store to your WooCommerce store, supercharged with AI.

The workflow doesn't just copy data—it intelligently enhances it. It fetches your Shopify catalog, uses AI to determine the best external source for additional product information (like Amazon for physical goods or G2 for software), scrapes that data, and then crafts a high-converting product description before pushing the complete, enriched product to WooCommerce.

This eliminates hours of manual research, copywriting, and data entry. It ensures your WooCommerce listings are as detailed and compelling as your Shopify ones, improving SEO and conversion rates while maintaining perfect inventory and data consistency across your sales channels.

How It Works

The automation follows a logical, multi-step process to ensure data is accurately transferred and enhanced.

1. Fetch & Initiate

The workflow is triggered, typically on a schedule (e.g., daily). It connects to your Shopify store via its API and retrieves a list of products. You can configure it to fetch all products or filter by collection, tag, or recent updates.

2. AI-Powered Research & Classification

For each product, an AI agent analyzes the title and category. It classifies the product type (e.g., "physical gadget," "software subscription," "digital download") and determines the most relevant external source to scrape for enriched data, such as Amazon product pages or software review sites.

3. Data Scraping & Collection

Using the BrowserAct node, the workflow executes a background browser task to navigate to the identified source URL. It scrapes key information like specifications, feature lists, customer reviews, and high-quality images that aren't in your original Shopify listing.

4. AI Content Synthesis

A second AI agent, acting as a professional e-commerce copywriter, takes the original Shopify description and the newly scraped data. It synthesizes them into a single, compelling, and SEO-optimized HTML description for WooCommerce. It also generates a logical SKU and formats image galleries.

5. WooCommerce Product Creation/Update

The workflow checks if a product with the same SKU already exists in WooCommerce. If not, it creates a new product with all the enriched data—title, description, price, categories, tags, images, and inventory status. If the product exists, it can be configured to update specific fields.

6. Error Handling & Notification

If any step fails (e.g., API error, scraping timeout, creation failure), the error is caught. The workflow then sends a detailed alert to a designated Slack channel, allowing your team to intervene quickly without losing track of the issue.

  3. WooCommerce Store Access: API keys (Consumer Key and Secret) from your WooCommerce site's settings.
  4. Google Gemini API Key: For the AI classification and content generation steps.
  5. BrowserAct Account & API Key: To power the web scraping steps.
  6. Slack Workspace: A channel to receive error notifications and success logs.

Quick Setup Guide

Follow these steps to get your sync automation running in under 30 minutes.

  1. Download & Import: Download the JSON template file from this page. In your n8n dashboard, go to "Workflows" and use the "Import from file" option to load it.
  2. Configure Credentials: In the workflow canvas, click on each node (Shopify, WooCommerce, Gemini, BrowserAct, Slack) and add your respective API keys and connection details in the credentials settings.
  3. Set Up BrowserAct Template: Ensure you have the "Shopify to WooCommerce Multi-Store Sync" template saved in your BrowserAct account, and note its ID for the node configuration.
  4. Test the Trigger: Set the initial Shopify "Get Many Products" node to fetch a limited number (like 1 or 2) for your first test run. Execute the workflow manually.
  5. Review & Adjust: Check the newly created product in WooCommerce. You may want to tweak the AI agent prompts in the "Create Product" node to better match your brand voice.
  6. Schedule & Deploy: Once satisfied, set the trigger to run on a schedule (e.g., every 6 hours) and activate the workflow.
