What This Workflow Does

If you're like most business owners and teams, you're probably using multiple productivity tools. Todoist for quick task capture and daily execution, Notion for detailed project planning and documentation. The problem? Tasks live in separate silos. Updates in one system don't reflect in the other, leading to missed deadlines, duplicated work, and team confusion.

This n8n automation solves that by creating a bidirectional sync between Todoist and Notion. When you create or update a task in either platform, it automatically updates in the other. The workflow uses Redis as a smart locking mechanism to prevent infinite loops, ensuring updates flow smoothly in both directions without creating duplicate or conflicting entries.

Beyond basic sync, this template treats Notion as your "source of truth" while maintaining Todoist's simplicity for daily execution. It preserves task relationships, handles priority mapping, and even includes safety features like marking tasks as "Obsolete" instead of deleting them—protecting your data while keeping both systems clean.

How It Works

The automation consists of several interconnected workflows that work together to maintain perfect synchronization between your task management systems.

1. Webhook Trigger System

Both Todoist and Notion send real-time webhooks when tasks change. These webhooks trigger the sync workflows instantly, ensuring changes propagate within seconds rather than hours or days. The system listens for creates, updates, and deletions in both platforms.

2. Redis Lock Mechanism

When a task update comes in from Todoist, Redis temporarily "locks" that task ID for 80 seconds. If the same update triggers a Notion webhook during this window, the system recognizes it's already processing that change and skips creating another update. This prevents the infinite loop problem common in bidirectional sync setups.

3. Intelligent Field Mapping

The workflow maps task properties between systems: Todoist priorities become Notion statuses, due dates sync accurately, and task names transfer cleanly. Todoist task IDs are stored in Notion properties to maintain the relationship, so the systems always know which tasks correspond to each other.

4. Daily Consistency Check

A separate workflow runs daily to compare all tasks between both systems and fix any inconsistencies. This catches any missed webhooks or edge cases, ensuring your data remains perfectly synchronized even if occasional webhook events get lost.
