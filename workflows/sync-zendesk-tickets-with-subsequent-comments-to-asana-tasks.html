Sync Zendesk tickets with subsequent comments to Asana tasks

Automatically create Asana tasks from Zendesk tickets and keep all follow-up comments synchronized between both platforms

What This Workflow Does

This automation bridges the gap between customer support (Zendesk) and project management (Asana) by automatically converting new support tickets into actionable tasks. When a customer submits a ticket in Zendesk, the workflow instantly creates a corresponding task in Asana with all relevant details. As the ticket progresses, every customer or agent comment gets added to the Asana task, maintaining complete context for all teams involved.

The solution eliminates manual copying of information between systems, reducing errors and saving significant time for support teams. It's particularly valuable for product-related support issues that require collaboration between customer support and product/engineering teams who primarily work in Asana.

How It Works

1. Trigger: New Zendesk ticket created

The workflow monitors your Zendesk instance for new tickets using webhooks or API polling. When a ticket matching your criteria appears, the workflow captures all relevant details including subject, description, requester information, and any custom fields.

2. Create Asana task with ticket details

The system formats the Zendesk ticket information into a properly structured Asana task. You can customize which fields transfer and how they're organized. The task typically includes a link back to the original Zendesk ticket for reference.

3. Monitor for new comments

After the initial task creation, the workflow continues monitoring the Zendesk ticket for new comments from both customers and support agents. Each new comment triggers an update to the corresponding Asana task.

4. Sync comments to Asana

New comments are added to the Asana task with attribution (who posted it) and timestamp information. The workflow maintains proper threading and can preserve formatting depending on your configuration.

Pro tip: Configure the workflow to only sync tickets with specific tags or from certain requesters to avoid cluttering Asana with routine support tickets.
