What This Workflow Does

This template solves the problem of inaccessible information trapped in PDF documents. Instead of manually searching through lengthy reports, manuals, or contracts, users can simply ask questions in a Telegram chat and get instant AI-generated answers based on the document's content.

The workflow automatically detects when a PDF is sent to your Telegram bot, processes it with AI embeddings, stores the knowledge in a vector database, and then responds to subsequent text questions with relevant, context-aware answers. It turns static documents into interactive knowledge bases.

This automation eliminates hours of manual document searching for support teams, educators, and businesses that distribute complex information. It provides 24/7 self-service access to document content, reducing repetitive queries and improving information accessibility.

How It Works

Step 1: PDF Detection & Processing

The Telegram trigger node monitors your bot for incoming messages. When a PDF document is detected, the workflow extracts the file, splits it into manageable chunks using a text splitter, and converts each chunk into numerical embeddings using an AI model.

Step 2: Vector Storage

These embeddings are stored in a vector database (like Pinecone), creating a searchable knowledge index of the document's content. This allows for semantic search—finding text that means the same thing, not just matching keywords.

Step 3: Question Handling

When users send text questions (without a document), the workflow searches the vector database for the most relevant text chunks related to their query. It then passes both the question and the retrieved context to an AI chat model to generate a coherent, accurate answer.

Step 4: Response Delivery

The AI-generated answer is sent back to the user through Telegram, completing the conversational loop. The entire process happens in seconds, making document interrogation feel like chatting with an expert.

What You'll Need

  1. A Telegram bot token (created via BotFather)
  2. Access to an AI model API (OpenAI, Groq, or similar)
  3. A vector database account (Pinecone, Supabase, Qdrant, or PostgreSQL with vector extension)
  4. An n8n instance (cloud or self-hosted)
  5. Basic understanding of API key configuration

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n instance.
  2. Configure Telegram: Create a bot via BotFather, obtain your token, and add it to the Telegram trigger node.
  3. Set up AI services: Add your API keys for the AI model (Groq/OpenAI) and vector database (Pinecone).
  4. Test with a PDF: Send a PDF to your bot, then ask questions about its content.
  5. Customize as needed: Swap AI models, change vector databases, or adjust text splitting parameters.

Pro tip: Start with non-sensitive documents for testing. Use shorter PDFs (under 50 pages) initially to optimize chunking settings before scaling to larger documents.
