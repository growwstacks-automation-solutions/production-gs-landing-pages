What This Workflow Does

This automation solves a common pain point for social media managers, marketing agencies, and content creators: manually checking TikTok analytics every day. Instead of logging into multiple accounts and copying numbers into spreadsheets, this n8n workflow automatically fetches TikTok account performance data using the TikTok API via RapidAPI and logs it directly into Google Sheets.

The workflow runs on a schedule you set (daily, weekly, etc.), pulls metrics like total videos, views, and engagement for each username in your tracking list, and organizes everything into a clean, timestamped spreadsheet. This creates a historical record of performance that's perfect for reporting, identifying trends, and proving ROI to clients or stakeholders.

Beyond basic tracking, this template serves as a foundation for more advanced analytics. You can extend it to calculate growth rates, compare performance across accounts, trigger alerts when metrics dip, or even feed the data into visualization tools like Google Data Studio for real-time dashboards.

How It Works

The automation follows a logical sequence to ensure reliable, hands-off operation.

1. Scheduled Trigger

The workflow begins with a Schedule Trigger node set to run at your preferred interval—daily at 9 AM, weekly on Monday, or any custom schedule. This eliminates manual initiation and ensures consistent data collection.

2. Read TikTok Usernames from Google Sheets

A Google Sheets node connects to your "Master List" spreadsheet, reading all TikTok usernames you want to monitor. This centralizes management—simply add or remove usernames from the sheet, and the automation adjusts automatically.

3. Loop Through Each Account

A Split In Batches or Loop node processes each username individually. This ensures clean, separate API calls and prevents data from different accounts from getting mixed up, even if one request fails.

4. Fetch Analytics via TikTok API

For each username, an HTTP Request node calls the TikTok API endpoint (hosted on RapidAPI) to retrieve current metrics. The request includes parameters like username and region, returning structured JSON data with video counts, view totals, and other engagement stats.

5. Transform and Structure Data

Function or Set nodes clean and format the API response, extracting only the relevant metrics (total_videos, total_views, average_views) and adding metadata like the fetch date, category tags, and account priority from your master list.

6. Append Results to Google Sheets

The formatted data for each account is appended as a new row in a dedicated "Analytics Log" Google Sheet. This creates a timestamped history, perfect for creating trend charts and performance reports.

7. Error Handling & Logging

If an API call fails (e.g., username not found, rate limit hit), the workflow catches the error, logs it to a separate "Error Log" sheet with details, and continues processing the next account without stopping the entire automation.

What You'll Need

  1. A free or paid n8n instance (cloud or self-hosted).
  2. A Google Cloud Project with the Google Sheets API enabled, and service account credentials (JSON key file) to allow n8n to read/write to your spreadsheets.
  3. A RapidAPI account and subscription to a TikTok API provider (like "TikTok API42" used in this template). You'll need an API key.
  4. A Google Sheet set up with at least two tabs: one for your list of TikTok usernames (with columns like username, category, priority) and one for the analytics log output.
  5. Basic familiarity with n8n's interface for pasting credentials and adjusting node settings.

Pro tip: Use a separate Google Sheet for your "master list" of usernames. This makes it easy to update tracked accounts without touching the workflow logic. You can also add columns for client name, campaign ID, or target metrics to enrich your reports.

Quick Setup Guide

Follow these steps to get this automation running in your n8n instance in under 15 minutes.

  1. Download the template using the button above and import it into your n8n workspace.
  2. Create your Google Sheets. Make one sheet with a "Usernames" tab (columns: username, category, priority) and another with an "Analytics Log" tab (columns: timestamp, username, total_videos, total_views, average_views, category).
  3. Configure the Google Sheets nodes. In the workflow, update the two Google Sheets nodes with your spreadsheet IDs and tab names. Add your Google service account JSON credentials in n8n's credentials manager.
  4. Set up the TikTok API connection. Sign up for the relevant TikTok API on RapidAPI, get your API key, and enter it into the HTTP Request node's headers (usually as "X-RapidAPI-Key").
  5. Adjust the schedule. Modify the Schedule Trigger node to run at your desired frequency (e.g., every day at 9 AM).
  6. Test and activate. Click "Execute Workflow" to test with one username. Check that data appears correctly in your Analytics Log sheet. Then toggle the workflow to "Active".
