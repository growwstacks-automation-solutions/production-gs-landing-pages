What This Workflow Does

Todoist is excellent for managing daily tasks, but it lacks a built-in way to review what you've accomplished over time. This automation solves that problem by connecting directly to Todoist's API, retrieving all your completed tasks from the past week, and sending them to you in a clean, organized email every Friday afternoon.

The workflow transforms raw task data into meaningful insights. Instead of just listing tasks, it groups them by the day they were completed, excludes projects you don't want to track (like grocery lists), and formats everything in an easy-to-read summary that helps you celebrate wins and identify productivity patterns.

For professionals, freelancers, and teams, this automated review creates accountability, provides documentation of work completed, and serves as a valuable planning tool for the week ahead—all without any manual effort after initial setup.

How It Works

1. Scheduled Trigger

Every Friday at 4 PM, the workflow automatically activates. You can also run it manually anytime you want an instant review. The schedule trigger ensures consistency without requiring you to remember to generate the report.

2. Todoist API Connection

The workflow uses Todoist's REST API to fetch all tasks completed in the last 7 days. Unlike basic integrations, this direct API access provides complete data including project names, due dates (if any), completion timestamps, and task metadata.

3. Data Processing & Filtering

A Code node processes the raw data: it groups tasks by completion date, excludes specific projects you've configured to ignore, calculates basic statistics (total tasks, tasks per day), and formats everything for email presentation.

4. Email Composition & Delivery

The Send Email node creates a professional HTML email with your completed tasks organized by day. It includes a summary section, visual separators between days, and clear formatting that works across all email clients. The email is sent via your configured SMTP service.

What You'll Need

  1. Todoist Account: Any paid Todoist plan (Premium, Business, or Pro) that provides API access. Free accounts have limited API capabilities.
  2. Todoist API Token: Available in your Todoist settings under Integrations → Developer. This authenticates the workflow with Todoist's servers.
  3. Email Service Credentials: SMTP settings from your email provider (Gmail, Outlook, SendGrid, etc.) or a dedicated email service integration in n8n.
  4. n8n Instance: Either n8n.cloud (hosted) or self-hosted n8n. The workflow works in both environments with minimal configuration.
  5. Basic n8n Knowledge: Understanding how to import workflows, create credentials, and test automations will ensure smooth setup.

Quick Setup Guide

  1. Download the template using the button above and import it into your n8n instance via the "Import from File" option.
  2. Create Todoist credentials in n8n: Add a new credential of type "HTTP Request (Auth)", select "Header Auth", and paste your Todoist API token with the format "Bearer YOUR_TOKEN".
  3. Configure email credentials: Set up SMTP credentials in n8n with your email provider's server details, or use a pre-built email service node if available.
  4. Test the workflow manually by clicking "Execute Workflow". Check your email inbox for the test summary and verify all tasks appear correctly.
  5. Activate the schedule by enabling the Schedule Trigger node. The default is Friday 4 PM, but you can adjust this to any day and time that suits your review rhythm.
  6. Customize project filtering by editing the Code node if you want to exclude specific Todoist projects from your weekly summary.
