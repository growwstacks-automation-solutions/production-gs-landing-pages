Track AI Model Executions with LangFuse Observability

Automatically log AI executions, token usage, latency, and insights into LangFuse for better performance monitoring and cost optimization.

Download Template JSON · n8n compatible · Free

What This Workflow Does

When your business uses AI models (like GPT-4, Claude, or custom LLMs), you need visibility into what's happening. How many tokens are being consumed? What's the latency? Which models perform best for specific tasks? Without tracking, you're flying blind—overspending on inefficient prompts, missing performance issues, and lacking data to make informed decisions.

This automation solves that by automatically capturing every AI execution and sending structured observability data to LangFuse. After an AI run completes, this workflow triggers, waits for metrics to settle, extracts execution details, normalizes the data, and posts it to LangFuse's ingestion API. You get a centralized dashboard showing token usage, latency, model performance, and cost trends—all without manual logging.

The result: you can optimize AI costs by identifying inefficient prompts, improve reliability by detecting failing models, and make strategic decisions based on actual performance data rather than guesses.

How It Works

Step 1: Trigger After AI Execution

The workflow is triggered by another workflow once an AI run finishes. It receives the execution ID as input, ensuring each AI call is tracked individually.

Step 2: Remove Duplicates & Wait for Data

A duplicate check ensures each execution is processed only once. Then a short wait (60-80 seconds) allows token totals and per-step metrics to become available in the execution data.

Step 3: Fetch Execution Metadata

The workflow retrieves the complete execution details—workflow name, execution time, total tokens consumed, and step-by-step breakdown.

Step 4: Structure the Data

A code node normalizes the raw execution into a clean array of "perModelRuns" containing model name, input/output previews, token counts, and latency measurements.

Step 5: Loop Through Each Run Step

The split-out and loop nodes iterate through each AI step in the execution, preparing individual observations for LangFuse.

Step 6: Prepare LangFuse Batch

For each step, a code node builds a JSON batch containing a trace-create (with stable ID trace-[executionId] grouped into session-[workflowId]) and a generation-create with model details, input/output, usage metrics, and timing data.

Step 7: Send to LangFuse

An HTTP request posts the batch to LangFuse's public ingestion API using Basic Auth with your public and secret keys. Optional short waits between sends prevent rate limiting.

What You'll Need

  1. A LangFuse Cloud project (or self-hosted instance) with API keys (public key and secret key)
  2. An n8n instance (cloud or self-hosted) with HTTP Request node available
  3. Existing workflows that call AI models and can trigger this observability workflow
  4. Basic understanding of how your AI executions are structured (workflow IDs, execution IDs)

Pro tip: Set up LangFuse dashboards and alerts before implementing this automation. Define what metrics matter most—cost per user, latency thresholds, error rates—so you know what to track.

Quick Setup Guide

  1. Download the template JSON file and import it into your n8n instance.
  2. Create a credential in n8n for LangFuse API: Generic Credential Type with Basic Auth. Username = your LangFuse public key, Password = your secret key.
  3. Update the HTTP Request node to use your LangFuse endpoint (default: https://cloud.langfuse.com/api/public/ingestion).
  4. Connect the trigger to your existing AI workflows—pass the execution ID as input parameter.
  5. Test with a single AI execution and verify data appears in your LangFuse dashboard.
  6. Adjust wait times if needed (some AI providers report token usage slower than others).
  7. Scale up by connecting all your AI workflows to this observability layer.
