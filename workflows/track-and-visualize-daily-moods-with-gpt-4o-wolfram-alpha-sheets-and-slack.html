Track and Visualize Daily Moods with AI & Automation

Transform subjective feelings into objective data. This workflow uses GPT-4o to analyze mood text, Wolfram Alpha to generate graphs, Google Sheets for logging, and Slack for sharing insights—fully automated.

What This Workflow Does

Manual mood tracking is inconsistent, time-consuming, and lacks analytical depth. This automation solves that by creating a seamless pipeline from raw emotional input to visualized insights. You simply describe your mood in a sentence. The system quantifies it, generates a graph, stores the data, and provides feedback—all without manual steps.

It’s designed for individuals seeking structured self-reflection, teams monitoring collective sentiment, or businesses wanting to analyze customer emotional feedback. The workflow turns qualitative experiences into quantifiable metrics, enabling trend analysis, proactive insights, and data-driven well-being decisions.

How It Works

Step 1: Mood Input & AI Analysis

A webhook or manual trigger captures your mood description text. GPT-4o analyzes the text, scoring it on two dimensions: valence (positivity) and energy (activity level). This creates a structured data point from unstructured language.

Step 2: Graph Generation via Wolfram Alpha

The valence and energy scores are sent to Wolfram Alpha as a query. Wolfram Alpha instantly generates a clean, visual graph plotting the emotional state, providing an immediate visual representation.

Step 3: Personalized Feedback Generation

A second AI agent reviews the analysis and graph, crafting a short, encouraging feedback message in your preferred language (e.g., Japanese). This adds a human-like, supportive layer to the raw data.

Step 4: Data Logging to Google Sheets

All components—user ID, mood text, scores, timestamp, Wolfram query, and feedback—are appended as a new row in a Google Sheet. This creates a permanent, queryable historical record.

Step 5: History Visualization & Slack Sharing

A separate endpoint can retrieve a user’s historical mood data from Sheets, format it for time-series plotting, send it to Wolfram Alpha for a trend line graph, and post the resulting visualization directly to a Slack channel for team awareness or personal accountability.

What You'll Need

  1. n8n Instance: A running n8n environment (cloud or self-hosted) to import and execute the workflow.
  2. OpenAI API Key: Credentials for GPT-4o access to power the sentiment analysis and feedback generation nodes.
  3. Google Sheets Setup: A Google Sheet with the correct header columns (userId, moodText, valence, energy, createdAt, wolframQuery, feedback) and its Sheet ID.
  4. Slack Workspace & Webhook: A Slack channel and webhook URL for posting historical graph visualizations.
  5. Wolfram Alpha App ID: A free App ID from the Wolfram|Alpha Developer Portal to enable graph generation API calls.

Quick Setup Guide

  1. Download & Import: Download the JSON template and import it into your n8n workspace.
  2. Configure Credentials: Add your OpenAI API key, Google Sheets authentication, and Slack webhook URL in the respective nodes.
  3. Set Wolfram Alpha App ID: Paste your free App ID into the two HTTP Request nodes that call Wolfram Alpha.
  4. Prepare Google Sheet: Create your Sheet, ensure headers match exactly, and paste the Sheet ID into the Google Sheets nodes.
  5. Test with Manual Trigger: Use the manual trigger node, modify the sample mood text, and execute the workflow to verify the entire pipeline.
  6. Activate Webhooks: Enable the webhook nodes for production use, providing the /mood and /history endpoints for external inputs.

Pro tip: Use a dedicated Slack channel for mood graphs to create a visual timeline that teams can review in meetings or individuals can reflect on weekly.
