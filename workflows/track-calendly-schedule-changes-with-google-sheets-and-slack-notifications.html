What This Workflow Does

Manual appointment tracking is a silent productivity killer. Every time someone books or cancels a meeting in Calendly, someone on your team has to manually update a spreadsheet, send notifications, or log the change. This workflow eliminates that entire process by creating an automated hub that captures every Calendly event in real-time.

The system listens for booking confirmations and cancellations via Calendly's webhooks, intelligently routes each event, extracts all relevant data (including guest information and custom questions), logs everything to organized Google Sheets tabs, and sends formatted Slack alerts to keep your team instantly informed. It even categorizes cancellations by urgency so you can prioritize follow-ups.

How It Works

The automation follows a logical flow that ensures no appointment data slips through the cracks.

1. Calendly Webhook Trigger

The workflow starts with a Calendly webhook trigger that listens for two key events: invitee.created (new bookings) and invitee.canceled (cancellations). When either occurs, the webhook instantly sends the complete event data to your n8n workflow without any polling delay.

2. Intelligent Event Routing

A Switch node examines the incoming event type and routes bookings down one path and cancellations down another. This separation allows each type to be processed with specialized logic tailored to its specific requirements and data structure.

3. Booking Processing Path

For new bookings, the workflow extracts over 20 data points including invitee details, event timing, meeting links, guest information, and custom form responses. It calculates derived fields like days until the event and flags urgent bookings (same-day or next-day) for priority handling.

4. Cancellation Processing Path

For cancellations, the system captures the reason, who canceled, and timing relative to the scheduled event. It categorizes cancellations into three types: Last Minute (within 24 hours), Standard (upcoming), and Past Event (already occurred). This categorization enables appropriate follow-up strategies.

5. Data Logging & Notification

All processed data flows to dedicated Google Sheets tabs—one for bookings, another for cancellations—creating a searchable historical record. Simultaneously, formatted Slack notifications deliver key information to your team channels with appropriate urgency indicators and quick-action links.

  4. n8n instance (self-hosted or cloud) to run the workflow

Pro tip: Before going live, create test bookings and cancellations in Calendly to verify the entire flow works correctly. Check that data appears in Google Sheets, Slack notifications arrive, and all fields populate as expected.

Quick Setup Guide

You can have this automation running in under 30 minutes by following these steps:

  1. Download and import the template JSON file into your n8n instance using the import function.
  2. Configure Calendly credentials in the webhook trigger node using OAuth2 or a personal access token from developer.calendly.com.
  3. Set up Google Sheets by creating a new spreadsheet with "Bookings" and "Cancellations" tabs, then connect using Google OAuth2 credentials.
  4. Connect Slack by creating a Slack app with chat:write permissions, installing it to your workspace, and adding credentials to both Slack nodes.
  5. Test thoroughly by creating and canceling test appointments in Calendly, then verifying data flows to Sheets and Slack correctly.
  6. Activate the workflow and let it run continuously to process all future Calendly events automatically.
