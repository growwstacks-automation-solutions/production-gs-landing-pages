What This Workflow Does

Manual price checking across multiple competitor websites is one of the most time-consuming tasks in e-commerce and retail. This automation solves that problem by continuously monitoring specific product pages for price changes, extracting the current pricing data, and alerting you immediately when prices drop or increase.

The workflow is particularly valuable for businesses practicing dynamic pricing, monitoring minimum advertised price (MAP) compliance, conducting competitive analysis, or simply wanting to ensure they're offering the best value to their customers. Instead of spending hours each week manually checking prices, this automation runs silently in the background, providing real-time market intelligence.

When a significant price change is detected, the system can trigger multiple actions—sending email alerts, updating internal databases, creating tasks for your team, or even automatically adjusting your own pricing in response to market movements.

How It Works

Step 1: Scheduled Monitoring

A Cron node triggers the workflow at regular intervals you configure—whether hourly, daily, or multiple times per day. This ensures continuous monitoring without manual intervention.

Step 2: Web Data Extraction

HTTP Request nodes fetch the latest product pages from your target websites. HTML Extract nodes then parse the page content using CSS selectors you specify to locate and extract the current price information accurately.

Step 3: Data Processing & Comparison

Function Item nodes process the extracted data, convert it to consistent formats, and compare it against previously stored prices. The system intelligently identifies whether a change is significant enough to warrant notification.

Step 4: Conditional Notifications

IF nodes route the workflow based on the price change analysis. If a meaningful change is detected, Send Email nodes dispatch alerts to your team. The system can also log all data for historical analysis and trend spotting.

Step 5: Data Storage & Maintenance

Execute Command and Write Binary File nodes manage the storage of price history, ensuring data persistence between workflow runs. This creates a valuable historical database for pricing trend analysis.

Pro tip: Start by monitoring 3-5 key competitor products that directly impact your sales. Once the automation is running smoothly, gradually expand to track dozens or hundreds of products across multiple competitors.

Quick Setup Guide

Importing and configuring this template takes about 15-20 minutes. Follow these steps to get your price tracking automation running:

  1. Download and import the JSON template into your n8n instance using the import workflow function.
  2. Configure the Cron node to set your preferred monitoring frequency (start with once daily for testing).
  3. Update the Function Item node labeled "Change me" with your target URLs, CSS selectors, and product identifiers.
  4. Set up email notifications by configuring the Send Email nodes with your SMTP credentials or email service connection.
  5. Test with one product first to ensure the extraction works correctly before adding more monitoring targets.
  6. Activate the workflow and verify you receive notifications when test price changes occur.

Important: Respect website terms of service and implement reasonable delays between requests to avoid overloading target servers. Consider using official APIs when available instead of scraping.
