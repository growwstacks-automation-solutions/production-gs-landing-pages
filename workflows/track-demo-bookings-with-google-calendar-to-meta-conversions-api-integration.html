What This Workflow Does

If you're running Meta Ads to generate sales demos, you know the frustration: someone clicks your ad, books a demo in your calendar, but Meta has no idea that conversion happened. Your ad reports show zero results, your cost-per-lead looks inflated, and you're making budget decisions with blind spots.

This n8n workflow solves that problem automatically. Every time a new "Schedule a Demo" event appears in your Google Calendar, it captures the details and sends them directly to Meta's Conversions API. This gives Meta's algorithm the conversion data it needs to optimize your ads for actual booked demos—not just link clicks or form submissions.

The result? More accurate reporting, better audience targeting, lower customer acquisition costs, and ultimately, more demos booked from your ad spend. It bridges the gap between your online ads and offline conversions that happen in your calendar.

How It Works

The automation follows a clean, logical sequence to ensure reliable data flow from calendar to ad platform.

1. Google Calendar Trigger

The workflow monitors your connected Google Calendar for new events. It uses OAuth2 for secure, ongoing access without storing passwords. You can specify which calendar to watch and set the polling interval based on your booking volume.

2. Event Filtering

Not every calendar event is a demo booking. The workflow filters events to only process those with specific keywords in the title—like "Demo," "Schedule a Call," or "Discovery Meeting." This prevents false positives from internal meetings or personal appointments.

3. Data Extraction & Formatting

For each qualifying event, the workflow extracts critical information: attendee email addresses, event start/end times, event title, and any custom parameters you've added to the calendar event description. It structures this data according to Meta's Conversions API specifications.

4. Meta Conversions API Submission

The formatted data is sent securely to Meta's server-side API using your Pixel ID and Access Token. This happens in real-time, ensuring your ad reports reflect the conversion within minutes of the booking being made.

5. Error Handling & Logging

The workflow includes built-in error handling. If the API call fails (due to invalid credentials, network issues, or data format problems), it logs the error details and can optionally send you an alert via email or Slack, so you never miss a conversion due to technical issues.

What You'll Need

  1. Meta Business Account Access: You'll need admin access to the Meta Business Manager where your ads are running.
  2. Meta Pixel ID: The unique identifier for your website's Meta Pixel (found in Events Manager).
  3. Meta Access Token: A long-lived system user access token with ads_management permissions for the Conversions API.
  4. Google Calendar Access: A Google account with access to the calendar where demo bookings are recorded.
  5. n8n Instance: Either n8n.cloud (hosted) or a self-hosted n8n installation.

Pro tip: Create a dedicated Google Calendar specifically for demo bookings. This makes filtering easier and prevents the workflow from processing irrelevant personal or internal meetings.

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure Google Calendar OAuth2 credentials in n8n's credentials section, granting access to the specific calendar you want to monitor.
  3. Obtain your Meta credentials: In Meta Events Manager, find your Pixel ID and generate a System User Access Token with the necessary permissions.
  4. Update the configuration node in the workflow with your Pixel ID, Access Token, and the exact event name you use for demo bookings (default is "Schedule").
  5. Test the workflow by creating a test event in your Google Calendar with the matching title. Check the workflow execution for errors and verify the event appears in Meta's Test Events tool.
  6. Activate the workflow and let it run. Monitor the first few executions to ensure everything processes smoothly.
