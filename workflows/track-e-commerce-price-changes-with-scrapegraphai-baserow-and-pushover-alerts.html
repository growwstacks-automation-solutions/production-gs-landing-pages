What This Workflow Does

This automation solves the tedious manual process of tracking competitor prices across multiple e-commerce sites. Retailers and e-commerce managers waste countless hours checking prices, only to miss crucial changes that impact their pricing strategy.

The workflow automatically scrapes product pages using ScrapeGraphAI, stores the extracted prices and product details in Baserow for historical analysis, and sends Pushover notifications when prices change beyond your defined threshold (typically 5-10%). It runs on your schedule - daily, weekly, or in real-time via webhook triggers.

How It Works

1. Product List Setup

Configure your list of products to monitor with their URLs and metadata. The workflow handles batch processing to avoid overloading sites with requests.

2. Automated Price Scraping

ScrapeGraphAI extracts current prices, product names, and availability status from each URL using customizable CSS selectors that adapt to different site structures.

3. Price Change Detection

The system compares new prices against Baserow's historical data, calculating percentage changes and identifying significant deviations.

4. Alert Notification

When thresholds are crossed, Pushover delivers mobile/desktop alerts with product details and change percentages for immediate action.

What You'll Need

  1. n8n instance (cloud or self-hosted)
  2. ScrapeGraphAI community node installed
  3. Baserow account with a prepared products table
  4. Pushover account for mobile/desktop alerts
  5. List of product URLs to monitor

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance
  2. Configure ScrapeGraphAI with your API key
  3. Connect your Baserow database with personal API token
  4. Set up Pushover with your user key and API token
  5. Add your product URLs to the initial Set node
  6. Adjust price change thresholds in the If node as needed
  7. Test with 2-3 products before full deployment

Pro tip: For high-volume monitoring, add proxy rotation in the HTTP Request nodes to avoid IP blocks from e-commerce sites.
