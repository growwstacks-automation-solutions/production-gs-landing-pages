What This Workflow Does

Community fundraising through WhatsApp groups is incredibly powerful but administratively challenging. When dozens of members contribute via MPESA, Airtel Money, or other mobile payment services, tracking each transaction manually becomes a nightmare for treasurers. Payment confirmations get buried in chat, amounts get transcribed incorrectly, and calculating running totals consumes hours each week.

This n8n automation solves these problems by creating an automated payment tracking system. When contributors forward their mobile payment confirmation SMS to the WhatsApp group, the workflow automatically extracts the payment details, logs them in a structured database, and can provide real-time totals upon request. It handles both MPESA and Airtel Money formats, distinguishes between different fundraising campaigns, and maintains an accurate, tamper-proof record of all contributions.

The system reduces administrative workload by 80-90%, eliminates errors that cause tension in community groups, and provides transparent financial tracking that builds trust among contributors. Treasurers can focus on the mission rather than financial administration.

How It Works

The automation connects WhatsApp to a payment processing engine through a simple but powerful workflow:

1. Payment Message Capture

When a payment confirmation SMS is forwarded to the designated WhatsApp number or group, Twilio captures the message and sends it to the n8n webhook. The system immediately timestamps the message and identifies the sender.

2. Payment Provider Identification

A switch node analyzes the message text using pattern recognition to determine if it's from MPESA, Airtel Money, or another service. Each provider has distinct SMS formats that the system is trained to recognize.

3. Data Extraction

Based on the identified provider, specialized parsing logic extracts the payment amount, sender details (from the SMS), transaction reference, and timestamp. This structured data is then prepared for storage.

4. Database Logging

The extracted payment information is saved to a database table (or Google Sheets/Airtable) with fields for amount, sender, campaign identifier, transaction ID, and date. This creates a searchable, permanent record.

5. Total Calculation & Response

When a treasurer requests a summary or after each payment, the system queries all payments for the current campaign, calculates the running total, and can send a WhatsApp reply with the current status.

6. Campaign Management

The workflow includes commands to start new campaigns, close existing ones, and clear test data, preventing mixing of different fundraising initiatives.

Pro tip: For production use, replace the demo data table with a proper SQL database or Google Sheets integration. This provides better data persistence, backup capabilities, and reporting options.

What You'll Need

  1. A Twilio account with WhatsApp capabilities enabled to receive messages from your fundraising group.
  2. An n8n instance (cloud or self-hosted) with webhook capabilities accessible from the internet.
  3. Basic understanding of mobile payment SMS formats for MPESA and Airtel Money to customize parsing if needed.
  4. A database or spreadsheet for storing payment records (the template uses a data table, but production systems should use SQL, Google Sheets, or Airtable).
  5. Administrative access to the WhatsApp group where fundraising occurs, or a dedicated WhatsApp number for payment confirmations.

Quick Setup Guide

  1. Download and import the JSON template into your n8n instance using the import workflow feature.
  2. Configure Twilio webhook to point to your n8n webhook URL for receiving WhatsApp messages.
  3. Set up your database by replacing the demo data table with your preferred storage solution (SQL recommended for production).
  4. Test with sample payments by forwarding MPESA and Airtel Money confirmation SMS to your WhatsApp number.
  5. Customize response messages to match your organization's tone and include relevant campaign identifiers.
  6. Train your treasurers on how to request totals and manage campaigns through simple WhatsApp commands.
  7. Run a pilot campaign with parallel manual tracking to verify accuracy before full deployment.

Pro tip: Start with a small test group before rolling out to your entire community. This allows you to iron out any parsing issues with local variations in mobile payment SMS formats.
