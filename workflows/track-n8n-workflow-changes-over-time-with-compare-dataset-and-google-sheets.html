What This Workflow Does

This automation solution provides version control for n8n workflows by systematically tracking all modifications. It solves the common problem of losing track of changes in collaborative automation environments, where multiple team members might edit the same workflows.

The template compares current workflow configurations against previous versions, logs all differences to Google Sheets, and can optionally notify relevant team members about significant changes. This creates an audit trail that helps teams maintain better control over their automation infrastructure.

How It Works

1. Workflow Snapshot Collection

The system first retrieves the current state of all workflows from your n8n instance. It captures the complete configuration including nodes, connections, and settings.

2. Change Detection

Using n8n's compare dataset node, the template identifies differences between the current workflow version and the previously stored version. It detects additions, deletions, and modifications at the node level.

3. Change Logging

All detected changes are formatted into a standardized report and appended to a Google Sheet. Each entry includes timestamps, change details, and the responsible user (when available).

Pro tip: Set up conditional formatting in Google Sheets to highlight critical changes like credential modifications or workflow activation status changes.
