What This Workflow Does

Manual SEO tracking is time-consuming and error-prone. This automation solves that by connecting the Ahrefs API directly to Google Sheets, creating a live dashboard of your website's performance metrics. Every time the workflow runs, it fetches the latest traffic estimates, backlink counts, domain authority scores, and other critical SEO data, then logs it neatly into predefined spreadsheet columns.

The system handles the entire process: from accepting a domain input via a simple web form, making the API call to Ahrefs, processing the JSON response, to appending clean, structured data into your Google Sheet. If the API call fails, it automatically sends an email alert so you never miss a data point. This transforms sporadic manual checks into consistent, scheduled reporting.

Beyond just data collection, this workflow creates a historical record. By running weekly or monthly, you build a timeline of SEO performance, enabling trend analysis, competitor benchmarking, and clear reporting for clients or management. It turns raw API data into actionable business intelligence.

How It Works

The workflow follows a logical sequence to ensure reliable data capture and error handling.

1. Form Trigger & Data Collection

The process begins with an n8n Form Trigger node, which creates a simple web form. Users or scheduled triggers submit a domain name (e.g., growwstacks.com). This input is captured and passed to the next step. You can also trigger this workflow manually from within n8n or set it on a schedule to run automatically for a predefined list of domains.

2. API Request to Ahrefs

The workflow sends the domain to the Website Traffic Checker - Ahref API via an HTTP Request node. It uses your RapidAPI key for authentication. The node is configured to handle the API's specific parameters and headers, ensuring a successful connection and data retrieval.

3. Success/Failure Logic Branch

An IF node checks the HTTP status code of the API response. If successful (status 200), the workflow proceeds to extract and parse the data. If it fails (any other status), the workflow branches to an Email node, which sends an alert with the domain and error code to a designated email address for immediate troubleshooting.

4. Data Parsing & Transformation

Two Code nodes process the successful API response. One extracts backlink-specific metrics (authority score, referring domains, total backlinks). The other extracts comprehensive traffic data (organic visits, bounce rate, pages per visit, traffic sources, etc.). These nodes transform the complex JSON into clean, column-ready arrays.

5. Google Sheets Logging

The final step appends the parsed data to two separate sheets in your Google Spreadsheet. The "Backlink Info" sheet logs authority metrics, while the "Traffic Data" sheet captures visit and engagement metrics. Each run adds a new row with a timestamp, building a historical dataset perfect for charting and analysis.

What You'll Need

  1. An n8n instance (cloud or self-hosted).
  2. A RapidAPI account with subscription to the "Website Traffic Checker - Ahref" API.
  3. Your RapidAPI key (x-rapidapi-key) for authentication.
  4. A Google Cloud Project with the Google Sheets API enabled.
  5. Google Service Account credentials (JSON key file) to connect n8n to your Sheets.
  6. A Google Spreadsheet with two sheets named "Backlink Info" and "Traffic Data" (or adjust the template to match your sheet names).
  7. Basic understanding of how to import and activate workflows in n8n.

Pro tip: Use a dedicated Google Service Account for automation, not a personal account. Share your target spreadsheet with the service account's email address as an editor. This keeps permissions secure and separate from individual user accounts.

Quick Setup Guide

Follow these steps to get your automated SEO tracker running in under 15 minutes.

  1. Download the template using the button above and import it into your n8n instance.
  2. Get your Ahrefs API key: Sign up at RapidAPI, subscribe to the "Website Traffic Checker - Ahref" API, and copy your x-rapidapi-key from the Endpoints section.
  3. Configure the HTTP Request node: Paste your API key into the Headers section under "x-rapidapi-key". Set "x-rapidapi-host" to "website-traffic-checker-ahref.p.rapidapi.com".
  4. Set up Google Sheets credentials: In n8n, create new credentials of type "Google Service Account". Upload your JSON key file from Google Cloud.
  5. Update the Google Sheets nodes: In both "Google Sheet: Backlink Info" and "Google Sheet: Traffic Data" nodes, select your credentials, enter your spreadsheet's Document ID, and specify the sheet names.
  6. Test with a single domain: Use the Form Trigger's test button or manually trigger the workflow with a domain you want to track. Check that data appears correctly in your sheets.
  7. Schedule it: Add a Schedule Trigger node to run the workflow weekly or monthly, or keep the Form Trigger for on-demand checks.
