Automate Training Feedback with Usertask & Airtable

What This Workflow Does

Manually processing training feedback is slow, error-prone, and often leads to missed opportunities for improvement or recognition. HR teams and training managers waste hours sorting through forms, deciding who needs to follow up, and trying to track actions.

This automation solves that by creating a seamless pipeline from feedback submission to resolution. It captures responses in Airtable, intelligently routes them based on sentiment, creates actionable tasks for poor ratings, and even shares positive feedback on LinkedIn to boost morale and visibility. The result is a closed-loop system that ensures every piece of feedback is acknowledged and acted upon, dramatically improving training program quality and participant satisfaction.

How It Works

The workflow is triggered by new feedback entries in an Airtable base. It then evaluates the feedback and directs it through different paths for appropriate handling.

Step 1: Feedback Capture & Trigger

The process starts when a new training feedback record is added to your Airtable. This could be from a post-training survey, a feedback form, or an integrated learning management system (LMS). The Airtable trigger node in n8n detects this new entry and passes the data (including rating, comments, and trainee details) into the workflow.

Step 2: Sentiment Analysis & Routing

A Switch node evaluates the numerical or categorical feedback rating. It categorizes the feedback into lanes: "Poor," "Fair/Good," and "Very Good/Excellent." This logical branching ensures the right action is taken for each type of response without human intervention.

Step 3: Task Creation & Notification

For poor feedback, the workflow creates an urgent task in Usertask (or a similar tool) assigned to the relevant trainer or HR manager. It simultaneously sends an email alert with the feedback details for immediate attention. For fair to good feedback, it may create a non-urgent follow-up task for later review. For excellent feedback, it documents the praise and can trigger a congratulatory email to the trainer and even post a recognition message on LinkedIn.

Step 4: Webhook for Task Results

A webhook node can be configured to listen for updates from Usertask. When a created task is marked as complete, the workflow can log the resolution, send a confirmation, or trigger further actions, closing the feedback loop entirely.

Pro tip: You can easily replace the Airtable trigger with a webhook from your survey tool (like Typeform or Google Forms) or LMS, making this workflow adaptable to almost any feedback source.

What You'll Need

  1. An n8n instance: You can use n8n.cloud (free tier available) or self-hosted n8n.
  2. Airtable base & API key: A table set up to receive training feedback, with fields for rating, comments, trainee, and session.
  3. Usertask account (or alternative): A task/project management tool where follow-up actions can be created. You can adapt this to Todoist, Asana, etc.
  4. Email sending credentials: SMTP settings (from Gmail, Outlook, etc.) or an account with a transactional email service like SendGrid.
  5. LinkedIn Developer Access (optional): If you want to enable automated recognition posts, you'll need LinkedIn API credentials.

Quick Setup Guide

  1. Import the template: Download the JSON file using the button above. In your n8n dashboard, go to "Workflows" and use the "Import from file" option.
  2. Configure the Airtable trigger: In the first node, add your Airtable API credentials and select the correct base and table. Set the trigger to watch for new records.
  3. Set up your Switch logic: Adjust the rating thresholds in the Switch node to match your feedback scale (e.g., 1-3 = Poor, 4-7 = Fair/Good, 8-10 = Excellent).
  4. Connect your action apps: In the Usertask nodes, add your credentials and set the desired project/list for tasks. In the Email node, add your SMTP details and customize the alert message.
  5. Test and activate: Add a test record to your Airtable feedback table. Watch the workflow run, check that a task is created and an email is sent (if applicable). Turn on the workflow once testing is successful.
