What This Workflow Does

This automation solves a common but time-consuming problem: manually transferring files from legacy FTP servers to modern cloud storage. Many businesses still rely on FTP for internal file sharing, client uploads, or system backups, but managing these files manually is error-prone and inefficient.

The workflow automatically connects to your FTP server, scans specified directories, downloads files, and uploads them to your Google Drive account. It handles the entire transfer process securely, providing reliable backups, effortless migration projects, and ongoing synchronization between on-premise storage and the cloud.

Whether you're phasing out an old FTP server, creating automated backups for compliance, or simply tired of manual file management, this automation eliminates the repetitive work while ensuring files are safely stored in Google Drive's accessible, searchable platform.

How It Works

The automation follows a logical sequence to ensure reliable file transfers:

1. Connect & List FTP Directory

The workflow establishes a secure connection to your FTP server using stored credentials. It scans the specified directory path and retrieves a complete list of all files and folders present.

2. Filter Files from Directories

Since FTP servers often contain both files and folders, the automation intelligently filters to process only actual files. This prevents attempting to "download" folders and ensures clean transfers.

3. Download File Contents

Each identified file is downloaded from the FTP server as binary data. The workflow maintains file names, preserves directory structures if configured, and handles various file types and sizes.

4. Upload to Google Drive

Downloaded files are securely uploaded to your specified Google Drive folder using OAuth2 authentication. The automation can create date-based subfolders, maintain organization, and provide upload confirmation.

Pro tip: Schedule this workflow to run daily or weekly for automatic backups. Combine it with email notification nodes to receive alerts when transfers complete or if any errors occur.

What You'll Need

  1. FTP Server Access: Host address, port (usually 21), username, and password. SFTP/FTPS credentials work for enhanced security.
  2. Google Drive Account: A Google account with Drive access. Business Google Workspace accounts work perfectly.
  3. n8n Instance: Self-hosted n8n or n8n.cloud account to run the workflow.
  4. Basic Folder Paths: Know the FTP directory path containing files and your target Google Drive folder ID.

Quick Setup Guide

Get this automation running in under 15 minutes:

  1. Download & Import: Download the template file and import it into your n8n instance.
  2. Configure FTP Credentials: Create FTP credentials in n8n with your server details and assign them to the FTP nodes.
  3. Set Up Google Drive: Create Google Drive OAuth2 credentials following n8n's guide and authorize the connection.
  4. Update Paths: Change the FTP directory path from the example to your actual folder. Update the Google Drive folder ID to your target location.
  5. Test & Schedule: Run the workflow manually with a few test files. Once verified, add a schedule trigger for automatic execution.
