Transform Data in Google Sheets

What This Workflow Does

Manual data management in Google Sheets is tedious, error-prone, and doesn't scale. This automation workflow solves that by handling four critical operations automatically: appending new rows, looking up specific values, updating existing records, and reading data for reporting.

Imagine you're managing a property rental spreadsheet. New listings come from a website form, you need to find all properties in a certain city, update their pricing based on market changes, and generate monthly reports. Doing this manually takes hours and risks mistakes. This workflow automates the entire process, ensuring data is always accurate and up-to-date.

The template demonstrates a complete data lifecycle within Google Sheets, but you can adapt it for countless business scenarios—customer databases, inventory tracking, lead management, financial reporting, or any spreadsheet that needs regular updates.

How It Works

Step 1: Generate or Receive Data

The workflow starts with a Set node that creates sample data (like new property listings). In your actual implementation, this could be replaced by nodes that fetch data from webhooks, forms, CRMs, databases, or other sources.

Step 2: Append New Rows to Google Sheets

The Google Sheets node takes the incoming data and adds it as a new row to your specified spreadsheet and range. You configure the spreadsheet ID and target range once, and all future data flows automatically to the correct location.

Step 3: Lookup Specific Values

Another Google Sheets node searches for specific values (like "Berlin" in our example) within a designated column. It returns all matching rows, enabling you to filter and work with subsets of your data.

Step 4: Update Existing Records

A Set node modifies the retrieved data—increasing rent prices by $100 in our example. This demonstrates how you can apply business logic or calculations before writing back to the sheet.

Step 5: Write Updates Back to Sheets

The updated records are mapped by their unique ID (or any key column) and written back to Google Sheets, ensuring only the targeted rows are modified without affecting other data.

Step 6: Read and Output Data

A final Google Sheets node reads a specified range (columns A-D in the template) and outputs the data. This can feed into reports, notifications, or other systems.

Quick Setup Guide

  1. Download the template using the button above and save the JSON file.
  2. Import into n8n: In your n8n instance, go to Workflows → Import from file and select the downloaded JSON.
  3. Configure Google Sheets connection: Click each Google Sheets node, authenticate with your Google account, and enter your spreadsheet ID and range.
  4. Adjust lookup values: In the "Google Sheets1" node, change the lookup value and column to match your data (e.g., search for "New York" in column "City").
  5. Test with sample data: Run the workflow once to verify it works with your spreadsheet structure.
  6. Replace sample data source: Swap the Set node with your actual data source (webhook, database, form, etc.).
  7. Activate the workflow: Set it to trigger on schedule, webhook, or manually based on your needs.

Pro tip: Start with a copy of your production spreadsheet for testing. Once the workflow works perfectly, switch to your live sheet. This prevents accidental data corruption during setup.
