How It Works

The automation follows a logical pipeline that transforms raw spreadsheet data into AI-ready intelligence.

1. Trigger & Data Extraction

The workflow activates when a new Excel file is available—either uploaded to a cloud folder, sent via email, or manually triggered. It connects to the specified workbook and worksheet, extracting all rows while preserving the original structure and context.

2. Data Cleaning & Normalization

Raw Excel data often contains inconsistencies: HTML tags from copied content, various date formats, mixed text cases, and missing values. This stage applies systematic cleaning: stripping formatting, standardizing dates to ISO format, normalizing text fields, and handling empty cells with default values.

3. Question/Answer Path Routing

For optimal vectorization, the workflow intelligently splits data into logical pairs—typically questions and answers, or prompts and responses. Using a switch node, it routes different data types through appropriate processing paths, preparing them for context-aware embedding generation.

4. AI Vector Generation with OpenAI

Clean text data is sent to OpenAI's embedding models (like text-embedding-3-small) which convert semantic meaning into numerical vectors. These vectors capture the conceptual relationships within your data, enabling similarity searches that understand context, not just keywords.

5. Data Enrichment & Merging

The original business data from Excel is merged with the newly generated AI vectors, creating enriched records that contain both human-readable information and machine-understandable embeddings. This preserves the full context while adding AI capabilities.

6. Storage in Supabase with Vector Indexing

Finally, enriched records are written to a Supabase PostgreSQL table with the pgvector extension enabled. The database is configured with appropriate indexes for fast similarity searches and full-text search capabilities, making the data immediately queryable for applications.

What You'll Need

  1. Supabase Account: A project with PostgreSQL and the pgvector extension enabled for vector storage and search.
  2. OpenAI API Key: Access to OpenAI's embedding models (text-embedding-3-small or similar) for generating vectors.
  3. Microsoft Excel Credentials: Authentication to access your source workbooks and worksheets (Office 365 or file path access).
  4. n8n Instance: Either self-hosted n8n or n8n.cloud account to run the automation.
  5. Sample Excel Structure: Understanding of your spreadsheet's columns and data types to configure the cleaning logic appropriately.

Pro tip: Start with a small, representative sample of your Excel data to test the cleaning logic before processing entire datasets. This helps identify edge cases and refine normalization rules.

Quick Setup Guide

Follow these steps to implement this automation in your environment:

  1. Download and Import: Download the template file and import it into your n8n instance using the workflow import function.
  2. Configure Excel Connection: Set up the Microsoft Excel node with your authentication credentials and specify the target workbook and worksheet.
  3. Set OpenAI Credentials: Add your OpenAI API key to the Embeddings node and select the appropriate model for your use case.
  4. Connect to Supabase: Configure the Supabase node with your project URL and service role key, ensuring the target table exists with vector column support.
  5. Adjust Data Cleaning: Review and modify the function nodes that handle data cleaning to match your specific Excel structure and quality issues.
  6. Test with Sample Data: Run the workflow with a small test file to verify all connections work and data flows correctly through each stage.
  7. Schedule or Trigger: Set up the trigger based on your needs—schedule regular runs, watch a folder for new files, or create a manual trigger for on-demand processing.

Vector search converts text (like customer notes, product descriptions, or support tickets) into numerical arrays called vectors using AI models like OpenAI's embeddings. Similar vectors are mathematically close together in high-dimensional space, allowing the system to find conceptually related content even when keywords don't match.

When you query this vectorized data, the system finds the closest matching vectors rather than doing keyword matching. This enables semantic search that understands meaning and context—perfect for finding similar support tickets, matching leads based on company descriptions, or categorizing inventory items by function rather than just product names.
