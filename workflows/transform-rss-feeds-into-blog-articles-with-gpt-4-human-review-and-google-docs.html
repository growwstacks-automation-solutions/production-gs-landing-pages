What This Workflow Does

Content teams and marketers spend countless hours monitoring industry news, researching topics, and drafting blog posts. This automation solves that problem by creating a continuous content pipeline that transforms RSS feed articles into polished, publication-ready blog posts.

The workflow automatically fetches the latest articles from your chosen RSS sources, uses GPT-4 to expand them into comprehensive blog posts with proper structure and SEO elements, sends drafts for human editorial review, and finally publishes approved content to Google Docs with team notifications. It maintains quality control through human-in-the-loop approval while eliminating 80% of the manual work.

How It Works

1. RSS Feed Monitoring

A schedule trigger checks your configured RSS feeds at regular intervals (default: every 6 hours). The RSS Read node fetches new articles, extracting titles, descriptions, publication dates, and source information.

2. AI Content Generation

Each article is passed to GPT-4 with a customized prompt that instructs the AI to create a well-structured blog post. The prompt includes your brand voice guidelines, target word count, SEO keywords, and specific formatting requirements.

3. Human Review Integration

The generated draft is sent to a human reviewer via the gotoHuman node. The reviewer receives the article with clear options to approve, reject, or request revisions. This ensures brand alignment and factual accuracy before publication.

4. Conditional Publishing

Based on the review decision, the workflow either creates a Google Doc with the approved content (including proper formatting and metadata) or routes rejected articles for revision. Approved articles trigger Slack notifications to your content team.

What You'll Need

  1. Self-hosted n8n instance (required for gotoHuman community node)
  2. OpenAI API key with GPT-4 access
  3. Google Workspace account for Docs integration
  4. gotoHuman account for human review workflow
  5. Slack workspace for team notifications
  6. RSS feed URLs from your content sources

Pro tip: Start with 2-3 high-quality RSS feeds from reputable industry sources. Test the AI output with different prompt variations before scaling to multiple feeds. The quality of your input RSS feeds directly impacts the quality of generated content.

Quick Setup Guide

Follow these steps to implement this automation in your n8n instance:

  1. Import the template into your n8n instance using the downloaded JSON file.
  2. Configure RSS sources by replacing the example URLs in the RSS Read node with your actual feed URLs.
  3. Connect OpenAI by adding your API credentials to the GPT-4 node and adjusting the content generation prompt for your brand.
  4. Set up Google Docs authentication and specify the folder where new documents should be created.
  5. Configure gotoHuman with your account details and customize the review template for your editorial team.
  6. Connect Slack and select the channel where publication notifications should be sent.
  7. Test the workflow with a single RSS item to ensure all connections work and the output meets your standards.
