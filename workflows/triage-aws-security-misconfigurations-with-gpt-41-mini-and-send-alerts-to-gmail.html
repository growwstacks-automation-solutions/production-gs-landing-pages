What This Workflow Does

This automation transforms your AWS security monitoring by automatically detecting, analyzing, and alerting on cloud misconfigurations. It connects AWS Security Hub findings with AI-powered analysis to prioritize risks and notify your team through email.

The workflow normalizes security findings across different AWS services (IAM, S3, EC2, RDS), uses GPT-4.1 Mini to assess severity with contextual understanding, logs results in Airtable for tracking, and sends actionable alerts via Gmail. This replaces manual review processes that often lead to alert fatigue and delayed responses.

How It Works

1. AWS Event Detection

The workflow begins when AWS Security Hub or Config detects a compliance issue. EventBridge routes these events to an SNS topic.

2. Webhook Processing

n8n receives the security finding via HTTP webhook, validates the request, and processes the subscription confirmation if needed.

3. Data Normalization

A code node transforms the raw AWS finding into a standardized JSON format, extracting key details like resource type, configuration issue, and affected account.

4. AI Prioritization

GPT-4.1 Mini analyzes the finding to assign a priority level (P0-P3) based on exploit potential, business impact, and provides remediation recommendations.

5. Alert Management

The workflow upserts the finding into Airtable to prevent duplicate alerts and sends a concise email notification with the AI assessment through Gmail.

What You'll Need

  1. An n8n instance accessible via HTTP/HTTPS
  2. AWS account with Security Hub enabled
  3. Permissions to create SNS topics and EventBridge rules
  4. OpenAI API key for GPT-4.1 Mini access
  5. Airtable base for logging findings
  6. Gmail account for sending alerts

Quick Setup Guide

  1. Import the JSON template into your n8n instance
  2. Configure the webhook endpoint with a secret token
  3. Set up AWS EventBridge rules forwarding to SNS
  4. Create an SNS subscription to your n8n webhook URL
  5. Connect your OpenAI, Airtable, and Gmail credentials
  6. Test with sample AWS security findings

Pro tip: Start with a small subset of critical AWS security rules before expanding coverage to avoid alert overload during initial testing.
