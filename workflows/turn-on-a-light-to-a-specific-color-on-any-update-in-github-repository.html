GitHub to Smart Light Automation

Get instant visual alerts for repository activity. This free n8n workflow turns your smart lights red when pull requests, issues, pushes, or other updates happen in your GitHub repo.

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation creates a physical notification system for your development team by connecting GitHub repository activity to smart home lighting. Whenever specific events occur in your GitHub repository—such as new pull requests, issue creations, code pushes, or releases—the workflow automatically triggers a connected smart light to change color, typically to red for immediate visual attention.

Instead of relying solely on email notifications or checking dashboards, team members receive ambient, real-time alerts through their physical environment. This is particularly valuable for distributed teams or situations where immediate awareness of repository changes can accelerate response times and improve collaboration.

The workflow serves as a bridge between your digital development workflow and your physical workspace, transforming abstract GitHub events into tangible signals that keep everyone informed without adding to notification fatigue.

How It Works

The automation follows a straightforward trigger-action pattern that connects two powerful platforms:

1. GitHub Webhook Trigger

The workflow starts with a GitHub Trigger node configured to listen for repository events. When you set up the webhook in your GitHub repository settings, it sends real-time notifications to n8n whenever specified events occur. You can filter for specific event types, branches, or users to customize what triggers the alert.

2. Event Processing & Filtering

Once triggered, the workflow can process the incoming GitHub webhook data. Optional filter nodes allow you to create conditional logic—for example, only triggering for events from specific team members, or only for critical issues labeled as "bug" or "urgent." This ensures your physical alerts are meaningful and not overwhelming.

3. Home Assistant Light Control

The processed event data then flows to a Home Assistant node, which sends commands to your smart lighting system. The template configures the light to turn on (if off) and change to a specific color—typically red for attention-grabbing alerts. You can customize this to different colors for different event types if desired.

4. Optional Notification Enhancement

While the primary output is the light change, you can easily extend the workflow to send parallel notifications to Slack, Microsoft Teams, or email, creating a multi-channel alert system that ensures critical updates never get missed.

What You'll Need

  1. A GitHub repository with admin access to configure webhooks
  2. n8n instance (cloud or self-hosted) with internet-accessible webhook URL
  3. Home Assistant instance with configured smart lights and API access
  4. GitHub personal access token or OAuth app credentials for authentication
  5. Home Assistant long-lived access token for API communication
  6. Compatible smart light (Philips Hue, LIFX, or any Home Assistant-supported device)

Pro tip: Start with a single repository and a dedicated test light to refine your event filters before rolling out to your entire team or multiple repositories. This prevents alert fatigue from too many triggers.

Quick Setup Guide

Follow these steps to implement this visual alert system in your environment:

  1. Download and import the template JSON file into your n8n instance using the import workflow feature.
  2. Configure GitHub credentials in n8n with a personal access token that has repo webhook permissions.
  3. Set up the GitHub webhook in your repository settings pointing to your n8n webhook URL (found in the GitHub Trigger node).
  4. Add Home Assistant credentials in n8n using a long-lived access token from your Home Assistant instance.
  5. Update the light entity ID in the Home Assistant node to match your specific smart light's identifier.
  6. Test the workflow by creating a test issue or pull request in your repository and verifying the light changes color.
  7. Activate the workflow and make adjustments to event filters or light colors based on your team's preferences.
