What This Workflow Does

Managing large document collections—internal wikis, policy manuals, API documentation, training materials—often leads to inefficient knowledge retrieval. Employees or customers spend valuable time searching, and AI chatbots trained on all documents can produce noisy, inaccurate answers due to irrelevant context.

This n8n workflow solves that by implementing a two-stage retrieval process. First, it uses Supabase to search file descriptions (metadata) matching the user query, filtering down to the most relevant files. Second, it performs a precise vector search only within those filtered files to fetch the exact document chunks needed. This layered approach dramatically improves AI answer quality, reduces processing load, and delivers context-aware responses faster.

The automation connects Supabase vector storage with OpenAI's language models, creating a production-ready chatbot that can answer questions based on your proprietary documents, ensuring consistency and reducing manual support overhead.

How It Works

Stage 1: Metadata Filtering

A custom code node calls a Supabase RPC function (match_files) that searches the vector embeddings of file descriptions. It returns a list of file IDs whose descriptions are semantically closest to the user's question, applying similarity thresholds and limiting results to keep the search focused.

Stage 2: Vector Chunk Retrieval

Using the filtered file IDs, another code node invokes the match_documents RPC function. This searches the vector embeddings of actual document chunks only within those selected files, again filtered by similarity. This ensures the AI receives only the most relevant pieces of content.

AI Answer Generation

The retrieved chunks, along with metadata like file names and URLs, are passed to an OpenAI message node. System instructions guide the AI to synthesize answers from the provided knowledge base, reference source documents, and produce comprehensive, accurate responses.

Database & Embedding Setup

Supporting workflows and SQL scripts prepare Supabase tables, create RPC functions, and generate embeddings from document summaries using OpenAI, completing the end-to-end system.

What You'll Need

  1. A Supabase account with vector extension enabled (PostgreSQL database).
  2. An OpenAI API key (or compatible LLM provider) for generating embeddings and answers.
  3. An n8n instance (cloud or self-hosted) to run the workflow.
  4. Your document files ready for processing (PDFs, text files, markdown, etc.).
  5. Basic SQL knowledge to run the provided setup scripts (or assistance from an automation expert).

Pro tip: Start with a small set of documents (5-10 files) to test the workflow and understand the embedding process. Scaling to hundreds of files is straightforward once the system is validated.

Quick Setup Guide

  1. Import the template: Download the JSON file and import it into your n8n workspace.
  2. Configure credentials: Replace placeholder credentials with your Supabase project details and OpenAI API key in the respective nodes.
  3. Set up Supabase: Run the provided SQL scripts in your Supabase database to create the files and documents tables, plus the match_files and match_documents RPC functions.
  4. Prepare documents: Use the helper workflow (included in the template notes) to upload your files, generate description summaries, and create vector embeddings stored in Supabase.
  5. Test the chatbot: Trigger the workflow with a sample question via the Manual Trigger node. Verify that the two-stage retrieval works and the OpenAI response is accurate.
  6. Deploy: Connect the workflow to a webhook, chat interface, or schedule for production use.
