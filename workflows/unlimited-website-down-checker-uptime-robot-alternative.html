How It Works

The automation follows a smart, fault-tolerant process that mimics how experienced DevOps engineers monitor critical services.

1. Scheduled Website Checks

A Schedule Trigger node runs at your configured interval (default: hourly). The workflow fetches your list of websites to monitor from a simple configuration node or database. Each URL is checked sequentially or in parallel using HTTP Request nodes.

2. Intelligent Failure Detection

When a website returns an error status code (like 500) or times out, the workflow doesn't immediately panic. It enters a "wait and verify" state using a Wait node, typically set to 300 seconds (5 minutes). This crucial step filters out temporary network issues, CDN hiccups, or brief server restarts.

3. Secondary Verification & Alerting

After the wait period, the workflow rechecks the problematic website. If it's still down, the system triggers alerts through your configured channels—Email for general notifications and Telegram for critical, immediate alerts to your phone. The alert includes the URL, error details, and timestamp.

4. Recovery Monitoring

Once alerted, the workflow continues monitoring the downed site. When it comes back online, you can optionally receive a "recovered" notification, giving you complete visibility into incident duration and resolution.

Pro tip: For mission-critical sites, set up a second instance of this workflow on a different server or cloud region. If your primary monitoring server fails, the backup will alert you—creating a self-monitoring system.

Quick Setup Guide

Importing and configuring this workflow takes about 10 minutes. Follow these steps:

  1. Download the template using the button above and save the JSON file.
  2. Import into n8n by clicking "+ Workflow" → "Import from file" and selecting the downloaded JSON.
  3. Configure your websites in the "Website List" node—add URLs you want to monitor.
  4. Set up email credentials in the Email node (Gmail or your SMTP server).
  5. Optional: Create Telegram bot via @BotFather, get your token and chat ID, and add them to the Telegram node.
  6. Adjust the wait time in the Wait node (300 seconds recommended to filter brief outages).
  7. Activate the workflow and test with a non-existent URL to verify alerts work.

Pro tip: Start by monitoring just one critical site. Once you confirm alerts work correctly, add your remaining sites to the list. This prevents configuration errors from causing alert storms.
