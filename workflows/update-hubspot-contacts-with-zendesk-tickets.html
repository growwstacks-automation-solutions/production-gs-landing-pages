What This Workflow Does

This automation solves a critical business problem: disconnected customer data between support and sales teams. When customers submit tickets in Zendesk, that valuable information often stays siloed in your support system. Sales and marketing teams in HubSpot miss crucial context about customer issues, recent interactions, and satisfaction levels.

The workflow automatically bridges this gap by syncing Zendesk ticket data to corresponding HubSpot contacts every 5 minutes. It captures new and updated tickets, enriches contact profiles with support history, and ensures everyone has a complete customer view. This eliminates manual data entry, reduces errors, and creates a unified customer experience across departments.

Beyond simple data transfer, this automation enables smarter business decisions. Sales teams can prioritize leads based on recent support interactions, marketing can segment campaigns by ticket categories, and customer success can provide personalized follow-ups. The result is improved conversion rates, higher customer satisfaction, and more efficient team collaboration.

How It Works

The automation runs on a scheduled basis, checking for updates and intelligently syncing data between systems.

1. Scheduled Trigger & Data Collection

A Cron node initiates the workflow every 5 minutes. A Function Item node checks which tickets have been created or modified since the last execution. The Zendesk node then collects all relevant ticket data and requester information.

2. Data Processing & Matching

A Set node extracts key contact information (email, name, external ID) from the ticket data. The Merge by Key node combines ticket details with requester information. An If node evaluates whether this corresponds to an existing ticket or requires creating new contact records.

3. HubSpot Integration & Updates

For existing tickets, the workflow updates the corresponding HubSpot contact with the latest ticket status and details. For new tickets, it creates or updates the HubSpot contact first, then creates a new ticket record in HubSpot while maintaining the connection back to Zendesk.

4. Synchronization & Completion

The Zendesk node updates external IDs to maintain cross-system references. Finally, a Function Item node sets the new last execution timestamp, ensuring the next run only processes fresh data.

Quick Setup Guide

Get this automation running in your environment with these simple steps:

  1. Download the template using the button above and import it into your n8n instance
  2. Configure credentials for both Zendesk and HubSpot in n8n's credential management
  3. Map your fields to ensure ticket data transfers to the correct HubSpot contact properties
  4. Set the sync frequency (default is every 5 minutes) based on your business needs
  5. Test with sample data to verify the workflow correctly identifies and updates contacts
  6. Activate the workflow and monitor initial runs to ensure proper synchronization

Pro tip: Start with a limited test group of contacts before rolling out to your entire database. This lets you verify field mappings and business logic without affecting all customer records.
