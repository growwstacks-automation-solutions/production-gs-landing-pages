What This Workflow Does

This automation solves a critical pain point for Managed Service Providers (MSPs) who use Syncro for ticketing and Clockify for time tracking. When technicians complete a ticket in Syncro, they often forget to archive the corresponding project in Clockify, leading to inaccurate time tracking and billing errors.

The workflow automatically monitors Syncro for ticket status changes and updates Clockify projects accordingly. When a ticket is marked complete in Syncro, the corresponding Clockify project archives. If the ticket reopens, the project automatically unarchives. This ensures your time tracking always matches your actual service delivery status.

How It Works

1. Syncro triggers the workflow

The workflow starts when a ticket status changes in Syncro. The n8n webhook captures the ticket ID, project name, and new status.

2. Status evaluation

The workflow checks whether the new status should trigger archiving (for completed/closed tickets) or unarchiving (for reopened/active tickets).

3. Clockify project update

Using the Clockify API, the workflow locates the matching project and updates its archived status to match the Syncro ticket status.

4. Confirmation and logging

The workflow logs the change in both systems and can optionally notify relevant team members via Slack or email.

Pro tip: Add a secondary trigger for ticket creation to automatically set up new Clockify projects for fresh tickets, completing your end-to-end automation.
