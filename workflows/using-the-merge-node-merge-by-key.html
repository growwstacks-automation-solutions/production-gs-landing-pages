Merge Data by Key with n8n

Combine customer data, orders, or product lists from different sources automatically using a common identifier.

What This Workflow Does

This template solves the common business problem of fragmented data. Companies often store customer information in a CRM, order history in an ecommerce platform, and support tickets in a helpdesk—all separate systems. Manually combining this data is time-consuming and error-prone.

The workflow automates data merging using a common key (like customer email or product ID). It brings together information from multiple sources, matches related records, and creates unified datasets. This gives you complete customer profiles, consolidated inventory reports, or enriched lead lists without manual spreadsheet work.

Whether you're syncing Salesforce contacts with Shopify orders, combining QuickBooks transactions with Stripe payments, or merging product catalogs from multiple suppliers, this template provides the foundation for reliable, automated data consolidation.

How It Works

The workflow follows a logical sequence to ensure clean, accurate merging:

Step 1: Data Collection

Two separate data sources provide the initial datasets. In the template, these are mock data nodes that you replace with your actual sources—database queries, API calls, or file imports from tools like Google Sheets, Airtable, or SaaS applications.

Step 2: Data Preparation

Critical conversion nodes transform the data into the proper n8n format. This step ensures each record is treated as a distinct item, preventing merging errors. You can add data cleaning here: standardizing date formats, trimming whitespace, or converting text cases.

Step 3: Key Identification

The workflow identifies the common field that will serve as the merge key. This could be email, customer ID, SKU, or any unique identifier present in both datasets. The template demonstrates how to select and validate this key field.

Step 4: Merge Execution

The merge node combines the datasets based on the selected key. Matched records are merged into single enriched items. You can choose what happens with unmatched records—keep them separate, discard them, or flag them for review.

Step 5: Output & Delivery

The merged data is formatted and sent to its destination: a database update, CSV export, notification email, or integration with another business system. The template shows how to structure the output for immediate use.

Pro tip: Always test your merge with a small sample dataset first. This helps identify issues with key field consistency before processing thousands of records.

  1. n8n instance (cloud or self-hosted) with basic workflow editing permissions
  2. Two or more data sources that share a common identifier field

  4. Sample data from each source to test the merge logic
  5. Destination for the merged data (database, spreadsheet, or another application)

Quick Setup Guide

Follow these steps to implement this merge workflow in your environment:

  1. Download and import the template JSON file into your n8n instance
  2. Replace mock data nodes with your actual data sources (APIs, database queries, or file imports)
  3. Configure the key field in both data streams to ensure they use the same identifier
  4. Adjust merge settings based on how you want to handle matched and unmatched records
  5. Test with sample data to verify matches occur correctly before full deployment
  6. Schedule or trigger the workflow based on your needs (hourly, daily, or on-demand)
  7. Monitor initial runs and adjust data cleaning steps if needed for optimal results

Pro tip: Add a validation step after the merge to check for unexpected duplicates or missing fields. A simple function node can flag records that don't meet quality thresholds.
