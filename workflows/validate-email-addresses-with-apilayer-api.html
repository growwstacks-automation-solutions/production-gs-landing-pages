What This Workflow Does

This n8n workflow automates the process of validating email addresses using APILayer's email verification API. It helps businesses ensure their email lists contain valid, deliverable addresses before sending marketing campaigns or important communications.

Invalid email addresses lead to bounced messages, hurt sender reputation, and waste marketing resources. This workflow checks email syntax, domain validity, mailbox existence, and other key indicators to filter out problematic addresses before they enter your CRM or email marketing platform.

How It Works

1. Email Input

The workflow accepts email addresses from various sources - form submissions, CSV imports, CRM exports, or direct API calls. You can configure the trigger based on your specific use case.

2. APILayer API Validation

Each email address is sent to APILayer's validation service which performs multiple checks including syntax validation, domain verification, SMTP check, and disposable email detection.

3. Result Processing

The workflow analyzes the API response to categorize emails as valid, risky, or invalid. You can configure actions based on the validation results - flagging bad addresses, moving them to quarantine, or triggering follow-up actions.

4. Output Routing

Validated emails are routed to their destination systems (CRM, email service provider, database) while invalid addresses are flagged for review or removed from your lists.

Syntax validation checks if an email follows proper formatting rules (user@domain.tld), while SMTP validation actually tests if the mailbox exists on the receiving server. Both are important for different reasons.

A marketing team might find syntax checks catch obvious typos immediately, while SMTP verification identifies abandoned accounts. Combining both methods provides the most accurate picture of email deliverability and helps prioritize follow-up actions.

  • Syntax checks format correctness
  • SMTP verifies mailbox existence
  • Use both for comprehensive validation
