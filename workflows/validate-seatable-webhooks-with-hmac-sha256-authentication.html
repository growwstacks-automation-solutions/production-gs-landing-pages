What This Workflow Does

In today's connected software landscape, webhooks are essential for real-time data flow between applications like Seatable and your business systems. However, without proper security, these data pipelines become vulnerable to injection attacks, data corruption, and unauthorized access.

This n8n workflow solves that critical security gap by implementing HMAC SHA256 validation for Seatable webhooks. It acts as a secure gatekeeper that verifies every incoming webhook request is genuinely from your Seatable instance before allowing the data to proceed to your automation logic. The workflow compares cryptographic signatures to ensure data integrity, preventing malicious actors from sending fake updates to your CRM, payment systems, or databases.

How It Works

The workflow follows a precise security verification process that happens in milliseconds for each webhook.

1. Webhook Trigger

The workflow starts with an n8n Webhook node listening for POST requests from Seatable. When Seatable sends data (like a new record or update), it includes a special x-seatable-signature header containing an HMAC SHA256 hash of the payload.

2. Signature Extraction

The workflow extracts this signature header and removes the sha256= prefix, leaving just the cryptographic hash that Seatable computed using your shared secret key.

3. HMAC Calculation

Using the Crypto node in n8n, the workflow independently calculates the HMAC SHA256 hash of the exact same raw request body, using the same secret key you've configured in Seatable.

4. Security Comparison

An If node performs a constant-time comparison between the signature from Seatable and the one calculated locally. If they match exactly, the webhook is authenticated.

5. Secure Response

Valid webhooks receive a 200 OK response and proceed to your custom processing logic. Invalid webhooks immediately receive a 403 Forbidden response and execution stops, protecting your systems.

Pro tip: Always store your HMAC secret in n8n's environment variables or a secrets manager—never hardcode it in the workflow. This allows for secure key rotation without modifying the workflow logic.

Quick Setup Guide

Follow these steps to implement secure webhook validation in under 15 minutes:

  1. Download the template using the button above and import it into your n8n instance.
  2. Configure the Webhook node with a unique path (or set to "manual" for testing).
  3. Set your secret key in the Crypto node by replacing the placeholder with your actual Seatable webhook secret.
  4. Test the validation by sending a sample payload from Seatable or using a tool like Postman with the correct signature header.
  5. Connect your logic to the "Add nodes for processing" section to handle validated data.
  6. Deploy the workflow and update your Seatable webhook URL to point to your n8n webhook endpoint.
