What This Workflow Does

How It Works

The quickstart workflow is designed as an interactive learning experience that demonstrates core n8n concepts through practical examples.

Step 1: Manual Trigger Initiation

The workflow begins with a manual trigger node, simulating how automations can be started by user action. This introduces you to n8n's execution model and shows how workflows can be triggered on-demand for testing and specific use cases.

Step 2: Data Processing with Set Node

A Set node demonstrates how to manipulate and structure data within n8n. You'll learn to rename fields, format values, and prepare data for downstream actions—essential skills for any automation that moves information between different systems.

Step 3: Working with Data Stores

The template includes interaction with a sample data store, teaching you how n8n can read from and write to databases or data tables. This introduces persistent data storage concepts crucial for workflows that need to remember information between executions.

Step 4: Documentation and Notes

Sticky note nodes within the workflow provide inline explanations and learning points. This models best practices for documenting your automations and shows how to make workflows understandable for team members who might need to modify them later.

Pro tip: After completing this tutorial, duplicate the workflow and try modifying one section—like changing the data fields or adding an extra processing step. The best way to learn n8n is through experimentation.

Quick Setup Guide

Follow these steps to import and run the quickstart tutorial in your n8n environment:

  1. Download the template using the button above to save the JSON file to your computer
  2. Open your n8n instance – Either log into n8n.cloud or access your self-hosted n8n
  3. Import the workflow – Click the "+" icon or "Import from file" option and select the downloaded JSON file
  4. Examine the workflow structure – Notice how nodes are connected and read the sticky note explanations
  5. Execute the workflow – Click "Execute Workflow" on the manual trigger node to run through the tutorial
  6. Follow along with the data flow – Watch how information moves from node to node in the execution panel
  7. Experiment with modifications – Try changing a field name or adding a simple condition to reinforce learning
