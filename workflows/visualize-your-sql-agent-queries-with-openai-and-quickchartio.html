What This Workflow Does

Business teams often struggle to interpret raw SQL query results. While SQL agents can answer questions about data, they typically return text or tables that require manual analysis and chart creation. This creates bottlenecks in decision-making and limits the accessibility of data insights across organizations.

This workflow solves this problem by automatically transforming SQL Agent responses into visual charts. When a user asks a data question, the system not only provides the answer but also generates an appropriate visualization using OpenAI's analysis capabilities and Quickchart.io's rendering engine. This bridges the gap between data extraction and data presentation, making insights immediately understandable.

The automation intelligently determines when a visualization would be helpful based on the query context, creates the optimal chart type for the data structure, and delivers both the answer and visual in a single response. This transforms how teams interact with their databases, turning complex queries into clear, actionable visual insights.

How It Works

Step 1: Question Analysis & SQL Execution

The workflow begins by extracting the user's question and passing it to a native SQL Agent. The agent connects to your database (PostgreSQL, MySQL, SQLite, etc.), executes the appropriate query, and returns the results in a human-readable format. This establishes the foundational data that will potentially be visualized.

Step 2: Visualization Decision Making

A Text Classifier node analyzes whether the query results would benefit from visual representation. It considers factors like data complexity, comparison needs, and the presence of trends or patterns. If visualization is deemed helpful, the workflow proceeds to chart generation; otherwise, it returns the SQL Agent's response as-is.

Step 3: AI-Powered Chart Definition

When visualization is needed, the workflow calls OpenAI via HTTP Request with structured output capabilities. The AI analyzes the query results and determines the optimal chart type (bar, line, pie, etc.), data mapping, labels, and styling parameters. This creates a complete chart definition tailored to the specific data.

Step 4: Chart Generation & Delivery

The chart definition is passed to Quickchart.io, which renders it as a high-quality image. The workflow then combines the SQL Agent's textual answer with the generated chart image, delivering a comprehensive response that includes both the data explanation and its visual representation.

Quick Setup Guide

  1. Import the template into your n8n instance using the downloaded JSON file
  2. Configure database credentials in the SQL Agent node for your PostgreSQL, MySQL, or other SQL database
  3. Add your OpenAI API key to the HTTP Request node that calls the chat completions endpoint
  4. Test with sample queries like "Show me monthly sales trends" or "Compare product category performance"
  5. Adjust classification thresholds if the system is over- or under-visualizing certain query types
  6. Deploy as a webhook or schedule to make it accessible to your team or integrate with chat applications
