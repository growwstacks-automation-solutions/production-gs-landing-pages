What This Workflow Does

This template introduces Gwen – an intelligent voice-powered marketing assistant that automates the entire content creation and delivery process. Gwen solves the time-consuming problem of producing consistent, high-quality marketing content by combining AI-powered text generation, image creation, and natural voice delivery into a single automated system.

Instead of spending hours writing blog posts, creating visuals, and recording audio, businesses can deploy Gwen to handle these tasks autonomously. The workflow takes content requests, researches topics, generates optimized text, creates accompanying images, and delivers the final results through human-like voice output – all without manual intervention.

The system is particularly valuable for marketing teams, content creators, and businesses looking to scale their content production while maintaining quality and consistency. By automating repetitive creative tasks, teams can focus on strategy and high-level direction rather than execution.

How It Works

The workflow operates through a modular system of interconnected tools managed by an AI agent. Here's the step-by-step process:

1. Content Request & Analysis

Gwen receives a content request through a webhook or interface. The AI agent analyzes the request to determine the appropriate action – whether it needs blog generation, image creation, image editing, or content search.

2. Tool Selection & Execution

Based on the analysis, Gwen selects and executes the appropriate sub-workflow. For blog posts, it conducts real-time research using Tavily, structures the content with OpenAI, and optimizes it for voice delivery. For images, it generates or edits visuals using DALL-E through OpenAI's API.

3. Content Generation & Refinement

The selected tools generate the requested content. Blog posts receive engaging intros, structured body content, and polished conclusions. Images are created based on detailed prompts and saved to Google Drive for easy access and organization.

4. Voice Conversion & Delivery

Once content is generated, ElevenLabs converts the text into natural-sounding speech. The voice output can be delivered through multiple channels – saved as audio files, sent via Telegram, or integrated into voice interfaces for immediate use.

5. Logging & Organization

All generated content – blogs, images, and audio files – are automatically logged in Google Sheets with metadata including creation date, topic, and file locations. This creates a searchable database of past content for future reference and repurposing.

Pro tip: Start by configuring the ElevenLabs integration first, as voice delivery is the most distinctive feature of this workflow. Test with simple prompts before scaling to complex content requests.

Quick Setup Guide

Follow these steps to get your voice-powered marketing assistant running in under 30 minutes:

  1. Import the template – Download the JSON file and import it into your n8n instance through the workflow import feature.
  2. Configure credentials – Set up API keys for OpenAI, ElevenLabs, Tavily, and Google services in n8n's credentials management.
  3. Connect sub-workflows – Import the blog post, create image, edit image, and search images sub-workflows and link them to the main Gwen agent node.
  4. Set up ElevenLabs webhook – Create a POST webhook in ElevenLabs pointing to your n8n instance with the correct body parameters for prompt and session ID.
  5. Test the integration – Run a simple test request to ensure all components communicate properly and content generates as expected.
  6. Customize for your needs – Adjust prompts, voice settings, and delivery channels to match your brand voice and content requirements.

Pro tip: Create a testing checklist: 1) Can Gwen generate a blog post? 2) Can it create an image? 3) Can it edit an existing image? 4) Does voice conversion work? 5) Is content being logged properly? Address each systematically.
