What This Workflow Does

Manually checking websites for updates is a time-consuming task that often gets overlooked, yet missing critical changes can have serious business consequences. Whether you're tracking competitor pricing, monitoring government regulations, watching for terms of service updates, or following industry announcements, staying informed requires constant vigilance.

This automation solves that problem by providing 24/7 website monitoring without manual effort. It automatically checks web pages for content changes, detects even minor modifications using hash tracking, archives snapshots of changed content to Google Drive, maintains a complete audit trail in Google Sheets, and sends instant email notifications to your team when updates occur.

The system works intelligently—it only alerts you when something actually changes, eliminating notification fatigue. By automating what was previously a manual, error-prone process, you gain reliable oversight while freeing up valuable employee time for more strategic work.

How It Works

The workflow follows a logical sequence that ensures efficient monitoring and reliable change detection.

1. Scheduled Trigger & URL Fetching

The automation begins with a scheduled trigger that runs at your specified intervals—daily, hourly, or whatever frequency makes sense for your monitoring needs. It then fetches the target webpage using HTTP requests, handling various content types and website structures.

2. Content Extraction & Hash Generation

Using HTML parsing, the workflow extracts the specific content you want to monitor—whether it's entire pages, specific sections, or particular data points. It then generates a unique cryptographic hash of this content, creating a digital fingerprint that represents the current state.

3. Change Detection via Hash Comparison

The system compares the newly generated hash against previously stored hashes. If the hash matches known values, the content hasn't changed, and the workflow stops here—no unnecessary alerts. If the hash differs, it proceeds to the notification and archiving steps.

4. Archiving & Notification

When changes are detected, the workflow automatically saves a timestamped snapshot of the new content to Google Drive for permanent reference. It then logs the change details—including timestamp, URL, and change summary—to a Google Sheet for audit trails and analysis. Finally, it sends email alerts through Gmail to notify relevant team members.

Quick Setup Guide

Follow these steps to implement this website monitoring automation in your environment.

  1. Download and import the template JSON file into your n8n instance using the import workflow feature.
  2. Configure Google connections by authenticating n8n with your Google Workspace account for Drive, Sheets, and Gmail access.
  3. Update target URLs in the variables node with the web pages you want to monitor.
  4. Adjust HTML selectors if you're monitoring specific page sections rather than entire pages.
  5. Set the schedule to match your monitoring needs—daily checks are common, but some use cases require more frequent monitoring.
  6. Test with known changes by manually updating a test page and verifying the automation detects it and sends proper notifications.
  7. Deploy and monitor the workflow, checking the Google Sheet audit log periodically to ensure it's capturing changes correctly.

Hash tracking creates a unique digital fingerprint (hash) of a webpage's content using algorithms like MD5 or SHA-256. Each time the automation runs, it generates a new hash of the current content and compares it to previous hashes. If the hash differs, the content has changed. This method is efficient and reliable for detecting even minor modifications.

The beauty of hash comparison is its efficiency. Instead of storing and comparing entire web pages (which could be megabytes of data), the system stores tiny hash strings (32-64 characters). This makes the automation fast, scalable, and resource-efficient even when monitoring hundreds of pages.
