What This Workflow Does

Manual SEO reporting is time-consuming and error-prone. Marketing teams often spend hours each week extracting data from Google Search Console, calculating week-over-week changes, and formatting reports—time that could be spent on strategy and optimization.

This automated workflow solves that problem by connecting directly to Google Search Console, pulling your organic search performance data, and generating a clean, actionable weekly report. It specifically segments brand versus non-brand traffic—a critical distinction for understanding true SEO growth—and calculates percentage changes for clicks, impressions, CTR, and average position.

The final output is a professionally formatted email summary that highlights wins (in green) and areas needing attention (in red), giving your team instant visibility into performance trends without touching a spreadsheet.

How It Works

The automation follows a logical sequence to transform raw search data into business intelligence.

1. Data Extraction & Scheduling

The workflow is triggered on a weekly schedule (typically Monday morning). It authenticates with Google Search Console using OAuth and requests performance data for your configured property and date range (last week vs. the previous week).

2. Brand vs Non-Brand Segmentation

Using query filtering logic, the workflow separates brand-related searches (containing your company name, product names, or trademarks) from non-brand, discovery-driven queries. This segmentation is crucial for understanding whether traffic growth comes from brand awareness or genuine SEO efforts.

3. Metric Calculation & Analysis

For each segment, the system calculates key metrics: total clicks, impressions, click-through rate (CTR), and average position. It then computes the week-over-week percentage change for each metric, providing immediate insight into trends.

4. Report Generation & Formatting

The calculated data is structured into a clean HTML table with color-coded cells—green for positive trends, red for negative ones. Additional commentary or highlights can be automatically generated based on threshold rules (e.g., "Non-brand clicks increased by 15%").

5. Distribution

The final report is sent via your chosen channel—typically email to stakeholders—but can be easily adapted to post to Slack, Microsoft Teams, or save to a Google Sheet for historical tracking.

Pro tip: Configure threshold alerts within the workflow to only send notifications when metrics change beyond a certain percentage (e.g., >10% drop in non-brand clicks). This reduces notification fatigue and highlights only significant changes.

What You'll Need

  1. A Google Search Console property verified for your website or specific site section.
  2. n8n instance (cloud or self-hosted) with the HTTP Request and Gmail/Email Send nodes available.
  3. Google Cloud Project with the Search Console API enabled and OAuth 2.0 credentials (for automated data access).
  4. Email sending credentials (Gmail/ SMTP) or webhook URLs for alternative destinations like Slack or Teams.
  5. A list of brand terms to filter queries (e.g., your company name, product names, common misspellings).

Quick Setup Guide

Get this automation running in your environment in under 30 minutes.

  1. Download & Import: Click the "Download Template" button above and import the JSON file into your n8n instance.
  2. Configure Google OAuth: In the HTTP Request node, set up credentials for the Google Search Console API using your Google Cloud Project client ID and secret.
  3. Set Your Property: Update the workflow configuration with your specific Google Search Console property URL (e.g., `sc-domain:yourdomain.com` or exact URL).
  4. Define Brand Filters: Edit the "Filter" node to include your specific brand keywords for accurate segmentation.
  5. Set Recipients & Schedule: Add the email addresses of report recipients in the Email Send node and set your preferred schedule (e.g., every Monday at 9 AM).
  6. Test & Activate: Run the workflow once manually to verify data flow and email formatting, then activate the schedule trigger.
