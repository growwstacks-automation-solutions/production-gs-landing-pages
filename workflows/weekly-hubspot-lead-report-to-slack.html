Weekly HubSpot Lead Report to Slack

What This Workflow Does

Manual sales funnel reporting is time-consuming, error-prone, and often delayed. Sales teams waste valuable hours every Monday extracting data from HubSpot, calculating weekly totals, and formatting updates for Slack. This creates inconsistent visibility, missed opportunities for quick follow-ups, and frustration across the organization.

This automation solves that by creating a hands-free weekly reporting system. It connects directly to your HubSpot account, filters for contacts who entered the "Lead" stage within the past 7 days, counts them precisely, and posts a clean, actionable summary to your designated Slack channel. Optional branches can also pull and summarize closed-won deals from the same period, giving a complete picture of funnel movement.

The result is a reliable, accurate snapshot that arrives automatically every Monday morning. Your team starts the week with immediate visibility into pipeline health, enabling faster conversations and more informed strategic decisions without any manual effort.

How It Works

1. Scheduled Trigger

A schedule node (typically set for Monday at 7:00 AM) initiates the workflow automatically. You can configure it for any day and time that fits your team's rhythm.

2. HubSpot Data Fetch

The workflow authenticates with your HubSpot account and queries the Contacts API. It specifically requests contacts with the property hs_v2_date_entered_lead populated, which tracks when a contact officially became a lead.

3. Date Filtering

A filter node isolates only those contacts whose lead entry date falls within the last 7 days (or your custom timeframe). This ensures the report is always current and relevant.

4. Data Aggregation

A summarize node counts the filtered contacts, providing the core metric: "X new leads this week."

5. Slack Notification

Finally, a Slack node posts a formatted message to your chosen channel. The message clearly states the lead count and can include emojis, trend context, or calls to action for the sales team.

Pro tip: Extend the workflow by adding a parallel branch to fetch and summarize "Closed-Won" deals from the same period. This gives your team both top-of-funnel (leads) and bottom-of-funnel (wins) visibility in a single report.

What You'll Need

  1. A running instance of n8n (cloud or self-hosted).
  2. HubSpot account with API access (OAuth2 credentials). Ensure you have contacts with the hs_v2_date_entered_lead property tracked.
  3. Slack workspace where you have permission to create an app and post to channels (OAuth2 credentials).
  4. Basic understanding of how to import a JSON workflow template and configure API connections within n8n.

Quick Setup Guide

You can have this automation running in under 15 minutes.

  1. Download the template using the button above.
  2. Import into n8n: In your n8n workspace, go to Workflows → Import from File and select the downloaded JSON.
  3. Configure HubSpot: Click the "HubSpot" node, add a new connection using OAuth2, and authenticate with your account. Ensure the node is set to fetch the correct contact properties.
  4. Configure Slack: Click the "Slack" node, add a new connection, and authenticate. Select the exact channel where the report should be posted.
  5. Set the schedule: Adjust the Schedule Trigger node to your preferred day and time (e.g., Mondays at 7 AM).
  6. Activate and test: Turn the workflow ON and run it once manually to verify the Slack message appears correctly.
