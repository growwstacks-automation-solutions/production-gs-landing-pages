Weekly n8n Executions Failures Report to Telegram

Automatically monitor and report failed workflow executions via Telegram alerts

Download Template JSON · n8n compatible · Free

What This Workflow Does

This automation solves the critical problem of undetected workflow failures in n8n. Many businesses only discover automation issues when downstream processes fail or customers complain. This template provides proactive monitoring by:

  • Running weekly checks of all workflow executions
  • Identifying and categorizing failures
  • Delivering consolidated reports to your Telegram channel

By implementing this solution, you'll reduce troubleshooting time by 60-80% compared to manual checks, while ensuring no critical failures go unnoticed for more than 7 days.

How It Works

1. Scheduled Trigger

The workflow activates every Monday morning to review the previous week's executions.

2. Fetch Execution Data

Connects to your n8n instance via API to retrieve execution logs for specified workflows.

3. Failure Analysis

Filters successful runs and groups failures by error type, workflow, and time period.

4. Report Generation

Creates an organized summary with counts, first occurrences, and error samples.

5. Telegram Notification

Sends the formatted report to your designated Telegram channel or chat.

Pro tip: For mission-critical workflows, modify the schedule trigger to run daily while maintaining the weekly summary format.
