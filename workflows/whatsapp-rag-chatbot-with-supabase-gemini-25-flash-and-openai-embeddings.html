What This Workflow Does

This automation transforms WhatsApp into an intelligent knowledge assistant that answers customer questions using your own documents. Instead of generic AI responses, it retrieves specific information from your knowledge base, policies, manuals, or FAQs to provide accurate, context-aware answers.

The system uses Retrieval-Augmented Generation (RAG) technology to combine document search with AI generation. When a user asks a question via WhatsApp, the workflow searches your uploaded documents in Supabase using OpenAI embeddings, finds the most relevant information, and uses Google's Gemini 2.5 Flash to generate a natural, helpful response.

Businesses use this to provide 24/7 customer support, internal knowledge sharing, product information dissemination, and training assistance—all through the familiar WhatsApp interface that customers and employees already use daily.

How It Works

1. Message Reception & Classification

When a WhatsApp message arrives via webhook, the workflow first determines if it's a document upload or a question. Document uploads trigger the ingestion pipeline, while questions initiate the retrieval and response process.

2. Document Processing (When Uploaded)

Uploaded documents are converted to text, split into manageable chunks, and processed through OpenAI's embedding model. These embeddings (numerical representations of text meaning) are stored in Supabase alongside the original text, creating a searchable knowledge base.

3. Query Processing & Semantic Search

For user questions, the workflow generates query embeddings using the same OpenAI model. It then performs a semantic search in Supabase to find the document chunks most relevant to the question, based on meaning rather than just keywords.

4. AI Response Generation

The retrieved context is passed to Google's Gemini 2.5 Flash LLM along with the original question. The AI synthesizes the information into a clear, helpful response that's tailored to your business tone and style.

5. WhatsApp Response Delivery

The generated answer is sent back to the user through WhatsApp, completing the conversation loop. The entire process typically takes 2-4 seconds, providing near-instant support.

What You'll Need

  1. WhatsApp Business API access (or Twilio WhatsApp Sandbox for testing)
  2. Supabase account for vector storage and document management
  3. OpenAI API key for generating text embeddings
  4. Google Gemini API access for the LLM response generation
  5. n8n instance (cloud or self-hosted) to run the workflow
  6. Your knowledge documents in PDF, Word, or text format

Pro tip: Start with your most frequently asked questions document. This gives the chatbot immediate value while you expand the knowledge base over time.

Quick Setup Guide

  1. Download and import the template JSON file into your n8n instance
  2. Configure credentials for WhatsApp, Supabase, OpenAI, and Gemini in n8n's credential manager
  3. Set up webhook in your WhatsApp Business API to point to your n8n workflow URL
  4. Upload initial documents by sending them to your WhatsApp number—the system will process and store them automatically
  5. Test with questions to verify the retrieval and response quality
  6. Refine prompts in the Gemini node to match your brand voice and response style
