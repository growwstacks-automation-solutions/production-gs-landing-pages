What This Workflow Does

This automation solves a common pain point in recruitment workflows - the repetitive task of creating and naming interview documentation for each candidate. Whenever a new event matching your criteria is created in Google Calendar (like an interview slot), the workflow automatically duplicates your standard interview notes template in Google Drive and renames it with relevant details from the calendar event.

For hiring teams, this means no more manual document creation, no more inconsistent file naming, and no more forgotten interview notes. The system ensures every interview has properly formatted documentation ready before the meeting begins, with all files consistently organized in your designated Google Drive folder.

How It Works

1. Calendar event detection

The workflow monitors your Google Calendar for new events matching your specified criteria (like events containing "interview" in the title). You can configure exactly which calendars and event types should trigger the automation.

2. Template duplication

When a matching event is detected, the system locates your predefined interview notes template in Google Drive and creates an identical copy in your specified destination folder.

3. Smart file renaming

The duplicated file is automatically renamed using details from the calendar event. A typical naming convention might be "CandidateName_Position_InterviewDate" - pulling these details directly from the event title, description, or attendee list.

Pro tip: Use consistent naming patterns in your calendar events (like "Interview: John Doe - Senior Developer") to enable more accurate automated file naming.
