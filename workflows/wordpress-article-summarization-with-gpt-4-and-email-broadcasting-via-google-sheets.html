WordPress Article Summarization with GPT-4 & Email Broadcast

Automatically fetch, summarize, and distribute your latest blog content to subscribers—zero manual work required.

What This Workflow Does

This automation solves the time-consuming task of manually sharing blog updates with your audience. Instead of copying, pasting, and sending emails every time you publish, this workflow automatically fetches your latest WordPress article, creates an AI-powered summary, and delivers it to your subscriber list via email.

For content creators, marketers, and businesses with active blogs, this represents a significant efficiency gain. You maintain consistent audience engagement without the weekly hours spent on newsletter preparation. The AI summarization ensures subscribers receive the core insights in a digestible format, increasing open rates and reducing unsubscribe rates from information overload.

The system operates on a schedule you define—daily, weekly, or whenever new content is published. It seamlessly connects your content repository (WordPress) with your audience management tool (Google Sheets) and delivery channel (email), creating a fully automated content distribution pipeline.

How It Works

Step 1: Scheduled Trigger

The workflow activates on a cron schedule you configure—for example, every Monday at 9 AM. This trigger initiates the entire process without manual intervention, ensuring consistent delivery timing for your audience.

Step 2: Fetch Latest WordPress Article

Using WordPress's REST API, the automation retrieves your most recent published post. It extracts the title, content, publication date, and permalink, preparing this data for the next processing stage.

Step 3: AI-Powered Summarization

The full article content is sent to OpenAI's GPT-4 model with specific instructions to create a concise, engaging summary. You can customize the output format—bullet points, paragraph summary, or key takeaways—to match your brand voice and audience preferences.

Step 4: Retrieve Subscriber List

The workflow accesses your Google Sheet containing email subscribers. It reads the email addresses, names (if available), and any segmentation data, preparing for personalized email delivery.

Step 5: Batch Email Delivery

Using either Gmail SMTP or your preferred email service, the system sends the summarized content to each subscriber. The SplitInBatches node ensures efficient sending without overwhelming your email service limits, while maintaining personalization through merge fields.

Step 6: Conditional Logic & Error Handling

Built-in checks ensure quality: articles below a minimum word count can be skipped, failed email deliveries are logged, and the system can notify you of completion or issues via your preferred channel (Slack, email, etc.).

What You'll Need

  1. WordPress website with the REST API enabled (standard for most installations)
  2. OpenAI API key with GPT-4 access (available from platform.openai.com)
  3. Google Sheets document containing subscriber emails (minimum columns: Email, optionally Name, SubscriptionDate)
  4. Email sending capability via Gmail SMTP (using app password) or a service like SendGrid/Mailgun
  5. n8n instance (self-hosted or cloud) to run the workflow
  6. Basic understanding of how to set API credentials in n8n (guided process)

Quick Setup Guide

Follow these steps to implement this automation in under 30 minutes:

  1. Download and import the JSON template into your n8n instance using the import function.
  2. Configure WordPress connection by adding your site URL to the HTTP Request node. No API key needed for public blogs.
  3. Add your OpenAI API key in the credentials section, then select GPT-4 as the model in the OpenAI node.
  4. Connect Google Sheets using OAuth2 authentication, then specify your spreadsheet ID and range containing emails.
  5. Set up email credentials for Gmail SMTP or your preferred email service in the Email Send node.
  6. Customize the schedule in the Cron node to match your desired publication rhythm (daily, weekly, etc.).
  7. Test with a single email to yourself before activating the full subscriber list send.

Pro tip: Before going live, add a "Test Mode" switch using an IF node that sends only to your email when enabled. This lets you verify summaries and formatting before broadcast.
